<p>
<p>Recently, we came across a <a href="https://twitter.com/h2jazi/status/1638923200527474691">tweet</a> about DarkBit ransomware. An Iranian APT group, named MuddyWater, is reportedly behind the DarkBit ransomware. In this blog we will explore the ransomware’s initial access method, the use of Cobalt Strike and the final ransomware payload.&nbsp;&nbsp;&nbsp;</p>



<h2>Initial Access Method</h2>



<p>The initial lure was delivered as an ISO file.&nbsp;</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="614" height="104" src="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure1.jpg" alt="" class="wp-image-25623" srcset="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure1.jpg 614w, https://labs.k7computing.com/wp-content/uploads/2023/04/Figure1-300x51.jpg 300w" sizes="(max-width: 614px) 100vw, 614px" /><figcaption>Figure 1 &#8211;&nbsp; ISOFile</figcaption></figure></div>



<p></p>



<p>The payload included a shortcut file (with a .doc extension) and a zip file.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="626" height="89" src="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure2.jpg" alt="" class="wp-image-25624" srcset="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure2.jpg 626w, https://labs.k7computing.com/wp-content/uploads/2023/04/Figure2-300x43.jpg 300w" sizes="(max-width: 626px) 100vw, 626px" /><figcaption>Figure 2 &#8211; Contents Inside ISO File<br></figcaption></figure></div>



<p></p>



<p>The shortcut was using PrintBrm.exe to unpack the HR-Update.zip and run it as shown below. PrintBrm.exe is a windows inbuilt <a href="https://learn.microsoft.com/en-us/previous-versions/windows/it-pro/windows-server-2012-r2-and-2012/jj134237(v=ws.11)">command line tool</a>.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="376" height="485" src="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure3.jpg" alt="" class="wp-image-25625" srcset="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure3.jpg 376w, https://labs.k7computing.com/wp-content/uploads/2023/04/Figure3-233x300.jpg 233w" sizes="(max-width: 376px) 100vw, 376px" /><figcaption>Figure 3 &#8211; Shortcut File</figcaption></figure></div>



<pre class="wp-block-code"><code>cmd.exe /c xcopy .\HR-Update.zip %TEMP% /h /y &amp;&amp; PrintBrm.exe -r -f %TEMP%\HR-Update.zip -d %TEMP%\unzip &amp; %TEMP%\unzip\HR-Update.exe</code></pre>



<pre class="wp-block-preformatted"></pre>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="714" height="71" src="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure4.jpg" alt="" class="wp-image-25629" srcset="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure4.jpg 714w, https://labs.k7computing.com/wp-content/uploads/2023/04/Figure4-300x30.jpg 300w" sizes="(max-width: 714px) 100vw, 714px" /><figcaption>Figure 4 &#8211;&nbsp; HR-Update.exe Running</figcaption></figure></div>



<p>HR-Update.exe was a Cobalt Strike beacon. Cobalt Strike, a penetration testing tool, can also be used by attackers for gaining a foothold in the system. The final ransomware payload is downloaded with the help of Cobalt Strike.</p>



<p>At the time of writing the blog, we were unable to get the exact DarkBit ransomware payload. So we are using another available sample that belonged to the same campaign.</p>



<h2>Analysis of Ransomware</h2>



<p>This DarkBit ransomware sample is written in GoLang. It contains command line arguments.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="393" height="300" src="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure5.jpg" alt="" class="wp-image-25656" srcset="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure5.jpg 393w, https://labs.k7computing.com/wp-content/uploads/2023/04/Figure5-300x229.jpg 300w" sizes="(max-width: 393px) 100vw, 393px" /><figcaption>Figure 5 &#8211;&nbsp; Ransomware Features</figcaption></figure></div>



<p></p>



<p>It also contains an inbuilt configuration file as shown in Figure 6.</p>



<p></p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" width="1024" height="686" src="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure6-1024x686.jpeg" alt="" class="wp-image-25657" srcset="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure6-1024x686.jpeg 1024w, https://labs.k7computing.com/wp-content/uploads/2023/04/Figure6-300x201.jpeg 300w, https://labs.k7computing.com/wp-content/uploads/2023/04/Figure6-768x514.jpeg 768w, https://labs.k7computing.com/wp-content/uploads/2023/04/Figure6.jpeg 1054w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption>Figure 6 &#8211; InBuilt Config</figcaption></figure></div>



<p>Further analysis revealed that they had obfuscated some dll names like advapi32.dll and functions like SystemFunction036.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="343" height="349" src="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure7-2.png" alt="" class="wp-image-25674" srcset="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure7-2.png 343w, https://labs.k7computing.com/wp-content/uploads/2023/04/Figure7-2-295x300.png 295w" sizes="(max-width: 343px) 100vw, 343px" /><figcaption>Figure 7-&nbsp; Obfuscation</figcaption></figure></div>



<p></p>



<p>Its dynamically resolving API at this address. Malware authors tend to dynamically resolve API to avoid static detections.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="484" height="332" src="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure9.png" alt="" class="wp-image-25637" srcset="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure9.png 484w, https://labs.k7computing.com/wp-content/uploads/2023/04/Figure9-300x206.png 300w" sizes="(max-width: 484px) 100vw, 484px" /><figcaption>Figure 8 &#8211;&nbsp; Dynamically Resolving API</figcaption></figure></div>



<p></p>



<p>CreateMutexW API is being used to check if an instance of the malware is already running. As can be seen in Figure 5 previously, they are also using multithreading.</p>



<p>As the customary prelude to file encryption, they are using vssadmin.exe to delete all the shadow copies.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="480" height="261" src="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure10.png" alt="" class="wp-image-25638" srcset="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure10.png 480w, https://labs.k7computing.com/wp-content/uploads/2023/04/Figure10-300x163.png 300w" sizes="(max-width: 480px) 100vw, 480px" /><figcaption>Figure 9 &#8211; Delete Shadow all</figcaption></figure></div>



<p></p>



<p>Here they are using SystemFunction036 (documented in MSDN as <a href="https://learn.microsoft.com/en-us/windows/win32/api/ntsecapi/nf-ntsecapi-rtlgenrandom">RtlGenRandom</a>) to generate a random key as shown in Figure 10.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="881" height="262" src="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure11.png" alt="" class="wp-image-25639" srcset="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure11.png 881w, https://labs.k7computing.com/wp-content/uploads/2023/04/Figure11-300x89.png 300w, https://labs.k7computing.com/wp-content/uploads/2023/04/Figure11-768x228.png 768w" sizes="(max-width: 881px) 100vw, 881px" /><figcaption>Figure 10 &#8211;&nbsp; SystemFunction036</figcaption></figure></div>



<p></p>



<p>SystemFunction036 is accessed multiple times in the code with varying buffer sizes passed to it.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="258" height="312" src="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure12.png" alt="" class="wp-image-25640" srcset="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure12.png 258w, https://labs.k7computing.com/wp-content/uploads/2023/04/Figure12-248x300.png 248w" sizes="(max-width: 258px) 100vw, 258px" /><figcaption>Figure 11 &#8211; Call To Dynamically Resolve API and Then SystemFunction036</figcaption></figure></div>



<p></p>



<p>Here we can see that the buffer size of 80 is made available for SystemFunction036.&nbsp;</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="278" height="356" src="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure13.png" alt="" class="wp-image-25641" srcset="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure13.png 278w, https://labs.k7computing.com/wp-content/uploads/2023/04/Figure13-234x300.png 234w" sizes="(max-width: 278px) 100vw, 278px" /><figcaption>Figure 12 &#8211; Encrypting Key</figcaption></figure></div>



<p></p>



<p>Later-on, the key used for encrypting the files is itself encrypted and attached to the encrypted files.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="227" height="354" src="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure13-1.png" alt="" class="wp-image-25676" srcset="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure13-1.png 227w, https://labs.k7computing.com/wp-content/uploads/2023/04/Figure13-1-192x300.png 192w" sizes="(max-width: 227px) 100vw, 227px" /><figcaption> Figure 13 &#8211;&nbsp; EncryptionAlgorithm </figcaption></figure></div>



<p></p>



<p>From Figure 13, we can see that It’s encrypting. It is likely using AES to encrypt the files, as strings related to the same functions can be found elsewhere in this same sample.</p>



<p>FindFirstFileW, FindNextFileW are used to iterate through the file system, to find the appropriate file and then encrypt it.</p>



<p>At that point it was observed that its writing file in chunks and not as a whole. For doing the same it&#8217;s using SetFilePointerEx API to move the file pointer to a specific address.</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" width="1024" height="270" src="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure15-1024x270.png" alt="" class="wp-image-25644" srcset="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure15-1024x270.png 1024w, https://labs.k7computing.com/wp-content/uploads/2023/04/Figure15-300x79.png 300w, https://labs.k7computing.com/wp-content/uploads/2023/04/Figure15-768x202.png 768w, https://labs.k7computing.com/wp-content/uploads/2023/04/Figure15.png 1301w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption>Figure 14 &#8211; SetFilePointerEx</figcaption></figure></div>



<p></p>



<p>It’s then using the WriteFile API.</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" width="1024" height="295" src="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure16-1024x295.png" alt="" class="wp-image-25645" srcset="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure16-1024x295.png 1024w, https://labs.k7computing.com/wp-content/uploads/2023/04/Figure16-300x87.png 300w, https://labs.k7computing.com/wp-content/uploads/2023/04/Figure16-768x222.png 768w, https://labs.k7computing.com/wp-content/uploads/2023/04/Figure16.png 1099w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption>Figure 15 &#8211;&nbsp; WriteFile</figcaption></figure></div>



<p></p>



<p>All these functions are called one after another, till all the files are encrypted.</p>



<p>After encrypting the file, the key is stored at the end of the file.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="617" height="461" src="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure17.png" alt="" class="wp-image-25646" srcset="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure17.png 617w, https://labs.k7computing.com/wp-content/uploads/2023/04/Figure17-300x224.png 300w" sizes="(max-width: 617px) 100vw, 617px" /><figcaption>Figure 16 &#8211;&nbsp; Ransomware key</figcaption></figure></div>



<p></p>



<p>The encrypted files are given ‘.darkbit’ extension and also a ransom note is dropped in the respective folders.</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" width="1024" height="397" src="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure19-1-1024x397.png" alt="" class="wp-image-25650" srcset="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure19-1-1024x397.png 1024w, https://labs.k7computing.com/wp-content/uploads/2023/04/Figure19-1-300x116.png 300w, https://labs.k7computing.com/wp-content/uploads/2023/04/Figure19-1-768x298.png 768w, https://labs.k7computing.com/wp-content/uploads/2023/04/Figure19-1.png 1279w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption>Figure 17&nbsp; &#8211; Ransomware Note</figcaption></figure></div>



<p></p>



<p>Tweet from a self-identifying DarkBit twitter handle, associating itself with&nbsp; <a href="https://twitter.com/DarkBitTW/status/1633842782065557504">MuddyWater</a></p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="486" height="114" src="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure20.png" alt="" class="wp-image-25649" srcset="https://labs.k7computing.com/wp-content/uploads/2023/04/Figure20.png 486w, https://labs.k7computing.com/wp-content/uploads/2023/04/Figure20-300x70.png 300w" sizes="(max-width: 486px) 100vw, 486px" /><figcaption>Figure 18 &#8211; Tweet on DarkBit</figcaption></figure></div>



<p>MuddyWaters uses different types of attacks for initial access like phishing email campaigns, using tools like MimiKatz to break into the system, etc. In this case, MuddyWaters made use of Cobalt Strike to get initial access into the system. From the ransomware note, we figured out that it was a politically motivated attack.</p>



<p>We at K7 Labs provide detection for DarkBit ransomware and all the latest threats. Users are advised to use a reliable security product such as “K7 Total Security” and keep it up-to-date to safeguard their devices.</p>
