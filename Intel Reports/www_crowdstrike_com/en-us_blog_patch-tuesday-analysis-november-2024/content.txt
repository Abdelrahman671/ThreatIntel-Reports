        <h1>November 2024 Patch Tuesday: Four Critical and Three Zero-Days Among 158 Vulnerabilities Patched</h1>
        
        <div class="publish_info">
            <p>November 12, 2024</p>

       <span><p>Microsoft has released security updates for 158 vulnerabilities in its November 2024 Patch Tuesday rollout. Among these are four <b>Critical</b> vulnerabilities (CVE-2024-43625, CVE-2024-49056, CVE-2024-43498, and CVE-2024-43639). Of these vulnerabilities, CVE-2024-49056 has been fully mitigated by Microsoft and requires no customer actions at this point. In addition, three zero-days have been identified (CVE-2024-43451, CVE-2024-49040, CVE-2024-49019). </p>
<h2>November 2024 Risk Analysis</h2>
<p>This month’s leading risk type is <a href="https://www.crowdstrike.com/cybersecurity-101/remote-code-execution-rce/">remote code execution</a> (59%), followed by <a href="https://www.crowdstrike.com/cybersecurity-101/privilege-escalation/">elevation of privilege</a> (29%).</p>
</span>

<div class="cmp-text" id="text-acebd52d4e" data-cmp-name="cmp-text-" data-cmp-is="text" data-target-location="false">
    
       <span><h2>Microsoft Discloses Vulnerability within Airlift.microsoft.com</h2>
<p>In an effort to provide additional transparency for Microsoft-hosted services, Microsoft has disclosed a <b>Critical</b> <a href="https://www.crowdstrike.com/tech-hub/?category&#61;endpoint-security">privilege escalation</a> vulnerability within airlift.microsoft.com (<a href="https://msrc.microsoft.com/update-guide/vulnerability/CVE-2024-49056">CVE-2024-49056</a>). This vulnerability has been fully mitigated by Microsoft and requires no customer interaction. Microsoft has stated that it is committed to transparency and has created a new classification of vulnerability that requires no customer interaction. More information about this commitment can be found <a href="https://msrc.microsoft.com/blog/2024/06/toward-greater-transparency-unveiling-cloud-service-cves/">here</a>.</p>
<h2>Two Zero-Days in Microsoft Windows</h2>
<p><a href="https://msrc.microsoft.com/update-guide/vulnerability/CVE-2024-49019">CVE-2024-49019</a> is an <b>Important</b> <a href="https://www.crowdstrike.com/tech-hub/?category&#61;endpoint-security">privilege escalation</a> vulnerability within the Active Directory certificate services platform. This platform is responsible for issuing and managing Public Key Infrastructure (PKI). More information about this service can be found <a href="https://learn.microsoft.com/en-us/windows-server/identity/ad-cs/active-directory-certificate-services-overview">here</a>. This vulnerability can allow an attacker to gain domain administrator privileges. This is one of the highest levels of privilege available in Active Directory. Only certificates generated using a <a href="https://learn.microsoft.com/en-us/windows-server/identity/ad-cs/certificate-template-concepts#certificate-template-versions">version 1 template</a> with a <b>source of subject name</b> field set to “Supplied in the request” are usable for this attack. In addition, the victim site must have overly broad <b>Enroll</b> permissions. Microsoft recommends removing Autoenroll permissions and auditing sites to ensure least privilege. </p>
<p><a href="https://msrc.microsoft.com/update-guide/vulnerability/CVE-2024-43451">CVE-2024-43451</a> is an <b>Important</b> <a href="https://www.crowdstrike.com/en-us/cybersecurity-101/social-engineering/spoofing-attack/?srsltid&#61;AfmBOopYlIa0pBhR7ZwF4gMyzgOZOGw7xyGf_TvoWt3BPCdy_xG7AMvs">spoofing</a> vulnerability within the <a href="https://www.crowdstrike.com/en-us/cybersecurity-101/identity-protection/windows-ntlm/">NTLM</a> handling system in Windows. This vulnerability exposes the victim’s NTLM hash to adversaries, allowing them to impersonate the victim. This vulnerability can be exploited by the user selecting (single-clicking), inspecting (right-clicking) or performing some other non-opening action on the malicious file. </p>
<table cellpadding="1" cellspacing="0" border="1"><caption>Table 1.Zero-days in Microsoft Windows</caption><tbody><tr class="top-row bg-orange"><td>Severity</td><td>CVSS Score</td><td>CVE</td><td>Description</td></tr><tr><td><b>Important</b></td><td>7.8</td><td>CVE-2024-49019</td><td>Active Directory Certificate Services Elevation of Privilege Vulnerability</td></tr><tr><td><b>Important</b></td><td>6.5</td><td>CVE-2024-43451</td><td>NTLM Hash Disclosure Spoofing Vulnerability</td></tr></tbody></table>
<h2>One Zero-Day in Microsoft Exchange Server</h2>
<p><a href="https://msrc.microsoft.com/update-guide/vulnerability/CVE-2024-49040">CVE-2024-49040</a> is an <b>Important</b> <a href="https://www.crowdstrike.com/en-us/cybersecurity-101/social-engineering/spoofing-attack/?srsltid&#61;AfmBOopYlIa0pBhR7ZwF4gMyzgOZOGw7xyGf_TvoWt3BPCdy_xG7AMvs">spoofing</a> vulnerability in which an attacker can bypass current spoofing protections within Exchange through use of a non-compliant <code>P2 FROM</code> header. This header is responsible for ensuring the integrity of the sender of an email. Once updated, the Exchange server will flag any non-compliant <code>P2 FROM</code> headers as suspicious. Additional information on this vulnerability can be found <a href="https://learn.microsoft.com/en-us/exchange/plan-and-deploy/post-installation-tasks/security-best-practices/exchange-non-compliant-p2from-detection?view&#61;exchserver-2019">here</a>.</p>
<table cellpadding="1" cellspacing="0" border="1"><caption>Table 2. Zero-day in Microsoft Exchange Server</caption><tbody><tr class="top-row bg-orange"><td>Severity</td><td>CVSS Score</td><td>CVE</td><td>Description</td></tr><tr><td><b>Important</b></td><td>7.5</td><td>CVE-2024-49040</td><td>Microsoft Exchange Server Spoofing Vulnerability</td></tr></tbody></table>
<h2>Two Critical Vulnerabilities in Microsoft Windows</h2>
<p><a href="https://msrc.microsoft.com/update-guide/vulnerability/CVE-2024-43639">CVE-2024-43639 </a>is a <b>Critical</b> remote code execution (<a href="https://www.crowdstrike.com/cybersecurity-101/remote-code-execution-rce/">RCE</a>) vulnerability affecting Microsoft Windows Kerberos and has a <b>CVSS</b> score of <b>9.8</b>. Successful exploitation of this vulnerability would allow an unauthenticated, remote attacker to use a specially crafted packet to leverage a cryptographic protocol within Windows Kerberos to allow for RCE. Due to the severe nature of this vulnerability, patching should be a high priority.</p>
<p><a href="https://msrc.microsoft.com/update-guide/vulnerability/CVE-2024-43625">CVE-2024-43625</a> is a <b>Critical</b> <a href="https://www.crowdstrike.com/tech-hub/?category&#61;endpoint-security">privilege escalation</a> vulnerability within the VMSwitch functionality of Hyper-V with a <b>CVSS</b> score of <b>8.1</b>. This vulnerability could allow an unprivileged attacker within a low-privileged Hyper-V host to traverse the security boundary to execute code within the Hyper-V execution environment. Notably, Microsoft states that this vulnerability is not within the System Center Virtual Machine Manager. This vulnerability has a high complexity requirement, requiring a significant amount of information about the operating environment of the victim Hyper-V host. <br />
</p>
<table cellpadding="1" cellspacing="0" border="1"><caption>Table 3. Critical vulnerabilities in Windows</caption><tbody><tr class="top-row bg-orange"><td>Severity</td><td>CVSS Score</td><td>CVE</td><td>Description</td></tr><tr><td><b>Critical</b></td><td>9.8</td><td>CVE-2024-43639</td><td>Windows Kerberos Remote Code Execution Vulnerability</td></tr><tr><td><b>Critical</b></td><td>8.1</td><td>CVE-2024-43625</td><td>Microsoft Windows VMSwitch Elevation of Privilege Vulnerability</td></tr></tbody></table>
<h2>One Critical Vulnerability in .Net and Visual Studio Code</h2>
<p><a href="https://msrc.microsoft.com/update-guide/vulnerability/CVE-2024-43498">CVE-2024-43498</a> is a <b>Critical</b> remote code execution (<a href="https://www.crowdstrike.com/cybersecurity-101/remote-code-execution-rce/">RCE</a>) vulnerability affecting Microsoft .Net and Visual Studio Code and has a <b>CVSS</b> score of <b>9.8</b>. Successful exploitation of this vulnerability would allow an unauthenticated, remote attacker to use a specially crafted request to a vulnerable .Net web app or by loading a specially crafted file into a vulnerable desktop application (Visual Studio Code).</p>
<table cellpadding="1" cellspacing="0" border="1"><caption>Table 4. <h2>Not All Relevant Vulnerabilities Have Patches: Consider Mitigation Strategies</h2>
<p>As we have learned with other notable vulnerabilities, such as <a href="https://www.crowdstrike.com/log4j2/">Log4j</a>, not every highly exploitable vulnerability can be easily patched. As is the case for the <a href="https://www.crowdstrike.com/blog/owassrf-exploit-analysis-and-recommendations/">ProxyNotShell</a> vulnerabilities, it’s critically important to develop a response plan for how to defend your environments when no patching protocol exists. </p>
<p>Regular review of your patching strategy should still be a part of your program, but you should also look more holistically at your organization&#39;s methods for cybersecurity and improve your overall security posture. </p>

<h3>About CVSS Scores</h3>
<p>The <a href="https://www.first.org/cvss/">Common Vulnerability Scoring System</a> (CVSS) is a free and open industry standard that CrowdStrike and many other cybersecurity organizations use to assess and communicate software vulnerabilities’ severity and characteristics. The CVSS Base Score ranges from 0.0 to 10.0, and the <a href="https://nvd.nist.gov/">National Vulnerability Database</a> (NVD) adds a severity rating for CVSS scores. Learn more about vulnerability scoring <a href="https://www.crowdstrike.com/en-us/epp-101/vulnerability-management-programs/">in this article</a>. </p>
