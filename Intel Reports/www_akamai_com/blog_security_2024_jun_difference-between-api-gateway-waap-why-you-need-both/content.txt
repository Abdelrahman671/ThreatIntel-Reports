    <h2>Introduction to Akamai API Gateway and Akamai App &amp; API Protector</h2>
<p>At Akamai, we're continuously evolving to address the dynamic <a href="/glossary/what-is-waap" target="_self">web application and API protection (WAAP)</a> challenges prevalent in today’s digital landscape. As a committed cybersecurity partner, we diligently survey the latest <a href="/glossary/how-do-apis-work" target="_self">application programming interface (API)</a> security threats so we can build robust solutions that best protect our valued customers.</p>
<p>Today, we’re highlighting the key capabilities of two Akamai solutions that can work together to bolster enterprise organizations’ security architecture: <a href="/glossary/what-is-an-api-gateway" target="_self">Akamai API Gateway</a>, our API traffic control solution, and <a href="/products/app-and-api-protector" target="_self">Akamai App &amp; API Protector</a>, our cloud WAAP solution.</p>
<h2>Understanding Akamai API Gateway</h2>
<p>API Gateway optimizes an organizational network’s API management and security, serving as a critical junction where incoming API requests are authenticated, authorized, and routed. This gateway acts as the first line of defense, scrutinizing API requests before they can interact with back-end systems — inspecting APIs to ensure that they are accessible, efficient, and secure by authenticating and authorizing traffic. </p>
<h2>Exploring Akamai App &amp; API Protector</h2>
<p>Our App &amp; API Protector transitions from the traditional <a href="/glossary/what-is-a-waf" target="_self">web application firewall (WAF)</a> and offers comprehensive protection tailored to both web applications and APIs. This solution embodies the next generation of WAAP technologies, providing a unified approach to secure assets from advanced security threats using automation and intelligence at scale. Further, it leverages adaptive security based on continuous learning from global traffic and threats, ensuring up-to-date defenses against evolving cybersecurity challenges. </p>
<p>Powered by strong, constantly updating technology and methodology, this holistic WAAP solution can evolve security postures for enterprise organizations while moving beyond the capabilities of traditional WAFs. </p>
<p>Although the terms “WAF” and “cloud WAAP solution” are often used interchangeably when discussing protection from the <a href="/glossary/what-is-owasp" target="_self">OWASP Top 10 Web Application Security Risks and the OWASP Top 10 API Security Risks,</a> we use the latter term as it incorporates protection for both web applications and APIs. </p>
<p>Figure 1 shows how API Gateway authenticates credentials for legitimate and malicious traffic intended for a customer’s environment, and how App &amp; API Protector then applies protections, visibilities, and edge cache scaling to that traffic, ensuring only legitimate traffic reaches the customer.</p>

    <h3>How API Gateway and WAAP work together</h3>
<ul>
<li><p>An API request’s journey starts at API Gateway, where it’s scrutinized, authenticated/authorized, and intelligently routed. </p>
</li>
<li><p>Once through the gateway, traffic flows into App &amp; API Protector, which conducts deeper security checks, including distributed<a href="/glossary/what-is-ddos" target="_self"> denial-of-service (DDoS)</a> attack mitigation, bot management, SSL and TLS encryption, and more. App &amp; API Protector applies security policies for mitigation and remediation of attack attempts. </p>
</li>
<li><p>Akamai’s layered security model allows only secure, legitimate requests to reach and interact with the customer’s critical back-end services and microservices.</p>
</li>
</ul>
<h3>How API Gateway and WAAP differ</h3>
<p>Although API Gateway and WAAP complement each other as integral components of Akamai’s security controls and architecture, they have distinct roles in mitigating API threats.</p>
<ul>
<li><p>API Gateway handles API traffic authentication, authorization, and routing to ensure traffic is optimized for runtime performance and initial security checks.</p>
</li>
<li><p>App &amp; API Protector (our WAAP solution) provides a deeper, more granular level of security. It protects against many malicious hacker approaches prevalent in the threat landscape, including sophisticated web attacks, API abuses, and more, with state-of-the-art security technologies like Akamai Adaptive Security Engine.</p>
</li>
</ul>
<h2>Why integrate API Gateway with your security stack?</h2>
<p>With API Gateway as the initial checkpoint for all your organization’s incoming API traffic, you get:</p>
<ul>
<li><p><b>Access control. </b>By validating JSON Web Tokens (JWT) and managing API keys at the edge, API Gateway prevents unnecessary traffic from reaching your core systems and offloads your identity provider while significantly reducing network latency.</p>
</li>
<li><p><b>Traffic management.</b> With the ability to set and enforce limits on API requests, your services stay operational even in the midst of excessive calls. It allows for fine-grained control over who can access what … and how often.</p>
</li>
<li><p><b>Enhanced reporting and policy enforcement.</b> API Gateway offers insights into API use patterns and can enforce policies like routing rules and API privacy settings, which ensures optimal performance and compliance with various standards.</p>
</li>
</ul>
<h2>App &amp; API Protector’s many-in-one security approach</h2>
<p>App &amp; API Protector is designed to be a fast, reliable, and comprehensive security solution, incorporating multiple sophisticated features and layers of defense within a single framework (Figure 2).</p>
