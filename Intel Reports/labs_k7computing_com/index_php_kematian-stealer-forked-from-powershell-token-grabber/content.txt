<p>
<p>Stealers are a widespread threat providing threat actors with access to a wealth of sensitive data which is then exfiltrated to them for further abuse. Kematian Stealer, a PowerShell based tool is one such sophisticated malware.</p>



<p>&nbsp;Recently we came across a <a href="https://x.com/suyog41/status/1802579698968322249">tweet</a> about Kematian Stealer. It was a PowerShell based Token-Grabber.</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" width="1024" height="520" src="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-1-1024x520.png" alt="" class="wp-image-27634" srcset="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-1-1024x520.png 1024w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-1-300x152.png 300w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-1-768x390.png 768w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-1.png 1030w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption>Figure 1: Execution_Flow</figcaption></figure></div>



<p></p>



<h2><strong>Binary Analysis</strong></h2>



<p>Let’s now analyse the malware in depth. The binary is a 64-bit portable executable and a loader file.</p>



<p>The loader written in C++ , contains an obfuscated script in its resource section.&nbsp;</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="818" height="192" src="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-2.png" alt="" class="wp-image-27636" srcset="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-2.png 818w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-2-300x70.png 300w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-2-768x180.png 768w" sizes="(max-width: 818px) 100vw, 818px" /><figcaption>Figure 2: Resource-Blob</figcaption></figure></div>



<p></p>



<p>The malware extracts the “112E9CAC33494A35D3547F4B3DCD2FD5” blob in the resource section,&nbsp; decrypts it, which is a batch file.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="782" height="608" src="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-3.png" alt="" class="wp-image-27638" srcset="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-3.png 782w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-3-300x233.png 300w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-3-768x597.png 768w" sizes="(max-width: 782px) 100vw, 782px" /><figcaption>Figure 3: Decryption_Loop</figcaption></figure></div>



<p></p>



<p>The above loop is used to decrypt the blob that was mentioned earlier. It was likely RC4.</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" width="1024" height="283" src="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-4-1024x283.png" alt="" class="wp-image-27640" srcset="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-4-1024x283.png 1024w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-4-300x83.png 300w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-4-768x212.png 768w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-4.png 1156w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption>Figure 4: Decrypted_Script</figcaption></figure></div>



<p></p>



<p>After decrypting, it tries to run the bat file with elevated&nbsp; privileges.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="760" height="128" src="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-5.png" alt="" class="wp-image-27642" srcset="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-5.png 760w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-5-300x51.png 300w" sizes="(max-width: 760px) 100vw, 760px" /><figcaption>Figure 5: Bat_File (am_admin)</figcaption></figure></div>



<p></p>



<p>The batch file containing the powershell_script is then executed.</p>



<p>On execution, it checks if the script is running with admin privileges. If not, it prompts the user to run the script with elevated privilege. If the script gets an elevated privilege, only then it moves on to the next function.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="694" height="163" src="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-6.png" alt="" class="wp-image-27644" srcset="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-6.png 694w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-6-300x70.png 300w" sizes="(max-width: 694px) 100vw, 694px" /><figcaption>Figure 6: Check_If_Admin</figcaption></figure></div>



<p></p>



<p>After that it runs the task function used for persistence. It creates persistence via the Windows&nbsp; Task Scheduler. First it creates a copy of the PowerShell script and places it in the %Appdata% folder with a filename percs.ps1.</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" width="1024" height="389" src="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-7-1024x389.png" alt="" class="wp-image-27647" srcset="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-7-1024x389.png 1024w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-7-300x114.png 300w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-7-768x292.png 768w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-7.png 1283w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption>Figure 7: Task_Creation</figcaption></figure></div>



<p></p>



<p>The script checks whether the directory, file, and task already exist before creating them. This prevents conflicts that would arise if multiple instances run simultaneously, potentially causing system instability or alerting the user of unusual behaviour.</p>



<p>Then it moves on to the data collection function called Grub.</p>



<h2><strong>Data collection</strong></h2>



<p>The grub function contains the main stealer code that’s mainly focused on system configuration and network environment information.</p>



<p>It begins with obtaining the system’s public IP by invoking the web request “Invoke-Web Request -Uri <a href="https://api.ipify.org">https://api.ipify.org</a>”, after obtaining the IP it stores it in a text file “ip.txt’ located in the users local application data directory “%LOCALAPPDATA%\Temp\ip.txt”.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="579" height="68" src="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-8.png" alt="" class="wp-image-27649" srcset="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-8.png 579w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-8-300x35.png 300w" sizes="(max-width: 579px) 100vw, 579px" /><figcaption>Figure 8: IP_Stealer</figcaption></figure></div>



<p></p>



<p>It then collects system information using the Windows command-line. PowerShell executes the Systeminfo.exe which retrieves the system information like OS Version, Host Name, System Model and more. After getting all the information it redirects the information to a text file named “system_info.txt” and stores it in the user&#8217;s “%LOCALAPPDATA%\Temp\ System_info.txt” location.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="906" height="703" src="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-9.png" alt="" class="wp-image-27651" srcset="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-9.png 906w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-9-300x233.png 300w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-9-768x596.png 768w" sizes="(max-width: 906px) 100vw, 906px" /><figcaption>Figure 9: System_Info_stealer</figcaption></figure></div>



<p></p>



<p>After collecting System info and System Public IP, it starts to collect System UUID and Mac addresses using WMI. It extracts the UUID and Mac address value from the WMI and stores it a text file named “uuid.txt” and “mac.txt” in the “%LOCALAPPDATA%\Temp\uuid.txt” and “%LOCALAPPDATA%\Temp\mac.txt” location.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="792" height="51" src="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-10.png" alt="" class="wp-image-27653" srcset="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-10.png 792w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-10-300x19.png 300w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-10-768x49.png 768w" sizes="(max-width: 792px) 100vw, 792px" /><figcaption>Figure 10: UUID_stealer</figcaption></figure></div>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="861" height="72" src="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-11.png" alt="" class="wp-image-27654" srcset="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-11.png 861w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-11-300x25.png 300w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-11-768x64.png 768w" sizes="(max-width: 861px) 100vw, 861px" /><figcaption>Figure 11: MAC_Stealer</figcaption></figure></div>



<p></p>



<p>After collecting the UUID and Mac address it collects the info about the system’s current username and hostname by using the system environment variable.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="349" height="60" src="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-12.png" alt="" class="wp-image-27655" srcset="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-12.png 349w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-12-300x52.png 300w" sizes="(max-width: 349px) 100vw, 349px" /><figcaption>Figure 12: User &amp; Host</figcaption></figure></div>



<p>&nbsp;</p>



<p>At last it collects the system netstat information by using the Windows command-line. The PowerShell script executes NETSTAT.exe and retrieves the network statistics, like active connections, listening ports with the associated Process IDs.</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" width="1024" height="702" src="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-13-1024x702.png" alt="" class="wp-image-27656" srcset="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-13-1024x702.png 1024w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-13-300x206.png 300w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-13-768x526.png 768w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-13.png 1042w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption>Figure 13: Netstat_Stealer</figcaption></figure></div>



<p></p>



<p>After that the author constructs a detailed and formatted message to be sent to a Discord channel using a web hook. The script includes system information about the victim (IP, username, hostname, UUID, MAC address) formatted as fields and visual elements like colour, thumbnail, and footer to make the message more appealing and structured. With this it sends the POST request to the specified Web Hook url that is mentioned within the JSON payload.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="850" height="774" src="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-14.png" alt="" class="wp-image-27657" srcset="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-14.png 850w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-14-300x273.png 300w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-14-768x699.png 768w" sizes="(max-width: 850px) 100vw, 850px" /><figcaption>Figure 14: Discord_Structure</figcaption></figure></div>



<p></p>



<p>Then it tries to terminate some Discord related process and also tries to remove some files if it exists, like Discord Token Protector etc. that could protect from malicious grabbers. To evade detection from security products, it checks the presence of Discord token protector.exe and secure.dat. If these files are present in the Discord token directory, the malware removes them.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="728" height="229" src="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-15.png" alt="" class="wp-image-27659" srcset="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-15.png 728w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-15-300x94.png 300w" sizes="(max-width: 728px) 100vw, 728px" /><figcaption>Figure 15: Discord_Kill</figcaption></figure></div>



<p></p>



<p>After that it checks if the particular directory exists or not, if it is available, it proceeds further else it creates a new directory “LOCALAPPDATA\Temp\percs”.</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" width="1024" height="133" src="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-16-1024x133.png" alt="" class="wp-image-27660" srcset="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-16-1024x133.png 1024w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-16-300x39.png 300w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-16-768x100.png 768w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-16.png 1284w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption>Figure 16: Downloading_Payload</figcaption></figure></div>



<p></p>



<p>After creating a particular directory, it tries to download a payload called main.exe. But unfortunately it’s not available in that particular web page; it redirects to the Kematian stealer GitHub page instead.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="843" height="48" src="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-17.png" alt="" class="wp-image-27662" srcset="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-17.png 843w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-17-300x17.png 300w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-17-768x44.png 768w" sizes="(max-width: 843px) 100vw, 843px" /><figcaption>Figure 17: Url_Redirection</figcaption></figure></div>



<p></p>



<p>At this stage of analysis, we understand that the stealer is a previous version of the Kematian stealer. Initially known as PowerShell-Token-Grabber; it was built by author KDot227 and now changed to Somali-Devs. In their recent updates they also mentioned about the author change in their source code and the GitHub page also redirects to the Kematian stealer GitHub page.</p>



<p>We got the main.exe from <a href="https://www.virustotal.com/gui/file/a1d1c36892c340a7ec9657ab7ed7b37aac76bd767ec4412ef1a3863c83263cfd/detection">Virus total </a>&nbsp;which was a python based executable. While decompiling the python executable, we came to know that this is where the browser stealer code is present. It focuses mainly on browser cookies, passwords, history details and the desktop screenshot.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="615" height="397" src="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-18.png" alt="" class="wp-image-27663" srcset="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-18.png 615w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-18-300x194.png 300w" sizes="(max-width: 615px) 100vw, 615px" /><figcaption>Figure 18: Targeted_Browsers</figcaption></figure></div>



<p></p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" width="1024" height="69" src="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-19-1024x69.png" alt="" class="wp-image-27664" srcset="https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-19-1024x69.png 1024w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-19-300x20.png 300w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-19-768x52.png 768w, https://labs.k7computing.com/wp-content/uploads/2024/07/Fig-19.png 1286w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption>Figure 19: Desktop_Grabber</figcaption></figure></div>



<p></p>



<p>It also targets Discord tokens; it tries to inject code into various discord clients to capture discord tokens, for that it tries to download JavaScript by the author KDot227 in the name of injection.js.</p>

<h2><strong>Data Exfiltration</strong></h2>



<p>After collecting all the required data, it then moves all the collected data from the application data directory to the newly created directory “LOCALAPPDATA\Temp\percs\”. It also tries to search for browser cookies, passwords and get the desktop screengrab; it was unable to retrieve the same as&nbsp; the webpage was not available. At last it compresses all the text files and zip the particular data directory.</p>

<p>Curl.exe is used for transferring the data along with a Json payload which contains the name and content. Finally, the grabber exfiltrates all the data to the Discord channel using a web hook.</p>
