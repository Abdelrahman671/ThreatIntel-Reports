<h2 id="default-enablement">Default enablement</h2>
<div class="IMPORTANT">
<p>Important</p>
<p>Windows Server 2025 is in preview. This information relates to a prerelease product that may be substantially modified before it's released. Microsoft makes no warranties, expressed or implied, with respect to the information provided here.</p>
</div>
<p>Starting in Windows 11, 22H2 and Windows Server 2025 (preview), Credential Guard is <a href="./#default-enablement" data-linktype="relative-path">enabled by default on devices which meet the requirements</a>.</p>
<p>System administrators can explicitly <a href="#enable-credential-guard" data-linktype="self-bookmark">enable</a> or <a href="#disable-credential-guard" data-linktype="self-bookmark">disable</a> Credential Guard using one of the methods described in this article. Explicitly configured values overwrite the default enablement state after a reboot.</p>
<p>If a device has Credential Guard explicitly turned off before updating to a newer version of Windows where Credential Guard is enabled by default, it will remain disabled even after the update.</p>
<div class="IMPORTANT">
<p>Important</p>
<p>For information about known issues related to default enablement, see <a href="considerations-known-issues" data-linktype="relative-path">Credential Guard: known issues</a>.</p>
</div>

<h3 id="verify-if-credential-guard-is-enabled">Verify if Credential Guard is enabled</h3>
<p>Checking Task Manager if <code>LsaIso.exe</code> is running isn't a recommended method for determining whether Credential Guard is running. Instead, use one of the following methods:</p>
<ul>
<li>System Information</li>
<li>PowerShell</li>
<li>Event Viewer</li>
</ul>
<h4 id="system-information">System Information</h4>
<p>You can use <em>System Information</em> to determine whether Credential Guard is running on a device.</p>
<ol>
<li>Select <strong>Start</strong>, type <code>msinfo32.exe</code>, and then select <strong>System Information</strong></li>
<li>Select <strong>System Summary</strong></li>
<li>Confirm that <strong>Credential Guard</strong> is shown next to <strong>Virtualization-based Security Services Running</strong></li>
</ol>
<h4 id="powershell">PowerShell</h4>
<p>You can use PowerShell to determine whether Credential Guard is running on a device. From an elevated PowerShell session, use the following command:</p>
<pre><code class="lang-powershell">(Get-CimInstance -ClassName Win32_DeviceGuard -Namespace root\Microsoft\Windows\DeviceGuard).SecurityServicesRunning
</code></pre>
<p>The command generates the following output:</p>
<ul>
<li><strong>0</strong>: Credential Guard is disabled (not running)</li>
<li><strong>1</strong>: Credential Guard is enabled (running)</li>
</ul>

<h3 id="disable-credential-guard-with-uefi-lock">Disable Credential Guard with UEFI lock</h3>
<p>If Credential Guard is enabled with UEFI lock, follow this procedure since the settings are persisted in EFI (firmware) variables.</p>
<div class="NOTE">
<p>Note</p>
<p>This scenario requires physical presence at the machine to press a function key to accept the change.</p>
</div>
<ol>
<li><p>Follow the steps in <a href="#disable-credential-guard" data-linktype="self-bookmark">Disable Credential Guard</a></p>
</li>
<li><p>Delete the Credential Guard EFI variables by using bcdedit. From an elevated command prompt, type the following commands:</p>
<pre><code class="lang-cmd">mountvol X: /s
copy %WINDIR%\System32\SecConfig.efi X:\EFI\Microsoft\Boot\SecConfig.efi /Y
bcdedit /create {0cb3b571-2f2e-4343-a879-d86a476d7215} /d "DebugTool" /application osloader
bcdedit /set {0cb3b571-2f2e-4343-a879-d86a476d7215} path "\EFI\Microsoft\Boot\SecConfig.efi"
bcdedit /set {bootmgr} bootsequence {0cb3b571-2f2e-4343-a879-d86a476d7215}
bcdedit /set {0cb3b571-2f2e-4343-a879-d86a476d7215} loadoptions DISABLE-LSA-ISO
bcdedit /set {0cb3b571-2f2e-4343-a879-d86a476d7215} device partition=X:
mountvol X: /d
</code></pre>
</li>
<li><p>Restart the device. Before the OS boots, a prompt appears notifying that UEFI was modified, and asking for confirmation. The prompt must be confirmed for the changes to persist.</p>
</li>
</ol>

</code></pre>

</tbody>
</table>
<p>Assign the policy to a group that contains as members the devices or users that you want to configure.</p>
<p>Alternatively, you can configure devices using a <a href="/en-us/mem/intune/configuration/custom-settings-configure" data-linktype="absolute-path">custom policy</a> with the <a href="/en-us/windows/client-management/mdm/policy-csp-deviceguard#enablevirtualizationbasedsecurity" data-linktype="absolute-path">DeviceGuard Policy CSP</a>.</p>
<table>
<thead>
<tr>
<th>Setting</th>
</tr>
</thead>
<tbody>
<tr>
<td><strong>Setting name</strong>: Turn On Virtualization Based Security<br><strong>OMA-URI</strong>: <code>./Device/Vendor/MSFT/Policy/Config/DeviceGuard/EnableVirtualizationBasedSecurity</code><br><strong>Data type</strong>: int<br><strong>Value</strong>: <code>0</code></td>
</tr>
</tbody>
</table>
<p>Once the policy is applied, restart the device.</p>
</section>
<section id="tabpanel_3_gpo" role="tabpanel" data-tab="gpo" aria-hidden="true" hidden="hidden">
<h3 id="disable--vbs-with-group-policy">Disable  VBS with group policy</h3>
<p>Configure the policy used to enable VBS to <strong>Disabled</strong>.</p>
<p>To configure a device with group policy, use the <a href="/en-us/previous-versions/windows/it-pro/windows-server-2008-r2-and-2008/cc731745(v=ws.10)" data-linktype="absolute-path">Local Group Policy Editor</a>. To configure multiple devices joined to Active Directory, <a href="/en-us/previous-versions/windows/it-pro/windows-server-2008-r2-and-2008/cc754740(v=ws.11)" data-linktype="absolute-path">create or edit</a> a group policy object (GPO) and use the following settings:</p>
<table>
<thead>
<tr>
<th>Group policy path</th>
<th>Group policy setting</th>
<th>Value</th>
</tr>
</thead>
<tbody>
<tr>
<td><strong>Computer Configuration\Administrative Templates\System\Device Guard\Turn on Virtualization Based Security</strong></td>
<td>Turn On Virtualization Based Security</td>
<td><strong>Disabled</strong></td>
</tr>
</tbody>
</table>
<p>Group policies can be <a href="/en-us/previous-versions/windows/it-pro/windows-server-2008-r2-and-2008/cc732979(v=ws.10)" data-linktype="absolute-path">linked</a> to domains or organizational units, <a href="/en-us/previous-versions/windows/it-pro/windows-server-2008-r2-and-2008/cc752992(v=ws.10)" data-linktype="absolute-path">filtered using security groups</a>, or <a href="/en-us/previous-versions/windows/it-pro/windows-server-2012-r2-and-2012/jj717288(v=ws.11)" data-linktype="absolute-path">filtered using WMI filters</a>.</p>
<p>Once the policy is applied, restart the device</p>
</section>
<section id="tabpanel_3_reg" role="tabpanel" data-tab="reg" aria-hidden="true" hidden="hidden">
<h3 id="disable-vbs-with-registry-settings">Disable VBS with registry settings</h3>

<tr>
<th>Setting</th>
</tr>
</thead>
<tbody>
<tr>

</tr>
<tr>
<td>Key path: <code>HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\DeviceGuard</code> <br>Key name: <code>RequirePlatformSecurityFeatures</code></td>
</tr>
</tbody>
</table>
<div class="IMPORTANT">
<p>Important</p>
<p>If you manually remove the registry settings, make sure to delete them all, otherwise the device might go into BitLocker recovery.</p>
</div>
<p>Restart the device to apply the change.</p>
