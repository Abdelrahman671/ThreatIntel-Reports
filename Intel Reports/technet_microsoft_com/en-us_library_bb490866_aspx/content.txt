							<div class="content "><h1 id="at">At</h1><div class="display-flex justify-content-space-between align-items-center flex-wrap-wrap page-metadata-container">
										<div class="margin-right-xxs">
											<ul class="metadata page-metadata" data-bi-name="page info" lang="en-us" dir="ltr"><li>Article</li><li>
														<time class="is-invisible" data-article-date aria-label="Article review date" datetime="2009-09-11T00:00:00.000Z" data-article-date-source="ms.date">09/11/2009</time>
													</li></ul>
										</div></div><nav id="center-doc-outline" class="doc-outline is-hidden-desktop display-none-print margin-bottom-sm" data-bi-name="intopic toc" aria-label="In this article">
											<h2 id="ms--in-this-article" class="title is-6 margin-block-xs">In this article</h2>
										</nav><!-- <content> --><p>Schedules commands and programs to run on a computer at a specified time and date. You can use <strong>at</strong> only when the Schedule service is running. Used without parameters, <strong>at</strong> lists scheduled commands.</p>
<h3 id="syntax">Syntax</h3>
<p><strong>at</strong> [<strong>\\<strong><em>ComputerName</em>] [{[<em>ID</em>] [</strong>/delete</strong>]|<strong>/delete</strong> [<strong>/yes</strong>]}]</p>
<p><strong>at</strong> [[<strong>\\<strong><em>ComputerName</em>] <em>hours</em></strong>:<strong><em>minutes</em> [/<strong>interactive</strong>] [{</strong>/every:<strong><em>date</em>[</strong>,</strong>...]|<strong>/next:<strong><em>date</em>[</strong>,</strong>...]}] <em>command</em>]</p>
<h3 id="parameters">Parameters</h3>
<p><strong>\\</strong> <em><strong>ComputerName</strong></em> &nbsp; <strong>:</strong> Specifies a remote computer. If you omit this parameter, <strong>at</strong> schedules the commands and programs on the local computer.</p>
<p><em><strong>ID</strong></em> &nbsp; <strong>:</strong> Specifies the identification number assigned to a scheduled command.</p>
<p><strong>/delete</strong> &nbsp; <strong>:</strong> Cancels a scheduled command. If you omit <em>ID</em>, all of the scheduled commands on the computer are canceled.</p>
<p><strong>/yes</strong> &nbsp; <strong>:</strong> Answers yes to all queries from the system when you delete scheduled events.</p>
<p><em><strong>hours</strong></em> <strong>:</strong> <em><strong>minutes</strong></em> &nbsp; <strong>:</strong> Specifies the time when you want to run the command. Time is expressed as <em>hours</em>**:**<em>minutes</em> in 24-hour notation (that is, 00:00 [midnight] through&nbsp;23:59).</p>
<p><strong>/interactive</strong> &nbsp; <strong>:</strong> Allows <em>command</em> to interact with the desktop of the user who is logged on at the time <em>command</em> runs.</p>
<p><strong>/every:</strong> &nbsp; <strong>:</strong> Runs <em>command</em> on every specified day or days of the week or month (for example, every Thursday, or the third day of every month).</p>
<p><em><strong>date</strong></em> &nbsp; <strong>:</strong> Specifies the date when you want to run the command. You can specify one or more days of the week (that is, type <strong>M</strong>,<strong>T</strong>,<strong>W</strong>,<strong>Th</strong>,<strong>F</strong>,<strong>S</strong>,<strong>Su</strong>) or one or more days of the month (that is, type 1 through 31). Separate multiple date entries with commas. If you omit <em>date</em>, <strong>at</strong> uses the current day of the month.</p>
<p><strong>/next:</strong> &nbsp; <strong>:</strong> Runs <em>command</em> on the next occurrence of the day (for example, next Thursday).</p>
<p><em><strong>command</strong></em> &nbsp; <strong>:</strong> Specifies the Windows command, program (that is, .exe or .com file), or batch program (that is, .bat or .cmd file) that you want to run. When the command requires a path as an argument, use the absolute path (that is, the entire path beginning with the drive letter). If the command is on a remote computer, specify Universal Naming Convention (UNC) notation for the server and share name, rather than a remote drive letter.</p>
<p><strong>/?</strong> &nbsp; <strong>:</strong> Displays help at the command prompt.</p>
<h3 id="remarks">Remarks</h3>
<ul>
<li><p><strong>Schtasks</strong> is a more powerful superset command-line scheduling tool and includes all the functionality found in the <strong>at</strong> command-line utility. Use <strong>schtasks</strong> instead of <strong>at</strong> for all command-line scheduling tasks. For more information about <strong>schtasks</strong>, see Related Topics.</p>
</li>
<li><p>Using <strong>at</strong>&nbsp;</p>
<p>To use <strong>at</strong>, you must be a member of the local Administrators group.</p>
</li>
<li><p>Loading Cmd.exe</p>
<p><strong>At</strong> does not automatically load Cmd.exe, the command interpreter, before running commands. If you are not running an executable (.exe) file, you must explicitly load Cmd.exe at the beginning of the command as follows:</p>
<p><strong>cmd /c dir &gt; c:\test.out</strong></p>
</li>
<li><p>Viewing scheduled commands</p>
<p>When you use <strong>at</strong> without command-line options, scheduled tasks appear in a table formatted similar to the following:</p>
<pre>Status&nbsp;&nbsp;ID&nbsp;&nbsp;&nbsp;Day&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Time&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Command Line
</pre></li>
</ul>
<p>OK&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;1&nbsp;&nbsp;&nbsp;&nbsp;Each F&nbsp;&nbsp;&nbsp;&nbsp;4:30 PM&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;net send group leads status due
OK&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;2&nbsp;&nbsp;&nbsp;&nbsp;Each M&nbsp;&nbsp;&nbsp;&nbsp;12:00 AM&nbsp;&nbsp;&nbsp;&nbsp;chkstor &gt; check.file
OK&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;3&nbsp;&nbsp;&nbsp;&nbsp;Each F&nbsp;&nbsp;&nbsp;&nbsp;11:59 PM&nbsp;&nbsp;&nbsp; backup2.bat </p>
<ul>
<li><p>Including identification number (<em>ID</em>)</p>
<p>When you include identification number (<em>ID</em>) with <strong>at</strong> at a command prompt, information for a single entry appears in a format similar to the following:</p>
<pre>Task ID:      1
</pre></li>
</ul>
<p>Status:       OK
Schedule:     Each  F
Time of Day:  4:30 PM
Command:      net send group leads status due</p>
<pre><code>After you schedule a command with **at**, especially a command that has command-line options, check that the command syntax is correct by typing **at** without command-line options. If the information in the Command Line column is incorrect, delete the command and retype it. If it is still incorrect, retype the command with fewer command-line options.
</code></pre>
<ul>
<li><p>Viewing results</p>
<p>Commands scheduled with <strong>at</strong> run as background processes. Output is not displayed on the computer screen. To redirect output to a file, use the redirection symbol (&gt;). If you redirect output to a file, you need to use the escape symbol (^) before the redirection symbol, whether you are using <strong>at</strong> at the command line or in a batch file. For example, to redirect output to Output.text, type:</p>
<p><strong>at 14:45 c:\test.bat ^&gt;c:\output.txt</strong></p>
<p>The current directory for the executing command is the systemroot folder.</p>
</li>
<li><p>Changing system time</p>
<p>If you change the system time at a computer after you schedule a command to run with <strong>at</strong>, synchronize the <strong>at</strong> scheduler with the revised system time by typing <strong>at</strong> without command-line options.</p>
</li>
<li><p>Storing commands</p>
<p>Scheduled commands are stored in the registry. As a result, you do not lose scheduled tasks if you restart the Schedule service.</p>
</li>
<li><p>Connecting to network drives</p>
<p>Do not use a redirected drive for scheduled jobs that access the network. The Schedule service might not be able to access the redirected drive, or the redirected drive might not be present if a different user is logged on at the time the scheduled task runs. Instead, use UNC paths for scheduled jobs. For example:</p>
<p><strong>at 1:00pm my_backup \\server\share</strong></p>
<p>Do not use the following syntax, where <strong>x:</strong> is a connection made by the user:</p>
<p><strong>at 1:00pm my_backup x:</strong></p>
<p>If you schedule an <strong>at</strong> command that uses a drive letter to connect to a shared directory, include an <strong>at</strong> command to disconnect the drive when you are finished using the drive. If the drive is not disconnected, the assigned drive letter is not available at the command prompt.</p>
</li>
</ul>
<h3 id="examples">Examples</h3>
<p>To display a list of commands scheduled on the Marketing server, type:</p>
<p><strong>at \\marketing</strong>&nbsp;</p>
<p>To learn more about a command with the identification number 3 on the Corp server, type:</p>
<p><strong>at \\corp 3</strong>&nbsp;</p>
<p>To schedule a net share command to run on the Corp server at 8:00 A.M. and redirect the listing to the Maintenance server, in the Reports shared directory, and the Corp.txt file, type:</p>
<p><strong>at \\corp 08:00 cmd /c "net share reports=d:\marketing\reports &gt;&gt; \\maintenance\reports\corp.txt"</strong>&nbsp;</p>
<p>To back up the hard drive of the Marketing server to a tape drive at midnight every five days, create a batch program called Archive.cmd, which contains the backup commands, and then schedule the batch program to run, type:</p>
<p><strong>at \\marketing 00:00 /every:5,10,15,20,25,30 archive</strong>&nbsp;</p>
<p>To cancel all commands scheduled on the current server, clear the <strong>at</strong> schedule information as follows:</p>
<p><strong>at /delete</strong>&nbsp;</p>
<p>To run a command that is not an executable (that is, .exe) file, precede the command with <strong>cmd /c</strong> to load Cmd.exe as follows:</p>
<p><strong>cmd /c dir &gt; c:\test.out</strong>&nbsp;</p>
