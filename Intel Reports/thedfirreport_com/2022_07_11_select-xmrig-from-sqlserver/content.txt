	<div class="entry-content-holder">
		<header class="entry-header">

		
				<div class="entry-meta">

					
		<span class="cat-links">
		
			<a class="category-color-60" href="https://thedfirreport.com/category/coinminer/">coinminer</a>

			<h1 class="entry-title">SELECT XMRig FROM SQLServer</h1>
		<div class="entry-meta">
			<span class="posted-on"><a href="https://thedfirreport.com/2022/07/11/select-xmrig-from-sqlserver/" rel="bookmark"><time class="entry-date published" datetime="2022-07-11T01:51:24+00:00">July 11, 2022</time></a></span>		</div><!-- .entry-meta -->

		
	</header><!-- .entry-header -->

	<div class="entry-content">
		
<p>In March 2022, we observed an intrusion on a public-facing Microsoft SQL Server. The end goal of this intrusion was to deploy a coin miner. Although deploying a coin miner on a vulnerable server after successful exploitation is a common objective for threat actors, this intrusion was slightly different and therefore more interesting.</p>



<p>US CERT recently published a malware analysis report related to XMRig coin miner on 23rd June 2022 (<a href="https://www.cisa.gov/uscert/ncas/analysis-reports/ar22-174a" target="_blank" rel="noreferrer noopener">https://www.cisa.gov/uscert/ncas/analysis-reports/ar22-174a</a>) and a security tip for defending against this threat (<a href="https://www.cisa.gov/uscert/ncas/tips/ST18-002" target="_blank" rel="noreferrer noopener">https://www.cisa.gov/uscert/ncas/tips/ST18-002</a>).</p>



<h2 class="wp-block-heading">Summary</h2>



<p>Over the month of March, we observed a cluster of activity targeting MSSQL servers. The activity started via password brute force attempts for the MSSQL SA account. These brute force attempts were observed repeatedly over the month. Examples included one cluster of 24,000 failed attempts from the same source, over a 27 hour effort, before they finally managed to guess the password. After having the correct credentials in their possession, the attackers then spawned a command shell via xp_cmdshell. According to <a href="https://docs.microsoft.com/en-us/sql/relational-databases/system-stored-procedures/xp-cmdshell-transact-sql?view=sql-server-ver15" target="_blank" rel="noopener">Microsoft documentation</a>, xp_cmdshell spawns a Windows command shell and passes in a string for execution.</p>



<p>Using xp_cmdshell, the threat actors were able to execute any command against the compromised server. They attempted to kill a bunch of AV programs by using taskkill.exe. The threat actors then wrote multiple commands to a batch file by using echo and redirecting the strings to a file named 1.bat. After the batch file was written they then proceeded to perform the same action echoing data into a file named bigfile.txt. After they finished writing to that file, they ran certutil to decode the base64 data into an executable file. This executable was a privilege escalation tool that was used to execute the batch file to make sure it executed with high enough permissions. They then executed the batch script. The commands included adding new users to the local administrators group, enabling RDP, enabling WDigest, and hiding the newly created admin accounts using the registry.</p>



<p>Once the threat actors had established persistence on the compromised host, they moved to their final objective, which was to install and run the XMRig miner. They dropped a Binary Managed Object Format (BMOF) file along with the miner itself, to do that. The threat actors used mofcomp.exe to decompile the BMOF binary and register a malicious class in the WMI repository. The event consumer of the newly created classes included a VBE script responsible for setting up and executing the XMRig miner with the correct settings.</p>



<p>No other activity beyond the mining was observed before the threat actors were evicted.</p>



<h2 class="wp-block-heading">Services</h2>



<p>We offer multiple services including a <a href="https://thedfirreport.com/services/" target="_blank" rel="noreferrer noopener">Threat Feed service</a> that tracks Command and Control frameworks such as Cobalt Strike, Sliver, BumbleBee, Covenant, Metasploit, Empire, PoshC2, etc. More information on this service and others can be found <a href="https://thedfirreport.com/services/" target="_blank" rel="noreferrer noopener">here</a>.</p>



<p>Artifacts for this case are limited due to the environment. A few log sources are available for this case under our <a href="https://www.patreon.com/thedfirreport" target="_blank" rel="noreferrer noopener">Security Researcher and Organization</a> services.</p>



<p>Analysis and reporting completed by <a href="https://twitter.com/_pete_0" target="_blank" rel="noreferrer noopener">@_pete_0</a> and <a href="https://twitter.com/Kostastsale" target="_blank" rel="noreferrer noopener">@kostastsale</a></p>



<h2 class="wp-block-heading">Initial Access</h2>



<p>The initial access took place via a brute-force attack, where the threat actors mainly targeted the System Admin (SA) account.</p>



<p>During the intrusions, we could see SQL Server event ID <strong>18456</strong> Failure Audit Events in the Windows application logs. We witnessed more than 24,000 attempts from the same source before the threat actors successfully guessed the username and password for the open SQL database.</p>
<p>Example of the failed brute force attempts:</p>



<figure class="wp-block-image"><a href="https://thedfirreport.com/wp-content/uploads/2022/07/12780-01.png"><img fetchpriority="high" decoding="async" class="aligncenter size-full wp-image-8041" src="https://thedfirreport.com/wp-content/uploads/2022/07/12780-01.png" alt="" width="590" height="330" srcset="https://thedfirreport.com/wp-content/uploads/2022/07/12780-01.png 590w, https://thedfirreport.com/wp-content/uploads/2022/07/12780-01-300x168.png 300w" sizes="(max-width: 590px) 100vw, 590px" /></a></figure>



<p>Followed by eventual successful logins.</p>



<figure class="wp-block-image"><a href="https://thedfirreport.com/wp-content/uploads/2022/07/12780-02.png"><img decoding="async" class="aligncenter size-full wp-image-8042" src="https://thedfirreport.com/wp-content/uploads/2022/07/12780-02.png" alt="" width="736" height="80" srcset="https://thedfirreport.com/wp-content/uploads/2022/07/12780-02.png 736w, https://thedfirreport.com/wp-content/uploads/2022/07/12780-02-300x33.png 300w" sizes="(max-width: 736px) 100vw, 736px" /></a></figure>



<p>It it likely that multiple successful logins were observed due to the automated access script that the threat actor was using.</p>



<h2 class="wp-block-heading">Execution</h2>



<p>In the next attack stage, the threat actors established a cmd shell via <a href="https://docs.microsoft.com/en-us/sql/relational-databases/system-stored-procedures/xp-cmdshell-transact-sql?view=sql-server-ver15" target="_blank" rel="noreferrer noopener">Extended SQL Stored Procedure</a> (xp_cmdshell). This process allows you to issue operating system commands directly to the Windows command shell using T-SQL code. An example of command execution following a successful authentication to SQL database using xp_cmdshell:</p>

<p>Additionally, we noticed the attackers dropping a file named &#8220;xitmf&#8221;. Looking into the file&#8217;s content, we noticed that the header began with &#8220;FOMB&#8221;. When flipping the header, it spells BMOF, which indicates a Binary Managed Object Format file. BMOF is a compiled version of a Managed Object Format (MOF) file. <a href="https://docs.microsoft.com/en-us/windows/win32/wmisdk/managed-object-format--mof-" target="_blank" rel="noreferrer noopener">As per Microsoft&#8217;s official documentation</a>:</p>



<blockquote class="wp-block-quote is-layout-flow wp-block-quote-is-layout-flow">

</blockquote>



<p>MOF files are compiled using the Windows compiler tool mofcomp. <a href="https://docs.microsoft.com/en-us/windows/win32/wmisdk/mofcomp" target="_blank" rel="noreferrer noopener">Mofcomp.exe</a> is also used to execute scripts by parsing the MOF statements and creates new classes as part of the WMI repository. </p>

<p>Looking into the VBScript, we notice that it is reaching out to the domain mymst007[.]info on port 4000 to download one more file and save it as temp file.</p>

<p>The final method of persistence we observed was the addition of an entry into the Image File Execution Option (<a href="https://docs.microsoft.com/en-us/archive/blogs/mithuns/image-file-execution-options-ifeo" target="_blank" rel="noreferrer noopener">IFEO</a>) registry key. By changing the Debugger value to a different executable, an attacker used IFEO to launch a program other than the intended one. In this case, threat actors modified the below registry key to launch the miner executable (smss.exe) instead of the svchost.exe binary.</p>



<pre><code>"cmd.exe" /c REG ADD "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Image File Execution Options\smss.exe" /f /v Debugger /t REG_SZ /d "C:\windows\system32\svchost.exe</code></pre>



<h2 class="ql-heading wp-block-heading" data-block-id="block-0e6b280f-c56b-4d81-b576-080f7947382f">Privilege Escalation</h2>
<p>The threat actors dropped a file named bigfile.txt which they used certutil to convert to bigfile.exe which we believe is a variation of NetworkServiceExploit.exe as seen below.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2022/07/12780-25.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-8069" src="https://thedfirreport.com/wp-content/uploads/2022/07/12780-25.png" alt="" width="1267" height="1216" srcset="https://thedfirreport.com/wp-content/uploads/2022/07/12780-25.png 1267w, https://thedfirreport.com/wp-content/uploads/2022/07/12780-25-300x288.png 300w, https://thedfirreport.com/wp-content/uploads/2022/07/12780-25-1024x983.png 1024w, https://thedfirreport.com/wp-content/uploads/2022/07/12780-25-768x737.png 768w" sizes="auto, (max-width: 1267px) 100vw, 1267px" /></a>This was used in this intrusion to run the batch file with the following command:</p>
<pre>"cmd.exe" /c %USERPROFILE%\AppData\bigfile.exe -i -c %USERPROFILE%\AppData\1.bat</pre>
<h2>Defense Evasion</h2>



<p>The threat actors attempted to kill antivirus tasks that could be running on the host. The commands targeted the below processes:</p>



<pre class="wp-block-preformatted">QQPCTray.exe
QQPCRTP.exe
QQPCMgr.exe
kavsvc.exe
alg.exe

SafeDogServerUI.exe
kxescore.exe
kxetray.exe
360safe.exe
360sd.exe
360rp.exe
360rps.exe
360tray.exe
ZhuDongFangYu.exe</pre>



<p>The privilege escalation tool the threat actors brought with them was written as a text file and then decoded using certutil into a binary file.</p>
<pre>"cmd.exe" /c certutil -decode %USERPROFILE%\AppData\bigfile.txt %USERPROFILE%\AppData\bigfile.exe</pre>
<p>As we can see from the contents of the 1.bat script, the threat actors are adding a new local administrator user and they proceed with hiding the user account by adding it to the registry using &#8220;<a href="https://attack.mitre.org/techniques/T1564/002/" target="_blank" rel="noreferrer noopener">Special Accounts</a>&#8220;.</p>



<pre><code>REG ADD "HKLM\Software\Microsoft\Windows NT\CurrentVersion\Winlogon\SpecialAccounts\Userlist" /v Adminv$ /t REG_DWORD /d 0</code></pre>



<p>Through the execution of the initial batch script, 1.bat, they also disabled the User Access Control(UAC) remote restriction by setting the registry key value to &#8220;1&#8221;.</p>



<pre><code>reg add "HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System" /v LocalAccountTokenFilterPolicy /t REG_DWORD /d 1 /f</code></pre>



<p>Threat actors also enabled Wdigest.</p>



<pre><code>reg add HKLM\SYSTEM\CurrentControlSet\Control\SecurityProviders\WDigest /v UseLogonCredential /t REG_DWORD /d 1 /f</code></pre>



<p>After many files were added to the system the threat actors included commands to remove them once their execution was finished.</p>
<pre>"cmd.exe" /c DEL %USERPROFILE%\AppData\1.bat<br />"cmd.exe" /c DEL %USERPROFILE%\AppData\bigfile.txt<br />"cmd.exe" /c DEL %USERPROFILE%\AppData\bigfile.exe</pre>
<h2 class="wp-block-heading">Credential Access</h2>



<p>During the initial access credentials were obtained via a brute-force attack against the exposed MSSQL server. No other credential access was observed during this intrusion, although the threat actors did enable WDigest to make later credential access easier.</p>



<h2 class="wp-block-heading">Command and Control</h2>



<p>We observed that the domain mymst007[.]info is used to download further payloads. The domain was created five years ago. We have seen similar reports that make mention of the same infrastructure. Attacks associated with this domain include the same or similar tactics techniques and procedures (TTPs).</p>
