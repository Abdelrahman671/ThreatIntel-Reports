												<div class="c-wysiwyg">
													<p>Satacom downloader, also known as LegionLoader, is a renowned malware family that emerged in 2019. It is known to use the technique of querying DNS servers to obtain the base64-encoded URL in order to receive the next stage of another malware family currently distributed by Satacom. The Satacom malware is delivered via third-party websites. Some of these sites do not deliver Satacom themselves, but use legitimate advertising plugins that the attackers abuse to inject malicious ads into the webpages. The malicious links or ads on the sites redirect users to malicious sites such as fake file-sharing services.</p>
<p>In this report we cover a recent malware distribution campaign related to the Satacom downloader. The main purpose of the malware that is dropped by the Satacom downloader is to steal BTC from the victim&#8217;s account by performing web injections into targeted cryptocurrency websites. The malware attempts to do this by installing an extension for Chromium-based web browsers, which later communicates with its C2 server, whose address is stored in the BTC transaction data.</p>
<p>The malicious extension has various JS scripts to perform browser manipulations while the user is browsing the targeted websites, including enumeration and manipulation with cryptocurrency websites. It also has the ability to manipulate the appearance of some email services, such as Gmail, Hotmail and Yahoo, in order to hide its activity with the victim&#8217;s cryptocurrencies shown in the email notifications.</p>
<h2 id="satacom-technical-analysis">Satacom technical analysis</h2>
<p>The initial infection begins with a ZIP archive file. It is downloaded from a website that appears to mimic a software portal that allows the user to download their desired (often cracked) software for free. The archive contains several legitimate DLLs and a malicious Setup.exe file that the user needs to execute manually to initiate the infection chain.</p>
<p>Various types of websites are used to spread the malware. Some of them are malicious websites with a hardcoded download link, while others have the &#8220;Download&#8221; button injected through a legitimate ad plugin. In this case, even legitimate websites may have a malicious &#8220;Download&#8221; link displayed on the webpage. At the time of writing, we saw the QUADS plugin being abused to deliver Satacom.</p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/06/01101851/Satacom_downloader_01.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-large wp-image-109811" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/06/01101851/Satacom_downloader_01.png" alt="Websites with embedded QUADS ad plugin " width="1024" height="601" /></a></p>
<p style="text-align: center"><em><strong>Websites with embedded QUADS ad plugin</strong> </em></p>
<p>The plugin is abused in the same way that other advertising networks are abused for malvertising purposes: the attackers promote ads that look like a &#8220;Download&#8221; button and redirect users to the attackers&#8217; websites.</p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/06/01101903/Satacom_downloader_02.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-large wp-image-109812" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/06/01101903/Satacom_downloader_02.png" alt="WP QUADS ad plugin within the website's content" width="1024" height="588" /></a></p>
<p style="text-align: center"><em><strong>WP QUADS ad plugin within the website&#8217;s content</strong></em></p>
<p>After the user clicks on the download button or link, there&#8217;s a chain of redirects that automatically takes them through various servers to reach a website masquerading as a file-sharing service to distribute the malware. In the screenshot below, we can see examples of websites that are the final destinations of the redirection chains.</p>

<p style="text-align: center"><em><strong>Fake &#8216;file-sharing&#8217; services</strong></em></p>
<p>After the user downloads and extracts the ZIP archive, which is about 7MB in size, a few binaries, EXE and DLL files are revealed. The DLLs are legitimate libraries, but the &#8216;Setup.exe&#8217; file is a malicious binary. It is about 450MB, but is inflated with null bytes to make it harder to analyze. The original size of the file without the added null bytes is about 5MB and it is an Inno Setup type file.</p>

<p style="text-align: center"><em><strong>Null bytes added to the PE file</strong></em></p>
<p>Inno Setup installers usually work as follows: at runtime the binary extracts a child installer to a temporary folder with the name &#8216;Setup.tmp&#8217;. Then it runs the child installer &#8216;Setup.tmp&#8217; file that needs to communicate with the primary installer with arguments pointing to the location of the original &#8216;Setup.exe&#8217; and its packages in order to retrieve the BIN data inside the &#8216;Setup.exe&#8217; file for the next step of the installation.</p>
<p>In the case of the Satacom installer, the Setup.tmp file, once running, creates a new PE DLL file in the Temp directory. After the DLL is created, the child installer loads it into itself and runs a function from the DLL.</p>
<p>It then decrypts the payload of Satacom and creates a new sub-process of &#8216;explorer.exe&#8217; in order to inject the malware into the &#8216;explorer.exe&#8217; process.</p>
<p>Based on the behavior we observed, we can conclude that the malware performs a common process injection technique on the remote &#8216;explorer.exe&#8217; process called process hollowing. This is a known technique used to evade detection by AV applications.</p>
<p>The malicious payload that&#8217;s injected into the &#8216;explorer.exe&#8217; process uses the RC4 encryption implementation to decrypt its configuration data, communication strings and data for the other dropped binaries on the victim&#8217;s machine. The encrypted data is stored inside the malicious payload.</p>
<p>The malware uses different hardcoded keys to decrypt the data at each step. There are four different RC4 keys that the malware uses to perform its actions, first decrypting the HEX string data to use it for its initial communication purposes.</p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29153505/Satacom_downloader_05.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-large wp-image-109815" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29153505/Satacom_downloader_05-1024x465.png" alt="RC4 keys (left pane) and encrypted HEX strings (right pane)" width="1024" height="465" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29153505/Satacom_downloader_05-1024x465.png 1024w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29153505/Satacom_downloader_05-300x136.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29153505/Satacom_downloader_05-768x348.png 768w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29153505/Satacom_downloader_05-386x175.png 386w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29153505/Satacom_downloader_05-370x168.png 370w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29153505/Satacom_downloader_05-617x280.png 617w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29153505/Satacom_downloader_05-800x363.png 800w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29153505/Satacom_downloader_05.png 1534w" sizes="(max-width: 1024px) 100vw, 1024px" /></a></p>

<h2 id="the-payload-malicious-browser-extension">The payload: malicious browser extension</h2>
<p>The Satacom downloader downloads various binaries to the victim&#8217;s machine. In this campaign we observed a PowerShell script being downloaded that installs a malicious Chromium-based browser extension that targets Google Chrome, Brave and Opera.</p>
<p>The extension installation script is responsible for downloading the extension in a ZIP archive file from a third-party website server. The PowerShell script downloads the archived file to the computer&#8217;s Temp directory and then extracts it to a folder inside the Temp directory.</p>
<p>After that, the script searches for the possible locations of shortcuts for each of the targeted browsers in such places as Desktop, Quick Launch and Start Menu. It also configures the locations of the browsers&#8217; installation files and the location of the extension on the computer.</p>
<p>Finally, the PS script recursively searches for any link (.LNK) file in the above locations and modifies the &#8220;Target&#8221; parameter for all existing browser shortcuts with the flag &#8220;&#8211;load-extension=[pathOfExtension]&#8221; so that the shortcut will load the browser with the malicious extension installed.</p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154019/Satacom_downloader_08.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-109818" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154019/Satacom_downloader_08.png" alt="Chrome shortcut with the extension parameter" width="973" height="377" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154019/Satacom_downloader_08.png 973w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154019/Satacom_downloader_08-300x116.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154019/Satacom_downloader_08-768x298.png 768w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154019/Satacom_downloader_08-452x175.png 452w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154019/Satacom_downloader_08-370x143.png 370w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154019/Satacom_downloader_08-723x280.png 723w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154019/Satacom_downloader_08-800x310.png 800w" sizes="(max-width: 973px) 100vw, 973px" /></a></p>
<p style="text-align: center"><em><strong>Chrome shortcut with the extension parameter</strong></em></p>
<p>After performing this action, the script closes any browser processes that may be running on the machine, so that the next time the victim opens the browser, the extension will be loaded into the browser and run while the user is browsing the internet.</p>
<p>This extension installation technique allows the threat actors to add the addon to the victim&#8217;s browser without their knowledge and without uploading it to the official extension stores, such as the Chrome Store, which requires the addon to meet the store&#8217;s requirements.</p>

<p style="text-align: center"><em><strong>Extension installation PowerShell script</strong></em></p>
<h3 id="malicious-extension-analysis">Malicious extension analysis</h3>
<p>After installation of the extension, we can analyze its functionality and features by checking specific files stored in the extension&#8217;s directory. If we take a look at the first lines of the &#8216;manifest.json&#8217; file, we&#8217;ll see that the extension disguises itself by naming the addon &#8220;Google Drive,&#8221; so even when the user accesses the browser addons, the only thing they will see is an addon named &#8220;Google Drive&#8221;, which looks like just another standard Google extension installed inside the browser.</p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154356/Satacom_downloader_10.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-109820" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154356/Satacom_downloader_10.png" alt="The manifest.json file settings" width="759" height="146" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154356/Satacom_downloader_10.png 759w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154356/Satacom_downloader_10-300x58.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154356/Satacom_downloader_10-370x71.png 370w" sizes="(max-width: 759px) 100vw, 759px" /></a></p>
<p style="text-align: center"><em><strong>The manifest.json file settings</strong></em></p>
<p>Another malicious extension file that always runs in the background when the user is browsing is &#8216;background.js&#8217;, which is responsible for initializing communication with the C2. If we take a closer look at the JavaScript code, we&#8217;ll find an interesting function call at the bottom of the script with a string variable that is the address of a bitcoin wallet.</p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154647/Satacom_downloader_11.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-109821" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154647/Satacom_downloader_11.png" alt="Background.js script snippet" width="854" height="99" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154647/Satacom_downloader_11.png 854w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154647/Satacom_downloader_11-300x35.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154647/Satacom_downloader_11-768x89.png 768w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154647/Satacom_downloader_11-370x43.png 370w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154647/Satacom_downloader_11-800x93.png 800w" sizes="(max-width: 854px) 100vw, 854px" /></a></p>
<p style="text-align: center"><em><strong>Background.js script snippet</strong></em></p>
<p>Looking at the script&#8217;s code, we can conclude that the extension is about to fetch another string from the hardcoded URL, into which the script inserts the bitcoin address. The JavaScript receives data in JSON format, which shows the wallet&#8217;s transaction activity, and then looks for a specific string within the latest transaction details.</p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154735/Satacom_downloader_12.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-109822" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154735/Satacom_downloader_12.png" alt="JSON of the transaction details" width="742" height="477" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154735/Satacom_downloader_12.png 742w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154735/Satacom_downloader_12-300x193.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154735/Satacom_downloader_12-272x175.png 272w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154735/Satacom_downloader_12-370x238.png 370w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154735/Satacom_downloader_12-436x280.png 436w" sizes="(max-width: 742px) 100vw, 742px" /></a></p>
<p style="text-align: center"><em><strong>JSON of the transaction details</strong></em></p>
<p>There are two strings on the page that contain the C2 address. The &#8220;script&#8221; string is a HEX string that contains the C2 host of the malware, and the &#8220;addr&#8221; string is the Base58-encoded C2 address. The reason for using the last cryptocurrency transaction of a specific wallet to retrieve the C2 address is that the server address can be changed by the threat actors at any time. Moreover, this trick makes it harder to disable the malware&#8217;s communication with its C2 server, since disabling wallets is much more difficult than blocking or banning IPs or domains. If the C2 server is blocked or taken down, the threat actors can simply change the &#8216;script&#8217; or &#8216;addr&#8217; string to a different C2 server by performing a new transaction. And since the extension always checks these strings to retrieve the C2, it will always ask for the new one if it&#8217;s ever changed.</p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154905/Satacom_downloader_13.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-109823" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154905/Satacom_downloader_13.png" alt="Decoded C2 address from the transaction details" width="588" height="229" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154905/Satacom_downloader_13.png 588w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154905/Satacom_downloader_13-300x117.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154905/Satacom_downloader_13-449x175.png 449w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154905/Satacom_downloader_13-370x144.png 370w" sizes="(max-width: 588px) 100vw, 588px" /></a></p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154946/Satacom_downloader_14.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-109824" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154946/Satacom_downloader_14.png" alt="Decoded C2 address from the transaction details" width="675" height="219" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154946/Satacom_downloader_14.png 675w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154946/Satacom_downloader_14-300x97.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154946/Satacom_downloader_14-539x175.png 539w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29154946/Satacom_downloader_14-370x120.png 370w" sizes="(max-width: 675px) 100vw, 675px" /></a></p>
<p style="text-align: center"><em><strong>Decoded C2 address from the transaction details</strong></em></p>
<p>The extension has several other scripts that are responsible for initializing the received commands and become functional after the C2 address is retrieved, because the scripts need to obtain some important information from the C2. For example, the C2 holds the BTC address that will be used when the BTC is transferred from the victim&#8217;s wallet to the threat actor&#8217;s wallet.</p>

<p style="text-align: center"><em><strong>Threat actor&#8217;s BTC wallet address</strong></em></p>
<p>To get hold of the victim&#8217;s cryptocurrency, the threat actors use web injects on the targeted websites. The web inject script is also provided by the C2 after the extension contacts it. In the following screenshot, we can see the &#8216;injections.js&#8217; script from the extension, which fetches the web inject script from the C2 server.</p>

<p style="text-align: center"><em><strong>The injections.js script</strong></em></p>
<p>After the addon contacts the C2 server – extracted as mentioned above – the server responds with the web inject script that will be used on the targeted websites.</p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155245/Satacom_downloader_17.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-large wp-image-109827" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155245/Satacom_downloader_17-1024x504.png" alt="Webinject script from C2 server" width="1024" height="504" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155245/Satacom_downloader_17-1024x504.png 1024w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155245/Satacom_downloader_17-300x148.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155245/Satacom_downloader_17-768x378.png 768w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155245/Satacom_downloader_17-355x175.png 355w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155245/Satacom_downloader_17-370x182.png 370w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155245/Satacom_downloader_17-568x280.png 568w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155245/Satacom_downloader_17-800x394.png 800w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155245/Satacom_downloader_17.png 1346w" sizes="(max-width: 1024px) 100vw, 1024px" /></a></p>

<p>If we take a closer look at the script, we can see that the threat actors are targeting various websites. In the version of the script shown above we can see that it targets Coinbase, Bybit, KuCoin, Huobi and Binance users.</p>
<p>Since the script within the C2 can be changed at any time, the threat actors can add or remove other web injection targets, as well as start targeting cryptocurrencies other than BTC, which makes this extension pretty dynamic and allows threat actors to control the malicious extension by changing the scripts.</p>
<p>If we look at the script, we can see that the extension performs various actions on the targeted websites. For example, it has the ability to retrieve the victims&#8217; addresses, obtain account information, bypass 2FA, and much more. Moreover, it&#8217;s capable of transferring BTC currency from the victim&#8217;s wallet to the attackers&#8217; wallet.</p>

<p style="text-align: center"><em><strong>Functions from the web inject script</strong></em></p>
<p>Looking at the full web inject script, we can conclude that the idea behind it is to steal BTC currencies from victims who have the malicious extension installed. The extension performs various actions on the account in order to remotely control it using the web inject scripts, and eventually the extension tries to withdraw the BTC currency to the threat actors&#8217; wallet. To circumvent the 2FA settings for transactions, the web inject script uses 2FA bypass techniques.</p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155407/Satacom_downloader_19.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-large wp-image-109829" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155407/Satacom_downloader_19-1024x690.png" alt="Snippet of the BTC withdrawal function from the web inject script" width="1024" height="690" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155407/Satacom_downloader_19-1024x690.png 1024w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155407/Satacom_downloader_19-300x202.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155407/Satacom_downloader_19-768x518.png 768w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155407/Satacom_downloader_19-260x175.png 260w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155407/Satacom_downloader_19-370x249.png 370w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155407/Satacom_downloader_19-416x280.png 416w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155407/Satacom_downloader_19-800x539.png 800w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155407/Satacom_downloader_19.png 1202w" sizes="(max-width: 1024px) 100vw, 1024px" /></a></p>
<p style="text-align: center"><em><strong>Snippet of the BTC withdrawal function from the web inject script</strong></em></p>
<p>Before stealing the cryptocurrency, the extension communicates with the C2 server to get the minimum BTC value. It then compares this value with the actual amount of money in the target wallet. If the wallet contains less cryptocurrency than the minimum amount received from the C2, it doesn&#8217;t withdraw any cryptocurrency from it.</p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155508/Satacom_downloader_20.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-109830" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155508/Satacom_downloader_20.png" alt="Minimum amount threshold from C2" width="320" height="64" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155508/Satacom_downloader_20.png 320w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155508/Satacom_downloader_20-300x60.png 300w" sizes="(max-width: 320px) 100vw, 320px" /></a></p>
<p style="text-align: center"><em><strong>Minimum amount threshold from C2</strong></em></p>
<p>The script also performs several other checks before stealing the BTC currency. For example, it also checks the BTC to USD exchange rate.</p>
<p>When the amount of BTC in the target wallet meets the C2 checks, the script performs the withdrawal function to steal the BTC currency from the victim.</p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155546/Satacom_downloader_21.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-109831" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155546/Satacom_downloader_21.png" alt="Performing balance check" width="690" height="793" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155546/Satacom_downloader_21.png 690w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155546/Satacom_downloader_21-261x300.png 261w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155546/Satacom_downloader_21-152x175.png 152w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155546/Satacom_downloader_21-370x425.png 370w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/05/29155546/Satacom_downloader_21-244x280.png 244w" sizes="(max-width: 690px) 100vw, 690px" /></a></p>
<p style="text-align: center"><em><strong>Performing balance check</strong></em></p>
<p>In addition to stealing BTC, the malicious extension performs additional actions to hide its activity.</p>
<p>For example, the malicious extension contains scripts that target three different email services: <em>Gmail</em>, <em>Hotmail </em>and <em>Yahoo</em>. The idea behind the scripts is to hide the email confirmation of the transaction performed by the malicious extension.</p>
<p>Each script makes visual changes to the emails once the victim reaches the email service&#8217;s page. It searches for pre-defined email titles and content, and when it finds them, it simply hides them from the victim by injecting HTML code into the message body. As a result, the victim is unaware that a specific transaction transferring crypto currency to the threat actors&#8217; wallet was made.</p>

<p>In addition, the extension can manipulate email threads from the targeted websites, so if the victim opens a thread from, for example, Binance, it can change the content of the emails and display a fake email thread that looks exactly like the real one. It also contains a placeholder for desired strings that the extension can inject into the content of the message page.</p>

<p style="text-align: center"><em><strong>Fake email thread template</strong></em></p>
<p>The malicious extension has many other JavaScripts and it&#8217;s capable of performing additional actions. For example, it can extract information through the browser, such as the system information, cookies, browser history, screenshots of opened tabs, and even receive commands from the C2 server.</p>

<p style="text-align: center"><em><strong>JavaScripts: requesting commands from the C2 (left pane) and taking screenshots (right pane)</strong></em></p>
<p>The purpose of the extension is to steal BTC and manipulate targeted cryptocurrency websites and email services to make the malware as stealthy as possible, so the victim doesn&#8217;t notice any information about the fraudulent transactions. The extension can update its functionality due to the technique used to retrieve the C2 server via the last transaction of a specific BTC wallet, which can be modified at any time by making another transaction to this wallet. This allows the threat actors to change the domain URL to a different one in case it&#8217;s banned or blocked by antivirus vendors.</p>
<h2 id="victims">Victims</h2>
<p>This campaign targets individual users around the world. According to our telemetry, in Q1 2023 users in the following countries were most frequently infected: Brazil, Algeria, Turkey, Vietnam, Indonesia, India, Egypt, Mexico.</p>
<h2 id="conclusions">Conclusions</h2>
<p>Satacom is a downloader that is still running campaigns and being developed by the threat actor behind it. This threat actor continues to distribute malware families using various techniques, such as ad injection via ad plugins for WordPress websites.</p>
<p>The recently distributed malware, which is a side-loaded extension for Chromium-based browsers, performs actions in the browser to manipulate the content of the targeted cryptocurrency website. The main purpose of this malicious extension is to steal cryptocurrency from victims and transfer it to the threat actors&#8217; wallet.</p>
<p>Moreover, since it is a browser extension, it can be installed in Chromium-based browsers on various platforms. Although the installation of the malicious extension and the infection chain described in this article are Windows-specific, if the threat actors want to target Linux and macOS users, they could easily do so, provided the victims use Chromium-based browsers.</p>
