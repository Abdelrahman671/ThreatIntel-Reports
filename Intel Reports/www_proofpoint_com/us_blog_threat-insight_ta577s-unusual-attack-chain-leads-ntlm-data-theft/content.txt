    <div class="node-full__body blog-content__body">  <h3 paraeid="{c8fd80b3-6f39-43f7-9597-7c6b68c74c49}{107}" paraid="1675912825">What happened&nbsp;</h3>

<p paraeid="{c8fd80b3-6f39-43f7-9597-7c6b68c74c49}{151}" paraid="237133442">Proofpoint identified notable cybercriminal threat actor TA577 using a new attack chain to demonstrate an uncommonly observed objective: stealing NT LAN Manager (NTLM) authentication information. This activity can be used for sensitive information gathering purposes and to enable follow-on activity.&nbsp;</p>

<p paraeid="{c8fd80b3-6f39-43f7-9597-7c6b68c74c49}{179}" paraid="353591951">Proofpoint identified at least two campaigns leveraging the same technique to steal NTLM hashes on 26 and 27 February 2024. Campaigns included tens of thousands of messages targeting hundreds of organizations globally. Messages appeared as replies to previous emails, known as thread hijacking, and contained zipped HTML attachments.&nbsp;</p>

<p paraeid="{c8fd80b3-6f39-43f7-9597-7c6b68c74c49}{213}" paraid="1104633967"><img alt="Figure 1" data-entity-type="file" data-entity-uuid="ffb71d60-4a02-47b0-81eb-e05f3b6b2bd0" height="538" src="/sites/default/files/inline-images/p1_1.png" width="1008" loading="lazy"></p>

<p paraeid="{c8fd80b3-6f39-43f7-9597-7c6b68c74c49}{222}" paraid="689951155"><em>Example message using thread hijacking containing a zipped attachment containing an HTML file.&nbsp;</em></p>

<p paraeid="{c8fd80b3-6f39-43f7-9597-7c6b68c74c49}{240}" paraid="599310872">Each .zip attachment has a unique file hash, and the HTMLs within the compressed files are customized to be specific for each recipient. When opened, the HTML file triggered a system connection attempt to a Server Message Block (SMB) server via a meta refresh to a file scheme URI ending in .txt. That is, the file would automatically contact an external SMB resource owned by the threat actor. Proofpoint has not observed malware delivery from these URLs, instead researchers assess with high confidence TA577’s objective is to capture NTLMv2 Challenge/Response pairs from the SMB server to steal NTLM hashes based on characteristics of the attack chain and tools used.&nbsp;&nbsp;</p>

<p paraeid="{d1b0b08c-46c4-46fd-b9b7-026ee80b322e}{55}" paraid="1016969785"><img alt="Figure 2" data-entity-type="file" data-entity-uuid="ed31a6fc-3b1b-4771-8849-38290c721144" height="282" src="/sites/default/files/inline-images/p2_1.png" width="706" loading="lazy"></p>

<p paraeid="{d1b0b08c-46c4-46fd-b9b7-026ee80b322e}{64}" paraid="1114506234"><em>Example HTML containing the URL (beginning with “file://”) pointing to the SMB resource.&nbsp;</em></p>

<p paraeid="{d1b0b08c-46c4-46fd-b9b7-026ee80b322e}{78}" paraid="33221315">These hashes could be exploited for password cracking or facilitate "<a data-entity-substitution="canonical" data-entity-type="node" data-entity-uuid="e74ce237-bb38-43f7-a124-8ff829e0172b" href="/us/threat-reference/lateral-movement" rel="noreferrer noopener" target="_blank" title="Lateral Movement">Pass-The-Hash</a>" attacks using other vulnerabilities within the targeted organization to move laterally within an impacted environment. Indications supporting this theory include artifacts on the SMB servers pointing towards the use of open-source toolkit <a href="https://github.com/fortra/impacket" rel="noreferrer noopener" target="_blank">Impacket</a> for the attack. The use of Impacket on the SMB server can be identified by the default NTLM server challenge "aaaaaaaaaaaaaaaa" and <a href="https://github.com/fortra/impacket/blob/985690e9d3d1012e18f1727701b44bbe953cb1c9/examples/smbserver.py#L94-L98" rel="noreferrer noopener" target="_blank">the default GUID</a> observed in the traffic. Such practices are uncommon in standard SMB servers.&nbsp;</p>

<p><img alt="Figure 3" data-entity-type="file" data-entity-uuid="887e19c8-aed4-44e4-9723-73606bdd4d1c" height="688" src="/sites/default/files/inline-images/Screenshot%202024-02-29%20at%209.38.13%20AM.png" width="1006" loading="lazy"></p>

<p paraeid="{d1b0b08c-46c4-46fd-b9b7-026ee80b322e}{154}" paraid="1887243279"><em>Observed packet capture (PCAP) from the TA577 campaign.&nbsp;&nbsp;</em></p>

<p paraeid="{d1b0b08c-46c4-46fd-b9b7-026ee80b322e}{164}" paraid="1291026234">Any allowed connection attempt to these SMB servers could potentially compromise NTLM hashes, along with revealing other sensitive information such as computer names, domain names, and usernames in clear text.&nbsp;</p>

<p paraeid="{d1b0b08c-46c4-46fd-b9b7-026ee80b322e}{174}" paraid="879013948">It is notable that TA577 delivered the malicious HTML in a zip archive to generate a local file on the host. If the file scheme URI was sent directly in the email body, the attack would not work on Outlook mail clients patched since July 2023. Disabling guest access to SMB does not mitigate the attack, since the file must attempt to authenticate to the external SMB server to determine if it should use guest access.&nbsp;</p>

<h3 paraeid="{d1b0b08c-46c4-46fd-b9b7-026ee80b322e}{220}" paraid="1723600638">Attribution&nbsp;</h3>

<p paraeid="{d1b0b08c-46c4-46fd-b9b7-026ee80b322e}{226}" paraid="1039982988">TA577 is a prominent cybercrime threat actor and one of the major Qbot affiliates before the botnet's disruption. It is considered an initial access broker (IAB) and Proofpoint has associated TA577 campaigns with follow-on ransomware infections including Black Basta.&nbsp; Recently, the actor favors Pikabot as an initial payload.&nbsp;&nbsp;</p>

<h3 paraeid="{2f52e0b3-a606-4633-9f64-cc8f9b10165a}{5}" paraid="253021488">Why it matters&nbsp;</h3>

<p paraeid="{2f52e0b3-a606-4633-9f64-cc8f9b10165a}{11}" paraid="1322057315">Proofpoint typically observes TA577 conducting attacks to deliver malware and has never observed this threat actor demonstrating the attack chain used to steal NTLM credentials first observed on 26 February. Recently, TA577 has been observed delivering Pikabot using a variety of attack chains.&nbsp;&nbsp;</p>

<p paraeid="{2f52e0b3-a606-4633-9f64-cc8f9b10165a}{37}" paraid="1338631984">The rate at which TA577 adopts and distributes new tactics, techniques, and procedures (TTPs) suggests the threat actor likely has the time, resources, and experience to rapidly iterate and test new delivery methods. TA577, in addition to other IABs appears to have the pulse of the threat landscape and know when and why specific attack chains stop being effective and will quickly create new methods to bypass detections and attempt to increase the effectiveness and likelihood of victim engagement with their payload delivery.&nbsp;</p>

<p paraeid="{2f52e0b3-a606-4633-9f64-cc8f9b10165a}{51}" paraid="169358886">Proofpoint researchers have also seen an increase in multiple threat actors abusing file scheme URIs to direct recipients to external file shares such as SMB and WebDAV to access remote content for malware delivery. Organizations should block outbound SMB to prevent exploitation identified in this campaign.&nbsp;&nbsp;</p>
