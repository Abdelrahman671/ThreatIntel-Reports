                            <h1 class="title-post" itemprop="name">Case Study: Tackling Extension Infections with a Budget-Friendly Scratch Project</h1>
                    <div class="date">
            <span itemprop="datePublished"><a href="https://threat.boutique/archive/2025-01" title="Show all posts made on this month">01 January 2025</a></span>
            - Posted in            <span itemprop="articleSection"><a href="https://threat.boutique/category/threat-analyze">Threat Analyze</a></span> by            <span itemprop="author"><a href="https://threat.boutique/author/b3vSclAuunBMsdt">Jok3r</a></span> -
            <span><a href="https://threat.boutique/2025/01/case-study-tackling-extension-infections-with-a-budget-friendly-scratch-project" rel="permalink">Permalink</a></span>
        </div>
                    <div class="featured-image">
                <a href="https://threat.boutique/2025/01/case-study-tackling-extension-infections-with-a-budget-friendly-scratch-project"><img src="https://threat.boutique/content/images/20250101194823-new.webp" alt="Case Study: Tackling Extension Infections with a Budget-Friendly Scratch Project"/></a>
            </div>
                                        <div class="post-body post-1735746480" itemprop="articleBody">
            <p>Backgound:
Following a <a href="https://www.cyberhaven.com/blog/cyberhavens-chrome-extension-security-incident-and-what-were-doing-about-it">notification from a company</a> that fell victim to a supply chain attack, where attackers compromised their Chrome extension, I realized the importance of addressing this issue. While I have discussed the potential threats stemming from such attack vectors in previous articles (<a href="https://threat.boutique/2024/08/mastering-browser-extensions-key-strategies-for-effective-analysis-and-threat-prevention">Article 1</a> and <a href="https://threat.boutique/2024/09/unseen-risks-navigating-hidden-threats-in-npm-nuget-pip-chrome-extensions-and-docker-images-for-everyday-tasks">Article 2</a>), this recent incident prompted me to share a project with the cybersecurity community. The project is designed to help organizations/firms with limited budgets and resources effectively combat this type of attack .</p>

<p>Recent supply chain story:
The attacker injected a typo-squatted domain into the extension for future malicious actions. However, the victim organization discovered the compromise before any significant damage occurred. Upon further investigation, other researchers also identified additional signs of compromise in other extensions.</p>

<p><strong><em>Project Important Points:</em></strong>
 - Threat modeling for extensions
 - Implement in technical scope</p>

<p><strong><em>Threat modeling for extensions:</em></strong>
 - Attacker Injecting typo domain
 - Attacker Injecting non typo domains
 - Attacker injecting js component which at least allow to work with encoded string 
 - Attacker compromising emails which was in source code
 - Attacker injecting over permission access capability</p>

<p><strong><em>Implement in technical scope:</em></strong>
First, place the extension file in CRX format into the 'extensions' folder. In the next step, run the script to check for the existence of the files. Then, the script will unarchive the CRX file and extract it to another location. After extraction, the script will check the 'manifest.json' for the presence of permission scopes, extract all domains and IP addresses, identify functions responsible for encoding/decoding , and extract any email addresses found within. 
<a href="https://github.com/lisajan-hash/chromeExtensionHunt3r">Project Scratch Github URL</a></p>

<p><strong><em>Conclusion:</em></strong>
As cybersecurity defenders, it is our responsibility not only to take defensive actions but also to manage the budget effectively to achieve desired results. This project can help in understanding core solutions while also saving resources for similar tasks.</p>
