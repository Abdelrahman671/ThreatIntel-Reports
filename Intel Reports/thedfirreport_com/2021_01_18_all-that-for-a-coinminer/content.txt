			<h1 class="entry-title">All That for a Coinminer?</h1>
		<div class="entry-meta">
			<span class="posted-on"><a href="https://thedfirreport.com/2021/01/18/all-that-for-a-coinminer/" rel="bookmark"><time class="entry-date published" datetime="2021-01-18T23:53:31+00:00">January 18, 2021</time></a></span>		</div><!-- .entry-meta -->

		
	</header><!-- .entry-header -->

	<div class="entry-content">
		<p>A threat actor recently brute forced a local administrator password using RDP and then dumped credentials using Mimikatz. They not only dumped <a href="https://adsecurity.org/?page_id=1821" target="_blank" rel="noopener">LogonPasswords</a> but they also exported all Kerberos tickets. The threat actor used Advanced IP Scanner to scan the environment before RDPing into multiple systems, including a Domain Controller. After an hour of moving around the environment, they deployed XMRig on the initial compromised system before logging off. The threat actor was active on the network for about 2 hours in total.</p>
<h3>MITRE ATT&amp;CK</h3>
<h3><a>Initial Access</a></h3>
<p>The threat actor logged in using RDP from an IP (92.118.13[.]103) that hadn&#8217;t attempted any previous logins. The account was created the previous day using a source IP of 54.38.67[.]132, which had been trying to brute force a local admin password. The threat actor used a workstation named winstation. During the intrusion, the threat actors also used 5.122.15[.]138 to login to one of the systems.</p>
<h3>Execution</h3>
<p>The threat actor copied svshost.exe to C:\naz\naz and then executed it. This PE creates &#8220;XMRig CPU mine.exe&#8221; and HideAll.bat in C:\Windows\PolicyDefinitions and then executes both of them.</p>
<p><img fetchpriority="high" decoding="async" class="alignnone size-full wp-image-2060" src="https://thedfirreport.com/wp-content/uploads/2021/01/44.png" alt="" width="744" height="404" srcset="https://thedfirreport.com/wp-content/uploads/2021/01/44.png 744w, https://thedfirreport.com/wp-content/uploads/2021/01/44-300x163.png 300w" sizes="(max-width: 744px) 100vw, 744px" /></p>
<h3><a>Defense Evasion</a></h3>
<p>The PE file that installs XMRig (svshost.exe) also has a script (HideAll.bat) imbedded in it, which is called at runtime. This is the contents of that batch file.</p>
<pre>attrib +h svshost.exe<br />attrib +h XMRig CPU mine.exe<br />attrib +h config.json<br />attrib +h HideAll.bat<br />attrib +h xmrig-notls.exe</pre>
<p>This script is copied to C:\Windows\PolicyDefitions\ and run, which causes the files specified to be hidden.</p>
<h2>Persistence</h2>
<p>Before the threat actor disconnected, they changed the user password.</p>
<pre>net user %USERNAME% ehs.123</pre>
<h2>Credential Access</h2>
<p><a href="https://github.com/gentilkiwi/mimikatz" target="_blank" rel="noopener">Mimikatz</a> was used to dump credentials from memory, as well as, export Kerberos tickets using the following command:</p>
<pre>mimikatz.exe", """log"" ""privilege::debug"" ""sekurlsa::logonpasswords"" ""sekurlsa::tickets /export"" ""exit"""</pre>
<p><img decoding="async" class="alignnone size-full wp-image-2086" src="https://thedfirreport.com/wp-content/uploads/2021/01/33.png" alt="" width="1274" height="420" srcset="https://thedfirreport.com/wp-content/uploads/2021/01/33.png 1274w, https://thedfirreport.com/wp-content/uploads/2021/01/33-300x99.png 300w, https://thedfirreport.com/wp-content/uploads/2021/01/33-1024x338.png 1024w, https://thedfirreport.com/wp-content/uploads/2021/01/33-768x253.png 768w" sizes="(max-width: 1274px) 100vw, 1274px" /></p>
<p>The threat actors used a vbs script named launch to execute mimikatz. This is the content of launch.vbs</p>
<pre>set shell=CreateObject("Shell.Application")<br />shell.ShellExecute "mimikatz.exe", """log"" ""privilege::debug"" ""sekurlsa::logonpasswords"" ""sekurlsa::tickets /export"" ""exit""", "", "runas", 0<br />set shell=nothing</pre>
<p>Since the log parameter was used, the output was saved to mimikatz.log</p>
<p><img decoding="async" class="alignnone size-full wp-image-2057" src="https://thedfirreport.com/wp-content/uploads/2021/01/11.png" alt="" width="390" height="155" srcset="https://thedfirreport.com/wp-content/uploads/2021/01/11.png 390w, https://thedfirreport.com/wp-content/uploads/2021/01/11-300x119.png 300w" sizes="(max-width: 390px) 100vw, 390px" /></p>
<p>The Kerberos tickets were saved to disk, due to the threat actor using sekurlsa::tickets /export.</p>
<p><img loading="lazy" decoding="async" class="alignnone size-full wp-image-2058" src="https://thedfirreport.com/wp-content/uploads/2021/01/k.png" alt="" width="890" height="131" srcset="https://thedfirreport.com/wp-content/uploads/2021/01/k.png 890w, https://thedfirreport.com/wp-content/uploads/2021/01/k-300x44.png 300w, https://thedfirreport.com/wp-content/uploads/2021/01/k-768x113.png 768w" sizes="auto, (max-width: 890px) 100vw, 890px" /></p>
<h3><a>Discovery</a></h3>
<p>Advanced IP Scanner was used to scan the environment.</p>
<p><img loading="lazy" decoding="async" class="alignnone size-full wp-image-2055" src="https://thedfirreport.com/wp-content/uploads/2021/01/12.png" alt="" width="656" height="372" srcset="https://thedfirreport.com/wp-content/uploads/2021/01/12.png 656w, https://thedfirreport.com/wp-content/uploads/2021/01/12-300x170.png 300w" sizes="auto, (max-width: 656px) 100vw, 656px" /></p>
<p>Task manager was opened multiple times. Possibly looking at logged in users and/or processes.</p>

<h3><a>Impact</a></h3>
<p>XMRig was running on the system, using some CPU but not enough to cause any issues. We tend to block mining endpoints, which may have lessened the impact of this intrusion. XMRig made connection attempts to 104.140.201[.]42 &amp; 104.142.244[.]186.</p>
<p>The threat actors have been using the associated Monero wallet for 738+ days and have netted around $5,159.</p>
