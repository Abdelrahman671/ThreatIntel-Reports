						<p>Mirai is a DDoS botnet that has gained a lot of media attraction lately due to high impact attacks such as on journalist <a href="http://krebsonsecurity.com/">Brian Krebs</a> and also for one of the biggest DDoS attacks on Internet against ISP <a href="http://dyn.com/blog/dyn-analysis-summary-of-friday-october-21-attack/">Dyn</a>, cutting off a major chunk of Internet, that took place last weekend (Friday 21 October 2016).</p>
<p>Besides the media coverage, Mirai is very interesting because we have both binary samples captured in the wild, but also because the source code was released recently &#8211; for sure we can expect many variants of Mirai code soon. Having both binary and source code allows us to study it in more detail.</p>
<p>It is quite amazing that we are in 2016 and still talking about worms, default/weak passwords and DDoS attacks: hello <a href="https://en.wikipedia.org/wiki/Morris_worm">Morris Worm</a> (1988) and <a href="https://en.wikipedia.org/wiki/MafiaBoy">Project Rivolta</a> (2000) to mention a few. </p>
<p><b>Source Code Analysis</b></p>
<p>We have compiled Mirai source code using our Tintorera, a VULNEX static analysis tool that generates intelligence while building C/C++ source code. This gives us the big picture fast.</p>
<p>From Tintorera we get an application detail summary counting compiled files, lines of code, comments, blanks and additional metrics; Tintorera also calculates the time needed to review the code. Mirai is a small project and not too complicated to review.  (Figure 1)</p>
<p><a href="https://simonroses.com/wp-content/uploads/2016/10/srf_mirai_sc1.png"><img decoding="async" src="https://simonroses.com/wp-content/uploads/2016/10/srf_mirai_sc1-300x125.png" alt="srf_mirai_sc1" width="300" height="125" /></a><br />
Figure 1</p>
<p>Mirai is using several functions from the Linux API, mostly related to network operations. (Figure 2)</p>
<p><a href="https://simonroses.com/wp-content/uploads/2016/10/srf_mirai_sc2.png"><img fetchpriority="high" decoding="async" src="https://simonroses.com/wp-content/uploads/2016/10/srf_mirai_sc2-300x258.png" alt="srf_mirai_sc2" width="300" height="258" /></a><br />
Figure 2</p>
<p>In the Tintorera intelligence report we have a list of files, functions names, basic blocks, cyclomatic complexity, API calls and inline assembly used by Mirai. By examining this list we can get an idea of the code. (Figure 3)</p>
<p><a href="https://simonroses.com/wp-content/uploads/2016/10/srf_mirai_sc3.png"><img decoding="async" src="https://simonroses.com/wp-content/uploads/2016/10/srf_mirai_sc3-300x169.png" alt="srf_mirai_sc3" width="300" height="169" /></a><br />
Figure 3</p>
<p>In file killer.c there is a function named killer_init that kills several services: telnet (port 23), ssh (port 22) and http (port 80) to prevent access to the compromised system by others. (Figure 4)</p>
<p><a href="https://simonroses.com/wp-content/uploads/2016/10/srf_mirai_sc4.png"><img loading="lazy" decoding="async" src="https://simonroses.com/wp-content/uploads/2016/10/srf_mirai_sc4-300x244.png" alt="srf_mirai_sc4" width="300" height="244" /></a><br />
Figure 4</p>
<p>In same file, killer.c, another function named memory_scan_match search memory for other Linux malwares. (Figure 5)</p>
<p><a href="https://simonroses.com/wp-content/uploads/2016/10/srf_mirai_sc5.png"><img loading="lazy" decoding="async" src="https://simonroses.com/wp-content/uploads/2016/10/srf_mirai_sc5-300x228.png" alt="srf_mirai_sc5" width="300" height="228" /></a><br />
Figure 5</p>
<p>In file scanner.c function named get_random_ip generates random IPs to attack while avoiding a white list addresses from General Electric, Hewlett-Packard, US Postal Service and US Department of Defense. (Figure 6)</p>
<p><a href="https://simonroses.com/wp-content/uploads/2016/10/srf_mirai_sc6.png"><img loading="lazy" decoding="async" src="https://simonroses.com/wp-content/uploads/2016/10/srf_mirai_sc6-300x197.png" alt="srf_mirai_sc6" width="300" height="197" /></a><br />
Figure 6</p>
<p>Mirai comes with a list of 62 default/weak passwords to perform brute force attacks on IoT devices. This list is setup in function scanner_init of file scanner.c. (Figure 7)</p>
<p><a href="https://simonroses.com/wp-content/uploads/2016/10/srf_mirai_sc7.png"><img loading="lazy" decoding="async" src="https://simonroses.com/wp-content/uploads/2016/10/srf_mirai_sc7-300x151.png" alt="srf_mirai_sc7" width="300" height="151"/></a><br />
Figure 7</p>
<p>In main.c file we can find the main function that prevents compromised devices to reboot by killing watchdog and starts the scanner to attack other IoT devices. In Figure 8 we see a callgraph of file main.c</p>
<p><a href="https://simonroses.com/wp-content/uploads/2016/10/srf_mirai_main_callgraph.png"><img loading="lazy" decoding="async" src="https://simonroses.com/wp-content/uploads/2016/10/srf_mirai_main_callgraph-300x17.png" alt="srf_mirai_main_callgraph" width="300" height="17" /></a><br />
Figure 8</p>
<p>Mirai offers offensive capabilities to launch DDoS attacks using UDP, TCP or HTTP protocols.  </p>
<p><b>Binary Analysis</b></p>
<p>Now let’s move to binary analysis. So far we have been able to study 19 different samples obtained in the wild for the following architectures: x86, ARM, MIPS, SPARC, Motorola 68020 and Renesas SH (SuperH). </p>
<p>For the binary analysis we have used <a href="http://www.vulnex.com/en/binsecsweeper.html">VULNEX BinSecSweeper platform</a> that allows analyzing binaries among other things/files in depth combining SAST and Big Data. </p>
<p>In Figure 9 we see a chart showing all the files magic to give us an idea of the file types/ architectures. All samples are 32 bits. </p>
<p><a href="https://simonroses.com/wp-content/uploads/2016/10/srf_mirai_bin2.png"><img loading="lazy" decoding="async" src="https://simonroses.com/wp-content/uploads/2016/10/srf_mirai_bin2-300x145.png" alt="srf_mirai_bin2" width="300" height="145" /></a><br />
Figure 9</p>
<p>By using BinSecSweeper we obtained a lot of information for each sample, similarities between them and different vulnerabilities. Currently not many Antivirus identify all the samples, so beware what Antivirus you use! In Figure 10 we have a visualization of file sizes in bytes.  </p>
<p><a href="https://simonroses.com/wp-content/uploads/2016/10/srf_mirai_bin1.png"><img loading="lazy" decoding="async" src="https://simonroses.com/wp-content/uploads/2016/10/srf_mirai_bin1-300x206.png" alt="srf_mirai_bin1" width="300" height="206" /></a><br />
Figure 10</p>
<p>We analyzed all section names in the samples and Figure 11 is the result.</p>
<p><a href="https://simonroses.com/wp-content/uploads/2016/10/srf_mirai_bin3.png"><img loading="lazy" decoding="async" src="https://simonroses.com/wp-content/uploads/2016/10/srf_mirai_bin3-300x206.png" alt="srf_mirai_bin3" width="300" height="206" /></a><br />
Figure 11</p>
<p>As mentioned before the samples are for different architectures so in this post we are not showing you the code analysis results.  </p>
<p>We have updated BinSecSweeper analysis engine to identify Mirai malware samples. A full binary analysis report is available from <a href="http://www.vulnex.com/en/services.html">VULNEX Cyber Intelligence Services</a> to our customers, please visit <a href="http://www.vulnex.com/">our website</a> or <a href="mailto:services@vulnex.com" target="_top">contact us</a>.  </p>
<p><b>Conclusions</b></p>
<p>Despite being a fairly simple code, Mirai has some interesting offensive and defensive capabilities and for sure it has made a name for itself. Now that the source code has been released, it is just a matter of time we start seeing variants of Mirai. </p>
<p>Mirai Botnet is a wakeup call to IoT vendors to secure their devices. Unfortunately millions of devices have been already deployed on Internet and there are insecure by default, so embrace yourself for more IoT attacks in the near future.</p>
<p><b>What do you think about IoT security?</b></p>
