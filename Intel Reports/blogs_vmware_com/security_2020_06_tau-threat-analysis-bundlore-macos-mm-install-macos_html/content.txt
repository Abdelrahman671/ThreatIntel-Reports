            <div class="entry-content">
              <p>The mm-install-&gt;macos variant of the Bundlore family of macOS adware has been around for many years in many variations and delivery methods. Recently, a variant with a novel installation method was discovered. Although most of the installation details were the same or similar to the samples analyzed in the blogs above, these new samples modified the <b>sudoers</b> file on the infected system to remove the password requirement for privilege escalation. The malware also utilizes a form of obfuscation not observed before in this family, hiding compressed data in a resource fork on a downloaded script file.</p>
<p>These samples were observed to be installed via a malicious chrome extension (crx file). This extension was pulled from an adware site http://download[.]mycouponsmartmac[.]com and was not publicly uploaded at the time of analysis.</p>
<p>After the MyCouponsmart extension is installed, javascript is injected into the browser that displays pop-up ads and redirects the user to a website requiring the user to download a fake Adobe Flash Player update. The software downloaded has a multi-stage installer that, once given authentication from the user, gathers system information and ultimately installs multiple adware programs as root. The installed program demonstrates persistence on the system and the capability to silently download and install software as root at any time.</p>
<p><a href="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/1_flash_update_page.png"><img decoding="async" class="alignnone size-full wp-image-55322" src="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/1_flash_update_page.png" alt="" width="2647" height="826" /></a></p>
<p><i>Figure </i><i>1</i><i>: Fake Flash Update Page</i></p>
<p>Interestingly, this page has a disclaimer included at the bottom informing the user that the installer may suggest installation of additional “free software offers” and that the Flash Player downloaded from the site is not affiliated with Adobe Flash.</p>
<p><a href="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/2_disclaimer.png"><img decoding="async" loading="lazy" class="alignnone size-full wp-image-55324" src="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/2_disclaimer.png" alt="" width="1113" height="155" /></a></p>
<p><i>Figure </i><i>2</i><i>: Fake Flash Update Page Disclaimer</i></p>
<p><b>Details<br />
</b>The analyzed samples were manually downloaded from a specially crafted URL from the site http://download[.]mycouponsmartmac[.]com. Each extension download URL uses a unique GUID, and changing this GUID results in the download of a different sample by hash.</p>
<p><a href="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/3_mycouponsmart_extension.png"><img decoding="async" loading="lazy" class="alignnone size-full wp-image-55326" src="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/3_mycouponsmart_extension.png" alt="" width="809" height="425" /></a></p>
<p><i>Figure </i><i>3</i><i>: Chrome Extension</i></p>
<p>After the downloaded MyCouponsmart extension is installed, it injects javascript code from <b>chrome-extension://background.js</b> into the browser which contains code to either pop up an advertisement or redirect the webpage. More details regarding this extension are covered in the Configuration Profiles section below.</p>
<p><a href="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/4_background_js.png"><img decoding="async" loading="lazy" class="alignnone size-full wp-image-55328" src="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/4_background_js.png" alt="" width="2232" height="1584" /></a></p>
<p><i>Figure </i><i>4</i><i>: Extension JavaScript</i></p>
<p>When the URL in the script above is visited, the user is redirected to another site (in this case, http://cshus[.]albertbucket[.]icu/) which at the time was offering a download for a fake Adobe Flash Update, which downloaded the file <b>AdobeFlashPlayer.zip</b> (SHA256: 98bbcced1edf5ee4d781664b8fe722262aefd1cc4e7aa22a271aa9720de56c15).</p>
<p>Immediately after the Flash zip file is downloaded, the browser is redirected to a site offering the download of another Chrome extension named “Search Manager”</p>
<p><a href="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/5_first_mycoupon_redirect.png"><img decoding="async" loading="lazy" class="alignnone size-full wp-image-55330" src="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/5_first_mycoupon_redirect.png" alt="" width="2655" height="1683" /></a></p>
<p><i>Figure </i><i>5</i><i>: Search Manager Extension</i></p>
<p>This zip file contained the disk image <b>AdobeFlashPlayer.dmg</b> (SHA256: f425e6b6ac74b2b3b2c8b20b56641dfa8bcdd325b3bcabe023970855cc7f129e) which was automatically mounted. The mounted DMG does not contain an installer; instead it displays an image containing an alias to a script in the mounted volume:</p>
<p><a href="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/6_flash_symlink_installer_img.png"><img decoding="async" loading="lazy" class="alignnone size-full wp-image-55332" src="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/6_flash_symlink_installer_img.png" alt="" width="1042" height="1042" /></a></p>
<p><i>Figure </i><i>6</i><i>: Flash Installer Script Shortcut</i></p>
<p>The script extracts compressed data containing a macOS .app in a hidden resource known as a <b>resource</b> <b>fork</b>. Resource forks were introduced in the early days of the Macintosh File System (MFS) and are deprecated but are still available, even in macOS Catalina. Resource forks were originally designed to allow an executable to store multiple resources within the file, yet remain separated from the executable data. Much like Alternative Data Streams (ADS) on Windows, the data is hidden from regular file and directory viewers, and multiple “streams” or “forks” are allowed on an individual file.</p>
<p>On macOS, these forks are implemented as an extended attribute (<b>xattr</b>) and can be enumerated or visualized using system tools such as <b>ls</b> and <b>xattr</b>. When <b>ls</b> is run with the <b>–</b><b>l</b> flag on a file that contains extended attributes, an “@” character will appear at the end of the file type and permissions listing:</p>
<p><a href="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/7_terminal_img-ls_l.png"><img decoding="async" loading="lazy" class="alignnone size-full wp-image-55334" src="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/7_terminal_img-ls_l.png" alt="" width="835" height="62" /></a></p>
<p><i>Figure </i><i>7</i><i>: Listing Attributes with ls</i></p>
<p>This indicates that the <b>Install.command</b> file has extended attributes, but does not tell us what <b>kind </b>of attributes they are. If we run <b>ls –l@</b> however, we can see the listed attributes and their sizes, including the ResourceFork:</p>
<p><a href="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/8_terminal_image-ls_l@.png"><img decoding="async" loading="lazy" class="alignnone size-full wp-image-55336" src="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/8_terminal_image-ls_l@.png" alt="" width="827" height="185" /></a></p>
<p><i>Figure </i><i>8</i><i>: Listing Extended Attributes with ls -l@</i></p>
<p>We can also use the tool <b>xattr</b><b> -p</b> to view the contents of the ResourceFork, which in this case is output in hexadecimal (I have used the system command <b>tail</b> below in order to truncate the results – this shows only the end of the resource contents):</p>
<p><a href="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/9_terminal_image-xattr_p.png"><img decoding="async" loading="lazy" class="alignnone size-full wp-image-55338" src="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/9_terminal_image-xattr_p.png" alt="" width="802" height="252" /></a></p>
<p><i>Figure </i><i>9</i><i>: Printing Extended Attributes with </i><i>xattr</i></p>
<p>The script from the analyzed sample is shown below. This script creates a temporary directory into which it will copy and execute the <b>mm-install-</b><b>macos</b> app. The command in the second line of the script takes the last 254kb of the resource fork on the <b>Install.command</b> file, unzips the contents, then further decompresses the data, copying it to the previously created temporary directory. The decompressed data is a macOS .app, which the script then executes in the background.</p>
<p><a href="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/10_terminal_img-install_script.png"><img decoding="async" loading="lazy" class="alignnone size-full wp-image-55340" src="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/10_terminal_img-install_script.png" alt="" width="1734" height="362" /></a></p>

<p><b>Configuration Profiles </b><br />
Highlighted in the table above, one of the methods of persistence and infection used by this variant is the creation of a custom configuration profile. Configuration profiles are typically used in enterprise, educational, or other distributed environments requiring centralized management and deployment of customized system configurations. In the case of this variant of Bundlore, the SearchMine component installed by mm-install-macos uses the configuration profile to lock several browser settings such as the default search page.</p>
<p>Because it installs these profiles from the command line with root privileges, the user is never notified. However, Apple announced this week at WWDC that they will be revoking the ability to silently install configuration profiles from the command line without user input, which will disable this ability of the malware.</p>
<p><a href="https://blog.malwarebytes.com/threat-analysis/2018/04/new-crossrider-variant-installs-configuration-profiles-on-macs/">This post</a> from MalwareBytes details a related malware sample called Crossrider that installs a similar profile, as well as how to list and remove any malicious profiles installed.  In both this Bundlore and the Crossrider samples, the configuration profile was named AdminPrefs. However, the malicious actors could easily change this name at any time by pushing a new installation script during the daily update check.</p>
<p>For example, the script originally downloads an “AdminPrefs” configuration profile template (also seen in the post referenced above) which it dynamically populates with system-specific information on the victim machine.</p>
<p>This script is configured to install different products depending on what is retrieved from the server. This allows the malware authors to dynamically change the malware installed on the system, which is all installed with root permissions due to the configuration during initial setup. <span data-ccp-props="{&quot;335559739&quot;:120,&quot;335559740&quot;:259}"> </span></p>
<p>Below is a sample of the analyzed script which shows the download and population of the profile template. In red, the base URL parameters are shown, along with the search domain that the browser will be configured to use. In blue, the script replaces the fields in the profile template with the custom parameters, and then installs the custom profile as seen in green.</p>
<p><a href="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/15_script_profile_template.png"><img decoding="async" loading="lazy" class="alignnone size-full wp-image-55350" src="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/15_script_profile_template.png" alt="" width="1214" height="720" /></a></p>
<p><i>Figure </i><i>15</i><i>: Update Script Template Creation</i></p>
<p>Once a profile is successfully installed, there will be a new icon in the System Preferences dialog as seen below:</p>
<p><a href="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/16_chrome_policy.png"><img decoding="async" loading="lazy" class="alignnone size-full wp-image-55352" src="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/16_chrome_policy.png" alt="" width="2818" height="1362" /></a></p>
<p><i>Figure </i><i>16</i><i>: Profiles Option Added To Preferences</i></p>
<p>The profiles installed depend on the browsers installed on the system, and which browser is set to default. On the analyzed system configured with Chrome as the default browser, the malware installed a profile that sets the home page, search provider, and new tab default page. As mentioned before and described in more detail below, it also installs the <b>MyCouponsmart</b> extension via means that render the user unable to remove it.</p>
<p>This profile configuration information is stored in plists on the system after loading. These per-user profile plists are located under a user directory under <b>/Library/</b><b>ManagedPreferences</b><b>/&lt;user&gt;/</b> and define browser defaults such as the default search and home page. As mentioned, this script additionally installed a Chrome extension that is unable to be removed by the user, even with administrative privileges. This is accomplished by using the Chrome <b>ExtensionInstallForceList</b> key which is provided for managed enterprise computers. According to <a href="https://cloud.google.com/docs/chrome-enterprise/policies/?policy=ExtensionInstallForcelist">Chrome documentation</a>:</p>
<p>[ExtensionInstallForceList] Specifies a list of apps and extensions that are installed silently, without user interaction, and which cannot be uninstalled nor disabled by the user. All permissions requested by the apps/extensions are granted implicitly, without user interaction, including any additional permissions requested by future versions of the app/extension.</p>
<p>This can be seen in the script below as highlighted in red, where the extension ID is <b>lfbenaabfliihodeianphjhhhcjgddlh</b> and points to the URL http://download[.]shoptimizelymac[.]com for updates:</p>
<p><a href="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/17_chrome_plist_profile.png"><img decoding="async" loading="lazy" class="alignnone size-full wp-image-55354" src="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/17_chrome_plist_profile.png" alt="" width="1614" height="1524" /></a></p>

When this installer script was run again a week after the initial infection, two additional components were installed – <b>macOSOTA</b> and <b>Periodikal</b>, which appear to be additional Bundlore samples (not covered here, but may be analyzed for a future blog post).</p>
<p><a href="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/20_installation_script_mojave.png"><img decoding="async" loading="lazy" class="alignnone size-full wp-image-55360" src="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/20_installation_script_mojave.png" alt="" width="2356" height="1966" /></a></p>

<p>Furthermore, unlike other adware families like <b>Smokyashan</b>, these additional components are not installed in the usual <b>Applications </b>folder, but instead are installed into the user’s Application directory located in <b>/Users/&lt;user&gt;/Applications</b>. This folder is not readily visible to the user unless they navigate directly to the directory in Finder.</p>
<p><a href="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/21_applications_user.png"><img decoding="async" loading="lazy" class="alignnone size-full wp-image-55362" src="http://blogs.vmware.com/security/wp-content/blogs.dir/26/files/2020/07/21_applications_user.png" alt="" width="1244" height="240" /></a></p>
<p><i>Figure </i><i>21</i><i>: User Applications Directory</i></p>
<p>Although this variant of Bundlore is not significantly different than others seen over the last year, the additional features of manipulating the sudoers file and installation of Configuration Profiles are less commonly seen. The main takeaway however is that adware is often dismissed, but samples like the Bundlore variant analyzed in this post are able to install anything as root without any notification to the user after initial authentication. Although the only programs observed to be installed by Bundlore thus far have been adware, with root access and no additional authentication required from the user, any software could be installed with full access to the file system.</p>
