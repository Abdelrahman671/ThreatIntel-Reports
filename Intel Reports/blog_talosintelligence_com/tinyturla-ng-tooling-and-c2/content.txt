    <h1 class="text-center">TinyTurla-NG in-depth tooling and command and control analysis</h1>

    <div class="text-center m-1">
        <div class="post-author">
            <span>By </span>
                <a href="https://blog.talosintelligence.com/author/asheer-malhotra/">Asheer Malhotra</a>, 
                <a href="https://blog.talosintelligence.com/author/holger-unterbrink/">Holger Unterbrink</a>,

        <div class="m-3">

    <section class="post-content-wrapper mt-5">
        <div class="post-content">
            <ul><li>Cisco Talos, in cooperation with <a href="https://cert.ngo/"><u>CERT.NGO</u></a>, has discovered new malicious components used by the Turla APT. New findings from Talos illustrate the inner workings of the command and control (C2) scripts deployed on the compromised WordPress servers utilized in the compromise we <a href="https://blog.talosintelligence.com/tinyturla-next-generation/"><u>previously disclosed</u></a>.</li><li>Talos also illustrates the post-compromise activity carried out by the operators of the <a href="https://blog.talosintelligence.com/tinyturla-next-generation/"><u>TinyTurla-NG (TTNG) backdoor</u></a> to issue commands to the infected endpoints. We found three distinct sets of PowerShell commands issued to TTNG to enumerate, stage and exfiltrate files that the attackers found to be of interest.</li><li>Talos has also discovered the use of another three malicious modules deployed via the initial implant, TinyTurla-NG, to maintain access, and carry out arbitrary command execution and credential harvesting.</li><li>One of these components is a modified agent/client from Chisel, an open-sourced attack framework, used to communicate with a separate C2 server to execute arbitrary commands on the infected systems.</li><li>Certificate analysis of the Chisel client used in this campaign indicates that another modified chisel implant has likely been created that uses a similar yet distinct certificate. This assessment is in line with Turla’s usage of multiple variants of malware families including TinyTurla-NG, TurlaPower-NG and other PowerShell-based scripts during this campaign.</li></ul><p>Talos, in cooperation with <a href="https://cert.ngo/"><u>CERT.NGO</u></a>, has discovered new malicious components used by the Turla APT in the compromise we’ve <a href="https://blog.talosintelligence.com/tinyturla-next-generation/"><u>previously disclosed</u></a>. The continued investigation also revealed details of the inner workings of the C2 scripts including handling of incoming requests and a WebShell component that allows the operators to administer the compromised C2 servers remotely.</p><h1 id="c2-server-analysis">C2 server analysis</h1><p>The command and control (C2) code is a PHP-based script that serves two purposes: It’s a handler for the TinyTurla-NG implants and web shell that the Turla operators can use to execute commands on the compromised C2 server. The C2 scripts obtained by Talos are complementary to the TinyTurla-NG (TTNG) and TurlaPower-NG implants and are meant to deliver executables and administrative commands to execute on infected systems.</p><p>On load, the PHP-based C2 script will perform multiple actions to create the file structure used to serve the TTNG backdoor. After receiving a request, the C2 script first checks if the logging directory exists, if not, it will create one. Next, the script checks for a specific COOKIE ID. If it exists and corresponds to the hardcoded value, then the C2 script will act as a web shell.</p><p>It will base64 decode the value of the $_COOKIE (not to be confused with the authentication COOKIE ID) entry and execute it on the C2 server as a command. These commands are either run using the exec(), passthru(), system(), or shell_exec() functions. It will also check if the variable specified is a resource and read its contents. Once the actions are complete, the output or resource is sent to the requestor and the PHP script will stop executing.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://lh7-us.googleusercontent.com/l_SKnGLRah-d24nq4gfPxsjd5_Y5RJhALweXOB31VPSRlgDsrcEbkOqQRVl1amnaEwgfTU9zifxJEDjiim4S1dUyEkG8-tNKH6U_Vw0aAkCG0K7L5PvXLNvJg6ig6VebhyrimdO4obFWQLpU5e4uFls" class="kg-image" alt="" loading="lazy" width="1600" height="1457"><figcaption><span style="white-space: pre-wrap;">C2 script’s web shell capability.</span></figcaption></figure><p>If there is an “id” provided in the HTTP request to the C2 server, the script will treat this as communication with an implant, such as TTNG or TurlaPower-NG. The “id” parameter is the same variable that is passed by the TTNG and TurlaPower-NG implants during communication with the C2 and creates the logging directory on the C2 server, as well. Depending on the next form value accompanying the “id”, the C2 will perform the following actions:</p><ul><li>"<strong>task</strong>": Write the content sent by the requestor to the “&lt;id&gt;/tasks.txt” file and record the requestor’s IP address and timestamp in the “&lt;id&gt;/_log.txt”. The contents of this file are then sent to the requestor in response to the “gettask” request. Adversaries use this mechanism to add more tasks to the list of tasks/commands that each C2 must send to their backdoor installations to execute on the infected endpoints.</li><li>"<strong>gettask</strong>": Send the contents of the “&lt;id&gt;/tasks.txt” file to the infected system requesting a new command to execute on the infected endpoint.</li><li>"<strong>result</strong>": Get the content of the HTTP(S) form and record it into the “&lt;id&gt;/result.txt” file. The C2 uses this mechanism to obtain and record the output of a command executed on an infected endpoint by the TTNG backdoor into a file on disk.</li><li>"<strong>getresult</strong>": Get the contents of the “&lt;id&gt;/result.txt” file from the C2 server. The adversaries use this to obtain the results of a command executed on the infected endpoint without having to access the C2 server.</li><li>"<strong>file</strong>" + "<strong>name</strong>": Save the contents of the file sent to the C2 server either in full or part to a file specified on the C2 server with the same “name” specified in the HTTP form.</li><li>"<strong>cat_file</strong>": Read the contents of a file specified by the requestor on the C2 server and respond with the contents.</li><li>"<strong>rm_file</strong>": Remove/delete a file specified by the requestor from the C2 server.</li></ul><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://lh7-us.googleusercontent.com/-wiL9sxTD6F0Zmt8sg0bKhAtau2EINzm80i_NyRombAsl5G3xkyWfUEGtA5zJMNijEqMHgdsSbK4DVKgFV7gRZmsq8OmDMAYBVC9F7SEhQjU35jM0wXln63flR21nVM9wmNISA-2XK4ksSUzdnpICUQ" class="kg-image" alt="" loading="lazy" width="1600" height="1424"><figcaption><span style="white-space: pre-wrap;">The C2 script’s request handling logic.</span></figcaption></figure><p>The HTTP form values accepted by the C2 server <code>task</code>, <code>cat_file</code>, <code>rm_file</code>, <code>get_result</code> and their corresponding operations on the C2 server indicate that these are part of an operational apparatus that allows the threat actors to feed the C2 server new commands and retrieve valuable information collected by the C2 server, <em>from a remote location</em>, without having to log into the C2 itself. Operationally, this is a tactic that is beneficial to the threat actors considering that all C2 servers discovered so far are websites compromised by the threat actor instead of being attacker-owned. Therefore, it would be beneficial for Turla’s operators to simply communicate over HTTPS masquerading as legitimate traffic instead of re-exploiting or accessing the servers through other means such as SSH thereby increasing their fingerprint on the compromised C2 servers.</p><p>This tactic can be visualized as:</p><figure class="kg-card kg-image-card"><img src="https://lh7-us.googleusercontent.com/HUPRV_gl4vZjB4FryMs1Y3IbbMzfcWcU3D_9fswydwIxgAVpabFpN9yn2GGBvobrZg19qtLAJz8DHZQmJ1yvEIrrY0cG9DvnFJJzc3SIWU215M969AAjoxIdW4yZAI0TBfWaAKA4wABUqAFpT7ebhXM" class="kg-image" alt="" loading="lazy" width="1600" height="860"></figure><h1 id="instrumenting-tinyturla-ng-to-carry-out-post-compromise-activity">Instrumenting TinyTurla-NG to carry out post-compromise activity</h1><p>The adversaries use TinyTurla-NG to perform additional reconnaissance to enumerate files of interest on the infected endpoints and then exfiltrate these files. They issued three distinct sets of modular PowerShell commands to TTNG:</p><ul><li><strong>Reconnaissance commands:</strong> Used to enumerate files in a directory specified by the operator. The directory listing is returned to the operator to select interesting files that can be exfiltrated.</li></ul><figure class="kg-card kg-image-card"><img src="https://lh7-us.googleusercontent.com/-yIw-M0KrbE0pYJ-wVOJlygJ2Q_RJfA7PSTtw0EJ3MjU7hUMLIqpxbMLIMiL1hTXegvUtNv4fxvN3Kfnd9AiZvvNwhoCP9nCqvRD-5MzYkTWUAitdCyuMYYPDf2MlyukErRwQq410WfGX6HQhZxwXWI" class="kg-image" alt="" loading="lazy" width="1296" height="777"></figure><p>PowerShell script/Command enumerates files in four locations specified by the C2 and sends the results back to it.</p><ul><li><strong>Copy file commands:</strong> Base64-encoded commands/scripts issued to the infected systems to copy over files of interest from their original location to a temporary directory, usually: C:\windows\temp\</li></ul><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://lh7-us.googleusercontent.com/fPHWBmyHy8mdAaBJ3myu1nyze9hpY0nblTFCuoYhSa5Mqx0NMkcr9K2_1usEmf6BjCs7oI2SKhDoE4mxRbZ6avCcGN6Arrd7B4TWmqKCRfYQ2I47zeUD-0SKSGz6FZerVQgVF0gVQ0_dsiRDTYNLZ1I" class="kg-image" alt="" loading="lazy" width="1221" height="716"><figcaption><span style="white-space: pre-wrap;">PowerShell script copies files to an intermediate location.</span></figcaption></figure><ul><li><strong>Exfiltration</strong> <strong>commands/scripts aka TurlaPower-NG: </strong>These scripts were used to finally exfiltrate the selected files to the C2 servers.</li></ul><p>The scripts used during enumeration, copying and exfiltration tasks contain hardcoded paths for files and folders of interest to Turla. These locations consisted of files and documents that were used and maintained by Polish NGOs to conduct their day-to-day operations. The actors also used these scripts to exfiltrate Firefox profile data, reinforcing our assessment that Turla made attempts to harvest credentials, along with data exfiltration.</p><p>While Tinyturla-NG itself is enough to perform a variety of unauthorized actions on the infected system using a combination of scripts described above, the attackers chose to deploy three more tools to aid in their malicious operations:</p><ul><li>Chisel: Modified copy of the Chisel client/agent.</li><li>Credential harvesting scripts: PowerShell-based scripts for harvesting Google Chrome or Microsoft Edge’s saved login data.</li><li>Tool for executing commands with elevated privileges: A binary that is meant to impersonate privilege levels of a specified process while executing arbitrary commands specified by the parent process.</li></ul><p>The overall infection activity once TTNG has been deployed looks like this:</p><figure class="kg-card kg-image-card"><img src="https://lh7-us.googleusercontent.com/0D_VPPzi0vXeUDXZTtOoAMJW-1DqxKG1aYNUWildZ7CZ6FvrTidljkhipxvhev8eQ6HiwRumr9Fvpocf9PvMmkUqB8Mc_lumC5BPZ9QyCLcuuo5X-8v_-WfnqkUMzxDAlTuIgLbegchb5hIAmc6mJFg" class="kg-image" alt="" loading="lazy" width="1600" height="1214"></figure><h1 id="using-chisel-as-another-means-of-persistent-access">Using Chisel as another means of persistent access</h1><p>Talos’ investigation uncovered that apart from TurlaPower-NG, the PowerShell-based file exfiltrator, the adversary also deployed another implant on infected systems. It’s a modified copy of the GoLang-based, open-source tunneling tool <a href="https://github.com/jpillora/chisel"><u>Chisel</u></a> stored in the location: C:\Windows\System32\TrustedWorker[.]exe</p><p>The modified Chisel malware is UPX compressed, as is common for Go binaries, and contains the C2 URL, port and communication certificate, and private keys embedded in the malware sample. Once it decrypts these artifacts, it continues to create a reverse SOCKS proxy connection to the C2 using the configuration: R:5000:socks</p><p>In the proxy:</p><ul><li><strong>“R”: </strong>Stands for remote port forwarding.</li><li><strong>“5000”</strong>: This is the port on the attacker machine that receives the connection from the infected system.</li><li><strong>“socks”</strong>: Specifies the usage of the SOCKS protocol.&nbsp;</li></ul><p>(The default local host and port for a socks remote in Chisel is 127[.]0[.]0[.]1:1080.)</p><p>The C2 server that the chisel sample contacts is: 91[.]193[.]18[.]120:443.</p><p>The TLS configuration consists of a client TLS certificate and key pair. The certificate is valid between Dec. 7, 2023 and Dec. 16, 2024. This validity falls in line with Talos’ assessment that the campaign began in December 2023. The issuer of the certificate is named “dropher[.]com” and the subject name is “blum[.]com”.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://lh7-us.googleusercontent.com/2Pi5BLDwjRFKJ-sLeLCYPViTgGFqK_ppHVNYkS4CnRNT05ecctQTw6-UzRT-bW3pxrwHqvYASFXfV10tKdbAEfME-EU3ymApwdefHG9vVMnOOYxqIa1wULReQrhaGj6PpUm0pp3AXolXZ89c3mRl_MQ" class="kg-image" alt="" loading="lazy" width="1600" height="1128"><figcaption><span style="white-space: pre-wrap;">TLS Certificate for the chisel malware used by Turla.&nbsp;</span></figcaption></figure><p>During our data analysis, we found another certificate which we assessed with high confidence was also generated by Turla operators, but it's unclear if this was a mistake or if they intended for the certificate to be used on another modified chisel implant.&nbsp;</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://lh7-us.googleusercontent.com/JW9Yv23xTi2UAsaaxqTBWN3tnMDXT-B3A3m3wWwSwhhDqoTWsq5tO4L25CSkZHu1wiSFzwRGrn-yBygRSjEg5fse1UeCE85eWhxhiTDXQpjppzx8b0mNwXNkmCC3_cVqCt312x9dNtaZGKnb_st03dY" class="kg-image" alt="" loading="lazy" width="1084" height="46"><figcaption><span style="white-space: pre-wrap;">Certificate issuer DN.</span></figcaption></figure><p>The new certificate has the same issuer but in this case, the common name is <em>blum[.]com</em> and the serial number is 0x1000. This certificate was generated one second before the one used in the modified chisel client/agent.</p><h1 id="additional-tools-for-elevated-process-execution-and-credential-harvesting">Additional tools for elevated process execution and credential harvesting</h1><p>Turla also deployed two more tools to aid their malicious operations on the infected systems. One is used to run arbitrary commands on the system and the other is used to steal Microsoft Edge browser’s login data.</p><p>The first tool is a small and simple Windows executable to create a new command line process on the system by impersonating the privilege level of another existing process. The tool will accept a target Process Identifier (PID) representing the process whose privilege level is to be impersonated and the command line that needs to be executed. Then, a new cmd[.]exe is spawned and used to execute arbitrary commands on the infected endpoint. The binary was compiled in early 2022 and was likely used in previous campaigns by Turla.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://lh7-us.googleusercontent.com/40kxBT0AwJC6-FHRCZClml9zAXD7qnDTSl4W27frQUuuSzuGkYFcAU6K8777E4fSAxl6rBDO3JPIDNmgYKlc_NMgPVAWKJqbGYOrx_veRdpbqKTFbqQbgAB1zy6EZF95r_nw02j48gE91aNNhKQw9RU" class="kg-image" alt="" loading="lazy" width="951" height="215"><figcaption><span style="white-space: pre-wrap;">The tool contains the embedded cmd[.]exe command line.</span></figcaption></figure><p>The second tool discovered by Talos is a PowerShell script residing at the location:</p>
<!--kg-card-begin: html-->
<pre>C:\windows\system32\edgeparser.ps1</pre>
<!--kg-card-end: html-->
<p>This script is used&nbsp;to find&nbsp; login data from Microsoft Edge located at:</p>
<!--kg-card-begin: html-->
<pre>%userprofile%\AppData\Local\Microsoft\Edge\User Data\Default\Login Data</pre>
<!--kg-card-end: html-->
<p>This data file and the corresponding decryption key for the login data extracted from the endpoint is archived into a ZIP file and stored in the directory:&nbsp;C:\windows\temp\&lt;filename&gt;.zip</p><p>The script can be used to obtain credentials for Google Chrome as well but has been modified to parse login data from: </p>
<!--kg-card-begin: html-->
<pre>%userprofile%\AppData\Local\Microsoft\Edge</pre>
<!--kg-card-end: html-->
<figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://lh7-us.googleusercontent.com/iBdbF4aW5ie9liIxGGzgVTRhy2r4rbYq_QKdTqhAZl101K0s7R9erX22MiHmorwExVdVA5N12nw8mttolFGr4fIBz4eqkVJGMhqg3WV4-4UB2JPFVwDkMzb7skGQQFOGjpUsVficylOEmprtotmpfUc" class="kg-image" alt="" loading="lazy" width="1249" height="1600"><figcaption><span style="white-space: pre-wrap;">PowerShell script obtaining key and login data to add to the archive for exfiltration.</span></figcaption></figure><p>TTNG uses the privilege elevation tool to run the PowerShell script using the command:</p>
<!--kg-card-begin: html-->
<pre>"C:\Windows\System32\i.exe" _PID_ "powershell -f C:\Windows\System32\edgeparser.ps1"</pre>
<!--kg-card-end: html-->
<p>This results in the tool spawning a new process with the command line:</p>
<!--kg-card-begin: html-->
<pre>C:\Windows\System32\cmd.exe /c "powershell -f C:\Windows\System32\edgeparser.ps1"</pre>
