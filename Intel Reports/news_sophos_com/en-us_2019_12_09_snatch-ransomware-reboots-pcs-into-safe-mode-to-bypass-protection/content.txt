	<div class="entry-content lg:prose-lg mx-auto prose max-w-4xl">
		<p>The Sophos Managed Threat Response (MTR) team and SophosLabs researchers have been investigating an ongoing series of ransomware attacks in which the ransomware executable forces the Windows machine to reboot into Safe Mode before beginning the encryption process. The attackers may be using this technique to circumvent endpoint protection, which often won&#8217;t run in Safe Mode.</p>
<p><iframe title="Sophos Snatch Ransomware Demo" src="https://player.vimeo.com/video/379844364?dnt=1&amp;app_id=122963" width="640" height="360" frameborder="0" allow="autoplay; fullscreen; picture-in-picture; clipboard-write"></iframe></p>
<p>In mid-October, the Sophos MTR team worked with a targeted organization to investigate and remediate a ransomware outbreak within their network. The ransomware, which calls itself Snatch, sets itself up as a service that will run during a Safe Mode boot. It quickly reboots the computer into Safe Mode, and in the rarefied Safe Mode environment, where most software (including security software) doesn&#8217;t run, Snatch encrypts the victims&#8217; hard drives.</p>
<figure id="attachment_62451" aria-describedby="caption-attachment-62451" style="width: 864px" class="wp-caption alignnone"><a href="https://news.sophos.com/wp-content/uploads/2019/12/snatch-reboot-collage-large.png"><img decoding="async" class="wp-image-62451 " src="https://news.sophos.com/wp-content/uploads/2019/12/snatch-reboot-collage-large.png" alt="Snatch ransomware reboots a test computer into Safe Mode before beginning to encrypt its hard drive" width="864" height="147" srcset="https://news.sophos.com/wp-content/uploads/2019/12/snatch-reboot-collage-large.png 3520w, https://news.sophos.com/wp-content/uploads/2019/12/snatch-reboot-collage-large.png?resize=300,51 300w, https://news.sophos.com/wp-content/uploads/2019/12/snatch-reboot-collage-large.png?resize=768,131 768w, https://news.sophos.com/wp-content/uploads/2019/12/snatch-reboot-collage-large.png?resize=1024,175 1024w, https://news.sophos.com/wp-content/uploads/2019/12/snatch-reboot-collage-large.png?resize=1536,262 1536w, https://news.sophos.com/wp-content/uploads/2019/12/snatch-reboot-collage-large.png?resize=2048,349 2048w" sizes="(max-width: 864px) 100vw, 864px" /></a><figcaption id="caption-attachment-62451" class="wp-caption-text">Snatch runs itself in an elevated permissions mode, sets registry keys that instructs Windows to run it following a Safe Mode reboot, then reboots the computer and starts encrypting the disk while it&#8217;s running in Safe Mode</figcaption></figure>
<p>Sophos analysts first encountered the Snatch ransomware about a year ago. The threat actor identities behind the ransomware appear to have been active since the summer of 2018. SophosLabs believes that the Safe Mode enhancement to this malware is a newly added feature.</p>
<p>SophosLabs feels that the severity of the risk posed by ransomware which runs in Safe Mode cannot be overstated, and that we needed to publish this information as a warning to the rest of the security industry, as well as to end users. As we continue to investigate new incidents, we will update this post, and may post a followup in the next few days.</p>
<p><span id="more-62404"></span></p>
<p>What we refer to as Snatch malware comprises a collection of tooling, which include a ransomware component and a separate data stealer, both apparently built by the criminals who operate the malware; a Cobalt Strike reverse-shell; and several publicly-available tools that aren&#8217;t inherently malicious, but used more conventionally by penetration testers, system administrators, or technicians.</p>
<p>One of a growing number of malware families we&#8217;ve encountered that have been programmed in Go, Snatch does not appear to be multiplatform. Created by Google, Go was designed to be able to produce programs that, in theory, could run under multiple operating systems.</p>
<p>However, the malware we&#8217;ve observed isn&#8217;t capable of running on platforms other than Windows. Snatch can run on most common versions of Windows, from 7 through 10, in 32- and 64-bit versions. The samples we&#8217;ve seen are also packed with the open source packer UPX to obfuscate their contents.</p>
<h2>Snatch&#8217;s threat actors job postings</h2>
<p>The threat actors behind this malware (who refer to themselves on criminal message boards as &#8220;Snatch Team&#8221;) appear to have adopted the <em>active automated attack</em> model, in which they seek to penetrate enterprise networks via automated brute-force attacks against vulnerable, exposed services, and then leverage that foothold to spread internally within the targeted organization&#8217;s network through human-directed action.</p>
<p>Online posts from criminal boards by suspected members of Snatch Team appear to support the assertion that this is the attacker&#8217;s<em> modus operandi</em>. A user (who goes by the online moniker BulletToothTony) soliciting assistance in this type of attack method, writing in a (translated from the original Russian language) message board posting titled &#8220;Snatch ransomware&#8221; that he is &#8220;Looking for affiliate partners with access to RDP\VNC\TeamViewer\WebShell\SQL inj [SQL injection] in corporate networks, stores and other companies.&#8221;</p>
<p><a href="https://news.sophos.com/wp-content/uploads/2019/12/ru1.png"><img loading="lazy" decoding="async" class="alignnone size-full wp-image-62551" src="https://news.sophos.com/wp-content/uploads/2019/12/ru1.png" alt="BulletToothTony: Looking for affiliate partners with access to RDP\VNC\TeamViewer\WebShell\SQLinj in corporate networks, stores and other companies" width="640" height="270" srcset="https://news.sophos.com/wp-content/uploads/2019/12/ru1.png 923w, https://news.sophos.com/wp-content/uploads/2019/12/ru1.png?resize=300,126 300w, https://news.sophos.com/wp-content/uploads/2019/12/ru1.png?resize=768,324 768w" sizes="(max-width: 640px) 100vw, 640px" /></a>Later in the same message thread, this user offers to (at no charge) train others in the use of the malware, allow prospective criminal partners to use their infrastructure, provide &#8220;the best students&#8221; with a customized server running Metasploit, and then says &#8220;we are looking for capable people to join our team.&#8221;</p>

Спасибо, ты такой <span class="css-901oao css-16my406 r-1qd0xha r-ad9z0x r-bcqeeo r-qvutc0">заботливый</span>.</p>
<p><a href="https://news.sophos.com/wp-content/uploads/2019/12/ru3.png"><img loading="lazy" decoding="async" class="alignnone size-full wp-image-62554" src="https://news.sophos.com/wp-content/uploads/2019/12/ru3.png" alt="We don't work with english speaking users sorry" width="640" height="233" srcset="https://news.sophos.com/wp-content/uploads/2019/12/ru3.png 760w, https://news.sophos.com/wp-content/uploads/2019/12/ru3.png?resize=300,109 300w" sizes="(max-width: 640px) 100vw, 640px" /></a>The threat actors have also innovated their crime in another important way: One piece of malware used in the Snatch attacks is capable of, and has been, stealing vast amounts of information from the target organizations.</p>
<h2>Deciphering the Snatch attack</h2>
<p>In one of the incidents, which targeted a large international company, the MTR team managed to obtain detailed logs from the targeted company that the ransomware had not been able to encrypt. The attackers initially accessed the company&#8217;s internal network by brute-forcing the password to an administrator&#8217;s account on a Microsoft Azure server, and were able to log in to the server using Remote Desktop (RDP).</p>
<p>Using the Azure server as a foothold, the attackers leveraged that administrator&#8217;s account to log into a domain controller (DC) machine on the same network, and then performed surveillance tasks on the target&#8217;s network over the course of several weeks.</p>
<p>The attackers query the list of users authorized to log in on the box, and write the results to a file. We also observed them dump WMIC system &amp; user data, process lists, and even the memory contents of the Windows LSASS service, to a file&#8230;</p>
<figure id="attachment_62462" aria-describedby="caption-attachment-62462" style="width: 640px" class="wp-caption alignnone"><a href="https://news.sophos.com/wp-content/uploads/2019/12/exfil-user-info.png"><img loading="lazy" decoding="async" class="size-full wp-image-62462" src="https://news.sophos.com/wp-content/uploads/2019/12/exfil-user-info.png" alt="" width="640" height="101" srcset="https://news.sophos.com/wp-content/uploads/2019/12/exfil-user-info.png 696w, https://news.sophos.com/wp-content/uploads/2019/12/exfil-user-info.png?resize=300,47 300w" sizes="(max-width: 640px) 100vw, 640px" /></a><figcaption id="caption-attachment-62462" class="wp-caption-text">User information stolen by Snatch</figcaption></figure>
<figure id="attachment_62475" aria-describedby="caption-attachment-62475" style="width: 511px" class="wp-caption alignnone"><a href="https://news.sophos.com/wp-content/uploads/2019/12/microsoftteams-image-4.png"><img loading="lazy" decoding="async" class="wp-image-62475 size-full" src="https://news.sophos.com/wp-content/uploads/2019/12/microsoftteams-image-4.png" alt="" width="511" height="139" srcset="https://news.sophos.com/wp-content/uploads/2019/12/microsoftteams-image-4.png 511w, https://news.sophos.com/wp-content/uploads/2019/12/microsoftteams-image-4.png?resize=300,82 300w" sizes="(max-width: 511px) 100vw, 511px" /></a><figcaption id="caption-attachment-62475" class="wp-caption-text">Snatch dumps lsass from memory then uploads the dump</figcaption></figure>
<p>&#8230;then upload them to their C2 server.</p>
<p>We&#8217;ve also observed that the attackers set up one-off Windows services to orchestrate specific tasks. These services have long randomized filenames, such as this one, which queries the list of running processes from the tasklist program, outputs it to a file in the temp directory, then runs a batch file (also located in the temp directory) that uploads the tasklist file to the C2 server.</p>
<p><img loading="lazy" decoding="async" class="alignnone size-full wp-image-62556" src="https://news.sophos.com/wp-content/uploads/2019/12/taskfile-service.png" alt="" width="640" height="257" srcset="https://news.sophos.com/wp-content/uploads/2019/12/taskfile-service.png 1018w, https://news.sophos.com/wp-content/uploads/2019/12/taskfile-service.png?resize=300,120 300w, https://news.sophos.com/wp-content/uploads/2019/12/taskfile-service.png?resize=768,308 768w" sizes="(max-width: 640px) 100vw, 640px" />In fact, it uses this same method to upload a lot of information to the C2 server. For instance, it uses this command to send the extracted user account and other profile information (the .txt file) back to the C2, and then executes a batch file it has created in the Windows temp directory.</p>
<p><a href="https://news.sophos.com/wp-content/uploads/2019/12/c2-comms.png"><img loading="lazy" decoding="async" class="alignnone size-full wp-image-62457" src="https://news.sophos.com/wp-content/uploads/2019/12/c2-comms.png" alt="" width="640" height="68" srcset="https://news.sophos.com/wp-content/uploads/2019/12/c2-comms.png 779w, https://news.sophos.com/wp-content/uploads/2019/12/c2-comms.png?resize=300,32 300w, https://news.sophos.com/wp-content/uploads/2019/12/c2-comms.png?resize=768,82 768w" sizes="(max-width: 640px) 100vw, 640px" /></a>The attackers installed surveillance software on about 200 machines, or roughly 5% of the computers on this particular organization&#8217;s internal network. The attackers installed several malware executables; The first group of files appears to be designed to give the attackers remote access to the machines without having to rely on the compromised Azure server.</p>
<p>The attackers also installed a free Windows utility called <strong>Advanced Port Scanner</strong> and used that tool to discover additional machines on the network they could target. Following this incident response, we were contacted by another company targeted by this same malware, and the investigation found a copy of Advanced Port Scanner on machines in that network, too.</p>
<figure id="attachment_62580" aria-describedby="caption-attachment-62580" style="width: 790px" class="wp-caption alignnone"><a href="https://news.sophos.com/wp-content/uploads/2019/12/microsoftteams-image-8.png"><img loading="lazy" decoding="async" class="wp-image-62580" src="https://news.sophos.com/wp-content/uploads/2019/12/microsoftteams-image-8.png" alt="" width="790" height="137" srcset="https://news.sophos.com/wp-content/uploads/2019/12/microsoftteams-image-8.png 798w, https://news.sophos.com/wp-content/uploads/2019/12/microsoftteams-image-8.png?resize=300,52 300w, https://news.sophos.com/wp-content/uploads/2019/12/microsoftteams-image-8.png?resize=768,134 768w" sizes="(max-width: 790px) 100vw, 790px" /></a><figcaption id="caption-attachment-62580" class="wp-caption-text">The Update_Collector.exe tool is involved in exfiltration of stolen data</figcaption></figure>
<p>Sophos analysts also found a tool we suspect was also created by the malware authors named <strong>Update_Collector.exe</strong>; The tool takes the data that had been collected using WMI to learn more about other machines and user accounts on the network, dumps that information to a file, and then uploads it to the attackers&#8217; command-and-control server. We came across copies on some of the compromised machines.</p>
<p>We also found a range of otherwise legitimate tools that have been adopted by criminals installed on machines within the target&#8217;s network, including <a href="https://processhacker.sourceforge.io/" target="_blank" rel="noopener noreferrer">Process Hacker</a>, <a href="https://www.iobit.com/en/advanceduninstaller.php" target="_blank" rel="noopener noreferrer">IObit Uninstaller</a>, <a href="https://www.microsoft.com/en-us/p/power-tool/9nblggh0cfd6?activetab=pivot:overviewtab" target="_blank" rel="noopener noreferrer">PowerTool</a>, and <a href="https://docs.microsoft.com/en-us/sysinternals/downloads/psexec" target="_blank" rel="noopener noreferrer">PsExec</a>. The attackers typically use them to try to disable AV products.</p>
<p>Subsequent hunts for related files revealed several other attacks in which precisely the same collection of tools was used in what appear to be opportunistic attacks against organizations located around the world, including the United States, Canada, and several European countries. All the organizations where these same files were found also were later discovered to have one or more computers with RDP exposed to the internet. Many of the components were found in the Downloads folder for an admin account on the infected system.</p>
<p>At some point during the attack, which may be several days to weeks after the initial network breach, the attacker downloads the ransomware component to the targeted machine(s). This component arrives on the system with a filename that includes the unique-to-each-victim five-character code and the word &#8220;<strong>_pack.exe</strong>&#8221; in the filename.</p>
<figure id="attachment_62560" aria-describedby="caption-attachment-62560" style="width: 839px" class="wp-caption alignnone"><a href="https://news.sophos.com/wp-content/uploads/2019/12/central1.png"><img loading="lazy" decoding="async" class="wp-image-62560 size-full" src="https://news.sophos.com/wp-content/uploads/2019/12/central1.png" alt="" width="839" height="402" srcset="https://news.sophos.com/wp-content/uploads/2019/12/central1.png 839w, https://news.sophos.com/wp-content/uploads/2019/12/central1.png?resize=300,144 300w, https://news.sophos.com/wp-content/uploads/2019/12/central1.png?resize=768,368 768w" sizes="(max-width: 839px) 100vw, 839px" /></a><figcaption id="caption-attachment-62560" class="wp-caption-text">As viewed through Sophos Central, the system is manipulated into downloading the ransomware file to disk, then executes it using PSEXEC</figcaption></figure>
<p>By the time the malware invokes the PSEXEC service to execute the ransomware, it has extracted itself into the Windows folder with the same five-character code followed by <strong>_unpack.exe.</strong></p>
<figure id="attachment_62561" aria-describedby="caption-attachment-62561" style="width: 476px" class="wp-caption alignnone"><a href="https://news.sophos.com/wp-content/uploads/2019/12/central2.png"><img loading="lazy" decoding="async" class="wp-image-62561 size-full" src="https://news.sophos.com/wp-content/uploads/2019/12/central2.png" alt="" width="476" height="464" srcset="https://news.sophos.com/wp-content/uploads/2019/12/central2.png 476w, https://news.sophos.com/wp-content/uploads/2019/12/central2.png?resize=300,292 300w" sizes="(max-width: 476px) 100vw, 476px" /></a><figcaption id="caption-attachment-62561" class="wp-caption-text">The &#8220;unpack&#8221; version ends up in the Windows directory</figcaption></figure>
<p>The ransomware installs itself as a Windows service called <strong>SuperBackupMan. </strong>The service description text, &#8220;This service make backup copy every day,&#8221; might help camouflage this entry in the Services list, but there&#8217;s no time to look. This registry key is set immediately before the machine starts rebooting itself.</p>
<p><a href="https://news.sophos.com/wp-content/uploads/2019/12/sbmsvc.png"><img loading="lazy" decoding="async" class="alignnone size-full wp-image-62559" src="https://news.sophos.com/wp-content/uploads/2019/12/sbmsvc.png" alt="" width="640" height="202" srcset="https://news.sophos.com/wp-content/uploads/2019/12/sbmsvc.png 857w, https://news.sophos.com/wp-content/uploads/2019/12/sbmsvc.png?resize=300,95 300w, https://news.sophos.com/wp-content/uploads/2019/12/sbmsvc.png?resize=768,243 768w" sizes="(max-width: 640px) 100vw, 640px" /></a>The SuperBackupMan service has properties that prevent it from being stopped or paused by the user while it&#8217;s running.</p>
<p><a href="https://news.sophos.com/wp-content/uploads/2019/12/notstop.png"><img loading="lazy" decoding="async" class="alignnone size-full wp-image-62563" src="https://news.sophos.com/wp-content/uploads/2019/12/notstop.png" alt="" width="640" height="205" srcset="https://news.sophos.com/wp-content/uploads/2019/12/notstop.png 668w, https://news.sophos.com/wp-content/uploads/2019/12/notstop.png?resize=300,96 300w" sizes="(max-width: 640px) 100vw, 640px" /></a>The malware then adds this key to the Windows registry so it will start up during a Safe Mode boot.</p>
<pre><strong>HKLM\SYSTEM\CurrentControlSet\Control\SafeBoot\Minimal\SuperBackupMan:Default:Service</strong></pre>
<p><a href="https://news.sophos.com/wp-content/uploads/2019/12/registry2.png"><img loading="lazy" decoding="async" class="alignnone size-full wp-image-62481" src="https://news.sophos.com/wp-content/uploads/2019/12/registry2.png" alt="" width="640" height="195" srcset="https://news.sophos.com/wp-content/uploads/2019/12/registry2.png 779w, https://news.sophos.com/wp-content/uploads/2019/12/registry2.png?resize=300,91 300w, https://news.sophos.com/wp-content/uploads/2019/12/registry2.png?resize=768,234 768w" sizes="(max-width: 640px) 100vw, 640px" /></a>Using the BCDEDIT tool on Windows, it issues a command that sets up windows operating system to boot in Safe Mode, and then immediately forces a reboot of the infected computer.</p>
<pre><strong>bcdedit.exe /set {current} safeboot minimal</strong>
<strong>shutdown /r /f /t 00</strong></pre>
<p>When the computer comes back up after the reboot, this time in Safe Mode, the malware uses the Windows component net.exe to halt the SuperBackupMan service, and then uses the Windows component vssadmin.exe to delete all the Volume Shadow Copies on the system, which prevents forensic recovery of the files encrypted by the ransomware.</p>
<pre><strong>net stop SuperBackupMan</strong>
<strong>vssadmin delete shadows /all /quiet</strong></pre>
<p>The ransomware then begins encrypting documents on the infected machine&#8217;s local hard drive.</p>
<h2>The impact of Snatch</h2>
<p>The ransomware appends a pseudorandom string of five alphanumeric characters to the encrypted files. This string appears both in the filename of (and hardcoded into) the ransomware executable, and in the ransom note, and appears to be unique to each targeted organization. For example, if the ransomware is named <strong>abcde</strong>x64.exe, the encrypted files would have the file extension .<strong>abcde</strong> appended to the original filename, and the ransom note uses a naming paradigm like README_<strong>ABCDE</strong>_FILES.txt or DECRYPT_<strong>ABCDE</strong>_DATA.txt</p>
<p><a href="https://news.sophos.com/wp-content/uploads/2019/12/ransnote.png"><img loading="lazy" decoding="async" class="alignnone size-full wp-image-62565" src="https://news.sophos.com/wp-content/uploads/2019/12/ransnote.png" alt="" width="640" height="225" srcset="https://news.sophos.com/wp-content/uploads/2019/12/ransnote.png 1146w, https://news.sophos.com/wp-content/uploads/2019/12/ransnote.png?resize=300,105 300w, https://news.sophos.com/wp-content/uploads/2019/12/ransnote.png?resize=768,269 768w, https://news.sophos.com/wp-content/uploads/2019/12/ransnote.png?resize=1024,359 1024w" sizes="(max-width: 640px) 100vw, 640px" /></a>The attackers were foiled in their attempts to infect machines protected by Sophos endpoint products with the ransomware payloads, or to kill the Sophos endpoint protection services and processes on machines that were attacked. But others were not so lucky. We reached out to <a href="https://www.coveware.com/" target="_blank" rel="noopener noreferrer">Coveware</a>, a company that specializes in extortion negotiations between ransomware victims and attackers. The company tells us they have negotiated with the Snatch criminals on 12 occasions between July and October on behalf of their clients. Ransom demands (in Bitcoin) ranged in value from $2,000 to $35,000, but trended up over that four month period.</p>

<p><a href="https://news.sophos.com/wp-content/uploads/2019/12/exclude-list-strings.png"><img loading="lazy" decoding="async" class="alignnone size-full wp-image-62567" src="https://news.sophos.com/wp-content/uploads/2019/12/exclude-list-strings.png" alt="" width="462" height="200" srcset="https://news.sophos.com/wp-content/uploads/2019/12/exclude-list-strings.png 462w, https://news.sophos.com/wp-content/uploads/2019/12/exclude-list-strings.png?resize=300,130 300w" sizes="(max-width: 462px) 100vw, 462px" /></a>The threat actors apparently actively monitor the systems running their agents, as we discovered while an analyst was manually running one of the samples from an instrumented, testbed computer.</p>
<p>When the analyst was logged out unexpectedly, the analyst suspected that the attackers identified the machine as a security research platform, so he wrote a message to the attackers and left it on the testbed&#8217;s desktop. Moments later, the attackers responded by logging the analyst out of the machine again and then blocking the IP address being used by the analyst from reconnecting to the Snatch C2 server.</p>
<p><a href="https://news.sophos.com/wp-content/uploads/2019/12/pgp-key.png"><img loading="lazy" decoding="async" class="alignnone size-full wp-image-62569" src="https://news.sophos.com/wp-content/uploads/2019/12/pgp-key.png" alt="" width="525" height="209" srcset="https://news.sophos.com/wp-content/uploads/2019/12/pgp-key.png 525w, https://news.sophos.com/wp-content/uploads/2019/12/pgp-key.png?resize=300,119 300w" sizes="(max-width: 525px) 100vw, 525px" /></a>We also discovered that the ransomware is using OpenPGP. The ransomware binaries we&#8217;ve inspected appear to have PGP Public Key blocks hardcoded into the files. (We&#8217;ve put those keys into <a href="https://github.com/sophoslabs/IoCs/blob/master/Ransomware-Snatch" target="_blank" rel="noopener noreferrer">the IoCs page for Snatch</a> on the SophosLabs Github.)</p>
<p>And the name &#8220;Snatch&#8221; doesn&#8217;t appear to be a coincidence. In earlier versions of the ransomware, the ransom note included an email address of &#8220;imBoristheBlade@protonmail.com&#8221; seems to be a reference to the Guy Ritchie movie Snatch (2000), in which a Rasputin-esque former-KGB agent character named Boris the Blade is beaten, shot, and stabbed throughout, often with little to no effect on his ability to get up and carry on fighting. Bullet Tooth Tony, the handle used by the message board poster, is another character that appears in the same movie.</p>
<h2>What can you do to prevent and detect a Snatch attack</h2>
<h3>Prevention</h3>
<ul>
<li>As we&#8217;ve been urging organizations to do for a while now, Sophos recommends that organizations of any size refrain from exposing the Remote Desktop interface to the unprotected internet. Organizations that wish to permit remote access to machines should put them behind a VPN on their network, so they cannot be reached by anyone who does not have VPN credentials.</li>
<li>The Snatch attackers also expressed interest in contracting with, or hiring, criminals who are capable of breaching networks using other types of remote access tools, such as VNC and TeamViewer, as well as those with experience using Web shells or breaking in to SQL servers using SQL injection techniques. It stands to reason that these types of internet-facing services also pose significant risks if left unattended.</li>
<li>Likewise, organizations should immediately implement multifactor authentication for users with administrative privileges, to make it more difficult for attackers to brute force those account credentials.</li>
<li>For Sophos customers, it is imperative that all users are running the most current endpoint protection, and enable the CryptoGuard feature within Intercept X.</li>
</ul>
<h3>Detection</h3>
<ul>
<li>The majority of initial access and footholds that we have observed are on unprotected and unmonitored devices. It&#8217;s extremely important for organizations of almost any size to perform regular and thorough inventory of devices, to ensure no gaps or &#8220;dark corners&#8221; exist on your network.</li>
<li>Execution of the Snatch ransomware occurred after threat actors had several days of undetected and uninhibited access to the network. A rigorous and mature threat hunting program would have greater potential to identify the threat actors prior to the execution of the ransomware executable.</li>
</ul>
<p><strong>Detection details</strong></p>
<p>Sophos products detect various components of Snatch and the files used in this attack under the following signatures:</p>
<p>Troj/Snatch-H<br />
Mal/Generic-R<br />
Troj/Agent-BCYI<br />
Troj/Agent-BCYN<br />
HPmal/GoRnSm-A<br />
HPmal/RansMaz-A<br />
PUA Detected: &#8216;PsExec&#8217;</p>
<p>Additional detection methods include anomalous behavior detection in MTR and Intercept X.</p>
<p>Indicators of compromise have been published <a href="https://github.com/sophoslabs/IoCs/blob/master/Ransomware-Snatch" target="_blank" rel="noopener noreferrer">on the SophosLabs Github</a> and will be updated as new indicators are discovered.</p>

				this.setSuccess(
					decodeURIComponent( 'You%20are%20already%20subscribed.%20Thank%20you%21' )
				);
			} else {
				this.setError(
					decodeURIComponent( 'We%20could%20not%20subscribe%20you.%20Please%20try%20again.' )
				);
			}
			})
			.catch((e) => {
			this.setError(
				decodeURIComponent( 'We%20could%20not%20subscribe%20you.%20Please%20try%20again.' )
			);
			})
			.finally(() => {
			this.reset();
			});
		},

		buttonText() {
		return this.loading ? decodeURIComponent( 'Please%20Wait' ) : decodeURIComponent( 'Subscribe' );
		},

		buttonDisabled() {
		return (
			this.subscribeTo.length < 1 ||
			!this.emailIsValid(this.email) ||
			this.loading
		);
		},
	};
	};
