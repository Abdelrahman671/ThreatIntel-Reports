	<h1 class="article-details__title">An Analysis of the BabLock (aka Rorschach) Ransomware</h1>
	<p class="article-details__description">This blog post analyzes a stealthy and expeditious ransomware called BabLock (aka Rorschach), which shares many characteristics with LockBit.</p>
	<p class="article-details__author-by">By: Don Ovid Ladores, Byron Gelera

            <div class="richText">
	
    


	
		<div>
			<p>A <a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/definition/ransomware">ransomware</a> called BabLock (aka Rorschach) has <a href="https://www.bleepingcomputer.com/news/security/new-rorschach-ransomware-is-the-fastest-encryptor-seen-so-far/">recently been making waves</a> due to its sophisticated and fast-moving attack chain that uses subtle yet effective techniques. Although primarily based on <a href="https://www.trendmicro.com/vinfo/us/security/news/ransomware-spotlight/ransomware-spotlight-lockbit">LockBit</a>, the ransomware is a hodgepodge of other different ransomware parts pieced together into what we now call BabLock (detected as Ransom.Win64.LOCKBIT.THGOGBB.enc). Note, however, that we do not believe that this ransomware originates from the threat actors behind LockBit, which is now in its third iteration.</p>
<p>In this blog entry, we look at its attack chain in detail and examine its likely origins.</p>
<h1><span class="body-subhead-title">Discovery</span></h1>
<p>In June 2022, we discovered a ransomware (which turned out to be BabLock) using what appeared to be a unique style of appending extensions, where instead of the normal “one sample, one extension” method commonly used in ransomware attacks, we discovered that the attackers were appending numerical increments from 00-99 on top of the fixed ransomware extension for this specific infection. As a result, even on a single infected machine, there could be multiple extension variations from a single execution.</p>
<p> </p>

		</div>
	

</div>
            <div class="image">
	
    


	<figure class="image-figure">
		<a id="42b37b" href="/content/dam/trendmicro/global/en/research/23/d/an-analysis-of-the-bablock-ransomware-/BabLock-1.png" target="_blank" rel="noopener noreferrer">
			<img src="/content/dam/trendmicro/global/en/research/23/d/an-analysis-of-the-bablock-ransomware-/BabLock-1.png" alt="Figure 1. The ransomware’s unique trait showing numerical increments for the extension"/>
		</a>
		<div class="caption-image-container ">
			<figcaption>Figure 1. The ransomware’s unique trait showing numerical increments for the extension</figcaption>
			
		</div>
	</figure>

</div>
            <div>




    
    
    <div class="richText">
	
    


	
		<div>
			<p>Our investigation found that the ransomware was always deployed as a multi-component package consisting mostly of the following files:</p>
<ul>
<li><span class="rte-red-bullet">The encrypted ransomware file, <i>config.ini</i></span></li>
<li><span class="rte-red-bullet">A malicious sideloaded DLL (DarkLoader, a <i>config.ini</i> decryptor and ransomware injector)</span></li>
<li><span class="rte-red-bullet">A non-malicious executable used to load the malicious DLL</span></li>
<li><span class="rte-red-bullet">A CMD file to execute the non-malicious binary using the correct password</span></li>
</ul>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		<a id="190017" href="/content/dam/trendmicro/global/en/research/23/d/an-analysis-of-the-bablock-ransomware-/BabLock-2.png" target="_blank" rel="noopener noreferrer">
			<img src="/content/dam/trendmicro/global/en/research/23/d/an-analysis-of-the-bablock-ransomware-/BabLock-2.png" alt="Figure 2. The main ransomware package found during one instance of infection"/>
		</a>
		<div class="caption-image-container ">
			<figcaption>Figure 2. The main ransomware package found during one instance of infection</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p>The DarkLoader DLL will check for specific commands, particularly<b> <i>--run</i>, </b>which checks for the correct 4-digit password needed to start the encryption process. Although it bears little significance to the unpacking of the contents of <i>config.ini </i>itself, the DLL will execute the fundamental ransomware routine if supplied correctly.</p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		<a id="47733b" href="/content/dam/trendmicro/global/en/research/23/d/an-analysis-of-the-bablock-ransomware-/BabLock-3.png" target="_blank" rel="noopener noreferrer">
			<img src="/content/dam/trendmicro/global/en/research/23/d/an-analysis-of-the-bablock-ransomware-/BabLock-3.png" alt="Figure 3. If the correct passcode is added to the command line, the ransomware will proceed with the whole encryption process"/>
		</a>
		<div class="caption-image-container ">
			<figcaption>Figure 3. If the correct passcode is added to the command line, the ransomware will proceed with the whole encryption process</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p>Once the DLL component is loaded by the non-malicious executable, it will immediately look for the <i>config.ini</i> file in the current executable’s path. Once this is found, the DLL decrypts <i>config.ini</i> and then executes <i>notepad.exe</i> with a certain set of command lines.</p>
<ul>
<li><span class="rte-red-bullet">For this particular campaign, we found a few notable and consistent patterns:</span></li>
<li><span class="rte-red-bullet">The main ransomware binary is usually delivered as an encrypted <i>config.ini</i> file.</span></li>
<li><span class="rte-red-bullet">DarkLoader is executed via DLL sideloading<b> </b>using legitimate executables.</span></li>
<li><span class="rte-red-bullet">The <i>config.ini</i> file is decrypted by a specially crafted loader designed specifically for these campaigns (detected as Trojan.Win64.DarkLoader)</span></li>
<li><span class="rte-red-bullet">BabLock appends a random number from 00 to 99 to the extension string per file within the same infected machine (for example, extn00-extn99 as extensions in the same infection).</span></li>
<li><span class="rte-red-bullet">Any DarkLoader DLL can be used to decrypt any encrypted ransomware <i>config.ini</i>, with no specific binary pairing needed.</span></li>
<li><span class="rte-red-bullet">The DarkLoader DLL uses Direct SysCall APIs to a select few, but important, calls to avoid API reading analysis.</span></li>
<li><span class="rte-red-bullet">The decrypted BabLock ransomware is always packed with VMProtect for anti-virtualization.</span></li>
<li><span class="rte-red-bullet">BabLock is loaded via the threat injection of a hooked API <i>Ntdll.RtlTestBit </i> to jump to memory containing the ransomware code.</span></li>
<li><span class="rte-red-bullet">There have been a few variations of the passcode for <i>--run</i> across different attacks, but all of them are still within a certain range of each other.</span></li>
</ul>

		</div>
	

</div>


    
    
    <div class="image">

			<img src="/content/dam/trendmicro/global/en/research/23/d/an-analysis-of-the-bablock-ransomware-/BabLock-4.png" alt="Figure 4. The command line argument supplied to notepad.exe to load and execute the ransomware on recent attacks."/>
		</a>
		<div class="caption-image-container ">

		</div>
	</figure>

</div>

			<img src="/content/dam/trendmicro/global/en/research/23/d/an-analysis-of-the-bablock-ransomware-/BabLock-5.png" alt="Figure 5. DLL using several direct SysCall instructions to avoid API reading techniques"/>

		<div class="caption-image-container ">

	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<h1><span class="body-subhead-title">Subtle but sophisticated</span></h1>
<p>Throughout our initial encounter with BabLock in June 2022, we searched for similar files and found that the earliest record of these files dated back to March 2022. After discovering this, we wanted to find out how it managed to stay under the radar for so long.                                                               </p>
<p>Since  June 2022, there have only been a handful of recorded incidents involving the ransomware, including the most recent one. Due to a low number count, no notable statistics involving region, industry, or victim profile have stood out as of the time of writing.<br />
</p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		<a id="a35432" href="/content/dam/trendmicro/global/en/research/23/d/an-analysis-of-the-bablock-ransomware-/BabLock-7.png" target="_blank" rel="noopener noreferrer">
			<img src="/content/dam/trendmicro/global/en/research/23/d/an-analysis-of-the-bablock-ransomware-/BabLock-7.png" alt="Figure 7. Distribution of incidents involving the BabLock ransomware"/>
		</a>
		<div class="caption-image-container ">
			<figcaption>Figure 7. Distribution of incidents involving the BabLock ransomware</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p>However, due to its notable features and characteristics, attacks related to BabLock can be easily identified. As we’ve already mentioned, after every file encryption, the ransomware appends a random number string between 00-99 to its hardcoded extension. This results in up to 100 different variations of the same ransomware extension.</p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		<a id="45f855" href="/content/dam/trendmicro/global/en/research/23/d/an-analysis-of-the-bablock-ransomware-/BabLock-8.png" target="_blank" rel="noopener noreferrer">
			<img src="/content/dam/trendmicro/global/en/research/23/d/an-analysis-of-the-bablock-ransomware-/BabLock-8.png" alt="Figure 8. Code snippet showing a random number string between 00-99 being appended to encrypted files"/>
		</a>
		<div class="caption-image-container ">
			<figcaption>Figure 8. Code snippet showing a random number string between 00-99 being appended to encrypted files</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p>It also has a fairly sophisticated execution routine:</p>
<ul>
<li><span class="rte-red-bullet">It uses a specific number code to execute properly.</span></li>
<li><span class="rte-red-bullet">It splits the package into multiple components.</span></li>
<li><span class="rte-red-bullet">It separates and hides the actual payload into an encrypted file.</span></li>
<li><span class="rte-red-bullet">It uses normal applications as loaders</span></li>
</ul>
<p>Finally, BabLock employs publicly available tools as part of its infection chain. We found that the most used tools were the following:</p>
<ul>
<li><span class="rte-red-bullet"><a href="https://github.com/jpillora/chisel">Chisel</a> - A transmission control protocol (TCP) and user datagram protocol (UDP) tunnel</span></li>
<li><span class="rte-red-bullet"><a href="https://github.com/shadow1ng/fscan">Fscan</a> - A scanning tool</span></li>
</ul>
<p>By using these two tools — combined with BabLock/LockBit possessing the capability to set active directory (AD) Group Policies for easier propagation — it’s possible for a malicious actor to navigate around a network without much effor</p>
<h1><span class="body-subhead-title">Comparing and contrasting BabLock to LockBit and other ransomware</span></h1>
<p>From our investigation, most of the routines used by BabLock are more closely related to Lockbit (2.0) than any other ransomware. Other researchers also mention <a href="https://www.computerweekly.com/news/365534897/Quick-acting-Rorschach-ransomware-appears-out-of-nowhere">similarities to ransomware such as Babuk, Yanluowang and others</a>.</p>
<p>Initially, we suspected it to be related to <a href="/en_us/research/21/e/what-we-know-about-darkside-ransomware-and-the-us-pipeline-attac.html">the DarkSide ransomware</a> due to ransom note similarities. However, unlike the DarkSide ransomware, BabLock removes shadow copies by executing the following command lines:</p>
<p><i><span class="blockquote">vssadmin.exe delete shadows /All /Quiet</span></i></p>
<p>Therefore, we immediately ruled this relationship out since it’s different to the way DarkSide does things, which is deleting shadow copies through Windows Management Instrumentation (WMI) and PowerShell (which is technically more sophisticated and difficult to detect through standard monitoring tools).</p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		<a id="6dcdf6" href="/content/dam/trendmicro/global/en/research/23/d/an-analysis-of-the-bablock-ransomware-/BabLock-9.png" target="_blank" rel="noopener noreferrer">
			<img src="/content/dam/trendmicro/global/en/research/23/d/an-analysis-of-the-bablock-ransomware-/BabLock-9.png" alt="Figure 9. The ransomware binary decrypts and executes the command line to delete shadow copies."/>
		</a>
		<div class="caption-image-container ">

		</div>
	</figure>

    <div class="richText">
	
    


	
		<div>
			<p>One of its common characteristics to <a href="https://chuongdong.com/reverse%20engineering/2022/03/19/LockbitRansomware/#checking-if-running-on-primary-domain-controller">Lockbit (2.0)</a> would be the use of the same group policy to generate a desktop drop path. Similarly, the use of vssadmin for deleting shadow copies is also a routine heavily used in LockBit attacks (albeit also a common routine for many modern ransomware). Still, the resemblance is uncanny. Furthermore, it is running the same commands to execute GPUpdate for the AD. Due to this, our detection for this ransomware is still under the LockBit family.</p>

		</div>
	

</div>

		<div>
			<p>From what we can tell, BabLock looks like a Frankenstein-like creation that is stitched together from different known ransomware families.</p>

		</div>
	

</div>

		<div>
			<h1><span class="body-subhead-title">Insights and conclusion</span></h1>
<p>Our first encounter with BabLock almost coincided with the <a href="/en_us/research/22/g/lockbit-ransomware-group-augments-its-latest-variant--lockbit-3-.html">release of Lockbit v3.0</a>. However, since most of its structure still resembles Lockbit v2.0, we surmise that this may be from another affiliate or group. With nearly a year since the release of LockBit v3.0, we have found no changes to the payload of the BabLock even with recent attacks, further solidifying our stance that they are neither connected nor closely affiliated with the actual LockBit group. What we do know is that the threat actor behind BabLock managed to take many of the base capabilities of LockBit v2.0 and added bits and pieces of different ransomware families to create their own unique variant, which could possibly be enhanced further in the future.</p>
<p><b><span class="body-subhead-title">Recommendations </span></b></p>
<p>Organizations can implement security frameworks to safeguard their systems from similar attacks, which systematically allocate resources to establish a robust defense strategy against ransomware. Below are some recommended guidelines that organizations may want to consider:</p>
<ul>
<li><span class="rte-red-bullet">Taking an inventory of assets and data </span></li>
<li><span class="rte-red-bullet">Identifying authorized and unauthorized devices and software </span></li>
<li><span class="rte-red-bullet">Auditing event and incident logs </span></li>
<li><span class="rte-red-bullet">Managing hardware and software configurations </span></li>
<li><span class="rte-red-bullet">Granting admin privileges and access only when necessary to an employee’s role </span></li>
<li><span class="rte-red-bullet">Monitoring network ports, protocols, and services </span></li>
<li><span class="rte-red-bullet">Establishing a software allowlist that only executes legitimate applications </span></li>
<li><span class="rte-red-bullet">Implementing data protection, backup, and recovery measures </span></li>
<li><span class="rte-red-bullet">Enabling multifactor authentication (MFA) </span></li>
<li><span class="rte-red-bullet">Deploying the latest versions of security solutions to all layers of the system, including email, endpoint, web, and network </span></li>
<li><span class="rte-red-bullet">Watching out for early signs of an attack such as the presence of suspicious tools in the system</span></li>
</ul>
<p>Implementing a multi-faceted approach can aid organizations in securing potential entry points into their systems such as endpoint, email, web, and network. With the help of security solutions that can identify malevolent elements and questionable activities, enterprises can be safeguarded from ransomware attacks.</p>
