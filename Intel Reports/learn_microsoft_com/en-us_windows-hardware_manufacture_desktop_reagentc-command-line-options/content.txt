											<h2 id="ms--in-this-article" class="title is-6 margin-block-xs">In this article</h2>
										</nav><!-- <content> --><p>You can use the REAgentC.exe tool to configure a Windows&nbsp;Recovery Environment (Windows&nbsp;RE) boot image and a push-button reset recovery image, and to administer recovery options and customizations. REAgentC comes with Windows, and you can run the <strong>REAgentC</strong> command on an offline Windows image or on a running Windows operating system.</p>
<div class="NOTE">
<p>Note</p>
<p>If you're using Windows PE 2.X, 3.X, or 4.X to configure recovery on an offline installation, you must use the Winrecfg.exe file from the Recovery folder of the Windows Assessment and Deployment Kit (Windows&nbsp;ADK). Winrecfg.exe supports only the offline operations that REAgentC.exe supports.</p>
</div>
<h2 id="reagentc-syntax">REAgentC syntax</h2>
<pre><code class="lang-cmd">reagentc.exe &lt;command&gt; &lt;arguments&gt;
</code></pre>
<h2 id="setreimage">/setreimage</h2>
<p>Sets the location of a Windows&nbsp;RE boot image.</p>
<p>Supports both online and offline operations.</p>
<h3 id="setreimage-syntax">/setreimage syntax</h3>
<pre><code class="lang-cmd">/setreimage /path &lt;path_to_Windows_RE_image&gt; [/target &lt;path_to_offline_image&gt;]
</code></pre>
<h3 id="setreimage-usage">/setreimage usage</h3>
<ul>
<li><p><code>/path</code> supports UNC paths to locations on the local disk. For example:</p>
<pre><code class="lang-cmd">Reagentc /setreimage /path S:\Recovery\WindowsRE`
</code></pre>
</li>
<li><p><code>/target</code> Specifies the location of the Windows image when you apply the setting offline. For example:</p>
<pre><code class="lang-cmd">Reagentc /setreimage /path T:\Recovery\WindowsRE /target W:\Windows
</code></pre>
</li>
</ul>
<h2 id="enable">/enable</h2>
<p>Enables a custom Windows RE boot image.</p>
<p>The <code>/enable</code> option runs automatically during the specialize configuration pass. If you don't specify a Windows RE boot image, the computer attempts to enable Windows RE by using the default Winre.wim file from the <code>\Windows\System32\Recovery</code> folder.</p>
<p>Supports both online and offline operations.</p>
<h3 id="enable-syntax">/enable syntax</h3>
<pre><code class="lang-cmd">/enable [/auditmode] [/osguid &lt;bcd_guid&gt;]
</code></pre>
<h3 id="enable-usage">/enable usage</h3>
<ul>
<li><p><code>/auditmode</code>: By default, the <code>/enable</code> option doesn't perform any actions when Windows is in audit mode. To override the default behavior and enable Windows RE from audit mode, specify the <code>/auditmode</code> option. For example:</p>
<pre><code class="lang-cmd">Reagentc /enable /auditmode
</code></pre>
<p>If you generalize the image after you use the <code>/enable</code> option in audit mode, Windows RE is disabled until you use the <code>/enable</code> option again or until after the specialize configuration pass runs.</p>
</li>
<li><p><code>/osguid {&lt;bcd_guid&gt;}</code> allows you to enable your custom Windows RE boot image from Windows PE. It can only be used after bcdboot.exe has been run. <code>&lt;bcd_guid&gt;</code> is the Boot Configuration Data (BCD) identifier of the target Windows installation, obtained by running <code>bcdedit -enum -v</code>.</p>
<pre><code class="lang-cmd">Reagentc /enable /osguid {00000000-0000-0000-0000-000000000000}
</code></pre>
</li>
</ul>
<h2 id="disable">/disable</h2>
<p>Disables any active Windows RE image that is mapped to the online image.</p>
<p>Supports only online operations.</p>
<h3 id="disable-usage">/disable usage</h3>
<pre><code class="lang-cmd">Reagentc /disable
</code></pre>
<h2 id="boottore">/boottore</h2>
<p>Specifies that Windows&nbsp;RE starts automatically the next time the system starts.</p>
<p>Supports only online operations.</p>
<h3 id="boottore-usage">/boottore usage</h3>
<pre><code class="lang-cmd">Reagentc /boottore
</code></pre>
<h2 id="setosimage">/setosimage</h2>
<blockquote>
<p>This setting isn't used in Windows&nbsp;10 or later</p>
</blockquote>
<p>Registers the location of a push-button reset image in an online or offline image. The recovery image must be in the Windows image (.wim) format.</p>
<p>Supports both online and offline operations.</p>
<h3 id="setosimage-syntax">/setosimage syntax</h3>
<pre><code class="lang-cmd">/setosimage /path &lt;path_to_recovery_image&gt; /index &lt;image_index&gt; [/target &lt;path_to_offline_image&gt;]
</code></pre>
<h3 id="setosimage-usage">/setosimage usage</h3>
<ul>
<li><p>The <code>/index</code> option specifies the index number of the recovery image to use from within a .wim file. For example:</p>
<pre><code class="lang-cmd">Reagentc /setosimage /path R:\RecoveryImage /index 1
</code></pre>
</li>
<li><p>The <code>/target</code> option specifies the location of the offline Windows image. For example:</p>
<pre><code class="lang-cmd">Reagentc /setosimage /path R:\RecoveryImage /index 1 /target W:\Windows
</code></pre>
</li>
</ul>
<h2 id="info">/info</h2>
<p>Displays the current status of Windows RE and any available recovery image on an online or offline image.</p>
<p>Supports both online and offline operations.</p>
<h3 id="info-syntax">/info syntax</h3>
<pre><code class="lang-cmd">/info [/target &lt;path_to_offline_image&gt;]
</code></pre>
<h3 id="info-usage">/info usage</h3>
<ul>
<li><p><code>/info</code> on its own returns the status of the online operating system:</p>
<pre><code class="lang-cmd">Reagentc /info
</code></pre>
</li>
<li><p>The <code>/target</code> option obtains configuration information about an offline image. For example:</p>
<pre><code class="lang-cmd">Reagentc /info /target W:\Windows
</code></pre>
</li>
</ul>
<h2 id="setbootshelllink">/setbootshelllink</h2>
<p>Registers the link to a custom tool that appears in the Windows Advanced startup menu.</p>
<p>Supports both online and offline operations.</p>
<h3 id="setbootshelllink-syntax">/setbootshelllink syntax</h3>
<pre><code class="lang-cmd">/setbootshelllink [/configfile &lt;path_to_BootShellXML&gt;] [/target &lt;path_to_offline_image&gt;]
</code></pre>
<h3 id="setbootshelllink-usage">/setbootshelllink usage</h3>
<ul>
<li><p>The BootShellXML file is an.xml file that contains the <code>&lt;BootShell&gt;</code> element and the <code>Name</code> and <code>Description</code> attributes that you want to appear in the link. For more information, see <a href="customize-windows-re?view=windows-11" data-linktype="relative-path">Customize Windows RE</a>:</p>
<pre><code class="lang-cmd">Reagentc /setbootshelllink /configfile F:\BootMenu\AddDiagnosticsToolToBootMenu.xml
</code></pre>
</li>
<li><p>The <code>/target</code> option specifies the location of the offline Windows image. If this argument isn't used, the running operating system is used. For example:</p>
<pre><code class="lang-cmd">Reagentc /setbootshelllink /target W:\Windows
</code></pre>
</li>
</ul>
