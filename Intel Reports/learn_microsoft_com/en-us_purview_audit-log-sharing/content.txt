<h2 id="the-sharepoint-sharing-schema">The SharePoint Sharing schema</h2>
<p>Sharing events (not including events related to sharing policy and sharing links) are different from file- and folder-related events in one primary way: one user is performing an action that has an effect on another user. For example, when a resource User A gives User B access to a file. In this example, User A is the <em>acting user</em> and User B is the<em>target user</em>. In the SharePoint File schema, the acting user's action only affects the file itself. When User A opens a file, the only information needed in the <strong>FileAccessed</strong> event is the acting user. To address this difference, there's a separate schema, called the <em>SharePoint Sharing schema</em> that captures more information about sharing events. This ensures that administrators have visibility into who shared a resource and the user the resource was shared with.</p>
<p>The Sharing schema provides two additional fields in an audit record related to sharing events:</p>
<ul>
<li><strong>TargetUserOrGroupType:</strong> Identifies whether the target user or group is a Member, Guest, SharePointGroup, SecurityGroup, or Partner.</li>
<li><strong>TargetUserOrGroupName:</strong> Stores the UPN or name of the target user or group that a resource was shared with (User B in the previous example).</li>
</ul>
<p>These two fields, in addition to other properties from the audit log schema such as User, Operation, and Date can tell the full story about <em>which</em> user shared <em>what</em> resource with <em>whom</em> and <em>when</em>.</p>
<p>There's another schema property that's important to the sharing story. When you export audit log search results, the <strong>AuditData</strong> column in the exported CSV file stores information about sharing events. For example, when a user shares a site with another user, this is accomplished by adding the target user to a SharePoint group. The <strong>AuditData</strong> column captures this information to provide context for administrators. See <a href="#step-2-use-the-powerquery-editor-to-format-the-exported-audit-log" data-linktype="self-bookmark">Step 2</a> for instructions on how to parse the information in the <strong>AuditData</strong> column.</p>
<h2 id="sharepoint-sharing-events">SharePoint sharing events</h2>
<p>Sharing is defined by when a user (the <em>acting</em> user) wants to share a resource with another user (the <em>target</em> user). Audit records related to sharing a resource with an external user (a user who is outside of your organization and doesn't have a guest account in your organization's Microsoft Entra ID) are identified by the following events, which are logged in the audit log:</p>
<ul>
<li><strong>SharingInvitationCreated:</strong> A user in your organization tried to share a resource (likely a site) with an external user. This results in an external sharing invitation sent to the target user. No access to the resource is granted at this point.</li>
<li><strong>SharingInvitationAccepted:</strong> The external user has accepted the sharing invitation sent by the acting user and now has access to the resource.</li>
<li><strong>AnonymousLinkCreated:</strong> An anonymous link (also called an "Anyone" link) is created for a resource. Because an anonymous link can be created and then copied, it's reasonable to assume that any document that has an anonymous link has been shared with a target user.</li>
<li><strong>AnonymousLinkUsed:</strong> As the name implies, this event is logged when an anonymous link is used to access a resource.</li>
<li><strong>SecureLinkCreated:</strong> A user has created a "specific people link" to share a resource with a specific person. This target user may be someone who is external to your organization. The person that the resource is shared with is identified in the audit record for the <strong>AddedToSecureLink</strong> event. The time stamps for these two events are nearly identical.</li>
<li><strong>AddedToSecureLink:</strong> A user was added to a specific people link. Use the <strong>TargetUserOrGroupName</strong> field in this event to identify the user added to the corresponding specific people link. This target user may be someone who is external to your organization.</li>
</ul>
<h2 id="sharing-auditing-work-flow">Sharing auditing work flow</h2>
<p>When a user (the acting user) wants to share a resource with another user (the target user), SharePoint (or OneDrive for Business) first checks if the email address of the target user is already associated with a user account in the organization's directory. If the target user is in the directory (and has a corresponding guest user account), SharePoint does the following things:</p>
<ul>
<li>Immediately assigns the target user permissions to access the resource by adding the target user to the appropriate SharePoint group, and logs an <strong>AddedToGroup</strong> event.</li>
<li>Sends a sharing notification to the email address of the target user.</li>
<li>Logs a <strong>SharingSet</strong> event. This event has a friendly name of "Shared file, folder, or site" under <strong>Sharing and access request activities</strong> in the activities picker of the audit log search tool. See the screenshot in <a href="#step-1-search-for-sharing-events-and-export-the-results-to-a-csv-file" data-linktype="self-bookmark">Step 1</a>.</li>
</ul>
<p>If a user account for the target user isn't in the directory, SharePoint does the following:</p>
<ul>
<li><p>Logs one of the following events, based on how the resource is shared:</p>
<ul>
<li><strong>AnonymousLinkCreated</strong></li>
<li><strong>SecureLinkCreated</strong></li>
<li><strong>AddedToSecureLink</strong></li>
<li><strong>SharingInvitationCreated</strong> (this event is logged only when the shared resource is a site)</li>
</ul>
</li>
<li><p>When the target user accepts the sharing invitation that's sent to them (by clicking the link in the invitation), SharePoint logs a <strong>SharingInvitationAccepted</strong> event and assigns the target user permissions to access the resource. If the target user is sent an anonymous link, the <strong>AnonymousLinkUsed</strong> event is logged after the target user uses the link to access the resource. For secure links, a <strong>FileAccessed</strong> event is logged when an external user uses the link to access the resource.</p>
</li>
</ul>
<p>Additional information about the target user is also logged, such as the identity of the user the invitation is to and the user who accepts the invitation. In some case, these users (or email addresses) can be different.</p>
<h2 id="how-to-identify-resources-shared-with-external-users">How to identify resources shared with external users</h2>
<p>A common requirement for administrators is creating a list of all resources that have been shared with users outside of the organization. By using sharing auditing in Office 365, administrators can generate this list. Here's how.</p>
<h3 id="step-1-search-for-sharing-events-and-export-the-results-to-a-csv-file">Step 1: Search for sharing events and export the results to a CSV file</h3>
<p>The first step is to search the audit log for sharing events. For more information (including the required permissions) about searching the audit log, see <a href="audit-search" data-linktype="relative-path">Search the audit log</a>.</p>
<p>Select the appropriate tab for the portal you're using. To learn more about the Microsoft Purview portal, see <a href="purview-portal" data-linktype="relative-path">Microsoft Purview portal</a>. To learn more about the Compliance portal, see <a href="purview-compliance-portal" data-linktype="relative-path">Microsoft Purview compliance portal</a>.</p>

<li><p>Select the <strong>Audit</strong> solution card. If the <strong>Audit</strong> solution card isn't displayed, select <strong>View all solutions</strong> and then select <strong>Audit</strong> from the <strong>Core</strong> section.</p>

<h3 id="step-2-use-the-powerquery-editor-to-format-the-exported-audit-log">Step 2: Use the PowerQuery Editor to format the exported audit log</h3>
<p>The next step is to use the JSON transform feature in the Power Query Editor in Excel to split each property in the <strong>AuditData</strong> column (which consists of a multi-property JSON object) into its own column. This lets you filter columns to view records related to sharing.</p>
<p>For step-by-step instructions, see "Step 2: Format the exported audit log using the Power Query Editor" in <a href="audit-log-export-records#step-2-format-the-exported-audit-log-using-the-power-query-editor" data-linktype="relative-path">Export, configure, and view audit log records</a>.</p>
<h3 id="step-3-filter-the-csv-file-for-resources-shared-with-external-users">Step 3: Filter the CSV file for resources shared with external users</h3>
<p>The next step is to filter the CSV for the different sharing-related events that were previously described in the <a href="#sharepoint-sharing-events" data-linktype="self-bookmark">SharePoint sharing events</a> section. Alternatively, you can filter the <strong>TargetUserOrGroupType</strong> column to display all records where the value of this property is <strong>Guest</strong>.</p>
<p>After you've followed the instructions in the previous step to prepare the CSV file by using the PowerQuery editor, do the following:</p>
<ol>
<li><p>Open the Excel file that you created in Step 2.</p>
</li>
<li><p>On the <strong>Home</strong> tab, select <strong>Sort &amp; Filter</strong>, and then select <strong>Filter</strong>.</p>
</li>
<li><p>In the <strong>Sort &amp; Filter</strong> dropdown list on the <strong>Operations</strong> column, clear all selections, then select one or more the following sharing-related events and then select <strong>Ok</strong>.</p>
<ul>
<li><strong>SharingInvitationCreated</strong></li>
<li><strong>AnonymousLinkCreated</strong></li>
<li><strong>SecureLinkCreated</strong></li>
<li><strong>AddedToSecureLink</strong></li>
</ul>
<p>Excel displays the rows for the events you selected.</p>
</li>
<li><p>Go to the column named <strong>TargetUserOrGroupType</strong> and select it.</p>
</li>
<li><p>In the <strong>Sort &amp; Filter</strong> dropdown list, clear all selections, then select <strong>TargetUserOrGroupType:Guest</strong>, and select <strong>Ok</strong>.</p>
<p>Now Excel displays the rows for sharing events AND where the target user is outside of your organization, because external users are identified by the value <strong>TargetUserOrGroupType:Guest</strong>.</p>
</li>
</ol>
