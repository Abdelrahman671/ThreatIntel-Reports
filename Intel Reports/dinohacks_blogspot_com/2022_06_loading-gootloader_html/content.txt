<p><span style="font-family: helvetica; font-size: medium;">&nbsp;</span></p><p><b><span style="color: #ffa400; font-family: helvetica; font-size: medium;"><i><span>Disclaimer:&nbsp;</span>Opinions expressed are solely my own. N<span>one of the ideas expressed in this blog post are shared, supported, or endorsed in any manner by my employer.</span></i></span></b></p><p><span style="font-family: helvetica; font-size: medium;">In this blog, I will be taking a look at the initial GootLoader sample (MD5:&nbsp;4dd369b5e028beebe3aa5c980960c502 , Sha256:&nbsp;c1029f0b5f4f6dfbe0fe656f075cbb5ccc2fc308087db21438d73394b75ea020).&nbsp;</span></p><p><span style="font-family: helvetica; font-size: medium;"><span>Available&nbsp;</span><span>here:&nbsp;<a href="https://bazaar.abuse.ch/sample/c1029f0b5f4f6dfbe0fe656f075cbb5ccc2fc308087db21438d73394b75ea020/">https://bazaar.abuse.ch/sample/c1029f0b5f4f6dfbe0fe656f075cbb5ccc2fc308087db21438d73394b75ea020/</a>&nbsp;</span></span></p><p><span style="font-family: helvetica; font-size: medium;">The sample is a javascript program which is meant to be executed in windows using wscript.exe.&nbsp;</span></p><p><span style="font-family: helvetica; font-size: medium;">Opening the sample in a text editor, it appears to look like jquery library 3.6.0 (screenshot below).</span></p><p><span style="font-family: helvetica; font-size: medium;"><br /></span></p><table align="center" cellpadding="0" cellspacing="0" class="tr-caption-container" style="margin-left: auto; margin-right: auto;"><tbody><tr><td style="text-align: center;"><a href="https://blogger.googleusercontent.com/img/a/AVvXsEhSukJE49nYcGi1c0e2LK-d7hldx7bDiTMPXoX0v0OZ5fqsy1QUadpQgDTRX6H8OihAoZXcgHmkt9xHGOoqozwv0K-MBYHlehqrkWzChhDbG5FHenMgi4uSbX2NpJdispPsaXC4OcY0DSU4jWzw5-VRiYKGe2GZAr2tOhPGbbwLum-VFL6R_-7XW5NWFw" style="margin-left: auto; margin-right: auto;"><span style="font-family: helvetica; font-size: medium;"><img data-original-height="1462" data-original-width="2744" src="https://blogger.googleusercontent.com/img/a/AVvXsEhSukJE49nYcGi1c0e2LK-d7hldx7bDiTMPXoX0v0OZ5fqsy1QUadpQgDTRX6H8OihAoZXcgHmkt9xHGOoqozwv0K-MBYHlehqrkWzChhDbG5FHenMgi4uSbX2NpJdispPsaXC4OcY0DSU4jWzw5-VRiYKGe2GZAr2tOhPGbbwLum-VFL6R_-7XW5NWFw=s16000" title="Fig 1. Sample in text editor." /></span></a></td></tr><tr><td class="tr-caption" style="text-align: center;"></td></tr></tbody></table><span style="font-family: helvetica; font-size: medium;">Comparing the sample with jqeury 3.6.0 downloaded from&nbsp;<a href="https://code.jquery.com/jquery-3.6.0.js,">https://code.jquery.com/jquery-3.6.0.js,</a> differences can be observed.&nbsp;<div><br /></div><div>The following lines shows the code inserted to create the sample:&nbsp;<p></p><script src="https://gist.github.com/nhegde610/e8ebe66f68fa50d14fc88af9f0f04eeb.js"></script><p><br /></p><p>One important thing to mention about javascript: If a variable is declared without keywords such as var, it will be treated as global variable.&nbsp;</p><p>After reading the code, following points can be observed:&nbsp;</p><p></p><ul style="text-align: left;"><li><u>Function seat07()</u> is incrementing a variable in a loop. The incremented variable is not referred in any part of the code. This is most likely done to increase the execution time and thus timeout sandbox detection. Commenting out the function would speed up the execution.</li><li><u>Function gone068()</u> contains the obfuscated string which is likely deobfuscated and executed. Generally, deobfuscated string would require one of the following to be executed:</li><ul><li>eval()</li><li>function constructor&nbsp;</li><li>window object&nbsp;&nbsp;</li></ul><li>It would be interesting to see how deobfuscated string is executed in this sample.</li><li><u>Function law1()</u> and <u>present4()</u> are quite interesting because they are accessing array elements and making a call. They could be used to call deobfuscating routine and execute the deobfuscated string.&nbsp;</li></ul><div><br /></div><div>After adding breakpoints and running the program in a javascript debugger, first layer of the GootLoader becomes clear:&nbsp;</div><div>&nbsp;</div></div><script src="https://gist.github.com/nhegde610/00ef111b52ae2c502944ee573b3f2677.js"></script><div><br /></div><div>Following points can be observed:</div><div><ul style="text-align: left;"><li>Deobfuscated strings are executed using function constructor.</li><li>It contains obfuscated string which needs to be deobfuscated.&nbsp;</li><li>It also seems to be accessing the registry: HKEY_CURRENT_USER&nbsp;&nbsp;</li></ul></div><div><br /></div><div>Deobfuscating the string in first layer reveals the second layer of GootLoader:&nbsp;</div><script src="https://gist.github.com/nhegde610/33b26baf7380c53c1b021698ba53b532.js"></script><div>&nbsp;</div><div><br /></div><div><br /></div><div>The following points can be noted:&nbsp;</div><div><ul style="text-align: left;"><li>It checks if the environment variable userdnsdomain is set which would indicate that the system is part of AD.&nbsp;</li><li>If active directory is set, then it would add the string "4173581" to parameter value in its requests.</li><li>The C2 communication would look something like this:</li><ul><li>GET request to server with path: test.php?zgntopmtyplqx=</li><li>The parameter will be &lt;random value&gt;4173581 if computer is part of AD otherwise only &lt;random value&gt;&nbsp;</li><li>If the http response status code is anything other than 200, it would sleep for 12345 milliseconds and check for another domain.&nbsp;</li><li>If the http response status code is 200, then it would look for the string :&nbsp; "@&lt;parameter value sent&gt;@" in the response text.&nbsp;</li><ul><li>if it is not present, then it will sleep for 23232 milliseconds.&nbsp;</li><li>If it is present, then it will do the following:</li><ul><li>Remove the string "@&lt;parameter value sent&gt;@" in the response text.&nbsp;</li><li>Response text is likely to be series of numbers which are parsed as integers then converted to ascii strings which are then deobfuscated.&nbsp;</li></ul></ul></ul></ul><div>If you are interested to read about this campaign, you can read the following blogs:</div><div><ul style="text-align: left;"><li><a href="https://news.sophos.com/en-us/2021/03/01/gootloader-expands-its-payload-delivery-options/">https://news.sophos.com/en-us/2021/03/01/gootloader-expands-its-payload-delivery-options/</a>&nbsp;</li><li><a href="https://thedfirreport.com/2022/05/09/seo-poisoning-a-gootloader-story/">https://thedfirreport.com/2022/05/09/seo-poisoning-a-gootloader-story/</a></li></ul><div><br /></div></div><div>You can also follow the twitter account:&nbsp;<a href="https://twitter.com/GootLoaderSites">https://twitter.com/GootLoaderSites</a> to get updates about the C2 server used by this campaign.&nbsp;</div><div><br /></div><div>In this blog, I used a debugger to deobfuscate the sample. There is a decoder script for this campaign available at:&nbsp;<a href="https://github.com/hpthreatresearch/tools/blob/main/gootloader/decode.py">https://github.com/hpthreatresearch/tools/blob/main/gootloader/decode.py</a>&nbsp;</div><div><br /></div><div>You can also read the blog that explains how this decoder script was written:&nbsp;<a href="https://threatresearch.ext.hp.com/tips-for-automating-ioc-extraction-from-gootloader-a-changing-javascript-malware/">https://threatresearch.ext.hp.com/tips-for-automating-ioc-extraction-from-gootloader-a-changing-javascript-malware/</a></div><div><br /></div><div>Have a nice Day!&nbsp;</div></div></span>
