<p>
<p>We came across a <a href="https://twitter.com/malwrhunterteam/status/1428031391728537602" target="_blank" rel="noreferrer noopener">tweet</a> about a lock screen app which installs ransomware by faking itself as a legitimate app, so that users are tricked into falling prey. This approach by the threat actor was quite interesting considering the fact that there were no ransom demands and the key to unlock the device has been hardcoded in the app.&nbsp; In this blog, we will be explaining the technical aspects of the app which was developed by an attacker<strong> </strong>to lock the screen of a user’s device.&nbsp;</p>



<h3><strong>Technical Details</strong></h3>



<p>Once the app “gbwhatsapp.apk“ ( Hash: 70273ee146260bafb1cc136a0249e2a2 ) is installed as shown in Figure 1, it seems to execute a shell command in its class ADRTLogCatReader. In the code shown in Figure 2, we can see it uses the command <strong>“logcat -v threadtime”</strong>.&nbsp; The shell command <strong>adb logcat -v threadtime</strong> displays the date, invocation time, priority, tag, and the PID and TID of the thread issuing the message.&nbsp;</p>



<div class="wp-block-image"><figure class="aligncenter size-large is-resized"><img loading="lazy" src="https://labs.k7computing.com/wp-content/uploads/2021/09/Figure1.jpg" alt="" class="wp-image-22646" width="391" height="560" srcset="https://labs.k7computing.com/wp-content/uploads/2021/09/Figure1.jpg 391w, https://labs.k7computing.com/wp-content/uploads/2021/09/Figure1-209x300.jpg 209w" sizes="(max-width: 391px) 100vw, 391px" /><figcaption><em>Figure 1:&nbsp; “gbwhatsapp.apk“ installed on the device</em></figcaption></figure></div>



<div class="wp-block-image"><figure class="aligncenter size-large is-resized"><img loading="lazy" src="https://labs.k7computing.com/wp-content/uploads/2021/09/Figure2-1024x252.jpg" alt="" class="wp-image-22647" width="1024" height="252" srcset="https://labs.k7computing.com/wp-content/uploads/2021/09/Figure2-1024x252.jpg 1024w, https://labs.k7computing.com/wp-content/uploads/2021/09/Figure2-300x74.jpg 300w, https://labs.k7computing.com/wp-content/uploads/2021/09/Figure2-768x189.jpg 768w, https://labs.k7computing.com/wp-content/uploads/2021/09/Figure2.jpg 1171w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption><em>Figure 2: Code using shell command</em></figcaption></figure></div>



<p>In the main activity, it looks like this app was developed on <strong>AIDE “com.aide.ui”</strong>, a tool to develop&nbsp;Android apps directly on an Android device as shown in Figure 3, and the main activity starts a&nbsp;service named <strong>MyService</strong>.&nbsp;</p>



<div class="wp-block-image"><figure class="aligncenter size-large is-resized"><img loading="lazy" src="https://labs.k7computing.com/wp-content/uploads/2021/09/Figure3-1024x191.jpg" alt="" class="wp-image-22649" width="1024" height="191" srcset="https://labs.k7computing.com/wp-content/uploads/2021/09/Figure3-1024x191.jpg 1024w, https://labs.k7computing.com/wp-content/uploads/2021/09/Figure3-300x56.jpg 300w, https://labs.k7computing.com/wp-content/uploads/2021/09/Figure3-768x144.jpg 768w, https://labs.k7computing.com/wp-content/uploads/2021/09/Figure3.jpg 1284w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption><em>Figure 3: ADRTLogCatReader&nbsp;communicates with com.aide.ui for debugging</em></figcaption></figure></div>



<p><strong>MyService</strong>&nbsp;invokes the malicious behaviour of the app. The idea behind this is displaying a lock screen window which cannot be exited until the correct key is entered. WindowManager displays a window on top of the screen and makes it persist using the function Toast.makeTest().show() as shown in Figure 4.</p>



<div class="wp-block-image"><figure class="aligncenter size-large is-resized"><img loading="lazy" src="https://labs.k7computing.com/wp-content/uploads/2021/09/Figure4-1024x227.jpg" alt="" class="wp-image-22651" width="1024" height="227" srcset="https://labs.k7computing.com/wp-content/uploads/2021/09/Figure4-1024x227.jpg 1024w, https://labs.k7computing.com/wp-content/uploads/2021/09/Figure4-300x67.jpg 300w, https://labs.k7computing.com/wp-content/uploads/2021/09/Figure4-768x170.jpg 768w, https://labs.k7computing.com/wp-content/uploads/2021/09/Figure4.jpg 1254w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption><em>Figure 4: Code to display lock screen&nbsp;</em></figcaption></figure></div>



<p>The lock screen consists of a note which can be seen from resource field “<strong>ransomware virus sistem lock</strong>” and “<strong>you sistem already lock dont any restart or uninstall thisfile cant damage and default virus for information call ask (Variants19crew)&#8230;.</strong>” as shown in Figure 5 and Figure 6.</p>



<div class="wp-block-image"><figure class="aligncenter size-large is-resized"><img loading="lazy" src="https://labs.k7computing.com/wp-content/uploads/2021/09/Figure5-1024x144.jpg" alt="" class="wp-image-22652" width="1024" height="144" srcset="https://labs.k7computing.com/wp-content/uploads/2021/09/Figure5-1024x144.jpg 1024w, https://labs.k7computing.com/wp-content/uploads/2021/09/Figure5-300x42.jpg 300w, https://labs.k7computing.com/wp-content/uploads/2021/09/Figure5-768x108.jpg 768w, https://labs.k7computing.com/wp-content/uploads/2021/09/Figure5.jpg 1298w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption><em>Figure 5: Code containing note to be displayed</em></figcaption></figure></div>



<div class="wp-block-image"><figure class="aligncenter size-large is-resized"><img loading="lazy" src="https://labs.k7computing.com/wp-content/uploads/2021/09/Figure6.jpg" alt="" class="wp-image-22653" width="396" height="666" srcset="https://labs.k7computing.com/wp-content/uploads/2021/09/Figure6.jpg 396w, https://labs.k7computing.com/wp-content/uploads/2021/09/Figure6-178x300.jpg 178w" sizes="(max-width: 396px) 100vw, 396px" /><figcaption><em>Figure 6: Lock screen display</em></figcaption></figure></div>



<p>Fortunately, there were no ransom demands and the key to unlock the device has also been hardcoded which can be used to get rid of this window as shown in Figure 7. When the user unlocks the device by entering the string <strong>“anonymous86”</strong>, it will kill the app.</p>



<div class="wp-block-image"><figure class="aligncenter size-large is-resized"><img loading="lazy" src="https://labs.k7computing.com/wp-content/uploads/2021/09/Figure7-1024x105.jpg" alt="" class="wp-image-22654" width="1024" height="105" srcset="https://labs.k7computing.com/wp-content/uploads/2021/09/Figure7-1024x105.jpg 1024w, https://labs.k7computing.com/wp-content/uploads/2021/09/Figure7-300x31.jpg 300w, https://labs.k7computing.com/wp-content/uploads/2021/09/Figure7-768x79.jpg 768w, https://labs.k7computing.com/wp-content/uploads/2021/09/Figure7.jpg 1428w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption><em>Figure 7: Code for </em>gbwhatsapp.apk<em> to unlock the device</em></figcaption></figure></div>



<p>There are scenarios where the malware author demands a ransom for the app “instagramgold.apk” as shown in Figure 8. The lock screen consists of a note which can also be seen from resource field “<strong>POOLS CLOSED</strong>” and “<strong>As you can see your phone has been hacked and with that, all of you information, right now as we speak is being uploaded to a magical place for exploitation. However this can be avoided. we&#8217;d like $1000.00 send to this bitcoin address bc1qj7a82wmtmm8a4vt2z93jxn5cemfjztnqffn0e9 within 24hrs or life gets worse. Cheerio</strong>”</p>



<div class="wp-block-image"><figure class="aligncenter size-large is-resized"><img loading="lazy" src="https://labs.k7computing.com/wp-content/uploads/2021/09/Figure8.jpg" alt="" class="wp-image-22655" width="398" height="663" srcset="https://labs.k7computing.com/wp-content/uploads/2021/09/Figure8.jpg 398w, https://labs.k7computing.com/wp-content/uploads/2021/09/Figure8-180x300.jpg 180w" sizes="(max-width: 398px) 100vw, 398px" /><figcaption><em>Figure 8: Lock screen demanding ransom</em></figcaption></figure></div>



<p>Fortunately, the key to get rid of this lock screen window is hardcoded. The device can be unlocked by entering the string “17317071” as shown in Figure 9 which kills the app.</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" width="1024" height="138" src="https://labs.k7computing.com/wp-content/uploads/2021/09/Figure9-1024x138.jpg" alt="" class="wp-image-22656" srcset="https://labs.k7computing.com/wp-content/uploads/2021/09/Figure9-1024x138.jpg 1024w, https://labs.k7computing.com/wp-content/uploads/2021/09/Figure9-300x41.jpg 300w, https://labs.k7computing.com/wp-content/uploads/2021/09/Figure9-768x104.jpg 768w, https://labs.k7computing.com/wp-content/uploads/2021/09/Figure9.jpg 1265w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption><em>Figure 9: Code for </em>instagramgold.apk<em> to unlock the device</em></figcaption></figure></div>



<p>This malware author is spreading a fake lock screen app resembling legitimate apps such as WhatsApp, GitHub, Netflix, Instagram Gold, Spotify, etc. However, while some of the apps of this type demand a ransom to unlock the device’s screen, a few others don’t. That said, we at K7 Labs are constantly monitoring such campaigns. We recommend users to avoid believing in such third party apps and break the chain in spreading these fake apps.</p>
