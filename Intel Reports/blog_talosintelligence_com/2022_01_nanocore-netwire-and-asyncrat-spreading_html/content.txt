    <h1 class="text-center">Nanocore, Netwire and AsyncRAT spreading campaign uses public cloud infrastructure</h1>

    <div class="text-center m-1">
        <div class="post-author">
            <span>By </span>
                <a href="https://blog.talosintelligence.com/author/chetan-raghuprasad/">Chetan Raghuprasad</a>, 
                <a href="https://blog.talosintelligence.com/author/vanja-svajcer/">Vanja Svajcer</a>
        </div>

        <br/>
        <time class="post-datetime" datetime="January 12, 2022 08:02">
            Wednesday, January 12, 2022 08:02
        </time>

        <div class="m-3">
                            <a href="/category/threat-spotlight/" class="category primary-category">
                                Threat Spotlight
                            </a>
                                                    <a href="/category/rat/" class="category primary-category">
                                RAT
                            </a>
                                </div>
    </div>

    <section class="post-content-wrapper mt-5">
        <div class="post-content">
            <ul><li>Cisco Talos discovered a malicious campaign in October 2021 delivering variants of Nanocore, Netwire and AsyncRATs targeting user's information.</li><li>According to Cisco Secure product telemetry, the victims of this campaign are primarily distributed across the United States, Italy and Singapore.</li><li>The actor used complex obfuscation techniques in the downloader script. Each stage of the deobfuscation process results with the decryption methods for the subsequent stages to finally arrive at the actual malicious downloader method.</li><li>The campaign is the latest example of threat actors abusing cloud services like Microsoft Azure and Amazon Web Services and are actively misusing them to achieve their malicious objectives.</li><li>The actor is using the DuckDNS dynamic DNS service to change domain names of the C2 hosts.</li></ul><h3 id="executive-summary">Executive Summary</h3><p>Threat actors are increasingly using cloud technologies to achieve their objectives without having to resort to hosting their own infrastructure. These types of cloud services like Azure and AWS allow attackers to set up their infrastructure and connect to the internet with minimal time or monetary commitments. It also makes it more difficult for defenders to track down the attackers' operations.</p><p>The threat actor in this case used cloud services to deploy and deliver variants of commodity RATs with the information stealing capability starting around Oct. 26, 2021. These variants of Remote Administration Tools (RATs) are packed with multiple features to take control over the victim's environment to execute arbitrary commands remotely and steal the victim's information.</p><p>The initial infection vector is a phishing email with a malicious ZIP attachment. These ZIP archive files contain an ISO image with a malicious loader in the form of JavaScript, a Windows batch file or Visual Basic script. When the initial script is executed on the victim's machine, it connects to a download server to download the next stage, which can be hosted on an Azure Cloud-based Windows server or an AWS EC2 instance.</p><p>To deliver the malware payload, the actor registered several malicious subdomains using DuckDNS, a free dynamic DNS service. The malware families associated with this campaign are variants of the Netwire, Nanocore and AsyncRAT remote access trojans.</p><p>Organizations should be inspecting outgoing connections to cloud computing services for malicious traffic. The campaigns described in this post demonstrate increasing usage of popular cloud platforms for hosting malicious infrastructure.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/img/a/AVvXsEi7xhO2xsatFHYVEgVnEvX1ke9YXqLub1hLgaZlEPW-52zT0KDH60oKDB19_WoEvtFLTVOWIWj3qwDNyGD3tGHaYv61u9Mqpjy9vJwgj2jTS--1pV5Ig4I5nAawApDLw054Dc93oIxvofb2lV41vgyjGA0JTWYV_HCozsetsIQbwslIrBCJpgXNl5RxLA.jpeg" class="kg-image" alt loading="lazy"><figcaption>Infection summary diagram.</figcaption></figure><h3 id="the-payload">The Payload</h3><p>The observed campaigns are using variants of <a href="https://blog.talosintelligence.com/2021/04/a-year-of-fajan-evolution-and-bloomberg.html%23more">Nanocore</a>, <a href="https://blog.talosintelligence.com/2021/09/operation-armor-piercer.html">Netwire</a> and <a href="https://blog.talosintelligence.com/2021/08/rat-campaign-targets-latin-america.html">AsyncRAT</a> as payloads. These are commodity RATs that were widely used in other campaigns.</p><h4 id="nanocorerat">NanocoreRAT</h4><p>Nanocore is a 32-bit .NET portable executable first seen in the wild in 2013. After 2017, there are leaked versions of Nanocore that are widely used by the threat actors in their campaigns.</p><p>Extracting the configuration information from the Nanocore clients samples associated with this campaign showed us they are using version 1.2.2.0, which is a leaked version with an Oct. 26, 2021 build date. The C2 server used is mback5338[.]duckdns[.]org, listening on the TCP port 7632. The build date correlates with the possible start of the campaign.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/img/a/AVvXsEh-zQj3JuLP17DI-ftpoiP1HXchL_nVypy2h_rJqBkVqXd9Wy0pQitXn7JqUKYDnuz8QWHDv7VP35PmzDryYew1Dh-i1jmJ3tzyDt_2OZZBFicx7h9RGE2d5XMvHqi1Mk1OJHgD3oof7xOo_ITTO1aRwKfT369DUc4GuK7MRhOPxEQlN7RWAxkwp53Kdg-s16000.png" class="kg-image" alt loading="lazy"><figcaption>Nanocore variant config file.</figcaption></figure><p>We have also observed other C2 domains and different port numbers used by different samples of Nanocore client associated with these campaigns:</p><ul><li>nanoboss[.]duckdns[.]org</li><li>justinalwhitedd554[.]duckdns[.]org</li></ul><p>The plugins included with the payload are the Client and SurveillanceEx plugins. The client plugin is used by the RAT to handle the communications with the C2 server and SurveillanceEX plugin provides video and audio capture and the remote desktop capability.</p><h4 id="netwirerat">NetwireRAT</h4><p>NetwireRAT is a known threat used by the threat actors to  steal victim's passwords, login credentials and credit card data. It has the capability to remotely execute the commands and collects filesystem information.</p><p>This trojan establishes persistance by writing the registry keys: <br>HKEY_CURRENT_USER\Software\NETwIRe\HostId<br>HKEY_CURRENT_USER\Software\NETwIRe\Install Date<br>HKEY_CURRENT_USER\SOfttware\Microsoft\WIndows\CurrentVersion\Run\SysWOW32 with its value as the path to the trojan.</p><h4 id="asyncrat">AsyncRAT</h4><p>AsyncRAT is a remote access tool meant to remotely monitor and control computers through secure encrypted connection. Threat actors in this campaign use the AsyncRAT client by setting its configuration to connect to the C2 server and provide the attacker with remote access to the victim's machine. Using some of its features such as keylogger, screen recorder, system configuration manager, the attacker can steal confidential data from the victim's machine.</p><p>AsyncRAT creates the mutex "AsyncMutex_6SI8OkPnk" as the infection marker in the victim's machine.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/img/a/AVvXsEgT2pmI-CBDyQaUBkwr4lpeB5_BDsZan78-UmF0qErCVQBd0AlMS9-9BMFGuh36Duj_OrlfViGAfgDdNN_-t42yXZpfBgkrXz0hdCfn1iUPhuE2gZuwBVlWBB0CnzonPxKNIDWDGJLOftFF56ZzcDwClDP89mC4-xu3hLEIZE-kmAclauGD-VzFokKHFQ-s16000.png" class="kg-image" alt loading="lazy"><figcaption>AsyncRAT variant mutex function.</figcaption></figure><p>The AsyncRAT config file is decrypted and contains the configuration information such as C2 domain. In this instance, the C2 domain is asyncmoney[.]duckdns[.]org using the TCP port 7829. We have observed that this variant of AsyncRAT communicates with the C2 domain via TCP ports 7840, 7841 and 7842.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/img/a/AVvXsEi0lW-E3OXGHEu-hDTv5th56CYahBnzdyjqpRodFEJQExJIV47laIw-3tZOUsSFY8MC5G9bWtJDYsBSIujLLWP3YlbJA9DZyV4TsjvtGEFMnOj7ThjEOxKrPWphxzB6QNLILRg-CJEw5LtvVGYtVP9X1nYLgmrpK3WI2YFsV73afbmvgDWwo5t9d4wnag-s16000.png" class="kg-image" alt loading="lazy"><figcaption>AsyncRAT variant C2 connection parameters.</figcaption></figure><h3 id="infection-chain">Infection chain</h3><p>The infection chain starts with an email that contains malicious ZIP documents. The ZIP file attachment is an ISO image file containing the loader in JavaScript, Visual Basic script or a Windows batch file format. The actor has attempted to entice recipients by purporting that the attachment is a fake invoice document.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/img/a/AVvXsEiVt8PCeC0hk9hieRJPws-rYtDUMULaaCCLgJmkvVkA7f3YkZLB2fJXkg7Rh8qcDPLsRWsJjJymobFitZtxM49TJ9rwuD0377YDLNMLSjCs80hyMRgw2EvDkJiUN-7xJ0zxpEonCZK01Tffwzs8DpaIYtiM7qZz9fMHxN1V4F4SmPB_LDAy00gA9Wp0vA-s16000.png" class="kg-image" alt loading="lazy"><figcaption>Phishing email example.</figcaption></figure><p>The initial few characters of the ZIP file names are randomly generated and could be specific to the campaign. Some of the observed ZIP file names are:</p><ul><li>WROOT_Invoice_Copy.zip</li><li>YUEOP_Invoice_Copy.zip</li><li>HOO8M_Invoice_Copy.zip</li><li>TROOS_Invoice_Copy.zip</li><li>TBROO1_Invoice_Copy.zip</li></ul><h4 id="javascript-downloader">JavaScript Downloader</h4><p>The downloader JavaScript is an obfuscated script that has four layers of obfuscation. The deobfuscation process is performed at each stage with every next stage generated as the result of the previous stage deobfuscation function.</p><h5 id="layer-1-deobfuscation">Layer 1 deobfuscation</h5><p>The first level of decryption is performed by the function 'ejv()', which iterates over each character of the obfuscated data into an array and performs a number of arithmetic operations to decrypt the character and returns the deobfuscated result.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/img/a/AVvXsEiP8RIm_dfzq48huhCAQ5pzBdsVGxZN5aotoy9UnLeYL_vMYH6IGC7na2Bq7qPShoEHwnK1os91uBPhBbngnKWWHhpqFyvjZbL4yJeyWxbINROww2q4L-ECYPosU4ec2MNQ1WuBt6t-LlxN2Zj45g9gIWaBmhWhCTZxsTdrFj6KDXXgbacoemp4iaBQ4Q-s16000.png" class="kg-image" alt loading="lazy"><figcaption>First level decryption function.</figcaption></figure><p>The function 'ejv()' generates the second-stage decryption routine.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/img/a/AVvXsEj-Ua3TN-_LHmPawHgp1q-utCF_ZNqabaEDerBjwSyWtN8dxlX0DaAMW9BtkLJDK3RsFYVpvAPPRVCIWy0PKR7dIV0AJSKZADzFuZOmyGCtpcqhMS4W4TeTSZFpLyHNfTcM99jGD7ikVSsoNWrpkedqP068c-5WKaBCl3f1AOqM0e1eOecUNkY19TzF4g.png" class="kg-image" alt loading="lazy"><figcaption>Second level decryption function.</figcaption></figure><h5 id="layer-2-deobfuscation">Layer 2 deobfuscation</h5><p>The remaining part of the encrypted contents of the JavaScript downloader are decrypted in two sub-phases in Layer 2 deobfuscation process. First, it is decrypted by the decryption function 'ejv()' and then the result is passed to the second-level decryption function.</p><p>The result of the deobfuscation process contains another decryption function 'Ox$()', which is the third layer decryption function.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/img/a/AVvXsEjtIU447usNbnUS7Yq5SpnVuJ270nKlS4su_mmPDccl0VqFIaOp0eg3rGVXmE_oefNH4CGwjXrxzoCXfOxchYDIBuoKDts_nfdH8UQoWm1VZ6P-mnE8GtsALqm8wrJhysJLNdH_H-QpADFiXtz_iufQQWZaLtW8vAfgVUbh1fqn5pHkX1FdzdAZ-DhJ0w.png" class="kg-image" alt loading="lazy"><figcaption>Third-level decryption function.</figcaption></figure><h5 id="layer-3-deobfuscation">Layer 3 deobfuscation</h5><p>The encrypted strings of the Layer 2 deobfuscation process are decrypted by the function 'Ox$()'. The decrypted result of the Layer 3 deobfuscation process is another obfuscated function which has multiple function calls returning values and a series of eval() functions calling the third-level decryption function 'Ox$()' to decrypt the malicious downloader code.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/img/a/AVvXsEhXPqRcTdxSHIv0mS9xHMu5rSATmFeWnn2gfyjCnRzb3T64x-GnWpZTmS2xH9b00fooT-ZvSoPCHLZ10LzFwFczFW6oSLxQcX92F1UhqDJ9GD2dsX0hfwUcJiSDDLzLYWw6KA0xUvMVvzk_PxXFDlNB-OwCwu_VSeYhYIYyyiTU6gHtquUyZ7atxZ26OA.png" class="kg-image" alt loading="lazy"><figcaption>Obfuscated malicious downloader code.</figcaption></figure><p>While analysing another sample of the JavaScript downloader of this campaign, we observed a slightly different result from the Phase 3 deobfuscation process. It is likely that the code is automatically generated and randomized to make the detection process more difficult.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/img/a/AVvXsEjL4DvMvEDsY_JVCT1Jds6LS5anPPWJyBU8H-q09RUQHoWY-cZpo_twTNzQr7eZcl9xJzRF7vlmhrvOKLj_BMl_dbW4S7REyzZMAwDm-4ranEnxwe_glD3bmImZ7ywR47Ne6mzd73JuGdK0TqcFRk8aJgVnNUOTuK0QloJZLQIaPBB-kq6nP8nLARxjMg.png" class="kg-image" alt loading="lazy"><figcaption>Obfuscated malicious downloader code.</figcaption></figure><h5 id="layer-4-deobfuscation">Layer 4 deobfuscation</h5><p>The final stage of the deobfuscation of malicious downloader code is performed in Layer 4,  with the help of a third-level decryption function and some of its self decryption logic within the code. We observed that the Layer 4 decrypted code is not just a downloader — it also performs other activities such as:</p><ul><li>Configures the Logon Auto Start registry key "HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run" to establish persistence.</li><li>Configures scheduled task jobs by invoking the schtasks.exe process.</li></ul><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/img/a/AVvXsEgDHIgp2hIYgHUKPvBgS_0WkPFI_7EkS53TgrwnrwrouyDdaDJC51rwV_ItbHJmVpiPPCY4RGhPL-nuowuqi5MekZ-ZFbw-U9nElTWMPADrzyw9Jvcq6L_YgPQydui7lYmOvMuigFv3CLZLd70FwRJbQSNNgsaUw3DTFJWxkLK1K_wXIur9G6OAr_TUyQ.png" class="kg-image" alt loading="lazy"></figure><ul><li>Downloads the payload from the download server with the URL http://gg1592661[.]duckdns[.]org:7924/vre. The payloads downloaded by the observed campaigns are the variants of Netwire, Nanocore and AsyncRAT remote access trojans, saved and executed from the user's temporary folder of the victim's machine.</li><li>The script attempts to interact with the Alternate Data Stream to hide the information about its source as downloaded from the internet.</li><li>Collects information from the registry key HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\ProductId to fingerprint the victim's machine.</li></ul><p>Aside from the JavaScript loader trojan, we have observed a Batch file downloader trojan and a VBScript downloader trojan in our Cisco Secure Endpoint telemetry.</p><h4 id="batch-file-downloader">Batch file downloader</h4><p>The batch script contains an obfuscated command that runs PowerShell to download and run a payload from a download server, in this instance, 13[.]78[.]209[.]105 on Azure Cloud.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/img/a/AVvXsEgEep0KOykLAEeN1GzVkwEKb48Dc_FHJw-cXIB7EFC_8CEGrEYHBe-7XfU5quq9JTmEipT8WGnnr7A26WVvMS3dMv4s4alvy2R1rAVABsMcQAF5ISMHL0t8KXpqb82VymDvJpl6xqTReZYFaOg0erFfpRXX__1zxqm4oqw8p39nfsaebEAf1sxOflIigg.png" class="kg-image" alt loading="lazy"><figcaption>Batch script downloader trojan.</figcaption></figure><h4 id="vbscript-downloader">VBScript downloader</h4><p>Obfuscated VB downloaders execute a PowerShell command which runs and connects to the download server, for example, to 52[.]27[.]15[.]250, running on AWS EC2.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/img/a/AVvXsEjfJBhmrd4GwvSSUoUvlB2tBtrvxMKKnel_5Rf-73d-jDK9l23ReupFAumeZcsBgJkYdv4mnFUxYgPFt8A6IKZNI2RZC-uXq7PmAOyRDoX_H8VmXwdSXbb6QRByonADfHy--8g06-YDPv7BUrIPDJdhBxaaR0L_cYLdQnMg9W8I1-npNGW3HrSvbD3Kdg.png" class="kg-image" alt loading="lazy"><figcaption>VBScript trojan downloader.</figcaption></figure><h4 id="powershell-dropper">PowerShell dropper</h4><p>Our investigation of download servers in Azure
