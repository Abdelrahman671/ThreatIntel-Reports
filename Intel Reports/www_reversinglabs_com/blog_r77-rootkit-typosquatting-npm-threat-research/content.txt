<div id="hs_cos_wrapper_module_1577203992502740" class="hs_cos_wrapper hs_cos_wrapper_widget hs_cos_wrapper_type_module" style="" data-hs-cos-general-type="widget" data-hs-cos-type="module">
<h2 class="post-header__secondary-head blog__post-secondary-h2">ReversingLabs discovered that one “s” was all that separated a legit npm package from a malicious twin that delivered the r77 rootkit —&nbsp;and was downloaded more than 700 times. </h2>

<div class="blog__post-content html-content">
<span id="hs_cos_wrapper_post_body" class="hs_cos_wrapper hs_cos_wrapper_meta_field hs_cos_wrapper_type_rich_text" style="" data-hs-cos-general-type="meta_field" data-hs-cos-type="rich_text"><p><img src="https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-npm-S-typo.webp?width=1400&amp;height=732&amp;name=blog-npm-S-typo.webp" width="1400" height="732" loading="lazy" alt="blog-npm-S-typo" style="height: auto; max-width: 100%; width: 1400px;" srcset="https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-npm-S-typo.webp?width=700&amp;height=366&amp;name=blog-npm-S-typo.webp 700w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-npm-S-typo.webp?width=1400&amp;height=732&amp;name=blog-npm-S-typo.webp 1400w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-npm-S-typo.webp?width=2100&amp;height=1098&amp;name=blog-npm-S-typo.webp 2100w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-npm-S-typo.webp?width=2800&amp;height=1464&amp;name=blog-npm-S-typo.webp 2800w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-npm-S-typo.webp?width=3500&amp;height=1830&amp;name=blog-npm-S-typo.webp 3500w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-npm-S-typo.webp?width=4200&amp;height=2196&amp;name=blog-npm-S-typo.webp 4200w" sizes="(max-width: 1400px) 100vw, 1400px"></p>
<p>ReversingLabs researchers have identified a new, malicious supply chain attack affecting the npm platform. The “typosquatting” campaign first appeared in August and pushed a malicious package, <em>node-hide-console-windows</em>, which downloaded a Discord bot that facilitated the planting of an open source rootkit, r77.</p>
<p>This is the first time ReversingLabs researchers have discovered a malicious open source package delivering rootkit functionality, and suggests that open source projects may increasingly be seen as an avenue by which to distribute malware.&nbsp;</p>
<!--more-->
<h2 style="font-weight: bold;">Typosquatting delivers open source malware</h2>
<p>This is just the latest in a string of malicious, supply chain attacks to leverage npm. In August, for example, ReversingLabs researchers uncovered evidence of additional malicious packages connected to npm’s campaign <a href="/blog/more-malicious-npm-packages-found-in-wake-of-jumpcloud-supply-chain-hack" rel="noopener" target="_blank">targeting cryptocurrency providers</a>. In July, we reported on <a href="/blog/operation-brainleeches-malicious-npm-packages-fuel-supply-chain-and-phishing-attacks" rel="noopener" target="_blank">Brainleeches</a>, another npm-based campaign that targeted both end users and developers with malicious code designed to harvest user data with phony Microsoft.com login forms and implant credential harvesting scripts in applications that inadvertently incorporate the npm packages.&nbsp;<br><br>As with those previous campaigns, this latest discovery relies heavily on a <a href="/glossary/typosquatting" rel="noopener">technique known as typosquatting</a>, in which cybercriminals stand up malicious packages with names that closely resemble legitimate and widely used open source modules.&nbsp;<br><br>In this case, the malicious package, named <span style="font-style: italic;">node-hide-console-windows</span>, mimics the legitimate npm package <a href="https://www.npmjs.com/package/node-hide-console-window?activeTab=readme" rel="noopener" target="_blank"><span style="font-style: italic;">node-hide-console-window</span></a>, which is used to toggle an application’s console window visibility. As you can see, these two packages are nearly identical, with only the letter “s” distinguishing the malicious package from the legitimate one.&nbsp;<br><br>As the threat research team has seen in prior campaigns, the attackers also took other steps to make the packages indistinguishable to hurried developers looking to add the node-hide-console-window functionality to their application. For example, the npm page for the malicious package was set up to look very similar to the page for the legitimate package. Also, the malicious npm package had 10 versions published, the same as <a href="https://www.npmjs.com/package/node-hide-console-window?activeTab=readme" rel="noopener" target="_blank"><span style="font-style: italic;">node-hide-console-window</span></a>.<br><br>This latest campaign began at the end of August.&nbsp;</p>
<a href="https://blog.reversinglabs.com/hubfs/Blog/blog-mind-the-s-figure-01.webp" class="lightbox-image"><img src="https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-mind-the-s-figure-01.webp?width=1400&amp;name=blog-mind-the-s-figure-01.webp" alt="Legitimate package’s versions" width="1400" style="width: 1400px;" srcset="https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-mind-the-s-figure-01.webp?width=700&amp;name=blog-mind-the-s-figure-01.webp 700w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-mind-the-s-figure-01.webp?width=1400&amp;name=blog-mind-the-s-figure-01.webp 1400w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-mind-the-s-figure-01.webp?width=2100&amp;name=blog-mind-the-s-figure-01.webp 2100w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-mind-the-s-figure-01.webp?width=2800&amp;name=blog-mind-the-s-figure-01.webp 2800w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-mind-the-s-figure-01.webp?width=3500&amp;name=blog-mind-the-s-figure-01.webp 3500w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-mind-the-s-figure-01.webp?width=4200&amp;name=blog-mind-the-s-figure-01.webp 4200w" sizes="(max-width: 1400px) 100vw, 1400px"></a>
<p style="text-align: center;"><span style="font-weight: bold; font-size: 16px;">Figure 1: Legitimate package’s versions<br><br></span></p>
<p>The malicious <span style="font-style: italic;">node-hide-console-windows</span> package attracted ReversingLabs researchers' attention during routine monitoring of the npm public repository. The team regularly scans npm packages using the <a href="/products/software-supply-chain-security" rel="noopener" target="_blank">ReversingLabs Software Supply Chain Security</a> platform. Packages that contain suspicious characteristics are identified and manually vetted by our researchers.&nbsp;<br><br>When our researchers discover malicious packages, we report them to npm maintainers so that they can be removed. We also conduct more in-depth analysis on our own to better understand malicious supply chain campaigns. The <span style="font-style: italic;">node-hide-console-windows</span> package displayed a number of peculiar behaviors our team often finds in malicious packages, prompting greater scrutiny.&nbsp;</p>
<a href="https://blog.reversinglabs.com/hubfs/Blog/blog-mind-the-s-figure-02.webp" class="lightbox-image"><img src="https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-mind-the-s-figure-02.webp?width=1400&amp;name=blog-mind-the-s-figure-02.webp" alt="Behavior indicators of node-hide-console-windows" width="1400" style="width: 1400px;" srcset="https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-mind-the-s-figure-02.webp?width=700&amp;name=blog-mind-the-s-figure-02.webp 700w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-mind-the-s-figure-02.webp?width=1400&amp;name=blog-mind-the-s-figure-02.webp 1400w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-mind-the-s-figure-02.webp?width=2100&amp;name=blog-mind-the-s-figure-02.webp 2100w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-mind-the-s-figure-02.webp?width=2800&amp;name=blog-mind-the-s-figure-02.webp 2800w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-mind-the-s-figure-02.webp?width=3500&amp;name=blog-mind-the-s-figure-02.webp 3500w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-mind-the-s-figure-02.webp?width=4200&amp;name=blog-mind-the-s-figure-02.webp 4200w" sizes="(max-width: 1400px) 100vw, 1400px"></a>
<p style="text-align: center;"><span style="font-weight: bold; font-size: 16px;">Figure 2: &nbsp;Behavior indicators of node-hide-console-windows<br><br></span></p>
<p>For example, in addition to the package’s suspiciously familiar name and version history, the maintainer account responsible for the package was newly created and not connected to any other npm projects. More to the point, upon inspection by RL's Software Supply Chain Security platform, malicious code was located inside of the <em>node-hide-console-windows</em> file <em>index.js</em>, which is specified as the main inside <em>package.json</em>. The code in that file wasn’t obfuscated nor did it look very trustworthy. When it ran, it fetched an executable that was detonated immediately thereafter.&nbsp;</p>
<h2 style="font-weight: bold;">DiscordRAT - An Open Source Trojan?</h2>
<p>So the research team took a look at the executable that the index.js file fetched. Our analysis quickly identified it as a copy of <a href="https://github.com/moom825/Discord-RAT-2.0" rel="noopener" target="_blank">DiscordRAT 2.0</a>, an open source <span style="font-style: italic;">“Discord Remote Administration Tool”</span> that is intended <span style="font-style: italic;">“for educational use only.”</span> In short: this is open source malware. &nbsp;<br><br>This was not the first time malicious actors have used a copy of open source malware available on GitHub in an actual attack. In May, for example, we observed a malicious npm package <a href="/blog/rats-found-hiding-in-the-npm-attic" rel="noopener"><span style="font-style: italic;">nodejs-encrypt-agent</span> that contained a malicious executable file which turned out to be TurkoRat</a>, an open source infostealer packaged with legitimate npm package <span style="font-style: italic;">pkg</span>. More recently, the third stage executable connected to the recent Roblox campaign was discovered to be a PyInstaller-compiled executable serving Luna Grabber, another open source infostealer.&nbsp;<br><br>For would-be attackers, <a href="https://github.com/moom825/Discord-RAT-2.0" rel="noopener" target="_blank">DiscordRAT 2.0</a> is very simple to use. Its authors provided detailed instructions in the <span style="font-style: italic;">readme.md</span> file that are easy to follow. The bot needs to be registered with the Discord developer portal and added into the Discord server or guild, through which a malicious actor wants to control it.&nbsp;<br><br>Before deploying it, a bot token must be acquired from the Discord developer portal along with a guild ID in which the bot was added. This is then included inside the malicious DiscordRAT 2.0 executable, which can be delivered to victims through various channels including as phishing email attachments, packages deposited on public repositories and targeting developers, and so on.</p>
<p>The team extracted both these identifiers from DiscordRAT’s configuration using the IDA Pro application. The bot token used in this malicious executable was:</p>

<h2 style="font-weight: bold;">Execute once, no further action needed</h2>
<p>DiscordRAT makes managing infected hosts easy. Once the malicious executable is run, it creates a channel in the associated Discord server- one channel for each victim. At the same time, an initial payload is sent to the infected host (Figure 5).</p>
<a href="https://blog.reversinglabs.com/hubfs/Blog/blog-mind-the-s-figure-05.webp" class="lightbox-image"><img src="https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-mind-the-s-figure-05.webp?width=1400&amp;name=blog-mind-the-s-figure-05.webp" alt="Starting payload sent to newly created Discord channel" width="1400" style="width: 1400px;" srcset="https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-mind-the-s-figure-05.webp?width=700&amp;name=blog-mind-the-s-figure-05.webp 700w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-mind-the-s-figure-05.webp?width=1400&amp;name=blog-mind-the-s-figure-05.webp 1400w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-mind-the-s-figure-05.webp?width=2100&amp;name=blog-mind-the-s-figure-05.webp 2100w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-mind-the-s-figure-05.webp?width=2800&amp;name=blog-mind-the-s-figure-05.webp 2800w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-mind-the-s-figure-05.webp?width=3500&amp;name=blog-mind-the-s-figure-05.webp 3500w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-mind-the-s-figure-05.webp?width=4200&amp;name=blog-mind-the-s-figure-05.webp 4200w" sizes="(max-width: 1400px) 100vw, 1400px"></a>
<p style="text-align: center;"><span style="font-weight: bold; font-size: 16px;">Figure 5: Starting payload sent to newly created Discord channel<br><br></span></p>
<p>Following that, the bot is waiting for a malicious actor to send additional commands. DiscordRAT provides series of commands for everything from extracting information to controlling the victim’s machine by disabling Windows Defender and firewall, killing processes, blocking use of the victim’s mouse and keyboard, as well as shutting down or “bluescreening” the infected device. (A long and complete list of these commands &nbsp;is available on <a href="https://github.com/moom825/Discord-RAT-2.0" rel="noopener" target="_blank">DiscordRAT's GitHub page</a>.)</p>
<h2 style="font-weight: bold;">Getting to the root(kit) of the problem</h2>
<p>As we continued our investigation of the package and executable, our interest was piqued when we noticed one of the commands named: <em>!rootkit</em>. This command allowed malicious actors to launch the <a href="https://github.com/bytecode77/r77-rootkit" rel="noopener" target="_blank">r77 rootkit</a> on the victim's machine.&nbsp;<br><br>Like DiscordRAT, r77 is an example of open source malware with extensive documentation that makes it easy to deploy, even by novice actors. r77 is a fileless ring 3 rootkit that is able to disguise files and processes and that can be bundled with other software or launched directly. &nbsp;r77 is a recent addition to DiscordRAT 2.0, with previous versions of that open source malware (<a href="https://github.com/moom825/Discord-RAT" rel="noopener" target="_blank">Discord-RAT</a>) lacking the ability to launch a rootkit. Also of interest: the DiscordRAT 2.0 executable we studied did not use the newest version of the r77 rootkit, but an older version of the rootkit.<br><br>In terms of functionality, when the bot receives the <span style="font-style: italic;">!rootkit </span>command, the r77 rootkit is launched and two registry subkeys are created to hide the bot's presence. One subkey is used to hide the path of the executable. The other is used to hide the bot’s process, meaning that the executable that launched the bot and the path to the executable are hidden from the user of the compromised device.<br><br>(We also discovered another command <span style="font-style: italic;">!unrootkit</span>, which removes the r77 rootkit from the victim’s machine.)&nbsp;<br><br>This wasn’t our first time encountering the r77 rootkit. The malware has been the star of a number of <a href="https://cybersecurity.att.com/blogs/labs-research/seroxen-rat-for-sale" rel="noopener" target="_blank">previous</a> malicious <a href="https://www.elastic.co/security-labs/elastic-security-labs-steps-through-the-r77-rootkit" rel="noopener" target="_blank">campaigns</a>. However this was the first time that ReversingLabs researchers found r77 hiding in a malicious open source package on npm.</p>
<h2 style="font-weight: bold;">But wait… there’s more!</h2>
<p>All 10 versions of <em>node-hide-console-windows</em> that we analyzed downloaded the same malicious executable serving DiscordRAT 2.0. Interestingly, the last two versions had an added malicious payload disguised as <em>visual code update</em>&nbsp;that was fetched by <em>index.js</em> in addition to the DiscordRAT. &nbsp;<br><br>Once again, this unexplained behavior was a major red flag that warranted more in-depth analysis: A random npm package downloading an update for Visual Studio Code from an arbitrary page not connected to the official page, paired up with various different suspicious behaviors.</p>
<a href="https://blog.reversinglabs.com/hubfs/Blog/blog-mind-the-s-figure-06.webp" class="lightbox-image"><img src="https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-mind-the-s-figure-06.webp?width=1400&amp;name=blog-mind-the-s-figure-06.webp" alt="Behavior indicators of the second downloaded executable" width="1400" style="width: 1400px;" srcset="https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-mind-the-s-figure-06.webp?width=700&amp;name=blog-mind-the-s-figure-06.webp 700w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-mind-the-s-figure-06.webp?width=1400&amp;name=blog-mind-the-s-figure-06.webp 1400w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-mind-the-s-figure-06.webp?width=2100&amp;name=blog-mind-the-s-figure-06.webp 2100w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-mind-the-s-figure-06.webp?width=2800&amp;name=blog-mind-the-s-figure-06.webp 2800w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-mind-the-s-figure-06.webp?width=3500&amp;name=blog-mind-the-s-figure-06.webp 3500w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/blog-mind-the-s-figure-06.webp?width=4200&amp;name=blog-mind-the-s-figure-06.webp 4200w" sizes="(max-width: 1400px) 100vw, 1400px"></a>
<p style="text-align: center;"><span style="font-weight: bold; font-size: 16px;">Figure 6: Behavior indicators of the second downloaded executable<br><br></span></p>
<p>In this case, the malicious payload associated with <em>visual code update </em>turned out to be a PyInstaller-compiled executable serving Blank-Grabber, an infostealer written in Python 3 that recently switched maintainers after its original author transferred control of the project to one of a contributor.<br><br>An open source package delivering a PyInstaller-compiled open source infostealer is something we saw recently with the <a href="/blog/fake-roblox-api-packages-luna-grabber-npm" rel="noopener" target="_blank">Roblox campaign</a>, though the deliverable there was LunaGrabber rather than Blank-Grabber. Still, the similarities suggest that malicious actors are turning to open source projects to plant malware while evading antivirus detection.&nbsp;</p>

<h2 style="font-weight: bold;">Conclusion and a call to action</h2>
<p>Compared with other <a href="/blog/iconburst-npm-software-supply-chain-attack-grabs-data-from-apps-websites" rel="noopener">malicious supply chain campaigns such as IconBurst</a><a href="/blog/iconburst-npm-software-supply-chain-attack-grabs-data-from-apps-websites" rel="noopener" target="_blank"></a>, the r77 rootkit campaign had a limited reach among developers. Before being detected and removed by npm maintainers, the <span style="font-style: italic;">node-hide-console-windows</span> was downloaded around <a href="https://npm-stat.com/charts.html?package=node-hide-console-windows&amp;from=2023-08-24&amp;to=2023-09-28" rel="noopener" target="_blank">700</a> times — a fraction of the downloads associated with the malicious IconBurst packages. So, was this a sophisticated, targeted campaign, or just noise — an unsophisticated lark by low skill script kiddies? It is unclear to the team at this point.<br><br>On one hand, our research shows that the malicious actor or actors made an effort to make their packages appear trustworthy. As we observed, the actor or actors behind this campaign fashioned an npm page that closely resembled the page for the legitimate package that was being typosquatted, and even created 10 versions of the malicious package to mirror the package they were mimicking.&nbsp;<br><br>On the other hand, all of the projects used in this campaign were well-documented, open source malware posted to major platforms like GitHub “for training purposes.” In other words, the entire campaign was fashioned from components that are freely and publicly available online. Such a campaign requires little effort by attackers to build and execute. The reliance on known and frequently deployed open source malware also increased the chances of detection by the targeted organizations.&nbsp;<br><br>Still, for the thousands of developers utilizing popular open source projects like <span style="font-style: italic;">node-hide-console-window</span>, such distinctions don’t matter. Regardless of how this campaign ranks in sophistication, these packages highlight an always present, and growing, danger lurking on public repositories such as npm or PyPI, as well as on GitHub. Something as dangerous and invasive as a rootkit is easily available to everyone, well documented, and ready for use.<br><br>Such availability makes supply chain attacks — once the province of well-resourced and sophisticated, nation-state or cybercriminal groups — accessible to low-skill malicious actors and “script kiddies.” For development organizations of all kinds, that’s even worse news than a slick, targeted and <a href="/blog/3cx-supply-chain-attack-targeted" rel="noopener">covert campaign like the one targeting 3CX</a> or <a href="/blog/sunburst-the-next-level-of-stealth" rel="noopener">SolarWinds</a>. That’s because it shows that the supply chain attack door is now open to low-stakes actors, who may fool incautious developers and infiltrate &nbsp;development pipelines in ways that reverberate far and wide.<br>&nbsp; &nbsp;&nbsp;<br>Like other recent campaigns we have highlighted, this campaign underscores the need for organizations to sharpen their tools for detecting risks from open source packages. As with this case, these attacks hinge on developer inattention to small details in naming &nbsp;— in this case a single letter “s.” But that’s not the only trick available to would-be supply chain attackers. The use of obfuscated code is a major warning sign. Other indicators include suspicious naming and package versioning, new packages with sketchy histories, smaller than expected downloads and dependencies, and more.&nbsp;<br><br>When using packages from public repositories in their projects, developers should keep an eye peeled for these small, but telling details to avoid a malicious package being introduced as a dependency in some larger project.&nbsp;<br><br>Development organizations also need to scrutinize the features and behaviors of the open source, third-party and commercial code they are relying on, in order to track dependencies and detect potential malicious payloads in them. ReversingLabs technology, <a href="/blog/reversinglabs-a1000-threat-analysis-and-hunting-solution-update-drives-secops-forward" rel="noopener">including our A1000 threat hunting</a> and <a href="/products/software-supply-chain-security" rel="noopener">Software Supply Chain Security</a> platforms, deliver the visibility teams need to discover malware.</p></span>
