          <h1>Malicious Macros Adapt to Use Microsoft Publisher to Push Ekipa RAT</h1>          
          
        </div>
        <div class="blog-details-header-right">
          
            
            <a href="#" class="theme-changer">Change theme to light</a>
            
            
            <img src="https://www.trustwave.com/hs-fs/hubfs/Web/Spiders/2024/spider_01-mob.png?width=450&amp;height=250&amp;name=spider_01-mob.png" loading="eager" alt="Malicious Macros Adapt to Use Microsoft Publisher to Push Ekipa RAT" class="hero-spider">
            
          
        </div>
      </div>
      <div class="toc-content content-wrapper">
        
        <div class="wrapper-main_content">
          
          <div class="blog-details-metadata">
            <span class="date-time">
              <time datetime="2022-12-21 06:00:00">December 21, 2022</time>
            </span>
            <span class="minread">9 Minute Read</span>
            
              <span class="blog-author">by Wojciech Cieslak</span>
            
          </div> 
          
          
          <div class="wrapper-blog-content">
            <span id="hs_cos_wrapper_post_body" class="hs_cos_wrapper hs_cos_wrapper_meta_field hs_cos_wrapper_type_rich_text" style="" data-hs-cos-general-type="meta_field" data-hs-cos-type="rich_text"><p>After Microsoft announced this year that macros from the Internet will be blocked by default in <a href="https://learn.microsoft.com/en-us/deployoffice/security/internet-macros-blocked">Office</a>, many threat actors have switched to different file types such as Windows Shortcut (LNK), ISO or ZIP files, to distribute their malware. Nevertheless, Office documents are still actively leveraged in many campaigns and pose a large risk to organizations, especially with threat actors continuously finding new ways to avoid detection.</p>
<!--more-->
<p>The Trustwave SpiderLabs’ Research Team has analyzed samples of an<span>&nbsp;</span><span>Ekipa</span><span>&nbsp;</span>Remote Access Trojan (RAT) in the<span>&nbsp;</span><span>wild, and</span><span>&nbsp;</span>found interesting techniques for the use of malicious Office documents. As shown in this research, the<span>&nbsp;</span><span>Ekipa</span><span>&nbsp;</span>RAT was added to a sophisticated threat actors’ cyber arsenal and<span>&nbsp;</span><span>used in the Russian – Ukraine war.&nbsp;</span><span></span></p>
<h3>Overview of Functionalities</h3>
<p><span>Ekipa</span><span>&nbsp;</span>is a Remote Access Trojan used for targeted attacks and can be purchased<span>&nbsp;</span><span>&nbsp;</span>on underground forums, as<span>&nbsp;</span><a href="https://cloudsek.com/threatintelligence/ekipa-remote-access-trojan-designed-by-russian-hacktivists-for-targeted-attacks/"><span>CloudSEK</span></a><span>&nbsp;</span>found in its research. The current price is set at $3,900, which is very high. The trojan leverages MS Office and Visual Basic for Applications as its main infection and operations vector. It also comes with a control panel and builders for:</p>
<ul>
<li><span></span>MS Word Macros</li>
<li><span></span>XLL Excel add-ins</li>
<li><span></span>MS Publisher Macros</li>
</ul>
<p>A Remote Access Trojan is capable of:</p>
<ul>
<li><span></span>Collecting information about a targeted system (basic system information, installed AV products,<span>&nbsp;</span><span>GPU</span><span>&nbsp;</span>and CPU information and more)</li>
<li><span></span>Browsing and downloading of files on attached drives</li>
<li><span></span>Dropping files</li>
<li><span></span>Executing files and commands.</li>
</ul>
<p>When used with malicious Word documents, the trojan’s main functions are implemented in a one-time VBA macro template. When the document is reopened, the server rejects the request to download the macro template and all subsequent requests for installation actions.</p>

<h3>Execution</h3>
<p>The initial Microsoft Word document “<span><a href="https://www.virustotal.com/gui/file/ba7c39cc4e349a852241b929c6046734ab3a8a94d19d0b8abb8f25023bbebfa0">Приказ&nbsp;№21 от&nbsp;29-03-2022.docx</a></span>” was observed in March 2022. Upon execution, it downloads the remote template from the URL:</p>
<pre style="word-wrap: break-word;"><code>hxxps[.]//roskazna[.]net/acpx/t.php?t=774b4bcb8d7287d011ac9cb2d7ff2a76659ca82a46e5df7783c9ff011d19b21e17393264b85072391adc0b57f0abea9e&amp;action=show_document&amp;z=1&amp;x=2500.</code></pre>
<p>This URL pattern matches URLs seen in other documents related to<span>&nbsp;</span><span>Ekipa</span><span>&nbsp;</span>RAT. They contact ‘<span>t.php</span>’ endpoint with parameter "t" which is the unique identifier ensuring that the remote template can be fetched only once for any given initial Word document file, and parameter “action” with “<span>show_document</span>” value.</p>
<p>The remote template executes the VBA RAT after the user decides to close the document. In the<span>&nbsp;</span><span>DocumentBeforeClose</span><span>&nbsp;</span>procedure, it cancels shutdown of the document and instead sets the<span>&nbsp;</span><span>Application.Visible</span><span>&nbsp;</span>property to False. Then it executes the main<span>&nbsp;</span><span>ConnectCP</span><span>&nbsp;</span><span>function</span>.</p>

<h3>Main Functionalities</h3>
<p>In the<span>&nbsp;</span><span>ConnectCP</span><span>&nbsp;</span>routine, the malicious macro collects information about the system and stores it in a JSON format. Next, it leverages<span>&nbsp;</span><span>SetTimer</span><span>&nbsp;</span>to set up a procedure (“<span>TimerProc</span>”) that will execute every 2.5 seconds. The time interval value is the ‘x’ parameter in the initial URL fetching the remote template.</p>
<p>The timer procedure executes the function responsible for sending the initially collected data about the system to the Command-and-Control server. In response the server returns a list of tasks for the trojan to execute.</p>

<h3><span>Reverse Shell Creation</span></h3>
<p><span>A new task with Task ID ‘~’ is responsible for creating a reverse shell for the attacker. It creates a ‘cmd.exe’ process with a modified&nbsp;</span><a href="https://learn.microsoft.com/en-us/windows/win32/api/processthreadsapi/ns-processthreadsapi-startupinfoa"><span>StartupInfoA</span></a><span>&nbsp;structure so that standard input and output is routed through two created pipes. One of them is used to send commands to Command Prompt and the second one to read the output.</span></p>

Cobalt Strike beacons with this watermark were dropped by<span>&nbsp;</span><span>Emotet</span><span>&nbsp;</span>and<span>&nbsp;</span><span>SVCReady</span><span>&nbsp;</span>botnets.</p>
<p>The<span>&nbsp;</span><span>Ekipa</span><span>&nbsp;</span>RAT is also being used in the Russian -Ukraine Conflict. While the analyzed Command and Control server azure-tech[.]pro seemed to be geo-fenced to only allow traffic from Ukraine, other documents were used in attacks against Russia. Documents communicating with<span>&nbsp;</span><span>kc-3</span>[.]<span>ru</span><span>&nbsp;</span>and<span>&nbsp;</span><span>roskazna</span>[.]net domains used lures targeting Russian recipients.</p>

<p>As shown in this research, the<span>&nbsp;</span><span>Ekipa</span><span>&nbsp;</span>RAT is actively being used to target Russian entities and individuals, which is in line with the Malwarebytes research.</p>
<p>Given that one of the servers appeared to be geofenced to only allow traffic from Ukraine, there is a small chance that it was used by two sides of this conflict.</p>
<p>It is interesting that while being sold on pro-Russian forums,<span>&nbsp;</span><span>Ekipa</span><span>&nbsp;</span>RAT is leveraged to target entities in Russia, which breaks the unwritten rule of this country’s hacker underground – don’t hack Russia.</p>
<h3>&nbsp;</h3>
<h3>Microsoft Publisher and XLL variants of<span>&nbsp;</span><span>Ekipa</span><span>&nbsp;</span>RAT</h3>
<p>We did not identify samples of those EKIPA RAT variants in the wild. The IOC section includes one Excel document with embedded macros that, based on the included URL pattern, is an<span>&nbsp;</span><span>Ekipa</span><span>&nbsp;</span>RAT loader, however the C2 server was inactive during our analysis.</p>
<p>Both XLL Excel add ins and Publisher variants are most likely a response to Microsoft blocking macros in files downloaded from Internet. While XLL files are widely used by threat actors, Microsoft Publisher (.pub) files are a niche.</p>
<p>Just as with other Microsoft office products, like Excel or Word, Publisher files can contain macros that will execute upon the opening or closing the file, which makes them interesting initial attack vectors from the threat actor’s point of view. When Microsoft blocked macros from executing in files downloaded from the Internet, it did not do so for the Publisher files.<img loading="lazy" src="https://www.trustwave.com/hs-fs/hubfs/Web/Blogs/SpiderLab/19462_image038.webp?width=479&amp;height=108&amp;name=19462_image038.webp" alt="19462_image038" width="479" height="108" style="display: block; margin-left: auto; margin-right: auto; height: auto; max-width: 100%; width: 479px;" srcset="https://www.trustwave.com/hs-fs/hubfs/Web/Blogs/SpiderLab/19462_image038.webp?width=240&amp;height=54&amp;name=19462_image038.webp 240w, https://www.trustwave.com/hs-fs/hubfs/Web/Blogs/SpiderLab/19462_image038.webp?width=479&amp;height=108&amp;name=19462_image038.webp 479w, https://www.trustwave.com/hs-fs/hubfs/Web/Blogs/SpiderLab/19462_image038.webp?width=719&amp;height=162&amp;name=19462_image038.webp 719w, https://www.trustwave.com/hs-fs/hubfs/Web/Blogs/SpiderLab/19462_image038.webp?width=958&amp;height=216&amp;name=19462_image038.webp 958w, https://www.trustwave.com/hs-fs/hubfs/Web/Blogs/SpiderLab/19462_image038.webp?width=1198&amp;height=270&amp;name=19462_image038.webp 1198w, https://www.trustwave.com/hs-fs/hubfs/Web/Blogs/SpiderLab/19462_image038.webp?width=1437&amp;height=324&amp;name=19462_image038.webp 1437w" sizes="(max-width: 479px) 100vw, 479px"></p>

<h3>&nbsp;</h3>
<h3><span>Conclusion</span><span></span></h3>
<p>The<span>&nbsp;</span><span>Ekipa</span><span>&nbsp;</span>RAT is a great example of that how threat actors are continuously changing their techniques to stay ahead of the<span>&nbsp;</span><span>defenders.</span><span>&nbsp;</span>As shown in this research, the creators of this malware are tracking changes in the security industry, like blocking macros from the internet by Microsoft, and shifting their tactics accordingly. It is also interesting to see how sophisticated threat actors adopt these new tools into their arsenal for a better chance of completing their objectives.</p>
