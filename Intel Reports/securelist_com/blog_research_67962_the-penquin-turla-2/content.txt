																							<h2>A Turla/Snake/Uroburos Malware for Linux</h2>											
											<div class="js-reading-content">
												<div class="c-wysiwyg">
													<p>Recently, an interesting malicious sample was uploaded to a multi-scanner service. This immediately triggered our interest because it appears to represent a previously unknown piece of a larger puzzle. That puzzle is &#8220;Turla&#8221;, one of the most complex APTs in the world.</p>
<p>We have written previously about the Turla APT with posts about their <a href="https://securelist.com/the-epic-turla-operation/65545/">Epic Turla operations</a>  and <a href="https://securelist.com/agent-btz-a-source-of-inspiration/58551/">Agent.btz inspiration</a>.</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2014/08/08075649/Uroburos_Epic.jpg" class="magnificImage"><img decoding="async" width="1600" height="1200" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2014/08/08075649/Uroburos_Epic.jpg" alt="Uroburos_Epic"  class="aligncenter size-full wp-image-65656" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2014/08/08075649/Uroburos_Epic.jpg 1600w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2014/08/08075649/Uroburos_Epic-300x225.jpg 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2014/08/08075649/Uroburos_Epic-1024x768.jpg 1024w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2014/08/08075649/Uroburos_Epic-768x576.jpg 768w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2014/08/08075649/Uroburos_Epic-1536x1152.jpg 1536w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2014/08/08075649/Uroburos_Epic-200x150.jpg 200w" sizes="(max-width: 1600px) 100vw, 1600px" /></a></p>
<p>So far, every single Turla sample we&#8217;ve encountered was designed for the Microsoft Windows family, 32 and 64 bit operating systems. The newly discovered Turla sample is unusual in the fact that it&#8217;s the <strong>first Turla sample targeting the Linux operating system</strong> that we have discovered.</p>
<p><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-67951" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2014/12/08065918/Penquin_1.jpg" alt="Penquin_1" width="126" height="159" /></p>
<p>This newly found Turla component supports Linux for broader system support at victim sites. The attack tool takes us further into the set alongside the <a href="http://www.baesystems.com/what-we-do-rai/the-snake-campaign">Snake rootkit</a> and components first associated with this actor a couple years ago. We suspect that this component was running for years at a victim site, but do not have concrete data to support that statement just yet.</p>
<p>The Linux Turla module is a C/C++ executable statically linked against multiple libraries, greatly increasing its file size. It was stripped of symbol information, more likely intended to increase analysis effort than to decrease file size. Its functionality includes hidden network communications, arbitrary remote command execution, and remote management. Much of its code is based on public sources.</p>

<h2 id="functional-description">Functional description</h2>
<p>The sample is a stealth backdoor based on the <a href="http://www.phenoelit.org/stuff/cd00r.c">cd00r sources</a>.</p>
<p>This Turla cd00r-based malware maintains stealth without requiring elevated privileges while running arbitrary remote commands. It can&#8217;t be discovered via netstat, a commonly used administrative tool. It uses techniques that don&#8217;t require root access, which allows it to be more freely run on more victim hosts. Even if a regular user with limited privileges launches it, it can continue to intercept incoming packets and run incoming commands on the system.</p>
<h3 id="startup-and-execution">Startup and Execution</h3>
<p>To start execution, the process requires two parameters: ID (a numeric value used as a part of the &#8220;magic packet for authentication&#8221;) and an existing network interface name. The parameters can be inputted two different ways: from STDIN, or from dropper a launching the sample. This is NOT a command-line parameter, it&#8217;s a real prompt asking the attacker user to provide the input parameters. After the ID and interface name are entered and the process launched, the backdoor&#8217;s process PID is returned. Here is a screenshot of this simple interface:</p>
<p><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-67952" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2014/12/08065915/Penquin_2.jpg" alt="Penquin_2" width="284" height="75" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2014/12/08065915/Penquin_2.jpg 284w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2014/12/08065915/Penquin_2-280x75.jpg 280w" sizes="auto, (max-width: 284px) 100vw, 284px" /></p>
<p>While there is no initial network callback, a section of code maintains a hardcoded c2 string &#8220;news-bbc.podzone[.]org&#8221;. This fully qualified domain name was first set up in 2010, suggesting that this binary is fairly recent in the string of Turla campaigns. Also, while we haven&#8217;t seen additional file download activity from this server by this tool, it likely participated as a file server of sorts.</p>
<h3 id="magic-packets-for-remote-command-execution">Magic Packets for Remote Command Execution</h3>
<p>The module statically links PCAP libraries, and uses this code to get a raw socket, applies a filter on it, and captures packets, checking for a specific condition (the *original cd00r first used this method, based on ports and SYN-packets). This condition is expressed here (it is based on the ID value input at startup by the attacker):</p>
<div style="color: #006a52;margin: 25px">ID = 123 Filter = (tcp[8:4] &#038; 0xe007ffff = 0xe003bebe) or (udp[12:4] &#038; 0xe007ffff = 0xe003bebe) ID = 321 Filter = (tcp[8:4] &#038; 0xe007ffff = 0x1bebe) or (udp[12:4] &#038; 0xe007ffff = 0x1bebe)</div>
<p>In simple terms, it checks for an ACK number in the TCP header, or the second byte from the UDP packet body.</p>
<p>If such a packet is received and the condition check is successful, execution jumps to the packet payload contents, and it creates a regular socket. The backdoor handles this socket as a file with read/write operations. It&#8217;s not the typical recv/send used in this code. It uses this new socket to connect to the source address of the &#8220;magic packets&#8221;. Then it reports its own PID and IP to the remote address, and starts an endless loop for receiving remote commands. When a command arrives, it is executed with a &#8220;/bin/sh -c &#8221; script.</p>
<p>Further analysis of the sample&#8217;s functionality will be updated here.</p>
<h2 id="conclusions">Conclusions</h2>
<p>Although Linux variants from the Turla framework were known to exist, we haven&#8217;t seen any in the wild yet.</p>
<p>This specific module appears to have been put together from public sources with some added functionality from the attackers. Some of the malicious code appears to be inactive, perhaps leftovers from older versions of the implant. Perhaps the most interesting part here is the unusual command and control mechanism based on TCP/UDP packets, as well as the C&#038;C hostname which fits previously known Turla activity.</p>
<p>The discovery of this Turla module rises one big question: how many other unknown Turla variants exist?</p>
<p><strong>Update:</strong> Since the publishing of this blogpost, we have discovered another Linux Turla module, which apparently represents a different malware generation than the previously known samples:</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2014/12/08065636/turla_linux_2.png" class="magnificImage"><img loading="lazy" decoding="async" class="alignnone size-full wp-image-68047" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2014/12/08065636/turla_linux_2.png" alt="turla_linux_2" width="964" height="663" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2014/12/08065636/turla_linux_2.png 964w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2014/12/08065636/turla_linux_2-300x206.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2014/12/08065636/turla_linux_2-768x528.png 768w" sizes="auto, (max-width: 964px) 100vw, 964px" /></a></p>
