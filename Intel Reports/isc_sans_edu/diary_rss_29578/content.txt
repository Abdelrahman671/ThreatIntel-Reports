<p><em><strong>Introduction</strong></em></p>

<p><a href="https://www.checkpoint.com/cyber-hub/threat-prevention/what-is-malware/icedid-malware/">IcedID</a> (also known as <a href="https://www.team-cymru.com/post/tracking-bokbot-icedid-infrastructuremapping-a-vast-and-currently-active-icedid-network">Bokbot</a>) is an information stealer/backdoor malware that can lead to other activity like Cobalt Strike and Virtual Network Computing (VNC) traffic.&nbsp; IcedID is often <a href="https://isc.sans.edu/diary/28884">distributed through email</a>, and we&#39;ve also seen it <a href="https://isc.sans.edu/diary/Google+ads+lead+to+fake+software+pages+pushing+IcedID+Bokbot/29344">delivered by fake software sites from Google ad traffic</a>.</p>

<p>For email-based distribution, we&#39;ve seen <a href="https://www.proofpoint.com/us/blog/threat-insight/onenote-documents-increasingly-used-to-deliver-malware">OneNote files</a> as an initial lure this month (<a href="https://twitter.com/k3dg3/status/1623333951069646857">here&#39;s one example</a>). But these distribution patterns occasionally change.&nbsp; For example, on Tuesday 2023-02-21, we found a distribution pattern using <span style="font-family:Courier New,Courier,monospace;">.url</span> files and WebDAV traffic for an IcedID infection.<br />
<br />
Today&#39;s diary reviews an infection from Thursday 2023-02-23 generated by one of those <span style="font-family:Courier New,Courier,monospace;">.url</span> files.</p>

<p><a href="https://isc.sans.edu/diaryimages/images/2023-02-24-ISC-diary-image-01.jpg"><img alt="" src="https://isc.sans.edu/diaryimages/images/2023-02-24-ISC-diary-image-01a.jpg" style="border-width: 2px; border-style: solid;" /></a><br />
<em>Shown above:&nbsp; Flow chart for the infection activity.</em></p>

<p><em><strong>The Discovery</strong></em></p>

<p>On Tuesday 2023-02-21, <a href="http://twitter.com/wwp96">@wwp96</a> tweeted about <a href="https://twitter.com/wwp96/status/1628126394487300096">an open directory</a> at <span style="font-family:Courier New,Courier,monospace;">hxxp://104.156.149[.]6/webdav/</span>.&nbsp; Searching VirusTotal revealed at least 22 <span style="font-family:Courier New,Courier,monospace;">.url</span> files that attempt to contact the server.&nbsp; These <span style="font-family:Courier New,Courier,monospace;">.url</span> files all use <span style="font-family:Courier New,Courier,monospace;">file:\\\\</span> instead of <span style="font-family:Courier New,Courier,monospace;">http://</span> for the URL, and they all grab a similarly-named <span style="font-family:Courier New,Courier,monospace;">.bat</span> file from the open directory.</p>

<p><a href="https://isc.sans.edu/diaryimages/images/2023-02-24-ISC-diary-image-02.jpg"><img alt="" src="https://isc.sans.edu/diaryimages/images/2023-02-24-ISC-diary-image-02a.jpg" style="border-width: 2px; border-style: solid;" /></a><br />
<em>Shown above:&nbsp; URL file and the associated BAT file.</em></p>

<p>The <span style="font-family:Courier New,Courier,monospace;">.bat</span> file runs a DLL installer for IcedID on the same server at <span style="font-family:Courier New,Courier,monospace;">\\104.156.149[.]6\webdav\host.dll</span>.</p>

<p><em><strong>WebDAV Traffic</strong></em></p>

<p>The <span style="font-family:Courier New,Courier,monospace;">.url</span> and <span style="font-family:Courier New,Courier,monospace;">.bat</span> files both use <a href="http://webdav.org/">WebDAV</a> to retrieve and run the malware.&nbsp; WebDAV stands for &quot;Web Distributed Authoring and Versioning,&quot; and it&#39;s a set of extensions to the HTTP protocol that allows users to access and edit files on a remote web server.</p>

<p>You can access WebDAV servers using Windows File Explorer. The image below shows what happened when we opened <span style="font-family:Courier New,Courier,monospace;">\\104.156.149[.]6\webdav\</span> in a File Explorer window.</p>

<p><a href="https://isc.sans.edu/diaryimages/images/2023-02-24-ISC-diary-image-03.jpg"><img alt="" src="https://isc.sans.edu/diaryimages/images/2023-02-24-ISC-diary-image-03a.jpg" style="border-width: 2px; border-style: solid;" /></a><br />
<em>Shown above:&nbsp; Opening the malicious WebDAV server in Windows File Explorer.</em></p>

<p>This WevDAV activity generated several HTTP <span style="font-family:Courier New,Courier,monospace;">PROPFIND</span> and <span style="font-family:Courier New,Courier,monospace;">GET</span> requests.&nbsp; While <span style="font-family:Courier New,Courier,monospace;">GET</span> requests are seen in almost any HTTP traffic, these <span style="font-family:Courier New,Courier,monospace;">PROPFIND</span> requests are specific to WebDAV.&nbsp; Reviewing our pcap of the IcedID infection in Wireshark, we can find several HTTP <span style="font-family:Courier New,Courier,monospace;">PROPFIND</span> requests over TCP port 80.</p>

<p><a href="https://isc.sans.edu/diaryimages/images/2023-02-24-ISC-diary-image-04.jpg"><img alt="" src="https://isc.sans.edu/diaryimages/images/2023-02-24-ISC-diary-image-04a.jpg" style="border-width: 2px; border-style: solid;" /></a><br />
<em>Shown above:&nbsp; Traffic from the infection filtered in Wireshark, highlighting the WebDAV PROPFIND requests.</em></p>

<p>Following TCP streams for any of the <span style="font-family:Courier New,Courier,monospace;">PROPFIND</span> requests reveals a Microsoft WebDAV user agent in the request headers.&nbsp; The WebDAV server returns an XML file with properties of the specified directory or file.</p>

<p><a href="https://isc.sans.edu/diaryimages/images/2023-02-24-ISC-diary-image-05.jpg"><img alt="" src="https://isc.sans.edu/diaryimages/images/2023-02-24-ISC-diary-image-05a.jpg" style="border-width: 2px; border-style: solid;" /></a><br />
<em>Shown above:&nbsp; TCP stream of a WebDAV PROPFIND request over HTTP.</em></p>

<p><em><strong>IcedID Traffic From The Infected Windows Host</strong></em></p>

<p>After the WebDAV activity, infection traffic was similar to previous IcedID infections.&nbsp; The only unusual activity was an HTTP <span style="font-family:Courier New,Courier,monospace;">GET</span> request using cURL to <span style="font-family:Courier New,Courier,monospace;">hxxp://mandalorecnote[.]com/images/</span> caused by the <span style="font-family:Courier New,Courier,monospace;">.bat</span> file.</p>

<p><a href="https://isc.sans.edu/diaryimages/images/2023-02-24-ISC-diary-image-06.jpg"><img alt="" src="https://isc.sans.edu/diaryimages/images/2023-02-24-ISC-diary-image-06a.jpg" style="border-width: 2px; border-style: solid;" /></a><br />
<em>Shown above:&nbsp; Traffic from the infection filtered in Wireshark.</em></p>

<p>The HTTP <span style="font-family:Courier New,Courier,monospace;">GET</span> request to <span style="font-family:Courier New,Courier,monospace;">hxxp://mandalorecnote[.]com/images/</span> returned a 12kB 64-bit DLL.&nbsp; This DLL doesn&#39;t appear to be used for the IcedID infection, and a cursory forensic investigation didn&#39;t find it saved to disk.&nbsp; It&#39;s likely a decoy file or decoy traffic, and the DLL doesn&#39;t immediately seem malicious.&nbsp; However, it&#39;s still an indicator for this specific wave of IcedID activity.</p>

<p><a href="https://isc.sans.edu/diaryimages/images/2023-02-24-ISC-diary-image-07.jpg"><img alt="" src="https://isc.sans.edu/diaryimages/images/2023-02-24-ISC-diary-image-07a.jpg" style="border-width: 2px; border-style: solid;" /></a><br />
<em>Shown above:&nbsp; 12 kB DLL returned from mandalorecnote[.]com, possible decoy traffic or file.</em></p>
