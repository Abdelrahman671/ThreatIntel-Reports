			<h1>DarkGate Loader Malware Delivered via Microsoft Teams</h1>

							<div class="wysiwyg"><p>Malspam campaigns involving DarkGate Loader have been on the rise since its author started advertising it as a Malware-as-a-Service offering on popular cybercrime forums in June 2023. Until now DarkGate Loader was seen delivered via traditional email malspam campaigns similar to those of Emotet. In August an operator started using Microsoft Teams to deliver the malware via HR-themed social engineering chat messages.</p>

			<div class="wysiwyg"><h2>Investigating the Senders</h2>
<p>Using <a href="https://compliance.microsoft.com/" target="_blank" rel="noreferrer noopener">Microsoft Purview&#8217;s eDiscovery tool</a> we searched for the senders (participants) in Microsoft Teams. </p>
<p>The senders of the external Microsoft Teams chat messages were identified as “Akkaravit Tattamanas” (63090101@my.buu.ac.th) and “ABNER DAVID RIVERA ROJAS” (adriverar@unadvirtual.edu.co). Truesec Threat Intelligence confirmed the accounts were compromised via an unknown malware and put up for sale on the Dark Web in August 2023.</p>
<p>Using <a href="https://aadinternals.com/osint/" target="_blank" rel="noreferrer noopener">AADInternal&#8217;s OSINT tool</a>, we could gather more information on the O365 tenant to which the accounts belong and use the listed domains to search for additional messages.</p>
<figure><img decoding="async" src="https://optimise2.assets-servd.host/jealous-emu/production/AAD1.png?w=691&#038;auto=compress%2Cformat&#038;fit=crop&#038;dm=1693999075&#038;s=f54f81701c8394971bce1c0e41234ca1" alt="" /></figure>
<figure><img decoding="async" src="https://optimise2.assets-servd.host/jealous-emu/production/AAD2.png?w=688&#038;auto=compress%2Cformat&#038;fit=crop&#038;dm=1693999078&#038;s=704d7baa4ccae3ffb0740ef29cd76e7d" alt="" /><figcaption>Figure 1: Screenshot from AADInternal&#8217;s OSINT tool with the sender&#8217;s O356 tenant details.</figcaption></figure>
<h2>HR-Themed Social Engineering Lure</h2>
<p>Both senders had an identical-sounding message with a link to an externally hosted file, &#8220;Changes to the vacation schedule.zip&#8221; (hosted on the senders SharePoint sites).</p>
<figure><img decoding="async" src="https://optimise2.assets-servd.host/jealous-emu/production/Teams_Message_2023-09-06-111840_xfme.png?w=1414&#038;auto=compress%2Cformat&#038;fit=crop&#038;dm=1693999120&#038;s=7e60d0ea24e5d7a25b18753104c8c0b9" alt="" /><figcaption>Figure 2: Screenshot of one of the MS Teams chat messages.</figcaption></figure>
<p>The SharePoint URLs hosting the remote attachment can be seen in the figure below.</p>
<figure><img decoding="async" src="https://optimise2.assets-servd.host/jealous-emu/production/URLs.png?w=3064&#038;auto=compress%2Cformat&#038;fit=crop&#038;dm=1693999779&#038;s=d309dd178c6537bd02577cbc44dc9d48" alt="" /><figcaption>Figure 3: URLs to the SharePoint sites hosting the remote ZIP file.</figcaption></figure>
<h2>Downloading the Malware</h2>
<p>Clicking the URL would take the victim to the SharePoint sites where the file “Changes to the vacation schedule.zip” could be downloaded.</p>
<figure><img decoding="async" src="https://optimise2.assets-servd.host/jealous-emu/production/Sharepoint-file.png?w=2614&#038;auto=compress%2Cformat&#038;fit=crop&#038;dm=1693999116&#038;s=efcd2491aa256b3d95c78d1d806ac9da" alt="" /><figcaption>Figure 4: Screenshot of a SharePoint site hosting the file &#8220;Changes to the vacation schedule.zip.&#8221;</figcaption></figure>
<p>The file was later identified by Microsoft Defender as malware “BAT/Tisifi.A#”.</p>
<figure><img decoding="async" src="https://optimise2.assets-servd.host/jealous-emu/production/MS-Defender2.png?w=602&#038;auto=compress%2Cformat&#038;fit=crop&#038;dm=1694000140&#038;s=83878fb42898f46235cf8ae97b9f587e" alt="" /><figcaption>Figure 5: Screenshot of MS Defender detecting the file as malicious.</figcaption></figure>
<h2>Analyzing the Malicious Files</h2>
<p>Using a combination of static and dynamic malware analysis our goal was to identify the final payload delivered in the campaign.</p>
<p>The ZIP file contains a malicious LNK file (shortcut) posing as a PDF document: &#8220;Changes to the vacation schedule.pdf.lnk.&#8221;</p>
<figure><img decoding="async" src="https://optimise2.assets-servd.host/jealous-emu/production/LNK-File_2023-09-06-113238_htnw.png?w=756&#038;auto=compress%2Cformat&#038;fit=crop&#038;dm=1693999958&#038;s=f1a70e6c31c7ec4aa126846290cc13d4" alt="" /><figcaption>Figure 6: Screenshot of the extracted LNK file as shown in File Explorer.</figcaption></figure>
<p>Using <a href="https://ericzimmerman.github.io" target="_blank" rel="noreferrer noopener">Eric Zimmerman’s </a>“LECmd.exe” to analyze the malicious LNK file, we can extract the command line it would execute upon opening.</p>
<figure><img decoding="async" src="https://optimise2.assets-servd.host/jealous-emu/production/Shell-cmd.png?w=3083&#038;auto=compress%2Cformat&#038;fit=crop&#038;dm=1694001838&#038;s=81f3aa58c7595bc0aaf5e8d79f1adb11" alt="" /><figcaption>Figure 7: Screenshot of the command executed after opening the LNK file.</figcaption></figure>
<p>The execution of the VBScript file in C:tgphasrxmp.vbs triggers the download and execution of the file hXXp:// 5[.]188[.]87[.]58:2351/wbzadczl</p>
<figure><img decoding="async" src="https://optimise2.assets-servd.host/jealous-emu/production/Wireshark.png?w=1027&#038;auto=compress%2Cformat&#038;fit=crop&#038;dm=1693999124&#038;s=0e051bb8a1da725b0a5ac9609e901931" alt="" /><figcaption>Figure 8: Wireshark trace of the VBScript file download.</figcaption></figure>
<p>The commands make use of a Windows version of cURL (renamed to wbza) to download and execute Autoit3.exe and the bundled script eszexz.au3. The pre-compiled AutoIT script hides the code in the middle of the file by looking for the magic bytes 0x4155332145413036 (AU3!EA06).</p>
<figure><img decoding="async" src="https://optimise2.assets-servd.host/jealous-emu/production/AuScript.png?w=817&#038;auto=compress%2Cformat&#038;fit=crop&#038;dm=1693999100&#038;s=04dd42bc71c6c33ebb24f8156d6c2e07" alt="" /><figcaption>Figure 9: Screenshot of the bundled AutoIT script file.</figcaption></figure>
<p>Upon executing the script, AutoIT drops a new file that contains shellcode, and before execution, it makes a check to see if Sophos antivirus is installed.</p>
<figure><img decoding="async" src="https://optimise2.assets-servd.host/jealous-emu/production/Sophos.png?w=1242&#038;auto=compress%2Cformat&#038;fit=crop&#038;dm=1693999119&#038;s=aada66965b320b6c9d869c308377ef8f" alt="" /><figcaption>Figure 10: The deobfuscated AutoIT script showing a check for Sophos antivirus.</figcaption></figure>
<p>If Sophos is not installed, additional code in the AutoIT script is deobfuscated to launch the shellcode.</p>
<figure><img decoding="async" src="https://optimise2.assets-servd.host/jealous-emu/production/DllCall.png?w=1578&#038;auto=compress%2Cformat&#038;fit=crop&#038;dm=1693999102&#038;s=4b46b045bcf363c1df1c44a6f927611e" alt="" /><figcaption>Figure 11: Screenshot of AutoIT shellcode execution.</figcaption></figure>
<p>When the shellcode is run, the first thing it does is load “byte by byte.” This technique is called stacked strings, to create a new file. It can be seen in the figure below that the first bytes of the created file are 0x4d and 0x5a, which indicates a Windows executable.</p>
<figure><img decoding="async" src="https://optimise2.assets-servd.host/jealous-emu/production/Shell.png?w=934&#038;auto=compress%2Cformat&#038;fit=crop&#038;dm=1694001339&#038;s=e9e4fc54a5fa8384964f923fc7182892" alt="" /><figcaption>Figure 12: Screenshot from Ghidra showing the shellcode&#8217;s use of stack strings to load a new Windows executable.</figcaption></figure>
<p>The payload could then be extracted from memory and analyzed with PE Studio from <a href="https://www.winitor.com" target="_blank" rel="noreferrer noopener">www.winitor.com</a>:</p>
<figure><img decoding="async" src="https://optimise2.assets-servd.host/jealous-emu/production/PEstudio.png?w=1016&#038;auto=compress%2Cformat&#038;fit=crop&#038;dm=1694002388&#038;s=953f452e41d3925f58104c206bbed461" alt="" /><figcaption>Figure 13: Screenshot from PE Studio showing technical details about the payload.</figcaption></figure>
<p>The payload was identified as “DarkGateLoader” on VirusTotal. After the identification of the malware, we found an excellent <a href="https://github.security.telekom.com/2023/08/darkgate-loader.html" target="_blank" rel="noreferrer noopener">writeup from Deutsche Telekom CERT</a> and used their <a href="https://github.com/telekom-security/malware_analysis/blob/main/darkgate/extractor.py" target="_blank" rel="noreferrer noopener">config extractor</a> on the AutoIT script file “eszexz.au3” to extract the DarkGate malware’s configuration:</p>
<figure><img decoding="async" src="https://optimise2.assets-servd.host/jealous-emu/production/Malware-Config.png?w=306&#038;auto=compress%2Cformat&#038;fit=crop&#038;dm=1693999113&#038;s=e620dfe0f44d7cdf3a28c3f44df122f7" alt="" /><figcaption>Figure 14: Configuration extracted from the DarkGate malware.</figcaption></figure>
<p>Further reading on the DarkGate Loader and DarkGate malware capabilities:</p>
<p><a href="https://github.security.telekom.com/2023/08/darkgate-loader.html" target="_blank" rel="noreferrer noopener">https://github.security.telekom.com/2023/08/darkgate-loader.html</a></p>
<p><a href="https://0xtoxin.github.io/threat%20breakdown/DarkGate-Camapign-Analysis/" target="_blank" rel="noreferrer noopener">https://0xtoxin.github.io/threat%20breakdown/DarkGate-Camapign-Analysis/</a></p>
<h2>Recommendations </h2>
<p>
This attack was detected due to the security awareness training of the recipients. Unfortunately, current Microsoft Teams security features such as <a href="https://learn.microsoft.com/en-us/microsoft-365/security/office-365-security/safe-attachments-for-spo-odfb-teams-about?view=o365-worldwide" target="_blank" rel="noreferrer noopener">Safe Attachments</a> or <a href="https://learn.microsoft.com/en-us/microsoft-365/security/office-365-security/safe-links-about?view=o365-worldwide#safe-links-settings-for-microsoft-teams" target="_blank" rel="noreferrer noopener">Safe Links</a> was not able to detect or block this attack. Right now, the only way to prevent this attack vector within Microsoft Teams is to only allow Microsoft Teams chat requests from specific external domains, albeit it might have business implications since all trusted external domains need to be whitelisted by an IT administrator.<br />
More on how these settings can be activated and used can be found here:<br />
<a href="https://learn.microsoft.com/en-us/microsoftteams/trusted-organizations-external-meetings-chat?tabs=organization-settings" target="_blank" rel="noreferrer noopener">https://learn.microsoft.com/en-us/microsoftteams/trusted-organizations-external-meetings-chat?tabs=organization-settings</a></p>
