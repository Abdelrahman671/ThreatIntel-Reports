        <div class="col col--body">
          <span id="hs_cos_wrapper_post_body" class="hs_cos_wrapper hs_cos_wrapper_meta_field hs_cos_wrapper_type_rich_text" style="" data-hs-cos-general-type="meta_field" data-hs-cos-type="rich_text"><p>The Sonatype Security Research team is currently tracking an ongoing campaign on the npm registry that uses npm packages to retrieve and exfiltrate your Kubernetes configuration and SSH keys to an external server.</p>
<!--more-->
<p>Our <a href="https://www.sonatype.com/products/software-supply-chain-management" rel="noopener" target="_blank"><span>automated malware detection systems</span></a> have identified at least 14 such packages thus far. We have been reporting these packages to the npm registry admins as these surface, following our analysis.</p>
<p>These packages, analyzed by Sonatype security researchers <a href="https://twitter.com/SnifferNandez" rel="noopener" target="_blank"><strong><span>Carlos Fernandez</span></strong></a> and <a href="https://www.linkedin.com/in/gustavo-rosa1/" rel="noopener" target="_blank"><strong><span>Gustavo Simoes</span></strong></a>, attempt to impersonate JavaScript libraries and components, such as ESLint plugins and TypeScript SDK tools. But, upon installation, multiple versions of the packages were seen running obfuscated code to collect and siphon sensitive files from the target machine.</p>
<p>Tracked as<strong> Sonatype-2023-4000</strong> and <strong>Sonatype-2023-4004</strong>, these packages are:</p>
<p style="padding-left: 40px;">@am-fe/hooks</p>
<p style="padding-left: 40px;">@am-fe/provider</p>
<p style="padding-left: 40px;">@am-fe/request</p>
<p style="padding-left: 40px;">@am-fe/utils</p>
<p style="padding-left: 40px;">@am-fe/watermark</p>
<p style="padding-left: 40px;">@am-fe/watermark-core</p>
<p style="padding-left: 40px;">@dynamic-form-components/mui</p>
<p style="padding-left: 40px;">@dynamic-form-components/shineout</p>
<p style="padding-left: 40px;">@expue/app</p>
<p style="padding-left: 40px;">@fixedwidthtable/fixedwidthtable</p>
<p style="padding-left: 40px;">@soc-fe/use</p>
<p style="padding-left: 40px;">@spgy/eslint-plugin-spgy-fe</p>
<p style="padding-left: 40px;">@virtualsearchtable/virtualsearchtable</p>
<p style="padding-left: 40px;">shineouts</p>
<p>Published in batches from different npm accounts, each of these packages scored no more than 200 downloads. A common factor linking these accounts was the use of the domain `app.threatest.com` within each of these malicious packages.</p>

<p>Secondly, the versions of the package do contain functional code and scripts borrowed from genuine open source packages, with some differences.</p>
<p>"The structure of the code looks like <em>typescript-sdk-tools</em> but the files are different," says Simoes.</p>

1872w" sizes="(max-width: 624px) 100vw, 624px"></p>
<p>Along with SSH keys and Kubernetes config, the script collects a system's basic fingerprinting information, such as username, IP address, hostname – a trend we have seen in <a href="/new-npm-poc-packages-target-paypal-zettle-airbnb-developers" rel="noopener" target="_blank"><span>basic PoC exploits</span></a> before. However, covert collection of these additional sensitive files with no warning whatsoever, combined with the deceptive metadata published to the npm registry for these packages should raise alarms and is a clear sign of malicious intent.</p>
<p>Fernandez also stressed that unauthorized access gained by the threat actor to a Kubernetes cluster could become problematic if they were to additionally <a href="https://www.akamai.com/blog/security-research/kubernetes-critical-vulnerability-command-injection" rel="noopener" target="_blank"><span>exploit recently disclosed vulnerabilities</span></a> against the cluster.</p>
<p>At the time of writing, the domain <strong>app.threatest[.]com</strong> being used by these packages was seen resolving to two Cloudflare IP addresses, 172.67.141.49, and 104.21.9.30, making it hard to attribute to a particular entity or known threat actors. The use of Mandarin in comments (i.e. "// 获取客户端外网 IP 地址" in line 49, which means "Get the client's external IP address") is also interesting but again, without additional insight, is by no means indicative of a specific threat actor.</p>
<p>We attempted to reach out multiple times to the purported publishers of these packages as well, both using the information present in the package metadata and the domain's WHOIS records, but did not hear back.</p>
<p>For all intents and purposes, based on the observations made so far, we are continuing to treat the packages in this campaign as malicious.</p>
