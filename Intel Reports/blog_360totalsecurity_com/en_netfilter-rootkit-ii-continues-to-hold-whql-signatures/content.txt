								<p><strong>1.Backgroud</strong></p>
<p>Recently, 360 Security Center discovered that a malicious driver &#8220;Netfilter rootkit&#8221; with WHQL signature was revealed in mid-June. WHQL signature means that after the hardware driver passed the Microsoft certification, Microsoft will add a &#8220;Hardware Compatibility Publisher&#8221; digital signature to the driver. The Netfilter rootkit has now been updated to the second generation and continues to hold the Microsoft signature. Moreover, the concealment of the upgraded Netfilter rootkit has increased so much that there is still no antivirus report on Virustotal.</p>
<p>In view of the fact that the second generation of Netfilter rootkit differs from the previous version in function and name, 360 Security Center named it &#8220;NetRedirect rootkit&#8221;. Although the NetRedirect rootkit has strong concealment and hazards, 360 Total Security can still achieve targeted defense and thorough investigation and killing, and fundamentally solve the user&#8217;s security problems.</p>
<p><a href="https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.1.png"><img class="aligncenter size-full wp-image-28634" src="https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.1.png" alt="" width="438" height="678" srcset="https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.1.png 438w, https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.1-194x300.png 194w" sizes="(max-width: 438px) 100vw, 438px" /></a></p>
<p>In fact, as early as June 25, the Microsoft Security Response Center stated that it had suspended the Netfilter rootkit account and reviewed other documents issued by it. “According to our zero-trust and layered defense security posture, we passed Microsoft Defender for Endpoint Built-in detection and blocking of this driver and related files.” However, its second-generation product, the NetRedirect rootkit, which is highly homologous and similar in behavior, still has a Microsoft signature, making it more concealed and difficult to detect and kill.</p>
<p>In addition, the harmfulness of NetRedirect rootkit has also been significantly improved. In view of the way that NetRedirect rootkit cloud controls the distribution of rootkits, the current malicious vendor is fully capable of not only being limited to the IP hijacking function, but also being able to implement any malicious rootkit execution on the infected devices.</p>
<ol start="2">
<li><strong> The hidden behavior of cloud-controlled malicious files and memory loading</strong></li>
</ol>
<p>Different from the previous generation of &#8220;Netfilter rootkit&#8221; verifying its own file md5 to implement file self-update, &#8220;NetRedirect rootkit&#8221; adopts the form of disguising the driver and the malicious driver, and the real malicious driver is stored on the Trojan C &amp; C server in a cloud-controlled manner, And the local masquerading as the driver of the WFP network filtering function is responsible for requesting malicious file data from the server, and calling the rootkit entry address in a concealed manner of memory loading.</p>
<p><a href="https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.2.png"><img class="aligncenter size-full wp-image-28640" src="https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.2.png" alt="" width="1322" height="1244" srcset="https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.2.png 1322w, https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.2-300x282.png 300w, https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.2-768x723.png 768w, https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.2-1024x964.png 1024w" sizes="(max-width: 1322px) 100vw, 1322px" /></a></p>
<p>The source of the &#8220;NetRedirect rootkit&#8221; is certain private server games. After the private server game runs, it will silently write to the driver registry service, release the NetRedirect.sys file to the %UserProfile%\AppData\Roaming directory, and load NetRedirect.sys .</p>
<p><a href="https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.3.png"><img class="aligncenter size-full wp-image-28646" src="https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.3.png" alt="" width="275" height="166" /></a></p>
<p>Subsequently, NetRedirect.sys, which has Microsoft&#8217;s signature, will request the real malicious driver from the server in the form of a socket:</p>
<p><a href="https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.4.png"><img class="aligncenter size-full wp-image-28652" src="https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.4.png" alt="" width="798" height="562" srcset="https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.4.png 798w, https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.4-300x211.png 300w, https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.4-768x541.png 768w" sizes="(max-width: 798px) 100vw, 798px" /></a></p>
<p>After obtaining the rootkit file data, the memory is self-loaded and the driver entry address is called:</p>
<p><a href="https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.5.png"><img class="aligncenter size-full wp-image-28658" src="https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.5.png" alt="" width="843" height="519" srcset="https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.5.png 843w, https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.5-300x185.png 300w, https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.5-768x473.png 768w" sizes="(max-width: 843px) 100vw, 843px" /></a></p>
<p>The Netfilter rootkit loaded in the memory is responsible for IP hijacking. It will repeatedly tamper with the HKCU\Software\Microsoft\Windows\CurrentVersion\Internet Settings\AutoConfigURL key value item, and finally achieve the purpose of IP hijacking:</p>
<p><a href="https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.6.png"><img class="aligncenter size-full wp-image-28664" src="https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.6.png" alt="" width="730" height="364" srcset="https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.6.png 730w, https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.6-300x150.png 300w" sizes="(max-width: 730px) 100vw, 730px" /></a></p>
<p>The partial hijacking list is as follows:</p>
<p><a href="https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.7.png"><img class="aligncenter size-full wp-image-28670" src="https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.7.png" alt="" width="763" height="364" srcset="https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.7.png 763w, https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.7-300x143.png 300w" sizes="(max-width: 763px) 100vw, 763px" /></a></p>
<p>It is monitored that the &#8220;NetRedirect rootkit&#8221; does not belong to any module&#8217;s memory thread, and tampering with the registry AutoConfigURL key value:</p>
<p><a href="https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.8.png"><img class="aligncenter size-full wp-image-28676" src="https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.8.png" alt="" width="547" height="234" srcset="https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.8.png 547w, https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.8-300x128.png 300w" sizes="(max-width: 547px) 100vw, 547px" /></a></p>
<p><a href="https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.9.png"><img class="aligncenter size-full wp-image-28682" src="https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.9.png" alt="" width="824" height="414" srcset="https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.9.png 824w, https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.9-300x151.png 300w, https://static.ts.360.com/blog/wp-content/uploads/2021/07/29.9-768x386.png 768w" sizes="(max-width: 824px) 100vw, 824px" /></a></p>
