			<h1 class="entry-title">BazarLoader to Conti Ransomware in 32 Hours</h1>
		<div class="entry-meta">
			<span class="posted-on"><a href="https://thedfirreport.com/2021/09/13/bazarloader-to-conti-ransomware-in-32-hours/" rel="bookmark"><time class="entry-date published" datetime="2021-09-13T00:13:02+00:00">September 13, 2021</time></a></span>		</div><!-- .entry-meta -->

	<div class="entry-content">
		<div class="markdown">
<h3 id="intro">Intro</h3>
<p>Conti is a top player in the ransomware ecosystem, being listed as 2nd overall in the <a href="https://www.coveware.com/blog/2021/7/23/q2-ransom-payment-amounts-decline-as-ransomware-becomes-a-national-security-priority" target="_blank" rel="nofollow noopener">Q2 2021 Coveware ransomware report</a>. The groups deploying this RaaS have only grown more <a href="https://twitter.com/AltShiftPrtScn/status/1431423623621918727?s=20" target="_blank" rel="noopener">prevalent</a>. Despite the group having it&#8217;s affiliate guide leaked, which revealed many techniques already covered in <a href="https://thedfirreport.com/2021/08/01/bazarcall-to-conti-ransomware-via-trickbot-and-cobalt-strike/" target="_blank" rel="nofollow noopener">previous reports</a>, the group&#8217;s using the ransomware are unlikely to let up any time soon.</p>
<p>In July we witnessed a BazarLoader campaign that deployed Cobalt Strike and ended with domain wide encryption using Conti ransomware.</p>
<h3 id="case-summary">Case Summary</h3>
<p>BazarLoader has continued to be one of the preeminent initial access brokers for ransomware threat actor access. For this intrusion we don&#8217;t know the initial campaign that deployed the malware but based on previous information, we can assess with high confidence that the delivery vector was a malicious email campaign. At the time of the intrusion, the group was <a href="https://twitter.com/malware_traffic/status/1412470165179092992" target="_blank" rel="nofollow noopener">favoring zip attachments</a> with malicious javascript files to download the BazarLoader malware. However BazarLoader has also been used with <a href="https://twitter.com/James_inthe_box/status/1422191753218576392?s=20" target="_blank" rel="nofollow noopener">Word</a> and <a href="https://thedfirreport.com/2021/03/08/bazar-drops-the-anchor/" target="_blank" rel="nofollow noopener">Excel</a> documents as well.</p>
<p>In this case we observed the initial activity beginning with a BazarLoader DLL. Upon initial execution on the beachhead, the malware made an initial connection to command and control, and then a few minutes later it performed discovery tasks on the host using Microsoft utilities like Net and Nltest to discover the domain and users of interest. like domain administrators. After this activity, the host went quiet for about one hour before downloading and executing a Cobalt Strike beacon DLL.</p>
<p>The threat actors used Cobalt Strike to run additional discovery tasks using Microsoft utilities like net, ping, systeminfo, and taskmanager. The threat actors then began using pass the hash with various accounts which continued several times throughout the intrusion. To see what machines were active in the environment, the threat actors scanned the network for SMB.</p>
<p>Around two and a half hours into the intrusion the threat actors began lateral movement. Lateral movement began by the threat actor transferring an executable to a remote system and then executing it using wmic. This was the primary lateral movement option favored by the threat actor, however PowerShell Cobalt Strike beacons, service executable Cobalt Strike beacons, and RDP were all used, but less commonly. Once on remote systems the threat actor used Cobalt Strike to dump lsass memory for further credentials.</p>
<p>After this phase completed, the threat actor&#8217;s activity faded but the Cobalt Strike continued to beacon out to the C2 server. About 12 hours later the threat actors became active again. From the domain controller the threat actors continued further lateral movement to more servers in the environment. They also continued further discovery activity running PowerShell scripts to discover the disk utilization of hosts, review user last login time per host, assess the installed anti-virus software, and track which hosts were online for the threat actors to target.</p>
<p>When the threat actors identified the file server, their method for data exfiltration was straightforward to a fault. They downloaded WinSCP from the project website, installed it on the file server and proceeded to exfiltrate data from the server using SCP to a VPS host they controlled in Romania.</p>
<p>Around 31 hours after initial access to the environment, the threat actors felt they were ready to complete their final objectives. RDP activity was seen from several hosts and an executable named test.exe was transferred to several endpoints. This test file was the Conti ransomware executable, and the threat actors decided to test in a controlled manner before running the full domain ransomware deployment. Like before, these &#8220;unit tests,&#8221; were performed using wmic to execute the files remotely on the endpoints.</p>
<p>The threat actors must have confirmed quickly that their tests were successful as within minutes they dropped test.exe renamed to backup.exe on two servers in the environment and executed manually via their RDP sessions. When executed in this manner the ransomware mounts all remote C$ drives in the local network and proceeds to encrypt the contents over the SMB connection. At this point, the Time to Ransom (TTR) for the threat actors was just shy of 32 hours since initial access.</p>

<h3 id="mitre-attck">MITRE ATT&amp;CK</h3>
<h3 id="initial-access">Initial Access</h3>
<p>In this case we did not observe the initial delivery for the malware. BazarLoader however tends to arrive in an environment via malicious email campaigns and in a few cases its been reported via <a href="https://unit42.paloaltonetworks.com/bazarloader-malware/" target="_blank" rel="nofollow noopener">call centers</a> social engineering users to load the malware. Seeing that this starts with a DLL file it is more likely that this was related to an email campaign using malicious <a href="https://twitter.com/malware_traffic/status/1412470165179092992" target="_blank" rel="nofollow noopener">zipped Javascript</a> files.</p>
<h3 id="execution">Execution</h3>

<pre><code>nltest /domain_trusts /all_trusts

<p>In addition to these command and control methods, one more network anomaly was observed. This was not used for primary command and control and the amount of data sent was small so we do not know the full intentions of the activity but several critical systems like domain controllers and file servers made connections to TOR nodes initiated by the threat actors.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/09/65c2a30ef8a12723f81341b984424b3c3d3e1c954d72b66a29f5f9c1a704502e.png" alt="enter image description here" /></p>
<h3 id="exfiltration">Exfiltration</h3>
<p>The threat actor on the second day of the intrusion downloaded WinSCP to the file server and proceeded to install the program there.</p>
<pre><code>C:\Users\REDACTED\AppData\Local\Temp\<span class="hljs-number">1</span>\is-HCFKT.tmp\WinSCP-<span class="hljs-number">5.19</span><span class="hljs-number">.1</span>-Setup.tmp<span class="hljs-string">" /SL5="</span>$A02B0,<span class="hljs-number">10288106</span>,<span class="hljs-number">864256</span>,C:\Users\USER\Desktop\WinSCP-<span class="hljs-number">5.19</span><span class="hljs-number">.1</span>-Setup.exe<span class="hljs-string">"</span>
</code></pre>
<p>The threat actor then proceeded to connect over port 22 to a server in Romania.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/09/f348f67ef89090bbfc0a6dbf0ed0a991bc5407ad7343330cc4278b8d13a24a08.png" alt="enter image description here" /></p>
<p>As the traffic was encrypted we can&#8217;t conclusively determine what data was exfiltrated. However we can infer that the choice to deploy on the file server was due to the data present and ease to move the data.</p>
<p>Another data point is that following the exfiltration canary documents present in the shares reported in as being opened from an IP on a Virtual Private Host provider in New York, USA.</p>
<h3 id="impact">Impact</h3>
<p>During the overnight hours of the 2nd day the threat actors began moving on their final objectives. This included testing their ransomware in the compromised environment before deploying across the domain.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/09/ca58f3e4e67ad87f182addb8f788c7a7ddd46ed64030d3b812f759310ea22d4d.png" alt="enter image description here" /></p>
<p>They initiated RDP connections and a Cobalt Strike beacon executable file to a endpoint not yet interacted with by the threat actors. The threat actor then transferred a Conti executable file to several endpoints named test.exe.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/09/f417213c761742e26088002aa58fcb7bbcaf9c6476210f70ffb8561dc62d0a65.png" alt="enter image description here" /></p>
<p>These test ransom files were then called remotely using wmic as seen in the previous lateral movement activity.</p>
<pre><code>C:\Windows\system32\cmd.exe /C wmic /node:"ENDPOINT" process <span class="hljs-operator"><span class="hljs-keyword">call</span> <span class="hljs-keyword">create</span> <span class="hljs-string">"C:\test.exe"</span></span>
</code></pre>
<p>After testing on several endpoints, the threat actors dropped a renamed version of the file on several servers in the environment and executed by hand using their RDP session.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/09/cf4d9744a30cc4546ade85d71cdf113cc8863008012c01e9474785d638a17d8f.png" alt="enter image description here" /></p>
<p>When executed in this manner, the ransomware payload attempts to spread laterally over SMB.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/09/823b02bd3a4ae97e0e3b1c7df9ee032c7fc955144f9b912703472a9ca24993ad.png" alt="enter image description here" /></p>
<p>From there, the threat actors left the environment with this note and domain wide encryption completed about 32 hours after the initial beachhead BazarLoader was executed.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2021/09/4b0ae07d39e5c2840b2ce4d44b5061a209d5b2144953eb2fec17635a49dd824e.png"><img loading="lazy" decoding="async" class="alignnone size-full wp-image-4039" src="https://thedfirreport.com/wp-content/uploads/2021/09/4b0ae07d39e5c2840b2ce4d44b5061a209d5b2144953eb2fec17635a49dd824e.png" alt="" width="1446" height="399" srcset="https://thedfirreport.com/wp-content/uploads/2021/09/4b0ae07d39e5c2840b2ce4d44b5061a209d5b2144953eb2fec17635a49dd824e.png 1446w, https://thedfirreport.com/wp-content/uploads/2021/09/4b0ae07d39e5c2840b2ce4d44b5061a209d5b2144953eb2fec17635a49dd824e-300x83.png 300w, https://thedfirreport.com/wp-content/uploads/2021/09/4b0ae07d39e5c2840b2ce4d44b5061a209d5b2144953eb2fec17635a49dd824e-1024x283.png 1024w, https://thedfirreport.com/wp-content/uploads/2021/09/4b0ae07d39e5c2840b2ce4d44b5061a209d5b2144953eb2fec17635a49dd824e-768x212.png 768w" sizes="auto, (max-width: 1446px) 100vw, 1446px" /></a></p>
