<p>In 2020 and 2021, Volexity identified multiple compromises related to a relatively unknown criminal threat actor that refers to itself as "XE Group". Volexity believes that XE Group is likely a Vietnamese-origin criminal threat actor whose intrusions follow an approximate pattern:</p>
<ul>
<li>Compromise of externally facing services via known exploits (e.g., Telerik UI vulnerabilities)</li>
<li>Monetization of these compromises through installation of password theft or credit card skimming code for web services related to these servers</li>
</ul>
<p>There has been previously reported XE Group activity in a blog <a href="https://blog.malwarebytes.com/threat-analysis/2020/07/credit-card-skimmer-targets-asp-net-sites/"> by Malwarebytes</a> from 2020; this post serves to provide additional  insight into XE Group and an update on its current operations.</p>
<h2>Analysis</h2>
<p>Volexity first encountered XE Group activity in early 2020 following a web server compromise at a customer site. The breach of the web server was automated, and it was remediated quickly after discovery, with no notable actions taken by the attacker. That one incident was a small blip on the radar and appeared rather isolated. Recently, however, Volexity detected XE Group activity across several of its customers by way of Volexity's Network Security Monitoring service. This time, the activity was coming from client systems rather than web servers. A deeper dive into the activity revealed clients were visiting compromised websites and potentially falling victim to a widespread credit card skimming operation.</p>
<p>After finding this new XE Group activity, Volexity investigated and quickly found infrastructure used by the attacker. In many cases, it was not difficult to enumerate the attackers' infrastructure using basic pivots such as WHOIS registrants and passive DNS. Much of this infrastructure still tied back to items described in the aforementioned Malwarebytes blog. The attacker uses email addresses relating to “xe[word]” and has consistently used the same custom nameservers (ns1.xegroups[.]com and ns2.xegroups[.]com) with their domains for a number of years.</p>
<p>An overview of the infrastructure used in the last three years is given in Figure 1 below. A full list can be found on Github <a href="https://github.com/volexity/threat-intel/blob/main/2021/2021-12-06%20-%20XEGroup/indicators/indicators.csv">here</a>.</p>
<p><img decoding="async" class="wp-image-2319 size-full aligncenter" src="https://www.volexity.com/wp-content/uploads/2021/11/maltego_replacement_image.jpg" alt="" width="966" height="595" srcset="https://www.volexity.com/wp-content/uploads/2021/11/maltego_replacement_image.jpg 966w, https://www.volexity.com/wp-content/uploads/2021/11/maltego_replacement_image-300x185.jpg 300w, https://www.volexity.com/wp-content/uploads/2021/11/maltego_replacement_image-768x473.jpg 768w" sizes="(max-width: 966px) 100vw, 966px" /></p>
<p style="text-align: center;"><em>Figure 1. XE Group infrastructure can be closely linked via passive DNS, WHOIS data, and other patterns</em></p>
<p>In the Figure 1, each red dot is a malware sample. A large number of the samples belong to a group of malware families that the attacker refers to collectively as “XEReverseShell”. Despite the singular name, in practice several of these offer different functionality. The malware families are mostly written in .NET and AutoIT, although the attacker sometimes uses loaders for their malware family written in C++.</p>
<p>Sometimes, the attacker even labels <a href="https://www.virustotal.com/gui/file/42b71d9548a8dfac0e4b3fd3946329972bdc0e156402ca47e6bb1cd58ab9f77c/details">their malware</a> as belonging to XE Group, as shown below:</p>
<p><img decoding="async" class="wp-image-2320 size-large aligncenter" src="https://www.volexity.com/wp-content/uploads/2021/11/image003-1024x99.png" alt="" width="1024" height="99" srcset="https://www.volexity.com/wp-content/uploads/2021/11/image003-1024x99.png 1024w, https://www.volexity.com/wp-content/uploads/2021/11/image003-300x29.png 300w, https://www.volexity.com/wp-content/uploads/2021/11/image003-768x74.png 768w, https://www.volexity.com/wp-content/uploads/2021/11/image003.png 1145w" sizes="(max-width: 1024px) 100vw, 1024px" /></p>
<p style="text-align: center;"><em>Figure 2. Sometimes the malware authors label malware with their brand.</em></p>
<p>Despite this, the bread and butter of attacks from XE Group is credit card skimming, and the other tools Volexity has identified are only used to help facilitate this purpose.</p>
<h2>Credit Card Skimming</h2>
<p>The most recent credit card skimming activity identified by Volexity was found based on suspicious redirects to the domain “object[.]fm”, a domain using the xegroups[.]com nameserver.  The code used to load the malicious JavaScript from this page reveals that the attacker uses an interesting technique: the JavaScript keyword "object" is used to populate the domain value:</p>
<p><img loading="lazy" decoding="async" class="wp-image-2321 size-full aligncenter" src="https://www.volexity.com/wp-content/uploads/2021/11/image005.png" alt="" width="951" height="155" srcset="https://www.volexity.com/wp-content/uploads/2021/11/image005.png 951w, https://www.volexity.com/wp-content/uploads/2021/11/image005-300x49.png 300w, https://www.volexity.com/wp-content/uploads/2021/11/image005-768x125.png 768w" sizes="auto, (max-width: 951px) 100vw, 951px" /></p>
<p style="text-align: center;"><em>Figure 3. Example code added to compromised sites to load malicious JavaScript</em></p>
<p>The effect of this code is to append a script to the loading page where the "src" value is as follows:</p>
<blockquote><p>typeof({}).fm/gmt.js</p></blockquote>
<p>Since typeof({}) evaluates to "object", this becomes:</p>
<blockquote><p>object[.]fm/gmt.js</p></blockquote>
<p>Volexity searched for other websites containing similar scripts to load additional JavaScript and found that the attacker is using the same method to load scripts from the domain "<strong>object.sbs</strong>".</p>
<p>The <a href="https://www.virustotal.com/gui/file/348e8494d32e6f7e8dcbe1f295ca9efd2af047d92d73eaf0091cd20efe48eb79">gmt.js file</a> loaded on infected sites is used for credit card skimming and shares much of the same code as the <a href="https://github.com/MBThreatIntel/skimmers/blob/master/null_gif_skimmer.js">original skimmer</a> identified by Malwarebytes. The effect of the credit card skimmer is that any form data submitted to pages loading the JavaScript in question is also exfiltrated to an attacker URL. An example of the type of form data stolen is shown below:</p>
<blockquote><p>{"rcgnAdultsCheckBoxon”:””,”firstNameTextBox”:”[name]”:,”lastNameTextBox”:”[surname]”:,”birthdateTextBox”:”[date]”,”genderCodeDropDown”:”[gender]”,”emailAddressTextBox”:”[email_address]”,”relationshipDropDown”:”[relation]”,”txtCardNumber”:”1111-2222-3333-4444:”,”ddlExpirationMonth”:”[month]”,”ddlExpirationYear:”[year],”txtSecurityCode”:”[code]"}</p></blockquote>
<p>The new version of the credit card skimming code shows slight differences to the 2020 version:</p>
<ul>
<li>There is additional use of “.join()” and .”replace()” to rebuild obfuscated strings.</li>
<li>Since the attacker is using "object" for their subdomains, they use another trick to obfuscate the domain name in the script itself:</li>
</ul>
<blockquote><p>k = typeof {}</p>
<p>r = location.toString().replace(/^(.*?\:\/\/).*?(\.).*$/g, '$1' + k+'$2')</p></blockquote>
<p>Here, "k" is a JavaScript Object; "r" takes the current location (e.g., the current URL) as a string and replaces the entire location with the contents of "k" (i.e., "object").</p>
<ul>
<li>The URI used to send stolen data to is pseudo randomized using arrays of words and random integers.</li>
<li>Functionality to look for passwords has been removed in the cases Volexity has identified.</li>
<li>Additional checks are done within the script to make sure the window has finished loading before key functionality is run by the script. This avoids a race condition between scripts on the infected page and the malicious JavaScript.</li>
<li>In older versions of the credit card skimmer, exfiltrated data was encoded in plain hex in the URL. In newer versions, colons and tilde characters are used to denote key value pairs and end of field markers respectively. A script to decode an exfiltration URL is provided on GitHub <a href="https://github.com/volexity/threat-intel/blob/main/2021/2021-12-06%20-%20XEGroup/scripts/decode_uri.py">here</a>.</li>
</ul>
<p>Additionally, in some cases the attacker appears to find the results of their standard credit card skimmer are insufficient or hard to parse, so they have included additional code to skim specific fields from compromised sites, <a href="https://www.virustotal.com/gui/file/f2642c6179a06c3a98d9e5499ebc14c249e34e61fb88d065b0125ca1ad7290d5/details">an example of which is given below</a>:</p>
<p><img loading="lazy" decoding="async" class="wp-image-2322 size-large aligncenter" src="https://www.volexity.com/wp-content/uploads/2021/11/image007-1024x403.png" alt="" width="1024" height="403" srcset="https://www.volexity.com/wp-content/uploads/2021/11/image007-1024x403.png 1024w, https://www.volexity.com/wp-content/uploads/2021/11/image007-300x118.png 300w, https://www.volexity.com/wp-content/uploads/2021/11/image007-768x302.png 768w, https://www.volexity.com/wp-content/uploads/2021/11/image007-1536x604.png 1536w, https://www.volexity.com/wp-content/uploads/2021/11/image007.png 1703w" sizes="auto, (max-width: 1024px) 100vw, 1024px" /></p>
<p style="text-align: center;"><em>Figure 4. Additional skimming code included on one compromised page to scrape specific fields</em></p>
<h2>Affected Websites</h2>
<p>After identifying a number of attacker command-and-control (C2) servers, Volexity searched various data sources—including its own telemetry—in order to discover websites that are compromised and hosting the credit card skimmer described above. Compromised sites vary a great deal in popularity and nature; examples of compromised sites include the following industries/sectors:</p>
<ul>
<li>Travel</li>
<li>Restaurant</li>
<li>Art &amp; Culture</li>
<li>Non-profit</li>
</ul>
<p>The variety of affected sites suggests no clear strategy on the attacker's part, and therefore suggests it may be a case of attempted bulk exploitation and indiscriminate attacks.</p>
<p>In one case, however, the attacker appears to have been able to compromise a web design company that specializes in designing and maintaining websites for storefronts. This means that each site owned by that company is currently serving a credit card skimmer.</p>
<h2>Who Is XE Group?</h2>
<p>Volexity believes XE Group is likely a Vietnamese-origin threat actor; several things led to this conclusion. First, the WHOIS information for several of the malicious domains is for an individual located in Vietnam:</p>
<p><img loading="lazy" decoding="async" class="wp-image-2323 size-large aligncenter" src="https://www.volexity.com/wp-content/uploads/2021/11/image009-1024x169.png" alt="" width="1024" height="169" srcset="https://www.volexity.com/wp-content/uploads/2021/11/image009-1024x169.png 1024w, https://www.volexity.com/wp-content/uploads/2021/11/image009-300x50.png 300w, https://www.volexity.com/wp-content/uploads/2021/11/image009-768x127.png 768w, https://www.volexity.com/wp-content/uploads/2021/11/image009-1536x254.png 1536w, https://www.volexity.com/wp-content/uploads/2021/11/image009.png 2008w" sizes="auto, (max-width: 1024px) 100vw, 1024px" /></p>
<p style="text-align: center;"><em>Figure 5. Screenshot from DomainTools Iris showing WHOIS information for “xemembers.com”</em></p>
<p>While this information is often faked, there is indeed <a href="https://github.com/xethanh">a user with the name</a> “Joe Nguyen”, username "xethanh", with an empty repository “xegroupdev” present on GitHub.</p>
<p><img loading="lazy" decoding="async" class="wp-image-2324 size-large aligncenter" src="https://www.volexity.com/wp-content/uploads/2021/11/image011-1024x505.png" alt="" width="1024" height="505" srcset="https://www.volexity.com/wp-content/uploads/2021/11/image011-1024x505.png 1024w, https://www.volexity.com/wp-content/uploads/2021/11/image011-300x148.png 300w, https://www.volexity.com/wp-content/uploads/2021/11/image011-768x379.png 768w, https://www.volexity.com/wp-content/uploads/2021/11/image011-1536x757.png 1536w, https://www.volexity.com/wp-content/uploads/2021/11/image011-2048x1010.png 2048w" sizes="auto, (max-width: 1024px) 100vw, 1024px" /></p>
<p style="text-align: center;"><em>Figure 6. GitHub profile of "Joe Nguyen" / "xethanh" which are terms seen in WHOIS records for known C2 addresses</em></p>
<p>This is a largely unused GitHub profile using a custom "XE" avatar and referencing both "Joe Nguyen" (the same name seen in the previously mentioned WHOIS records) and "xegroupdev".</p>
<p>A user with username “xethanh” previously existed on the now-abandoned forum crdclub[.]su which advertised stolen credit card information:</p>
<p><img loading="lazy" decoding="async" class="wp-image-2325 size-large aligncenter" src="https://www.volexity.com/wp-content/uploads/2021/11/image013-1024x42.png" alt="" width="1024" height="42" srcset="https://www.volexity.com/wp-content/uploads/2021/11/image013-1024x42.png 1024w, https://www.volexity.com/wp-content/uploads/2021/11/image013-300x12.png 300w, https://www.volexity.com/wp-content/uploads/2021/11/image013-768x31.png 768w, https://www.volexity.com/wp-content/uploads/2021/11/image013-1536x62.png 1536w, https://www.volexity.com/wp-content/uploads/2021/11/image013-2048x83.png 2048w" sizes="auto, (max-width: 1024px) 100vw, 1024px" /></p>
<p style="text-align: center;"><em>Figure 7. Post on crdclub[.]su shows a user with the same handle seen on Github ("xethanh") advertising stolen credit card data</em></p>
<p>Users with the same name exist on other carding forums as well, such as cybercarders[.]su and cardingforum[.]co. In a non-malware twist, there is also a YouTube profile with the handle "xethanh" which uses the same name and XE avatar as the Github profile.</p>
<p><img loading="lazy" decoding="async" class="wp-image-2326 aligncenter" src="https://www.volexity.com/wp-content/uploads/2021/11/image015_cropped.png" alt="" width="706" height="303" srcset="https://www.volexity.com/wp-content/uploads/2021/11/image015_cropped.png 974w, https://www.volexity.com/wp-content/uploads/2021/11/image015_cropped-300x129.png 300w, https://www.volexity.com/wp-content/uploads/2021/11/image015_cropped-768x330.png 768w" sizes="auto, (max-width: 706px) 100vw, 706px" /></p>
<p style="text-align: center;"><em>Figure 8. YouTube profile for user "xethanh" with the same avatar and username as the previously mentioned Github profile</em></p>
<p>Crucially, in the About section of the YouTube profile, there is a link to xegroups[.]com/xethanh. The site xegroups[.]com is a C2 for older malware samples and a nameserver for newer infrastructure used by the group. While the name "Joe Nguyen" may be a pseudonym, it seems most likely that the user behind the GitHub and YouTube accounts is the attacker.</p>
<p>It could be that XE Groups is a compromised organization of some kind, but this seems unlikely since a number of the tools used by the attacker over a long period of time has used "xe[word]" nomenclature in descriptions, original filenames, and PDB strings. Furthermore, a number of the malware files discovered in VirusTotal were first observed being uploaded from Vietnamese web users, many of whom only have a single VirusTotal upload. It is likely these were uploaded by the attacker in order to test detection rates of their malware prior to deploying them in real environments.</p>
<h2>Conclusion</h2>
<p>XE Group's credit card skimming operation has been ongoing since at least early 2020, using a relatively limited set of infrastructure. The attacker primarily focuses on compromising IIS environments and uses their access to deploy credit card skimming JavaScript code on affected websites.</p>
<p>There is a relatively clear trail of evidence to help identify XE Group. The attacker appears to be of Vietnamese origin. They often use the brand “XE Group”, and a number of "xe[word]" themed domains have been registered and linked to the group. Volexity has identified a likely persona on carding forums associated with this activity, which suggests the attacker monetizes stolen credit card data through sales rather than direct use of stolen cards themselves. The persona used for the GitHub and carding account, and several of the domains, have a history going back to 2013, which suggests the attacker may have been attempting similar attacks for up to eight years, with only one significant public mention of their activity. The oldest malware sample Volexity was able to identify <a href="https://www.virustotal.com/gui/file/02c48917b15015ddd02738bc1f480f9c6379165618435855030f4c63ce372485">relating to XE Group dates back to late 2014</a>.</p>
<p>To prevent these specific attacks from being successful, Volexity recommends the following:</p>
<ul>
<li>Block related network indicators provided (<a href="https://github.com/volexity/threat-intel/blob/main/2021/2021-12-06%20-%20XEGroup/indicators/indicators.csv">link</a>).</li>
<li>Use signatures provided (<a href="https://github.com/volexity/threat-intel/blob/main/2021/2021-12-06%20-%20XEGroup/indicators/yara.yar">link</a>) to identify related activity.</li>
</ul>
