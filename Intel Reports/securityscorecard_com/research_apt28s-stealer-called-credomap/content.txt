					<h2>Executive summary: CredoMap APT28 Malware</h2>
<p>CredoMap is a stealer developed by the Russian APT28/Sofacy/Fancy Bear that was<br />
used to target users in Ukraine in the context of the ongoing war between Russia and Ukraine. The <a href="https://securityscorecard.com/blog/what-is-malware/">malware</a> was initially discovered by <a href="https://blog.google/threat-analysis-group/update-on-cyber-activity-in-eastern-europe/">Google</a> and <a href="https://cert.gov.ua/article/341128">CERT-UA</a>. The threat actor weaponized a document to exploit the Follina (CVE-2022-30190) vulnerability that would result in downloading the .NET stealer. The malware aims to steal the credentials and cookies from Google Chrome, Mozilla Firefox, and Microsoft Edge. The data exfiltration is done by sending information to a possibly compromised C2 server via the IMAP email protocol.</p>
<h2>Analysis and findings of CredoMap APT28 Malware</h2>
<p>SHA256: 2318ae5d7c23bf186b88abecf892e23ce199381b22c8eb216ad1616ee8877933</p>
<p>The process retrieves the path of the current executable and then connects to a hard-coded C2 server (162.241.216.236) on port 143 (IMAP) using hard-coded credentials:</p>
<figure id="attachment_1771" aria-describedby="caption-attachment-1771" style="width: 874px" class="wp-caption aligncenter"><img fetchpriority="high" decoding="async" class="size-full wp-image-1771" src="https://securityscorecard.com/wp-content/uploads/2024/01/figure-1.png" alt="figure-1" width="874" height="230" srcset="https://securityscorecard.com/wp-content/uploads/2024/01/figure-1.png 874w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-1-300x79.png 300w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-1-768x202.png 768w" sizes="(max-width: 874px) 100vw, 874px" /><figcaption id="caption-attachment-1771" class="wp-caption-text">Figure 1</figcaption></figure>
<figure id="attachment_1772" aria-describedby="caption-attachment-1772" style="width: 852px" class="wp-caption aligncenter"><img decoding="async" class="size-full wp-image-1772" src="https://securityscorecard.com/wp-content/uploads/2024/01/figure-2.webp" alt="figure-2" width="852" height="305" srcset="https://securityscorecard.com/wp-content/uploads/2024/01/figure-2.webp 852w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-2-300x107.webp 300w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-2-768x275.webp 768w" sizes="(max-width: 852px) 100vw, 852px" /><figcaption id="caption-attachment-1772" class="wp-caption-text">Figure 2</figcaption></figure>
<p>The malware creates a TcpClient object, obtains a client stream for reading and writing, and then reads the response from the server:</p>
<figure id="attachment_1773" aria-describedby="caption-attachment-1773" style="width: 650px" class="wp-caption aligncenter"><img decoding="async" class="size-full wp-image-1773" src="https://securityscorecard.com/wp-content/uploads/2024/01/figure-3.png" alt="figure-3" width="650" height="370" srcset="https://securityscorecard.com/wp-content/uploads/2024/01/figure-3.png 650w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-3-300x171.png 300w" sizes="(max-width: 650px) 100vw, 650px" /><figcaption id="caption-attachment-1773" class="wp-caption-text">Figure 3</figcaption></figure>
<p>The binary performs the login operation and reads the response using the Read method:</p>
<figure id="attachment_1774" aria-describedby="caption-attachment-1774" style="width: 761px" class="wp-caption aligncenter"><img decoding="async" class="size-full wp-image-1774" src="https://securityscorecard.com/wp-content/uploads/2024/01/figure-4.png" alt="figure-4" width="761" height="261" srcset="https://securityscorecard.com/wp-content/uploads/2024/01/figure-4.png 761w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-4-300x103.png 300w" sizes="(max-width: 761px) 100vw, 761px" /><figcaption id="caption-attachment-1774" class="wp-caption-text">Figure 4</figcaption></figure>
<p>It selects the INBOX folder using the SELECT command and performs multiple function calls that steal the browsers’ credentials and cookies:</p>
<figure id="attachment_1775" aria-describedby="caption-attachment-1775" style="width: 807px" class="wp-caption aligncenter"><img decoding="async" class="size-full wp-image-1775" src="https://securityscorecard.com/wp-content/uploads/2024/01/figure-5.png" alt="figure-5" width="807" height="182" srcset="https://securityscorecard.com/wp-content/uploads/2024/01/figure-5.png 807w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-5-300x68.png 300w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-5-768x173.png 768w" sizes="(max-width: 807px) 100vw, 807px" /><figcaption id="caption-attachment-1775" class="wp-caption-text">Figure 5</figcaption></figure>
<figure id="attachment_1776" aria-describedby="caption-attachment-1776" style="width: 762px" class="wp-caption aligncenter"><img decoding="async" class="size-full wp-image-1776" src="https://securityscorecard.com/wp-content/uploads/2024/01/figure-6.webp" alt="figure-6" width="762" height="243" srcset="https://securityscorecard.com/wp-content/uploads/2024/01/figure-6.webp 762w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-6-300x96.webp 300w" sizes="(max-width: 762px) 100vw, 762px" /><figcaption id="caption-attachment-1776" class="wp-caption-text">Figure 6</figcaption></figure>
<p>The sample verifies if the file “\Google\Chrome\User Data\Default\Network\Cookies” exists in the Local AppData folder by calling the File.Exists function:</p>
<figure id="attachment_1777" aria-describedby="caption-attachment-1777" style="width: 975px" class="wp-caption aligncenter"><img decoding="async" class="size-full wp-image-1777" src="https://securityscorecard.com/wp-content/uploads/2024/01/figure-7.png" alt="figure-7" width="975" height="70" srcset="https://securityscorecard.com/wp-content/uploads/2024/01/figure-7.png 975w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-7-300x22.png 300w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-7-768x55.png 768w" sizes="(max-width: 975px) 100vw, 975px" /><figcaption id="caption-attachment-1777" class="wp-caption-text">Figure 7</figcaption></figure>
<p>The File.Copy method is used to copy the above file to a new file called “cc”:</p>
<figure id="attachment_1778" aria-describedby="caption-attachment-1778" style="width: 975px" class="wp-caption aligncenter"><img decoding="async" class="size-full wp-image-1778" src="https://securityscorecard.com/wp-content/uploads/2024/01/figure-8.png" alt="figure-8" width="975" height="129" srcset="https://securityscorecard.com/wp-content/uploads/2024/01/figure-8.png 975w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-8-300x40.png 300w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-8-768x102.png 768w" sizes="(max-width: 975px) 100vw, 975px" /><figcaption id="caption-attachment-1778" class="wp-caption-text">Figure 8</figcaption></figure>
<p>The malicious binary opens a connection to the Cookies database and executes an SQL query that extracts some fields:</p>
<figure id="attachment_1779" aria-describedby="caption-attachment-1779" style="width: 975px" class="wp-caption aligncenter"><img decoding="async" class="size-full wp-image-1779" src="https://securityscorecard.com/wp-content/uploads/2024/01/figure-9.png" alt="figure-9" width="975" height="61" srcset="https://securityscorecard.com/wp-content/uploads/2024/01/figure-9.png 975w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-9-300x19.png 300w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-9-768x48.png 768w" sizes="(max-width: 975px) 100vw, 975px" /><figcaption id="caption-attachment-1779" class="wp-caption-text">Figure 9</figcaption></figure>
<p>The process opens and reads the file called “Local/Google/Chrome/User Data/Local State” using File.ReadAllText. It extracts the Base64-encoded random key that is encrypted with DPAPI from JSON([“os_crypt”][“encrypted_key”]). The key is Base64-decoded and decrypted via a function call to ProtectedData.Unprotect:</p>
<figure id="attachment_1780" aria-describedby="caption-attachment-1780" style="width: 975px" class="wp-caption aligncenter"><img decoding="async" class="size-full wp-image-1780" src="https://securityscorecard.com/wp-content/uploads/2024/01/figure-10.png" alt="figure-10" width="975" height="80" srcset="https://securityscorecard.com/wp-content/uploads/2024/01/figure-10.png 975w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-10-300x25.png 300w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-10-768x63.png 768w" sizes="(max-width: 975px) 100vw, 975px" /><figcaption id="caption-attachment-1780" class="wp-caption-text">Figure 10</figcaption></figure>
<p>The binary creates an AESEngine object, an AEADParameters object containing the decrypted AES-128 key and the nonce (12 bytes), and calls the GcmBlockCipher.Init function with a “False” parameter (decryption operation):</p>
<figure id="attachment_1781" aria-describedby="caption-attachment-1781" style="width: 899px" class="wp-caption aligncenter"><img decoding="async" class="size-full wp-image-1781" src="https://securityscorecard.com/wp-content/uploads/2024/01/figure-11.webp" alt="figure-11" width="899" height="175" srcset="https://securityscorecard.com/wp-content/uploads/2024/01/figure-11.webp 899w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-11-300x58.webp 300w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-11-768x149.webp 768w" sizes="(max-width: 899px) 100vw, 899px" /><figcaption id="caption-attachment-1781" class="wp-caption-text">Figure 11</figcaption></figure>
<p>The “encrypted_value” extracted from the Cookies database is decrypted using the ProcessBytes and DoFinal methods:</p>
<figure id="attachment_1782" aria-describedby="caption-attachment-1782" style="width: 975px" class="wp-caption aligncenter"><img decoding="async" class="size-full wp-image-1782" src="https://securityscorecard.com/wp-content/uploads/2024/01/figure-12.png" alt="figure-12" width="975" height="87" srcset="https://securityscorecard.com/wp-content/uploads/2024/01/figure-12.png 975w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-12-300x27.png 300w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-12-768x69.png 768w" sizes="(max-width: 975px) 100vw, 975px" /><figcaption id="caption-attachment-1782" class="wp-caption-text">Figure 12</figcaption></figure>
<p>The resulting values are stored in a dictionary that has the keys as “host_key” with values “name=&lt;Decrypted encrypted_value&gt;;”, as highlighted in the figure below.</p>
<figure id="attachment_1783" aria-describedby="caption-attachment-1783" style="width: 805px" class="wp-caption aligncenter"><img decoding="async" class="size-full wp-image-1783" src="https://securityscorecard.com/wp-content/uploads/2024/01/figure-13.png" alt="figure-13" width="805" height="285" srcset="https://securityscorecard.com/wp-content/uploads/2024/01/figure-13.png 805w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-13-300x106.png 300w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-13-768x272.png 768w" sizes="(max-width: 805px) 100vw, 805px" /><figcaption id="caption-attachment-1783" class="wp-caption-text">Figure 13</figcaption></figure>
<p>Finally, the process serializes the dictionary to a JSON string using JsonConvert.SerializeObject:</p>
<figure id="attachment_1784" aria-describedby="caption-attachment-1784" style="width: 803px" class="wp-caption aligncenter"><img decoding="async" class="size-full wp-image-1784" src="https://securityscorecard.com/wp-content/uploads/2024/01/figure-14.png" alt="figure-14" width="803" height="86" srcset="https://securityscorecard.com/wp-content/uploads/2024/01/figure-14.png 803w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-14-300x32.png 300w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-14-768x82.png 768w" sizes="(max-width: 803px) 100vw, 803px" /><figcaption id="caption-attachment-1784" class="wp-caption-text">Figure 14</figcaption></figure>
<p>The data exfiltration occurs by issuing a valid IMAP APPEND command. The “From” field is set to the username obtained from the Environment.UserName property, the “Subject” field is set to the current date and time on the computer obtained from the DateTime.UtcNow property, and the JSON string is also included in the command (see figure 15).</p>
<figure id="attachment_1785" aria-describedby="caption-attachment-1785" style="width: 714px" class="wp-caption aligncenter"><img decoding="async" class="size-full wp-image-1785" src="https://securityscorecard.com/wp-content/uploads/2024/01/figure-15.webp" alt="figure-15" width="714" height="398" srcset="https://securityscorecard.com/wp-content/uploads/2024/01/figure-15.webp 714w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-15-300x167.webp 300w" sizes="(max-width: 714px) 100vw, 714px" /><figcaption id="caption-attachment-1785" class="wp-caption-text">Figure 15</figcaption></figure>
<p>The malware verifies if the file “\Google\Chrome\User Data\Default\Login Data” exists in the Local AppData folder using File.Exists:</p>
<figure id="attachment_1786" aria-describedby="caption-attachment-1786" style="width: 975px" class="wp-caption aligncenter"><img decoding="async" class="size-full wp-image-1786" src="https://securityscorecard.com/wp-content/uploads/2024/01/figure-16.png" alt="figure-16" width="975" height="96" srcset="https://securityscorecard.com/wp-content/uploads/2024/01/figure-16.png 975w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-16-300x30.png 300w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-16-768x76.png 768w" sizes="(max-width: 975px) 100vw, 975px" /><figcaption id="caption-attachment-1786" class="wp-caption-text">Figure 16</figcaption></figure>
<p>The File.Copy function is utilized to copy the above file to a new file called “cp”:</p>
<figure id="attachment_1787" aria-describedby="caption-attachment-1787" style="width: 975px" class="wp-caption aligncenter"><img decoding="async" class="size-full wp-image-1787" src="https://securityscorecard.com/wp-content/uploads/2024/01/figure-17.png" alt="figure-17" width="975" height="177" srcset="https://securityscorecard.com/wp-content/uploads/2024/01/figure-17.png 975w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-17-300x54.png 300w, https://securityscorecard.com/wp-content/uploads/2024/01/figure-17-768x139.png 768w" sizes="(max-width: 975px) 100vw, 975px" /><figcaption id="caption-attachment-1787" class="wp-caption-text">Figure 17</figcaption></figure>
<p>The binary opens a connection to the Login Data database and executes an SQL query that extracts the “action_url”, “username_value”, and “password_value” fields:</p>
