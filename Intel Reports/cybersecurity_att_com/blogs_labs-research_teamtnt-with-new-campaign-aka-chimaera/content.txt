							<div class="blog-title-date-author-area">
								<h1>TeamTNT with new campaign aka &#8220;Chimaera&#8221;</h1>
								<div class="date">September 8, 2021 &nbsp;|&nbsp;
									<a href="/blogs/author/ofer-caspi">Ofer Caspi</a></div>
							</div>
							<div class="blog-body">
								<h2>Executive summary</h2>

<p>LevelBlue Labs&trade; has discovered a new campaign by threat group TeamTNT that is targeting multiple operating systems and applications. The campaign uses multiple shell/batch scripts, new open source tools, a cryptocurrency miner, the TeamTNT IRC bot, and more.</p>

<p>Alien Labs research indicates the command and control (C&amp;C) server used in this newly discovered campaign contains infection statistics that suggest TeamTNT has been running this campaign since July 25, 2021, and that it is responsible for thousands of infections globally.</p>

<h2>Key takeaways:</h2>

<ul>
	<li>TeamTNT is using new, open source tools to steal usernames and passwords from infected machines.</li>
	<li>The group is targeting various operating systems including: Windows, different Linux distributions including Alpine (used for containers), AWS, Docker, and Kubernetes.</li>
	<li>The campaign has been active for approximately one month and is responsible for &nbsp;thousands of infections globally.</li>
	<li>As of August 30, 2021, many malware samples still have zero antivirus (AV) detections and others have low detection rates.</li>
</ul>

<h2>Background</h2>

<p>TeamTNT has been one of the most active threat groups since mid 2020. Their activity typically uses open source tools for malicious activity. A partial list of imported tools contains:</p>

<ul>
	<li>Masscan and port scanner to search for new infection candidates</li>
	<li><a href="https://github.com/gianlucaborello/libprocesshider" target="_blank">libprocesshider</a>&nbsp;for executing their bot directly from memory</li>
	<li>7z to decompress downloaded files</li>
	<li><a href="https://github.com/b374k/b374k" target="_blank">b374k shell</a>&nbsp;which is a php web administrator that can be used to control infected systems</li>
	<li>Lazagne, an open-source tool for multiple web operating systems, which is used to collect stored credentials from numerous applications</li>
</ul>

<p>Several recent publications, such the <a href="https://www.trendmicro.com/vinfo/us/security/news/cybercrime-and-digital-threats/teamtnt-activities-probed?utm_source=trendmicroresearch&amp;utm_medium=smk&amp;utm_campaign=0721_teamtnt" target="_blank">one by TrendMicro</a>, have described in detail TeamTNT campaigns, including the tools and techniques they use. One of the most <a href="https://unit42.paloaltonetworks.com/teamtnt-operations-cloud-environments/" target="_blank">recent findings</a> (June 4, 2021) came from Palo Alto&nbsp;researchers who discovered the TeamTNT Chimaera repository. In July 2021, TeamTNT began running the Chimaera campaign using new tools, and they began publishing infection statistics publicly on their website for the first time (Figures 1, 2).</p>

<p>As of the publishing of this report, many of the samples analyzed by Alien Labs have zero or low detection on <a href="https://www.virustotal.com/gui/file/caeb6eb1ee40fc4ac1da020a9a7542cffe55d29339306f6adf2d1e20e638538a/detection" target="_blank">VirusTotal</a>. However, defenders can be proactive in hardening their systems. For example, due to the recent, high profile attacks on Kubernetes &mdash; including those executed by TeamTNT &mdash;&nbsp; the National Security Agency (NSA) and the Cybersecurity and Infrastructure Security Agency (CISA) published&nbsp;&ldquo;<a href="https://www.nsa.gov/Press-Room/News-Highlights/Article/Article/2716980/nsa-cisa-release-kubernetes-hardening-guidance/" target="_blank">Kubernetes Hardening Guidance</a>&rdquo; in August of this year. Defenders should reference this guide to understand how to better defend against attacks like those used by TeamTNT.</p>

<p style="text-align:center"><img alt="Chimaera stats on main screen" src="https://cdn-cybersecurity.att.com/blog-content/chimaera_home.png" /></p>

<p style="text-align:center">Figure 1. TeamTNT C&amp;C website showing infection statistics</p>

<p style="text-align:center"><img alt="Chimaera C&amp;C" src="https://cdn-cybersecurity.att.com/blog-content/chimaera_cc.jpg" /></p>

<p style="text-align:center">Figure 2. TeamTNT C&amp;C website showing Chimaera campaign statistics</p>

<h2>Analysis of components used in the &ldquo;Chimaera&rdquo; campaign</h2>

<h3>New credentials stealer (&ldquo;Lazagne&rdquo; component)</h3>

<p>The malicious script starts its activity by modifying the bash history file. This hides any future commands executed from users using the &ldquo;history&rdquo; command on Linux.</p>

<p>The script then installs its dependencies (&lsquo;curl&rsquo;, &lsquo;bash&rsquo;, &lsquo;wget&rsquo;, &lsquo;pip&rsquo;, &lsquo;py3-pip&rsquo;, &lsquo;python3-pip&rsquo;). As seen in figure 3, supported operating systems include different Linux distributions, such as Alpine Linux which is typically used in containers.</p>

<p><img alt="Lazagne stage 1" src="https://cdn-cybersecurity.att.com/blog-content/lazagne_stage_1.jpg" /></p>

<p style="text-align:center">Figure 3. The first stage of the Lazagne component.</p>

<p>Once the malware is finished with its &ldquo;pre-setup,&rdquo; it downloads the second phase of the attack from its C&amp;C, which includes another bash script (&lsquo;run.sh&rsquo;) along with the Lazagne project, as seen in figure 4.</p>

<p><img alt="Lazagne stage 2" src="https://cdn-cybersecurity.att.com/blog-content/lazagne_stage_2.jpg" /></p>

<p style="text-align:center">Figure 4. Second stage of the Lazagne component (&lsquo;run.sh&rsquo;).</p>

<p>Lazagne is an open-source project available for different operating systems (Windows, Linux, and MacOS). Its developer describes the Lazagne tool as an application that can be used to retrieve multiple passwords stored on a local machine. Due to its capabilities, the tool has been added as a post exploitation module to the <a href="https://github.com/n1nj4sec/pupy/" target="_blank">pupy</a> project.</p>

<p>It supports a wide range of programs, such as browsers (Chrome, Firefox, Opera, etc.), Sysadmin programs (such as CoreFTP, Putty, OpenSSH, etc. ), Wifi password, mail programs, databases, etc. The full list of supported programs can be found on the <a href="https://github.com/AlessandroZ/LaZagne" target="_blank">Lazagne page</a> on Github.</p>

<p>In this phase two of the attack, the second malicious script executes the Lazagne tool, saves its output into &ldquo;laZagne.out.txt,&rdquo; and uploads it to the C&amp;C using the curl command. At the end of the execution, the malware deletes any file that has been downloaded.</p>

<h3>Windows component &ndash; Set up a cryptocurrency miner</h3>

<p>For Windows operating systems, the attackers use a malicious script that downloads all the tools required for unpacking and executing the Xmrig miner. This includes the 7z tool for decompressing downloaded files and Nssm to add the miner as a service.&nbsp;(See figure <span dir="RTL">5</span>.)</p>

<p><img alt="Chimaera windows module" src="https://cdn-cybersecurity.att.com/blog-content/chimaera_windows_component.jpg" /></p>

<p style="text-align:center">Figure 5. Windows module</p>

<p>The malware will setup the miner and then the miner will persist it in the system in two ways: 1) by adding itself as a service&nbsp;if the malware gains admin privileges or 2) by adding the batch file to the startup folder. (See figure <span dir="RTL">6</span>.)</p>

<p><img alt="Chimaera windows persistence" src="https://cdn-cybersecurity.att.com/blog-content/chimaera_windows_persistence.jpg" /></p>

<p style="text-align:center">Figure 6. Windows module - persistence</p>

<h3>Kubernetes root payload component</h3>

<p>This component is mainly responsible for installing a cryptocurrency miner on infected devices, allowing the attacker to connect remotely to the system using SSH. (See figure 7)</p>

<p><img alt="Kubernetes root payload" src="https://cdn-cybersecurity.att.com/blog-content/kubernetes_root_payload.jpg" /></p>

<p style="text-align:center">Figure 7. Kubernetes root payload</p>

<p>The malicious script uses the following steps to achieve its goal:</p>

<ul>
	<li>Disabling or uninstalling security products on infected machines, such as Aegis Authenticator, quartz, and Alibaba services (AliSecGuard, AliYunDun, AliNet etc.). (Figures 8, 9)</li>
	<li>Adding the attacker&rsquo;s RSA-key to the list of known SSH host (allowing the attacker to connect the machine through SSH without the need of user/password in the system).</li>
	<li>Installing missing required tools for crypto mining.</li>
	<li>Modifying the host file.</li>
	<li>Setting up the XMRig crypto miner.</li>
	<li>Adding persistence for the XMR miner.</li>
	<li>Removing itself.</li>
</ul>

<p><img alt="Chimaera stopping security services" src="https://cdn-cybersecurity.att.com/blog-content/chimaera_stopping_security_services.jpg" /></p>

<p style="text-align:center">Figure 8. Stopping security services</p>

<h3>TeamTNT AWS stealer</h3>

<p>Similar to the other TeamTNT components, the AWS stealer (see figure 11) first installs missing dependencies. It then collects information from infected devices and stores the informaiton in a temporary file &ldquo;/var/tmp/TeamTNT_AWS_STEALER.txt&rdquo;.</p>

<p><img alt="TeamTNT AWS stealer" src="https://cdn-cybersecurity.att.com/blog-content/teamtnt_aws_stealer.jpg" /></p>

<p style="text-align:center">Figure 11. AWS stealer</p>

<p>This information includes:</p>

<ul>
	<li>AWS default region</li>
	<li>AWS access key Id</li>
	<li>AWS secret access key</li>
	<li>AWS session token</li>
	<li>AWS user credentials</li>
	<li>AWS root credentials</li>
	<li>Shared credentials file</li>
	<li>Container credential relative URI</li>
</ul>

<p>When finished, the malware uploads all of the stored information to its C&amp;C using curl command, and then it cleans up its traces.</p>

<h2>Conclusion</h2>

<p>LevelBlue&nbsp;Labs has discovered new malicious files distributed by the threat actor TeamTNT. As researches have observed of TeamTNT in older campaigns, they are focusing on stealing cloud systems credentials, using infected systems for cryptocurrency mining, and abusing victim&rsquo;s machines to search and spread to other vulnerable systems. The use of open-source tools like Lazagne allows TeamTNT to stay below the radar for a while, making it more difficult for anti-virus companies to detect.</p>

<h2>Recommended actions</h2>

<ol>
	<li>Keep your software with the latest security updates.</li>
	<li>Keep minimal exposure to the Internet on Linux servers and IoT devices and use a properly configured firewall.</li>
	<li>Monitor network traffic, outbound port scans, and unreasonable bandwidth usage.</li>
</ol>
