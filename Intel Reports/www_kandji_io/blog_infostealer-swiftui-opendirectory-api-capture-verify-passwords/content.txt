            <span id="hs_cos_wrapper_post_body" class="hs_cos_wrapper hs_cos_wrapper_meta_field hs_cos_wrapper_type_rich_text" style="" data-hs-cos-general-type="meta_field" data-hs-cos-type="rich_text"><p>On July 29, <a href="https://x.com/4n6bexaminer?lang=en" rel="noopener" target="_blank"><span>@4n6Bexaminer</span></a> tweeted about <a href="https://x.com/4n6Bexaminer/status/1817871971893551538" rel="noopener" target="_blank"><span>a new macOS stealer</span></a>. Moments later, <a href="https://x.com/Huntio?lang=en" rel="noopener" target="_blank"><span>Hunt.io</span></a> tweeted about the same new malware and then released <a href="https://hunt.io/blog/macos-malware-impersonates-the-unarchiver-app-to-steal-user-data" rel="noopener" target="_blank"><span>a blog post</span></a> about it on July 30. That post focused primarily on the malicious bash scripts that were downloaded from the command-and-control (C2) server and then executed as the second stage.</p>
<!--more-->
<p>We wanted to take a close look at the first stage—specifically at the stealer’s <a href="https://en.wikipedia.org/wiki/Dropper_(malware)" rel="noopener" target="_blank"><span>dropper</span></a>, which is written in Swift and leverages APIs not seen in other recent stealers to capture and verify the user’s password. Many detection methods focus on OSAscript; this malware takes a different approach to evade detection.&nbsp;</p>
<h2><span style="font-size: 40px; color: #333333;">Password Prompt</span></h2>
<h3><strong><span style="font-size: 32px; color: #333333;">sub_1000061f8</span></strong></h3>
<p>We will start with the lock image seen in the prompt.</p>
<p><img src="https://blog.kandji.io/hs-fs/hubfs/lock%20modal.png?width=600&amp;height=682&amp;name=lock%20modal.png" width="600" height="682" loading="lazy" alt="lock modal" style="height: auto; max-width: 100%; width: 600px; margin-left: auto; margin-right: auto; display: block;" srcset="https://blog.kandji.io/hs-fs/hubfs/lock%20modal.png?width=300&amp;height=341&amp;name=lock%20modal.png 300w, https://blog.kandji.io/hs-fs/hubfs/lock%20modal.png?width=600&amp;height=682&amp;name=lock%20modal.png 600w, https://blog.kandji.io/hs-fs/hubfs/lock%20modal.png?width=900&amp;height=1023&amp;name=lock%20modal.png 900w, https://blog.kandji.io/hs-fs/hubfs/lock%20modal.png?width=1200&amp;height=1364&amp;name=lock%20modal.png 1200w, https://blog.kandji.io/hs-fs/hubfs/lock%20modal.png?width=1500&amp;height=1705&amp;name=lock%20modal.png 1500w, https://blog.kandji.io/hs-fs/hubfs/lock%20modal.png?width=1800&amp;height=2046&amp;name=lock%20modal.png 1800w" sizes="(max-width: 600px) 100vw, 600px"></p>
<p>The lock image is initialized using the <code>Image.init(_:bundle:)</code>&nbsp;call. The <code>resizable</code> method is then passed, which returns a pointer to the Image view.</p>
<pre>10000642c&nbsp; &nbsp; int64_t Image = Image.init(_:bundle:)('lock', 0xe400000000000000, 0)<br>&lt;...&gt;<br>100006468&nbsp; &nbsp; int64_t resizeView = Image.resizable(capInsets:resizingMode:)(x22, Image, v0, v1, v2, v3)</pre>
<p>The prompt text is created using <code>StringInterpolation</code> and a function (which I renamed <code>getNameOfApp</code>()) that queries for the name of the application.&nbsp;</p>
<pre>10000658c    LocalizedStringKey.Strin...(literalCapacity:interpolationCount:)(0x43, 1)<br>1000065ac&nbsp; &nbsp; <span style="color: #37761d;">// A user password must be entered to allow the "</span><br>1000065ac&nbsp; &nbsp; LocalizedStringKey.StringInterpolation.appendLiteral(_:)(-0x2fffffffffffffd2, -0x7ffffffeffff52d0)<br>1000065b0&nbsp; &nbsp; int64_t AppName_3 = <span style="color: #ff0201;">getNameOfApp()</span><br>1000065b8&nbsp; &nbsp; LocalizedStringKey.StringInterpolation.appendInterpolation(_:)()<br>1000065c0&nbsp; &nbsp; _swift_bridgeObjectRelease(AppName_3)<br>1000065dc&nbsp; &nbsp; <span style="color: #37761d;">// " application to run.</span><br>1000065dc&nbsp; &nbsp; LocalizedStringKey.StringInterpolation.appendLiteral(_:)(-0x2fffffffffffffeb, -0x7ffffffeffff52a0)<br>1000065e4&nbsp; &nbsp; LocalizedStringKey.init(stringInterpolation:)(x20)</pre>
<h3><strong><span style="font-size: 32px; color: #333333;">getNameOfApp()</span></strong></h3>
<p>This function uses the <code>[NSBundle mainBundle]</code> object to query for the <code>CFBundleDisplayName</code> inside the Info.plist of the application bundle.
