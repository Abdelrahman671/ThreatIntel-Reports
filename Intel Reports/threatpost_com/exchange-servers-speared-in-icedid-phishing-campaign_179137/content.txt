																														<div class="c-article__intro" name="overview">
											<p>The ever-evolving malware shows off new tactics that use email thread hijacking and other obfuscation techniques to provide advanced evasion techniques.</p>
										</div>
																				<div class="c-article__content js-reading-content">
											<p>The ever-evolving <a href="https://threatpost.com/icedid-web-forms-google-urls/165347/" target="_blank" rel="noopener">banking trojan IcedID</a> is back again with a phishing campaign that uses previously compromised Microsoft Exchange servers to send emails that appear to come from legitimate accounts. Attackers also are using stealthy new payload-delivery tactics to spread the modular malware.</p>
<p>Researchers from <a href="https://www.intezer.com/" target="_blank" rel="noopener">Intezer</a> earlier this month uncovered the campaign, which employs thread hijacking to send malicious messages from stolen Exchange accounts, thus adding an extra level of evasion to the campaign&#8217;s malicious intent, wrote researchers <a href="https://www.intezer.com/author/jkennedy/" target="_blank" rel="noopener">Joakim Kennedy</a> and <a href="https://www.intezer.com/author/ryanrobinson/" target="_blank" rel="noopener">Ryan Robinson</a> <a href="https://www.intezer.com/blog/research/conversation-hijacking-campaign-delivering-icedid/" target="_blank" rel="noopener">in a blog post</a> published Monday.</p>
<p>The actors behind IcedID – as well as other spearphishers – have previously used phishing emails that &#8220;reuse previously stolen emails to make the lure more convincing,&#8221; researchers wrote. However, this time the threat has evolved in a couple of key ways that make it even more dangerous to targets, which include organizations within energy, healthcare, law and pharmaceutical sectors, researchers noted.</p>
<p><a href="https://threatpost.com/infosec-insider-subscription-page/?utm_source=ART&amp;utm_medium=ART&amp;utm_campaign=InfosecInsiders_Newsletter_Promo/" target="_blank" rel="noopener"><img decoding="async" class="aligncenter wp-image-168544 size-full" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/103/2021/07/10165815/infosec_insiders_in_article_promo.png" alt="Infosec Insiders Newsletter" width="700" height="50" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/103/2021/07/10165815/infosec_insiders_in_article_promo.png 700w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/103/2021/07/10165815/infosec_insiders_in_article_promo-300x21.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/103/2021/07/10165815/infosec_insiders_in_article_promo-680x50.png 680w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/103/2021/07/10165815/infosec_insiders_in_article_promo-140x10.png 140w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/103/2021/07/10165815/infosec_insiders_in_article_promo-540x39.png 540w" sizes="(max-width: 700px) 100vw, 700px" /></a></p>
<p>Not only is the threat actor now using compromised Microsoft Exchange servers to send the phishing emails from the account that they stole from, but the delivery of the malicious payload also has shifted in a way that can execute malware without the user even knowing, researchers said.</p>
<p>&#8220;The payload has also moved away from using office documents to the use of ISO files with a Windows LNK file and a DLL file,&#8221; researchers wrote. &#8220;The use of ISO files allows the threat actor to bypass the <a href="https://attack.mitre.org/techniques/T1553/005/" target="_blank" rel="noopener">Mark-of-the-Web</a> controls, resulting in execution of the malware without warning to the user.&#8221;</p>
<p>Previously the infection chain most commonly associated with IcedID phishing campaigns has been an email with an attached password-protected ZIP archive that contains a macro-enabled Office document, which executes the IcedID installer.</p>
<h2><b>Breakdown of the Attack Chain</b></h2>
<p>The new campaign starts with a phishing email that includes a message about an important document and includes a password-protected ZIP archive file attached, the password for which is included in the email body.</p>
<p>The email seems extra convincing to users because it uses what&#8217;s called &#8220;thread hijacking,&#8221; in which attackers use a portion of a previous thread from a legitimate email found in the inbox of the stolen account.</p>
<p>&#8220;By using this approach, the email appears more legitimate and is transported through the normal channels which can also include security products,&#8221; researchers wrote.</p>
<p>The majority of the originating Exchange servers that researchers observed in the campaign appear to be unpatched and publicly exposed, &#8220;making the ProxyShell vector a good theory,&#8221; they wrote. <a href="https://threatpost.com/exchange-servers-attack-proxyshell/168661/" target="_blank" rel="noopener">ProxyShell</a> is a remote-code execution (RCE) bug discovered in Exchange Servers last year that has since been patched but has been <a href="https://threatpost.com/proxyshell-attacks-unpatched-exchange-servers/168879/" target="_blank" rel="noopener">throttled by attackers</a>.</p>
<p>Once unzipped, the attached file includes a single &#8220;ISO&#8221; file with the same file name as the ZIP archive that was created not that long before the email was sent. That ISO file includes two files: a LNK file named &#8220;document&#8221; and a DLL file named &#8220;main,&#8221; also prepared relatively recently and potentially used in previous phishing email, researchers said.</p>
<p>When a user double clicks the LNK file, it uses &#8220;regsvr32&#8221; to execute the DLL file, which allows for proxy execution of malicious code in main.dll for defense evasion, they wrote in the post. The DLL file is a loader for the IcedID payload.</p>
<p>The loader will locate the encrypted payload, which is stored in the resource section of the binary, through the technique API hashing. The resulting hash is then compared with a hardcoded hash, locating the call for FindResourceA, which is dynamically called to fetch the encrypted payload, researchers wrote.</p>
<p>The ultimate step in the attack chain is that the IcedID &#8220;Gziploader&#8221; payload is decoded and placed in memory and then executed. The GZiploader fingerprints the machine and sends a beacon to the command-and-control (C2) server – located at yourgroceries[.]top<i>.</i> – with information about the infected host, which then can be used for further nefarious activity.</p>
<h2><b>Evolution of a Threat</b></h2>
<p>Researchers at IBM first discovered IcedID <a href="https://threatpost.com/new-icedid-trojan-targets-us-banks/128851/" target="_blank" rel="noopener">back in 2017</a> as a trojan targeting banks, payment card providers, mobile services providers, payroll, web mail and e-commerce sites.</p>
<p>The malware has <a href="https://threatpost.com/botnet-operators-team-up-to-leverage-icedid-trickbot-trojans/132392/" target="_blank" rel="noopener">evolved over the years</a> and already has a storied history of clever obfuscation. For example, it <a href="https://threatpost.com/icedid-banker-adding-steganography-covid-19-theme/156718/" target="_blank" rel="noopener">resurfaced</a> during the <a href="https://blogs.juniper.net/en-us/threat-research/covid-19-and-fmla-campaigns-used-to-install-new-icedid-banking-malware" target="_blank" rel="noopener">COVID-19 campaign</a> with new functionality that uses steganography – the practice of hiding code within images to stealthily infect victims – as well as other enhancements.</p>
<p>The new campaign is evidence of its <a href="https://threatpost.com/spam-icedid-banking-trojan-variant/167250/" target="_blank" rel="noopener">further evolution</a> and could signify that IcedID is indeed becoming, <a href="https://threatpost.com/icedid-banking-trojan-surges-emotet/165314/" target="_blank" rel="noopener">as many fear</a>, the new <a href="https://threatpost.com/emotet-takedown-infrastructure-netwalker-offline/163389/" target="_blank" rel="noopener">Emotet</a> – a modular threat that began as a trojan but steadily evolved into one of the most dangerous malwares ever seen.</p>
<p>&#8220;This attack shows how much effort attackers put in all the time to evade detection and why defense in depth is necessary,&#8221; observed Saumitra Das, CTO and co-founder at security firm <a href="https://u7061146.ct.sendgrid.net/ls/click?upn=4tNED-2FM8iDZJQyQ53jATURk7nu5DOXPXjQHtUbQPB-2Bo-3Dj4oZ_6bll2uIcECOBsx1gx1IC2zx-2FnKyCXka4AgKvEYqpnW0-2BDbBUicS42bKww9XV5LeOm8YSoCZbw6XkWDSfAMcb8GJOp9iX7pVlW-2BkiIYpN1sif0KFuJYXLhOJYPn-2B9Sn-2Baowev7BWgf7-2Fsft7vhZ-2FleI4B1CtNAbekmGpeBhmEFQ0kWRSkTE0FzXKh-2Bz963fPXZn0hvo6ZGATosJpWWMJIx2kznvRhglY0WQkeZMakpGSSCIz9LKVoA7IXOHVn5P16MOaoTEh1LFaqgv30hL1UfNg9Za-2FKpoEtnwzBDLz4DtQVA3dFYwDxuvZKeD9Y8Hi4WQLnSai8UFna4-2BIEwYtA0NcX5KrsjsbSEnjBzFNfZ-2B0-3D" target="_blank" rel="noopener">Blue Hexagon</a>, in an email to Threatpost.</p>
<p>This time and effort, in turn, shows a level of sophistication on the part of those behind IcedID in that they have thorough knowledge of contemporary email protections and are continuously adding new tactics as security also grows and evolves, he said.</p>
<p>&#8220;Many email security systems use reputation of senders to block malicious email without being able to assess the email itself,&#8221; Das noted. &#8220;Here, they used compromised Exchange servers to make it through.&#8221;</p>
<p>The group&#8217;s use of obfuscated file formats to deliver malware, as well as the final payload&#8217;s delivery over the network, also demonstrate that the threat actors know how to evade signature and sandboxes, he added.</p>
<p>&#8220;These attacks often go much deeper than simply stealing data,&#8221; concurred Chris Clements, vice president of solutions architecture at security firm <a href="https://u7061146.ct.sendgrid.net/ls/click?upn=4tNED-2FM8iDZJQyQ53jATUc1h7F6EeKyqQHDAzxY6FeBG4AZ1lNaZ-2Fme9HKLAKT7PZQLK_6bll2uIcECOBsx1gx1IC2zx-2FnKyCXka4AgKvEYqpnW0-2BDbBUicS42bKww9XV5LeOm8YSoCZbw6XkWDSfAMcb8IRiPIGKWMahkivu0WTh5PX5dG77IJVWKxIQtQJVv-2BIYMmRr5z7OIF8mKih-2F25UI0RQa6-2Bdcn0eyt9a-2F-2BxbdAQ8flodV7haNCcr-2BW1iLqgw0DYt7ntjLmuD7PDGwxwwHSq2gHGWVXVmYGWcDbHq95V0DcFYQggLtmHop2EFskxujGp5A7HFr4-2Bzu8HP-2Fn84dnll5nv7EwsYGa4Z-2BkWEdDcrCAY75JBexQSBfFsv2LbL-2Bn1Qz-2FYzen2NsuzLcfAC1av2zq9EhGfkk9KycL0qVySQ-3D" target="_blank" rel="noopener">Cerberus Sentinel</a><b>, </b>in an email to Threatpost. &#8220;The cybercriminals take the time to read through the mailboxes to understand the inter-organization relationships and operating procedures.</p>
<p>&#8220;To protect themselves from similar attacks, it&#8217;s critical that organizations ensure that they apply security patches promptly and thoroughly in their environment,&#8221; he added. However, what is historically true for patching remains true now: that it&#8217;s &#8220;a task that&#8217;s easier said than done,&#8221; Clemens acknowledged.</p>
<p>&#8220;It really takes a cultural approach to cybersecurity to plan for failures in defenses like patch management,&#8221; he said.</p>
