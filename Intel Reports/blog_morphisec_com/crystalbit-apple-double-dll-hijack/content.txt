This has included the evolution of adware, PUA, and fraudulent software bundle delivery beyond a consumer problem into a significant attack vector on enterprise employees.</p>
<p><!--more--></p>
<p>In this vein, Morphisec Labs has seen a more than 800 percent increase in preventing attacks from adware and fraudulent software bundles among protected enterprises.</p>
<p>In this blog, we will technically dive into one significant attack campaign example that happened in the second week of May. During this attack, the adversary abuses two legitimate vendor applications, such as CrystalBit and Apple, as part of a <strong>dll double hijack</strong> attack chain that starts with a fraudulent software bundle and eventually leads to a persistent miner and in some cases spyware deployment.</p>
<p>The abuse of the Apple push notification executable (APSDaemon.exe) is certainly not new. Over the course of more than a year, adversaries have deployed a legitimate and signed copy of the application together with a malicious AppleVersions.dll that is soon loaded by the daemon. In most cases, the deployment was part of a second stage of an attack and rarely have been seen as part of an infiltration stage.</p>
<p>What caught our attention is the use of similar techniques by abusing <a href="https://crystalidea.com/blog/malware-abusing-anytoiso-and-crystalbit-solutions" rel="noopener" target="_blank">AnyToIso and CrystalBit</a> software as part of the first delivery stage of the attack.</p>
<h2>DLL HIJACK Technical Overview</h2>
<div><img src="https://blog.morphisec.com/hubfs/%5BTHREAT%20POST%5D%20CrystalBit%20%20Apple,%20Double%20dll%20hijack-4.png" alt="apple dll hijack"></div>
<p>&nbsp;</p>
<p>The attack chain consists of:</p>
<ol>
<li>Downloading a software bundle from a fraudulent site (DVD plugins, browsers, Excel plugins, codecs, etc...).</li>
<ul>
<li>Format: [random letters and digits]{7,8}_SETUP.zip</li>
</ul>
<li>Elevated Execution of a signed AnyToIso / CrystalBit application by the victim, which leads to a malicious msimg32.dll hijack (the malicious DLL is bundled together with the executable)<br><img src="https://blog.morphisec.com/hs-fs/hubfs/%5BTHREAT%20POST%5D%20CrystalBit%20%20Apple,%20Double%20dll%20hijack-1.png?width=443&amp;name=%5BTHREAT%20POST%5D%20CrystalBit%20%20Apple,%20Double%20dll%20hijack-1.png" alt="CrystalBit  Apple hijack" width="443" style="width: 443px; display: block; margin-left: auto; margin-right: auto;" srcset="https://blog.morphisec.com/hs-fs/hubfs/%5BTHREAT%20POST%5D%20CrystalBit%20%20Apple%2C%20Double%20dll%20hijack-1.png?width=222&amp;name=%5BTHREAT%20POST%5D%20CrystalBit%20%20Apple%2C%20Double%20dll%20hijack-1.png 222w, https://blog.morphisec.com/hs-fs/hubfs/%5BTHREAT%20POST%5D%20CrystalBit%20%20Apple%2C%20Double%20dll%20hijack-1.png?width=443&amp;name=%5BTHREAT%20POST%5D%20CrystalBit%20%20Apple%2C%20Double%20dll%20hijack-1.png 443w, https://blog.morphisec.com/hs-fs/hubfs/%5BTHREAT%20POST%5D%20CrystalBit%20%20Apple%2C%20Double%20dll%20hijack-1.png?width=665&amp;name=%5BTHREAT%20POST%5D%20CrystalBit%20%20Apple%2C%20Double%20dll%20hijack-1.png 665w, https://blog.morphisec.com/hs-fs/hubfs/%5BTHREAT%20POST%5D%20CrystalBit%20%20Apple%2C%20Double%20dll%20hijack-1.png?width=886&amp;name=%5BTHREAT%20POST%5D%20CrystalBit%20%20Apple%2C%20Double%20dll%20hijack-1.png 886w, https://blog.morphisec.com/hs-fs/hubfs/%5BTHREAT%20POST%5D%20CrystalBit%20%20Apple%2C%20Double%20dll%20hijack-1.png?width=1108&amp;name=%5BTHREAT%20POST%5D%20CrystalBit%20%20Apple%2C%20Double%20dll%20hijack-1.png 1108w, https://blog.morphisec.com/hs-fs/hubfs/%5BTHREAT%20POST%5D%20CrystalBit%20%20Apple%2C%20Double%20dll%20hijack-1.png?width=1329&amp;name=%5BTHREAT%20POST%5D%20CrystalBit%20%20Apple%2C%20Double%20dll%20hijack-1.png 1329w" sizes="(max-width: 443px) 100vw, 443px"></li>
<li>Msimg32.dll writes a number of files that include the Apple push application and the malicious AppleVersions.dll to a “super” hidden directory under %windir%/SysWOW64/Speech/Engines/. In some cases it also writes data.dll. Both the directory name within the Speech/Engines and the APSDaemon.exe file name are randomized under given constraints</li>
<ul>
<li>The APSDaemon.exe new name starts with “<em>fb_”</em> or “<em>rb_”</em>, (“fb_&lt;number&gt;.exe”).</li>
<li>The directory name starts with “<em>Q-”.</em><br><br><img src="https://blog.morphisec.com/hubfs/%5BTHREAT%20POST%5D%20CrystalBit%20%20Apple,%20Double%20dll%20hijack-2.png" alt="syswow64" style="display: block; margin-left: auto; margin-right: auto;"></li>
</ul>
<li>Msimg32.dll creates a scheduled task for persistence, in most cases we have observed a GoogleUpdateTask that will execute every 15 minutes. Note: the scheduled task is created with system privileges, you will need system privs to view it.<br><br><strong>Note</strong><span style="background-color: transparent;">: to access the folder you will need to disable hidden privileges (attrib -h -s).</span></li>
</ol>
<div><img src="https://blog.morphisec.com/hubfs/%5BTHREAT%20POST%5D%20CrystalBit%20%20Apple,%20Double%20dll%20hijack-3.png" alt="Google update task machine"></div>
<p>&nbsp;</p>
<h3>CoinLoader Shellcode</h3>
<p>VirusTotal and other 3rd party tools have classified some of the AppleVersions.dll versions in a very generic way, while some dll versions have got a clear CoinLoader classification, yet with a relatively low detection ratio. We have decided to investigate some of those and identified interesting techniques to extract modules and functions from within the memory (known but rare).</p>
<p>More specifically, their shellcode implemented a variation of the <strong>Fowler–Noll–Vo</strong> hash algorithm to compare module names and function names during the iteration over the Process environment block structure. This significantly increased their chances to evade security vendors that are looking for the regular hash signatures in memory (all leading vendors are looking for the ROR-13 hash patterns as those are implemented as part of the default code injection framework unless the framework is recompiled).</p>
<div><img src="https://blog.morphisec.com/hubfs/%5BTHREAT%20POST%5D%20CrystalBit%20%20Apple,%20Double%20dll%20hijack.png" alt="dll framework"></div>
<p>&nbsp;</p>
<h2>Conclusions</h2>
<p>Malware has become much more evasive without relation to its <strong>type or category.&nbsp;</strong>This evasiveness manifests itself through whitelisting bypass, <a href="/fileless-malware-attacks" rel="noopener" target="_blank">fileless techniques</a> and in-memory execution. Looking for suspicious memory patterns is not enough and will not hold for long, which makes prevention a clear necessity.</p>
<p>COVID-19 is the perfect time for adversaries to go hunt for new and less protected environments, WFH (work from home) is such an environment without a proper network protection stack, without proper hardening and without proper IT management and enforcement.</p>
<p>Morphisec protects such environments without applying any sort of detection by executing Moving Target Defense against the same adversaries.</p>
