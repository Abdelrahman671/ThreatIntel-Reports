			<h1 class="entry-title">BumbleBee: Round Two</h1>
		<div class="entry-meta">
			<span class="posted-on"><a href="https://thedfirreport.com/2022/09/26/bumblebee-round-two/" rel="bookmark"><time class="entry-date published" datetime="2022-09-26T01:05:36+00:00">September 26, 2022</time></a></span>		</div><!-- .entry-meta -->

		
	</header><!-- .entry-header -->

	<div class="entry-content">
		
<p>In this intrusion from May 2022, the threat actors used <a href="https://malpedia.caad.fkie.fraunhofer.de/details/win.bumblebee" target="_blank" rel="noreferrer noopener">BumbleBee</a> as the initial access vector. BumbleBee has been <a href="https://symantec-enterprise-blogs.security.com/blogs/threat-intelligence/bumblebee-loader-cybercrime" target="_blank" rel="noreferrer noopener">identified</a> as an initial access vector utilized by <a href="https://blog.google/threat-analysis-group/exposing-initial-access-broker-ties-conti/" target="_blank" rel="noreferrer noopener">several ransomware affiliates</a>.</p>



<p>In this intrusion, we see the threat actor use BumbleBee to deploy Cobalt Strike and Meterpreter. The threat actor then used RDP and SMB to move around the network looking at backup systems and file shares before being evicted from the network.</p>



<h2 class="wp-block-heading">Case Summary</h2>



<p>The intrusion began with the delivery of an ISO file containing a LNK file and a BumbleBee payload in the form of a hidden DLL file. A user on a workstation mounted the ISO file and executed the LNK file, running the Bumblebee payload.</p>



<p>Around 15 minutes after the execution of BumbleBee, multiple processes were spawned with the goal of injecting Meterpreter into each of them. After the threat actors gained access with Meterpreter, they began conducting reconnaissance on the workstation and network, including querying domain controllers, mapping domain joined computers, enumerating Active Directory trusts, and listing Domain Admin accounts. All of this first wave of discovery relied on built in Windows utilities like nltest, arp, net, ping, nbtstat, and nslookup.</p>



<p>BumbleBee executed under a user with local administrator privileges on all workstations in the environment. At around six hours after initial execution, we observed a new process created that was then used to host a Cobalt Strike beacon, from the same command and control server observed in a prior <a href="https://thedfirreport.com/2022/08/08/bumblebee-roasts-its-way-to-domain-admin/" target="_blank" rel="noreferrer noopener">BumbleBee case</a>. This beacon reprised discovery activity, but also cut a common command short <code>net user /dom</code> instead of <code>/domain</code>, whether from keyboard laziness or a trick to trip-up detections. The threat actor then used their access to execute procdump via a remote service creation with the intention of dumping credentials from LSASS from an adjacent workstation on the network.</p>



<p>Next, the threat actors moved laterally via RDP to a server. A new local user, sql_admin, was created and added to the local administrator&#8217;s group and AnyDesk remote access software was installed. Through the AnyDesk session, the threat actor was observed connecting to a file share and accessing multiple documents related to cyber insurance and spreadsheets with passwords.</p>



<p>A second round of enumeration was observed on the beachhead using AdFind, which was executed via the Cobalt Strike beacon on the system. Following this second round of enumeration, the threat actor moved latterly to a server hosting backups, via RDP and interacted with the backup console. From the backup system, the threat actors also opened internet explorer and attempted to load the environment&#8217;s mail server, likely checking for Outlook Web Access.</p>



<p>A third round of enumeration, this time taking place from the first lateral server host, was observed via a script named ‘1.bat’ that would ping all computers in the environment. Following this third round of enumeration the threat actors were evicted from the environment and no further impact was observed.</p>



<p>We assess with medium confidence this intrusion was related to pre-ransomware activity due to the tool set and techniques the actor displayed.</p>

<p>We also have artifacts and IOCs available from this case such as pcaps, memory captures, files, event logs including Sysmon, Kape packages, and more, under our <a href="https://www.patreon.com/thedfirreport" target="_blank" rel="noreferrer noopener">Security Researcher and Organization</a> services.</p>

<h2 class="wp-block-heading">Initial Access</h2>



<p>The BumbleBee malware has been following the trend of using the effective combination of utilizing an .iso image containing a .lnk and .dll file. We have observed the same behavior with other major malware distributors in previous reports:</p>



<ul class="wp-block-list">
<li>IcedID &#8211; <a href="https://thedfirreport.com/2022/04/04/stolen-images-campaign-ends-in-conti-ransomware/" target="_blank" rel="noreferrer noopener">Stolen Images Campaign Ends in Conti Ransomware</a></li>
<li>BazarLoader &#8211; <a href="https://thedfirreport.com/2021/12/13/diavol-ransomware/" target="_blank" rel="noreferrer noopener">Diavol Ransomware</a></li>
</ul>



<p>Using the event log, &#8220;Microsoft-Windows-VHDMP-Operational.evtx&#8221;, we can quickly find when the user mounted the .iso.</p>



<figure class="wp-block-image"><a href="https://thedfirreport.com/wp-content/uploads/2022/09/14373-001.png"><img decoding="async" class="aligncenter size-full wp-image-10002" src="https://thedfirreport.com/wp-content/uploads/2022/09/14373-001.png" alt="" width="1359" height="747" srcset="https://thedfirreport.com/wp-content/uploads/2022/09/14373-001.png 1359w, https://thedfirreport.com/wp-content/uploads/2022/09/14373-001-300x165.png 300w, https://thedfirreport.com/wp-content/uploads/2022/09/14373-001-1024x563.png 1024w, https://thedfirreport.com/wp-content/uploads/2022/09/14373-001-768x422.png 768w" sizes="(max-width: 1359px) 100vw, 1359px" /></a></figure>



<p>Upon clicking the LNK file the BumbleBee payload was executed.</p>

<h2 class="wp-block-heading">Execution</h2>



<p>Following the user mounting the .iso file, they clicked on a .lnk file <code>documents.lnk</code>. As noted in previous <a href="https://thedfirreport.com/2022/04/25/quantum-ransomware/#:~:text=Typical%20end%20user%20perspective%20after%20opening%20the%20ISO%20file%3A" target="_blank" rel="noreferrer noopener">reports</a>, the .dll is hidden from the user unless they display hidden items in explorer like so:</p>



<figure class="wp-block-image"><a href="https://thedfirreport.com/wp-content/uploads/2022/09/14373-003.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-10004" src="https://thedfirreport.com/wp-content/uploads/2022/09/14373-003.png" alt="" width="750" height="191" srcset="https://thedfirreport.com/wp-content/uploads/2022/09/14373-003.png 750w, https://thedfirreport.com/wp-content/uploads/2022/09/14373-003-300x76.png 300w" sizes="auto, (max-width: 750px) 100vw, 750px" /></a></figure>



<p>The .lnk contains instructions to execute a specific exported function with the BumbleBee DLL file.</p>



<figure class="wp-block-image"><a href="https://thedfirreport.com/wp-content/uploads/2022/09/14373-004.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-10005" src="https://thedfirreport.com/wp-content/uploads/2022/09/14373-004.png" alt="" width="783" height="130" srcset="https://thedfirreport.com/wp-content/uploads/2022/09/14373-004.png 783w, https://thedfirreport.com/wp-content/uploads/2022/09/14373-004-300x50.png 300w, https://thedfirreport.com/wp-content/uploads/2022/09/14373-004-768x128.png 768w" sizes="auto, (max-width: 783px) 100vw, 783px" /></a></figure>



<p>When the .lnk was doubled clicked by the user, the BumbleBee malware <code>tamirlan.dll</code> was executed:</p>



<pre class="wp-block-preformatted">C:\Windows\System32\rundll32.exe tamirlan.dll,EdHVntqdWt</pre>



<p>The output of <a href="https://github.com/EricZimmerman/LECmd" target="_blank" rel="noreferrer noopener">LECmd.exe</a>, when used on<code>documents.lnk</code>, provided additional context to where and when this .lnk file was created:</p>



<pre class="wp-block-preformatted">&gt;&gt; Tracker database block
   Machine ID: user-pc
   MAC Address: 9a:5b:d6:3e:47:ec
   MAC Vendor: (Unknown vendor)
   Creation: &lt;REDACTED DATE&gt;</pre>



<p>Approximately 5 seconds after execution, the <code>rundll32.exe</code> process contacted the IP <code>154.56.0.221</code>. More information on this traffic is covered in the Command and Control section below.</p>



<p>An interesting tactic of note, was the use of WMI and COM function calls to start the process, used to inject into. The BumbleBee loader uses WMI to start new process by calling COM functions to create a new process. Below you can see the COM instance creation followed by defining the WMI namespace and WMI object being created &#8211; &#8220;Win32_Process&#8221;.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2022/09/14373-005.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-10006" src="https://thedfirreport.com/wp-content/uploads/2022/09/14373-005.png" alt="" width="758" height="874" srcset="https://thedfirreport.com/wp-content/uploads/2022/09/14373-005.png 758w, https://thedfirreport.com/wp-content/uploads/2022/09/14373-005-260x300.png 260w" sizes="auto, (max-width: 758px) 100vw, 758px" /></a></p>





<p>Analysis of the loader found that a function of the malware chooses 1 of 3 target processes before injecting the supplied code:</p>



<pre class="wp-block-preformatted">C:\Program Files\Windows Mail\wabmig.exe
C:\Program Files\Windows Mail\wab.exe
C:\Program Files\Windows Photo Viewer\ImagingDevices.exe</pre>



<figure class="wp-block-image"><a href="https://thedfirreport.com/wp-content/uploads/2022/09/14373-006.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-10007" src="https://thedfirreport.com/wp-content/uploads/2022/09/14373-006.png" alt="" width="760" height="100" srcset="https://thedfirreport.com/wp-content/uploads/2022/09/14373-006.png 760w, https://thedfirreport.com/wp-content/uploads/2022/09/14373-006-300x39.png 300w" sizes="auto, (max-width: 760px) 100vw, 760px" /></a></figure>



<p>This resulted in new processes not being a child of BumbleBee, but rather WmiPrvSE.exe.</p>



<figure class="wp-block-image"><a href="https://thedfirreport.com/wp-content/uploads/2022/09/14373-007.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-10008" src="https://thedfirreport.com/wp-content/uploads/2022/09/14373-007.png" alt="" width="1315" height="151" srcset="https://thedfirreport.com/wp-content/uploads/2022/09/14373-007.png 1315w, https://thedfirreport.com/wp-content/uploads/2022/09/14373-007-300x34.png 300w, https://thedfirreport.com/wp-content/uploads/2022/09/14373-007-1024x118.png 1024w, https://thedfirreport.com/wp-content/uploads/2022/09/14373-007-768x88.png 768w" sizes="auto, (max-width: 1315px) 100vw, 1315px" /></a></figure>



<p>In this intrusion, an instance of <code>C:\Program Files\Windows Photo Viewer\ImagingDevices.exe</code> was created and accessed by the BumbleBee <code>rundll32.exe</code> process. Shortly after this interaction, the process started communicating to a Meterpreter C2 <code>3.85.198.66</code>. This process spawned <code>cmd.exe</code> and several typical discovery commands that are covered in more detail below.</p>



<p>The second process, was spawned the WMI technique was an instance of <code>C:\Program Files\Windows Mail\wabmig.exe</code>. This process was used to host both a session to another Meterpreter C2 <code>50.16.62.87</code> and a Cobalt Strike C2 server <code>45.153.243.142,</code> which was then used to conduct the majority of additional activity including credential dumping and discovery exercises highlighted below. The pivot to using Cobalt Strike began around 6 hours after the execution of the BumbleBee loader.</p>

<h2 class="wp-block-heading">Collection</h2>



<p>The threat actor accessed multiple documents and folders from a remote file server. The SMB share was accessed through a compromised server via an AnyDesk session.</p>

<p>The lsass dump file ran remotely, was copied to the beachhead through the admin share <em>C$.</em></p>



<p>After being copied, the file was zipped using 7za.exe (7-zip), in preparation for exfiltration.</p>
