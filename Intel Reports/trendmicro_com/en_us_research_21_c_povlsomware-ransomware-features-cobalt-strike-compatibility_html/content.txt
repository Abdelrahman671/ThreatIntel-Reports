	<h1 class="article-details__title">Povlsomware Ransomware Features Cobalt Strike Compatibility</h1>
	<p class="article-details__description">Povlsomware is a proof-of-concept (POC) ransomware first released in November 2020 which, according to their Github page, is used to “securely” test the ransomware protection capabilities of security vendor products. </p>
	<p class="article-details__author-by">By: Don Ovid Ladores
		
			<time class="article-details__date">March 01, 2021</time>
		
		
		<span>Read time:&nbsp;</span><span class="eta"></span> (<span class="words"></span> words)
	</p>

</div>
            <div class="richText">
	
    


	
		<div>
			<p>Povlsomware (<a href="https://www.trendmicro.com/vinfo/tmr/?/us/threat-encyclopedia/malware/ransom.msil.povlsom.thbaoba">Ransom.MSIL.POVLSOM.THBAOBA</a>)  is a proof-of-concept (POC) <a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/definition/ransomware">ransomware</a> first released in November 2020 which, according to their Github page, is used to “securely” test the ransomware protection capabilities of security vendor products. Povlsomware has not garnered much attention at the moment, being talked about <a href="https://id-ransomware.blogspot.com/2020/12/povlsomware-ransomware.html">in only a few sites</a> — however, it has some interesting characteristics, notably its compatibility with the post-exploitation tool Cobalt Strike (which has been linked to other ransomware families such as <a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/news/cybercrime-and-digital-threats/the-state-of-ransomware-2020-s-catch-22">Ryuk</a> and <a href="https://www.trendmicro.com/en_us/research/21/a/an-overview-of-the-doppelpaymer-ransomware.html">DoppelPaymer</a>) — giving the malware capabilities beyond what its seemingly simple infection routine shows. Furthermore, because the malware is open-source, anyone can modify and use it as part of their attack chain.</p>

		</div>
	

</div>
            <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/21/c/povlsomware-ransomware-features-cobalt-strike-compatibility/Povlsomeware-1.png" alt="Figure 1. The ReadMe page from the Povlsomware GitHub page"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 1. The ReadMe page from the Povlsomware GitHub page</figcaption>
			
		</div>
	</figure>

</div>
            <div>




    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/21/c/povlsomware-ransomware-features-cobalt-strike-compatibility/Povlsomeware-2.png" alt="Figure 2. Ransom note from a sample found in the wild written in Russian"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 2. Ransom note from a sample found in the wild written in Russian</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p>At first glance, Povlsomware seems to follow a fairly typical ransomware routine. First, it will check to see if it is running in privileged mode, after which it will attempt to delete backups via Windows Management Instrumentation (WMI).</p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/21/c/povlsomware-ransomware-features-cobalt-strike-compatibility/Povlsomeware-3.png" alt="Figure 3. Removing backups via WMI"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 3. Removing backups via WMI</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p>Based on the latest source code, the files it tries to encrypt are fairly standard, which includes commonly used file types such as word documents, PDF files, audio and video files, and RAR archives.</p>
<p>It will also avoid directories containing the following strings:</p>
<ul>
<li><span class="rte-red-bullet">All Users\\Microsoft\\</span></li>
<li><span class="rte-red-bullet">AppData\\</span></li>
<li><span class="rte-red-bullet">C:\\Program Files</span></li>
<li><span class="rte-red-bullet">\\Povlsomware\\</span></li>
<li><span class="rte-red-bullet">\\Povlsomware-master\\</span></li>
<li><span class="rte-red-bullet">C:\\ProgramData\\</span></li>
<li><span class="rte-red-bullet">$Recycle.Bin</span></li>
<li><span class="rte-red-bullet">\\source\\</span></li>
<li><span class="rte-red-bullet">Temporary Internet Files</span></li>
<li><span class="rte-red-bullet">C:\\Windows</span></li>
</ul>
<p>After it is done with encryption, Povlsomware will display the list of affected files along with the ransom note.</p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/21/c/povlsomware-ransomware-features-cobalt-strike-compatibility/Povlsomeware-4.png" alt="Figure 4. The ransom note displayed by Povlsomware after it has encrypted its victim’s files"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 4. The ransom note displayed by Povlsomware after it has encrypted its victim’s files</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p>One interesting characteristic about Povlsomware is that it is an extensionless ransomware — that is, it does not append additional extension names to the files it encrypts. This means that encrypted files will still look the same when viewed through a directory but will not work properly when the user tries to open them.</p>
<p>Beyond its routine, the more concerning aspect of Povlsomware is how the authors claim it is coded —its Cobalt Strike compatibility allows it to perform in-memory loading and execution. This “execute-assembly” feature of Cobalt Strike allows the execution of a payload binary through memory only and does not drop any payload binaries into the victim’s system. This makes the payload more difficult to detect (since it does not rely on the tools found in the target system) and analyze due to the lack of payload binaries being dropped.</p>

		</div>
	

</div>


    
    
    <div class="image">

    <div class="richText">
	
    


	
		<div class="responsive-table-wrap">
			<p>What makes Povlsomware notable is not what it has already accomplished — there have been few, if any, notable incidents involving the malware at the time of writing — but how it can potentially be used. Although ostensibly advertised as an “educational” tool, its access to Cobalt Strike capabilities and its open-source nature make it a potentially dangerous malware in the hands of an experienced ransomware operator. </p>
<h1><span class="body-subhead-title">Security recommendations and solutions</span></h1>
<p>Ransomware attacks can come in many forms. At the simplest end of the spectrum, you have infection vectors such as phishing emails that will deploy the malware if the user accesses them. More complicated campaigns, on the other hand, might make use of complex infrastructure involving the use of tools such as Cobalt Strike or Mimikatz as part of the infection routine.</p>
<ul>
<li><span class="rte-red-bullet">Given the wide array of tools and techniques that ransomware operators use, organizations and individual users have to be vigilant on all fronts to protect their machines and systems from ransomware attacks. The following security best practices can help with this:</span></li>
<li><span class="rte-red-bullet">Limiting access to shared network drives and turning off file sharing lessens the probability that a ransomware infection spreads to other devices.</span></li>
<li><span class="rte-red-bullet">Employing authentication options that include multi-factor authentication strengthens the security of user accounts, minimizing opportunities for account theft or compromise.</span></li>
<li><span class="rte-red-bullet">Enforcing the principle of least privilege — where users only have access to the sections of the system that they need— can help prevent attackers from running tools and software used by ransomware.</span></li>
<li><span class="rte-red-bullet">Using application controls can help prevent attacks that leverage remote access software.</span></li>
<li><span class="rte-red-bullet">Regular patching and updating of software and systems reduce the chance for an attacker to exploit a vulnerability that is part of the infection routine.</span></li>
<li><span class="rte-red-bullet">Regularly backing up files, preferably <a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/news/virtualization-and-cloud/best-practices-backing-up-data">using the 3-2-1 rule</a>, still remains an effective method of protecting data in the event of a successful ransomware infection.</span></li>
</ul>
<p>Organizations can also take advantage of security technologies such as <a href="/en_us/business/products/detection-response/xdr.html">Trend Micro XDR</a><a href="https://www.trendmicro.com/en_us/business/products/detection-response/xdr.html"><sup>TM</sup></a>, which collects and correlates data across endpoints, emails, cloud workloads, and networks to provide better context to attacks in a single console. This allows security and IT teams to respond to ransomware and other threats faster and more effectively.</p>
