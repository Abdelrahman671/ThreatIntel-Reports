<h2 class="heading" id="anchor-1">Overview</h2>
<p>Kimsuky&#8217;s HappyDoor malware is not a commonly known malware. AhnLab first collected its sample in 2021, and continued monitoring revealed that it had been used up to the present day in 2024 in data breaches. Investigation results hint that the threat actor has been patching the malware continuously. As shown in the image below, the version (4.2) and the numbers assumed to be the creation date (Jan. 12 2024) are hard-coded into the version information, and the latest sample also displays the malware&#8217;s name (happy).</p>
<figure class="image">
 <img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2024/07/kimsuky-img.png" alt=""></p>
<p style="text-align: center; margin-top: 5px;"><span style="color: #6B7684; font-size: 14px;">Binary internal version information</span></p>
</figure>
<p>This &#8220;happy&#8221; string is also found in the Export DLL name and the code&#8217;s debug string, as shown below. Given the information, AhnLab SEcurity intelligence Center (ASEC) has named this malware &#8220;HappyDoor&#8221;.</p>
<figure class="image aligncenter">
 <img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2024/06/image-64.png?resize=1024%2C379&amp;ssl=1" alt=""></p>
<p style="text-align: center; margin-top: 5px;"><span style="color: #6B7684; font-size: 14px;">Export DLL name (left) and debug string inside (right)</span></p>
</figure>
<h2 class="heading" id="anchor-2">Distribution Method and Changes</h2>
<h3 class="heading" id="anchor-2-1">1. Distribution Method</h3>
<p>The Kimsuky group has distributed various malware strains via spear phishing email attacks in the past and is continuing to do so to this day. Some of the major cases include the installation of AppleSeed and AlphaSeed malware, and their method of distribution was introduced in a previous report (<a href="https://asec.ahnlab.com/en/30532/">Analysis Report on Kimsuky Group&#8217;s APT Attacks (AppleSeed, PebbleDash)</a>) and also in the 2023 threat trend report (<a href="https://asec.ahnlab.com/en/60054/">Trend Analysis on Kimsuky Group’s Attacks Using AppleSeed</a>). Recently, a detailed report was published in AhnLab TIP (<a href="https://atip.ahnlab.com/intelligence/view?id=435fa2d7-8d4f-4d40-9bf7-cc504cace709">Kimsuky Group&#8217;s Bait Files Used to Distribute AppleSeed &amp; HappyDoor Backdoors (December 2023 &#8211; April 2024)</a>).</p>
<p>HappyDoor in this case is also being distributed via an email attachment just like the previous method of distribution. This attachment file contains a compressed file, and the latter carries a JScript or a dropper (executable file). Once that is run, HappyDoor is created and executed along with normal bait files. The following is an example of JScript code (AMSI) decrypted when the script is executed:</p>
<figure class="table">
<table>

<h4 class="heading" id="anchor-3-3-2">3.2. Backdoor</h4>
<p>When receiving backdoor commands from the threat actor, HappyDoor usually operates via&nbsp;<a href="https://atcptest.wpcomstaging.com/en/76800/#%EC%95%94%ED%98%B8%ED%99%94%EB%90%9C_%EB%B0%B1%EB%8F%84%EC%96%B4_%EB%AA%85%EB%A0%B9">encrypted packets</a>. The action&#8217;s encryption is decided by the&nbsp;<a href="https://atcptest.wpcomstaging.com/en/76800/#Notapad_%EB%A0%88%EC%A7%80%EC%8A%A4%ED%8A%B8%EB%A6%AC_%EA%B5%AC%EC%A1%B0">Notepad registry</a>&#8216;s &#8220;<em>Backdoor Packet Encryption ON/OFF&#8221; </em>value, and because it is set to &#8220;1&#8221; (ON) by default, changing it to &#8220;0&#8221; (OFF) allows HappyDoor to receive backdoor commands using unencrypted packets. Upon receiving a packet from the threat actor&#8217;s server, HappyDoor performs the backdoor feature that corresponds to the &#8220;CMD ID&#8221; number of the <a href="https://atcptest.wpcomstaging.com/en/76800/#%ED%8C%A8%ED%82%B7_%EA%B5%AC%EC%A1%B0">packet</a>.</p>

<h2 class="heading" id="anchor-4">Conclusion</h2>
<p>While monitoring threat groups that are known to be sponsored by North Korea, ASEC recently identified an attack case presumed to be the Kimsuky group&#8217;s doing. Mainly taking on the disguise of a professor or an academic institution, the threat actor has been using social engineering techniques like spear phishing to distribute emails with attachments that, once run, install a backdoor and may also install additional malware.</p>
<p>The threat actor uses the additionally installed malware strains and proxy tools, such as Ngrok, to establish a proxy network environment and may install more programs like VNC malware strains, Chrome Remote Desktop, or RDP Wrapper to ameliorate its underwhelming remote control features. AhnLab also found various records of the threat actor installing diverse malware strains to escalate privilege and steal account information. Because of this, the users targeted by the Kimsuky group&#8217;s attacks are at the risk of having various information stolen from their infected environments, and there have been actual cases where the threat actor stole the users&#8217; certificates.</p>
<p>Users must be particularly cautious against attachments in emails from unknown sources and executable files downloaded from web pages. Security administrators in companies must enhance monitoring for software in use and apply patches for any security vulnerabilities in the programs. Users should also apply the latest patch for OS and programs such as internet browsers, and update V3 to the latest version to prevent malware infection in advance.</p>
