    <h2>Executive summary</h2>
<ul>
<li><p>Akamai security researchers uncovered a new crypto mining campaign, which has been active since the start of 2023.</p>
</li>
<li><p>The malware is spread over SSH protocol using a custom Mirai botnet that was modified by the threat actors.</p>
</li>
<li><p>The capabilities of the new botnet, NoaBot, include a wormable self-spreader and an SSH key backdoor to download and execute additional binaries or spread itself to new victims.</p>
</li>
<li><p>As part of the attack, a modified version of the XMRig miner is dropped. The miner obfuscates its configuration and also uses a custom mining pool to avoid exposing the wallet address used by the miner.</p>
</li>
<li><p>We have seen evidence to link the botnet to the P2PInfect worm, which was <a href="https://unit42.paloaltonetworks.com/peer-to-peer-worm-p2pinfect" target="_blank" rel="nofollow noreferrer" referrerpolicy="no-referrer">discovered by Unit 42</a> in July 2023.</p>
</li>
<li><p>The malware obfuscation and custom code show a high level of operation security, which usually indicates mature threat actors, but the naming of the malware’s binaries and some its included strings are quite childish. This complicates attribution.</p>
</li>
<li><p>We saw more than 800 different attacking IPs in 2023, spread evenly across the world.</p>
</li>
<li><p>We have <a href="https://github.com/akamai/akamai-security-research/tree/main/malware/noabot" target="_blank" rel="nofollow noreferrer" referrerpolicy="no-referrer">published</a> indicators of compromise (IOCs), queries, signatures, and scripts that can be used to test for infection.</p>
</li>
</ul>

    <h2>Introduction</h2>
<p>NoaBot is yet another <a href="https://en.wikipedia.org/wiki/Mirai_(malware)" target="_blank" rel="nofollow noreferrer" referrerpolicy="no-referrer" title="Mirai">Mirai</a>-based botnet. The Mirai <a href="/glossary/what-is-a-botnet" target="_self" title="botnet" rel="nofollow noreferrer" referrerpolicy="no-referrer">botnet</a> is a wormable botnet that targets Linux-based Internet of Things (IoT) devices. It is used for distributed denial-of-service <a title="(DDoS) attacks" href="/glossary/what-is-ddos" target="_self">(DDoS) attacks</a>. The original Mirai botnet was <a href="https://web.archive.org/web/20160905023500/http://blog.malwaremustdie.org/2016/08/mmd-0056-2016-linuxmirai-just.html" target="_blank" rel="nofollow noreferrer" referrerpolicy="no-referrer" title="identified">identified</a> in 2016, but its source code has been made <a href="https://github.com/jgamblin/Mirai-Source-Code/tree/3273043e1ef9c0bb41bd9fcdc5317f7b797a2a94" target="_blank" rel="nofollow noreferrer" referrerpolicy="no-referrer" title="public">public</a>, and many variants can be seen nowadays.</p>
<p>We first detected the NoaBot campaign at the beginning of 2023. Since then, we’ve seen two evolutions of the malware, which consist of additional obfuscations or a change of command and control (C2) and mining pool domains (Figure 1). We’ve also seen several incidents that drop samples of the <a href="https://unit42.paloaltonetworks.com/peer-to-peer-worm-p2pinfect" target="_blank" rel="nofollow noreferrer" referrerpolicy="no-referrer" title="P2PInfect worm">P2PInfect worm</a>, which hints that the two campaigns are related.</p>

    <h2>The botnet</h2>
<p>The NoaBot botnet has most of the capabilities of the original Mirai botnet (such as a scanner module and an attacker module, hiding its process name, etc.), but we can also see many differences from Mirai’s <a href="https://github.com/jgamblin/Mirai-Source-Code/tree/3273043e1ef9c0bb41bd9fcdc5317f7b797a2a94" target="_blank" rel="nofollow noreferrer" referrerpolicy="no-referrer">original source code</a>. First and foremost, the malware’s spreader is based in SSH, not based in Telnet like Mirai.</p>
<p>The SSH scanner seems to be custom made, and quite peculiar — once a connection is established, the botnet simply sends the string “hi” and terminates the connection (Figure 2). Quick termination makes sense in the context of a scanner, since it doesn’t need to keep the connection open, just to verify it exists. Why does it bother sending “hi,” though? That’s a mystery.<br />
</p>

    <h2>Mirai is too old and developed some Rust</h2>
<p>The most recent incidents we’ve seen also had samples of <a href="https://unit42.paloaltonetworks.com/peer-to-peer-worm-p2pinfect" target="_blank" rel="nofollow noreferrer" referrerpolicy="no-referrer">P2PInfect</a> — a peer-to-peer self-replicating worm written in Rust. While P2PInfect was first seen in July 2023, we’ve seen NoaBot activity since January 2023, which means it predates P2PInfect by a bit. Why switch from Mirai to something else? Something that might even be custom? We have no clear answer, but a few guesses.</p>
<p>First, custom code is more difficult to reverse engineer than repurposed code because it is modified. Second, the threat actors seem quite tech-savvy, so it could be that they’re trying their hand at malware development out of curiosity or boredom (or both). Finally, given that P2PInfect targets Redis servers, it could simply be a case of different tools for different purposes.</p>
<p>How do we know that it’s the same threat actors, not just some sort of collaboration? We aren’t 100% certain, but we’re close. It all boils down to the technical professionalism in the malware, coupled with a teenager’s maturity level in terms of inside jokes, including inserting profanities in the miner’s name, embedding gaming pop song lyrics in malware binaries, and sending “hi” while scanning for open ports. </p>
<p>P2PInfect carries on that tradition. It seems like a sophisticated tool, but it uses a Unix socket and names it “NunzombiE” (Figure 8). And while NunzombiE is obfuscated and decoded during run time, the worm also has embedded strings for “fast_vuln_file” and “slow_vuln_file” — perfectly legit strings to contain in an executable, no red flags here.</p>

    <h2>Mitigation, detection, emulation</h2>
<p>The malware’s method of lateral movement is via plain old SSH credentials dictionary attacks.</p>
<p>Restricting arbitrary internet SSH access to your network greatly diminishes the risks of infection. In addition, using strong (not default or randomly generated) passwords also makes your network more secure, as the malware uses a basic list of guessable passwords. We’ve shared the credential sets used by the malware in our <a href="https://github.com/akamai/akamai-security-research/tree/main/malware/noabot/credentials" target="_blank" rel="nofollow noreferrer" referrerpolicy="no-referrer">GitHub repository</a>.</p>
<p>There is not much to say about detecting the malware, beyond looking for its binary names. It runs from a randomly generated folder under /lib, so process names are probably the way to go. It should also be possible to detect its cron job, if one was installed. An IOC CSV file is also available at our repository, as well as some YARA signatures for the mine.</p>
<p>In case you want to test your environment against the botnet’s SSH spreader, we’ve created a <a href="https://github.com/akamai/akamai-security-research/tree/main/malware/noabot/infection_monkey_noabot_profile.conf" target="_blank" rel="nofollow noreferrer" referrerpolicy="no-referrer">configuration file</a> for the <a href="/infectionmonkey" target="_self">Infection Monkey</a>, our open source adversary emulation platform (we’ve included a brief explanation on how to use the Infection Monkey in the Appendix). Keep in mind that since the malware uses a very large credential set, it would be impractical for us to test them all (the malware doesn’t care about computation or time costs, but we do). Instead, we opted to include only the more common credentials in the configuration. If you want to include more credentials (or different ones), feel free to build upon this configuration and include your own modifications.</p>
<p>This Infection Monkey configuration also adds a masquerade string that will cause our YARA signatures to trigger on the Monkey agent payload, so you could use it to test that detection as well.<br />
</p>
<p>Once the Infection Monkey finishes testing your environment, it’ll create a report of all the machines it managed to breach. You can also use its cryptojacking plug-in if you want to take the attack simulation to the next level and test for stronger credentials. </p>

    <h2>Summary</h2>
<p>On the surface, NoaBot isn’t a very sophisticated campaign — it’s “just” a Mirai variant and an XMRig cryptominer, and they’re a dime a dozen nowadays. However, the obfuscations added to the malware and the additions to the original source code paint a vastly different picture of the threat actors’ capabilities. Despite the technical skills of the threat actors, they seem to have some immature naming conventions (a Unix socket called “NunzombiE,” for example) that persist even across different malware samples and binaries. We used this characteristic to associate NoaBot with P2PInfect, and perhaps this detection clue will hold true for future malware campaigns.</p>
