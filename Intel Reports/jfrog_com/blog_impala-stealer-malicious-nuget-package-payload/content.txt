			<h1>Analyzing Impala Stealer &#8211; Payload of the first NuGet attack campaign</h1>
<p><img fetchpriority="high" decoding="async" class="alignnone size-full wp-image-113281" src="data:image/svg+xml,%3Csvg%20xmlns='http://www.w3.org/2000/svg'%20viewBox='0%200%201200%20430'%3E%3C/svg%3E" alt="" width="1200" height="430" data-lazy-src="https://speedmedia.jfrog.com/08612fe1-9391-4cf3-ac1a-6dd49c36b276/media.jfrog.com/wp-content/uploads/2023/04/06192849/1200x230.png" /><noscript><img fetchpriority="high" decoding="async" class="alignnone size-full wp-image-113281" src="https://speedmedia.jfrog.com/08612fe1-9391-4cf3-ac1a-6dd49c36b276/media.jfrog.com/wp-content/uploads/2023/04/06192849/1200x230.png" alt="" width="1200" height="430" /></noscript></p>
<p>In this blog post, we’ll provide a detailed analysis of a malicious payload we’ve dubbed “Impala Stealer”, a custom crypto stealer which was used as the payload for the NuGet malicious packages campaign we’ve exposed <a href="https://jfrog.com/blog/attackers-are-starting-to-target-net-developers-with-malicious-code-nuget-packages/">in our previous post</a>. The sophisticated campaign targeted .NET developers via NuGet malicious packages, and the JFrog Security team was able to detect and report it as part of our regular activity of exposing supply chain attacks.</p>
<p>Using the typosquatting technique, the attackers propagated 13 malicious packages which impersonated legitimate packages, some of which are very popular. The attackers leveraged the NuGet package structure, which allows inserting PowerShell scripts that will be executed upon installation, in order to perform a two-stage attack. In the first stage, the <code>init.ps1</code> PowerShell script bundled in the malicious package is automatically executed upon installation, in order to download and execute a Windows executable. In the second stage, the downloaded executable installs itself as a persistent backdoor, before targeting the <a href="https://www.exodus.com/" target="_blank" rel="noopener">Exodus Wallet</a> desktop application by using code injection, in order to gain access to the user’s cryptocurrency accounts.</p>
<p>Since the payload was referred to by (some of) the malicious packages as <strong>Impala</strong>, a name we also saw in the payload’s debug strings, we decided to name this custom payload <strong>Impala Stealer</strong>.</p>

<h2>Impala Stealer technical analysis</h2>
<p>The main payload of the Impala Stealer is an executable file. In this section we will elaborate on the payload’s actions, and what we were able to conclude from its analysis.</p>
<p>The payload’s capabilities can be divided into three types:</p>
<ol>
<li>Persistency</li>
<li>Auto-updating</li>
<li>Crypto-stealing</li>
</ol>
<h3>Payload format</h3>
<p>The payload is an executable that seems to be a .NET application compiled <span style="text-decoration: underline">natively</span> using .NET <a href="https://learn.microsoft.com/en-us/dotnet/core/deploying/native-aot/" target="_blank" rel="noopener">Ahead of Time</a> (AoT) compilation, which is a process that converts the .NET intermediate language into a native language (in our case &#8211; x86-64). This AoT compilation was probably done as an <span style="text-decoration: underline">obfuscation</span> step, as native code is harder to reverse engineer than intermediate language binaries:<br />
<img decoding="async" class="wp-image-113253 aligncenter" src="data:image/svg+xml,%3Csvg%20xmlns='http://www.w3.org/2000/svg'%20viewBox='0%200%20800%20162'%3E%3C/svg%3E" alt="View of strings from within the binary showing that it is a .NET application compiled AOT" width="800" height="162" data-lazy-src="https://speedmedia.jfrog.com/08612fe1-9391-4cf3-ac1a-6dd49c36b276/media.jfrog.com/wp-content/uploads/2023/04/06191520/image1-1.png" /><noscript><img decoding="async" class="wp-image-113253 aligncenter" src="https://speedmedia.jfrog.com/08612fe1-9391-4cf3-ac1a-6dd49c36b276/media.jfrog.com/wp-content/uploads/2023/04/06191520/image1-1.png" alt="View of strings from within the binary showing that it is a .NET application compiled AOT" width="800" height="162" /></noscript></p>
<p style="text-align: center"><img decoding="async" class="aligncenter wp-image-113254" src="data:image/svg+xml,%3Csvg%20xmlns='http://www.w3.org/2000/svg'%20viewBox='0%200%20800%20524'%3E%3C/svg%3E" alt="View of strings from within the binary showing that it is a .NET application compiled AOT" width="800" height="524" data-lazy-src="https://speedmedia.jfrog.com/08612fe1-9391-4cf3-ac1a-6dd49c36b276/media.jfrog.com/wp-content/uploads/2023/04/06191547/image2.png" /><noscript><img decoding="async" class="aligncenter wp-image-113254" src="https://speedmedia.jfrog.com/08612fe1-9391-4cf3-ac1a-6dd49c36b276/media.jfrog.com/wp-content/uploads/2023/04/06191547/image2.png" alt="View of strings from within the binary showing that it is a .NET application compiled AOT" width="800" height="524" /></noscript><em>View of strings from within the binary showing that it is a .NET application compiled AOT</em></p>
<p>The payload also embeds two additional binary executables:</p>
<ol>
<li>A custom <strong>updater</strong> app that we will analyze later in this post</li>
<li><a href="https://github.com/Zerthox/rasar" target="_blank" rel="noopener">Rasar</a>, a tool used for extracting and compressing <a href="https://github.com/electron/asar" target="_blank" rel="noopener">Electron Archives</a></li>
</ol>
<h3>High-level overview of Impala Stealer</h3>
<p style="text-align: center"><img decoding="async" class="aligncenter wp-image-113255 size-full" src="data:image/svg+xml,%3Csvg%20xmlns='http://www.w3.org/2000/svg'%20viewBox='0%200%20412%20782'%3E%3C/svg%3E" alt="Impala Stealer and Updater flow graph" width="412" height="782" data-lazy-src="https://speedmedia.jfrog.com/08612fe1-9391-4cf3-ac1a-6dd49c36b276/media.jfrog.com/wp-content/uploads/2023/04/06191613/image3.png" /><noscript><img decoding="async" class="aligncenter wp-image-113255 size-full" src="https://speedmedia.jfrog.com/08612fe1-9391-4cf3-ac1a-6dd49c36b276/media.jfrog.com/wp-content/uploads/2023/04/06191613/image3.png" alt="Impala Stealer and Updater flow graph" width="412" height="782" /></noscript><em>Impala Stealer and Updater flow graph</em></p>
<h3>Installation and auto-update mechanism</h3>
<p>Upon execution, the payload first checks whether the directory <code>%USERPROFILE%\.nuget</code> exists or not. If it doesn’t exist, the payload exits immediately. This is probably a way to make sure that the payload was dropped into the system via NuGet.</p>
<p style="text-align: center"><img decoding="async" class="aligncenter wp-image-113256" src="data:image/svg+xml,%3Csvg%20xmlns='http://www.w3.org/2000/svg'%20viewBox='0%200%20925%20128'%3E%3C/svg%3E" alt="Checking for the existence of the %USERPROFILE%\.nuget folder" width="925" height="128" data-lazy-src="https://speedmedia.jfrog.com/08612fe1-9391-4cf3-ac1a-6dd49c36b276/media.jfrog.com/wp-content/uploads/2023/04/06191642/image4.png" /><noscript><img decoding="async" class="aligncenter wp-image-113256" src="https://speedmedia.jfrog.com/08612fe1-9391-4cf3-ac1a-6dd49c36b276/media.jfrog.com/wp-content/uploads/2023/04/06191642/image4.png" alt="Checking for the existence of the %USERPROFILE%\.nuget folder" width="925" height="128" /></noscript><em>Checking for the existence of the %USERPROFILE%\.nuget folder</em></p>
<p>Afterwards, the payload drops another executable (which was embedded in the original payload) to <code>%LOCALAPPDATA%\Squirrel-2021\Updater.exe</code>, and adds this path to the registry key found at <code>HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Run</code>, in order to make sure that it will be executed <a href="https://learn.microsoft.com/en-us/windows/win32/setupapi/run-and-runonce-registry-keys" target="_blank" rel="noopener">every time a user logs in to the system</a>.</p>
<p style="text-align: center"><img decoding="async" class="aligncenter wp-image-113257" src="data:image/svg+xml,%3Csvg%20xmlns='http://www.w3.org/2000/svg'%20viewBox='0%200%20900%20412'%3E%3C/svg%3E" alt="Installation of the malware via the Run registry key" width="900" height="412" data-lazy-src="https://speedmedia.jfrog.com/08612fe1-9391-4cf3-ac1a-6dd49c36b276/media.jfrog.com/wp-content/uploads/2023/04/06191712/image5.png" /><noscript><img decoding="async" class="aligncenter wp-image-113257" src="https://speedmedia.jfrog.com/08612fe1-9391-4cf3-ac1a-6dd49c36b276/media.jfrog.com/wp-content/uploads/2023/04/06191712/image5.png" alt="Installation of the malware via the Run registry key" width="900" height="412" /></noscript><em>Installation of the malware via the Run registry key</em></p>
<p>This executable, which we dubbed “The Updater,” is a .NET executable that tries to download an executable from a remote location, then saves it to the path <code>%PROGRAMDATA%\XboxGameBar\RuntimeBroker.exe</code>, and finally executes it.</p>
<p style="text-align: center"><img decoding="async" class="alignnone wp-image-113258" src="data:image/svg+xml,%3Csvg%20xmlns='http://www.w3.org/2000/svg'%20viewBox='0%200%20900%20153'%3E%3C/svg%3E" alt="" width="900" height="153" data-lazy-src="https://speedmedia.jfrog.com/08612fe1-9391-4cf3-ac1a-6dd49c36b276/media.jfrog.com/wp-content/uploads/2023/04/06191746/image6.png" /><noscript><img decoding="async" class="alignnone wp-image-113258" src="https://speedmedia.jfrog.com/08612fe1-9391-4cf3-ac1a-6dd49c36b276/media.jfrog.com/wp-content/uploads/2023/04/06191746/image6.png" alt="" width="900" height="153" /></noscript></p>
<p style="text-align: center"><img decoding="async" class="aligncenter wp-image-113259" src="data:image/svg+xml,%3Csvg%20xmlns='http://www.w3.org/2000/svg'%20viewBox='0%200%20900%20310'%3E%3C/svg%3E" alt="Action performed by the updating mechanism" width="900" height="310" data-lazy-src="https://speedmedia.jfrog.com/08612fe1-9391-4cf3-ac1a-6dd49c36b276/media.jfrog.com/wp-content/uploads/2023/04/06191809/image7.png" /><noscript><img decoding="async" class="aligncenter wp-image-113259" src="https://speedmedia.jfrog.com/08612fe1-9391-4cf3-ac1a-6dd49c36b276/media.jfrog.com/wp-content/uploads/2023/04/06191809/image7.png" alt="Action performed by the updating mechanism" width="900" height="310" /></noscript><em>Action performed by the updating mechanism</em></p>
<p>The URL that is passed to the <code>DownloadData</code> function is the same URL used by the PowerShell script in the NuGet package to download the payload. Hence, we concluded that this executable probably keeps the malware up to date, and protects it from deletion.</p>
<h3>Achieving Persistency through Code Injection</h3>
<p>After creating the Updater executable, the payload performs more actions to base itself in the system. If Discord or Microsoft Visual Studio Code are installed on the system, the payload injects JavaScript code into them, causing them to run <code>RuntimeBroker.exe</code> (the main Impala executable, downloaded by the Updater) on startup.</p>
<p>In Discord, JavaScript code is added to <code>%LOCALAPPDATA%\Discord\app-&lt;discord version&gt;\modules\discord_voice-2\discord_voice\index.js</code>:</p>
<p style="text-align: center"><img decoding="async" class="aligncenter wp-image-113263" src="data:image/svg+xml,%3Csvg%20xmlns='http://www.w3.org/2000/svg'%20viewBox='0%200%20700%20325'%3E%3C/svg%3E" alt="Code injected into Discord" width="700" height="325" data-lazy-src="https://speedmedia.jfrog.com/08612fe1-9391-4cf3-ac1a-6dd49c36b276/media.jfrog.com/wp-content/uploads/2023/04/06191845/image8.png" /><noscript><img decoding="async" class="aligncenter wp-image-113263" src="https://speedmedia.jfrog.com/08612fe1-9391-4cf3-ac1a-6dd49c36b276/media.jfrog.com/wp-content/uploads/2023/04/06191845/image8.png" alt="Code injected into Discord" width="700" height="325" /></noscript><em>Code injected into Discord</em></p>
<p>In Visual Studio Code, JavaScript code is added to <code>%LOCALAPPDATA%\Programs\Microsoft</code> VS <code>Code\resources\app\out\main.js</code>:</p>
<p style="text-align: center"><img decoding="async" class="aligncenter wp-image-113264" src="data:image/svg+xml,%3Csvg%20xmlns='http://www.w3.org/2000/svg'%20viewBox='0%200%20700%20288'%3E%3C/svg%3E" alt="Code injected into VS Code" width="700" height="288" data-lazy-src="https://speedmedia.jfrog.com/08612fe1-9391-4cf3-ac1a-6dd49c36b276/media.jfrog.com/wp-content/uploads/2023/04/06191912/image9.png" /><noscript><img decoding="async" class="aligncenter wp-image-113264" src="https://speedmedia.jfrog.com/08612fe1-9391-4cf3-ac1a-6dd49c36b276/media.jfrog.com/wp-content/uploads/2023/04/06191912/image9.png" alt="Code injected into VS Code" width="700" height="288" /></noscript><em>Code injected into VS Code</em></p>
<h3>“Liberating” funds from your Exodus Wallet</h3>
<p>After basing itself on the system, Impala begins execution of its main payload.</p>
<p>The payload searches for an installation of the <a href="https://www.exodus.com/" target="_blank" rel="noopener">Exodus Wallet</a> Desktop Application. If found, the malware then looks for the file <code>%LOCALAPPDATA%\exodus\app-&lt;exodus version&gt;\resources\app.asar</code>. This file is an <a href="https://github.com/electron/asar" target="_blank" rel="noopener">Electron Archive</a>, containing resources, web pages and JavaScript code for the Exodus app. The payload creates a backup of the file, and then extracts it, using an embedded executable copy of <a href="https://github.com/Zerthox/rasar" target="_blank" rel="noopener">Rasar</a>.</p>
<p>After extracting the archive, the payload tries to download a JavaScript code snippet from an online paste website:</p>
<p style="text-align: center"><img decoding="async" class="aligncenter wp-image-113266" src="data:image/svg+xml,%3Csvg%20xmlns='http://www.w3.org/2000/svg'%20viewBox='0%200%20784%2084'%3E%3C/svg%3E" alt="Paste URL from the payload binary" width="784" height="84" data-lazy-src="https://speedmedia.jfrog.com/08612fe1-9391-4cf3-ac1a-6dd49c36b276/media.jfrog.com/wp-content/uploads/2023/04/06191935/image10.png" /><noscript><img decoding="async" class="aligncenter wp-image-113266" src="https://speedmedia.jfrog.com/08612fe1-9391-4cf3-ac1a-6dd49c36b276/media.jfrog.com/wp-content/uploads/2023/04/06191935/image10.png" alt="Paste URL from the payload binary" width="784" height="84" /></noscript><em>Code injected into VS Code</em></p>
<p>At the time of writing, this paste was already deleted, and so we couldn’t obtain its contents, but we can speculate about its functionality from the performed operations:</p>
<p>The payload edits 3 of the extracted Exodus app’s JavaScript files:</p>
<ol>
<li>In <code>src\static\wallet.html</code> Impala adds the url <a href="https://discord.com/" target="_blank" rel="noopener">https://discord.com/</a> to the <a href="https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Content-Security-Policy/connect-src" target="_blank" rel="noopener">Content-Security-Policy</a>, allowing pages to connect to Discord domains &#8211;  .
<p style="text-align: center"><img decoding="async" class="aligncenter wp-image-113267" src="data:image/svg+xml,%3Csvg%20xmlns='http://www.w3.org/2000/svg'%20viewBox='0%200%20900%2016'%3E%3C/svg%3E" alt="Code injected into Exodus’ src\static\wallet.html" width="900" height="16" data-lazy-src="https://speedmedia.jfrog.com/08612fe1-9391-4cf3-ac1a-6dd49c36b276/media.jfrog.com/wp-content/uploads/2023/04/06191959/image11.png" /><noscript><img decoding="async" class="aligncenter wp-image-113267" src="https://speedmedia.jfrog.com/08612fe1-9391-4cf3-ac1a-6dd49c36b276/media.jfrog.com/wp-content/uploads/2023/04/06191959/image11.png" alt="Code injected into Exodus’ src\static\wallet.html" width="900" height="16" /></noscript><em>Code injected into Exodus’ src\static\wallet.html</em></p>
<p><span style="font-size: 16px">This is performed in order to allow Impala to exfiltrate sensitive data via a hardcoded Discord webhook: </span><em style="font-size: 16px">https[:]//discord[.]com/api/webhooks/1076330498026115102/MLkgrUiivlgAoFWyvkSpLsBE3DMaDZd9cxPK3k9XQPyh6dw55jktV6qfDgxbs5AaY7Py<br />
</em></li>
<li>In <code>src\app\main\index.js</code> Impala adds <code>discord.com</code> to the wallet domains:
<p style="text-align: center"><img decoding="async" class="wp-image-113268 aligncenter" src="data:image/svg+xml,%3Csvg%20xmlns='http://www.w3.org/2000/svg'%20viewBox='0%200%20800%20171'%3E%3C/svg%3E" alt="Code injected into Exodus’ src\app\main\index.js" width="800" height="171" data-lazy-src="https://speedmedia.jfrog.com/08612fe1-9391-4cf3-ac1a-6dd49c36b276/media.jfrog.com/wp-content/uploads/2023/04/06192034/image12.png" /><noscript><img decoding="async" class="wp-image-113268 aligncenter" src="https://speedmedia.jfrog.com/08612fe1-9391-4cf3-ac1a-6dd49c36b276/media.jfrog.com/wp-content/uploads/2023/04/06192034/image12.png" alt="Code injected into Exodus’ src\app\main\index.js" width="800" height="171" /></noscript><em>Code injected into Exodus’ src\app\main\index.js</em></p>
<p>This is probably done for the same reason as mentioned above (exfiltration through Discord).</li>
<li style="text-align: center">In <code>src\app\wallet\index.js</code> Impala adds the code that it pulled from the online paste after an existing call to <code>await this._loadLightningCreds()</code>.<img decoding="async" class="aligncenter wp-image-113269" src="data:image/svg+xml,%3Csvg%20xmlns='http://www.w3.org/2000/svg'%20viewBox='0%200%20800%20280'%3E%3C/svg%3E" alt="Code injected into Exodus’ src\app\wallet\index.js" width="800" height="280" data-lazy-src="https://speedmedia.jfrog.com/08612fe1-9391-4cf3-ac1a-6dd49c36b276/media.jfrog.com/wp-content/uploads/2023/04/06192108/image13.png" /><noscript><img decoding="async" class="aligncenter wp-image-113269" src="https://speedmedia.jfrog.com/08612fe1-9391-4cf3-ac1a-6dd49c36b276/media.jfrog.com/wp-content/uploads/2023/04/06192108/image13.png" alt="Code injected into Exodus’ src\app\wallet\index.js" width="800" height="280" /></noscript>
<p><em>Code injected into Exodus’ src\app\wallet\index.js</em></li>
</ol>
<p>Since we weren’t able to get the injected JavaScript code from the online paste, we can’t exactly say what it was meant to do.  However, we examined the code in <code>src\app\wallet\index.js</code>, and were able to tell that the call to <code>_loadLightningCreds</code> is done after the Exodus Wallet user inserts their password to login to the wallet. At that point, the <code>_loadLightningCreds</code> uses the credentials to decrypt several files.</p>
<p>The code that was supposed to be added from the deleted paste might have been used to steal the user’s credentials for Exodus and other cryptocurrency exchange platforms, or use them to decrypt important files of the Exodus wallet. After injecting the malicious code to the JavaScript files, the payload compresses them back into the “app.asar” file, causing Exodus to run the malicious code when it runs.</p>
<p>After injecting the malicious code into Exodus, Impala sends a message to the hardcoded Discord webhook mentioned earlier, containing the following data:</p>
<ul>
<li style="list-style-type: none">
<ul>
<li>content:<br />
<code>``css<br />
</code><code>user's Exodus has been pwned sorry g<br />
</code><code>```</code></li>
<li>username: <code>Exodus Detection by Impala</code></li>
</ul>
</li>
</ul>
<ul>
<li style="list-style-type: none"></li>
</ul>
<ul>
<li>avatar_url: <em>https[:]//www.startpage[.]com/av/proxy-image?piurl=https%3A%2F%2Fupload.wikimedia.org%2Fwikipedia%2Fen%2F9%2F9b%2FTame_Impala_-_Currents.png&amp;sp=1676686318T1f4463dbc9a9476b9d1c842a525a4dcfb610c3d6875da7a162ab7fc8136a17b2</em></li>
</ul>
<p>This message is probably used to notify the attacker about a successful attack.</p>
<h2>Conclusion</h2>
<p>In conclusion, supply chain attacks, which have been on the rise in the last few years, should also be considered a serious risk for NuGet developers. Our research team had exposed for the first time a sophisticated attack, in which several malicious packages were propagated, relying on typosquatting techniques and impersonating legitimate packages. According to our analysis, the attack’s payload mainly targets the Exodus Wallet application by injecting it with malicious code meant to leak the victim’s credentials to cryptocurrency exchanges.</p>
<p>.NET developers — see our previous blog post for tips on how to identify and protect yourself from <a href="https://jfrog.com/blog/attackers-are-starting-to-target-net-developers-with-malicious-code-nuget-packages/">malicious NuGet packages</a>.</p>
