</style><p>Today is Microsoft's November 2024 Patch Tuesday, which includes security updates for 91 flaws, including four zero-days, two of which are actively exploited.</p>

<p>This Patch Tuesday fixed four critical vulnerabilities, which include two remote code execution and two elevation of privileges flaws.</p>

<p>The number of bugs in each vulnerability category is listed below:</p>

<ul style="list-style-type:square"><li>26 Elevation of Privilege vulnerabilities</li>
	<li>2 Security Feature Bypass vulnerabilities</li>
	<li>52 Remote Code Execution vulnerabilities</li>
	<li>1 Information Disclosure vulnerability</li>
	<li>4 Denial of Service vulnerabilities</li>
	<li>3 Spoofing vulnerabilities</li>
</ul><p>This count does not include two Edge flaws that were previously fixed on November 7th.</p>

<p>To learn more about the non-security updates released today, you can review our dedicated articles on the new <a href="https://www.bleepingcomputer.com/news/microsoft/windows-11-kb5046617-and-kb5046633-cumulative-updates-released/">Windows 11 KB5046617 and KB5046633 cumulative updates</a>&nbsp;and the&nbsp;<a href="https://www.bleepingcomputer.com/news/microsoft/windows-10-kb5046613-update-released-with-fixes-for-printer-bugs/">Windows 10 KB5046613 update</a>.</p>

<h2>Four zero-days disclosed</h2>

<p>This month's Patch Tuesday fixes four zero-days, two of which were actively exploited in attacks, and three were publicly disclosed.</p>

<p>Microsoft classifies a zero-day flaw as one that is publicly disclosed or actively exploited while no official fix is available.</p>

<p>The two actively exploited zero-day vulnerabilities in today's updates are:</p>

<p style="margin-left:40px"><strong><a href="http://msrc.microsoft.com/update-guide/en-US/advisory/CVE-2024-43451" target="_blank" rel="nofollow noopener">CVE-2024-43451</a> - NTLM Hash Disclosure Spoofing Vulnerability</strong></p>

<p style="margin-left:40px">Microsoft has fixed a vulnerability that exposes NTLM hashes to remote attackers with minimal interaction with a malicious file.</p>

<p style="margin-left:40px">"This vulnerability discloses a user's NTLMv2 hash to the attacker who could use this to authenticate as the user," explained Microsoft.</p>

<p style="margin-left:40px">"Minimal interaction with a malicious file by a user such as selecting (single-click), inspecting (right-click), or performing an action other than opening or executing could trigger this vulnerability," continued Microsoft.</p>

<p style="margin-left:40px">Microsoft says Israel Yeshurun of ClearSky Cyber Security discovered this vulnerability and that it was publicly disclosed, but did not share any further details.</p>

<p style="margin-left:40px"><strong><a href="http://msrc.microsoft.com/update-guide/en-US/advisory/CVE-2024-49039" target="_blank" rel="nofollow noopener">CVE-2024-49039</a> -&nbsp;Windows Task Scheduler Elevation of Privilege Vulnerability</strong></p>

<p style="margin-left:40px">A specially crafted application could be executed that elevates privilege to Medium Integrity level.</p>

<p style="margin-left:40px">"In this case, a successful attack could be performed from a low privilege&nbsp;<a href="https://docs.microsoft.com/windows/win32/secauthz/appcontainer-isolation" rel="nofollow noopener">AppContainer</a>. The attacker could elevate their privileges and execute code or access resources at a higher integrity level than that of the AppContainer execution environment," explained Microsoft.</p>

<p style="margin-left:40px">Microsoft says that exploiting this vulnerability would allow attackers to&nbsp;execute RPC functions that are normally restricted to privileged accounts.</p>

<p style="margin-left:40px">The flaw was discovered by Vlad Stolyarov and Bahare Sabouri of Google's Threat Analysis Group.</p>

<p style="margin-left:40px">It is not known how the flaw was exploited in attacks.</p>

<p>The other three vulnerabilities that were publicly disclosed&nbsp;but not exploited in attacks are:</p>

<p style="margin-left:40px"><strong><a href="http://msrc.microsoft.com/update-guide/vulnerability/CVE-2024-49040" target="_blank" rel="nofollow noopener">CVE-2024-49040</a> - Microsoft Exchange Server Spoofing Vulnerability</strong></p>

<p style="margin-left:40px">Microsoft has fixed a Microsoft Exchange vulnerability that allows threat actors to spoof the sender's email address in emails to local recipients.</p>

<p style="margin-left:40px">"Microsoft is aware of a vulnerability (<a data-linktype="external" href="https://msrc.microsoft.com/update-guide/vulnerability/CVE-2024-49040" rel="nofollow noopener">CVE-2024-49040</a>) that allows attackers to run spoofing attacks against Microsoft Exchange Server," explains a <a href="https://learn.microsoft.com/en-us/exchange/plan-and-deploy/post-installation-tasks/security-best-practices/exchange-non-compliant-p2from-detection?view=exchserver-2019" target="_blank" rel="nofollow noopener">related advisory</a> by Microsoft.</p>

<p style="margin-left:40px">"The vulnerability is caused by the current implementation of the&nbsp;<code>P2 FROM</code>&nbsp;header verification, which happens in transport."</p>

<p style="margin-left:40px">Starting with this month's Microsoft Exchange security updates, Microsoft is now detecting and flagging spoofed emails with an alert prepended to the email body that states, "Notice: This email appears to be suspicious. Do not trust the information, links, or attachments in this email without verifying the source through a trusted method."</p>

<p style="margin-left:40px">Microsoft says the flaw was discovered by <a href="https://msrc.microsoft.com/update-guide/vulnerability/CVE-2024-49040" target="_blank" rel="nofollow noopener">Slonser</a> at Solidlab, who publicly disclosed the flaw in <a href="https://blog.slonser.info/posts/email-attacks/" target="_blank" rel="nofollow noopener">this article</a>.</p>

<p style="margin-left:40px"><strong><a href="http://msrc.microsoft.com/update-guide/vulnerability/CVE-2024-49019" target="_blank" rel="nofollow noopener">CVE-2024-49019</a> -&nbsp;Active Directory Certificate Services Elevation of Privilege Vulnerability</strong></p>

<p style="margin-left:40px">Microsoft fixed a flaw that allows attackers to gain domain administrator privileges by abusing&nbsp;built-in default version 1 certificate templates.</p>

<p style="margin-left:40px">"Check if you have published any certificates created using a&nbsp;<a href="https://learn.microsoft.com/en-us/windows-server/identity/ad-cs/certificate-template-concepts#certificate-template-versions" rel="nofollow noopener">version 1 certificate template</a>&nbsp;where the&nbsp;<strong>Source of subject name</strong>&nbsp;is set to "Supplied in the request" and the&nbsp;<strong>Enroll</strong>&nbsp;permissions are granted to a broader set of accounts, such as domain users or domain computers," explains Microsoft.</p>

<p style="margin-left:40px">"An example is the built-in&nbsp;<strong>Web Server</strong>&nbsp;template, but it is not vulnerable by default due to its restricted&nbsp;<strong>Enroll</strong>&nbsp;permissions."</p>

<p style="margin-left:40px">The flaw was discovered by&nbsp;<a href="https://x.com/louscicchitano/" target="_blank" rel="nofollow noopener">Lou Scicchitano</a>, <a href="https://x.com/slobtresix0/" target="_blank" rel="nofollow noopener">Scot Berner</a>, and <a href="https://x.com/bandrel/" target="_blank" rel="nofollow noopener">Justin Bollinger</a> with TrustedSec, who disclosed the&nbsp;"EKUwu" vulnerability in October.</p>

<p style="margin-left:40px">"Using built-in default version 1 certificate templates, an attacker can craft a CSR to include application policies that are preferred over the configured Extended Key Usage attributes specified in the template," reads <a href="http://trustedsec.com/blog/ekuwu-not-just-another-ad-cs-esc" target="_blank" rel="nofollow noopener">TrustedSec's report</a>.</p>

<p style="margin-left:40px">"The only requirement is enrollment rights, and it can be used to generate client authentication, certificate request agent, and codesigning certificates using the&nbsp;<strong><em>WebServer</em></strong>&nbsp;template."</p>

<p style="margin-left:40px">As explained above,&nbsp;<strong>CVE-2024-43451</strong> was also publicly disclosed.</p>

function cz_strip_tags(input, allowed) {
  allowed = (((allowed || '') + '')
    .toLowerCase()
    .match(/<[a-z][a-z0-9]*>/g) || [])
    .join(''); // making sure the allowed arg is a string containing only tags in lowercase (<a><b><c>)
  var tags = /<\/?([a-z][a-z0-9]*)\b[^>]*>/gi,
    commentsAndPhpTags = /<!--[\s\S]*?-->|<\?(?:php)?[\s\S]*?\?>/gi;
  return input.replace(commentsAndPhpTags, '')
    .replace(tags, function($0, $1) {
      return allowed.indexOf('<' + $1.toLowerCase() + '>') > -1 ? $0 : '';
    });
}
function cz_br2nl(str) {
	var regex = /<br\s*[\/]?>/gi;
    //var pure_str = str.replace(regex,"\n");
    var pure_str = str.replace(regex,"");
    return cz_strip_tags(pure_str,'');

$(window).scroll(function(event){
    scrollTop = $(this).scrollTop();

    if ( scrollTop < 0 ) {
        scrollTop = 0;
    }
    if ( scrollTop > $('body').height() - $(window).height() ) {
        scrollTop = $('body').height() - $(window).height();
    }

    if (scrollTop >= prevScrollTop && scrollTop) {
        $('header').addClass("nav-up");

	 $('header').removeClass("nav-up");

	}
	else
	{
	 $('header').removeClass("nav-up");
	 clearInterval(main_nav_hide_timer);
	}
    }

    prevScrollTop = scrollTop;
});
