It also covers other tactics, techniques and procedures (TTPs) observed during this attack.\r\n\r\nIn addition, we discuss Repellent Scorpius' connection to a historical incident involving data exfiltration, predating the group's operation under the Cicada3301 brand, as well as the ransomware group\u2019s plans going forward. Despite its recent inception, it is quickly picking up pace by setting up an affiliate program and recruiting partners. This has increased its number of victims according to the leak site.\r\n\r\nThere is an intriguing background associated with the name under which the ransomware group operates. According to Wikipedia, the name 3301 refers to three sets of highly complex and mysterious puzzles that first appeared on 4chan between 2012-2014, all signed with the pseudonym 3301. The third set of these puzzles remains unsolved to this day.\r\n\r\nBased on the timeline from a Unit 42 Incident Response engagement, we estimate that the ransomware group began their operations in May 2024. Owing to the absence of other reports, we believe that this may be the beginning of their operations.\r\n\r\nWhile the incidents may have begun around that time, we started to observe leak site activity in June. Despite a lack of activity on the leak site for around a month since June 19, the ransomware group has resumed operations.\r\n\r\nOf note, we have observed signs that the group has data obtained in older compromise incidents. It is unclear whether this means that the threat actor previously operated using differently branded ransomware, or whether they have purchased or inherited data from other ransomware groups.\r\n\r\n[caption id=\"attachment_136643\" align=\"alignnone\" width=\"900\"] Figure 1. Cicada3301 leak site as of July 2024.[\/caption]\r\n\r\nRepellent Scorpius employs a double extortion scheme of encrypting systems. This entails stealing data and threatening to publish it if the victim doesn\u2019t pay the ransom.\r\n\r\nUnit 42 has evidence to suggest that the Repellent Scorpius operators have developed a RaaS affiliate program. It operates a control panel for affiliates and ransom payment pages for victims, and actively recruits initial access brokers (IAB) and network intruders on Russian-language cybercrime forums.\r\n\r\nGiven the limited number of victims, it might be too early to suggest whether this ransomware group targets a particular sector or region. Having said that, one of the points in the FAQ section on the affiliate panel website says, \u201cIt is strictly prohibited to target the CIS countries.\u201d (Translated from Russian.)\r\n\r\nKrakenLabs posted a screenshot on X (formerly Twitter), displaying a Russian translated post by the Repellent Scorpius ransomware group on an underground forum to recruit partners for their affiliate program.\r\nIncident Attack Lifecycle\r\nWe have mapped the attack stages captured from our incident response engagement to the MITRE ATT&amp;CK\u00ae framework tactics, which we summarize below.\r\nInitial Access\r\nMultiple Remote Desktop Protocol (RDP) logon events were captured on a given host. Based on investigation findings and the group\u2019s modus operandi, we assess that attackers achieved initial access through stolen credentials, possibly purchased from an IAB.\r\n\r\nThe public IP address predominantly associated was 103.42.240[.]37, as an RDP server with the hostname: WIN-RMM48SHAUPR. This IP address is associated with a Pakistan-based hosting provider 0DAYHOST (SMC-PRIVATE) LIMITED, while the autonomous system name indicates that Serverius Holding B.V. controls the IP address allocation.\r\nExecution\r\nUnit 42 investigators observed attackers employing a batch script named 1.bat to execute the ransomware payload against multiple hosts within the client network. Details regarding the ransomware payload, along with its arguments, are below.\r\n\r\n[caption id=\"attachment_136645\" align=\"alignnone\" width=\"900\"] Figure 2. Batch script with multiple ransomware command executions.[\/caption]\r\nLateral Movement\r\nPsExec is a legitimate tool that attackers leveraged to execute the ransomware payload against different hosts within the network. The tool is embedded within the ransomware payload and later extracted, which we describe in further detail below. It was executed through the following PowerShell command:\r\npowershell -Command C:\\Users\\Public\\psexec0.exe -accepteula -s -d \"C:\\Users\\Public\\locker.exe\" --no_impl --key &lt;redacted&gt; -p \\\\&lt;hostname&gt;\\C$\r\nCollection\r\nUnit 42 investigators found the creation of the following file C:\\ProgramData\\found_shares.txt. There have been previous occurrences of PowerView, a PowerSploit PowerShell module, storing file share enumeration results in the same file path and multiple ransomware intrusions have leveraged this technique.\r\nExfiltration\r\nUnit 42 investigators identified Rclone (a legitimate open-source utility) as the tool used for exfiltration. Attackers installed the tool in the ProgramData file path (C:\\ProgramData\\rclone.exe), along with the configuration file (C:\\ProgramData\\rclone.conf).\r\n\r\nWe observed 91.238.181[.]238 was the public IP address attackers used for exfiltration activity. This IP address comes from a hosting provider called VDS&amp;VPN services.\r\n\r\nThe IP address in question has previously been flagged for Cobalt Strike activity (watermark: 674054486) and was potentially linked to other ransomware groups such as Bashful Scorpius (aka Nokoyawa) and Ambitious Scorpius (aka ALPHV\/BlackCat) in 2023. This IP address was also observed trying to exploit ScreenConnect vulnerabilities, (CVE-2024-1708 and CVE-2024-1709) in February 2024.\r\nImpact (Encryptor)\r\nThe ransomware is a 64-bit binary written in the programing language Rust, which accepts the following command-line arguments:\r\nUSAGE:\r\n\r\nlocker.exe [FLAGS] [OPTIONS]\r\n\r\nAdditional Information:\r\n\r\n--key Sets the keys for activation (Required parameter)\r\n\r\n-p, --path Sets the path to the file or directory to be encrypted\r\n\r\n-s, --sleep Sleep is indicated in seconds\r\n\r\n--no_local Skip encrypting data stored locally on this device\r\n\r\n--no_net Skip encryption of network data\r\n\r\n--no_impl Don't use impersonation\r\nFigure 2 shows the threat actors used a batch script to execute the ransomware multiple times against a list of hard-coded directory paths in the victim network. The encryptor requires a key parameter to begin execution, which has been redacted from the image.\r\n\r\nThe binary performs a key validation routine, in which it attempts to decrypt an embedded ransom note using the ChaCha20 stream cipher [PDF].\r\n\r\nThe ransomware note is Base64-decoded. It is then decrypted using the first 32 bytes of the submitted key as the ChaCha20 secret key and the last 12 bytes of the submitted key as the nonce. Then it is Base64-decoded a final time.\r\n\r\nThe encryptor will validate the decryption process by checking whether the string ***is_ok*** exists in the decrypted data. If the validation is successful, execution proceeds.\r\n\r\nThe encryptor contains a legitimate copy of PsExec embedded within itself, which it will extract and save to the location C:\\Users\\Public\\psexec0.exe. The malware will then create a copy of itself in the C:\\Users\\Public\\ directory.\r\n\r\nOnce copied, it will use the PsExec binary to execute itself several more times, using hard-coded credentials stolen from the victim network during the preceding incursion. This may be an attempt to get the encryptor to run with higher privileges.\r\nC:\\Users\\Public\\psexec0.exe -accepteula -s -d \"C:\\Users\\Public\\&lt;encryptor&gt;\" --no_impl --key &lt;key&gt;\r\n\r\nC:\\Users\\Public\\psexec0.exe -accepteula -u &lt;username&gt; -p &lt;password&gt; -s -d \"C:\\Users\\Public\\&lt;encryptor&gt;\" --no_impl --key &lt;key&gt;\r\nNext, the encryptor will run a series of commands to terminate services and processes, delete shadow copies and disable recovery features among other tasks. A list of the executed commands is below, and a full list of targeted processes and services is in the appendix.\r\n\r\n\r\n\r\nCommand\r\nPurpose\r\n\r\n\r\ncmd \/C fsutil behavior set SymlinkEvaluation R2L:1\r\nEnables remote to local symbolic links\r\n\r\n\r\ncmd \/C fsutil behavior set SymlinkEvaluation R2R:1\r\nEnables remote to remote symbolic links\r\n\r\n\r\ncmd \/C iisreset.exe \/stop\r\nStops Internet Information Services (IIS)\u00a0\r\n\r\n\r\ncmd \/C vssadmin.exe Delete Shadows \/all \/quiet\r\nDeletes volume shadow copies using VSSAdmin.exe\r\n\r\n\r\ncmd \/C wmic.exe Shadowcopy Delete\r\nDeletes volume shadow copies using the Windows Management Instrumentation Command-Line (WMIC) utility\r\n\r\n\r\ncmd \/C bcdedit \/set {default}\r\nPossibly a misused command, as it requires additional parameters to execute properly\r\n\r\n\r\ncmd \/C bcdedit \/set {default} recoveryenabled No\r\nDisables the automatic recover feature for the default boot entry\r\n\r\n\r\ncmd \/C \u201cfor \/F 'tokens=*' %1 in ('wevtutil.exe el') DO wevtutil.exe cl %1\u201d\r\nClears Windows Event Logs\r\n\r\n\r\ncmd \/C reg add HKEY_LOCAL_MACHINE\\SYSTEM\\CurrentControlSet\\Services\\LanmanServer\\Parameters \/v MaxMpxCt \/d 65535 \/t REG_DWORD \/f\r\nSets the number of concurrent network requests to the maximum allowed\r\n\r\n\r\ncmd \/C sc stop &lt;service&gt;\r\nStops the specified service\r\n\r\n\r\ncmd \/C taskkill \/IM &lt;process&gt;* \/F\r\nForcefully terminates the specified process\r\n\r\n\r\n\r\nOnce the sample completes the above processes, it begins the encryption routine. By default, the ransomware will check for the existence of drives from A:\\ to Z:\\. The sample encrypts all files within detected drives, excluding files with specific extensions or files located in directories matching keywords. This information is listed in the appendix.\r\n\r\nThe encryptor renames files with a new extension before starting the encryption process. In the sample analyzed by Unit 42 the extension was kcr5umw.\r\n\r\nThe encryption process is composed of two sequences. First, the encryptor will read the contents of the target file, encrypt the contents using ChaCha20 with a randomly generated key secret and nonce bytes, and write the result back to the file. Second, the ChaCha20 key and nonce are encrypted using a hard-coded RSA public key, and the result is appended to the file. Finally, the extension is appended to the end of the encrypted data.\r\n\r\nOnce encryption of all files is complete, the sample will write the ransom note, named in the format RECOVER-&lt;encrypted_file_extension&gt;-DATA.txt. The contents of the note are as follows.\r\n*************************************\r\n\r\n*** Welcome to Cicada3301 ***\r\n\r\n*************************************\r\n\r\n** What Happened? **\r\n\r\n----------------------------------------------\r\n\r\nYour computers and servers are encrypted, your backups are deleted.\r\n\r\nWe use strong encryption algorithms, so you won't be able to decrypt your data.\r\n\r\nYou can recover everything by purchasing a special data recovery program from us.\r\n\r\nThis program will restore your entire network.\r\n\r\n** Data Leak **\r\n\r\n----------------------------------------------\r\n\r\nWe have downloaded more than %SIZE% GB of your company data.\r\n\r\nContact us, or we will be forced to publish all your data on the Internet\r\n\r\nand send it to all regulatory authorities in your country, as well as to your customers, partners, and competitors.\r\n\r\nWe are ready to:\r\n\r\n- Provide you with proof that the data has been stolen;\r\n\r\n- Delete all stolen data;\r\n\r\n- Help you rebuild your infrastructure and prevent similar attacks in the future;\r\n\r\n** What Guarantees? **\r\n\r\n----------------------------------------------\r\n\r\nOur reputation is of paramount importance to us.\r\n\r\nFailure to fulfill our obligations means not working with you, which is against our interests.\r\n\r\nRest assured, our decryption tools have been thoroughly tested and are guaranteed to unlock your data.\r\n\r\nShould any problems arise, we are here to support you. As a goodwill gesture,\r\n\r\nwe are willing to decrypt one file for free.\r\n\r\n** How to Contact us? **\r\n\r\n----------------------------------------------\r\n\r\nUsing TOR Browser:\r\n\r\n1) You can download and install the TOR browser from this site: https:\/\/torproject.org\/\r\n\r\n2) Open our website: &lt;redacted&gt;\r\n\r\nWARNING: DO NOT MODIFY or attempt to restore any files on your own. This can lead to their permanent loss.\r\nLinks to Historical Incident\r\nUnit 42 is aware of at least one case where Repellent Scorpius had access to a victim\u2019s data, which attackers likely took in an incident several years prior. A forensic review of the victim\u2019s environment identified no recent signs of compromise. A quick walkthrough of some of the TTPs observed during that incident were as follows:\r\n\r\n\r\n\r\nMITRE Tactic\r\nDescription\r\n\r\n\r\nExecution\r\n\r\n\r\n \tWinRAR to extract certain tools from its archive.\u00a0\r\n \tCertutil leveraged for payload download.\u00a0\r\n\r\n\r\n\r\n\r\nPersistence\r\nMultiple scheduled tasks were set up for hourly execution of different commands.\r\n\r\n\r\nCredential access\r\nWe observed the presence of tools such as Mimikatz and Impacket-based executables, primarily used for extracting credentials.\r\n\r\n\r\nDiscovery\r\n\r\n\r\n \tExecution of ADRecon PowerShell script to gather information and extract artifacts from a given Active Directory, and a Rubeus based executable.\r\n \tSome of the tools or built-in commands attackers used were wmic, nslookup, ping, ipconfig, net, quser, qwinsta and SoftPerfect Network Scanner.\r\n\r\n\r\n\r\n\r\nCommand and control\r\n\r\n\r\n \tReverse tunnel with adversary server via SSH\r\n \tPowerShell command to send the victim IP address and hostname to a given hard-coded domain, via POST request.\r\n \tMultiple other tools were used in this scenario, including Plink, GOST and a SOCKS proxy tool.\r\n\r\n\r\n\r\n\r\n\r\n As previously mentioned, it is unclear how the Repellent Scorpius group possessed this data. However, we observed certain overlaps with another attack carried out by an affiliate that deployed BlackCat ransomware, reported in March 2022.\r\n\r\n Examples include attackers using ADRecon and SoftPerfect Network Scanner tools, setting up a reverse SSH tunnel and creating similar scheduled tasks. That said, there was no evidence that BlackCat was deployed in this incident, likely due to the fact that different stages of the attack were thwarted.\r\n\r\n While we did come across a few filename-based overlaps, we observed no substantial TTP overlaps between the recent ransomware incident and the historical one.\r\nNew Version of Encryptor\r\nUnit 42 researchers found an updated Cicada3301 encryptor in late July 2024, which had some differences from the previously analyzed version.\r\n\r\nThreat authors added a new command-line argument, --no-note. When this argument is invoked, the encryptor will not write the ransom note to the system.\r\n\r\nInstead of running the embedded PsExec binary directly via PowerShell, the encryptor will create a randomly named .bat file in the C:\\Users\\Public directory, which executes using \u201ccmd.exe \/C\u201d.                 <h2><a id="post-136638-_n41yi9jwwr88"></a>Executive Summary</h2>
<p>Repellent Scorpius is a new ransomware-as-a-service (RaaS) group that distributes Cicada3301 ransomware. The ransomware group appears to have first emerged in May 2024, with a multi-extortion operation.</p>
<p>This report based on Unit 42 Incident Response engagements provides a technical analysis of the ransomware employed by the Repellent Scorpius group. It also covers other tactics, techniques and procedures (TTPs) observed during this attack.</p>
<p>In addition, we discuss Repellent Scorpius' connection to a historical incident involving data exfiltration, predating the group's operation under the Cicada3301 brand, as well as the ransomware group’s plans going forward. Finally, we provide a walkthrough of an updated encryptor obtained through external sources, highlighting the differences from its previous variant. Unit 42 anticipates a rise in Cicada3301 ransomware activity, leading to an increase in the number of victims.</p>

<h2><a id="post-136638-_b18h6peqprv3"></a>Repellent Scorpius Threat Overview</h2>
<p>Repellent Scorpius (distributors of Cicada3301 ransomware) is a new threat group that has recently emerged in the wild. Despite its recent inception, it is quickly picking up pace by setting up an affiliate program and recruiting partners. This has increased its number of victims according to the leak site.</p>
<p>There is an intriguing background associated with the name under which the ransomware group operates. According to <a href="https://en.wikipedia.org/wiki/Cicada_3301" target="_blank" rel="noopener">Wikipedia</a>, the name 3301 refers to three sets of highly complex and mysterious puzzles that first appeared on 4chan between 2012-2014, all signed with the pseudonym 3301. The third set of these puzzles remains unsolved to this day.</p>
<p>Based on the timeline from a Unit 42 Incident Response engagement, we estimate that the ransomware group began their operations in May 2024. Owing to the absence of other reports, we believe that this may be the beginning of their operations.</p>
<p>While the incidents may have begun around that time, we started to observe leak site activity in June. Despite a lack of activity on the leak site for around a month since June 19, the ransomware group has resumed operations.</p>
<p>Of note, we have observed signs that the group has data obtained in older compromise incidents. It is unclear whether this means that the threat actor previously operated using differently branded ransomware, or whether they have purchased or inherited data from other ransomware groups.</p>
<figure id="attachment_136643" aria-describedby="caption-attachment-136643" style="width: 900px" class="wp-caption alignnone"><img decoding="async" class="wp-image-136643" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2024/09/word-image-766319-136638-1.png" alt="Screenshot of Cicada3301 leak site showing multiple posts with publication dates. Cicada3301 leak site as of July 2024.</figcaption></figure>
<p>Repellent Scorpius employs a double extortion scheme of encrypting systems. This entails stealing data and threatening to publish it if the victim doesn’t pay the ransom.</p>
<p>Unit 42 has evidence to suggest that the Repellent Scorpius operators have developed a RaaS affiliate program. It operates a control panel for affiliates and ransom payment pages for victims, and actively recruits initial access brokers (IAB) and network intruders on Russian-language cybercrime forums.</p>
<p>Given the limited number of victims, it might be too early to suggest whether this ransomware group targets a particular sector or region. Having said that, one of the points in the FAQ section on the affiliate panel website says, “<span style="font-family: 'courier new', courier, monospace;">It is strictly prohibited to target the CIS countries</span>.” (Translated from Russian.)</p>
<p>KrakenLabs <a href="https://x.com/KrakenLabs_Team/status/1812805505594847449" target="_blank" rel="noopener">posted a screenshot on X</a> (formerly Twitter), displaying a Russian translated post by the Repellent Scorpius ransomware group on an underground forum to recruit partners for their affiliate program.</p>
<h2><a id="post-136638-_dgfxry9r7u1e"></a>Incident Attack Lifecycle</h2>
<p>We have mapped the attack stages captured from our incident response engagement to the <a href="https://attack.mitre.org/" target="_blank" rel="noopener">MITRE ATT&amp;CK® framework</a> tactics, which we summarize below.</p>
<h3><a id="post-136638-_ufr92wipdtul"></a>Initial Access</h3>
<p>Multiple Remote Desktop Protocol (RDP) logon events were captured on a given host. Based on investigation findings and the group’s modus operandi, we assess that attackers achieved initial access through stolen credentials, possibly purchased from an IAB.</p>
<p>The public IP address predominantly associated was <span style="font-family: 'courier new', courier, monospace;">103.42.240[.]37</span>, as an RDP server with the hostname: <span style="font-family: 'courier new', courier, monospace;">WIN-RMM48SHAUPR</span>. This IP address is associated with a Pakistan-based hosting provider <span style="font-family: 'courier new', courier, monospace;">0DAYHOST (SMC-PRIVATE) LIMITED</span>, while the autonomous system name indicates that <span style="font-family: 'courier new', courier, monospace;">Serverius Holding B.V.</span> controls the IP address allocation.</p>
<h3><a id="post-136638-_qcov6ae0nu2j"></a>Execution</h3>
<p>Unit 42 investigators observed attackers employing a batch script named <span style="font-family: 'courier new', courier, monospace;">1.bat</span> to execute the ransomware payload against multiple hosts within the client network. Details regarding the ransomware payload, along with its arguments, are below.</p>

<h3><a id="post-136638-_a9flpiseois1"></a>Collection</h3>
<p>Unit 42 investigators found the creation of the following file <span style="font-family: 'courier new', courier, monospace;">C:\ProgramData\found_shares.txt</span>. <h3><a id="post-136638-_sshao7v36ocg"></a>Exfiltration</h3>
<p>Unit 42 investigators identified <a href="https://rclone.org/" target="_blank" rel="noopener">Rclone</a> (a legitimate open-source utility) as the tool used for exfiltration. Attackers installed the tool in the <em>ProgramData</em> file path (<span style="font-family: 'courier new', courier, monospace;">C:\ProgramData\rclone.exe</span>), along with the configuration file (<span style="font-family: 'courier new', courier, monospace;">C:\ProgramData\rclone.conf</span>).</p>
<p>We observed <span style="font-family: 'courier new', courier, monospace;">91.238.181[.]238</span> was the public IP address attackers used for exfiltration activity. This IP address comes from a hosting provider called <span style="font-family: 'courier new', courier, monospace;">VDS&amp;VPN services</span>.</p>
<p>The IP address in question has previously been flagged for Cobalt Strike activity (watermark: 674054486) and was potentially <a href="https://twitter.com/TLP_R3D/status/1718188502406385955" target="_blank" rel="noopener">linked to other ransomware groups</a> such as Bashful Scorpius (aka Nokoyawa) and Ambitious Scorpius (aka ALPHV/BlackCat) in 2023. This IP address was also observed trying to exploit ScreenConnect vulnerabilities, (<a href="https://unit42.paloaltonetworks.com/connectwise-threat-brief-cve-2024-1708-cve-2024-1709/" target="_blank" rel="noopener">CVE-2024-1708 and CVE-2024-1709) in February 2024</a>.</p>

<p>Figure 2 shows the threat actors used a batch script to execute the ransomware multiple times against a list of hard-coded directory paths in the victim network. The encryptor requires a key parameter to begin execution, which has been redacted from the image.</p>
<p>The binary performs a key validation routine, in which it attempts to decrypt an embedded ransom note using the <a href="https://cr.yp.to/chacha/chacha-20080128.pdf" target="_blank" rel="noopener">ChaCha20 stream cipher [PDF]</a>.</p>
<p>The ransomware note is Base64-decoded. It is then decrypted using the first 32 bytes of the submitted key as the ChaCha20 secret key and the last 12 bytes of the submitted key as the nonce. Then it is Base64-decoded a final time.</p>
<p>The encryptor will validate the decryption process by checking whether the string <span style="font-family: 'courier new', courier, monospace;">***is_ok***</span> exists in the decrypted data. If the validation is successful, execution proceeds.</p>
<p>The encryptor contains a legitimate copy of PsExec embedded within itself, which it will extract and save to the location <span style="font-family: 'courier new', courier, monospace;">C:\Users\Public\psexec0.exe</span>. The malware will then create a copy of itself in the <span style="font-family: 'courier new', courier, monospace;">C:\Users\Public\</span> directory.</p>
<p>Once copied, it will use the <span style="font-family: 'courier new', courier, monospace;">PsExec</span> binary to execute itself several more times, using hard-coded credentials stolen from the victim network during the preceding incursion. By default, the ransomware will check for the existence of drives from <span style="font-family: 'courier new', courier, monospace;">A:\</span> to <span style="font-family: 'courier new', courier, monospace;">Z:\</span>. <p>The encryptor renames files with a new extension before starting the encryption process. In the sample analyzed by Unit 42 the extension was <span style="font-family: 'courier new', courier, monospace;">kcr5umw</span>.</p>
<p>The encryption process is composed of two sequences. First, the encryptor will read the contents of the target file, encrypt the contents using ChaCha20 with a randomly generated key secret and nonce bytes, and write the result back to the file. Second, the ChaCha20 key and nonce are encrypted using a hard-coded RSA public key, and the result is appended to the file. Finally, the extension is appended to the end of the encrypted data.</p>

<h2><a id="post-136638-_cmf45jb45f8w"></a>New Version of Encryptor</h2>
<p>Unit 42 researchers found an updated Cicada3301 encryptor in late July 2024, which had some differences from the previously analyzed version.</p>
<p>Threat authors added a new command-line argument, <span style="font-family: 'courier new', courier, monospace;">--no-note</span>. When this argument is invoked, the encryptor will not write the ransom note to the system.</p>
<p>Instead of running the embedded <span style="font-family: 'courier new', courier, monospace;">PsExec</span> binary directly via PowerShell, the encryptor will create a randomly named <span style="font-family: 'courier new', courier, monospace;">.bat</span> file in the <span style="font-family: 'courier new', courier, monospace;">C:\Users\Public</span> directory, which executes using <span style="font-family: 'courier new', courier, monospace;">“cmd.exe /C”</span>. Included in the created <span style="font-family: 'courier new', courier, monospace;">.bat</span> file is a line to delete the script after execution is complete.</p>
<p>The most recent samples do not have hard-coded usernames or passwords in the binary but still retain the capability to execute <span style="font-family: 'courier new', courier, monospace;">PsExec</span> using these credentials if they exist.</p>

<h2><a id="post-136638-_rt54on20d5fv"></a>Conclusion</h2>
<p>Although it may not currently appear to be widespread, Repellent Scorpius is actively hiring IAB and network intruders. It has also recently set up a RaaS affiliate program. Therefore, we can expect to see attackers posting a growing list of active incidents and victims on their leak site in the near future.</p>
<p>The TTPs highlighted here are from specific incident response engagements. Considering that the Cicada3301 ransomware is relatively new, we expect that its TTPs will change and evolve over time.</p>
