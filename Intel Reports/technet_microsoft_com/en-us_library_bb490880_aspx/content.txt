										</nav><!-- <content> --><p>Starts a new instance of the command interpreter, Cmd.exe. Used without parameters, <strong>cmd</strong> displays Windows&nbsp;XP version and copyright information.</p>
<h3 id="syntax">Syntax</h3>
<p><strong>cmd</strong> [[{<strong>/c</strong>|<strong>/k</strong>}] [<strong>/s</strong>] [<strong>/q</strong>] [<strong>/d</strong>] [{<strong>/a</strong>|<strong>/u</strong>}] [<strong>/t:<strong><em>fg</em>] [</strong>/e:</strong>{<strong>on</strong>|<strong>off</strong>}] [<strong>/f:</strong>{<strong>on</strong>|<strong>off</strong>}] [<strong>/v:</strong>{<strong>on</strong>|<strong>off</strong>}] <em>string</em>]</p>
<h3 id="parameters">Parameters</h3>
<p><strong>/c</strong> &nbsp; <strong>:</strong> Carries out the command specified by <em>string</em> and then stops.</p>
<p><strong>/k</strong> <strong>:</strong> Carries out the command specified by <em>string</em> and continues.</p>
<p><strong>/s</strong> &nbsp; <strong>:</strong> Modifies the treatment of <em>string</em> after <strong>/c</strong> or <strong>/k</strong>.</p>
<p><strong>/q</strong> <strong>:</strong> Turns the echo off.</p>
<p><strong>/d</strong> &nbsp; <strong>:</strong> Disables execution of AutoRun commands.</p>
<p><strong>/a</strong> <strong>:</strong> Creates American National Standards Institute (ANSI) output.</p>
<p><strong>/u</strong> <strong>:</strong> Creates Unicode output.</p>
<p><strong>/t:</strong> <em><strong>fg</strong></em> &nbsp; <strong>:</strong> Sets the foreground <em>f</em> and background <em>g</em> colors. The following tables lists valid hexadecimal digits that you can use as the values for <em>f</em> and <em>g</em>.</p>

<p><strong>/e:on</strong> &nbsp; <strong>:</strong> Enables command extensions.</p>
<p><strong>/e:off</strong> &nbsp; <strong>:</strong> Disables commands extensions.</p>
<p><strong>/f:on</strong> &nbsp; <strong>:</strong> Enables file and directory name completion.</p>
<p><strong>/f:off</strong> &nbsp; <strong>:</strong> Disables file and directory name completion.</p>
<p><strong>/v:on</strong> &nbsp; <strong>:</strong> Enables delayed environment variable expansion.</p>
<p><strong>/v:off</strong> &nbsp; <strong>:</strong> Disables delayed environment variable expansion.</p>
<p><em><strong>string</strong></em> &nbsp; <strong>:</strong> Specifies the command you want to carry out.</p>
<p><strong>/?</strong> &nbsp; <strong>:</strong> Displays help at the command prompt.</p>
<h3 id="remarks">Remarks</h3>
<ul>
<li><p>Using multiple commands</p>
<p>You can use multiple commands separated by the command separator <strong>&amp;&amp;</strong> for <em>string</em>, but you must enclose them in quotation marks (for example, <strong>"<em><strong>command</strong></em>&amp;&amp;<em><strong>command</strong></em>&amp;&amp;<em><strong>command</strong></em>"</strong>).</p>
</li>
<li><p>Processing quotation marks</p>
<p>If you specify <strong>/c</strong> or <strong>/k</strong>, <strong>cmd</strong> processes the remainder of <em>string</em> and quotation marks are preserved only if all of the following conditions are met:</p>
<ul>
<li><p>You do not use <strong>/s</strong>.</p>
</li>
<li><p>You use exactly one set of quotation marks.</p>
</li>
<li><p>You do not use any special characters within the quotation marks (for example: &amp;&lt;&gt;( ) @ ^ |).</p>
</li>
<li><p>You use one or more white-space characters within the quotation marks.</p>
</li>
<li><p>The <em>string</em> within quotation marks is the name of an executable file.</p>
</li>
</ul>
<p>If the previous conditions are not met, <em>string</em> is processed by examining the first character to verify whether or not it is an opening quotation mark. If the first character is an opening quotation mark, it is stripped along with the closing quotation mark. Any text following the closing quotation marks is preserved.</p>
</li>
<li><p>Executing registry subkeys</p>
<p>If you do not specify <strong>/d</strong> in <em>string</em>, Cmd.exe looks for the following registry subkeys:</p>
<p><strong>HKEY_LOCAL_MACHINE\Software\Microsoft\Command</strong> <strong>Processor\AutoRun\REG_SZ</strong>&nbsp;</p>
<p><strong>HKEY_CURRENT_USER\Software\Microsoft\Command</strong> <strong>Processor\AutoRun</strong>&nbsp;<strong>REG_EXPAND_SZ</strong></p>
<p>If either one or both registry subkeys are present, they are executed before all other variables.</p>
<p>**&nbsp;Caution**</p>
<ul>
<li>Incorrectly editing the registry may severely damage your system. Before making changes to the registry, you should back up any valued data on the computer.</li>
</ul>
</li>
<li><p>Enabling and disabling command extensions</p>
<p>Command extensions are enabled by default in Windows&nbsp;XP. You can disable them for a particular process by using <strong>/e:off</strong>. You can enable or disable extensions for all <strong>cmd</strong> command-line options on a computer or user session by setting the following <strong>REG_DWORD</strong> values:</p>
<p><strong>HKEY_LOCAL_MACHINE\Software\Microsoft\Command</strong> <strong>Processor\EnableExtensions\REG_DWORD</strong>&nbsp;</p>
<p><strong>HKEY_CURRENT_USER\Software\Microsoft\Command</strong> <strong>Processor\EnableExtensions\REG_DWORD</strong>&nbsp;</p>
<p>Set the <strong>REG_DWORD</strong> value to either <strong>0Ã1</strong> (that is, enabled) or <strong>0Ã0</strong> (that is, disabled) in the registry by using Regedit.exe. User-specified settings take precedence over computer settings, and command-line options take precedence over registry settings.</p>
<p>**&nbsp;Caution**</p>
<ul>
<li>Incorrectly editing the registry may severely damage your system. Before making changes to the registry, you should back up any valued data on the computer.</li>
</ul>
<p>When you enable command extensions, the following commands are affected:</p>
<ul>
<li><p><strong>assoc</strong>&nbsp;</p>
</li>
<li><p><strong>call</strong>&nbsp;</p>
</li>
<li><p><strong>chdir</strong> (<strong>cd</strong>)</p>
</li>
<li><p><strong>color</strong>&nbsp;</p>
</li>
<li><p><strong>del</strong> (<strong>erase</strong>)</p>
</li>
<li><p><strong>endlocal</strong>&nbsp;</p>
</li>
<li><p><strong>for</strong>&nbsp;</p>
</li>
<li><p><strong>ftype</strong>&nbsp;</p>
</li>
<li><p><strong>goto</strong>&nbsp;</p>
</li>
<li><p><strong>if</strong>&nbsp;</p>
</li>
<li><p><strong>mkdir</strong> (<strong>md</strong>)</p>
</li>
<li><p><strong>popd</strong>&nbsp;</p>
</li>
<li><p><strong>prompt</strong>&nbsp;</p>
</li>
<li><p><strong>pushd</strong>&nbsp;</p>
</li>
<li><p><strong>set</strong>&nbsp;</p>
</li>
<li><p><strong>setlocal</strong>&nbsp;</p>
</li>
<li><p><strong>shift</strong>&nbsp;</p>
</li>
<li><p><strong>start</strong> (also includes changes to external command processes)</p>
</li>
</ul>
<p>For more information about these commands, see Related Topics.</p>
</li>
<li><p>Enabling delayed environment variable expansion</p>
<p>If you enable delayed environment variable expansion, you can use the exclamation character to substitute the value of an environment variable at run time.</p>
</li>
<li><p>Enabling file and directory name completion</p>
<p>File and directory name completion is not enabled by default. You can enable or disable file name completion for a particular process of the <strong>cmd</strong> command with <strong>/f:</strong>{<strong>on</strong>|<strong>off</strong>}. You can enable or disable file and directory name completion for all processes of the <strong>cmd</strong> command on a computer or user logon session by setting the following <strong>REG_DWORD</strong> values:</p>
<p><strong>HKEY_LOCAL_MACHINE\Software\Microsoft\Command</strong> <strong>Processor\CompletionChar\REG_DWORD</strong>&nbsp;</p>
<p><strong>HKEY_LOCAL_MACHINE\Software\Microsoft\Command</strong> <strong>Processor\PathCompletionChar\REG_DWORD</strong>&nbsp;</p>
<p><strong>HKEY_CURRENT_USER\Software\Microsoft\Command</strong> <strong>Processor\CompletionChar\REG_DWORD</strong>&nbsp;</p>
<p><strong>HKEY_CURRENT_USER\Software\Microsoft\Command</strong> <strong>Processor\PathCompletionChar\REG_DWORD</strong>&nbsp;</p>
<p>To set the <strong>REG_DWORD</strong> value, run Regedit.exe and use the hexadecimal value of a control character for a particular function (for example, <strong>0Ã9</strong> is TAB and <strong>0Ã08</strong> is BACKSPACE). User-specified settings take precedence over computer settings, and command-line options take precedence over registry settings.</p>
<p>**&nbsp;Caution**</p>
<ul>
<li>Incorrectly editing the registry may severely damage your system. Before making changes to the registry, you should back up any valued data on the computer.</li>
</ul>
<p>If you enable file and directory name completion by using <strong>/f:on</strong>, use CTRL+D for directory name completion and CTRL+F for file name completion. To disable a particular completion character in the registry, use the value for white space [<strong>0Ã20</strong>] because it is not a valid control character.</p>
<p>When you press CTRL+D or CTRL+F, <strong>cmd</strong> processes file and directory name completion. These key combination functions append a wildcard character to <em>string</em> (that is, if one is not present), build a list of paths that match, and then display the first matching path. If none of the paths match, the file and directory name completion function beeps and does not change the display. To move through the list of matching paths, press CTRL+D or CTRL+F repeatedly. To move through the list backwards, press the SHIFT key and CTRL+D or CTRL+F simultaneously. To discard the saved list of matching paths and generate a new list, edit <em>string</em> and press CTRL+D or CTRL+F. If you switch between CTRL+D and CTRL+F, the saved list of matching paths is discarded and a new list is generated. The only difference between the key combinations CTRL+D and CTRL+F is that CTRL+D only matches directory names and CTRL+F matches both file and directory names. If you use file and directory name completion on any of the built-in directory commands (that is, <strong>CD</strong>, <strong>MD</strong>, or <strong>RD</strong>), directory completion is assumed.</p>
<p>File and directory name completion correctly processes file names that contain white space or special characters if you place quotation marks around the matching path.</p>
<p>The following special characters require quotation marks: &amp; &lt; &gt; [ ] { } ^ = ; ! ' + , ` ~ [white space]</p>
<p>If the information that you supply contains spaces, use quotation marks around the text (for example, <strong>"<em><strong>Computer&nbsp;Name</strong></em>"</strong>).</p>
<p>If you process file and directory name completion from within <em>string</em>, any part of the [<em>Path</em>] to the right of the cursor is discarded (that is, at the point in <em>string</em> where the completion was processed).</p>
</li>
</ul>
