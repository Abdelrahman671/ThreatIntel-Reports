	<h1 class="article-details__title">Kasseika Ransomware Deploys BYOVD Attacks, Abuses PsExec and Exploits Martini Driver  </h1>
	<p class="article-details__description">In this blog, we detail our investigation of the Kasseika ransomware and the indicators we found suggesting that the actors behind it have acquired access to the source code of the notorious BlackMatter ransomware.  </p>
	<p class="article-details__author-by">By: Emmanuel Panopio, Christian Jason Geollegue, Julius Keith Estrellado, Christian Alpuerto, Shawn Austin Santos, Emmanuel Roll, Rhio Manaog, Gerald Fernandez, Don Ovid Ladores, Raighen Sanchez, Raymart Yambot, Francesca Villasanta, Sophia Nilette Robles

            <div class="richText">
	
    


	
		<div>
			<p>Following an increase in bring-your-own-vulnerable-driver (BYOVD) attacks launched by ransomware groups in 2023, the Kasseika ransomware is among the latest groups to take part in the trend. Kasseika joins <a href="https://www.trendmicro.com/vinfo/tmr/?/ph/security/news/ransomware-spotlight/ransomware-spotlight-akira">Akira</a>, <a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/news/ransomware-spotlight/ransomware-spotlight-blackbyte">BlackByte</a>, and <a href="/en_us/research/22/e/avoslocker-ransomware-variant-abuses-driver-file-to-disable-anti-Virus-scans-log4shell.html">AvosLocker</a> in using the tactic that allows threat actors to terminate antivirus processes and services for the deployment of <a href="/en_us/what-is/ransomware.html">ransomware</a>. In this case we investigated, the <a href="https://twitter.com/TrendMicroRSRCH/status/1735509992571040141">Kasseika</a> ransomware abused Martini driver to terminate the victim machine’s antivirus-related processes.  </p>
<p>In our analysis of the Kasseika ransomware attack chain, we observed indicators that resemble the <a href="https://www.cisa.gov/news-events/cybersecurity-advisories/aa21-291a">BlackMatter</a> ransomware. These indicators include pseudo-ransom extensions and the use of extension string.README.txt as the ransom note file name and format.  </p>
<p>A closer look revealed that majority of the source code used by BlackMatter was used in this attack. Based on our research, the BlackMatter source code is not widely available, so its use in this Kasseika ransomware attack is suggestive of a mature actor in a limited group that acquired or bought access to it. </p>
<p>BlackMatter respawned from DarkSide, which is known to have been used as the basis for ALPHV, more popularly known as <a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/news/ransomware-spotlight/ransomware-spotlight-blackcat">BlackCat</a>. Since its <a href="https://www.zdnet.com/article/blackmatter-ransomware-to-shut-down-affiliates-transferring-victims-to-lockbit/">shutdown</a> in 2021, other ransomware groups have been observed using similar techniques and tools to BlackMatter, while a more exclusive group of ransomware operators are able to access its old code and apply it to <a href="/en_us/research/22/g/lockbit-ransomware-group-augments-its-latest-variant--lockbit-3-.html">new strains</a>.   </p>

		</div>
	

</div>
            <div class="image">
	
    


	<figure class="image-figure">

    <div class="richText">
	
    


	
		<div>
			<p><span class="body-subhead-title">Targeted phishing links via email for initial access </span></p>
<p>In the Kasseika ransomware case that we investigated, we observed that it used targeted phishing techniques for initial access, as well as to gather credentials from one of the employees of its target company. It then uses remote administration tools (RATs) to gain privileged access and move laterally within its target network. </p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		<a class="bs-modal" id="34f52d" data-modal-title="Figure 2. PsExec Command to execute malicious .bat file " href="/content/dam/trendmicro/global/en/research/24/a/kasseika-ransomware/kasseika-fig2-psexec.jpg">
			<img src="/content/dam/trendmicro/global/en/research/24/a/kasseika-ransomware/kasseika-fig2-psexec.jpg" alt="Figure 2. PsExec Command to execute malicious .bat file "/>
		</a>
		<div class="caption-image-container ">
			<figcaption>Figure 2. PsExec Command to execute malicious .bat file  (click to enlarge)</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p><span class="body-subhead-title">Abusing PsExec for execution </span></p>
<p>Kasseika abused the legitimate Windows RAT PsExec to execute its malicious files. PsExec was originally designed for network management, but its misuse allows threat actors to remotely deploy a malicious .bat file, as in this case.  </p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/24/a/kasseika-ransomware/kasseika-fig3-terminates.jpg" alt="Figure 3. Kasseika terminates Martini runtime "/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 3. Kasseika terminates Martini runtime </figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p>The Kasseika ransomware initially uses a batch script to load its malicious entities. The script begins by checking for the existence of the process named <i>Martini.exe</i>. If found, it then proceeds to terminate it to ensure that there is only one instance of the process running on the machine.  </p>
<p><span class="body-subhead-title">Kasseika’s KILLAV mechanism for defense evasion </span></p>
<p>Upon further analysis,<i> Martini.exe</i> first verifies whether the <i>Martini.sys</i> driver was successfully downloaded to the affected system. The signed driver<i> Martini.sys</i>, originally labeled as <i>viragt64.sys</i>, is part of VirIT Agent System developed by TG Soft. By exploiting its vulnerabilities, Kasseika leverages this driver to effectively disable various security tools. If <i>Martini.sys</i> does not exist, the malware will <a href="https://attack.mitre.org/techniques/T1010/">terminate itself</a> and not proceed with its intended routine. </p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/24/a/kasseika-ransomware/kasseika-fig4-martinisys.jpg" alt="Figure 4. “Martini.sys” file properties and certificate information "/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 4. “Martini.sys” file properties and certificate information </figcaption>

					<span class="material-symbols-outlined">download</span>
				</a>
			</div>
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<p>After confirming the presence of the system file, Kasseika proceeds to create a service and then initiates it. </p>

		</div>
	

</div>

			<p><span class="body-subhead-title">Kasseika payload analysis </span></p>
<p>The Kasseika ransomware is a 32-bit Windows PE file packed by Themida. Themida-packed binaries are known to have formidable code obfuscation and anti-debugging techniques, making it hard to reverse-engineer them. </p>

		</div>

			<p>Before encryption, Kasseika terminates all processes and services that are currently accessing Windows Restart Manager. Kasseika first starts a new session, modifying the <i>Owner </i>value from the registry keys in the following list. It then starts enumerating session hashes (<i>SessionHash</i>) of processes and services from the registry keys in the same list. After termination, it retrieves the paths of the terminated files that will be checked later for encryption:  </p>
<ul>
<li><span class="rte-legal-text"><b><i> HKEY_CURRENT_USER\Software\Microsoft\RestartManager\Session{numbers}</i></b></span></li>
<li><span class="rte-legal-text"><b><i>Owner = {hex values} </i></b></span></li>
<li><span class="rte-legal-text"><b> </b></span></li>
<li><span class="rte-legal-text"><b><i>HKEY_CURRENT_USER\Software\Microsoft\RestartManager\Session{numbers}</i></b></span></li>
<li><span class="rte-legal-text"><b><i>SessionHash = {hex values} </i></b></span></li>
<li><span class="rte-legal-text"><b> </b></span></li>
<li><span class="rte-legal-text"><b><i>HKEY_CURRENT_USER\Software\Microsoft\RestartManager\Session{numbers}</i></b></span></li>
<li><span class="rte-legal-text"><b><i>Sequence = 0x01 </i></b></span></li>
<li><span class="rte-legal-text"><b> </b></span></li>
<li><span class="rte-legal-text"><b><i>HKEY_CURRENT_USER\Software\Microsoft\RestartManager\Session{numbers}</i></b></span></li>
<li><span class="rte-legal-text"><b><i>RegFiles{numbers} = {encrypted path and file} </i></b></span></li>
<li><span class="rte-legal-text"><b> </b></span></li>
<li><span class="rte-legal-text"><b><i>HKEY_CURRENT_USER\Software\Microsoft\RestartManager\Session{numbers}</i></b></span></li>
<li><span class="rte-legal-text"><b><i>RegFilesHash = {hex values} </i></b></span></li>
</ul>

		</div>

    <div class="richText">
	
    


	
		<div>
			<p>The Kasseika ransomware deletes the shadow copies of the affected system by using Windows Management Instrumentation command-line (WMIC) queries to enumerate them. </p>
<ul>
<li><span class="blockquote">SELECT * Win32_ShadowCopies </span></li>
</ul>
<p>The Kasseika ransomware then decrypts its encrypted extension by first retrieving a hard-coded string from <i>CryptoPP::StringSinkTemplate</i>. Next, it uses Base64 to encode the first nine characters of the string. Finally, since the characters “+”, “/”, and “=” in Base64 are not compatible in a file extension, the ransomware replaces them with “a”, “l”, and “e”, respectively.  </p>

		</div>
	

</div>

			<p>Kasseika retrieves its encryption algorithm key, ChaCha20, together with the RSA encryption algorithm from open-source C++ library CryptoPP. Kasseika then generates a modified version of the ChaCha20 matrix that consists of randomly generated bytes. The matrix is copied to a buffer that will be encrypted by the RSA public key, after which the encrypted buffer is written into the modified version of the ChaCha20 matrix. The Kasseika ransomware then uses the modified ChaCha20 matrix to encrypt target files.    </p>

		</div>

<p><span class="body-subhead-title">Security Recommendations </span></p>
<p>The following is a list of measures that organizations can employ as best practices to minimize the chances of falling victim to ransomware attacks such as those launched by the Kasseika ransomware: </p>
<ul>
<li><span class="rte-red-bullet">Only grant employees administrative rights and access when necessary. </span></li>
<li> </li>
<li><span class="rte-red-bullet">Ensure that security products are updated regularly and perform period scans.  </span></li>
<li> </li>
<li><span class="rte-red-bullet">Secure regular backups of critical data in case of any loss. </span></li>
<li> </li>
<li><span class="rte-red-bullet">Exercise good email and website safety practices — download attachments, select URLs, and execute programs only from trusted sources. </span></li>
<li> </li>
<li><span class="rte-red-bullet">Encourage users to alert the security team of potentially suspicious emails and files and use tools to block malicious emails. </span></li>
<li> </li>
<li><span class="rte-red-bullet">Conduct regular user education around the dangers and signals of social engineering. </span></li>
</ul>
<p> A multilayered approach can help organizations guard possible entry points into their system (endpoint, email, web, and network). Security solutions can detect malicious components and suspicious behavior, which can help protect enterprises.   </p>
