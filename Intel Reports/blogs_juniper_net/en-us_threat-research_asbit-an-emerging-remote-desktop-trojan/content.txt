<h1>Introduction</h1>
<p>&nbsp;</p>
<p>Juniper Threat Labs is currently monitoring an emerging Chinese Remote Desktop Trojan called Asbit. It’s a remote access Trojan being advertised on its developer’s website as a “Fast Remote Desktop”. This RAT first made its appearance in 2021 and kept updating its infrastructure and features as it went along. It uses a number of strategies to avoid endpoint and network detection. By using DNS over HTTPS (DoH) to resolve the IP addresses of its control servers, it aims to get past network DNS filters.</p>
<p>The domain <code>asbit[.]cn</code> was registered on January 31, 2021. This domain was used to host the malware and its modules. It also hosts the threat actor’s website. The website is still live at the time of this publication. Recently, the threat actor registered another domain and website,<code> rdlite[.]com</code>. The new website looks like a copy of the previous website.</p>
<p>&nbsp;</p>

<h1>Timeline</h1>
<p>To provide an understanding of the Asbit operation, below is a timeline of their infrastructure. The domain, <code>asbit[.]cn</code> was registered on January 31, 2021. At that time, it resolved to IP <code>47.111.81.199</code>.  Later in the year 2021, the threat actors registered <code>mitm[.]work</code> and <code>fmt[.]ink</code>. At this time, the servers are hosted on Linux servers. In 2022, they registered the domain <code>def[.]cab, rfb[.]ink and rdlite[.]com</code>. Around June of 2022, we also see a shift of the OS they used in hosting these servers. Based on third-party server fingerprinting services like Shodan and Cencys, we found that they shifted to Windows OS and used IIS. It’s unclear to us why they shifted to Windows.</p>

<h1>Early Version of asbit RAT</h1>
<p>The earliest version of the malware we could find dates back to May 2021. It included a series of loader modules which will eventually download the core module. The malware is typically bundled with installers. It starts by dropping the loader module that installs another DLL as a service. The installed service further downloads and executes the core module. The core module has remote desktop functionality and other capabilities, including command execution and keystroke injection.</p>
<p>&nbsp;</p>
<p>The malicious installer (<code>806523a9e4021cd72691fbdeb229339a24984b7ca79c3860ba468c832bf95c4d</code>) drops the loader modules <code>C:\2.0.50727</code> and <code>C:\4.0.30319</code>. The files appear to be folders of the .NET framework, but they are in fact files. This is likely done by the author to obscure itself and try to evade endpoint detection. It also drops a malicious <code>libexpat.dll</code> (<code>018656e61c7c5fc8d5983a32507fbcf379dd9e2b5dd7d67b8e44590d46a8c51f</code>) and a benign <code>XLSP.dll</code> (<code>cca2ea493c8402ed24c9c93328066ae0ce451059e2f954929f24ab7481119b6c</code>).</p>
<figure id="attachment_27976" class="thumbnail wp-caption aligncenter" style="width: 466px"><img loading="lazy" class="size-full wp-image-27976" src="https://blogs.juniper.net/wp-content/uploads/2022/08/procmon_drop_files.png" alt="" width="466" height="136" srcset="https://blogs.juniper.net/wp-content/uploads/2022/08/procmon_drop_files.png 466w, https://blogs.juniper.net/wp-content/uploads/2022/08/procmon_drop_files-300x88.png 300w" sizes="(max-width: 466px) 100vw, 466px" /><figcaption class="caption wp-caption-text">The installer drops the components in the C:\ directory.</figcaption></figure>
<h2>Loader Module</h2>
<p>The loader module starts with the dropped C:\2.0.50727 or C:\4.0.30319. It executes either of these files using <a href="https://docs.microsoft.com/en-us/dotnet/framework/tools/regasm-exe-assembly-registration-tool">RegAsm.exe</a>, an assembly registration tool that is used to load .NET assembly files.</p>
<table width="624">
<tbody>
<tr>
<td width="624"><code>C:\Windows\Microsoft.NET\Framework\v4.0.30319\RegAsm.exe /u "C:\4.0.30319"</code></td>
</tr>
</tbody>
</table>
<p>The <code>"/u"</code> parameter, which means <code>"Unregister</code>&#8220;, is part of the command to install this module. This could be interpreted as being used to remove the module. However, the module itself has an <code>“Unregister”</code> function that does the opposite. It creates a service for <code>libexpat.dll</code> that further downloads additional modules, including the core module.</p>
<figure id="attachment_27970" class="thumbnail wp-caption aligncenter" style="width: 637px"><img loading="lazy" class=" wp-image-27970" src="https://blogs.juniper.net/wp-content/uploads/2022/08/asbit_unregister-1024x401.png" alt="" width="637" height="250" srcset="https://blogs.juniper.net/wp-content/uploads/2022/08/asbit_unregister-1024x401.png 1024w, https://blogs.juniper.net/wp-content/uploads/2022/08/asbit_unregister-300x117.png 300w, https://blogs.juniper.net/wp-content/uploads/2022/08/asbit_unregister-768x300.png 768w, https://blogs.juniper.net/wp-content/uploads/2022/08/asbit_unregister.png 1122w" sizes="(max-width: 637px) 100vw, 637px" /><figcaption class="caption wp-caption-text">Unregister function in the loader module that installs 2 DLLs as a service.</figcaption></figure>
<p>Once the <code>libexpat.dll</code> service is installed and running, it downloads 2 additional modules. One is a module that creates a web proxy (<code>f223ceb9830ed18d823f42d9d8d5c09429782e0bcdbc62da5f00c881964e7041</code>) and the other is the core module (<code>4b1efadc55c0cc3471e0945804125317b2a75772393541f680b1617f8a42773d</code>).</p>
<h3></h3>
<h2>Second Loader Module</h2>
<p>This module installs a browser proxy by downloading a <a href="https://developer.mozilla.org/en-US/docs/Web/HTTP/Proxy_servers_and_tunneling/Proxy_Auto-Configuration_PAC_file">Proxy Auto-Configuration</a> (PAC) file from <code>47.111.81.199</code>. This allows the threat actor to act as a Browser-in-the-Middle by making all requests from the browser proxied to the threat actor’s server. Installation of the PAC file also includes installation of the threat actor’s certificate.</p>

<p>The new core module includes functions such as capture screenshots, inject keystrokes and execute commands, but we noticed a much cleaner programming style hinting active development of the malware. We also noticed it added several functions, particularly handlers, for example, vnc, rfb and p2p functions.</p>
<p><img loading="lazy" class="alignnone size-full wp-image-27985" src="https://blogs.juniper.net/wp-content/uploads/2022/08/asbit_handlers.png" alt="" width="724" height="261" srcset="https://blogs.juniper.net/wp-content/uploads/2022/08/asbit_handlers.png 724w, https://blogs.juniper.net/wp-content/uploads/2022/08/asbit_handlers-300x108.png 300w" sizes="(max-width: 724px) 100vw, 724px" /></p>
<p>&nbsp;</p>
<p>Another interesting function is the use of DNS over HTTP (DoH) to query the ip address of its control servers. For instance, the domain name of the file server where it hosts the rdp module could be any of the following domains:</p>
<ul>
<li>a.rdlite.com:1024</li>
<li>b.rdlite.com:1024</li>
<li>w.rdlite.com:1024</li>
</ul>
<p>However, it does not use the traditional way of getting the IP address, which is connecting to the DNS at port 53 using UDP. Rather, it gets the IP address using DNS over HTTP (DoH).</p>
<p>Below is a query for the IP address of <code>ddns.b.rdlite.com</code> using Alibaba DoH server.</p>
<table width="624">
<tbody>
<tr>
<td width="624"><code><code>GET /resolve?name=ddns.b.rdlite.com&amp;short=true&amp;_=28042000 HTTP/1.1<br />
Host: 223.5.5.5<br />
Connection: Keep-Alive</code>["222.210.157.249"]</code></td>
</tr>
</tbody>
</table>
<p>Currently, it can either use AliDns or DnsPod DoH servers.</p>
<p><img loading="lazy" class="size-large wp-image-27971 aligncenter" src="https://blogs.juniper.net/wp-content/uploads/2022/08/doh_alidns-1024x319.png" alt="" width="1024" height="319" srcset="https://blogs.juniper.net/wp-content/uploads/2022/08/doh_alidns-1024x319.png 1024w, https://blogs.juniper.net/wp-content/uploads/2022/08/doh_alidns-300x94.png 300w, https://blogs.juniper.net/wp-content/uploads/2022/08/doh_alidns-768x239.png 768w, https://blogs.juniper.net/wp-content/uploads/2022/08/doh_alidns.png 1055w" sizes="(max-width: 1024px) 100vw, 1024px" /></p>
<p><img loading="lazy" class="size-full wp-image-27972 aligncenter" src="https://blogs.juniper.net/wp-content/uploads/2022/08/doh_dnspod.png" alt="" width="913" height="352" srcset="https://blogs.juniper.net/wp-content/uploads/2022/08/doh_dnspod.png 913w, https://blogs.juniper.net/wp-content/uploads/2022/08/doh_dnspod-300x116.png 300w, https://blogs.juniper.net/wp-content/uploads/2022/08/doh_dnspod-768x296.png 768w" sizes="(max-width: 913px) 100vw, 913px" /></p>
<p>Using DoH makes the malware more resilient against network filters that rely on traditional DNS queries for blocking domains. For example, a network filter that blocks the domain ddns.b.rdlite.com won’t be able to block the request because it can only see the HTTP request to DoH servers (e.g., Alibaba or DnsPod). The way to block this request is to directly block the IP address associated with the domain.</p>
<p>It’s interesting to note however that the malware author did not use HTTPS which will make it more resilient as such requests are encrypted.</p>
<p>&nbsp;</p>
<h2>RDP Client Module</h2>
<p>&nbsp;</p>
<p>This module is mainly for remote desktop control as the name implies. The old versions relied on commercial or free remote desktop applications, such as TightVNC or UltraVNC while the latest version implements its own RDP. It does this by using the built-in Windows RDP application. It works by creating an RDP Session instance by using the CLSID (<code>9B78F0E6-3E05-4A5B-B2E8-E743A8956B65</code>) which is the <a href="https://docs.microsoft.com/en-us/previous-versions/windows/desktop/rdp/rdpsession">RDPSession Class</a> in Windows.</p>
<p><img loading="lazy" class="size-large wp-image-27977 aligncenter" src="https://blogs.juniper.net/wp-content/uploads/2022/08/RDP_CLSID-1024x226.png" alt="" width="1024" height="226" srcset="https://blogs.juniper.net/wp-content/uploads/2022/08/RDP_CLSID-1024x226.png 1024w, https://blogs.juniper.net/wp-content/uploads/2022/08/RDP_CLSID-300x66.png 300w, https://blogs.juniper.net/wp-content/uploads/2022/08/RDP_CLSID-768x169.png 768w, https://blogs.juniper.net/wp-content/uploads/2022/08/RDP_CLSID.png 1106w" sizes="(max-width: 1024px) 100vw, 1024px" /></p>
<p>The RDP modules control server is <code>v2.team.rfb.ink:5100</code>, and it also uses DoH to determine the IP of its control server.</p>
<p><img loading="lazy" class="size-full wp-image-27979 aligncenter" src="https://blogs.juniper.net/wp-content/uploads/2022/08/team_service_doh.png" alt="" width="615" height="226" srcset="https://blogs.juniper.net/wp-content/uploads/2022/08/team_service_doh.png 615w, https://blogs.juniper.net/wp-content/uploads/2022/08/team_service_doh-300x110.png 300w" sizes="(max-width: 615px) 100vw, 615px" /></p>
<h1>Conclusion</h1>
<p>In this blog, we have detailed the evolution of an emerging Chinese remote desktop trojan. We have identified some of its capabilities and its infrastructure and how it tries to evade endpoint and network detection. We believe that this RAT is in its early development but it’s actively being developed. It’s likely that we will see future versions of this malware with enhanced capabilities and new ways of evading endpoint and network detection.</p>
<p><a href="https://www.juniper.net/us/en/products/security/advanced-threat-prevention.html">Juniper ATP Cloud</a> detects this malware using Machine Learning based on behavioral analysis engine.</p>
<p><img loading="lazy" class=" wp-image-27978 aligncenter" src="https://blogs.juniper.net/wp-content/uploads/2022/08/SkyATP-1024x618.png" alt="" width="779" height="470" srcset="https://blogs.juniper.net/wp-content/uploads/2022/08/SkyATP-1024x618.png 1024w, https://blogs.juniper.net/wp-content/uploads/2022/08/SkyATP-300x181.png 300w, https://blogs.juniper.net/wp-content/uploads/2022/08/SkyATP-768x463.png 768w, https://blogs.juniper.net/wp-content/uploads/2022/08/SkyATP.png 1363w" sizes="(max-width: 779px) 100vw, 779px" /></p>
