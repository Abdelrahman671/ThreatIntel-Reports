<P class="lia-align-justify lia-indent-padding-left-30px"><EM>We will provide additional details on timeline in the upcoming weeks.</EM></P>
<P class="lia-align-justify">&nbsp;</P>
<P class="lia-align-justify">It’s a challenging time in software security; migration to the modern cloud, the largest number of remote workers ever, and a global pandemic impacting staffing and supply chains all contribute to changes in organizations. Unfortunately, these changes also give bad actors opportunities to exploit organizations:</P>
<P class="lia-align-left">&nbsp;</P>
<TABLE style="width: 88%; border: 1px solid black; margin-left: auto; margin-right: auto;">
<TBODY>
<TR>
<TD width="100%" height="112px">
<P class="lia-align-left" data-unlink="true"><FONT color="#000000"><EM>“Cybercriminals are targeting and attacking all sectors of critical infrastructure, including healthcare and public health, information technology (IT), financial services, and energy sectors. Ransomware attacks are increasingly successful, crippling governments and businesses, and the profits from these attacks are soaring.” </EM></FONT></P>
<P class="lia-align-left" data-unlink="true"><FONT color="#800080">- </FONT><A href="https://query.prod.cms.rt.microsoft.com/cms/api/am/binary/RWMFIi#page=1" target="_self" rel="noopener noreferrer">Microsoft Digital Defense Report</A>, Oct 2021</P>
</TD>
</TR>
</TBODY>
</TABLE>
<P class="lia-align-left">&nbsp;</P>
<P class="lia-align-justify">For years Microsoft Office has shipped powerful automation capabilities called active content, the most common kind are macros. While we provided a notification bar to warn users about these macros, users could still decide to enable the macros by clicking a button. Bad actors send macros in Office files to end users who unknowingly enable them, malicious payloads are delivered, and the impact can be severe including malware, compromised identity, data loss, and remote access. See more in <A href="https://www.microsoft.com/security/blog/2016/03/22/new-feature-in-office-2016-can-block-macros-and-help-prevent-infection/" target="_blank" rel="noopener noreferrer">this blog post</A>.</P>
<P class="lia-align-left">&nbsp;</P>
<TABLE style="width: 88%; border: 1px solid black; margin-left: auto; margin-right: auto;">
<TBODY>
<TR>
<TD width="100%">
<P class="lia-align-left"><FONT color="#000000"><EM>"A wide range of threat actors continue to target our customers by sending documents and luring them into enabling malicious macro code.&nbsp; Usually, the malicious code is part of a document that originates from the internet (email attachment, link, internet download, etc.).&nbsp; Once enabled, the malicious code gains access to the identity, documents, and network of the person who enabled it."</EM></FONT></P>
<P class="lia-align-left"><FONT color="#000000">- Tom Gallagher, Partner Group Engineering Manager, Office Security</FONT></P>
</TD>
</TR>
</TBODY>
</TABLE>
<P class="lia-align-left">&nbsp;</P>
<P class="lia-align-justify">For the protection of our customers, we need to make it more difficult to enable macros in files obtained from the internet.</P>
<P class="lia-align-left">&nbsp;</P>
<H2 class="lia-align-left" id="toc-hId-336413958"><STRONG>Changing Default Behavior</STRONG></H2>
<P class="lia-align-left"><BR />We’re introducing a default change for five Office apps that run macros:</P>
<P class="lia-align-left">&nbsp;</P>
<P class="lia-align-center"><STRONG>VBA macros obtained from the internet will now be blocked by default.</STRONG></P>
<P class="lia-align-left">&nbsp;</P>
<P class="lia-align-justify">For macros in files obtained from the internet, users will no longer be able to enable content with a click of a button. A message bar will appear for users notifying them with a button to learn more. The default is more secure and is expected to keep more users safe including home users and information workers in managed organizations.</P>
<P class="lia-align-left">&nbsp;</P>
<TABLE style="width: 88%; border: 1px solid black; margin-left: auto; margin-right: auto;">
<TBODY>
<TR>
<TD width="100%">
<P class="lia-align-left"><FONT color="#000000"><EM>"We will continue to adjust our user experience for macros, as we’ve done here, to make it more difficult to trick users into running malicious code via social engineering while maintaining a path for legitimate macros to be enabled where appropriate via Trusted Publishers and/or Trusted Locations.”</EM> </FONT></P>
<P class="lia-align-left"><FONT color="#000000">- Tristan Davis, Partner Group Program Manager, Office Platform</FONT></P>
</TD>
</TR>
</TBODY>
</TABLE>
<P class="lia-align-left">&nbsp;</P>
<P class="lia-align-justify">This change only affects Office on devices running Windows and only affects the following applications: Access, Excel, PowerPoint, Visio, and Word. The change will begin rolling out in Version 2203, starting with Current Channel (Preview) in early April 2022. Later, the change will be available in the other update channels, such as Current Channel, Monthly Enterprise Channel, and Semi-Annual Enterprise Channel.</P>
<P class="lia-align-left">&nbsp;</P>
<P class="lia-align-left">At a future date to be determined, we also plan to make this change to Office LTSC, Office 2021, Office 2019, Office 2016, and Office 2013.</P>
<P class="lia-align-left">&nbsp;</P>
<H2 class="lia-align-left" id="toc-hId--1471040505"><STRONG>End User Experience</STRONG></H2>
<P class="lia-align-justify"><BR />Once a user opens an attachment or downloads from the internet an untrusted Office file containing macros, a message bar displays a Security Risk that the file contains Visual Basic for Applications (VBA) macros obtained from the internet with a <STRONG>Learn More</STRONG> button.</P>
<P class="lia-align-left">&nbsp;</P>
<P class="lia-align-left"><span class="lia-inline-image-display-wrapper lia-image-align-inline" image-alt="A message bar displays a Security Risk showing blocked VBA macros from the internet" style="width: 774px;"><img src="https://techcommunity.microsoft.com/t5/image/serverpage/image-id/342371i5535164A2FB05196/image-size/large?v=v2&amp;px=999" role="button" title="VBAMacrosBlockedTrustBar.png" alt="A message bar displays a Security Risk showing blocked VBA macros from the internet" li-image-url="https://techcommunity.microsoft.com/t5/image/serverpage/image-id/342371i5535164A2FB05196?v=v2" li-image-display-id="'342371i5535164A2FB05196'" li-message-uid="'3071805'" li-messages-message-image="true" li-bindable="" class="lia-media-image" tabindex="0" li-bypass-lightbox-when-linked="true" li-use-hover-links="false" /><span class="lia-inline-image-caption" onclick="event.preventDefault();">A message bar displays a Security Risk showing blocked VBA macros from the internet</span></span></P>
<P class="lia-align-left">&nbsp;</P>
<P class="lia-align-justify">The<STRONG> Learn More</STRONG> button goes to <A href="https://go.microsoft.com/fwlink/?linkid=2185272" target="_blank" rel="noopener noreferrer">an article for end users and information workers</A> that contains information about the security risk of bad actors using macros, safe practices to prevent phishing &amp; malware, and instructions on how to enable these macros by saving the file and removing the Mark of the Web (MOTW).</P>
<P class="lia-align-left">&nbsp;</P>
<H2 class="lia-align-left" id="toc-hId-1016472328"><STRONG>What is Mark of the Web (MOTW)?</STRONG></H2>
<P class="lia-align-justify"><BR />The MOTW is an attribute added to files by Windows when it is sourced from an untrusted location (Internet or Restricted Zone). The files must be saved to a NTFS file system, the MOTW is not added to files on FAT32 formatted devices.</P>
<P class="lia-align-left">&nbsp;</P>
<H3 class="lia-align-left" id="toc-hId-1707033802"><STRONG>IT A</STRONG><STRONG>dministrator Options</STRONG></H3>
<P class="lia-align-left"><BR />This chart shows the evaluation flow for Office files with VBA macros and MOTW:<BR /><BR /></P>
<P class="lia-align-left"><span class="lia-inline-image-display-wrapper lia-image-align-inline" image-alt="Evaluation flow for Office files with VBA macros and MOTW" style="width: 999px;"><img src="https://techcommunity.microsoft.com/t5/image/serverpage/image-id/342776i5D9AB520C0DDF0F1/image-size/large?v=v2&amp;px=999" role="button" title="VBAMacrosTrustWorkflow-GreyContainer.png" alt="Evaluation flow for Office files with VBA macros and MOTW" li-image-url="https://techcommunity.microsoft.com/t5/image/serverpage/image-id/342776i5D9AB520C0DDF0F1?v=v2" li-image-display-id="'342776i5D9AB520C0DDF0F1'" li-message-uid="'3071805'" li-messages-message-image="true" li-bindable="" class="lia-media-image" tabindex="0" li-bypass-lightbox-when-linked="true" li-use-hover-links="false" /><span class="lia-inline-image-caption" onclick="event.preventDefault();">Evaluation flow for Office files with VBA macros and MOTW</span></span></P>
<P class="lia-align-justify"><BR />Organizations can use the “Block macros from running in Office files from the Internet” policy to prevent users from inadvertently opening files from the internet that contain macros. <STRONG>Microsoft recommends enabling this policy, and if you do enable it, your organization won’t be affected by this default change.</STRONG></P>
<P class="lia-align-left">&nbsp;</P>
<TABLE style="width: 88%; border: 1px solid black; margin-left: auto; margin-right: auto;">
<TBODY>
<TR>
<TD width="100%">
<P><FONT color="#000000"><EM>“Setting policy is a powerful tool for IT Admins to protect their organizations. For years we’ve recommended blocking macros obtained from the internet in our <A href="https://techcommunity.microsoft.com/t5/microsoft-security-baselines/bg-p/Microsoft-Security-Baselines" target="_blank">security baselines</A>, and many customers have done so. I’m pleased Microsoft is taking the next step to securing everyone with this policy by default!” </EM></FONT></P>
<P><FONT color="#000000"><EM>-</EM> Hani Saliba, Partner Director of Engineering, Office Calc</FONT></P>
</TD>
</TR>
</TBODY>
</TABLE>
<P class="lia-align-left">&nbsp;</P>
<P class="lia-align-left">Additionally, there are two other options to know your files are safe:</P>
<UL>
<LI class="lia-align-left">Opening files from a Trusted Location</LI>
<LI class="lia-align-left">Opening files with digitally signed macros and providing the certificate to the user, who then installs it as a Trusted Publisher on their local machine</LI>
</UL>
<P class="lia-align-justify"><BR />To learn more about how to get ready for this change and recommendations for managing VBA macros in Office files, <A href="https://go.microsoft.com/fwlink/p/?linkid=2185771" target="_self" rel="noopener noreferrer">read this article for Office admins</A>.</P>

        function createCookie(name,value,days) {
            var expires = "";
            if (days) {
                var date = new Date();
                date.setTime(date.getTime() + (days*24*60*60*1000));
                expires = "; expires=" + date.toUTCString();
            }
            document.cookie = name + "=" + value + expires + "; path=/";
        }

        function readCookie(name) {
            var nameEQ = name + "=";
            var ca = document.cookie.split(';');
            for(var i=0;i < ca.length;i++) {
                var c = ca[i];
                while (c.charAt(0)==' ') c = c.substring(1,c.length);
                if (c.indexOf(nameEQ) == 0) return c.substring(nameEQ.length,c.length);
            }
            return null;
        }
