It was assigned &lt;a href=\&#34;https://msrc.microsoft.com/update-guide/en-US/vulnerability/CVE-2024-21320\&#34; target=\&#34;_blank\&#34; rel=\&#34;nofollow noreferrer\&#34;>CVE-2024-21320&lt;/a> with a CVSS score of 6.5.&lt;/p>\n&lt;/li>\n&lt;/ul>\n&lt;ul>\n&lt;li>&lt;p>The vulnerability can trigger an authentication coercion — an attack in which a victim is coerced into sending credentials (usually in the form of NTLM hashes over SMB) to an attacker’s machine.     <ul>
<li><p>Akamai security researcher <a href="/blog?author=tomer-peled" target="_self">Tomer Peled</a> recently discovered a spoofing vulnerability in Microsoft Themes. It was assigned <a href="https://msrc.microsoft.com/update-guide/en-US/vulnerability/CVE-2024-21320" target="_blank" rel="nofollow noreferrer">CVE-2024-21320</a> with a CVSS score of 6.5.</p>
</li>
</ul>
<ul>
<li><p>The vulnerability can trigger an authentication coercion — an attack in which a victim is coerced into sending credentials (usually in the form of NTLM hashes over SMB) to an attacker’s machine. The attacker can later crack the credentials offline.</p>
</li>
<li><p><b>To exploit this vulnerability, the attacker only needs the victim to download a theme file to their computer. When the user views the file in Explorer, the browser will automatically send Server Message Block (SMB) handshake packets that contain credentials to the attacker’s server.</b></p>
</li>
<li><p>All Windows versions are affected, as Themes is a built-in feature in the Windows operating system. </p>
</li>
<li><p>Microsoft fixed this vulnerability in <a href="/blog/security-research/akamai-perspective-patch-tuesday-january-2024" target="_self">January’s Patch Tuesday 2024</a>.</p>
</li>
<li><p>We provide a <a rel="nofollow noreferrer" href="https://github.com/akamai/akamai-security-research/tree/main/PoCs/CVE-2024-21320" target="_blank">proof-of-concept</a> (PoC) Theme file, as well as a PoC video, and present several ways to mitigate this vulnerability.</p>
</li>
</ul>

<div data-cmp-is="text" class="cmp-text" data-cmp-data-layer="{&#34;text-e5d58b8b50&#34;:{&#34;@type&#34;:&#34;akamai/components/text&#34;,&#34;repo:modifyDate&#34;:&#34;2024-03-05T14:04:47Z&#34;,&#34;xdm:text&#34;:&#34;&lt;p>Since the days of Windows XP, Microsoft has provided several previsual personalization options out of the box — colors, fonts, and cursors, to name a few.     <p>Since the days of Windows XP, Microsoft has provided several previsual personalization options out of the box — colors, fonts, and cursors, to name a few. This personalization is easy for users to do. They can view the installed themes simply by right-clicking on the desktop, selecting “personalize,” and clicking on “Themes.” Theme files have the suffix <i>.theme</i> and they can be created using <a href="https://learn.microsoft.com/en-us/windows/win32/controls/themesfileformat-overview" target="_blank" rel="nofollow noreferrer">this</a> overview from MSDN.</p>
<p>This seemingly innocuous nicety can provide a home for some sneaky vulnerabilities. In our <a href="/blog/security-research/akamai-perspective-patch-tuesday-september-2023" target="_self">analysis</a> of September’s  Patch Tuesday 2023, we briefly discussed the impact of <a href="https://msrc.microsoft.com/update-guide/vulnerability/CVE-2023-38146" target="_blank" rel="nofollow noreferrer">CVE-2023-38146</a>, a vulnerability within Themes. While analyzing the vulnerability, we decided to “play” with the values in a Theme file, and discovered there is a lack of validation in certain parameters.<br />
</p>
<p><b>By abusing this lack of validation, we can perform an attack with effectively zero user interaction: All the users must do is download a malicious Themes file.</b> Once the user views the file in Explorer, the exploit begins.</p>
<p>We have documented what you need to know about this vulnerability, including mitigations and a PoC. Read on.</p>

    <p>Since these actions start automatically, it's a great place for attackers to look for ways to manipulate the outcome. One of the ways attackers can do this is by changing the value of each of these three parameters to a UNC path pointing at an attacker-controlled server, hoping the victim's machine will try to authenticate against their machine.</p>
<p>We found that the path for the image can be any legitimate path, including a UNC path. Changing the values of “<i>BrandImage</i>” or “<i>Wallpaper</i>” yielded a connection from the victim's machine leading to an authentication coercion attack. Namely, as part of the connection to the remote server, the client performs an SMB negotiation during which their NTLM credentials are sent.</p>

      <h2 class="cmp-title__text">The consequences of an NTLM leak<br />
</h2>
    
</div>

    


</div>
<div class="text aem-GridColumn aem-GridColumn--default--12">
<div data-cmp-is="text" class="cmp-text" data-cmp-data-layer="{&#34;text-b9fa973972&#34;:{&#34;@type&#34;:&#34;akamai/components/text&#34;,&#34;repo:modifyDate&#34;:&#34;2024-03-05T05:11:50Z&#34;,&#34;xdm:text&#34;:&#34;&lt;p>With a victim’s NTLM credentials, an attacker can perform an NTLM relay attack — an attack on systems that accept NTLM as access credentials. By relaying the NTLM hashes, an attacker could authenticate as a legitimate user and thus gain access to systems they otherwise would not have access to.&lt;/p>\r\n&lt;p>Another option is to supply the NTLM credentials as input to a password-cracking application, such as John the Ripper, to try and crack the victim’s password using brute force.&lt;/p>\r\n&lt;p>An example of how great the impact can be is the infamous&amp;nbsp;&lt;a href=\&#34;https://msrc.microsoft.com/update-guide/vulnerability/CVE-2023-23397\&#34; target=\&#34;_blank\&#34; rel=\&#34;nofollow noreferrer\&#34;>Outlook vulnerability&lt;/a>, which was initially disclosed to Microsoft in March 2023. A similar outcome was found and it has been actively exploited in the wild, proving that this vector of attack is still relevant and lucrative for attackers.&lt;/p>\r\n&lt;p>The vulnerability from March allowed an attacker to send an email to the victim and trigger an audio file download. The path to that audio file could point anywhere, including to a remote server (using a UNC path). Akamai researcher &lt;a href=\&#34;/content/akamai/en/blog.html?author=ben-barnea\&#34; target=\&#34;_self\&#34;>Ben Barnea&lt;/a> wrote extensively about this vulnerability and how he found &lt;a href=\&#34;/content/akamai/en/blog/security-research/important-outlook-vulnerability-bypass-windows-api.html\&#34; target=\&#34;_self\&#34;>multiple&lt;/a> &lt;a href=\&#34;/content/akamai/en/blog/security-research/2023/dec/chaining-vulnerabilities-to-achieve-rce-part-one.html\&#34; target=\&#34;_self\&#34;>bypasses&lt;/a> to the patch for it.&lt;/p>\r\n&#34;}}">
    <p>With a victim’s NTLM credentials, an attacker can perform an NTLM relay attack — an attack on systems that accept NTLM as access credentials. By relaying the NTLM hashes, an attacker could authenticate as a legitimate user and thus gain access to systems they otherwise would not have access to.</p>
<p>Another option is to supply the NTLM credentials as input to a password-cracking application, such as John the Ripper, to try and crack the victim’s password using brute force.</p>
<p>An example of how great the impact can be is the infamous <a href="https://msrc.microsoft.com/update-guide/vulnerability/CVE-2023-23397" target="_blank" rel="nofollow noreferrer">Outlook vulnerability</a>, which was initially disclosed to Microsoft in March 2023. A similar outcome was found and it has been actively exploited in the wild, proving that this vector of attack is still relevant and lucrative for attackers.</p>
<p>The vulnerability from March allowed an attacker to send an email to the victim and trigger an audio file download. The path to that audio file could point anywhere, including to a remote server (using a UNC path). Akamai researcher <a href="/blog?author=ben-barnea" target="_self">Ben Barnea</a> wrote extensively about this vulnerability and how he found <a href="/blog/security-research/important-outlook-vulnerability-bypass-windows-api" target="_self">multiple</a> <a href="/blog/security-research/chaining-vulnerabilities-to-achieve-rce-part-one" target="_self">bypasses</a> to the patch for it.</p>

<div class="text text__pb-20 aem-GridColumn aem-GridColumn--default--12">
<div data-cmp-is="text" class="cmp-text" data-cmp-data-layer="{&#34;text-01eacb8a4f&#34;:{&#34;@type&#34;:&#34;akamai/components/text&#34;,&#34;repo:modifyDate&#34;:&#34;2024-03-05T05:14:27Z&#34;,&#34;xdm:text&#34;:&#34;&lt;p>In their patch, Microsoft added a call to a function that checks whether the input is a UNC path (“&lt;i>PathIsUNC&lt;/i>”), and a registry value to check whether UNC paths are allowed for Theme file use (Figure 6). If the path is a UNC path and UNC paths are not allowed, then the thumbnail will not be loaded. If they are allowed, the thumbnail will be created with the BrandImage and Wallpaper.&lt;/p>\r\n&lt;p>The value “&lt;i>DisableThumbnailOnNetworkFolder&lt;/i>” can be found in the registry location: “&lt;i>HKCU/Software/Microsoft/Windows/CurrentVersion/Policies/Explorer/Mitigation&lt;/i>”.The value is checked in the newly added function “&lt;i>IsUNCPathAllowedForThumbnailImage()&lt;/i>” inside the library &lt;i>themeui.dll&lt;/i>.&lt;/p>\r\n&#34;}}">
    <p>In their patch, Microsoft added a call to a function that checks whether the input is a UNC path (“<i>PathIsUNC</i>”), and a registry value to check whether UNC paths are allowed for Theme file use (Figure 6). If the path is a UNC path and UNC paths are not allowed, then the thumbnail will not be loaded. If they are allowed, the thumbnail will be created with the BrandImage and Wallpaper.</p>
<p>The value “<i>DisableThumbnailOnNetworkFolder</i>” can be found in the registry location: “<i>HKCU/Software/Microsoft/Windows/CurrentVersion/Policies/Explorer/Mitigation</i>”.The value is checked in the newly added function “<i>IsUNCPathAllowedForThumbnailImage()</i>” inside the library <i>themeui.dll</i>.</p>

      <h2 class="cmp-title__text">Mitigation<br />

<div class="text aem-GridColumn aem-GridColumn--default--12">
<div data-cmp-is="text" class="cmp-text" data-cmp-data-layer="{&#34;text-3695e457bd&#34;:{&#34;@type&#34;:&#34;akamai/components/text&#34;,&#34;repo:modifyDate&#34;:&#34;2024-03-05T05:19:47Z&#34;,&#34;xdm:text&#34;:&#34;&lt;p>On Windows 11, it is possible to block the use of NTLM authentication in SMB with remote machines via Group Policies. To do so, administrators should &lt;a href=\&#34;https://techcommunity.microsoft.com/t5/storage-at-microsoft/smb-ntlm-blocking-now-supported-in-windows-insider/ba-p/3916206\&#34; target=\&#34;_blank\&#34; rel=\&#34;nofollow noreferrer\&#34;>edit&lt;/a> Administrative Templates &amp;gt; Network &amp;gt; Lanman Workstation &amp;gt; Block NTLM.&lt;/p>\r\n&lt;p>Microsoft suggested using another policy named “Restrict NTLM” that will help mitigate this vulnerability. The policy can be activated by using &lt;a href=\&#34;https://msrc.microsoft.com/update-guide/en-US/vulnerability/CVE-2024-21320\&#34; target=\&#34;_blank\&#34; rel=\&#34;nofollow noreferrer\&#34;>Microsoft’s manual&lt;/a>.&lt;/p>\r\n&lt;p>Using microsegmentation, network administrators can block SMB traffic to remote locations outside the network. Such connections are not usually expected to take place, as SMB is mostly used for communication with Domain Controllers and file servers. See our comprehensive &lt;a href=\&#34;/content/akamai/en/blog/security-research/segmentation-from-a-practical-perspective.html#ms-rpc\&#34; target=\&#34;_self\&#34;>blog post on microsegmentation&lt;/a> for further elaboration on segmenting SMB traffic.&lt;/p>\r\n&#34;}}">
    <p>On Windows 11, it is possible to block the use of NTLM authentication in SMB with remote machines via Group Policies. To do so, administrators should <a href="https://techcommunity.microsoft.com/t5/storage-at-microsoft/smb-ntlm-blocking-now-supported-in-windows-insider/ba-p/3916206" target="_blank" rel="nofollow noreferrer">edit</a> Administrative Templates &gt; Network &gt; Lanman Workstation &gt; Block NTLM.</p>
<p>Microsoft suggested using another policy named “Restrict NTLM” that will help mitigate this vulnerability. The policy can be activated by using <a href="https://msrc.microsoft.com/update-guide/en-US/vulnerability/CVE-2024-21320" target="_blank" rel="nofollow noreferrer">Microsoft’s manual</a>.</p>
<p>Using microsegmentation, network administrators can block SMB traffic to remote locations outside the network. Such connections are not usually expected to take place, as SMB is mostly used for communication with Domain Controllers and file servers. See our comprehensive <a href="/blog/security-research/segmentation-from-a-practical-perspective#ms-rpc" target="_self">blog post on microsegmentation</a> for further elaboration on segmenting SMB traffic.</p>

<div class="text aem-GridColumn aem-GridColumn--default--12">
<div data-cmp-is="text" class="cmp-text" data-cmp-data-layer="{&#34;text-d2511bc4a4&#34;:{&#34;@type&#34;:&#34;akamai/components/text&#34;,&#34;repo:modifyDate&#34;:&#34;2024-03-05T05:22:48Z&#34;,&#34;xdm:text&#34;:&#34;&lt;p>Authentication coercion attacks are well known and used extensively by attackers for &lt;a href=\&#34;/content/akamai/en/glossary/what-is-lateral-movement.html\&#34; target=\&#34;_self\&#34;>lateral movement&lt;/a> and &lt;a href=\&#34;/content/akamai/en/glossary/what-is-credential-stuffing.html\&#34; target=\&#34;_self\&#34;>credential stuffing&lt;/a>. For example, a few years ago, the group called &lt;a href=\&#34;https://www.cisa.gov/news-events/alerts/2018/03/15/russian-government-cyber-activity-targeting-energy-and-other-critical\&#34; target=\&#34;_blank\&#34; rel=\&#34;nofollow noreferrer\&#34;>Dragonfly&lt;/a> used modified LNK files to obtain credentials over SMB. Triggering this kind of attack using Theme files is a surprising new vector.&lt;/p>\r\n&lt;p>This vulnerability stresses the importance of an &lt;a href=\&#34;/content/akamai/en/glossary/what-is-phishing.html\&#34; target=\&#34;_self\&#34;>anti-phishing protocol&lt;/a> inside organizations, as attackers can easily send a benign-looking file that will initiate this attack.&lt;/p>\r\n&lt;p>We are happy to see Microsoft attempting to mitigate this attack vector with the addition of relevant group policies. We advise defenders to update their endpoints with the latest &lt;a href=\&#34;https://msrc.microsoft.com/update-guide/en-US/vulnerability/CVE-2024-21320\&#34; target=\&#34;_blank\&#34; rel=\&#34;nofollow noreferrer\&#34;>security patch&lt;/a>.&lt;/p>\r\n&#34;}}">
    <p>Authentication coercion attacks are well known and used extensively by attackers for <a href="/glossary/what-is-lateral-movement" target="_self">lateral movement</a> and <a href="/glossary/what-is-credential-stuffing" target="_self">credential stuffing</a>. For example, a few years ago, the group called <a href="https://www.cisa.gov/news-events/alerts/2018/03/15/russian-government-cyber-activity-targeting-energy-and-other-critical" target="_blank" rel="nofollow noreferrer">Dragonfly</a> used modified LNK files to obtain credentials over SMB. Triggering this kind of attack using Theme files is a surprising new vector.</p>
<p>This vulnerability stresses the importance of an <a href="/glossary/what-is-phishing" target="_self">anti-phishing protocol</a> inside organizations, as attackers can easily send a benign-looking file that will initiate this attack.</p>
<p>We are happy to see Microsoft attempting to mitigate this attack vector with the addition of relevant group policies. We advise defenders to update their endpoints with the latest <a href="https://msrc.microsoft.com/update-guide/en-US/vulnerability/CVE-2024-21320" target="_blank" rel="nofollow noreferrer">security patch</a>.</p>
