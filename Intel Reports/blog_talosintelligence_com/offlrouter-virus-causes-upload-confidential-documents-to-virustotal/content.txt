    <h1 class="text-center">OfflRouter virus causes Ukrainian users to upload confidential documents to VirusTotal</h1>

    <div class="text-center m-1">
        <div class="post-author">
            <span>By </span>
                <a href="https://blog.talosintelligence.com/author/vanja-svajcer/">Vanja Svajcer</a>
        </div>

        <br/>
        <time class="post-datetime" datetime="April 17, 2024 07:59">
            Wednesday, April 17, 2024 07:59
        </time>

        <div class="m-3">
                        <a href="https://blog.talosintelligence.com/category/threat-spotlight/" class="category primary-category">
                            Threat Spotlight
                        </a>
                                            <a href="https://blog.talosintelligence.com/category/threats/" class="category primary-category">
                            Threats
                        </a>
                            </div>
    </div>

    <section class="post-content-wrapper mt-5">
        <div class="post-content">
            <ul><li>During a threat-hunting exercise, Cisco Talos discovered documents with potentially confidential information originating from Ukraine. The documents contained malicious VBA code, indicating they may be used as lures to infect organizations.&nbsp;</li><li>The results of the investigation have shown that the presence of the malicious code is due to the activity of a rare multi-module virus that's delivered via the .NET interop functionality to infect Word documents.&nbsp;</li><li>The virus, named OfflRouter, has been active in Ukraine since 2015 and remains active on some Ukrainian organizationsâ networks, based on over 100 original infected documents uploaded to VirusTotal from Ukraine and the documentsâ upload dates.&nbsp;</li><li>We assess that OfflRouter is the work of an inventive but relatively inexperienced developer, based on the unusual choice of the infection mechanism, the apparent lack of testing and mistakes in the code.&nbsp;</li><li>The authorâs design choices may have limited the spread of the virus to very few organizations while allowing it to remain active and undetected for a long period of time.&nbsp;</li></ul><p>As a part of a regular threat hunting exercise, Cisco Talos monitors files uploaded to open-source repositories for potential lures that may target government and military organizations. Lures are created from legitimate documents by adding content that will trigger malicious behavior and are often used by threat actors.&nbsp;</p><p>For example, malicious document lures with externally referenced templates written in Ukrainian language are used by the <a href="https://blog.talosintelligence.com/gamaredon-apt-targets-ukrainian-agencies/" rel="noreferrer noopener"><u>Gamaredon group</u></a> as an initial infection vector. Talos has previously discovered military theme lures in Ukrainian and Polish, mimicking the official PowerPoint and Excel files, to launch the so-called â<a href="https://blog.talosintelligence.com/malicious-campaigns-target-entities-in-ukraine-poland/" rel="noreferrer noopener"><u>Picasso loader</u></a><u>,â</u> which installs remote access trojans (RATs) onto victims' systems.&nbsp;</p><p>In July 2023, threat actors attempted to use lures related to the NATO summit in Vilnius to install the <a href="https://blogs.blackberry.com/en/2023/07/romcom-targets-ukraine-nato-membership-talks-at-nato-summit" rel="noreferrer noopener"><u>Romcom remote access trojan</u></a>. These are just some of the reasons why hunting for document lures is vital to any threat intelligence operation.&nbsp;&nbsp;</p><p>In February 2024, Talos discovered several documents with content that seems to originate from Ukrainian local government organizations and the Ukrainian National Police uploaded to VirusTotal. The documents contained VBA code to drop and run an executable with the name `ctrlpanel.exe`, which raised our suspicion and prompted us to investigate further.&nbsp;&nbsp;&nbsp;</p><p>Eventually, we discovered over 100 uploaded documents with potentially confidential information about government and police activities in Ukraine. The analysis of the code showed unexpected results â instead of lures used by advanced actors, the uploaded documents were infected with a multi-component VBA macro virus OfflRouter, created in 2015. The virus is still active in Ukraine and is causing potentially confidential documents to be uploaded to publicly accessible document repositories.&nbsp;&nbsp;&nbsp;</p><h2 id="attribution">Attribution&nbsp;</h2><p>Although the virus is active in Ukraine, there are no indications that it was created by an author from that region. Even the debugging database string used to name the virus âE:\Projects\OfflRouter2\OfflRouter2\obj\Release\ctrlpanel.pdbâ present in the ctrlpanel.exe does not point to a non-English speaker.&nbsp;</p><p>From the choice of the infection mechanism, VBA code generation, several mistakes in the code, and the apparent lack of testing, we estimate that the author is an inexperienced but inventive programmer.&nbsp;&nbsp;</p><p>The choices made during the development limited the virus to a specific geographic location and allowed it to remain active for almost 10 years.&nbsp;</p><h2 id="offlrouter-has-been-confined-to-ukraine">OfflRouter has been confined to Ukraine&nbsp;</h2><p>According to earlier research by the <a href="https://www.csirt.gov.sk/wp-content/uploads/2021/08/analysis_offlrouter.pdf" rel="noreferrer noopener"><u>Slovakian government CSIRT (Computer Security Incident Response Team) team</u></a>, some infected documents were already publicly available on the Ukrainian National Police website in 2018.&nbsp;&nbsp;</p><p>The newly discovered infected documents are written in Ukrainian, which may have contributed to the fact that the virus is rarely seen outside Ukraine. Since the malware has no capabilities to spread by email, it can only be spread by sharing documents and removable media, such as USB memory sticks with infected documents. The inability to spread by email and the initial documents in Ukrainian are additional likely reasons the virus stayed confined to Ukraine.&nbsp;&nbsp;</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/2024/04/data-src-image-5238cf56-e25e-4464-996c-29242926392d.png" class="kg-image" alt="" loading="lazy" width="800" height="727" srcset="https://blog.talosintelligence.com/content/images/size/w600/2024/04/data-src-image-5238cf56-e25e-4464-996c-29242926392d.png 600w, https://blog.talosintelligence.com/content/images/2024/04/data-src-image-5238cf56-e25e-4464-996c-29242926392d.png 800w" sizes="(min-width: 720px) 720px"><figcaption><span style="white-space: pre-wrap;">One of the documents was recently uploaded to VirusTotal from Ukraine.&nbsp;</span></figcaption></figure><p>The virus targets only documents with the filename extension .doc, the default extension for the OLE2 documents, and it will not try to infect other filename extensions. The default Word document filename extension for the more recent Word versions is .docx, so few documents will be infected as a result.&nbsp;&nbsp;</p><p>This is a possible mistake by the author, although there is a small probability that the malware was specifically created to target a few organizations in Ukraine that still use the .doc extension, even if the documents are internally structured as <a href="https://en.wikipedia.org/wiki/Office_Open_XML" rel="noreferrer noopener"><u>Office Open XML documents</u></a>.&nbsp;&nbsp;</p><p>Other issues prevented the virus from spreading more successfully and most of them are found in the executable module, ctrlpanel.exe, dropped and executed by the VBA part of the code.&nbsp;&nbsp;</p><p>When the virus is run, it attempts to set the value Ctrlpanel of the registry key HKLM\Software\Microsoft\Windows\CurrentVersion\Run so that it runs on the Windows boot. An internal global string _RootDir is used as the value, however, the string only contains the folder where the ctrlpanel.exe is found and not its full path, which makes this auto-start measure fail.&nbsp;&nbsp;</p><p>One interesting concept in the .NET module is the entire process of infecting documents. As a part of the infection process, the VBA code is generated by combining the code taken from the hard-coded strings in the module with the encoded bytes of the ctrlpanel.exe binary. This makes the generated code the same for every infection cycle and rather easy to detect. Having a VBA code generator in the .NET code has more potential to make the infected documents more difficult to detect.&nbsp;&nbsp;</p><p>Once launched, the executable module stays active in memory and uses threading timers to execute two background worker objects, the first one tasked with the infection of documents and the second one with checking for the existence of the potential plugin modules for the .NET module.&nbsp;&nbsp;</p><p>The infection background worker enumerates the mounted drives and attempts to find documents to infect by using the Directory.Getfiles function with the string search pattern â*.docâ as a parameter.&nbsp;&nbsp;</p><p>One of the parameters of the function is the SearchOption parameter which specifies the option to search in subdirectories or only the in the root folder. For fixed drives, the module chooses to search only the root folder, which is an unusual choice, as it is quite unlikely that the root folder will hold any documents to infect.&nbsp;&nbsp;</p><p>For removable drives, the module also searches all subfolders, which likely makes it more successful. Finally, it checks the list of recent documents in Word and attempts to infect them, which contributes to the success of the virus spreading to other documents on fixed drives.&nbsp;&nbsp;</p><figure https://blog.talosintelligence.com/content/images/2024/04/data-src-image-24887024-10a6-4462-ab56-3f7373e5e6a0.png 658w"><figcaption><i><em class="italic" style="white-space: pre-wrap;">The choice of document filename extensions is limited to .doc.</em></i></figcaption></figure><h2 id="offlrouter-vba-code-drops-and-executes-the-main-executable-module">OfflRouter VBA code drops and executes the main executable module&nbsp;</h2><p>The VBA part of the virus runs when a document is opened, provided macros are enabled, and it contains code to drop and run the executable module ctrlpanel.exe. </p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/2024/04/data-src-image-64d8a34f-7eb1-4aaa-bf25-930d50f4cec1.png" class="kg-image" alt="" loading="lazy" width="640" height="366" srcset="https://blog.talosintelligence.com/content/images/size/w600/2024/04/data-src-image-64d8a34f-7eb1-4aaa-bf25-930d50f4cec1.png part creates and executes the executable module of the virus.</em></i></figcaption></figure><p>&nbsp;The beginning of the macro code defines a function that checks if the file already exists and if it does not, it opens it for writing and calls functions CheckHashX, which at first glance looks like containing junk code to reset the value of the variable `Y`. However, the variable Y is defined as a private property with an overridden setter function, which converts the variable value assignment into appending the supplied value to the end of the opened executable module C:\Users\Public\ctrlpanel.exe. Every code line that looks like an assignment appends the assigned value to the end of the file, and that is how the executable module is written.&nbsp;&nbsp;</p><p>This technique is likely implemented to make the detection of the embedded module a bit more difficult, as the executable mode is not stored as a contiguous block, but as a sequence of integer values that look like being assigned to a variable.&nbsp;&nbsp;</p><p>To a more experienced analyst, this code looks like garbage code generated by polymorphic engines, and it raises the question of why the author has not extended the code generation to pseudo-randomize the code.&nbsp;&nbsp;</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/2024/04/data-src-image-1867f9e5-fd42-4411-aa92-da4cc9d4e4e0.jpeg" class="kg-image" alt="" loading="lazy" width="1600" height="828" style="white-space: pre-wrap;">Infected Word documents drop the .NET component that infects other documents</em></i><span style="white-space: pre-wrap;">.&nbsp;</span></figcaption></figure><p>The virus is unique, as it consists of VBA and executable modules with the infection logic contained in the PE executable .NET module.&nbsp;&nbsp;</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/2024/04/data-src-image-d8d5b00c-9675-401c-9022-be09c5e7d976.png" class="kg-image" alt="" loading="lazy" width="712" has an overridden setter function to write into a file.</em></i></figcaption></figure><h2 id="ctrlpanelexe-net-module">Ctrlpanel.exe .NET module&nbsp;</h2><p>The unique infection method used by ctrlpanel.exe is through the <a href="https://learn.microsoft.com/en-us/dotnet/csharp/advanced-topics/interop/how-to-access-office-interop-objects" rel="noreferrer noopener"><u>Office Interop</u></a> classes of .NET VBProject interface Microsoft.Vbe.Interop and the Microsoft.Office.Interop.Word class that exposes the functionality of the Word application.&nbsp;&nbsp;</p><p>The Interop.Word class is used to instantiate a Document class which allows access to the VBA macro code and the addition of the code to the target document file using the standard Word VBA functions.&nbsp;&nbsp;</p><p>When the .NET module ctrlpanel.exe is launched, it attempts to open a mutex ctrlpanelapppppp to check if another module instance is already running on the system. If the mutex already exists, the process will terminate.&nbsp;&nbsp;</p><p>Suppose no other module instances are running. In that case, ctrlpanel.exe creates the mutex named âctrlpanelappppppâ, attempts to set the registry run key so the module runs on system startup, and finally initializes two timers to run associated background timer callbacks â VBAClass_Timer_Callback and PluginClass_TimerCallback, implemented to start the Run function of the classes VBAClass and PluginClass, respectively.&nbsp;</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/2024/04/data-src-image-0885c4f8-6ded-4f2e-9742-23650e2c25ae.png" class="kg-image" alt="" loading="lazy" width="705" height="276" has two threads of execution.</em></i></figcaption></figure><p>The VBAClass_Timer_Callback will be called one second after the creation of VBAClass_Timer timer and the PluginClass_Timer_Callback three seconds after the creation of the PluginClass_Timer.&nbsp;&nbsp;</p><p>The full functionality of the executable module is implemented by two classes, VBAClass and PluginClass, specifically within their respective functions Backgroundworker_DoWork.&nbsp;</p><h4 id="vbaclass-is-tasked-with-generating-vba-code-and-infecting-other-word-documents"><em>VBAClass is tasked with generating VBA code and infecting other Word documents&nbsp;</em></h4><p>&nbsp;The background worker function runs in an infinite loop and contains two major parts, the first is tasked with the document infection and the second with finding the documents to infect, the logic we already described above.&nbsp;&nbsp;</p><p>The infection iterates through a list of the document candidates to infect and uses an innovative method to check the document infection marker to avoid multiple infection processes â the function checks the document creation metadata, adds the creation times, and checks the value of the sum. If the sum is zero, the document is considered already infected.&nbsp;&nbsp;</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/2024/04/data-src-image-50d981a0-7698-4302-a2b8-3c17e3519750.png" class="kg-image" alt="" loading="lazy" width="636" height="85" srcset="https://blog.talosintelligence.com/content/images/size/w600/2024/04/data-src-image-50d981a0-7698-4302-a2b8-3c17e3519750.png 600w, https://blog.talosintelligence.com/content/images/2024/04/data-src-image-50d981a0-7698-4302-a2b8-3c17e3519750.png 636w"><figcaption><i><em class="italic" style="white-space: pre-wrap;">The file system time of creation is used as an infection marker.</em></i></figcaption></figure><p>If the document to be infected is created in the Word 97-2003 binary format (OLE2), the document will be saved with the same name in Microsoft Office Open XML format with enabled macros (DOCX).&nbsp;&nbsp;</p><p>The infection uses the traditional VBA class infection routine. It accesses the Visual Basic code module of the first VBComponent and then adds the code generated by the function MyScript to the documentâs code module. After that, the infected document is saved.&nbsp;&nbsp;</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/2024/04/data-src-image-1edf41b4-715c-42b1-bd31-0ca35194bc9a.png" class="kg-image" alt="" loading="lazy" width="760" height="135" srcset="https://blog.talosintelligence.com/content/images/size/w600/2024/04/data-src-image-1edf41b4-715c-42b1-bd31-0ca35194bc9a.png 600w, https://blog.talosintelligence.com/content/images/2024/04/data-src-image-1edf41b4-715c-42b1-bd31-0ca35194bc9a.png 760w" sizes="(min-width: 720px) 720px"><figcaption><i><em class="italic" style="white-space: pre-wrap;">The target document is converted to .docx and then infected.</em></i></figcaption></figure><p>The code generation function MyScript contains static strings and instructions to dynamically generate code that will be added to infected documents. It opens its own executable ctrlpanel.exe for reading and reads the file 32-bit by 32-bit value which gets converted to decimal strings that can be saved as VBA code. For every repeating 32-bit value, most commonly for zero, the function creates a for loop to write the repeating value to the dropped file. The purpose is unclear, but it is likely to achieve a small saving in the size of the code and compress it.&nbsp;&nbsp;</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/2024/04/data-src-image-91a167cc-099c-4077-b6fd-62aafb0ac97b.png" class="kg-image" alt="" loading="lazy" width="632" height="155" srcset="https://blog.talosintelligence.com/content/images/size/w600/2024/04/data-src-image-91a167cc-099c-4077-b6fd-62aafb0ac97b.png 600w, https://blog.talosintelligence.com/content/images/2024/04/data-src-image-91a167cc-099c-4077-b6fd-62aafb0ac97b.png 632w"><figcaption><i><em class="italic" style="white-space: pre-wrap;">Repeating 32-bit values are âcompressed.â</em></i></figcaption></figure><p>For every 4,096 bytes, the code generates a new CheckHashX subroutine to break the code into chunks. The purpose of this is not clear.&nbsp;&nbsp;</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/2024/04/data-src-image-84b434a5-518a-4c1d-9d00-1fb107f17832.png" class="kg-image" alt="" loading="lazy" width="620" height="270" srcset="https://blog.talosintelligence.com/content/images/size/w600/2024/04/data-src-image-84b434a5-518a-4c1d-9d00-1fb107f17832.png 600w, https://blog.talosintelligence.com/content/images/2024/04/data-src-image-84b434a5-518a-4c1d-9d00-1fb107f17832.png 620w"><figcaption><i><em class="italic" style="white-space: pre-wrap;">The VBA code infecting files is dynamically created.</em></i></figcaption></figure><p>&nbsp;After the file is infected, the background worker adds the infection marker file by setting the values of hour, minute, second, and millisecond creation times to zero.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/2024/04/data-src-image-0b43b5ec-7b08-4a85-bbba-aeaefaa48e31.png" class="kg-image" alt="" loading="lazy" width="533" height="154"><figcaption><i><em class="italic" style="white-space: pre-wrap;">The infection market is set after the infected file has been copied to its original location.</em></i></figcaption></figure><h4 id="pluginclass-is-tasked-with-discovering-and-loading-plugins"><em>PluginClass is tasked with discovering and loading plugins&nbsp;</em></h4><p>Ctrlpanel.exe can also search for potential plugins present on removable media, which is very unusual for simple viruses that infect documents. This may indicate that the authorâs aims were a bit more ambitious than the simple VBA infection.&nbsp;</p><p>Searching for the plugins in removable drives is unusual as it requires the plugins to be delivered to the system on a physical media, such as a USB drive or a CD-ROM. The plugin loader searches for any files with the filename extension .orp, decodes its name using Base64 decoding function, decodes the file content using Base64 decoding, copies the file into the c:\users\public\tools folder with the previously decoded name and the extension .exe and finally executes the plugin.&nbsp;</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.talosintelligence.com/content/images/2024/04/data-src-image-b52ac17d-cf67-43f6-8b7f-cab1c39139c7.png" class="kg-image" alt="" loading="lazy" width="853" height="326" srcset="https://blog.talosintelligence.com/content/images/size/w600/2024/04/data-src-image-b52ac17d-cf67-43f6-8b7f-cab1c39139c7.png 600w, https://blog.talosintelligence.com/content/images/2024/04/data-src-image-b52ac17d-cf67-43f6-8b7f-cab1c39139c7.png 853w" sizes="(min-width: 720px) 720px"><figcaption><i><em class="italic" style="white-space: pre-wrap;">Any plugins found on a removable drive are decoded, copied to drive C: and launched.</em></i></figcaption></figure><p>If the plugins are already present on an infected machine, ctrlpanel.exe will try to encode them using Base64 encoding, copy them to the root of the attached removable media with the filename extension â.orpâ and set the newly created file attributes to the values system and hidden so that they are not displayed by default by Windows File Explorer.&nbsp;</p><h3 id="it-is-unclear-if-the-initial-vector-is-a-document-or-the-executable-module-ctrlpanelexe">It is unclear if the initial vector is a document or the executable module ctrlpanel.exe&nbsp;</h3><p>The advantage of the two-module virus is that it can be spread as a standalone executable or as an infected document. It may even be advantageous to initially spread as an executable as the module can run standalone and set the registry keys to allow execution of the VBA code and changing of the default saved file formats to doc before infecting documents. That way, the infection may be a bit stealthier.&nbsp;&nbsp;</p>
