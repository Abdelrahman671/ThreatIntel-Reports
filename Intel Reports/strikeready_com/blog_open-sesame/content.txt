        <div class="blog-content row justify-content-center" style="width: 100%; margin: 0 auto;">
            <div class="col-md-12">
                <p>The Bitter APT Group has a storied history of targeted cyber espionage, primarily focusing on South Asia. Their operations have been characterized by persistent attempts to infiltrate governmental and industrial entities to gather sensitive information. The group has shown an ability to bypass security technologies by leveraging obscure file formats, as well as leveraging password-protected or otherwise encrypted payloads.</p>

                <p>It can be difficult to track all the tools an actor uses, and disambiguate ownership between associated teams, especially without hundreds of incident responders on the bench.</p>

                <p>That being said, sometimes the attacker makes it easy, and they leave a copy of their backdoors on a misconfigured server, which was the case of <code>libraofficeonline.com</code>. In addition to the payloads being available, the zip was also available, named <code>op.zip</code>, probably short for <code>operation.zip</code>.  A zip file is especially useful when dealing with backdoors that may be interpreted by a handler, such as <code>.php</code> files.  The analysis of some of these interesting files is given below.</p>

                            <li>The zip file named op.zip (SHA256: b8beb5e27fc339772b63ed454ec054a16b554e5c354eab8de7b4addbe238f403) was found on a command and control server, and was likely mistakenly exposed by the actor. It contains multiple files with functionalities ranging from basic backdoors to information stealers. Among these files, one exciting file is the msws.msi, which further drops the payload of ORPCBackdoor.</li>
                        </ul>

                        <p style="background-color: #fff; margin-bottom: 1rem !important;">

                        <ul style="list-style-type: disc;">
                            <li>This file is dropping a cab file which contains a payload of ORCPBackdoor (SHA256: 74ba5883d989566a94e7c6c217b17102f054ffbe98bc9c878a7f700f9809e910)</li>

                            <li>It acts as a delivery mechanism, dropping multiple files and contributing to espionage activities.</li>
                        </ul>
                    </li>


                    <li style="margin-bottom: 1rem !important;">
                        <b class="color-green">SearchApp.jpg</b>

                        <ul style="list-style-type: disc;">

                            <li>Downloads command into <code>C:\Users\[user]\AppData\Roaming\commands.txt</code>, executes them, and uploads the results back to the server.</li>

                            <li>Sends an encoded "hello" message, receives commands, decodes them, and executes various operations such as listing directories, running shells, deleting files, and more.</li>

                        <ul style="list-style-type: disc;">
                            <li>A browser credential stealer.</li>
                            <li>Extracts information from Firefox and Thunderbird stores it in <code>C:\Users\Public\Documents\ats</code> and creates a zip file of this folder.</li>

                            <li>Targets Chrome, Edge, and Brave browsers, saves the data in <code>C:\Users\Public\Documents\als</code> and zips the folder.</li>

                            <li>The application captures screenshots, saves them locally, resizes the images, and sends them to a remote server located at <code>windowphotoviewer[.]com</code> on port <code>443</code>.</li>

                        </ul>
