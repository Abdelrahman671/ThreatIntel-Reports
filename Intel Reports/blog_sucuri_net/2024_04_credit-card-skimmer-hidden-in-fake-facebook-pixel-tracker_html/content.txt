</figure></div><div class="entry-container"><div class="entry-content-wrap"><div class="entry-content"><p>In recent months, we have encountered a number of cases where attackers inject malware into website software that allows for custom or miscellaneous code — for example, the miscellaneous scripts area of the Magento admin panel, or WordPress plugins such as <a href="https://blog.sucuri.net/2024/03/sign1-malware-analysis-campaign-history-indicators-of-compromise.html">Custom CSS &amp; JS</a>.</p><p>Custom script editors are popular with bad actors because they allow for external third party (and malicious) JavaScript and can easily pretend to be benign by leveraging naming conventions that match popular scripts like <a href="https://blog.sucuri.net/2022/08/examining-less-common-wordpress-credit-card-skimmers.html" target="_blank" rel="noopener">Google Analytics</a> or libraries like <a href="https://blog.sucuri.net/2021/11/woocommerce-skimmer-spoofs-checkout-page.html" target="_blank" rel="noopener">JQuery</a>. Furthermore, they can be embedded in any site without any real verification of their validity. Once those scripts are injected, they introduce additional (and sometimes outright malicious) functionality to the site where they have been inserted.</p><p>We recently encountered a rather interesting case of this: the attackers took that a step further by embedding a credit card skimmer in a well-concealed fake Facebook Pixel tracker script. Facebook Pixel tracking scripts use the same concept as Google Tags — but instead of adding functionality to the site, they measure, optimize and build audiences for ad campaigns.</p><p>Let’s take a closer look!</p><h2>Inspecting the Facebook pixel script</h2><p>Below we can see a normal Facebook Pixel script.</p><p><picture decoding="async" class="aligncenter wp-image-33598 size-full">

However, after a bit of scrutiny we can see the use of “<b>atob</b>” (a favorite among online pick-pockets, which translates a base64 encoded string to regular text).</p><p>You’ll also notice a couple of strings which appear to be random letters. The data entered into the fake form is then sent to the hackers controlled script on yet another compromised site: <strong>hxxps://www.donjuguetes[.]es/health_check.php</strong></p><h2>Symptoms of compromise</h2><p>Unfortunately, because this type of attack uses a valid script template the symptoms may not present themselves in an obvious manner. Because credit card stealers often wait for keywords such as ‘<b>checkout</b>’ or ‘<b>onepage</b>’, they may not become visible until the checkout page has loaded.</p><p>Since most checkout pages are dynamically generated based on cookie data and other variables passed to the page, these scripts evade public scanners and the only way to identify the malware is to check the page source or watch network traffic. These scripts run silently in the background.</p><p>That being said, leveraging an <a href="https://sucuri.net/malware-detection-scanning/">Intrusion Detection System</a> and website monitoring is an excellent way to catch known skimmers and other website malware at the early stages before they impact your site and traffic.</p><h2>Protecting your site from credit card skimmers</h2><p>There are a number of steps you can take to protect your ecommerce site from credit card skimmers.</p><ul><li aria-level="1"><b>Keep your site up to date</b>. The most common method of infection is via outdated software: attackers are always checking for vulnerabilities in outdated plugins and themes and leverage automation to quickly exploit vulnerable environments. You can prevent this by simply <a href="https://blog.sucuri.net/2022/09/a-guide-to-virtual-patching-for-website-vulnerabilities.html" target="_blank" rel="noopener">patching your site</a> and ensuring software is using the latest security updates.</li><li aria-level="1"><b>Review admin accounts and keep passwords updated.</b> Attackers commonly gain access via accounts with weak passwords. Once they have gained access using an admin account, they can make unlimited changes to the site content. Attackers will often add additional admin accounts that they can use later. It is important to routinely review the admin accounts that are configured to confirm they are all valid, and to change passwords on a regular basis. Ensure all <a href="https://blog.sucuri.net/2024/01/how-to-make-strong-password.html" target="_blank" rel="noopener">passwords are strong</a> and unique from other credentials.</li><li aria-level="1"><b>Leverage file integrity and website monitoring.</b> <a href="https://blog.sucuri.net/2021/05/server-side-scans-and-file-integrity-monitoring.html">File integrity monitoring</a> checks and compares files against a known baseline to detect any suspicious or unexpected changes. This essentially functions as an early detection system. If a hacker gains access to your website and modifies files, or if malware alters your site’s code, it will flag these changes, allowing for rapid response and minimizing potential damage.</li><li aria-level="1"><b>Protect your site with a web application firewall. </b>A properly configured <a href="https://sucuri.net/website-firewall/" target="_blank" rel="noopener">website firewall</a> can help to block malicious traffic by preventing hack attempts from reaching the hosting server.</li></ul><p>If you believe that your site is infected with skimmers or other malware, reach out or <a href="https://sucuri.net/live-chat/">start a chat</a>!
