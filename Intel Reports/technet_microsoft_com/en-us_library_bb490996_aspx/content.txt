							<div class="content "><h1 id="schtasks">Schtasks</h1><div class="display-flex justify-content-space-between align-items-center flex-wrap-wrap page-metadata-container">
										<div class="margin-right-xxs">
											<ul class="metadata page-metadata" data-bi-name="page info" lang="en-us" dir="ltr"><li>Article</li><li>
														<time class="is-invisible" data-article-date aria-label="Article review date" datetime="2009-09-11T00:00:00.000Z" data-article-date-source="ms.date">09/11/2009</time>
													</li></ul>

											<h2 id="ms--in-this-article" class="title is-6 margin-block-xs">In this article</h2>
										</nav><!-- <content> --><p>Schedules commands and programs to run periodically or at a specific time. Adds and removes tasks from the schedule, starts and stops tasks on demand, and displays and changes scheduled tasks.</p>
<p>To view the command syntax, click the following command:</p>
<p>schtasks create</p>
<p>Creates a new scheduled task.</p>
<h3 id="syntax">Syntax</h3>
<p><strong>schtasks</strong>&nbsp;<strong>/create</strong>&nbsp;<strong>/tn</strong>&nbsp;<em>TaskName</em>&nbsp;<strong>/tr</strong>&nbsp;<em>TaskRun</em>&nbsp;<strong>/sc</strong>&nbsp;<em>schedule</em>&nbsp;[<strong>/mo</strong>&nbsp;<em>modifier</em>]&nbsp;[<strong>/d</strong>&nbsp;<em>day</em>]&nbsp;[<strong>/m</strong>&nbsp;<em>month</em>[<strong>,<strong><em>month</em>...]&nbsp;[</strong>/i</strong>&nbsp;<em>IdleTime</em>]&nbsp;[<strong>/st</strong>&nbsp;<em>StartTime</em>]&nbsp;[<strong>/sd</strong>&nbsp;<em>StartDate</em>]&nbsp;[<strong>/ed</strong>&nbsp;<em>EndDate</em>]&nbsp;[<strong>/s</strong>&nbsp;<em>computer</em> [<strong>/u</strong> [<em>domain</em>\]<em>user</em>&nbsp;<strong>/p</strong>&nbsp;<em>password</em>]]&nbsp;[<strong>/ru</strong> {[<em>Domain</em>\]<em>User</em> | <strong>"System"</strong>} [<strong>/rp</strong>&nbsp;<em>Password</em>]]&nbsp;<strong>/?</strong>&nbsp;</p>
<h3 id="parameters">Parameters</h3>
<p><strong>/tn</strong> &nbsp; <em><strong>TaskName</strong></em> &nbsp; <strong>:</strong> Specifies a name for the task.</p>
<p><strong>/tr</strong> &nbsp; <em><strong>TaskRun</strong></em> &nbsp; <strong>:</strong> Specifies the program or command that the task runs. Type the fully qualified path and file name of an executable file, script file, or batch file. If you omit the path, SchTasks.exe assumes that the file is in the <em>Systemroot</em>\System32 directory.</p>
<p><strong>/sc</strong> &nbsp; <em><strong>schedule</strong></em> &nbsp; <strong>:</strong> Specifies the schedule type. Valid values are MINUTE, HOURLY, DAILY, WEEKLY, MONTHLY, ONCE, ONSTART, ONLOGON, ONIDLE.</p>
<table>
<colgroup>
<col>
<col>
</colgroup>

<tr class="odd">

<td><p>Specifies the time unit for the schedule.</p></td>
</tr>
<tr class="even">
<td><p><strong>ONCE</strong></p></td>
<td><p>The task runs once at a specified date and time.</p></td>
</tr>
<tr class="odd">
<td><p><strong>ONSTART</strong></p></td>
<td><p>The task runs every time the system starts. You can specify a start date, or run the task the next time the system starts.</p></td>
</tr>
<tr class="even">
<td><p><strong>ONLOGON</strong></p></td>
<td><p>The task runs whenever a user (any user) logs on. You can specify a date, or run the task the next time the user logs on.</p></td>
</tr>
<tr class="odd">
<td><p><strong>ONIDLE</strong></p></td>
<td><p>The task runs whenever the system is idle for a specified period of time. You can specify a date, or run the task the next time the system is idle.</p></td>
</tr>
</tbody>
</table>
<p><strong>/mo</strong> &nbsp; <em><strong>modifier</strong></em> &nbsp; <strong>:</strong> Specifies how often the task runs within its schedule type. This parameter is required for a MONTHLY schedule. This parameter is valid, but optional, for a MINUTE, HOURLY, DAILY, or WEEKLY schedule. The default value is&nbsp;1.</p>

<h3 id="remarks">Remarks</h3>
<ul>
<li><p>Using the <strong>/s</strong>, <strong>/u</strong>, and <strong>/p</strong> command-line options</p>
<p>The <strong>/u</strong> and <strong>/p</strong> command-line options are available only when you use <strong>/s</strong>. You must use <strong>/p</strong> with <strong>/u</strong> to provide the user's password.</p>
</li>
<li><p>SchTasks.exe does not verify program file locations or user account passwords. If you do not enter the correct file location or the correct password for the user account, the task is created, but it does not run. Also, if the password for an account changes or expires, and you do not change the password saved in the task, then the task does not run.</p>
</li>
<li><p>The NT Authority\System account does not have interactive logon rights. Users do not see and cannot interact with programs run with system permissions.</p>
</li>
<li><p>Each task runs only one program. However, you can create a batch file that starts multiple tasks, and then schedule a task that runs the batch file.</p>
</li>
<li><p>You can test a task as soon as you create it. Use the <strong>run</strong> operation to test the task and then check the SchedLgU.txt file (<em>Systemroot</em>\SchedLgU.txt) for errors.</p>
</li>
</ul>

<h5 id="to-schedule-a-task-to-run-every-20minutes">To schedule a task to run every 20&nbsp;minutes</h5>
<p>The following command schedules a security script, Sec.vbs, to run every 20&nbsp;minutes. Because the command does not include a starting date or time, the task starts 20&nbsp;minutes after the command completes, and runs every 20&nbsp;minutes thereafter whenever the system is running. Please enter the password
************</pre>

<p>SUCCESS: The Scheduled Task "Security Script" has successfully been created.&nbsp;</p>

<p><strong>schtasks</strong>&nbsp;<strong>/create</strong>&nbsp;<strong>/tn</strong>&nbsp;<em>TaskName</em>&nbsp;<strong>/tr</strong>&nbsp;<em>TaskRun</em>&nbsp;<strong>/sc onidle</strong>&nbsp;<strong>/i</strong><em>IdleTime</em>&nbsp;[<strong>/sd</strong>&nbsp;<em>StartDate</em>]&nbsp;[<strong>/s</strong>&nbsp;<em>computer</em> [<strong>/u</strong> [<em>domain</em>\]<em>user</em>&nbsp;<strong>/p</strong>&nbsp;<em>password</em>]]&nbsp;[<strong>/ru</strong> {[<em>Domain</em>\]<em>User</em> | <strong>"System"</strong>} [<strong>/rp</strong>&nbsp;<em>Password</em>]]&nbsp;</p>
<h4 id="example-7">Example</h4>
<h5 id="to-schedule-a-task-that-runs-whenever-the-computer-is-idle">To schedule a task that runs whenever the computer is idle</h5>
<p>The following command schedules the MyApp program to run whenever the computer is idle. It uses the required <strong>/i</strong> parameter to specify that the computer must remain idle for ten minutes before the task starts.</p>
<p><strong>schtasks /create /tn "My App" /tr c:\apps\myapp.exe /sc onidle /i 10</strong></p>
<h3 id="more-examples">More examples</h3>
<p>To view the example, click the example name:</p>
<p>To create a task that runs with System permissions</p>
<p>The following command schedules the MyApp program to run with permissions of the NT Authority\System account. In this example, the task is scheduled to run on the first day of every month, but you can use any schedule type for a task run with system permissions.</p>
<p>The command uses the <strong>/ru "System"</strong> parameter to specify the system security context. Because system tasks do not use a password, the <strong>/rp</strong> parameter is omitted.</p>
<p><strong>schtasks /create /tn "My App" /tr c:\apps\myapp.exe /sc monthly /d 1 /ru "System"</strong></p>
<p>In response, SchTasks.exe displays an informational message and a success message. It does not prompt for a password.</p>
<pre>INFO: The task will be created under user name ("NT AUTHORITY\SYSTEM").
SUCCESS: The Scheduled task "My App" has successfully been created.</pre>
<p>To create a task that runs more than one program</p>
<p>Each task runs only one program. However, you can create a batch file that runs multiple programs and then schedule a task to run the batch file. The following procedure demonstrates this method:</p>
<ol>
<li><p>Create a batch file that starts the programs you want to run.</p>
<p>In this example, you create a batch file that starts Event Viewer (Eventvwr.exe) and System Monitor (Perfmon.exe).</p>
<ul>
<li><p>Open a text editor, such as Notepad.</p>
</li>
<li><p>Type the name and fully qualified path to the executable file for each program. In this case, the file includes the following statements.</p>
<p><strong>C:\Windows\System32\Eventvwr.exe</strong><br>
<strong>C:\Windows\System32\Perfmon.exe</strong></p>
</li>
<li><p>Save the file as MyApps.bat.</p>
</li>
</ul>
</li>
<li><p>Use SchTasks.exe to create a task that runs MyApps.bat.</p>
<p>The following command creates the Monitor task, which runs whenever anyone logs on. It uses the <strong>/tn</strong> parameter to name the task, and the <strong>/tr</strong> parameter to run MyApps.bat. It uses the <strong>/sc</strong> parameter to indicate the OnLogon schedule type and the <strong>/ru</strong> parameter to specify the Administrator account.</p>
<p><strong>schtasks /create /tn Monitor /tr C:\MyApps.bat /sc onlogon /ru Reskit\Administrator</strong></p>
<p>As a result of this command, whenever a user logs on to the computer, the task starts both Event Viewer and System Monitor.</p>
</li>
</ol>

<p>Changes one or more of the following properties of a task.</p>
<ul>
<li><p>The program that the task runs (<strong>/tr</strong>).</p>
</li>
<li><p>The user account under which the task runs (<strong>/ru</strong>).</p>
</li>
<li><p>The password for the user account (<strong>/rp</strong>).</p>
</li>

<h3 id="parameters-1">Parameters</h3>
<p><strong>/tn</strong> &nbsp; <em><strong>TaskName</strong></em> &nbsp; <strong>:</strong> Identifies the task to be changed. Enter the task name.</p>
<p><strong>/s</strong> &nbsp; <em><strong>Computer</strong></em> &nbsp; <strong>:</strong> Specifies the name or IP address of a remote computer (with or without backslashes). The default is the local computer.</p>
<p><strong>/u</strong> <strong>[</strong> <em><strong>domain</strong></em> <strong>\]</strong> <em><strong>user</strong></em> &nbsp; <strong>:</strong> Runs the command with the permissions of the specified user account. <p><strong>/tr</strong> &nbsp; <em><strong>TaskRun</strong></em> &nbsp; <strong>:</strong> Changes the program that the task runs. Enter the fully qualified path and file name of an executable file, script file, or batch file. If you omit the path, SchTasks.exe assumes that the file is in the <em>Systemroot</em>\System32 directory. The specified program replaces the original program run by the task.</p>
<p><strong>/ru</strong> <strong>[</strong> <em><strong>Domain</strong></em> <strong>\]</strong> <em><strong>User</strong></em> <strong>|</strong> <strong>"System"</strong> &nbsp; <strong>:</strong> Changes the user account for the task.</p>

<h3 id="examples-1">Examples</h3>
<h4 id="to-change-the-program-that-a-task-runs">To change the program that a task runs</h4>
<p>The following command changes the program that the Virus Check task runs from VirusCheck.exe to VirusCheck2.exe. This command uses the <strong>/tn</strong> parameter to identify the task and the <strong>/tr</strong> parameter to specify the new program for the task. (You cannot change the task name.)</p>
<p><strong>schtasks /change /tn "Virus Check" /tr C:\VirusCheck2.exe</strong></p>
<p>In response, SchTasks.exe displays the following success message:</p>
<pre>SUCCESS: The parameter of the Scheduled Task "Virus Check" has been changed.</pre>
<p>As a result of this command, the Virus Check task now runs VirusCheck2.exe.</p>
<h4 id="to-change-the-password-for-a-remote-task">To change the password for a remote task</h4>
<p>The following command changes the password of the user account for the RemindMe task on the remote computer, Svr01. The command uses the <strong>/tn</strong> parameter to identify the task and the <strong>/s</strong> parameter to specify the remote computer. It uses the <strong>/rp</strong> parameter to specify the new password, p@ssWord3.</p>
<p>This procedure is required whenever the password for a user account expires or changes. If the password saved in a task is no longer valid, then the task does not run.</p>
<p><strong>schtasks /change /tn RemindMe /s Svr01 /rp p@ssWord3</strong></p>
<p>In response, SchTasks.exe displays the following success message:</p>
<pre>SUCCESS: The parameter of the Scheduled Task "RemindMe" has been changed.</pre>
<p>As a result of this command, the RemindMe task now runs under its original user account, but with a new password.</p>
<h4 id="to-change-the-program-and-user-account-for-a-task">To change the program and user account for a task</h4>
<p>The following command changes the program that a task runs and changes the user account under which the task runs. Essentially, it uses an old schedule for a new task. This command changes the Notepad task, which starts Notepad.exe every morning at 9:00 a.m., to start Internet Explorer instead.</p>
<p>The command uses the <strong>/tn</strong> parameter to identify the task. It uses the <strong>/tr</strong> parameter to change the program that the task runs and the <strong>/ru</strong> parameter to change the user account under which the task runs.</p>
<p>The <strong>/rp</strong> parameter, which provides the password for the user account, is omitted. You must provide a password for the account, but you can use the <strong>/rp</strong> parameter and type the password in clear text, or wait for SchTasks.exe to prompt you for a password, and then enter the password in obscured text.</p>
<p><strong>schtasks /change /tn Notepad /tr "c:\program files\Internet Explorer\iexplore.exe" /ru DomainX\Admin01</strong></p>
<p>In response, SchTasks.exe requests the password for the user account. It obscures the text you type, so the password is not visible.</p>
<pre>Please enter the password for DomainX\Admin01: *********</pre>
<p>Note that the <strong>/tn</strong> parameter identifies the task and that the <strong>/tr</strong> and <strong>/ru</strong> parameters change the properties of the task. You cannot use another parameter to identify the task and you cannot change the task name.</p>
<p>In response, SchTasks.exe displays the following success message:</p>
<pre>SUCCESS: The parameter of the Scheduled Task "Notepad" has been changed.</pre>
<p>As a result of this command, the RemindMe task now runs under its original user account, but with a new password.</p>
<h4 id="to-change-a-program-to-the-system-account">To change a program to the System account</h4>
<p>The following command changes the SecurityScript task so that it runs with permissions of the NT Authority\System account. It uses the <strong>/ru ""</strong> parameter to indicate the System account.</p>
<p><strong>schtasks /change /tn SecurityScript /ru ""</strong></p>
<p>In response, SchTasks.exe displays the following success message:</p>
<pre>SUCCESS: The parameter of the Scheduled Task "SecurityScript" has been changed.</pre>
<p>Because tasks run with System account permissions do not require a password, SchTasks.exe does not prompt for one.</p>

<h4 id="to-run-a-task-on-a-remote-computer">To run a task on a remote computer</h4>

<p>In this case, SchTasks.exe displays the following error message:</p>
<pre>ERROR: Unable to run the Scheduled Task "Update".</pre>
<p>To find the cause of the error, look in the Scheduled Tasks transaction log, C:\Windows\SchedLgU.txt on Svr01. In this case, the following entry appears in the log:</p>
<pre>"Update.job" (update.exe) 3/26/2001 1:15:46 PM ** ERROR **
The attempt to log on to the account associated with the task failed, therefore, the task did not run.
The specific error is:
0x8007052e: Logon failure: unknown user name or bad password.
Verify that the task's Run-as name and password are valid and try again.</pre>
<p>Apparently, the user name or password in the task is not valid on the system. The following <strong>schtasks /change</strong> command updates the user name and password for the Update task on Svr01:</p>
<p><strong>schtasks /change /tn Update /s Svr01 /ru Administrator /rp PassW@rd3</strong></p>
<p>After the <strong>change</strong> command completes, the <strong>run</strong> command is repeated. This time, the Update.exe program starts and SchTasks.exe displays the following message:</p>
<pre>SUCCESS: The Scheduled Task "Update" is running......</pre>

<p><strong>/u</strong> <strong>[</strong> <em><strong>domain</strong></em> <strong>\]</strong> <em><strong>user</strong></em> &nbsp; <strong>:</strong> Runs the command with the permissions of the specified user account. By default, the command runs with the permissions of the user logged on to the computer running SchTasks.</p>
<p><strong>/p</strong> &nbsp; <em><strong>password</strong></em> &nbsp; <strong>:</strong> Specifies the password of the user account specified in the <strong>/u</strong> parameter. This parameter is required when the <strong>/u</strong> parameter is used.</p>
<p><strong>/?</strong> <strong>:</strong> Displays help at the command prompt.</p>
<h3 id="remarks-5">Remarks</h3>
<ul>
<li><p>Using the <strong>/s</strong>, <strong>/u</strong>, and <strong>/p</strong> command-line options</p>
<p>The <strong>/u</strong> and <strong>/p</strong> command-line options are available only when you use <strong>/s</strong>. You must use <strong>/p</strong> with <strong>/u</strong> to provide the user's password.</p>
</li>
<li><p>The <strong>query</strong> operation lists all tasks scheduled for the computer, not just the tasks scheduled by the current user.</p>
</li>
</ul>
<h3 id="examples-5">Examples</h3>
<h4 id="to-display-the-scheduled-tasks-on-the-local-computer">To display the scheduled tasks on the local computer</h4>
<p>The following commands display all tasks scheduled for the local computer. These commands produce the same result and can be used interchangeably.</p>
<p><strong>schtasks</strong></p>
<p><strong>schtasks /query</strong></p>
<p>In response, SchTasks.exe displays the tasks in the default, siple table format, as shown in the following table:</p>
<pre>TaskName                  Next Run Time            Status
========================= ======================== ==============
Microsoft Outlook         At logon time            
SecureScript              14:42:00 PM , 2/4/2001   </pre>
<h4 id="to-display-advanced-properties-scheduled-tasks">To display advanced properties scheduled tasks</h4>
<p>The following command requests a detailed display of the tasks on the local computer. It uses the <strong>/v</strong> parameter to request a detailed (verbose) display and the <strong>/fo LIST</strong> parameter to format the display as a list for easy reading. You can use this command to verify that a task you created has the intended recurrence pattern.</p>
<p><strong>schtasks /query /fo LIST /v</strong></p>
<p>In response, SchTasks.exe displays a detailed property list for all tasks. The following display shows the task list for a task scheduled to run at 4:00&nbsp;A.M. on the last Friday of every month:</p>
<pre>HostName:                                  RESKIT01

<h4 id="to-log-tasks-scheduled-for-a-remote-computer">To log tasks scheduled for a remote computer</h4>
<p>The following command requests a list of tasks scheduled for a remote computer, and adds the tasks to a comma-separated log file on the local computer. You can use this command format to collect and track tasks that are scheduled for multiple computers.</p>
<p>The command uses the <strong>/s</strong> parameter to identify the remote computer, Reskit16, the <strong>/fo</strong> parameter to specify the format and the <strong>/nh</strong> parameter to suppress the column headings. The <strong>&gt;&gt;</strong> append symbol redirects the output to the task log, p0102.csv, on the local computer, Svr01. Because the command runs on the remote computer, the local computer path must be fully qualified.</p>
<p><strong>schtasks /query /s Reskit16 /fo csv /nh &gt;&gt; \\svr01\data\tasklogs\p0102.csv</strong></p>
<p>In response, SchTasks.exe adds the tasks scheduled for the Reskit16 computer to the p0102.csv file on the local computer, Svr01.</p>
<h3 id="remarks-6">Remarks</h3>
<ul>
<li><p>SchTasks.exe performs operations similar to those in Scheduled Tasks in Control Panel. You can use either tool to create, delete, configure, or display scheduled tasks.</p>
</li>
<li><p>Typing <strong>schtasks</strong> without any parameters performs a query.</p>
</li>
<li><p>The user must be a member of the Administrators group on the computer that the command affects.</p>
</li>
<li><p>To verify that a scheduled task ran or to find out why a scheduled task did not run, see the Task Scheduler service transaction log, <em>Systemroot</em>\SchedLgU.txt. This log records attempted runs initiated by all tools that use the service, including Scheduled Tasks and SchTasks.exe.</p>
</li>
<li><p>On rare occasions, task files become corrupted. Corrupted tasks do not run. When you try to perform an operation on corrupted tasks, SchTasks.exe displays the following error message:</p>
<pre>ERROR: The data is invalid.</pre>
<p>You cannot recover corrupted tasks. To restore the task scheduling features of the system, use SchTasks.exe or Scheduled Tasks to delete the tasks from the system and reschedule them.</p>
</li>
<li><p>SchTasks.exe replaces At.exe, a tool included in previous versions of Windows.</p>
</li>
</ul>
