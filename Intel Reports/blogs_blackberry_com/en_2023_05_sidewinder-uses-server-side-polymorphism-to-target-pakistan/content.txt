    <h3><b>Summary</b></h3>
<p>The BlackBerry Threat Research and Intelligence team has been actively tracking and monitoring the <a href="https://malpedia.caad.fkie.fraunhofer.de/actor/razor_tiger" target="_blank" title="sidewinder attack group info">SideWinder</a> APT group, which has led to the discovery of their latest campaign targeting Pakistan government organizations.</p>
<p>In this campaign, the SideWinder advanced persistent threat (APT) group used a server-based polymorphism technique to deliver the next stage payload.</p>

<h3><b><br>
Technical Analysis</b></h3>
<p><b>Context</b></p>
<p>The SideWinder APT group, also known as Razor Tiger, Rattlesnake, and T-APT-04, has been actively targeting Pakistan government organizations since at least 2012.</p>
<p>One of the oldest nation-state threat actors, SideWinder is believed to <a href="https://attack.mitre.org/groups/G0121/" target="_blank" title="mitre attack info on apt group sidewinder">originate from India</a>. Active since at least 2012, the group has been observed targeting military, government, and business entities, with a particular focus on Pakistan, Afghanistan, China, and Nepal. SideWinder primarily makes use of <a href="https://cdn-cybersecurity.att.com/docs/global-perspective-of-the-sidewinder-apt.pdf" target="_blank" title="sidewinder apt group attack techniques">email spear-phishing, document exploitation, and DLL side-loading techniques</a> in an attempt to avoid detection and deliver targeted implants. &nbsp;</p>
<p>Through our threat hunting efforts, the BlackBerry Threat Research and Intelligence team discovered a new malware campaign by the SideWinder group. This campaign utilized a server-side polymorphism technique. The use of this technique allows the threat actor to potentially bypass traditional signature-based antivirus (AV) detection to deliver the next stage payload.</p>

<h3><b><br>
What is Server-Side Polymorphism?</b></h3>
<p><a href="https://www.csoonline.com/article/2633048/stopping-malware-that-mutates-on-demand.html" target="_blank" title="explanation of server-side polymorphism">Server-side polymorphism</a> is a technique used by threat actors and other distributors of malware to attempt to evade detection by antivirus scanners. Polymorphic (literally “many shapes”) malware is malicious code that alters its appearance through encryption and obfuscation, making sure that no two samples look the same. It is hard for traditional or legacy AV software based on signatures to catch this type of malware, because the transformation code is not visible for security analysis. Although futuristic-sounding, it’s actually an older technique that has been used by threat actors since the early 1990s. &nbsp;</p>
<h3><b>Campaign Analysis</b></h3>
<p>The SideWinder APT group’s new campaign leveraging server-side polymorphism to deliver the next stage payload began in late November 2022. The malicious documents used in this campaign were created to target Pakistan government officials. The documents were designed to trick Pakistan officials by displaying convincing content relevant to their interests.</p>
<p>During the investigation, the BlackBerry Threat Research and Intelligence team analyzed the documents used by the threat group to identify various artefacts used in this campaign to potentially locate other files of interest. The first malicious lure we examined was a document titled “GUIDELINES FOR BEACON JOURNAL – 2023 PAKISTAN NAVY WAR COLLEGE (PNWC)”.</p>

<h3><b><br>
Weaponization</b></h3>
<p>The next stage payload “file.rtf”, a rich text document file, can only be downloaded by users in the Pakistani IP range. It is important to note that in both instances, only the name of the file “file.rtf” and the file type are the same; however, the contents, file size and the file hash are different. This is an example of server-based polymorphism, where each time the server responds with a different version of file, so bypassing the victim’s antivirus scanner (presuming the antivirus uses signature-based detection).</p>
<p>If the user is not in the Pakistani IP range, the server returns an 8-byte RTF file (<b>file.rtf</b>) that contains a single string: {\rtf1 }. However, if the user is within the Pakistani IP range, the server then returns the RTF payload, which varies between 406KB – 414KB in size.</p>

<h3><b>Network Infrastructure</b></h3>
<p>SideWinder’s campaign command-and-control (C2) infrastructure is only live for short periods of time. Non-Pakistani IP responses from the systems hosting RTF files have been identical since at least January 2021, with an 8-byte file with {\rtf1 } as the content. Following the relationships in <a href="https://www.virustotal.com/gui/file/1955c6914097477d5141f720c9e8fa44b4fe189e854da298d85090cbc338b35a/relations" target="_blank" title="virustotal distribution infrastructure of sidewinder campaigns">VirusTotal</a> shows the distribution infrastructure and the longevity of similar campaigns. 28 domains have been seen in the wild hosting this empty RTF file, all with similar URLs used for hosting.</p>
<p>For these campaigns, SideWinder also uses predictable URL structures when hosting their malicious files:<br>
</p>
<ul>
<li>First stage - */2/0/0/*/files-*/(hta|file.rtf)</li>
<li>Second stage - */3/1/1/*/files-*/</li>
</ul>
<p>The longevity of these tactics, techniques, and procedures (TTPs)&nbsp;– nearly 2 years – gives us confidence that they can be utilized for the detection of future campaigns.</p>
<p>In mid-March 2023, we discovered a new configured server delivering the payload. This server was different in that it was configured so that a victim in Turkey could receive a second-stage payload. This shows that this threat actor is also now targeting organizations in Turkey.</p>
