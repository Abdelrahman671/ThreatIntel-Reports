			<h1 class="entry-title">Hancitor Continues to Push Cobalt Strike</h1>
		<div class="entry-meta">
			<span class="posted-on"><a href="https://thedfirreport.com/2021/06/28/hancitor-continues-to-push-cobalt-strike/" rel="bookmark"><time class="entry-date published" datetime="2021-06-28T01:28:27+00:00">June 28, 2021</time></a></span>		</div><!-- .entry-meta -->

		
	</header><!-- .entry-header -->

	<div class="entry-content">
		
<blockquote>
<p>First observed in 2014, Hancitor (also known as Chanitor and Tordal) is a downloader trojan that has been used to deliver multiple different malware such as Pony, Vawtrak, and DELoader. <a href="https://digital.nhs.uk/cyber-alerts/2016/cc-678" target="_blank" rel="noopener">[1]</a></p>
</blockquote>
<p>Here&#8217;s some great write ups on Hancitor:</p>
<p>Binary Defense &#8211; <a href="https://www.binarydefense.com/analysis-of-hancitor-when-boring-begets-beacon">Analysis of Hancitor – When Boring Begets Beacon &#8211; Binary Defense</a></p>
<p>Group IB &#8211; <a href="https://blog.group-ib.com/hancitor-cuba-ransomware">Connecting the Bots: Hancitor fuels Cuba Ransomware Operations (group-ib.com)</a></p>
<p>Unit 42 &#8211; <a href="https://unit42.paloaltonetworks.com/hancitor-infections-cobalt-strike/">Recent Hancitor Infections Use Cobalt Strike and a Network Ping Tool (paloaltonetworks.com)</a></p>
<h3>Case Summary</h3>
<p>In this short intrusion, the threat actor gained initial access on a system through a maldoc campaign which made use of the Hancitor downloader. The first-stage DLL, which was dropped by a malicious Word document, attempted to download multiple malware payloads on the beachhead system, including Ficker Stealer. In addition, a Cobalt Strike beacon payload was downloaded, and deployed to perform post-exploitation activities. Once inside the target environment, port scans and a large amount of ICMP traffic was observed&#8211;to identify additional active systems. After about 20 minutes, the threat actor copied a batch script file and DLL file to another system using the SMB protocol. A remote service was installed to start the batch file, which executed the Cobalt Strike shellcode-embedded DLL. On the second compromised system, various discovery-related commands were executed before going silent. The threat actors were evicted before completing their mission.</p>
<h3>Services</h3>
<p>We offer multiple services including a Threat Feed service which tracks Command and Control frameworks such as Cobalt Strike, Metasploit, Empire, PoshC2, etc. More information on this service and others can be found <a href="https://thedfirreport.com/services/" target="_blank" rel="noopener">here</a>. One of the Cobalt Strike servers used in this intrusion was known about as far back as February and the other 2 were added to our Threat Feed on 5/20/21.</p>
<p>We also have artifacts available from this case such as pcaps, memory captures, files, Kape packages, and more, under our <a href="https://www.patreon.com/thedfirreport" target="_blank" rel="noopener">Security Researcher and Organization</a> services.</p>



<h3> <strong>Timeline</strong> </h3>

<h3>MITRE ATT&amp;CK v9</h3>
<h3>Initial Access </h3>



<p>The Hancitor malware was embedded in a macro-based Word document. This single-paged document contained a picture with instructions, attempting to lure the victim into enabling macros.  </p>



<figure class="wp-block-image size-large"><a href="https://thedfirreport.com/wp-content/uploads/2021/06/image-21.png"><img decoding="async" class="alignnone wp-image-3264" src="https://thedfirreport.com/wp-content/uploads/2021/06/image-21.png" alt="" width="996" height="624" srcset="https://thedfirreport.com/wp-content/uploads/2021/06/image-21.png 996w, https://thedfirreport.com/wp-content/uploads/2021/06/image-21-300x188.png 300w, https://thedfirreport.com/wp-content/uploads/2021/06/image-21-768x481.png 768w" sizes="(max-width: 996px) 100vw, 996px" /></a></figure>



<p> <br />When the macro was enabled, the infection chain started, and the first-stage Hancitor DLL was dropped to disk. </p>
<p>Reviewing the macro we can see that in sub yyy (towards the bottom) content within the document is being copied and used to create a file object by sub xxx which then is executed by the shell call to rundll32.</p>

<h3>Command and Control </h3>



<p>In the network traffic, we can identify a data stream pattern that is distinctive to Hancitor malware. </p>



<p><a href="https://thedfirreport.com/wp-content/uploads/2021/06/003_cnc-1.png"><img loading="lazy" decoding="async" class="alignnone size-full wp-image-3312" src="https://thedfirreport.com/wp-content/uploads/2021/06/003_cnc-1.png" alt="" width="1600" height="240" srcset="https://thedfirreport.com/wp-content/uploads/2021/06/003_cnc-1.png 1600w, https://thedfirreport.com/wp-content/uploads/2021/06/003_cnc-1-300x45.png 300w, https://thedfirreport.com/wp-content/uploads/2021/06/003_cnc-1-1024x154.png 1024w, https://thedfirreport.com/wp-content/uploads/2021/06/003_cnc-1-768x115.png 768w, https://thedfirreport.com/wp-content/uploads/2021/06/003_cnc-1-1536x230.png 1536w" sizes="auto, (max-width: 1600px) 100vw, 1600px" /></a></p>
<p>First, the malware performed a lookup of the external IP-address of the infected system (1). This was followed by Hancitor C2 traffic, sent via HTTP POST requests, which included unique attributes of the infected system, such as hostname and username information (2). </p>



<p>Hancitor then attempted to download additional malware.
