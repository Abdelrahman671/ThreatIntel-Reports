</div></div><div class="col-md-9 blog-main"><div class="single-post-content"><p>Researchers at Palo Alto Networks recently published a report regarding the <a href="https://researchcenter.paloaltonetworks.com/2018/09/unit42-new-konni-malware-attacking-eurasia-southeast-asia/" target="_blank" rel="noopener nofollow noreferrer">NOKKI</a> malware, which has shared code with KONNI and, although not in the report by Palo Alto, <a href="https://malware.lu/assets/files/articles/RAP003_KimJongRAT-Stealer_Analysis.1.0.pdf" target="_blank" rel="noopener nofollow noreferrer">KimJongRAT</a> (discovered by <a href="https://twitter.com/r00tbsd" target="_blank" rel="noopener nofollow noreferrer">Paul Rascagnères</a> of Cisco Talos in 2013), and another report on how there is evidence of the <a href="https://researchcenter.paloaltonetworks.com/2018/10/unit42-nokki-almost-ties-the-knot-with-dogcall-reaper-group-uses-new-malware-to-deploy-rat/" target="_blank" rel="noopener nofollow noreferrer">NOKKI malware connecting to the North Korean threat actor</a> known as APT37, Reaper, or Group123.</p>
<blockquote class="twitter-tweet" data-width="550" data-dnt="true">
<p lang="en" dir="ltr">New report by <a href="https://twitter.com/PaloAltoNtwks?ref_src=twsrc%5Etfw">@PaloAltoNtwks</a> on <a href="https://twitter.com/hashtag/NOKKI?src=hash&amp;ref_src=twsrc%5Etfw">#NOKKI</a> malware with similarities to <a href="https://twitter.com/hashtag/KONNI?src=hash&amp;ref_src=twsrc%5Etfw">#KONNI</a> also has code relations to <a href="https://twitter.com/hashtag/KimJongRAT?src=hash&amp;ref_src=twsrc%5Etfw">#KimJongRAT</a> in a report from 2013 by <a href="https://twitter.com/r00tbsd?ref_src=twsrc%5Etfw">@r00tbsd</a> <a href="https://t.co/Rc0T8a4o6h">https://t.co/Rc0T8a4o6h</a> <a href="https://t.co/QkjiaQJ2wF">pic.twitter.com/QkjiaQJ2wF</a></p>
<p>&mdash; Jay Rosenberg (@jaytezer) <a href="https://twitter.com/jaytezer/status/1045670058704478208?ref_src=twsrc%5Etfw">September 28, 2018</a></p></blockquote>
<p><script data-minify="1" async src="https://149520725.v2.pressablecdn.com/wp-content/cache/min/1/widgets.js?ver=1740498858" charset="utf-8"></script></p>
<p>The malicious document related to NOKKI, using VBScript, downloads a newly discovered malware named Final1stspy, due to the PDB string inside. As noted by Palo Alto Networks, Final1stspy comes in 2 components, the EXE named “LoadDll” with the sole purpose of loading up a DLL payload, internally named “hadowexecute.” After collecting information about the infected computer, the end result is that the DOGCALL malware, also known as ROKRAT, is downloaded as the final payload, thus being <strong>one of the links between NOKKI and APT37</strong>.</p>
<h3><strong>LINKS THROUGH CODE REUSE</strong></h3>
<p>The DLL payload component under the hash that was listed in the report was not available on VirusTotal. We created some YARA signatures from the code of “LoadDll” and did a hunt via VirusTotal. Since the EXE component shares code with the DLL, the YARA hunt led us to find an earlier version of the DLL component of Final1stspy with 2/67 detections, compilation timestamp of May 21, 2018, and first upload to VirusTotal date of July 11, 2018. This is an earlier version than described in the Palo Alto report. After obtaining the hadowexecute DLL component, we checked to see if there was any code reuse in the <a href="https://analyze.intezer.com/#/files/2011b9aa61d280ca9397398434af94ec26ddb6ab51f5db269f1799b46cf65a76" target="_blank" rel="noopener noreferrer">Intezer Analyze&#x2122;</a> system.</p>
<p><strong>        </strong></p>
<p><a href="https://149520725.v2.pressablecdn.com//wp-content/uploads/2018/10/analyze_final1stspy_group123.png" data-slb-group="post-images" data-slb-active="1" data-slb-asset="658765743" data-slb-internal="0" data-slb-group="slb"><img fetchpriority="high" decoding="async" class="aligncenter wp-image-2000 size-large" src="data:image/svg+xml,%3Csvg%20xmlns='http://www.w3.org/2000/svg'%20viewBox='0%200%201024%20475'%3E%3C/svg%3E" alt="Intezer Analyze&#x2122; system." width="1024" height="475" data-lazy-src="https://149520725.v2.pressablecdn.com//wp-content/uploads/2018/10/analyze_final1stspy_group123-1024x475.png" /><noscript><img fetchpriority="high" decoding="async" class="aligncenter wp-image-2000 size-large" src="https://149520725.v2.pressablecdn.com//wp-content/uploads/2018/10/analyze_final1stspy_group123-1024x475.png" alt="Intezer Analyze&#x2122; system." width="1024" height="475" /></noscript></a></p>
<p style="text-align: center;"><a href="https://analyze.intezer.com/#/files/2011b9aa61d280ca9397398434af94ec26ddb6ab51f5db269f1799b46cf65a76" target="_blank" rel="noopener noreferrer">Intezer Analyze&#x2122;</a></p>
<p>We see that there is some code shared between the EXE component of Final1stspy and other code that has been seen before in the FreeMilk campaign which utilized ROKRAT.</p>
<p>Let’s see the shared code between ROKRAT and Final1stspy by doing comparison of these functions in IDA. If you take a look, there is an identical match between them.</p>

<p>This function is unique code that has only been seen before in Group123’s ROKRAT and the DLL component of Final1stspy. The identical function gathers information about the operating system and stores it in the same format.</p>
<h3><strong>CONCLUSION</strong></h3>
<p>The evidence in Group123 being the threat actor involved here does not only lie in the final delivered payload, but in the code itself. This code reuse provides more evidence towards the relationship of KimJongRAT, KONNI, NOKKI, Final1stspy, ROKRAT, and APT37.</p>

jQuery(document).ready(function($) {
    var $slider = $('.testimonials-slider');
    var scrollSpeed = 1; // Lower values for slower scrolling
    var isDragging = false;
    var startX = 0, scrollStart = 0;
    var scrollInterval;
    
    if ($slider.length) {
        var $items = $slider.children();
        var scrollWidth = $slider[0].scrollWidth;
        var viewportWidth = $slider[0].clientWidth;

        $slider.append($items.clone()); // Clone items for smooth looping
        $slider.append($items.clone()); // Clone twice to ensure seamless loop

        function scrollLeft() {
            if (isDragging) return; // Stop scrolling while dragging

            var currentPosition = $slider.scrollLeft();
            var newPosition = currentPosition + scrollSpeed;

            // Instead of resetting instantly, adjust position dynamically
            if (newPosition >= scrollWidth) {
                newPosition = newPosition - scrollWidth; // Adjust position smoothly
            }

            $slider.scrollLeft(newPosition);
        }

        // Auto-scroll
        function startScrolling() {
            if (!scrollInterval) {
                scrollInterval = setInterval(scrollLeft, 20);
            }
        }

        function stopScrolling() {
            clearInterval(scrollInterval);
            scrollInterval = null; 
        }

        startScrolling(); // Start auto-scroll on load

        // Mouse & Touch Drag Support
        $slider.on('mousedown touchstart', function(e) {
            stopScrolling(); // Pause while interacting

            startX = e.type === 'touchstart' ? e.originalEvent.touches[0].pageX : e.pageX;
            scrollStart = $slider.scrollLeft();
            isDragging = true;
        });

        $slider.on('mousemove touchmove', function(e) {
            if (!isDragging) return;

            var x = e.type === 'touchmove' ? e.originalEvent.touches[0].pageX : e.pageX;
            var distance = startX - x;
            $slider.scrollLeft(scrollStart + distance);
        });

        $slider.on('mouseup touchend mouseleave', function() {
            isDragging = false;
            startScrolling(); // Resume auto-scroll after release
        });

        // Adjust on window resize
        $(window).on('resize', function() {
            viewportWidth = $slider[0].clientWidth; // Update viewport width dynamically
            scrollWidth = $slider[0].scrollWidth;
        });
    }
});
