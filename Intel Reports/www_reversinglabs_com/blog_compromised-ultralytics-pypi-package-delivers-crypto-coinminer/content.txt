<h2 class="post-header__secondary-head blog__post-secondary-h2">A compromised build environment led to a malicious deployment of a popular AI library that had the potential of delivering other malware.</h2>

<div class="blog__post-content html-content">
<span id="hs_cos_wrapper_post_body" class="hs_cos_wrapper hs_cos_wrapper_meta_field hs_cos_wrapper_type_rich_text" style="" data-hs-cos-general-type="meta_field" data-hs-cos-type="rich_text"><h2><img src="https://www.reversinglabs.com/hs-fs/hubfs/Blog/XMRig-Coinminer.webp?width=1400&amp;height=732&amp;name=XMRig-Coinminer.webp" width="1400" height="732" loading="lazy" alt="XMRig-Coinminer" style="height: auto; max-width: 100%; width: 1400px;" srcset="https://www.reversinglabs.com/hs-fs/hubfs/Blog/XMRig-Coinminer.webp?width=700&amp;height=366&amp;name=XMRig-Coinminer.webp 700w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/XMRig-Coinminer.webp?width=1400&amp;height=732&amp;name=XMRig-Coinminer.webp 1400w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/XMRig-Coinminer.webp?width=2100&amp;height=1098&amp;name=XMRig-Coinminer.webp 2100w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/XMRig-Coinminer.webp?width=2800&amp;height=1464&amp;name=XMRig-Coinminer.webp 2800w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/XMRig-Coinminer.webp?width=3500&amp;height=1830&amp;name=XMRig-Coinminer.webp 3500w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/XMRig-Coinminer.webp?width=4200&amp;height=2196&amp;name=XMRig-Coinminer.webp 4200w" sizes="(max-width: 1400px) 100vw, 1400px"><span style="font-size: 24px; font-weight: normal;">Executive Summary</span></h2>
<p>On December 4, a malicious version 8.3.41 of the popular AI library <em>ultralytics</em> â€” which has almost 60 million downloads â€” was published to the Python Package Index (PyPI) package repository. The package contained downloader code that was downloading the <em>XMRig</em> coinminer. The compromise of the project's build environment was achieved by exploiting a known and previously reported GitHub Actions script injection.</p>
<!--more-->
<h2 style="font-size: 24px; font-weight: normal;">Discussion</h2>
<p>On December 4th, a new version 8.3.41 of a popular AI library <em>ultralytics</em> was released on GitHub and also published to PyPI package repository. Similar to a recent <a href="/blog/malicious-pypi-crypto-pay-package-aiocpa-implants-infostealer-code" rel="noopener">RL research team post </a>regarding the <span style="font-style: italic;">aiocpa</span> incident, the content in the GitHub repository didnâ€™t match the content of the matching PyPI package. Malicious actors managed to compromise the build environment related to the mentioned project and injected the malicious code after the code review part of the process was finished.</p>
<p>What made the <em>ultralytics</em> incident worse: Project maintainers didnâ€™t properly locate the compromise and as a result, on December 5, version 8.3.42, which was supposed to address the incident and serve as a safe version to upgrade from the malicious one, ended up containing the same malicious code. Finally a clean version, 8.3.43, was published on the same day, resolving this supply chain attack.</p>
<p>This supply chain compromise had the potential to impact a huge number of users since <span style="font-style: italic;">ultralytics</span> (see <a href="https://secure.software/pypi/packages/ultralytics" rel="noopener" target="_blank">RL Spectra Assure Community listing</a>) is a GitHub project with more than 30,000 stars â€” and the PyPI package shows about 60 million downloads.&nbsp;</p>
<h2 style="font-weight: normal; font-size: 24px;">Infection vector</h2>
<p>Unlike the <a href="/blog/malware-found-in-solana-npm-library-with-50m-downloads" rel="noopener">recent compromise of a trusted npm package @solana/web3.js</a> (<a href="https://secure.software/npm/packages/@solana/web3.js" rel="noopener" target="_blank">See Spectra Assure Community listing</a>), which also had a similar impact radius but was caused by a compromise of one of the maintainer accounts, in this case intrusion into the build environment was achieved by a more sophisticated vector, by exploiting a known <a href="https://github.com/ultralytics/actions/security/advisories/GHSA-7x29-qqmq-v6qc" rel="noopener" target="_blank">GitHub Actions Script Injection</a> that was previously reported by the security researcher <a href="https://www.linkedin.com/in/adnanekhan/" rel="noopener" target="_blank">Adnan Khan</a>.</p>
<a href="https://blog.reversinglabs.com/hubfs/Blog/ultralytics-blog-fig1.png" class="lightbox-image"><img src="https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig1.png?width=1400&amp;name=ultralytics-blog-fig1.png" alt="Comment on the GitHub issue pages explaining the infection vector" width="1400" style="width: 1400px;" srcset="https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig1.png?width=700&amp;name=ultralytics-blog-fig1.png 700w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig1.png?width=1400&amp;name=ultralytics-blog-fig1.png 1400w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig1.png?width=2100&amp;name=ultralytics-blog-fig1.png 2100w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig1.png?width=2800&amp;name=ultralytics-blog-fig1.png 2800w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig1.png?width=3500&amp;name=ultralytics-blog-fig1.png 3500w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig1.png?width=4200&amp;name=ultralytics-blog-fig1.png 4200w" sizes="(max-width: 1400px) 100vw, 1400px"></a>
<p style="font-weight: bold; font-size: 16px; text-align: center;">Figure 1: Comment on the GitHub issue pages explaining the infection vector</p>
<p>With this GitHub Actions Script injection, a malicious actor can create a fork of any repository that uses <em>ultralytics/actions</em>, and by crafting a pull request from a branch that has injection payload code in its title, he can achieve arbitrary code execution.&nbsp;<br><br>Two maliciously crafted pull requests, <a href="https://github.com/ultralytics/ultralytics/pull/18018" rel="noopener" target="_blank">#18018</a> and <a href="https://github.com/ultralytics/ultralytics/pull/18020" rel="noopener" target="_blank">#18020</a>, were designed to enable backdoor access to the compromised environment. Figure 2 shows the malicious code embedded in the name of the forked branch.</p>
<a href="https://blog.reversinglabs.com/hubfs/Blog/ultralytics-blog-fig2.png" class="lightbox-image"><img src="https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig2.png?width=1400&amp;name=ultralytics-blog-fig2.png" alt="Malicious pull requests designed to trigger execution of the payload code" width="1400" style="width: 1400px;" srcset="https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig2.png?width=700&amp;name=ultralytics-blog-fig2.png 700w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig2.png?width=1400&amp;name=ultralytics-blog-fig2.png 1400w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig2.png?width=2100&amp;name=ultralytics-blog-fig2.png 2100w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig2.png?width=2800&amp;name=ultralytics-blog-fig2.png 2800w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig2.png?width=3500&amp;name=ultralytics-blog-fig2.png 3500w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig2.png?width=4200&amp;name=ultralytics-blog-fig2.png 4200w" sizes="(max-width: 1400px) 100vw, 1400px"></a>
<p style="font-weight: bold; font-size: 16px; text-align: center;">Figure 2: Malicious pull requests designed to trigger execution of the payload code</p>
<p>The user account behind this pull request, openimbot, and the remote connection that was established after the execution of the malicious payload was initiated from Hong Kong, based on information provided by <em>ultralytics</em> maintainers.</p>
<a href="https://blog.reversinglabs.com/hubfs/Blog/ultralytics-blog-fig3.png" class="lightbox-image"><img src="https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig3.png?width=1400&amp;name=ultralytics-blog-fig3.png" alt="User data behind the malicious pull request" width="1400" style="width: 1400px;" srcset="https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig3.png?width=700&amp;name=ultralytics-blog-fig3.png 700w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig3.png?width=1400&amp;name=ultralytics-blog-fig3.png 1400w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig3.png?width=2100&amp;name=ultralytics-blog-fig3.png 2100w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig3.png?width=2800&amp;name=ultralytics-blog-fig3.png 2800w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig3.png?width=3500&amp;name=ultralytics-blog-fig3.png 3500w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig3.png?width=4200&amp;name=ultralytics-blog-fig3.png 4200w" sizes="(max-width: 1400px) 100vw, 1400px"></a>
<p style="font-weight: bold; font-size: 16px; text-align: center;">Figure 3: User data behind the malicious pull request</p>
<p>This GitHub user, <em>openimbot</em>, has an interesting contribution history, with a long period of inactivity, between the end of August and beginning of December, when the reported attack was executed. While researchers could conclude that the account was overtaken sometime in this period, that may not be a correct conclusion.&nbsp;</p>
<a href="https://blog.reversinglabs.com/hubfs/Blog/ultralytics-blog-fig4.png" class="lightbox-image"><img src="https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig4.png?width=1400&amp;name=ultralytics-blog-fig4.png" alt="Contributions history of the openimbot account which initiated the compromise" width="1400" style="width: 1400px;" srcset="https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig4.png?width=700&amp;name=ultralytics-blog-fig4.png 700w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig4.png?width=1400&amp;name=ultralytics-blog-fig4.png 1400w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig4.png?width=2100&amp;name=ultralytics-blog-fig4.png 2100w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig4.png?width=2800&amp;name=ultralytics-blog-fig4.png 2800w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig4.png?width=3500&amp;name=ultralytics-blog-fig4.png 3500w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig4.png?width=4200&amp;name=ultralytics-blog-fig4.png 4200w" sizes="(max-width: 1400px) 100vw, 1400px"></a>
<p style="font-weight: bold; font-size: 16px; text-align: center;">Figure 4: Contributions history of the <em>openimbot</em> account, which initiated the compromise</p>
<p>The RL Spectra Assure platform can help prevent this type of attack and quickly pinpoint the malicious content inserted into the package. Figure 5 shows the file-based behavior differences between non-malicious version 8.3.40 and malicious version 8.3.41.</p>
<a href="https://blog.reversinglabs.com/hubfs/Blog/ultralytics-blog-fig5.png" class="lightbox-image"><img src="https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig5.png?width=1400&amp;name=ultralytics-blog-fig5.png" alt="Behavior differences" width="1400" style="width: 1400px;" srcset="https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig5.png?width=700&amp;name=ultralytics-blog-fig5.png 700w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig5.png?width=1400&amp;name=ultralytics-blog-fig5.png 1400w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig5.png?width=2100&amp;name=ultralytics-blog-fig5.png 2100w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig5.png?width=2800&amp;name=ultralytics-blog-fig5.png 2800w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig5.png?width=3500&amp;name=ultralytics-blog-fig5.png 3500w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig5.png?width=4200&amp;name=ultralytics-blog-fig5.png 4200w" sizes="(max-width: 1400px) 100vw, 1400px"></a>
<p style="font-weight: bold; font-size: 16px; text-align: center;">Figure 5: Behavior differences between non-malicious version 8.3.40 and malicious version 8.3.41</p>
<p>From the behavior diff, security teams can conclude that the malicious code was inserted into files <em>downloads.py </em>and <em>model.py</em>. Figure 6 shows that the code inserted into the <em>model.py</em> file is designed to check the machine type of the system on which the code is executed. This type of behavior is often used by malware to deliver payload specific to the infected machine.&nbsp;</p>
<a href="https://blog.reversinglabs.com/hubfs/Blog/ultralytics-blog-fig6.png" class="lightbox-image"><img src="https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig6.png?width=1400&amp;name=ultralytics-blog-fig6.png" alt="New behavior introduced in model.py file" width="1400" style="width: 1400px;" srcset="https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig6.png?width=700&amp;name=ultralytics-blog-fig6.png 700w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig6.png?width=1400&amp;name=ultralytics-blog-fig6.png 1400w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig6.png?width=2100&amp;name=ultralytics-blog-fig6.png 2100w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig6.png?width=2800&amp;name=ultralytics-blog-fig6.png 2800w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig6.png?width=3500&amp;name=ultralytics-blog-fig6.png 3500w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig6.png?width=4200&amp;name=ultralytics-blog-fig6.png 4200w" sizes="(max-width: 1400px) 100vw, 1400px"></a>
<p style="font-weight: bold; font-size: 16px; text-align: center;">Figure 6: New behavior introduced in model.py file</p>
<p>Inspection of the source code confirms the above assumption. The code designed to download platform-specific payload is visible in Figure 7.</p>
<a href="https://blog.reversinglabs.com/hubfs/Blog/ultralytics-blog-fig7.png" class="lightbox-image"><img src="https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig7.png?width=1400&amp;name=ultralytics-blog-fig7.png" alt="Code responsible for downloading and executing payload" width="1400" style="width: 1400px;" srcset="https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig7.png?width=700&amp;name=ultralytics-blog-fig7.png 700w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig7.png?width=1400&amp;name=ultralytics-blog-fig7.png 1400w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig7.png?width=2100&amp;name=ultralytics-blog-fig7.png 2100w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig7.png?width=2800&amp;name=ultralytics-blog-fig7.png 2800w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig7.png?width=3500&amp;name=ultralytics-blog-fig7.png 3500w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig7.png?width=4200&amp;name=ultralytics-blog-fig7.png 4200w" sizes="(max-width: 1400px) 100vw, 1400px"></a>
<p style="font-weight: bold; font-size: 16px; text-align: center;">Figure 7: Code responsible for downloading and executing payload</p>
<p>File system-related behavior changes visible in Figure 8 suggest that the code that performs the actual downloading of the file and writing it to disk is located in the <em>downloads.py</em> file. The <a href="https://www.diffchecker.com/ZY07xESP/" rel="noopener" target="_blank">code difference</a> between <span style="font-style: italic;">downloads.py</span> file from versions 8.3.40 and 8.3.41 confirm that conclusion.&nbsp;</p>
<a href="https://blog.reversinglabs.com/hubfs/Blog/ultralytics-blog-fig8.png" class="lightbox-image"><img src="https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig8.png?width=1400&amp;name=ultralytics-blog-fig8.png" alt="New behavior introduced in downloads.py file" width="1400" style="width: 1400px;" srcset="https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig8.png?width=700&amp;name=ultralytics-blog-fig8.png 700w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig8.png?width=1400&amp;name=ultralytics-blog-fig8.png 1400w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig8.png?width=2100&amp;name=ultralytics-blog-fig8.png 2100w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig8.png?width=2800&amp;name=ultralytics-blog-fig8.png 2800w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig8.png?width=3500&amp;name=ultralytics-blog-fig8.png 3500w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/ultralytics-blog-fig8.png?width=4200&amp;name=ultralytics-blog-fig8.png 4200w" sizes="(max-width: 1400px) 100vw, 1400px"></a>
<p style="font-weight: bold; font-size: 16px; text-align: center;">Figure 8: New behavior introduced in downloads.py file</p>
<p>While in this case, based on the present information the RL research team has, it seems that the malicious payload served was simply an XMRig miner, and that the malicious functionality was aimed at cryptocurrency mining. But it is not hard to imagine what the potential impact and the damage could be if threat actors decided to plant more aggressive malware like backdoors or remote-access trojans (RATs).</p>
