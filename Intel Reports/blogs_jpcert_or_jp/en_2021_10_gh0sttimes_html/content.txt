<section class="entry-content clearfix">
<p>An attack group BlackTech has been actively conducting attacks against Japanese organisations since 2018. Although it is not as prominent as before, JPCERT/CC is still seeing some cases as of now. This article introduces the details of the malware Gh0stTimes, which is used by this group.</p>
<h3>Gh0stTimes overview</h3>
<p>Gh0stTimes is customised based on Gh0st RAT and has been used in some attack cases since 2020. Figure 1 shows the comparison of Gh0stTimes and Gh0st RAT code.</p>
<p><figure class="mt-figure mt-figure-center"><a class="mt-asset-link" href="https://blogs.jpcert.or.jp/en/.assets/gh0sttimes-fig1.png"><img alt="Comparison of Gh0stTimes and Gh0st RAT code" class="asset asset-image at-xid-2002916 mt-image-center" height="464" src="https://blogs.jpcert.or.jp/en/.assets/thumbnail/gh0sttimes-fig1-640wri.png" style="display: block; margin-left: auto; margin-right: auto;" width="640"/></a><figcaption>Figure 1: Comparison of Gh0stTimes and Gh0st RAT (CFileManager) code<br/>(Left: Gh0stTimes / Right: Gh0st RAT)</figcaption></figure></p>
<p>Both sets of code are functions for file operation, and they are almost identical. Many of the Gh0st RAT functions are upgraded in Gh0stTimes, but some parts of the code are just kept as is. The next sections explain the features of Gh0stTimes.</p>
<ul>
<li>Communication protocol</li>
<li>Commands</li>
<li>Dummy code</li>
<li>C2 server control panel</li>
</ul>
<h3>Communication protocol</h3>
<p>Just like Gh0st RAT, Gh0stTimes communicates with C2 servers with its custom protocol, but the packet format is different. Figure 2 shows the flow of communication.</p>
<p><figure class="mt-figure mt-figure-center"><a class="mt-asset-link" href="https://blogs.jpcert.or.jp/en/.assets/gh0sttimes-fig2.png"><img alt="Gh0stTimes communication flow" class="asset asset-image at-xid-2002917 mt-image-center" height="242" src="https://blogs.jpcert.or.jp/en/.assets/thumbnail/gh0sttimes-fig2-640wri.png" style="display: block; margin-left: auto; margin-right: auto;" width="640"/></a><figcaption>Figure 2: Gh0stTimes communication flow</figcaption></figure></p>
<p>At the beginning of its communication with a C2 server, Gh0stTimes sends an authentication ID and data (The "Key" in Figure 2) to generate an encryption key for the following communication. The C2 server checks the authentication ID and only accepts the communication with certain IDs. Figure 3 shows an example of the specific authentication IDs.</p>
<p><figure class="mt-figure mt-figure-center"><a class="mt-asset-link" href="https://blogs.jpcert.or.jp/en/.assets/gh0sttimes-fig3.png"><img alt="Gh0stTimes authentication ID sample" class="asset asset-image at-xid-2002918 mt-image-center" height="380" src="https://blogs.jpcert.or.jp/en/.assets/thumbnail/gh0sttimes-fig3-640wri.png" style="display: block; margin-left: auto; margin-right: auto;" width="640"/></a><figcaption>Figure 3: Gh0stTimes authentication ID sample</figcaption></figure></p>
<p>After the successful authentication, the communication that follows is encrypted with the key provided at the beginning of the communication. The next round of communication includes the information of infected hosts, such as hostname, username and processor name (Figure 4).</p>
<p><figure class="mt-figure mt-figure-center"><a class="mt-asset-link" href="https://blogs.jpcert.or.jp/en/.assets/gh0sttimes-fig4.png"><img alt="Information of infected hosts sent by Gh0stTimes" class="asset asset-image at-xid-2002919 mt-image-center" height="386" src="https://blogs.jpcert.or.jp/en/.assets/thumbnail/gh0sttimes-fig4-640wri.png" style="display: block; margin-left: auto; margin-right: auto;" width="640"/></a><figcaption>Figure 4: Information of infected hosts sent by Gh0stTimes</figcaption></figure></p>
<p>After sending the information of infected hosts, commands are exchanged. See Appendix A for the format of data exchanged. When exchanging commands, the data is RC4-encrypted and then zlib-compressed. Gh0stTimes uses its custom RC4 algorithm, which has XOR 0xAC process over the encrypted data.</p>
<p><figure class="mt-figure mt-figure-center"><a class="mt-asset-link" href="https://blogs.jpcert.or.jp/en/.assets/gh0sttimes-fig5.png"><img alt=" Part of Gh0stTimes code to encrypt data with RC4" class="asset asset-image at-xid-2002920 mt-image-center" height="172" src="https://blogs.jpcert.or.jp/en/.assets/thumbnail/gh0sttimes-fig5-640wri.png" style="display: block; margin-left: auto; margin-right: auto;" width="640"/></a><figcaption>Figure 5: Part of Gh0stTimes code to encrypt data with RC4</figcaption></figure></p>

<h3>Commands</h3>
<p>Gh0stTimes is equipped with the following 5 types of commands:</p>
<ul>
<li>FileManager (command number 0x1): File operation</li>
<li>ShellManager (command number 0x28): Remote shell execution</li>
<li>PortmapManager (command number 0x32): C2 server redirect function</li>
<li>UltraPortmapManager (command number 0x3F): Proxy function</li>
<li>No name (command number 0): End communication</li>
</ul>
<p><figure class="mt-figure mt-figure-center"><a class="mt-asset-link" href="https://blogs.jpcert.or.jp/en/.assets/gh0sttimes-fig6.png"><img alt="List of commands" class="asset asset-image at-xid-2002921 mt-image-center" height="299" src="https://blogs.jpcert.or.jp/en/.assets/thumbnail/gh0sttimes-fig6-640wri.png" style="display: block; margin-left: auto; margin-right: auto;" width="640"/></a><figcaption>Figure 6: List of commands</figcaption></figure></p>
<p>ShellManager and FileManager are the same as Gh0st RAT's original functions. FileManager has multiple functions to operate files on infected hosts. (See Appendix B for details.) <br/>
PortmapManager and UltraPortmapManager are unique to Gh0stTimes, which indicates that its relay function has been enhanced compared to Gh0st RAT.</p>
<h3>Dummy code</h3>
<p>Some types of malware that BlackTech use contains dummy code, which may make analysis difficult. Gh0stTimes has such code (Figure 7), but it does not have much impact to the analysis.</p>
<p><figure class="mt-figure mt-figure-center"><a class="mt-asset-link" href="https://blogs.jpcert.or.jp/en/.assets/gh0sttimes-fig7.png"><img alt="Gh0stTimes dummy code sample" class="asset asset-image at-xid-2002922 mt-image-center" height="439" src="https://blogs.jpcert.or.jp/en/.assets/thumbnail/gh0sttimes-fig7-640wri.png" style="display: block; margin-left: auto; margin-right: auto;" width="640"/></a><figcaption>Figure 7: Gh0stTimes dummy code sample</figcaption></figure></p>
