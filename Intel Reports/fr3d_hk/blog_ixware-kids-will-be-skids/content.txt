			<h1 class="title">IXWare - Kids will be skids</h1>
		</a>

				<!-- Creation date -->
		<h6 class="card-subtitle mb-3 text-muted">October 6, 2020 - Reading time: 17 minutes</h6>
		
		<!-- Full content -->
		<p>IXWare is what happens when inexperienced malware developers create malware as a service. We'll be analysing IXWare and how it's used to attack players of the online video game Roblox.</p>
<p><!-- pagebreak --></p>
<div id="page_content">
<h2 style="box-sizing: border-box; margin-top: 0px; margin-bottom: 0.5rem; font-weight: 500; line-height: 1.2; font-size: 2rem; color: #212529; font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, 'Noto Sans', sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol', 'Noto Color Emoji'; background-color: #ffffff;">Foreword</h2>
<hr>
<p>After releasing my last blog post covering DiamondFox I have been very busy behind the scenes working on my threat tracker (<a href="http://threatshare.io/">threatshare.io</a>) and my personal site (<a href="https://fr3d.hk/">fr3d.hk</a>). A huge thank you as always to <a href="https://twitter.com/steved3">Steved3 </a>for reviewing and editing this post. The threat landscape has mostly stayed the same and not many things have caught my eye until this piece of malware.</p>
<h2 style="box-sizing: border-box; margin-top: 0px; margin-bottom: 0.5rem; font-weight: 500; line-height: 1.2; font-size: 2rem; color: #212529; font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, 'Noto Sans', sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol', 'Noto Color Emoji'; background-color: #ffffff;">Overview</h2>
<hr style="color: #000000; font-family: sans-serif; font-size: 16px;">
<p style="color: #000000; font-family: sans-serif; font-size: 16px;">IXWare is a MaaS (Malware as a service) that offers the ability to steal Windows users' passwords. It became widespread after they moved to a central website handling all of their users' needs. The malware seems to be appealing to actors attacking the video game Roblox as it offers many techniques to recover infected victims' accounts.</p>
<p style="color: #000000; font-family: sans-serif; font-size: 16px;"><img src="https://fr3d.hk/blog/bl-content/uploads/pages/3fac8bd212e22425b8a39924d46a7e41/maxresdefault.jpg" alt=""></p>
<p style="color: #000000; font-family: sans-serif; font-size: 16px;">IXWare is sold on a Roblox hacking forum that caters to reselling of stolen accounts. The malware sells for 10 euros a month and 25 euros for 3 months. On purchase, you're given access to the webpanel. In the below screenshot you will see pricing and features of the malware.</p>
<p style="color: #000000; font-family: sans-serif; font-size: 16px;"><img src="https://fr3d.hk/blog/bl-content/uploads/pages/3fac8bd212e22425b8a39924d46a7e41/mockup1.png" alt="" width="873" height="496">Purchase Page &amp; Features</p>
<p>Below is a list of features (some don't exist/work):</p>
<ul>
<li>Core Features</li>
<li>Webpanel Builder</li>
<li>Webpanel Delivery</li>
<li>Stable &amp; Reliable</li>
<li>Clean Log Interface</li>
<li>Mutex (Single Instance Application)</li>
<li>UAC Bypass (Windows 7, 8.1 &amp; 10)</li>
<li>Critical System-Process (BSoD when killed)</li>
<li>Melt File</li>
<li>Anti Debug/Anti VM/Sandbox</li>
<li>Process Blocker</li>
<li>Website Blocker</li>
<li>Screenshot Logger</li>
<li>Protected Stub</li>
<li>Advanced Keylogger</li>
<li>Clipboard Logging</li>
<li>Window Logging</li>
<li>Browser Password Recoveries</li>
</ul>
<p>Here's a video of the creator demoing another feature of a JavaScript cookie logger for the Roblox game.</p>
<div style="width: 100%; height: 0px; position: relative; padding-bottom: 56.250%;"><iframe src="https://streamable.com/e/3icqt9" frameborder="0" width="100%" height="100%" allowfullscreen="" style="width: 100%; height: 100%; position: absolute; left: 0px; top: 0px; overflow: hidden;"></iframe></div>
<p>The creators of the malware also provide reviews for their product on TrustPilot. Along with the reviews they also offer a TOS that they believe will protect them from law enforcement by stating that the malware is only to be used on a computer you own etc. This TOS will not do anything for them as the main creator <em>icorexÂ </em>or <em>ghostelutsch</em> is based in Germany</p>
<p>IXWare is written in C# and is dependent on the .NET 4.0 framework. It is easy to find samples in the wild that aren't obfuscated.</p>
<h2 style="box-sizing: border-box; margin-top: 0px; margin-bottom: 0.5rem; font-weight: 500; line-height: 1.2; font-size: 2rem; color: #212529; font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, 'Noto Sans', sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol', 'Noto Color Emoji'; background-color: #ffffff;">Anti-VM</h2>
<hr style="color: #000000; font-family: sans-serif; font-size: 16px;">
<p style="color: #000000; font-family: sans-serif; font-size: 16px;">Malware will employ different tactics to avoid being run in virtual machines (VM). This is usually done to evade analysis. The malware begins by getting the running services and checking their names to see if it's running within a VM.Â </p>
<p style="color: #000000; font-family: sans-serif; font-size: 16px;"><img src="https://fr3d.hk/blog/bl-content/uploads/pages/3fac8bd212e22425b8a39924d46a7e41/antivmservices.png" alt="">Anti-VM - Services</p>
<p style="color: #000000; font-family: sans-serif; font-size: 16px;">If anti-VM is enabled within the malware config, we'll see a new task being created named <em>CheckVM.Â </em>This task calls another function named <em>DetectVM.</em>Â <em>DetectVM</em> uses the classic anti-VM method of creating a management object, and then checking the specifics of the system to see whether they match. Here are the following checks.</p>
<p><img src="https://fr3d.hk/blog/bl-content/uploads/pages/3fac8bd212e22425b8a39924d46a7e41/antivmchecks.png" alt="">Anti-VM - Checks</p>
<p>If any of these checks succeed then the malware will exit.</p>
<h2 style="box-sizing: border-box; margin-top: 0px; margin-bottom: 0.5rem; font-weight: 500; line-height: 1.2; font-size: 2rem; color: #212529; font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, 'Noto Sans', sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol', 'Noto Color Emoji'; background-color: #ffffff;">UAC Bypass</h2>
<hr style="color: #000000; font-family: sans-serif; font-size: 16px;">
<p style="color: #000000; font-family: sans-serif; font-size: 16px;">If the malware has not exited because of anti-VM checks, it will proceed to check if it is running as an elevated process by calling <em>IsElevated.Â </em>If the process isn't elevated then the malware will proceed to attempt a few different UAC bypasses. The malware is able to attempt a UAC bypass on the following windows versions.</p>
<ul>
<li style="color: #000000; font-family: sans-serif; font-size: 16px;">Windows 7 (CompMgmtLauncher)</li>
<li style="color: #000000; font-family: sans-serif; font-size: 16px;">Windows Vista (CompMgmtLauncher)</li>
<li style="color: #000000; font-family: sans-serif; font-size: 16px;">Windows 8 (CompMgmtLauncher)</li>
<li style="color: #000000; font-family: sans-serif; font-size: 16px;">Windows 10 (fodhelper)</li>
</ul>
<p>The bypass methods are all very similar, a simple Google search will give you more than enough information on how these exploits work.</p>
<h2 style="box-sizing: border-box; margin-top: 0px; margin-bottom: 0.5rem; font-weight: 500; line-height: 1.2; font-size: 2rem; color: #212529; font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, 'Noto Sans', sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol', 'Noto Color Emoji'; background-color: #ffffff;">Startup &amp; Melt</h2>
<hr style="color: #000000; font-family: sans-serif; font-size: 16px;">
<p style="color: #000000; font-family: sans-serif; font-size: 16px;">To maintain persistence the malware will make sure that it is run at startup, and disappear when the malware is first run. If the melting functionality is enabled, the malware copies itself to the temp path with a random file name, and sets the file attributes to hidden. Once this has been completed the malware begins setting up its persistence.</p>
<p style="color: #000000; font-family: sans-serif; font-size: 16px;"><img src="https://fr3d.hk/blog/bl-content/uploads/pages/3fac8bd212e22425b8a39924d46a7e41/startup.png" alt="">Startup</p>
<p style="color: #000000; font-family: sans-serif; font-size: 16px;">We see the malware setting a registry value in <i>SOFTWARE\\Microsoft\\Windows\\CurrentVersion\\Run</i> set to the location of the malware. If the malware has the melting functionality enabled within its configuration, it will create a <em>schtasksÂ </em>process with the following argumentsÂ </p>
<p style="color: #000000; font-family: sans-serif; font-size: 16px;"><i>/create /tn "svchost" /sc ONLOGON /tr filename /rl HIGHEST /f</i></p>
<p style="color: #000000; font-family: sans-serif; font-size: 16px;">This task will maintain persistence upon the computer logging on.</p>
<h2 style="box-sizing: border-box; margin-top: 0px; margin-bottom: 0.5rem; font-weight: 500; line-height: 1.2; font-size: 2rem; color: #212529; font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, 'Noto Sans', sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol', 'Noto Color Emoji'; background-color: #ffffff;">Persistence</h2>
<hr>
<p>IXWare offers persistence by making the malware a critical process that will blue screen the computer if ended. This is done by first checking if persistence is enabled within the malware config and then calling the <em>criticalProcess</em> function. The <em>criticalProcess </em>function will useÂ <em>NtSetInformationProcess</em> to mark the malware process as critical.Â </p>
<p><img src="https://fr3d.hk/blog/bl-content/uploads/pages/3fac8bd212e22425b8a39924d46a7e41/criticalprocess.png" alt="">criticalProcess</p>
<p><em>NtSetInformationProcess</em> will use a handle on the current process along with the <em>processInformationClass</em> parameter set to 29 which indicatesÂ <em>BreakOnTermination.Â </em>Along withÂ <em>BreakOnTermination</em> another parameter is used namedÂ <em>numÂ </em>which sets the process to critical when it is 1. In addition to doing this there is also an event called <em>SessionEnd </em>which does the same process described above except that <em>num</em> is set to 0 so that the process is no longer critical.Â </p>
<h2 style="box-sizing: border-box; margin-top: 0px; margin-bottom: 0.5rem; font-weight: 500; line-height: 1.2; font-size: 2rem; color: #212529; font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, 'Noto Sans', sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol', 'Noto Color Emoji'; background-color: #ffffff;">Disable Windows Defender</h2>
<hr>
<p>To avoid detection by the in built anti-virus IXWare will attempt to disable it. The function that handles this will check if the current user is an administrator on the system and if not it will return. If the user is an admin then the malware begins by editing some registry keys to disable Windows Defender.</p>
<p><img src="https://fr3d.hk/blog/bl-content/uploads/pages/3fac8bd212e22425b8a39924d46a7e41/disablewd.png" alt="">Disable Windows Defender</p>
<p>After these registry values have been set the malware calls a function namedÂ <em>CheckDefender.Â </em>This function uses powershell to check the settings of Windows Defender. Then going through each preference it will update it with a new setting that will disable any enabled functionality. The code for this is long so I have pasted it <a href="https://pastebin.com/raw/Vq7wi4Ft">here</a>.Â </p>
<p><img src="https://fr3d.hk/blog/bl-content/uploads/pages/3fac8bd212e22425b8a39924d46a7e41/checkwd.png" alt="">Further Disabling</p>
<p>After this function returns the malware finishes off by updating many different registry keys to maintain the disabled Windows Defender.Â </p>
<p><img src="https://fr3d.hk/blog/bl-content/uploads/pages/3fac8bd212e22425b8a39924d46a7e41/wdreg.png" alt="">More disabling</p>
<p>Once these registry values have been updated the malware will continue with its functionality.</p>
<h2 style="box-sizing: border-box; margin-top: 0px; margin-bottom: 0.5rem; font-weight: 500; line-height: 1.2; font-size: 2rem; color: #212529; font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, 'Noto Sans', sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol', 'Noto Color Emoji'; background-color: #ffffff;">Recoveries</h2>
<hr>
<p>The main functionality of the malware is to steal browser and application passwords. IXWare concentrates on chromium browsers as they are easy to steal from, along with supporting theft from Discord. The stealing functionality is quite lacking even though it is one of the main features of the malware. To steal from chromium browsers the malware will begin by enumerating through all supported browsers.</p>
<p><img src="https://fr3d.hk/blog/bl-content/uploads/pages/3fac8bd212e22425b8a39924d46a7e41/chromiumgrab.png" alt="">Chromium Browsers</p>
<p>Then going through each of the supported browsers the malware will attempt to get any accounts stored in the browser profile. Browsers store their login information within an SQLite file, the contents in this database are encrypted using AES with a key that is saved in theÂ <em>local state</em> folder. We can see IXWare retrieving this key using regex.</p>
<p><img src="https://fr3d.hk/blog/bl-content/uploads/pages/3fac8bd212e22425b8a39924d46a7e41/masterykey.png" alt="">Getting Key</p>
<p>Once it has the key it will useÂ <em>CryptoAPI</em> to unprotect it so that it can be used to decrypt login credentials. The malware will then enumerate theÂ <em>logins</em> table and proceed to get url, username and a decrypted password. These passwords are then written into a text file with a filename of a random string of characters in the temp path.Â The malware also allows for the user to steal Discord accounts by grabbing the discord token files within the app data path.</p>
<h2 style="box-sizing: border-box; margin-top: 0px; margin-bottom: 0.5rem; font-weight: 500; line-height: 1.2; font-size: 2rem; color: #212529; font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, 'Noto Sans', sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol', 'Noto Color Emoji'; background-color: #ffffff;">Cookie Stealer</h2>
<hr style="color: #000000; font-family: sans-serif; font-size: 16px;">
<p style="color: #000000; font-family: sans-serif; font-size: 16px;">To steal cookies for the game Roblox the malware employs a very simple tactic. It enumerates the running processes and looks for a process with the name "RobloxPlayerBeta", and gets the command line arguments used to run this process and from this will grab the auth token.</p>
<p style="color: #000000; font-family: sans-serif; font-size: 16px;"><img src="https://fr3d.hk/blog/bl-content/uploads/pages/3fac8bd212e22425b8a39924d46a7e41/roblox.png" alt="">Get Roblox Cookie</p>
<p style="color: #000000; font-family: sans-serif; font-size: 16px;">After the auth token has been stolen from the running process, the malware sends this auth token to the C2 so it can be manipulated into a usable cookie. After the C2 has replied with the desired cookie, the malware will writes into the temp path with a random filename.</p>
<h2 style="box-sizing: border-box; margin-top: 0px; margin-bottom: 0.5rem; font-weight: 500; line-height: 1.2; font-size: 2rem; color: #212529; font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, 'Noto Sans', sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol', 'Noto Color Emoji'; background-color: #ffffff;">C2 Communications</h2>
<hr>
<p>Due to the fact that the malware is a MaaS and all customers will be using the same webpanel, the communications within different samples are quite similar. The C2 communications are done in one POST request to the C2 gate. Specifics of the infected system are sent in the request headers.</p>
<p><img src="https://fr3d.hk/blog/bl-content/uploads/pages/3fac8bd212e22425b8a39924d46a7e41/c2headers.png" alt="">C2 Headers</p>
<p>This begins with the malware requesting another file on the C2 namedÂ <em>UT.php,Â </em>this file returns a unix timestamp. This timestamp is then combined with the number "877692". Once this has been done the string is SHA265 hashed and encrypted. Each header value is encrypted with the function <em>encrypt,</em> this function uses AES (CBC) to encrypt all strings with the first 32 bytes of a constant key. Once done, the malware gets the contents of the password logs, and puts this along with the headers into a multipart form that is sent to the <em>upload.php </em>file on the C2.Â </p>
<h2 style="box-sizing: border-box; margin-top: 0px; margin-bottom: 0.5rem; font-weight: 500; line-height: 1.2; font-size: 2rem; color: #212529; font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, 'Noto Sans', sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol', 'Noto Color Emoji'; background-color: #ffffff;">Epilogue</h2>
<hr>
<p>This malware is what happens when code is copied and pasted with no innovation. Most of what I have described in this post can probably be found on GitHub and simple YARA rules will be able to pick up this malware easily. I have also not chosen to describe the flaws in this malware as it is still actively in development. I hope this post will be of use and as always thank you for reading!</p>
