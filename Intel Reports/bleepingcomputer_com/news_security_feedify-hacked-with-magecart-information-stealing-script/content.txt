          <h1>Feedify Hacked with Magecart Information Stealing Script</h1>
          <div class="cz-news-story-title-section">
            <div class="cz-news-title-left-area">
                            By <h6><a rel="author" href="https://www.bleepingcomputer.com/author/lawrence-abrams/" class="author"><span itemprop="author" itemscope itemtype="https://schema.org/Person"><span itemprop="name">Lawrence Abrams</span></span></a></h6>
            </div>
	    <meta itemprop="articleSection" content="Security"><div class="cz-news-title-right-area">
              <ul><li class="cz-news-date">September 12, 2018</li>
                <li class="cz-news-time">12:51 PM</li>
                
                <li class="cz-news-comment"><a href="#comment_form">0</a></li>
              </ul></div>
          </div>
	  <div class="articleBody">
	   
<p style="text-align:center"><img alt="" height="511" src="https://www.bleepstatic.com/images/news/security/f/feedify/magecart/eye.jpg" width="1267"></p>

<p>A script used by the customer engagement service Feedify has been hacked to include the malicious MageCart script. MageCart is malicious code used by attackers to steal&nbsp;credit card details&nbsp;and other information from e-commerce sites when a user submits a form.</p>

<p>In order to use the Feedify service, e-commerce sites need to add a Feedify&nbsp;JavaScript script to their site. If the Feedify&nbsp;script is compromised with MageCart, any visitors who go to e-commerce site that uses the Feedify&nbsp;script will also load the malicious code.</p>

<p>This hack was first noticed by a security researcher named Placebo who posted about it yesterday on Twitter. When Placebo posted about it, MageCart had already been removed from the Feedify&nbsp;script.</p>

<blockquote class="twitter-tweet" align="center" data-lang="en">
<p dir="ltr" lang="en">MageCart on Feedify. A customer engagement tool. According to there website 4000+ website use there tooling/code. Fixed today after I notified them.<a href="https://twitter.com/ydklijnsma?ref_src=twsrc%5Etfw" rel="nofollow noopener">@ydklijnsma</a> <a href="https://twitter.com/GossiTheDog?ref_src=twsrc%5Etfw" rel="nofollow noopener">@GossiTheDog</a> <a href="https://t.co/K2czXkUoHD" rel="nofollow noopener">pic.twitter.com/K2czXkUoHD</a></p>
&mdash; Placebo (@Placebo52510486) <a href="https://twitter.com/Placebo52510486/status/1039585013057118209?ref_src=twsrc%5Etfw" rel="nofollow noopener">September 11, 2018</a></blockquote>

<p>When researching this story, I created a Feedify account to test what scripts their customers were being instructed to add. When testing the service, customers are instructed to add the following snippet of code to their site.</p>

<div style="text-align:center">
<figure class="image" style="display:inline-block"><img alt="" height="209" src="https://www.bleepstatic.com/images/news/security/f/feedify/magecart/feedify-script-to-add.jpg" width="633"><figcaption>Caption</figcaption></figure></div>

<p>When examining the&nbsp;https://cdn.feedify.net/getjs/feedbackembad-min-1.0.js script, though, I saw that MageCart was still embedded in the script as shown by the highlighted section below.</p>

<div style="text-align:center">
<figure class="image" style="display:inline-block"><img alt="" height="719" src="https://www.bleepstatic.com/images/news/security/f/feedify/magecart/compromised-feedify-script-2.jpg" width="1290"><figcaption>Caption</figcaption></figure></div>

<p>A partial deobfuscation of the script shows that any submitted information will be sent to the URL https://info-stat.ws/js/slider.js.</p>

<div style="text-align:center">
<figure class="image" style="display:inline-block"><img alt="" height="981" src="data:image/gif;base64,R0lGODlhAQABAAAAACH5BAEKAAEALAAAAAABAAEAAAICTAEAOw==" width="1479" data-src="https://www.bleepstatic.com/images/news/security/f/feedify/magecart/partially-deobfuscated.jpg" class="b-lazy"><figcaption>Caption</figcaption></figure></div>

<p>To confirm that this was indeed MageCart, I contacted&nbsp;<a href="https://twitter.com/ydklijnsma" rel="nofollow noopener">Yonathan Klijnsma</a>&nbsp;of RiskIQ who further confirmed that the Feedify script was still compromised.&nbsp;Klijnsma&nbsp;told BleepingComputer that the script had been reinfected 15 minutes prior to my contacting him.&nbsp;</p>

<blockquote class="twitter-tweet" align="center" data-lang="en">
<p dir="ltr" lang="en">FYI: Feedify is re-infected with MageCart since about an hour ago, exact time of infection is: Wed, 12 Sep 2018 14:16:02 GMT.<br><br>
URL: hxxps://cdn[.]feedify[.]net/getjs/feedbackembad-min-1.0.js<br><br>
/cc <a href="https://twitter.com/Placebo52510486?ref_src=twsrc%5Etfw" rel="nofollow noopener">@Placebo52510486</a> <a href="https://twitter.com/GossiTheDog?ref_src=twsrc%5Etfw" rel="nofollow noopener">@GossiTheDog</a> <a href="https://twitter.com/_Feedify?ref_src=twsrc%5Etfw" rel="nofollow noopener">@_feedify</a> <a href="https://t.co/4DtpP3l0Wd" rel="nofollow noopener">https://t.co/4DtpP3l0Wd</a></p>
&mdash; Yonathan Klijnsma (@ydklijnsma) <a href="https://twitter.com/ydklijnsma/status/1039896950223429632?ref_src=twsrc%5Etfw" rel="nofollow noopener">September 12, 2018</a></blockquote>

<p>Currently the malicious code has been removed from the&nbsp;https://feedify.net/getjs/feedbackembad-min-1.0.js, but it is still present in&nbsp;https://cdn.feedify.net/getjs/feedbackembad-min-1.0.js.</p>

<p>BleepingComputer&nbsp;has contacted Feedify&nbsp;for further information, but has not received a response at the time of this publication.</p>

<h3>MageCart&nbsp;used in recent British Airways hack</h3>

<p>RiskIQ&nbsp;also discovered that a&nbsp;script used by <a href="https://www.bleepingcomputer.com/news/security/british-airways-fell-victim-to-card-scraping-attack/" target="_blank" rel="nofollow noopener">British Airways was also recently compromised</a>&nbsp;by the MageCart&nbsp;script. This allowed attackers to steal payment and other sensitive information from approximately 380,000 individuals.</p>

<p style="text-align:center"><img alt="" height="977" src="data:image/gif;base64,R0lGODlhAQABAAAAACH5BAEKAAEALAAAAAABAAEAAAICTAEAOw==" width="1600" data-src="https://www.bleepstatic.com/images/news/u/1100723/Modernizr_malicious_tag.png" class="b-lazy"></p>

<p>In the British Airways hack, the compromised script was the Modernizr JavaScript library, which airline's site was using.</p>

function cz_strip_tags(input, allowed) {
  allowed = (((allowed || '') + '')
    .toLowerCase()
    .match(/<[a-z][a-z0-9]*>/g) || [])
    .join(''); // making sure the allowed arg is a string containing only tags in lowercase (<a><b><c>)
  var tags = /<\/?([a-z][a-z0-9]*)\b[^>]*>/gi,
    commentsAndPhpTags = /<!--[\s\S]*?-->|<\?(?:php)?[\s\S]*?\?>/gi;
  return input.replace(commentsAndPhpTags, '')
    .replace(tags, function($0, $1) {
      return allowed.indexOf('<' + $1.toLowerCase() + '>') > -1 ? $0 : '';
    });
}
function cz_br2nl(str) {
	var regex = /<br\s*[\/]?>/gi;
    //var pure_str = str.replace(regex,"\n");
    var pure_str = str.replace(regex,"");
    return cz_strip_tags(pure_str,'');
}
