<h2 class="wp-block-heading" id="h-introduction">Introduction</h2>



<p>On 26 August 2023, cybersecurity researcher Randy McEoin published<sup data-fn="f35ef7ee-4c65-469f-b3bb-9606f17193e5" class="fn"><a href="#f35ef7ee-4c65-469f-b3bb-9606f17193e5" id="f35ef7ee-4c65-469f-b3bb-9606f17193e5-link">1</a></sup> an analysis of a <strong>new malicious JavaScript framework deployed on compromised websites to deliver further malware using the drive-by download technique</strong>. The newly discovered malware was named ClearFake due to the clear text JavaScript injected into the compromised website, which was not obfuscated in the early version as is usually the case for Javascript malware.</p>



<p>ClearFake is <strong>another “fake updates” threat leveraging social engineering to trick the user into running a fake web browser update</strong>, as for SocGholish and FakeSG malware. By linking the “fake updates” lure to the <strong>watering hole technique</strong>, ClearFake operators target a wide range of users and conduct effective, scalable malware distribution campaigns.</p>



<p>From our telemetry and customers’ feedback, we observed an increasing number of communications to ClearFake infrastructure at the end of September 2023. At the same time, we identified several hundred websites injected by ClearFake.</p>



<p>Sekoia.io’s Threat &amp; Detection Research (TDR) team investigated this emerging threat and shares in this blog post our <strong>analysis of ClearFake, the malware delivered, as well as tracking opportunities</strong>.</p>

<h2 class="wp-block-heading" id="h-malware-delivered-by-clearfake">Malware delivered by ClearFake</h2>



<p>On 30 September 2023, Sekoia.io analysts ran the infection chain until retrieving the final payload downloaded by the victim.</p>



<h3 class="wp-block-heading" id="h-suspicious-filename">Suspicious filename</h3>



<p>For Microsoft Edge’s visitors, ClearFake delivered a malicious Windows Application Packaging Project (APPX file) from Dropbox.</p>



<p>The payload’s name “<em>MlсrоsоftЕdgеSеtup.appx</em>” is a masquerading of the legitimate Microsoft Edge installer and uses UTF-8 Cyrillic character for the characters “c”, “e”, “o” and “E”. Escaping unicode characters returns the following result:</p>



<pre class="wp-block-code"><code>Ml\u0441r\u043Es\u043Eft\u0415dg\u0435S\u0435tup.appx</code></pre>



<figure class="wp-block-table aligncenter"><table><tbody><tr><td><strong>ASCII</strong></td><td><strong>Unicode</strong></td><td><strong>Name</strong></td></tr><tr><td><em>с</em></td><td>\u0441</td><td>Cyrillic Small Letter Es</td></tr><tr><td>e</td><td>\u0435</td><td>Cyrillic Small Letter Ie</td></tr><tr><td><em>о</em></td><td>\u043E</td><td>Cyrillic Small Letter O</td></tr><tr><td><em>Е</em></td><td>\u0415</td><td>Cyrillic Capital Letter Ie</td></tr></tbody></table></figure>



<p>Cyrillic characters are invisible to the user. Sekoia.io assess with high confidence that the <strong>use of lookalike characters aims at avoiding static detection patterns</strong> based on the filename, without raising the potential victim’s suspicions.</p>



<p>It is interesting to note that <strong>SocGholish operators successfully leveraged this technique in 2022</strong>, as identified by Red Canary<sup data-fn="4ca6b6c8-0d7e-4f62-8b46-5899975d59dc" class="fn"><a href="#4ca6b6c8-0d7e-4f62-8b46-5899975d59dc" id="4ca6b6c8-0d7e-4f62-8b46-5899975d59dc-link">3</a></sup>. As this obfuscation method is not widely used, it is legitimate to ask ourselves if the SocGholish operators are also behind the new ClearFake malware.</p>



<h3 class="wp-block-heading" id="h-appx-file">APPX file</h3>



<p>Windows Apps are ZIP archive files that store executable files and other additional ones including XML (<em>AppxManifest.xml </em>and <em>AppxBlockMap.xml</em>), P7X (<em>AppxSignature.p7x</em>), as well as other optional files and repositories.<br></p>



<p>The APPX file delivered by ClearFake (MD5: <em>a7900cdbb2912d76aa6329c5c41d8609</em>) is signed by “<em>STECH CONSULTANCY LIMITED</em>” and contains in particular the following executables:</p>



<ul class="wp-block-list">
<li><span style="background-color:#FAFAFA;border-radius:3px;padding:3px;font-size:.9em;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">\MlсrоsоftЕdgеSеtup\AI_STUBS\AiStubX64.exe</span> (MD5: <em>e89f448e8f41a590c51d34948bdc9c1e)</em></li>



<li><span style="background-color:#FAFAFA;border-radius:3px;padding:3px;font-size:.9em;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">\MlсrоsоftЕdgеSеtup\VFS\AppData\.exe</span> (MD5: <em>d113b3debc7e0a2da4369dd8d1dbad53</em>)</li>
</ul>



<p>Once executed, the Windows App reads the APPX manifest’s entry point containing the <em>AiStubX64 </em>executable and then executes it. The<em> AiStubX64.exe </em>process <strong>copies the <em>KSPSService </em>executable located in the Virtual File System (VFS) and then launches it</strong>. The payload (<em>KSPSService.exe</em>) deployed by the APPX file <strong>turned out to be a sample of HijackLoader</strong>. More technical information on this execution flow can be found in the Microsoft documentation<sup data-fn="3cf3fe19-9916-4c09-89ab-6aeddc9deb03" class="fn"><a href="#3cf3fe19-9916-4c09-89ab-6aeddc9deb03" id="3cf3fe19-9916-4c09-89ab-6aeddc9deb03-link">4</a></sup> and FINSIN’s analysis<sup data-fn="3131029d-6bf1-4315-8c56-900b16251f3f" class="fn"><a href="#3131029d-6bf1-4315-8c56-900b16251f3f" id="3131029d-6bf1-4315-8c56-900b16251f3f-link">5</a></sup>.</p>



<p>The APPX file also contains a legitimate Microsoft Edge installer (<em>MicrosoftEdgeUpdateSetup.exe</em> MD5: <em>58d8d75b0ca5e316862ed81cdb2d0c67</em>) and a PowerShell script (<em>chrome.ps1</em> MD5: <em>bfe16fc5d100757bd9dec4ef1aa42913</em>), downloading a legitimate Edge installer from transfer[.]sh and executing it. Both codes are executed when the user runs the Windows App file. Sekoia.io analysts believe that installing the legitimate web browser alongside the malware once again avoids any suspicion from the victim.</p>



<p>As mentioned by SentinelOne<sup data-fn="d238da43-90e5-4725-8838-d215302ad95c" class="fn"><a href="#d238da43-90e5-4725-8838-d215302ad95c" id="d238da43-90e5-4725-8838-d215302ad95c-link">6</a></sup>, APPX files are regularly used in malware campaigns to deploy the payload on the infected host, including BazarBackdoor, Emotet or Magniber ransomware. <strong>Although this technique is not new, Sekoia.io believes its use improves the rate of successful compromise</strong> by reducing the detection of the malicious payload’s execution.</p>



<h3 class="wp-block-heading" id="h-overview-of-hijackloader">Overview of HijackLoader</h3>



<p>First observed in the wild in July 2023 by Zscaler ThreatLabz<sup data-fn="ee2be818-5227-4ee9-ab62-697713e79ebc" class="fn"><a href="#ee2be818-5227-4ee9-ab62-697713e79ebc" id="ee2be818-5227-4ee9-ab62-697713e79ebc-link">7</a></sup>, <strong>HijackLoader is a modular loader downloading and executing an obfuscated payload</strong>. It implements several evasion techniques, including code injection, use of syscalls, Windows API hashing and Heaven’s gate. In recent months, HijackLoader delivered numerous commodity malware, including Danabot, Lumma, Raccoon, Redline, Remcos, SystemBC and Vidar.</p>



<p>Once executed, the HijackLoader sample deployed through the APPX file downloads its obfuscated payload from the adversary infrastructure “<em>hxxps://server2-slabx.ocmtancmi2c5t[.]live/osmesis/1829973585.png</em>”. The payload loaded by HijackLoader is a Raccoon sample communicating with its Command &amp; Control (C2) server “<em>128.140.101[.]125</em>”.</p>



<p>In August 2023, Rapid7 observed<sup data-fn="77fb2ffa-a58c-47c6-8955-b3b1779c59dd" class="fn"><a href="#77fb2ffa-a58c-47c6-8955-b3b1779c59dd" id="77fb2ffa-a58c-47c6-8955-b3b1779c59dd-link">8</a></sup> that the new <strong>IDAT Loader malware was delivered by ClearFake</strong>. Based on the code similarities between IDAT Loader and HijackLoader, and given the overlap in the C2 infrastructures, <strong>Sekoia.io assess with high confidence that the same threat group operates both loaders</strong>.</p>

<h2 class="wp-block-heading" id="h-conclusion">Conclusion</h2>



<p>First seen in the wild in July 2023, <strong>ClearFake is another “fake updates” threat that quickly became widespread </strong>due to the effective lure targeting a wide audience, as well as the watering hole technique used to distribute the malware via numerous compromised websites.</p>



<p>Given the <strong>ongoing development and the use of cutting-edge techniques</strong>, such as the blockchain technology to store malicious payloads, this <strong>threat must be closely monitored by organisations</strong>, as the malware delivered by ClearFake can be used to gain access to the victim’s network.</p>



<p>The <strong>tactics, techniques and procedures leveraged by the ClearFake operators overlap with those of SocGholish ones</strong> (tracked as TA569), in particular the use of watering holes, “fake updates&#8221; lures, Keitaro TDS, Dropbox file hosting service and the masquerading of filename with cyrillic characters. Considering this, <strong>Sekoia.io further assess ClearFake and SocGholish are possibly operated by the same threat group. Gathering additional evidence may help to confirm or refute this hypothesis</strong>.</p>



<p>To provide our customers with actionable intelligence, we will continue to monitor the evolution of ClearFake and other malware it delivers.</p>
