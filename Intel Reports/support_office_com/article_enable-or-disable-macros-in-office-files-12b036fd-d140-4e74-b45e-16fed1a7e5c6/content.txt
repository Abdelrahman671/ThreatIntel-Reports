		<h1 id="page-header" class="">Enable or disable macros in Microsoft 365 files</h1>
	</header>
	<section class="supAppliesToSection" ms.cmpgrp="applies_to" aria-label="Products this article applies to" dir="ltr">
		<div id="supAppliesToTableContainer">
			<details id="supAppliesToList"><summary aria-label="Press the Enter key to expand the full list of Office versions that this help article applies to." class="contentToggle">Applies To</summary><span class="appliesToItem">Excel for Microsoft 365</span> <span class="appliesToItem">Word for Microsoft 365</span> <span class="appliesToItem">Outlook for Microsoft 365</span> <span class="appliesToItem">PowerPoint for Microsoft 365</span> <span class="appliesToItem">Access for Microsoft 365</span> <span class="appliesToItem">Excel 2024</span> <span class="appliesToItem">Word 2024</span> <span class="appliesToItem">Outlook 2024</span> <span class="appliesToItem">PowerPoint 2024</span> <span class="appliesToItem">Access 2024</span> <span class="appliesToItem">Visio Professional 2024</span> <span class="appliesToItem">Visio Standard 2024</span> <span class="appliesToItem">Excel 2021</span> <span class="appliesToItem">Word 2021</span> <span class="appliesToItem">Outlook 2021</span> <span class="appliesToItem">PowerPoint 2021</span> <span class="appliesToItem">Access 2021</span> <span class="appliesToItem">Visio Professional 2021</span> <span class="appliesToItem">Visio Standard 2021</span> <span class="appliesToItem">Excel 2019</span> <span class="appliesToItem">Word 2019</span> <span class="appliesToItem">Outlook 2019</span> <span class="appliesToItem">PowerPoint 2019</span> <span class="appliesToItem">Access 2019</span> <span class="appliesToItem">Visio Professional 2019</span> <span class="appliesToItem">Visio Standard 2019</span> <span class="appliesToItem">Excel 2016</span> <span class="appliesToItem">Word 2016</span> <span class="appliesToItem">Outlook 2016</span> <span class="appliesToItem">PowerPoint 2016</span> <span class="appliesToItem">Access 2016</span> <span class="appliesToItem">Visio Professional 2016</span> <span class="appliesToItem">Visio Standard 2016</span> </details>

	<article class="ocpArticleContent"><section class="ocpIntroduction">
        <table aria-label="" class="banded">
          <tbody>
            <tr>
              <td></td>
              <td>
                <p>
                  <b class="ocpLegacyBold">Guided support in your browser can provide digital solutions for Office problems</b>
                </p>
                <p>
                  <a href="https://go.microsoft.com/fwlink/?linkid=2248452" target="_blank" role="button" class="ocpExternalLink supLinkButton" data-bi-type="anchor">Try guided support</a>
                </p>
              </td>
            </tr>
          </tbody>
        </table>
        <p id="bmbacktotop">A macro is a series of commands used to automate a repeated task and can be run when you have to perform the task. However, some macros can pose a security risk by introducing viruses or malicious software to your computer. </p>
        <div class="ocpAlert">
    <p class="ocpAlertSection"><b class="ocpWarning">Warning: </b>Never enable macros in a Microsoft 365 file unless you're sure you know what those macros do and you want the functionality they provide. <b class="ocpLegacyBold">You don't need to enable macros to view or edit the file.</b> For more info see <a href="/en-us/office/protect-yourself-from-macro-viruses-a3f3576a-bfef-4d25-84dc-70d18bde5903" class="ocpArticleLink" data-bi-type="anchor">Protect yourself from macro viruses.</a></p>
  </div>
      </section><section aria-labelledby="ID0EDN" class="ocpSection">
        <h2 id="ID0EDN" class="">Make a document trusted to enable macros</h2>
        
          <p>If you see a security warning when you open a document or try to run a macro, you can choose to make it a trusted document and enable macros. This example is on an Excel workbook. </p>
          <p>
             <picture><img class="fluentImage" src="https://support.content.office.net/en-us/media/bfa9a2e4-9717-4722-aee3-ae0c8c0b2fb3.png" loading="lazy" alt="Macros disabled warning bar"></picture>
          </p>
          <ol type="1" itemscope="" itemtype="http://schema.org/ItemList">
    <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>Select <b class="ocpUI">Enable Content</b>.</p>
            <meta itemprop="position" content="1">
    </li>
    <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>In the <b class="ocpUI">Security Warning</b> dialog, select <b class="ocpUI">Yes </b>to make the document trusted.</p>
              <p>
                 <picture><img class="fluentImage" src="https://support.content.office.net/en-us/media/af55e369-aa85-48bf-a773-f91fbc77d7b2.png" loading="lazy" alt="Trusted document security warning"></picture>
              </p>
            <meta itemprop="position" content="2">
    </li>
  </ol>
          <p>Macros are now enabled on this document. To revoke a trusted document, you need to clear all trusted documents. For more info, see <a href="/en-us/office/trusted-documents-cf872bd8-47ec-4c02-baa5-1fdba1a11b53" class="ocpArticleLink" data-bi-type="anchor">Trusted documents</a>.</p>
          <div class="ocpAlert">
    <p class="ocpAlertSection"><b class="ocpTip">Tip: </b>Rather than enabling macros for each document, you can create a trusted location to store trusted documents. Microsoft 365 will then not check them with the Trust Center. For more info, see <a href="/en-us/office/add-remove-or-change-a-trusted-location-in-microsoft-office-7ee1cdc2-483e-4cbb-bcb3-4e7c67147fb4" class="ocpArticleLink" data-bi-type="anchor">Add, remove, or change a trusted location in Microsoft Office</a>.</p>
  </div>
        
      </section><section aria-labelledby="ID0EDH" class="ocpSection">
        <h2 id="ID0EDH" class="">Change macro settings in the Trust Center</h2>
        
          <p>Macro settings are located in the Trust Center. However, if your device is managed by your work or school the system administrator might prevent anyone from changing settings.</p>
          <div class="ocpAlert">
    <p class="ocpAlertSection"><b class="ocpImportant">Important: </b>When you change your macro settings in the Trust Center, they are changed only for the Microsoft 365 app that you are currently using. The macro settings are not changed for all your Microsoft 365 apps.</p>
  </div>
          <ol type="1" itemscope="" itemtype="http://schema.org/ItemList">
    <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>Select the <b class="ocpUI">File</b> tab and choose <b class="ocpUI">Options</b>.</p>
            <meta itemprop="position" content="1">
    </li>
    <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>Select <b class="ocpUI">Trust Center</b>, and then choose <b class="ocpUI">Trust Center Settings</b>.</p>
            <meta itemprop="position" content="2">
    </li>
    <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>In the <b class="ocpUI">Trust Center</b>, select <b class="ocpUI">Macro Settings</b>.</p>
              <p>
                 <picture><img class="fluentImage" src="https://support.content.office.net/en-us/media/d512cfe1-8b56-4460-bee1-52a09f09d52c.png" loading="lazy" alt="Trust center macro settings"></picture>
              </p>
              <div class="ocpAlert">
        <p class="ocpAlertSection"><b class="ocpTip">Tip: </b>If you have the <a href="/en-us/office/show-the-developer-tab-e1192344-5e56-4d45-931b-e5fd9bea2d45" class="ocpArticleLink" data-bi-type="anchor">developer tab</a> on your ribbon, select the <b class="ocpUI">Developer</b> tab and then choose<b class="ocpUI"> </b> <picture><img class="fluentImage" src="https://support.content.office.net/en-us/media/3ac269ef-4b9d-4d9b-a2b7-b6d5f63b296d.png" loading="lazy" alt="Macro security icon"></picture> <b class="ocpUI">Macro Security</b> to open the <b class="ocpUI">Trust Center</b> quickly. </p>
      </div>
            <meta itemprop="position" content="3">
    </li>
    <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>Make the selections that you want, then select <b class="ocpUI">OK</b>.</p>
            <meta itemprop="position" content="4">
    </li>
  </ol>
        
      </section><section aria-labelledby="" class="ocpSection">
        
        
          <p>The following are Macro setting options. Not all apps have the same choices. </p>
          <div class="ocpAlert">
    <p class="ocpAlertSection"><b class="ocpNote">Note: </b>The options are slightly different in Excel, we'll call those out as we go.</p>
  </div>
          <ul>
    <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>
                <b class="ocpRunInHead">Disable all macros without notification</b>    Macros and security alerts about macros are disabled.<br aria-hidden="true"><br aria-hidden="true">
	In Excel this option is <b class="ocpUI">Disable VBA macros without notification</b> and it only applies to VBA macros.</p>
            <meta itemprop="position" content="1">
    </li>
    <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>
                <b class="ocpRunInHead">Disable all macros with notification</b>    Macros are disabled, but security alerts appear if there are macros present. </p>
              <p>
                 <picture><img class="fluentImage" src="https://support.content.office.net/en-us/media/bfa9a2e4-9717-4722-aee3-ae0c8c0b2fb3.png" loading="lazy" alt="Macros disabled warning bar"></picture>
              </p>
              <p>Use this setting to enable macros on a case-by-case basis.<br aria-hidden="true"><br aria-hidden="true">
	In Excel this option is <b class="ocpUI">Disable VBA macros with notification</b> and it only applies to VBA macros.</p>
            <meta itemprop="position" content="2">
    </li>
    <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>
                <b class="ocpRunInHead">Disable all macros except digitally signed macros</b>    Macros are disabled, and security alerts appear if there are unsigned macros present. However, if the macro is digitally signed by a trusted publisher, the macro just runs. If the macro is signed by a publisher you haven't trusted yet, you are given the opportunity to enable the signed macro and trust the publisher.<br aria-hidden="true"><br aria-hidden="true">
	In Excel this option is <b class="ocpUI">Disable VBA macros except digitally signed macros </b>and it only applies to VBA macros.</p>
            <meta itemprop="position" content="3">
    </li>
    <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>
                <b class="ocpRunInHead">Enable all macros (not recommended, potentially dangerous code can run)</b>    All macros run without confirmation. This setting makes your computer vulnerable to malicious code.<br aria-hidden="true"><br aria-hidden="true">
	In Excel this option is <b class="ocpUI">Enable VBA macros (not recommended, potentially dangerous code can run) </b>and it only applies to VBA macros.</p>
            <meta itemprop="position" content="4">
    </li>
    <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>
                Excel also has a checkbox for<b class="ocpUI"> Enable Excel 4.0 macros when VBA macros are enabled</b>. If you select this checkbox all of the above settings for VBA macros will also apply to Excel 4.0 (XLM) macros. If this checkbox is not selected XLM macros are disabled without notification.</p>
            <meta itemprop="position" content="5">
    </li>
    <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>
                <b class="ocpRunInHead">Trust access to the VBA project object model</b>   Block or allow programmatic access to the <a href="https://msdn.microsoft.com/en-us/library/bb190538.aspx" target="_blank" class="ocpExternalLink" data-bi-type="anchor">Visual Basic for Applications (VBA)</a> object model from an automation client. This security option is for code written to automate a Microsoft 365 program and manipulate the VBA environment and object model. It is a per-user and per-application setting, and denies access by default, hindering unauthorized programs from building harmful self-replicating code. For automation clients to access the VBA object model, the user running the code must grant access. To turn on access, select the check box.</p>
              <div class="ocpAlert">
        <p class="ocpAlertSection"><b class="ocpNote">Note: </b>Microsoft Access has no <b class="ocpUI">Trust access to the VBA project model object</b> option.</p>
      </div>
            <meta itemprop="position" content="6">
    </li>
  </ul>
