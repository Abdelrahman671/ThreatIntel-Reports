<div class="wpb_wrapper">
<p>
<p>We at K7 Labs recently found an interesting new .NET loader which downloads and executes <a href="https://github.com/Loksie/KoiVM-Virtualization#koivm"><strong>KoiVM</strong></a> virtualized binary, which in turn drops Remcos RAT and Agent Tesla based on the availability of its C2. The samples under consideration uses <a href="https://hastebin.com/about.md"><strong>hastebin</strong></a> URLs as its C2 server to download the next stage payloads. The overall flow of this multistage malware can be observed in the following flow diagram.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="785" height="557" src="https://labs.k7computing.com/wp-content/uploads/2022/12/1.png" alt="" class="wp-image-24822" srcset="https://labs.k7computing.com/wp-content/uploads/2022/12/1.png 785w, https://labs.k7computing.com/wp-content/uploads/2022/12/1-300x213.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/12/1-768x545.png 768w" sizes="(max-width: 785px) 100vw, 785px" /><figcaption>Figure 1: Execution Flow</figcaption></figure></div>



<p>The initial downloader is dropped through spam emails containing attachments of the names “New Orders.zip” or “Export Invoice &#8211; 8026137.zip”. The Zip contains a .NET executable with the same name as the Zip file and disguises itself as a calculator application. However, it is actually a multistage downloader.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="810" height="271" src="https://labs.k7computing.com/wp-content/uploads/2022/12/2.png" alt="" class="wp-image-24823" srcset="https://labs.k7computing.com/wp-content/uploads/2022/12/2.png 810w, https://labs.k7computing.com/wp-content/uploads/2022/12/2-300x100.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/12/2-768x257.png 768w" sizes="(max-width: 810px) 100vw, 810px" /><figcaption>Figure 2: Original Name of Downloader</figcaption></figure></div>



<div style="height:51px" aria-hidden="true" class="wp-block-spacer"></div>



<h3><strong>Stage-1 (Downloader Analysis)</strong></h3>



<p>The downloader initially starts to decode the C2 using an interesting decoding routine given below.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="918" height="76" src="https://labs.k7computing.com/wp-content/uploads/2022/12/3.png" alt="" class="wp-image-24824" srcset="https://labs.k7computing.com/wp-content/uploads/2022/12/3.png 918w, https://labs.k7computing.com/wp-content/uploads/2022/12/3-300x25.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/12/3-768x64.png 768w" sizes="(max-width: 918px) 100vw, 918px" /><figcaption>Figure 3: C2 decoding routine</figcaption></figure></div>



<p>Each character of the C2 string is XOR’ed with the index value of the corresponding character to obtain the C2 address. We can easily mimic this in Python using the code given below.</p>



<pre class="wp-block-code"><code><span style="color:#0175a7" class="has-inline-color">“””
Code to decode C2 URL’s
“””
 
c2servers = ""
decoded = r"huvsw?)(`hy\u007fioga&gt;r}~;gw`7w{huwquISW\u000fLQRW&#91;\u0013\u0005\u0004DL]&#91;US&#91;]\u001aVYZ\u0017K&#91;L\u0013^_N5,7!1&lt;)"
for c in range(0, len(decoded)):
    c2servers += chr(ord(decoded&#91;c]) ^ c)
print(c2servers.replace(",", "\n"))
 
Extracted C2’s:
 
hxxps://hastebin&#91;.]com/raw/nasijojiru
hxxps://hastebin&#91;.]com/raw/caqumubuyo
</span></code></pre>



<p>Once the C2 address&nbsp; is decoded, it sends a GET request to download the encoded 2<sup>nd</sup> stage KoiVM Droppers. After receiving the response from the server, the downloader starts its multistage decoding routine. It base64 decodes the response and decompresses it in memory using the DeflateStream class. The resultant buffer is XORed with the hardcoded key in the stage-1 downloader “<strong>M4use</strong>” to get the final decoded stage-2 KoiVM dropper binaries.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="901" height="414" src="https://labs.k7computing.com/wp-content/uploads/2022/12/4.png" alt="" class="wp-image-24825" srcset="https://labs.k7computing.com/wp-content/uploads/2022/12/4.png 901w, https://labs.k7computing.com/wp-content/uploads/2022/12/4-300x138.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/12/4-768x353.png 768w" sizes="(max-width: 901px) 100vw, 901px" /><figcaption>Figure 4: Payload decoding flow</figcaption></figure></div>



<div style="height:3px" aria-hidden="true" class="wp-block-spacer"></div>



<h3><strong>Stage2 (Virtualized Droppers)</strong></h3>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="951" height="514" src="https://labs.k7computing.com/wp-content/uploads/2022/12/5.png" alt="" class="wp-image-24826" srcset="https://labs.k7computing.com/wp-content/uploads/2022/12/5.png 951w, https://labs.k7computing.com/wp-content/uploads/2022/12/5-300x162.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/12/5-768x415.png 768w" sizes="(max-width: 951px) 100vw, 951px" /><figcaption>Figure 5: KoiVM Dropper</figcaption></figure></div>



<p>The stage-2 payload is highly obfuscated and virtualized with KoiVM. It is used along with <a href="https://github.com/yck1509/ConfuserEx">ConfuserEx</a> to virtualize the execution of the sample. It changes all the IL-Instruction to the byte format understandable only by the KoiVM Runtime.&nbsp;</p>



<p>As stated in KoiVM <a href="https://github.com/Loksie/KoiVM-Virtualization#:~:text=first%20one%20is%20certainly%20ridiculous%20as%20it%20will%20%22merge%22%20with%20cex%20and%20virtualize%20every%20single%20method%2C%20including%20protections%20from%20ConfuserEX%2C%20however%20note%20that%20this%20might%20KILL%20your%20performan">Readme</a>, virtualization with KoiVM can be done in two ways</p>



<ol><li>Virtualize only the methods which we select</li><li>Virtualize all the functions including ConfuserEx integrity protection</li></ol>



<p>The stage-2 dropper payloads had chosen the 2<sup>nd</sup> option to virtualize all the functions, which made our analysis harder. Since Win32API and structs are accessed using <strong>PInvoke</strong> in C# and it can’t be virtualized or obfuscated, we were able to identify the API’s and correlate the behavior of this KoiVM dropper. The sample imports all the API’s which are required for Process Injection and In-memory execution.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="828" height="507" src="https://labs.k7computing.com/wp-content/uploads/2022/12/6.png" alt="" class="wp-image-24827" srcset="https://labs.k7computing.com/wp-content/uploads/2022/12/6.png 828w, https://labs.k7computing.com/wp-content/uploads/2022/12/6-300x184.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/12/6-768x470.png 768w" sizes="(max-width: 828px) 100vw, 828px" /><figcaption>Figure 6: Imports accessed through PInvoke</figcaption></figure></div>



<p>The encoded stage-3 payload is found in the resource section of the KoiVM binary. On analyzing the blob, we found an interesting string pattern which seems to be repeating. When Null bytes are XOR’ed with a key, the resultant value is the key itself. Since the 3<sup>rd</sup> stage payload has many NULL bytes we are able to extract the XOR key used for decoding. Similarly, the KoiVM sample downloaded from the other hastebin URL (second C2 address) had a similar pattern. There are two different final 3<sup>rd</sup> stage payloads which are dropped based on the C2 address accessed , of which the first binary is XOR decoded using the key <strong>“Jus3ify”</strong> and the second binary is XOR decoded using the key “<strong>Monito3</strong>“.&nbsp;</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="626" height="506" src="https://labs.k7computing.com/wp-content/uploads/2022/12/7.png" alt="" class="wp-image-24828" srcset="https://labs.k7computing.com/wp-content/uploads/2022/12/7.png 626w, https://labs.k7computing.com/wp-content/uploads/2022/12/7-300x242.png 300w" sizes="(max-width: 626px) 100vw, 626px" /><figcaption>Figure 7: Decoded stage-3 payloads</figcaption></figure></div>



<p>The key can also be identified by debugging the KoiVM Runtime using <a href="https://github.com/dnSpyEx">dnSpyEx</a> and stepping into the yielder function “<strong>SelectIterator</strong>” as shown in image below. We were able to view payload data and key as plaintext because all functions of KoiVM dropper binary are only virtualized and not the calls to string methods.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="797" height="365" src="https://labs.k7computing.com/wp-content/uploads/2022/12/8.png" alt="" class="wp-image-24829" srcset="https://labs.k7computing.com/wp-content/uploads/2022/12/8.png 797w, https://labs.k7computing.com/wp-content/uploads/2022/12/8-300x137.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/12/8-768x352.png 768w" sizes="(max-width: 797px) 100vw, 797px" /><figcaption>Figure 8: XOR key in memory</figcaption></figure></div>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="646" height="443" src="https://labs.k7computing.com/wp-content/uploads/2022/12/9.png" alt="" class="wp-image-24830" srcset="https://labs.k7computing.com/wp-content/uploads/2022/12/9.png 646w, https://labs.k7computing.com/wp-content/uploads/2022/12/9-300x206.png 300w" sizes="(max-width: 646px) 100vw, 646px" /><figcaption>Figure 9: XOR decoded payload in memory</figcaption></figure></div>

<h3><strong>Stage 3</strong></h3>



<h4><strong>Agent Tesla</strong></h4>



<p>Using <a href="https://github.com/horsicq/Detect-It-Easy">Detect it Easy</a><strong> </strong>we were able to identify that stage-3 payload is obfuscated with <strong>.Net Reactor</strong>, thus we used .<a href="https://github.com/SychicBoy/NETReactorSlayer">NetSlayer</a> to de-obfuscate the sample to analyze further.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="856" height="469" src="https://labs.k7computing.com/wp-content/uploads/2022/12/10.png" alt="" class="wp-image-24831" srcset="https://labs.k7computing.com/wp-content/uploads/2022/12/10.png 856w, https://labs.k7computing.com/wp-content/uploads/2022/12/10-300x164.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/12/10-768x421.png 768w" sizes="(max-width: 856px) 100vw, 856px" /><figcaption>Figure 10: Trying to de-virtualize using .NET Slayer</figcaption></figure></div>



<p>The tool was not able to completely de-obfuscate the sample, for example we could see that the Agent Tesla binary has implemented control flow flattening, but the tool was not able to unflatten it. The strings are present in raw hex form using string interning.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="297" height="428" src="https://labs.k7computing.com/wp-content/uploads/2022/12/11.png" alt="" class="wp-image-24832" srcset="https://labs.k7computing.com/wp-content/uploads/2022/12/11.png 297w, https://labs.k7computing.com/wp-content/uploads/2022/12/11-208x300.png 208w" sizes="(max-width: 297px) 100vw, 297px" /><figcaption>Figure 11: Control flow flattening implemented in Agent Tesla</figcaption></figure></div>



<p>The Agent Tesla malware has the capability to log keystrokes, steal browser cookies and crypto wallets and send it to C2. All the strings are saved as raw bytes by using string interning and they are accessed with respective index and length using a class method.</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" width="1024" height="423" src="https://labs.k7computing.com/wp-content/uploads/2022/12/12-1024x423.png" alt="" class="wp-image-24833" srcset="https://labs.k7computing.com/wp-content/uploads/2022/12/12-1024x423.png 1024w, https://labs.k7computing.com/wp-content/uploads/2022/12/12-300x124.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/12/12-768x317.png 768w, https://labs.k7computing.com/wp-content/uploads/2022/12/12.png 1192w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption>Figure 12: Configuration stored using string interning</figcaption></figure></div>



<p>On dumping the strings, we got a configuration file and confirmed it as <strong>Agent Tesla</strong> malware.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="630" height="316" src="https://labs.k7computing.com/wp-content/uploads/2022/12/13.png" alt="" class="wp-image-24834" srcset="https://labs.k7computing.com/wp-content/uploads/2022/12/13.png 630w, https://labs.k7computing.com/wp-content/uploads/2022/12/13-300x150.png 300w" sizes="(max-width: 630px) 100vw, 630px" /><figcaption>Figure 13: Tesla Configuration</figcaption></figure></div>



<p>Agent Tesla is an info stealing malware, which collects keystrokes, browser cookies, and system information. The collected data is sent as an attachment to a mail id &#8211; peterashley202@gmail[.]com.</p>



<div style="height:3px" aria-hidden="true" class="wp-block-spacer"></div>



<h4><strong>Remcos RAT&nbsp;</strong></h4>



<p>On viewing the strings from stage-2 payload (the KoiVM payload2 from the second hastebin URL), we were able to identify the final payload to be Remcos RAT which was confirmed by extracting the configuration from KoiVM payload2’s&nbsp; resource section.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="514" height="167" src="https://labs.k7computing.com/wp-content/uploads/2022/12/14.png" alt="" class="wp-image-24835" srcset="https://labs.k7computing.com/wp-content/uploads/2022/12/14.png 514w, https://labs.k7computing.com/wp-content/uploads/2022/12/14-300x97.png 300w" sizes="(max-width: 514px) 100vw, 514px" /><figcaption>Figure 14: Remcos Agent String</figcaption></figure></div>



<p>The RC4 encrypted configuration of Remcos RAT is saved in the resource section as “SETTINGS”.&nbsp;</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="783" height="306" src="https://labs.k7computing.com/wp-content/uploads/2022/12/15.png" alt="" class="wp-image-24836" srcset="https://labs.k7computing.com/wp-content/uploads/2022/12/15.png 783w, https://labs.k7computing.com/wp-content/uploads/2022/12/15-300x117.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/12/15-768x300.png 768w" sizes="(max-width: 783px) 100vw, 783px" /><figcaption>Figure 15: Remcos RAT encrypted config stored in resource</figcaption></figure></div>



<p>The first byte in the configuration file is the <strong>length of RC4 key</strong>(n). The next n bytes are the RC4 key followed by the payload bytes.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="858" height="566" src="https://labs.k7computing.com/wp-content/uploads/2022/12/16.png" alt="" class="wp-image-24837" srcset="https://labs.k7computing.com/wp-content/uploads/2022/12/16.png 858w, https://labs.k7computing.com/wp-content/uploads/2022/12/16-300x198.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/12/16-768x507.png 768w" sizes="(max-width: 858px) 100vw, 858px" /><figcaption>Figure 16: Remcos Configuration</figcaption></figure></div>



<p>Remcos RAT steals browser cookies, takes current window screenshots and sends it to the C2 present in Configuration. It establishes a listener connection with the C2 and waits for the attacker to send commands to execute.</p>



<p>We at K7 Labs provide detection against latest threats and also for this newer variant of Loader. Users are advised to use a reliable security product such as “K7 Total Security” and keep it up-to-date so as to safeguard their devices.</p>
