							<div class="content "><h1 id="copy">Copy</h1><div class="display-flex justify-content-space-between align-items-center flex-wrap-wrap page-metadata-container">
										<div class="margin-right-xxs">
											<ul class="metadata page-metadata" data-bi-name="page info" lang="en-us" dir="ltr"><li>Article</li><li>
														<time class="is-invisible" data-article-date aria-label="Article review date" datetime="2009-09-11T00:00:00.000Z" data-article-date-source="ms.date">09/11/2009</time>
													</li></ul>

											<h2 id="ms--in-this-article" class="title is-6 margin-block-xs">In this article</h2>
										</nav><!-- <content> --><p>Copies one or more files from one location to another.</p>
<h3 id="syntax">Syntax</h3>
<p><strong>copy</strong> [<strong>/d</strong>] [<strong>/v</strong>] [<strong>/n</strong>] [{<strong>/y</strong>|<strong>/-y</strong>}] [<strong>/z</strong>] [{<strong>/a</strong>|<strong>/b</strong>}] <em>Source</em> [{<strong>/a</strong>|<strong>/b</strong>}] [<strong>+</strong>&nbsp;<em>Source</em> [{<strong>/a</strong>|<strong>/b</strong>}] [<strong>+</strong> ...]] [<em>Destination</em> [{<strong>/a</strong>|<strong>/b</strong>}]]</p>
<h3 id="parameters">Parameters</h3>
<p><strong>/d</strong> &nbsp; <strong>:</strong> Allows the encrypted files being copied to be saved as decrypted files at the destination.</p>
<p><strong>/v</strong> &nbsp; <strong>:</strong> Verifies that new files are written correctly.</p>
<p><strong>/n</strong> &nbsp; <strong>:</strong> Uses a short file name, if available, when copying a file with a name longer than eight characters, or with a file extension longer than three characters.</p>
<p><strong>/y</strong> &nbsp; <strong>:</strong> Suppresses prompting to confirm that you want to overwrite an existing destination file.</p>
<p><strong>/-y</strong> &nbsp; <strong>:</strong> Prompts you to confirm that you want to overwrite an existing destination file.</p>
<p><strong>/z</strong> &nbsp; <strong>:</strong> Copies networked files in restartable mode.</p>
<p><strong>/a</strong> &nbsp; <strong>:</strong> Indicates an ASCII text file.</p>
<p><strong>/b</strong> &nbsp; <strong>:</strong> Indicates a binary file.</p>
<p><em><strong>Source</strong></em> &nbsp; <strong>:</strong> Required. Specifies the location from which you want to copy a file or set of files. <em>Source</em> can consist of a drive letter and colon, a folder name, a file name, or a combination of these.</p>
<p><em><strong>Destination</strong></em> &nbsp; <strong>:</strong> Required. Specifies the location to which you want to copy a file or set of files. <em>Destination</em> can consist of a drive letter and colon, a folder name, a file name, or a combination of these.</p>
<p><strong>/?</strong> &nbsp; <strong>:</strong> Displays help at the command prompt.</p>
<h3 id="remarks">Remarks</h3>
<ul>
<li><p>You can copy an ASCII text file that uses an end-of-file character (that is, CTRL+Z) to indicate the end of the file.</p>
</li>
<li><p>Using <strong>/a</strong>&nbsp;</p>
<p>When <strong>/a</strong> precedes a list of files on the command line, it applies to all files listed until <strong>copy</strong> encounters <strong>/b</strong>. In this case, <strong>/b</strong> applies to the file preceding <strong>/b</strong>.</p>
<p>When <strong>/a</strong> follows a list of files on the command line, it applies to all listed files until <strong>copy</strong> encounters <strong>/b</strong>. In this case, <strong>/b</strong> applies to the file preceding <strong>/b</strong>.</p>
<p>The effect of <strong>/a</strong> depends on its position in the command-line string. When <strong>/a</strong> follows <em>Source</em>, <strong>copy</strong> treats the file as an ASCII file and copies data that precedes the first end-of-file character.</p>
<p>When <strong>/a</strong> follows <em>Destination</em>, <strong>copy</strong> adds an end-of-file character as the last character of the file.</p>
</li>
<li><p>Using <strong>/b</strong>&nbsp;</p>
<p><strong>/b</strong> directs the command interpreter to read the number of bytes specified by the file size in the directory. <strong>/b</strong> is the default value for <strong>copy</strong>, unless <strong>copy</strong> combines files.</p>
<p>When <strong>/b</strong> precedes a list of files on the command line, it applies to all listed files until <strong>copy</strong> encounters <strong>/a</strong>. In this case, <strong>/a</strong> applies to the file preceding <strong>/a</strong>.</p>
<p>When <strong>/b</strong> follows a list of files on the command line, it applies to all listed files until <strong>copy</strong> encounters <strong>/a</strong>. In this case, <strong>/a</strong> applies to the file preceding <strong>/a</strong>.</p>
<p>The effect of <strong>/b</strong> depends on its position in the commandline string. When <strong>/b</strong> follows <em>Source</em>, <strong>copy</strong> copies the entire file, including any end-of-file character.</p>
<p>When <strong>/b</strong> follows <em>Destination</em>, <strong>copy</strong> does not add an end-of-file character.</p>
</li>
<li><p>Using <strong>/v</strong>&nbsp;</p>
<p>If a write operation cannot be verified an error message appears. Although recording errors rarely occur with <strong>copy</strong>, you can use <strong>/v</strong> to verify that critical data has been correctly recorded. The <strong>/v</strong> command-line option also slows down the <strong>copy</strong> command, because each sector recorded on the disk must be checked.</p>
</li>
<li><p>Using <strong>/y</strong> and <strong>/-y</strong>&nbsp;</p>
<p>If <strong>/y</strong> is preset in the COPYCMD environment variable, you can override this setting by using <strong>/-y</strong> at the command line. By default, you are prompted when you replace this setting, unless the <strong>copy</strong> command is executed in a batch script.</p>
</li>
<li><p>Appending files</p>
<p>To append files, specify a single file for <em>Destination</em>, but multiple files for <em>Source</em> (using wildcard characters or <em>file1</em>**+<em><strong>file2</strong></em>+**<em>file3</em> format).</p>
</li>
<li><p>Using <strong>/z</strong>&nbsp;</p>
<p>If the connection is lost during the copy phase (for example, if the server going offline severs the connection), <strong>copy /z</strong> resumes after the connection is reestablished. <strong>/z</strong> also displays the percentage of the copy operation that is completed for each file.</p>
</li>
<li><p>Copying to and from devices</p>
<p>You can substitute a device name for one or more occurrences of <em>Source</em> or <em>Destination</em>.</p>
</li>
<li><p>Using or omitting <strong>/b</strong> when copying to a device</p>
<p>When <em>Destination</em> is a device (for example, Com1 or Lpt1), <strong>/b</strong> copies data to the device in binary mode. In binary mode, <strong>copy /b</strong> copies all characters (that is, including special characters, such as CTRL+C, CTRL+S, CTRL+Z, and carriage return) to the device as data. However, if you omit <strong>/b</strong>, data is copied to the device in ASCII mode. In ASCII mode, special characters might cause Windows&nbsp;XP to combine files during the copying process. For more information, see "Combining files."</p>
</li>
<li><p>Using the default destination file</p>
<p>If you do not specify a destination file, a copy is created with the same name, creation date, and creation time as the original file, placing the new copy in the current directory on the current drive. If the source file is on the current drive and in the current directory and you do not specify a different drive or directory for the destination file, the <strong>copy</strong> command stops and displays the following error message:</p>
<p>File cannot be copied onto itself</p>
<p>0 File(s) copied</p>
</li>
<li><p>Combining files</p>
<p>If you specify more than one <em>Source</em>, separating entries with a plus sign (+), <strong>copy</strong> combines the files into a single file. If you use wildcard characters (that is, * or ?) in <em>Source</em>, but you specify a single file name in <em>Destination</em>, <strong>copy</strong> combines all files matching the file name in <em>Source</em> and creates a single file with the file name specified in <em>Destination</em>. In either case, <strong>copy</strong> assumes the combined files are ASCII files unless you use <strong>/b</strong>.</p>
</li>
<li><p>Copying zero-length files</p>
<p><strong>Copy</strong> does not copy files that are 0 bytes long. Use <strong>xcopy</strong> to copy these files.</p>
</li>
<li><p>Changing the time and date of a file</p>
<p>If you want to assign the current time and date to a file without modifying the file, use the following syntax:</p>
<p><strong>copy /b</strong>&nbsp;<em>Source</em>**+**,,</p>
<p>The commas indicate the omission of the <em>Destination</em> parameter.</p>
</li>
<li><p>Copying files in subdirectories</p>
<p>To copy all of a directory's files and subdirectories, use the <strong>xcopy</strong> command. For information about <strong>xcopy</strong>, see Related Topics.</p>
</li>
<li><p>The <strong>copy</strong> command, with different parameters, is available from the Recovery Console.</p>
</li>
</ul>
<h3 id="examples">Examples</h3>
<p>To copy a file called Memo.doc to Letter.doc in the current drive and ensure that an end-of-file character is at the end of the copied file, type:</p>
<p><strong>copy memo.doc letter.doc /a</strong></p>
<p>To copy a file named Robin.typ from the current drive and directory to an existing directory named Birds that is located on drive C, type:</p>
<p><strong>copy robin.typ c:\birds</strong></p>
<p>If the Birds directory does not exist, the file Robin.typ is copied into a file named Birds that is located in the root directory on the disk in drive C.</p>
<p>To copy several files into one file, list several <em>Source</em> files, separate the file names with a plus sign (+), and specify a <em>Destination</em> file that you want to contain the resulting combined file. For example, to combine Mar89.rpt, Apr89.rpt, and May89.rpt, which are located on the current drive and directory, and place them in a file named Report on the current drive and directory, type:</p>
<p><strong>copy mar89.rpt + apr89.rpt + may89.rpt report</strong></p>
<p>When you combine files, <strong>copy</strong> marks the destination file with the current date and time. If you omit <em>Destination</em>, the files are combined and stored under the name of the file specified first. For example, to combine all four files in Report when a file named Report already exists, type:</p>
<p><strong>copy report + mar89.rpt + apr89.rpt + may89.rpt</strong></p>
<p>You can also combine several files into one file by using wildcard characters (that is, * or ?). For example, to combine all files in the current directory on the current drive that have the extension .txt into one file named Combin.doc, type:</p>
<p><strong>copy *.txt combin.doc</strong></p>
<p>If you want to combine several binary files into one file by using wildcard characters, include <strong>/b</strong>. This prevents Windows&nbsp;XP from treating CTRL+Z as an end-of-file character. For example, type:</p>
<p><strong>copy /b *.exe combin.exe</strong></p>
<h4 id="caution">&nbsp;Caution</h4>
<ul>
<li>If you combine binary files, the resulting file might be unusable due to internal formatting.</li>
</ul>
<p>In the following example, <strong>copy</strong> combines each file that has a .txt extension with its corresponding .ref file. The result is a file with the same file name but with a .doc extension. <strong>Copy</strong> combines File1.txt with File1.ref to form File1.doc, and then <strong>copy</strong> combines File2.txt with File2.ref to form File2.doc, and so on. For example, type:</p>
<p><strong>copy *.txt + *.ref *.doc</strong></p>
<p>To combine all files with the .txt extension, and then combine all files with the .ref extension into one file named Combin.doc, type:</p>
<p><strong>copy *.txt + *.ref combin.doc</strong></p>
