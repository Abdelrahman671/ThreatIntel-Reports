<div class="p1"><div class="p-bs">
	<h2>Summary</h2>
	<div class="p-bs-intro">Tracking Vidar Infrastructure</div>
	<ul>
		<li>Vidar, a malware that evolved from Arkei, stands out as one of the first stealers capable of extracting information from 2FA Software and the Tor Browser.</li>
		<li>Vidar's C2 servers utilize HTTP over TLS, including hardcoded subject and issuer-distinguished names (DNs) on certificates. This allowed Censys to detect 22 unique IP addresses linked to Vidar campaigns.</li>
		<li>Vidar has been associated with Scattered Spider, known for targeting large organizations and IT help desks. </li>
	</ul>
</div>
<div class="p-tb p-tbb">
	<h2>Introduction</h2>
<p>Stealers are trojans that collect credentials, notable files, and tokens from an infected computer and upload the data back to attacker-controlled infrastructure. Today, we will discuss one of the more advanced stealers: <a href="https://malpedia.caad.fkie.fraunhofer.de/details/win.vidar" target="_blank" rel="noopener">Vidar</a>. Vidar is a piece of malware originating from the <a href="https://malpedia.caad.fkie.fraunhofer.de/details/win.arkei_stealer" target="_blank" rel="noopener">Arkei Stealer</a> but uses new methods to find and direct traffic to the attacker.</p>
</div><div class="p-tb">
	<h2><span style="font-weight: 400">Vidar Operational Details</span></h2>
<p>Vidar uses common network communication methods, and once in place, it will connect to a Telegram server to fetch the URL of the Command and Control (C2) server. In the following two screenshots, you will see examples of this C2 distribution method via Telegram or, if that fails, a backup Steam account.</p>
<div id="attachment_3334" style="width: 276px" class="wp-caption aligncenter"><img aria-describedby="caption-attachment-3334" class="wp-image-3334" src="https://censys.com/wp-content/uploads/2023/11/Vidar-Telegram-Pointer.png" alt="Example of a Telegram account pointing to the Vidar C2 server" width="266" height="194" /><p id="caption-attachment-3334" class="wp-caption-text">Example of a Telegram account pointing to the Vidar C2 server</p></div>
<div id="attachment_3335" style="width: 498px" class="wp-caption aligncenter"><img aria-describedby="caption-attachment-3335" class="wp-image-3335" src="https://censys.com/wp-content/uploads/2023/11/Vidar-Steam-Pointer.png" alt="Example of Steam account pointing to the Vidar C2 server" width="488" height="142" /><p id="caption-attachment-3335" class="wp-caption-text">Example of Steam account pointing to the Vidar C2 server</p></div>
<p>Once the C2 server connection has been established, Vidar will start the process of exfiltrating data from the host to the attacker-owned server.</p>
<p><img class="aligncenter wp-image-3336" src="https://censys.com/wp-content/uploads/2023/11/Vidar-DLLs.png" alt="Here, we see seven different HTTP GET requests made to download several legitimate DLLs" width="212" height="203" /></p>
<p>Here, we see seven different HTTP GET requests made to the C2, which downloads several legitimate DLLs:</p>
<ul>
<li style="font-weight: 400">/sqlite3.dll</li>
<li style="font-weight: 400">/freebl3.dll</li>
<li style="font-weight: 400">/mozglue.dll</li>
<li style="font-weight: 400">/msvcp140.dll</li>
<li style="font-weight: 400">/nss3.dll</li>
<li style="font-weight: 400">/softokn3.dll</li>
<li style="font-weight: 400">/vcruntime140.dll</li>
</ul>
<p><img loading="lazy" class="aligncenter wp-image-3337" src="https://censys.com/wp-content/uploads/2023/11/Vidar-Desktop-Screenshot.png" alt="A desktop screenshot submitted to Vidar's C2" width="590" height="332" /></p>
<p>Vidar then takes a screenshot of the user&#8217;s desktop, collects information about the user’s system (browser cookies, passwords, etc…), and sends it all over the C2’s HTTPS connection via a multipart form data POST request. Note that these servers will only allow POST requests from specific user agents such as the example below.</p>
<p><img loading="lazy" class="aligncenter size-full wp-image-3339" src="https://censys.com/wp-content/uploads/2023/11/Vidar-User-Agent.png" alt="" width="1937" height="229" /><img loading="lazy" class="aligncenter size-full wp-image-3340" src="https://censys.com/wp-content/uploads/2023/11/Vidar-Payload.png" alt="" width="2148" height="416" /></p>
<p>Because this C2 uses TLS, we can view its specific hardcoded subject and issuer-distinguished names (DNs) on the host’s certificate:</p>
<p><img loading="lazy" class="aligncenter wp-image-3341 size-full" src="https://censys.com/wp-content/uploads/2023/11/Vidar-Subject-and-Issuer-DNs.png" alt="Vidar's hardcoded subject and issuer-distinguished names" width="1452" height="326" /></p>
<p>This is particularly noteworthy, as it can provide a method for identifying these C2 servers, which can be found with the following Censys search query:</p>
<p><a href="https://search.censys.io/search?resource=hosts&amp;sort=RELEVANCE&amp;per_page=25&amp;virtual_hosts=INCLUDE&amp;q=services.tls.certificates.leaf_data.subject_dn%3A+%22C%3DXX%2C+ST%3DNY%2C+L%3DNY%2C+O%3DStaticIP%2C+OU%3DprivateIP%22" target="_blank" rel="noopener"><span style="font-weight: 400">services.tls.certificates.leaf_data.subject_dn: &#8220;C=XX, ST=NY, L=NY, O=StaticIP, OU=privateIP&#8221;</span></a></p>
<p>If the reader wishes to automate a system to pull down a list of known Vidar C2 servers, the following <a href="https://github.com/censys/censys-python" target="_blank" rel="noopener">Censys CLI</a> command can be used:</p>
</div><figure class="p-cb ">
	<pre>censys search --pages -1 --virtual-hosts INCLUDE 'services.tls.certificates.leaf_data.subject_dn: "C=XX, ST=NY, L=NY, O=StaticIP, OU=privateIP"' | jq -r '.[] | if try(.name) then .name else .ip end'</pre>
</figure><div class="p-tb">
	<h2>Vidar&#8217;s Scope on the Internet</h2>
<p><strong>Note:</strong> For this study, we define a &#8220;host&#8221; as a unique collection of service data associated with an IP address and one or more host names. We consolidate hostnames serving the same service data as their bare IP counterparts for deduplication purposes. Censys Search will sometimes show separate entries for the same physical IP address for multiple hostnames.</p>
<p>At the time of writing, <strong>Censys observed <a href="https://search.censys.io/search/report?resource=hosts&amp;q=services.tls.certificates.leaf_data.subject_dn%3A+%22C%3DXX%2C+ST%3DNY%2C+L%3DNY%2C+O%3DStaticIP%2C+OU%3DprivateIP%22&amp;virtual_hosts=INCLUDE&amp;field=ip&amp;num_buckets=50" target="_blank" rel="noopener">22 unique IP addresses</a> associated with a Vidar campaign</strong> (some with multiple hostnames) which can be seen <a href="https://search.censys.io/search?resource=hosts&amp;sort=RELEVANCE&amp;per_page=25&amp;virtual_hosts=INCLUDE&amp;q=services.tls.certificates.leaf_data.subject_dn%3A+%22C%3DXX%2C+ST%3DNY%2C+L%3DNY%2C+O%3DStaticIP%2C+OU%3DprivateIP%22" target="_blank" rel="noopener">within Censys search results</a>.</p>
<p><img loading="lazy" class="aligncenter wp-image-3342" src="https://censys.com/wp-content/uploads/2023/11/Vidar-Stats.png" alt="Vidar C2 Stats" width="753" height="381" /></p>
<p>Interestingly, most of these C2 services are isolated to two distinct internet providers within two countries: <strong><a href="https://search.censys.io/search?resource=hosts&amp;sort=RELEVANCE&amp;per_page=25&amp;virtual_hosts=INCLUDE&amp;q=autonomous_system.asn%3D24940+and+services.tls.certificates.leaf_data.subject_dn%3A+%22C%3DXX%2C+ST%3DNY%2C+L%3DNY%2C+O%3DStaticIP%2C+OU%3DprivateIP%22" target="_blank" rel="noopener">AS24940 (HETZNER-AS)</a> with 21 distinct hosts (19 located in Germany and 2 located in Finland)</strong> and <strong>a single host running in <a href="https://search.censys.io/search?resource=hosts&amp;sort=RELEVANCE&amp;per_page=25&amp;virtual_hosts=INCLUDE&amp;q=autonomous_system.asn%3D202448+and+services.tls.certificates.leaf_data.subject_dn%3A+%22C%3DXX%2C+ST%3DNY%2C+L%3DNY%2C+O%3DStaticIP%2C+OU%3DprivateIP%22" target="_blank" rel="noopener">AS202448 (MVPS)</a></strong> in the country of Finland.</p>
<h2><span style="font-weight: 400">Why Vidar Matters</span></h2>
<p>This malware is a tool of choice for <a href="https://www.cisa.gov/sites/default/files/2023-11/aa23-320a_scattered_spider.pdf" target="_blank" rel="noopener">Scattered Spider</a>, a cybercriminal organization known for targeting large companies and IT help desks. Along with their <a href="https://www.trellix.com/about/newsroom/stories/research/scattered-spider-the-modus-operandi/#:~:text=Despite%20this%20change%20in%20targets%2C%20Scattered%20Spider%20continues%20to%20leverage%20a%20variety%20of%20social%20engineering%20tactics%2C" target="_blank" rel="noopener">ability to social engineer</a> some of the largest organizations, Scattered Spider engages in data theft for extortion and has been known to deploy ransomware alongside Vidar. High-profile targets like <a href="https://www.wired.com/story/mgm-ceasars-hack-ransomware/" target="_blank" rel="noopener">MGM and Caesars have fallen victim</a> to their attacks, underscoring the severity of the threat.In response to these recent attacks, the FBI and CISA have <a href="https://www.cisa.gov/sites/default/files/2023-11/aa23-320a_scattered_spider.pdf" target="_blank" rel="noopener">issued recommendations</a> for organizations running critical infrastructure to mitigate and reduce the likelihood and impact of attacks by Scattered Spider actors.</p>
