This is true for a number of different types of cyberattacks, and is especially true for ransomware attacks. Tracking Akira ransomware attacks over the past several months, Huntress analysts have identified a number of indicators that stand out as precursors to an Akira ransomware attack.</p><p>It should be noted that in some incidents, the Huntress agent is added after the ransomware file encryption has occurred. In others, the Huntress agent has not been installed completely across the infrastructure. In still other instances, older versions of Windows may still be in use within the infrastructure. All of these conditions severely limit visibility into the incident. </p><p>â</p><h2>Attacks</h2><p>Huntress analysts have observed threat actors creating new user accounts, either via type 3 network connections, moving laterally within the infrastructure, or via access to exposed MSSQL servers (via <strong>[.highlight]sqlservr.exe[.highlight]</strong>), either standalone or part of Veeam installations.</p><p>While Huntress analysts have observed threat actors creating new user accounts as an initial step in an attack that has led to Akira ransomware being deployed, these new user accounts have not always been used to access the targeted endpoint. Many times, an already-compromised account will be used to log into the endpoint. This may need to be preceded by the threat actor enabling RDP, but this has not always been the case. RDP can be enabled on endpoints via the following command:</p><p><strong>[.highlight]reg add &quot;HKLM\SYSTEM\CurrentControlSet\Control\Terminal Server&quot; /v fDenyTSConnections /t REG_DWORD /d 0 /f[.highlight]</strong></p><p>Note that the use of this command is not unique to Akira ransomware threat actors. Huntress analysts have observed this command being used during <a href="https://www.huntress.com/blog/lolbin-to-inc-ransomware" target="_blank">INC</a> and <a href="https://www.huntress.com/blog/readtext34-ransomware-incident" target="_blank">ReadText34</a> ransomware incidents.</p><p>In December 2023, Huntress observed the following command being issued against a targeted endpoint:</p><p><strong>[.highlight]reg add[.highlight]<br/>[.highlight]HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon\SpecialAccounts\UserList[.highlight]<br/>[.highlight]/t REG_DWORD /v [USER] /d 0 /f 1[.highlight]</strong></p><p>This command prevents the designated user account from being displayed on the Welcome screen, essentially hiding it from view.</p><p>Analysts have observed threat actor access via RDP originating from a workstation named <strong>[.highlight]WIN-JGRMF8L11HO[.highlight]</strong>. This workstation name will be visible in <strong>[.highlight]Microsoft-Windows-Security-Auditing/4624[.highlight]</strong> (successful login) or <strong>[.highlight]Microsoft-Windows-Security-Auditing/4625[.highlight]</strong> (failed login attempt) event records, as well as within <strong>[.highlight]Microsoft-Windows-Security-Auditing/4776[.highlight]</strong> event records, indicating that an attempt was made to validate credentials for an account. All of these event records may be found within the Security Event Log, depending on the audit configuration employed on the endpoint. Huntress analysts have observed this workstation name in log entries during multiple incidents, going back to 19 April 2024, and several instances have been confirmed as Akira ransomware infections.</p><p>In several instances, Huntress has detected Cloudflared tunnels being installed on endpoints within the customer infrastructure:</p><p><strong>[.highlight]C:\ProgramData\cloudflared.exe tunnel run --token REDACTED[.highlight]</strong></p><p>When the file encryption malware is finally deployed, the command line appears as follows:</p><div class="w-embed"><code>C:\ProgramData\w.exe  -p=&lt;path&gt; -n=2</code></div><p>The exact path may vary depending upon the incident, and there may be multiple instances of the command seen in EDR telemetry. However, the purpose of sharing observed indicators from earlier stages of the attack cycle is to assist customers and others in the community by providing earlier warning of an impending attack, in hopes that file encryption malware deployment will be obviated, or at the very least, significantly inhibited.</p><h2>Recommendations</h2><p>Huntress recommends that organizations start with a thorough asset inventory, of both endpoints and exposed services. This should be followed by attack surface reduction, and monitoring via MDR and SIEM. Monitoring for and updating indicators identified as precursors to or being used during the early stages of an attack can inhibit or even obviate the threat actorâs follow-on efforts.</p><h2>IOCs</h2><div class="w-embed"><div style="font-family: var(--primary-font-family)"><table>

  /**
   * Finds and returns the submit button for the form containing the given element.
   * @param {HTMLElement} element - The HTML element inside the form.
   * @returns {HTMLButtonElement | null} The submit button of the form, or null if not found.
   */
  function findFormSubmitButton(element) {
    // First, ensure the element is valid
    if (!(element instanceof HTMLElement)) {
      console.error("Invalid element provided");
      return null;
    }

    // Find the closest form ancestor
    const neverbounceForm = element.closest("form");

    // If no form is found, return null
    if (!neverbounceForm) {
      console.warn("No form found for the given element");
      return null;
    }

    // Look for a submit button within the form
    const submitButton = neverbounceForm.querySelector('button[type="submit"], input[type="submit"]');

    // Return the submit button or null if not found
    return submitButton || null;
  }

  /**
   * Hook into field registration events
   * The nb:registered event is fired on the body every time
   * _nb.fields.registerListener is called and during page load for
   * each field auto registered.
   *
   * All `nb:` prefixed events contain a `detail` object
   * with the following params:
   *   `id` - a reference ID for the field & form
   *   `result` - a result object
   *   `error` - error object from request
   */
  document.querySelector("body").addEventListener("nb:registered", function (event) {
    // Get field using id from registered event
    let field = document.querySelector('[data-nb-id="' + event.detail.id + '"]');

    // Handle clear events (input has changed or an API error was returned)
    field.addEventListener("nb:clear", function (e) {
      // Check for errors
      if (e.detail.result && e.detail.result.isError()) {
        if (e.detail.result.isThrottled()) {
          // Do stuff when the verification is throttled
        }

        // Do stuff when other API errors occur
        // - Our recommendation is to hide any loaders and treat these emails the same way you would treat an Unknown email
        const submitButton = findFormSubmitButton(e.target);
        if (submitButton) {
          submitButton.disabled = "";
        }
        e.target.setCustomValidity("");
      } else {
        if (!e.target.value) {
          e.target.setCustomValidity("");
        } else {
          e.target.setCustomValidity("Please provide your company email. Free domain emails do not apply.");
        }
      }
      // Do stuff when input changes, (e.g. hide loader)
    });

    // Handle loading status (API request has been made)
    field.addEventListener("nb:loading", function (e) {
      // Do stuff while waiting on API response
      const submitButton = findFormSubmitButton(e.target);
      if (submitButton) {
        submitButton.disabled = "disabled";
      }
    });

    // Handle results (API call has succeeded)
    field.addEventListener("nb:result", function (e) {
      if (e.detail.result.is(_nb.settings.getAcceptedStatusCodes())) {

      // Do stuff when input doesn't even look like an email (i.e. missing @ or no .com/.net/etc...)
    });

      // Get the ID for this field
      var id = e.detail.id;

      // Get result object
      var result = e.detail.result;

        if (ignoreFreeDomains) {

    async function hubSpotFormSubmitHandler(eventForm) {
      if (eventForm.getAttribute("data-hubspot-form-id")) {
        var url = `https://api.hsforms.com/submissions/v3/integration/submit/3911692/${eventForm.getAttribute("data-hubspot-form-id")}`;

        var formData = new FormData(eventForm);

        let fields = [];
        let context = {};

        const contextFields = ["hutk", "ipAddress", "pageUri", "pageId", "pageName"];
        const companyFields = ["name"];
        const companyPrefix = "0-2/";
        const contractFields = ["title"];
        const contractPrefix = "2-14533744/";

        Array.from(formData.entries()).forEach((entry) => {
          const elements = document.getElementsByName(entry[0]);
          if (elements) {
            if (elements[0]) {
              const fieldName = elements[0].getAttribute("data-hubspot-fieldname") ?? elements[0].getAttribute("data-wfhsfieldname") ?? entry[0];
              if (elements[0]?.type == "hidden" && contextFields.includes(fieldName)) {
                if (!!entry[1]) {
                  context[fieldName] = entry[1];
                } else {
                  if (entry[0] == "pageUri") {
                    context.pageUri = window.location.href;
                  }
                }
              } else {
                let field = {
                  name: fieldName,
                  value: entry[1],
                };
                if (companyFields.includes(fieldName)) field.name = companyPrefix + fieldName;
                if (contractFields.includes(fieldName)) field.name = contractPrefix + fieldName;
                fields.push(field);
              }
            }
          }
        });

        const data = { fields: fields, context: context };

        const headers = new Headers();
        headers.append("Content-Type", "application/json");

        const response = await fetch(url, {
          method: "POST",
          body: JSON.stringify(data),
          headers: headers,
        });
        if (response?.status && response.status < 400) {
          return response;
          // setSuccessState(eventForm, successElements, failElements)
          // setSubmitButtonValue(submitButton, submitButtonValue)
        } else {
          console.error("Form submit error", response);
          throw new Error("Form submit Error");
          // setSubmitButtonValue(submitButton, submitButtonValue)
          // setFailedState(eventForm, successElements, failElements)
        }
      } else {
        console.log(`No HubSpot form id defined for ${eventForm.id}`);
        throw new Error(`No HubSpot form id defined for ${eventForm.id}`);
      }
    }

        let failElements = formBlock.querySelectorAll('[class*="w-form-fail"]');

        resetFormState(successElements, failElements);

        setSubmitButtonValue(submitButton, submitButtonWait, true);
        try {
          if (typeof handleFormSubmit !== "function") {
            throw new Error("handleFormSubmit is not a function");
          }

          hubSpotPreFormSubmitHandler(eventForm);

          if (typeof handlePreFormSubmit === "function") {
            try {
              handlePreFormSubmit(eventForm);
            } catch (error) {
              console.log("handlePreFormSubmit Error", error);

          } else {
            console.log("No pre form submit handler present");

          const result = await handleFormSubmit(eventForm);

          setSubmitButtonValue(submitButton, submitButtonValue);

            }

          }
          setSubmitButtonValue(submitButton, submitButtonValue);
          setFailedState(eventForm, successElements, failElements);
        }
      } catch (error) {
        console.log("formSubmitListener error", error);
      }
    }
