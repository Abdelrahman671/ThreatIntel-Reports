<h1 class="post-title entry-title"><a href="https://www.blackhillsinfosec.com/attacking-exchange-with-mailsniper/">Attacking Exchange with MailSniper</a></h1>
<div class="entry-content">
<p><a href="https://www.blackhillsinfosec.com/?team=beau-bullock">Beau Bullock</a> //
</p>
<p>I’ve added in a few modules to <a href="https://github.com/dafthack/MailSniper">MailSniper</a>&nbsp;that will assist in remote attacks against organizations that are hosting an externally facing Exchange server (OWA or EWS). Specifically, the modules are Get-GlobalAddressList, Invoke-PasswordSprayOWA, and Invoke-PasswordSprayEWS.</p>
<h3 class="wp-block-heading" id="h.zactdpbtqh9z">Get-GlobalAddressList</h3>
<p>Very often on external penetration tests we perform a reconnaissance phase that might yield us some email addresses or usernames of an organization. If we can successfully find valid credentials for any one of them, and the organization has an Outlook Web Access or Exchange Web Services portal it is possible to download the entire Global Address List from the Exchange server. So, from one valid credential we can now have access to all email addresses for every employee of an organization.</p>
<p>In trying to improve on the method <a href="https://twitter.com/OrOneEqualsOne">Carrie Roberts</a>&nbsp;wrote about in her <a href="https://www.blackhillsinfosec.com/?p=5282">blog post</a>&nbsp;regarding gathering the Global Address List from OWA manually I&#8217;ve automated this task into MailSniper. &nbsp;<a href="https://twitter.com/fullmetalcache">Brian Fehrman</a>&nbsp;found something very interesting in OWA. There is a function called FindPeople that will allow you to pull back the entire GAL with a single request. Unfortunately, this function is only implemented in Exchange version 2013. In testing, Get-GlobalAddressList that utilizes the FindPeople function was able to pull 4282 email addresses from a remote OWA portal in 10 seconds.</p>
<p><em><strong>The OWA “FindPeople” method requires you are using PowerShell version 3 or higher.</strong></em></p>
<p>For cases where the Exchange version is less than 2013 Get-GlobalAddressList fails back to enumerating the GAL from Exchange Web Services. This method can take a bit longer due to the fact that EWS will only let you search 100 results at a time. To get around this restriction I basically search AA through ZZ then sort/uniq the results.</p>
<p>To use it import the module into a PowerShell version 3 session then run something like this:</p>
<pre class="wp-block-preformatted">Get-GlobalAddressList&nbsp;-ExchHostname mail.domain.com -UserName

domain\username -Password Fall2016 -OutFile global-address-list.txt</pre>
<p></p>
<p>If Exchange version is 2013 it should look something like this:</p>
<figure class="wp-block-image aligncenter"><img decoding="async" src="https://blackhillsinfosec.com/wp-content/uploads/legacy/AHX4XgbWCOHsLKVrYiNxaJJgUkHMEaFNaA5PUhoZs3_rRbdqQfn308ImlIohdbtpvvE4tAMMVIngfgYXQKeUoegiQwYcoIyZUzVL518qYzd4vHSyecvBd7SzkNgMyApM5ChrDyto" alt /></figure>
<p>After obtaining the full email list you can then feed that back into password spraying attacks where you will likely gain more valid credentials.</p>
<p>Speaking of password spraying…</p>
<h3 class="wp-block-heading" id="h.rejgo8f7cere">Invoke-PasswordSprayOWA &amp; Invoke-PasswordSprayEWS</h3>
<p>I wrote in two modules for password spraying Outlook Web Access and Exchange Web Services to MailSniper. Password spraying is an attack where instead of trying to brute force many password attempts for a single user account we try one password across many user accounts. This helps avoid account lockout and will still result in us obtaining valid credentials as users still pick passwords like “Fall2016”. Both of the functions are multi-threaded. Just pass the -Threads option and specify a number of threads (15 seems to be a pretty good starting point).</p>
<p><em><strong>Both functions have a similar structure but one thing to note is that Invoke-PasswordSprayOWA requires PowerShell version 3 or higher.</strong></em></p>
<p>To use Invoke-PasswordSprayOWA import the module into a PowerShell version 3 session then run something like this:</p>
<pre class="wp-block-preformatted">Invoke-PasswordSprayOWA&nbsp;-ExchHostname mail.domain.com -UserList

.\userlist.txt -Password Fall2016 -Threads 15&nbsp;-OutFile owa-sprayed-creds.txt</pre>
<p></p>
<p>To use Invoke-PasswordSprayEWS import the module into a PowerShell session then run something like this:</p>
<pre class="wp-block-preformatted">Invoke-PasswordSprayEWS -ExchHostname mail.domain.com -UserList

.\userlist.txt -Password Fall2016 -Threads 15 -OutFile ews-sprayed-creds.txt</pre>
<p></p>
<p>You should start to see credentials populate in the terminal as MailSniper finds valid creds:</p>
<figure class="wp-block-image aligncenter"><img decoding="async" src="https://blackhillsinfosec.com/wp-content/uploads/legacy/cGNbTxtHRnVqcUPULKJjzCl7RXt2PNByp_RVg00JlhGh6k0P8iR353Mgyuv4h6ZwsIdEp1qlP0l-UuY-4fV8h6-t15WxdAg5dkh0TFw5ijmsjccmoUKULF_0Vf-fm-6NUkX1jpKW" alt /></figure>
<p>In testing I’ve noticed the EWS password spraying method is significantly faster. Both Invoke-PasswordSprayOWA and using Burp Intruder with 15 threads took about 1 hour and 45 minutes to complete spraying 10,000 users. Spraying that same list of users against EWS took only 9 minutes and 28 seconds.</p>
<p>For more information about MailSniper check out this <a href="https://www.blackhillsinfosec.com/?p=5296">blog post.</a></p>
