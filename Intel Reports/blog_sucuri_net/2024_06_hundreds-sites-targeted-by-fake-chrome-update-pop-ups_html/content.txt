</figure></div><div class="entry-container"><div class="entry-content-wrap"><div class="entry-content"><p><a href="https://blog.sucuri.net/2024/03/new-wave-of-socgholish-infections-impersonates-wordpress-plugins.html" target="_blank" rel="noopener">Fake Browser Update</a> campaigns are known for their deceptive tactics used by hackers to trick users into downloading malicious software. These campaigns typically involve injecting malicious code into a website, which then displays a popup message urging users to update their web browser. Clicking on the provided link usually results in downloading malware, such as a <a href="https://blog.sucuri.net/2024/02/remote-access-trojan-rat-types-mitigation-removal.html" target="_blank" rel="noopener">remote access trojan</a> or an infostealer.</p><p>One of the most notorious examples of this type of malware is <a href="https://blog.sucuri.net/2022/08/socgholish-5-years-of-massive-website-infections.html" target="_blank" rel="noopener">SocGholish</a>. However, our research team has been tracking a new campaign that has been active since late April 2024. This campaign follows a similar pattern but includes some unique characteristics that make it particularly concerning.</p><h2>How the malware works</h2><p>The infection process for this new fake browser update campaign begins with the injection of malicious code into vulnerable websites. Once the website is compromised, visitors are presented with the following misleading popup message a few seconds after the webpage loads:</p><p><picture decoding="async" class="aligncenter wp-image-34021 size-medium">

</p><p>The message, written in poor English, reads: <b>Warning Exploit Chrome Detect. Update Chrome Browser</b> and includes a large blue <b>Update</b> button. The pop-up is displayed even to users who are not using the Chrome browser, highlighting its deceptive (and amateurish) nature.</p><p>When a user clicks on the <b>Update </b>button, they are redirected to one of several malicious URLs designed to initiate a malware download. The URLs involved in this campaign include:</p><ul><li aria-level="1">hxxps://photoshop-adobe[.]shop/download/<strong>dwnl.php</strong></li><li aria-level="1">hxxps://brow-ser-update[.]top/download/<strong>dwnl.php</strong></li><li aria-level="1">hxxps://tinyurl[.]com/uoiqwje3</li></ul><p>Creation dates for the malicious domains indicate that the campaign could have been distributed as early as March:</p><ul><li aria-level="1">brow-ser-update[.]top &#8211; created on May 3, 2024.</li><li aria-level="1">photoshop-adobe[.]shop &#8211; created on March 14, 2024</li></ul><p>Although these URLs are no longer functional, they previously served malicious downloads commonly named <b>GoogleChrome-x86.msix</b> from server <a href="https://urlscan.io/search/#page.ip:%22185.196.9.156%22"><b>185.196.9[.]156</b></a>.</p><p><picture loading="lazy" decoding="async" class="aligncenter size-full wp-image-34022">

Sucuri’s <a href="https://sitecheck.sucuri.net/" target="_blank" rel="noopener">SiteCheck remote website scanner</a> detects this threat as <b>malware.fake_update.3</b>.</p><p><picture loading="lazy" decoding="async" class="aligncenter size-full wp-image-34023">

</p><h2>Technical details</h2><p>The malicious code injected into compromised websites is designed to execute a popup message using the following legitimate WordPress plugin: <b>Hustle – Email Marketing, Lead Generation, Optins, Popups</b> This plugin is commonly used for creating popups and opt-in forms, making it an ideal tool for attackers to exploit.</p><p>In the page source, the malicious injections appear as follows:</p><p><picture loading="lazy" decoding="async" class="aligncenter size-full wp-image-34024">

</picture>
</p><p>The code is often found in JSON files located in the <b>wp-content/uploads</b> directory, such as:</p><ul><li aria-level="1">wp-content/uploads/2024/04/<b>hustle-popup-20240425-111136-[redacted]-ffsf.json</b></li><li aria-level="1">wp-content/uploads/2024/05/<b>import.json</b></li></ul><p>The injected code may also be found stored in the <b>wp_hustle_modules_meta</b> table within the WordPress database.</p><p>Initially, we suspected a vulnerability within the Hustle plugin itself. However, further investigation revealed that the compromised sites were running the latest version of the plugin. A review of historical data from the Web Archive indicated that the affected sites did not have the Hustle plugin installed before the end of April. This suggests that the attackers gained access to the WordPress admin interface, installed the plugin, and then used its &#8220;Import&#8221; functionality to upload the malicious popup code.</p><p>This campaign underscores a growing trend among hackers to leverage legitimate plugins for malicious purposes. By doing so, they can evade detection by file scanners, as most plugins store their data within the WordPress database.</p><p>The addition of a single plugin can easily go unnoticed on a typical WordPress site, which often uses 15 or more plugins. This tactic has been employed in other notable WordPress infection campaigns, such as the <a href="https://blog.sucuri.net/2024/04/javascript-malware-switches-to-server-side-redirects-dns-txt-records-tds.html" target="_blank" rel="noopener">VexTrio DNS TXT redirects using the WPCode plugin</a> and the <a href="https://blog.sucuri.net/2024/03/sign1-malware-analysis-campaign-history-indicators-of-compromise.html" target="_blank" rel="noopener">Sign1 malware exploiting the Simple Custom CSS and JS plugin</a>.</p><h2>Protecting your site from fake chrome browser updates</h2><p>As a website owner, it’s important to take a proactive approach to security to mitigate risk from threats:</p><ol><li aria-level="1">Employ a “use it or lose it” policy on your website. That means regularly review all plugins and <b>remove any components that you don’t recognize</b> or aren’t in use.</li><li aria-level="1">Generate <a href="https://blog.sucuri.net/2024/01/how-to-make-strong-password.html" target="_blank" rel="noopener">strong and unique passwords</a> for all of your accounts, including admins, FTP, database, and hosting.</li><li aria-level="1">Regularly <a href="https://sucuri.net/malware-detection-scanning/" target="_blank" rel="noopener">monitor your website</a> and check for suspicious activity or unexpected website admin users.</li><li aria-level="1">Consider using <a href="https://blog.sucuri.net/2022/04/the-case-for-2fa-by-default-for-wordpress.html" target="_blank" rel="noopener">2FA</a> and <a href="https://docs.sucuri.net/website-firewall/whitelist-and-blacklist/protected-page/" target="_blank" rel="noopener">restricting access to your WordPress admin and sensitive pages</a> to allow access to only trusted IP addresses.</li><li aria-level="1">Always keep your website software patched and up-to-date, including your core CMS, plugins, themes, or any other extensible components.</li><li aria-level="1">Use a <a href="https://sucuri.net/website-firewall/" target="_blank" rel="noopener">web application firewall</a> to help prevent vulnerability exploits, malicious code, and hack attempts.</li></ol><p>If you think your website has been infected with malware but you’re not sure what to do next, we can help!
