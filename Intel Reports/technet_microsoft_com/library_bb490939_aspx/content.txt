							<div class="content "><h1 id="using-netsh">Using Netsh</h1><div class="display-flex justify-content-space-between align-items-center flex-wrap-wrap page-metadata-container">
										<div class="margin-right-xxs">
											<ul class="metadata page-metadata" data-bi-name="page info" lang="en-us" dir="ltr"><li>Article</li><li>
														<time class="is-invisible" data-article-date aria-label="Article review date" datetime="2009-09-11T00:00:00.000Z" data-article-date-source="ms.date">09/11/2009</time>
													</li></ul>

											<h2 id="ms--in-this-article" class="title is-6 margin-block-xs">In this article</h2>
										</nav><!-- <content> --><p><strong>Netsh</strong> is a command-line scripting utility that allows you to, either locally or remotely, display or modify the network configuration of a computer that is currently running. <strong>Netsh</strong> also provides a scripting feature that allows you to run a group of commands in batch mode against a specified computer. <strong>Netsh</strong> can also save a configuration script in a text file for archival purposes or to help you configure other servers.</p>
<h3 id="netsh-contexts">Netsh contexts</h3>
<p><strong>Netsh</strong> interacts with other operating system components using dynamic-link library (DLL) files. Each Netsh helper DLL provides an extensive set of features called a context, which is a group of commands specific to a networking component. These contexts extend the functionality of <strong>netsh</strong> by providing configuration and monitoring support for one or more services, utilities, or protocols. For example, Dhcpmon.dll provides <strong>netsh</strong> the context and set of commands necessary to configure and manage DHCP servers.</p>
<p>To run a <strong>netsh</strong> command, you must start <strong>netsh</strong> from the Cmd.exe prompt and change to the context that contains the command you want to use. The contexts that are available to you depend on which networking components you have installed. For example, if you type <strong>dhcp</strong> at the Netsh command prompt, you change to the DHCP context, but if you do not have DHCP installed the following message appears:</p>
<p>The following command was not found: dhcp.</p>
<p>For more information about Netsh contexts, see the following topics:</p>
<ul>
<li><p><a href="https://technet.microsoft.com/en-us/library/bb490940(v=technet.10)" data-linktype="external">Netsh commands for AAAA</a>&nbsp;</p>
</li>
<li><p><a href="https://technet.microsoft.com/en-us/library/bb490941(v=technet.10)" data-linktype="external">Netsh commands for DHCP</a>&nbsp;</p>
</li>
<li><p><a href="https://technet.microsoft.com/en-us/library/bb490942(v=technet.10)" data-linktype="external">Netsh diagnostic (diag) commands</a>&nbsp;</p>
</li>
<li><p><a href="https://technet.microsoft.com/en-us/library/bb490943(v=technet.10)" data-linktype="external">Netsh commands for Interface IP</a>&nbsp;</p>
</li>
<li><p><a href="https://technet.microsoft.com/en-us/library/bb490944(v=technet.10)" data-linktype="external">Netsh commands for RAS</a>&nbsp;</p>
</li>
<li><p><a href="https://technet.microsoft.com/en-us/library/bb490945(v=technet.10)" data-linktype="external">Netsh commands for Routing</a>&nbsp;</p>
</li>
<li><p><a href="https://technet.microsoft.com/en-us/library/bb490946(v=technet.10)" data-linktype="external">Netsh commands for WINS</a>&nbsp;</p>
</li>
</ul>
<h3 id="using-multiple-contexts">Using multiple contexts</h3>
<p>A context can exist within a context. For example, within the Routing context, you can change to the IP and IPX subcontexts.</p>
<p>To display a list of commands and subcontexts that you can use within a context, at the <strong>netsh</strong> prompt, type the context name, and then type either <strong>/?</strong> or <strong>help</strong>. For example, to display a list of subcontexts and commands that you can use in the Routing context, at the <strong>netsh</strong> prompt (that is, netsh&gt;), type either of the following:</p>
<p><strong>routing&nbsp;/?</strong></p>
<p><strong>routing&nbsp;help</strong></p>
<p>To perform tasks in another context without changing from your current context, type the context path of the command you want to use at the <strong>netsh</strong> prompt. For example, to add the Local Area Connection interface in the IGMP context without changing to the IGMP context, at the <strong>netsh</strong> prompt, type:</p>
<p><strong>routing ip igmp add interface "Local Area Connection" startupqueryinterval=21</strong></p>
<h3 id="running-netsh-commands-from-the-cmdexe-command-prompt">Running Netsh commands from the Cmd.exe command prompt</h3>
<p>When you run Netsh from the Cmd.exe command prompt, <strong>netsh</strong> uses the following syntax. To run these Netsh commands on a remote Windows&nbsp;2000&nbsp;Server, you must first use Remote Desktop Connection to connect to a Windows&nbsp;2000&nbsp;Server that is running Terminal Server. There might be functional differences between Netsh context commands on Windows&nbsp;2000 and Windows&nbsp;XP.</p>
<p>To view the command syntax, click the following command:</p>
<p>netsh&nbsp;</p>
<p>Netsh is a command-line scripting utility that allows you to, either locally or remotely, display or modify the network configuration of a currently running computer. Used without parameters, <strong>netsh</strong> opens the Netsh.exe command prompt (that is, netsh&gt;).</p>
<h4 id="syntax">Syntax</h4>
<p><strong>netsh</strong>&nbsp;[<strong>-a</strong>&nbsp;<em>AliasFile</em>] [<strong>-c</strong>&nbsp;<em>Context</em>] [<strong>-r</strong>&nbsp;<em>RemoteComputer</em>] [{<em>NetshCommand</em>|<strong>-f</strong>&nbsp;<em>ScriptFile</em>}]</p>
<h4 id="parameters">Parameters</h4>
<p><strong>-a</strong> &nbsp; <strong>:</strong> Returns you to the <strong>netsh</strong> prompt after running <em>AliasFile</em>.</p>
<p><em><strong>AliasFile</strong></em> &nbsp; <strong>:</strong> Specifies the name of the text file that contains one or more <strong>netsh</strong> commands.</p>
<p><strong>-c</strong> &nbsp; <strong>:</strong> Changes to the specified <strong>netsh</strong> context.</p>
<p><em><strong>Context</strong></em> &nbsp; <strong>:</strong> Specifies the <strong>netsh</strong> context. The following table lists the available <strong>netsh</strong> contexts.</p>
<table>
<colgroup>
<col>
<col>
</colgroup>
<thead>
<tr class="header">
<th><p>Context</p></th>
<th><p>Description</p></th>
</tr>
</thead>
<tbody>
<tr class="odd">
<td><p><a href="https://technet.microsoft.com/en-us/library/bb490940(v=technet.10)" data-linktype="external">AAAA</a></p></td>
<td><p>Shows and sets the configuration of the authentication, authorization, accounting, and auditing (AAAA) database used by the Internet Authentication Service (IAS) and the Routing and Remote Access service.</p></td>

<p><strong>-r</strong> &nbsp; <strong>:</strong> Configures a remote computer.</p>
<p><em><strong>RemoteComputer</strong></em> &nbsp; <strong>:</strong> Specifies the remote computer to configure.</p>
<p><em><strong>NetshCommand</strong></em> &nbsp; <strong>:</strong> Specifies the <strong>netsh</strong> command that you want to run.</p>
<p><strong>-f</strong> &nbsp; <strong>:</strong> Exits Netsh.exe after running the script.</p>
<p><em><strong>ScriptFile</strong></em> &nbsp; <strong>:</strong> Specifies the script that you want to run.</p>
<p><strong>/?</strong> <strong>:</strong> Displays help at the command prompt.</p>
<h4 id="remarks">Remarks</h4>
<ul>
<li><p>Using <strong>-r</strong>&nbsp;</p>
<p>If you specify <strong>-r</strong> followed by another command, <strong>netsh</strong> executes the command on the remote computer and then returns to the Cmd.exe command prompt. If you specify <strong>-r</strong> without another command, <strong>netsh</strong> opens in remote mode. The process is similar to using <strong>set machine</strong> at the Netsh command prompt. When you use <strong>-r</strong>, you set the target computer for the current instance of <strong>netsh</strong> only. After you exit and reenter <strong>netsh</strong>, the target computer is reset as the local computer. You can run <strong>netsh</strong> commands on a remote computer by specifiying a computer name stored in WINS, a UNC name, an internet name to be resolved by the DNS server, or a numerical IP address.</p>
</li>
</ul>
<h3 id="running-netsh-commands-from-the-netshexe-command-prompt">Running Netsh commands from the Netsh.exe command prompt</h3>
<p><strong>Netsh</strong> uses the following standard commands in all contexts that you can run from a Netsh.exe command prompt (that is, netsh&gt;). To run these Netsh commands on a remote Windows&nbsp;2000&nbsp;Server, you must first use Remote Desktop Connection to connect to a Windows&nbsp;2000&nbsp;Server that is running Terminal Server. There might be functional differences between Netsh context commands on Windows&nbsp;2000 and Windows&nbsp;XP.</p>
<p>To view the command syntax, click a command:</p>
<p>..</p>
<p>Moves to the context that is one level up.</p>
<h4 id="syntax-1">Syntax</h4>
<p><strong>..</strong>&nbsp;</p>
<h4 id="parameters-1">Parameters</h4>
<p><strong>/?</strong> <strong>:</strong> Displays help at the command prompt.</p>
<p>abort</p>
<p>Discards any changes made in offline mode. <strong>Abort</strong> has no effect in online mode.</p>
<h4 id="syntax-2">Syntax</h4>
<p><strong>abort</strong></p>
<h4 id="parameters-2">Parameters</h4>
<p><strong>/?</strong> <strong>:</strong> Displays help at the command prompt.</p>
<p><strong>add helper</strong></p>
<p>Installs the helper DLL in <strong>netsh</strong>.</p>
<h4 id="syntax-3">Syntax</h4>
<p><strong>add helper</strong>&nbsp;<em>DLLName</em>&nbsp;</p>
<h4 id="parameters-3">Parameters</h4>
<p><em><strong>DLLName</strong></em> &nbsp; <strong>:</strong> Required. Specifies the name of the helper DLL you want to install.</p>
<p><strong>/?</strong> <strong>:</strong> Displays help at the command prompt.</p>
<p><strong>alias</strong></p>
<p>Adds an alias that consists of a user-defined character string, which <strong>netsh</strong> treats as equivalent to another character string. Used without parameters, <strong>alias</strong> displays all available aliases.</p>
<h4 id="syntax-4">Syntax</h4>
<p><strong>alias</strong>&nbsp;[<em>AliasName</em>] [<em>string1</em> [<em>string2</em> ...]]</p>
<h4 id="parameters-4">Parameters</h4>
<p><strong>alias</strong> <strong>[</strong> <em><strong>AliasName</strong></em> <strong>]</strong> <strong>:</strong> Displays the specified alias.</p>
<p><strong>alias</strong> &nbsp; <strong>[</strong> <em><strong>AliasName</strong></em> <strong>][</strong> <em><strong>string1</strong></em> <strong>[</strong> <em><strong>string2</strong></em> <strong>...]]</strong> <strong>:</strong> Sets <em>AliasName</em> to the specified string(s).</p>
<p><strong>/?</strong> <strong>:</strong> Displays help at the command prompt.</p>
<h4 id="examples">Examples</h4>
<p>The following <strong>netsh</strong> sample script sets two <strong>netsh</strong> aliases, Shaddr and Shp, and then leaves the Netsh command prompt in the Interface IP context:</p>
<pre>alias shaddr show interface ip addr
alias shp show helpers
interface ip</pre>
<p>If you type <strong>shaddr</strong> at the Netsh command prompt, Netsh.exe interprets this as the command <strong>show interface ip addr</strong>. If you type <strong>shp</strong> at the Netsh command prompt, Netsh.exe interprets this as the command <strong>show helpers</strong>.</p>
<p>bye</p>
<p>Exits Netsh.exe.</p>
<h4 id="syntax-5">Syntax</h4>
<p><strong>bye</strong></p>
<h4 id="parameters-5">Parameters</h4>
<p><strong>/?</strong> <strong>:</strong> Displays help at the command prompt.</p>
<p>commit</p>
<p>Commits any changes made in the offline mode to the router. <strong>Commit</strong> has no effect in online mode.</p>
<h4 id="syntax-6">Syntax</h4>
<p><strong>commit</strong></p>
<h4 id="parameters-6">Parameters</h4>
<p><strong>/?</strong> <strong>:</strong> Displays help at the command prompt.</p>
<p>delete helper</p>
<p>Removes the helper DLL from <strong>netsh</strong>.</p>
<h4 id="syntax-7">Syntax</h4>
<p><strong>delete helper</strong>&nbsp;<em>DLLName</em></p>
<h4 id="parameters-7">Parameters</h4>
<p><em><strong>DLLName</strong></em> &nbsp; <strong>:</strong> Required. Specifies the name of the helper DLL you want to uninstall.</p>
<p><strong>/?</strong> <strong>:</strong> Displays help at the command prompt.</p>
<p>dump</p>
<p>Creates a script that contains the current configuration. If you save this script to a file, you can use the file to restore configuration settings that have been changed. Used without parameters, <strong>dump</strong> displays all of the <strong>netsh</strong> context configurations.</p>
<h4 id="syntax-8">Syntax</h4>
<p><strong>dump</strong> [<em>FileName</em>]</p>
<h4 id="parameters-8">Parameters</h4>
<p><strong>[</strong> <em><strong>FileName</strong></em> <strong>]</strong> <strong>:</strong> Specifies the name of the file to which you want to redirect output.</p>
<p><strong>/?</strong> <strong>:</strong> Displays help at the command prompt.</p>
<p>exec</p>
<p>Loads a script file and runs commands from it.</p>
<h4 id="syntax-9">Syntax</h4>
<p><strong>exec</strong>&nbsp;<em>ScriptFile</em></p>
<h4 id="parameters-9">Parameters</h4>
<p><em><strong>ScriptFile</strong></em> &nbsp; <strong>:</strong> Required. Specifies the name of the script that you want to load and run.</p>
<h4 id="remarks-1">Remarks</h4>
<ul>
<li>The <em>ScriptFile</em> can run on one or more computers.</li>
</ul>
<p>exit</p>
<p>Exits Netsh.exe.</p>
<h4 id="syntax-10">Syntax</h4>
<p><strong>exit</strong></p>
<h4 id="parameters-10">Parameters</h4>
<p><strong>/?</strong> <strong>:</strong> Displays help at the command prompt.</p>
<p>help</p>
<p>Displays help.</p>
<h4 id="syntax-11">Syntax</h4>
<p>{<strong>/?</strong>|<strong>?</strong>|<strong>help</strong>|<strong>h</strong>}</p>
<h4 id="parameters-11">Parameters</h4>
<p><strong>none</strong></p>
<p>offline</p>
<p>Sets the current mode to offline.</p>
<h4 id="syntax-12">Syntax</h4>
<p><strong>offline</strong></p>
<h4 id="parameters-12">Parameters</h4>
<p><strong>/?</strong> <strong>:</strong> Displays help at the command prompt.</p>
<h4 id="remarks-2">Remarks</h4>
<ul>
<li><p>Changes that you make in this mode are saved, but you need to run the <strong>commit</strong> or <strong>online</strong> command to set the changes in the router.</p>
</li>
<li><p>When you switch from offline mode to online mode, changes that you made in offline mode are reflected in the configuration that is currently running.</p>
</li>
<li><p>Changes that you make in online mode are immediately reflected in the configuration that is currently running.</p>
</li>
</ul>
<p>online</p>
<p>Sets the current mode to online.</p>
<h4 id="syntax-13">Syntax</h4>
<p><strong>online</strong></p>
<h4 id="parameters-13">Parameters</h4>
<p><strong>/?</strong> <strong>:</strong> Displays help at the command prompt.</p>
<h4 id="remarks-3">Remarks</h4>
<ul>
<li><p>Changes that you make in online mode are immediately reflected in the configuration that is currently running.</p>
</li>
<li><p>When you switch from offline mode to online mode, changes that you made in offline mode are reflected in the configuration that is currently running.</p>
</li>
</ul>
<p>popd</p>
<p>Restores a context from the stack.</p>
<h4 id="syntax-14">Syntax</h4>
<p><strong>popd</strong>&nbsp;</p>
<h4 id="parameters-14">Parameters</h4>
<p><strong>/?</strong> <strong>:</strong> Displays help at the command prompt.</p>
<h4 id="remarks-4">Remarks</h4>
<ul>
<li>Used in conjunction with <strong>pushd</strong>, <strong>popd</strong> enables you to change the context, run the command in the new context, and then resume the prior context.</li>
</ul>
<h4 id="examples-1">Examples</h4>
<p>The following sample script changes a context from the root context to the <strong>interface ip</strong> context, adds a static IP route, and then returns to the root context:</p>
<pre>netsh&gt;</pre>
<p><strong>pushd</strong></p>
<pre>netsh&gt;</pre>
<p><strong>interface ip</strong></p>
<pre>netsh interface ip&gt;</pre>
<p><strong>set address local static 10.0.0.9 255.0.0.0 10.0.0.1 1</strong></p>
<pre>netsh interface ip&gt;</pre>
<p><strong>popd</strong>&nbsp;</p>
<pre>netsh&gt;</pre>
<p>pushd</p>
<p>Saves the current context on a first-in-last-out (FILO) stack.</p>
<h4 id="syntax-15">Syntax</h4>
<p><strong>pushd</strong>&nbsp;</p>
<h4 id="parameters-15">Parameters</h4>
<p><strong>/?</strong> <strong>:</strong> Displays help at the command prompt.</p>
<h4 id="remarks-5">Remarks</h4>
<ul>
<li>Used in conjunction with <strong>popd</strong>, <strong>pushd</strong> enables you to change the context, run the command in the new context, and then resume the prior context.</li>
</ul>
<p>quit</p>
<p>Exits Netsh.exe.</p>
<h4 id="syntax-16">Syntax</h4>
<p><strong>quit</strong></p>
<h4 id="parameters-16">Parameters</h4>
<p><strong>/?</strong> <strong>:</strong> Displays help at the command prompt.</p>
<p>set file</p>
<p>Copies the Command Prompt window output to a file.</p>
<h4 id="syntax-17">Syntax</h4>
<p><strong>set file</strong> {<strong>open</strong>&nbsp;<em>FileName</em>|<strong>append</strong>&nbsp;<em>FileName</em>|<strong>close</strong>}</p>
<h4 id="parameters-17">Parameters</h4>
<p><strong>open</strong> &nbsp; <em><strong>FileName</strong></em> &nbsp; <strong>:</strong> Sends the Command Prompt window output to the specified file.</p>
<p><strong>append</strong> &nbsp; <em><strong>FileName</strong></em> &nbsp; <strong>:</strong> Appends the Command Prompt window output to the specified existing file.</p>
<p><strong>close</strong> &nbsp; <strong>:</strong> Stops sending output and closes a file.</p>
<p><strong>/?</strong> <strong>:</strong> Displays help at the command prompt.</p>
<h4 id="remarks-6">Remarks</h4>
<ul>
<li><p>Using <em>FileName</em>&nbsp;</p>
<p>If the specified <em>FileName</em> does not currently exist, <strong>netsh</strong> creates a new file with that name. If the specified <em>FileName</em> does currently exist, <strong>netsh</strong> overwrites the existing data.</p>
</li>
</ul>
<h4 id="examples-2">Examples</h4>
<p>To create a new log file called Session.log and copy all succeeding <strong>netsh</strong> input and output to Session.log, type:</p>
<p><strong>set file open c:\session.log</strong></p>
<p>set machine</p>
<p>Sets the current computer on which to perform configuration tasks. Used without parameters, <strong>set machine</strong> sets the local computer.</p>
<h4 id="syntax-18">Syntax</h4>
<p><strong>set machine</strong> [[<em>ComputerName</em>**=**]<em>string</em>]</p>
<h4 id="parameters-18">Parameters</h4>
<p><em><strong>ComputerName</strong></em> &nbsp; <strong>:</strong> Specifies the name of the computer on which to perform configuration tasks.</p>
<p><em><strong>string</strong></em> &nbsp; <strong>:</strong> Specifies the location of the remote computer.</p>
<p><strong>/?</strong> <strong>:</strong> Displays help at the command prompt.</p>
<h4 id="remarks-7">Remarks</h4>
<ul>
<li><p>Performing configuration tasks on multiple computers</p>
<p>You can run commands on multiple computers from a single script. You can use <strong>set machine</strong> in a script to specify a destination computer (for example, ComputerA), and then run the commands that follow <strong>set machine</strong> on this computer (that is, ComputerA). You can then use <strong>set machine</strong> to specify another destination computer (for example, ComputerB), and then run commands on this computer (that is, ComputerB).</p>
</li>
</ul>
<p>set mode</p>
<p>Sets the current mode to online or offline.</p>
<h4 id="syntax-19">Syntax</h4>
<p><strong>set mode</strong> {<strong>online</strong>|<strong>offline</strong>}</p>
<h4 id="parameters-19">Parameters</h4>
<p><strong>online</strong> &nbsp; <strong>:</strong> Sets the current mode to online.</p>
<p><strong>offline</strong> &nbsp; <strong>:</strong> Sets the current mode to offline.</p>
<p><strong>/?</strong> <strong>:</strong> Displays help at the command prompt.</p>
<p>show&nbsp;</p>
<p>Displays alias, helper, and mode information.</p>
<h4 id="syntax-20">Syntax</h4>
<p><strong>show</strong> {<strong>alias</strong>|<strong>helper</strong>|<strong>mode</strong>}</p>
<h4 id="parameters-20">Parameters</h4>
<p><strong>alias</strong> &nbsp; <strong>:</strong> Lists all defined aliases.</p>
<p><strong>helper</strong> &nbsp; <strong>:</strong> Lists all top-level helpers.</p>
<p><strong>mode</strong> &nbsp; <strong>:</strong> Displays the current mode.</p>
<p><strong>/?</strong> <strong>:</strong> Displays help at the command prompt.</p>
<p>unalias</p>
<p>Deletes the specified alias.</p>
<h4 id="syntax-21">Syntax</h4>

<h4 id="parameters-21">Parameters</h4>

<p><strong>/?</strong> <strong>:</strong> Displays help at the command prompt.</p>
