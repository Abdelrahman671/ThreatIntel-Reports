										</nav><!-- <content> --><p>The <strong>AddMonitor</strong> function installs a local port monitor and links the configuration, data, and monitor files.</p>
<h2 id="syntax">Syntax</h2>
<pre><code class="lang-C++">BOOL AddMonitor(
  _In_ LPTSTR pName,
  _In_ DWORD  Level,
  _In_ LPBYTE pMonitors
);
</code></pre>
<h2 id="parameters">Parameters</h2>
<dl> <dt>
<p><em>pName</em> [in]</p>
</dt> <dd>
<p>A pointer to a null-terminated string that specifies the name of the server on which the monitor should be installed. For systems that support only local installation of monitors, this string should be <strong>NULL</strong>.</p>
</dd> <dt>
<p><em>Level</em> [in]</p>
</dt> <dd>
<p>The version of the structure to which <em>pMonitors</em> points. This value must be 2.</p>
</dd> <dt>
<p><em>pMonitors</em> [in]</p>
</dt> <dd>
<p>A pointer to a <a href="monitor-info-2" data-linktype="relative-path"><strong>MONITOR_INFO_2</strong></a> structure. If the <strong>pEnvironment</strong> member of the <em>pMonitors</em> structure is <strong>NULL</strong>, the current environment of the caller (client), not of the destination (server), is used.</p>
<p>Note that the call will fail if the environment does not match the environment of the server, that is, you can only add a monitor that was written for the architecture of the server.</p>
</dd> </dl>
<h2 id="return-value">Return value</h2>
<p>If the function succeeds, the return value is a nonzero value.</p>
<p>If the function fails, the return value is zero.</p>
<h2 id="remarks">Remarks</h2>
<div class="NOTE">
<p>Note</p>
<p>This is a blocking or synchronous function and might not return immediately. How quickly this function returns depends on run-time factors such as network status, print server configuration, and printer driver implementation factors that are difficult to predict when writing an application. Calling this function from a thread that manages interaction with the user interface could make the application appear to be unresponsive.</p>
</div>
<p>The caller must have the <a href="/en-us/windows/desktop/SecAuthZ/authorization-constants" data-linktype="absolute-path">SeLoadDriverPrivilege</a>.</p>
<p>Before an application calls the <strong>AddMonitor</strong> function, all files required by the monitor must be copied to the SYSTEM32 directory.</p>
<p>To determine the port monitors that are currently installed, call the <a href="enummonitors" data-linktype="relative-path"><strong>EnumMonitors</strong></a> function.</p>
<p>To remove a monitor added by <strong>AddMonitor</strong>, call the <a href="deletemonitor" data-linktype="relative-path"><strong>DeleteMonitor</strong></a> function.</p>
