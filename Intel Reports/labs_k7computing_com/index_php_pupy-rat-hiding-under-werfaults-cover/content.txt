<div class="wpb_wrapper">
<p>
<p></p>



<p>We at K7 Labs recently identified an interesting technique used by threat actors to execute a Remote Admin Tool. We all know that <strong>WerFault.exe</strong> is used for the Windows Error Reporting. This blog describes how threat actors use the legitimate WerFault.exe to execute Pupy RAT on the victims’ machine.&nbsp;</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="855" height="737" src="https://labs.k7computing.com/wp-content/uploads/2023/01/Figure1.png" alt="" class="wp-image-25160" srcset="https://labs.k7computing.com/wp-content/uploads/2023/01/Figure1.png 855w, https://labs.k7computing.com/wp-content/uploads/2023/01/Figure1-300x259.png 300w, https://labs.k7computing.com/wp-content/uploads/2023/01/Figure1-768x662.png 768w" sizes="(max-width: 855px) 100vw, 855px" /><figcaption>Figure 1: Execution flow</figcaption></figure></div>



<p></p>



<h2><strong>Analysis of Binary&nbsp;</strong></h2>



<h3>Stage 1 &#8211; WerFault Execution</h3>



<p>Recently we came across an ISO image, <strong>recent inventory &amp; our specialties.iso</strong> from a twitter feed. The ISO<strong> </strong>contains four files, a legitimate WerFault.exe,a malicious DLL named faultrep.dll, a shortcut file named recent inventory &amp; our specialties.lnk<strong> </strong>and a XLS file named File.xls. The shortcut file has the same name as the ISO image. When the victim opens that shortcut file, it uses scriptrunner.exe LOLBin via cmd to execute WerFault.exe from the ISO.&nbsp;</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="405" height="392" src="https://labs.k7computing.com/wp-content/uploads/2023/01/Figure2.png" alt="" class="wp-image-25162" srcset="https://labs.k7computing.com/wp-content/uploads/2023/01/Figure2.png 405w, https://labs.k7computing.com/wp-content/uploads/2023/01/Figure2-300x290.png 300w" sizes="(max-width: 405px) 100vw, 405px" /><figcaption>Figure 2: ISO &amp; shortcut file</figcaption></figure></div>



<p></p>



<h3>Stage 2 &#8211; Pupy RAT loader</h3>



<p>Originally, Faultrep.dll is the name of DLL used by WerFault.exe is, which is present in the default windows folder. When WerFault.exe starts executing, it uses DLL Side-Loading technique to load the Faultrep.dll from the ISO and it has a dummy export function <strong>WerpInitiateCrashReporting </strong>similar to the original DLL.This malicious Faultrep.dll is compiled in C.</p>



<p>The DLL has a custom API resolving function with two arguments, DLL hash and Function hash.&nbsp;</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="572" height="189" src="https://labs.k7computing.com/wp-content/uploads/2023/01/Figure3.png" alt="" class="wp-image-25163" srcset="https://labs.k7computing.com/wp-content/uploads/2023/01/Figure3.png 572w, https://labs.k7computing.com/wp-content/uploads/2023/01/Figure3-300x99.png 300w" sizes="(max-width: 572px) 100vw, 572px" /><figcaption>Figure 3: API Resolving</figcaption></figure></div>



<p></p>



<p>We noticed that this loader uses the same API resolving function as <a href="https://www.crowdstrike.com/blog/guloader-malware-analysis/#:~:text=DJB2%20Hashes%20for%20Windows%20API%20Resolution">Guloader</a>. The DLLs resolved were kernel32 and advapi32.&nbsp;</p>



<p>After resolving the APIs, it starts to serve its purpose. Using the resolved function <strong>CreateThread, </strong>it creates two threads. The first thread opens a lure excel sheet named <strong>file.xls</strong> from the ISO.&nbsp;</p>



<p></p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="857" height="710" src="https://labs.k7computing.com/wp-content/uploads/2023/01/Figure4.png" alt="" class="wp-image-25164" srcset="https://labs.k7computing.com/wp-content/uploads/2023/01/Figure4.png 857w, https://labs.k7computing.com/wp-content/uploads/2023/01/Figure4-300x249.png 300w, https://labs.k7computing.com/wp-content/uploads/2023/01/Figure4-768x636.png 768w" sizes="(max-width: 857px) 100vw, 857px" /><figcaption>Figure 4: First thread opening Excel sheet</figcaption></figure></div>



<p>While manually resolving the function, we found that one of the functions it resolved was <strong>SystemFunction032 </strong>from the advapi32.dll. This function is undocumented in MSDN and on further searching we found the documentation on <a href="https://source.winehq.org/WineAPI/SystemFunction032.html">WineAPI</a>. With that documentation, we understood that the function is used for RC4 encryption and accepts two arguments: key<em> </em>and <em>data. </em>On further analysis we found the RC4 decryption function which contains the data and hard coded string as key.&nbsp;</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="754" height="233" src="https://labs.k7computing.com/wp-content/uploads/2023/01/Figure5.png" alt="" class="wp-image-25165" srcset="https://labs.k7computing.com/wp-content/uploads/2023/01/Figure5.png 754w, https://labs.k7computing.com/wp-content/uploads/2023/01/Figure5-300x93.png 300w" sizes="(max-width: 754px) 100vw, 754px" /><figcaption>Figure 5: Second thread doing RC4 decryption</figcaption></figure></div>



<p></p>



<p>The data is pointed to the address of the overlay. So we dumped the encrypted overlay data and using the key we further decrypted it. After decrypting the data, we confirmed that the data is a PE file with the magic bytes.</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" width="1024" height="593" src="https://labs.k7computing.com/wp-content/uploads/2023/01/Figure6-1024x593.png" alt="" class="wp-image-25166" srcset="https://labs.k7computing.com/wp-content/uploads/2023/01/Figure6-1024x593.png 1024w, https://labs.k7computing.com/wp-content/uploads/2023/01/Figure6-300x174.png 300w, https://labs.k7computing.com/wp-content/uploads/2023/01/Figure6-768x445.png 768w, https://labs.k7computing.com/wp-content/uploads/2023/01/Figure6.png 1084w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption>Figure 6: RC4 Decryption</figcaption></figure></div>



<p></p>



<p>We dumped the decrypted output data to a PE file. It was compiled with C &amp; Python and found that it is a <strong>Pupy RAT. </strong>This RAT is loaded into the memory and executed while WerFault.exe was executing in the front.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="819" height="325" src="https://labs.k7computing.com/wp-content/uploads/2023/01/Figure7-1.png" alt="" class="wp-image-25168" srcset="https://labs.k7computing.com/wp-content/uploads/2023/01/Figure7-1.png 819w, https://labs.k7computing.com/wp-content/uploads/2023/01/Figure7-1-300x119.png 300w, https://labs.k7computing.com/wp-content/uploads/2023/01/Figure7-1-768x305.png 768w" sizes="(max-width: 819px) 100vw, 819px" /><figcaption>Figure 7: Decrypted PE file</figcaption></figure></div>



<p></p>



<h3>Stage 3 &#8211; Pupy RAT</h3>



<p>Pupy RAT is an open-source cross platform Remote Admin Tool available in <a href="https://github.com/n1nj4sec/pupy">Github</a> According to the <a href="https://www.recordedfuture.com/pupyrat-malware-analysis">sources</a>, since 2013 it has possibly been used by APT33 and APT35 from Iran for&nbsp; cyber espionage operations like the one that was discovered in 2020 and targeted a major European energy organisation.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="1012" height="710" src="https://labs.k7computing.com/wp-content/uploads/2023/01/Figure8.png" alt="" class="wp-image-25169" srcset="https://labs.k7computing.com/wp-content/uploads/2023/01/Figure8.png 1012w, https://labs.k7computing.com/wp-content/uploads/2023/01/Figure8-300x210.png 300w, https://labs.k7computing.com/wp-content/uploads/2023/01/Figure8-768x539.png 768w" sizes="(max-width: 1012px) 100vw, 1012px" /><figcaption>Figure 8: Pupy RAT Github</figcaption></figure></div>



<p></p>



<p>It was executed from memory and based on the analysis of ReflectiveLoader function, is capable of executing any PE file in-memory, remotely. It tries to make a C2 connection in the background when the victim believes WerFault is running. Since the C2 was down at the time of analysis, RAT was unable to establish a connection for carrying out any further malicious activity. With the XLS sheet in Chinese, we believe that the victim is from China.&nbsp;</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="554" height="163" src="https://labs.k7computing.com/wp-content/uploads/2023/01/Figure9.png" alt="" class="wp-image-25170" srcset="https://labs.k7computing.com/wp-content/uploads/2023/01/Figure9.png 554w, https://labs.k7computing.com/wp-content/uploads/2023/01/Figure9-300x88.png 300w" sizes="(max-width: 554px) 100vw, 554px" /><figcaption>Figure 9: Pupy RAT C2 connection</figcaption></figure></div>



<p></p>



<p>We at K7 Labs provide detection against latest threats and also for this newer variant of Loader. Users are advised to use a reliable security product such as “K7 Total Security” and keep it up-to-date so as to safeguard their devices.</p>
