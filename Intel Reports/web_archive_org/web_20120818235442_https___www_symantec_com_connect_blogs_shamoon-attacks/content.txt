  <h1 class="node-title">The Shamoon Attacks</h1>
  
  <div class="node-posted">

            <p>W32.Disttrack is a new threat that is being used in specific targeted attacks against at least one organization in the energy sector.&nbsp; It is a destructive malware that corrupts files on a compromised computer and overwrites the MBR (Master Boot Record) in an effort to render a computer unusable.</p>
<p>W32.Disttrack consists of several components:</p>
<ol>
<li>Dropper&mdash;the main component and source of the original infection. It drops a number of other modules.</li>
<li>Wiper&mdash;this module is responsible for the destructive functionality of the threat.</li>
<li>Reporter&mdash;this module is responsible for reporting infection information back to the attacker.
<p>		&nbsp;</p></li>
</ol>
<h3>Dropper Component</h3>
<p>The Dropper component performs the following actions:</p>
<ul>
<li>Copies itself to <em>%System%\trksvr.exe</em></li>
<li>Drops the following files embedded into resources:
<ul>
<li>A 64-bit version of the dropper component: <em>%System%\trksrv.exe</em> (contained in the &ldquo;X509&rdquo; resource)</li>
<li>Reporter component: <em>%System%\netinit.exe</em> (contained in the &quot;PKCS7&quot; resource)</li>
<li>Wiper component: <em>%System%\[NAME SELECTED FROM LIST].exe</em> (contained in the &quot;PKCS12&quot; resource)<br/>
				<strong>Note:</strong> The name of the component is selected from the following list:
<ul>
<li>caclsrv</li>
<li>certutl</li>
<li>clean</li>
<li>ctrl</li>
<li>dfrag</li>
<li>dnslookup</li>
<li>dvdquery</li>
<li>event</li>
<li>extract</li>
<li>findfile</li>
<li>fsutl</li>
<li>gpget</li>
<li>iissrv</li>
<li>ipsecure</li>
<li>msinit</li>
<li>ntx</li>
<li>ntdsutl</li>
<li>ntfrsutil</li>
<li>ntnw</li>
<li>power</li>
<li>rdsadmin</li>
<li>regsys</li>
<li>routeman</li>
<li>rrasrv</li>
<li>sacses</li>
<li>sfmsc</li>
<li>sigver</li>
<li>smbinit</li>
<li>wcscript</li>
</ul>
</li>
</ul>
</li>
<li>Copies itself to the following network shares:
<ul>
<li>ADMIN$</li>
<li>C$\\WINDOWS</li>
<li>D$\\WINDOWS</li>
<li>E$\\WINDOWS</li>
</ul>
</li>
<li>Creates a task to execute itself</li>
<li>Creates the following service to start itself whenever Windows starts:
<ul>
<li><strong>Service name:</strong> TrkSvr</li>
<li><strong>Display name:</strong> Distributed Link Tracking Server</li>
<li><strong>Image path:</strong> %System%\trksvr.exe</li>
</ul>
</li>
</ul>
<h3>&nbsp;</h3>
<h3>Wiper Component</h3>
<p>The Wiper component includes the following functionality:</p>
<ul>
<li>Deletes an existing driver from the following location and overwrites it with another legitimate driver:
<ul>
<li><em>%System%\drivers\drdisk.sys</em></li>
<li>The device driver is a clean disk driver that enables user-mode applications to read and write to disk sectors. The driver is used to overwrite the computer&rsquo;s MBR but may be used for legitimate purposes.</li>
<li>The file is digitally signed</li>
</ul>
</li>
<li>Executes the following commands that collect file names, which will be overwritten and writes them to <em>f1.inf</em> and <em>f2.inf</em>:</li>
</ul>
<p><img src="/web/20120818235442im_/https://www.symantec.com/connect/imagebrowser/view/image/2425641/_original" alt="" src="/web/20120818235442im_/https://www.symantec.com/connect/imagebrowser/view/image/2425641/_original"/></p>
<p>&nbsp;</p>
<ul>
<li>Files from the <em>f1.inf</em> and <em>f2.inf</em> will be overwritten with the JPEG image shown below. Overwritten files are thus rendered useless.</li>
</ul>
<p><img src="/web/20120818235442im_/https://www.symantec.com/connect/imagebrowser/view/image/2425801/_original" alt="" src="/web/20120818235442im_/https://www.symantec.com/connect/imagebrowser/view/image/2425801/_original"/></p>
<p><strong><em>Figure 1.</em></strong><em> Image used to overwrite files</em></p>
<ul>
<li>Finally, the component will overwrite the MBR so that the compromised computer can no longer start</li>
</ul>
<p>The following string that points to the location of debug symbols was left in the Wiper component of this threat and gives an idea of where the component was located on the developer&rsquo;s computer:<br/>
	C:\Shamoon\ArabianGulf\wiper\release\wiper.pdb</p>
<p>&nbsp;</p>
<h3>Reporter Component</h3>
<p>The Reporter component is responsible for sending infection information back to the attacker. Information is sent as a HTTP GET request and is structured as follows:<br/>
	http://[DOMAIN]/ajax_modal/modal/data.asp?mydata=[MYDATA]&amp;uid=[UID]&amp;state=[STATE]</p>
<p>The following data is sent to the attacker:</p>
<ul>
<li>[DOMAIN]&mdash;a domain name</li>
<li>[MYDATA]&mdash;a number that specifies how many files were overwritten</li>
<li>[UID]&mdash;the IP address of the compromised computer</li>
<li>[STATE]&mdash;a random number</li>
</ul>
<p>Threats with such destructive payloads are unusual and are not typical of targeted attacks. Symantec Security Response is continuing to analyze this threat and will post more information as it becomes available. Symantec customers are protected from this threat, which our security products detect as <a href="https://web.archive.org/web/20120818235442/http://www.symantec.com/security_response/writeup.jsp?docid=2012-081608-0202-99&amp;om_rssid=sr-mixed30days">W32.Disttrack.</a></p>
