Its double extortion strategy significantly amplifies the threat it poses. Unit 42 researchers have been monitoring the Howling Scorpius ransomware group over the past year.\r\n\r\nHowling Scorpius targets small to medium-sized businesses in North America, Europe and Australia, across various sectors. Affected industries include education, consulting, government, manufacturing, telecommunications, technology and pharmaceuticals.\r\n\r\nOur research reveals that Howling Scorpius maintains and operates encryptors for Windows and Linux operating systems. We identified variants specifically designed for ESXi hosts. In addition, our findings have shown that this group is actively upgrading and enhancing its tool set, thus posing a greater risk for organizations.\r\n\r\nPalo Alto Networks customers are better protected against Akira ransomware from the Howling Scorpius ransomware group through the following products and services:\r\n\r\n \tCortex XDR and XSIAM\r\n \tCloud-Delivered Security Services for the Next-Generation Firewall, such as Advanced WildFire\r\n \tCortex Xpanse\r\n\r\nThe Unit 42 Incident Response team has responded to several Howling Scorpius ransomware incidents since the group first emerged in 2023. If you think you might have been compromised or have an urgent matter, contact the Unit 42 Incident Response team.\r\n\r\n\r\n\r\nRelated Unit 42 Topics\r\nCybercrime, Ransomware\r\n\r\n\r\n\r\nHowling Scorpius Overview\r\nFirst observed in March 2023 [PDF], Akira is a RaaS group we track as Howling Scorpius. This group employs a double extortion strategy, exfiltrating critical data from a network before executing its encryption process. This double extortion tactic allows the group to leak stolen data even if victims recover their systems without paying, maximizing the pressure to comply.\r\n\r\nHowling Scorpius operates a Tor-based leak site for Akira ransomware. The group uses the site to list victims and exfiltrate stolen data if they refuse to comply with ransom demands.\r\n\r\nThe Akira leak site has a retro-green look. Howling Scorpius also operates a separate Tor-based negotiation site, which victims can access using a dedicated password provided by the group. Figure 1 shows a screenshot of the Akira ransomware leak site.\r\n\r\n[caption id=\"attachment_137684\" align=\"alignnone\" width=\"900\"] Figure 1. Screenshot of the Akira Ransomware leak site in a Tor browser, from November 2024.[\/caption]\r\n\r\nThe Akira ransomware leak site displays a text-based console with a list of commands. The leaks command returns a list of victims who did not pay and includes links to download .torrent files. Viewers can then use these .torrent files to download the released data for those victims who did not pay their ransom.\r\n\r\nThis console also includes a news command that lists all compromised companies that it says date back as far as April 2023. The site describes the news command as \u201cupcoming data releases,\u201d and the results end with the most recent victims.\r\n\r\nThe group primarily targets small to medium-sized businesses across various regions and industries.\r\nTargeted Regions\r\nWhile Howling Scorpius has targeted organizations globally since 2023, the U.S. has emerged as the most affected country, according to Akira leak site data. Figure 2 highlights the top 10 affected countries based on this leak site data from March 2023-October 2024.\r\n\r\n[caption id=\"attachment_137686\" align=\"alignnone\" width=\"900\"] Figure 2. A column chart showing the countries impacted by Howling Scorpius from March 2023-October 2024.[\/caption]\r\nTargeted Industries\r\nAkira leak site data shows the group has impacted several industries, including manufacturing, professional and legal services, wholesale, retail and construction. Figure 3 shows the top 10 industries affected by this ransomware from March 2023-October 2024.\r\n\r\n[caption id=\"attachment_137688\" align=\"alignnone\" width=\"900\"] Figure 3. The distribution of the top 10 sectors affected by Howling Scorpius from March 2023-October 2024.[\/caption]\r\nTechnical Analysis of the Akira Ransomware Attack Lifecycle\r\nBelow is a technical analysis of Howling Scorpius operations mapped to the different stages of a cyberattack\u2019s lifecycle.\r\nInitial Access\r\nHowling Scorpius affiliates employ various methods to gain initial access to organizations. These include exploiting vulnerable virtual private network (VPN) services that lack multi-factor authentication (MFA) using valid accounts, often purchased through initial access brokers on the dark web.\r\n\r\nAffiliates also target external-facing services like Remote Desktop Protocol (RDP), and they conduct spear phishing campaigns.\r\n\r\nFigure 4 shows an alert raised by Cortex XDR for an example of a remote service creation. This specific alert involves using a service component of PsExec named PSEXESVC.exe to run a process from a remote system.\r\n\r\n[caption id=\"attachment_137690\" align=\"alignnone\" width=\"395\"] Figure 4. Cortex XDR alert for remote service creation from an uncommon source.[\/caption]\r\n\r\nThe security community has documented Howling Scorpius exploiting vulnerabilities in Cisco products, such as CVE-2020-3259 and CVE-2023-20269.\r\nCredentials Access\r\nLocal Credential Access Techniques\r\nHowling Scorpius affiliates employ various credential access techniques to extract credentials for privilege escalation. Mimikatz and LaZagne are their primary tools.\r\n\r\nAffiliates also often create a MiniDump of the LSASS process memory leveraging comsvcs.dll. Figure 5 shows an example of Cortex XDR detecting an example of comsvcs.dll used for this type of memory dump.\r\n\r\n[caption id=\"attachment_137692\" align=\"alignnone\" width=\"700\"] Figure 5. Cortex XDR detection alert of comsvcs.dll MiniDump of LSASS.[\/caption]\r\nKereberoasting\r\nHowling Scorpius affiliates employ the Kerberoasting attack to achieve control over service accounts and exploit credentials stored in memory.\r\nExtracting Credentials for Domain Control\r\nThe group\u2019s affiliates focus on extracting credentials from the Active Directory database to pursue comprehensive domain control. They copy the SYSTEM registry hive and NTDS.dit file from the domain controller (DC) to obtain a complete listing of user accounts and their corresponding domain password hashes.\r\nExploiting Compromised vCenter Instances\r\nIn cases where affiliates compromise a vCenter instance, they will perform the following activities:\r\n\r\n \tShutting down the DC's virtual machine (VM)\r\n \tCopying the DC's Virtual Machine Disk (VMDK) files to another VM they created beforehand\r\n \tExtracting the NTDT.dit and SYSTEM registry hive files (as reported by Rewterz)\r\n\r\nPersistence\r\nHowling Scorpius affiliates created new domain accounts to establish persistence. These accounts give these affiliates another form of access that does not require them to deploy tools or malware on the targeted systems. In addition, CISA reported [PDF] that the affiliates created new administrative domain accounts named itadm.\r\nDiscovery and Lateral Movement\r\nHowling Scorpius affiliates' lateral movement within compromised networks primarily involves exploiting remote services such as Remote Desktop Procol (RDP) and Server Message Block (SMB). The group also employs remote service creation and Windows Management Instrumentation (WMI) to further its reach.\r\n\r\nThese affiliates use network scanning tools like NetScan and Advanced IP Scanner to map the network and identify potential critical assets in the targeted organization for lateral movement. They also execute PowerShell and Windows Net Commands to query Active Directory for information on additional users and administrators.\r\nDefense Evasion\r\nBring Your Own Driver\r\nHowling Scorpius affiliates use tools that abuse the Zemana antimalware driver to terminate antimalware-related processes. Figure 6 below shows information from an alert raised in Cortex XDR for attempting to create the malicious Zemana driver.\r\n\r\n[caption id=\"attachment_137694\" align=\"alignnone\" width=\"500\"] Figure 6. Cortex XDR alert for the attempt to use the Zemana antimalware driver.[\/caption]\r\nAnti Virus Disablement\r\nAffiliates have also tried to disable Windows Defender Real-Time Protection using PowerShell, and they tried to uninstall the EDR agents installed on infected systems.\r\nBring Your Own VM\r\nAffiliates sometimes create their own VMs. Within these VMs, they disable security tools. They then mount the hypervisor host's storage drives onto the VM, shutting down any processes using those files to unlock running VM files. After successfully mounting the drives and unlocking all targeted files, they execute the ransomware within the new VM (as reported by CyberCX), bypassing the host's security tools.\r\nExfiltration\r\nHowling Scorpius affiliates usually exfiltrate data from compromised hosts using WinRAR and a combination of WinSCP, RClone and FileZilla, through the File Transfer Protocol (FTP). Below is an example of a data exfiltration attempt we observed:\r\n\"C:\\Program Files\\WinRAR\\WinRAR.exe\" a -ep1 -scul -r0 -iext -imon1 -- . \"[REDACTED]\\Company\\[REDACTED]\" [REDACTED]\\Company\\HR \"[REDACTED]\\Company\\Human Resources Management - HR\"\r\nAkira Ransomware Encryptors\r\nThis section details the different encryptors for Akira ransomware that Howling Scorpius uses for Windows and Linux operating systems.\r\nRansom Note\r\nUpon successful encryption, Akira ransomware encryptors create a ransom note named akira_readme.txt that provides victims instructions for how to interact with the group. This file includes links to both the leak site and the negotiation site.\r\n\r\nThe file also contains a unique code that victims must enter on the negotiation site to facilitate communication with the attackers and potential ransom discussions. Figure 7 shows an example of the akira_readme.txt file.\r\n\r\n[caption id=\"attachment_137696\" align=\"alignnone\" width=\"900\"] Figure 7. An example of the akira_readme.txt file content.[\/caption]\r\nWindows Variant\r\nExecution\r\nUpon execution, the Windows variant of the Akira ransomware encryptor will attempt to delete shadow copies using the following PowerShell command:\r\n\r\n \tpowershell.exe -Command \"Get-WmiObject Win32_Shadowcopy | Remove-WmiObject\"\r\n\r\nCommand-Line Arguments\r\nThe Windows variant of the Akira ransomware encryptor uses the following command-line arguments:\r\n\r\n \t-p\\--encryption_path \u2013 Contains the root directory of the encryption process\r\n \t-s\\--share_file \u2013 Contains the targeted network drive path\r\n \t-n\\--encryption_percent \u2013 Controls the amount of data to be encrypted within each file\r\n \t--fork \u2013 Creates a child process for the encryption process\r\n \t-l \u2013 Writes the list of drives into the log file\r\n \t-localonly \u2013 Prevents the encryption of remote drives\r\n \t-e\/\u2013exclude \u2013 Contains files to exclude from the encryption process\r\n\r\nFigure 8 below shows the Windows encryptor for the Akira ransomware detected and prevented by Cortex XDR.\r\n\r\n[caption id=\"attachment_137698\" align=\"alignnone\" width=\"500\"] Figure 8. Windows encryptor for Akira ransomware detected by Cortex XDR.[\/caption]\r\nEncryption\r\nAkira ransomware\u2019s Windows variant uses a hybrid approach to encrypt data. It encrypts the content of the files using the ChaCha20 algorithm.\r\n\r\nThe threat then encrypts the ChaCha20 key using a hard-coded RSA public key. The encryptor supports full and partial encryption, controlled through the aforementioned command-line parameter.\r\n\r\nAvast published a decryptor in June 2023 exploiting a vulnerability in Akira's encryption scheme. However, CyberCX found a sample in VirusTotal that revealed that Howling Scorpius had patched this vulnerability within three days of its public disclosure.\r\n\r\nIn February 2024, we identified updates in the Howling Scorpius codebase. These updates included implementing support for the KCipher2 algorithm alongside ChaCha20. Encrypted files would use the .akira extension.\r\n\r\nThe list of the targeted file extensions and excluded directories the Howling Scorpius Windows encryptor uses can be found in Appendix A.\r\nThe Megazord Variant\r\nIn August 2023, a new strain of ransomware called Megazord appeared. This strain, written in Rust, has a ransom note with content similar to that of Akira ransomware and points to the same negotiation site. This indicates Howling Scorpius is also the same group behind Megazord.\r\n\r\nBesides being written in Rust, Megazord variants differ from Akira encryptors by the following characteristics:\r\n\r\n \tUsing a different file extension for encrypted files \u2013 .powerranges\r\n \tUsing a different name for the ransom note \u2013 powerranges.txt\r\n\r\nIn addition, Megazord encryptors execute several commands to terminate and stop a list of services and processes that could affect the encryption process. For the complete list of commands executed by Megazord encryptors, please view Appendix B.\r\n\r\nThe Megazord strain has a new layer of protection, requiring a password as an execution condition (defined by the \u2013id command-line argument). Figure 9 demonstrates how Cortex XDR detects and prevents Megazord.\r\n\r\n[caption id=\"attachment_137700\" align=\"alignnone\" width=\"500\"] Figure 9. Megazord encryptor detected by Cortex XDR.[\/caption]\r\nUpdated Version\r\nWhile looking for additional Megazord encryptors, we came across two samples that were compiled in March 2024, which had two new command-line arguments affecting the execution flow of the encryptor. The command-line argument \u2013proc allows the attackers to turn off the termination of processes and services, and the \u2013dirs command-line argument allows the attackers to ignore blocklisted directories.\r\n\r\nFigure 10 shows the updated help menu from a Megazord sample.\r\n\r\n[caption id=\"attachment_137702\" align=\"alignnone\" width=\"900\"] Figure 10. Megazord variant help menu.[\/caption]\r\nThe Possibility of Different Operators Sharing the Megazord Ransomware\r\nAnother unique sample we found differs primarily by its ransom note. This new ransom note raises the possibility that Megazord might not be exclusive to Howling Scorpius, although we cannot confirm this yet.\r\n\r\nThe new ransom note contains distinct language and a different means of communicating via Telegram, which hints at the involvement of a different threat actor. Figure 11 shows the new ransom note.\r\n\r\n[caption id=\"attachment_137704\" align=\"alignnone\" width=\"900\"] Figure 11. The new Megazord ransom note.[\/caption]\r\nLinux\/ESXi Variant\r\nBased on the internal strings and naming conventions we observed in the Linux\/ESXi variants of Akira ransomware, we assess that these samples were initially designed to run on ESXi systems. Some samples we encountered executed ESXCLI commands, strengthening our assessment. Figure 12 shows an example of an internal string found in one of the Linux\/ESXi variants.\r\n\r\n[caption id=\"attachment_137706\" align=\"alignnone\" width=\"900\"] Figure 12. An example of an internal string of Linux\/ESXi samples.[\/caption]\r\nExecution\r\nIn some of the Akira Linux variants we have encountered, attackers changed the syslog logs directory to \/tmp. It\u2019s likely they did this to disable logging and disable the Core Dump file using the following ESXCLI commands:\r\n\r\n \t\/bin\/sh -c 'esxcli system syslog config set --logdir=\/tmp'\r\n \t\/bin\/sh -c 'esxcli system syslog reload'\r\n \t\/bin\/sh -c 'esxcli system coredump file set --unconfigure'\r\n\r\nCommand-Line Arguments\r\nThe Linux\/ESXi variant of the Akira ransomware encryptor uses the following command-line arguments:\r\n\r\n \t-p\\--encryption_path \u2013 Specifies the root directory of the encryption process\r\n \t-s\\--share_file \u2013 Specifies the targeted network drive path\r\n \t-n\\--encryption_percent \u2013 Controls the amount of data to be encrypted within each file\r\n \t--fork \u2013 Creates a child process for the encryption process\r\n\r\nFigure 13 demonstrates the detection and prevention of the Linux\/ESXi variant by Cortex XDR.\r\n\r\n[caption id=\"attachment_137708\" align=\"alignnone\" width=\"500\"] Figure 13. Howling Scorpius Linux\/ESXi encryptor detected by Cortex XDR.[\/caption]\r\nEncryption\r\nAkira ransomware's Linux\/ESXi variant uses a hybrid encryption approach to lock data, the same as its Windows variant. The Linux\/ESXi variant encrypts the symmetric key used to encrypt the content of the targeted files with an embedded RSA public key.\r\n\r\nThis variant uses several symmetric encryption algorithms for the targeted file encryption, such as AES, CAMELLIA, DES and IDEA. Like the Windows version, this variant supports full and partial encryption controlled through the aforementioned command-line parameters.\r\n\r\nThe list of targeted file extensions and excluded directories by Akira ransomware's Linux\/ESXi encryptor can be found in Appendix C.\r\nAkira v2\r\nIn April 2024, CISA's #StopRansomware efforts [PDF] revealed a new variant of the Akira ransomware's Linux\/ESXi encryptor called Akira_v2. This Rust-based variant introduces a new command-line argument set and expanded capabilities.\r\n\r\nLike Megazord, Akira_v2 also adds a new layer of protection by requesting a password using the \u2013id argument as a run condition. In addition, by using the --vmonly argument, Akira_v2 adds the ability to encrypt VM files only.\r\n\r\nFigure 14 shows the help menu unique to this variant.\r\n\r\n[caption id=\"attachment_137710\" align=\"alignnone\" width=\"878\"] Figure 14. Akira_v2 help menu.[\/caption]\r\n\r\nThis variant targets the following file extensions:\r\n\r\n \t.vmdk\r\n \t.vmem\r\n \t.vmx\r\n \t.log\r\n \t.vswp\r\n \t.vmsd\r\n \t.vmsn\r\n\r\nBy using the \u2013stopvm argument, the variant adds the ability to turn off running VMs. It does so by executing the following command:\r\n\r\n \tvim-cmd vmsvc\/getallvms | tail -n +2 | awk '{system(\"vim-cmd vmsvc\/power.off \" $1)}'.\r\n\r\nAlso, Akira_v2 uses yet another ransom note file, named akiranew.txt, which still points to the same negotiation site used for the original version of Akira ransomware. Akira_v2 also changes the extension added to encrypted files to .akiranew.\r\n\r\nFigure 15 demonstrates how Cortex XDR detects and prevents the Akira_v2 variant.\r\n\r\n[caption id=\"attachment_137712\" align=\"alignnone\" width=\"500\"] Figure 15. Howling Scorpius\u2019s Akira_v2 encryptor detected by Cortex XDR.[\/caption]\r\nConclusion\r\nThis threat assessment demonstrates how Akira ransomware operates, solidifying Howling Scorpius' position among the top five most active ransomware groups despite its relatively recent emergence.                 <h2><a id="post-137681-_4lt92rr5muov"></a>Executive Summary</h2>
<p>Emerging in early 2023, the Howling Scorpius ransomware group is the entity behind the Akira ransomware-as-a-service (RaaS), which has consistently ranked in recent months among <a href="https://unit42.paloaltonetworks.com/unit-42-ransomware-leak-site-data-analysis/" target="_blank" rel="noopener">the top five most active</a> ransomware groups. Its double extortion strategy significantly amplifies the threat it poses. Unit 42 researchers have been monitoring the Howling Scorpius ransomware group over the past year.</p>
<p>Howling Scorpius targets small to medium-sized businesses in North America, Europe and Australia, across various sectors. Affected industries include education, consulting, government, manufacturing, telecommunications, technology and pharmaceuticals.</p>
<p>Our research reveals that Howling Scorpius maintains and operates encryptors for Windows and Linux operating systems. We identified variants specifically designed for ESXi hosts. In addition, our findings have shown that this group is actively upgrading and enhancing its tool set, thus posing a greater risk for organizations.</p>
<p>Palo Alto Networks customers are better protected against Akira ransomware from the Howling Scorpius ransomware group through the following products and services:</p>
<ul>
<li><a href="https://www.paloaltonetworks.com/cortex/cortex-xdr" target="_blank" rel="noopener"><strong>Cortex XDR</strong></a> and <a href="https://docs-cortex.paloaltonetworks.com/p/XSIAM" target="_blank" rel="noopener"><strong>XSIAM</strong></a></li>
<li><a href="https://docs.paloaltonetworks.com/cdss" target="_blank" rel="noopener"><strong>Cloud-Delivered Security Services</strong></a> for the <a href="https://docs.paloaltonetworks.com/ngfw" target="_blank" rel="noopener"><strong>Next-Generation Firewall</strong></a>, such as <a href="https://docs.paloaltonetworks.com/wildfire" target="_blank" rel="noopener"><strong>Advanced WildFire</strong></a></li>
<li><a href="https://www.paloaltonetworks.com/cortex/cortex-xpanse" target="_blank" rel="noopener"><strong>Cortex Xpanse</strong></a></li>
</ul>

If you think you might have been compromised or have an urgent matter, contact the <a href="https://start.paloaltonetworks.com/contact-unit42.html" target="_blank" rel="noopener"><strong>Unit 42 Incident Response team</strong></a>.</p>

<h2><a id="post-137681-_y6qimc4unhv5"></a>Howling Scorpius Overview</h2>
<p><a href="https://www.cisa.gov/sites/default/files/2024-04/aa24-109a-stopransomware-akira-ransomware_2.pdf" target="_blank" rel="noopener">First observed in March 2023 [PDF]</a>, Akira is a RaaS group we track as Howling Scorpius. This group employs a double extortion strategy, exfiltrating critical data from a network before executing its encryption process. This double extortion tactic allows the group to leak stolen data even if victims recover their systems without paying, maximizing the pressure to comply.</p>
<p>Howling Scorpius operates a Tor-based leak site for Akira ransomware. The group uses the site to list victims and exfiltrate stolen data if they refuse to comply with ransom demands.</p>
<p>The Akira leak site has a retro-green look. Howling Scorpius also operates a separate Tor-based negotiation site, which victims can access using a dedicated password provided by the group. Figure 1 shows a screenshot of the Akira ransomware leak site.</p>
<figure id="attachment_137684" aria-describedby="caption-attachment-137684" style="width: 900px" class="wp-caption alignnone"><img  class="wp-image-137684 lozad"  data-src="https://unit42.paloaltonetworks.com/wp-content/uploads/2024/11/word-image-494478-137681-1.png" alt="Screenshot of a computer screen displaying a message from a hacker group named &quot;AKIRA&quot; on a dark-themed interface with green text. <p>The Akira ransomware leak site displays a text-based console with a list of commands. The <span style="font-family: 'courier new', courier, monospace;">leaks</span> command returns a list of victims who did not pay and includes links to download <span style="font-family: 'courier new', courier, monospace;">.torrent</span> files. Viewers can then use these <span style="font-family: 'courier new', courier, monospace;">.torrent</span> files to download the released data for those victims who did not pay their ransom.</p>
<p>This console also includes a news command that lists all compromised companies that it says date back as far as April 2023. The site describes the <span style="font-family: 'courier new', courier, monospace;">news</span> command as “upcoming data releases,” and the results end with the most recent victims.</p>
<p>The group primarily targets small to medium-sized businesses across various regions and industries.</p>

<h2><a id="post-137681-_wven14kmgum2"></a>Technical Analysis of the Akira Ransomware Attack Lifecycle</h2>
<p>Below is a technical analysis of Howling Scorpius operations mapped to the different stages of a <a href="https://www.paloaltonetworks.com/cyberpedia/how-to-break-the-cyber-attack-lifecycle" target="_blank" rel="noopener">cyberattack’s lifecycle</a>.</p>
<h3><a id="post-137681-_oxb4uthmhw3o"></a>Initial Access</h3>
<p>Howling Scorpius affiliates employ various methods to gain initial access to organizations. These include exploiting vulnerable virtual private network (VPN) services that lack multi-factor authentication (MFA) using <a href="https://attack.mitre.org/techniques/T1078/" target="_blank" rel="noopener">valid accounts</a>, often purchased through initial access brokers on the dark web.</p>
<p>Affiliates also target <a href="https://attack.mitre.org/techniques/T1133/" target="_blank" rel="noopener">external-facing services</a> like Remote Desktop Protocol (RDP), and they conduct spear phishing campaigns.</p>
<p>Figure 4 shows an alert raised by Cortex XDR for an example of a remote service creation. This specific alert involves using a service component of <a href="https://learn.microsoft.com/en-us/sysinternals/downloads/psexec#introduction" target="_blank" rel="noopener">PsExec</a> named <span style="font-family: 'courier new', courier, monospace;">PSEXESVC.exe</span> to run a process from a remote system.</p>
<figure id="attachment_137690" aria-describedby="caption-attachment-137690" style="width: 395px" class="wp-caption alignnone"><img  class="wp-image-137690 lozad"  data-src="https://unit42.paloaltonetworks.com/wp-content/uploads/2024/11/word-image-505702-137681-4.png" alt="A screenshot from Cortex XDR displaying a service start notification by a remote host, using PSExec.exe from the C:\WINDOWS directory, and statistics about its remote service activity listed under XDR Analytics BIOC." width="395" height="290" /><figcaption id="caption-attachment-137690" class="wp-caption-text">Figure 4. Cortex XDR alert for remote service creation from an uncommon source.</figcaption></figure>
<p>The security community has documented Howling Scorpius <a href="https://www.cisa.gov/news-events/cybersecurity-advisories/aa24-109a" target="_blank" rel="noopener">exploiting vulnerabilities</a> in Cisco products, such as <a href="https://nvd.nist.gov/vuln/detail/CVE-2020-3259" target="_blank" rel="noopener">CVE-2020-3259</a> and <a href="https://nvd.nist.gov/vuln/detail/CVE-2023-20269" target="_blank" rel="noopener">CVE-2023-20269</a>.</p>
<h3><a id="post-137681-_kbqvboxn0cqe"></a>Credentials Access</h3>
<h4><a id="post-137681-_uzwdf4cfvrb6"></a>Local Credential Access Techniques</h4>
<p>Howling Scorpius affiliates employ various credential access techniques to extract credentials for privilege escalation. <a href="https://attack.mitre.org/software/S0002/" target="_blank" rel="noopener">Mimikatz</a> and <a href="https://attack.mitre.org/software/S0349/" target="_blank" rel="noopener">LaZagne</a> are their primary tools.</p>
<p>Affiliates also often create a <a href="https://attack.mitre.org/techniques/T1003/001/" target="_blank" rel="noopener"><span style="font-family: 'courier new', courier, monospace;">MiniDump</span> of the LSASS process memory</a> leveraging <span style="font-family: 'courier new', courier, monospace;">comsvcs.dll</span>. Figure 5 shows an example of Cortex XDR detecting an example of <span style="font-family: 'courier new', courier, monospace;">comsvcs.dll</span> used for this type of memory dump.</p>
<figure id="attachment_137692" aria-describedby="caption-attachment-137692" style="width: 700px" class="wp-caption alignnone"><img  class="wp-image-137692 lozad"  data-src="https://unit42.paloaltonetworks.com/wp-content/uploads/2024/11/word-image-508384-137681-5.png" alt="Alert icon with an &quot;A&quot; inside a pink triangle, indicating a security notification about a memory dump performed using comsvcs.dll on Lsass.exe, commonly associated with unauthorized access attempts. Below the icon is the file name &quot;rundll32.exe&quot; and a command line path for further technical details." width="700" height="393" data-srcset="https://unit42.paloaltonetworks.com/wp-content/uploads/2024/11/word-image-508384-137681-5.png 980w, https://unit42.paloaltonetworks.com/wp-content/uploads/2024/11/word-image-508384-137681-5-784x440.png 784w, https://unit42.paloaltonetworks.com/wp-content/uploads/2024/11/word-image-508384-137681-5-768x431.png 768w" sizes="(max-width: 700px) 100vw, 700px" /><figcaption id="caption-attachment-137692" class="wp-caption-text">Figure 5. Cortex XDR detection alert of <span style="font-family: 'courier new', courier, monospace;">comsvcs.dll</span> MiniDump of LSASS.</figcaption></figure>
<h4><a id="post-137681-_7w8v7zfyma2i"></a>Kereberoasting</h4>
<p><a href="https://www.cisa.gov/news-events/cybersecurity-advisories/aa24-109a" target="_blank" rel="noopener">Howling Scorpius affiliates employ</a> the <a href="https://attack.mitre.org/techniques/T1558/003/" target="_blank" rel="noopener">Kerberoasting</a> attack to achieve control over service accounts and exploit credentials stored in memory.</p>
<h4><a id="post-137681-_ib238dhf978r"></a>Extracting Credentials for Domain Control</h4>
<p>The group’s affiliates focus on extracting credentials from the <a href="https://en.wikipedia.org/wiki/Active_Directory" target="_blank" rel="noopener">Active Directory</a> database to pursue comprehensive domain control. They copy the <a href="https://attack.mitre.org/techniques/T1003/002/" target="_blank" rel="noopener">SYSTEM registry hive</a> and <a href="https://attack.mitre.org/techniques/T1003/003/" target="_blank" rel="noopener"><span style="font-family: 'courier new', courier, monospace;">NTDS.dit</span></a> file from the <a href="https://en.wikipedia.org/wiki/Domain_controller" target="_blank" rel="noopener">domain controller</a> (DC) to obtain a complete listing of user accounts and their corresponding domain password hashes.</p>
<h4><a id="post-137681-_g7vl4g4czbj1"></a>Exploiting Compromised vCenter Instances</h4>
<p>In cases where affiliates compromise a vCenter instance, they will perform the following activities:</p>
<ul>
<li>Shutting down the DC's virtual machine (VM)</li>
<li>Copying the DC's <a href="https://en.wikipedia.org/wiki/VMDK" target="_blank" rel="noopener">Virtual Machine Disk</a> (VMDK) files to another VM they created beforehand</li>
<li>Extracting the <span style="font-family: 'courier new', courier, monospace;">NTDT.dit</span> and SYSTEM registry hive files (as reported by <a href="https://www.rewterz.com/threat-advisory/akira-ransomware-exfiltrates-domain-controller-files-by-privilege-escalation" target="_blank" rel="noopener">Rewterz</a>)</li>
</ul>
<h3><a id="post-137681-_eb5m3itklzh3"></a>Persistence</h3>
<p>Howling Scorpius affiliates created <a href="https://attack.mitre.org/versions/v14/techniques/T1136/002/" target="_blank" rel="noopener">new domain accounts</a> to establish persistence. These accounts give these affiliates another form of access that does not require them to deploy tools or malware on the targeted systems. In addition, <a href="https://www.cisa.gov/sites/default/files/2024-04/aa24-109a-stopransomware-akira-ransomware_2.pdf" target="_blank" rel="noopener">CISA reported [PDF]</a> that the affiliates created new administrative domain accounts named <span style="font-family: 'courier new', courier, monospace;">itadm</span>.</p>
<h3><a id="post-137681-_xvavuqe3l2ui"></a>Discovery and Lateral Movement</h3>
<p>Howling Scorpius affiliates' lateral movement within compromised networks primarily involves exploiting remote services such as <a href="https://learn.microsoft.com/en-us/troubleshoot/windows-server/remote/understanding-remote-desktop-protocol" target="_blank" rel="noopener">Remote Desktop Procol (RDP)</a> and <a href="https://learn.microsoft.com/en-us/previous-versions/windows/it-pro/windows-server-2012-r2-and-2012/hh831795(v=ws.11)" target="_blank" rel="noopener">Server Message Block (SMB)</a>. The group also employs remote service creation and <a href="https://learn.microsoft.com/en-us/windows/win32/wmisdk/wmi-start-page" target="_blank" rel="noopener">Windows Management Instrumentation (WMI)</a> to further its reach.</p>
<p>These affiliates use network scanning tools like <a href="https://www.softperfect.com/products/networkscanner/" target="_blank" rel="noopener">NetScan</a> and <a href="https://www.advanced-ip-scanner.com/" target="_blank" rel="noopener">Advanced IP Scanner</a> to map the network and identify potential critical assets in the targeted organization for lateral movement. They also execute PowerShell and <a href="https://learn.microsoft.com/en-us/troubleshoot/windows-server/networking/net-commands-on-operating-systems" target="_blank" rel="noopener">Windows Net Commands</a> to query Active Directory for information on additional users and administrators.</p>
<h3><a id="post-137681-_3l6dgch46psr"></a>Defense Evasion</h3>
<h4><a id="post-137681-_wscodjg2iki7"></a>Bring Your Own Driver</h4>

<h4><a id="post-137681-_tc6dbwgczfee"></a>Anti Virus Disablement</h4>
<p>Affiliates have also tried to disable Windows Defender Real-Time Protection using PowerShell, and they tried to uninstall the EDR agents installed on infected systems.</p>
<h4><a id="post-137681-_wk6xzsci6gbc"></a>Bring Your Own VM</h4>
<p>Affiliates sometimes create their own VMs. Within these VMs, they disable security tools. They then mount the <a href="https://en.wikipedia.org/wiki/Hypervisor" target="_blank" rel="noopener">hypervisor</a> host's storage drives onto the VM, shutting down any processes using those files to unlock running VM files. After successfully mounting the drives and unlocking all targeted files, they execute the ransomware within the new VM (as reported by <a href="https://cybercx.com.au/blog/akira-ransomware/" target="_blank" rel="noopener">CyberCX</a>), bypassing the host's security tools.</p>

<h3><a id="post-137681-_f7x68pymq252"></a>Windows Variant</h3>
<h4><a id="post-137681-_qwk9r7k1m09s"></a>Execution</h4>
<p>Upon execution, the Windows variant of the Akira ransomware encryptor will attempt to delete shadow copies using the following PowerShell command:</p>
<ul>
<li><span style="font-family: 'courier new', courier, monospace;">powershell.exe -Command "Get-WmiObject Win32_Shadowcopy | Remove-WmiObject"</span></li>
</ul>
<h4><a id="post-137681-_v0zba9pp9qa1"></a>Command-Line Arguments</h4>
<p>The Windows variant of the Akira ransomware encryptor uses the following command-line arguments:</p>
<ul>
<li><span style="font-family: 'courier new', courier, monospace;">-p\--encryption_path</span> – Contains the root directory of the encryption process</li>
<li><span style="font-family: 'courier new', courier, monospace;">-s\--share_file</span> – Contains the targeted network drive path</li>
<li><span style="font-family: 'courier new', courier, monospace;">-n\--encryption_percent</span> – Controls the amount of data to be encrypted within each file</li>
<li><span style="font-family: 'courier new', courier, monospace;">--fork</span> – Creates a child process for the encryption process</li>
<li><span style="font-family: 'courier new', courier, monospace;">-l</span> – Writes the list of drives into the log file</li>
<li><span style="font-family: 'courier new', courier, monospace;">-localonly</span> – Prevents the encryption of remote drives</li>
<li><span style="font-family: 'courier new', courier, monospace;">-e/–exclude</span> – Contains files to exclude from the encryption process</li>
</ul>

<h4><a id="post-137681-_d0u6yvd0lbnn"></a>Encryption</h4>
<p>Akira ransomware’s Windows variant uses a hybrid approach to encrypt data. It encrypts the content of the files using the <a href="https://en.wikipedia.org/wiki/ChaCha20-Poly1305" target="_blank" rel="noopener">ChaCha20</a> algorithm.</p>
<p>The threat then encrypts the ChaCha20 key using a hard-coded RSA public key. The encryptor supports full and partial encryption, controlled through the aforementioned command-line parameter.</p>
<p><a href="https://decoded.avast.io/threatresearch/decrypted-akira-ransomware/#how_to" target="_blank" rel="noopener">Avast published a decryptor</a> in June 2023 exploiting a vulnerability in Akira's encryption scheme. However, <a href="https://cybercx.com.au/blog/akira-ransomware/" target="_blank" rel="noopener">CyberCX</a> found a sample in VirusTotal that revealed that Howling Scorpius had patched this vulnerability within three days of its public disclosure.</p>
<p>In February 2024, we identified <a href="https://github.com/PaloAltoNetworks/Unit42-timely-threat-intel/blob/main/2024-02-27-IOCs-for-Akira-Ransomware.txt" target="_blank" rel="noopener">updates in the Howling Scorpius codebase</a>. These updates included implementing support for the <a href="https://en.wikipedia.org/wiki/KCipher-2" target="_blank" rel="noopener">KCipher2</a> algorithm alongside ChaCha20. Encrypted files would use the .akira extension.</p>
<p>The list of the targeted file extensions and excluded directories the Howling Scorpius Windows encryptor uses can be found in <a href="#post-137681-_hbmx1kf754k6">Appendix A</a>.</p>
<h3><a id="post-137681-_hjbeheg93vwg"></a>The Megazord Variant</h3>
<p>In August 2023, a new strain of ransomware called Megazord appeared. This strain, written in <a href="https://www.rust-lang.org/" target="_blank" rel="noopener">Rust</a>, has a ransom note with content similar to that of Akira ransomware and points to the same negotiation site. This indicates Howling Scorpius is also the same group behind Megazord.</p>
<p>Besides being written in Rust, Megazord variants differ from Akira encryptors by the following characteristics:</p>
<ul>
<li>Using a different file extension for encrypted files – <span style="font-family: 'courier new', courier, monospace;">.powerranges</span></li>
<li>Using a different name for the ransom note – <span style="font-family: 'courier new', courier, monospace;">powerranges.txt</span></li>
</ul>
<p>In addition, Megazord encryptors execute several commands to terminate and stop a list of services and processes that could affect the encryption process. For the complete list of commands executed by Megazord encryptors, please view <a href="#post-137681-_fxmdjsw2h5or">Appendix B</a>.</p>
<p>The Megazord strain has a new layer of protection, requiring a password as an execution condition (defined by the <span style="font-family: 'courier new', courier, monospace;">–id</span> command-line argument). Figure 9 demonstrates how Cortex XDR detects and prevents Megazord.</p>

<h4><a id="post-137681-_sf7vq7stqqrn"></a>Encryption</h4>
<p>Akira ransomware's Linux/ESXi variant uses a hybrid encryption approach to lock data, the same as its Windows variant. The Linux/ESXi variant encrypts the symmetric key used to encrypt the content of the targeted files with an embedded RSA public key.</p>
<p>This variant uses several symmetric encryption algorithms for the targeted file encryption, such as <a href="https://en.wikipedia.org/wiki/Advanced_Encryption_Standard" target="_blank" rel="noopener">AES</a>, <a href="https://en.wikipedia.org/wiki/Camellia_(cipher)" target="_blank" rel="noopener">CAMELLIA</a>, <a href="https://en.wikipedia.org/wiki/Data_Encryption_Standard" target="_blank" rel="noopener">DES</a> and <a href="https://en.wikipedia.org/wiki/International_Data_Encryption_Algorithm" target="_blank" rel="noopener">IDEA</a>. Like the Windows version, this variant supports full and partial encryption controlled through the aforementioned command-line parameters.</p>
<p>The list of targeted file extensions and excluded directories by Akira ransomware's Linux/ESXi encryptor can be found in <a href="#post-137681-_ifjvlpe9zc6j">Appendix C</a>.</p>
<h4><a id="post-137681-_42k5lh89razm"></a>Akira v2</h4>
<p>In April 2024, CISA's<a href="https://www.cisa.gov/sites/default/files/2024-04/aa24-109a-stopransomware-akira-ransomware_2.pdf" target="_blank" rel="noopener"> #StopRansomware efforts [PDF]</a> revealed a new variant of the Akira ransomware's Linux/ESXi encryptor called Akira_v2. This Rust-based variant introduces a new command-line argument set and expanded capabilities.</p>
<p>Like Megazord, Akira_v2 also adds a new layer of protection by requesting a password using the <span style="font-family: 'courier new', courier, monospace;">–id</span> argument as a run condition. In addition, by using the <span style="font-family: 'courier new', courier, monospace;">--vmonly</span> argument, Akira_v2 adds the ability to encrypt VM files only.</p>

<h2><a id="post-137681-_2an8ryq91inv"></a>Conclusion</h2>
<p>This threat assessment demonstrates how Akira ransomware operates, solidifying Howling Scorpius' position among the top five most active ransomware groups despite its relatively recent emergence. <span style="font-weight: 400;">The group’s developers and affiliates appear to be actively developing new strains and capabilities, as well as making ongoing changes to the toolkit, which contributes to the persistence and prevalence of the ransomware.</span></p>
<p>We showed how the group used different ransomware variants in tandem, its infection vectors and activity within an infected organization. This group's recent focus on virtualization hosts to affect more endpoints and circumvent security measures means organizations should take the threat seriously and prepare against it.</p>

(function($) {

	var subscription_form_button = document.getElementById('unit42footerSubscription_form_button');
	window.captchaComplete = function() {
		subscribeSuccess = true;
		if ($(mail).val() != '' && isEmail($(mail).val())) {
			$(subscription_form_button).removeClass('is-disabled');
		}
		setTimeout(function() {
			$(email).focus();
			$('.g-recaptcha iframe').attr('tabindex', '-1');
		}, 100)
	}
	window.captchaExpires = function() {
		subscribeSuccess = false;
		$(subscription_form_button).addClass('is-disabled', true);
	} 
	$(subscription_form).submit(function(e) {
		e.preventDefault();
		e.stopImmediatePropagation();
		updateEmailMask();
		var success = true;
		var form = $(this);
		var mail = form.find('input[name="Email"]');
		
		if (mail.val() === '') {
			mail.addClass('has-error');
			showError(1);
			success = false;	
		}  else if (!isEmail(mail.val())){
			showError(2);
			success = false;
		}
		else {
			mail.removeClass('has-error');
			$('.error-mail').addClass('d-none');
		}

		if (!subscribeSuccess) {
			$('.error-recaptcha').removeClass('d-none');
		} else {
			$('.error-recaptcha').addClass('d-none');
		}

		if (success && subscribeSuccess) {
			$.ajax({
				type: 'POST',
				url: form.attr('action'),
				data: form.serialize(),
				beforeSend: function() {
                 form.find('button').addClass('is-loading');
    			},
				success: function(msg) {
					form.find('.form-success-message').html('<p class="success-message">You have been successfully subscribed</p>');
					form.find('button').removeClass('is-loading');
					$(email).val('');
					clearError();
				},
				error: function(jqXHR, textStatus, errorThrown) {
					$(subscription_form_button).addClass('is-disabled', true);
					form.find('button').removeClass('is-loading');
					
				}
			});
		}
		return false;
	});
	function showError(error_type){
		if(error_type == 1) {
			$('.error-mail').text("Please enter the email address.").addClass('error-show');
			$(subscription_form_button).addClass('is-disabled');
		} else if(error_type == 2){
			$('.error-mail').text("Please provide a valid e-mail address.").addClass('error-show');
			$(subscription_form_button).addClass('is-disabled');
		}	
		$(subscription_form_button).removeClass('is-loading');
	}
	function clearError(){
		$('.error-mail').text("").removeClass('error-show');;
		$(subscription_form_button).removeClass('is-loading');
		$(subscription_form_button).removeClass('is-disabled');
	}
	
	$(email).on('input', function (event) { 
		var email = $(this).val();
		if (isEmail(email) ) {			
			clearError();
		} else if(email == ""){
			clearError();
		} else{
			showError(2);			  
		}
    });

	function isEmail(email) {
		var re = /^(([^<>()\[\]\\.,;:\s@"]+(\.[^<>()\[\]\\.,;:\s@"]+)*)|(".+"))@((\[[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\])|(([a-zA-Z\-0-9]+\.)+[a-zA-Z]{2,}))$/;
		return re.test(String(email).toLowerCase());
	}
	
	var captcha_loaded = false;
         if(!captcha_loaded){
            // recaptcha on foucs call
            $(document).on('change paste keyup', '#newsletter-email', function () {
               if($('.g-recaptcha').hasClass('d-none')){
                $('.g-recaptcha').removeClass('d-none');

                  script.type = 'text/javascript';
                  script.src = 'https://www.google.com/recaptcha/api.js?hl=en_US';
                  head.appendChild(script);
               }
            });
        }
	
	function updateEmailMask() {
		var email = $("#unit42footerSubscription_form input[name='Email']").val();
		if (email && email.trim() != '') {
			var maskedEmail = maskEmailAddress(email);
			$("#unit42footerSubscription_form input[name='emailFormMask']").val(maskedEmail);
		}
	}

	function maskEmailAddress (emailAddress) {
		function mask(str) {
			var strLen = str.length;
			if (strLen > 4) {
				return str.substr(0, 1) + str.substr(1, strLen - 1).replace(/\w/g, '*') + str.substr(-1,1);
			}
			return str.replace(/\w/g, '*');
		}
		return emailAddress.replace(/([\w.]+)@([\w.]+)(\.[\w.]+)/g, function (m, p1, p2, p3) {
			return mask(p1) + '@' + mask(p2) + p3;
		});

		return emailAddress;
	}
