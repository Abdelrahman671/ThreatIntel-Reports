        <h1>August 2024 Patch Tuesday: Six Zero-Days and Six Critical Vulnerabilities Amid 85 CVEs</h1>
        
        <div class="publish_info">
            <p>August 14, 2024</p>

       <span><p>Microsoft has released security updates for 85 vulnerabilities in its August 2024 Patch Tuesday rollout. These include six actively exploited zero-days (CVE-2024-38213, CVE-2024-38193, CVE-2024-38189, CVE-2024-38178, CVE-2024-38107, CVE-2024-38106). Among the updates is a fix for one of the vulnerabilities related to a ‘downgrade’ attack (CVE-2024-21302). Six of the vulnerabilities are rated <b>Critical</b> in severity, while the remaining 79 are rated <b>Important</b> or <b>Moderate</b>.</p>
<h2>August 2024 Risk Analysis</h2>
<p>This month’s leading risk type is <a href="https://www.crowdstrike.com/cybersecurity-101/privilege-escalation/">elevation of privilege</a> (37%) followed by <a href="https://www.crowdstrike.com/cybersecurity-101/remote-code-execution-rce/">remote code execution</a> (35%) and <a href="https://www.crowdstrike.com/cybersecurity101/secops/data-leakage/">information disclosure</a> (8%).</p>
<p><img src="/content/dam/crowdstrike/www/en-us/wp/2024/08/PT-Aug-2024-blog-Fig-1.png" alt="Figure 1. Breakdown of August 2024 Patch Tuesday attack types" width="468"/></p>
<p>Figure 1. Breakdown of August 2024 Patch Tuesday attack types</p>
<p> </p>
<p>Windows products received the most patches this month with 43, followed by Extended Security Update (ESU) with 21 and Microsoft Office with 8.</p>
<p><img src="/content/dam/crowdstrike/www/en-us/wp/2024/08/PT-Aug-2024-Fig-2.png" alt="Figure 2. Breakdown of product families affected by August 2024 Patch Tuesday" width="468"/></p>
<p>Figure 2. Breakdown of product families affected by August 2024 Patch Tuesday</p>
<p> </p>
<h2>Actively Exploited Zero-Day Vulnerability in Microsoft Project</h2>
<p>Microsoft Project received a patch for <a href="https://msrc.microsoft.com/update-guide/en-US/advisory/CVE-2024-38189">CVE-2024-38189</a>, which has a severity of <b>Important</b> and a <b>CVSS</b> score of <b>8.8</b>. This vulnerability allows an attacker to perform remote code execution by convincing a victim to click a link or open a malicious file. Microsoft advises customers to block macros from running in Office products and enable notifications.</p>

<p><i>Table 1. Zero-day in Microsoft Project</i></p>
<h2>Actively Exploited Zero-Day Vulnerabilities in Windows Ancillary Function Driver for WinSock</h2>
<p>Windows Ancillary Function Driver for WinSock received a patch for <a href="https://msrc.microsoft.com/update-guide/en-US/advisory/CVE-2024-38193">CVE-2024-38193</a>, with a severity of <b>Important</b> and a <b>CVSS</b> score of <b>7.8</b>. These elevation of privilege vulnerabilities allow an attacker with low-level authentication to elevate access to obtain SYSTEM privileges. Windows Ancillary Function Driver for WinSock is primarily responsible for handling network-related functions. Typically, these kinds of vulnerabilities are chained with remote code execution vulnerabilities to compromise systems. Details of the vulnerability, including the proof of concept for exploiting it, have not been publicly disclosed by Microsoft.</p>
<table><tbody><tr><td>Severity</td><td>CVSS Score</td><td>CVE</td><td>Description</td></tr><tr><td>Important</td><td>7.8</td><td>CVE-2024-38193</td><td>Windows Ancillary Function Driver for WinSock Elevation of Privilege Vulnerability</td></tr></tbody></table>
<p><i>Table 2. Zero-days in Windows Ancillary Function Driver for WinSock and Windows Power Dependency Coordinator</i></p>
<h2>Actively Exploited Zero-Day Vulnerabilities in Windows Power Dependency Coordinator</h2>
<p>Windows Power Dependency Coordinator received a patch for <a href="https://msrc.microsoft.com/update-guide/en-US/advisory/CVE-2024-38107">CVE-2024-38107</a>, with a severity of <b>Important</b> and a <b>CVSS</b> score of <b>7.8</b>. This elevation of privilege vulnerability allows an attacker with low-level authentication to elevate access to obtain SYSTEM privileges. The attack is focused on the management of power states like sleep and wake functions in Windows. Typically, these kinds of vulnerabilities are chained with remote code execution vulnerabilities to compromise systems. Details of the vulnerability, including the proof of concept for exploiting it, have not been publicly disclosed by Microsoft.</p>
<table><tbody><tr><td>Severity</td><td>CVSS Score</td><td>CVE</td><td>Description</td></tr><tr><td>Important</td><td>7.8</td><td>CVE-2024-38107</td><td>Windows Power Dependency Coordinator Elevation of Privilege Vulnerability</td></tr></tbody></table>
<p><i>Table 3. Zero-days in Windows Ancillary Function Driver for WinSock and Windows Power Dependency Coordinator</i></p>
<h2>Actively Exploited Zero-Day Vulnerability in Scripting Engine</h2>
<p>Scripting Engine received a patch for <a href="https://msrc.microsoft.com/update-guide/en-US/advisory/CVE-2024-38178">CVE-2024-38178</a>, which has a severity of <b>Important</b> and a <b>CVSS</b> score of <b>7.5</b>. This vulnerability requires an authenticated user to be using Edge in Internet Explorer mode and clicking on a compromised URL. This allows an unauthenticated attacker to initiate remote code execution.</p>
<table><tbody><tr><td>Severity</td><td>CVSS Score</td><td>CVE</td><td>Description</td></tr><tr><td>Important</td><td>7.5</td><td>CVE-2024-38178</td><td>Scripting Engine Memory Corruption Vulnerability</td></tr></tbody></table>
<p><i>Table 4. Zero-day in Scripting Engine</i></p>
<h2>Actively Exploited Zero-Day Vulnerability in Windows Kernel</h2>
<p>The Windows kernel received a patch for <a href="https://msrc.microsoft.com/update-guide/en-US/advisory/CVE-2024-38106">CVE-2024-38106</a>, which has a severity of <b>Important</b> and a <b>CVSS</b> score of <b>7.0</b>. This is an elevation of privilege vulnerability that leads to SYSTEM privileges. This attack is highly complex and requires the attacker to win a race condition; however, some race conditions may be easier than others. Details of the vulnerability, including a proof of concept, have not been publicly disclosed by Microsoft.</p>
<table><tbody><tr><td>Severity</td><td>CVSS Score</td><td>CVE</td><td>Description</td></tr><tr><td>Important</td><td>7.0</td><td>CVE-2024-38106</td><td>Windows Kernel Elevation of Privilege Vulnerability</td></tr></tbody></table>
<p><i>Table 5. Zero-day in the Windows kernel</i></p>
<h2>Actively Exploited Zero-Day Vulnerability in Windows Mark of the Web Security</h2>
<p>Windows Mark of the Web Security received a patch for <a href="https://msrc.microsoft.com/update-guide/en-US/advisory/CVE-2024-38213">CVE-2024-38213</a>, which has a severity of <b>Important</b> and a <b>CVSS</b> score of <b>6.5</b>. A vulnerability in the Windows Defender SmartScreen allows an attacker to bypass security warnings when a malicious file is downloaded. This security feature has been bypassed multiple times over the years (<a href="https://www.crowdstrike.com/blog/patch-tuesday-analysis-march-2023/">March 2023</a>, <a href="https://www.crowdstrike.com/blog/patch-tuesday-analysis-july-2023/">July 2023</a>, <a href="https://www.crowdstrike.com/blog/patch-tuesday-analysis-november-2023/">November 2023</a>, <a href="https://www.crowdstrike.com/blog/patch-tuesday-analysis-february-2024/">February 2024</a>), making it a prime target for threat actors utilizing it in phishing attacks.</p>
<table><tbody><tr><td>Severity</td><td>CVSS Score</td><td>CVE</td><td>Description</td></tr><tr><td>Moderate</td><td>6.5</td><td>CVE-2024-38213</td><td>Windows Mark of the Web Security Feature Bypass Vulnerability</td></tr></tbody></table>
<p><i>Table 6. Zero-day in Windows Mark of the Web Security</i></p>
<h2>Critical Vulnerabilities in Windows Network Virtualization, TCP/IP, Reliable Multicast Transport Driver, Azure Health Bot and Secure Boot</h2>
<p><a href="https://msrc.microsoft.com/update-guide/en-US/advisory/CVE-2024-38063">CVE-2024-38063</a> is a <b>Critical</b> remote code execution (RCE) vulnerability affecting Windows TCP/IP and has a <b>CVSS</b> score of <b>9.8</b>. Successful exploitation of this vulnerability allows an attacker to execute code remotely. Microsoft has not provided other details but suggests disabling IPv6 as only IPv6 packets can be abused to exploit this vulnerability. The attack complexity is low, and exploitation is more likely, per Microsoft.</p>
<p><a href="https://msrc.microsoft.com/update-guide/en-US/advisory/CVE-2024-38140">CVE-2024-38140</a> is a <b>Critical</b> RCE vulnerability affecting Windows Reliable Multicast Transport Driver (RMCAST) and has a <b>CVSS</b> score of <b>9.8</b>. The vulnerability is only exploitable if there is a program listening on a Pragmatic General Multicast (PGM) port. If PGM is installed or enabled, but no active software is listening, then this vulnerability is not exploitable. Most organizations have controls (e.g., a firewall) in place to protect access to any ports at the network level.</p>
<p><a href="https://msrc.microsoft.com/update-guide/en-US/advisory/CVE-2024-38109">CVE-2024-38109</a> is a <b>Critical</b> elevation of privilege vulnerability affecting Azure Health Bot and has a <b>CVSS</b> score of <b>9.1</b>. This vulnerability allows an attacker to access Azure’s internal metadata service (IMDS) by utilizing server-side request forgery (SSRF). Successfully exploiting this vulnerability allows the attacker to gain elevated privileges over the network.</p>
<p><a href="https://msrc.microsoft.com/update-guide/en-US/advisory/CVE-2024-38159">CVE-2024-38159</a> and <a href="https://msrc.microsoft.com/update-guide/en-US/advisory/CVE-2024-38160">CVE-2024-38160</a> are <b>Critical</b> RCE vulnerabilities affecting Windows Network Virtualization, and both have a <b>CVSS</b> score of <b>9.1</b>. Successful exploitation of these vulnerabilities could allow the attacker to interact with other tenants’ applications and content. This can only be achieved with a high level of control over the machine. Microsoft has provided mitigation factors in order to successfully secure virtual machines, such as the importance of Hyper-V on virtualization infrastructures.</p>
<p><a href="https://msrc.microsoft.com/update-guide/en-US/advisory/CVE-2023-40547">CVE-2023-40547</a> is a <b>Critical</b> RCE vulnerability affecting Secure Boot and has a <b>CVSS</b> score of <b>8.8</b>. Microsoft has included this vulnerability as it has been heavily involved in developing and implementing a security standard in Secure Boot. Secure Boot is designed to prevent unauthorized code from running during the boot process by ensuring that only software signed by trusted keys is executed. This particular vulnerability poses a risk to the integrity of Secure Boot and Microsoft will apply a Secure Boot Advanced Targeting (SBAT) update to block vulnerable Linux boot loaders that could impact Windows security.</p>
<table><tbody><tr><td>Severity</td><td>CVSS Score</td><td>CVE</td><td>Description</td></tr><tr><td>Critical</td><td>9.8</td><td>CVE-2024-38063</td><td>Windows TCP/IP Remote Code Execution Vulnerability</td></tr><tr><td>Critical</td><td>9.8</td><td>CVE-2024-38140</td><td>Windows Reliable Multicast Transport Driver (RMCAST) Remote Code Execution Vulnerability</td></tr><tr><td>Critical</td><td>9.1</td><td>CVE-2024-38109</td><td>Azure Health Bot Elevation of Privilege Vulnerability</td></tr><tr><td>Critical</td><td>9.1</td><td>CVE-2024-38159</td><td>Windows Network Virtualization Remote Code Execution Vulnerability</td></tr><tr><td>Critical</td><td>9.1</td><td>CVE-2024-38160</td><td>Windows Network Virtualization Remote Code Execution Vulnerability</td></tr><tr><td>Critical</td><td>8.8</td><td>CVE-2023-40547</td><td>Redhat: CVE-2023-40547 Shim – RCE in HTTP boot support may lead to secure boot bypass</td></tr></tbody></table>
<p><i>Table 7. Critical vulnerabilities in Windows and Azure</i></p>
<h2>Vulnerabilities with Existing Proof of Concept in Windows Line Printer Daemon (LPD) Service, Windows Update Stack and Windows Secure Kernel Mode</h2>
<p><a href="https://msrc.microsoft.com/update-guide/en-US/advisory/CVE-2024-38199">CVE-2024-38199</a> is an <b>Important</b> RCE vulnerability affecting Windows Line Printer Daemon (LPD) and has a <b>CVSS</b> score of <b>9.8</b>. An unauthenticated attacker sending specially crafted print tasks to a shared vulnerable Windows Line Printer Daemon (LPD) service could result in remote code execution. The good news is that Microsoft has deprecated LPD since Windows Server 2012, and it is not installed or enabled by default.</p>
<p><a href="https://msrc.microsoft.com/update-guide/en-US/advisory/CVE-2024-38202">CVE-2024-38202</a> is an <b>Important</b> elevation of privilege vulnerability affecting Windows Update Stack and has a <b>CVSS</b> score of <b>7.3</b>. Microsoft was notified of an elevation of privilege vulnerability in Windows Backup. This vulnerability could potentially enable an attacker with basic user privileges to reintroduce previously patched vulnerabilities or circumvent some features of virtualization-based security (VBS).</p>
<p>While there have been no known attempts to exploit these vulnerabilities in the wild, a public presentation at Black Hat last week (“Windows Downdate: Downgrade Attacks Using Windows Updates” by SafeBreach) showcased their use in a vulnerability chain in order to compromise a system. Due to the complexity of the attack, there are no security updates to mitigate this threat as of yet. Microsoft has issued an advisory with recommended actions to help reduce the risk of exploitation until the security update is available.</p>
<p><a href="https://msrc.microsoft.com/update-guide/en-US/advisory/CVE-2024-21302">CVE-2024-21302</a> is an <b>Important</b> elevation of privilege vulnerability affecting Windows Secure Kernel Mode and has a <b>CVSS</b> score of <b>6.7</b>. Microsoft was notified of a secure kernel mode elevation vulnerability in Windows-based systems supporting VBS. This vulnerability could enable an attacker with administrator privileges to replace current versions of Windows system files with outdated versions. While there have been no known attempts to exploit this vulnerability in the wild, the same Black Hat presentation mentioned above showcased its use in a vulnerability chain in order to compromise a system. Microsoft has released a security update that includes opt-in revocation policy mitigations to address this vulnerability. Microsoft encourages customers running affected versions of Windows to review the advisory and knowledge base.</p>

<h2>Not All Relevant Vulnerabilities Have Patches: Consider Mitigation Strategies</h2>
<p>As we have learned with other notable vulnerabilities, such as <a href="https://www.crowdstrike.com/log4j2/">Log4j</a>, not every highly exploitable vulnerability can be easily patched. As is the case for the <a href="https://www.crowdstrike.com/blog/owassrf-exploit-analysis-and-recommendations/">ProxyNotShell</a> vulnerabilities, it’s critically important to develop a response plan for how to defend your environments when no patching protocol exists.</p>
<p>Regular review of your patching strategy should still be a part of your program, but you should also look more holistically at your organization’s methods for cybersecurity and improve your overall security posture.</p>

<h3>About CVSS Scores</h3>
<p>The <a href="https://www.first.org/cvss/">Common Vulnerability Scoring System</a> (CVSS) is a free and open industry standard that CrowdStrike and many other cybersecurity organizations use to assess and communicate software vulnerabilities’ severity and characteristics. The CVSS Base Score ranges from 0.0 to 10.0, and the <a href="https://nvd.nist.gov/">National Vulnerability Database</a> (NVD) adds a severity rating for CVSS scores.
