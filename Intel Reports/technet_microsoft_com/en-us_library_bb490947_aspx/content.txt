							<div class="content "><h1 id="netstat">Netstat</h1><div class="display-flex justify-content-space-between align-items-center flex-wrap-wrap page-metadata-container">
										<div class="margin-right-xxs">
											<ul class="metadata page-metadata" data-bi-name="page info" lang="en-us" dir="ltr"><li>Article</li><li>
														<time class="is-invisible" data-article-date aria-label="Article review date" datetime="2009-09-11T00:00:00.000Z" data-article-date-source="ms.date">09/11/2009</time>
													</li></ul>
										</div></div><nav id="center-doc-outline" class="doc-outline is-hidden-desktop display-none-print margin-bottom-sm" data-bi-name="intopic toc" aria-label="In this article">
											<h2 id="ms--in-this-article" class="title is-6 margin-block-xs">In this article</h2>
										</nav><!-- <content> --><p>Displays active TCP connections, ports on which the computer is listening, Ethernet statistics, the IP routing table, IPv4 statistics (for the IP, ICMP, TCP, and UDP protocols), and IPv6 statistics (for the IPv6, ICMPv6, TCP over IPv6, and UDP over IPv6 protocols). Used without parameters, <strong>netstat</strong> displays active TCP connections.</p>
<h3 id="syntax">Syntax</h3>
<p><strong>netstat</strong>&nbsp;[<strong>-a</strong>] [<strong>-e</strong>] [<strong>-n</strong>] [<strong>-o</strong>] [<strong>-p</strong>&nbsp;<em>Protocol</em>] [<strong>-r</strong>] [<strong>-s</strong>] [<em>Interval</em>]</p>
<h3 id="parameters">Parameters</h3>
<p><strong>-a</strong> &nbsp; <strong>:</strong> Displays all active TCP connections and the TCP and UDP ports on which the computer is listening.</p>
<p><strong>-e</strong> &nbsp; <strong>:</strong> Displays Ethernet statistics, such as the number of bytes and packets sent and received. This parameter can be combined with <strong>-s</strong>.</p>
<p><strong>-n</strong> &nbsp; <strong>:</strong> Displays active TCP connections, however, addresses and port numbers are expressed numerically and no attempt is made to determine names.</p>
<p><strong>-o</strong> &nbsp; <strong>:</strong> Displays active TCP connections and includes the process ID (PID) for each connection. You can find the application based on the PID on the <strong>Processes</strong> tab in Windows Task Manager. This parameter can be combined with <strong>-a</strong>, <strong>-n</strong>, and <strong>-p</strong>.</p>
<p><strong>-p</strong> &nbsp; <em><strong>Protocol</strong></em> &nbsp; <strong>:</strong> Shows connections for the protocol specified by <em>Protocol</em>. In this case, the <em>Protocol</em> can be <strong>tcp</strong>, <strong>udp</strong>, <strong>tcpv6</strong>, or <strong>udpv6</strong>. If this parameter is used with <strong>-s</strong> to display statistics by protocol, <em>Protocol</em> can be <strong>tcp</strong>, <strong>udp</strong>, <strong>icmp</strong>, <strong>ip</strong>, <strong>tcpv6</strong>, <strong>udpv6</strong>, <strong>icmpv6</strong>, or <strong>ipv6</strong>.</p>
<p><strong>-s</strong> &nbsp; <strong>:</strong> Displays statistics by protocol. By default, statistics are shown for the TCP, UDP, ICMP, and IP protocols. If the IPv6 protocol for Windows&nbsp;XP is installed, statistics are shown for the TCP over IPv6, UDP over IPv6, ICMPv6, and IPv6 protocols. The <strong>-p</strong> parameter can be used to specify a set of protocols.</p>
<p><strong>-r</strong> &nbsp; <strong>:</strong> Displays the contents of the IP routing table. This is equivalent to the <strong>route print</strong> command.</p>
<p><em><strong>Interval</strong></em> &nbsp; <strong>:</strong> Redisplays the selected information every <em>Interval</em> seconds. Press CTRL+C to stop the redisplay. If this parameter is omitted, <strong>netstat</strong> prints the selected information only once.</p>
<p><strong>/?</strong> &nbsp; <strong>:</strong> Displays help at the command prompt.</p>
<h3 id="remarks">Remarks</h3>
<ul>
<li><p>Parameters used with this command must be prefixed with a hyphen (<strong>-</strong>) rather than a slash (<strong>/</strong>).</p>
</li>
<li><p><strong>Netstat</strong> provides statistics for the following:</p>
<ul>
<li><p>Proto</p>
<p>The name of the protocol (TCP or UDP).</p>
</li>
<li><p>Local Address</p>
<p>The IP address of the local computer and the port number being used. The name of the local computer that corresponds to the IP address and the name of the port is shown unless the <strong>-n</strong> parameter is specified. If the port is not yet established, the port number is shown as an asterisk (*).</p>
</li>
<li><p>Foreign Address</p>
<p>The IP address and port number of the remote computer to which the socket is connected. The names that corresponds to the IP address and the port are shown unless the <strong>-n</strong> parameter is specified. If the port is not yet established, the port number is shown as an asterisk (*).</p>
</li>
<li><p>(state)</p>
<p>Indicates the state of a TCP connection. The possible states are as follows:</p>
<p>CLOSE_WAIT</p>
<p>CLOSED</p>
<p>ESTABLISHED</p>
<p>FIN_WAIT_1</p>
<p>FIN_WAIT_2</p>
<p>LAST_ACK</p>
<p>LISTEN</p>
<p>SYN_RECEIVED</p>
<p>SYN_SEND</p>
<p>TIMED_WAIT</p>
<p>For more information about the states of a TCP connection, see RFC 793.</p>
</li>
</ul>
</li>
<li><p>This command is available only if the <strong>Internet Protocol (TCP/IP)</strong> protocol is installed as a component in the properties of a network adapter in Network Connections</p>
</li>
</ul>
<h3 id="examples">Examples</h3>
<p>To display both the Ethernet statistics and the statistics for all protocols, type the following command:</p>
<p><strong>netstat -e -s</strong></p>
<p>To display the statistics for only the TCP and UDP protocols, type the following command:</p>
<p><strong>netstat -s -p tcp udp</strong></p>
<p>To display active TCP connections and the process IDs every 5 seconds, type the following command:</p>
<p><strong>nbtstat -o 5</strong></p>
<p>To display active TCP connections and the process IDs using numerical form, type the following command:</p>
<p><strong>nbtstat -n -o</strong></p>
