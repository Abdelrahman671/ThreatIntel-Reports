    <h1 class="text-center">Masslogger campaigns exfiltrates user credentials</h1>

    <div class="text-center m-1">
        <div class="post-author">
            <span>By </span>
                <a href="https://blog.talosintelligence.com/author/nick-biasini/">Nick Biasini</a>

        <br/>
        <time class="post-datetime" datetime="February 17, 2021 08:00">
            Wednesday, February 17, 2021 08:00
        </time>

        <div class="m-3">
                            <a href="/category/securex-3/" class="category primary-category">

                            </a>

    <section class="post-content-wrapper mt-5">
        <div class="post-content">
            <p>By <a href="https://www.google.com/url?q=https://twitter.com/vanjasvajcer&amp;sa=D&amp;source=editors&amp;ust=1613558376141000&amp;usg=AOvVaw2b9iUY2soly67oFZ5Wy2nm">Vanja Svajcer</a>. <br></p><h3 id="news-summary">News summary <br></h3><ul><li>As protection techniques develop, attackers are finding it harder to successfully attack their targets and must find creative ways to succeed.</li><li>Cisco Talos recently discovered a campaign utilizing a variant of the Masslogger trojan designed to retrieve and exfiltrate user credentials from multiple sources such as Microsoft Outlook, Google Chrome and instant messengers.</li><li>Apart from the initial email attachment, all the stages of the attacks are fileless and they only occur in volatile memory.</li><li>These threats demonstrate several techniques of the <a href="https://attack.mitre.org/&amp;sa=D&amp;source=editors&amp;ust=1613558376142000&amp;usg=AOvVaw2w9QlS_0G_8gA9kTp65smA">MITRE ATT&amp;CK framework</a>, most notably <a href="https://attack.mitre.org/techniques/T1566/&amp;sa=D&amp;source=editors&amp;ust=1613558376142000&amp;usg=AOvVaw2mM9T9_A5-DcWi9nl9oICW">T1566</a>  Phishing, <a href="https://attack.mitre.org/techniques/T1059/001/&amp;sa=D&amp;source=editors&amp;ust=1613558376143000&amp;usg=AOvVaw3NEBSPEMPyvcafIwAIq1N-">T1059.001</a> and <a href="https://attack.mitre.org/techniques/T1059/007/&amp;sa=D&amp;source=editors&amp;ust=1613558376143000&amp;usg=AOvVaw1FnEJHPJgZsLP_3ycj1aCS">T1059.007</a>  Command and Scripting Interpreters, <a href="https://attack.mitre.org/techniques/T1140/&amp;sa=D&amp;source=editors&amp;ust=1613558376143000&amp;usg=AOvVaw1CErSKihrgfjoNnu7ylGrX">T1140</a>  Deobfuscate/Decode Files or Information, <a href="https://attack.mitre.org/techniques/T1497/&amp;sa=D&amp;source=editors&amp;ust=1613558376143000&amp;usg=AOvVaw0hFzQ57p1wlrv5UmKJ-ZTI">T1497</a>  Virtualization/Sandbox Evasion, <a href="https://attack.mitre.org/techniques/T1555/003/&amp;sa=D&amp;source=editors&amp;ust=1613558376144000&amp;usg=AOvVaw0iw4cflB1MtkbOzcsZU8_N">T1555.003</a>  Credentials from Web Browsers, <a href="https://attack.mitre.org/techniques/T1115/&amp;sa=D&amp;source=editors&amp;ust=1613558376144000&amp;usg=AOvVaw3fvykbP3u4Rfqh6rw_F7-8">T1115 </a> Clipboard Data, <a href="https://attack.mitre.org/techniques/T1056/001/&amp;sa=D&amp;source=editors&amp;ust=1613558376144000&amp;usg=AOvVaw0EanwDVxiMCUt7MZZ3gj01">T1056.001</a>  Keylogging and <a href="https://attack.mitre.org/techniques/T1048/003/&amp;sa=D&amp;source=editors&amp;ust=1613558376145000&amp;usg=AOvVaw24HrUayTMOnWwEJDxre3dd">T1048.003</a>  Exfiltration Over Unencrypted/Obfuscated Non-C2 Protocol.</li></ul><p>Attackers are constantly reinventing ways to monetize their tools. Cisco Talos recently discovered an interesting campaign affecting Windows systems and targeting users in Turkey, Latvia and Italy, although similar campaigns by the same actor have also been targeting users in Bulgaria, Lithuania, Hungary, Estonia, Romania and Spain in September, October and November 2020.</p><p>The actor employs a multi-modular approach that starts with the initial phishing email and carries through to the final payload. The adversaries behind this campaign likely do this to evade detection. But it can also be a weakness, as there are plenty of opportunities for defenders to break the killchain.</p><h4 id="what-s-new">What's new? </h4><p>Although operations of the Masslogger trojan have been previously documented, we found the new campaign notable for using the compiled HTML file format to start the infection chain. This file format is typically used for Windows Help files, but it can also contain active script components, in this case JavaScript, which launches the malware's processes.</p><h4 id="how-did-it-work">How did it work? </h4><p>The infection starts with an email message containing a legitimate-looking subject line that seems to relate to a business. The email contains a RAR attachment with a slightly unusual filename extension.</p><p>The usual filename extension for RAR files is .rar. However, RAR-compressed archives can also be split into multi-volume archives. In this case, the filename creates files with the RAR extension named "r00" and onwards with the .chm file extension. This naming scheme is used by the Masslogger campaign, presumably to bypass any programs that would block the email attachment based on its file extension.</p><p>CHM is a compiled HTML file that contains an embedded HTML file with JavaScript code to start the active infection process. Every stage of the infection is obfuscated to avoid detection using simple signatures.</p><p>The second stage is a PowerShell script that eventually deobfuscates into a downloader and downloads and loads the main PowerShell loader. The Masslogger loaders seem to be hosted on compromised legitimate hosts with a filename containing one letter and one number concatenated with the filename extension .jpg. For example, "D9.jpg".</p><p>The main payload is a variant of the Masslogger trojan designed to retrieve and exfiltrate user credentials from a variety of sources, targeting home and business users. Masslogger can be configured as a keylogger, but in this case, the actor has disabled this functionality.</p><h4 id="so-what">So what? </h4><p>While most of the public attention seems to be focused on ransomware attacks, big game hunting and APTs, it is important to keep in mind that crimeware actors are still active and can inflict significant damage to organizations by stealing users' credentials. The credentials themselves have value on the dark web and actors sell them for money or use them in other attacks.</p><p>Based on the IOCs we retrieved, we have moderate confidence that this actor has previously used other payloads such as <a href="https://blog.talosintelligence.com/custom-dropper-hide-and-seek&amp;sa=D&amp;source=editors&amp;ust=1613558376147000&amp;usg=AOvVaw3bRZazyMAi_9ApyAOiV1bv">AgentTesla</a>, <a href="https://blog.talosintelligence.com/my-little-formbook&amp;sa=D&amp;source=editors&amp;ust=1613558376147000&amp;usg=AOvVaw1XcjzdYecnsbzKwecjJjOC">Formbook</a> and AsyncRAT in campaigns starting as early as April 2020.</p><h2 id="technical-case-overview">Technical case overview</h2><h3 id="introduction">Introduction </h3><p>Masslogger is a spyware program written in .NET with a focus on stealing user credentials, mostly from the browsers but also from several popular messaging applications and email clients. It was released in April 2020 and sold on underground forums for a moderate price with a few licensing options.</p><p>The exfiltration of data takes place over one or more of these channels:</p><ul><li>FTP (plain text over default port 21), the configuration contains user credentials.</li><li>HTTP  Using a PHP-based control panel.</li><li>SMTP  The user has to specify email address, server and credentials to use it. <br>We won't dig deep into the functionality of the final Masslogger payload, as this was previously <a href="https://fr3d.hk/blog/masslogger-frankenstein-s-creation&amp;sa=D&amp;source=editors&amp;ust=1613558376148000&amp;usg=AOvVaw2Qj6JdzZvYHNC7iNw7rUQs">well-described by other researchers</a>. Instead, we'll focus on the infection vector and the memory-only delivery chain before the final stage is loaded. In case of commodity spyware such as Masslogger, it is the infection chain and contextual information that distinguish the individual actors behind each campaign.</li></ul><p>The infection chain we follow seems to focus on business users, with email being the infection vector. The email contains a RAR attachment with a compiled HTML (.chm) attachment. The rest of the chain is split between JavaScript, PowerShell and .NET.</p><h4 id="email-as-an-infection-vector">Email as an infection vector </h4><p>The latest campaign began in mid-January. Based on the combination of discovered emails and file names, we believe it was targeting organizations in Turkey, Latvia and Italy. We have observed similar campaigns happening in several instances before, starting no later than September 2020. In previous campaigns, the actor was targeting users in Bulgaria, Lithuania, Hungary, Estonia, Romania and Spain.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-LgwY0C6muL4/YCzlizN0ouI/AAAAAAAAAms/G5kf83itUqAW9AhcpaQ3PZHITK6I1cTpACPcBGAYYCw/s16000/image4.jpg" class="kg-image" alt loading="lazy"></figure><p><i>European countries targeted by the observed Masslogger campaigns from September 2020.</i> <br></p><p>The email is written in the language of the targeted recipient's top-level domain. The first example is an email targeted at users in Turkey with the subject "Domestic customer inquiry" and the body "At the request of our customer, please send your attached best quotes."</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-BGfRNifmdtY/YCzmc91W-gI/AAAAAAAAAm0/7IamfKAw_LgGDG_04Iuc46CpawGO-lKcgCLcBGAsYHQ/s16000/image17.png" class="kg-image" alt loading="lazy"></figure><p><i>Email campaign example targeting users in Turkey.</i> <br></p><p>Some earlier campaigns were purported to be a request to open and sign a memorandum of understanding. The actor attempted to make emails more credible by adding a link to the legitimate scanning application to the email footer "Shipped with Genius Scan for iOS."</p><p>For the campaigns in September, October and November, the adversaries sent emails containing a subject line that translates to "MOU Information" with the text "Please return it signed and stamped. Best regards," in the body.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-8kqX7Bv40SM/YCzm3ZToVSI/AAAAAAAAAm8/l7N0hlVASRQ6o1XnYoUOQ7eVizqsQaSWQCLcBGAsYHQ/s16000/image13.png" class="kg-image" alt loading="lazy"></figure><p><i>An example of an earlier email targeting users in Spain.</i></p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-q7FduIurjJo/YCznNQbcLEI/AAAAAAAAAnE/SPjBh1o9yQIFTcuRB7hbhbx9btf3n6CtACLcBGAsYHQ/s16000/image7.png" class="kg-image" alt loading="lazy"></figure><p><i>An example of an earlier email targeting users in Bulgaria.</i> <br></p><p>The attachment file name for the latest campaign is chosen according to the email subject, with possible random strings prepended, for example, "70727_YK90054_Teknik_Cizimler." The attachment filename extension is chosen to bypass simple blockers that attempt to block RAR attachments using its default filename extension ".rar". The actor changes the filename extension to RAR multi-volume filename extensions, starting from ".r00". WinRAR and other RAR-capable unarchivers will still open the file without problems.</p><p>The attached RAR archive contains a single file with the ".chm" filename extension. CHM stands for "compiled HTML files," and it is one of the default formats for Windows Help files. Compiled HTML files can be easily created using the Windows HTML Help executable program hh.exe. The same program can be used with the command line option "-decompile" to extract the embedded and compressed HTML files.</p><p>When the user opens the attachment with the default application, a simple HTML page is displayed, containing the text "Customer service, Please Wait"</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-XgZxa_4U14w/YCznq6c83pI/AAAAAAAAAnM/kaSebyJBhkQkp0DS-jkeTzvd-0AY9bRRACLcBGAsYHQ/w427-h640/image16.png" class="kg-image" alt loading="lazy"></figure><p><i>The HTML page displayed when the .CHM attachment is opened.</i> <br></p><p>When the CHM file is decompiled and the HTML file extracted, it contains lightly obfuscated JavaScript code to create an HTML page. The HTML content is escaped and reversed, so it is easy to deobfuscate.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-MFnwnUj9VJQ/YCzoMxQLDnI/AAAAAAAAAnY/3-fLhBCaRZYOcfKa9SQZsojJG9fnT0CQgCLcBGAsYHQ/s16000/image9.png" class="kg-image" alt loading="lazy"></figure><p><i>Obfuscated JavaScript code in the decompiled HTML file.</i> <br></p><p>The HTML code contains an ActiveX object containing PowerShell code obfuscated in a similar way to strings in the JavaScript code of the CHM file.<br></p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-l5orefuRUUg/YCzoxiR0cnI/AAAAAAAAAng/827RPez7MLQ5ownAP_ttry66YeE71BBhACLcBGAsYHQ/s16000/image15.png" class="kg-image" alt loading="lazy"></figure><p><i>HTML page with an ActiveX object embedded and PowerShell code.</i> <br></p><p>When deobfuscated, we can observe a PowerShell downloader stage, which simply connects to the download server, usually a compromised legitimate host. The download server hosts the next stage of the infection.<br></p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-CzUI_nSHkvc/YCzpAx4kJKI/AAAAAAAAAnk/I8BlXSIW5ZIB2Q3ePnZBpy_7_ZZB6vb1wCLcBGAsYHQ/s16000/image18.png" class="kg-image" alt loading="lazy"></figure><p><i>PowerShell downloader stage.</i> <br></p><p>The URL to download the next stage ends in the path with the format [1Letter][1 to 2-digit number].jpg, for example, hxxp://sinetcol[.]co/D7.jpg. This stage is encoded with a simple hexadecimal encoding scheme and is converted to code by first splitting the downloaded content using the character "^" as the delimiter and then adding ASCII representation of each number to a string variable. Eventually, the string containing PowerShell code is piped into the Invoke-Expression (IEX) cmdlet. This is the PowerShell loader.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-WaW45TLTa-0/YCzpbSUhCiI/AAAAAAAAAnw/kiQFtz0d99s5bG18AyKXmC36dRuaCDhXACLcBGAsYHQ/s16000/image10.png" class="kg-image" alt loading="lazy"></figure><p><i>Encoded PowerShell loader</i> </p><p>The PowerShell loader contains two encoded .NET assemblies. The first one is a DLL and the other an executable.</p><h4 id="loader-dll-and-final-masslogger-payload">Loader DLL and final Masslogger payload</h4><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-i4fQAw3D9iY/YCzqoNwWisI/AAAAAAAAAn8/HfC0bAW0uC0uZ8puqgXYG-glkCPVGM9SgCLcBGAsYHQ/s16000/image8.png" class="kg-image" alt loading="lazy"></figure><p><i>Start of the PowerShell loader.</i> <br></p><p>The PowerShell loader first decodes the .NET DLL and then deobfuscates the string "System.AppDomain" to get the reference to its method "GetCurrentDomain." The loader then creates a byte array where it stores the Masslogger loader before it invokes the GetCurrentDomain function to get the context of execution and the process where the script is executing.</p><p>The acquired domain is then used to load the .NET DLL assembly into the powershell.exe process space with the assembly name "Waves.dll." Waves employs a Costura loader, an open-source reflective assembly loader alternative to <a href="https://github.com/dotnet/ILMerge&amp;sa=D&amp;source=editors&amp;ust=1613558376153000&amp;usg=AOvVaw2q8JU4QosE_ps8ZpkWhzeb">ILMerge</a> and is obfuscated with DotNetGuard obfuscator, all to make analysis and detection more difficult.</p><p>Once the DLL is loaded as a .NET assembly, the PowerShell loader calls the method tasked with creating a msbuild.exe process, injecting the final payload into its process space and launching it.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-Oo0MdGScVpM/YCzq0Ef_PgI/AAAAAAAAAoA/cvogmJ4F7585AuRakSpytyw5KwCZTeJ6wCLcBGAsYHQ/s16000/image1.png" class="kg-image" alt loading="lazy"></figure><p><i>After decoding, the DLL loader assembly is created and loaded.</i> <br></p><p>The Masslogger payload is stored in memory as a buffer compressed with gzip. The buffer is decompressed by the DLL loader. The internal assembly name of the payload is "service-med-star.gr", which is a concatenation of the username and the server used for FTP credentials exfiltration.</p><p>Masslogger is a credential stealer and keylogger with the ability to exfiltrate data through SMTP, FTP or HTTP protocols. For the first two, no additional server-side components are required, while the exfiltration over HTTP is done through the Masslogger control panel web application.</p><p>This version of Masslogger contains the functionality to target and retrieve credentials from the following applications:</p><ul><li>Pidgin messenger client</li><li>FileZilla FTP client</li><li>Discord</li><li>NordVPN</li><li>Outlook</li><li>FoxMail</li><li>Thunderbird</li><li>FireFox</li><li>QQ Browser</li><li>Chromium based browsers (Chrome, Chromium, Edge, Opera, Brave)</li></ul><p>The configuration for a payload is stored as an encrypted array of strings within the payload itself. Although the configuration is encrypted and the payload obfuscated with an unknown obfuscator it is still possible to find code used to decrypt the configuration as <a href="https://mariohenkel.medium.com/decrypt-masslogger-2-4-0-0-configuration-eff3ee0720a7&amp;sa=D&amp;source=editors&amp;ust=1613558376156000&amp;usg=AOvVaw3NSKBHLbEjQPdwKFih48d1">previously documented</a> by Mario Henkel.</p><p>The configuration is decrypted using the standard .NET framework functionality. This allows us to place a breakpoint to the beginning of the method System.Security.Cryptography.AesCryptoServiceProvider and step back to the configuration decryption function within the payload body and trace the value returned to the caller after each configuration string is decrypted.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-vxSRwb8tV08/YCzrAis0GKI/AAAAAAAAAoI/QNz20AVPjxElWU4kir0UFostAzXOM07hACLcBGAsYHQ/s16000/image5.png" class="kg-image" alt loading="lazy"></figure><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-Z8oRZ_5KVvE/YCzrxfaH5ZI/AAAAAAAAAoY/zFt4SG-jBZoDO37ZLQlYqkF2Iigj33csACLcBGAsYHQ/s16000/image12.png" class="kg-image" alt loading="lazy"></figure><p><i>Breakpoints placed to decrypt the payload configuration.</i> <br></p><p>The decrypted configuration is parsed by Masslogger to configure the trojan to target a specific set of applications and exhibit functionality. In our case, the Masslogger version we are dealing with is 3.0.7563.31381 and the exfiltration is conducted over FTP, with med-star.gr as the FTP exfiltration server.</p><p>Although the payload is configured to use FTP, the actor has installed a version of Masslogger control panel on the same server with the URL hxxps://www[.]med-star[.]gr/panel/?/login.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-gQ2FxiVcICo/YCztGEGr9nI/AAAAAAAAAok/gOKmNUIIaCgFJpK-oUTN8-AsWf3UeB2fgCLcBGAsYHQ/s16000/image11.png" class="kg-image" alt loading="lazy"></figure><p><i>Masslogger control panel login screen.</i> </p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-kmuzw5ajob4/YCztTrnPR4I/AAAAAAAAAoo/6_14Cv7tK3QnK1bnrKwdnFdPVsk8Q01XACLcBGAsYHQ/s16000/image2.png" class="kg-image" alt loading="lazy"></figure><p><i>Decrypted Masslogger configuration strings.</i> <br></p><p>Once the credentials from targeted applications are retrieved, they are uploaded to the exfiltration server with a filename containing the username, two-letter country ID, unique machine ID and the timestamp for when the file was created.</p><p>Uploaded credential files begin with the information about the user and the infected system, configuration options and processes running, followed by the retrieved credentials delimited by lines containing targeted application names.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-croxIkxHhTQ/YCzthn8Mm7I/AAAAAAAAAow/ksMWQlNWsP04890BcrHYVeBUilytLtmywCLcBGAsYHQ/s16000/image6.png" class="kg-image" alt loading="lazy"></figure><p><i>The beginning of an uploaded exfiltrated credentials file.</i> </p><h4 id="conclusion">Conclusion <br> </h4><p>This recently discovered Masslogger campaign  which we attribute to an actor launching similar credential-stealing campaigns  back to at least September 2020. There is moderate confidence the author has previously used AgentTesla with similar goals in April 2020. The campaigns are targeted to several European countries, shifting its focus monthly. In our research, we detected email messages targeting Turkey, Latvia, Lithuania, Bulgaria, Hungary, Estonia, Romania, Italy and Spain, as well as messages written in English.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-9ylEpUfjLaE/YCzttZu6PxI/AAAAAAAAAo4/uwP_DZOOJlgEJAHRxeLl-w6uce8aD3BIACLcBGAsYHQ/s16000/image19.jpg" class="kg-image" alt loading="lazy"></figure><p><i>Masslogger campaign modules.</i> <br></p><p>The observed campaign is almost entirely executed and present only in memory, which emphasizes the importance of conducting regular and background memory scans. The only component present on disk is the attachment and the compiled HTML help file.</p><p>Users are advised to configure their systems for logging PowerShell events such as module loading and executed script blocks as they will show executed code in its deobfuscated format. Talos will continue to track similar campaigns to make sure adequate protection is included in Cisco Secure products.</p><h3 id="coverage">Coverage <br></h3><p>Ways our customers can detect and block this threat are listed below.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-N-NDPUPqW9w/YCzt9VgdLlI/AAAAAAAAApE/gB5FW8WJAj4nJ5q52CxSTLZev9fDSJvhACLcBGAsYHQ/w308-h400/image14.jpg" class="kg-image" alt loading="lazy"></figure><p>Advanced Malware Protection (<a href="https://www.cisco.com/c/en/us/products/security/advanced-malware-protection&amp;sa=D&amp;source=editors&amp;ust=1613558376161000&amp;usg=AOvVaw0aYnze42N4C77H8Pearb13">AMP</a>) is ideally suited to prevent the execution of the malware used by these threat actors.
