					<h3 class="name"><span style="line-height:50px;">Increase Security with TPM, Secure Boot, and Trusted Boot</span></h3>
		
				
		<div class="content">
						<p><img class="align-right" style="float:right;" alt=" " src="/resized-image/__size/320x240/__key/communityserver-discussions-components-files/489/0003.strategies_5F00_for_5F00_protecting-border.jpg" />Control systems must be secure by design and should have a hardware root of trust as the foundation of all the security constructs in the control system. Emerson PLC/PAC controllers come with Trusted Platform Module (TPM) technology that enables hardware root of trust. All PLC/PAC boot firmware is signed by Emerson with the private key stored in the TPM to ensure only Emerson-signed firmware will run on the hardware. Patches supplied by Emerson are also signed for verification purposes prior to loading. Here&rsquo;s why TPM is important.</p>
<p style="padding-left:30px;"><strong>Trusted Platform Module<br /></strong>The Trusted Platform Module (TPM) is a separate hardware module with a dedicated microcontroller providing cryptographic key generation and key storage capability. Since each TPM chip has a unique and secret RSA key pair burned in, it can perform platform authentication. Software can use the TPM to authenticate other hardware devices. The TPM can be used for both encryption and decryption operations and is an excellent source of entropy for random number generation. The random number generator makes it virtually impossible for any other system to guess the generated sequence. This capability, when combined with the server public key, creates an encrypted link between two ends. The TPM generates a non-repeatable number that makes it difficult for outside influences to decipher the data being transmitted. The TPM can be implemented on any computer platform and is required by the United States Department of Defense (TPM version 1.2 or higher) for many of their devices, including phones and computers. TPM forms a &ldquo;hardware root of trust&rdquo; when used in conjunction with BIOS. Root of Trust (RoT) is a set of functions in the trusted computing module that is always trusted by the computer&#39;s operating system (OS). The RoT serves as a separate computer engine, controlling the trusted computing platform cryptographic processor on the device in which it is embedded. TPM allows secure storage and reporting of security metrics that can be used to randomly validate the system&rsquo;s configurations to ensure changes haven&rsquo;t&nbsp;occurred. Remote Attestation, an authentication process, can be facilitated when the TPM creates a nearly unforgeable hash key that is a signature of the hardware and software configuration. This could allow third-party systems to verify that the software has not been changed.</p>
<p style="padding-left:30px;"><strong>Secure Boot<br /></strong>With Secure Boot, the control system firmware checks that the system boot loader is signed with a cryptographic key authorized by Emerson and stored in a database contained in the firmware. It is used by UEFI (Unified Extensible Firmware Interface) in conjunction with BIOS for controlled boot to prevent the execution of unsigned programs.</p>
<p style="padding-left:30px;"><strong>Trusted Boot<br /></strong>Trusted Boot takes over where Secure Boot leaves off. Trusted Boot verifies the digital signature of the OS. In turn, the OS verifies the components it will use in the startup process, such as startup files and boot drivers. If a file has been modified, the boot loader detects the change, then refuses to load the corrupt component. Trusted Boot will only use trusted software, often implemented by using signed and certified software from the manufacturer, resulting in proper configuration and patch management.</p>
<p>This discussion is one of a series of blogs discussing security in industrial PLC/PAC control systems. If you&rsquo;d like to see others, <a href="/products/control-safety-systems/f/plc-pac-systems-industrial-computing-forum" rel="noopener noreferrer" target="_blank">click here</a>.</p>

    var container = j('#header-5256_container');
    var popup;
    var hoverTimer; // Variable to hold the timer for delayed hide

    function showLinksPopUp(parent, children) {
        var links = j('<ul></ul>').append(children);

        if (popup) {
            popup.glowPopUpPanel('empty', '')
                .glowPopUpPanel('append', links)
                .glowPopUpPanel('show', parent, {
                    // Positioning options
                    // Positioning options
                position: {
                    my: 'left top',  // Popup's top left corner
                    at: 'left bottom',  // Position at the bottom left corner of the parent
                    of: parent,  // Relative to the parent element
                    collision: 'flipfit' // Adjust position to fit in the viewport
                }

                });
        } else {
            popup = j('<div></div>')
                .glowPopUpPanel({
                    cssClass: 'links-popup-panel',
                    zIndex: 1500,
                    hideOnDocumentClick: true
                })
                .glowPopUpPanel('append', links)
                .glowPopUpPanel('show', parent, {
                 // Positioning options
                position: {
                    my: 'left top',  // Popup's top left corner
                    at: 'left bottom',  // Position at the bottom left corner of the parent
                    of: parent,  // Relative to the parent element
                    collision: 'flipfit' // Adjust position to fit in the viewport
                }

                });
        }
    }

    function hidePopup() {
        if (popup) {
            popup.glowPopUpPanel('hide');
        }
    }

    // Function to clear the hover timer
    function clearHoverTimer() {
        if (hoverTimer) {
            clearTimeout(hoverTimer);
            hoverTimer = null;
        }
    }

    // Use mouseenter instead of mouseover to avoid rapid toggling
    container.on('mouseenter', '.navigation-list-item a', function(e) {
        clearHoverTimer();
        var elm = j(e.target);
        var children = elm.data('_link_children');
        if (!children) {
            children = elm.closest('li').find('ul.children > li');
            elm.data('_link_children', children);
        }
        if (children && children.length > 0) {
            showLinksPopUp(elm, children);
        } else {
            hidePopup();
        }
    });

    // Add mouseleave event to hide popup with a delay
    container.on('mouseleave', '.navigation-list-item a', function() {
        hoverTimer = setTimeout(function() {
            if (!popup.is(':hover')) {
                hidePopup();
            }
        }, 3); // Slight delay to allow moving to the popup panel
    });

    // Add mouseenter event to clear the timer if re-entered before hide
    j(document).on('mouseenter', '.links-popup-panel', function() {
        clearHoverTimer();
    });

    // Add mouseleave event to hide popup with a delay
    j(document).on('mouseleave', '.links-popup-panel', function() {
        hoverTimer = setTimeout(hidePopup, 3); // Slight delay to allow moving to the navigation link
    });
