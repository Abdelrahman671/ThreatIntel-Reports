&lt;h2&gt;Typical Attack Anatomy&lt;/h2&gt;

&lt;h2&gt;The procSettings DLL&lt;/h2&gt;
To find out more about what’s inside this core DLL, we performed a static examination of the DLL. This did not shed light on its functionality and rules that govern its activity. One of the things we did see is that this DLL is Borland Delphi compiled and that it imports different functions from different DLLs. This suggests that procSettings is the DLL that holds most of the logic of the malware and its implementation.

A dynamic analysis we ran allowed us to examine the exported function THetholdImplementationIntercept. We saw that first the malware created a mutex with the name CodeCall.Net Mutey in order to prevent multiple instances of the malware running at the same time.

Next, the malware ran a check to discern whether the targeted bank application was installed on the infected device. It did that by searching the software directory under %appdatalocal%.

If the software the attackers are interested in is indeed installed on the device, it further checks if the file C:\ProgramData\OkApp.is exists. This file is one of the malware’s files, used as an indicator; this file is empty of content.

 	&lt;li&gt;Next, ZE Loader loads the encrypted file &#039;operationB&#039;, decrypts and unpacks it. The encryption and unpacking methods are the same as before. This file is a malicious DLL that is responsible for setting an outbound RDP connection to the C&amp;C.&lt;/li&gt;
&lt;/ol&gt;
&lt;img src=&quot;https://images-cdn.welcomesoftware.com/Zz0yMWQzNTUxYjBjODMxMWVjODdmMDE5NGRhMzAyNzYwNw==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiMjFkMzU1MWIwYzgzMTFlYzg3ZjAxOTRkYTMwMjc2MDciXSwiZXhwIjoxNjMyNDE2MDcyfQ.S5OpCfEtwOudLgMoDg_OfoLgKVDpbsW27UJaDqGSJh8&quot; alt=&quot;Fig10.png&quot; width=&quot;754.7679999999999&quot; height=&quot;322&quot; /&gt;

&lt;em&gt;Figure 10: ZE Loader opens an outbound RDP connection&lt;/em&gt;
&lt;h2&gt;OperationB DLL&lt;/h2&gt;
We began with a static examination of the malicious DLL &#039;OperationB.&#039; Examining the DLL’s resource section, we saw that it contained some legitimate RDP DLLs, including the right ones for each Windows architecture, as well as RDP configuration files.

&lt;img src=&quot;https://images-cdn.welcomesoftware.com/Zz0yYzAyZTBhMDBjODMxMWVjODdmMDE5NGRhMzAyNzYwNw==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiMmMwMmUwYTAwYzgzMTFlYzg3ZjAxOTRkYTMwMjc2MDciXSwiZXhwIjoxNjMyNDE2MDcyfQ.jbn0D2Tv8Iq9B6WGPhSj3TMlht-Q-TP91ehjXI_qNbo&quot; alt=&quot;Fig11.png&quot; width=&quot;345&quot; height=&quot;260&quot; /&gt;

&lt;em&gt;Figure 11: RDP files used by ZE Loader&lt;/em&gt;

&lt;img src=&quot;https://images-cdn.welcomesoftware.com/Zz0zNTBmZTkyMjBjODMxMWVjODUwY2E1YWIxOTFiYjQ1MQ==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiMzUwZmU5MjIwYzgzMTFlYzg1MGNhNWFiMTkxYmI0NTEiXSwiZXhwIjoxNjMyNDE2MDcyfQ.Za2vNLULBuED5Cb3O9iYeApNu0gbocqKxd-YSRO5nac&quot; alt=&quot;Fig12.png&quot; width=&quot;792&quot; height=&quot;643&quot; /&gt;

&lt;em&gt;Figure 12: RDP configuration as used by ZE Loader&lt;/em&gt;

Dynamically running this malicious DLL, we see that it begins by saving the RDP DLL and its configuration on disk under a randomly generated directory; in this case, saved under %programFiles%.
&lt;h2&gt;Manipulating Security Settings&lt;/h2&gt;
In the next step, ZE Loader manipulates some security settings to enable the attacker to have undisturbed remote access to the infected device.

ZE Loader searches for the service &#039;TermService&#039;. This service allows RDP connections to stream to and from the client device. ZE Loader sets its configuration settings to SERVICE_AUTO_START with the path of the RDP DLL file it already saved on disk.

Next, ZE Loader changes the settings of the infected device to allow and establish multiple RDP connections to and from that device. The following settings are toggled to &#039;true&#039;:
&lt;ul&gt;
 	&lt;li&gt;HKLM\System\CurrentControlSet\Control\Terminal Server\fDenyTSConnection&lt;/li&gt;
 	&lt;li&gt;HKLM\System\CurrentControlSet\Control\Terminal Server\Licensing Core\EnableConCurrentSessions&lt;/li&gt;
 	&lt;li&gt;HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon\AllowMultipuleTSSession&lt;/li&gt;
&lt;/ul&gt;
&lt;img src=&quot;https://images-cdn.welcomesoftware.com/Zz00MzI2YjNjNDBjODMxMWVjODUwY2E1YWIxOTFiYjQ1MQ==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiNDMyNmIzYzQwYzgzMTFlYzg1MGNhNWFiMTkxYmI0NTEiXSwiZXhwIjoxNjMyNDE2MDcyfQ.FOwTPudm_cn5WI8t40_7lm1x6x5k15OGnauzNyfFlTI&quot; alt=&quot;Fig13.png&quot; width=&quot;845&quot; height=&quot;203&quot; /&gt;

&lt;em&gt;Figure 13: RDP configuration allows connections to and from the infected device&lt;/em&gt;

Additional RDP settings are configured to enable the attacker to eventually use the remote access to the infected device without much effort.

&lt;img src=&quot;https://images-cdn.welcomesoftware.com/Zz01MmU3MTdkNjBjODMxMWVjYTY1MzBmZTFhYzIyMmFkMQ==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiNTJlNzE3ZDYwYzgzMTFlY2E2NTMwZmUxYWMyMjJhZDEiXSwiZXhwIjoxNjMyNDE2MDcyfQ.mS5pq8FweiwhqWxkHfIMa7pNqilQ968OliAenEqbXk8&quot; alt=&quot;Fig14.png&quot; width=&quot;848&quot; height=&quot;133.41199606686334&quot; /&gt;

&lt;em&gt;Figure 14: RDP configuration bypasses security on the infected device&lt;/em&gt;

The malware adds a new user account to the victim’s local area network settings with the name Administart0r and password 123mudar. To ensure it is allowed to perform admin actions on the device, the malware adds the new malicious user to the localgroup &#039;administradores&#039;.

&lt;img src=&quot;https://images-cdn.welcomesoftware.com/Zz02MGUwMTk0NjBjODMxMWVjODdmMDE5NGRhMzAyNzYwNw==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiNjBlMDE5NDYwYzgzMTFlYzg3ZjAxOTRkYTMwMjc2MDciXSwiZXhwIjoxNjMyNDE2MDcyfQ.3PzxKzREEo_5r5c44Uj6IX7pjbuoNEB2p5SBvMMtBb0&quot; alt=&quot;Fig15.png&quot; width=&quot;546&quot; height=&quot;267&quot; /&gt;

&lt;em&gt;Figure 15: ZE Loader adds a user to the administrator&#039;s local group&lt;/em&gt;

In the last step of the malware, before an attack is performed, ZE Loader further sets a new rule in the firewall that allows anyone to use RDP connections.

&lt;img src=&quot;https://images-cdn.welcomesoftware.com/Zz02YWQ1Yjc5NDBjODMxMWVjODdmMDE5NGRhMzAyNzYwNw==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiNmFkNWI3OTQwYzgzMTFlYzg3ZjAxOTRkYTMwMjc2MDciXSwiZXhwIjoxNjMyNDE2MDcyfQ.prdkkoW-LCpElg4vUnhx_394gf2CY0iYbtTcK_8bLDg&quot; alt=&quot;Fig16.png&quot; width=&quot;848&quot; height=&quot;256.10738255033556&quot; /&gt;

&lt;em&gt;Figure 16: ZE Loader creates firewall rule to allow RDP connections for all&lt;/em&gt;
&lt;h2&gt;Going Into Action Mode&lt;/h2&gt;
Once it is resident on the infected device and all the preparations are in place, ZE Loader begins monitoring the victim’s activity on the web browser, waiting for them to authenticate an online banking session or access a designated banking application on the desktop. To do that, it monitors running processes and will kill the corresponding process if one is started:

&lt;img src=&quot;https://images-cdn.welcomesoftware.com/Zz03YjBjMGU0MjBjODMxMWVjODUwY2E1YWIxOTFiYjQ1MQ==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiN2IwYzBlNDIwYzgzMTFlYzg1MGNhNWFiMTkxYmI0NTEiXSwiZXhwIjoxNjMyNDE2MDcyfQ.3DHlJgp4zb2pO4mKSBVVSdp7EgnOJ-kC-cf72AOYSCE&quot; alt=&quot;Fig17.png&quot; width=&quot;846&quot; height=&quot;275&quot; /&gt;

&lt;img src=&quot;https://images-cdn.welcomesoftware.com/Zz04NDNiNzhmZTBjODMxMWVjODdmMDE5NGRhMzAyNzYwNw==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiODQzYjc4ZmUwYzgzMTFlYzg3ZjAxOTRkYTMwMjc2MDciXSwiZXhwIjoxNjMyNDE2MDcyfQ.Hyx4HZ1z5HempZQxrcUfnR1imD7ze2mBMpt8hz6a2Fk&quot; alt=&quot;Fig17b.png&quot; width=&quot;823&quot; height=&quot;423&quot; /&gt;

&lt;em&gt;Figure 17: ZE Loader kills the process of designated banking apps if any are opened&lt;/em&gt;

After killing the app processes, it loads an encrypted string fetched from the file &#039;Host.hst.&#039; This file contains the encrypted domain name: &#039;controlefinaceiro2021.duckdns.org.&#039;

To trick the victim into believing the app did open, the malware sets up a new window to pop up with app images. It loads and decrypts an image that corresponds to the targeted bank brand from the encrypted images directory: /JDK_SDK.

&lt;img src=&quot;https://images-cdn.welcomesoftware.com/Zz04ZGY5N2M2MDBjODMxMWVjYTY1MzBmZTFhYzIyMmFkMQ==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiOGRmOTdjNjAwYzgzMTFlY2E2NTMwZmUxYWMyMjJhZDEiXSwiZXhwIjoxNjMyNDE2MDcyfQ.caNQTyawR8VuCO_F7VoZxYVMYE7nrddICSbK29IuleA&quot; alt=&quot;Fig18a.png&quot; width=&quot;811&quot; height=&quot;140&quot; /&gt;

&lt;img src=&quot;https://images-cdn.welcomesoftware.com/Zz05NWU1OGVjODBjODMxMWVjYTY1MzBmZTFhYzIyMmFkMQ==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiOTVlNThlYzgwYzgzMTFlY2E2NTMwZmUxYWMyMjJhZDEiXSwiZXhwIjoxNjMyNDE2MDcyfQ.FE_s-09PDy4UAWKEfQfprU6KzTDIZSi8owYUwfYC_p8&quot; alt=&quot;Fig18b.png&quot; width=&quot;790&quot; height=&quot;188&quot; /&gt;

&lt;em&gt;Figure 18: ZE Loader loading fake images from its locally stored trove&lt;/em&gt;

As part of the attack, the malware presents different pages/images that mimic bank applications in order to trick the victim into entering their credentials into data fields in the image. The attacker uses those to either take the session over on web browsers or access the application remotely through the victim’s device using an RDP connection.
&lt;h2&gt;ZE Loader’s Cryptography&lt;/h2&gt;
ZE Loader uses a couple of cryptographic algorithms as part of its execution and to hide assets and files. The following are the main findings from our analysis:
&lt;h2&gt;Decrypt(data, IV_array, IV_size, size)&lt;/h2&gt;
This function is responsible for decrypting the different assets of the malware, including DLL files, embedded shellcode, images, etc.

The function’s available parameters are:
&lt;ul&gt;
 	&lt;li&gt;Data: the encrypted data to be decrypted&lt;/li&gt;
 	&lt;li&gt;IV_array: array of values needed for the decryption process&lt;/li&gt;
 	&lt;li&gt;IV_size: length of the IV array&lt;/li&gt;
 	&lt;li&gt;Size: size of the encrypted data.&lt;/li&gt;
&lt;/ul&gt;
&lt;img src=&quot;https://images-cdn.welcomesoftware.com/Zz1hMjBlYmQ5NjBjODMxMWVjODdmMDE5NGRhMzAyNzYwNw==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiYTIwZWJkOTYwYzgzMTFlYzg3ZjAxOTRkYTMwMjc2MDciXSwiZXhwIjoxNjMyNDE2MDcyfQ.CvG24jkDSc61vL-W5nDzGlpIV7jgfwk8D3bbuwVQ7Uc&quot; alt=&quot;Fig19.png&quot; width=&quot;677&quot; height=&quot;574&quot; /&gt;

&lt;em&gt;Figure 19: ZE Loader’s decryption function parameters&lt;/em&gt;
&lt;h2&gt;Command_or_decrypt(command, encrypted_str, result)&lt;/h2&gt;
This function is responsible for the decryption of strings embedded in the sample. The available parameters of the function are:
&lt;ul&gt;
 	&lt;li&gt;Command: there are two types of commands for this function — C &amp; D&lt;/li&gt;
 	&lt;li&gt;Encrypted str: the encrypted string&lt;/li&gt;
 	&lt;li&gt;Result: array that will contain the decrypted string.&lt;/li&gt;
&lt;/ul&gt;
&lt;img src=&quot;https://images-cdn.welcomesoftware.com/Zz1hYmQxYzdlMjBjODMxMWVjODRjMTNmNWZhYjEyMzE0OA==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiYWJkMWM3ZTIwYzgzMTFlYzg0YzEzZjVmYWIxMjMxNDgiXSwiZXhwIjoxNjMyNDE2MDcyfQ.BKaMBPG_1KMA8B7T67rjJhhEobZcHyurbl74LszhqOU&quot; alt=&quot;Fig20.png&quot; width=&quot;765&quot; height=&quot;482&quot; /&gt;

&lt;em&gt;Figure 20: ZE Loader’s string decryption function parameters&lt;/em&gt;
&lt;h2&gt;Decrypt_image(image_path, decrypted_image, key)&lt;/h2&gt;
This function is responsible for decrypting images that the malware keeps locally, hidden in the directory JDK_SDK. The decryption algorithm the malware uses is the BlowFish encryption algorithm with the hard-coded key &#039;1&#039;. Blowfish is a symmetric-key block cipher that provides a good encryption rate in software and was likely used for that reason. The parameters of the function are:
&lt;ul&gt;
 	&lt;li&gt;Image_path: path of the encrypted image&lt;/li&gt;
 	&lt;li&gt;Decrypted_image: the decrypted image after the decryption process&lt;/li&gt;
 	&lt;li&gt;Key: key for the decryption algorithm; the key is the hard-coded char &#039;1&#039;.&lt;/li&gt;
&lt;/ul&gt;
&lt;em&gt;&lt;img src=&quot;https://images-cdn.welcomesoftware.com/Zz1iN2Y1ZTdmNzBjODMxMWVjODUwY2E1YWIxOTFiYjQ1MQ==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiYjdmNWU3ZjcwYzgzMTFlYzg1MGNhNWFiMTkxYmI0NTEiXSwiZXhwIjoxNjMyNDE2MDcyfQ.sxpTRm2xZeDLaDW_EP8-l6SyZHeNvwNYzBZWuSn2IRo&quot; alt=&quot;Fig21.png&quot; width=&quot;597&quot; height=&quot;431&quot; /&gt;&lt;/em&gt;

&lt;em&gt;Figure 21: ZE Loader’s image decryption function and its parameters&lt;/em&gt;
&lt;h2&gt;Piecing It Together&lt;/h2&gt;
The malware keeps encrypted images that mimic its various targets’ websites and designated applications locally in the &#039;JDK_SDK&#039; directory. After decrypting that directory, we were able to access a wide range of targets. On top of popular banks, the malware targets some blockchain platforms and cryptocurrency exchange platforms.

The images also led to insights regarding some of the sophisticated ways the attacker overcomes two-factor authentication challenges in order to steal user credentials. For example, one of the malware’s assets named &#039;coin.tlb&#039; is a file that contains two encrypted strings. &lt;span class=&quot;&quot;&gt;After decrypting the strings, we found the two strings below:&lt;/span&gt;

&lt;h2&gt;Remote Overlay Trojans Still Going Strong&lt;/h2&gt;
While it is a dated threat, remote overlay Trojans are an enduring staple in the cyber crime arena. Prolific in Latin America, they also target European countries where the same languages are spoken, so as to maximize the reach of their attacks. The strength of attacks that leverage this malware type is the remote access to user devices. Adding manual work in real time allows attackers to extract critical transaction elements from their victims and finalize transactions that are otherwise adequately protected.

While it lacks sophistication on the code level, its overall scheme continues to work. To mitigate the risk of remote overlay Trojans, here are some things users can do:
&lt;ul&gt;
 	&lt;li&gt;Do not open &lt;a href=&quot;https://securityintelligence.com/news/wine-themed-phishing-emails-2020/&quot; target=&quot;_blank&quot; rel=&quot;noopener&quot;&gt;unsolicited emails&lt;/a&gt; and don’t click links or attachments inside such messages&lt;/li&gt;
 	&lt;li&gt;Do not log in to bank accounts from an email that appears to urge action&lt;/li&gt;
 	&lt;li&gt;When in doubt, call your bank&lt;/li&gt;
 	&lt;li&gt;Have an antivirus installed on your device and turn on automatic updates&lt;/li&gt;
 	&lt;li&gt;Keep your operating system and all programs up to date&lt;/li&gt;
 	&lt;li&gt;Delete applications that are not in use&lt;/li&gt;
 	&lt;li&gt;Disable remote connections to your device. Press Windows + X à click &#039;System&#039;. From the left sidebar click &#039;Remote Desktop&#039; and make sure the remote desktop option is toggled off.&lt;/li&gt;
&lt;/ul&gt;

<html><body><p>IBM Trusteer closely follows developments in the financial cyber crime arena. Recently, we discovered a new remote overlay malware that is more persistent and more sophisticated than most current-day codes. In this post we will dive into the technical details of the sample we worked on and present ZE Loader&rsquo;s capabilities and features. The parts that differ from other malware of this kind are:</p>
<ul>
<li>Installation of a backdoor to the victim&rsquo;s device</li>
<li>Remaining stealthy in the guise of legitimate software</li>
<li>Holding permanent assets on the victim&rsquo;s device</li>
<li>Stealing user credentials.</li>
</ul>
<p>Another aspect we examine here is the malware&rsquo;s algorithms used in the encryption of its resources and events. We will suggest some tactics to detect the presence of ZE Loader on infected devices to mitigate its potential impact.</p>
<h2>Overlay Malware Is an Enduring Threat</h2>
<p>Overlay malware is not a new threat, nor is it very sophisticated. Yet, this malware category, which typically spreads in Latin America, Spain and Portugal, is an enduring one. We keep seeing it used in attacks on online banking users in those regions, and its success fuels the interest of cyber criminals to continue using it.</p>
<p>In the case of ZE Loader, we did see some new features that push the typical boundaries of <a href="https://securityintelligence.com/posts/vizom-malware-targets-brazilian-bank-customers-remote-overlay/"  target="_blank" rel="noopener nofollow" >overlay Trojans</a>. For example, most malware in this category does not keep assets on the infected device, but ZE Loader does. In most cases, this sort of malware does not go to the lengths of hiding its presence; its lifecycle is short and the effort is futile. ZE Loader does use some stealth tactics.</p>
<h2>Typical Attack Anatomy</h2>
<p>A remote overlay attack follows a rather familiar path. Once the user becomes infected &mdash; usually via malspam, phishing pages or malicious attachments &mdash; the malware is installed on the target device. In most cases, the malware begins monitoring browser window names for a targeted bank&rsquo;s site. It then goes into action upon access to a hard-coded list of entities. With the regional focus of this malware type, it mostly goes after local banks.</p>
<p>Once the user lands on a targeted website, the attacker is notified in real-time. The attacker can then take over the device remotely using the remote access feature. As the victim accesses their online banking account, the attacker can see their activity and choose a time to interject. To trick users into divulging authentication codes or other personal data, attackers display full-screen overlay images that keep the victim from continuing the banking session. In the background, the attacker initiates a fraudulent money transfer from the compromised account and leverages the victim&rsquo;s presence in real-time to obtain the required information to complete it.</p>
<p>It&rsquo;s not an automated fraud scheme, but it is one that keeps working in certain parts of the world, which makes it a risk that banks must continue to reckon with.</p>

<h2>ZE Loader&rsquo;s Execution and Post-Infection Behavior</h2>

<p>In a recent campaign we analyzed, the attackers were using several payload options to infect the victim&rsquo;s device. These payloads&rsquo; folders contained binary files from legitimate applications. Once executed, the seemingly benign applications would load the malware&rsquo;s malicious DLL.</p>

<p>When we viewed the ZE Loader attack from an anatomy perspective, the elements interact as follows:</p>
<p><amp-img src="https://images-cdn.welcomesoftware.com/Zz01NjE4NWI1YjBjODIxMWVjOTUyYmI5YjkyODhjZjJiMQ==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiNTYxODViNWIwYzgyMTFlYzk1MmJiOWI5Mjg4Y2YyYjEiXSwiZXhwIjoxNjMyNDE2MDcyfQ.JYP8q-khRuqaZFFYsr04DOBSuAFgz4P3eR6eiTF7VDI" layout="intrinsic" class="" alt="Fig4.png" width="532.8635170603675" height="322" lightbox="lightbox"></amp-img></p>
<p><em>Figure 4: ZE Loader&rsquo;s attack anatomy</em></p>
<p>Running the legitimate program used as ZE Loader&rsquo;s front also loads the malicious DLL. In this case, it is DVDSetting.dll, and we can see in the image below that the legitimate software imports that DLL.</p>
<p><amp-img src="https://images-cdn.welcomesoftware.com/Zz05ZmVmNjM4NjBjODIxMWVjODRjMTNmNWZhYjEyMzE0OA==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiOWZlZjYzODYwYzgyMTFlYzg0YzEzZjVmYWIxMjMxNDgiXSwiZXhwIjoxNjMyNDE2MDcyfQ.u5jKTdHZuULsazfgggVbee3zysm0_b6IU2XH3W-eT9Y" layout="intrinsic" class="" alt="Fig5.png" width="590" height="571" lightbox="lightbox"></amp-img></p>

Inside DVDSettings.dll there is also some embedded shellcode, also in encrypted form, which is responsible for unpacking and running the procSettings UPX-packed DLL post decryption.</p>
<p><amp-img src="https://images-cdn.welcomesoftware.com/Zz1hYjc2ZWIyYTBjODIxMWVjYTM4YjE1NjdiMTIwZjUzZg==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiYWI3NmViMmEwYzgyMTFlY2EzOGIxNTY3YjEyMGY1M2YiXSwiZXhwIjoxNjMyNDE2MDcyfQ.Rmuvh2mkl3-Z05GYLSTglfWtVoVt7OqJV3JtzA_YHG4" layout="intrinsic" class="" alt="Fig6.png" width="711" height="331" lightbox="lightbox"></amp-img></p>

<p><amp-img src="https://images-cdn.welcomesoftware.com/Zz1mYmZhMzhjMjBjODIxMWVjODUwY2E1YWIxOTFiYjQ1MQ==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiZmJmYTM4YzIwYzgyMTFlYzg1MGNhNWFiMTkxYmI0NTEiXSwiZXhwIjoxNjMyNDE2MDcyfQ.ze1iU3IwP3AsoIqJo6t_qQQkHt062apObSnCzkavfaA" layout="intrinsic" class="" alt="Fig7.png" width="837" height="529" lightbox="lightbox"></amp-img></p>

<p>Next, the decrypted shellcode unpacks the decrypted procSettings DLL file and then calls the entry point of procSettings DLL.</p>
<h2>The procSettings DLL</h2>
<p>To find out more about what&rsquo;s inside this core DLL, we performed a static examination of the DLL. This did not shed light on its functionality and rules that govern its activity. One of the things we did see is that this DLL is Borland Delphi compiled and that it imports different functions from different DLLs. This suggests that procSettings is the DLL that holds most of the logic of the malware and its implementation.</p>
<p>A dynamic analysis we ran allowed us to examine the exported function THetholdImplementationIntercept. We saw that first the malware created a mutex with the name CodeCall.Net Mutey in order to prevent multiple instances of the malware running at the same time.</p>
<p>Next, the malware ran a check to discern whether the targeted bank application was installed on the infected device. It did that by searching the software directory under %appdatalocal%.</p>
<p>If the software the attackers are interested in is indeed installed on the device, it further checks if the file C:\ProgramData\OkApp.is exists. This file is one of the malware&rsquo;s files, used as an indicator; this file is empty of content.</p>
<p><amp-img src="https://images-cdn.welcomesoftware.com/Zz0wYmU5ODlhZTBjODMxMWVjOTUyYmI5YjkyODhjZjJiMQ==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiMGJlOTg5YWUwYzgzMTFlYzk1MmJiOWI5Mjg4Y2YyYjEiXSwiZXhwIjoxNjMyNDE2MDcyfQ.ED0kiCqSmDJWKkFrQr7m3lXAuWqiHeUbjcXxLIPPJqI" layout="intrinsic" class="" alt="Fig8.png" width="641.3121869782972" height="322" lightbox="lightbox"></amp-img></p>
<p><em>Figure 8: ZE Loader&rsquo;s indicator file that checks for previous infection</em></p>

<ol>
<li>First, ZE Loader checks that it is running with administrator privileges.</li>
</ol>
<p><amp-img src="https://images-cdn.welcomesoftware.com/Zz0xODE5NDQ4YTBjODMxMWVjODUwY2E1YWIxOTFiYjQ1MQ==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiMTgxOTQ0OGEwYzgzMTFlYzg1MGNhNWFiMTkxYmI0NTEiXSwiZXhwIjoxNjMyNDE2MDcyfQ.K2V_WvyYqz_o5szOcDh_m0Zf52sbC6egnYFxWg9EFww" layout="intrinsic" class="" alt="Fig9.png" width="848" height="307.4458874458874" lightbox="lightbox"></amp-img></p>

</ol>

<h2>Manipulating Security Settings</h2>
<p>In the next step, ZE Loader manipulates some security settings to enable the attacker to have undisturbed remote access to the infected device.</p>
<p>ZE Loader searches for the service &lsquo;TermService&rsquo;. This service allows RDP connections to stream to and from the client device. ZE Loader sets its configuration settings to SERVICE_AUTO_START with the path of the RDP DLL file it already saved on disk.</p>
<p>Next, ZE Loader changes the settings of the infected device to allow and establish multiple RDP connections to and from that device. The following settings are toggled to &lsquo;true&rsquo;:</p>
<ul>
<li>HKLM\System\CurrentControlSet\Control\Terminal Server\fDenyTSConnection</li>
<li>HKLM\System\CurrentControlSet\Control\Terminal Server\Licensing Core\EnableConCurrentSessions</li>
<li>HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon\AllowMultipuleTSSession</li>
</ul>
<p><amp-img src="https://images-cdn.welcomesoftware.com/Zz00MzI2YjNjNDBjODMxMWVjODUwY2E1YWIxOTFiYjQ1MQ==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiNDMyNmIzYzQwYzgzMTFlYzg1MGNhNWFiMTkxYmI0NTEiXSwiZXhwIjoxNjMyNDE2MDcyfQ.FOwTPudm_cn5WI8t40_7lm1x6x5k15OGnauzNyfFlTI" layout="intrinsic" class="" alt="Fig13.png" width="845" height="203" lightbox="lightbox"></amp-img></p>
<p><em>Figure 13: RDP configuration allows connections to and from the infected device</em></p>
<p>Additional RDP settings are configured to enable the attacker to eventually use the remote access to the infected device without much effort.</p>
<p><amp-img src="https://images-cdn.welcomesoftware.com/Zz01MmU3MTdkNjBjODMxMWVjYTY1MzBmZTFhYzIyMmFkMQ==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiNTJlNzE3ZDYwYzgzMTFlY2E2NTMwZmUxYWMyMjJhZDEiXSwiZXhwIjoxNjMyNDE2MDcyfQ.mS5pq8FweiwhqWxkHfIMa7pNqilQ968OliAenEqbXk8" layout="intrinsic" class="" alt="Fig14.png" width="848" height="133.41199606686334" lightbox="lightbox"></amp-img></p>
<p><em>Figure 14: RDP configuration bypasses security on the infected device</em></p>
<p>The malware adds a new user account to the victim&rsquo;s local area network settings with the name Administart0r and password 123mudar. To ensure it is allowed to perform admin actions on the device, the malware adds the new malicious user to the localgroup &lsquo;administradores&rsquo;.</p>
<p><amp-img src="https://images-cdn.welcomesoftware.com/Zz02MGUwMTk0NjBjODMxMWVjODdmMDE5NGRhMzAyNzYwNw==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiNjBlMDE5NDYwYzgzMTFlYzg3ZjAxOTRkYTMwMjc2MDciXSwiZXhwIjoxNjMyNDE2MDcyfQ.3PzxKzREEo_5r5c44Uj6IX7pjbuoNEB2p5SBvMMtBb0" layout="intrinsic" class="" alt="Fig15.png" width="546" height="267" lightbox="lightbox"></amp-img></p>
<p><em>Figure 15: ZE Loader adds a user to the administrator&rsquo;s local group</em></p>
<p>In the last step of the malware, before an attack is performed, ZE Loader further sets a new rule in the firewall that allows anyone to use RDP connections.</p>
<p><amp-img src="https://images-cdn.welcomesoftware.com/Zz02YWQ1Yjc5NDBjODMxMWVjODdmMDE5NGRhMzAyNzYwNw==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiNmFkNWI3OTQwYzgzMTFlYzg3ZjAxOTRkYTMwMjc2MDciXSwiZXhwIjoxNjMyNDE2MDcyfQ.prdkkoW-LCpElg4vUnhx_394gf2CY0iYbtTcK_8bLDg" layout="intrinsic" class="" alt="Fig16.png" width="848" height="256.10738255033556" lightbox="lightbox"></amp-img></p>
<p><em>Figure 16: ZE Loader creates firewall rule to allow RDP connections for all</em></p>
<h2>Going Into Action Mode</h2>
<p>Once it is resident on the infected device and all the preparations are in place, ZE Loader begins monitoring the victim&rsquo;s activity on the web browser, waiting for them to authenticate an online banking session or access a designated banking application on the desktop. To do that, it monitors running processes and will kill the corresponding process if one is started:</p>
<p><amp-img src="https://images-cdn.welcomesoftware.com/Zz03YjBjMGU0MjBjODMxMWVjODUwY2E1YWIxOTFiYjQ1MQ==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiN2IwYzBlNDIwYzgzMTFlYzg1MGNhNWFiMTkxYmI0NTEiXSwiZXhwIjoxNjMyNDE2MDcyfQ.3DHlJgp4zb2pO4mKSBVVSdp7EgnOJ-kC-cf72AOYSCE" layout="intrinsic" class="" alt="Fig17.png" width="846" height="275" lightbox="lightbox"></amp-img></p>
<p><amp-img src="https://images-cdn.welcomesoftware.com/Zz04NDNiNzhmZTBjODMxMWVjODdmMDE5NGRhMzAyNzYwNw==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiODQzYjc4ZmUwYzgzMTFlYzg3ZjAxOTRkYTMwMjc2MDciXSwiZXhwIjoxNjMyNDE2MDcyfQ.Hyx4HZ1z5HempZQxrcUfnR1imD7ze2mBMpt8hz6a2Fk" layout="intrinsic" class="" alt="Fig17b.png" width="823" height="423" lightbox="lightbox"></amp-img></p>
<p><em>Figure 17: ZE Loader kills the process of designated banking apps if any are opened</em></p>
<p>After killing the app processes, it loads an encrypted string fetched from the file &lsquo;Host.hst.&rsquo; This file contains the encrypted domain name: &lsquo;controlefinaceiro2021.duckdns.org.&rsquo;</p>
<p>To trick the victim into believing the app did open, the malware sets up a new window to pop up with app images. It loads and decrypts an image that corresponds to the targeted bank brand from the encrypted images directory: /JDK_SDK.</p>
<p><amp-img src="https://images-cdn.welcomesoftware.com/Zz04ZGY5N2M2MDBjODMxMWVjYTY1MzBmZTFhYzIyMmFkMQ==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiOGRmOTdjNjAwYzgzMTFlY2E2NTMwZmUxYWMyMjJhZDEiXSwiZXhwIjoxNjMyNDE2MDcyfQ.caNQTyawR8VuCO_F7VoZxYVMYE7nrddICSbK29IuleA" layout="intrinsic" class="" alt="Fig18a.png" width="811" height="140" lightbox="lightbox"></amp-img></p>
<p><amp-img src="https://images-cdn.welcomesoftware.com/Zz05NWU1OGVjODBjODMxMWVjYTY1MzBmZTFhYzIyMmFkMQ==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiOTVlNThlYzgwYzgzMTFlY2E2NTMwZmUxYWMyMjJhZDEiXSwiZXhwIjoxNjMyNDE2MDcyfQ.FE_s-09PDy4UAWKEfQfprU6KzTDIZSi8owYUwfYC_p8" layout="intrinsic" class="" alt="Fig18b.png" width="790" height="188" lightbox="lightbox"></amp-img></p>
<p><em>Figure 18: ZE Loader loading fake images from its locally stored trove</em></p>
<p>As part of the attack, the malware presents different pages/images that mimic bank applications in order to trick the victim into entering their credentials into data fields in the image. The attacker uses those to either take the session over on web browsers or access the application remotely through the victim&rsquo;s device using an RDP connection.</p>
<h2>ZE Loader&rsquo;s Cryptography</h2>
<p>ZE Loader uses a couple of cryptographic algorithms as part of its execution and to hide assets and files. The following are the main findings from our analysis:</p>
<h2>Decrypt(data, IV_array, IV_size, size)</h2>
<p>This function is responsible for decrypting the different assets of the malware, including DLL files, embedded shellcode, images, etc.</p>
<p>The function&rsquo;s available parameters are:</p>
<ul>
<li>Data: the encrypted data to be decrypted</li>
<li>IV_array: array of values needed for the decryption process</li>
<li>IV_size: length of the IV array</li>
<li>Size: size of the encrypted data.</li>
</ul>
<p><amp-img src="https://images-cdn.welcomesoftware.com/Zz1hMjBlYmQ5NjBjODMxMWVjODdmMDE5NGRhMzAyNzYwNw==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiYTIwZWJkOTYwYzgzMTFlYzg3ZjAxOTRkYTMwMjc2MDciXSwiZXhwIjoxNjMyNDE2MDcyfQ.CvG24jkDSc61vL-W5nDzGlpIV7jgfwk8D3bbuwVQ7Uc" layout="intrinsic" class="" alt="Fig19.png" width="677" height="574" lightbox="lightbox"></amp-img></p>
<p><em>Figure 19: ZE Loader&rsquo;s decryption function parameters</em></p>
<h2>Command_or_decrypt(command, encrypted_str, result)</h2>
<p>This function is responsible for the decryption of strings embedded in the sample. The available parameters of the function are:</p>
<ul>
<li>Command: there are two types of commands for this function &mdash; C &amp; D</li>
<li>Encrypted str: the encrypted string</li>
<li>Result: array that will contain the decrypted string.</li>
</ul>
<p><amp-img src="https://images-cdn.welcomesoftware.com/Zz1hYmQxYzdlMjBjODMxMWVjODRjMTNmNWZhYjEyMzE0OA==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiYWJkMWM3ZTIwYzgzMTFlYzg0YzEzZjVmYWIxMjMxNDgiXSwiZXhwIjoxNjMyNDE2MDcyfQ.BKaMBPG_1KMA8B7T67rjJhhEobZcHyurbl74LszhqOU" layout="intrinsic" class="" alt="Fig20.png" width="765" height="482" lightbox="lightbox"></amp-img></p>
<p><em>Figure 20: ZE Loader&rsquo;s string decryption function parameters</em></p>
<h2>Decrypt_image(image_path, decrypted_image, key)</h2>
<p>This function is responsible for decrypting images that the malware keeps locally, hidden in the directory JDK_SDK. The decryption algorithm the malware uses is the BlowFish encryption algorithm with the hard-coded key &lsquo;1&rsquo;. Blowfish is a symmetric-key block cipher that provides a good encryption rate in software and was likely used for that reason. The parameters of the function are:</p>
<ul>
<li>Image_path: path of the encrypted image</li>
<li>Decrypted_image: the decrypted image after the decryption process</li>
<li>Key: key for the decryption algorithm; the key is the hard-coded char &lsquo;1&rsquo;.</li>
</ul>
<p><em><amp-img src="https://images-cdn.welcomesoftware.com/Zz1iN2Y1ZTdmNzBjODMxMWVjODUwY2E1YWIxOTFiYjQ1MQ==?token=eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJzdWIiOlsiYjdmNWU3ZjcwYzgzMTFlYzg1MGNhNWFiMTkxYmI0NTEiXSwiZXhwIjoxNjMyNDE2MDcyfQ.sxpTRm2xZeDLaDW_EP8-l6SyZHeNvwNYzBZWuSn2IRo" layout="intrinsic" class="" alt="Fig21.png" width="597" height="431" lightbox="lightbox"></amp-img></em></p>
<p><em>Figure 21: ZE Loader&rsquo;s image decryption function and its parameters</em></p>

<p>The images also led to insights regarding some of the sophisticated ways the attacker overcomes two-factor authentication challenges in order to steal user credentials. <h2>Remote Overlay Trojans Still Going Strong</h2>
<p>While it is a dated threat, remote overlay Trojans are an enduring staple in the cyber crime arena. Prolific in Latin America, they also target European countries where the same languages are spoken, so as to maximize the reach of their attacks. The strength of attacks that leverage this malware type is the remote access to user devices. Adding manual work in real time allows attackers to extract critical transaction elements from their victims and finalize transactions that are otherwise adequately protected.</p>
<p>While it lacks sophistication on the code level, its overall scheme continues to work. To mitigate the risk of remote overlay Trojans, here are some things users can do:</p>
<ul>
<li>Do not open <a href="https://securityintelligence.com/news/wine-themed-phishing-emails-2020/"  target="_blank" rel="noopener nofollow" >unsolicited emails</a> and don&rsquo;t click links or attachments inside such messages</li>
<li>Do not log in to bank accounts from an email that appears to urge action</li>
<li>When in doubt, call your bank</li>
<li>Have an antivirus installed on your device and turn on automatic updates</li>
<li>Keep your operating system and all programs up to date</li>
<li>Delete applications that are not in use</li>
<li>Disable remote connections to your device. Press Windows + X &agrave; click &lsquo;System&rsquo;. From the left sidebar click &lsquo;Remote Desktop&rsquo; and make sure the remote desktop option is toggled off.</li>
</ul>
