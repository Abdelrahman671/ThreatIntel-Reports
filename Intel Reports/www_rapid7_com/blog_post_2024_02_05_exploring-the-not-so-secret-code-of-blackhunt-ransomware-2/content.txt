                            <h3></h3><p>It seems like every week, the cybersecurity landscape sees the emergence of yet another ransomware variant, with Black Hunt being one of the latest additions. <a href="https://twitter.com/RakeshKrish12/status/1597839380558716928">Initially</a> reported by cybersecurity researchers in 2022, this new threat has quickly made its presence known. In a recent incident, Black Hunt ransomware wreaked havoc by compromising around 300 companies in Paraguay.</p><p>Rapid7 Labs consistently monitors emerging threats, and this new ransomware variant caught our attention for several reasons. The behavior and potential impact of this new variant raised concerns among our team, prompting us to conduct a thorough analysis to better understand its capabilities and potential risks.</p><p>During our analysis we found notable similarities between Black Hunt ransomware and LockBit, which suggested that it uses leaked code of Lockbit. In addition, it uses some techniques similar to REvil ransomware.</p><h3 id="technical-analysis">Technical Analysis</h3><p>In this analysis we examined the Black Hunt sample shared on X (formerly Twitter), by <a href="https://twitter.com/malwrhunterteam/status/1744499152011104549">MalwareHunterTeam</a>. In our investigation we found some interesting techniques and features used by this malware. The recent Black Hunt sample is a C++ executable, which widely reuses the leaked Lockbit ransomware code and shares similarities with several other recently spotted ransomware families.The execution of the ransomware on an infected machine starts with a check for a file named Vaccine.txt under directory C:\ProgramData path</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.rapid7.com/content/images/2024/02/Screenshot-2024-02-05-at-10.29.13-AM.png" class="kg-image"><figcaption>Figure 1 - Check for the existence of Vaccine.txt file</figcaption></figure><p>If the file is found, malware terminates its execution. This file detection acts as an anti-exploitation flag for the ransomware. As this is not a well-known anti-sandbox/anti-VM technique, we assume that this file is in use by the threat actor (TA) itself. It is either created by the victim which is instructed to create it after the ransom is paid or dropped by the decryptor, if one is sent to the victim to decrypt the <a href="https://www.rapid7.com/fundamentals/data-encryption/">encrypted data</a>. Adding that check is logical if the ransomware operators consider scenarios where their persistence mechanism and the ransomware binary remain on the system even after the ransom is paid and files are decrypted. Moreover, the fact that all files dropped by the ransomware are placed in the C:\ProgramData Â directory further supports our assessment that this file is associated with the Black Hunt operation.</p><p>Next, the malware adjusts the following privileges to processes Access Token by using the `AdjustTokenPrivileges` API function:</p><!--kg-card-begin: markdown--><table>

This function retrieves the command-line string for the current process, including the program name and any arguments passed during startup. Following this, another function call is made to `CMD_ARGS`.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://blog.rapid7.com/content/images/2024/02/Screenshot-2024-02-05-at-10.31.40-AM.png" class="kg-image"><figcaption>Figure 4</figcaption></figure><p>The function processes a command-line string and checks if any of the following arguments were passed:</p><!--kg-card-begin: markdown--><table>
