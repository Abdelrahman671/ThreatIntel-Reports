<h2 class="wp-block-heading" id="1">Executive Summary</h2>



<ul class="wp-block-list">
<li>In late 2023, Sygnia’s Incident Response team was engaged by a client whose network was compromised and was leveraged to penetrate one of its business partner’s network.</li>



<li>During the investigation, several servers, workstations, and users were found to be compromised by a threat actor who deployed various tools to communicate with a set of C2 servers.</li>



<li>One of these tools was identified as a variant of Demodex, a rootkit previously associated with the threat group known as GhostEmperor.</li>



<li>GhostEmperor is a sophisticated China-nexus threat group known to target mostly South-East Asian telecommunication and government entities, first disclosed by Kaspersky in a <a href="https://securelist.com/ghostemperor-from-proxylogon-to-kernel-mode/104407/" target="_blank" rel="noreferrer noopener">blog</a> published in September 2021.</li>



<li>GhostEmperor employs a multi-stage malware to achieve stealth execution and persistence and utilizes several methods to impede analysis process.</li>



<li>Usually, once the threat group gains initial access to the victim’s network by using vulnerabilities such as ProxyLogon, a batch file is executed to initiate the infection chain.</li>



<li>In this blog we describe a new infection chain deployed by GhostEmperor, which includes several loading schemes and various obfuscation techniques utilized by the threat group.</li>
</ul>



<h2 class="wp-block-heading" id="2">Introduction</h2>



<p>During Sygnia’s analysis of the forensic findings extracted from the victim’s environment, the team found strong resemblance to the multi-stage tool which was described in Kaspersky’s blog from 2021. However, our investigation yielded some alterations in the infection chain and a slightly different C++ DLL variant.</p>



<p>Among these alterations, the variant we analyzed incorporates an EDR evasion technique and uses a reflective loader to execute the Core-Implant. Additionally, we identified the use of different file names and registry keys. The variant we encountered appears to have been compiled in July 2021, suggesting it might be a more recent version than the one originally analyzed by Kaspersky.</p>



<p>This blog post focuses on the key differences we identified and analyzed in the infection chain and the loading scheme operations.</p>

<h2 class="wp-block-heading" id="3">WMIExec</h2>



<p>WMIExec is a command-line tool used for executing commands on remote Windows systems through Windows Management Instrumentation (WMI).<br>It is part of the Impacket Toolkit, which is an open-source collection of modules written in Python for&nbsp;programmatically constructing and manipulating network protocols, that is&nbsp;commonly used by threat actors and red teams.</p>



<p>During our investigation, we observed that the threat actor used this tool to run a batch file, initiating the infection chain on the victim’s compromised machine. The output logs were saved to a file located at <code>c:\windows\temp</code> using a local SMB path. The following command was executed:<br></p>



<p>cmd.exe /Q /c c:\windows\vss\1.bat &gt; \127.0.0.1\C$\Windows\Temp[generated_string] 2&gt;&amp;1</p>



<figure class="wp-block-image size-large"><img decoding="async" width="1024" height="393" src="data:image/svg+xml,%3Csvg%20xmlns='http://www.w3.org/2000/svg'%20viewBox='0%200%201024%20393'%3E%3C/svg%3E" alt="Snippet showcasing the WMIExec command being executed on a victim machine with batch script '1.bat'" class="wp-image-3629" data-lazy-srcset="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-2_31-1024x393.jpg 1024w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-2_31-300x115.jpg 300w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-2_31-768x294.jpg 768w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-2_31.jpg 1500w" data-lazy-sizes="(max-width: 1024px) 100vw, 1024px" data-lazy-src="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-2_31-1024x393.jpg" /><noscript><img decoding="async" width="1024" height="393" src="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-2_31-1024x393.jpg" alt="Snippet showcasing the WMIExec command being executed on a victim machine with batch script '1.bat'" class="wp-image-3629" srcset="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-2_31-1024x393.jpg 1024w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-2_31-300x115.jpg 300w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-2_31-768x294.jpg 768w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-2_31.jpg 1500w" sizes="(max-width: 1024px) 100vw, 1024px" /></noscript><figcaption class="wp-element-caption"><em>Snippet showcasing the WMIExec command being executed on a victim machine with batch script &#8216;1.bat&#8217;</em></figcaption></figure>



<h2 class="wp-block-heading" id="4">Batch File</h2>



<p>The batch file starts the infection by installing the malware and obtaining persistency by the following actions:</p>



<p>It starts by dropping a CAB file named “1.cab” to C:\Windows\Web. CAB is a compressed archive format commonly utilized in Windows to bundle multiple files.<br>The batch file then uses expand.exe &#8211; a native Windows tool used for file extraction from compressed Cabinet files (.cab), to extract these four files:</p>



<ul class="wp-block-list">
<li>prints1m.dll – Service DLL.</li>



<li>Service.ps1 – encrypted Powershell.</li>



<li>config.REG – registry dump of AES decryption key.</li>



<li>AesedMemoryBinX64.REG &#8211; registry dump of AES-encrypted shellcode containing the Core-Implant.</li>
</ul>



<p>Next, the batch file imports the two registry files using the <code><strong>reg.exe import [file]</strong></code> command to set two registry keys with encrypted values, which will be used later to execute the next stage.</p>

<p>The threat actor employs several LOLBins such as reg.exe and expand.exe within the batch file to achieve stealthiness as they are legitimate and signed Microsoft built-in tools which do not arouse any suspicion.</p>

<h2 class="wp-block-heading" id="5">PowerShell script</h2>



<p>The decrypted PowerShell script creates a new service named “WdiSystem” that loads the malicious Service DLL (prints1m.dll). It also creates a service group called “WdiSystemhost” and runs the malicious service within this group. By running the malicious service within the context of the “WdiSystem<strong>h</strong>ost” service group, the threat actor masquerades the malware’s execution as a legitimate Windows system process, as it resembles the authentic and legitimate WdiSystem<strong>H</strong>ost (“Windows Diagnostic System Host” service).</p>

<h2 class="wp-block-heading" id="6">Prints1m.dll &#8211; Service DLL</h2>



<p>This Service DLL dynamically loads all of the necessary functions it requires for operation by navigating through an internal OS structure named Process Environment Block, which contains the already loaded libraries and functions in the process.</p>



<p>The Kernel32 library, loaded by default in every process, is used by the malware to access the LoadLibraryA function, which is responsible for loading DLLs into the process.</p>



<p>Subsequently, an encrypted configuration located at the DLL’s data section (offset 0x4050) is decrypted using a custom decryption scheme, which contains the following parameters:</p>



<ul class="wp-block-list">
<li>Initial sleep time.</li>



<li>Registry paths of the shellcode location (which was established by the batch file).</li>



<li>A list of module and function names required for operation (offset 0x45F0).</li>
</ul>



<p>The service uses this list to create an in-memory Import Address Table, loading the modules it requires using the LoadLibraryA function, and traverses each module’s export table to obtain the necessary functions.</p>

<h2 class="wp-block-heading" id="7">EDR Evasion and Anti-User-Mode Hooking Technique</h2>



<p>Antivirus and EDR solutions typically inject DLLs into the address space of running applications to facilitate user-mode hooking, thus identifying and preventing malicious activity within the processes.<br>During our investigation we observed that the threat actor added an evasion technique to the Service DLL by setting a specific mitigation policy to the process:</p>



<figure class="wp-block-image size-large is-resized"><img decoding="async" width="1024" height="159" src="data:image/svg+xml,%3Csvg%20xmlns='http://www.w3.org/2000/svg'%20viewBox='0%200%201024%20159'%3E%3C/svg%3E" alt="Calling SetMitigationPolicy with ProcessSignaturePolicy as parameter to set the mitigation policy" class="wp-image-3605" style="width:840px;height:auto" data-lazy-srcset="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-13-1024x159.jpg 1024w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-13-300x47.jpg 300w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-13-768x119.jpg 768w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-13.jpg 1500w" data-lazy-sizes="(max-width: 1024px) 100vw, 1024px" data-lazy-src="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-13-1024x159.jpg" /><noscript><img decoding="async" width="1024" height="159" src="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-13-1024x159.jpg" alt="Calling SetMitigationPolicy with ProcessSignaturePolicy as parameter to set the mitigation policy" class="wp-image-3605" style="width:840px;height:auto" srcset="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-13-1024x159.jpg 1024w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-13-300x47.jpg 300w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-13-768x119.jpg 768w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-13.jpg 1500w" sizes="(max-width: 1024px) 100vw, 1024px" /></noscript><figcaption class="wp-element-caption"><em>Calling SetMitigationPolicy with ProcessSignaturePolicy as parameter to set the mitigation policy</em></figcaption></figure>



<p>Mitigation policies, such as ASLR, DEP and CFG, are security measures implemented by the OS to mitigate attacks and vulnerabilities such as Buffer Overflows and Code Injections. Some of these mitigation policies are enabled in the process by default. In our investigation, the threat actor set up a particular mitigation named “ProcessSignaturePolicy” which forbid loading DLLs that are not signed by Microsoft to the process.<br>This means that any security solution trying to inject a DLL not signed by Microsoft will fail to do so. This technique helps circumvent analysis tools by limiting user-mode hooking.</p>



<figure class="wp-block-image size-large"><img decoding="async" width="1024" height="783" src="data:image/svg+xml,%3Csvg%20xmlns='http://www.w3.org/2000/svg'%20viewBox='0%200%201024%20783'%3E%3C/svg%3E" alt="Service’s mitigation policies" class="wp-image-3606" data-lazy-srcset="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-14-1024x783.jpg 1024w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-14-300x229.jpg 300w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-14-768x587.jpg 768w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-14.jpg 1500w" data-lazy-sizes="(max-width: 1024px) 100vw, 1024px" data-lazy-src="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-14-1024x783.jpg" /><noscript><img decoding="async" width="1024" height="783" src="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-14-1024x783.jpg" alt="Service’s mitigation policies" class="wp-image-3606" srcset="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-14-1024x783.jpg 1024w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-14-300x229.jpg 300w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-14-768x587.jpg 768w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-14.jpg 1500w" sizes="(max-width: 1024px) 100vw, 1024px" /></noscript><figcaption class="wp-element-caption"><em>Service’s mitigation policies</em></figcaption></figure>



<p>The fact that many antivirus vendors employ DLLs with a legitimate Microsoft signature, and that some security solutions inject their DLLs prior to the invocation of SetProcessMitigationPolicy, limits the effectiveness of this method.</p>



<h2 class="wp-block-heading" id="8">Shellcode and Reflective loader</h2>



<p>The Service DLL reads two encrypted registry keys that were set by the batch file:<br>“AKey” – an AES decryption key<br>“inputlog” – an AES-encrypted shellcode containing the core-implant.</p>



<figure class="wp-block-image size-large"><img decoding="async" width="1024" height="223" src="data:image/svg+xml,%3Csvg%20xmlns='http://www.w3.org/2000/svg'%20viewBox='0%200%201024%20223'%3E%3C/svg%3E" alt="Snippet from Sandbox execution of the threat actor's malicious service showing the read activity performed by the service of the two registry keys." class="wp-image-3607" data-lazy-srcset="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-15-1024x223.jpg 1024w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-15-300x65.jpg 300w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-15-768x167.jpg 768w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-15.jpg 1500w" data-lazy-sizes="(max-width: 1024px) 100vw, 1024px" data-lazy-src="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-15-1024x223.jpg" /><noscript><img decoding="async" width="1024" height="223" src="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-15-1024x223.jpg" alt="Snippet from Sandbox execution of the threat actor's malicious service showing the read activity performed by the service of the two registry keys." class="wp-image-3607" srcset="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-15-1024x223.jpg 1024w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-15-300x65.jpg 300w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-15-768x167.jpg 768w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-15.jpg 1500w" sizes="(max-width: 1024px) 100vw, 1024px" /></noscript><figcaption class="wp-element-caption"><em>Snippet from Sandbox execution of the threat actor&#8217;s malicious service showing the read activity performed by the service of the two registry keys</em></figcaption></figure>



<p>The service employs the AES algorithm to decrypt the encrypted shellcode retrieved from the &#8220;inputlog&#8221; registry key. It sets the decryption key from the &#8220;AKey&#8221; value and uses a null byte array as the Initialization Vector (IV). The shellcode consists of a Position-Independent shellcode functioning as a reflective loader, alongside a corrupted Portable Executable (PE) file, positioned at offset 0x4000. Certain headers within the PE file have been deliberately stripped to enhance resistance to analysis and detection. Specifically, the &#8220;MZ&#8221; and &#8220;PE&#8221; headers have been nullified, and the DOS Stub has been removed.</p>



<figure class="wp-block-image size-large"><img decoding="async" width="1024" height="234" src="data:image/svg+xml,%3Csvg%20xmlns='http://www.w3.org/2000/svg'%20viewBox='0%200%201024%20234'%3E%3C/svg%3E" alt="Jump\trampoline at the Start of the shellcode." class="wp-image-3608" data-lazy-srcset="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-16-1024x234.jpg 1024w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-16-300x69.jpg 300w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-16-768x176.jpg 768w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-16.jpg 1500w" data-lazy-sizes="(max-width: 1024px) 100vw, 1024px" data-lazy-src="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-16-1024x234.jpg" /><noscript><img decoding="async" width="1024" height="234" src="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-16-1024x234.jpg" alt="Jump\trampoline at the Start of the shellcode." class="wp-image-3608" srcset="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-16-1024x234.jpg 1024w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-16-300x69.jpg 300w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-16-768x176.jpg 768w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-16.jpg 1500w" sizes="(max-width: 1024px) 100vw, 1024px" /></noscript><figcaption class="wp-element-caption"><em>Jump\trampoline at the Start of the shellcode</em></figcaption></figure>



<figure class="wp-block-image size-large"><img decoding="async" width="1024" height="602" src="data:image/svg+xml,%3Csvg%20xmlns='http://www.w3.org/2000/svg'%20viewBox='0%200%201024%20602'%3E%3C/svg%3E" alt="Corrupted PE file located at offset 0x4000" class="wp-image-3609" data-lazy-srcset="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-17-1024x602.jpg 1024w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-17-300x176.jpg 300w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-17-768x452.jpg 768w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-17.jpg 1500w" data-lazy-sizes="(max-width: 1024px) 100vw, 1024px" data-lazy-src="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-17-1024x602.jpg" /><noscript><img decoding="async" width="1024" height="602" src="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-17-1024x602.jpg" alt="Corrupted PE file located at offset 0x4000" class="wp-image-3609" srcset="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-17-1024x602.jpg 1024w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-17-300x176.jpg 300w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-17-768x452.jpg 768w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-17.jpg 1500w" sizes="(max-width: 1024px) 100vw, 1024px" /></noscript><figcaption class="wp-element-caption"><em>Corrupted PE file located at offset 0x4000</em></figcaption></figure>



<p>The shellcode loads the core-implant DLL using a reflective loader which performs the following steps:</p>



<ul class="wp-block-list">
<li>Allocates memory for the core-implant DLL.</li>



<li>Parses the custom PE headers of the core-implant.</li>



<li>Moves each section to its proper location in the allocated memory.</li>
</ul>



<figure class="wp-block-image size-large is-resized"><img decoding="async" width="1024" height="368" src="data:image/svg+xml,%3Csvg%20xmlns='http://www.w3.org/2000/svg'%20viewBox='0%200%201024%20368'%3E%3C/svg%3E" alt="Code snippet parsing DLL sections and relocating them to the appropriate memory locations" class="wp-image-3610" style="width:840px;height:auto" data-lazy-srcset="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-18-1024x368.jpg 1024w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-18-300x108.jpg 300w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-18-768x276.jpg 768w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-18.jpg 1500w" data-lazy-sizes="(max-width: 1024px) 100vw, 1024px" data-lazy-src="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-18-1024x368.jpg" /><noscript><img decoding="async" width="1024" height="368" src="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-18-1024x368.jpg" alt="Code snippet parsing DLL sections and relocating them to the appropriate memory locations" class="wp-image-3610" style="width:840px;height:auto" srcset="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-18-1024x368.jpg 1024w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-18-300x108.jpg 300w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-18-768x276.jpg 768w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-18.jpg 1500w" sizes="(max-width: 1024px) 100vw, 1024px" /></noscript><figcaption class="wp-element-caption"><em>Code snippet parsing DLL sections and relocating them to the appropriate memory locations</em></figcaption></figure>



<ul class="wp-block-list">
<li>Performs relocation of the code and data sections to match the new base address.</li>



<li>Resolves the import table.</li>



<li>Sets proper memory protections.</li>
</ul>



<figure class="wp-block-image size-large"><img decoding="async" width="1024" height="388" src="data:image/svg+xml,%3Csvg%20xmlns='http://www.w3.org/2000/svg'%20viewBox='0%200%201024%20388'%3E%3C/svg%3E" alt="Code snippet applying correct protections for each section" class="wp-image-3611" data-lazy-srcset="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-19-1024x388.jpg 1024w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-19-300x114.jpg 300w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-19-768x291.jpg 768w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-19.jpg 1500w" data-lazy-sizes="(max-width: 1024px) 100vw, 1024px" data-lazy-src="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-19-1024x388.jpg" /><noscript><img decoding="async" width="1024" height="388" src="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-19-1024x388.jpg" alt="Code snippet applying correct protections for each section" class="wp-image-3611" srcset="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-19-1024x388.jpg 1024w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-19-300x114.jpg 300w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-19-768x291.jpg 768w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-19.jpg 1500w" sizes="(max-width: 1024px) 100vw, 1024px" /></noscript><figcaption class="wp-element-caption"><em>Code snippet applying correct protections for each section</em></figcaption></figure>



<ul class="wp-block-list">
<li>Executes the now-ready Core-Implant by calling its Entry Point.</li>
</ul>



<h2 class="wp-block-heading" id="9">Core-Implant</h2>



<p>The Core-Implant handles two main tasks &#8211; managing Command and Control (C2) communication and installing the Demodex kernel rootkit. To load Demodex, the threat actor had to bypass the Driver Signature Enforcement (DSE) security feature, which blocks unsigned drivers.</p>



<p>To do that, the threat actor leveraged “Cheat Engine”, an open-source tool used for video game cheating, and utilized its signed driver, dbk64.sys, to manipulate memory and execute code in kernel space. the threat actor used this driver to map and execute a shellcode in kernel space which patches the IOCTL Dispatcher of the dbk64.sys driver. This modification adds functionality to the driver that enables it to load the Demodex driver.</p>



<p>An analysis of the Core-Implant’s metadata shows that the threat actor modified the compilation and export-table timestamp of the Core-Implant to 12 Feb 2016. However, the timestamp of the debug section is set to 02 July 2021, which might indicate that this is the actual time this implant was created.</p>



<figure class="wp-block-image size-large"><img decoding="async" width="1024" height="328" src="data:image/svg+xml,%3Csvg%20xmlns='http://www.w3.org/2000/svg'%20viewBox='0%200%201024%20328'%3E%3C/svg%3E" alt="Core-Implant’s timestamps retrieved from PE Studio" class="wp-image-3612" data-lazy-srcset="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-20-1024x328.jpg 1024w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-20-300x96.jpg 300w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-20-768x246.jpg 768w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-20.jpg 1500w" data-lazy-sizes="(max-width: 1024px) 100vw, 1024px" data-lazy-src="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-20-1024x328.jpg" /><noscript><img decoding="async" width="1024" height="328" src="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-20-1024x328.jpg" alt="Core-Implant’s timestamps retrieved from PE Studio" class="wp-image-3612" srcset="https://www.sygnia.co/wp-content/uploads/2024/07/Picture-20-1024x328.jpg 1024w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-20-300x96.jpg 300w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-20-768x246.jpg 768w, https://www.sygnia.co/wp-content/uploads/2024/07/Picture-20.jpg 1500w" sizes="(max-width: 1024px) 100vw, 1024px" /></noscript><figcaption class="wp-element-caption"><em>Core-Implant’s timestamps retrieved from PE Studio</em></figcaption></figure>
