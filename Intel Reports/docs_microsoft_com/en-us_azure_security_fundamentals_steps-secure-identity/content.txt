											<h2 id="ms--in-this-article" class="title is-6 margin-block-xs">In this article</h2>
										</nav><!-- <content> --><p>If you're reading this document, you're aware of the significance of security. You likely already carry the responsibility for securing your organization. If you need to convince others of the importance of security, send them to read the latest <a href="https://www.microsoft.com/security/business/security-intelligence-report" data-linktype="external">Microsoft Digital Defense Report</a>.</p>
<p>This document helps you get a more secure posture using the capabilities of Microsoft Entra ID by using a five-step checklist to improve your organization's protection against cyber-attacks.</p>
<p>This checklist helps you quickly deploy critical recommended actions to protect your organization immediately by explaining how to:</p>
<ul>
<li>Strengthen your credentials</li>
<li>Reduce your attack surface area</li>

<h2 id="before-you-begin-protect-privileged-accounts-with-mfa">Before you begin: Protect privileged accounts with MFA</h2>
<p>Before you begin this checklist, make sure you don't get compromised while you're reading this checklist. In Microsoft Entra we observe 50 million password attacks daily, yet only a fraction of users and administrators are using strong authentications such as multifactor authentication (MFA). These statistics are based on data as of August 2021. In Microsoft Entra ID, users who have privileged roles, such as administrators, are the root of trust to build and manage the rest of the environment. Implement the following practices to minimize the effects of a compromise.</p>
<p>Attackers who get control of privileged accounts can do tremendous damage, so it's critical to <a href="../../active-directory/authentication/how-to-authentication-find-coverage-gaps" data-linktype="relative-path">protect these accounts before proceeding</a>. Enable and require <a href="../../active-directory/authentication/concept-mfa-howitworks" data-linktype="relative-path">Microsoft Entra multifactor authentication (MFA)</a> for all administrators in your organization using <a href="../../active-directory/fundamentals/concept-fundamentals-security-defaults" data-linktype="relative-path">Microsoft Entra Security Defaults</a> or <a href="../../active-directory/conditional-access/howto-conditional-access-policy-admin-mfa" data-linktype="relative-path">Conditional Access</a>. It's critical.</p>
<p>All set? Let's get started on the checklist.</p>
<h2 id="step-1-strengthen-your-credentials">Step 1: Strengthen your credentials</h2>
<p>Although other types of attacks are emerging, including consent phishing and attacks on nonhuman identities, password-based attacks on user identities are still the most prevalent vector of identity compromise. Well-established spear phishing and password spray campaigns by adversaries continue to be successful against organizations that don't implement multifactor authentication (MFA) or other protections against this common tactic.</p>
<p>As an organization you need to make sure that your identities are validated and secured with MFA everywhere. In 2020, the <a href="https://www.ic3.gov/Media/PDF/AnnualReport/2020_IC3Report.pdf" data-linktype="external">Federal Bureau of Investigation (FBI) Internet Crime Complaint Center (IC3) Report</a> identified phishing as the top crime type for victim complaints. The number of reports doubled compared to the previous year. Phishing poses a significant threat to both businesses and individuals, and credential phishing was used in many of the most damaging attacks last year. Microsoft Entra multifactor authentication (MFA) helps safeguard access to data and applications, providing another layer of security by using a second form of authentication. Organizations can enable multifactor authentication with Conditional Access to make the solution fit their specific needs. Take a look at this deployment guide to see how you how to <a href="../../active-directory/authentication/concept-mfa-howitworks" data-linktype="relative-path">plan, implement, and roll-out Microsoft Entra multifactor authentication</a>.</p>
<h3 id="make-sure-your-organization-uses-strong-authentication">Make sure your organization uses strong authentication</h3>
<p>To easily enable the basic level of identity security, you can use the one-select enablement with <a href="../../active-directory/fundamentals/concept-fundamentals-security-defaults" data-linktype="relative-path">Microsoft Entra security defaults</a>. Security defaults enforce Microsoft Entra multifactor authentication for all users in a tenant and blocks sign-ins from legacy protocols tenant-wide.</p>
<p>If your organization has Microsoft Entra ID P1 or P2 licenses, then you can also use the <a href="../../active-directory/conditional-access/howto-conditional-access-insights-reporting" data-linktype="relative-path">Conditional Access insights and reporting workbook</a> to help you discover gaps in your configuration and coverage. From these recommendations, you can easily close this gap by creating a policy using the new Conditional Access templates experience. <a href="../../active-directory/conditional-access/concept-conditional-access-policy-common" data-linktype="relative-path">Conditional Access templates</a> are designed to provide an easy method to deploy new policies that align with Microsoft recommended <a href="identity-management-best-practices" data-linktype="relative-path">best practices</a>, making it easy to deploy common policies to protect your identities and devices.</p>
<h3 id="start-banning-commonly-attacked-passwords-and-turn-off-traditional-complexity-and-expiration-rules">Start banning commonly attacked passwords and turn off traditional complexity, and expiration rules.</h3>
<p>Many organizations use traditional complexity and password expiration rules. <a href="https://www.microsoft.com/research/publication/password-guidance/" data-linktype="external">Microsoft's research</a> shows, and <a href="https://pages.nist.gov/800-63-3/sp800-63b.html" data-linktype="external">National Institute of Standards and Technology (NIST) Special Publication 800-63B Digital Identity Guidelines</a> state, that these policies cause users to choose passwords that are easier to guess. We recommend you use <a href="../../active-directory/authentication/concept-password-ban-bad" data-linktype="relative-path">Microsoft Entra password protection</a> a dynamic banned password feature using current attacker behavior to prevent users from setting passwords that can easily be guessed. This capability is always on when users are created in the cloud, but is now also available for hybrid organizations when they deploy <a href="../../active-directory/authentication/concept-password-ban-bad-on-premises" data-linktype="relative-path">Microsoft Entra password protection for Windows Server Active Directory</a>. In addition, we recommend you remove expiration policies. Password change offers no containment benefits as cyber criminals almost always use credentials as soon as they compromise them. Refer to the following article to <a href="/en-us/microsoft-365/admin/manage/set-password-expiration-policy" data-linktype="absolute-path">Set the password expiration policy for your organization</a>.</p>
<h3 id="protect-against-leaked-credentials-and-add-resilience-against-outages">Protect against leaked credentials and add resilience against outages</h3>
<p>The simplest and recommended method for enabling cloud authentication for on-premises directory objects in Microsoft Entra ID is to enable <a href="../../active-directory/hybrid/how-to-connect-password-hash-synchronization" data-linktype="relative-path">password hash synchronization (PHS)</a>. If your organization uses a hybrid identity solution with pass-through authentication or federation, then you should enable password hash sync for the following two reasons:</p>
<ul>
<li>The <a href="../../active-directory/identity-protection/overview-identity-protection" data-linktype="relative-path">Users with leaked credentials report</a> in Microsoft Entra ID warns of publicly exposed username and password pairs. An incredible volume of passwords is leaked via phishing, malware, and password reuse on third-party sites that are later breached. Microsoft finds many of these leaked credentials and tells you, in this report, if they match credentials in your organization â but only if you enable <a href="../../active-directory/hybrid/how-to-connect-password-hash-synchronization" data-linktype="relative-path">password hash sync</a> or have cloud-only identities.</li>
<li>If an on-premises outage happens, like a ransomware attack, you can <a href="../../active-directory/hybrid/choose-ad-authn" data-linktype="relative-path">switch over to using cloud authentication using password hash sync</a>. This backup authentication method allows you to continue accessing apps configured for authentication with Microsoft Entra ID, including Microsoft 365. In this case, IT staff doesn't need to resort to shadow IT or personal email accounts to share data until the on-premises outage is resolved.</li>
</ul>
<p>Passwords are never stored in clear text or encrypted with a reversible algorithm in Microsoft Entra ID. For more information on the actual process of password hash synchronization, see <a href="../../active-directory/hybrid/how-to-connect-password-hash-synchronization#detailed-description-of-how-password-hash-synchronization-works" data-linktype="relative-path">Detailed description of how password hash synchronization works</a>.</p>
<h3 id="implement-ad-fs-extranet-smart-lockout">Implement AD FS extranet smart lockout</h3>
<p>Smart lockout helps lock out bad actors that try to guess your users' passwords or use brute-force methods to get in. Smart lockout can recognize sign-ins that come from valid users and treat them differently than ones of attackers and other unknown sources. Attackers get locked out, while your users continue to access their accounts and be productive. Organizations, which configure applications to authenticate directly to Microsoft Entra ID benefit from Microsoft Entra smart lockout. Federated deployments that use AD FS 2016 and AD FS 2019 can enable similar benefits using <a href="/en-us/windows-server/identity/ad-fs/operations/configure-ad-fs-extranet-smart-lockout-protection" data-linktype="absolute-path">AD FS Extranet Lockout and Extranet Smart Lockout</a>.</p>
<h2 id="step-2-reduce-your-attack-surface-area">Step 2: Reduce your attack surface area</h2>
<p>Given the pervasiveness of password compromise, minimizing the attack surface in your organization is critical. Disable the use of older, less secure protocols, limit access entry points, moving to cloud authentication, exercise more significant control of administrative access to resources, and embrace Zero Trust security principles.</p>
<h3 id="use-cloud-authentication">Use Cloud Authentication</h3>
<p>Credentials are a primary attack vector. The practices in this blog can reduce the attack surface by using cloud authentication, deploy MFA, and use passwordless authentication methods. You can deploy passwordless methods such as Windows Hello for Business, Phone Sign-in with the Microsoft Authenticator App or FIDO.</p>
<h3 id="block-legacy-authentication">Block legacy authentication</h3>
<p>Apps using their own legacy methods to authenticate with Microsoft Entra ID and access company data, pose another risk for organizations. Examples of apps using legacy authentication are POP3, IMAP4, or SMTP clients. Legacy authentication apps authenticate on behalf of the user and prevent Microsoft Entra ID from doing advanced security evaluations. The alternative, modern authentication, reduces your security risk, because it supports multifactor authentication and Conditional Access.</p>
<p>We recommend the following actions:</p>
<ol>
<li>Discover legacy authentication in your organization with Microsoft Entra sign-in logs and Log Analytics workbooks.</li>
<li>Setup SharePoint Online and Exchange Online to use modern  authentication.</li>
<li>If you have Microsoft Entra ID P1 or P2 licenses, use Conditional Access policies to block legacy authentication. For Microsoft Entra ID Free tier, use Microsoft Entra Security Defaults.</li>
<li>Block legacy authentication if you use AD FS.</li>
<li>Block Legacy Authentication with Exchange Server 2019.</li>
<li>Disable legacy authentication in Exchange Online.</li>
</ol>
<p>For more information, see the article <a href="../../active-directory/conditional-access/block-legacy-authentication" data-linktype="relative-path">Blocking legacy authentication protocols in Microsoft Entra ID</a>.</p>
<h3 id="block-invalid-authentication-entry-points">Block invalid authentication entry points</h3>
<p>Using the <em>verify explicitly principle</em>, you should reduce the impact of compromised user credentials when they happen. For each app in your environment, consider the valid use cases: which groups, which networks, which devices and other elements are authorized â then block the rest. With Microsoft Entra Conditional Access, you can control how authorized users access their apps and resources based on specific conditions you define.</p>
<p>For more information on how to use Conditional Access for your Cloud Apps and user actions, see <a href="../../active-directory/conditional-access/concept-conditional-access-cloud-apps" data-linktype="relative-path">Conditional Access Cloud apps, actions, and authentication context</a>.</p>
<h3 id="review-and-govern-admin-roles">Review and govern admin roles</h3>
<p>Another Zero Trust pillar is the need to minimize the likelihood a compromised account can operate with a privileged role. This control can be accomplished by assigning the least amount of privilege to an identity. If youâre new to Microsoft Entra roles, this article helps you understand Microsoft Entra roles.</p>
<p>Privileged roles in Microsoft Entra ID should be cloud only accounts in order to isolate them from any on-premises environments and donât use on-premises password vaults to store the credentials.</p>
<h3 id="implement-privilege-access-management">Implement Privilege Access Management</h3>
<p>Privileged Identity Management (PIM) provides a time-based and approval-based role activation to mitigate the risks of excessive, unnecessary, or misused access permissions to important resources. These resources include resources in Microsoft Entra ID, Azure, and other Microsoft Online Services such as Microsoft 365 or Microsoft Intune.</p>
<p>Microsoft Entra Privileged Identity Management (PIM) helps you minimize account privileges by helping you:</p>
<ul>
<li>Identify and manage users assigned to administrative roles.</li>
<li>Understand unused or excessive privilege roles you should remove.</li>
<li>Establish rules to make sure privileged roles are protected by multifactor authentication.</li>
<li>Establish rules to make sure privileged roles are granted only long enough to accomplish the privileged task.</li>
</ul>
<p>Enable Microsoft Entra PIM, then view the users who are assigned administrative roles and remove unnecessary accounts in those roles. For remaining privileged users, move them from permanent to eligible. Finally, establish appropriate policies to make sure when they need to gain access to those privileged roles, they can do so securely, with the necessary change control.</p>
<p>Microsoft Entra built-in and custom roles operate on concepts similar to roles found in the role-based access control system for Azure resources (Azure roles). The difference between these two role-based access control systems is:</p>
<ul>
<li>Microsoft Entra roles control access to Microsoft Entra resources such as users, groups, and applications using the Microsoft Graph API</li>
<li>Azure roles control access to Azure resources such as virtual machines or storage using Azure Resource Management</li>
</ul>
<p>Both systems contain similarly used role definitions and role assignments. However, Microsoft Entra role permissions can't be used in Azure custom roles and vice versa. As part of deploying your privileged account process, follow the best practice to create at least two emergency accounts to make sure you still have access to Microsoft Entra ID if you lock yourself out.</p>
<p>For more information, see the article <a href="../../active-directory/privileged-identity-management/pim-deployment-plan" data-linktype="relative-path">Plan a Privileged Identity Management deployment</a> and securing privileged access.</p>
<h3 id="restrict-user-consent-operations">Restrict user consent operations</h3>
<p>Itâs important to understand the various Microsoft Entra application consent experiences, the types of permissions and consent, and their implications on your organizationâs security posture. While allowing users to consent by themselves does allow users to easily acquire useful applications that integrate with Microsoft 365, Azure, and other services, it can represent a risk if not used and monitored carefully.</p>
<p>Microsoft recommends restricting user consent to allow end-user consent only for apps from verified publishers and only for permissions you select. If end-user consent is restricted, previous consent grants will still be honored but all future consent operations that an administrator must perform. For restricted cases, users can request admin consent through an integrated admin consent request workflow or through your own support processes. Before restricting end-user consent, use our recommendations to plan this change in your organization. For applications you wish to allow all users to access, consider granting consent on behalf of all users, making sure users who didn't yet individually consent can access the app. If you donât want these applications to be available to all users in all scenarios, use application assignment and Conditional Access to restrict user access to specific apps.</p>
<p>Make sure users can request admin approval for new applications to reduce user friction, minimize support volume, and prevent users from signing up for applications using non-Microsoft Entra credentials. Once you regulate your consent operations, administrators should audit app and consent permissions regularly.</p>
<p>For more information, see the article <a href="../../active-directory/develop/consent-framework" data-linktype="relative-path">Microsoft Entra consent framework</a>.</p>

<h2 id="step-5-enable-end-user-self-service">Step 5: Enable end-user self-service</h2>

<h3 id="implement-self-service-password-reset">Implement self-service password reset</h3>
<p>Microsoft Entra ID's <a href="../../active-directory/authentication/tutorial-enable-sspr" data-linktype="relative-path">self-service password reset (SSPR)</a> offers a simple means for IT administrators to allow users to reset or unlock their passwords or accounts without helpdesk or administrator intervention. <h3 id="implement-self-service-group-and-application-access">Implement self-service group and application access</h3>
<p>Microsoft Entra ID can allow nonadministrators to manage access to resources, using security groups, Microsoft 365 groups, application roles, and access package catalogs. <a href="../../active-directory/enterprise-users/groups-self-service-management" data-linktype="relative-path">Self-service group management</a> enables group owners to manage their own groups, without needing to be assigned an administrative role. Users can also create and manage Microsoft 365 groups without relying on administrators to handle their requests, and unused groups expire automatically. <a href="../../active-directory/governance/entitlement-management-overview" data-linktype="relative-path">Microsoft Entra entitlement management</a> further enables delegation and visibility, with comprehensive access request workflows and automatic expiration. <p><a name="implement-azure-ad-access-reviews"></a></p>
<h3 id="implement-microsoft-entra-access-reviews">Implement Microsoft Entra access reviews</h3>

Regular oversight by the users themselves, resource owners, and other reviewers ensure that users don't retain access for extended periods of time when they no longer need it.</p>
<h3 id="implement-automatic-user-provisioning">Implement automatic user provisioning</h3>
<p>Provisioning and deprovisioning are the processes that ensure consistency of digital identities across multiple systems. These processes are typically applied as part of <a href="../../active-directory/governance/what-is-identity-lifecycle-management" data-linktype="relative-path">identity lifecycle management</a>.</p>
<p>Provisioning is the processes of creating an identity in a target system based on certain conditions. Deprovisioning is the process of removing the identity from the target system, when conditions are no longer met. Synchronization is the process of keeping the provisioned object, up to date, so that the source object and target object are similar.</p>
<p>Microsoft Entra ID currently provides three areas of automated provisioning. They are:</p>
<ul>
<li>Provisioning from an external nondirectory authoritative system of record to Microsoft Entra ID, via <a href="../../active-directory/governance/what-is-provisioning#hr-driven-provisioning" data-linktype="relative-path">HR-driven provisioning</a></li>
<li>Provisioning from Microsoft Entra ID to applications, via <a href="../../active-directory/governance/what-is-provisioning#app-provisioning" data-linktype="relative-path">App provisioning</a></li>
<li>Provisioning between Microsoft Entra ID and Active Directory Domain Services, via <a href="../../active-directory/governance/what-is-provisioning#inter-directory-provisioning" data-linktype="relative-path">inter-directory provisioning</a></li>
</ul>
<p>Find out more here: What is provisioning with Microsoft Entra ID?</p>
<h2 id="summary">Summary</h2>
<p>There are many aspects to a secure Identity infrastructure, but this five-step checklist helps you to quickly accomplish a safer and secure identity infrastructure:</p>
<ul>
<li>Strengthen your credentials</li>
<li>Reduce your attack surface area</li>
<li>Automate threat response</li>
<li>Utilize cloud intelligence</li>
<li>Enable end-user self-service</li>
</ul>
<p>We appreciate how seriously you take security and hope this document is a useful roadmap to a more secure posture for your organization.</p>
