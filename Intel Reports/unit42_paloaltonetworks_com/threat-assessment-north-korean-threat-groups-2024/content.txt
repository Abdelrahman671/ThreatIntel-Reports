Since then, multiple variants of the malware have been observed in the wild. Most RustBucket infections are composed of three stages.\r\n\r\nThe first stage usually is an AppleScript file contained inside an application or inside a ZIP archive masquerading as a legitimate file. This AppleScript file is responsible for retrieving the second stage downloader.\r\n\r\nThe second stage downloader masquerades as a PDF viewer application. Some variants of this second stage downloader are written in Swift, while others are written in Objective-C.\r\n\r\nThe third stage is the final payload retrieved by the second stage downloader. Figure 2 shows an alert from Cortex XDR that blocks a RustBucket sample from downloading the next stage of malware.\r\n\r\n[caption id=\"attachment_136661\" align=\"alignnone\" width=\"1920\"] Figure 2. Cortex XDR alert on preventing RustBucket download activity.[\/caption]\r\n\r\nThe third stage payloads are Mach-O binaries written in Rust, hence the name RustBucket. Later variants of stage three employ persistence via a LaunchAgent, a feature that did not exist in older variants. Stage three has two main commands:\r\n\r\n \tDownload and execute a file\r\n \tSelf-terminate the malware\r\n\r\nKANDYKORN\r\nMalware type: Backdoor\r\n\r\nGroup affiliation: Alluring Pisces\r\n\r\nFirst seen: 2023\r\n\r\nOS type: macOS\r\n\r\nDescription:\r\n\r\nFirst discovered in 2023, KANDYKORN is the payload of a five-stage infection chain targeting macOS systems. Known infections of KANDYKORN start with social engineering, tricking the victim into downloading a malicious ZIP archive containing a malicious Python script. If the victim executes the Python file, it downloads stage two of the infection, which is a second Python script that is saved into a folder named _log.\r\n\r\nThe second stage of the infection involves two additional Python scripts. The first Python script saved to the _log directory downloads another script saved to the \/Users\/Shared\/ directory, which in turn downloads a stage three file, saving it as \/Users\/shared\/.sld.\r\n\r\nStage three of the infection is a downloader and loader dubbed SUGARLOADER. For persistence, SUGARLOADER saves itself as \/Users\/shared\/.log.\r\n\r\nUpon execution, SUGARLOADER checks for the existence of a configuration file at \/Library\/Caches\/com.apple.safari.ck. If that configuration file is missing, SUGARLOADER downloads it using a default IP address provided in the command line.\r\n\r\nThe configuration file at \/Library\/Caches\/com.apple.safari.ck contains the location to download the next stage from.                 <h2><a id="post-136656-_4lt92rr5muov"></a><strong>Executive Summary</strong></h2>
<p><a href="https://attack.mitre.org/groups/G0032/" target="_blank" rel="noopener">Lazarus</a> has been used in public reporting as an umbrella term for threat actors from the Democratic People's Republic of Korea (DPRK), commonly referred to as North Korea. However, many of these threat actors can be <a href="https://cloud.google.com/blog/topics/threat-intelligence/mapping-dprk-groups-to-government/" target="_blank" rel="noopener">classified into different groups under the Reconnaissance General Bureau (RGB)</a> of the Korean People's Army.</p>
<p>Over the years, the RGB has revealed at least six threat groups that we designate as:</p>
<ul>
<li>Alluring Pisces (<a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2018/03/07180244/Lazarus_Under_The_Hood_PDF_final.pdf" target="_blank" rel="noopener">Bluenoroff [PDF]</a>)</li>
<li>Gleaming Pisces (<a href="https://www.microsoft.com/en-us/security/blog/2022/12/06/dev-0139-launches-targeted-attacks-against-the-cryptocurrency-industry/" target="_blank" rel="noopener">Citrine Sleet</a>)</li>
<li>Jumpy Pisces (<a href="https://www.justice.gov/opa/pr/north-korean-government-hacker-charged-involvement-ransomware-attacks-targeting-us-hospitals" target="_blank" rel="noopener">Andariel</a>)</li>
<li>Selective Pisces (<a href="https://www.hhs.gov/sites/default/files/dprk-cyber-espionage.pdf" target="_blank" rel="noopener">TEMP.Hermit [PDF]</a>)</li>

<h2><a id="post-136656-_94xgafg6sykn"></a><strong>North Korean Threat Groups Under the RGB</strong></h2>
<p>North Korean threat group activity is often <a href="https://www.justice.gov/opa/pr/north-korean-regime-backed-programmer-charged-conspiracy-conduct-multiple-cyber-attacks-and" target="_blank" rel="noopener">referred to as Lazarus or the Lazarus Group</a> in public reports. However, most of this activity is <a href="https://cloud.google.com/blog/topics/threat-intelligence/mapping-dprk-groups-to-government/" target="_blank" rel="noopener">reportedly conducted by groups under the RGB</a>, an organization that falls under the General Staff Bureau of the DPRK Korean People's Army.</p>
<p>These groups support the North Korean government through a combination of espionage, financial gain and geopolitical disruption. Some of the significant operations executed by these groups across the years include:</p>
<ul>
<li>The <a href="https://www.secureops.com/wp-content/uploads/2021/06/Sony-Breach-Analysis-v4.pdf" target="_blank" rel="noopener">Sony Pictures Hack in 2014 [PDF]</a></li>
<li>The <a href="https://www.csoonline.com/article/563017/wannacry-explained-a-perfect-ransomware-storm.html" target="_blank" rel="noopener">WannaCry ransomware attacks in 2017</a></li>
<li><a href="https://www.clearskysec.com/wp-content/uploads/2020/08/Dream-Job-Campaign.pdf" target="_blank" rel="noopener">Operation Dream Job [PDF]</a></li>
<li>Numerous <a href="https://www.elliptic.co/blog/how-the-lazarus-group-is-stepping-up-crypto-hacks-and-changing-its-tactics" target="_blank" rel="noopener">cryptocurrency exchange attacks</a></li>
</ul>
<p>These groups have been reportedly <a href="https://www.usna.edu/CyberCenter/_files/documents/Operation-Blockbuster-Report.pdf" target="_blank" rel="noopener">active as early as 2007 [PDF]</a>. Activity under the RGB can be categorized into at least six threat groups:</p>
<ul>
<li>Alluring Pisces (aka <a href="https://services.google.com/fh/files/misc/apt38-un-usual-suspects.pdf" target="_blank" rel="noopener">APT38 [PDF]</a>, <a href="https://apt.securelist.com/apt/bluenoroff" target="_blank" rel="noopener">Bluenoroff</a>, <a href="https://x.com/MsftSecIntel/status/1722316021841764414" target="_blank" rel="noopener">Sapphire Sleet</a>): This group has targeted financial institutions, cryptocurrency businesses and ATMs. It has also conducted significant cyber heists.</li>
<li>Gleaming Pisces (aka <a href="https://www.microsoft.com/en-us/security/blog/2022/12/06/dev-0139-launches-targeted-attacks-against-the-cryptocurrency-industry/" target="_blank" rel="noopener">Citrine Sleet</a>): This group performed attacks targeting the cryptocurrency industry and is known for its association with the <a href="https://www.cisa.gov/news-events/analysis-reports/ar21-048e" target="_blank" rel="noopener">AppleJeus</a> campaign.</li>
<li>Jumpy Pisces (aka <a href="https://www.justice.gov/opa/pr/north-korean-government-hacker-charged-involvement-ransomware-attacks-targeting-us-hospitals" target="_blank" rel="noopener">Andariel</a>, <a href="https://www.justice.gov/opa/pr/three-north-korean-military-hackers-indicted-wide-ranging-scheme-commit-cyberattacks-and" target="_blank" rel="noopener">Hidden Cobra</a>, <a href="https://www.microsoft.com/en-us/security/blog/2024/07/25/onyx-sleet-uses-array-of-malware-to-gather-intelligence-for-north-korea/" target="_blank" rel="noopener">Onyx Sleet</a>): This group has primarily conducted cyberespionage, but it has also conducted ransomware activity.</li>
<li>Selective Pisces (aka <a href="https://www.microsoft.com/en-us/security/security-insider/diamond-sleet" target="_blank" rel="noopener">Diamond Sleet</a>, <a href="https://www.mandiant.com/sites/default/files/2021-09/rpt-apt38-2018-web_v5-1.pdf" target="_blank" rel="noopener">TEMP.Hermit [PDF]</a>, <a href="https://www.microsoft.com/en-us/security/blog/2022/09/29/zinc-weaponizing-open-source-software/" target="_blank" rel="noopener">ZINC</a>): This group has targeted media, defense and IT organizations. It focuses on espionage, financial gain and network destruction.</li>
<li>Slow Pisces (aka <a href="https://github.blog/security/vulnerability-research/security-alert-social-engineering-campaign-targets-technology-industry-employees/" target="_blank" rel="noopener">Jade Sleet</a>, <a href="https://cloud.google.com/blog/topics/threat-intelligence/north-korea-supply-chain/" target="_blank" rel="noopener">UNC4899</a>): This group has targeted blockchain and cryptocurrency companies. It was also involved in a supply chain attack targeting a U.S.-based software platform and is known for distributing a series of malicious applications called <a href="https://www.cisa.gov/news-events/cybersecurity-advisories/aa22-108a" target="_blank" rel="noopener">TraderTraitor</a>.</li>
<li>Sparkling Pisces (aka <a href="https://services.google.com/fh/files/misc/apt43-report-en.pdf" target="_blank" rel="noopener">APT43 [PDF]</a>, <a href="https://www.microsoft.com/en-us/security/security-insider/intelligence-reports/digital-threats-from-east-asia-increase-in-breadth-and-effectiveness" target="_blank" rel="noopener">Emerald Sleet</a>, <a href="https://thehackernews.com/2024/03/n-korea-linked-kimsuky-shifts-to.html" target="_blank" rel="noopener">Kimsuky</a>, <a href="https://blogs.microsoft.com/on-the-issues/2019/12/30/microsoft-court-action-against-nation-state-cybercrime/" target="_blank" rel="noopener">THALLIUM</a>): This group conducts intelligence collection and has used cybercrime to fund espionage.</li>
</ul>
<p>These groups have evolved over the years, and we often find overlaps in the tactics, techniques and tools. Figure 1 shows a simplified organizational chart for these groups under the RGB.</p>

<h2><a id="post-136656-_2fjhzsc3y2yx"></a><strong>Recent North Korean Malware Arsenal Analysis</strong></h2>
<h3><a id="post-136656-_abmk1tzy4v2"></a>MacOS Malware</h3>
<h4><a id="post-136656-_6agte1yqoxng"></a><strong>RustBucket</strong></h4>
<p><strong>Malware type:</strong> Backdoor</p>
<p><strong>Group affiliation:</strong> Alluring Pisces</p>
<p><strong>First seen:</strong> 2023</p>
<p><strong>OS type:</strong> macOS</p>
<p><strong>Description:</strong></p>
<p>RustBucket is macOS malware <a href="https://thehackernews.com/2023/07/beware-new-rustbucket-malware-variant.html" target="_blank" rel="noopener">first reported in 2023</a>. Since then, multiple variants of the malware have been observed in the wild. Most RustBucket infections are composed of three stages.</p>
<p>The first stage usually is an <a href="https://developer.apple.com/library/archive/documentation/AppleScript/Conceptual/AppleScriptLangGuide/introduction/ASLR_intro.html" target="_blank" rel="noopener">AppleScript</a> file contained inside an application or inside a ZIP archive masquerading as a legitimate file. This AppleScript file is responsible for retrieving the second stage downloader.</p>
<p>The second stage downloader masquerades as a PDF viewer application. Some variants of this second stage downloader are written in <a href="https://developer.apple.com/swift/" target="_blank" rel="noopener">Swift</a>, while others are written in <a href="https://developer.apple.com/library/archive/documentation/Cocoa/Conceptual/ProgrammingWithObjectiveC/Introduction/Introduction.html" target="_blank" rel="noopener">Objective-C</a>.</p>
<p>The third stage is the final payload retrieved by the second stage downloader. Figure 2 shows an alert from Cortex XDR that blocks a RustBucket sample from downloading the next stage of malware.</p>
<figure id="attachment_136661" aria-describedby="caption-attachment-136661" style="width: 1920px" class="wp-caption alignnone"><img decoding="async" class="wp-image-136661 size-large" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2024/09/word-image-568322-136656-2-1920x469.png" alt="A screenshot showing a user interface with two panels in Cortex XDR. The left panel displays technical information. The right panel presents the alert information. There is an overlaid screenshot of a section of code. " width="1920" height="469" srcset="https://unit42.paloaltonetworks.com/wp-content/uploads/2024/09/word-image-568322-136656-2-1920x469.png 1920w, https://unit42.paloaltonetworks.com/wp-content/uploads/2024/09/word-image-568322-136656-2-786x192.png 786w, https://unit42.paloaltonetworks.com/wp-content/uploads/2024/09/word-image-568322-136656-2-768x188.png 768w, https://unit42.paloaltonetworks.com/wp-content/uploads/2024/09/word-image-568322-136656-2-1536x375.png 1536w, https://unit42.paloaltonetworks.com/wp-content/uploads/2024/09/word-image-568322-136656-2.png 2048w" sizes="(max-width: 1920px) 100vw, 1920px" /><figcaption id="caption-attachment-136661" class="wp-caption-text">Figure 2. Cortex XDR alert on preventing RustBucket download activity.</figcaption></figure>
<p>The third stage payloads are <a href="https://developer.apple.com/library/archive/documentation/Performance/Conceptual/CodeFootprint/Articles/MachOOverview.html" target="_blank" rel="noopener">Mach-O</a> binaries written in Rust, hence the name RustBucket. Later variants of stage three employ persistence via a <a href="https://www.linkedin.com/pulse/maximizing-efficiency-macos-how-use-launchagents-run-scheduled/" target="_blank" rel="noopener">LaunchAgent</a>, a feature that did not exist in older variants. Stage three has two main commands:</p>
<ul>
<li>Download and execute a file</li>
<li>Self-terminate the malware</li>
</ul>
<h4><a id="post-136656-_6bdmoncw1sjt"></a><strong>KANDYKORN</strong></h4>
<p><strong>Malware type:</strong> Backdoor</p>
<p><strong>Group affiliation:</strong> Alluring Pisces</p>
<p><strong>First seen:</strong> 2023</p>
<p><strong>OS type:</strong> macOS</p>
<p><strong>Description:</strong></p>
<p>First discovered in 2023, KANDYKORN is the payload of a five-stage infection chain targeting macOS systems. Known infections of KANDYKORN start with social engineering, tricking the victim into downloading a malicious ZIP archive containing a malicious Python script. If the victim executes the Python file, it downloads stage two of the infection, which is a second Python script that is saved into a folder named <span style="font-family: 'courier new', courier, monospace;">_log</span>.</p>
<p>The second stage of the infection involves two additional Python scripts. The first Python script saved to the <span style="font-family: 'courier new', courier, monospace;">_log</span> directory downloads another script saved to the <span style="font-family: 'courier new', courier, monospace;">/Users/Shared/</span> directory, which in turn downloads a stage three file, saving it as <span style="font-family: 'courier new', courier, monospace;">/Users/shared/.sld</span>.</p>
<p>Stage three of the infection is a downloader and loader dubbed SUGARLOADER. For persistence, SUGARLOADER saves itself as <span style="font-family: 'courier new', courier, monospace;">/Users/shared/.log</span>.</p>
<p>Upon execution, SUGARLOADER checks for the existence of a configuration file at <span style="font-family: 'courier new', courier, monospace;">/Library/Caches/com.apple.safari.ck</span>. If that configuration file is missing, SUGARLOADER downloads it using a default IP address provided in the command line.</p>
<p>The configuration file at <span style="font-family: 'courier new', courier, monospace;">/Library/Caches/com.apple.safari.ck</span> contains the location to download the next stage from. In Figure 3, we see part of a Cortex XDR alert that reveals the installation of this configuration file.</p>
<figure id="attachment_136663" aria-describedby="caption-attachment-136663" style="width: 786px" class="wp-caption alignnone"><img decoding="async" class="wp-image-136663 size-medium" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2024/09/word-image-571611-136656-3-786x231.png" alt="Cortex XDR logs. Screenshot showing two rows in a table. The first row is File Create and the second is File Write." width="786" height="231" srcset="https://unit42.paloaltonetworks.com/wp-content/uploads/2024/09/word-image-571611-136656-3-786x231.png 786w, https://unit42.paloaltonetworks.com/wp-content/uploads/2024/09/word-image-571611-136656-3-768x226.png 768w, https://unit42.paloaltonetworks.com/wp-content/uploads/2024/09/word-image-571611-136656-3.png 844w" sizes="(max-width: 786px) 100vw, 786px" /><figcaption id="caption-attachment-136663" class="wp-caption-text">Figure 3. Section of a Cortex XDR alert revealing SUGARLOADER installing its configuration file.</figcaption></figure>
<p>Cortex XDR detects SUGARLOADER installing its configuration file and alerts on staged malware activity as shown below in Figure 4.</p>
<figure id="attachment_136665" aria-describedby="caption-attachment-136665" style="width: 786px" class="wp-caption alignnone"><img decoding="async" class="wp-image-136665 size-medium" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2024/09/word-image-574905-136656-4-786x251.png" alt="Security alert from Cortex XDR Agent indicating Staged Malware Activity with reference number, related to Sugarloader configuration file installation." width="786" height="251" srcset="https://unit42.paloaltonetworks.com/wp-content/uploads/2024/09/word-image-574905-136656-4-786x251.png 786w, https://unit42.paloaltonetworks.com/wp-content/uploads/2024/09/word-image-574905-136656-4-768x245.png 768w, https://unit42.paloaltonetworks.com/wp-content/uploads/2024/09/word-image-574905-136656-4.png 960w" sizes="(max-width: 786px) 100vw, 786px" /><figcaption id="caption-attachment-136665" class="wp-caption-text">Figure 4. Staged malware activity alert in Cortex XDR for SUGARLOADER.</figcaption></figure>
<p>After installing its configuration file, SUGARLOADER downloads a malware binary for HLOADER.</p>
<p>HLOADER functions as the persistence mechanism for KANDYKORN. HLOADER attempts to masquerade as <a href="https://discord.com/">Discord</a> by replacing the legitimate application and renaming itself <span style="font-family: 'courier new', courier, monospace;">Discord</span>. Figure 5 shows the Cortex XDR preventing this name change by HLOADER.</p>
<figure id="attachment_136667" aria-describedby="caption-attachment-136667" style="width: 600px" class="wp-caption alignnone"><img decoding="async" class="wp-image-136667" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2024/09/word-image-577962-136656-5.png" alt="Graphic showing a high severity alert for malware in Cortex XDR. The alert action is 'Prevented (Blocked)' and is represented by a pink circle and a shield inside a triangle warning symbol." width="600" height="286" srcset="https://unit42.paloaltonetworks.com/wp-content/uploads/2024/09/word-image-577962-136656-5.png 2048w, https://unit42.paloaltonetworks.com/wp-content/uploads/2024/09/word-image-577962-136656-5-786x375.png 786w, https://unit42.paloaltonetworks.com/wp-content/uploads/2024/09/word-image-577962-136656-5-1467x700.png 1467w, https://unit42.paloaltonetworks.com/wp-content/uploads/2024/09/word-image-577962-136656-5-768x366.png 768w, https://unit42.paloaltonetworks.com/wp-content/uploads/2024/09/word-image-577962-136656-5-1536x733.png 1536w" sizes="(max-width: 600px) 100vw, 600px" /><figcaption id="caption-attachment-136667" class="wp-caption-text">Figure 5. Alert from Cortex XDR preventing HLOADER from naming itself Discord for persistence.</figcaption></figure>
<p>If the legitimate Discord application already exists on the victim's host, HLOADER will rename the legitimate Discord file to a different name, so it can take over the <span style="font-family: 'courier new', courier, monospace;">Discord</span> file name. Figure 6 shows two actions from a Cortex XDR alert where HLOADER renamed the legitimate Discord app to a new name (the bottom file event). It then renamed itself to take the place of the legitimate Discord file (the top file event).</p>
<figure id="attachment_136669" aria-describedby="caption-attachment-136669" style="width: 800px" class="wp-caption alignnone"><img decoding="async" class="wp-image-136669" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2024/09/word-image-581649-136656-6.png" alt="Screenshot of a file event log from Cortex XDR showing details of file events, including file name, description, and associated categories." width="800" height="109" srcset="https://unit42.paloaltonetworks.com/wp-content/uploads/2024/09/word-image-581649-136656-6.png 1754w, https://unit42.paloaltonetworks.com/wp-content/uploads/2024/09/word-image-581649-136656-6-786x107.png 786w, https://unit42.paloaltonetworks.com/wp-content/uploads/2024/09/word-image-581649-136656-6-768x104.png 768w, https://unit42.paloaltonetworks.com/wp-content/uploads/2024/09/word-image-581649-136656-6-1536x208.png 1536w" sizes="(max-width: 800px) 100vw, 800px" /><figcaption id="caption-attachment-136669" class="wp-caption-text">Figure 6. File events from a Cortex XDR alert showing HLOADER renaming itself and the legitimate Discord file.</figcaption></figure>
<p>Because Discord usually boots with the operating system, if this file renaming is successful, HLOADER will run instead of the legitimate Discord application upon booting or rebooting. If Discord is already installed on the victim's system, HLOADER will also execute the newly renamed legitimate Discord application when booting or rebooting.</p>
<p>In the final stage of the attack, SUGARLOADER downloads KANDYKORN and loads it into memory by using <a href="https://attack.mitre.org/techniques/T1620/" target="_blank" rel="noopener">reflective loading</a>. KANDYKORN is the final payload and possesses several capabilities, including information gathering, data exfiltration and arbitrary command execution.</p>
<h4><a id="post-136656-_3su1wmjpaj64"></a><strong>SmoothOperator</strong></h4>
<p><strong>Malware type: </strong>Backdoor</p>
<p><strong>Group affiliation: </strong>Undetermined, under RGB</p>
<p><strong>First seen: </strong>2023</p>
<p><strong>OS type: </strong>macOS</p>
<p><strong>Description:</strong></p>
<p>In the beginning of 2023, multiple vendors discovered Trojanized macOS installers for the legitimate 3CX client application known as <a href="https://www.3cx.com/user-manual/installation-windows/" target="_blank" rel="noopener">3CXDesktopApp</a>. These Trojanized installers contained multi-staged malware called <a href="https://www.sentinelone.com/blog/smoothoperator-ongoing-campaign-trojanizes-3cx-software-in-software-supply-chain-attack/" target="_blank" rel="noopener">SmoothOperator</a>.</p>
<p>SmoothOperator can execute payloads and extract data related to 3CX from infected hosts. It is written in Objective-C and targets 64-bit Intel-based macOS users.</p>
<p>The Trojanized component of SmoothOperator inside the 3CXDesktopApp application is a module called <span style="font-family: 'courier new', courier, monospace;">libffmpeg.dylib</span>, which is a legitimate dependency that appears to have been altered or tampered with by the threat actors. The main purpose of this tampered <span style="font-family: 'courier new', courier, monospace;">libffmpeg.dylib</span> file is to collect the infected device’s environment information and to deliver additional payloads.</p>
<p>When downloading an additional payload, the module writes the payload into a file named <span style="font-family: 'courier new', courier, monospace;">UpdateAgent</span> and executes it. Below, Figure 7 shows disassembled code from a tampered <span style="font-family: 'courier new', courier, monospace;">libffmpeg.dylib</span> file related to saving the follow-up payload as <span style="font-family: 'courier new', courier, monospace;">UpdateAgent</span>.</p>

<p><span style="font-family: 'courier new', courier, monospace;">UpdateAgent</span> collects the victim's 3CX account information, then it removes itself. <h3><a id="post-136656-_7u2h81jumfue"></a>Linux Malware</h3>
<h4><a id="post-136656-_2ikqlqk3xlfo"></a><strong>OdicLoader</strong></h4>
<p><strong>Malware type:</strong> Downloader</p>
<p><strong>Group affiliation:</strong> Selective Pisces</p>
<p><strong>First seen:</strong> 2023</p>
<p><strong>OS type:</strong> Linux</p>
<p><strong>Description:</strong></p>
<p>OdicLoader is an ELF downloader that masquerades as a PDF file by using the <a href="https://www.compart.com/en/unicode/U+2024" target="_blank" rel="noopener"><span style="font-family: 'courier new', courier, monospace;">U+2024</span> Unicode character</a> (hexadecimal <span style="font-family: 'courier new', courier, monospace;">0xE2 0x80 0xA4</span>) instead of a period (hexadecimal <span style="font-family: 'courier new', courier, monospace;">0x2e</span>) with a <span style="font-family: 'courier new', courier, monospace;">pdf</span> file extension. <a href="https://attack.mitre.org/techniques/T1204/002/" target="_blank" rel="noopener">This technique</a> can deceive the file manager in a graphical Linux environment, causing the fake PDF file to execute as an ELF when double-clicked instead of opening with a PDF viewer.</p>
<p>When executed, OdicLoader opens a decoy PDF with the system's default PDF viewer using <span style="font-family: 'courier new', courier, monospace;"><a href="https://linux.die.net/man/1/xdg-open" target="_blank" rel="noopener">xdg-open</a></span>, then it downloads and executes the next stage payload.</p>
<p>ESET reported OdicLoader as part of a North Korean threat campaign named <a href="https://www.welivesecurity.com/2023/04/20/linux-malware-strengthens-links-lazarus-3cx-supply-chain-attack/" target="_blank" rel="noopener">Operation DreamJob</a>. <h4><a id="post-136656-_ewxdb8904h0p"></a>Comebacker</h4>
<p><strong>Malware type:</strong> Backdoor and downloader</p>
<p><strong>Group affiliation:</strong> Selective Pisces</p>
<p><strong>First seen:</strong> 2020</p>
<p><strong>OS type:</strong> Windows</p>

<p>Attackers originally used Comebacker malware as part of a <a href="https://www.microsoft.com/en-us/security/blog/2021/01/28/zinc-attacks-against-security-researchers/" target="_blank" rel="noopener">campaign targeting security researchers in 2020</a>. <h4><a id="post-136656-_jhcva4ew2x2b"></a><strong>CollectionRAT</strong></h4>

<p>These capabilities include:</p>
<ul>
<li>Manipulating processes and files</li>
<li>Executing arbitrary commands</li>
<li>Exfiltrating data</li>
<li>Downloading and executing additional payloads</li>
<li>Removing itself from an infected host upon instruction from the C2 server</li>
</ul>

				return str.substr(0, 1) + str.substr(1, strLen - 1).replace(/\w/g, '*') + str.substr(-1,1);
			}
			return str.replace(/\w/g, '*');
		}
		return emailAddress.replace(/([\w.]+)@([\w.]+)(\.[\w.]+)/g, function (m, p1, p2, p3) {
			return mask(p1) + '@' + mask(p2) + p3;
		});

		return emailAddress;
