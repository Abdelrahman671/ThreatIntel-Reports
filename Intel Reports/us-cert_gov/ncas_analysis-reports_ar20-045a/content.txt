<h3>Summary</h3>
<div class="cma-section">
<h5>Description</h5>
<p>This Malware Analysis Report (MAR) is the result of analytic efforts between Department of Homeland Security (DHS), the Federal Bureau of Investigation (FBI), and the Department of Defense (DoD). Working with U.S. Government partners, DHS, FBI, and DoD identified Trojan malware variants used by the North Korean government. This malware variant has been identified as BISTROMATH. The U.S. Government refers to malicious cyber activity by the North Korean government as HIDDEN COBRA. For more information on HIDDEN COBRA activity, visit https[:]//www[.]us-cert.gov/hiddencobra.<br><br>
			<br><br>
			DHS, FBI, and DoD are distributing this MAR to enable network defense and reduce exposure to North Korean government malicious cyber activity.<br><br>
			<br><br>
			This MAR includes malware descriptions related to HIDDEN COBRA, suggested response actions and recommended mitigation techniques. Users or administrators should flag activity associated with the malware and report the activity to the Cybersecurity and Infrastructure Security Agency (CISA) or the FBI Cyber Watch (CyWatch), and give the activity the highest priority for enhanced mitigation.<br><br>
			<br><br>
			This report looks at multiple versions of a full-featured RAT implant executable and multiple versions of the CAgent11 GUI implant controller/builder. These samples performs simple XOR network encoding and are capable of many features including conducting system surveys, file upload/download, process and command execution, and monitoring the microphone, clipboard, and the screen. The GUI controllers allow interaction with the implant as well as the option to dynamically build new implants with customized options. The implants are loaded with a trojanized executable containing a fake bitmap which decodes into shellcode which loads the embedded implant.</p>
<p>For a downloadable copy of IOCs, see <a href="https://www.us-cert.gov/sites/default/files/publications/MAR-10265965-1.v1.stix.xml">MAR-101265965-1.v1.stix</a>.</p>

			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;enabled_options += "%s\\" % truncate_nullterm_str(options[34])<br><br>
			&nbsp;&nbsp;&nbsp;enabled_options += "%s\n" % truncate_nullterm_str(options[35])<br><br>
			<br><br>
			&nbsp;&nbsp;&nbsp;str = "Mutex\n"<br><br>
			&nbsp;&nbsp;&nbsp;if len(truncate_nullterm_str(options[36])) == 0:<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;disabled_options += str<br><br>
			&nbsp;&nbsp;&nbsp;else:<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;enabled_options += "Mutex: %s\n" % truncate_nullterm_str(options[36])<br><br>
			<br><br>
			&nbsp;&nbsp;&nbsp;print("\nDisabled Options:")<br><br>
			&nbsp;&nbsp;&nbsp;print(disabled_options)<br><br>
			<br><br>
			&nbsp;&nbsp;&nbsp;print("\nEnabled Options:")<br><br>
			&nbsp;&nbsp;&nbsp;print(enabled_options)<br><br>
			<br><br>
			<br><br>
			def main():<br><br>
			&nbsp;&nbsp;&nbsp;parser = argparse.ArgumentParser()<br><br>
			&nbsp;&nbsp;&nbsp;parser.add_argument('filename')<br><br>
			&nbsp;&nbsp;&nbsp;args = parser.parse_args()<br><br>
			<br><br>
			&nbsp;&nbsp;&nbsp;with open(args.filename, 'rb') as f:<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;exe = f.read()<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;PE_header_pos = struct.unpack('&lt;i', exe[0x3c:0x3c+4])[0]<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;PE_header_len = struct.unpack('&lt;i', exe[PE_header_pos+0x54:PE_header_pos+0x54+4])[0]<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;PE_header_length = struct.unpack('&lt;h', exe[PE_header_pos+0x14:PE_header_pos+0x14+2])[0]<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;section_headers_pos = PE_header_pos + PE_header_length + 0x18<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;num_headers = struct.unpack('&lt;h', exe[PE_header_pos+0x6:PE_header_pos+0x6+2])[0]<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;curr_header_pos = section_headers_pos<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;bitmap_pos = PE_header_len<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;for i in range(num_headers):<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;header_len = struct.unpack('&lt;i', exe[curr_header_pos+0x10:curr_header_pos+0x10+4])[0]<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;bitmap_pos += header_len<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;curr_header_pos += 0x28<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;key1 = struct.unpack('&lt;I', exe[bitmap_pos+0x3a:bitmap_pos+0x3a+4])[0]<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;bitmap_len = len(exe) - bitmap_pos<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;bitmap_header_len = struct.unpack('&lt;H', exe[bitmap_pos+0x3e:bitmap_pos+0x3e+2])[0]<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;key2 = struct.unpack('&lt;I', exe[bitmap_pos+0x36:bitmap_pos+0x36+4])[0]<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;bitmap_len -= bitmap_header_len<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;bitmap_len -= 0x036<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;print("[ ] Decoding %d Bytes with:" % bitmap_len)<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;print(" &nbsp;&nbsp;&nbsp;Key1: %s" % hex(key1))<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;print(" &nbsp;&nbsp;&nbsp;Key2: %s" % hex(key2))<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;dec = decode(0,exe[bitmap_pos+bitmap_header_len+0x36:],bitmap_len,key1,key2)<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;print("[+] Decoding Complete!")<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;parse_options(dec[0:0xd56-0x36])<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;payload_pos = 0xd56-0x36+offset<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;print("[ ] Found embedded payload, extracting..")<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;with open(args.filename + "_payload.exe", 'wb') as out:<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;out.write(dec[payload_pos:])<br><br>
			&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;print("[+] Wrote %d Bytes to %s" % (len(dec[payload_pos:]), args.filename + "_payload.exe"))<br><br>
			<br><br>
			if __name__ == '__main__':<br><br>
			&nbsp;&nbsp;&nbsp;main()<br><br>
			--End Decryption and Extraction Python3 Script--</p>

<h3>Recommendations</h3>
<div class="cma-section">
<p>CISA recommends that users and administrators consider using the following best practices to strengthen the security posture of their organization's systems. Any configuration changes should be reviewed by system owners and administrators prior to implementation to avoid unwanted impacts.</p>
<ul>
<li>Maintain up-to-date antivirus signatures and engines.</li>
<li>Keep operating system patches up-to-date.</li>
<li>Disable File and Printer sharing services. If these services are required, use strong passwords or Active Directory authentication.</li>
<li>Restrict users' ability (permissions) to install and run unwanted software applications. Do not add users to the local administrators group unless required.</li>
<li>Enforce a strong password policy and implement regular password changes.</li>
<li>Exercise caution when opening e-mail attachments even if the attachment is expected and the sender appears to be known.</li>
<li>Enable a personal firewall on agency workstations, configured to deny unsolicited connection requests.</li>
<li>Disable unnecessary services on agency workstations and servers.</li>
<li>Scan for and remove suspicious e-mail attachments; ensure the scanned attachment is its "true file type" (i.e., the extension matches the file header).</li>
<li>Monitor users' web browsing habits; restrict access to sites with unfavorable content.</li>
<li>Exercise caution when using removable media (e.g., USB thumb drives, external drives, CDs, etc.).</li>
<li>Scan all software downloaded from the Internet prior to executing.</li>
<li>Maintain situational awareness of the latest threats and implement appropriate Access Control Lists (ACLs).</li>
</ul>
<p>Additional information on malware incident prevention and handling can be found in National Institute of Standards and Technology (NIST) Special Publication 800-83, <strong>"Guide to Malware Incident Prevention &amp; Handling for Desktops and Laptops".</strong></p>
</div>

<h3>Document FAQ</h3>
<div class="cma-section">
<p><strong>What is a MIFR?</strong> A Malware Initial Findings Report (MIFR) is intended to provide organizations with malware analysis in a timely manner. In most instances this report will provide initial indicators for computer and network defense. To request additional analysis, please contact CISA and provide information regarding the level of desired analysis.</p>
<p><strong>What is a MAR?</strong> A Malware Analysis Report (MAR) is intended to provide organizations with more detailed malware analysis acquired via manual reverse engineering. To request additional analysis, please contact CISA and provide information regarding the level of desired analysis.</p>
<p><strong>Can I edit this document?</strong> This document is not to be edited in any way by recipients. All comments or questions related to this document should be directed to the CISA at 1-888-282-0870 or <a href="mailto:soc@us-cert.gov">soc@us-cert.gov</a>.</p>
<p><strong>Can I submit malware to CISA?</strong> Malware samples can be submitted via three methods:</p>
<ul>
<li>Web: <a href="https://malware.us-cert.gov">https://malware.us-cert.gov</a></li>
<li>E-Mail: <a href="mailto:submit@malware.us-cert.gov">submit@malware.us-cert.gov</a></li>
<li>FTP: ftp.malware.us-cert.gov (anonymous)</li>
</ul>
<p>CISA encourages you to report any suspicious activity, including cybersecurity incidents, possible malicious code, software vulnerabilities, and phishing-related scams. Reporting forms can be found on CISA's homepage at <a href="http://www.us-cert.gov">www.us-cert.gov</a>.</p>
