					<div id="mw-content-text" class="mw-body-content"><div class="mw-content-ltr mw-parser-output" lang="en" dir="ltr"><div class="shortdescription nomobile noexcerpt noprint searchaux" style="display:none">Malware framework for attacking industrial control systems</div>
<p>
<b>Industroyer</b><sup id="cite_ref-1" class="reference"><a href="#cite_note-1"><span class="cite-bracket">&#91;</span>1<span class="cite-bracket">&#93;</span></a></sup> (also referred to as <b>Crashoverride</b>) is a <a href="/wiki/Malware" title="Malware">malware</a> framework considered to have been used in the cyberattack on <a href="/wiki/Ukraine" title="Ukraine">Ukraine</a>’s power grid on December 17, 2016.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2"><span class="cite-bracket">&#91;</span>2<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-&#93;_Pavel_Polityuk,_Oleg_Vukmanovic_and_Stephen_Jewkes:_Ukraine&#39;s_power_outage_was_a_cyber_attack:_Ukrenergo_3-0" class="reference"><a href="#cite_note-]_Pavel_Polityuk,_Oleg_Vukmanovic_and_Stephen_Jewkes:_Ukraine&#39;s_power_outage_was_a_cyber_attack:_Ukrenergo-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-We_live_security_4-0" class="reference"><a href="#cite_note-We_live_security-4"><span class="cite-bracket">&#91;</span>4<span class="cite-bracket">&#93;</span></a></sup>
The attack cut a fifth of <a href="/wiki/Kyiv" title="Kyiv">Kyiv</a>, the capital, off power for one hour and is considered to have been a large-scale test.<sup id="cite_ref-Motherboard_5-0" class="reference"><a href="#cite_note-Motherboard-5"><span class="cite-bracket">&#91;</span>5<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-6" class="reference"><a href="#cite_note-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup> The Kyiv incident was the second cyberattack on Ukraine's power grid in two years. The <a href="/wiki/December_2015_Ukraine_power_grid_cyberattack" class="mw-redirect" title="December 2015 Ukraine power grid cyberattack">first attack</a> occurred on December 23, 2015.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7"><span class="cite-bracket">&#91;</span>7<span class="cite-bracket">&#93;</span></a></sup> Industroyer is the first ever known malware specifically designed to attack <a href="/wiki/Electrical_grid" title="Electrical grid">electrical grids</a>.<sup id="cite_ref-Dragos_8-0" class="reference"><a href="#cite_note-Dragos-8"><span class="cite-bracket">&#91;</span>8<span class="cite-bracket">&#93;</span></a></sup>
At the same time, it is the fourth malware publicly revealed to target <a href="/wiki/Industrial_control_systems" class="mw-redirect" title="Industrial control systems">industrial control systems</a>, after <a href="/wiki/Stuxnet" title="Stuxnet">Stuxnet</a>, <a href="/wiki/Havex" title="Havex">Havex</a>, and <a href="/wiki/BlackEnergy" title="BlackEnergy">BlackEnergy</a>.
</p>
<meta property="mw:PageProp/toc" />
<div class="mw-heading mw-heading2"><h2 id="Discovery_and_naming">Discovery and naming</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Industroyer&amp;action=edit&amp;section=1" title="Edit section: Discovery and naming"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The malware was discovered by Slovak internet security company <a href="/wiki/ESET" title="ESET">ESET</a>. ESET and most of the cybersecurity companies detect it under the name “Industroyer”.<sup id="cite_ref-Virustotal1_9-0" class="reference"><a href="#cite_note-Virustotal1-9"><span class="cite-bracket">&#91;</span>9<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-Virustotal2_10-0" class="reference"><a href="#cite_note-Virustotal2-10"><span class="cite-bracket">&#91;</span>10<span class="cite-bracket">&#93;</span></a></sup>
Cybersecurity firm Dragos named the malware “Crashoverride”.<sup id="cite_ref-Dragos_8-1" class="reference"><a href="#cite_note-Dragos-8"><span class="cite-bracket">&#91;</span>8<span class="cite-bracket">&#93;</span></a></sup>
In 2022, the Russian hacker group <a href="/wiki/Sandworm_(hacker_group)" title="Sandworm (hacker group)">Sandworm</a> initiated a blackout in Ukraine using a variant of Industroyer aptly dubbed Industroyer2.<sup id="cite_ref-11" class="reference"><a href="#cite_note-11"><span class="cite-bracket">&#91;</span>11<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading2"><h2 id="Description">Description</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Industroyer&amp;action=edit&amp;section=2" title="Edit section: Description"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The detailed analysis of Industroyer<sup id="cite_ref-We_live_security2_12-0" class="reference"><a href="#cite_note-We_live_security2-12"><span class="cite-bracket">&#91;</span>12<span class="cite-bracket">&#93;</span></a></sup>
revealed that the malware was designed to disrupt the working processes of industrial control systems, specifically those used in <a href="/wiki/Electrical_substation" class="mw-redirect" title="Electrical substation">electrical substations</a>.
Industroyer is modular malware; its main components are the following:
</p>
<ul><li><b>A main <a href="/wiki/Backdoor_(computing)" title="Backdoor (computing)">backdoor</a></b>  is used to control all other components of the malware. It connects to its remote Command &amp; Control servers in order to receive commands from the attackers.</li>
<li><b>An additional backdoor</b> provides an alternative persistence mechanism that allows the attackers to regain access to a targeted network in case the main backdoor is detected and/or disabled.</li>
<li><b>A launcher component</b> is a separate executable responsible for launching the payload components and the data wiper component. The launcher component contains a specific activation time and date; analyzed samples contained two dates: December 17, 2016 and December 20, 2016. (Note: the former date was the date the attack actually went ahead.)</li>
<li><b>Four <a href="/wiki/Payload" title="Payload">payload</a> components</b> target particular industrial <a href="/wiki/Communication_protocols" class="mw-redirect" title="Communication protocols">communication protocols</a> specified in the following standards: <a href="/wiki/IEC_60870-5-101" class="mw-redirect" title="IEC 60870-5-101">IEC 60870-5-101</a>, <a href="/wiki/IEC_60870-5-104" class="mw-redirect" title="IEC 60870-5-104">IEC 60870-5-104</a>, <a href="/wiki/IEC_61850" title="IEC 61850">IEC 61850</a>, and OLE for Process Control Data Access (<a href="/wiki/OPC_Data_Access" title="OPC Data Access">OPC Data Access</a>). The functionalities of the payload components include mapping the network, and then issuing commands to the specific industrial control devices.</li>
<li><b>A data wiper component</b> is designed to erase system-crucial <a href="/wiki/Registry_keys" class="mw-redirect" title="Registry keys">Registry keys</a> and overwrite files to make the system unbootable and recovery from the attack harder.</li></ul>
