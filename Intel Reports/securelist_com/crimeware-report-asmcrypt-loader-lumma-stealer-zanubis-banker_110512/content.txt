													<h2 id="introduction">Introduction</h2>
<p>As long as cybercriminals want to make money, they&#8217;ll keep making malware, and as long as they keep making malware, we&#8217;ll keep analyzing it, publishing reports and providing protection. Last month we covered a wide range of cybercrime topics. For example, we published a private report on a new malware found on underground forums that we call ASMCrypt (related to the <a href="https://securelist.com/doublefinger-loader-delivering-greetingghoul-cryptocurrency-stealer/109982/" target="_blank" rel="noopener">DoubleFinger loader</a>). But there&#8217;s more going on in the cybercrime landscape, so we also published reports on new versions of the Lumma stealer and Zanubis Android banking trojan. This blog post contains excerpts from those reports.</p>
<p>If you want to learn more about our crimeware reporting service, please contact us at <a href="mailto:crimewareintel@kaspersky.com" target="_blank" rel="noopener">crimewareintel@kaspersky.com</a>.</p>
<h2 id="asmcrypt">ASMCrypt</h2>
<p>As mentioned in our <a href="https://securelist.com/emotet-darkgate-lokibot-crimeware-report/110286/" target="_blank" rel="noopener">previous blog post</a>, we monitor many underground forums. On one of them we saw an ad, promoting a new cryptor/loader variant called ASMCrypt. The idea behind this type of malware is to load the final payload without the loading process or the payload itself being detected by AV/EDR, etc. This sounds a lot like the DoubleFinger loader we discussed <a href="https://securelist.com/doublefinger-loader-delivering-greetingghoul-cryptocurrency-stealer/109982/" target="_blank" rel="noopener">here</a>.</p>
<p>In fact, after careful analysis, we believe with a high degree of confidence that ASMCrypt is an evolved version of DoubleFinger. However, ASMCrypt works slightly differently and is more of a &#8220;front&#8221; for the actual service that runs on the TOR network.</p>
<p>So how does it work? First the buyer obtains the ASMCrypt binary, which connects to the malware&#8217;s backend service over the TOR network using hardcoded credentials. If everything is okay, the options menu is shown:</p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/09/06172830/crimeware-report-asmcrypt-lumma-01.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-110524" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/09/06172830/crimeware-report-asmcrypt-lumma-01.png" alt="" width="663" height="355" /></a></p>
<p>The buyer can choose from the following options:</p>
<ul>
<li>Stealth or invisible injection method;</li>
<li>The process the payload should be injected into;</li>
<li>Folder name for startup persistence;</li>
<li>Stub type: either the malware itself masquerading as Apple QuickTime, or a legitimate application that sideloads the malicious DLL.</li>
</ul>
<p>After selecting all the desired options and pressing the build button, the application creates an encrypted blob hidden inside a .png file. This image must be uploaded to an image hosting site. The malicious DLL (or binary) from the last bullet point above is also created and will be distributed by the cybercriminals.</p>
<p>When the malicious DLL is executed on a victim system, it downloads the .png file, decrypts it, loads it into memory and then executes it.</p>
<h2 id="lumma">Lumma</h2>
<p>The Arkei stealer, written in C++, first appeared in May 2018 and has been forked/rebranded several times over the last couple of years. It has been known as Vidar, Oski, Mars and now Lumma, which has a 46% overlap with Arkei. Over time, the main functionality of all the variants has remained the same: stealing cached files, configuration files and logs from crypto wallets. It can do this by acting as a browser plugin, but it also supports the standalone Binance application.</p>
<p>But first the infection vector. Lumma is distributed via a spoofed website that mimics a legitimate .docx to .pdf site. When a file is uploaded, it is returned with the double extension .pdf.exe.</p>
<p>Lumma itself first appeared on our radar in August 2022, when we detected new samples. Around the same time, cybersecurity enthusiast Fumik0_ <a href="https://twitter.com/fumik0_/status/1559474920152875008" target="_blank" rel="noopener">tweeted</a> that Lumma was a &#8220;fork/refactor&#8221; of Mars. Since then, Lumma has undergone a number of changes, some of which we will highlight below:</p>
<ul>
<li>We found only one sample (MD5 6b4c224c16e852bdc7ed2001597cde9d) that had the functionality to collect the system process list. The same sample also used a different URL to communicate with the C2 (/winsock instead of /socket.php);</li>
<li>We also found one sample (MD5 844ab1b8a2db0242a20a6f3bbceedf6b) that appears to be a debugging version. When certain code fragments are reached, a notification is sent to the C2. Again, it uses a different URL (/windbg).</li>
<li>In a more recent sample (MD5 a09daf5791d8fd4b5843cd38ae37cf97), the attackers changed the User-Agent field to &#8220;HTTP/1.1&#8221;. It is unclear why this was done;</li>
<li>While all previous samples, including the three mentioned above, downloaded additional libraries from the C2 for 32-bit systems so that specific browser-related files (e.g. passwords and the like) could be parsed, MD5 5aac51312dfd99bf4e88be482f734c79 simply uploads the entire database to the C2;</li>
<li>MD5 d1f506b59908e3389c83a3a8e8da3276 has a string encryption algorithm. They are now hex encoded and encrypted with an XOR key (first 4 bytes of the string).</li>
<li>One of the biggest changes we saw involved MD5 c2a9151e0e9f4175e555cf90300b45c9. This sample supports dynamic configuration files retrieved from the C2. The configuration is Base64 encoded and XORed with the first 32 bytes of the configuration file.</li>
</ul>
<div id="attachment_110525" style="width: 824px" class="wp-caption aligncenter"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/09/06172823/crimeware-report-asmcrypt-lumma-02.png" class="magnificImage"><img loading="lazy" decoding="async" aria-describedby="caption-attachment-110525" class="size-full wp-image-110525" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/09/06172823/crimeware-report-asmcrypt-lumma-02.png" width="814" height="653" /></a><p id="caption-attachment-110525" class="wp-caption-text">Code snippet of the &#8220;debugging&#8221; sample</p></div>
<h2 id="zanubis">Zanubis</h2>
<p>Zanubis, an Android banking trojan, first <a href="https://www.entdark.net/2022/09/zanubis-latam-banking-trojan.html" target="_blank" rel="noopener">appeared</a> around August 2022, targeting financial institution and cryptocurrency exchange users in Peru. Zanubis&#8217;s main infection path is through impersonating legitimate Peruvian Android applications and then tricking the user into enabling the Accessibility permissions in order to take full control of the device.</p>
<p>We spotted more recent samples of Zanubis  in the wild around April 2023. The malware was disguised as the official Android application for the Peruvian governmental organization SUNAT (Superintendencia Nacional de Aduanas y de Administración Tributaria). We explored the new design and features of the malware, which seemed to have undergone several phases of evolution to reach a new level of sophistication.</p>
<p>Zanubis is obfuscated with the help of Obfuscapk, a popular obfuscator for Android APK files. After the victim grants Accessibility permissions to the malicious app, thus allowing it to run in the background, the malware uses WebView to load a legitimate SUNAT website used for looking up debts. The intention here is to lead the unsuspecting user to believe that the app is part of the SUNAT ecosystem of services.</p>
<p>Communication with the C2 relies on WebSockets and the library called Socket.IO. The latter allows the malware to establish a persistent connection to the C2, which provides failover options (from WebSockets to HTTP and vice versa). Another advantage is that it provides the C2 with a scalable environment where all new infections by Zanubis can receive commands (also called <em>events</em>) on a massive scale from the C2 if required. Once the malware starts, the implant calls a function to check the connection to the C2. It establishes two connections to the same C2 server, but they perform different types of actions, and the second connection is established only if requested by the C2.</p>
<p>Intentionally, Zanubis doesn&#8217;t count with a pre-populated and hardcoded list of applications to target. In recent years, malware developers have tended to add or remove the names of applications from the target list. To set the targeted applications on the implant, the C2 sends the event <em>config_packages</em>. The JSON object sent with the event contains an array specifying the applications that the malware should monitor. The malware parses the list of targeted applications each time an event occurs on the screen, such as an app opening, which the malware detects using the <em>onAccessibilityEvent</em> function. Once an application on the list is found running on the device, Zanubis takes one of two actions, depending on its configuration, to steal the victim&#8217;s information: logging events/keys, or recording the screen.</p>
<p>Previously, we mentioned initializing the second connection from the infected device, which provides further options for the C2. After Zanubis establishes this new connection, it sends a <em>VncInit</em> event to the server to inform it that initialization of the second feature set is complete, and it will send information about screen rendering, such as the display size, every second. We can assume that this is a way for the operators to take control of, or backdoor, the infected phone.</p>
<p>An interesting feature in the second set is the <em>bloqueoUpdate</em> event. This is one of the most invasive – and persuasive – actions taken by the malware: it pretends to be an Android update, thus blocking the phone from being used. As the &#8220;update&#8221; runs, the phone remains unusable to the point that it can&#8217;t be locked or unlocked, as the malware monitors those attempts and blocks them.</p>
<div id="attachment_110630" style="width: 186px" class="wp-caption aligncenter"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/09/13104747/crimeware-report-zanubis.png" class="magnificImage"><img loading="lazy" decoding="async" aria-describedby="caption-attachment-110630" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/09/13104747/crimeware-report-zanubis-176x300.png" alt="" width="176" height="300" class="size-medium wp-image-110630" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/09/13104747/crimeware-report-zanubis-176x300.png 176w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/09/13104747/crimeware-report-zanubis-103x175.png 103w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/09/13104747/crimeware-report-zanubis-293x500.png 293w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/09/13104747/crimeware-report-zanubis-164x280.png 164w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/09/13104747/crimeware-report-zanubis.png 385w" sizes="(max-width: 176px) 100vw, 176px" /></a><p id="caption-attachment-110630" class="wp-caption-text">Fake update locking the user out of the phone</p></div>
<p>According to our analysis, the targeted applications are banks and financial entities in Peru. This fact, in conjunction with our telemetry data, leads us to determine that Zanubis targets users in that country specifically. The list of targeted applications contains more than 40 package names. The samples of Zanubis collected to date are capable of infecting any Android phone, but they were all written with Spanish as the system language in mind.</p>
<h2 id="conclusion">Conclusion</h2>
<p>Malware is constantly evolving, as is illustrated by the Lumma stealer, which has multiple variations with varying functionality. Zanubis also aspires to become a fully armed banking trojan that could inflict financial losses and steal the personal data of mobile users. This constant change in malicious code and cybercriminal TTPs is a challenge for defense teams. To protect itself, an organization must learn about new threats as soon as they emerge. Intelligence reports can help you stay on top of the latest malicious tools and attacker TTPs. If you&#8217;d like to stay up to date on the latest TTPs being used by criminals, or have questions about our private reports, please contact us at <a href="mailto:crimewareintel@kaspersky.com" target="_blank" rel="noopener">crimewareintel@kaspersky.com</a>.</p>
