<h1 style="text-align: left">Latest Golden Chickens MaaS Tools Updates and Observed Attacks</h1>
</div>
</div>
<div class="et_pb_module et_pb_text et_pb_text_82  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">
<h2>Executive Summary</h2>
<ul>
<li>Throughout March and April, <a href="https://quointelligence.eu/">QuoIntelligence</a> observed four attacks utilizing various tools from the Golden Chickens (GC) Malware-as-a-Service (MaaS) portfolio. We are now declassifying our findings for the general public.</li>
<li>Overall, we attribute the separately conducted campaigns with confidence varying from low to moderate to GC05, GC06.tmp, and <strong>FIN6</strong>.</li>
<li>During our analysis of the attacks, we uncovered the GC MaaS Operator, <strong>Badbullzvenom</strong>, created new variants of three existing tools in the service portfolio with notable code updates to TerraLoader, VenomLNK, and more_eggs.</li>
<li>TerraLoader. A multipurpose loader written in PureBasic.
<ul>
<li>Updates – the new variant uses different string de/obfuscation, brute-forcing implementation, and anti-analysis techniques.</li>
</ul>
</li>
<li>VenomLNK. A Windows shortcut file likely generated by a newer version of the VenomKit building kit.
<ul>
<li>Updates – the new variant uses a new volume serial number, an evolved execution scheme, and only the local path to the Windows command prompt.</li>
</ul>
</li>
<li>more_eggs. A backdoor malware written in JavaScript (JS)
<ul>
<li>Updates – the new variant includes a minimum delay before executing or retrying an action, and cleans up memory after using it.</li>
</ul>
</li>
<li>In April, we detected two new attacks sharing similar characteristics of previously observed attack activity attributed to FIN6 – a financially-motivated threat actor group. Based on our analysis of the new campaigns, we assess attribution to FIN6 with low to moderate confidence.</li>
<li>The uncovered campaigns highlight that Badbullzvenom is still highly active in the business of its MaaS, and that FIN6 is still one of Badbullzvenom’s recurrent customers.</li>
</ul>
</div>
</div>
<div class="et_pb_module et_pb_text et_pb_text_83  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">
<h2>Introduction</h2>
<p>Throughout March and April, QuoIntelligence observed four attacks (i.e. <em>sightings</em>) utilizing various tools from the Golden Chickens (GC) Malware-as-a-Service (MaaS) portfolio &#8211; we are now declassifying our findings, after first notifying clients on 22 May . Further, during our analysis of the sightings, we confirmed the GC MaaS Operator, Badbullzvenom, released improved variants with code updates to three tools in the service portfolio:</p>
<ul>
<li><strong>TerraLoader.</strong> A multipurpose loader written in PureBasic. TerraLoader is a flagship product of GC MaaS service portfolio.</li>
<li><strong>more_eggs.</strong> A backdoor malware capable of beaconing to a fixed command and control (C2) server and executing additional payloads downloaded from an external Web resource. The backdoor is written in JavaScript (JS).</li>
<li><strong>VenomLNK.</strong> A Windows shortcut file likely generated by a newer version of the VenomKit building kit.</li>
</ul>
</div>
</div>
<div class="et_pb_module et_pb_text et_pb_text_84  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">

<div class="et_pb_module et_pb_text et_pb_text_85  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">
<h2>The Golden Chickens</h2>
<p>Since 2018, QuoItelligence has tracked the evolution of the GC MaaS, the activities of its Operator <em>Badbullzvenom</em>, as well as the different threat actors using the MaaS – including top-tier, financially-motivated threat actors such as FIN6 and the Cobalt Group. The GC MaaS remains as a preferred service provider for top-tier e-crime threat actor groups due to Badbullzvenom/the Operator’s consistent updates and improvements of tools and its ability to maintain underlying network infrastructure. Although GC tools have primarily been used to compromise organizations in the retail and financial sector, one notable outlier includes a potentially targeted attack against a chemical company.</p>
</div>
</div>
<div class="et_pb_module et_pb_text et_pb_text_86  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">
<h2>Technical Analysis</h2>
</div>
</div>
<div class="et_pb_module et_pb_text et_pb_text_87  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">
<h3>Latest Sightings Related to GC MaaS</h3>
<p>Throughout March and April, QuoIntelligence has observed four sightings utilizing various tools from the GC MaaS portfolio. Overall, we attribute the separately conducted campaigns with confidence varying from low to moderate to GC05, GC06.tmp, and FIN6. To clarify GC05 and GC06.tmp, we categorize the multiple GC MaaS clients as GCxx based on their overall motives, means, and opportunities. Additionally, we append .tmp to the GC categorization to represent that we are investigating their exact singular attribution.</p>
</div>
</div>
<div class="et_pb_module et_pb_text et_pb_text_88  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">
<h3>Sighting 1 GC06.tmp: Excel 4.0 Macro Sheet Used to Deliver GC MaaS Infection Chain</h3>
<p>On 6 March, QuoIntelligence <a href="https://twitter.com/malz_intel/status/1235964282921275393">detected</a> a new malicious Microsoft Excel document leading to the download of GC MaaS tools. Following our preliminary analysis, we confirmed the malicious document (maldoc) leads to an attack kill-chain which entirely relies on GC MaaS tools. Based on our telemetry, we assess with moderate to high confidence this targeted attack was against a large German chemical company.</p>
<p>Upon further analysis, we do not attribute the maldoc to the GC MaaS toolset as it is clear the employed technique is a well-documented abuse of a legacy functionality in Microsoft Office known as <a href="https://malware.pizza/2020/05/12/evading-av-with-excel-macros-and-biff8-xls/">Excel 4.0 Macro Sheet</a>. The Macro Sheet was obviously adapted to use the downloaded .<em>ocx</em> file – the typical file extension of TerraLoader.</p>
</div>
</div>
<div class="et_pb_module et_pb_text et_pb_text_89  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">
<p><img decoding="async" class="wp-image-1677 aligncenter size-full" src="https://quointelligence.eu/wp-content/uploads/2020/07/Figure-2.png" alt="" width="2340" height="196" srcset="https://quointelligence.eu/wp-content/uploads/2020/07/Figure-2.png 2340w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-2-300x25.png 300w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-2-1024x86.png 1024w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-2-150x13.png 150w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-2-768x64.png 768w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-2-1536x129.png 1536w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-2-2048x172.png 2048w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-2-1080x90.png 1080w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-2-1280x107.png 1280w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-2-980x82.png 980w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-2-480x40.png 480w" sizes="(max-width: 2340px) 100vw, 2340px" /></p>
<p style="text-align: center"><em>Figure 2: Output of tools to parse Microsoft document OLE objects</em></p>
</div>
</div>
<div class="et_pb_module et_pb_text et_pb_text_90  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">
<p>The Macro Sheet contains formulas in cells to perform actions, including <em>Run on open</em> (Auto_Open) and execute shell commands and web requests. Once the document is opened, the Macro Sheet’s code is triggered, and it initiates the infection routine to download and execute the next stage payload which is a TerraLoader variant.</p>
<p>The attack chain consists of multiple known GC tools which are:</p>
<ul>
<li><strong>TerraLoader</strong>. A multipurpose loader, written in PureBasic. TerraLoader is essentially a flagship product of GC MaaS service portfolio.</li>
<li><strong>lite_more_eggs</strong>. A lite version of more_eggs used as a loader, written in JavaScript.</li>
<li><strong>more_eggs.</strong> A backdoor malware capable of beaconing to a fixed command and control (C2) server and executing additional payloads downloaded from an external Web resource. The backdoor is written in JavaScript.</li>
<li><strong>TerraStealer</strong>. An information stealer (also known as SONE, StealerOne) written in PureBasic.</li>
</ul>
</div>
</div>
<div class="et_pb_module et_pb_text et_pb_text_91  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">
<p><img loading="lazy" loading="lazy" decoding="async" class="wp-image-1678 aligncenter size-full" src="https://quointelligence.eu/wp-content/uploads/2020/07/Figure-3.png" alt="" width="962" height="172" srcset="https://quointelligence.eu/wp-content/uploads/2020/07/Figure-3.png 962w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-3-300x54.png 300w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-3-150x27.png 150w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-3-768x137.png 768w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-3-480x86.png 480w" sizes="(max-width: 962px) 100vw, 962px" /></p>
<p style="text-align: center"><em>Figure 3 – Kill-Chain of Sighting 1</em></p>
</div>
</div>
<div class="et_pb_module et_pb_text et_pb_text_92  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">
<p>Consistent with <a href="https://quointelligence.eu/2020/01/the-chicken-keeps-laying-new-eggs-uncovering-new-gc-maas-tools-used-by-top-tier-threat-actors/">our earlier observation</a>, attacks relying on lite_more_eggs result in a variant of more_eggs dropped on the victim&#8217;s the system. In this case, neither TerraLoader nor more_eggs were digitally signed, and the observed more_eggs variant version is the older “2.0b”.</p>
</div>
</div>
<div class="et_pb_module et_pb_text et_pb_text_93  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">
<p><img loading="lazy" loading="lazy" decoding="async" class="wp-image-1679 aligncenter size-full" src="https://quointelligence.eu/wp-content/uploads/2020/07/Figure-4.png" alt="" width="2260" height="568" srcset="https://quointelligence.eu/wp-content/uploads/2020/07/Figure-4.png 2260w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-4-300x75.png 300w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-4-1024x257.png 1024w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-4-150x38.png 150w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-4-768x193.png 768w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-4-1536x386.png 1536w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-4-2048x515.png 2048w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-4-1080x271.png 1080w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-4-1280x322.png 1280w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-4-980x246.png 980w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-4-480x121.png 480w" sizes="(max-width: 2260px) 100vw, 2260px" /></p>
<p style="text-align: center"><em>Figure 4 – configuration of more_eggs delivered by the lite_more_eggs sample</em></p>
</div>
</div>
<div class="et_pb_module et_pb_text et_pb_text_94  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">
<p>We have observed three occurrences involving the same highlighted attack kill-chain of GC attributed tools, resulting specifically in the older “2.0b” variant of more_eggs. Although this activity is not distinct enough, we are temporally attributing these sightings to GC06.tmp.</p>
</div>
</div>
<div class="et_pb_module et_pb_text et_pb_text_95  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">
<h3>Sighting 2 &#8211; GC05: A New Campaign with Familiar Tactics, Techniques, and Procedures (TTPs)</h3>
<p>On 10 April, QuoIntelligence detected a new VenomLNK variant. The VenomLNK file is contained within a Zip archive; both themed as a financial document, and likely delivered to a targeted user as an email attachment or link. While the observed filenames indicate the exploitation of a financial theme, we cannot confirm the victimology at this time.</p>
</div>
</div>

<div class="et_pb_text_inner">
<p>The attack’s kill-chain involves an obfuscated JS scriptlet dropping a TerraLoader variant and decoy Microsoft Word document. While the decoy document appears in the screen on the user’s system, the TerraLoader is running and dropping a more_eggs variant. Finally, the more_eggs delivers a final payload of the information-stealer tracked by QuoItelligence as TerraStealer, two tools already attributed to the GC MaaS.</p>
</div>
</div>
<div class="et_pb_module et_pb_text et_pb_text_98  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">
<p><img loading="lazy" loading="lazy" decoding="async" class="wp-image-1683 aligncenter size-full" src="https://quointelligence.eu/wp-content/uploads/2020/07/Figure-6.png" alt="" width="962" height="312" srcset="https://quointelligence.eu/wp-content/uploads/2020/07/Figure-6.png 962w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-6-300x97.png 300w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-6-150x49.png 150w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-6-768x249.png 768w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-6-480x156.png 480w" sizes="(max-width: 962px) 100vw, 962px" /></p>

<div class="et_pb_module et_pb_text et_pb_text_99  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">
<p>Pivoting on our initial sample, we obtained additional VenomLNK files which are all similar except for the C2 URLs and contain slight modifications from earlier known variants. Further, we determined that our initial sighting was part of a campaign which likely began on 11 March through 14 April. Based on our observations, the campaign aligns with activities and TTPs we previously attributed to GC05; a threat actor we have tracked since September 2019 who leverages the GC MaaS extensively, especially VenomLNK, more_eggs, and TerraStealer.</p>
</div>
</div>
<div class="et_pb_module et_pb_text et_pb_text_100  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">
<h3>FIN6: A Familiar and Returning GC MaaS Customer</h3>
<p>In April, we processed two sightings of attacks sharing similar characteristics of previously observed activity attributed to the financially-motivated threat actor group known as FIN6. Since 2018, QuoIntelligence was able to attribute with high confidence the use of GC MaaS tools such as more_eggs, TerraLoader, and TerraTV to FIN6 campaigns. Based on our analysis of the new campaigns, it is evident that FIN6 remains a customer of the GC MaaS. Although FIN6 is known to primarily target the financial and retail sectors, we cannot confirm the victimology of these campaigns at this time.</p>
</div>
</div>
<div class="et_pb_module et_pb_text et_pb_text_101  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">
<h3>Sighting 3 &#8211; ‘Fake Job’ Spearphishing Delivering VenomLNK</h3>
<p>On 8 April, we became aware of a new variant of VenomLNK and its original Zip archive. Both filenames aligned with the theme for the known fake job campaign attributed to FIN6, by both researchers at <a href="https://securityintelligence.com/news/threat-actor-using-fake-linkedin-job-offers-to-deliver-more_eggs-backdoor/">IBM-X Force</a> and <a href="https://www.proofpoint.com/us/threat-insight/post/fake-jobs-campaigns-delivering-moreeggs-backdoor-fake-job-offers">Proofpoint</a>, conducted since at least the middle of 2018. The original Zip archive, named <em>assistant_buyer.zip</em>, contained the VenomLNK variant named <em>Job Offering.lnk</em>. During analysis, the C2 URL was not serving the next stage payload of the kill-chain. Based on our telemtry, the first alleged execution of the attack occurred on 7 and 8 April, suggesting the sighting was likely part of new activity. However, due to lack of further pieces of evidence on the kill-chain, we currently attribute the sighting to FIN6 with low confidence.</p>
</div>
</div>
<div class="et_pb_module et_pb_text et_pb_text_102  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">
<h3>Sighting 4 &#8211; TerraLoader Directly Injecting Metasploit’s Meterpreter</h3>
<p>On 27 April, QuoIntelligence detected a new variant of TerraLoader having a modified payload delivery mechanism which decrypts the included payload (shellcode) and loads it directly into memory. During analysis, we identified two DLLs in memory – one very likely OpenSSL and the other highly likely Meterpreter, which is a full-featured backdoor. The Meterpreter uses HTTPS to callback to an attacker-controlled asset. Further aligning with the detection timeframe, the TerraLoader variant included a kill-switch of year 2020 – a feature that disallows the execution of a malware sample beyond a hardcoded date, time, or year value. As we have already <a href="https://quointelligence.eu/2020/01/the-chicken-keeps-laying-new-eggs-uncovering-new-gc-maas-tools-used-by-top-tier-threat-actors/">noted</a>, the kill-switch is a common feature of the Operator’s arsenal aimed at enforcing his own <em>licensing</em> with his customers.</p>
</div>
</div>
<div class="et_pb_module et_pb_text et_pb_text_103  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">
<p><img loading="lazy" loading="lazy" decoding="async" class="wp-image-1684 aligncenter size-full" src="https://quointelligence.eu/wp-content/uploads/2020/07/Figure-7.png" alt="" width="810" height="256" srcset="https://quointelligence.eu/wp-content/uploads/2020/07/Figure-7.png 810w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-7-300x95.png 300w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-7-150x47.png 150w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-7-768x243.png 768w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-7-480x152.png 480w" sizes="(max-width: 810px) 100vw, 810px" /></p>
<p style="text-align: center"><em>Figure 7 –Sighting 4 &#8211; TerraLoader direct memory injection</em></p>
</div>
</div>
<div class="et_pb_module et_pb_text et_pb_text_104  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">
<p>Previously in April 2019, we identified FIN6 as the only GC MaaS customer using a variation of the approach described above. Further to the attribution of the April 2019 case, the involved C2 domain, registered in January 2019,  is also a domain we observed in attack activity we already attributed earlier, with high confidence to FIN6. In April 2020, we detected another attack with the same approach from 2019. The activity of all three cases are described as follows:</p>
<ul>
<li><strong>April 2019:</strong> Involves initially generating an apparent stager executable, likely with Metasploit tools, for use by TerraLoader to inject into another process and download Meterpreter.</li>
<li><strong>April 2020:</strong> Similarly, this activity involves a generated stager executable used by TerraLoader to inject into another process (wermgr.exe) and download the next stage payload, which is a Meterpreter.</li>
</ul>
</div>
</div>
<div class="et_pb_module et_pb_text et_pb_text_105  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">
<p><img loading="lazy" loading="lazy" decoding="async" class="wp-image-1687 aligncenter size-full" src="https://quointelligence.eu/wp-content/uploads/2020/07/Figure-8.png" alt="" width="926" height="196" srcset="https://quointelligence.eu/wp-content/uploads/2020/07/Figure-8.png 926w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-8-300x63.png 300w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-8-150x32.png 150w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-8-768x163.png 768w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-8-480x102.png 480w" sizes="(max-width: 926px) 100vw, 926px" /></p>
<p style="text-align: center"><em>Figure 8 – April 2019 &amp; 2020 &#8211; TerraLoader process injection </em></p>
</div>
</div>
<div class="et_pb_module et_pb_text et_pb_text_106  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">
<ul>
<li><strong>April 2020:</strong> Differently, this activity involves TerraLoader loading obfuscated shellcode directly into the memory of itself, already including the Meterpreter payload, and executing it. Both TerraLoader variants detected in April included a kill-switch of year 2020, indicating recent or ongoing activity.</li>
</ul>
<p>A reasonable hypothesis for the new approach of using obfuscated shellcode, instead of injecting into another process, could likely be to increase stealth and evade detection by security solutions such as Anti-Virus. As such, TerraLoader is known to be fully undetectable, so decrypting and executing code within the same memory space will increase the likelihood of being undetected by most Anti-Virus solutions.</p>
</div>
</div>
<div class="et_pb_module et_pb_text et_pb_text_107  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">
<h2>GC MaaS Toolset Updates</h2>
<h3>TerraLoader</h3>
<p>The TerraLoader variant observed in Sighting 2, spanning from 11 March to 14 April, contains some notable feature changes, which we previously observed only twice in December 2019. The new variant uses a different string de/obfuscation, brute-forcing implementation, and anti-analysis techniques.</p>
<p><strong>String de/obfuscation</strong></p>
<ul>
<li>The latest variants store strings RC4 (a stream cipher) encrypted as raw bytes and seems to entirely use the same stream cipher for decryption. In early variants, deobfuscation was achieved through XOR-decryption on strings stored as hex streams. </li>
</ul>
<p><strong>Brute-forcing Implementation</strong></p>
<ul>
<li>In new variants, only the first half of the string encryption key is stored in the malware. The second half of the string encryption key is brute-forced – calculated at runtime by counting up from zero until it is found. As soon as the bruteforcing is able to decrypt a specific ciphertext to a specific plaintext, which are both stored in the malware, the key is successfully found.</li>
<li>From an analysis perspective, earlier variants used XOR obfuscation which can be bypassed quickly, however, the latest variants use RC4 so the same bruteforce search for the actual key needs to be performed to successfully decrypt all strings.</li>
</ul>
<p><strong>Anti-analysis Techniques</strong></p>
<ul>
<li>Checks where in memory <em>ntdll.dll</em> (a Microsoft file that contains NT kernel functions) is loaded.</li>
<li>Checks hash of executable (exe) name against a whitelist (pre-calculated hashes) including <em>regsvr32.exe</em>, using ZwQueryInformationProcess.</li>
<li>Checks hash of loaded DLLs against a blacklist. (pre-calculated hashes)</li>
<li>Compares hash of Dynamic-link library (DLL) extension, expects <em>.ocx</em>, and exe name (expects <em>regsvr32.exe</em>) against pre-calculated hash values. To do so, Process Environment Block (PEB) is used to know where a process exists in memory.</li>
<li>Uses <em>NtQueryInformationProcess</em> to check if a debugger is present on the system.</li>
<li>Dynamic function address resolution continues to perform lookup by hash (CRC32), but additionally uses an XOR value to make direct hash value comparison impractical.</li>
</ul>
</div>
</div>
<div class="et_pb_module et_pb_text et_pb_text_108  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">
<h3>more_eggs</h3>
<p>On 29 April, we detected a new variant of TerraLoader which contains a msxml.exe (a Windows command line utility that invokes the Microsoft XML Parser for transformation) and new more_eggs version, “6.6b”, embedded in its <em>.data</em> section.</p>
</div>
</div>
<div class="et_pb_module et_pb_text et_pb_text_109  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">
<p style="text-align: center"><em><img loading="lazy" loading="lazy" decoding="async" class="wp-image-1688 aligncenter size-full" src="https://quointelligence.eu/wp-content/uploads/2020/07/Figure-9.png" alt="" width="2360" height="638" srcset="https://quointelligence.eu/wp-content/uploads/2020/07/Figure-9.png 2360w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-9-300x81.png 300w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-9-1024x277.png 1024w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-9-150x41.png 150w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-9-768x208.png 768w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-9-1536x415.png 1536w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-9-2048x554.png 2048w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-9-1080x292.png 1080w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-9-1280x346.png 1280w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-9-980x265.png 980w, https://quointelligence.eu/wp-content/uploads/2020/07/Figure-9-480x130.png 480w" sizes="(max-width: 2360px) 100vw, 2360px" /></em></p>
<p style="text-align: center"><em>Figure 9 – more_eggs configuration of the latest variant</em></p>
</div>
</div>
<div class="et_pb_module et_pb_text et_pb_text_110  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">
<p>The latest variant of more_eggs is 6.6b, one iteration above the last known version “6.6a”, was observed during the campaign from 11 March through 14 April. Besides the typical customized more_eggs configuration variables (version number <em>BV</em>, C2 address <em>Gate</em>, and part of the ciphering key used to encrypt C2 communications, <em>Rkey</em>), the latest variant contains two notable main code changes:</p>
<ul>
<li>Introduces minimum delay before executing or retrying an action.</li>
<li>Attempts to cleanup memory by assigning empty values to variables after using them. In general, it is not clear how effective this approach is in JavaScript; however, this does at least hinder a JavaScript debugger.</li>
</ul>
</div>
</div>
<div class="et_pb_module et_pb_text et_pb_text_111  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">
<h3>VenomLNK</h3>
<p><em>Sighting 3</em> utilized an updated variant of VenomLNK as an initial attack vector in a targeted campaign. We have observed VenomLNK used in various campaigns involving different infection chains.</p>
<p>Metadata analysis of the LNK file allows key information to be extracted about the direct link to another file and the execution process. In general, LNK files have a small file size but contain valuable information such as shortcut target file, file location and name, and the program that opens the target file.</p>
<p>The VenomLNK files obtained from the campaign were all similar and contain slight modifications from earlier known samples which are:</p>
<ul>
<li>Uses a new volume serial number: 0xcae82342. The Serial Number is dependent of the hard drive the LNK file was created on.</li>
<li>Evolution of the execution scheme: /v /c set &#8220;z1=times”. The command line input places the first variable in double quotes, which can often break detection-based security solutions.</li>
<li>Only uses the Local Path (C:\Windows\System32\cmd.exe) to the Windows command prompt, dissimilar from earlier variant which also included the Relative Path (&#8230;&#8230;..\Windows\System32\cmd.exe)</li>
</ul>
</div>
</div>
<div class="et_pb_module et_pb_text et_pb_text_112  et_pb_text_align_left et_pb_bg_layout_light">
<div class="et_pb_text_inner">
<h2>Conclusion</h2>
<p>The GC MaaS continues to offer a versatile catalog of attack tools and underlying C2 infrastructure to fulfill the entire attack kill-chain. The Operator continues to regularly evolve and improve the toolset within his service portfolio, and adapt new techniques over time, such as in the campaign leveraging TerraLoader to directly inject a payload into memory. We expect the MaaS will continue to prove its success and profitability, through at least its returning customers and the known top-tier e-crime threat actors who have utilized the available services. </p>
</div>
