		<h1>PsiXBot: The Evolution Of A Modular .NET Bot</h1>
<h2>Summary</h2>
<p>In this blog we will share our analysis of a modular piece of malware which is referred to by the author as PsiXBot. The malware first surfaced in 2017 but has recently undergone significant developments of its core and modules, which include the logging of keystrokes and stealing of Outlook and browser credentials. With these new developments done and the first large scale distributions observed in the wild, PsiXBot has officially made its debut in the malware ecosystem.</p>
<h2>Introduction</h2>
<p>Fox-IT actively monitors cyber criminal activity on a daily basis in order to proactively identify threats that are relevant to our customers. On the 21st of February 2019 we noticed <em>SmokeLoader</em>, a popular bot used to install additional malware on infected machines for a fee, push a task in order to distribute a .NET malware sample. Further research on the sample revealed a bot with a modular nature and capabilities, such as stealing data from infected hosts as well as receiving download &amp; execute tasks. Our interest was further peaked when the <a href="http://www.malware-traffic-analysis.net/2019/03/16/index.html">Spelevo Exploit Kit</a> started distributing the same malware on the 16th of March at which point we decided to further investigate this piece of malware, resulting in the findings below.</p>
<p>Having seen it evolve since 2017 to now getting out from beta versioning, we observe its being distributed by multiple infection vectors, such as exploit kits and malware loaders.</p>
<h2>Analysis</h2>
<p>During routine threat research activity, we stumbled upon a <a href="https://twitter.com/James_inthe_box/status/1035209707244244997">tweet</a> on what seemed to be an early version of the malware known as <em>PsiXBot</em>. In the same Twitter thread a <a href="https://twitter.com/mesa_matt/status/1035213536258080768">link</a> was also shared to a very early version of that same malware.</p>
<p>An overview of the versions are found below:</p>
<ul>
<li>Mid 2017: First version spotted in the wild (<em>SHA256: d2ee07bf04947cac64cc372123174900725525c20211e221110b9f91b7806332</em>);</li>
<li>August 2018: Updated version spotted (<em>SHA256: ce0e46fa1c5b463ed4a070a05594a79203ed2dd5df96cece9f875e2957fda4fa</em>);</li>
<li>Early 2019: The latest version is now being distributed via different infection vectors (<em>SHA256: ca30c42334fcc693320772b4ce1df26fe5f1d0110bc454ec6388d79dffea4ae8</em>).</li>
</ul>
<p>The illustration below displays the code structure of the versions: the first version to the left and the most recent one to the right.</p>
<p><a href="https://i0.wp.com/blog.fox-it.com/wp-content/uploads/2019/03/versions.png?ssl=1"><img data-recalc-dims="1" fetchpriority="high" decoding="async" data-attachment-id="2419" data-permalink="https://blog.fox-it.com/2019/03/27/psixbot-the-evolution-of-a-modular-net-bot/versions/" data-orig-file="https://i0.wp.com/blog.fox-it.com/wp-content/uploads/2019/03/versions.png?fit=1321%2C1438&amp;ssl=1" data-orig-size="1321,1438" data-comments-opened="0" data-image-meta="{&quot;aperture&quot;:&quot;0&quot;,&quot;credit&quot;:&quot;&quot;,&quot;camera&quot;:&quot;&quot;,&quot;caption&quot;:&quot;&quot;,&quot;created_timestamp&quot;:&quot;0&quot;,&quot;copyright&quot;:&quot;&quot;,&quot;focal_length&quot;:&quot;0&quot;,&quot;iso&quot;:&quot;0&quot;,&quot;shutter_speed&quot;:&quot;0&quot;,&quot;title&quot;:&quot;&quot;,&quot;orientation&quot;:&quot;0&quot;}" data-image-title="versions" data-image-description="" data-image-caption="" data-medium-file="https://i0.wp.com/blog.fox-it.com/wp-content/uploads/2019/03/versions.png?fit=276%2C300&amp;ssl=1" data-large-file="https://i0.wp.com/blog.fox-it.com/wp-content/uploads/2019/03/versions.png?fit=800%2C871&amp;ssl=1" src="https://i0.wp.com/blog.fox-it.com/wp-content/uploads/2019/03/versions.png?resize=800%2C871&#038;ssl=1" alt="" width="800" height="871" class="alignnone size-large wp-image-2419" /></a></p>
<p>Note that the name <em>PsiX</em> is derived from the name of the assembly. Taking a look to the PDB path, the same naming is again present: <code>G:\WORK\MONEY\BOT\NoName\PsiX\obj\Release\NoNameBot.pdb</code></p>
<p>Additionally, taking a look at the sample displayed in the center of the image above, the name &#8220;Radius&#8221; can be spotted as well. Part or whole name can be also observed among the C&amp;Cs such as: <code>radcall.bit</code>, <code>radbot.bit</code> and <code>rrradiusspace.bit</code>.</p>
<p>The differences we observed across the versions are mostly:</p>
<ul>
<li>New commands supported by adding different modules;</li>
<li>Encryption of the strings with AES;</li>
<li>Version number updated from <code>Beta 1.0.0</code> to <code>1.0.1</code>.</li>
</ul>
<p>The analysis below is centered on the most recent version.</p>
<p>As you could have already seen, the malware is written in .NET and it is not obfuscated. Typically it&#8217;s distributed within a dropper which hides the main payload. Once executed the <em>Main()</em> function is invoked. It first verifies that it is the only running instance by looking to a hardcoded mutex (for the sample analyzed: <code>gfdhfyf543543cdsdfsdf</code>), then it executes a loop to mimic sleep function before activating.</p>
<p>Most of the strings are encrypted with AES by a hardcoded string key. For this sample the key is: <code>1243hewqr8n1220g321&amp;amp;^*&amp;amp;^Tb0</code></p>
<p>The malware also checks the language settings of the victim, if the language is set to <code>ru-RU</code> (Russian) the malware will exit. For all other language settings the malware will continue its malicious activity.</p>
<p>An additional check is relating the filename to ensure it is matching the one configured. In instances where it is not, it proceeds with the installation process, which is done by invoking the <em>CopyEx</em> method via <em>WMI</em> and invoking the copied binary again via WMI. The installation path is: <code>Local\Microsoft\.exe</code> in the <code>%APPDATA%</code> folder.</p>
<p>After the installation the malware contacts the configured C&amp;Cs, that are initialized by the following code:</p>
<p>[code lang=c]<br />
public static string[] valid = new string[]<br />
{<br />
    &quot;pppoe.bit&quot;,<br />
    &quot;weather0.bit&quot;,<br />
    &quot;mygranny.bit&quot;,<br />
    &quot;six6.bit&quot;,<br />
    &quot;learncpp.bit&quot;<br />
};<br />
[/code]</p>
<p>In order to communicate with the <em>.bit</em> domains it uses hardcoded DNS servers (<code>193.37.213.223</code> for the sample used in this analysis). Upon the DNS resolution, it sends a ping to the C&amp;Cs in order to identify the first one that is up.</p>
<p>The bot reports to its C&amp;C some information gathered from the infected host. An example of the string used in the request is:</p>
<p>[code lang=text]<br />
action=call&amp;user_name=john&amp;bot_id=D63BAFF79F3A3504C70DC3298EE74C68&amp;av=N/A&amp;os_major=Microsoft Windows 7 Home Basic N &amp;permissions=User&amp;os_bit=64&amp;cpu=Intel(R) Core(TM) i7-6820HQ CPU @ 2.70GHz&amp;gpu=Standard VGA Graphics Adapter&amp;ram=2048&amp;hdd=C:12345/67890&amp;version=1.0.1&amp;user_group=Admin&amp;pc_net=4.0<br />
[/code]</p>
<p>The meaning of the parameters are:</p>
<ul>
<li><em>action</em>: the purpose of the request;</li>
<li><em>user_name</em>: username of the victim&#8217;s host;</li>
<li><em>bot_id</em>: unique string to identify the infected host;</li>
<li><em>av</em>: name of the AntiVirus software installed;</li>
<li><em>os_major</em>: name of the OS installed;</li>
<li><em>permissions</em>: username&#8217;s permissions;</li>
<li><em>os_bit</em>: OS architecture;</li>
<li><em>cpu</em>: CPU model;</li>
<li><em>gpu</em>: GPU model;</li>
<li><em>ram</em>: RAM available;</li>
<li><em>hdd</em>: HDD Serial Number;</li>
<li><em>version</em>: version number of the malware (latest one is 1.0.1);</li>
<li><em>user_group</em>: User group name the username is part of;</li>
<li><em>pc_net</em>: version of .NET framework installed.</li>
</ul>
<p>The data transfer is encrypted with RC4 using a hardcoded key which for this sample is <code>63a6a2eea47f74b9d25d50879214997a</code>. It is interesting to note that the author encrypted most of the strings, except the RC4 encryption key, C&amp;Cs and DNS servers which are in plain-text.</p>
<p>The C&amp;C answers with a JSON string, like the following one:</p>
<p>[code lang=c]<br />
{<br />
    result_code: [<br />
        {<br />
            &quot;result_code&quot;: &quot;200&quot;,<br />
        }<br />
    ]<br />
}<br />
[/code]</p>
<p>If the server returns a valid response, the malware sleeps for 95 seconds before requesting a new command to execute. This is done by sending the data:</p>
<p>[code lang=text]<br />
action=command&amp;bot_id=D63BAFF79F3A3504C70DC3298EE74C68<br />
[/code]</p>
<p>An example of a response is the following one:</p>
<p>[code lang=c]<br />
{<br />
  &quot;result_code&quot;: [<br />
    {<br />
      &quot;result_code&quot;: &quot;200&quot;<br />
    }<br />
  ],<br />
  &quot;commands&quot;: [<br />
    {<br />
      &quot;command_id&quot;: &quot;1485&quot;,<br />
      &quot;command_action&quot;: &quot;GetSteallerPasswords&quot;,<br />
      &quot;command_data&quot;: &quot;&quot;,<br />
      &quot;command_arg&quot;: &quot;&quot;<br />
    },<br />
    {<br />
      &quot;command_id&quot;: &quot;1486&quot;,<br />
      &quot;command_action&quot;: &quot;StartSchedulerModule&quot;,<br />
      &quot;command_data&quot;: &quot;&quot;,<br />
      &quot;command_arg&quot;: &quot;&quot;<br />
    }<br />
  ]<br />
}<br />
[/code]</p>
<p>In the example above the C&amp;C asks for the execution of two commands. The <code>command_action</code> value is the exact name of the method that must be invoked. The malware will resolve the method dynamically in accordance to this value. In order to avoid hardcoded strings, the author implemented an easy way to upgrade the malware: if the method name sent by the C&amp;C is not present in the malware during execution, the method/call is simply ignored.<br />
For one particular method invocation the malware uses a type named <em>SukaBlyat</em>, which is an offensive term used as Russian slang.</p>
<p>The two received commands are used in order to ask the C&amp;C server for additional modules. The data transfered requesting the module is:</p>
<p>[code lang=text]<br />
action=module&amp;bot_id=D63BAFF79F3A3504C70DC3298EE74C68&amp;module_action=SchedulerModule<br />
[/code]</p>
<p>Subsequently, the module is downloaded and executed, while sending the following command:</p>
<p>[code lang=text]<br />
action=result&amp;command_id=1485&amp;command_result=6E756C6C<br />
[/code]</p>
<p>The commands currently supported are:</p>
<p>[code lang=text]<br />
Download<br />
DownloadAndExecute<br />
Execute<br />
GetInstalledSoft<br />
GetKeylogs<br />
GetOutlook<br />
GetProcessesList<br />
GetScreenShot<br />
GetSteallerCookies<br />
GetSteallerPasswords<br />
StartAndroidModule<br />
StartBTC<br />
StartComplexModule<br />
StartKeylogger<br />
StartNewComplexModule<br />
StartSchedulerModule<br />
StopProcess<br />
[/code]</p>
<h2>Modules</h2>
<p>The modules available for this recent version of the bot are:</p>
<ul>
<li><em>BrowserModule (assembly name stMod.exe)</em>: used to dump passwords or cookies from a variety of browsers as well as from FileZilla FTP client. It accepts an argument to specify which is the data to be dumped: <code>-passes</code> for the password or <code>-cookies</code> for the cookies. The program returns a string with all the stolen information. It seems to be based on the QuasarRAT project;</p>
</li>
<li>
<p><em>BTCModule (assembly name LESHI.exe)</em>: accepts an argument and a cryptocurrency address. The supported address types are: <code>-btc</code>, <code>-ether</code>, <code>-ltc</code>, <code>-monero</code> and <code>-ripple</code>. Once such an address is configured, the program proceeds to monitor the clipboard every 3 seconds and verifies if the copied text is a valid address and of which type. If the check is successful the malware replaces the text with one of the configured wallet addresses;</p>
</li>
<li>
<p><em>ComplexModule (assembly name Client.exe)</em>: an old version of the open source rat QuasarRAT. In particular the <em>xclient</em> string (which is part of this module namespace) was present in a fork from 2016 (see <code>https://github.com/GeekGalaxy/QuasarRAT</code>). Also within the decompiled source code we retrieved the type name <em>QuasarClient</em>;</p>
</li>
<li>
<p><em>KeyLoggerModule (assembly name KeyLoggerModule.exe)</em>: uses the <em>SetWindowsHookEx</em> API in order to set a global hook and intercept keystrokes. The intercepted keys are saved in a file named <code>Logger.log</code>;</p>
</li>
<li>
<p><em>NewComplexModule (assembly name RemoteClient.exe)</em>: implements a remote desktop like program. It allows streaming the desktop user, interacting with it and starting the browser. The code does not seem to be anything publicly available;</p>
</li>
<li>
<p><em>OutlookModule (assembly name OutlookPasswordRecovery.dll)</em>:  dumps the Outlook passwords and returns a string with the information retrieved.</p>
</li>
<li>
<p><em>SchedulerModule (assembly name Scheduler.exe)</em>: used to ensure persistence. It just creates a scheduled task to run the bot each 60 seconds.</p>
</li>
</ul>
<h2>Distribution</h2>
<p>Typically, there are two ways of spreading such a strain of malware: infecting new unwitting victims and leveraging existing compromised systems. Fox-IT observes the PsiXBot actors are able to do both &#8211; delivering their malware via malspam campaigns or exploit kits (such as the Spelevo Exploit Kit) as well as using services offered on underground markets to load malware on (pre-infected) devices such as SmokeLoader.</p>
