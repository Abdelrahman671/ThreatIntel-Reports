          <h1>Kali Linux 2024.4 released with 14 new tools, deprecates some features</h1>
          <div class="cz-news-story-title-section">
            <div class="cz-news-title-left-area">
                            By <h6><a rel="author" href="https://www.bleepingcomputer.com/author/lawrence-abrams/" class="author"><span itemprop="author" itemscope itemtype="https://schema.org/Person"><span itemprop="name">Lawrence Abrams</span></span></a></h6>
            </div>
	    <meta itemprop="articleSection" content="Security"><meta itemprop="articleSection" content="Linux"><div class="cz-news-title-right-area">
              <ul><li class="cz-news-date">December 16, 2024</li>
                <li class="cz-news-time">04:33 PM</li>
                
                <li class="cz-news-comment"><a href="#comment_form">0</a></li>
              </ul></div>
          </div>
	  <div class="articleBody">
	   
<p style="text-align:center"><img alt="Kali Linux" height="900" src="https://www.bleepstatic.com/content/hl-images/2022/03/11/kali-linux.jpg" width="1600"></p>

<p>Kali Linux has released version 2024.4, the fourth and final version of 2024, and it is now available with fourteen new tools,&nbsp;numerous improvements, and deprecates&nbsp;some features.</p>

<p>Kali Linux is a distribution created for cybersecurity professionals and ethical hackers to perform penetration testing, ethical hacking, vulnerability research, security audits, and red team exercises.</p>

<p>With this final release of 2024, the Kali Team brings increased Raspberry Pi support, a new default Python version, and the end of i386 builds.</p>

<h2>Fourteen new tools in Kali Linux 2024.4</h2>

<p>It is not a new Kali release without new tools to play with, and&nbsp;Kali 2024.4&nbsp;does not disappoint.</p>

<p>The fourteen new tools released in this release are:</p>

<ul><li><a href="https://www.kali.org/tools/bloodyad/" target="_blank" rel="nofollow noopener">bloodyad</a> - Active Directory privilege escalation framework&nbsp;<em>(Submitted by&nbsp;<a href="https://gitlab.com/Arszilla" target="_blank" rel="nofollow noopener">@Arszilla</a>)</em></li>
	<li><a href="https://www.kali.org/tools/certi/" target="_blank" rel="nofollow noopener">certi</a> - Ask for certificates to ADCS and discover templates&nbsp;<em>(Submitted by&nbsp;<a href="https://gitlab.com/Arszilla" target="_blank" rel="nofollow noopener">@Arszilla</a>)</em></li>
	<li><a href="https://www.kali.org/tools/chainsaw/" target="_blank" rel="nofollow noopener">chainsaw</a> - Rapidly search and hunt through Windows forensic artefacts&nbsp;<em>(Submitted by&nbsp;<a href="https://gitlab.com/Arszilla" target="_blank" rel="nofollow noopener">@Arszilla</a>)</em></li>
	<li><a href="https://www.kali.org/tools/findomain/" target="_blank" rel="nofollow noopener">findomain</a> - Fastest and most complete solution for domain recognition&nbsp;<em>(Submitted by&nbsp;<a href="https://gitlab.com/Arszilla" target="_blank" rel="nofollow noopener">@Arszilla</a>)</em></li>
	<li><a href="https://www.kali.org/tools/hexwalk/" target="_blank" rel="nofollow noopener">hexwalk</a> - Hex analyzer, editor and viewer</li>
	<li><a href="https://www.kali.org/tools/linkedin2username/" target="_blank" rel="nofollow noopener">linkedin2username</a> - Generate username lists for companies on LinkedIn</li>
	<li><a href="https://www.kali.org/tools/mssqlpwner/" target="_blank" rel="nofollow noopener">mssqlpwner</a> - Interact and pwn MSSQL servers</li>
	<li><a href="https://www.kali.org/tools/openssh-ssh1/" target="_blank" rel="nofollow noopener">openssh-ssh1</a> - Secure SHell (SSH) client for legacy SSH1 protocol</li>
	<li><a href="https://www.kali.org/tools/proximoth/" target="_blank" rel="nofollow noopener">proximoth</a> - Control frame attack vulnerability detection tool&nbsp;<em>(Submitted by&nbsp;<a href="https://gitlab.com/TechnicalUserX" target="_blank" rel="nofollow noopener">@TechnicalUserX</a>)</em></li>
	<li><a href="https://www.kali.org/tools/python-pipx/" target="_blank" rel="nofollow noopener">python-pipx</a> - Execute binaries from Python packages in isolated environments</li>
	<li><a href="https://www.kali.org/tools/sara/" target="_blank" rel="nofollow noopener">sara</a> - RouterOS Security Inspector (Submitted by&nbsp;<a href="https://gitlab.com/casterbyte" target="_blank" rel="nofollow noopener">@casterbyte</a>)</li>
	<li><a href="https://www.kali.org/tools/web-cache-vulnerability-scanner/" target="_blank" rel="nofollow noopener">web-cache-vulnerability-scanner</a> - Go-based CLI tool for testing for web cache poisoning&nbsp;<em>(Submitted by&nbsp;<a href="https://gitlab.com/Arszilla" target="_blank" rel="nofollow noopener">@Arszilla</a>)</em></li>
	<li><a href="https://www.kali.org/tools/xsrfprobe/" target="_blank" rel="nofollow noopener">xsrfprobe</a> - An advanced Cross Site Request Forgery (CSRF/XSRF) audit and exploitation toolkit.</li>
	<li><a href="https://www.kali.org/tools/nmap/#zenmap" target="_blank" rel="nofollow noopener">zenmap</a> - The Network Mapper (nmap) front end (zenmap-kbx is no longer needed!)</li>
</ul><p>Kali says they also bumped the Linux Kernel to version 6.11.</p>

<h2>The end of the i386 kernel and images</h2>

<p>With the release of Kali 2024.4,&nbsp;there will no longer be any i386 images released as Debian, which Kali is based on,&nbsp;dropped support for 32-bit builds in October.</p>

<p>However, as 32-bit programs can still run on the &nbsp;x86-64 images, Kali Linux has not removed any i386 packages from the distro.</p>

<div class="fan_quote">
<p>"Despite being long obsolete, this architecture remained supported in software for years. 2019 was the year when major Linux distributions (<a href="https://fedoraproject.org/wiki/Changes/Stop_Building_i686_Kernels" target="_blank" rel="nofollow noopener">Fedora 31</a>&nbsp;&amp;&nbsp;<a href="https://wiki.ubuntu.com/BionicBeaver/ReleaseNotes" target="_blank" rel="nofollow noopener">Ubuntu</a>) started to drop it. Finally, in&nbsp;<a href="https://lists.debian.org/debian-release/2024/10/msg00064.html" target="_blank" rel="nofollow noopener">October 2024</a>, Debian stopped building a&nbsp;<code>i386</code>&nbsp;kernel (and OS images, as a consequence). Kali Linux, being based on Debian,&nbsp;<a href="https://www.kali.org/blog/end-of-i386-kernel-and-images/" target="_blank" rel="nofollow noopener">follow suit</a>:&nbsp;<strong>images and releases will no longer be created for this platform</strong>.</p>

<p>It&rsquo;s important to note that this is not an instant death for i386 though. This is not how architectures die. The i386 kernel and images are gone, however&nbsp;<em><strong>i386 packages in general are not removed</strong>&nbsp;from the repository</em>. It means that it&rsquo;s&nbsp;<strong>still possible to run i386 programs on a 64-bit system</strong>. Either directly via the package manager (APT supports installation of i386 packages on a amd64 system), or via&nbsp;<a href="https://hub.docker.com/r/kalilinux/kali-rolling/tags" target="_blank" rel="nofollow noopener">i386 Docker images</a>."</p>
&#10070; The Kali Team</div>

<h2>New default Python version&nbsp;3.12</h2>

<p>With this release, Python version 3.12 is now the default Python interpreter, and installing packages via&nbsp;pip command is disallowed by default.</p>

<p>As explained by the Kali Team in <a href="https://www.kali.org/blog/python-externally-managed/" target="_blank" rel="nofollow noopener">July 2023</a>, running pip as root to install system-wide packages can cause conflicts with a distro's built-in package manager, like <code>apt</code>.</p>

<p>"There is a major change with this new Python version:&nbsp;<strong>installing third-party Python packages via&nbsp;<code>pip</code>&nbsp;is now strongly discouraged and disallowed by default</strong>," explains the Kali Team.</p>

<p>"This change has been coming for a long time,&nbsp;<a href="https://www.kali.org/blog/python-externally-managed/" rel="nofollow noopener">we wrote about it 18 months ago already</a>, been given little reminders in each release blog post since and we gave another push about it in the&nbsp;<a href="https://www.kali.org/blog/kali-linux-2024-3-release/" rel="nofollow noopener">2024.3 release blog post</a>. Now it&rsquo;s finally effective."</p>

<p>For those who wish to use a program like pip, Kali now includes a <code>pipx&nbsp;</code>command that acts as a replacement. Instructions on using the pipx command can be <a href="https://www.kali.org/docs/general-use/python3-external-packages/" target="_blank" rel="nofollow noopener">found here</a>.</p>

<h2 id="deprecations-in-the-ssh-client-dsa-keys">SSH DSA keys are now&nbsp;deprecated</h2>

<p>Kali Linux 2024.4 comes with OpenSSH (9.8p1), which deprecates SSH DSA keys.</p>

<p>For those who need to support older keys for legacy systems, the new build includes the SSH1 client, which the Kali team says is an SSH client frozen at version 7.5.</p>

<p>However, the developers warn that if you are using tools that do not know about the <code>ssh1</code> command, they will no longer work properly with legacy systems requiring DSA keys.</p>

<p>"If you target very old SSH servers, you might need to use this client, assuming you are using the SSH client directly from the command-line," explains the Kali Team.</p>

<p>"However, if you use it&nbsp;<em>indirectly</em>&nbsp;(via some tool that uses SSH), it&rsquo;s possible that the tool does not know about the ssh1 command, so in practice you will lose support for DSA keys with this new Kali release. If you are in this situation, talk to us (via our&nbsp;<a href="https://discord.kali.org/" target="_blank" rel="nofollow noopener">our Discord server</a>&nbsp;or our&nbsp;<a href="https://bugs.kali.org/" target="_blank" rel="nofollow noopener">bug tracker</a>), and we might be able to help."</p>

<h2>Increased Raspberry Pi Imager support</h2>

<p>The Raspberry Pi Imager is a tool that allows you to find supported Raspberry Pi images and easily write them to a microSD card.</p>

<p>With this release of Kali Linux, you can now use the imager to pre-configure settings that will automatically be applied to the Kali Linux Raspberry Pi image when it's written to the microSD card.</p>

<p>These settings can include a custom hostname, login credentials, SSH keys, Wi-Fi network configuration, and locale settings, making it much easier to get the image set up as needed before it's ever booted up for the first time.</p>

<div style="text-align:center">
<figure class="image" style="display:inline-block"><img alt="Raspberry Pi Imager support" height="600" src="https://www.bleepstatic.com/images/news/linux/k/kali-linux/2024.4/raspberry-pi-imager-custom-image-2%5B1%5D.png" width="643"><figcaption><strong>Raspberry Pi Imager support</strong><br><em>Source: Kali</em></figcaption></figure></div>

<h2>Desktop changes</h2>

<p>With this release, Kali Linux includes the Gnome 47 desktop environment and greater support for color customization.</p>

<p>"We are excited to announce that the latest update to the GNOME Desktop, GNOME 47, is now available!," announced the Kali Team.</p>

<p>"This update brings numerous changes and desktop enhancements, but the most notable feature is the new support for accent color customization. You can now choose your favorite color for window and shell widgets, giving you more control over your desktop&rsquo;s look and feel."</p>

<p>The developers also introduced a new&nbsp;system-monitor panel extension and login theme.</p>

<div style="text-align:center">
<figure class="image" style="display:inline-block"><img alt="New Kali Linux 2024.4 login theme" height="600" src="https://www.bleepstatic.com/images/news/linux/k/kali-linux/2024.4/kali-login-theme.jpg" width="960"><figcaption><strong>New Kali Linux 2024.4 login theme</strong><br><em>Source: Kali</em></figcaption></figure></div>

<h2>How to get Kali Linux 2024.4</h2>

<p>To start using Kali Linux 2024.4, you can upgrade your existing installation, <a href="https://www.kali.org/get-kali/" target="_blank" rel="nofollow noopener">select a platform</a>,&nbsp;or&nbsp;<a href="https://cdimage.kali.org/kali-2024.4/" target="_blank" rel="nofollow noopener">directly download ISO images</a>&nbsp;for new installs and live distributions.</p>

<p>For those updating from a previous version,&nbsp;you can use the following commands to upgrade to the latest version.</p>

<pre>
<code class="language-xml">echo "deb http://http.kali.org/kali kali-rolling main contrib non-free non-free-firmware" | sudo tee /etc/apt/sources.list

sudo apt update &amp;&amp; sudo apt -y full-upgrade

cp -vrbi /etc/skel/. ~/

[ -f /var/run/reboot-required ] &amp;&amp; sudo reboot -f</code></pre>

<p>If you are running Kali on the Windows Subsystem for Linux, upgrade to WSL2 for a better experience, including the ability to use graphical apps.</p>

<p>You can check the WSL version used by Kali with the 'wsl -l -v' command in a Windows command prompt.</p>

<p>Once done upgrading, you can check if the upgrade was successful by using the following command:</p>

<pre>
<code>grep VERSION /etc/os-release</code></pre>

function cz_strip_tags(input, allowed) {
  allowed = (((allowed || '') + '')
    .toLowerCase()
    .match(/<[a-z][a-z0-9]*>/g) || [])
    .join(''); // making sure the allowed arg is a string containing only tags in lowercase (<a><b><c>)
  var tags = /<\/?([a-z][a-z0-9]*)\b[^>]*>/gi,
    commentsAndPhpTags = /<!--[\s\S]*?-->|<\?(?:php)?[\s\S]*?\?>/gi;
  return input.replace(commentsAndPhpTags, '')
    .replace(tags, function($0, $1) {
      return allowed.indexOf('<' + $1.toLowerCase() + '>') > -1 ? $0 : '';
    });
}
function cz_br2nl(str) {
	var regex = /<br\s*[\/]?>/gi;
    //var pure_str = str.replace(regex,"\n");
    var pure_str = str.replace(regex,"");
    return cz_strip_tags(pure_str,'');
}
