													<h2 id="introduction">Introduction</h2>
<p>In May 2024, we detected a campaign exclusively targeting victims in Italy. We were rather surprised by this, as cybercriminals typically select a broader target to maximize their profits. For example, a certain type of malware might target users in France and Spain, with the phishing emails written in both of the respective languages. However, for such a campaign, the malware&#8217;s code includes no particular checks to ensure it only runs in France and Spain. What sets this campaign apart is that, at various stages of the infection chain, checks are made to ensure that only Italian users are infected. This prompted us to investigate further and discover that the attackers were delivering a new RAT as the final payload that we dubbed SambaSpy.</p>
<h2 id="infection-chain">Infection chain</h2>
<p>When we started our investigation, we discovered two (slightly) different infection chains, as can be seen in the two figures below.</p>
<div id="attachment_113862" style="width: 980px" class="wp-caption aligncenter"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17152431/sambaspy_rat_targets_italian_users_08.png" class="magnificImage"><img loading="lazy" decoding="async" aria-describedby="caption-attachment-113862" class="size-full wp-image-113862" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17152431/sambaspy_rat_targets_italian_users_08.png" alt="SambaSpy infection chain 1" width="970" height="356" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17152431/sambaspy_rat_targets_italian_users_08.png 970w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17152431/sambaspy_rat_targets_italian_users_08-300x110.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17152431/sambaspy_rat_targets_italian_users_08-768x282.png 768w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17152431/sambaspy_rat_targets_italian_users_08-954x350.png 954w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17152431/sambaspy_rat_targets_italian_users_08-740x272.png 740w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17152431/sambaspy_rat_targets_italian_users_08-763x280.png 763w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17152431/sambaspy_rat_targets_italian_users_08-800x294.png 800w" sizes="(max-width: 970px) 100vw, 970px" /></a><p id="caption-attachment-113862" class="wp-caption-text">SambaSpy infection chain 1</p></div>

<p>Let&#8217;s discuss the second case in more detail as that infection chain is more elaborate. First, the victim receives an email from a German email address. The email was, however,  written in Italian and looked like it came from a legitimate Italian real estate company.</p>

<p>The email urges the receiver to view an invoice by clicking on an embedded link. Then the user is redirected to a malicious website which, during our analysis and according to other open sources, redirects to FattureInCloud, a legitimate Italian cloud solution used to store and manage digital invoices and quotes, where a legitimate invoice is displayed. Unfortunately, we were unable to view this file directly. Luckily, we found such an invoice on the urlscan.io website as can be seen in the figure below.</p>

<p>Getting redirected to a legitimate resource seemed like a dead end but our telemetry provided information that some users were redirected by the same URL to a malicious web server running behind ngrok. This server provided an HTML page containing JS code with comments in Brazilian Portuguese. It redirects users to a malicious OneDrive URL but only if they are running Edge, Firefox, or Chrome with their language set to Italian. If the users don&#8217;t pass these checks, they stay on the page.</p>

Users that match the target end up at a PDF document hosted on Microsoft OneDrive, luring the user to click on a hyperlink with the text &#8220;VISUALIZZA DOCUMENTO&#8221;, meaning &#8220;view document.&#8221; Finally, the URL redirects to a malicious JAR file hosted on MediaFire, which is either a dropper or a downloader.</p>
<h2 id="the-downloader">The downloader</h2>
<p>The downloader runs some checks to see whether or not it is running in a VM.It then checks that the environment is set to Italian again and if not, exits. If all the checks are passed, the downloader downloads the final stage and executes it.</p>

<h2 id="the-dropper">The dropper</h2>
<p>The dropper does exactly the same thing as a downloader besides downloading the malware, as it is embedded inside the resources of the JAR file.</p>
<h2 id="sambaspy">SambaSpy</h2>
<p>SambaSpy is a full-featured RAT developed in Java and obfuscated using the Zelix KlassMaster protector. The strings are encrypted and class names and methods are obfuscated to prevent analysis and detection. The malware exhibits a wide range of functionalities. Below, we provided a partial description. For the full report, please contact us.</p>
<p>The many malware&#8217;s features include:</p>
<ul>
<li>File system management;</li>
<li>Process management;</li>
<li>Uploading/downloading files;</li>
<li>Webcam control;</li>
<li>Logging keystrokes and controlling the clipboard;</li>
<li>Grabbing screenshots;</li>
<li>Remote desktop management;</li>
<li>Password stealing;</li>
<li>Loading additional plugins at runtime;</li>
<li>Starting a remote shell;</li>
<li>Interacting with the victim;</li>
</ul>
<p>Let&#8217;s look at some of these in more detail:</p>
<h3 id="loading-plugins">Loading plugins</h3>
<p>The plugin loading mechanism is fairly simple. A class is supplied to the RAT and loaded using URLClassLoader to open a file on the disk previously downloaded by the RAT. It then invokes the addURL() method in the loaded class. See the figure below for the complete code.</p>

<h3 id="logging-keystrokes-and-clipboard">Logging keystrokes and clipboard</h3>
<p>SambaSpy uses the JNativeHook library to log every keystroke from the victim. Then, upon key release, each keystroke event is sent to the C2. It also uses Java Abstract Window native libraries to steal or modify the victim&#8217;s clipboard content.</p>
<h3 id="stealing-browser-credentials">Stealing browser credentials</h3>
<p>The RAT can steal credentials from major browsers including Chrome, Edge, Opera, Brave, Iridium and Vivaldi.</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17153643/sambaspy_rat_targets_italian_users_09.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-large wp-image-113868" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17153643/sambaspy_rat_targets_italian_users_09-1024x335.png" alt="" width="1024" height="335" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17153643/sambaspy_rat_targets_italian_users_09-1024x335.png 1024w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17153643/sambaspy_rat_targets_italian_users_09-300x98.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17153643/sambaspy_rat_targets_italian_users_09-768x251.png 768w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17153643/sambaspy_rat_targets_italian_users_09-1071x350.png 1071w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17153643/sambaspy_rat_targets_italian_users_09-740x242.png 740w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17153643/sambaspy_rat_targets_italian_users_09-857x280.png 857w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17153643/sambaspy_rat_targets_italian_users_09-800x261.png 800w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17153643/sambaspy_rat_targets_italian_users_09.png 1414w" sizes="(max-width: 1024px) 100vw, 1024px" /></a></p>
<h3 id="remote-desktop-control">Remote desktop control</h3>
<p>SambaSpy implements a custom remote control system. It uses the Java Abstract Window library&#8217;s <a href="https://docs.oracle.com/javase/8/docs/api/java/awt/Robot.html" target="_blank" rel="noopener">Robot class</a> to control the mouse and keyboard from the system, along with the <a href="https://docs.oracle.com/javase/8/docs/api/java/awt/GraphicsDevice.html" target="_blank" rel="noopener">GraphicsDevice class</a> to provide a screen display under the attacker&#8217;s control.</p>
<h2 id="who-is-behind-the-campaign">Who is behind the campaign?</h2>
<p>We have yet to correlate the attackers behind this campaign to known actors. Nevertheless, there are indications that the threat actor speaks Brazilian Portuguese.</p>
<p>The language found in the malicious artifacts (i.e., code comments and error messages), and in the various sites utilized by the threat actor, was identified as Brazilian Portuguese as it features words not used in other dialects of the language.</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17153852/sambaspy_rat_targets_italian_users_05.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-113869" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17153852/sambaspy_rat_targets_italian_users_05.png" alt="" width="828" height="75" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17153852/sambaspy_rat_targets_italian_users_05.png 828w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17153852/sambaspy_rat_targets_italian_users_05-300x27.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17153852/sambaspy_rat_targets_italian_users_05-768x70.png 768w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17153852/sambaspy_rat_targets_italian_users_05-740x67.png 740w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17153852/sambaspy_rat_targets_italian_users_05-800x72.png 800w" sizes="(max-width: 828px) 100vw, 828px" /></a></p>
<p>We also know that the attacker is not exclusively interested in Italian targets, but has extended their activities to Spain and Brazil. During our investigation, we discovered other malicious domains attributed to the same attacker used in other campaigns. However, the infection chain involving these domains doesn&#8217;t include any language checks similar to those we observed in the Italian campaign.</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17151826/sambaspy_rat_targets_italian_users_01.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-113860" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17151826/sambaspy_rat_targets_italian_users_01.png" alt="" width="638" height="340" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17151826/sambaspy_rat_targets_italian_users_01.png 638w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17151826/sambaspy_rat_targets_italian_users_01-300x160.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/17151826/sambaspy_rat_targets_italian_users_01-525x280.png 525w" sizes="(max-width: 638px) 100vw, 638px" /></a></p>
<p>The domains were attributed to the same attacker because they were used to manage or distribute other variants of the downloader observed in Italy.</p>
<h2 id="conclusion">Conclusion</h2>
<p>There are several interesting aspects to this campaign. First, this campaign specifically targeted Italians by ensuring at multiple stages of the infection process that Italian was set as the language of the victim&#8217;s system. Second, there are various connections with Brazil, such as language artifacts in the code and domains targeting Brazilian users. This aligns with the fact that attackers from Latin America often target European countries with closely related languages, namely Italy, Spain, and Portugal.</p>
<p>Third, it is interesting to observe how the attackers based their distribution campaign on a legitimate document shared online, taking advantage of the brand of the company that generated the document. It is important to highlight that the company is completely unrelated to the campaign, as they are not directly involved in sending the messages or distributing the malware. They do not even appear to have published the invoice. Rather a customer presumably did so.</p>
<p>The attacker behind this campaign continuously changes its obfuscation methods, phishing emails, and C2 endpoints. Nevertheless, they reuse infrastructure-related second-level domains by creating new subdomains, which gives us a better overview of the attacker. This information can be used in future attribution efforts, while enhancing malware detection.</p>
