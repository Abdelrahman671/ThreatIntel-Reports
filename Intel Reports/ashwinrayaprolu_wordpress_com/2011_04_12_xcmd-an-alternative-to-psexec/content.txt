					<h1 class="entry-title">xCmd an Alternative to&nbsp;PsExec</h1>
		
		<div class="entry-meta">
						<span class="posted-on"><a href="https://ashwinrayaprolu.wordpress.com/2011/04/12/xcmd-an-alternative-to-psexec/" rel="bookmark"><time class="entry-date published" datetime="2011-04-12T17:30:14-07:00">April 12, 2011</time><time class="updated" datetime="2011-04-12T17:30:58-07:00">April 12, 2011</time></a></span> <span class="byline"><span class="author vcard"><a class="url fn n" href="https://ashwinrayaprolu.wordpress.com/author/ashwinrayaprolu/">ashwinrayaprolu</a></span></span>
			<span class="entry-categories"><a href="https://ashwinrayaprolu.wordpress.com/category/utilities/" rel="category tag">Utilities</a></span><span class="entry-tags"><a href="https://ashwinrayaprolu.wordpress.com/tag/alternative/" rel="tag">Alternative</a>, <a href="https://ashwinrayaprolu.wordpress.com/tag/psexec/" rel="tag">PsExec</a>, <a href="https://ashwinrayaprolu.wordpress.com/tag/windows-remote-execution/" rel="tag">Windows Remote Execution</a>, <a href="https://ashwinrayaprolu.wordpress.com/tag/xcmd/" rel="tag">xCmd</a></span>		</div><!-- .entry-meta -->
	</header><!-- .entry-header -->

	<div class="entry-content">
		<p>I recently had to use PsExec for one of my project and it had to integrated into program. I mean invocation of PsExec had to be done from a coding language. Then i faced problem of application getting hanged. To verify my code i did call PsExec from Java, Csharp, VB.net. None of them worked and program hanged on calling PSExec.</p>
<p>There is a problem with PsExec OutputStream when it is called from a programming language. None of the versions of PsExec looked good with this bug. So i had tpo find out an alternative,  Here is one good tool named XCmd by &#8220;<a href="http://www.codeguru.com/member.php/155/">Zoltan Csizmadia</a>&#8221;</p>
<h1>Execute Applications on Remote Systems</h1>
<div>
<ul>
<li><span style="font-size:15px;font-weight:bold;">Overview</span></li>
</ul>
</div>
<div id="display_content">
<p>This program allows you to execute applications on remote systems without installing any client <a id="itxthook0" rel="nofollow" href="http://www.codeguru.com/Cpp/I-N/network/remoteinvocation/article.php/c5433#">software</a>. You can start a command prompt or just execute a command or exe  on a remote machine. The only restriction is you must be an administrator 😦</p>
<p>Everybody knows the cool tools from Sysinternals (www.sysinternals.com). One of my favorites are PSEXEC, PSKILL and PSLIST,&#8230; 🙂<br />
I was always wonder how they could query every kind of information or execute commands on a remote machine without installing any client software.</p>
<h3>Features</h3>
<ul>
<li>With this program you can run as many remote commands on the remote machine as you want.  (PSEXEC supports only one remote command on the remote machine at the same time)</li>
<li>You can execute internal commands (dir,..) directly.<br />
xCmd.exe \\remote dir</li>
<li>You can start a light &#8220;telnet&#8221; connection with a remote machine without any telnet server<br />
xCmd.exe \\remote cmd</li>
</ul>
<h3>Usage</h3>
<p><span style="font-family:'Courier New';">xCmd v1.0 for NT4/2000 &#8211; executes commands remotely<br />
Freeware! 2001 Zoltan Csizmadia, zoltan_csizmadia@yahoo.com</span></p>
<p>Usage: xCmd.exe \\computer [options] command/exe arguments</p>
<p>Options:<br />
/D:directory         Set working directory<br />
Default: Remote &#8220;%SystemRoot%\System32&#8221;<br />
/IDLE                Idle priority class<br />
/NORMAL              Normal priority class<br />
/HIGH                High priority class<br />
/REALTIME            Realtime priority class<br />
/C                   Copy the specified program to the remote machine&#8217;s<br />
&#8220;%SystemRoot%\System32&#8221; directory<br />
Commands&#8217;s exe file must be absolute to local machine<br />
/USER:user           User for remote connection<br />
/PWD:{password|*}    Password for remote connection<br />
/NOWAIT              Don&#8217;t wait for remote process to terminate</p>
<p>Examples:<br />
xCmd.exe \\remote cmd<br />
xCmd.exe \\remote /user:administrator dir c:\ <a href="/"><br />
</a> xCmd.exe \\remote /user:somebody /pwd:* /d:d:\ test1.exe /p1 /p2<br />
xCmd.exe \\remote /c /user:somebody /pwd:* /d:d:\ test2.exe /whatever</p>
<p>&#8211; Input is passed to remote machine when you press the ENTER.<br />
&#8211; Ctrl-C terminates the remote process<br />
&#8211; Command and file path arguments have to be absolute to remote machine<br />
If you are using /c option, command exe file path must be absolute to<br />
local machine, but the arguments must be absolute to remote machine</p>
<h3>How does it work?</h3>
<ol>
<li>The xCmd.exe is console application and when you start it, the program will extract a xCmdSvc.exe from its resources.</li>
<li>xCmd.exe creates a service on the remote machine (that&#8217;s the reason, you must be an administrator</li>
<li>xCmd.exe starts the remote service (#2)</li>
<li>xCmd.exe and xCmdSvc.exe will communicate via named pipes</li>
<li>xCmd.exe send a packet to the service what to execute</li>
<li>xCmdSvc.exe starts the command and redirect stdout, stderr, stdin to 3 named pipes.</li>
<li>xCmd.exe listens these 3 named pipes (#6), redirect them to its stdout, stderr, stdin</li>
</ol>
</div>
<p>I have downloaded his code and converted to Visual Studio 2008 Solution and fixed a small bug which comes with iostream.h inclusion in new version of C++.</p>
