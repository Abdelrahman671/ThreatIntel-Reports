                        <h1 class="entry-title">Outbreak of Follina in Australia</h1>                            <div class="entry-meta">

<p>Our threat hunters have been busy searching for abuse of the recently-released zero-day remote code execution bug in Microsoft Office (<code>CVE-2022-30190</code>). As part of their investigations, they found evidence of a threat actor hosting malicious payloads on what appears to be an <code>Australian VOIP telecommunications provider</code> with a presence in the <code>South Pacific nation of Palau</code>.</p>



<p>Further analysis indicated that targets in <code>Palau</code> were sent malicious documents that, when opened, exploited this vulnerability, causing victim computers to contact the provider’s website, download and execute the malware, and subsequently become infected.</p>



<h1 class="wp-block-heading">Key Observations</h1>



<p>This threat was a complex multi-stage operation utilizing <code>LOLBAS</code> (Living off the Land Binaries And Scripts), which allowed the attacker to initialize the attack using the <code>CVE-2022-30190</code> vulnerability within the <code>Microsoft Support Diagnostic Tool</code>. This vulnerability enables threat actors to run malicious code without the user downloading an executable to their machine which might be detected by endpoint detection.</p>



<p>Multiple stages of this malware were signed with a legitimate company certificate to add additional legitimacy and minimize the chance of detection.</p>



<h1 class="wp-block-heading">First stage</h1>



<p>The compromised website, as pictured in the screenshot below, was used to host <code>robots.txt</code> which is an executable which was disguised as “robots.txt”. We believe the name was used to conceal itself from detection if found in network logs. Using the Diagnostics Troubleshooting Wizard (<code>msdt.exe</code>), this file &#8220;robots.txt&#8221; was downloaded and saved as the file (<code>Sihost.exe</code>) and then executed.</p>

<h1 class="wp-block-heading">Second Stage, Sihost.exe</h1>



<p>When the renamed &#8220;robots.txt&#8221; &#8211; &#8220;Sihost.exe&#8221; &#8211; was executed by msdt.exe it downloaded the second stage of the attack which was a loader with the hash <code>b63fbf80351b3480c62a6a5158334ec8e91fecd057f6c19e4b4dd3febaa9d447</code>. This executable was then used to download and decrypt the third stage of the attack, an encrypted file stored as ‘<code>favicon.svg</code>’ on the same web server.</p>



<h1 class="wp-block-heading">Third stage, favicon.svg</h1>



<p>After this file has been decrypted, it is used to download the fourth stage of the attack from <code>palau.voipstelecom.com[.]au.</code> These files are named <code>Sevntx64.exe</code> and <code>Sevntx.lnk</code>, which are then executed on the victims&#8217; machine.</p>

<h1 class="wp-block-heading">Fourth Stage, Sevntx64.exe and Sevntx64.lnk</h1>



<p>When the file is executed, it loads a <code>66kb</code> shellcode from the <code>AsyncRat</code> malware family; <code>Sevntx64.exe</code> is signed with the same compromised certificate as seen previously in &#8220;robots.txt&#8221;.</p>

<h1 class="wp-block-heading">Final Stage, AsyncRat</h1>



<p>When the executable is loaded, the machine has been fully compromised with AsyncRat; the trojan is configured to communicate with the server <code>palau[.]voipstelecom[.]com[.]au</code> on port <code>443</code>.&nbsp;</p>



<p>AsyncRat SHA256: </p>



<p><code>aba9b566dc23169414cb6927ab5368b590529202df41bfd5dded9f7e62b91479</code></p>

<p>We highly recommend Avast Software to protect against the latest threats, and Microsoft patches to protect your Windows systems from the latest <code>CVE-2022-30190</code> vulnerability.</p>
