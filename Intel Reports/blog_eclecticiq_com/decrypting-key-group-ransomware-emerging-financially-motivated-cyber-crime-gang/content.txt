<h2 style="text-align: left;">Executive Summary&nbsp;</h2>
<p style="text-align: left;"><br>The Key Group ransomware family was first revealed on January 6, 2023, continuing their operations since then. EclecticIQ researchers assess with high confidence, the Key Group ransomware gang is primarily a Russian speaking, financially motivated threat group using Telegram channel keygroup777Tg for the negotiation of ransoms.[<a href="#b1" rel="noopener">1</a>]&nbsp;<br><br>Key Group has an additional private (invite only) Telegram channel to share information between members such as doxing and offensive tool sharing. According to Telegram messages, EclecticIQ analysts assess with low confidence, that since June 29, 2023 threat actors are likely using NjRAT - a remote administration tool (RAT) - to remotely access victim devices. [<a href="#b2" rel="noopener">2</a>]&nbsp;<br><br>Key Group ransomware uses CBC-mode Advanced Encryption Standard (AES) to encrypt files and sends personally identifiable information (PII) of victim devices to threat actors. The ransomware uses the same static AES key and initialization vector (IV) to recursively encrypt victim data and change the name of encrypted files with the keygroup777tg extension. &nbsp;<br><br>EclecticIQ analysts assess that Key Group ransomware can be classified as a low-sophisticated threat actor. The ransomware samples contained multiple cryptographic mistakes that enabled EclecticIQ to create a decryption tool for this specific ransomware version built in August 03,2023. &nbsp;<a href="https://www.eclecticiq.com/hubfs/_blogs/corporate-blog/2023/Redline%20Stealer/Redline%20Stealer_Figure%201.png" rel="noopener"></a></p>
<!--more-->
<h2 style="text-align: left;">Who is Key Group? &nbsp;&nbsp;</h2>
<p><span>Key Group or KEYGROUP777, is a Russian-speaking cybercrime actor focusing on financial gain by selling Personal Identifying Information (PII) or initial access to compromised devices and obtaining ransom money. Key Group actors use private Telegram channels to share information about offensive tools and communicate with their members. The owner of the group uses the alias DARKZEUS.&nbsp;<br><br>According to researchers from BI.ZONE, older versions of Key Group ransomware used Chaos ransomware builder 4.0 to build the ransomware sample and target Russian victims. [3]&nbsp;<br></span></p>

<h2><span>Technical Analysis of Key Group Ransomware&nbsp;</span></h2>
<p style="font-weight: bold;">Encryption method</p>
<p><span>Key Group ransomware uses AES encryption, implemented in C#, using the RijndaelManaged class, which is a symmetric encryption algorithm.&nbsp;<br><br>Figure 5 shows the AES_Encrypt function. This function encryps victim data using the AES algorithm in Cipher Block Chaining (CBC) mode with a given static password. The password is derived from a key using the Password-Based Key Derivation Function 2 (PBKDF2) with a fixed salt.&nbsp;</span></p>
<span style="font-size: 0.9375rem;"><br></span><span style="font-size: 0.9375rem; background-color: transparent;">&nbsp;&nbsp;</span><br>
<p style="text-align: left;"><a href="https://www.eclecticiq.com/hubfs/_blogs/corporate-blog/2023/Key%20Group%20Ransomware/Key%20Group%20Ransomware_image4.png" rel="noopener"><img src="https://blog.eclecticiq.com/hs-fs/hubfs/_blogs/corporate-blog/2023/Key%20Group%20Ransomware/Key%20Group%20Ransomware_image4.png?width=682&amp;height=364&amp;name=Key%20Group%20Ransomware_image4.png" width="682" height="364" loading="lazy" alt="Key Group Ransomware_image4" style="height: auto; max-width: 100%; width: 682px; margin-left: auto; margin-right: auto; display: block;" srcset="https://blog.eclecticiq.com/hs-fs/hubfs/_blogs/corporate-blog/2023/Key%20Group%20Ransomware/Key%20Group%20Ransomware_image4.png?width=341&amp;height=182&amp;name=Key%20Group%20Ransomware_image4.png 341w, https://blog.eclecticiq.com/hs-fs/hubfs/_blogs/corporate-blog/2023/Key%20Group%20Ransomware/Key%20Group%20Ransomware_image4.png?width=682&amp;height=364&amp;name=Key%20Group%20Ransomware_image4.png 682w, https://blog.eclecticiq.com/hs-fs/hubfs/_blogs/corporate-blog/2023/Key%20Group%20Ransomware/Key%20Group%20Ransomware_image4.png?width=1023&amp;height=546&amp;name=Key%20Group%20Ransomware_image4.png 1023w, https://blog.eclecticiq.com/hs-fs/hubfs/_blogs/corporate-blog/2023/Key%20Group%20Ransomware/Key%20Group%20Ransomware_image4.png?width=1364&amp;height=728&amp;name=Key%20Group%20Ransomware_image4.png 1364w, https://blog.eclecticiq.com/hs-fs/hubfs/_blogs/corporate-blog/2023/Key%20Group%20Ransomware/Key%20Group%20Ransomware_image4.png?width=1705&amp;height=910&amp;name=Key%20Group%20Ransomware_image4.png 1705w, https://blog.eclecticiq.com/hs-fs/hubfs/_blogs/corporate-blog/2023/Key%20Group%20Ransomware/Key%20Group%20Ransomware_image4.png?width=2046&amp;height=1092&amp;name=Key%20Group%20Ransomware_image4.png 2046w" sizes="(max-width: 682px) 100vw, 682px"></a></p>
<p style="text-align: center;"><span style="font-size: 14px;">Figure 5 - AES_Encrypt function inside Key Group ransomware.</span>&nbsp;<br><br></p>
<p style="text-align: left;"><span style="font-size: 0.9375rem;">A brief overview of how the code works:&nbsp;<br><span style="font-weight: bold;"></span></span></p>
<ul>
<li style="text-align: left;"><span style="font-size: 0.9375rem;"><span style="font-weight: bold;">RijndaelManaged Configuration:</span> The code configures a RijndaelManaged object with a 256-bit key size and a 128-bit block size. Rijndael is the algorithm that AES is based on, and these are standard key and block sizes for AES. </span></li>
<li style="text-align: left;"><span style="font-size: 0.9375rem;"><span style="font-weight: bold;">Initialize Salt:</span> A predefined salt value is created. Salting is a technique used to add an additional layer of randomness to passwords. </span></li>
<li style="text-align: left;"><span style="font-size: 0.9375rem;"><span style="font-weight: bold;">Password Derivation: </span>The code uses the Rfc2898DeriveBytes class to derive the encryption key and IV from the provided static password and salt. The 1000 parameter specifies the number of iterations to use in the PBKDF2 algorithm, which helps make brute-force attacks more difficult. </span></li>
<li style="text-align: left;"><span style="font-size: 0.9375rem;"><span style="font-weight: bold;">Encryption:</span> The CryptoStream is used to perform the encryption, writing the encrypted data into the MemoryStream.&nbsp;</span></li>
</ul>
<p><span style="font-size: 0.9375rem;"><span data-contrast="auto" xml:lang="EN-GB" lang="EN-GB"><span>Figure </span><span>6</span><span> </span><span>shows</span><span> the </span></span><span data-contrast="auto" xml:lang="EN-GB" lang="EN-GB"><span>EncryptFile</span><span> </span></span><span data-contrast="auto" xml:lang="EN-GB" lang="EN-GB"><span>function</span><span>.
