    <div class="node-full__body blog-content__body">  <h3 paraeid="{93975240-51e7-4c76-ab16-19f7924bb3fc}{202}" paraid="582032292">What happened&nbsp;</h3>

<p paraeid="{93975240-51e7-4c76-ab16-19f7924bb3fc}{208}" paraid="861108519">Proofpoint researchers identified a campaign impersonating the British postal carrier Royal Mail delivering Prince ransomware. Prince is a ransomware variant freely available on GitHub with a “disclaimer” that it is only designed for educational purposes.&nbsp;</p>

<p paraeid="{93975240-51e7-4c76-ab16-19f7924bb3fc}{224}" paraid="1958806946">The campaign occurred in mid-September and targeted people in the UK and the U.S. The activity was low-volume and impacted a small number of organizations.&nbsp;&nbsp;</p>

<p paraeid="{93975240-51e7-4c76-ab16-19f7924bb3fc}{242}" paraid="1150671405">Notably, in most cases the messages appear to originate via contact forms posted on the target organizations’ websites, indicating the actor does not exclusively target organizations via email directly, but also from public contact forms.&nbsp;&nbsp;</p>

<p paraeid="{f3bbb704-b36e-46b2-84e9-a1973ac8c4d0}{19}" paraid="96819809">Additionally, it appears there are no decryption mechanisms once files are encrypted, and there is no capability for data exfiltration, thus the ultimate outcome of the attack would be destructive rather than typical ransomware.&nbsp;</p>

<h3 paraeid="{f3bbb704-b36e-46b2-84e9-a1973ac8c4d0}{45}" paraid="2078045694">Campaign details&nbsp;</h3>

<p paraeid="{f3bbb704-b36e-46b2-84e9-a1973ac8c4d0}{51}" paraid="617030391">The emails all contained a sender or reply-to Proton Mail email address, using different email addresses for each message and contact form submission.&nbsp;</p>

<p paraeid="{f3bbb704-b36e-46b2-84e9-a1973ac8c4d0}{63}" paraid="1062876863"><img alt="Figure 1" data-entity-type="file" data-entity-uuid="e0319dc3-1d3a-45e4-8376-fd44d1d3a6d9" height="785" src="/sites/default/files/inline-images/Screenshot%202024-09-27%20at%207.57.10%E2%80%AFAM.png" width="916" loading="lazy"></p>

<p paraeid="{f3bbb704-b36e-46b2-84e9-a1973ac8c4d0}{72}" paraid="1686513381"><em>Email lure impersonating Royal Mail.&nbsp;&nbsp;</em></p>

<p paraeid="{f3bbb704-b36e-46b2-84e9-a1973ac8c4d0}{78}" paraid="993384314">Royal Mail is a brand regularly impersonated by malicious actors. The company provides a <a href="https://www.royalmail.com/help/scam-examples" rel="noreferrer noopener" target="_blank">helpful list of common scams</a> to watch out for that abuse their brand.&nbsp;&nbsp;</p>

<p paraeid="{f3bbb704-b36e-46b2-84e9-a1973ac8c4d0}{89}" paraid="1820311478">Messages contained a unique PDF attachment that also impersonated Royal Mail. The PDF included a link that led to the download of a ZIP file hosted on Dropbox (ex. PACKAGE-0074752.zip).&nbsp;&nbsp;</p>

<p paraeid="{f3bbb704-b36e-46b2-84e9-a1973ac8c4d0}{121}" paraid="1884074016"><img alt="Figure 2" data-entity-type="file" data-entity-uuid="00508028-504f-4d84-878d-724609017295" height="419" src="/sites/default/files/inline-images/Screenshot%202024-09-27%20at%207.57.18%E2%80%AFAM.png" width="608" loading="lazy"></p>

<p paraeid="{f3bbb704-b36e-46b2-84e9-a1973ac8c4d0}{130}" paraid="1932385154"><em>PDF containing a Dropbox URL.&nbsp;</em></p>

<p paraeid="{f3bbb704-b36e-46b2-84e9-a1973ac8c4d0}{136}" paraid="180489385">The ZIP contained another password-protected ZIP file (ex: invoice.zip) and a text file (ex: privacy notice.txt) that contained the password needed to open the password-protected ZIP file.&nbsp;</p>

<p><img alt="Figure 3" data-entity-type="file" data-entity-uuid="40ad61ba-5671-48a6-a4db-db8fe9055db9" height="116" src="/sites/default/files/inline-images/Screenshot%202024-09-27%20at%207.57.30%E2%80%AFAM.png" width="702" loading="lazy"></p>

<p paraeid="{f3bbb704-b36e-46b2-84e9-a1973ac8c4d0}{159}" paraid="1230138921"><em>Content of downloaded ZIP file which contains second ZIP and “privacy notice” text file.&nbsp;</em></p>

<p paraeid="{f3bbb704-b36e-46b2-84e9-a1973ac8c4d0}{173}" paraid="1159520784">The second ZIP file contained a shortcut (LNK) file which, if executed, extracted and ran JavaScript extracted from the shortcut itself by performing the following functions:&nbsp;</p>

<ul>
	<li paraeid="{f3bbb704-b36e-46b2-84e9-a1973ac8c4d0}{191}" paraid="1780142108">It tried to locate the shortcut file in either the %temp% or the current directory and assigned the path to a variable via a “for loop”&nbsp;</li>
	<li paraeid="{f3bbb704-b36e-46b2-84e9-a1973ac8c4d0}{206}" paraid="1186615123">It used the “findstr” command to find the JavaScript code embedded in the shortcut, wrote it to a file (for clarity we will call it JS1.js) in %temp%, and executed it using WScript</li>
</ul>

<p><img alt="Figure 4" data-entity-type="file" data-entity-uuid="cd995fb2-8d4e-49ba-a396-b40b62a8fa85" height="106" src="/sites/default/files/inline-images/Screenshot%202024-09-27%20at%209.44.31%E2%80%AFAM.png" width="575" loading="lazy"></p>

<p paraeid="{f3bbb704-b36e-46b2-84e9-a1973ac8c4d0}{244}" paraid="627812366"><em>Example of “for loop” used in the shortcut.&nbsp;</em></p>

<p paraeid="{f3bbb704-b36e-46b2-84e9-a1973ac8c4d0}{250}" paraid="295318816"><img alt="Figure 5" data-entity-type="file" data-entity-uuid="3636a041-6130-4692-99b8-7adca6664426" height="629" src="/sites/default/files/inline-images/Screenshot%202024-09-27%20at%209.45.10%E2%80%AFAM.png" width="924" loading="lazy"></p>

<p paraeid="{f3bbb704-b36e-46b2-84e9-a1973ac8c4d0}{255}" paraid="1576628412"><em>Shortcut opened in Hex editor.&nbsp;</em></p>

<p paraeid="{a3ebf8bb-fea5-4f92-8936-2077ec386a73}{12}" paraid="964882833">The JavaScript, which is heavily obfuscated, performed the following actions:&nbsp;</p>

<ul>
	<li paraeid="{a3ebf8bb-fea5-4f92-8936-2077ec386a73}{22}" paraid="996447904">It wrote four files to %temp% directory with seemingly random names. For readability we will call them: PS1.ps1, PS2.ps1, PS3.ps1, and JS2.js.&nbsp;</li>
	<li paraeid="{a3ebf8bb-fea5-4f92-8936-2077ec386a73}{52}" paraid="1032897260">It used PowerShell to run PS1.ps1 and PS2.ps1&nbsp;</li>
</ul>

<p paraeid="{a3ebf8bb-fea5-4f92-8936-2077ec386a73}{62}" paraid="276923181">Example PowerShell command initiated by the JavaScript code:&nbsp;</p>

<p style="font-family: Courier; margin-left: 70px;
    margin-right: 217px;">C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe -ep bypass -nop -c "\PS1.ps1"</p>

<p paraeid="{a3ebf8bb-fea5-4f92-8936-2077ec386a73}{82}" paraid="820655142">PS1.ps1 was a highly obfuscated version of a well-known <a href="https://www.mdsec.co.uk/2018/06/exploring-powershell-amsi-and-logging-evasion/" rel="noreferrer noopener" target="_blank">AMSI Bypass</a>. In the sample analyzed, it was de-obfuscated to:&nbsp;</p>

<p style="font-family: Courier; margin-left: 70px;
    margin-right: 217px;">[Ref].Assembly.GetType('System.Management.Automation.Utils.Amsi').GetField('amsiInitFailed', 'NonPublic,Static').SetValue($null, $true)</p>

<p paraeid="{a3ebf8bb-fea5-4f92-8936-2077ec386a73}{123}" paraid="976645574">PS2.ps1 was a highly obfuscated version of a well-documented <a href="https://0x00-0x00.github.io/research/2018/10/31/How-to-bypass-UAC-in-newer-Windows-versions.html" rel="noreferrer noopener" target="_blank">Windows Connection Manager UAC bypass</a>. This essentially wrote an INF file and opened it with Windows Connection Manager (CMSTP) to invoke WScript to run the previously created JS2.js.&nbsp;&nbsp;</p>

<p paraeid="{a3ebf8bb-fea5-4f92-8936-2077ec386a73}{156}" paraid="746911879">JS2.js again ran the PS1.ps1 AMSI Bypass, and then ran PS3.ps1. PS3.ps1, which was again heavily obfuscated, did the following:&nbsp;</p>

<ul>
	<li paraeid="{a3ebf8bb-fea5-4f92-8936-2077ec386a73}{178}" paraid="116869193">Deleted all previously dropped files&nbsp;</li>
	<li paraeid="{a3ebf8bb-fea5-4f92-8936-2077ec386a73}{188}" paraid="902570365">Wrote three Base64 strings to the registry (keys varied between the scripts)&nbsp;</li>
	<li paraeid="{a3ebf8bb-fea5-4f92-8936-2077ec386a73}{202}" paraid="926124213">Used PowerShell to create a Scheduled Task that ran every 20 minutes, but only if the computer was connected to power and had been idle for 15 minutes, meaning the first run was, at the earliest, after 20 minutes&nbsp;</li>
</ul>

<p paraeid="{a3ebf8bb-fea5-4f92-8936-2077ec386a73}{232}" paraid="1067628221">The Scheduled Task ran an encoded PowerShell command that:&nbsp;</p>

<ul>
	<li paraeid="{a3ebf8bb-fea5-4f92-8936-2077ec386a73}{242}" paraid="2018398224">Created an AES decrypt function that used the AES key and Initialization Vector from the previously written registry keys&nbsp;</li>
	<li paraeid="{946ac298-362e-430a-ac35-3e2021023908}{17}" paraid="196716524">Used the decrypt function to decrypt the third Base64 encoded registry key, decoded the resulting Base64, which was compressed with Gzip, and extracted that data to a new Base64 string. This final string was decoded to bytes and loaded as a .NET assembly.&nbsp;</li>
</ul>

<p paraeid="{946ac298-362e-430a-ac35-3e2021023908}{31}" paraid="2099367466">The loaded .NET assembly, which was likely obfuscated with some form of the <a href="https://yck1509.github.io/ConfuserEx/" rel="noreferrer noopener" target="_blank">ConfuserEx</a> obfuscator that is available as open source, was a binary built with the open source ransomware builder “Prince Ransomware” <a href="https://github.com/SecDbg/Prince-Ransomware" rel="noreferrer noopener" target="_blank">available on GitHub</a>. This connection was first identified due to the observed sample downloading the same PNG from Imgur, and setting the PNG as the background, exactly as Prince Ransomware does in the configuration example on GitHub.&nbsp;</p>

<p paraeid="{946ac298-362e-430a-ac35-3e2021023908}{73}" paraid="879788746">While encrypting files, the ransomware displayed the Windows Update splash screen to the user, added the file extension ".womp" to the encrypted files, and wrote the ransom note “Decryption Instructions.txt” to the desktop.&nbsp;</p>

<p paraeid="{946ac298-362e-430a-ac35-3e2021023908}{101}" paraid="3966818"><img alt="Figure 6" data-entity-type="file" data-entity-uuid="6c8346f9-48d1-45b9-8865-778f433ec42c" src="/sites/default/files/inline-images/Screenshot%202024-09-27%20at%209.46.04%E2%80%AFAM.png" width="1274" height="1096" loading="lazy"></p>

<p paraeid="{946ac298-362e-430a-ac35-3e2021023908}{110}" paraid="771392270"><em>Ransom note displayed to user.&nbsp;</em></p>

<p paraeid="{946ac298-362e-430a-ac35-3e2021023908}{116}" paraid="1532380356">The ransom note claimed that files were exfiltrated and that the threat actor would automatically decrypt all files if 0.007 Bitcoins (roughly USD $400) was paid to a specified wallet. However, no such capability exists due to the following:&nbsp;</p>

<ul>
	<li paraeid="{946ac298-362e-430a-ac35-3e2021023908}{150}" paraid="395900080">Based on the characteristics observed in the campaign, there was no way for the threat actor to identify which victims have paid to perform the decryption because there were no unique identifiers between encryptions&nbsp;</li>
	<li paraeid="{946ac298-362e-430a-ac35-3e2021023908}{178}" paraid="1048777041">While Prince Ransomware builder does create a decryptor when building the ransomware binary, there is no listed threat actor contact or instructions to request for the files to be decrypted</li>
	<li paraeid="{946ac298-362e-430a-ac35-3e2021023908}{210}" paraid="794398194">No code in the detonation chain indicated that it was capable of exfiltrating data. The only network activity was the download of the desktop background&nbsp;</li>
</ul>

<p paraeid="{946ac298-362e-430a-ac35-3e2021023908}{232}" paraid="1130102764">Based on the lack of a link to determine which user has paid to have their files decrypted and which infected computer belongs to the user who paid, paired with the lack of communication instructions, makes this a destructive attack distributed by threat actors with likely no intention to decrypt any files, even if the victim paid. It is unclear if this is a mistake by the threat actor, or if the attack was designed to be destructive.&nbsp;</p>

<p paraeid="{35c9326c-be65-4287-8b33-78b52297cc34}{7}" paraid="557273823">Since all filenames, variables, and the final binary differ between the two observed shortcuts, it is likely that a custom builder service is used to construct the detonation chain from shortcut to the execution of the ransomware.&nbsp;</p>

<h3 paraeid="{35c9326c-be65-4287-8b33-78b52297cc34}{33}" paraid="1138742925">Attribution&nbsp;&nbsp;</h3>

<p paraeid="{35c9326c-be65-4287-8b33-78b52297cc34}{39}" paraid="1816678965">Proofpoint does not attribute this activity to a tracked threat actor. Because the ransomware is openly available on GitHub, it can be used and modified by various threat actors. According to the Prince Ransomware creator, SecDbg, Prince can bypass Windows Defender with modifications to its code not listed on GitHub. The creator states on GitHub:&nbsp;&nbsp;</p>

<p paraeid="{35c9326c-be65-4287-8b33-78b52297cc34}{65}" paraid="1208959699">“If, for whatever reason, bypassing Windows Defender is a priority for you, contact me on Telegram and I will accept payment for any changes you may require.”&nbsp;</p>

<p><img alt="Figure 7" data-entity-type="file" data-entity-uuid="6f561edf-8802-4ff8-9d8f-8b815e7e8632" height="576" src="/sites/default/files/inline-images/Screenshot%202024-09-27%20at%209.46.50%E2%80%AFAM.png" width="881" loading="lazy"></p>

<p paraeid="{35c9326c-be65-4287-8b33-78b52297cc34}{76}" paraid="1726365771"><em>Screenshot of GitHub account advertising Prince Ransomware.&nbsp;&nbsp;</em></p>

<p paraeid="{35c9326c-be65-4287-8b33-78b52297cc34}{82}" paraid="972475486">The developer also offers an information stealer called ThunderKitty on GitHub, where they advertise a “Paid version.” This suggests that SecDbg may offer a paid service for customizing the malware to bypass security measures. Given these circumstances, it seems plausible that SecDbg’s services could include a builder service for the detonation chain from shortcut to execution of the ransomware.&nbsp;</p>

<p paraeid="{35c9326c-be65-4287-8b33-78b52297cc34}{134}" paraid="762847075">GitHub and other open-source code repositories often host a variety of different malware and hacking tools. Despite claims from the accounts’ creators that the malware should only be used for educational purposes, threat actors often use these tools in malicious campaigns.&nbsp;</p>

<h3 paraeid="{35c9326c-be65-4287-8b33-78b52297cc34}{134}" paraid="762847075">Why it matters&nbsp;&nbsp;</h3>

<p paraeid="{35c9326c-be65-4287-8b33-78b52297cc34}{146}" paraid="1230297309">Although ransomware threats typically do not originate directly as an initial payload in email threats, it is still occasionally observed, especially from ransomware variants that are freely available either via repositories like GitHub or <a href="https://www.proofpoint.com/us/blog/threat-insight/security-brief-millions-messages-distribute-lockbit-black-ransomware" rel="noreferrer noopener" target="_blank">via leaked builders</a> of custom ransomware variants.&nbsp;&nbsp;</p>

<p paraeid="{35c9326c-be65-4287-8b33-78b52297cc34}{163}" paraid="69522536">Using contact forms to initially contact the recipient means the actors do not need to identify a contact email directly, and the email could be received by multiple different people who have access to a contact form recipient alias. That could include people who use work email addresses to receive mail forwarded from contact forms and websites unrelated to their job function or employer. Contact form malware delivery is a technique that has previously been used by other actors – TA578 for example <a href="https://www.proofpoint.com/us/blog/threat-insight/latrodectus-spider-bytes-ice" rel="noreferrer noopener" target="_blank">has consistently used</a> contact forms with complaint-related content to target organizations globally.&nbsp;&nbsp;</p>

<p paraeid="{35c9326c-be65-4287-8b33-78b52297cc34}{184}" paraid="898870412">Regardless of whether the email originates from a contact form or direct email, the Proton Mail sender or reply-to, paired with the Royal Mail-branded lure, should immediately warrant suspicion from the recipient. Organizations should train users to recognize such tactics and to report them to internal security operations when observed.&nbsp;&nbsp;</p>
