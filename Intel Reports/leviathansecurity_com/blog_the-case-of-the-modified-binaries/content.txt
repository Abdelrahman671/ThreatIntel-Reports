<div class="sqs-html-content">
  <h2 style="white-space:pre-wrap;">SUMMARY</h2><p class="" style="white-space:pre-wrap;">After creating and using a new exitmap module,&nbsp;I found downloaded binaries being patched through a Tor exit node in Russia.&nbsp; Tor is a wonderful tool for protecting the identity of journalists, their sources, and even regular users around the world; however, anonymity does not guarantee security. &nbsp;</p><h2 style="white-space:pre-wrap;">BACKGROUND</h2><p class="" style="white-space:pre-wrap;">At DerbyCon this year I gave a&nbsp;<a href="http://www.youtube.com/watch?v=LjUN9MACaTs" target="_blank">presentation</a>&nbsp;of my binary patching framework,&nbsp;<a href="https://github.com/secretsquirrel/the-backdoor-factory" target="_blank">BDF</a>.&nbsp; Many binaries are hosted without any transport layer security encryption. Some binaries are&nbsp;signed to prevent modification, but most are not. During that presentation, I talked about the MITM patching of binaries during download, and showed how easy it was using&nbsp;<a href="https://github.com/secretsquirrel/BDFProxy" target="_blank">BDFProxy</a>. I also mentioned that similar techniques are probably already in use on the Internet.</p><p class="" style="white-space:pre-wrap;">I had only circumstantial evidence until recently.</p><h2 style="white-space:pre-wrap;">MICROSOFT UPDATES ERROR</h2><p class="" style="white-space:pre-wrap;">I tested BDFProxy against&nbsp;a number of binaries and update processes, including Microsoft Windows Automatic updates.&nbsp; The good news is that if an entity is actively patching Windows PE files for Windows Update, the update verification process detects it, and you will receive error code 0x80200053. &nbsp;</p>

Very possible.</p></li><li><p class="" style="white-space:pre-wrap;">The file was patched during download. Improbable.</p></li><li><p class="" style="white-space:pre-wrap;">MS certificate verification is broken.
