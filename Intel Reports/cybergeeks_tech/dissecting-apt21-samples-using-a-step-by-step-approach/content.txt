<p style="font-size:25px">Summary</p>



<p>In this blog post we&#8217;re presenting a detailed analysis of 2 malicious files (a backdoor known as &#8220;Travelnet&#8221;) linked to an APT (Advanced Persistent Threat) actor called APT21.</p>



<p>APT21 , also known as Zhenbao or Hammer Panda, is a group of suspected state sponsored hackers of Chinese origin.</p>



<p>According to multiple online sources, that we have referenced in the article, APT 21 historically targeted the Russian government and groups which seek greater autonomy or independence from China, such as those from Tibet or Xinjiang.</p>



<p> The first file is a dropper used to register a malicious DLL (NetTraveler trojan) as a service. The main purpose of the trojan is to gather information about the environment such as user name, host name, IP address of the host, Windows OS version, different configurations of the CPU, information about memory consumption, the list of processes. The malicious process is interested in .doc, .docx, .xls, .xlsx, .txt, .rtf, .pdf files on disk and also on USB drives and network shares in order to exfiltrate them. During the entire infection, multiple .ini configuration files are created and also the malware has the capability to download and execute additional files on the infected machine. The data is compressed using a custom Lempel-Ziv-based algorithm and encoded with a modified Base64 algorithm before it will be exfiltrated to the Command and Control server.</p>

RegQueryValueExA API is used to retrieve the type and data for netsvcs (svchost.exe) associated with &#8220;HKEY_LOCAL_MACHINE\SOFTWARE\WOW6432Node\Microsoft\Windows NT\CurrentVersion\Svchost&#8221;:</p>

<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" decoding="async" width="548" height="531" src="https://cybergeeks.tech/wp-content/uploads/2020/11/68.jpg" alt="" class="wp-image-428" srcset="https://cybergeeks.tech/wp-content/uploads/2020/11/68.jpg 548w, https://cybergeeks.tech/wp-content/uploads/2020/11/68-300x291.jpg 300w" sizes="auto, (max-width: 548px) 100vw, 548px" /><figcaption>Figure 77</figcaption></figure></div>



<p>The same request as in Figure 74 is performed once more in order to keep the server in the loop for every new step.</p>



<p>Case 3: (RESET)</p>



<p>The following files are deleted: enumfs.ini, dnlist.ini, &#8220;C:\WINDOWS\system32\udidx.ini&#8221;, uenumfs.ini and stat_t.ini. Same request displayed in Figure 74 is used to contact the C2 server (this step is done in every case).</p>



<p>Case 4: (UPLOAD)</p>



<p>This case is identical to Case 2 (UPDATE) with the difference that no files/registry keys are deleted. </p>



<p>After the execution flow passes all cases, the process sleeps for 60 seconds and then it goes back in the loop.</p>



<p><strong>Thread2 activity</strong></p>



<p>RegisterClassA function is used to register a window class for use in CreateWindow/CreateWindowEx calls, it creates a windows using CreateWindowExA (windows class name is &#8220;NTMainWndClass&#8221;, 0x80000000 &#8211; WS_POPUP style). Also, the window procedure used in RegisterClassA API call (sub_10004535) is called 5 times as follows (one for each type of message): 0x81 (WM_NCCREATE), 0x83 (WM_NCCREATE), 0x01 (WM_CREATE), 0x05 (WM_SIZE) and 0x03 (WM_SIZE). We should also mention the following calls: ShowWindow (Sets the specified window&#8217;s show state), UpdateWindow (it sends a WM_PAINT message to the window), GetMessage (gets a message from the calling thread&#8217;s message queue) and TranslateMessage (translates messages into character messages):</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" decoding="async" width="574" height="743" src="https://cybergeeks.tech/wp-content/uploads/2020/11/69.jpg" alt="" class="wp-image-431" srcset="https://cybergeeks.tech/wp-content/uploads/2020/11/69.jpg 574w, https://cybergeeks.tech/wp-content/uploads/2020/11/69-232x300.jpg 232w" sizes="auto, (max-width: 574px) 100vw, 574px" /><figcaption>Figure 78</figcaption></figure></div>



<p>The malware is interested in WM_DEVICECHANGE (0x219) messages with a parameter of DBT_DEVICEARRIVAL (0x8000) which means that for example a new USB drive has been plugged in or a network shared folder is mounted on the system:</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" decoding="async" width="1024" height="228" src="https://cybergeeks.tech/wp-content/uploads/2020/11/70-1024x228.jpg" alt="" class="wp-image-432" srcset="https://cybergeeks.tech/wp-content/uploads/2020/11/70-1024x228.jpg 1024w, https://cybergeeks.tech/wp-content/uploads/2020/11/70-300x67.jpg 300w, https://cybergeeks.tech/wp-content/uploads/2020/11/70-768x171.jpg 768w, https://cybergeeks.tech/wp-content/uploads/2020/11/70.jpg 1059w" sizes="auto, (max-width: 1024px) 100vw, 1024px" /><figcaption>Figure 79</figcaption></figure></div>



<p>&#8220;USearch&#8221; and &#8220;UTypeLimit&#8221; values parsed from dnlist.ini are expected to be set to &#8220;True&#8221;, also &#8220;UAuto&#8221; value is &#8220;False&#8221; in dnlist.ini (this could indicate if the exfiltration of the targeted files should be automatically or not). The attacker is also interested in &#8220;Types&#8221; parameter (the targeted extensions) and we&#8217;ll see why in a bit. The idea is to scan each and every device inserted and also the network shares mounted on the host and create a &#8220;file system&#8221; structure in uenumfs.ini file (as it did in Thread1):</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" decoding="async" width="308" height="182" src="https://cybergeeks.tech/wp-content/uploads/2020/11/71.jpg" alt="" class="wp-image-433" srcset="https://cybergeeks.tech/wp-content/uploads/2020/11/71.jpg 308w, https://cybergeeks.tech/wp-content/uploads/2020/11/71-300x177.jpg 300w" sizes="auto, (max-width: 308px) 100vw, 308px" /><figcaption>Figure 80</figcaption></figure></div>



<p>As in the first case, this search will apply for every directory found on the drive, recursively. The process creates a &#8220;C:\Users\&lt;Username&gt;\AppData\Local\Temp\ntvba00.tmp\&#8221; directory and its attribute is set to hidden. The following file is also created: &#8220;C:\Windows\SysWOW64\uenumfs.ini&#8221; (the content of it will be similar to enumfs.ini):</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" decoding="async" width="1024" height="175" src="https://cybergeeks.tech/wp-content/uploads/2020/11/72-1024x175.jpg" alt="" class="wp-image-434" srcset="https://cybergeeks.tech/wp-content/uploads/2020/11/72-1024x175.jpg 1024w, https://cybergeeks.tech/wp-content/uploads/2020/11/72-300x51.jpg 300w, https://cybergeeks.tech/wp-content/uploads/2020/11/72-768x131.jpg 768w, https://cybergeeks.tech/wp-content/uploads/2020/11/72.jpg 1145w" sizes="auto, (max-width: 1024px) 100vw, 1024px" /><figcaption>Figure 81</figcaption></figure></div>



<p>For each file found on the USB drive/network share, the process compares it&#8217;s extension with the list mentioned before: .doc, .docx, .xls, .xlsx, .txt, .rtf, .pdf:</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" decoding="async" width="1024" height="139" src="https://cybergeeks.tech/wp-content/uploads/2020/11/73-1024x139.jpg" alt="" class="wp-image-435" srcset="https://cybergeeks.tech/wp-content/uploads/2020/11/73-1024x139.jpg 1024w, https://cybergeeks.tech/wp-content/uploads/2020/11/73-300x41.jpg 300w, https://cybergeeks.tech/wp-content/uploads/2020/11/73-768x104.jpg 768w, https://cybergeeks.tech/wp-content/uploads/2020/11/73.jpg 1057w" sizes="auto, (max-width: 1024px) 100vw, 1024px" /><figcaption>Figure 82</figcaption></figure></div>



<p>Let&#8217;s suppose that &#8220;C:\eula.1028.txt&#8221; (for the sake of simplicity) is a targeted file. The malware calculates a hash (MD5) of a combination between the filename and last modified timestamp (please note the initialize variables which correspond to MD5 algorithm):</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" decoding="async" width="1024" height="205" src="https://cybergeeks.tech/wp-content/uploads/2020/11/74-1024x205.jpg" alt="" class="wp-image-436" srcset="https://cybergeeks.tech/wp-content/uploads/2020/11/74-1024x205.jpg 1024w, https://cybergeeks.tech/wp-content/uploads/2020/11/74-300x60.jpg 300w, https://cybergeeks.tech/wp-content/uploads/2020/11/74-768x154.jpg 768w, https://cybergeeks.tech/wp-content/uploads/2020/11/74.jpg 1053w" sizes="auto, (max-width: 1024px) 100vw, 1024px" /><figcaption>Figure 83</figcaption></figure></div>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" decoding="async" width="388" height="295" src="https://cybergeeks.tech/wp-content/uploads/2020/11/75.jpg" alt="" class="wp-image-437" srcset="https://cybergeeks.tech/wp-content/uploads/2020/11/75.jpg 388w, https://cybergeeks.tech/wp-content/uploads/2020/11/75-300x228.jpg 300w" sizes="auto, (max-width: 388px) 100vw, 388px" /><figcaption>Figure 84</figcaption></figure></div>



<p>After the function is finished the following result will represent the hash (unique identifier) corresponding to eula.1028.txt file:</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" decoding="async" width="523" height="38" src="https://cybergeeks.tech/wp-content/uploads/2020/11/76.jpg" alt="" class="wp-image-438" srcset="https://cybergeeks.tech/wp-content/uploads/2020/11/76.jpg 523w, https://cybergeeks.tech/wp-content/uploads/2020/11/76-300x22.jpg 300w" sizes="auto, (max-width: 523px) 100vw, 523px" /><figcaption>Figure 85</figcaption></figure></div>



<p>Now &#8220;C:\eula.1028.txt&#8221; is copied to &#8220;C:\Users\&lt;Username&gt;\AppData\Local\Temp\ntvba00.tmp\U2007-11-07-12-00-5f7a78e7927532ba2a930ec8d47e252a.txt&#8221; (hidden file) &#8211; 2007 (year), 11 (month), 07 (day), 12 (hour), 00 (minute), 5f7a78e7927532ba2a930ec8d47e252a is the hash computed above (all values correspond to last modified timestamp):</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" decoding="async" width="1024" height="146" src="https://cybergeeks.tech/wp-content/uploads/2020/11/77-1024x146.jpg" alt="" class="wp-image-439" srcset="https://cybergeeks.tech/wp-content/uploads/2020/11/77-1024x146.jpg 1024w, https://cybergeeks.tech/wp-content/uploads/2020/11/77-300x43.jpg 300w, https://cybergeeks.tech/wp-content/uploads/2020/11/77-768x109.jpg 768w, https://cybergeeks.tech/wp-content/uploads/2020/11/77.jpg 1250w" sizes="auto, (max-width: 1024px) 100vw, 1024px" /><figcaption>Figure 86</figcaption></figure></div>



<p>The process creates &#8220;C:\Windows\SysWOW64\udidx.ini&#8221; file and will add all hashes computed as explained before:</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" decoding="async" width="1024" height="164" src="https://cybergeeks.tech/wp-content/uploads/2020/11/78-1024x164.jpg" alt="" class="wp-image-440" srcset="https://cybergeeks.tech/wp-content/uploads/2020/11/78-1024x164.jpg 1024w, https://cybergeeks.tech/wp-content/uploads/2020/11/78-300x48.jpg 300w, https://cybergeeks.tech/wp-content/uploads/2020/11/78-768x123.jpg 768w, https://cybergeeks.tech/wp-content/uploads/2020/11/78.jpg 1116w" sizes="auto, (max-width: 1024px) 100vw, 1024px" /><figcaption>Figure 87</figcaption></figure></div>



<p>Last modified timestamp of the new file is set as the value extracted from the initial file:</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" decoding="async" width="364" height="508" src="https://cybergeeks.tech/wp-content/uploads/2020/11/79.jpg" alt="" class="wp-image-441" srcset="https://cybergeeks.tech/wp-content/uploads/2020/11/79.jpg 364w, https://cybergeeks.tech/wp-content/uploads/2020/11/79-215x300.jpg 215w" sizes="auto, (max-width: 364px) 100vw, 364px" /><figcaption>Figure 88</figcaption></figure></div>
