    <section class="post-content-wrapper mt-5">
        <div class="post-content">
            <p>This post was authored by <a href="https://www.twitter.com/SecurityBeard/">Warren Mercer</a>, <a href="https://www.twitter.com/r00tbsd">Paul Rascagneres</a> and with contributions from Jungsoo An.</p><h2 id="executive-summary-earlier-this-year-talos-published-2-articles-concerning-south-korean-threats-the-first-one-was-about-the-use-of-a-malicious-hwp-document-which-dropped-downloaders-used-to-retrieve-malicious-payloads-on-several-compromised-websites-one-of-the-website-was-a-compromised-government-website-we-named-this-case-evil-new-years-the-second-one-was-about-the-analysis-and-discovery-of-the-rokrat-malware-">Executive Summary <br> Earlier this year, Talos published 2 articles concerning South Korean threats. The <a href="https://blog.talosintelligence.com/korean-maldoc">first one</a> was about the use of a malicious HWP document which dropped downloaders used to retrieve malicious payloads on several compromised websites. One of the website was a compromised government website. We named this case "Evil New Years". The <a href="https://blog.talosintelligence.com/introducing-rokrat">second one</a> was about the analysis and discovery of the ROKRAT malware.</h2><p>This month, Talos discovered a new ROKRAT version. This version contains technical elements that link the two previous articles. This new sample contains code from the two publications earlier this year:</p><ul><li>It contains the same reconnaissance code used;</li><li>Similar PDB pattern that the "Evil New Years" samples used;</li><li>it contains the same cloud features and similar copy-paste methods that ROKRAT used;</li><li>It uses cloud platform as C&amp;C but not exactly the same. This version uses pcloud, box, dropbox and yandex. <br></li></ul><p>We also discovered that this new version of ROKRAT shares code with Freenki, a downloader used in the FreeMilk campaign.</p><p>The campaign started, unsurprisingly, with a malicious HWP document. This document was alleged to be written by a lawyer who claims to represent the "Citizens' Alliance for North Korean Human Rights and Reunification of Korean Peninsula". It mentions a meeting of this group that took place the 1st of November at Seoul. Due to the content of this malicious document we can assume that the targets are interested by the situation in North Korea. This malicious document drops and executes a new version of ROKRAT.</p><h2 id="hwp-malicious-document-as-with-the-previous-rokrat-campaigns-we-described-the-infection-vector-used-with-this-actor-is-a-malicious-hwp-document-the-hwp-files-are-created-using-hangul-word-processor-a-popular-alternative-to-microsoft-office-for-south-korean-users-developed-by-hancom-here-is-a-screenshot-of-the-malicious-document-">HWP Malicious Document <br> As with the previous ROKRAT campaigns we described the infection vector used with this actor is a malicious HWP document. The HWP files are created using Hangul Word Processor, a popular alternative to Microsoft Office for South Korean users developed by Hancom. Here is a screenshot of the malicious document:</h2><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-oZY_XkhLCak/WhyBjcvv1DI/AAAAAAAAAMY/EtxUBxnlW0I7Zw5dvU4cVLCmMy3vVYzEwCLcBGAs/s640/image7.png" class="kg-image" alt loading="lazy"></figure><p><br> The malicious document mentions the "Community of North Korean human right and unification". We first observed his campaign during November 2017. The document was alleged to be written by a lawyer who has been representing the community known as ' ()'.</p><p>The purpose of the document is to arrange a meeting to discuss about items which are related to 'North Korean Human Rights Act' and 'Enactment of a law' which passed in last 2016 in South Korea.</p><p>Based on the meeting date (1st Nov 2017), this decoy document could be delivered to the stakeholders in the community '' by pretending to be a request to join the discussion for finding better ideas/ways to let more people be interested in their activity before Nov 2017.</p><p>The HWP file contains an OLE object named BIN0001.OLE. Once extracted and uncompressed (zlib), we obtain the following script:</p><pre><code>const strEncode = "TVqQAAMAAAAEAAAA//8AALgAAAAAAAAAQAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAA6AAAAA4fug4AtAnNIbgBTM0hV[...redacted...]AAAAAAAAAAAAAAAAAAAAAA="
DIM outFile
DIM base64Decoded
DIM shell_obj
SET shell_obj = CreateObject("WScript.Shell")
DIM fso
SET fso = CreateObject("Scripting.FileSystemObject")
outFile = "c:\ProgramData\HncModuleUpdate.exe"
base64Decoded = decodeBase64(strEncode)
IF NOT(fso.FileExists(outFile)) then
writeBytes outFile, base64Decoded
shell_obj.run outFile
END IF
WScript.Quit()
private function decodeBase64(base64)
DIM DM, EL
SET DM = CreateObject("Microsoft.XMLDOM")
SET EL = DM.createElement("tmp")
EL.DataType = "bin.base64"
EL.Text = base64
decodeBase64 = EL.NodeTypedValue
end function
private Sub writeBytes(file, bytes)
DIM binaryStream
SET binaryStream = CreateObject("ADODB.Stream")
binaryStream.Type = 1
binaryStream.Open
binaryStream.Write bytes
binaryStream.SaveToFile file, 1
End Sub
</code></pre><p>The purpose is to decode, using the base64 algorithm, the content of the strEncode variable. The decoded data is stored in the c:\ProgramData\HncModuleUpdate.exe file and executed. The binary is the ROKRAT dropper. The specific filename 'HncModuleUpdate' may fool a user into thinking this is a Hancom software.</p><h2 id="stage-1-dropper-the-purpose-of-the-dropper-is-to-extract-the-resource-named-sbs-this-resource-contains-malicious-shellcode-additionally-the-dropper-executes-a-new-cmd-exe-process-injects-the-extracted-resource-and-executes-it-the-code-injection-is-performed-by-the-virtualalloc-writeprocessmemory-and-createremotethread-apis-">Stage 1: Dropper <br> The purpose of the dropper is to extract the resource named SBS. This resource contains malicious shellcode. Additionally, the dropper executes a new cmd.exe process, injects the extracted resource and executes it. The code injection is performed by the VirtualAlloc(), WriteProcessMemory() and CreateRemoteThread() APIs:</h2><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-UK3O08XFp2k/WhyBximnWLI/AAAAAAAAAMc/s5N3fKQB__ks4xpcTOH6GtWfOMVzQS_TwCLcBGAs/s640/image6.png" class="kg-image" alt loading="lazy"></figure><p><br> Once executed, the shellcode will decoded a PE file, will load it in the memory of cmd.exe and finally will execute it. This payload is a new variant of ROKRAT.</p><p>Additionally, one of the analysed droppers displays a picture to the user:</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-UdRh6pJR0FU/WhyB6um-i6I/AAAAAAAAAMg/Rujz_rpXM8AIMWxHayfmA7v0sdKxbxYawCLcBGAs/s400/image20.png" class="kg-image" alt loading="lazy"></figure><p><br> The people in the pictures are about the Korean war and people related to independence troops during the "independence movement". The image on the top left comes from <a href="https://ko.wikipedia.org/wiki/%EB%B0%B1%EC%84%A0%EC%97%BD">Wikipedia</a>. The picture in the middle left comes from <a href="http://m.blog.daum.net/_blog/_m/articleView.do?blogid=0UCYA&amp;articleno=1545">this blog</a>. And the bottom left image comes from this <a href="http://m.ohmynews.com/NWS_Web/Mobile/at_pg.aspx?CNTN_CD=A0000939170#cb">news website</a>. The decoy image seems to be a set of public pictures.</p><h2 id="stage-2-rokrat">Stage 2: ROKRAT</h2><h3 id="similarities-with-the-evil-new-years-maldoc-this-variant-of-rokrat-contains-similar-code-with-the-evil-new-years-downloader-the-information-collected-during-the-reconnaissance-phase-is-similar-the-malware-uses-the-following-registry-key-to-get-the-machine-type-hklm-system-currentcontrolset-services-mssmbios-data-smbiosdata-the-system-manufacturer-value-is-used-to-identify-the-type-of-machine-here-is-the-graph-flow-of-the-evil-new-years-downloader-">Similarities With the "Evil New Years" MalDoc <br> This variant of ROKRAT contains similar code with the "Evil New Years" downloader. The information collected during the reconnaissance phase is similar. The malware uses the following registry key to get the machine type: HKLM\System\CurrentControlSet\Services\mssmbios\Data\SMBiosData. The "System manufacturer" value is used to identify the type of machine. Here is the graph flow of the "Evil New Years" downloader:</h3><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-ThSGoJ2fiL8/WhyCFM6M3SI/AAAAAAAAAMo/kS5m9vVHIhUAhkMPEhb-YiPH5-WISQ0hgCLcBGAs/s640/image10.png" class="kg-image" alt loading="lazy"></figure><p><br> The graph flow of the ROKRAT variant:</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-sshO1piH2Z4/WhyCKaO_YUI/AAAAAAAAAMs/T-ioks5MAWQP9yqvUiBKHLO88h3IIGBQgCLcBGAs/s640/image13.png" class="kg-image" alt loading="lazy"></figure><p><br> The graph flows are 99% similar. Additionally, the machine type is described with the following strings:</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-P3vttCd3--8/WhyCQUU2stI/AAAAAAAAAMw/yqLRBtpi1DUNwbOJ8WiPWn9nC3rtishggCLcBGAs/s640/image8.png" class="kg-image" alt loading="lazy"></figure><p><br> The code appears to be based on this <a href="http://www.rohitab.com/discuss/topic/35915-win32-api-to-get-system-information/?p%3D10075283">forum</a> post describing the use of the Win32 APIs used. The source code only considers the following type:<br></p><pre><code>default:   lpString = "(Other)";               break;
case 0x02: lpString = "(Unknown)";             break;
case 0x03: lpString = "(Desktop)";             break;
case 0x04: lpString = "(Low Profile Desktop)"; break;
case 0x06: lpString = "(Mini Tower)";          break;
case 0x07: lpString = "(Tower)";               break;

case 0x09: lpString = "(Laptop)";              break;
case 0x0A: lpString = "(Notebook)";            break;

</code></pre><p>Notice the () used by the ROKRAT author too. Some values are ignored as we can see from the <a href="http://www.dmtf.org/sites/default/files/standards/documents/DSP0134_2.7.0.pdf">SMBIOS documentation</a>:</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-0Ere0jsIaBM/WhyCWDThHsI/AAAAAAAAAM0/hGb9Yd4MiZ85bo8q1mlGuZ8VzRrFI690wCLcBGAs/s640/image19.png" class="kg-image" alt loading="lazy"></figure><p><br> The missing values are also omitted from the forum post.</p><p>Another similarity is the PDB path. The "Evil New Year" sample contained the following PDB path:</p><pre><code>e:\Happy\Work\Source\version 12\T+M\Result\DocPrint.pdb</code></pre><p>This new ROKRAT variant contains the following PDB:<br></p><pre><code>d:\HighSchool\version 13\2ndBD\T+M\T+M\Result\DocPrint.pdb</code></pre><p>We clearly have the similar pattern.<br></p><h3 id="anti-sandbox-this-rokrat-variant-contain-anti-sandbox-tricks-this-is-performed-by-checking-if-the-following-libraries-are-loaded-">Anti-Sandbox <br> This ROKRAT variant contain anti-sandbox tricks. This is performed by checking if the following libraries are loaded:</h3><ul><li>SbieDll.dll (sandboxie library)</li><li>Dbghelp.dll (Microsoft debugging tools)</li><li>Api_log.dll (threatAnalyzer / GFI SandBox)</li><li>Dir_watch.dll (threatAnalyzer / GFI SandBox)</li></ul><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-tMpcUMOjVY4/WhyCeLjiTtI/AAAAAAAAAM4/55qQDK6rByglGRQeSmTpw6kLM3ksCkJ9QCLcBGAs/s640/image4.png" class="kg-image" alt loading="lazy"></figure><h3 id="anti-debug-this-rokrat-version-contains-anti-debug-tricks-for-example-it-uses-the-following-nop-technique-">Anti-Debug<br> This ROKRAT version contains anti-debug tricks. For example it uses the following NOP technique:</h3><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/--2thdwNFFWk/WhyCxluop1I/AAAAAAAAANA/ZvScFYWEfmEX8V0XKU8Ab3e6RR9Nl3vFACLcBGAs/s640/image5.png" class="kg-image" alt loading="lazy"></figure><p><br> nop dword ptr [eax+eax+00h] is a 5 bytes NOP: 0x0F1F440000. But this opcode is not correctly supported by Immunity Debugger, the assembly is replaced by "???" in red in the screenshot:</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-FOQkjTQlNvA/WhyC3zsnVTI/AAAAAAAAANE/WmpI7gwqSBM75SG_ls-3cHjVzcKC50JTgCLcBGAs/s640/image2.png" class="kg-image" alt loading="lazy"></figure><h3 id="screenshots-feature-the-two-rokrat-versions-performed-screenshots-it-s-interesting-to-note-similarities-between-the-two-versions-especially-the-filename-of-the-saved-screenshot-here-is-the-april-rokrat-version-">Screenshots Feature <br> The two ROKRAT versions performed screenshots. It's interesting to note similarities between the two versions. Especially the filename of the saved screenshot, here is the April ROKRAT version:</h3><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-9U3juJSttOw/WhyC_xXCS7I/AAAAAAAAANI/QFDm3a7rfxQcmAM_ly8YHcSBuD4WW6FegCLcBGAs/s640/image17.png" class="kg-image" alt loading="lazy"></figure><p><br> And the code of the November version:</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/--n6yXqONkEs/WhyDFe5MlnI/AAAAAAAAANM/3XsNWR3OdBUI3VK6FQLIhAGsopYx4WT5ACLcBGAs/s640/image11.png" class="kg-image" alt loading="lazy"></figure><p><br> The pattern is exactly the same: %s%04X%04X.tmp. The two %04X are random values. And the %s contains a temporary path (obtained with GetTempPath()). In both sample, the string length is 0x12C (300). This part is clearly a copy-paste.<br></p><h3 id="browser-password-stealer-one-of-the-analysed-november-rokrat-samples-contained-a-browser-stealing-capability-the-malware-is-able-to-extract-the-stored-passwords-from-internet-explorer-chrome-and-firefox-for-chrome-and-firefox-the-malware-queries-the-sqlite-database-containing-the-url-username-and-password-">Browser Password Stealer <br> One of the analysed November ROKRAT samples contained a browser stealing capability. The malware is able to extract the stored passwords from Internet Explorer, Chrome and Firefox. For Chrome and Firefox, the malware queries the sqlite database containing the URL, username and password:</h3><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-1c4E_cI5198/WhyDL5Xq05I/AAAAAAAAANQ/pkyAlBQDvAgq3n9AX1XpGuRd8uLEQhbVACLcBGAs/s640/image3.png" class="kg-image" alt loading="lazy"></figure><p><br> Additionally, ROKRAT supports the Microsoft Vault mechanism. Vault was implemented in Windows 7, it contains any sensitive data (like the credentials) of Internet Explorer. Here is the initialization of the Vault APIs:</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-kgM45XXgNKw/WhyDSVHJD5I/AAAAAAAAANY/y18ZaMeY3XAiJJt0c2qUT88YUc-HQfPtACLcBGAs/s640/image14.png" class="kg-image" alt loading="lazy"></figure><p><br>The ROKRAT implementation is largely based on the <a href="https://www.codeproject.com/Articles/1167943/The-Secrets-of-Internet-Explorer-Credentials">following project</a>. This is a change of tactic for ROKRAT when compared with previous samples/versions. This time the actor is specifically targeting information which would be used for additional compromises and maybe even on potential personal accounts. The method used by the ROKRAT actors was also out of the ordinary as they embedded the whole SQLite library into their executable to allow the SQLite browsing attempts used for Firefox &amp; Google Chrome.</p><p>During our investigation, we discovered that the browser password stealer code is exactly the same as the code used during the <a href="https://researchcenter.paloaltonetworks.com/2017/10/unit42-freemilk-highly-targeted-spear-phishing-campaign/">FreeMilk</a> campaign described by Unit 42. In this article, the author already noticed C2 infrastructure overlap between FreeMilk and ROKRAT. In addition, we can add that some code overlap is present between the 2 samples:</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-wgeOLIL7OBQ/WhyDX-7egxI/AAAAAAAAANc/bj86udF7beAhD0Q3kWPDDLsJPNjWn-FBwCLcBGAs/s640/image12.png" class="kg-image" alt loading="lazy"></figure><p><br> On the left, we have the ROKRAT sample and on the right the FreeMilk sample. We can notice that in addition to the code, the author copy-pasted English typos such as "IE Registery".<br></p><h3 id="cloud-platforms-used-as-cc-finally-this-rokrat-version-uses-cloud-platforms-in-exactly-the-same-way-as-our-previous-analysis-this-time-the-author-did-not-use-social-network-platforms-but-different-cloud-providers-">Cloud Platforms Used As C&amp;C <br> Finally, this ROKRAT version uses cloud platforms in exactly the same way as our previous analysis. This time, the author did not use social network platforms, but different cloud providers:</h3><h4 id="pcloud">pcloud</h4><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-PrgljjxiAFo/WhyEcoSdFdI/AAAAAAAAAN4/bdK6iwGJ8moRLm9NDzv_5CVZx82QlV0NQCLcBGAs/s640/image18.png" class="kg-image" alt loading="lazy"></figure><h4 id="box">Box</h4><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-mEPlwuN1nzk/WhyEywpiFYI/AAAAAAAAAOA/WJFSGK8Ylp8IQE6Vru-WMQWd57IGHsBkwCLcBGAs/s640/image15.png" class="kg-image" alt loading="lazy"></figure><h4 id="dropbox">Dropbox</h4><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-DFU-dyBrdNg/WhyFAJdvf6I/AAAAAAAAAOM/oP6_ri4qfBQZa3-1tuSvysODAQyRZT73gCLcBGAs/s640/image9.png" class="kg-image" alt loading="lazy"></figure><h4 id="yandex">Yandex</h4><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-obpjJED7PA8/WhyFL04Jk0I/AAAAAAAAAOQ/5c1siYoQcdAt8UgR9YIuTXiawtZlN1DhACLcBGAs/s640/image16.png" class="kg-image" alt loading="lazy"></figure><h2 id="conclusion-this-campaign-shows-that-the-actor-behind-rokrat-is-still-active-based-on-the-pdb-it-could-be-the-13th-version-of-this-malware-this-actor-made-the-decision-only-to-use-legitimate-cloud-platforms-but-changed-some-from-the-last-incarnation-from-an-attacker-s-perspective-it-s-an-interesting-choice-the-flow-is-encrypted-by-default-with-https-and-the-malicious-flow-can-be-difficult-to-find-in-the-middle-of-legitimate-traffic-to-these-platforms-we-can-also-determine-that-the-actor-likes-to-use-code-already-available-on-the-internet-in-various-repositories-we-mentioned-throughout-this-post-ie-github-code-project-and-other-public-forums-">Conclusion <br> This campaign shows that the actor behind ROKRAT is still active. Based on the PDB, it could be the 13th version of this malware. This actor made the decision only to use legitimate cloud platforms, but changed some from the last incarnation. From an attacker's perspective it's an interesting choice, the flow is encrypted by default with HTTPS and the malicious flow can be difficult to find in the middle of legitimate traffic to these platforms. We can also determine that the actor likes to use code already available on the internet in various repositories we mentioned throughout this post ie; GitHub, Code Project and other public forums.</h2><p>Based on source code copy-paste, we remain highly confident that the author of ROKRAT is also behind, or working with those behind, the FreeMilk spear phishing campaign. This is further proven by the fact that ROKRAT shares code with Freenki downloader used in the FreeMilk campaign.</p><p>Moreover, the actor is always interested by the same pattern of targets, the decoy documents refer to precise elements related to the geopolitical situation between North and South Korea. Generally, the documents reference the Ministry of Unification or the situation of North Korean citizens. They frequently contain specific references to real meetings or conferences, showing a high knowledge of current events in North and South Korea.</p><p>Together this information helps us to understand the profile of the targeted systems and the interests of the threat actor.</p><h2 id="coverage-additional-ways-our-customers-can-detect-and-block-this-threat-are-listed-below-">Coverage <br>Additional ways our customers can detect and block this threat are listed below.</h2><figure class="kg-card kg-image-card"><img loading="lazy"></figure><p>Advanced Malware Protection (<a href="https://www.cisco.com/c/en/us/products/security/advanced-malware-protection">AMP</a>) is ideally suited to prevent the execution of the
