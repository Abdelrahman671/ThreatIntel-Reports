										</nav><!-- <content> --><p>Enables you to retrieve information about event logs and publishers. You can also use this command to install and uninstall event manifests, to run queries, and to export, archive, and clear logs.</p>
<h2 id="syntax">Syntax</h2>
<pre><code>wevtutil [{el | enum-logs}] [{gl | get-log} &lt;Logname&gt; [/f:&lt;Format&gt;]]
[{sl | set-log} &lt;Logname&gt; [/e:&lt;Enabled&gt;] [/i:&lt;Isolation&gt;] [/lfn:&lt;Logpath&gt;] [/rt:&lt;Retention&gt;] [/ab:&lt;Auto&gt;] [/ms:&lt;MaxSize&gt;] [/l:&lt;Level&gt;] [/k:&lt;Keywords&gt;] [/ca:&lt;Channel&gt;] [/c:&lt;Config&gt;]]
[{ep | enum-publishers}]
[{gp | get-publisher} &lt;Publishername&gt; [/ge:&lt;Metadata&gt;] [/gm:&lt;Message&gt;] [/f:&lt;Format&gt;]] 
[{im | install-manifest} &lt;Manifest&gt;] [/rf:&lt;Path&gt;] [/mf:&lt;Path&gt;] [/pf:&lt;Path&gt;] 
[{um | uninstall-manifest} &lt;Manifest&gt;] [{qe | query-events} &lt;Path&gt; [/lf:&lt;Logfile&gt;] [/sq:&lt;Structquery&gt;] [/q:&lt;Query&gt;] [/bm:&lt;Bookmark&gt;] [/sbm:&lt;Savebm&gt;] [/rd:&lt;Direction&gt;] [/f:&lt;Format&gt;] [/l:&lt;Locale&gt;] [/c:&lt;Count&gt;] [/e:&lt;Element&gt;]]

<h2 id="options">Options</h2>
<table>
<thead>
<tr>
<th>Option</th>
<th>Description</th>
</tr>
</thead>
<tbody>
<tr>
<td>/f:&lt;Format&gt;</td>
<td>Specifies that the output should be either XML or text format. If &lt;Format&gt; is XML, the output is displayed in XML format. If &lt;Format&gt; is Text, the output is displayed without XML tags. The default is Text.</td>
</tr>
<tr>
<td>/e:&lt;Enabled&gt;</td>
<td>Enables or disables a log. &lt;Enabled&gt; can be true or false.</td>
</tr>
<tr>
<td>/i:&lt;Isolation&gt;</td>
<td>Sets the log isolation mode. &lt;Isolation&gt; can be system, application or custom. The isolation mode of a log determines whether a log shares a session with other logs in the same isolation class. If you specify system isolation, the target log will share at least write permissions with the System log. If you specify application isolation, the target log will share at least write permissions with the Application log. If you specify custom isolation, you must also provide a security descriptor by using the <strong>/ca</strong> option.</td>
</tr>
<tr>
<td>/lfn:&lt;Logpath&gt;</td>
<td>Defines the log file name. &lt;Logpath&gt; is a full path to the file where the Event Log service stores events for this log.</td>
</tr>
<tr>
<td>/rt:&lt;Retention&gt;</td>
<td>Sets the log retention mode. &lt;Retention&gt; can be true or false. The log retention mode determines the behavior of the Event Log service when a log reaches its maximum size. If an event log reaches its maximum size and the log retention mode is true, existing events are retained, and incoming events are discarded. If the log retention mode is false, incoming events overwrite the oldest events in the log.</td>
</tr>
<tr>
<td>/ab:&lt;Auto&gt;</td>
<td>Specifies the log auto-backup policy. &lt;Auto&gt; can be true or false. If this value is true, the log will be backed up automatically when it reaches the maximum size. If this value is true, the retention (specified with the <strong>/rt</strong> option) must also be set to true.</td>
</tr>
<tr>
<td>/ms:&lt;MaxSize&gt;</td>
<td>Sets the maximum size of the log in bytes. The minimum log size is 1048576 bytes (1024KB) and log files are always multiples of 64KB, so the value you enter will be rounded off accordingly.</td>
</tr>
<tr>
<td>/l:&lt;Level&gt;</td>
<td>Defines the level filter of the log. &lt;Level&gt; can be any valid level value. This option is only applicable to logs with a dedicated session. You can remove a level filter by setting &lt;Level&gt; to 0.</td>
</tr>
<tr>
<td>/k:&lt;Keywords&gt;</td>
<td>Specifies the keywords filter of the log. &lt;Keywords&gt; can be any valid 64-bit keyword mask. This option is only applicable to logs with a dedicated session.</td>
</tr>
<tr>
<td>/ca:&lt;Channel&gt;</td>
<td>Sets the access permission for an event log. &lt;Channel&gt; is a security descriptor that uses the Security Descriptor Definition Language (SDDL). For more information about SDDL format, see the Microsoft Developers Network (MSDN) Web site (<a href="../../" data-linktype="relative-path">https://msdn.microsoft.com</a>).</td>
</tr>
<tr>
<td>/c:&lt;Config&gt;</td>
<td>Specifies the path to a configuration file. This option will cause log properties to be read from the configuration file defined in &lt;Config&gt;. If you use this option, you must not specify a &lt;Logname&gt; parameter. The log name will be read from the configuration file.</td>
</tr>
<tr>
<td>/ge:&lt;Metadata&gt;</td>
<td>Gets metadata information for events that can be raised by this publisher. &lt;Metadata&gt; can be true or false.</td>
</tr>
<tr>
<td>/gm:&lt;Message&gt;</td>
<td>Displays the actual message instead of the numeric message ID. &lt;Message&gt; can be true or false.</td>
</tr>
<tr>
<td>/lf:&lt;Logfile&gt;</td>
<td>Specifies that the events should be read from a log or from a log file. &lt;Logfile&gt; can be true or false. If true, the parameter to the command is the path to a log file.</td>
</tr>
<tr>
<td>/sq:&lt;Structquery&gt;</td>
<td>Specifies that events should be obtained with a structured query. &lt;Structquery&gt; can be true or false. If true, &lt;Path&gt; is the path to a file that contains a structured query.</td>
</tr>
<tr>
<td>/q:&lt;Query&gt;</td>
<td>Defines the XPath query to filter the events that are read or exported. If this option is not specified, all events will be returned or exported. This option is not available when <strong>/sq</strong> is true.</td>
</tr>
<tr>
<td>/bm:&lt;Bookmark&gt;</td>
<td>Specifies the path to a file that contains a bookmark from a previous query.</td>
</tr>
<tr>
<td>/sbm:&lt;Savebm&gt;</td>
<td>Specifies the path to a file that is used to save a bookmark of this query. The file name extension should be .xml.</td>
</tr>
<tr>
<td>/rd:&lt;Direction&gt;</td>
<td>Specifies the direction in which events are read. &lt;Direction&gt; can be true or false. If true, the most recent events are returned first.</td>
</tr>
<tr>
<td>/l:&lt;Locale&gt;</td>
<td>Defines a locale string that is used to print event text in a specific locale. Only available when printing events in text format using the <strong>/f</strong> option.</td>
</tr>
<tr>
<td>/c:&lt;Count&gt;</td>
<td>Sets the maximum number of events to read.</td>
</tr>
<tr>
<td>/e:&lt;Element&gt;</td>
<td>Includes a root element when displaying events in XML. &lt;Element&gt; is the string that you want within the root element. For example, <strong>/e:root</strong> would result in XML that contains the root element pair &lt;root&gt;.</td>
</tr>
<tr>
<td>/ow:&lt;Overwrite&gt;</td>
<td>Specifies that the export file should be overwritten. &lt;Overwrite&gt; can be true or false. If true, and the export file specified in &lt;Exportfile&gt; already exists, it will be overwritten without confirmation.</td>
</tr>
<tr>
<td>/bu:&lt;Backup&gt;</td>
<td>Specifies the path to a file where the cleared events will be stored. Include the .evtx extension in the name of the backup file.</td>
</tr>
<tr>
<td>/r:&lt;Remote&gt;</td>
<td>Runs the command on a remote computer. &lt;Remote&gt; is the name of the remote computer. The <strong>im</strong> and <strong>um</strong> parameters do not support remote operation.</td>
</tr>
<tr>
<td>/u:&lt;Username&gt;</td>
<td>Specifies a different user to log on to a remote computer. &lt;Username&gt; is a user name in the form domain\user or user. This option is only applicable when the <strong>/r</strong> option is specified.</td>
</tr>
<tr>
<td>/p:&lt;Password&gt;</td>
<td>Specifies the password for the user. If the <strong>/u</strong> option is used and this option is not specified or &lt;Password&gt; is *, the user will be prompted to enter a password. This option is only applicable when the <strong>/u</strong> option is specified.</td>
</tr>
<tr>
<td>/a:&lt;Auth&gt;</td>
<td>Defines the authentication type for connecting to a remote computer. &lt;Auth&gt; can be Default, Negotiate, Kerberos or NTLM. The default is Negotiate.</td>
</tr>
<tr>
<td>/uni:&lt;Unicode&gt;</td>
<td>Displays the output in Unicode. &lt;Unicode&gt; can be true or false. If &lt;Unicode&gt; is true then the output is in Unicode.</td>
</tr>

<h2 id="remarks">Remarks</h2>
<ul>
<li><p>Using a configuration file with the sl parameter</p>
<p>The configuration file is an XML file with the same format as the output of wevtutil gl &lt;Logname&gt; /f:xml. To shows the format of a configuration file that enables retention, enables autobackup, and sets the maximum log size on the Application log:</p>
<pre><code>&lt;?xml version=1.0 encoding=UTF-8?&gt;
&lt;channel name=Application isolation=Application
xmlns=https://schemas.microsoft.com/win/2004/08/events&gt;
&lt;logging&gt;
&lt;retention&gt;true&lt;/retention&gt;
&lt;autoBackup&gt;true&lt;/autoBackup&gt;
&lt;maxSize&gt;9000000&lt;/maxSize&gt;
&lt;/logging&gt;
&lt;publishing&gt;
&lt;/publishing&gt;
&lt;/channel&gt;
</code></pre>
</li>
</ul>
<h2 id="examples">Examples</h2>
<p>List the names of all logs:</p>
<pre><code>wevtutil el
</code></pre>
<p>Display configuration information about the System log on the local computer in XML format:</p>
<pre><code>wevtutil gl System /f:xml
</code></pre>
<p>Use a configuration file to set event log attributes (see Remarks for an example of a configuration file):</p>
<pre><code>wevtutil sl /c:config.xml
</code></pre>
<p>Display information about the Microsoft-Windows-Eventlog event publisher, including metadata about the events that the publisher can raise:</p>
<pre><code>wevtutil gp Microsoft-Windows-Eventlog /ge:true
</code></pre>
<p>Install publishers and logs from the myManifest.xml manifest file:</p>
<pre><code>wevtutil im myManifest.xml
</code></pre>
<p>Uninstall publishers and logs from the myManifest.xml manifest file:</p>
<pre><code>wevtutil um myManifest.xml
</code></pre>
<p>Display the three most recent events from the Application log in textual format:</p>
<pre><code>wevtutil qe Application /c:3 /rd:true /f:text
</code></pre>
<p>Display the status of the Application log:</p>
<pre><code>wevtutil gli Application
</code></pre>
<p>Export events from System log to C:\backup\system0506.evtx:</p>
<pre><code>wevtutil epl System C:\backup\system0506.evtx
</code></pre>
<p>Clear all of the events from the Application log after saving them to C:\admin\backups\a10306.evtx:</p>
<pre><code>wevtutil cl Application /bu:C:\admin\backups\a10306.evtx
</code></pre>
<p>Archive the specified (<em>.evtx</em>) log file in a self-contained format. A subdirectory (LocaleMetaData) is created and all locale-specific information is saved in that subdirectory:</p>
<pre><code>wevtutil archive-log "C:\backup\Application.evtx" /locale:en-us
</code></pre>
