<p><br>
Something that always fascinated me was running code directly from memory. From <a href="https://www.adlice.com/runpe-hide-code-behind-legit-process/">Process Hollowing</a> (aka RunPE) to <code>PTRACE</code> <a href="https://blog.xpnsec.com/linux-process-injection-aka-injecting-into-sshd-for-fun/">injection</a>. I had some success playing around with it in <code>C</code> in the past, without using any of the previous mentioned methods, but unfortunately the code is lost somewhere in the forums of <code>VXHeavens</code> (sadly no longer online) but the code was buggy and worked only with Linux 32bit systems (I wish I knew about <a href="http://man7.org/linux/man-pages/man3/shm_open.3.html">shm_open</a> back then, which is sort of an alternative for the syscall we are using in this post, mainly targeting older systems where <code>memfd_create</code> is not available).</p>

<h2 id="overview-and-code">Overview and code</h2>

<p>Recently, I have been trying to code in <code>assembly</code> a bit, I find it very interesting and I believe every developer should understand at least the basics of it. I chose <a href="https://flatassembler.net">FASM</a> as my assembler because I think it is very simple, powerful and I like its concepts (like same source, same output). More information about its design can be found <a href="https://flatassembler.net/docs.php?article=design">here</a>. Anyway, I have written a small tool, <code>memrun</code>, that allows you to run ELF files from memory using the <a href="http://man7.org/linux/man-pages/man2/memfd_create.2.html">memfd_create</a> syscall, which is available in Linux where kernel version is <code>&gt;= 3.17</code>.</p>

<p>What happens with <code>memfd_create</code> is that it acts like <code>malloc</code> syscall but will return a file descriptor that references an anonymous file (which does not exists in the disk) and we can pass it to <code>execve</code> and execute it from memory. There are a couple in-depth articles about it around the internet already so I will not get too deep into it. A nice one by <code>magisterquis</code> can be found at his <a href="https://magisterquis.github.io/2018/03/31/in-memory-only-elf-execution.html">page</a></p>

<p>The assembly code might look too big but there are some things we need to take care in this case that we don&rsquo;t need to when writing in a HLL like <code>Go</code> (as you can see in its example below). Also it&rsquo;s nice if you want to use the code for an exploit, you can just adjust the assembly instructions to your needs. Both examples are for <code>x86_64</code> only:</p>

	<span class="nx">_</span> <span class="p">=</span> <span class="nx">syscall</span><span class="p">.</span><span class="nf">Exec</span><span class="p">(</span><span class="nx">fdPath</span><span class="p">,</span> <span class="p">[]</span><span class="kt">string</span><span class="p">{</span><span class="nx">displayName</span><span class="p">},</span> <span class="kc">nil</span><span class="p">)</span>
<span class="p">}</span>

	<span class="nx">lenArgs</span> <span class="o">:=</span> <span class="nb">len</span><span class="p">(</span><span class="nx">os</span><span class="p">.</span><span class="nx">Args</span><span class="p">)</span>

<h2 id="see-it-in-action">See it in action</h2>

<p>Allow me to show it in action. Let&rsquo;s start by creating a simple target file in <code>C</code>, named <code>target.c</code>. The file will try to open itself for reading and if it can&rsquo;t, it will print a message forever every 5 seconds. We will execute it from memory:</p>

<div class="highlight"><pre class="chroma"><code class="language-c" data-lang="c"><span class="cp">#include</span> <span class="cpf">&lt;stdio.h&gt;</span><span class="cp">
</span><span class="cp">#include</span> <span class="cpf">&lt;unistd.h&gt;</span><span class="cp">
</span><span class="cp"></span>
<span class="kt">int</span> <span class="nf">main</span><span class="p">(</span><span class="kt">int</span> <span class="n">argc</span><span class="p">,</span> <span class="kt">char</span> <span class="o">**</span><span class="n">argv</span><span class="p">)</span>
<span class="p">{</span>
  <span class="n">printf</span><span class="p">(</span><span class="s">&#34;My process ID : %d</span><span class="se">\n</span><span class="s">&#34;</span><span class="p">,</span> <span class="n">getpid</span><span class="p">());</span>

  <span class="n">FILE</span> <span class="o">*</span><span class="n">myself</span> <span class="o">=</span> <span class="n">fopen</span><span class="p">(</span><span class="n">argv</span><span class="p">[</span><span class="mi">0</span><span class="p">],</span> <span class="s">&#34;r&#34;</span><span class="p">);</span>
  <span class="k">if</span> <span class="p">(</span><span class="n">myself</span> <span class="o">==</span> <span class="nb">NULL</span><span class="p">)</span> <span class="p">{</span>
        <span class="k">while</span><span class="p">(</span><span class="mi">1</span><span class="p">)</span> <span class="p">{</span>
                <span class="n">printf</span><span class="p">(</span><span class="s">&#34;I can&#39;t find myself, I must be running from memory!</span><span class="se">\n</span><span class="s">&#34;</span><span class="p">);</span>
                <span class="n">sleep</span><span class="p">(</span><span class="mi">5</span><span class="p">);</span>
        <span class="p">}</span>
  <span class="p">}</span> <span class="k">else</span> <span class="p">{</span>
        <span class="n">printf</span><span class="p">(</span><span class="s">&#34;I am just a regular boring file being executed from the disk...</span><span class="se">\n</span><span class="s">&#34;</span><span class="p">);</span>
  <span class="p">}</span>

  <span class="k">return</span> <span class="mi">0</span><span class="p">;</span>
<span class="p">}</span></code></pre></div>

<p>Now we build <code>target.c</code>:




<div class="alert ">
  
<pre>
$ gcc target.c -o target
</pre>

</div>

</p>

<p>We should also build our <code>FASM</code> or <code>Go</code> tool, I will use the assembly one here:




<div class="alert ">
  
<pre>
$ fasm memrun.asm
flat assembler  version 1.73.04  (16384 kilobytes memory, x64)
4 passes, 1221 bytes.
</pre>

</div>

</p>

<p>Running the file normally gives us this:




<div class="alert ">
  
<pre>
$ ./target
My process ID : 4944
I am just a regular boring file being executed from the disk...
</pre>

</div>

</p>

<p>But using <code>memrun</code> to run it will be totally different:




<div class="alert ">
  
<pre>
$ ./memrun target MASTER_HACKER_PROCESS_NAME_1337
My process ID : 4945
I can't find myself, I must be running from memory!
I can't find myself, I must be running from memory!
</pre>

</div>

</p>

<p>Furthermore, if you look for its pid with <code>ps</code> utility, this is what you get:




<div class="alert ">
  
<pre>
$ ps -f 4945
UID        PID  PPID  C STIME TTY      STAT   TIME CMD
guitmz    4945  4842  0 15:31 pts/0    S+     0:00 MASTER_HACKER_PROCESS_NAME_1337
</pre>

</div>

</p>

<p>Finally, let&rsquo;s check the process directory:




<div class="alert ">
  
<pre>
$ ls -l /proc/4945/{cwd,exe}
lrwxrwxrwx 1 guitmz guitmz 0 Mar 27 15:38 /proc/4945/cwd -> /home/guitmz/memrun/assembly
lrwxrwxrwx 1 guitmz guitmz 0 Mar 27 15:38 /proc/4945/exe -> /memfd: (deleted)
</pre>

</div>

</p>

<p>Note the <code>/memfd: (deleted)</code> part, no actual file in disk for this process :)</p>

<p>For those who know, this can be an interesting technique to run stealthy binaries in Linux, you can go even further by giving it a proper name (like a real Linux process) and detach it from the <code>tty</code> and change its <code>cwd</code> with some simple approches. Tip: <code>fork</code> is your friend :)</p>
