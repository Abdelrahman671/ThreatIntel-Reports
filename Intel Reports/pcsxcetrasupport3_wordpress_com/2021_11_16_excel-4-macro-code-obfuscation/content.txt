											<h2 class="entry-title"><a href="https://pcsxcetrasupport3.wordpress.com/2021/11/16/excel-4-macro-code-obfuscation/" rel="bookmark">Excel 4 macro code&nbsp;obfuscation</a></h2>					
					<div class="entry-meta">
						<span class="meta-prep meta-prep-author">Posted on</span> <a href="https://pcsxcetrasupport3.wordpress.com/2021/11/16/excel-4-macro-code-obfuscation/" title="6:19 pm" rel="bookmark"><span class="entry-date">November 16, 2021</span></a> <span class="meta-sep">by</span> <span class="author vcard"><a class="url fn n" href="https://pcsxcetrasupport3.wordpress.com/author/pcsxcetrasupport3/" title="View all posts by pcsxcetrasupport3">pcsxcetrasupport3</a></span>					</div><!-- .entry-meta -->

					<div class="entry-content">
						<p>This sample comes from a Twitter thread located <a href="https://twitter.com/fr0s7_/status/1458481326294769674">Here</a> by Frost @fr0s7_ and appears to be&nbsp; “BazarLoader”</p>
<p>Since this is a Xlsb file I usually just open it up in my Office 2010 Pro sandbox and then convert to Xlsm and unzip it so I can just view as xml. </p>
<p>The first thing I always do is take a quick look with a hex editor looking for anything of interest.</p>
<p><a href="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/binheader.png"><img width="690" height="643" title="BinHeader" style="border-width:0;padding-top:0;padding-right:0;padding-left:0;display:inline;background-image:none;" alt="BinHeader" src="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/binheader_thumb.png?w=690&#038;h=643" border="0"></a></p>
<p>As we can see from the first 2 bytes we have a “PK” or zip file format.</p>
<p>Once we “UnZip” the file and navigate to the xl folder we can verify this is a binary file and it also contains a Excel 4 macro folder named “macrosheets”.</p>
<p><a href="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/bin-xlfolder.png"><img width="693" height="312" title="Bin-xlFolder" style="border-width:0;padding-top:0;padding-right:0;padding-left:0;display:inline;background-image:none;" alt="Bin-xlFolder" src="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/bin-xlfolder_thumb.png?w=693&#038;h=312" border="0"></a></p>
<p><a href="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/bin-sharedstr.png"><img width="691" height="782" title="Bin-SharedStr" style="border-width:0;padding-top:0;padding-right:0;padding-left:0;display:inline;background-image:none;" alt="Bin-SharedStr" src="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/bin-sharedstr_thumb.png?w=691&#038;h=782" border="0"></a></p>
<p>If we look at the SharedStrings.bin file we can see that strings are in a Unicode format and not that easy to see where they split up at.</p>
<p><a href="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/sheet1-bin.png"><img width="594" height="812" title="Sheet1-bin" style="border-width:0;padding-top:0;padding-right:0;padding-left:0;display:inline;background-image:none;" alt="Sheet1-bin" src="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/sheet1-bin_thumb.png?w=594&#038;h=812" border="0"></a></p>
<p>Looking at sheet1.bin in the macrosheets folder we can see it is not human readable.&nbsp; </p>
<p>This is the point where I usually convert the file.</p>
<p><a href="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/xml-header.png"><img width="598" height="759" title="Xml-Header" style="border-width:0;padding-top:0;padding-right:0;padding-left:0;display:inline;background-image:none;" alt="Xml-Header" src="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/xml-header_thumb.png?w=598&#038;h=759" border="0"></a></p>
<p>Here we can see we still have a “PK” file but you can clearly see the data is presented a little differently.</p>
<p><a href="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/xml-folder.png"><img width="652" height="258" title="xml-Folder" style="border-width:0;padding-top:0;padding-right:0;padding-left:0;display:inline;background-image:none;" alt="xml-Folder" src="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/xml-folder_thumb.png?w=652&#038;h=258" border="0"></a></p>
<p>Once we unzip and navigate to the xl folder here it now looks a little different.</p>
<p><a href="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/xmlsharedstr.png"><img width="664" height="318" title="XmlSharedStr" style="border-width:0;padding-top:0;padding-right:0;padding-left:0;display:inline;background-image:none;" alt="XmlSharedStr" src="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/xmlsharedstr_thumb.png?w=664&#038;h=318" border="0"></a></p>
<p><a href="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/xmlsharedstr-2.png"><img width="668" height="137" title="XmlSharedStr-2" style="border-width:0;padding-top:0;padding-right:0;padding-left:0;display:inline;background-image:none;" alt="XmlSharedStr-2" src="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/xmlsharedstr-2_thumb.png?w=668&#038;h=137" border="0"></a></p>
<p>And now if we look at the SharedStrings.xml file it is a little different.</p>
<p>By the counts there are 34 indexed shared strings. Each appears to be randomly generated strings.</p>
<p><a href="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/xmlsharedstr-3.png"><img width="678" height="338" title="XmlSharedStr-3" style="border-width:0;padding-top:0;padding-right:0;padding-left:0;display:inline;background-image:none;" alt="XmlSharedStr-3" src="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/xmlsharedstr-3_thumb.png?w=678&#038;h=338" border="0"></a></p>
<p>I wrote a tool to aid in extracting and indexing the shared string from the xml file.</p>
<p>When I first parsed the shared strings I ended up with 0-37 index values instead of 0-33.<br />Turns out the tool stumbled on a rare random Char value I was using to split on.</p>
<p><a href="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/macrosheet-1.png"><img width="685" height="314" title="MacroSheet-1" style="border-width:0;padding-top:0;padding-right:0;padding-left:0;display:inline;background-image:none;" alt="MacroSheet-1" src="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/macrosheet-1_thumb.png?w=685&#038;h=314" border="0"></a></p>
<p>Here we see the xml version of the macro code. Like the shared strings it is hard to see thru all of the xml tags what is there so I wrote a parser for those too.</p>
<p><a href="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/macrosheet-2.png"><img width="691" height="382" title="MacroSheet-2" style="border-width:0;padding-top:0;padding-right:0;padding-left:0;display:inline;background-image:none;" alt="MacroSheet-2" src="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/macrosheet-2_thumb.png?w=691&#038;h=382" border="0"></a></p>
<p>This tool is designed to extract values to aid in better viewing what is happening without all of the xml tags. In this case some are left.</p>
<p><a href="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/macrosheet-3_b.png"><img width="692" height="405" title="MacroSheet-3_b" style="border-width:0;padding-top:0;padding-right:0;padding-left:0;display:inline;background-image:none;" alt="MacroSheet-3_b" src="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/macrosheet-3_b_thumb.png?w=692&#038;h=405" border="0"></a></p>
<p>Here we see what the values are.</p>
<p><a href="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/decode-1.png"><img width="697" height="339" title="Decode-1" style="border-width:0;padding-top:0;padding-right:0;padding-left:0;display:inline;background-image:none;" alt="Decode-1" src="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/decode-1_thumb.png?w=697&#038;h=339" border="0"></a></p>
<p>If we look at the highlighted values in green&nbsp; we see that it is looking for the string in cell ‘E11’ then we are taking the char at the index and taking so many chars. “MID(E11,12,1)” . In vbs the index start at 1 but in this the index starts at 0. </p>
<p>So now we know the first char code was converted to “S” and now we see the first extracted letter is “h” and the next to letter is “e” and then the next 2 are at the same index and is “l”.</p>
<p>Now we have the word “Shell” extracted.</p>
<p>This would be a pain to do by hand, but now that we understand how it works what else is available to extract this data.</p>
<p>The Answer is “XLMMacroDeobfuscator” located <a href="https://github.com/DissectMalware/XLMMacroDeobfuscator">here</a> .</p>
<p><a href="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/deobf-1.png"><img width="686" height="476" title="DeObf-1" style="border-width:0;padding-top:0;padding-right:0;padding-left:0;display:inline;background-image:none;" alt="DeObf-1" src="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/deobf-1_thumb.png?w=686&#038;h=476" border="0"></a></p>
<p>As we can see here this tool does a great job of presenting us with the deobfuscated strings.</p>
<p>The version I’m using here is from October 3rd 2021 before it was updated several more times. The version number stayed the same so you need to verify by the install/ file date. </p>
<p>Using the latest version as of November 12th 2021 it only returned the eval result. Also notice in the screen shot that showed the data it is a “Partial Evaluation” where in the updated version it is a “Full Evaluation”.</p>
<p>I have not looked at the byte format for the Macro sheet data but I have looked at the shared strings in the binary format.</p>
<p>Do to the lack of information that I can find on the file format let’s take a quick look at the data in this file as shown below. Notice the patterns.</p>
<p><a href="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/ss-bin-1.png"><img width="696" height="453" title="SS-Bin-1" style="border-width:0;padding-top:0;padding-right:0;padding-left:0;display:inline;background-image:none;" alt="SS-Bin-1" src="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/ss-bin-1_thumb.png?w=696&#038;h=453" border="0"></a></p>
<p>In the original sample I wrote an extraction tool for we can see how it is laid out slightly different.</p>
<p><a href="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/ss-bin-2.png"><img width="701" height="564" title="SS-Bin-2" style="border-width:0;padding-top:0;padding-right:0;padding-left:0;display:inline;background-image:none;" alt="SS-Bin-2" src="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/ss-bin-2_thumb.png?w=701&#038;h=564" border="0"></a></p>
<p><a href="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/file-1.png"><img width="710" height="148" title="file-1" style="border-width:0;padding-top:0;padding-right:0;padding-left:0;display:inline;background-image:none;" alt="file-1" src="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/file-1_thumb.png?w=710&#038;h=148" border="0"></a></p>
<p>Although the file in my original sample was labeled qut.xml it was not an xml file at all. So you can not count on a file name or extension for searches.</p>
<p><a href="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/file-2.png"><img width="707" height="677" title="file-2" style="border-width:0;padding-top:0;padding-right:0;padding-left:0;display:inline;background-image:none;" alt="file-2" src="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/file-2_thumb.png?w=707&#038;h=677" border="0"></a></p>
<p>And here is what it looks like in the Hex editor.</p>
<p><a href="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/file-3.png"><img width="713" height="370" title="File-3" style="border-width:0;padding-top:0;padding-right:0;padding-left:0;display:inline;background-image:none;" alt="File-3" src="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/file-3_thumb.png?w=713&#038;h=370" border="0"></a></p>
<p><a href="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/byteresearch-1-b.png"><img width="715" height="564" title="ByteResearch-1-b" style="border-width:0;padding-top:0;padding-right:0;padding-left:0;display:inline;background-image:none;" alt="ByteResearch-1-b" src="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/byteresearch-1-b_thumb.png?w=715&#038;h=564" border="0"></a></p>
<p>Lets take a look at format for this sample then we will go back and look at the one from the beginning.</p>
<p>We can see the first 3 bytes of the data appear to be a fixed Header value.</p>
<p>The next 4 bytes are the “Count”. If I understand correctly, it is the total times the string/chars are referenced. </p>
<p>The next 4 bytes are the “Unique Count”. These should be the total number of strings shown in the cells.</p>
<p>Next it gets interesting. </p>
<p>The first byte is always 0x13 Next we have 1 or 2 bytes (Unknown). Perhaps it is a data type ? It appears that it could be 1 or 2 bytes then a null byte depending on the string.</p>
<p>Next we have the length of the string as displayed in the cell. It uses at least 2 bytes. <br />So the first is only 1 char then value is 0x0100 or in reverse order 0x0001.</p>
<p>After that we have 2 null bytes. Then finally the Unicode bytes for the string.</p>
<p>Now lets go back to our first file that we extracted from this sample.</p>
<p><a href="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/firstsst-b.png"><img width="609" height="686" title="FirstSST-b" style="border-width:0;padding-top:0;padding-right:0;padding-left:0;display:inline;background-image:none;" alt="FirstSST-b" src="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/firstsst-b_thumb.png?w=609&#038;h=686" border="0"></a></p>
<p>Notice how everything is aligned but the area in the red box.</p>
<p><a href="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/firstsst-2.png"><img width="613" height="339" title="FirstSST-2" style="border-width:0;padding-top:0;padding-right:0;padding-left:0;display:inline;background-image:none;" alt="FirstSST-2" src="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/firstsst-2_thumb.png?w=613&#038;h=339" border="0"></a></p>
<p>If we look at the string under index 16 we see it is 531 characters long.</p>
<p>531 = 0x0213 and our value in the data is 0x1302.</p>
<p><a href="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/firstsst-3.png"><img width="620" height="505" title="FirstSST-3" style="border-width:0;padding-top:0;padding-right:0;padding-left:0;display:inline;background-image:none;" alt="FirstSST-3" src="https://pcsxcetrasupport3.wordpress.com/wp-content/uploads/2021/11/firstsst-3_thumb.png?w=620&#038;h=505" border="0"></a></p>
<p>Now everything lines up.</p>
<p>Here we see the first byte&nbsp; 0X13 then 2 unknown bytes then a null byte then 2 bytes for the length and then a double null and finally the start of out Unicode string values.</p>
<p>So in this sample we have extra 0x13 in a place that will break the tool. </p>
<p>At this point the tool will work on a few but will need a total rewrite based on this new information.</p>
<p>There have been plenty of samples that I have looked at where you did not even need to look at the VBA or macro code. All you needed to do was extract the shared strings to get the urls or paths used.</p>
<p>That is it for this one I hope you learned from this as much as I did.</p>
