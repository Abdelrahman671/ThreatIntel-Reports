                     <p>Authored by Yashvi Shah</p>
<p>Agent Tesla functions as a Remote Access Trojan (RAT) and an information stealer built on the .NET framework. It is capable of recording keystrokes, extracting clipboard content, and searching the disk for valuable data. The acquired information can be transmitted to its command-and-control server via various channels, including HTTP(S), SMTP, FTP, or even through a Telegram channel.</p>
<p>Generally, Agent Tesla uses deceptive emails to infect victims, disguising as business inquiries or shipment updates. Opening attachments triggers malware installation, concealed through obfuscation. The malware then communicates with a command server to extract compromised data.</p>
<p>The following heat map shows the current prevalence of Agent Tesla on field:</p>
<p><img decoding="async" loading="lazy" class="aligncenter wp-image-173860 size-full" src="https://www.mcafee.com/blogs/wp-content/uploads/2023/09/heatmap.png" alt="" width="1137" height="690" srcset="https://www.mcafee.com/blogs/wp-content/uploads/2023/09/heatmap.png 1137w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/heatmap-300x182.png 300w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/heatmap-1024x621.png 1024w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/heatmap-768x466.png 768w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/heatmap-205x124.png 205w" sizes="(max-width: 1137px) 100vw, 1137px" /></p>
<p style="text-align: center;"><em>Figure 1: Agent Tesla heat map</em></p>
<p>McAfee Labs has detected a variation where Agent Tesla was delivered through VBScript (VBS) files, showcasing a departure from its usual methods of distribution. VBS files are script files used in Windows for automating tasks, configuring systems, and performing various actions. They can also be misused by cybercriminals to deliver malicious code and execute harmful actions on computers.</p>
<h2><strong>Technical Analysis</strong></h2>
<p>The examined VBS file executed numerous PowerShell commands and then leveraged steganography to perform process injection into RegAsm.exe as shown in Figure 2. Regasm.exe is a Windows command-line utility used to register .NET assemblies as COM components, allowing interoperability between different software. It can also be exploited by malicious actors for purposes like process injection, potentially enabling covert or unauthorized operations.</p>
<p><img decoding="async" loading="lazy" class="aligncenter wp-image-173874 size-full" src="https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Injection_chain.png" alt="" width="1348" height="623" srcset="https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Injection_chain.png 1348w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Injection_chain-300x139.png 300w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Injection_chain-1024x473.png 1024w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Injection_chain-768x355.png 768w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Injection_chain-205x95.png 205w" sizes="(max-width: 1348px) 100vw, 1348px" /></p>
<p style="text-align: center;"><em>Figure 2: Infection Chain</em></p>
<p>VBS needs scripting hosts like wscript.exe to interpret and execute its code, manage interactions with the user, handle output and errors, and provide a runtime environment. When the VBS is executed, wscript invokes the initial PowerShell command.</p>
<p><img decoding="async" loading="lazy" class="aligncenter wp-image-173888 size-full" src="https://www.mcafee.com/blogs/wp-content/uploads/2023/09/ProcessTree.png" alt="" width="992" height="186" srcset="https://www.mcafee.com/blogs/wp-content/uploads/2023/09/ProcessTree.png 992w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/ProcessTree-300x56.png 300w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/ProcessTree-768x144.png 768w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/ProcessTree-205x38.png 205w" sizes="(max-width: 992px) 100vw, 992px" /></p>
<p style="text-align: center;"><em>Figure 3: Process Tree</em></p>
<h3><strong>First PowerShell command</strong></h3>
<p>The first PowerShell command is encoded as illustrated here:</p>
<p><img decoding="async" loading="lazy" class="aligncenter wp-image-173930 size-full" src="https://www.mcafee.com/blogs/wp-content/uploads/2023/09/First-Powershell.png" alt="" width="1826" height="820" srcset="https://www.mcafee.com/blogs/wp-content/uploads/2023/09/First-Powershell.png 1826w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/First-Powershell-300x135.png 300w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/First-Powershell-1024x460.png 1024w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/First-Powershell-768x345.png 768w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/First-Powershell-1536x690.png 1536w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/First-Powershell-205x92.png 205w" sizes="(max-width: 1826px) 100vw, 1826px" /></p>
<p style="text-align: center;"><em>Figure 4: Encoded First PowerShell</em></p>
<p>Obfuscating PowerShell commands serves as a defense mechanism employed by malware authors to make their malicious intentions harder to detect. This technique involves intentionally obfuscating the code by using various tricks, such as encoding, replacing characters, or using convoluted syntax. This runtime decoding is done to hide the true nature of the command from static analysis tools that examine the code without execution. Upon decoding, achieved by substituting occurrences of &#8216;#@$#&#8217; with &#8216;A&#8217; and subsequently applying base64-decoding, we successfully retrieved the decrypted PowerShell content as follows:</p>
<p><img decoding="async" loading="lazy" class="aligncenter wp-image-173916 size-full" src="https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Decoded-first-powershell-1.png" alt="" width="1825" height="287" srcset="https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Decoded-first-powershell-1.png 1825w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Decoded-first-powershell-1-300x47.png 300w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Decoded-first-powershell-1-1024x161.png 1024w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Decoded-first-powershell-1-768x121.png 768w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Decoded-first-powershell-1-1536x242.png 1536w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Decoded-first-powershell-1-205x32.png 205w" sizes="(max-width: 1825px) 100vw, 1825px" /></p>
<p style="text-align: center;"><em>Figure 5: Decoded content</em></p>
<h3><strong>Second PowerShell Command</strong></h3>
<p>The deciphered content serves as the parameter passed to the second instance of PowerShell..</p>
<p><img decoding="async" loading="lazy" class="aligncenter wp-image-173944 size-large" src="https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Second-PowerShell-1024x209.png" alt="" width="1024" height="209" srcset="https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Second-PowerShell-1024x209.png 1024w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Second-PowerShell-300x61.png 300w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Second-PowerShell-768x156.png 768w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Second-PowerShell-1536x313.png 1536w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Second-PowerShell-205x42.png 205w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Second-PowerShell.png 1822w" sizes="(max-width: 1024px) 100vw, 1024px" /></p>
<p style="text-align: center;"><em>Figure 6: Second PowerShell command</em></p>
<p>Deconstructing this command line for clearer comprehension:</p>
<p><img decoding="async" loading="lazy" class="aligncenter wp-image-173958 size-full" src="https://www.mcafee.com/blogs/wp-content/uploads/2023/09/disassembled-command.png" alt="" width="1797" height="575" srcset="https://www.mcafee.com/blogs/wp-content/uploads/2023/09/disassembled-command.png 1797w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/disassembled-command-300x96.png 300w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/disassembled-command-1024x328.png 1024w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/disassembled-command-768x246.png 768w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/disassembled-command-1536x491.png 1536w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/disassembled-command-205x66.png 205w" sizes="(max-width: 1797px) 100vw, 1797px" /></p>
<p style="text-align: center;"><em>Figure 7: Disassembled command</em></p>
<h3><strong>Steganography </strong></h3>
<p>As observed, the PowerShell command instructs the download of an image, from the URL that is strore in variable &#8220;imageURL.&#8221; The downloaded image is 3.50 MB in size and is displayed below:</p>
<p><img decoding="async" loading="lazy" class="aligncenter wp-image-174269 size-full" src="https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Downloaded-Image.png" alt="" width="800" height="594" srcset="https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Downloaded-Image.png 800w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Downloaded-Image-300x223.png 300w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Downloaded-Image-768x570.png 768w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Downloaded-Image-174x129.png 174w" sizes="(max-width: 800px) 100vw, 800px" /></p>
<p>&nbsp;</p>
<p style="text-align: center;"><em>Figure 8: Downloaded image</em></p>
<p>This image serves as the canvas for steganography, where attackers have concealed their data. This hidden data is extracted and utilized as the PowerShell commands are executed sequentially. The commands explicitly indicate the presence of two markers, &#8216;&lt;&lt;BASE64_START&gt;&gt;&#8217; and &#8216;&lt;&lt;BASE64_END&gt;&gt;&#8217;. The length of the data is stored in variable ‘base64Length’. The data enclosed between these markers is stored in ‘base64Command’. The subsequent images illustrate these markers and the content encapsulated between them.</p>
<p><img decoding="async" loading="lazy" class="aligncenter wp-image-173986 size-full" src="https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Steganography.png" alt="" width="1236" height="420" srcset="https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Steganography.png 1236w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Steganography-300x102.png 300w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Steganography-1024x348.png 1024w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Steganography-768x261.png 768w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Steganography-205x70.png 205w" sizes="(max-width: 1236px) 100vw, 1236px" /></p>
<p style="text-align: center;"><em>Figure 9: Steganography</em></p>
<p>After obtaining this data, the malware proceeds with decoding procedures. Upon examination, it becomes apparent that the decrypted data is a .NET DLL file. In the subsequent step, a command is executed to load this DLL file into an assembly.</p>
<p><img decoding="async" loading="lazy" class="aligncenter wp-image-174000 size-full" src="https://www.mcafee.com/blogs/wp-content/uploads/2023/09/dllpayload.png" alt="" width="607" height="359" srcset="https://www.mcafee.com/blogs/wp-content/uploads/2023/09/dllpayload.png 607w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/dllpayload-300x177.png 300w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/dllpayload-205x121.png 205w" sizes="(max-width: 607px) 100vw, 607px" /></p>
<p style="text-align: center;"><em>Figure 10: DLL obtained from steganography</em></p>
<h3><strong>Process Injection into RegAsm.exe</strong></h3>
<p>This DLL serves two purposes:</p>
<ol>
<li>Downloading and decoding the final payload</li>
<li>Injecting it into RegAsm.exe</li>
</ol>
<p><img decoding="async" loading="lazy" class="aligncenter wp-image-174014 size-full" src="https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Dll-loaded.png" alt="" width="1171" height="788" srcset="https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Dll-loaded.png 1171w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Dll-loaded-300x202.png 300w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Dll-loaded-1024x689.png 1024w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Dll-loaded-768x517.png 768w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Dll-loaded-192x129.png 192w" sizes="(max-width: 1171px) 100vw, 1171px" /></p>
<p style="text-align: center;"><em>Figure 11: DLL loaded</em></p>
<p>In Figure 11, at marker 1, a parameter named &#8216;QBXtX&#8217; is utilized to accept an argument for the given instruction. As we proceed with the final stage of the PowerShell command shown in Figure 7, the sequence unfolds as follows:</p>

<p>In Figure 11, another activity is evident at marker 3, where the process path for the upcoming process injection is specified. The designated process path for the process injection is :</p>
<p><strong><em>&#8220;C:\Windows\Microsoft.NET\Framework\v4.0.30319\RegAsm.exe&#8221;.</em></strong></p>
<p>Since RegAsm.exe is a legitimate Windows tool, it&#8217;s less likely to raise suspicion from security solutions. Injecting .NET samples into it allows attackers to effectively execute their malicious payload within a trusted context, making detection and analysis more challenging.</p>
<p>Process injection involves using Windows API calls to insert code or a payload into the memory space of a running process. This allows the injected code to execute within the context of the target process. Common steps include allocating memory, writing code, creating a remote thread, and executing the injected code. In this context, the DLL performs a sequence of API calls to achieve process injection:</p>
<p><img decoding="async" loading="lazy" class="aligncenter wp-image-174070 size-full" src="https://www.mcafee.com/blogs/wp-content/uploads/2023/09/processinjection.png" alt="" width="934" height="899" srcset="https://www.mcafee.com/blogs/wp-content/uploads/2023/09/processinjection.png 934w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/processinjection-300x289.png 300w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/processinjection-768x739.png 768w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/processinjection-134x129.png 134w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/processinjection-24x24.png 24w" sizes="(max-width: 934px) 100vw, 934px" /></p>
<p style="text-align: center;"><em>Figure 15: Process Injection</em></p>
<p>By obscuring the sequence of API calls and their intended actions through obfuscation techniques, attackers aim to evade detection and make it harder for security researchers to unravel the true behavior of the malicious code. The function ‘hU0H4qUiSpCA13feW0’ is used for replacing content. For example,</p>
<p>&#8220;kern!&#8221;.Replace(&#8220;!&#8221;, &#8220;el32&#8221;)  à  kernel32</p>
<p>Class1.hU0H4qUiSpCA13feW0(&#8220;qllocEx&#8221;, &#8220;q&#8221;, &#8220;VirtualA&#8221;) à VirtualAllocEx</p>
<p>As a result, these functions translate into the subsequent API calls:</p>
<ol>
<li><strong>CreateProcessA : </strong>This API call is typically employed to initiate the creation of a new process, rather than for process injection. In the context of process injection, the focus is generally on targeting an existing process and injecting code into it.</li>
<li><strong>VirtualAllocEx</strong>: This is often used in process injection to allocate memory within the target process to host the injected code.</li>
<li><strong>ReadProcessMemory</strong>: This is used to read the memory of a target process. It is typically used in reflective DLL injection to read the contents of a DLL from the injector&#8217;s memory and write it into the target process.</li>
<li><strong>GetThreadContext</strong>: This API is used to retrieve the context (registers, flags, etc.) of a thread within a target process. It&#8217;s useful for modifying thread execution flow during injection.</li>
<li><strong>Wow64GetThreadContext</strong>: This is like GetThreadContext, but it&#8217;s used when dealing with 32-bit processes on a 64-bit system.</li>
<li><strong>SetThreadContext</strong>: This API is used to set the context of a thread within a target process. This can be useful for modifying the execution flow.</li>
<li><strong>Wow64SetThreadContext</strong>: Like SetThreadContext, but for 32-bit processes on a 64-bit system.</li>
<li><strong>ZwUnmapViewOfSection</strong>: This is used to unmap a section of a process&#8217;s virtual address space, which could potentially be used to remove a DLL loaded into a target process during injection.</li>
<li><strong>WriteProcessMemory</strong>: This is used to write data into the memory of a target process. It&#8217;s commonly used for injecting code or data into a remote process.</li>
<li><strong>ResumeThread</strong>: This is used to resume the execution of a suspended thread, often after modifying its context or injecting code.</li>
</ol>
<p>Upon successful injection of the malware into RegAsm.exe, it initiates its intended operations, primarily focused on data theft from the targeted system.</p>
<p>The ultimate executable is heavily obfuscated. It employs an extensive array of switch cases and superfluous code, strategically intended to mislead researchers and complicate analysis. <h2><strong>Exfiltration:</strong></h2>
<p>Agent Tesla employs significant obfuscation techniques to evade initial static analysis attempts. This strategy conceals its malicious code and actual objectives. Upon successful decoding, we were able to scrutinize its internal operations and functionalities, including the use of SMTP for data exfiltration.</p>
<p>The observed sample utilizes SMTP as its chosen method of exfiltration. This protocol is frequently favored due to its minimal overhead demands on the attacker. SMTP reduces overhead for attackers because it is efficient, widely allowed in networks, uses existing infrastructure, causes minimal anomalies, leverages compromised accounts, and appears less suspicious compared to other protocols. A single compromised email account can be used for exfiltration, streamlining the process, and minimizing the need for complex setups. They can achieve their malicious goals with just a single email account, simplifying their operations.</p>
<p><img decoding="async" loading="lazy" class="aligncenter wp-image-174196 size-large" src="https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Function-calls-made-for-exfiltration.-1024x615.png" alt="" width="1024" height="615" srcset="https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Function-calls-made-for-exfiltration.-1024x615.png 1024w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Function-calls-made-for-exfiltration.-300x180.png 300w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Function-calls-made-for-exfiltration.-768x462.png 768w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Function-calls-made-for-exfiltration.-205x123.png 205w, https://www.mcafee.com/blogs/wp-content/uploads/2023/09/Function-calls-made-for-exfiltration..png 1080w" sizes="(max-width: 1024px) 100vw, 1024px" /></p>

This orchestrated sequence highlights how the malware establishes a connection through SMTP, facilitating its intended operations.</p>
<p>Following these steps, the malware efficiently establishes a login using acquired credentials. Once authenticated, it commences the process of transmitting the harvested data to a designated email address associated with the malware itself.</p>
<h3><strong>Summary</strong><strong>:</strong></h3>
<p>The infection process of Agent Tesla involves multiple stages. It begins with the initial vector, often using email attachments or other social engineering tactics. Once executed, the malware employs obfuscation to avoid detection during static analysis. The malware then undergoes decoding, revealing its true functionality. It orchestrates a sequence of PowerShell commands to download and process a hidden image containing encoded instructions. These instructions lead to the extraction of a .NET DLL file, which subsequently injects the final payload into the legitimate process &#8216;RegAsm.exe&#8217; using a series of API calls for process injection. This payload carries out its purpose of data theft, including targeting browsers and email clients for sensitive information. The stolen data is exfiltrated via SMTP communication, providing stealth and leveraging email account. Overall, Agent Tesla&#8217;s infection process employs a complex chain of techniques to achieve its data-stealing objectives.</p>
