<p><strong>Summary</strong></p>



<p>We analyzed a new version of ChromeLoader (also known as Choziosi Loader) that was seen in the wild in recent weeks.</p>



<p>This ChromeLoader campaign that appears to have started in December 2021 has become widespread and has spawned multiple versions, making atomic indicators ineffective for detections.</p>



<p>In our analysis we will be discussing the capabilities of this loader, as well as trying to dig a little deeper, in order to find some indicators that will be more difficult for the threat actor to alter without making significant changes to the malware&#8217;s architecture (at least compared to extracting domains, IP and hashes as only IOCs)</p>



<p>We will be starting our analysis with the execution of the obfuscated powershell that ultimately downloads the malicious extension on the host.</p>



<p>We have managed to extract some interesting strings from the different stages of the malware, as well as discovering a few modules that are not in use at the moment (or are not working properly), but could give a hint on what functionality will be added to the malware in the future.</p>



<p>We will not be talking about the dropper in this analysis, as it is already quite well documented and no significant changes were seen for new versions of Chromeloader.</p>



<p>For more information on the dropper, a collection of atomic indicators and a comparison between different versions seen in the wild, we recommend you read the article put out by Unit42, as it is quite extensive: </p>



<p><a href="https://unit42.paloaltonetworks.com/chromeloader-malware/#post-123828-_rk3otl9durd">https://unit42.paloaltonetworks.com/chromeloader-malware/#post-123828-_rk3otl9durd</a></p>



<p>Ok, let&#8217;s get to it.</p>



<p><strong>Static Analysis</strong></p>



<p>We start our analysis with a powershell command that will connect to the C2 server used for the installation (different from the C2 used for data exfil) to download the first malicious payload, as well as set the ground for the extension&#8217;s installation. </p>



<p>While the initial powershell is heavily obfuscated, there are a few strings of interest that we can identify with little effort. One of them is the function that builds the URL for the installation C2 server. We can also see the <code>Invoke-Expression</code>&nbsp;cmdlet.</p>



<figure class="wp-block-image aligncenter size-large"><img fetchpriority="high" decoding="async" width="1024" height="399" src="https://cybergeeks.tech/wp-content/uploads/2022/08/image-1024x399.png" alt="" class="wp-image-3851" srcset="https://cybergeeks.tech/wp-content/uploads/2022/08/image-1024x399.png 1024w, https://cybergeeks.tech/wp-content/uploads/2022/08/image-300x117.png 300w, https://cybergeeks.tech/wp-content/uploads/2022/08/image-768x299.png 768w, https://cybergeeks.tech/wp-content/uploads/2022/08/image.png 1152w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption><sub>Image 1</sub>  <sub>&#8211; Installer URL visible in the initial Powershell</sub></figcaption></figure>



<p>By changing one of the visible iex expressions to Write-Output, we were able to make good progress without bothering with decrypting the script.</p>



<figure class="wp-block-image aligncenter size-full"><img decoding="async" width="605" height="214" src="https://cybergeeks.tech/wp-content/uploads/2022/08/image-28.png" alt="" class="wp-image-3879" srcset="https://cybergeeks.tech/wp-content/uploads/2022/08/image-28.png 605w, https://cybergeeks.tech/wp-content/uploads/2022/08/image-28-300x106.png 300w" sizes="(max-width: 605px) 100vw, 605px" /><figcaption><sub>Image 2</sub></figcaption></figure>



<p>NOTE: this method is a lot faster than manual deobfuscation, but can miss details that could be relevant for the analysis ( there could be multiple &#8220;invokeExpressions&#8221; that were obfuscated and we could not see for instance). Just something to keep in mind.</p>



<p>Running the script like this downloads a C# script from the installation server present in the command. During normal execution it would then invoke it.</p>



<p>The C# script gives us some good hints about the capability of the malware.</p>



<p>Some interesting function names:</p>



<ul class="wp-block-list"><li>getGoogSearchUri</li><li>hookSearchNavigation</li><li>runChromeOrEdge</li><li>runFirefox</li><li>runThread</li></ul>



<p>Some interesting methods:</p>



<ul class="wp-block-list"><li>paneConditionChromeOrEdge</li><li>editConditionChromeOrEdge</li><li>toolbarConditionFirefox</li><li>comboboxConditionFirefox</li><li>editboxConditionFirefox</li></ul>



<p>While previous analysis of the malware concluded that it would only affect Chrome browsers, we can see here that it checks to see if Chrome, Edge or Firefox is installed and can hook any of these.</p>



<p>It seems to be mostly interested in the user&#8217;s search history, as it intercepts searches done on google and then redirects them to Bing .</p>



<p>It also intercepts keyboard keys to account for the users that use the keyboard to navigate the results.</p>

<p>In the string from image 6 , the URL is built using the following variables:</p>



<ul class="wp-block-list"><li>Domain</li><li>TID is a hardcoded value in the script; the value was the same every time for this version of the malware so maybe it is used for versioning</li><li>U is an unique identifier for the user</li><li>We could not determine what &#8220;ist&#8221; is at this time</li></ul>



<p>Once the C# code finishes, it is followed by a series of additional Powershell commands that will build it, load it into memory and run it.</p>



<p>Powershell is used again to build the URL from which the next payload will be downloaded.</p>



<figure class="wp-block-image aligncenter size-full"><img loading="lazy" decoding="async" width="843" height="99" src="https://cybergeeks.tech/wp-content/uploads/2022/08/image-5.png" alt="" class="wp-image-3856" srcset="https://cybergeeks.tech/wp-content/uploads/2022/08/image-5.png 843w, https://cybergeeks.tech/wp-content/uploads/2022/08/image-5-300x35.png 300w, https://cybergeeks.tech/wp-content/uploads/2022/08/image-5-768x90.png 768w" sizes="auto, (max-width: 843px) 100vw, 843px" /><figcaption><sub>Image 7 &#8211; Powershell is used to build the C# code</sub></figcaption></figure>



<p>The commands suggest that a new archived file will be downloaded and expanded in a new folder made in the user&#8217;s APPDATA , that is randomly generated by the script via an XOR operation.</p>



<p>The following function sets up the key for the encryption:</p>

<p>Indeed, after arming the sample and detonating it , we can see a new folder in AppData called &#8220;chrome_glass&#8221;:</p>

<p>We can see how the script&nbsp; downloaded the C# code and then built it into a .dll in the Temp folder:</p>



<figure class="wp-block-image aligncenter size-full"><img loading="lazy" decoding="async" width="753" height="566" src="https://cybergeeks.tech/wp-content/uploads/2022/08/image-10.png" alt="" class="wp-image-3861" srcset="https://cybergeeks.tech/wp-content/uploads/2022/08/image-10.png 753w, https://cybergeeks.tech/wp-content/uploads/2022/08/image-10-300x225.png 300w" sizes="auto, (max-width: 753px) 100vw, 753px" /><figcaption><sub>Image 11 &#8211; the stager is built and executed in the user&#8217;s Temp folder, and then deleted</sub></figcaption></figure>



<p>&nbsp;There are also some evasion mechanisms here that are worth pointing out:</p>



<ul class="wp-block-list"><li>The files are downloaded, ran and then deleted</li><li>The PSScript Policy test runs to ensure that the Temp folder is writable and that the files can be deleted</li><li>A new directory is created with a randomly generated name, to ensure that the files cannot be retrieved by tools such as DirWatch</li></ul>



<p>Let`s also look at registry changes. We have caught hints from the powershell script from earlier that the installer will also write a value in &#8220;HKCU:\Software\CodeSector\&#8221;.</p>



<p>And indeed, we see a new registry Key being added:</p>



<figure class="wp-block-image aligncenter size-full"><img loading="lazy" decoding="async" width="783" height="200" src="https://cybergeeks.tech/wp-content/uploads/2022/08/image-11.png" alt="" class="wp-image-3862" srcset="https://cybergeeks.tech/wp-content/uploads/2022/08/image-11.png 783w, https://cybergeeks.tech/wp-content/uploads/2022/08/image-11-300x77.png 300w, https://cybergeeks.tech/wp-content/uploads/2022/08/image-11-768x196.png 768w" sizes="auto, (max-width: 783px) 100vw, 783px" /><figcaption><sub>Image 12 &#8211; A new registry key is added</sub></figcaption></figure>

<p><strong>Network indicators:</strong></p>



<p>We know that the extension is using QUIC as a transport protocol for fast and encrypted communication. But, since we control the execution, we can force Chrome to dump the SSL Keys so we can load them in Wireshark and decrypt the traffic.&nbsp;</p>



<figure class="wp-block-image aligncenter size-full"><img loading="lazy" decoding="async" width="684" height="340" src="https://cybergeeks.tech/wp-content/uploads/2022/08/image-21.png" alt="" class="wp-image-3872" srcset="https://cybergeeks.tech/wp-content/uploads/2022/08/image-21.png 684w, https://cybergeeks.tech/wp-content/uploads/2022/08/image-21-300x149.png 300w" sizes="auto, (max-width: 684px) 100vw, 684px" /><figcaption><sub>Image 22 &#8211; Decrypted QUIC traffic</sub></figcaption></figure>
