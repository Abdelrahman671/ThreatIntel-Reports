			<h1 class="entry-title">Qbot Likes to Move It, Move It</h1>
		<div class="entry-meta">
			<span class="posted-on"><a href="https://thedfirreport.com/2022/02/07/qbot-likes-to-move-it-move-it/" rel="bookmark"><time class="entry-date published" datetime="2022-02-07T01:02:36+00:00">February 7, 2022</time></a></span>		</div><!-- .entry-meta -->

		
	</header><!-- .entry-header -->

	<div class="entry-content">
		<div class="markdown">
<p id="intro"><a href="https://malpedia.caad.fkie.fraunhofer.de/details/win.qakbot" target="_blank" rel="nofollow noopener">Qbot</a> (aka QakBot, Quakbot, Pinkslipbot ) has been around for a long time having first been observed back in 2007. More info on Qbot can be found at the following links: <a href="https://www.microsoft.com/security/blog/2021/12/09/a-closer-look-at-qakbots-latest-building-blocks-and-how-to-knock-them-down/">Microsoft</a> &amp; <a href="https://redcanary.com/threat-detection-report/threats/qbot/" target="_blank" rel="noopener">Red Canary</a></p>
<p>In this case, from October 2021, we will break down how Qbot quickly spread across all workstations in an environment, while stealing browser information and emails. While the case is nearly 5 months old, Qbot infections in the past week have followed the same pattern.</p>
<h3 id="case-summary">Case Summary</h3>
<p>We did not observe the initial access for this case but assess with medium to high confidence that a malicious email campaign was used to deliver an Excel (xls) document. Following the opening of the xls document, the initial Qbot DLL loader was downloaded and saved to disk. Interestingly, the name of the DLL contained a .html extension to disguise the portable executable nature of the payload. Once executed, the Qbot process creates a scheduled task to elevate itself to system.</p>
<p>Qbot injected into many processes but one favorite in this intrusion, was Microsoft Remote Assistance (msra.exe). Within minutes of landing on the beachhead, a series of discovery commands were executed using Microsoft utilities. Around the same time, LSASS was access by Qbot to collect credentials from memory.</p>
<p>Thirty minutes after initial access, Qbot was observed collecting data from the beachhead host including browser data and emails from Outlook. At around 50 minutes into the infection, the beachhead host copied a Qbot dll to an adjacent workstation, which was then executed by remotely creating a service. Minutes later, the beachhead host did the same thing to another adjacent workstation and then another, and before we knew it, all workstations in the environment were compromised.</p>
<p>Qbot followed it&#8217;s normal process on each machine. Servers were not accessed in this intrusion. After this activity, normal beaconing occurred but no further actions on objectives were seen.</p>
<h3 id="services">Services</h3>
<p>We offer multiple services including a <a href="https://thedfirreport.com/services/" target="_blank" rel="nofollow noopener">Threat Feed service</a> which tracks Command and Control frameworks such as Qbot, Cobalt Strike, BazarLoader, Covenant, Metasploit, Empire, PoshC2, etc. More information on this service and others can be found <a href="https://thedfirreport.com/services/" target="_blank" rel="nofollow noopener">here</a>.</p>
<p>We also have artifacts and IOCs available from this case such as memory captures, files, event logs including Sysmon, Kape packages, and more, under our <a href="https://www.patreon.com/thedfirreport" target="_blank" rel="nofollow noopener">Security Researcher and Organization</a> services.</p>
<h3 id="timeline">Timeline</h3>
<p><img fetchpriority="high" decoding="async" class="size-full wp-image-5497 aligncenter" src="https://thedfirreport.com/wp-content/uploads/2022/02/Qbot-Likes-to-Move-It-Move-It.png" alt="" width="1088" height="1244" srcset="https://thedfirreport.com/wp-content/uploads/2022/02/Qbot-Likes-to-Move-It-Move-It.png 1088w, https://thedfirreport.com/wp-content/uploads/2022/02/Qbot-Likes-to-Move-It-Move-It-262x300.png 262w, https://thedfirreport.com/wp-content/uploads/2022/02/Qbot-Likes-to-Move-It-Move-It-896x1024.png 896w, https://thedfirreport.com/wp-content/uploads/2022/02/Qbot-Likes-to-Move-It-Move-It-768x878.png 768w" sizes="(max-width: 1088px) 100vw, 1088px" /></p>
<p>Analysis and reporting completed by <a href="https://twitter.com/iiamaleks" target="_blank" rel="noreferrer noopener">@iiamaleks</a></p>
<p>Reviewed by and <a href="https://twitter.com/MetallicHack" target="_blank" rel="noreferrer noopener">@MetallicHack</a> &amp; <a href="https://twitter.com/tas_kmanager" target="_blank" rel="noreferrer noopener">@tas_kmanager</a></p>
<h3 id="mitre-attck">MITRE ATT&amp;CK</h3>
<h3 id="initial-access">Initial Access</h3>
<p>We assess with medium to high confidence that the QBot infection was delivered to the system via a malspam campaign through a hidden 4.0 Macro&#8217;s in Excel.</p>
<p>We believe <a href="https://www.joesandbox.com/analysis/501226/0/html" target="_blank" rel="noopener">this</a> is the xls file that lead to the Qbot infection, due to the overlap in time period, download url, and file name.</p>
<p><img decoding="async" class="aligncenter" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2022/02/c75b23a8f84a84cfa827cda6067dad9acd974dc3ca2c9292f975a544877ad03e" alt="enter image description here" width="1176" height="822" /></p>
<h3 id="execution">Execution</h3>
<p>The QBot dll was executed on the system and shortly after, injected into the msra.exe process.</p>

<h3 id="privilege-escalation">Privilege Escalation</h3>

<h3 id="defense-evasion">Defense Evasion</h3>
<p>QBot was observed injecting into msra.exe process on multiple systems.</p>
<p><img loading="lazy" decoding="async" class="aligncenter" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2022/02/fb686791c94301af96c8f40cef5293369788f22010ee09f17ccf2bf615a4779f" alt="enter image description here" width="1477" height="111" /></p>
<p>Multiple folders were added to the Windows Defender Exclusions list in order to prevent the Qbot dll placed inside of it from being detected. The newly dropped dll was then executed and process injected into msra.exe.</p>
<p><img loading="lazy" decoding="async" class="alignnone size-full wp-image-5483" src="https://thedfirreport.com/wp-content/uploads/2022/02/workstation.png" alt="" width="1645" height="302" srcset="https://thedfirreport.com/wp-content/uploads/2022/02/workstation.png 1645w, https://thedfirreport.com/wp-content/uploads/2022/02/workstation-300x55.png 300w, https://thedfirreport.com/wp-content/uploads/2022/02/workstation-1024x188.png 1024w, https://thedfirreport.com/wp-content/uploads/2022/02/workstation-768x141.png 768w, https://thedfirreport.com/wp-content/uploads/2022/02/workstation-1536x282.png 1536w" sizes="auto, (max-width: 1645px) 100vw, 1645px" /></p>

<h3 id="command-and-control">Command and Control</h3>
<p>Qbot uses a tiered infrastructure, often using other compromised systems as first tier proxy points for establishing a constantly changing list of C2 endpoints. <p>With this type of setup the list of C2 from October 2021, has in large rotated out of use. To keep up to date on current Qbot C2 endpoints you can check out our <a href="https://thedfirreport.com/services/" target="_blank" rel="nofollow noopener">Threat Feed &amp; All Intel service</a> as we track these changing lists daily.</p>
