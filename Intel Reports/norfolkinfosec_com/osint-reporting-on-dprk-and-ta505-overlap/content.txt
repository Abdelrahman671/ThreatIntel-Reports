<p style="font-size:12px;text-align:left">Yesterday, at SAS2019, BAE Systems <a href="https://www.wired.com/story/atm-hacks-swift-network/">presented findings</a> related to DPRK SWIFT heist activity that took place in 2018. As part of this research (a leaked video of the presentation is available online), BAE included two key points not previously disclosed in the public domain:<br><br>&#8211; The existence of a PowerShell backdoor attributable to DPRK, which the researchers dubbed PowerBrace<br>&#8211; A possible overlap between TA505 intrusions and DPRK intrusions, suggesting a possible hand-off between the two groups.<br><br>This blog will leave a full analysis of those two points and the supporting context to the people that found them, as it&#8217;s theirs to share; however, data that may support such conclusions <em><u>have</u> </em>been available in open source for quite some time. <br><br>In early January, VNCert<a href="http://www.vncert.gov.vn/baiviet.php?id=109"> issued an alert</a> regarding attacks targeting financial institutions, containing a mix of DPRK IOCs (including a keylogger referred to as PSLogger <a href="https://norfolkinfosec.com/a-lazarus-keylogger-pslogger/">previously analyzed by this blog</a>), TA505 IOCs (previously <a href="https://ti.360.net/blog/articles/excel-4.0-macro-utilized-by-ta505-to-target-financial-institutions-recently-en/">published by 360 TIC</a>), and a handful of PowerShell scripts that are generally identical aside from a handful of configuration changes. Furthermore, the aforementioned keylogger was first uploaded by a submitter (fabd7a52) in Pakistan in December 2018. That same submitter acted as the first uploader for one of the PowerShell samples identified below (b88d4d72fdabfc040ac7fb768bf72dcd), further corroborating a possible link.<br><br>Given the multi-sourced reporting overlaps and the additional Pakistan findings mentioned above, this blog assesses that the PowerShell scripts in question likely belong to the same family of DPRK-attributable malware reported by BAE systems. <br><br>A listing of selected IOCs is below the fold, alongside a few brief notes (and a script) for <em>how</em> to analyze the PowerShell malware.</p>



<span id="more-175"></span>



<p><strong><u>IOCs from VNCert</u></strong></p>



<p style="font-size:12px;text-align:left"><u>TA505:</u><br><br>These contain infrastructure overlaps with reporting from the same month found here: https://ti.360.net/blog/articles/excel-4.0-macro-utilized-by-ta505-to-target-financial-institutions-recently-en/<br><br>MD5: 5B7244C47104F169B0840440CDEDE788 <br>MD5: cc29adb5b78300b0f17e566ad461b2c7 <br>MD5: E00499E21F9DCF77FC990400B8B3C2B5 <br>MD5: 53F7BE945D5755BB628DEECB71CDCBF2 <br>MD5: 9c35e9aa9255aa2214d704668b039ef6 <br>MD5: 2e0d13266b45024153396f002e882f15<br>MD5: 26f09267d0ec0d339e70561a610fb1fd<br>MD5: 09e4f724e73fccc1f659b8a46bfa7184 <br><br><br><u>DPRK:</u><br>HSMBalance.exe MD5:34404a3fb9804977c6ab86cb991fb130 &#8211; Keylogger <br>ICAS.ps1 MD5: b12325a1e6379b213d35def383da2986 &#8211; Possible PowerBrace<br>MD5: 8a41520c89dce75a345ab20ee352fef0 &#8211; Possible PowerBrace<br>MD5: 7c651d115109fd8f35fddfc44fd24518 &#8211; Possible PowerBrace<br>MD5: b88d4d72fdabfc040ac7fb768bf72dcd &#8211; Possible PowerBrace<br>MD5: 3be75036010f1f2102b6ce09a9299bca &#8211; Possible PowerBrace <br><br>Several hashes were omitted: these were EML files that belong to specific financial organizations. Others were not on VirusTotal or were not read properly by OCR. </p>



<p style="font-size:12px"><strong>A Few Notes on the PowerShell Backdoor</strong></p>



<p style="font-size:12px;text-align:left">MD5 Used:  b12325a1e6379b213d35def383da2986  (ICAS.ps1)<br>C2: 192.95.14.128<br><br>As previously mentioned, this blog will not be publishing a full analysis of this backdoor in deference to the people who first found it; however, in the interest of helping analysts who need the data, there are a few key points to mention:<br><br>&#8211; The backdoor uses a configuration file that includes two C2 servers and a series of Base64 encoded commands<br>&#8211; Most of the malware&#8217;s function names have been replaced with MD5 hashes</p>



<figure class="wp-block-image"><img fetchpriority="high" decoding="async" width="1024" height="346" src="https://norfolkinfosec.com/wp-content/uploads/2019/04/PB_1-1-1024x346.png" alt="" class="wp-image-177" srcset="https://norfolkinfosec.com/wp-content/uploads/2019/04/PB_1-1-1024x346.png 1024w, https://norfolkinfosec.com/wp-content/uploads/2019/04/PB_1-1-300x101.png 300w, https://norfolkinfosec.com/wp-content/uploads/2019/04/PB_1-1-768x259.png 768w, https://norfolkinfosec.com/wp-content/uploads/2019/04/PB_1-1-210x71.png 210w, https://norfolkinfosec.com/wp-content/uploads/2019/04/PB_1-1.png 1336w" sizes="(max-width: 980px) 100vw, 980px" /><figcaption><strong>A snippet of the encoded configuration and obfuscated functions. Right click and open in a new tab to expand.</strong></figcaption></figure>



<p style="font-size:12px;text-align:left">A script below has been included that performs the Base64 transformation on values where it can find them. To analyze this script, this blog then recommends the following process:<br><br>1) Using an easily identifiable command name (decoded by the script), locate that command&#8217;s use in a function<br>2) Identify references between that command and other functions<br>3) Rename those other functions<br>4) Repeat<br><br>An example of decoded data (with variables and functions renamed manually) is below:</p>



<figure class="wp-block-image"><img decoding="async" width="1009" height="451" src="https://norfolkinfosec.com/wp-content/uploads/2019/04/PB_2.png" alt="" class="wp-image-178" srcset="https://norfolkinfosec.com/wp-content/uploads/2019/04/PB_2.png 1009w, https://norfolkinfosec.com/wp-content/uploads/2019/04/PB_2-300x134.png 300w, https://norfolkinfosec.com/wp-content/uploads/2019/04/PB_2-768x343.png 768w, https://norfolkinfosec.com/wp-content/uploads/2019/04/PB_2-210x94.png 210w" sizes="(max-width: 980px) 100vw, 980px" /><figcaption><strong>A portion of the partially decoded and deobfuscated PowerShell backdoor</strong></figcaption></figure>



<p style="font-size:12px">A script to assist with this is here:</p>



<pre class="wp-block-code"><code>import base64
import re

c = open("c:\\users\\[username]\\desktop\\[filename]").readlines()

line_list = []

for line in c:
    #print(line)
    try:
        enc = re.search("(?&lt;=\$\(\[Text.Encoding\]::Unicode.GetString\(\[Convert\]::FromBase64String\().*?(?=\))",line).group()
		print(line)
		print(enc)
		d = ('"' + base64.b64decode(enc) + '"')
		e = (re.sub("\$\(\[Text.Encoding\]::Unicode.GetString\(\[Convert\]::FromBase64String\(.*?\)\)\)",d,line))
		f = re.sub("\0","",e)
		line_list.append(f)
        
    except:
        line_list.append(line)


with open("c:\\users\\[username]\\desktop\\laz_decoded.ps1","wt") as t:
    for unit in line_list:
        t.write(unit)</code></pre>
