<p>The Morphisec Labs team has been tracking an ongoing RAT delivery campaign that started in February this year. This campaign is unique in its heavy use of the <a href="https://en.wikipedia.org/wiki/AutoHotkey" rel="nofollow noopener" target="_blank">AutoHotKey </a>scripting languageâ€”a fork of the AutoIt language frequently used for testing purposes.&nbsp;</p>
<p><!--more--></p>
<p>Starting in February, we identified at least four versions of the RAT delivery campaign, each of which includes multiple advancements and adaptations over the past three months.</p>
<p>In this blog post, we dive into the details of each attack chain, highlighting interesting and rare techniques the attackers use, including:</p>
<ul>
<li aria-level="1">Manifest flow hijack through <a href="https://www.vbsedit.com/" rel="nofollow noopener" target="_blank"><span>VbsEdit</span></a> manipulation&nbsp;</li>
<li aria-level="1">UAC bypass</li>
<li aria-level="1">Emulator bypass</li>
<li aria-level="1">Tampering with Microsoft Defender and other antivirus products</li>
<li aria-level="1">In-place compilation</li>
<li aria-level="1">Delivery through text share services</li>
</ul>
<h2>Technical details:</h2>
<p>The RAT delivery campaign starts from an AutoHotKey compiled script. This is a standalone executable that contains the following: the AHK interpreter, the <span style="font-weight: bold;">AHK script</span>, and any files it has incorporated via the <a href="https://www.autohotkey.com/docs/commands/FileInstall.htm" rel="nofollow noopener" target="_blank">FileInstall</a> command. In this campaign, the attackers incorporate malicious scripts/executables alongside a legitimate application to disguise their intentions.</p>

<h3>VjW0rm/Houdini chain</h3>
<p>The attack chain that delivers the VjW0rm and Houdini RAT is the first one we saw using this Loader. This attack chain was first used utilizes thein February 2021 and is still in use as of today. We observed several changes over time and we will describe them below:</p>
<h4>Version 1</h4>
<p><strong>First seen:</strong> February 17, 2021<br><strong>Hash:</strong> 40e8b99b36739c397f8f0da2ab40f62b3af3da8b3f43fc2537841a9bf9105584</p>

<p style="text-align: center; font-size: 14px;"><strong>Figure 3: </strong><span style="font-weight: normal;">The</span> first version of the AHK script</p>
<p>First, the <em>AHK script</em> drops a legitimate application to the <em>%appdata%</em> directory and executes it. Next, it drops two files into the <em>%programdata% </em>directory. The first file is called <em>conhost.exe </em>and the second file is called <em>conhost.exe.manifest (malicious manifest). </em>Then the script executes the legitimate conhost.exe application, which leads to the execution of the malicious manifest through a path hijack.</p>
<p>Those files are the outcome of a tool called <a href="https://www.vbsedit.com/" rel="nofollow noopener" target="_blank"><span>VbsEdit</span></a>. The attacker uses VBsEdit to convert the VjW0rm and Houdini VBScript into an executable.</p>

<p>The tool creates a manifest (XML) file that holds the base64 encoded VBScript and information about how to execute the script. This manifest file needs to be located alongside the launcher (called conhost.exe). The launcher itself is a legitimate tool without any detections in VirusTotal.</p>

<h2>Fingerprinting the campaign</h2>
<p>In this campaign, we described various attack chains. We can attribute them to the same actor based on the following:</p>
<ul>
<li aria-level="1">They all drop a legitimate application before performing any malicious activity.</li>
<li aria-level="1">They have the same resource naming convention across all of the versions: *.MP4, KELLVBS.VBS, CONHOST.EXE, etc.</li>
<li aria-level="1">The AHK script has a strong resemblance across all of the chains, using the same commands: FileInstall, run, sleep, and drop the files to the %ProgramData% directory.</li>
<li aria-level="1">In several attack chains, we observed the same directory spamming technique.</li>
<li aria-level="1">They use the same scripts and UAC bypass technique to disable the Defender (in different stages).</li>
</ul>
<h2>Conclusion</h2>
<p>As threat actors study baseline security controls like emulators, antivirus, and UAC, they develop techniques to bypass and evade them. The technique changes detailed in this report did not affect the impact of these campaigns. The tactical goals remained the same. Rather, the technique changes were to bypass passive security controls. A common denominator among these evasive techniques is the abuse of process memory because it's typically a static and predictable target for the adversary.</p>
<p>We still need these baseline controls to keep the automated attacks at bay. But the manual tradecraft employed by innovative attackers like this one requires a modern approach to security. <span>Morphisec</span>&nbsp;offers control and visibility of these baseline controls while adding <a href="https://www.morphisec.com/products/morphisec-guard-endpoint-protection" rel="noopener">advanced breach prevention for in-memory exploits</a> and evasive fileless techniques like those used in these campaigns.
