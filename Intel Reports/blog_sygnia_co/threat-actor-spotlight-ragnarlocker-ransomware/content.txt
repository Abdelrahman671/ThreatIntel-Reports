<h2 class="wp-block-heading" id="1">Key Takeaways</h2>



<ul class="wp-block-list">
<li>The FBI’s Internet Crime Complaint Center (IC3) recently published <a href="https://www.ic3.gov/Media/PDF/AnnualReport/2022_IC3Report.pdf" rel="noreferrer noopener" target="_blank">their internet crime report for 2022</a>. The report indicates that during 2022 there was an increase in ransomware attacks, and the reported cases resulted in a loss of more than $34.3 million.</li>



<li>The report indicates that during 2022, the IC3 received 870 complaints regarding ransomware infection from organizations belonging to 14 out of 16 critical infrastructure sectors (e.g., healthcare, critical manufacturing, energy services etc.).</li>



<li>These statistics prompted Sygnia to publish this blog on RagnarLocker, a threat actor which uses double-extortion tactics since early 2020 and targets, among others, the critical infrastructure sector.</li>



<li>The blog post describes RagnarLocker ransomware attack flow:
<ul class="wp-block-list">
<li>Initial access was obtained by exploiting a known vulnerability in external-facing remote service. This reinforces the need for patch management and prioritization of commonly exploited vulnerabilities, especially regarding external facing systems.&nbsp;</li>



<li>Sygnia’s investigation revealed several customized batch scripts utilized by RagnarLocker operators to conduct reconnaissance activities against the Windows Event Logs and to deploy the ransomware binary in the network.</li>



<li>RagnarLocker utilized a remote administration tool that is less common with ransomware groups, Remote Manipulator System (RMS), as a Command &amp; Control mechanism. The blog provides information on how to investigate the forensic artifacts left by this tool on the compromised host.&nbsp;</li>



<li>Another remote access tool, AnyDesk, was used for data exfiltration. Using legitimate administrative tools allowed the threat actor to remain under the radar and bypass security controls while siphoning terabytes of sensitive data.</li>
</ul>
</li>
</ul>



<h2 class="wp-block-heading" id="2">Introduction</h2>



<p>RagnarLocker is both the name of a ransomware strain and of a criminal group which develops and operates it. Their data leakage blog appeared in April 2020, but although they’re an experienced group, RagnarLocker never made it to the top 10 ransomware strains. That being said, the group’s targeting of organizations in the critical infrastructure sectors (as <a href="https://www.cisa.gov/topics/critical-infrastructure-security-and-resilience/critical-infrastructure-sectors" rel="noreferrer noopener" target="_blank">defined by CISA</a>) makes them very dangerous.&nbsp;</p>



<p>The FBI has released two flash notifications about the group’s activities (<a href="https://www.waterisac.org/system/files/articles/FLASH-MU-000140-MW.pdf" rel="noreferrer noopener" target="_blank">Flash MU-000140-MW</a> on November 19, 2020 and <a href="https://www.ic3.gov/Media/News/2022/220307.pdf" rel="noreferrer noopener" target="_blank">Flash CU-000163-MW</a> on March 7, 2022), the last of which warned that as of January 2022, “at least 52 entities across 10 critical infrastructure sectors [were] affected by RagnarLocker ransomware”.</p>



<p>Sygnia investigated a double-extortion attack which involved RagnarLocker ransomware and wanted to shed light on some of the group’s interesting Tactics, Techniques and Procedures (TTPs) to assist organizations in mitigating similar attacks.&nbsp;</p>



<h2 class="wp-block-heading" id="3">Ragnarlocker attack flow</h2>



<p>A RagnarLocker compromise is very similar to the attack flow of other ransomware groups. Initial access was obtained by exploiting a software vulnerability in an external facing service. <a href="https://www.ic3.gov/Media/PDF/AnnualReport/2022_IC3Report.pdf" rel="noreferrer noopener" target="_blank">Based on IC3</a>, this is one of the top three initial access vectors utilized by ransomware groups. Moreover, the exploited service is part of CISA’s <a href="https://www.cisa.gov/known-exploited-vulnerabilities-catalog" rel="noreferrer noopener" target="_blank">Known Exploited Vulnerabilities (KEV) catalog</a>, a fact which demonstrates the importance of prioritizing the patching of these vulnerabilities.</p>



<p>After gaining initial access, the threat actor used the folder ‘C:\users\public’ as a staging directory.</p>



<p>During the discovery phase, several tools were executed from this folder. The threat actor combined commercial tools (e.g., Famatech’s Advanced Port Scanner, which was utilized to scan the internal networks), tools developed by Microsoft (e.g., Sysinternal’s PsInfo which was used to gather information about the operation systems along with Dsquery which was used to query the active directory) and custom tools.&nbsp;</p>



<h2 class="wp-block-heading" id="4">Utilizing windows event logs for discovery</h2>



<p>One of RagnarLocker’s custom tools was a PowerShell script called ‘logs.ps1’, which was used to gather information about remote desktop sessions from Windows Event logs. The script takes a list of computer names (‘name.txt’) to query and performs a search against the System event log (event IDs 7001, 7002) and the Microsoft-Windows-TerminalServices-LocalSessionManager/Operational Event log (event IDs 21, 25) for IP addresses and usernames. This information is used by the threat actor to determine which accounts are allowed to perform Remote Desktop connections and between which systems.&nbsp;</p>



<figure class="wp-block-table aligncenter"><table><tbody><tr><td>$k=&#8221;===========================================================================================================&#8221;<br>$s=&#8221;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8212;&#8211;&#8220;<br>$n1=&#8221;7001 7002&#8243;<br>$n2=&#8221;21 25&#8243;<br>$name = @(Get-Content name.txt)<br>&nbsp;&nbsp;&nbsp;&nbsp;ForEach($PCname in $name)<br>&nbsp;&nbsp;&nbsp;&nbsp;{&nbsp;<br>$k&gt;&gt;event.txt<br>$PCname &gt;&gt; event.txt<br>$n1 &gt;&gt;event.txt<br>Invoke-Command -ScriptBlock {Get-EventLog -LogName System | ?{(7001,7002) -contains $_.EventID }| %{<br>(new-object -Type PSObject -Property @{<br>TimeCreated = $_.TimeGenerated<br>ID=$_.EventID<br>ClientIP = $_.Message -replace &#8216;(?smi).*Source Network Address:*&#8217;<br>&nbsp;<br>})<br>} | sort TimeCreated -Descending | Select TimeCreated, ID, ClientIP } -ComputerName $PCname &gt;&gt; event.txt<br>&nbsp;<br>$s&gt;&gt;event.txt<br>&nbsp;<br>$n2 &gt;&gt;event.txt<br>&nbsp;<br>Invoke-Command -ScriptBlock {Get-WinEvent -LogName Microsoft-Windows-TerminalServices-LocalSessionManager/Operational | ?{(21,25) -contains $_.ID }| %{<br>(new-object -Type PSObject -Property @{<br>TimeCreated = $_.TimeCreated<br>ID=$_.ID<br>ClientIP = $_.Message -replace &#8216;(?smi).*Source Network Address:*&#8217;<br>UserName = $_.Message -replace &#8216;(?smi).*User:*&#8217;<br>})<br>} | sort TimeCreated -Descending | Select TimeCreated, ID, ClientIP, Username } -ComputerName $PCname &gt;&gt; event.txt<br>}</td></tr></tbody></table><figcaption class="wp-element-caption"><em>Figure 1: Reconstruction of ‘logs.ps1’ PowerShell script, which extracts IP addresses and usernames from both System and Terminal Services event logs</em></figcaption></figure>



<p>Lateral movement was mostly based on RDP, utilizing the ‘logs.ps1’ script output to identify target systems. After compromising a new system, the threat actor used the Windows Task Manager to dump the LSASS process and obtain credentials.&nbsp;</p>



<h2 class="wp-block-heading" id="5">Command &amp; control mechanisms</h2>



<p>The threat actors used two legitimate remote administration tools to establish a command &amp; control channel and exfiltrate data: Remote Manipulator System (RMS) and AnyDesk.</p>



<p>RMS is a Russian remote desktop software developed by TektonIT, that it is free for non-commercial use. The tool was reportedly used by several threat actors, such as <a href="https://e.cyberint.com/hubfs/Report%20Legit%20Remote%20Access%20Tools%20Turn%20Into%20Threat%20Actors%20Tools/CyberInt_Legit%20Remote%20Access%20Tools%20Turn%20Into%20Threat%20Actors'%20Tools_Report.pdf" rel="noreferrer noopener" target="_blank">TA505</a> (the group behind, among others, Locky ransomware), <a href="https://ssu.gov.ua/uploads/files/DKIB/Technical%20report%20Armagedon.pdf" rel="noreferrer noopener" target="_blank">Gamaredon</a> (a Russian espionage group with ties to the FSB) and many other smaller cybercriminal groups.</p>



<p>RMS allows the user to carry out a variety of tasks including full control of remote hosts, file transfer, power control, and terminal usage. The software can connect directly to an IP address (on port TCP/5650), or it can use ‘internet ID’ connection, in which case the traffic will be routed through RMS relay servers (on port TCP/5655 with a fallback to port 443). The latter is the preferred method as it can traverse NAT and maintains an open connection with the compromised system.</p>



<p>RMS was not installed under its default location (‘C:\Program Files (x86)\Remote Manipulator System &#8211; Host’), but instead it was installed under ‘C:\Program Files (x86)\System’, probably to hide the tool’s presence on the compromised host.&nbsp;</p>



<p>During the installation process, a service called ‘TektonIT &#8211; RMS Host’ was created on the system.</p>


<div class="wp-block-image">
<figure class="aligncenter size-full"><img decoding="async" width="587" height="386" src="data:image/svg+xml,%3Csvg%20xmlns='http://www.w3.org/2000/svg'%20viewBox='0%200%20587%20386'%3E%3C/svg%3E" alt="" class="wp-image-1169" data-lazy-srcset="https://www.sygnia.co/wp-content/uploads/2024/01/figure02.png 587w, https://www.sygnia.co/wp-content/uploads/2024/01/figure02-300x197.png 300w" data-lazy-sizes="(max-width: 587px) 100vw, 587px" data-lazy-src="https://www.sygnia.co/wp-content/uploads/2024/01/figure02.png" /><noscript><img decoding="async" width="587" height="386" src="https://www.sygnia.co/wp-content/uploads/2024/01/figure02.png" alt="" class="wp-image-1169" srcset="https://www.sygnia.co/wp-content/uploads/2024/01/figure02.png 587w, https://www.sygnia.co/wp-content/uploads/2024/01/figure02-300x197.png 300w" sizes="(max-width: 587px) 100vw, 587px" /></noscript><figcaption class="wp-element-caption">Figure 2: Snippet from Windows event log EID 7045, showing the RMS service which was created during installation</figcaption></figure></div>


<p>RagnarLocker threat actors used the tool ‘HideUL’ (‘Hide from Uninstall List’) as a defense evasion technique. The tool masked the presence of RMS on the compromised servers, by removing it from the ‘Add/Remove Programs’ interface of Windows. This was achieved by deleting the RMS entry from the registry key: ‘HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Uninstall’.</p>



<p>Another remote administration tool which was utilized by the RagnarLocker threat actor was AnyDesk. The tool was widely deployed using a batch script (see below), which allowed the threat actor to maintain access as well as to exfiltrate sensitive information.</p>



<p>The threat actors used SoftPerfect’s Lan Search Pro to perform a keyword search through the network and identified sensitive documents to exfiltrate. For example, they searched documents containing the word “confidential” and exfiltrated them.</p>



<h2 class="wp-block-heading" id="6">Ransomware deployment routine</h2>



<p>RagnarLocker operators heavily used PsExec as part of their ransomware deployment routine. First, they used the ‘net’ command to create a local user called ‘Defau1t’ and add it to the ‘local administrators’ group on at least 40 systems.&nbsp;</p>



<p>Next, a batch script named ‘any.bat’ was executed by PsExec. The script installed the remote desktop software AnyDesk, configured it to start on Windows startup and set a password for unattended access. The same password was used to configure the ‘Defau1t’ user account.</p>

<h2 class="wp-block-heading" id="9">Defending against Ragnarlocker</h2>



<p>Ransomware families share TTPs. The reasons for that are varied (e.g., the popularity of the Ransomware-as-a-Service tactic, reusing of tools and techniques to lower the costs of an attack, etc.), but by following the guidelines described below, you will improve your overall posture against most ransomware families.</p>



<ul class="wp-block-list">
<li>Update systems on a regular basis. Prioritize external facing systems and utilize <a href="https://www.cisa.gov/known-exploited-vulnerabilities-catalog" rel="noreferrer noopener" target="_blank">CISA Known Exploited Vulnerabilities (KEV) catalogue</a> as part of the patch management workflow.</li>



<li>Define a list of approved remote administration tools for your organization and utilize application control solutions to block or monitor the use of unauthorized tools.</li>



<li>Implement two factor authentication on all external facing interfaces.</li>



<li>Limit outbound internet access for servers to reduce the risk of data exfiltration and command and control (C2) communication. For servers that require outbound connectivity, restricting the outbound communication based on specific destinations and ports can help reduce the risk. Secure Web Gateways (SWG) can be used to protect endpoints.&nbsp;</li>



<li>Implement a PIM/PAM solution to securely manage privileged accounts to minimize the risk of privileged account misuse. Utilize Just-In-Time (JIT) access to further improve security in the environment.&nbsp;</li>
</ul>



<h2 class="wp-block-heading" id="10">Hunting for Ragnarlocker</h2>



<ul class="wp-block-list">
<li><strong>Look for processes and scripts executed from suspicious locations</strong>, such as the ‘c:\users\public’, ‘c:\PerfLogs’ and temp folders.&nbsp;</li>



<li><strong>Look for abnormal service execution</strong> based on name, executed program, and frequency of execution.</li>



<li><strong>Hunt for local accounts and group changes.</strong> This can be achieved by analyzing command-line values for processes such as cmd.exe, net.exe, net1.exe, wmic.exe and powershell.exe. A supplementary approach is to use Windows event logs as the hunt data source. Available logs include Event ID 4720 (A user account was created) and Event ID 4732 (a member was added to a security enabled local group).</li>



<li>L<strong>ook for suspicious LSASS memory access.</strong> Focus on abnormal cross-process activity, such as rundll32.exe or taskmgr.exe which open handle to lsass.exe. Another hunt scenario is to search for suspicious child process of lsass.exe, such as cmd.exe and powershell.exe</li>



<li><strong>Hunt for creation of LSASS dump file in temp folders.</strong> This can be achieved by analyzing file creation events where the Windows task manager process (taskmgr.exe) creates a .dmp file (lsass.dmp) in folders such as %LocalAppData%\temp.</li>



<li>Using the available network telemetry, look for data uploads to IP addresses which belong to RMS, AnyDesk, and sites like mega.io.</li>



<li><strong>Hunt for anomalous PowerShell commands.</strong> The RagnarLocker operators executed PowerShell remotely, without using encoded commands. Statistical analysis of module logging (event ID 4103) and script block logging (event ID 4104) event logs can help identify outliers that represent anomalous PowerShell commands. By cross-referencing these outliers with process execution logs (event ID 4688 or Sysmon event ID 1), it’s possible to detect malicious activities.</li>
</ul>
