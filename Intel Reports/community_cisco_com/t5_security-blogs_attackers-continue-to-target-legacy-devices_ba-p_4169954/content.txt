		<div class="lia-message-body-content">
			
				
					
					
						<P>Attackers will always target the "low hanging fruit": devices that have passed <A href="https://www.cisco.com/c/en/us/products/eos-eol-policy.html" target="_self" rel="nofollow noopener noreferrer">end-of-software maintenance and end-of-support</A>. A few years ago, Cisco described the <A href="https://blogs.cisco.com/security/evolution-of-attacks-on-cisco-ios-devices" target="_self" rel="nofollow noopener noreferrer">evolution of attacks against infrastructure devices.</A>&nbsp;All of the attacks discussed in that article targeted devices that have reached the end-of-sale milestone for several years! Too<SPAN>&nbsp;</SPAN>many organizations are relying on seriously outdated network components and operating systems—thus providing even more opportunity for adversaries to infiltrate or attack their network.&nbsp;<SPAN>In fact, some organizations continue to run network infrastructure software versions that are more than 8 years old (which likely are exposed to many unpatched known vulnerabilities).</SPAN></P>
<H2 id="toc-hId-1251862712">Adversarial Tactics and Techniques</H2>
<P>The malware used in the aforementioned <A href="https://blogs.cisco.com/security/evolution-of-attacks-on-cisco-ios-devices" target="_self" rel="nofollow noopener noreferrer">evolved Cisco IOS attacks</A> show increasing levels of complexity in the type of modifications made to legacy Cisco IOS, the behavior of Command and Control (C2) communication, and the platforms they target. The following are some of the most noticeable tactics and techniques used by adversaries in those attacks:</P>
<UL>
<LI>Automated exfiltration via traffic duplication by using modified SPAN ports.</LI>
<LI>Weaken encryption:&nbsp;<SPAN>Diffie-Hellman keyspace was reduced by attackers.</SPAN></LI>
<LI><SPAN>Legacy Cisco IOS Software was modified to disable crypto hardware acceleration.</SPAN></LI>
<LI><SPAN>The ROMMON on the targeted Cisco device was modified to ensure persistence of the command and control channel.</SPAN></LI>
<LI><SPAN>Attackers&nbsp;leveraged modified ROMMON code in order to inject binary code into the in-memory Cisco IOS image to support data exfiltration.</SPAN></LI>
<LI><SPAN>The usage of&nbsp;adversary-controlled TFTP servers in order to load malicious software to the targeted infrastructure device.</SPAN></LI>
<LI><SPAN>Attackers used the Simple Network Management Protocol (SNMP) with stolen credentials to retrieve the compromised device configuration.</SPAN></LI>
<LI><SPAN>Attackers&nbsp;exfiltrated device-specific data via ICMP packets.</SPAN></LI>
<LI><SPAN>Crafted ICMP packets were also used to trigger unwanted device behavior that helped the attacker to further manipulate the compromised device.</SPAN></LI>
<LI><SPAN>Some of the modified software images included keylogging mechanisms designed to capture the network administrator keystrokes.</SPAN></LI>
</UL>
<P>The Cisco PSIRT has also seen multiple attacks against legacy protocols, such as <A href="https://blog.talosintelligence.com/2017/02/cisco-coverage-for-smart-install-client.html" target="_self" rel="nofollow noopener noreferrer">Smart Install</A>. According to <A href="https://www.shodan.io/search?query=smart+install+client+active" target="_self" rel="nofollow noopener noreferrer">Shodan</A>, there are <A href="https://www.shodan.io/search?query=smart+install+client+active" target="_self" rel="nofollow noopener noreferrer">thousands of devices</A> that are still running Smart Install.&nbsp;Cisco Smart Install is a legacy feature that provides zero-touch deployment for new switches, typically access layer switches. Customers who are seeking more than zero-touch deployment should consider deploying the Cisco Network Plug and Play solution instead.</P>
<P>The main weakness of the Smart Install protocol is the lack of authorization or authentication mechanisms in between the client and the director. This can allow a client to process crafted Smart Install protocol messages as if these messages were from the Smart Install director. The following are some of the techniques used by attackers leveraging Smart Install enabled devices:</P>
<UL>
<LI>Attackers changed the TFTP server address on&nbsp;integrated branch clients (IBC).</LI>
<LI>Attackers copied arbitrary files from the affected device to an attacker-controlled TFTP server</LI>
<LI>The target device's startup configuration (startup-config) file was replaced with a file that the attacker prepared. Attackers then forced a reload of the IBC after a defined time interval; subsequently, booting with the new startup-config.</LI>
<LI>In some cases, adversaries loaded attacker-supplied firmware onto the compromised device.</LI>
<LI>Attackers were able to execute high-privileged CLI commands on the affected device (including do-exec CLI commands).</LI>
</UL>
<P>&nbsp;</P>
<H2 id="toc-hId--555591751">A Call to Action</H2>
<P><SPAN>Upgrading infrastructure devices is a big undertaking and in some cases requires network downtime. However, the costs of ignoring the problem of aging infrastructure and running legacy protocols can run much higher. Modern network infrastructure devices now come with numerous security features and capabilities that mitigate all the aforementioned attacks. The<A href="https://www.cisco.com/c/en/us/about/trust-center/technology-built-in-security.html#~processes" target="_self" rel="nofollow noopener noreferrer"> Cisco Secure Development Lifecycle (SDL)</A> applies industry-leading practices and technology to build trustworthy solutions that have fewer field-discovered product security incidents.&nbsp;Because network infrastructure devices play a fundamental role in the growth and trajectory of a business, forward-looking leaders and engineers must have a resilience strategy that includes investing in updating IT infrastructure.&nbsp;</SPAN></P>
