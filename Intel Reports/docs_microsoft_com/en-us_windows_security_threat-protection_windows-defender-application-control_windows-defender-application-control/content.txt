											<h2 id="ms--in-this-article" class="title is-6 margin-block-xs">In this article</h2>
										</nav><!-- <content> --><div class="NOTE">
<p>Note</p>
<p>Some capabilities of App Control for Business are only available on specific Windows versions. Learn more about <a href="feature-availability" data-linktype="relative-path">App Control feature availability</a>.</p>
</div>
<p>With thousands of new malicious files created every day, using traditional methods like antivirus solutions-signature-based detection to fight against malware-provides an inadequate defense against new attacks.</p>
<p>In most organizations, information is the most valuable asset, and ensuring that only approved users have access to that information is imperative. However, when a user runs a process, that process has the same level of access to data that the user has. As a result, sensitive information could easily be deleted or transmitted out of the organization if a user knowingly or unknowingly runs malicious software.</p>
<p>Application control can help mitigate these types of security threats by restricting the applications that users are allowed to run and the code that runs in the System Core (kernel). Application control policies can also block unsigned scripts and MSIs, and restrict Windows PowerShell to run in <a href="/en-us/powershell/module/microsoft.powershell.core/about/about_language_modes" data-linktype="absolute-path">Constrained Language Mode</a>.</p>
<p>Application control is a crucial line of defense for protecting enterprises given today's threat landscape, and it has an inherent advantage over traditional antivirus solutions. Specifically, application control moves away from an application trust model where all applications are assumed trustworthy to one where applications must earn trust in order to run. Many organizations, like the Australian Signals Directorate, understand the significance of application control and frequently cite application control as one of the most effective means for addressing the threat of executable file-based malware (.exe, .dll, etc.).</p>
<div class="NOTE">
<p>Note</p>
<p>Although application control can significantly harden your computers against malicious code, we recommend that you continue to maintain an enterprise antivirus solution for a well-rounded enterprise security portfolio.</p>
</div>
<p>Windows 10 and Windows 11 include two technologies that can be used for application control depending on your organization's specific scenarios and requirements:</p>
<ul>
<li><strong>App Control for Business</strong>; and</li>
<li><strong>AppLocker</strong></li>
</ul>
<h2 id="app-control-and-smart-app-control">App Control and Smart App Control</h2>
<p>Starting in Windows 11 version 22H2, <a href="https://support.microsoft.com/topic/what-is-smart-app-control-285ea03d-fa88-4d56-882e-6698afdb7003" data-linktype="external">Smart App Control</a> provides application control for consumers. Smart App Control is based on App Control, allowing enterprise customers to create a policy that offers the same security and compatibility with the ability to customize it to run line-of-business (LOB) apps. To make it easier to implement this policy, an <a href="design/example-appcontrol-base-policies" data-linktype="relative-path">example policy</a> is provided. The example policy includes <strong>Enabled:Conditional Windows Lockdown Policy</strong> option that isn't supported for App Control enterprise policies. This rule must be removed before you use the example policy. To use this example policy as a starting point for creating your own policy, see <a href="design/create-appcontrol-policy-for-lightly-managed-devices#create-a-custom-base-policy-using-an-example-app-control-base-policy" data-linktype="relative-path">Create a custom base policy using an example App Control base policy</a>.</p>
<p>Smart App Control is only available on clean installation of Windows 11 version 22H2 or later, and starts in evaluation mode. Smart App Control is automatically turned off for enterprise managed devices unless the user has turned it on first. To turn off Smart App Control across your organization's endpoints, you can set the <strong>VerifiedAndReputablePolicyState</strong> (DWORD) registry value under <code>HKLM\SYSTEM\CurrentControlSet\Control\CI\Policy</code> as shown in the following table. After you change the registry value, you must either restart the device or use <a href="operations/citool-commands#refresh-the-app-control-policies-on-the-system" data-linktype="relative-path">CiTool.exe -r</a> for the change to take effect.</p>

<div class="IMPORTANT">
<p>Important</p>
<p>Once you turn Smart App Control off, it can't be turned on without resetting or reinstalling Windows.</p>
</div>
<h3 id="smart-app-control-enforced-blocks">Smart App Control Enforced Blocks</h3>
<p>Smart App Control enforces the <a href="design/microsoft-recommended-driver-block-rules" data-linktype="relative-path">Microsoft Recommended Driver Block rules</a> and the <a href="design/applications-that-can-bypass-appcontrol" data-linktype="relative-path">Microsoft Recommended Block Rules</a>, with a few exceptions for compatibility considerations. The following aren't blocked by Smart App Control:</p>
<ul>
<li>Infdefaultinstall.exe</li>
<li>Microsoft.Build.dll</li>
<li>Microsoft.Build.Framework.dll</li>
<li>Wslhost.dll</li>
</ul>
