<h1><span id="hs_cos_wrapper_name" class="hs_cos_wrapper hs_cos_wrapper_meta_field hs_cos_wrapper_type_text" style="" data-hs-cos-general-type="meta_field" data-hs-cos-type="text">Malicious helpers: VS Code Extensions observed stealing sensitive information</span></h1>
<div id="hs_cos_wrapper_module_1577203992502740" class="hs_cos_wrapper hs_cos_wrapper_widget hs_cos_wrapper_type_module" style="" data-hs-cos-general-type="widget" data-hs-cos-type="module">
<h2 class="post-header__secondary-head blog__post-secondary-h2">Two newly discovered extensions on the VS Code Marketplace are designed to steal sensitive information, showing that open source attacks are expanding.</h2>
</div>
</div>
<div class="blog-author__short">
<div class="blog-author__short-info">

<div id="hs_cos_wrapper_module_1705062190107298" class="hs_cos_wrapper hs_cos_wrapper_widget hs_cos_wrapper_type_module" style="" data-hs-cos-general-type="widget" data-hs-cos-type="module"></div>
<div class="blog__post-content html-content">
<span id="hs_cos_wrapper_post_body" class="hs_cos_wrapper hs_cos_wrapper_meta_field hs_cos_wrapper_type_rich_text" style="" data-hs-cos-general-type="meta_field" data-hs-cos-type="rich_text"><p><img src="https://www.reversinglabs.com/hs-fs/hubfs/Blog/VS-Code-Extensions.webp?width=1400&amp;height=732&amp;name=VS-Code-Extensions.webp" width="1400" height="732" loading="lazy" alt="Malicious helpers: VS Code Extensions observed stealing sensitive information" style="height: auto; max-width: 100%; width: 1400px;" srcset="https://www.reversinglabs.com/hs-fs/hubfs/Blog/VS-Code-Extensions.webp?width=700&amp;height=366&amp;name=VS-Code-Extensions.webp 700w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/VS-Code-Extensions.webp?width=1400&amp;height=732&amp;name=VS-Code-Extensions.webp 1400w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/VS-Code-Extensions.webp?width=2100&amp;height=1098&amp;name=VS-Code-Extensions.webp 2100w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/VS-Code-Extensions.webp?width=2800&amp;height=1464&amp;name=VS-Code-Extensions.webp 2800w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/VS-Code-Extensions.webp?width=3500&amp;height=1830&amp;name=VS-Code-Extensions.webp 3500w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/VS-Code-Extensions.webp?width=4200&amp;height=2196&amp;name=VS-Code-Extensions.webp 4200w" sizes="(max-width: 1400px) 100vw, 1400px"></p>
<p>In the last few years, there has been a <a href="/blog/the-state-of-software-supply-chain-security-2024-key-takeaways" rel="noopener">dramatic rise (1300%) in supply chain attacks across multiple public repositories</a>. ReversingLabs’ researchers have been monitoring them daily to detect malicious packages. After packages are detected, the team notifies administrators for these public repositories, and encourages them to take the offending packages down if they are still up. With these threat hunting efforts, the RL research team is trying to raise awareness of the threat posed to software producers and their customers when malicious packages are added to the development cycle.&nbsp;<br><br>Up until recently, malicious actors were mostly active in two public repositories: npm and PyPI. ReversingLabs has discovered several malicious campaigns in those repositories in recent months. In January, I wrote about <a href="/blog/gitgot-cybercriminals-using-github-to-store-stolen-data" rel="noopener">malicious npm packages that leverage GitHub to store stolen Base64-encrypted SSH keys</a>. And in February, RL researcher <span>Petar Kirhmajer</span> <a href="/blog/attackers-leverage-pypi-to-sideload-malicious-dlls" rel="noopener">wrote about malicious PyPI packages that were observed using sideloading to execute code</a>.<br><br>As we have expanded our monitoring efforts to include other popular, public repositories such as RubyGems and most notably NuGet, we have come across malicious campaigns affecting users of those package managers as well. RL threat researcher Karlo Zanki <a href="/blog/iamreboot-malicious-nuget-packages-exploit-msbuild-loophole" rel="noopener">wrote in October 2023 about the newest malicious NuGet packages</a> where malicious functionality is placed inside the <span style="font-style: italic;">&lt;packageID&gt;</span>.targets file in the “build'' directory. More recently, one of our researchers <a href="/blog/suspicious-nuget-package-grabs-data-from-industrial-systems" rel="noopener">discovered another suspicious NuGet package</a><a href="/blog/suspicious-nuget-package-grabs-data-from-industrial-systems" rel="noopener" style="font-style: italic;">, SqzrFramework480</a>, that may be targeting developers working with technology made by a China-based firm that does industrial- and digital equipment manufacturing.&nbsp;<br><br>In the last couple of months, our attention expanded to include the Visual Studio Code Marketplace (or VS Code Marketplace). VS Code Marketplace is a popular, online platform where developers publish, share, and install extensions that enhance Microsoft’s Visual Studio Code (VS Code) open-source code editor. Historically, VS Code Marketplace hasn’t been a popular destination for malicious actors interested in pushing malicious wares via open source packages - at least based on our observations. There have only been a few research blogs that exposed malicious VS Code extensions, the majority of which were published in the first half of 2023 year.<br><br>The ReversingLabs research team decided to dig into the VS Code Marketplace and see if threats might be lurking there. In the process, the team discovered detected multiple, malicious extensions engineered to steal data and linked to the same author: <span style="font-style: italic;">VSAnalysistest</span>. These extensions — named <span style="font-style: italic;">clipboard-helper-vscode, code-ai-assistant; codegpt-helper, </span>and <span style="font-style: italic;">mycodegpt-assistant — </span>were removed from VS Marketplace at the time this research took place. &nbsp;<br><br>While these extensions were not very complicated, the RL research team believed it was worth documenting what we discovered so that organizations that rely on the VS Code Marketplace are informed about the presence of supply chain threats on that platform. Here's what we found — and steps development organizations can take to identify VS Code Marketplace threats.</p>
<!--more-->
<h2 style="font-size: 24px; font-weight: normal;">Discussion</h2>
<p>Previously observed malicious VS Code extensions mentioned in various blogs published last year ranged from <a href="https://blog.checkpoint.com/securing-the-cloud/malicious-vscode-extensions-with-more-than-45k-downloads-steal-pii-and-enable-backdoors/" rel="noopener" target="_blank">simple beacon-backs used in red team operations, to infostealers grabbing sensitive data and exfiltrating it using a Discord webhook</a> to <a href="https://www.aquasec.com/blog/can-you-trust-your-vscode-extensions/" rel="noopener" target="_blank">downloaders executing possibly malicious code</a>.&nbsp;<br><br>The newly discovered malicious extensions are simple in their design. The difference from previously reported VS Code campaigns? Code provenance. In the previously documented malicious VS Code campaigns, the code in the malicious extensions closely resembled malicious code often found on other open source platforms, for example: in malicious npm packages (Figure 1). However, in the extensions we discovered, the code is different and appears to be borrowed from tutorials for writing VS Code extensions published by Microsoft.</p>
<a href="https://blog.reversinglabs.com/hubfs/Blog/vscode-blog-figure-01.webp" class="lightbox-image"><img src="https://www.reversinglabs.com/hs-fs/hubfs/Blog/vscode-blog-figure-01.webp?width=1400&amp;name=vscode-blog-figure-01.webp" alt="code found in malicious VS Code extension" width="1400" style="width: 1400px;" srcset="https://www.reversinglabs.com/hs-fs/hubfs/Blog/vscode-blog-figure-01.webp?width=700&amp;name=vscode-blog-figure-01.webp 700w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/vscode-blog-figure-01.webp?width=1400&amp;name=vscode-blog-figure-01.webp 1400w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/vscode-blog-figure-01.webp?width=2100&amp;name=vscode-blog-figure-01.webp 2100w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/vscode-blog-figure-01.webp?width=2800&amp;name=vscode-blog-figure-01.webp 2800w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/vscode-blog-figure-01.webp?width=3500&amp;name=vscode-blog-figure-01.webp 3500w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/vscode-blog-figure-01.webp?width=4200&amp;name=vscode-blog-figure-01.webp 4200w" sizes="(max-width: 1400px) 100vw, 1400px"></a>
<p style="font-weight: bold; font-size: 16px; text-align: center;">Figure 1: code found in malicious VS Code extension k3s0externobyes.k3s0externobyes</p>
<p>The VS Code extension that raised some red flags and prompted further investigation was <span style="font-style: italic;">clipboard-helper-vscode</span>, a package by name that may be intended to fool incautious developers into installing it. This extension is advertised as a Visual Studio Code add-on “designed to enhance the clipboard functionality within VS Code." It only consists of one command that the user can use when an extension is added to the VS Code: "helloWorld".<br><br>The package was flagged due to a threat hunting policy applied by the <a href="/products/software-supply-chain-security" rel="noopener">ReversingLabs Spectra Assure platform</a> that identified a Discord webhook string in the <span style="font-style: italic;">extension.js </span>file.&nbsp;<br><br>By the time RL researchers detected the extension, it had already been removed from the marketplace, so there was no need to report it to maintainers. It is unclear whether it was removed by the author(s) or was taken down as a result of automated scanning by Microsoft, or notification by another third-party firm.</p>
<a href="https://blog.reversinglabs.com/hubfs/Blog/vscode-blog-figure-02.webp" class="lightbox-image"><img src="https://www.reversinglabs.com/hs-fs/hubfs/Blog/vscode-blog-figure-02.webp?width=1400&amp;name=vscode-blog-figure-02.webp" alt="Threat hunting policy of clipboard-helper-vscode" width="1400" style="width: 1400px;" srcset="https://www.reversinglabs.com/hs-fs/hubfs/Blog/vscode-blog-figure-02.webp?width=700&amp;name=vscode-blog-figure-02.webp 700w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/vscode-blog-figure-02.webp?width=1400&amp;name=vscode-blog-figure-02.webp 1400w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/vscode-blog-figure-02.webp?width=2100&amp;name=vscode-blog-figure-02.webp 2100w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/vscode-blog-figure-02.webp?width=2800&amp;name=vscode-blog-figure-02.webp 2800w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/vscode-blog-figure-02.webp?width=3500&amp;name=vscode-blog-figure-02.webp 3500w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/vscode-blog-figure-02.webp?width=4200&amp;name=vscode-blog-figure-02.webp 4200w" sizes="(max-width: 1400px) 100vw, 1400px"></a>
<p style="font-weight: bold; font-size: 16px; text-align: center;">Figure 2: Threat hunting policy of clipboard-helper-vscode</p>
<p>When it comes to packages found on other open source platforms like npm, the presence of a Discord webhook has a high correlation with malicious activity. That’s because Discord is often used as a platform to transfer stolen information, attempting to disguise the malicious data exfiltration as benign behavior. That was the case with <span style="font-style: italic;">clipboard-helper-vscode</span> extension as well.<br><br>In the VS Code extension, authors can declare “<a href="https://code.visualstudio.com/api/references/activation-events" rel="noopener">activation events</a>” in the activationEvents field of <span style="font-style: italic;">package.json</span>. When the activation events happen, the extension is activated, and the function named <a href="https://code.visualstudio.com/api/get-started/extension-anatomy#:~:text=activate%20is%20executed%20when%20your%20registered%20Activation%20Event%20happens" rel="noopener" style="font-style: italic;">activate</a> exported by the extension entry file is executed. In the case of the <span style="font-style: italic;">clipboard-helper-vscode</span> extension, when the command <span style="font-style: italic;">chatgpt.updateAPIkey</span> is executed (Figure 3), the extension is activated and the function <span style="font-style: italic;">activate</span> in extension’s entry file <span style="font-style: italic;">extension.js</span> is executed.&nbsp;<br><br></p>
<p><img src="https://www.reversinglabs.com/hs-fs/hubfs/Blog/vscode-blog-figure-03.webp?width=446&amp;height=104&amp;name=vscode-blog-figure-03.webp" width="446" height="104" loading="lazy" alt="activation events in clipboard-helper-vscode" style="height: auto; max-width: 100%; width: 446px; margin-left: auto; margin-right: auto; display: block;" srcset="https://www.reversinglabs.com/hs-fs/hubfs/Blog/vscode-blog-figure-03.webp?width=223&amp;height=52&amp;name=vscode-blog-figure-03.webp 223w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/vscode-blog-figure-03.webp?width=446&amp;height=104&amp;name=vscode-blog-figure-03.webp 446w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/vscode-blog-figure-03.webp?width=669&amp;height=156&amp;name=vscode-blog-figure-03.webp 669w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/vscode-blog-figure-03.webp?width=892&amp;height=208&amp;name=vscode-blog-figure-03.webp 892w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/vscode-blog-figure-03.webp?width=1115&amp;height=260&amp;name=vscode-blog-figure-03.webp 1115w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/vscode-blog-figure-03.webp?width=1338&amp;height=312&amp;name=vscode-blog-figure-03.webp 1338w" sizes="(max-width: 446px) 100vw, 446px"></p>
<p style="font-weight: bold; font-size: 16px; text-align: center;"><br>Figure 3: activation events in clipboard-helper-vscode</p>
<h2 style="font-size: 24px; font-weight: normal;">Clipboard-helper-vscode: threat or test?</h2>
<p>Upon activation of an extension, the <em>clipboard-helper-vscode</em> checks every two seconds to see if the stored clipboard data has changed. If it has, the new clipboard data is sent over a Discord webhook. The data exfiltration capabilities of this extension are limited, however the counter for max sending number was set to “3”.<br>&nbsp;<br>That limit, coupled with the structure of the code and the name of the extension and the publisher, <em>VSAnalysistest</em>, suggest that this extension was intended only for testing purposes. If its purpose was malicious, it may be that the extension was still in the early stages of development when it was discovered, and that later versions would be more subtle and fully featured.&nbsp;</p>
<a href="https://blog.reversinglabs.com/hubfs/Blog/vscode-blog-figure-04.webp" class="lightbox-image"><img src="https://www.reversinglabs.com/hs-fs/hubfs/Blog/vscode-blog-figure-04.webp?width=1400&amp;name=vscode-blog-figure-04.webp" alt="extension.js file from clipboard-helper-vscode" width="1400" style="width: 1400px;" srcset="https://www.reversinglabs.com/hs-fs/hubfs/Blog/vscode-blog-figure-04.webp?width=700&amp;name=vscode-blog-figure-04.webp 700w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/vscode-blog-figure-04.webp?width=1400&amp;name=vscode-blog-figure-04.webp 1400w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/vscode-blog-figure-04.webp?width=2100&amp;name=vscode-blog-figure-04.webp 2100w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/vscode-blog-figure-04.webp?width=2800&amp;name=vscode-blog-figure-04.webp 2800w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/vscode-blog-figure-04.webp?width=3500&amp;name=vscode-blog-figure-04.webp 3500w, https://www.reversinglabs.com/hs-fs/hubfs/Blog/vscode-blog-figure-04.webp?width=4200&amp;name=vscode-blog-figure-04.webp 4200w" sizes="(max-width: 1400px) 100vw, 1400px"></a>
<p style="text-align: center;"><span style="font-weight: bold; font-size: 16px;">Figure 4: extension.js file from clipboard-helper-vscode</span></p>
<h2 style="font-size: 24px; font-weight: normal;">Code-ai-assistant: phishing for credentials</h2>
<p>Another extension published by the same author, <span style="font-style: italic;">code-ai-assistant</span>, functions in a similar way to <span style="font-style: italic;">clipboard-helper-vscode</span>. The extension is marketed as “an innovative VS Code extension designed to enhance coding efficiency and streamline your development workflow.” In reality, the extension is used for exfiltrating data.<br><br>There are more activation events for it. When one of these events happens, the extension is activated, and a function <span style="font-style: italic;">activate</span> in the file <span style="font-style: italic;">extension.js</span> is executed. Developers that deployed the <span style="font-style: italic;">code-ai-assistant</span> VS Code extension are prompted with an input box asking for an “API key”. Placeholder text inside prompts the developer to enter their GitHub password, which is then saved and exported over a Discord webhook when the <span style="font-style: italic;">helloWord</span> command is invoked. As with the <span style="font-style: italic;">clipboard-helper-vscode</span> extension, this suggests that <span style="font-style: italic;">code-ai-assistant </span>may simply be test code: <span style="font-style: italic;">helloWorld</span> is the first command found in VS Code tutorial <a href="https://code.visualstudio.com/api/get-started/your-first-extension" rel="noopener" target="_blank">Your First Extension</a>.<br><br>The remaining two detected extensions, <span style="font-style: italic;">codegpt-helper</span>; and mycodegpt-assistant, are similar to two already described. The extension <span style="font-style: italic;">codegpt-helper</span> is very similar to the code-ai-assistant extension. The API key user credentials are entered in the input box and then sent over the Discord webhook. The extension <span style="font-style: italic;">mycodegpt-assistant</span> takes the “easycode.openAI ApiKey” and sends it, again over the Discord webhook.</p>
<h2 style="font-size: 24px; font-weight: normal;">Impact is limited</h2>
<p>The two malicious extensions we discovered were published between late 2023 and the end of January 2024. At the time of publication, the extensions have been removed from VS Code Marketplace. Before being removed, they were downloaded a small number of times. Based on statistics provided by <a href="https://github.com/microsoft/vscode-vsce" rel="noopener" target="_blank">vsce</a>, all four VS Code extensions together were installed about 50 times, and downloaded far fewer, about five times.<br><br>That means the impact of this malicious “campaign” is almost certainly limited. It is hard to tell without more research how they were removed from the VS Code Marketplace, or by whom. Microsoft <a href="https://code.visualstudio.com/docs/editor/extension-marketplace#_can-i-trust-extensions-from-the-marketplace" rel="noopener" target="_blank">has some safety measures</a> for extensions that are being published on VS Code Marketplace, so there is possibility they have been removed by VS Code Marketplace administrators after being flagged as malicious by their automatic virus scanner.<br><br>There is also a possibility that they have been removed by the author, or reported by someone and then removed by VS Code Marketplace administrators - possibly evidence that the extensions were intended as experiments or meant for educational purposes and removed soon after being posted.</p>
<h2 style="font-size: 24px; font-weight: normal;">Conclusion</h2>
<p>Malicious packages are nothing new. ReversingLabs researchers detect a few dozen malicious packages every day on various public repositories. These malicious packages vary from simple infostealers and downloaders to reverse shells and more complex and interesting packages.&nbsp;<br><br>What is new is that the attacks are leveraging less-popular repositories like Microsoft’s VS Code Marketplace. Their presence on the VS Code Marketplace suggests that developers looking for extensions and other shared code need to be aware of the risk of software supply chain attacks. Like other open source package managers, VS Code Marketplace has some features for detecting malicious extensions. However, malicious actors still have an incentive to sidestep detection and publish malicious software, even if it is just for a short time.&nbsp;<br><br>In this case, the two Visual Studio Code extensions were simple in design and borrowed heavily from sample code published by Microsoft as an introduction to VS Code. That suggests these extensions were likely experiments or — at least — early iterations of planned malware. Their short lifespan and the limited number of downloads make clear that this campaign was not a major threat to VS Code Marketplace developers.&nbsp;<br><br>What these extensions do prove is that malicious actors are finding new methods for writing software and new systems for their publishing. Developers as well as security experts need to be wary of threats always lurking on all public code repositories, not just “the usual suspects.” Before including a public library in their development cycle, developers need to conduct security assessments to see if those libraries and extensions are safe to use and free of malicious or suspicious features, such as Discord webhooks.&nbsp;<br><br>Fortunately, developers have tools to help them detect malicious content before being included in their software. Check out <a href="/products/software-supply-chain-security" rel="noopener">ReversingLabs Spectra Assure</a> to learn more about how we work with developers to ensure their code is secure.&nbsp;</p>
