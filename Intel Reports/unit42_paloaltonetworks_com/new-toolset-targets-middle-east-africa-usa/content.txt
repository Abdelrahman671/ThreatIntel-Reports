A Network Provider module is a DLL component implementing the interface provided by Microsoft to support additional types of network protocols during the authentication process.\r\n\r\nThis technique is pretty well documented. Sergey Polak demonstrated the technique at BlackHat back in 2004 at his session titled \u201cCapturing Windows Passwords using the Network Provider API.\u201d In 2020, researcher Grzegorz Tworek uploaded his tool NPPSpy to GitHub, which also implements this technique.\r\n\r\nDue to the file naming patterns of the DLL module, and as a reference to the previous research and tools, Unit 42 researchers named this malware family Ntospy. The threat actor registers the Ntospy DLL module as a Network Provider module to hijack the authentication process, to get access to the user credentials every time the victim attempts to authenticate to the system.\r\n\r\nFigure 1 illustrates the path of the processes the malware used during the authentication process to load the malicious DLL module in an MS Exchange Server environment.\r\n\r\n[caption id=\"attachment_131407\" align=\"aligncenter\" width=\"600\"] Figure 1. Main function of the malware sample.[\/caption]\r\n\r\nWith that pattern, the threat communicates with the C2 server by adding additional subdomains to build the DNS query. <p>Unit 42 researchers observed a series of apparently related attacks against organizations in the Middle East, Africa and the U.S. We will discuss a set of tools used in the course of the attacks that reveal clues about the threat actors’ activity. We are sharing this research to provide detection, prevention and hunting recommendations to help organizations strengthen their overall security posture.</p>
<p>&nbsp;</p>
<p>These tools were used to perform the following activities:</p>
<ul>
<li>Establish backdoor capabilities</li>
<li>For command and control (C2)</li>
<li>Steal user credentials.</li>
<li>Exfiltrate confidential information</li>
</ul>
<p>Unit 42 is sharing these results with the purpose of helping organizations defend against the tools observed here.</p>
<p>We assess with medium confidence that this threat activity cluster aligns to nation-state related threat actors due to the nature of the organizations that were compromised, the TTPs observed and the customization of the tool set. We have not confirmed a particular nation-state or threat group.</p>
<p>Tools that were used in this cluster were the following:</p>
<ul>
<li>A new backdoor we’ve named Agent Racoon
<ul>
<li>This malware family is written using the .NET framework and leverages the domain name service (DNS) protocol to create a covert channel and provide different backdoor functionalities. Threat actors have used this along with the other two tools in multiple attacks targeting organizations across the U.S., Middle East and Africa. Its C2 infrastructure dates back to 2020.</li>
</ul>
</li>
<li>A new tool we’ve named Ntospy
<ul>
<li>This malware is a Network Provider DLL module designed to steal user credentials.</li>
</ul>
</li>
<li>A customized version of Mimikatz called Mimilite</li>
</ul>
<p>The compromised organizations belong to the following industries:</p>
<ul>
<li>Education</li>
<li>Real estate</li>
<li>Retail</li>
<li>Non-profit organizations</li>
<li>Telecom companies</li>
<li>Governments</li>
</ul>
<p>Based on unique similarities in tools as well as tactics, techniques and procedures (TTPs), we are tracking this <a href="https://unit42.paloaltonetworks.com/from-activity-to-formal-naming/">threat activity cluster</a> as CL-STA-0002.</p>
<p>What follows is a detailed description of the activity we observed as well as characteristics of the tool set.</p>
<p>Palo Alto Networks customers receive protection from these threats through <a href="https://www.paloaltonetworks.com/cortex/cortex-xdr" target="_blank" rel="noopener">Cortex XDR</a> as well as <a href="https://docs.paloaltonetworks.com/advanced-url-filtering/administration" target="_blank" rel="noopener">Advanced URL Filtering</a>, <a href="https://docs.paloaltonetworks.com/dns-security">DNS Security</a> and <a href="https://docs.paloaltonetworks.com/wildfire">Advanced Wildfire</a>. Organizations can engage the Unit 42 Incident Response team for specific assistance with this threat and others.</p>

<h2><a id="post-131403-_ksyqmuosq9kn"></a><strong>Activity Summary</strong></h2>
<p>The threat actor used temporary directories such as <span style="font-family: 'courier new', courier, monospace;">C:\Windows\Temp and C:\Temp</span> to deploy specific components of their tool set across the different affected organizations. They used the following similar filenames for batch and PowerShell scripts:</p>
<ul>
<li><span style="font-family: 'courier new', courier, monospace;">c:\windows\temp\crs.ps1</span></li>
<li><span style="font-family: 'courier new', courier, monospace;">c:\windows\temp\ebat.bat</span></li>
<li><span style="font-family: 'courier new', courier, monospace;">c:\windows\temp\install.bat</span></li>
<li><span style="font-family: 'courier new', courier, monospace;">c:\windows\temp\mslb.ps1</span></li>
<li><span style="font-family: 'courier new', courier, monospace;">c:\windows\temp\pb.ps1</span></li>
<li><span style="font-family: 'courier new', courier, monospace;">c:\windows\temp\pb1.ps1</span></li>
<li><span style="font-family: 'courier new', courier, monospace;">c:\windows\temp\pscan.ps1</span></li>
<li><span style="font-family: 'courier new', courier, monospace;">c:\windows\temp\set_time.bat</span></li>
<li><span style="font-family: 'courier new', courier, monospace;">c:\windows\temp\usr.ps1</span></li>
</ul>
<p>While the attackers commonly used <a href="#post-131403-_wven14kmgum2" target="_blank" rel="noopener">Ntospy</a> across the affected organizations, the <a href="https://unit42.paloaltonetworks.com/?p=131403&amp;preview=true#post-131403-_uoz73eltqdwp" target="_blank" rel="noopener">Mimilite</a> tool and the <a href="#post-131403-_50t4n6xlnq0" target="_blank" rel="noopener">Agent Racoon</a> malware have only been found in nonprofit and government-related organizations’ environments.</p>
<p>After each attack session, the threat actor leveraged <span style="font-family: 'courier new', courier, monospace;">cleanmgr.exe</span> to clean up the environment used during the session.</p>
<h2><a id="post-131403-_wven14kmgum2"></a><strong>Gaining Access to Credentials with Ntospy</strong></h2>
<p>To perform credential theft, the threat actor used a custom DLL module implementing a <a href="https://learn.microsoft.com/en-us/windows/win32/secauthn/network-provider-api" target="_blank" rel="noopener">Network Provider</a>. A Network Provider module is a DLL component implementing the interface provided by Microsoft to support additional types of network protocols during the authentication process.</p>
<p>This technique is pretty well documented. Sergey Polak demonstrated the technique at BlackHat back in 2004 at his session titled “<a href="https://www.blackhat.com/presentations/win-usa-04/bh-win-04-polak/bh-win-04-polak2.pdf" target="_blank" rel="noopener">Capturing Windows Passwords using the Network Provider API</a>.” In 2020, researcher <a href="https://twitter.com/0gtweet" target="_blank" rel="noopener">Grzegorz Tworek</a> uploaded his tool <a href="https://github.com/gtworek/PSBits/tree/master/PasswordStealing/NPPSpy" target="_blank" rel="noopener">NPPSpy</a> to GitHub, which also implements this technique.</p>
<p>Due to the file naming patterns of the DLL module, and as a reference to the previous research and tools, Unit 42 researchers named this malware family Ntospy. The threat actor registers the Ntospy DLL module as a Network Provider module to hijack the authentication process, to get access to the user credentials every time the victim attempts to authenticate to the system.</p>
<p>Figure 1 illustrates the path of the processes the malware used during the authentication process to load the malicious DLL module in an MS Exchange Server environment.</p>
<figure id="attachment_131407" aria-describedby="caption-attachment-131407" style="width: 600px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-131407" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/11/word-image-131403-1.png" alt="Image 1 is a screenshot of the different paths of the processes loading the DLL module in the Microsoft Exchange Server environment." width="600" height="183" /><figcaption id="caption-attachment-131407" class="wp-caption-text">Figure 1. Image path of processes loading the malicious DLL component in an MS Exchange environment.</figcaption></figure>
<p>The threat actor’s implementation of this technique has some unique features. They created different versions of the Ntospy malware over the time frame we observed. They all share similarities, such as the following:</p>
<ul>
<li>Using filenames with Microsoft patch patterns.</li>
<li><span style="font-family: 'courier new', courier, monospace;">.msu</span> extensions pretending to be Microsoft Update Package files to store the received credentials in cleartext.</li>
<li>RichPE header hashes that link different samples to the same compilation environment.</li>
</ul>
<p>To install the DLL module, the threat actor registers a new Network Provider called <span style="font-family: 'courier new', courier, monospace;">credman</span>. They do so by using an installation script found at <span style="font-family: 'courier new', courier, monospace;">C:\Windows\Temp\install.bat</span> that installs the Network Provider by using <span style="font-family: 'courier new', courier, monospace;">reg.exe</span>. The malware then sets the DLL module path by pointing to the malicious DLL module <span style="font-family: 'courier new', courier, monospace;">c:\windows\system32\ntoskrnl.dll</span>.</p>
<p><img decoding="async" class="aligncenter wp-image-131409" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/11/word-image-131403-2.png" alt="Many lines of code. Attacker registers as credman." width="600" height="298" /></p>

<h2><a id="post-131403-_50t4n6xlnq0"></a><strong>Agent Racoon Backdoor</strong></h2>
<p>The Agent Racoon malware family is built to provide backdoor capabilities. It is written using the .NET framework, and leverages DNS to establish a covert channel with the C2 server. Unit 42 researchers named the malware family Agent Racoon due to some references found within the code of the identified samples, as shown in Figure 7.</p>
<figure id="attachment_131421" aria-describedby="caption-attachment-131421" style="width: 616px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-131421" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/11/word-image-131403-8.png" alt="Image 7 is a screenshot of the .NET project details. There are entries for Assembly name, which is raccoon; Default namespace, which is agent; Target framework, which is .NET Framework 4; Output type, which is Console Application; and Startup object, which is agent.Program. Auto-generate binding redirects is not selected." width="616" height="188" /><figcaption id="caption-attachment-131421" class="wp-caption-text">Figure 7. .NET Project details.</figcaption></figure>
<p>When executed, the threat has some predefined settings such as:</p>
<ul>
<li>The base domain used to create the DNS covert channel</li>
<li>A unique key per sample, used as a seed to generate an encryption password to encrypt the DNS communication</li>
<li>A fallback DNS server if no DNS server can be read from the compromised system</li>
</ul>
<p>All the C2 domains identified fulfill the same base pattern, with unique values for the four character identifier across different samples:</p>
<p style="padding-left: 40px;"><span style="font-family: 'courier new', courier, monospace;">[4 characters].telemetry.[domain].com</span></p>
<p>The value of <span style="font-family: 'courier new', courier, monospace;">Program.dns_ip</span> is different for each sample found, which could indicate that the threat actor is building the binary with specific settings gathered from the targeted environment.</p>
<figure id="attachment_131423" aria-describedby="caption-attachment-131423" style="width: 900px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-131423" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/11/word-image-131403-9.png" alt="Image 8 is a screenshot of many lines of code. Main function of the malware sample.</figcaption></figure>
<p>With that pattern, the threat communicates with the C2 server by adding additional subdomains to build the DNS query. It uses Internationalizing Domain Names for Applications’ (IDNA) domain names with Punycode encoding. This encoding type is a representation of Unicode values over the ASCII encoding for internet hostnames.</p>
<p>The domain names follow the pattern below:</p>
<p style="padding-left: 40px;"><span style="font-family: 'courier new', courier, monospace;">[random_val].a.[4 characters].telemetry.[domain].com</span></p>
<p>The screenshot from Wireshark in Figure 9 illustrates a complete DNS query:</p>
<figure id="attachment_131425" aria-describedby="caption-attachment-131425" style="width: 900px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-131425" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/11/word-image-131403-10.png" alt="Image 9 is a screenshot of many lines of code. <p>To manage the communication with the C2 server, the malware uses a communication loop shown in Figure 10.</p>
<figure id="attachment_131427" aria-describedby="caption-attachment-131427" style="width: 603px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-131427" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/11/word-image-131403-11.png" alt="Image 10 is a screenshot of many lines of code. <p>The following are some main features of the communication loop above:</p>
<ul>
<li>The communication loop finishes when the answer <span style="font-family: 'courier new', courier, monospace;">xn--cc</span> is received from the C2 server, or a communication error occurs.</li>
<li>The randomized delay between messages can have multiple reasons:
<ul>
<li>To avoid network spikes.</li>
<li>To avoid potential network congestion.</li>
<li>To provide randomness as an attempt to avoid network beaconing detection.</li>
</ul>
</li>
<li>The encryption of all the communication messages through <span style="font-family: 'courier new', courier, monospace;">Program.Util.RC</span>.</li>
</ul>
<p>The encryption routine implements a stream cipher that takes the initial unique key per sample <span style="font-family: 'courier new', courier, monospace;">Program.key</span> (<span style="font-family: 'courier new', courier, monospace;">this.defaultkey</span>), as shown in Figure 11. It then creates a 1-byte encryption key to later encrypt the message with an XOR.</p>
<figure id="attachment_131429" aria-describedby="caption-attachment-131429" style="width: 391px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-131429" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/11/word-image-131403-12.png" alt="Image 11 is a screenshot of many lines of code. Two red arrows point to instances of (this.defaultkey) and one red arrow points to the “message” line inside brackets. <p>Depending on the length of the message sent to the C2 server, different subdomains are added to the query, as shown in the code snippet in Figure 12.</p>

<h2><a id="post-131403-_9km75qqrfzx"></a><strong>Data Exfiltration</strong></h2>
<p>Unit 42 researchers also identified the collection and successful exfiltration of confidential information, such as emails from MS Exchange environments, using PowerShell snap-ins to dump the emails.</p>
<p><img decoding="async" class="aligncenter wp-image-131441" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/11/word-image-131403-18.png" alt="PowerShell snap-in code to dump emails" width="700" height="93" /></p>
<p>In the search criteria from the command above, the threat actor used similar commands to search through different folders, mailboxes and dates to dump those emails.</p>
<p>After dumping the emails, the threat actor tried to compress the <span style="font-family: 'courier new', courier, monospace;">.pst</span> file with a command-line RAR tool before exfiltrating it:</p>
<p><img decoding="async" class="aligncenter wp-image-131443" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/11/word-image-131403-19.png" alt="Command-line RAR tool to compress the .pst file. " width="700" height="75" /></p>
<p>However, the threat actor canceled the attempt to compress the <span style="font-family: 'courier new', courier, monospace;">.pst</span> file by using the tool <span style="font-family: 'courier new', courier, monospace;">taskkill.exe</span> approximately eight minutes later.</p>
<p><img decoding="async" class="aligncenter wp-image-131445" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/11/word-image-131403-20.png" alt="PowerShell snap-in code to dump emails" width="700" height="41" /></p>
<p>Eventually the threat actor discarded the usage of <span style="font-family: 'courier new', courier, monospace;">raren.exe</span> and simply renamed the <span style="font-family: 'courier new', courier, monospace;">.pst</span> file, moving it to the IIS root directory and mimicking an error log in a compressed file to download it through the web server.</p>
<p><img decoding="async" class="aligncenter wp-image-131447" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/11/word-image-131403-21.png" alt="Command-line RAR tool to compress the .pst file. " width="700" height="64" /></p>
<p>And finally, the <span style="font-family: 'courier new', courier, monospace;">ai.pst</span> file is removed.</p>
<p><img decoding="async" class="aligncenter wp-image-131449" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/11/word-image-131403-22.png" alt="Use of tool taskkill to cancel the compression attempt. " width="700" height="45" /></p>
<p>This process is repeated for several mailboxes with different search criteria.</p>
<p>In addition to the email exfiltration, Unit 42 researchers identified exfiltration of the victim’s <a href="https://learn.microsoft.com/en-us/windows-server/storage/folder-redirection/folder-redirection-rup-overview" target="_blank" rel="noopener">Roaming Profile</a>. A Roaming Profile is used to serve the same profile to the user when logging in from different computers from the same Active Directory environment.</p>
<p>To exfiltrate this, the threat actor compressed the directory by using the standalone version of the 7-Zip tool (which they dropped into the system using <span style="font-family: 'courier new', courier, monospace;">certutil.exe</span>), and split the compressed file into chunks of 100 MB.</p>
<p><img decoding="async" class="aligncenter wp-image-131451" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/11/word-image-131403-23.png" alt="Removing the ai dot pst file. " width="700" height="48" /></p>
<p><img decoding="async" class="aligncenter wp-image-131453" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/11/word-image-131403-24.png" alt="Compressing the directory. Splitting the file into segments of 100 MB. " width="700" height="109" /></p>
<p>Later, following the same procedure, the threat actor exfiltrated the content.</p>
<p><img decoding="async" class="aligncenter wp-image-131455" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/11/word-image-131403-25.png" alt="Exfiltration of content. " width="700" height="103" /></p>
<h2><a id="post-131403-_cefib6i8eknk"></a><strong>Conclusion</strong></h2>
<p>Our hope in sharing the descriptions of this tool set is that readers can use this information to search their networks to identify other possible attacks using these tools. This tool set is not yet associated with a specific threat actor, and not entirely limited to a single cluster or campaign.</p>
<p>As mentioned at the beginning of this article, we found an overlapping Ntospy sample with SHA256 <span style="font-family: 'courier new', courier, monospace;">bcd2bdea2bfecd09e258b8777e3825c4a1d98af220e7b045ee7b6c30bf19d6df</span> with a <a href="https://www.paloaltonetworks.com/blog/security-operations/through-the-cortex-xdr-lens-uncovering-a-new-activity-group-targeting-governments-in-the-middle-east-and-africa/" target="_blank" rel="noopener">previously identified</a> threat activity cluster CL-STA-0043. However, the overlaps are not limited to that sample.</p>
<p>We have also identified two compromised organizations in common across both activity clusters. Some of the TTPs match on both clusters, such as the MS Exchange PowerShell snap-ins and one of the Network Provider DLL modules.</p>
<p>Unit 42 researchers believe this threat activity cluster aligns with medium confidence to nation-state related threat actors for the following reasons:</p>
<ul>
<li>The detection and defense evasion techniques used</li>
<li>The exfiltration activity observed</li>
<li>The victimology</li>
<li>The customization level of the tools used</li>
<li>The TTPs observed</li>
</ul>

		if ($(mail).val() != '' && isEmail($(mail).val())) {
			$(subscription_form_button).removeClass('is-disabled');
		}
		setTimeout(function() {
			$(email).focus();
			$('.g-recaptcha iframe').attr('tabindex', '-1');
		}, 100)
	}
	window.captchaExpires = function() {
		subscribeSuccess = false;
		$(subscription_form_button).addClass('is-disabled', true);
	} 
	$(subscription_form).submit(function(e) {
		e.preventDefault();
		e.stopImmediatePropagation();
		updateEmailMask();
		var success = true;
		var form = $(this);
		var mail = form.find('input[name="Email"]');
		
		if (mail.val() === '') {
			mail.addClass('has-error');
			showError(1);
			success = false;	
		}  else if (!isEmail(mail.val())){
			showError(2);
			success = false;
		}
		else {
			mail.removeClass('has-error');
			$('.error-mail').addClass('d-none');
		}

		if (!subscribeSuccess) {
			$('.error-recaptcha').removeClass('d-none');
		} else {
			$('.error-recaptcha').addClass('d-none');
		}

		if (success && subscribeSuccess) {
			$.ajax({
				type: 'POST',
				url: form.attr('action'),
				data: form.serialize(),

				success: function(msg) {
					form.find('.form-success-message').html('<p class="success-message">You have been successfully subscribed</p>');
					form.find('button').removeClass('is-loading');
					$(email).val('');
					clearError();
				},
				error: function(jqXHR, textStatus, errorThrown) {
					$(subscription_form_button).addClass('is-disabled', true);
					form.find('button').removeClass('is-loading');
					
				}
			});
		}
		return false;
	});
	function showError(error_type){
		if(error_type == 1) {
			$('.error-mail').text("Please enter the email address.").addClass('error-show');
			$(subscription_form_button).addClass('is-disabled');
		} else if(error_type == 2){
			$('.error-mail').text("Please provide a valid e-mail address.").addClass('error-show');
			$(subscription_form_button).addClass('is-disabled');
		}	
		$(subscription_form_button).removeClass('is-loading');
	}
	function clearError(){
		$('.error-mail').text("").removeClass('error-show');;
		$(subscription_form_button).removeClass('is-loading');
		$(subscription_form_button).removeClass('is-disabled');
	}
	
	$(email).on('input', function (event) { 
		var email = $(this).val();
		if (isEmail(email) ) {			
			clearError();
		} else if(email == ""){
			clearError();
		} else{
			showError(2);			  
		}
    });

	function isEmail(email) {
		var re = /^(([^<>()\[\]\\.,;:\s@"]+(\.[^<>()\[\]\\.,;:\s@"]+)*)|(".+"))@((\[[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\])|(([a-zA-Z\-0-9]+\.)+[a-zA-Z]{2,}))$/;
		return re.test(String(email).toLowerCase());
	}

               }
            });
        }
	
	function updateEmailMask() {
		var email = $("#unit42footerSubscription_form input[name='Email']").val();
		if (email && email.trim() != '') {
			var maskedEmail = maskEmailAddress(email);
			$("#unit42footerSubscription_form input[name='emailFormMask']").val(maskedEmail);
		}
	}

	function maskEmailAddress (emailAddress) {
		function mask(str) {
			var strLen = str.length;
			if (strLen > 4) {
				return str.substr(0, 1) + str.substr(1, strLen - 1).replace(/\w/g, '*') + str.substr(-1,1);
			}
			return str.replace(/\w/g, '*');
		}
		return emailAddress.replace(/([\w.]+)@([\w.]+)(\.[\w.]+)/g, function (m, p1, p2, p3) {
			return mask(p1) + '@' + mask(p2) + p3;
		});

		return emailAddress;
	}
