<p><em><strong>Introduction</strong></em></p>

<p>TA551 (also known as Shathak) represents e threat actor behind malspam that has pushed different families of malware over the past few years.&nbsp; TA551 previously distributed <a href="https://www.malware-traffic-analysis.net/2020/12/03/index2.html">Ursnif</a>, <a href="https://unit42.paloaltonetworks.com/valak-evolution/">Valak</a>, and <a href="https://unit42.paloaltonetworks.com/ta551-shathak-icedid/">IcedID</a>.</p>

<p>TA551 stopped sending IcedID sometime in June 2021 and <a href="https://www.malware-traffic-analysis.net/2021/06/30/index.html">began pushing Trickbot</a>.</p>

<p>By July 2021, TA551 stopped sending Trickbot and <a href="https://www.malware-traffic-analysis.net/2021/07/21/index.html">began pushing BazarLoader</a> (sometimes called BazaLoader).&nbsp; TA551 continues to push BazarLoader, and Cobalt Strike is often follow-up malware for these infections.</p>

<p>Today&#39;s diary reviews a TA551 BazarLoader infection followed by Cobalt Strike on Tuesday 2021-08-10.</p>

<p><a href="https://isc.sans.edu/diaryimages/images/2021-08-11-ISC-diary-image-00.jpg"><img alt="" src="https://isc.sans.edu/diaryimages/images/2021-08-11-ISC-diary-image-00a.jpg" style="border-width: 2px; border-style: solid;" /></a><br />
<em>Shown above:&nbsp; Flow chart for the chain of events in the infection on 2021-08-10.</em></p>

<p><em><strong>From email to document</strong></em></p>

<p>Examples of TA551 emails from Tuesday 2021-08-10 are not yet publicly available, but a recent example was submitted to VirusTotal from a wave last week on 2021-08-04.&nbsp; These emails have different passwords each day, and we often see different passwords for different emails during the same day.&nbsp; These emails spoof replies to previously valid emails, but they no longer include message text from the email chain.&nbsp; We only see subject lines and spoofed sending addresses from the previously valid emails.</p>

<p><a href="https://isc.sans.edu/diaryimages/images/2021-08-11-ISC-diary-image-02.jpg"><img alt="" src="https://isc.sans.edu/diaryimages/images/2021-08-11-ISC-diary-image-02a.jpg" style="border-width: 2px; border-style: solid;" /></a><br />
<em>Shown above:&nbsp; An example of TA551 malspam from 2021-08-04.</em></p>

<p>Attachments are currently named <em><strong>request.zip</strong></em> or <em><strong>info.zip</strong></em>.&nbsp; Potential victims would open these password-protected zip archives on a vulnerable Windows host using the password supplied in the message text.&nbsp; The extracted document uses a template that tells potential victims to enable macros.</p>

<p><a href="https://isc.sans.edu/diaryimages/images/2021-08-11-ISC-diary-image-03.jpg"><img alt="" src="https://isc.sans.edu/diaryimages/images/2021-08-11-ISC-diary-image-03.jpg" style="border-width: 2px; border-style: solid;" /></a><br />
<em>Shown above:&nbsp; Extracting the malicious TA551 Word doc from the attached zip archive.</em></p>

<p><em><strong>Kicking off an infection</strong></em></p>

<p>On a vulnerable Windows host, a victim would enable macros on the extracted document.&nbsp; Using an example from 2021-08-10, the document dropped an HTA file in the same directory as the document.&nbsp; This HTA file contains HTML and script designed to retrieve a malicious DLL to infect a vulnerable Windows host with BazarLoader.</p>

<p><a href="https://isc.sans.edu/diaryimages/images/2021-08-11-ISC-diary-image-05.jpg"><img alt="" src="https://isc.sans.edu/diaryimages/images/2021-08-11-ISC-diary-image-05b.jpg" style="border-width: 2px; border-style: solid;" /></a><br />

<p><a href="https://isc.sans.edu/diaryimages/images/2021-08-11-ISC-diary-image-06.jpg"><img alt="" src="https://isc.sans.edu/diaryimages/images/2021-08-11-ISC-diary-image-06a.jpg" style="border-width: 2px; border-style: solid;" /></a><br />
<em>Shown above:&nbsp; HTA file opened in Wordpad to show some of the malicious script.</em></p>

<p><em><strong>BazarLoader to Cobalt Strike</strong></em></p>

<p><a href="https://isc.sans.edu/diaryimages/images/2021-08-11-ISC-diary-image-07.jpg"><img alt="" src="https://isc.sans.edu/diaryimages/images/2021-08-11-ISC-diary-image-07a.jpg" style="border-width: 2px; border-style: solid;" /></a><br />
<em>Shown above:&nbsp; TCP stream showing the vulnerable host retrieving a malicious DLL for BazarLoader</em></p>

<p><a href="https://isc.sans.edu/diaryimages/images/2021-08-11-ISC-diary-image-08.jpg"><img alt="" src="https://isc.sans.edu/diaryimages/images/2021-08-11-ISC-diary-image-08a.jpg" style="border-width: 2px; border-style: solid;" /></a><br />
<em>Shown above:&nbsp; Traffic from the infection filtered in Wireshark.</em></p>

<p>After the infected host retrieved a DLL for BazarLoader, HTTPS traffic began for Bazar Command and Control (C2) activity.&nbsp; A malicious DLL for Cobalt Strike was sent through Bazar C2 traffic, then HTTPS traffic to <em><strong>xagadi[.]com</strong></em> began over <em><strong>23.106.223[.]174</strong></em> for Cobalt Strike.</p>

<p><em><strong>Cobalt Strike tunneling through HTTPS</strong></em></p>

<p>In recent weeks, we&#39;ve noticed HTTPS traffic acting as a tunnel for Cobalt Strike activity.&nbsp; Cobalt Stike URLs within this HTTPS traffic spoof commonly-used domains like <em><strong>bing.com</strong></em> or <em><strong>google</strong></em>.</p>

<p>Images below show decrypted HTTPS traffic from <a href="https://app.any.run/tasks/7ee2b245-5fd8-4d61-9d17-df4591a3cb03">Any.Run&#39;s sandbox analysis of the Cobalt Strike binary</a> found on our infected lab host. The pcap from this sandbox analysis has a decryption key, so we can see the actual URLs spoofing <em><strong>bing.com</strong></em> within HTTPS traffic to <em><strong>xagadi[.]com</strong></em>.</p>

<p><a href="https://isc.sans.edu/diaryimages/images/2021-08-11-ISC-diary-image-09.jpg"><img alt="" src="https://isc.sans.edu/diaryimages/images/2021-08-11-ISC-diary-image-09a.jpg" style="border-width: 2px; border-style: solid;" /></a><br />
<em>Shown above:&nbsp; Decrypted HTTPS traffic from any.run&#39;s sandbox analysis of Cobalt Strike sample from our infected Windows host.</em></p>

<p><a href="https://isc.sans.edu/diaryimages/images/2021-08-11-ISC-diary-image-10.jpg"><img alt="" src="https://isc.sans.edu/diaryimages/images/2021-08-11-ISC-diary-image-10a.jpg" style="border-width: 2px; border-style: solid;" /></a><br />
<em>Shown above:&nbsp; HTTP stream for initial GET request caused by the Cobalt Strike sample.</em></p>

<p><a href="https://isc.sans.edu/diaryimages/images/2021-08-11-ISC-diary-image-11.jpg"><img alt="" src="https://isc.sans.edu/diaryimages/images/2021-08-11-ISC-diary-image-11a.jpg" style="border-width: 2px; border-style: solid;" /></a><br />
<em>Shown above:&nbsp; HTTP stream for initial POST request caused by the Cobalt Strike sample.</em></p>

<p>We started seeing this HTTPS tunneling from Cobalt Strike samples this month (August 2021), but it might have started earlier.&nbsp; <a href="https://app.any.run/tasks/5e623021-e00e-4685-b361-d31eecc5b728/">Here&#39;s a similar sample of Cobalt Stike</a> from Monday 2021-08-09.&nbsp; It generated the same type of activity: URLs spoofing <em><strong>google.es</strong></em> tunneled through HTTPS traffic to <em><strong>gojihu[.]com</strong></em> and <em><strong>yuxicu[.]com</strong></em>, originally reported <a href="https://twitter.com/Unit42_Intel/status/1424829355704922114">here</a>.</p>

<p><em><strong>Final words</strong></em></p>

<p>For the past two months or so, I&#39;ve seen more BazarLoader being pushed than ever before.&nbsp; BazarLoader is currently sent through at least three different campaigns:</p>

<ul>
	<li>TA551 (Shathak) - example in this diary</li>
	<li>&quot;Stolen Images Evidence&quot; campaign - more info and a recent example <a href="https://github.com/pan-unit42/tweets/blob/master/2021-07-29-IOCs-for-BazarLoader-CobaltStrike-PrintNightmare.txt">here</a>.</li>
	<li>BazarCall - more info <a href="https://unit42.paloaltonetworks.com/bazarloader-malware/">here</a> and <a href="https://www.microsoft.com/security/blog/2021/07/29/bazacall-phony-call-centers-lead-to-exfiltration-and-ransomware/">here</a>.</li>
</ul>

<p>BazarLoader is commonly followed by Cobalt Strike when an infected host is part of an Active Directory (AD) environment.&nbsp; These infections reportedly deliver ransomware as a final payload in real-world environments (<a href="https://thedfirreport.com/2021/08/01/bazarcall-to-conti-ransomware-via-trickbot-and-cobalt-strike/">here</a> is one such example).</p>

<p>But decent spam filters and best security practices can help you avoid BazarLoader. Default security settings in Windows 10 and Microsoft Office 2019 should prevent these types of infections from happening.</p>

<p>However, it&#39;s a &quot;cat-and-mouse&quot; game.&nbsp; Malware developers create new ways to circumvent security measures, while vendors update their software, applications, and endpoint protection to address these new developments.&nbsp; Furthermore, mass-distribution methods like malspam remain cheap and profitable for cyber criminals.</p>

<p>Malware samples from this wave of TA551 and pcaps from the associated traffic can be found at <a href="https://www.malware-traffic-analysis.net/2021/08/10/index.html">here</a>.</p>
