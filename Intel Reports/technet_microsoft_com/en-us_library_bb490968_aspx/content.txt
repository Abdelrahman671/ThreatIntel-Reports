							<div class="content "><h1 id="ping">Ping</h1><div class="display-flex justify-content-space-between align-items-center flex-wrap-wrap page-metadata-container">
										<div class="margin-right-xxs">
											<ul class="metadata page-metadata" data-bi-name="page info" lang="en-us" dir="ltr"><li>Article</li><li>
														<time class="is-invisible" data-article-date aria-label="Article review date" datetime="2009-09-11T00:00:00.000Z" data-article-date-source="ms.date">09/11/2009</time>
													</li></ul>
										</div></div><nav id="center-doc-outline" class="doc-outline is-hidden-desktop display-none-print margin-bottom-sm" data-bi-name="intopic toc" aria-label="In this article">
											<h2 id="ms--in-this-article" class="title is-6 margin-block-xs">In this article</h2>
										</nav><!-- <content> --><p>Verifies IP-level connectivity to another TCP/IP computer by sending Internet Control Message Protocol (ICMP) Echo Request messages. The receipt of corresponding Echo Reply messages are displayed, along with round-trip times. Ping is the primary TCP/IP command used to troubleshoot connectivity, reachability, and name resolution. Used without parameters, <strong>ping</strong> displays help.</p>
<h3 id="syntax">Syntax</h3>
<p><strong>ping</strong>&nbsp;[<strong>-t</strong>] [<strong>-a</strong>] [<strong>-n</strong>&nbsp;<em>Count</em>] [<strong>-l</strong> <em>Size</em>] [<strong>-f</strong>] [<strong>-i</strong>&nbsp;<em>TTL</em>] [<strong>-v</strong>&nbsp;<em>TOS</em>] [<strong>-r</strong>&nbsp;<em>Count</em>] [<strong>-s</strong>&nbsp;<em>Count</em>] [{<strong>-j</strong>&nbsp;<em>HostList</em> | <strong>-k</strong> <em>HostList</em>}] [<strong>-w</strong>&nbsp;<em>Timeout</em>] [<em>TargetName</em>]</p>
<h3 id="parameters">Parameters</h3>
<p><strong>-t</strong> &nbsp; <strong>:</strong> Specifies that ping continue sending Echo Request messages to the destination until interrupted. To interrupt and display statistics, press CTRL-BREAK. To interrupt and quit ping, press CTRL-C.</p>
<p><strong>-a</strong> &nbsp; <strong>:</strong> Specifies that reverse name resolution is performed on the destination IP address. If this is successful, ping displays the corresponding host name.</p>
<p><strong>-n</strong> &nbsp; <em><strong>Count</strong></em> &nbsp; <strong>:</strong> Specifies the number of Echo Request messages sent. The default is 4.</p>
<p><strong>-l</strong> <em><strong>Size</strong></em> &nbsp; <strong>:</strong> Specifies the length, in bytes, of the Data field in the Echo Request messages sent. The default is 32. The maximum <em>size</em> is 65,527.</p>
<p><strong>-f</strong> &nbsp; <strong>:</strong> Specifies that Echo Request messages are sent with the Don't Fragment flag in the IP header set to 1. The Echo Request message cannot be fragmented by routers in the path to the destination. This parameter is useful for troubleshooting path Maximum Transmission Unit (PMTU) problems.</p>
<p><strong>-i</strong> <em><strong>TTL</strong></em> &nbsp; <strong>:</strong> Specifies the value of the TTL field in the IP header for Echo Request messages sent. The default is the default TTL value for the host. For Windows&nbsp;XP hosts, this is typically 128. The maximum <em>TTL</em> is 255.</p>
<p><strong>-v</strong> &nbsp; <em><strong>TOS</strong></em> &nbsp; <strong>:</strong> Specifies the value of the Type of Service (TOS) field in the IP header for Echo Request messages sent. The default is 0. <em>TOS</em> is specified as a decimal value from 0 to 255.</p>
<p><strong>-r</strong> &nbsp; <em><strong>Count</strong></em> &nbsp; <strong>:</strong> Specifies that the Record Route option in the IP header is used to record the path taken by the Echo Request message and corresponding Echo Reply message. Each hop in the path uses an entry in the Record Route option. If possible, specify a <em>Count</em> that is equal to or greater than the number of hops between the source and destination. The <em>Count</em> must be a minimum of 1 and a maximum of 9.</p>
<p><strong>-s</strong> &nbsp; <em><strong>Count</strong></em> &nbsp; <strong>:</strong> Specifies that the Internet Timestamp option in the IP header is used to record the time of arrival for the Echo Request message and corresponding Echo Reply message for each hop. The <em>Count</em> must be a minimum of 1 and a maximum of 4.</p>
<p><strong>-j</strong> &nbsp; <em><strong>HostList</strong></em> &nbsp; <strong>:</strong> Specifies that the Echo Request messages use the Loose Source Route option in the IP header with the set of intermediate destinations specified in <em>HostList</em>. With loose source routing, successive intermediate destinations can be separated by one or multiple routers. The maximum number of addresses or names in the host list is 9. The host list is a series of IP addresses (in dotted decimal notation) separated by spaces.</p>
<p><strong>-k</strong> &nbsp; <em><strong>HostList</strong></em> &nbsp; <strong>:</strong> Specifies that the Echo Request messages use the Strict Source Route option in the IP header with the set of intermediate destinations specified in <em>HostList</em>. With strict source routing, the next intermediate destination must be directly reachable (it must be a neighbor on an interface of the router). The maximum number of addresses or names in the host list is 9. The host list is a series of IP addresses (in dotted decimal notation) separated by spaces.</p>
<p><strong>-w</strong> &nbsp; <em><strong>Timeout</strong></em> &nbsp; <strong>:</strong> Specifies the amount of time, in milliseconds, to wait for the Echo Reply message that corresponds to a given Echo Request message to be received. If the Echo Reply message is not received within the time-out, the "Request timed out" error message is displayed. The default time-out is 4000 (4 seconds).</p>
<p><em><strong>TargetName</strong></em> &nbsp; <strong>:</strong> Specifies the destination, which is identified either by IP address or host name.</p>
<p><strong>/?</strong> &nbsp; <strong>:</strong> Displays help at the command prompt.</p>
<h3 id="remarks">Remarks</h3>
<ul>
<li><p>You can use <strong>ping</strong> to test both the computer name and the IP address of the computer. If pinging the IP address is successful, but pinging the computer name is not, you might have a name resolution problem. In this case, ensure that the computer name you are specifying can be resolved through the local Hosts file, by using Domain Name System (DNS) queries, or through NetBIOS name resolution techniques.</p>
</li>
<li><p>This command is available only if the <strong>Internet Protocol (TCP/IP)</strong> protocol is installed as a component in the properties of a network adapter in Network Connections</p>
</li>
</ul>
<h3 id="examples">Examples</h3>
<p>The following example shows <strong>ping</strong> command output:</p>
<p>C:\&gt;ping example.microsoft.com</p>
<p>Pinging example.microsoft.com [192.168.239.132] with 32 bytes of data:</p>
<p>Reply from 192.168.239.132: bytes=32 time=101ms TTL=124</p>
<p>Reply from 192.168.239.132: bytes=32 time=100ms TTL=124</p>
<p>Reply from 192.168.239.132: bytes=32 time=120ms TTL=124</p>
<p>Reply from 192.168.239.132: bytes=32 time=120ms TTL=124</p>
<p>To ping the destination 10.0.99.221 and resolve 10.0.99.221 to its host name, type:</p>
<p><strong>ping -a 10.0.99.221</strong></p>
<p>To ping the destination 10.0.99.221 with 10 Echo Request messages, each of which has a Data field of 1000 bytes, type:</p>
<p><strong>ping -n 10 -l 1000 10.0.99.221</strong></p>
<p>To ping the destination 10.0.99.221 and record the route for 4 hops, type:</p>
<p><strong>ping -r 4 10.0.99.221</strong></p>
<p>To ping the destination 10.0.99.221 and specify the loose source route of 10.12.0.1-10.29.3.1-10.1.44.1, type:</p>
<p><strong>ping -j 10.12.0.1 10.29.3.1 10.1.44.1 10.0.99.221</strong></p>
