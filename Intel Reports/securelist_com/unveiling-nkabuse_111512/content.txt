												<div class="c-wysiwyg">
													<p>During an incident response performed by Kaspersky&#8217;s Global Emergency Response Team (<a href="https://www.kaspersky.com/enterprise-security/incident-response" target="_blank" rel="noopener">GERT</a>) and GReAT, we uncovered a novel multiplatform threat named &#8220;NKAbuse&#8221;. The malware utilizes <a href="https://nkn.org" target="_blank" rel="noopener">NKN technology</a> for data exchange between peers, functioning as a potent implant, and equipped with both flooder and backdoor capabilities. Written in Go, it is flexible enough to generate binaries compatible with various architectures.</p>
<p>Our analysis suggests that the primary target of NKAbuse is Linux desktops. However, in view of its ability to infect MIPS and ARM systems, it also poses a threat to IoT devices.</p>
<p>NKAbuse infiltrates systems by uploading an implant to the victim host. The malware establishes persistence through a cron job and installs itself in the host&#8217;s home folder. Its capabilities span flooding to backdoor access to remote administration (RAT), offering a range of features.</p>
<h2 id="a-new-kind-of-network">A new kind of network</h2>
<p>NKN, short for &#8220;New Kind of Network&#8221;, functions as a peer-to-peer (P2P) and blockchain-oriented network protocol that prioritizes decentralization and privacy. The NKN network currently has more than 180,000 official nodes. It offers diverse routing algorithms designed to optimize data transmission by selecting the shortest node trajectory to reach its intended destination.</p>
<div id="attachment_111515" style="width: 1034px" class="wp-caption aligncenter"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13160959/Unveiling_NKAbuse_01.png" class="magnificImage"><img loading="lazy" decoding="async" aria-describedby="caption-attachment-111515" class="size-large wp-image-111515" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13160959/Unveiling_NKAbuse_01-1024x459.png" alt="NKN data routing diagram" width="1024" height="459" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13160959/Unveiling_NKAbuse_01-1024x459.png 1024w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13160959/Unveiling_NKAbuse_01-300x135.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13160959/Unveiling_NKAbuse_01-768x344.png 768w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13160959/Unveiling_NKAbuse_01-390x175.png 390w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13160959/Unveiling_NKAbuse_01-370x166.png 370w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13160959/Unveiling_NKAbuse_01-624x280.png 624w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13160959/Unveiling_NKAbuse_01-800x359.png 800w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13160959/Unveiling_NKAbuse_01.png 1512w" sizes="(max-width: 1024px) 100vw, 1024px" /></a><p id="caption-attachment-111515" class="wp-caption-text">NKN data routing diagram</p></div>
<p>Historically, malware operators have exploited new and emerging communication protocols like NKN to link up with their command-and-control servers (C2) or bot masters. This threat (ab)uses the NKN public blockchain protocol to carry out a large set of flooding attacks and act as a backdoor inside Linux systems.</p>
<h2 id="a-not-so-new-attack-vector">A not-so-new attack vector</h2>
<p>Evidence collected and analyzed by GERT suggests that this attack exploited an old vulnerability related to Struts2 (CVE-2017-5638 &#8211; Apache Struts2), targeting a financial company.</p>

<p style="text-align: center"><em><strong>Log evidence obtained from the exploited WebService</strong></em></p>
<p>The excerpt from the audit logs shown above is the same as that referenced in <a href="https://github.com/vulhub/vulhub/blob/master/struts2/s2-048/README.md" target="_blank" rel="noopener">the vulhub POC S2-048</a>. The vulnerability allows the attackers to execute commands on the server by passing the command in a header identified as &#8220;shell&#8221; and sending the instructions to Bash for execution. After the vulnerability is exploited, a command is executed on the system to download the initial script.</p>
<h2 id="a-new-multiplatform-implant">A new multiplatform implant</h2>
<p>The malware is typically installed on the victim&#8217;s device by executing a remote shell script that downloads and executes the contents of the <strong>setup.sh</strong> shell script hosted by the attacker remotely. The setup process checks the OS type and, depending on that, it downloads the second stage, which is the actual malware implant. The implant is downloaded from the same server; it is named &#8220;<strong>app_linux_{ARCH}&#8221;, </strong>where &#8220;{ARCH}&#8221; is the target OS architecture. The downloaded implant is placed into the temporary <strong>/tmp</strong> directory and then executed. There are eight architectures hosted on that server and supported by the malware:</p>
<ul>
<li>386</li>
<li>arm64</li>
<li>arm</li>
<li>amd64</li>
<li>mips</li>
<li>mipsel</li>
<li>mips64</li>
<li>mips64el</li>
</ul>
<p>This analysis will focus on the amd64 (x86-64) version.</p>
<p>Once executed, the malware checks if it is the only instance running and moves itself to a safe place instead of remaining in the volatile /tmp directory. The directory chosen by the implant to reside in is /root/.config/StoreService/. Another set of directories created inside that destination path is files and .cache. Then the implant retrieves the infected machine&#8217;s IP address by sending a GET request to <strong>ifconfig.me</strong>, loads the default configuration, which checks if it is located inside the .cache directory and, if not, loads certain hardcoded settings.</p>
<div id="attachment_111516" style="width: 677px" class="wp-caption aligncenter"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13161240/Unveiling_NKAbuse_02.png" class="magnificImage"><img loading="lazy" decoding="async" aria-describedby="caption-attachment-111516" class="size-full wp-image-111516" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13161240/Unveiling_NKAbuse_02.png" alt=" Loading and setting up the default configuration" width="667" height="562" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13161240/Unveiling_NKAbuse_02.png 667w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13161240/Unveiling_NKAbuse_02-300x253.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13161240/Unveiling_NKAbuse_02-208x175.png 208w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13161240/Unveiling_NKAbuse_02-370x312.png 370w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13161240/Unveiling_NKAbuse_02-332x280.png 332w" sizes="(max-width: 667px) 100vw, 667px" /></a><p id="caption-attachment-111516" class="wp-caption-text">Loading and setting up the default configuration</p></div>
<p>This configuration is then saved to a new cache structure, which holds other important repeatedly reused settings, such as the generated private key.</p>
<p>NKAbuse makes use of <strong>cron jobs</strong> to survive reboots. To achieve that, it needs to be root. It checks that the current user ID is 0 and, if so, proceeds to parse the current crontab, adding itself for every reboot.</p>
<h2 id="a-new-communication">A new communication</h2>
<p>NKAbuse utilizes the NKN protocol to communicate with the bot master and receive/send information. To do this, the malware implant creates a new account and a new <strong>multiclient</strong>, which enables it to send and receive data from multiple clients concurrently, increasing the reliability of its communications with the bot master.</p>
<p>The NKN account is created with the default config options, and then the multiclient is initialized with an identifier which in our sample is a 64 character string representing the <strong>public key and remote address</strong> used by the malware.</p>
<div id="attachment_111517" style="width: 711px" class="wp-caption aligncenter"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13161312/Unveiling_NKAbuse_03.png" class="magnificImage"><img loading="lazy" decoding="async" aria-describedby="caption-attachment-111517" class="size-full wp-image-111517" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13161312/Unveiling_NKAbuse_03.png" alt="NKAbuse setting up the NKN client structure with the help of a hardcoded public key" width="701" height="410" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13161312/Unveiling_NKAbuse_03.png 701w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13161312/Unveiling_NKAbuse_03-300x175.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13161312/Unveiling_NKAbuse_03-299x175.png 299w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13161312/Unveiling_NKAbuse_03-370x216.png 370w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13161312/Unveiling_NKAbuse_03-479x280.png 479w" sizes="(max-width: 701px) 100vw, 701px" /></a><p id="caption-attachment-111517" class="wp-caption-text">NKAbuse setting up the NKN client structure with the help of a hardcoded public key</p></div>

The handler contains 42 or so cases, each performing different actions depending on the &#8220;code&#8221; sent, and waits for more messages to arrive.</p>

<p>All these payloads historically have been used by botnets, so, when combined with the NKN as the communication protocol, the malware can asynchronously wait for the master to launch a combined attack. It is important to note that the last type of payload attack diverts from the others. NKAbuse overflows a DNS server with junk DNS requests (type <em>AAAA</em>), causing it to try to resolve &#8220;<strong>{JUNK}.google.com</strong>&#8221; subdomains, where {JUNK} is a randomly generated subdomain name in the <em>0-9a-zA-Z</em> format.</p>
<h2 id="a-new-backdoor-with-rat-capabilities">A new backdoor with RAT capabilities</h2>
<p>NKAbuse has multiple features that turn it into a powerful backdoor or a remote access trojan (RAT), not just a DDoS tool. In fact, most of the message commands mentioned above are, in one way or another, used for persistence, command execution, or information gathering.</p>
<p>The malware implant establishes a structure named &#8220;<strong>Heartbeat&#8221;</strong>, which talks to the bot master at regular intervals. It contains a number of other structures that store information about the infected host: the PID, the victim&#8217;s IP address, free memory, current configuration, and so on.</p>
<p>Another feature of this malware is the ability to make <strong>screenshots</strong> of the infected machine. It uses an open-source project to determine the display bounds and then capture an image of the current screen, in order to convert it to PNG and send to the bot master.</p>
<div id="attachment_111518" style="width: 527px" class="wp-caption aligncenter"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13161347/Unveiling_NKAbuse_04.png" class="magnificImage"><img loading="lazy" decoding="async" aria-describedby="caption-attachment-111518" class="size-full wp-image-111518" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13161347/Unveiling_NKAbuse_04.png" alt="Capturing screenshots" width="517" height="158" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13161347/Unveiling_NKAbuse_04.png 517w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13161347/Unveiling_NKAbuse_04-300x92.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/12/13161347/Unveiling_NKAbuse_04-370x113.png 370w" sizes="(max-width: 517px) 100vw, 517px" /></a><p id="caption-attachment-111518" class="wp-caption-text">Capturing screenshots</p></div>
<p>NKAbuse can also create files with specific content, remove files from the file system, and fetch a file list from a specific path. It can get a list of processes running in the system and even a detailed list of the available network interfaces. Another common feature that makes this implant a full-fledged backdoor is the ability to run system commands. These are executed on behalf of the current user, and the output is sent via NKN to the botmaster.</p>
<h2 id="a-new-threat">A new threat</h2>
<p>Although relatively rare, new cross-platform flooders and backdoors like NKAbuse stand out through their utilization of less common communication protocols. This particular implant appears to have been meticulously crafted for integration into a botnet, yet it can adapt to functioning as a backdoor in a specific host. Moreover, its use of blockchain technology ensures both reliability and anonymity, which indicates the potential for this botnet to expand steadily over time, seemingly devoid of an identifiable central controller.</p>
<p>Additionally, it was confirmed that the malware has no self-propagation functionality, which means the initial infection vector is delivered by someone who exploits a vulnerability to deploy the sample.</p>
<p>Our telemetry data shows that there are victims in Colombia, Mexico, and Vietnam. All Kaspersky products detect the threat as <strong>HEUR:Backdoor.Linux.NKAbuse.a.</strong></p>

With any requests on the subject, please contact <strong><a href="mailto:crimewareintel@kaspersky.com">crimewareintel@kaspersky.com</a>.</strong></p>
