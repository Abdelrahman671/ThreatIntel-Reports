                                                    <div class="single-column-right">
                                                <p>A 19 year old, yet major, vulnerability was recently <a href="https://research.checkpoint.com/extracting-code-execution-from-winrar/">found</a> by Check Point Research in the popular web application, WinRAR, that could potentially put over <a href="https://www.win-rar.com/start.html?&amp;L=0" target="_blank" rel="noopener noreferrer">500 million users</a> at risk. The exploit works by simply extracting an archive from an innocent looking ACE file which could lead to a remote code execution.</p>
<p>Following the discovery, it was not long at all before the Check Point Research team, as well as a <a href="https://twitter.com/buffaloverflow/status/1099628244406550529?s=19" target="_blank" rel="noopener noreferrer">researcher</a> from NCC Group, spotted malware sample leveraging this vulnerability in the wild. In this case, the malware is an ace archive containing a JS file that is, in turn, written to startup. It will download and run an executable when initiated. The downloaded executable is a .NET RAT called Orcus.</p>
<p>Fortunately, Check Point customers were already protected against this type of attack before the publication of this WinRAR vulnerability was made last week. By taking a closer look at how Check Point SandBlast Agent works we can get a better understanding of both the attack and protection against it.</p>
<p><a href="//blog.checkpoint.com/wp-content/uploads/2019/02/win-itw1.png"><img decoding="async" width="1181" height="627" class="aligncenter size-full wp-image-24166" src="//blog.checkpoint.com/wp-content/uploads/2019/02/win-itw1.png" alt="" srcset="https://blog.checkpoint.com/wp-content/uploads/2019/02/win-itw1.png 1181w, https://blog.checkpoint.com/wp-content/uploads/2019/02/win-itw1-250x133.png 250w, https://blog.checkpoint.com/wp-content/uploads/2019/02/win-itw1-768x408.png 768w, https://blog.checkpoint.com/wp-content/uploads/2019/02/win-itw1-700x372.png 700w, https://blog.checkpoint.com/wp-content/uploads/2019/02/win-itw1-120x64.png 120w" sizes="(max-width: 1181px) 100vw, 1181px" /></a></p>
<p style="text-align: center;"><strong>Figure 1:</strong> Forensics Report from Sand Blast Agent (in detect only mode to showcase the whole attack) highlighting the script drop and execution upon reboot. (<a href="https://forensics.checkpoint.com/wrarexp/tree.html" target="_blank" rel="noopener noreferrer">Click here for the full report</a>).</p>
<p>&nbsp;</p>
<p>The attack begins by the user extracting an archive using WinRAR believing he is extracting a rbxm file which is a 3D model for Roblox, a popular online gaming platform. In addition to the 3D model, there is a hidden JS file with custom path to user’s startup folder as highlighted below.</p>
<p><a href="//blog.checkpoint.com/wp-content/uploads/2019/02/win-itw2.png"><img decoding="async" class="aligncenter wp-image-24167" src="//blog.checkpoint.com/wp-content/uploads/2019/02/win-itw2.png" alt="" width="647" height="426" srcset="https://blog.checkpoint.com/wp-content/uploads/2019/02/win-itw2.png 1060w, https://blog.checkpoint.com/wp-content/uploads/2019/02/win-itw2-250x165.png 250w, https://blog.checkpoint.com/wp-content/uploads/2019/02/win-itw2-768x506.png 768w, https://blog.checkpoint.com/wp-content/uploads/2019/02/win-itw2-700x462.png 700w, https://blog.checkpoint.com/wp-content/uploads/2019/02/win-itw2-120x79.png 120w" sizes="(max-width: 647px) 100vw, 647px" /></a></p>
<p style="text-align: center;"><strong>Figure 2:</strong> WinRAR archive containing JS file.</p>
<p>Meanwhile, we can see from the <a href="https://forensics.checkpoint.com/wrarexp/tree.html" target="_blank" rel="noopener noreferrer">Sand Blast Agent Forensics report</a> that the JS payload file is actually extracted to the user’s start-up folder where it will then be launched once the user reboots the machine or simply logs off and logs back on.</p>
<p><a href="//blog.checkpoint.com/wp-content/uploads/2019/02/win-itw3.png"><img loading="lazy" decoding="async" class="aligncenter wp-image-24168" src="//blog.checkpoint.com/wp-content/uploads/2019/02/win-itw3.png" alt="" width="721" height="345" srcset="https://blog.checkpoint.com/wp-content/uploads/2019/02/win-itw3.png 1344w, https://blog.checkpoint.com/wp-content/uploads/2019/02/win-itw3-250x119.png 250w, https://blog.checkpoint.com/wp-content/uploads/2019/02/win-itw3-768x367.png 768w, https://blog.checkpoint.com/wp-content/uploads/2019/02/win-itw3-700x334.png 700w, https://blog.checkpoint.com/wp-content/uploads/2019/02/win-itw3-120x57.png 120w" sizes="(max-width: 721px) 100vw, 721px" /></a></p>
<p style="text-align: center;"><strong>Figure 3:</strong> WinRAR process writing JS file to startup folder.</p>
<p>&nbsp;</p>
<p><a href="//blog.checkpoint.com/wp-content/uploads/2019/02/win-itw4.png"><img loading="lazy" decoding="async" class="aligncenter wp-image-24169" src="//blog.checkpoint.com/wp-content/uploads/2019/02/win-itw4.png" alt="" width="739" height="430" srcset="https://blog.checkpoint.com/wp-content/uploads/2019/02/win-itw4.png 1431w, https://blog.checkpoint.com/wp-content/uploads/2019/02/win-itw4-250x145.png 250w, https://blog.checkpoint.com/wp-content/uploads/2019/02/win-itw4-768x447.png 768w, https://blog.checkpoint.com/wp-content/uploads/2019/02/win-itw4-700x407.png 700w, https://blog.checkpoint.com/wp-content/uploads/2019/02/win-itw4-120x70.png 120w" sizes="(max-width: 739px) 100vw, 739px" /></a></p>
<p style="text-align: center;"><strong>Figure 4:</strong> Execution of JS file, download and launch of the Orcus RAT.</p>
<p>&nbsp;</p>
<p>SandBlast Agent is able to catch such activity through the SandBlast Agent Behavioral Guard protection name Gen.Win.WrarExp.A.</p>
<p>By performing Advanced Behavioral Analysis along with threat extraction and sandboxing techniques remotely on public or private cloud servers, SandBlast Agent is able to use a low-overhead, non-intrusive approach to protect users against modern malware techniques.</p>
<p>The exploitation of vulnerabilities in such commonly used applications like WinRAR highlights once again how easily users can be exposed to malware when downloading files. By doing so, they can inadvertently put an enterprise’s entire IT network at risk of infection.</p>
<p>When suspicious events do occur, it is essential that organizations have immediate access to the information required to fully understand and triage attacks to quickly identify source and scope, and to determine the best path of resolution.</p>
<p><a href="http://www.checkpoint.com/press/2016/server-room-airport-check-point-extends-zero-day-security-systems" target="_blank" rel="noopener noreferrer">Check Point SandBlast Agent</a> is a progressive new solution that extends advanced threat prevention to endpoint devices to defend against zero-day and targeted threats. With the capture and automatic analysis of complete forensics data, SandBlast Agent provides actionable attack insight and context to enable rapid remediation in the event of a breach.</p>
