Browser extensions have become integral to our online experience. From productivity tools to entertainment add-ons, these small software modules offer customized features to suit individual preferences. Unfortunately, extensions can prove useful to malicious actors as well.

Capitalizing on the favorable characteristics of an add-on, an attacker can leverage attributes like persistence, seamless installation, elevated privileges and unencrypted data exposure to distribute and operate banking trojans.

In November 2023, security researchers at IBM Security Trusteer found new widespread &lt;a href=&quot;https://www.ibm.com/topics/malware?mhsrc=ibmsearch_a&amp;mhq=malware&quot; target=&quot;_blank&quot; rel=&quot;noopener&quot;&gt;malware&lt;/a&gt; dubbed Fakext that uses a malicious Edge extension to perform man-in-the-browser and web-injection attacks.

Here’s what cyber professionals need to know about the Fakext campaign and the different attacks the extension performs. Lastly, we will explore some indicators of compromise (IOCs) and a remediation guide for this malware.
&lt;h2&gt;Fakext campaign targeting Latin America&lt;/h2&gt;
Since the start of November 2023, our team has seen over 35,000 infected sessions, primarily originating from Latin America (LATAM), with a smaller number from Europe and North America. The extensive number of infected sessions indicates an exceptionally successful and widespread campaign. We have also seen that when Fakext injects content onto the screen, such as error messages, user forms and notifications, it is displayed in Spanish.

The list of targeted banks extracted from the initial loader comprises 14 banks operating in LATAM, particularly in Mexico. Furthermore, the loader is programmed to halt code execution if the current website does not match the specified targets. These collective observations strongly indicate that this variant is tailored to specifically target banks in LATAM. However, the methods employed here are generic, and with slight content alterations could pose a threat to other regions. We are already aware of &lt;a href=&quot;https://securityintelligence.com/posts/grandoreiro-malware-now-targeting-banks-in-spain/&quot; target=&quot;_blank&quot; rel=&quot;noopener&quot;&gt;previous instances&lt;/a&gt; where malware originating in Latin America has transitioned to Spain and subsequently spread to other parts of Europe.
&lt;h2&gt;Step 1: Infection&lt;/h2&gt;
The sole purpose of the extension is to provide a persistent mechanism to inject scripts into the victim’s HTML page.

The loader script is fetched from one of the many command and control (C2) servers the threat actor maintains and runs in the current page context. In addition to regular HTTP traffic, Fakext uses Telegram’s application programming interface (API) as another communication channel with the C2 servers. The current state of the injection and even screenshots are sent using Telegram.

Fakext downloads the fingerprintJS library as a legitimate external resource from its official content delivery network (CDN) and uses it to generate the victim’s user ID. The browser’s fingerprint is added as an HTML document attribute named “fkr-client-uid,” which signals that the extension is installed and running.

The loader script then looks for the previously mentioned ID and the current page URL to see if it’s one of the targeted banks and fetches extra modules, depending on the outcome.

There are two main modules that Fakext runs on targeted sites:
&lt;ul&gt;
 	&lt;li&gt;A form grabber that logs all input fields on the page&lt;/li&gt;
 	&lt;li&gt;An overlay that injects content onto the page to alter victims’ behavior for further fraud opportunities.&lt;/li&gt;
&lt;/ul&gt;
&lt;h2&gt;Step 2: Evasion&lt;/h2&gt;
This malware tries to hide its network traffic with seemingly legitimate domain names that are similar to known CDNs and frameworks, such as:
&lt;ul&gt;
 	&lt;li&gt;fastify[.]sbs (like fastify[.]io)&lt;/li&gt;
 	&lt;li&gt;jschecks[.]com&lt;/li&gt;
 	&lt;li&gt;cdn[.]jsassets[.]sbs&lt;/li&gt;
 	&lt;li&gt;javascrip12[.]com&lt;/li&gt;
 	&lt;li&gt;fastify[.]elfaker[.]workers[.]dev&lt;/li&gt;
&lt;/ul&gt;
For a full list of IOCs, see the &lt;a&gt;IOCs section&lt;/a&gt; below.

The threat actor uses &lt;a href=&quot;https://workers.cloudflare.com/&quot; target=&quot;_blank&quot; rel=&quot;noopener&quot;&gt;Cloudflare’s workers&lt;/a&gt; to distribute the web injections. The extension itself (which currently has over 10,000 users) describes itself as a tool to help facilitate the use of Mexico&#039;s SAT portal, which is a government tax agency website.

&lt;img src=&quot;https://images1.cmp.optimizely.com/Zz0wMGUyMzRjYWJlZDgxMWVlOTI1YzRlMmVhMWY3YTE0MA==&quot; alt=&quot;A Fakext malware infection vector through the SATiD extension page from the Edge store&quot; width=&quot;673&quot; height=&quot;501&quot; /&gt;
&lt;p class=&quot;wc-small&quot; style=&quot;font-size: 13px; line-height: 20px;&quot;&gt;&lt;em&gt;Figure 1: SATiD extension page from the Edge store&lt;/em&gt;&lt;/p&gt;
Fakext also uses popular anti-debugging techniques we have already seen in &lt;a href=&quot;https://securityintelligence.com/posts/trickbot-bolsters-layered-defenses-prevent-injection/&quot;&gt;past web injections&lt;/a&gt;. The use of code obfuscation, native function overrides and deliberate code sections designed to crash development tools collectively contribute to rendering the code more challenging to detect and analyze.
&lt;h2&gt;Step 3: Interception&lt;/h2&gt;
Fakext runs a generic form grabber on the current page that hooks into all input fields and waits for an input event. Once a keypress occurs, the entire input element, including style, ID, type and value, is sent to the C2 server.

In addition, the current page URL is sent, which allows the fraudster to know the exact type and owner of the credentials they have stolen.

In the case of specific targets with known HTML page structures and element IDs, only the pertinent inputs are intercepted. These fields are identified by their specific IDs hardcoded in the script, suggesting that certain injections were customized exclusively for selected targets.
&lt;p class=&quot;wc-small&quot; style=&quot;font-size: 13px; line-height: 20px;&quot;&gt;&lt;em&gt;&lt;img src=&quot;https://images1.cmp.optimizely.com/Zz0yNTgyMWI2MGJlZDgxMWVlOGViNGRhMWZhNzk2ZmIwOA==&quot; alt=&quot;An example of the GET request code that Fakext malware uses to exfiltrate data.&quot; width=&quot;720&quot; height=&quot;105.95317725752508&quot; /&gt;Figure 2: Example GET request with exfiltrated data&lt;/em&gt;&lt;/p&gt;

&lt;h2&gt;Step 4: Data theft&lt;/h2&gt;
In some targets in the lists, Fakext uses a different attack vector. In those cases, it injects an overlay onto the page that matches the current page styling and prevents the user from continuing the usual behavior.

Under the false presence of the bank’s IT support, the popup prompts the user to download a legitimate remote access tool (RAT) and provides the fraudster with the tool’s credentials.

The rest of the page is dimmed and unresponsive and the prompt can’t be removed.

This injection constantly sends information to the C2 servers about the current state of the overlay, such as which popup page the user is on, which banking page the user is on (pre or post-login) and what type of RAT the user installed.

With RAT credentials, knowledge of the user, banking app state and the ability to inject certain pages onto the victim’s screen (such as a fake one-time password (OTP) page), the fraudster can perform transactions and other types of financial fraud.

The victim can’t identify that external content was injected, and the whole overlay seems like a legitimate security procedure.

In addition, an optional credit card information form is often presented for further data theft.

&lt;h2&gt;Remediation and general guidelines&lt;/h2&gt;
If installed, immediately remove the “SATiD” add-on from your Edge browser.

Users should practice vigilance when using banking apps. This includes contacting their bank to report potentially suspicious activity on their accounts, not downloading software from unknown sources and following best practices for &lt;a href=&quot;https://securityintelligence.com/x-force/how-to-keep-secrets-safe-password-primer/&quot; target=&quot;_blank&quot; rel=&quot;noopener&quot;&gt;password hygiene&lt;/a&gt; and email security hygiene.

We emphasize that legitimate banking apps do not ask you to download a remote access tool and provide the credentials to someone else. In addition, it’s important to periodically review the extensions you have installed. If you no longer use a particular extension or you found an extension that you aren’t familiar with, consider removing it to reduce the potential attack surface.

<!DOCTYPE html PUBLIC "-//W3C//DTD HTML 4.0 Transitional//EN" "http://www.w3.org/TR/REC-html40/loose.dtd">
<html><body><p><em>This article was made possible thanks to contributions from Itzhak Chimino, Michael Gal and Liran Tiebloom.</em></p>
<p>Browser extensions have become integral to our online experience. From productivity tools to entertainment add-ons, these small software modules offer customized features to suit individual preferences. Unfortunately, extensions can prove useful to malicious actors as well.</p>
<p>Capitalizing on the favorable characteristics of an add-on, an attacker can leverage attributes like persistence, seamless installation, elevated privileges and unencrypted data exposure to distribute and operate banking trojans.</p>
<p>In November 2023, security researchers at IBM Security Trusteer found new widespread <a href="https://www.ibm.com/topics/malware?mhsrc=ibmsearch_a&amp;mhq=malware"  target="_blank" rel="noopener nofollow" >malware</a> dubbed Fakext that uses a malicious Edge extension to perform man-in-the-browser and web-injection attacks.</p>
<p>Here&rsquo;s what cyber professionals need to know about the Fakext campaign and the different attacks the extension performs. Lastly, we will explore some indicators of compromise (IOCs) and a remediation guide for this malware.</p>
<h2>Fakext campaign targeting Latin America</h2>
<p>Since the start of November 2023, our team has seen over 35,000 infected sessions, primarily originating from Latin America (LATAM), with a smaller number from Europe and North America. The extensive number of infected sessions indicates an exceptionally successful and widespread campaign. We have also seen that when Fakext injects content onto the screen, such as error messages, user forms and notifications, it is displayed in Spanish.</p>
<p>The list of targeted banks extracted from the initial loader comprises 14 banks operating in LATAM, particularly in Mexico. Furthermore, the loader is programmed to halt code execution if the current website does not match the specified targets. These collective observations strongly indicate that this variant is tailored to specifically target banks in LATAM. However, the methods employed here are generic, and with slight content alterations could pose a threat to other regions. We are already aware of <a href="https://securityintelligence.com/posts/grandoreiro-malware-now-targeting-banks-in-spain/"  target="_blank" rel="noopener nofollow" >previous instances</a> where malware originating in Latin America has transitioned to Spain and subsequently spread to other parts of Europe.</p>
<h2>Step 1: Infection</h2>
<p>The sole purpose of the extension is to provide a persistent mechanism to inject scripts into the victim&rsquo;s HTML page.</p>
<p>The loader script is fetched from one of the many command and control (C2) servers the threat actor maintains and runs in the current page context. In addition to regular HTTP traffic, Fakext uses Telegram&rsquo;s application programming interface (API) as another communication channel with the C2 servers. The current state of the injection and even screenshots are sent using Telegram.</p>
<p>Fakext downloads the fingerprintJS library as a legitimate external resource from its official content delivery network (CDN) and uses it to generate the victim&rsquo;s user ID. The browser&rsquo;s fingerprint is added as an HTML document attribute named &ldquo;fkr-client-uid,&rdquo; which signals that the extension is installed and running.</p>
<p>The loader script then looks for the previously mentioned ID and the current page URL to see if it&rsquo;s one of the targeted banks and fetches extra modules, depending on the outcome.</p>
<p>There are two main modules that Fakext runs on targeted sites:</p>
<ul>
<li>A form grabber that logs all input fields on the page</li>
<li>An overlay that injects content onto the page to alter victims&rsquo; behavior for further fraud opportunities.</li>
</ul>
<h2>Step 2: Evasion</h2>
<p>This malware tries to hide its network traffic with seemingly legitimate domain names that are similar to known CDNs and frameworks, such as:</p>
<ul>
<li>fastify[.]sbs (like fastify[.]io)</li>
<li>jschecks[.]com</li>
<li>cdn[.]jsassets[.]sbs</li>
<li>javascrip12[.]com</li>
<li>fastify[.]elfaker[.]workers[.]dev</li>
</ul>
<p>For a full list of IOCs, see the <a >IOCs section</a>&nbsp;below.</p>
<p>The threat actor uses <a href="https://workers.cloudflare.com/" target="_blank" rel="noopener nofollow"  target="_blank" rel="noopener nofollow" >Cloudflare&rsquo;s workers</a> to distribute the web injections. The extension itself (which currently has over 10,000 users) describes itself as a tool to help facilitate the use of Mexico&rsquo;s SAT portal, which is a government tax agency website.</p>
<p><amp-img src="https://images1.cmp.optimizely.com/Zz0wMGUyMzRjYWJlZDgxMWVlOTI1YzRlMmVhMWY3YTE0MA==" layout="intrinsic" class="" alt="A Fakext malware infection vector through the SATiD extension page from the Edge store" width="673" height="501" lightbox="lightbox"></amp-img></p>
<p class="wc-small" style="font-size: 13px; line-height: 20px;"><em>Figure 1: SATiD extension page from the Edge store</em></p>
<p>Fakext also uses popular anti-debugging techniques we have already seen in <a href="https://securityintelligence.com/posts/trickbot-bolsters-layered-defenses-prevent-injection/" >past web injections</a>. The use of code obfuscation, native function overrides and deliberate code sections designed to crash development tools collectively contribute to rendering the code more challenging to detect and analyze.</p>
<h2>Step 3: Interception</h2>
<p>Fakext runs a generic form grabber on the current page that hooks into all input fields and waits for an input event. Once a keypress occurs, the entire input element, including style, ID, type and value, is sent to the C2 server.</p>
<p>In addition, the current page URL is sent, which allows the fraudster to know the exact type and owner of the credentials they have stolen.</p>
<p>In the case of specific targets with known HTML page structures and element IDs, only the pertinent inputs are intercepted. These fields are identified by their specific IDs hardcoded in the script, suggesting that certain injections were customized exclusively for selected targets.</p>
<p class="wc-small" style="font-size: 13px; line-height: 20px;"><em><amp-img src="https://images1.cmp.optimizely.com/Zz0yNTgyMWI2MGJlZDgxMWVlOGViNGRhMWZhNzk2ZmIwOA==" layout="intrinsic" class="" alt="An example of the GET request code that Fakext malware uses to exfiltrate data." width="720" height="105.95317725752508" lightbox="lightbox"></amp-img>Figure 2: Example GET request with exfiltrated data</em></p>
<h2>Step 4: Data theft</h2>
<p>In some targets in the lists, Fakext uses a different attack vector. In those cases, it injects an overlay onto the page that matches the current page styling and prevents the user from continuing the usual behavior.</p>
<p>Under the false presence of the bank&rsquo;s IT support, the popup prompts the user to download a legitimate remote access tool (RAT) and provides the fraudster with the tool&rsquo;s credentials.</p>
<p><amp-img src="https://images1.cmp.optimizely.com/Zz01MDU3OTczNGJlZDgxMWVlOTlmNzY2MmE3N2YxNWU1NA==" layout="intrinsic" class="" alt="The Fakext prompt to install &ldquo;security software&rdquo; before continuing with bank operations" width="691" height="431" lightbox="lightbox"></amp-img></p>
<p class="wc-small" style="font-size: 13px; line-height: 20px;"><em>Figure 3: Prompt to install &ldquo;security software&rdquo; before continuing with bank operations.</em></p>
<p>The rest of the page is dimmed and unresponsive and the prompt can&rsquo;t be removed.</p>
<p><amp-img src="https://images2.cmp.optimizely.com/Zz04MWFiNGVmY2JlZDgxMWVlOGEwNDllNzllOWRmN2VhOQ==" layout="intrinsic" class="" alt="Fakext's instructions on how to download and install TeamViewer." width="709" height="372" lightbox="lightbox"></amp-img></p>
<p class="wc-small" style="font-size: 13px; line-height: 20px;"><em>Figure 4: Instructions on how to download and install TeamViewer.</em></p>
<p><amp-img src="https://images3.cmp.optimizely.com/Zz1jNTMyYzJjMmJlZDgxMWVlOTk5ZDkyMjhlZGM3MTgzMg==" layout="intrinsic" class="" alt="Instructions recognize the credentials the victim needs to provide." width="708" height="365" lightbox="lightbox"></amp-img></p>
<p class="wc-small" style="font-size: 13px; line-height: 20px;"><em>Figure 5: Instructions recognize the credentials the victim needs to provide.</em></p>
<p>This injection constantly sends information to the C2 servers about the current state of the overlay, such as which popup page the user is on, which banking page the user is on (pre or post-login) and what type of RAT the user installed.</p>
<p>With RAT credentials, knowledge of the user, banking app state and the ability to inject certain pages onto the victim&rsquo;s screen (such as a fake one-time password (OTP) page), the fraudster can perform transactions and other types of financial fraud.</p>
<p><amp-img src="https://images1.cmp.optimizely.com/Zz1lODY0NzdkNmJlZDgxMWVlYjU2N2RhMWZhNzk2ZmIwOA==" layout="intrinsic" class="" alt="Fakext using a false token input." width="702" height="342" lightbox="lightbox"></amp-img></p>
<p class="wc-small" style="font-size: 13px; line-height: 20px;"><em>Figure 6: Fake token input.</em></p>
<p>Native security measures, such as content security policy (CSP), secure socket layer (SSL) certificates or cross-origin resource sharing (CORS) limitation, don&rsquo;t remediate this threat because the browser extension overrides them.</p>
<p>The victim can&rsquo;t identify that external content was injected, and the whole overlay seems like a legitimate security procedure.</p>
<p>In addition, an optional credit card information form is often presented for further data theft.</p>
<p><amp-img src="https://images2.cmp.optimizely.com/Zz0wMDQ4ZGFhMGMwZjQxMWVlOTg0MWFlMjY3NDFjYjgyNQ==" layout="intrinsic" class="" alt="The Fakext IT support loading page." width="720" height="399.17864476386035" lightbox="lightbox"></amp-img></p>
<p class="wc-small" style="font-size: 13px; line-height: 20px;"><span class="resolved"><em>Figure 7: IT support loading page</em></span></p>
<p><amp-img src="https://images2.cmp.optimizely.com/Zz0yMTM4MmJhMmJlZDkxMWVlYTJjOWFlMDZkNzNiNTU0MQ==" layout="intrinsic" class="" alt="Credit card theft form for Fakext." width="475" height="651" lightbox="lightbox"></amp-img></p>
<p class="wc-small" style="font-size: 13px; line-height: 20px;"><em>Figure 8: Credit card theft form.</em></p>

<h2></h2>
<h2>Remediation and general guidelines</h2>
<p>If installed, immediately remove the &ldquo;SATiD&rdquo; add-on from your Edge browser.</p>
<p>Users should practice vigilance when using banking apps. This includes contacting their bank to report potentially suspicious activity on their accounts, not downloading software from unknown sources and following best practices for&nbsp;<a href="https://securityintelligence.com/x-force/how-to-keep-secrets-safe-password-primer/"  target="_blank" rel="noopener nofollow" >password hygiene</a>&nbsp;and email security hygiene.</p>
<p>We emphasize that legitimate banking apps do not ask you to download a remote access tool and provide the credentials to someone else. In addition, it&rsquo;s important to periodically review the extensions you have installed. If you no longer use a particular extension or you found an extension that you aren&rsquo;t familiar with, consider removing it to reduce the potential attack surface.</p>
