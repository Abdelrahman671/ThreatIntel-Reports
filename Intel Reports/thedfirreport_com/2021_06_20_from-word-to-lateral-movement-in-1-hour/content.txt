			<h1 class="entry-title">From Word to Lateral Movement in 1 Hour</h1>
		<div class="entry-meta">
			<span class="posted-on"><a href="https://thedfirreport.com/2021/06/20/from-word-to-lateral-movement-in-1-hour/" rel="bookmark"><time class="entry-date published" datetime="2021-06-20T23:27:55+00:00">June 20, 2021</time></a></span>		</div><!-- .entry-meta -->

<h3>Introduction </h3>



<p>In May 2021, we observed a threat actor conducting an intrusion utilizing the IcedID payloads for initial access. They later performed a number of techniques from host discovery to lateral movement, using RDP and SMB to access the file servers within an enterprise domain.</p>
<p>IcedID (known as BokBot) first observed in 2017, continues to be an active and capable threat against both individuals and organizations. The <a href="https://www.binarydefense.com/icedid-gziploader-analysis/" target="_blank" rel="noopener">IcedID</a> malware utilizes a modular malware framework and incorporates a number of anti-forensic and defense evasion capabilities. This malware has like others before it moved into the initial access broker market being used as an entry point for follow on activity like Cobalt Strike, and has lead to multiple domain wide ransomware deployments such as <a href="https://thedfirreport.com/2021/03/29/sodinokibi-aka-revil-ransomware/" target="_blank" rel="noopener">Revil</a> and <a href="https://thedfirreport.com/2021/05/12/conti-ransomware/" target="_blank" rel="noopener">Conti</a>.</p>





<h3>Summary </h3>



<p>We assess with medium confidence that the initial IcedID infection was delivered via a malspam campaign, which included an attachment with a password protected zip archive. Once extracted, the user would find a Word document with a macro, which upon execution, would deliver the initial DLL loader. Discovered in 2017, what started as a commodity malware, IcedID is now currently being deployed as an initial access broker by ransomware threat actors.   </p>



<p>In this case, the threat actor appeared to have specific goals, and did not waste any time. Within 35 minutes after the initial infection, they made their way in to the network via a Cobalt Strike Beacon deployed from the IcedID infected host.  </p>



<p>The first task of the threat actor was to enumerate the network by establishing a list of the domain admins using living off the land techniques, such as net.exe. A freely available tool Adfind.exe was also utilized to further enumerate the domain. The threat actor was also observed stealing credentials from the lsass.exe process. </p>
<p>Five minutes after the above discovery activity, we observed the actors moving laterally to other hosts on the network with the credentials of a domain administrator account. In this case, Cobalt Strike was also used to create the administrative token, and attempted to install a service using a windows service executable. The service was tasked to run an encoded PowerShell command which would download and execute the Cobalt Strike beacon over HTTP.</p>



<p>Based on the name of the hosts that the threat actors decided to pivot, we judge that they were able to digest the ‘AdFind’ results and focus on, what they believed to be, important targets – critical assets such as file servers, domain controllers, etc. It is also worth mentioning that even after the unsuccessful remote execution attempt against a few servers due to AV, the actors decided to connect via RDP and spend over an hour looking for valuable data before disconnecting and leaving the network. </p>



<p>No exfiltration of data or impact to the systems was observed but at least one command and control. It is unclear why the actors decided not to continue with their operation. No attempt was made to clean up the intrusion by the actors &#8211; artifacts that were deployed were still in operation, including C2 implants. </p>

The Cobalt Strike server used in this attack was added to our Threat Feed on 5/7/21.</p>

<h3>MITRE ATT&amp;CK v9</h3>

<h3> </h3>
<h3>Credential Access </h3>



<p>The LSASS process was accessed by an unusual process “wuauclt.exe” on the beachhead host. This was the Cobalt Strike Beacon and was used to access the credentials.  </p>



<pre>EventID: 10 <br /><br />Description: Process Access  <br />SourceImage: "C:\Windows\system32\WUAUCLT.exe"  <br />TargetImage: “C:\Windows\system32\lsass.exe” <br /><br />GrantedAccess: 0x1FFFFF <br />CallTrace: C:\Windows\SYSTEM32\ntdll.dll+9c584|C:\Windows\System32\KERNELBASE.dll+2730e|UNKNOWN(0000015E81B60D3D) </pre>



<p>This is not the first time we have observed this process (wuauclt.exe) being used. In our previous report with another IcedID infection leading to <a href="https://thedfirreport.com/2021/03/29/sodinokibi-aka-revil-ransomware/" target="_blank" rel="noopener">Sodinokibi ransomware</a> we also observed the same process being used. </p>



<p>The same process was also observed invoking PowerShell scripts: </p>





<p><img loading="lazy" decoding="async" class="alignnone size-full wp-image-3246" src="https://thedfirreport.com/wp-content/uploads/2021/06/1.png" alt="" width="498" height="418" srcset="https://thedfirreport.com/wp-content/uploads/2021/06/1.png 498w, https://thedfirreport.com/wp-content/uploads/2021/06/1-300x252.png 300w" sizes="auto, (max-width: 498px) 100vw, 498px" /></p>
<p>“Wuauclt.exe” is normally used for the Microsoft Windows Update Service and this was an attempt to blend into the OS environment. </p>



<h3>Discovery </h3>



<p>Discovery commands were run by IcedID during the initial execution on the beachhead. These commands use the Microsoft Windows built-in commands and utilities, such as WMIC, ipconfig, etc. The aim was to determine the installed Anti-virus software, network configuration, domain configuration and user accounts. The following are the commands that were executed: </p>



<pre>WMIC /Node: localhost /Namespace: \\root \SecurityCenter2 Path AntiVirusProduct Get * /Format: List  <br />ipconfig /all  <br />systeminfo  <br />net config workstation  <br />nltest /domain_trusts  <br />nltest /domain_trusts /all_trusts  <br />net view /all /domain<br />net view /all </pre>



<p>Using the information gathered, the IcedID operator was able to focus on specific targets, obtaining access to the privileged accounts and the high value hosts. </p>



<p>Once the IcedID operators were able to establish a C2 session to the initial compromised host, the operators were observed executing the following command: </p>



<pre>net group "domain admins" /DOMAIN </pre>



<p>Interestingly, we observed the operator deploying and utilizing AdFind to collect information about the hosts on the network. AdFind is an Active Directory query tool developed by <a href="https://thedfirreport.com/2020/05/08/adfind-recon/" target="_blank" rel="noopener">JoeWare</a>, a useful utility for system administrators, but also  popular among <a href="https://attack.mitre.org/software/S0552/" target="_blank" rel="noopener">threat actors</a>. </p>
<pre>cmd.exe /C C:\Recovery\AdFind.exe -f objectcategory=computer -csv name cn OperatingSystem dNSHostName &gt; C:\Recovery\DOMAIN.csv</pre>





<p>AdFind was transferred and executed on the beachhead host. The threat actor placed the AdFind binary and the results in the ‘C:\Recovery’ folder. We assess this folder location was chosen to avoid raising suspicion, as compared to executing from a user or temporary folder location. </p>

<h3>Lateral Movement </h3>



<p>The threat actors attempted and successfully managed to pivot laterally to various hosts on the domain. This was achieved by connecting via SMB and starting a service that would execute an encrypted PowerShell command with embedded Cobalt Strike SMB beacons.</p>

<p>The PowerShell is base64 encoded. Decoding the PowerShell shows that the SMB pipe is named \\.\pipe\halfduplux_9e.  </p>
