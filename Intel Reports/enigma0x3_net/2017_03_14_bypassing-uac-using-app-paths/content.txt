			data = JSON.parse( e.data );

				return;

			iframe.setAttribute( 'scrolling', 'no' );
			iframe.setAttribute( 'sandbox', 'allow-storage-access-by-user-activation allow-scripts allow-same-origin allow-top-navigation-by-user-activation' );

		<h1 class="entry-title">Bypassing UAC using App&nbsp;Paths</h1>
		<div class="entry-meta">
			<a href="https://enigma0x3.net/2017/03/14/bypassing-uac-using-app-paths/" title="2:35 pm" rel="bookmark"><time class="entry-date" datetime="2017-03-14T14:35:46+00:00">March 14, 2017</time></a><span class="byline"> by <span class="author vcard"><a class="url fn n" href="https://enigma0x3.net/author/enigma0x3/" title="View all posts by enigma0x3" rel="author">enigma0x3</a></span></span>					</div><!-- .entry-meta -->
	</header><!-- .entry-header -->

	<div class="entry-content">
		<p><span style="font-weight:400;">Over the past several months, I&#8217;ve taken an interest in </span><a href="https://msdn.microsoft.com/en-us/library/windows/desktop/dn742497(v=vs.85).aspx" target="_blank"><span style="font-weight:400;">Microsoft’s User Account Control (UAC) </span></a><span style="font-weight:400;">feature in Windows. While Microsoft doesn&#8217;t define UAC as a </span><a href="https://msdn.microsoft.com/en-us/library/cc751383.aspx" target="_blank"><span style="font-weight:400;">security boundary</span></a><span style="font-weight:400;">, bypassing this protection is still something attackers frequently need to do. The recent </span><a href="https://wikileaks.org/ciav7p1/cms/page_3375231.html" target="_blank"><span style="font-weight:400;">Vault7 leak</span></a><span style="font-weight:400;"> confirms that bypassing UAC is operationally interesting, even to nation states, as several UAC bypasses/notes were detailed in the dump. On the purposefully public side, check out the </span><a href="https://github.com/hfiref0x/UACME" target="_blank"><span style="font-weight:400;">UACME project</span></a><span style="font-weight:400;"> by </span><a href="https://twitter.com/hfiref0x" target="_blank"><span style="font-weight:400;">@hfiref0x</span></a><span style="font-weight:400;"> for a great collection of existing techniques.</span></p>
<p><span style="font-weight:400;">Microsoft seems to have a renewed interest in UAC, finally fixing many of the issues highlighted by publicly disclosed bypasses. While these fixes are only in </span><a href="http://www.winhelponline.com/blog/microsoft-fixes-eventvwr-exe-uac-bypass-exploit-windows-10-creators-update/" target="_blank"><span style="font-weight:400;">newer versions of Windows</span></a><span style="font-weight:400;">, the active response from Microsoft drives me to continue searching for UAC bypasses. I&#8217;ve </span><a href="https://enigma0x3.net/2016/07/22/bypassing-uac-on-windows-10-using-disk-cleanup/" target="_blank"><span style="font-weight:400;">previously blogged</span></a><span style="font-weight:400;"> about two </span><a href="https://enigma0x3.net/2016/08/15/fileless-uac-bypass-using-eventvwr-exe-and-registry-hijacking/" target="_blank"><span style="font-weight:400;">different bypass techniques</span></a><span style="font-weight:400;">, and this post will highlight an alternative method that also doesn&#8217;t rely on the IFileOperation/DLL hijacking approach. This technique works on Windows 10 build 15031, where the vast majority of public bypasses have been patched.</span></p>
<p><span style="font-weight:400;">As some of you may know, there are some Microsoft signed binaries that auto-elevate due to their manifest. You can read more about these binaries and their manifests </span><a href="https://technet.microsoft.com/en-us/magazine/2009.07.uac.aspx" target="_blank"><span style="font-weight:400;">here</span></a><span style="font-weight:400;">. While searching for more of these auto-elevating binaries by using the SysInternals tool &#8220;</span><a href="https://technet.microsoft.com/en-us/sysinternals/bb897441.aspx" target="_blank"><span style="font-weight:400;">sigcheck</span></a><span style="font-weight:400;">&#8220;, I came across &#8220;sdclt.exe&#8221; and verified that it auto-elevates due to its manifest:</span></p>
<p><a href="https://enigma0x3.net/wp-content/uploads/2017/03/auto_elevate.png"><img data-attachment-id="557" data-permalink="https://enigma0x3.net/2017/03/14/bypassing-uac-using-app-paths/auto_elevate/#main" data-orig-file="https://enigma0x3.net/wp-content/uploads/2017/03/auto_elevate.png" data-orig-size="1662,978" data-comments-opened="1" data-image-meta="{&quot;aperture&quot;:&quot;0&quot;,&quot;credit&quot;:&quot;&quot;,&quot;camera&quot;:&quot;&quot;,&quot;caption&quot;:&quot;&quot;,&quot;created_timestamp&quot;:&quot;0&quot;,&quot;copyright&quot;:&quot;&quot;,&quot;focal_length&quot;:&quot;0&quot;,&quot;iso&quot;:&quot;0&quot;,&quot;shutter_speed&quot;:&quot;0&quot;,&quot;title&quot;:&quot;&quot;,&quot;orientation&quot;:&quot;0&quot;}" data-image-title="auto_elevate" data-image-description="" data-image-caption="" data-medium-file="https://enigma0x3.net/wp-content/uploads/2017/03/auto_elevate.png?w=300" data-large-file="https://enigma0x3.net/wp-content/uploads/2017/03/auto_elevate.png?w=690" tabindex="0" role="button" class="alignnone size-full wp-image-557" src="https://enigma0x3.net/wp-content/uploads/2017/03/auto_elevate.png?w=690&#038;h=406" alt="" width="690" height="406" srcset="https://enigma0x3.net/wp-content/uploads/2017/03/auto_elevate.png?w=690&amp;h=406 690w, https://enigma0x3.net/wp-content/uploads/2017/03/auto_elevate.png?w=1380&amp;h=812 1380w, https://enigma0x3.net/wp-content/uploads/2017/03/auto_elevate.png?w=150&amp;h=88 150w, https://enigma0x3.net/wp-content/uploads/2017/03/auto_elevate.png?w=300&amp;h=177 300w, https://enigma0x3.net/wp-content/uploads/2017/03/auto_elevate.png?w=768&amp;h=452 768w, https://enigma0x3.net/wp-content/uploads/2017/03/auto_elevate.png?w=1024&amp;h=603 1024w" sizes="(max-width: 690px) 100vw, 690px" /></a></p>
<p><i><span style="font-weight:400;">*Note: This only works on Windows 10. The manifest for sdclt.exe in Windows 7 has the requestedExecutionLevel set to “AsInvoker”, preventing auto-elevation when started from medium integrity.</span></i></p>
<p><span style="font-weight:400;">When observing the execution flow of sdclt.exe, it becomes apparent that this binary starts control.exe in order to open up a Control Panel item in high-integrity context:</span></p>
<p><a href="https://enigma0x3.net/wp-content/uploads/2017/03/starting_control.png"><img data-attachment-id="558" data-permalink="https://enigma0x3.net/2017/03/14/bypassing-uac-using-app-paths/starting_control/#main" data-orig-file="https://enigma0x3.net/wp-content/uploads/2017/03/starting_control.png" data-orig-size="2122,826" data-comments-opened="1" data-image-meta="{&quot;aperture&quot;:&quot;0&quot;,&quot;credit&quot;:&quot;&quot;,&quot;camera&quot;:&quot;&quot;,&quot;caption&quot;:&quot;&quot;,&quot;created_timestamp&quot;:&quot;0&quot;,&quot;copyright&quot;:&quot;&quot;,&quot;focal_length&quot;:&quot;0&quot;,&quot;iso&quot;:&quot;0&quot;,&quot;shutter_speed&quot;:&quot;0&quot;,&quot;title&quot;:&quot;&quot;,&quot;orientation&quot;:&quot;0&quot;}" data-image-title="starting_control" data-image-description="" data-image-caption="" data-medium-file="https://enigma0x3.net/wp-content/uploads/2017/03/starting_control.png?w=300" data-large-file="https://enigma0x3.net/wp-content/uploads/2017/03/starting_control.png?w=690" tabindex="0" role="button" class="alignnone size-full wp-image-558" src="https://enigma0x3.net/wp-content/uploads/2017/03/starting_control.png?w=690&#038;h=269" alt="" width="690" height="269" srcset="https://enigma0x3.net/wp-content/uploads/2017/03/starting_control.png?w=690&amp;h=269 690w, https://enigma0x3.net/wp-content/uploads/2017/03/starting_control.png?w=1380&amp;h=538 1380w, https://enigma0x3.net/wp-content/uploads/2017/03/starting_control.png?w=150&amp;h=58 150w, https://enigma0x3.net/wp-content/uploads/2017/03/starting_control.png?w=300&amp;h=117 300w, https://enigma0x3.net/wp-content/uploads/2017/03/starting_control.png?w=768&amp;h=299 768w, https://enigma0x3.net/wp-content/uploads/2017/03/starting_control.png?w=1024&amp;h=399 1024w" sizes="(max-width: 690px) 100vw, 690px" /></a></p>
<p>I became curious how sdclt.exe obtains the path to control.exe. Looking again at the execution flow, sdclt.exe queries the App Path key for control.exe within the HKEY_CURRENT_USER hive.</p>
<p><a href="https://enigma0x3.net/wp-content/uploads/2017/03/app_path_query.png"><img data-attachment-id="559" data-permalink="https://enigma0x3.net/2017/03/14/bypassing-uac-using-app-paths/app_path_query/#main" data-orig-file="https://enigma0x3.net/wp-content/uploads/2017/03/app_path_query.png" data-orig-size="2304,374" data-comments-opened="1" data-image-meta="{&quot;aperture&quot;:&quot;0&quot;,&quot;credit&quot;:&quot;&quot;,&quot;camera&quot;:&quot;&quot;,&quot;caption&quot;:&quot;&quot;,&quot;created_timestamp&quot;:&quot;0&quot;,&quot;copyright&quot;:&quot;&quot;,&quot;focal_length&quot;:&quot;0&quot;,&quot;iso&quot;:&quot;0&quot;,&quot;shutter_speed&quot;:&quot;0&quot;,&quot;title&quot;:&quot;&quot;,&quot;orientation&quot;:&quot;0&quot;}" data-image-title="app_path_query" data-image-description="" data-image-caption="" data-medium-file="https://enigma0x3.net/wp-content/uploads/2017/03/app_path_query.png?w=300" data-large-file="https://enigma0x3.net/wp-content/uploads/2017/03/app_path_query.png?w=690" tabindex="0" role="button" class="alignnone size-full wp-image-559" src="https://enigma0x3.net/wp-content/uploads/2017/03/app_path_query.png?w=690&#038;h=112" alt="" width="690" height="112" srcset="https://enigma0x3.net/wp-content/uploads/2017/03/app_path_query.png?w=690&amp;h=112 690w, https://enigma0x3.net/wp-content/uploads/2017/03/app_path_query.png?w=1380&amp;h=224 1380w, https://enigma0x3.net/wp-content/uploads/2017/03/app_path_query.png?w=150&amp;h=24 150w, https://enigma0x3.net/wp-content/uploads/2017/03/app_path_query.png?w=300&amp;h=49 300w, https://enigma0x3.net/wp-content/uploads/2017/03/app_path_query.png?w=768&amp;h=125 768w, https://enigma0x3.net/wp-content/uploads/2017/03/app_path_query.png?w=1024&amp;h=166 1024w" sizes="(max-width: 690px) 100vw, 690px" /></a></p>
<p><span style="font-weight:400;">Calls to HKEY_CURRENT_USER (or HKCU) from a high integrity process are particularly interesting. This often means that an elevated process is interacting with a registry location that a medium integrity process can tamper with. In this case, I saw that “sdclt.exe” was querying </span><b>HKCU:\Software\Microsoft\Windows\CurrentVersion\App Paths\control.exe. </b><span style="font-weight:400;">If you aren’t familiar with App Paths in Windows, you can read more about the topic </span><a href="https://msdn.microsoft.com/en-us/library/windows/desktop/ee872121(v=vs.85).aspx#appPaths" target="_blank"><span style="font-weight:400;">here</span></a><span style="font-weight:400;">. </span></p>
<p><span style="font-weight:400;">As it stands, sdclt.exe looks for the App Path of control.exe in the HKCU hive. Essentially, this binary is asking “what is the full path of control.exe?”. If that key isn’t found, it continues the typical Windows search order. Since it is searching in a place that can be modified, we can populate the key. </span></p>
<p><a href="https://enigma0x3.net/wp-content/uploads/2017/03/reg_mods.png"><img data-attachment-id="560" data-permalink="https://enigma0x3.net/2017/03/14/bypassing-uac-using-app-paths/reg_mods/#main" data-orig-file="https://enigma0x3.net/wp-content/uploads/2017/03/reg_mods.png" data-orig-size="1560,644" data-comments-opened="1" data-image-meta="{&quot;aperture&quot;:&quot;0&quot;,&quot;credit&quot;:&quot;&quot;,&quot;camera&quot;:&quot;&quot;,&quot;caption&quot;:&quot;&quot;,&quot;created_timestamp&quot;:&quot;0&quot;,&quot;copyright&quot;:&quot;&quot;,&quot;focal_length&quot;:&quot;0&quot;,&quot;iso&quot;:&quot;0&quot;,&quot;shutter_speed&quot;:&quot;0&quot;,&quot;title&quot;:&quot;&quot;,&quot;orientation&quot;:&quot;0&quot;}" data-image-title="reg_mods" data-image-description="" data-image-caption="" data-medium-file="https://enigma0x3.net/wp-content/uploads/2017/03/reg_mods.png?w=300" data-large-file="https://enigma0x3.net/wp-content/uploads/2017/03/reg_mods.png?w=690" tabindex="0" role="button" loading="lazy" class="alignnone size-full wp-image-560" src="https://enigma0x3.net/wp-content/uploads/2017/03/reg_mods.png?w=690&#038;h=285" alt="" width="690" height="285" srcset="https://enigma0x3.net/wp-content/uploads/2017/03/reg_mods.png?w=690&amp;h=285 690w, https://enigma0x3.net/wp-content/uploads/2017/03/reg_mods.png?w=1380&amp;h=570 1380w, https://enigma0x3.net/wp-content/uploads/2017/03/reg_mods.png?w=150&amp;h=62 150w, https://enigma0x3.net/wp-content/uploads/2017/03/reg_mods.png?w=300&amp;h=124 300w, https://enigma0x3.net/wp-content/uploads/2017/03/reg_mods.png?w=768&amp;h=317 768w, https://enigma0x3.net/wp-content/uploads/2017/03/reg_mods.png?w=1024&amp;h=423 1024w" sizes="(max-width: 690px) 100vw, 690px" /></a></p>
<p><span style="font-weight:400;">Guess what happens once sdclt.exe is started again? You guessed it. Sdclt.exe queried our newly created App Paths key for control.exe, resulting in cmd.exe getting returned.</span></p>
<p><a href="https://enigma0x3.net/wp-content/uploads/2017/03/execution.png"><img data-attachment-id="561" data-permalink="https://enigma0x3.net/2017/03/14/bypassing-uac-using-app-paths/execution-3/#main" data-orig-file="https://enigma0x3.net/wp-content/uploads/2017/03/execution.png" data-orig-size="2294,454" data-comments-opened="1" data-image-meta="{&quot;aperture&quot;:&quot;0&quot;,&quot;credit&quot;:&quot;&quot;,&quot;camera&quot;:&quot;&quot;,&quot;caption&quot;:&quot;&quot;,&quot;created_timestamp&quot;:&quot;0&quot;,&quot;copyright&quot;:&quot;&quot;,&quot;focal_length&quot;:&quot;0&quot;,&quot;iso&quot;:&quot;0&quot;,&quot;shutter_speed&quot;:&quot;0&quot;,&quot;title&quot;:&quot;&quot;,&quot;orientation&quot;:&quot;0&quot;}" data-image-title="execution" data-image-description="" data-image-caption="" data-medium-file="https://enigma0x3.net/wp-content/uploads/2017/03/execution.png?w=300" data-large-file="https://enigma0x3.net/wp-content/uploads/2017/03/execution.png?w=690" tabindex="0" role="button" loading="lazy" class="alignnone size-full wp-image-561" src="https://enigma0x3.net/wp-content/uploads/2017/03/execution.png?w=690&#038;h=137" alt="" width="690" height="137" srcset="https://enigma0x3.net/wp-content/uploads/2017/03/execution.png?w=690&amp;h=137 690w, https://enigma0x3.net/wp-content/uploads/2017/03/execution.png?w=1380&amp;h=274 1380w, https://enigma0x3.net/wp-content/uploads/2017/03/execution.png?w=150&amp;h=30 150w, https://enigma0x3.net/wp-content/uploads/2017/03/execution.png?w=300&amp;h=59 300w, https://enigma0x3.net/wp-content/uploads/2017/03/execution.png?w=768&amp;h=152 768w, https://enigma0x3.net/wp-content/uploads/2017/03/execution.png?w=1024&amp;h=203 1024w" sizes="(max-width: 690px) 100vw, 690px" /></a></p>
<p><span style="font-weight:400;">Looking at Process Explorer (or whoami /groups), I was able to confirm that cmd.exe is indeed high integrity:</span></p>
<p><a href="https://enigma0x3.net/wp-content/uploads/2017/03/elevation.png"><img data-attachment-id="562" data-permalink="https://enigma0x3.net/2017/03/14/bypassing-uac-using-app-paths/elevation/#main" data-orig-file="https://enigma0x3.net/wp-content/uploads/2017/03/elevation.png" data-orig-size="2008,670" data-comments-opened="1" data-image-meta="{&quot;aperture&quot;:&quot;0&quot;,&quot;credit&quot;:&quot;&quot;,&quot;camera&quot;:&quot;&quot;,&quot;caption&quot;:&quot;&quot;,&quot;created_timestamp&quot;:&quot;0&quot;,&quot;copyright&quot;:&quot;&quot;,&quot;focal_length&quot;:&quot;0&quot;,&quot;iso&quot;:&quot;0&quot;,&quot;shutter_speed&quot;:&quot;0&quot;,&quot;title&quot;:&quot;&quot;,&quot;orientation&quot;:&quot;0&quot;}" data-image-title="elevation" data-image-description="" data-image-caption="" data-medium-file="https://enigma0x3.net/wp-content/uploads/2017/03/elevation.png?w=300" data-large-file="https://enigma0x3.net/wp-content/uploads/2017/03/elevation.png?w=690" tabindex="0" role="button" loading="lazy" class="alignnone size-full wp-image-562" src="https://enigma0x3.net/wp-content/uploads/2017/03/elevation.png?w=690&#038;h=230" alt="" width="690" height="230" srcset="https://enigma0x3.net/wp-content/uploads/2017/03/elevation.png?w=690&amp;h=230 690w, https://enigma0x3.net/wp-content/uploads/2017/03/elevation.png?w=1380&amp;h=460 1380w, https://enigma0x3.net/wp-content/uploads/2017/03/elevation.png?w=150&amp;h=50 150w, https://enigma0x3.net/wp-content/uploads/2017/03/elevation.png?w=300&amp;h=100 300w, https://enigma0x3.net/wp-content/uploads/2017/03/elevation.png?w=768&amp;h=256 768w, https://enigma0x3.net/wp-content/uploads/2017/03/elevation.png?w=1024&amp;h=342 1024w" sizes="(max-width: 690px) 100vw, 690px" /></a></p>
<p><span style="font-weight:400;">It is important to note that this technique does not allow for parameters, meaning it requires your payload to be placed on disk someplace. If you try to give the binary any parameters (e.g, C:\Windows\System32\cmd.exe /c calc.exe), it will interpret the entire string as the lpFile value to the </span><a href="https://msdn.microsoft.com/en-us/library/windows/desktop/bb759784(v=vs.85).aspx" target="_blank"><span style="font-weight:400;">ShellExecuteInfo</span></a><span style="font-weight:400;"> structure, which is then passed over to </span><a href="https://msdn.microsoft.com/en-us/library/windows/desktop/bb762154(v=vs.85).aspx" target="_blank"><span style="font-weight:400;">ShellExecuteEx</span></a><span style="font-weight:400;">. Since that value doesn’t exist, it will not execute.</span></p>
<p><span style="font-weight:400;">To demonstrate this technique, you can find a script here: <a href="https://raw.githubusercontent.com/enigma0x3/Misc-PowerShell-Stuff/master/Invoke-AppPathBypass.ps1">https://raw.githubusercontent.com/enigma0x3/Misc-PowerShell-Stuff/master/Invoke-AppPathBypass.ps1</a></span></p>
<p>The script takes a full path to your payload. C:\Windows\System32\cmd.exe is a good one to validate. It will automatically add the keys, start sdclt.exe and then cleanup.</p>
<p><span style="font-weight:400;">This particular technique can be remediated or fixed by setting the UAC level to “Always Notify” or by removing the current user from the Local Administrators group. Further, if you would like to monitor for this attack, you could utilize methods/signatures to look for and alert on new registry entries in </span><b>HKCU\Microsoft\Windows\CurrentVersion\App Paths\Control.exe</b><span style="font-weight:400;">.</span></p>
