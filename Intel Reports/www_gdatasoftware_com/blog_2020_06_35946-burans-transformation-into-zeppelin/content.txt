id="blogpage-content-container"> <div class="row"> <div class="column fce small-12"> <p class="intro-text">Ransomware is still evolving. Evidence for this can be seen every day. Our analysts have taken a look at Buran and Zeppelin, a particularly devastating exhibit of this evolution. </p> </div> </div> <div class="row hide-for-large"> <div class="column small-12"> <div class="pagenews-progress"> <div class="reading-time"> <span class="eta"></span> <span class="word-count"></span> </div> <div data-sticky-container id="mobile-progress-container" class="hide-on-large"> <progress value="0" class="sticky" data-sticky data-margin-top="0" data-sticky-on="small" data-margin-bottom="0" data-top-anchor="mobile-progress-container" data-btm-anchor="site-footer:top"></progress> </div> </div> <nav class="mobile-menu"> <ul class="pagenews-menu"> <li> <a href="#c203444" data-scroll-link> <span>Attack Vector</span> </a> </li> <li> <a href="#c203445" data-scroll-link> <span>Installation</span> </a> </li> <li> <a href="#c203503" data-scroll-link> <span>Second Instance </span> </a> </li> <li> <a href="#c203510" data-scroll-link> <span>Third instance</span> </a> </li> <li> <a href="#c203516" data-scroll-link> <span>Information for fellow researchers</span> </a> </li> <li> <a href="#c203517" data-scroll-link> <span>References</span> </a> </li> </ul> </nav> </div> </div> <div class="row"> <div id="c203443" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><p>Ransomware made a strong comeback in 2019 after its hiatus in 2018. Many high-profile attacks were reported by the end of 2019<sup>[1]</sup>. According to Emsisoft, in U.S. alone, the victims of ransomware include at least 113 government agencies, 89 educational establishments and 764 healthcare providers. The total amount of ransom demands tallies over $7.5 billion . <sup>[2]</sup>. In a report by Coveware, the average cost of ransom payment increased by 104% from third to fourth quarter of 2019<sup>[3]</sup>. It is therefore hardly surprising that cybercriminals are enticed once again into developing and creating new ransomware variants. Amongst the prevalent ransomware last year was the Buran ransomware that emerged early May 2019 and continues to proliferate until now. In a matter of just 9 months, this ransomware released over 5 updates by changing its code and attack vectors in order to stay stealthy and cause more damage. &nbsp;By the end of last year, a new variant of ransomware known as Zeppelin was released. Upon initial analysis of Zeppelin, certain behaviors and parts of its source code have been found to have similarities with Buran. This led us to identify Zeppelin as a new variant of Buran.</p></div><div class="ce-gallery row small-up-1 medium-up-1 large-up-1"><div class="column column-block"><figure class="image"><a href="/fileadmin/web/general/images/blog/2020/03_2020/1_Zeppelin_Overview.SVG" class="fancybox"><img class="image-embed-item " alt="Buran and Zeppelin ransomware Overview" src="/fileadmin/web/general/images/blog/2020/03_2020/1_Zeppelin_Overview.SVG" width="300" height="168" loading="lazy" /></a><figcaption class="image-caption"> Buran and Zeppelin ransomware Overview </figcaption></figure></div></div></div></div> <div data-magellan-target="c203444" class="small-12 column"><div class="row"> <div id="c203444" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-above"><div class="ce-bodytext"><header class=" "><h2 class=" "> Attack Vector </h2></header><p>Zeppelin is reaching its target networks primarily through phishing emails. These emails contain macro-enabled documents that will initiate the download and execution of the ransomware file on the victim’s machine. &nbsp;Moreover, other Zeppelin samples were also distributed through malicious advertisements (malvertising) that are designed to trick its victims into clicking fake advertisements which will trigger the download of the malicious file. Lastly, Zeppelin, like other ransomware, utilizes the use of public remote desktop software via web interfaces to remotely control a victim’s machine and execute the ransomware.</p></div></div></div> </div></div> <div data-magellan-target="c203445" class="small-12 column"><div class="row"> <div id="c203445" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><header class=" "><h2 class=" "> Installation </h2></header><p>Like Buran, Zeppelin will allocate a space in memory. When &nbsp;executed, it will perform its decryption routine. However, compared to Buran’s straight forward routine, Zeppelin has some changes to its code. For instance, it now harvests application programming interface(APIs) that it will use later by loading it in the stack. After decrypting, it will re-write the decrypted code to the base address of the file and execute it. It uses this obfuscation technique to make the analysis and signature detection of the file difficult.</p></div><div class="ce-gallery row small-up-1 medium-up-1 large-up-1"><div class="column column-block"><figure class="image"><a href="/fileadmin/_processed_/7/b/2_API_Harvest_e1a3b98e2e.png" class="fancybox"><img class="image-embed-item " alt="Harvesting of API" src="/fileadmin/_processed_/7/b/2_API_Harvest_c2fd7dbbd6.png" width="1440" height="491" loading="lazy" /></a><figcaption class="image-caption"> Harvesting of API </figcaption></figure></div></div></div></div> <div id="c203474" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-image ce-center ce-above"><div class="ce-gallery row small-up-1 medium-up-2 large-up-2"><div class="column column-block"><figure class="image"><a href="/fileadmin/web/general/images/blog/2020/03_2020/3_Decrypt_PE.png" class="fancybox"><img class="image-embed-item " alt="Decryption a PE file in the memory" src="/fileadmin/_processed_/a/f/3_Decrypt_PE_6cd769d812.png" width="715" height="854" loading="lazy" /></a><figcaption class="image-caption"> Jump at the decrypted PE file code </figcaption></figure></div><div class="column column-block"><figure class="image"><a href="/fileadmin/web/general/images/blog/2020/03_2020/4_Jump_To_decrypted_code.png" class="fancybox"><img class="image-embed-item " src="/fileadmin/_processed_/1/d/4_Jump_To_decrypted_code_2d7e45b217.png" width="715" height="827" loading="lazy" alt="" /></a></figure></div></div></div></div> <div id="c203446" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><p>The main similarities of Zeppelin with Buran are its several system checks. It will first attempt to connect to the internet to make a query to hxxp://<i>geoiptool.com. </i>This is<i></i>a valid web service that checks the geolocation of a system with the use of an IP address,<i></i>to verify where the file is currently being executed. If found to be running in either Ukraine, Belarus, Kazakhstan or Russian Federation, it won’t proceed with its infection and terminate instantly. The malware authors did this to make sure that the ransomware won’t infect any user living at the mentioned countries. This could be a hint that the ransomware originated from any of these countries.</p></div><div class="ce-gallery row small-up-1 medium-up-1 large-up-1"><div class="column column-block"><figure class="image"><a href="/fileadmin/_processed_/e/e/5_connect_geoip_e171454bcf.png" class="fancybox"><img class="image-embed-item " alt="Harvesting of API" src="/fileadmin/_processed_/e/e/5_connect_geoip_2efb45630d.png" width="1440" height="691" loading="lazy" /></a><figcaption class="image-caption"> Harvesting of API </figcaption></figure></div></div></div></div> <div id="c203497" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-image ce-center ce-above"><div class="ce-gallery row small-up-1 medium-up-1 large-up-1"><div class="column column-block"><figure class="image"><a href="/fileadmin/web/general/images/blog/2020/03_2020/7_country_check.png" class="fancybox"><img class="image-embed-item " alt="Country Protection Check" src="/fileadmin/web/general/images/blog/2020/03_2020/7_country_check.png" width="1233" height="94" loading="lazy" /></a><figcaption class="image-caption"> Country Protection Check </figcaption></figure></div></div></div></div> <div id="c203498" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><p>Zeppelin also creates a registry key that will be used to store data. In the early variants of Zeppelin, it still creates a “Buran” registry key which was later changed to Zeppelin. This is one of the links between Buran and Zeppelin. Compared to Buran, which just creates several instances of itself, Zeppelin drops an executable file inside the %APPDATA% directory with a filename randomly chosen from a list of possible names. To ensure its persistence, it adds an autorun key to the registry that points to the path of the dropped file.</p></div><div class="ce-gallery row small-up-1 medium-up-2 large-up-2"><div class="column column-block"><figure class="image"><a href="/fileadmin/_processed_/5/f/8_Zeppelin_Registry_c2c9f9d387.png" class="fancybox"><img class="image-embed-item " alt="Zeppelin Registry Entry" src="/fileadmin/_processed_/5/f/8_Zeppelin_Registry_b2802cba72.png" width="715" height="262" loading="lazy" /></a><figcaption class="image-caption"> Zeppelin Registry Entry </figcaption></figure></div><div class="column column-block"><figure class="image"><a href="/fileadmin/_processed_/8/d/9_create_copy_94bddf1cd5.png" class="fancybox"><img class="image-embed-item " alt="Creation of a copy of itself in a different location" src="/fileadmin/_processed_/8/d/9_create_copy_28e08271d7.png" width="715" height="278" loading="lazy" /></a><figcaption class="image-caption"> Creation of a copy of itself in a different location </figcaption></figure></div></div></div></div> <div id="c203501" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><p>Compared to Buran, which just creates several instances of itself, Zeppelin drops an executable file inside the %APPDATA% directory with a filename randomly chosen from a list of possible names. To ensure its persistence, it adds an autorun key to the registry that points to the path of the dropped file. The dropped file is a copy of itself which will be executed by using the “Shell Execute” API with “-start” argument.</p></div><div class="ce-gallery row small-up-1 medium-up-1 large-up-1"><div class="column column-block"><figure class="image"><a href="/fileadmin/_processed_/f/3/11_shellexecute_start_argument_f084a29510.png" class="fancybox"><img class="image-embed-item " alt="Execution of its copy with “-start” " src="/fileadmin/_processed_/f/3/11_shellexecute_start_argument_d075233aa4.png" width="1440" height="503" loading="lazy" /></a><figcaption class="image-caption"> Execution of its copy with “-start” </figcaption></figure></div></div></div></div> </div></div> <div data-magellan-target="c203503" class="small-12 column"><div class="row"> <div id="c203503" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-intext-right"><header class=" "><h2 class=" "> Second Instance </h2></header><div class="image-gallery-wrap"><div class="ce-gallery row small-up-1 medium-up-1 large-up-1"><div class="column column-block"><figure class="image"><a href="/fileadmin/_processed_/2/c/12_iplogger_header_b3a2b648ad.png" class="fancybox"><img class="image-embed-item " alt="Discovery of victim’s IP address using iplogger.org" src="/fileadmin/_processed_/2/c/12_iplogger_header_81c8c6e6a8.png" width="300" height="114" loading="lazy" /></a><figcaption class="image-caption"> Discovery of victim’s IP address using iplogger.org </figcaption></figure></div></div></div><p>Upon execution of the dropped copy, it will decrypt the contents of its ransom note, then store it in an allocated memory space for later use. Meanwhile, it will connect once again to the Internet and make a query to geoiptools.com to recheck where it was executed. After that, it will initiate a connection to iplogger.org, once again a legitimate web service used to track IP addresses, with the user-agent field id set to “ZEPPELIN” and the referrer field containing the unique ID of the victim. The malware author can use the IPLogger service to view the list of victims Zeppelin ransomware has.</p></div></div> <div id="c203504" class="fce column large-12 small-12 frame-layout-0 " ><p>The processes running in the victim’s system will be checked against a list of applications associated with monitoring system processes and services, database, backups and web services. Monitors clipboard for cryptocurrency address </figcaption></figure></div></div><div class="ce-bodytext"><p>The second instance of Zeppelin enables the malware author to drop a version of Clipbanker in the %appdata%\local\temp directory and &nbsp;execute it as “winupas.exe”. This clipbanker is responsible for monitoring the system’s clipboard for any strings that matches a cryptocurrency address. &nbsp;If a match is identified, clipbanker will replace the string to that of the malware author’s cryptocurrency address so that any amount of cryptocurrency to be transferred will be redirected to the malware author’s address. After that, Zeppelin will create another instance of itself with “-agent 0” argument.</p></div></div></div> </div></div> <div data-magellan-target="c203510" class="small-12 column"><div class="row"> <div id="c203510" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-intext-left"><header class=" "><h2 class=" "> Third instance </h2></header><div class="image-gallery-wrap"><div class="ce-gallery row small-up-1 medium-up-1 large-up-1"><div class="column column-block"><figure class="image"><a href="/fileadmin/_processed_/a/8/15_Drivetype_228b4db50f.png" class="fancybox"><img class="image-embed-item " alt="Listing of all available drives" src="/fileadmin/_processed_/a/8/15_Drivetype_f5426f6a24.png" width="300" height="143" loading="lazy" /></a><figcaption class="image-caption"> Listing of all available drives </figcaption></figure></div></div></div><p>The third instance of Zeppelin is mainly for file encryption. First it will check available drives in the system by iterating drives from Z:\ to A:\. It only looks for certain drive types which are: unknown, removable, fixed, remote and RAM disk drives.</p><p>Then, all directories except Windows Operating System-related, Internet browsers and among other folders, will be traversed to encrypt all files in it. These whitelisted folders and its files are avoided to ensure the proper execution of the malware.</p><p>&nbsp;</p></div></div> <div id="c203511" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-above"><div class="ce-gallery row small-up-1 medium-up-1 large-up-1"><div class="column column-block"><figure class="image"><img class="image-embed-item " alt="Whitelisted File Paths" src="/fileadmin/_processed_/1/d/Buran_WhitelistFilePaths_2b068f0416.png" width="1440" height="333" loading="lazy" /><figcaption class="image-caption"> Whitelisted File Paths </figcaption></figure></div></div><div class="ce-bodytext"></div></div></div> <div id="c203512" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-intext-right"><div class="image-gallery-wrap"><div class="ce-gallery row small-up-1 medium-up-1 large-up-1"><div class="column column-block"><figure class="image"><a href="/fileadmin/_processed_/2/7/Buran_FileWhitelist_3288bfb111.png" class="fancybox"><img class="image-embed-item " src="/fileadmin/_processed_/2/7/Buran_FileWhitelist_9b4a0a6216.png" width="200" height="143" loading="lazy" alt="" /></a></figure></div></div></div><p>One of the evident changes in Zeppelin is that the infection coverage is wider as it infects more filetypes than Buran. For instance, Zeppelin not only infects document files but also executable files with “.exe” extension. This makes Zeppelin more destructive than Buran as it renders the victim’s machine pretty much unusable by encrypting all software installed, unless the installation path is included in the whitelisted file paths. Every Zeppelin encrypted file can easily be distinguished by an infection marker “ZEPPELIN” that can be seen at the beginning of the file’s content. This infection marker makes it distinct from Buran, but at the same time an indication that they are from the same family as they both leave infection markers at the start of each file using the same encryption routine. After all files in the directory are encrypted, a ransom note in text file format will be dropped. Lastly, it will open a ransom note using notepad.exe to inform the victim of the infection.</p></div></div> <div id="c203513" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-image class=" "> Conclusion </h2></header><p>In this day where we create faster solutions and detections, malware authors also adapt to this by creating and releasing more malware updates to make sure that it stays relevant. This is evident in ransomware campaigns as malware authors get an extra motivation by gaining huge sums of money in exchange for file recovery. Normally, ransomware only infects document files which is also the case with Buran. However, Zeppelin takes things a step further by targeting not only document related files but also applications and tools installed in the victim’s system . This extent of damage gives Zeppelin more leverage for the victim to pay the ransom. &nbsp;With this, delivering more advanced detections and solutions that will withstand fast-paced changes of ransomware is needed.
