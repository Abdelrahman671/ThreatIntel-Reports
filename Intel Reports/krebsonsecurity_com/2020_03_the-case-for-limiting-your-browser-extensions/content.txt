					<h1 class="entry-title">The Case for Limiting Your Browser Extensions</h1>
				</div>
							
			
			</header><!-- .entry-header -->

	<div id="primary" class="site-content">
		<div id="content" role="main">

			
					<article id="post-50640" class="post-50640 post type-post status-publish format-standard hentry category-breadcrumbs tag-212b3d4039ab5319ec-js tag-blue-shield-of-california tag-cndpps tag-domaintools-com tag-frankomedison1020gmail-com tag-icontent tag-linkojager tag-metrext tag-monetizus tag-page-ruler-extension tag-peter-newnham tag-thisadsfor">

		<div class="adt">
			
			
				
				<span class="date updated">March 3, 2020</span>
					
        </div>
		<div class="adt-comment">
		<span><a class="link-comments" href="https://krebsonsecurity.com/2020/03/the-case-for-limiting-your-browser-extensions/#comments">71 Comments</a></span> 
        </div>    <div class="clear"></div>

							<div class="entry-content">
												<p>Last week, KrebsOnSecurity reported to health insurance provider <a href="https://www.blueshieldca.com/" target="_blank" rel="noopener"><strong>Blue Shield of California</strong></a> that its Web site was flagged by multiple security products as serving malicious content. Blue Shield quickly removed the unauthorized code. An investigation determined it was injected by a browser extension installed on the computer of a Blue Shield employee who&#8217;d edited the Web site in the past month.</p>
<p>The incident is a reminder that browser extensions &#8212; however useful or fun they may seem when you install them &#8212; typically have a great deal of power and can effectively read and/or write all data in your browsing sessions. And as we&#8217;ll see, it&#8217;s not uncommon for extension makers to sell or lease their user base to shady advertising firms, or in some cases abandon them to outright cybercriminals.</p>
<p><img decoding="async" class="aligncenter wp-image-45005" src="https://krebsonsecurity.com/wp-content/uploads/2018/09/browsers.png" alt="" width="599" height="110" /></p>
<p>The health insurance site was compromised after an employee at the company edited content on the site while using a Web browser equipped with a once-benign but now-compromised extension which quietly injected code into the page.</p>
<p>The extension in question was <a href="https://chrome.google.com/webstore/detail/page-ruler/emliamioobfffbgcfdchabfibonehkme?hl=en" target="_blank" rel="noopener">Page Ruler</a>, a Chrome addition with some 400,000 downloads. Page Ruler lets users measure the inch/pixel width of images and other objects on a Web page. But the extension was <a href="https://github.com/wrakky/page-ruler/issues/101" target="_blank" rel="noopener">sold by the original developer</a> a few years back, and for some reason it&#8217;s still available from the Google Chrome store despite multiple recent reports from people blaming it for spreading malicious code.</p>
<p>How did a browser extension lead to a malicious link being added to the health insurance company Web site? This compromised extension tries to determine if the person using it is typing content into specific Web forms, such as a blog post editing system like <strong>WordPress</strong> or <strong>Joomla</strong>.</p>
<p>In that case, the extension silently adds a request for a javascript link to the end of whatever the user types and saves on the page. When that altered HTML content is saved and published to the Web, the hidden javascript code causes a visitor&#8217;s browser to display ads under certain conditions.</p>
<p>Who exactly gets paid when those ads are shown or clicked is not clear, but there are a few clues about who&#8217;s facilitating this. The malicious link that set off antivirus alarm bells when people tried to visit Blue Shield California downloaded javascript content from a domain called <strong>linkojager[.]org</strong>.</p>
<p>The file it attempted to download &#8212; <strong>212b3d4039ab5319ec.js</strong> &#8212; appears to be named after an affiliate identification number designating a specific account that should get credited for serving advertisements. A simple Internet search shows this same javascript code is present on <a href="https://krebsonsecurity.com/wp-content/uploads/2020/03/212b3d4039ab5319ecjs.txt" target="_blank" rel="noopener">hundreds of other Web sites</a>, no doubt inadvertently published by site owners who happened to be editing their sites with this Page Ruler extension installed.</p>
<p>If we download a copy of that javascript file and view it in a text editor, we can see the following message toward the end of the file:</p>
<blockquote><p>[NAME OF EXTENSION HERE]&#8217;s development is supported by advertisements that are added to some of the websites you visit. During the development of this extension, I&#8217;ve put in thousands of hours adding features, fixing bugs and making things better, not mentioning the support of all the users who ask for help.</p>
<p>Ads support most of the internet we all use and love; without them, the internet we have today would simply not exist. Similarly, without revenue, this extension (and the upcoming new ones) would not be possible.</p>
<p>You can disable these ads now or later in the settings page. You can also minimize the ads appearance by clicking on partial support button. Both of these options are available by clicking \&#8217;x\&#8217; button in the corner of each ad. In both cases, your choice will remain in effect unless you reinstall or reset the extension.</p></blockquote>
<p>This appears to be boilerplate text used by one or more affiliate programs that pay developers to add a few lines of code to their extensions. The opt-out feature referenced in the text above doesn&#8217;t actually work because it points to a domain that no longer resolves &#8212; <strong>thisadsfor[.]us</strong>. But that domain is still useful for getting a better idea of what we&#8217;re dealing with here.</p>
<p>Registration records maintained by <a href="https://www.domaintools.com" target="_blank" rel="noopener">DomainTools</a> [an advertiser on this site] say it was originally registered to someone using the email address <strong>frankomedison1020@gmail.com</strong>. A reverse WHOIS search on that unusual name turns up several other interesting domains, including <strong>icontent[.]us</strong>.</p>
<p>icontent[.]us is currently not resolving either, but <a href="https://web.archive.org/web/20170421092836/http://icontent.us/#features" target="_blank" rel="noopener">a cached version of it at Archive.org</a> shows it once belonged to an advertising network called <strong>Metrext</strong>, which marketed itself as an analytics platform that let extension makers track users in real time.</p>
<div id="attachment_50642" style="width: 607px" class="wp-caption aligncenter"><a class="lightbox" href="https://krebsonsecurity.com/wp-content/uploads/2020/02/metrext.png"><img aria-describedby="caption-attachment-50642" decoding="async" loading="lazy" class="wp-image-50642" src="https://krebsonsecurity.com/wp-content/uploads/2020/02/metrext.png" alt="" width="597" height="376" /></a><p id="caption-attachment-50642" class="wp-caption-text">An archived copy of the content once served at icontent[.]us promises &#8220;plag&#8217;n&#8217;play&#8221; capability.</p></div>
<p>&#8220;Three lines into your product and it&#8217;s in live,&#8221; iContent enthused. &#8220;High revenue per user.&#8221;</p>
<p>Another domain tied to Frank Medison is <strong>cdnpps[.]us</strong>, which currently redirects to the domain &#8220;<strong>monetizus[.]com</strong>.&#8221; Like its competitors, Monetizus&#8217; site is full of grammar and spelling errors: &#8220;Use Monetizus Solutions to bring an extra value to your toolbars, addons and extensions, without loosing an audience,&#8221; the company says in a banner at the top of its site.</p>
<div id="attachment_50747" style="width: 601px" class="wp-caption aligncenter"><a class="lightbox" href="https://krebsonsecurity.com/wp-content/uploads/2020/03/monetizus.png"><img aria-describedby="caption-attachment-50747" decoding="async" loading="lazy" class="wp-image-50747" src="https://krebsonsecurity.com/wp-content/uploads/2020/03/monetizus.png" alt="" width="591" height="335" /></a><p id="caption-attachment-50747" class="wp-caption-text">Be sure not to &#8220;loose&#8221; out on sketchy moneymaking activities!</p></div>
<p>Contacted by KrebsOnSecurity, Page Ruler&#8217;s original developer <strong>Peter Newnham</strong> confirmed he sold his extension to MonetizUs in 2017.</p>
<p>&#8220;They didn&#8217;t say what they were going to do with it but I assumed they were going to try to monetize it somehow, probably with the scripts their website mentions,&#8221; Newnham said.<span id="more-50640"></span></p>
<p>&#8220;I could have probably made a lot more running ad code myself but I didn&#8217;t want the hassle of managing all of that and Google seemed to be making noises at the time about cracking down on that kind of behaviour so the one off payment suited me fine,&#8221; Newnham said. &#8220;Especially as I hadn&#8217;t updated the extension for about 3 years and work and family life meant I was unlikely to do anything with it in the future as well.&#8221;</p>
<p>Monetizus did not respond to requests for comment.</p>
<p>Newnham declined to say how much he was paid for surrendering his extension. But it&#8217;s not difficult to see why developers might sell or lease their creation to a marketing company: Many of these entities offer the promise of a hefty payday for extensions with decent followings. For example, one competing extension monetization platform called <strong>AddonJet</strong> claims it can offer revenues of up to $2,500 <em>per day</em> for every 100,000 user in the United States (see screenshot below).</p>
<div id="attachment_50671" style="width: 606px" class="wp-caption aligncenter"><a class="lightbox" href="https://krebsonsecurity.com/wp-content/uploads/2020/02/addonjt.png"><img aria-describedby="caption-attachment-50671" decoding="async" loading="lazy" class="wp-image-50671" src="https://krebsonsecurity.com/wp-content/uploads/2020/02/addonjt.png" alt="" width="596" height="451" /></a><p id="caption-attachment-50671" class="wp-caption-text">Read here how its work!</p></div>
<p>I hope it&#8217;s obvious by this point, but readers should be extremely cautious about installing extensions &#8212; sticking mainly to those that are actively supported and respond to user concerns. Personally, I do not make much use of browser extensions. In almost every case I’ve considered installing one I’ve been sufficiently spooked by the permissions requested that I ultimately decided it wasn’t worth the risk.</p>
<p>If you’re the type of person who uses multiple extensions, it may be wise to adopt a risk-based approach going forward. Given the high stakes that typically come with installing an extension, consider carefully whether having the extension is truly worth it. This applies equally to plug-ins designed for Web site content management systems like <strong>WordPress</strong> and<strong> Joomla</strong>.</p>
<p>Do not agree to update an extension if it suddenly requests more permissions than a previous version. This should be a giant red flag that something is not right. If this happens with an extension you trust, you&#8217;d be well advised to remove it entirely.</p>
<p>Also, never download and install an extension just because some Web site says you need it to view some type of content. Doing so is almost always a high-risk proposition. Here, Rule #1 from <a href="https://krebsonsecurity.com/2011/05/krebss-3-basic-rules-for-online-safety/" target="_blank" rel="noopener">KrebsOnSecurity’s Three Rules of Online Safety</a> comes into play: “If you didn’t go looking for it, don’t install it.” Finally, in the event you do wish to install something, make sure you’re getting it directly from the entity that produced the software.</p>
<p>Google Chrome users can see any extensions they have installed by clicking the three dots to the right of the address bar, selecting “More tools” in the resulting drop-down menu, then “Extensions.” In Firefox, click the three horizontal bars next to the address bar and select “Add-ons,” then click the “Extensions” link on the resulting page to view any installed extensions.</p>

			<header class="comment-meta comment-author vcard">
				<cite class="fn">ThaCrip </cite><a href="https://krebsonsecurity.com/2020/03/the-case-for-limiting-your-browser-extensions/comment-page-2/#comment-506292"><time datetime="2020-03-13T07:55:19-04:00">March 13, 2020</time></a>			</header><!-- .comment-meta -->

			
			<section class="comment-content comment">
				<p>&#8220;Do not agree to update an extension if it suddenly requests more permissions than a previous version. This should be a giant red flag that something is not right. If this happens with an extension you trust, you’d be well advised to remove it entirely.&#8221;</p>
<p>In general that&#8217;s good advice but I don&#8217;t think it&#8217;s always true as a popular ad-blocking extension (uBlock Origin by Raymond Hill (which is a recommended extension by Mozilla)) did that not all that long ago (i.e. <a href="https://github.com/gorhill/uBlock/wiki/Permissions#access-ip-address-and-hostname-information" rel="nofollow ugc">https://github.com/gorhill/uBlock/wiki/Permissions#access-ip-address-and-hostname-information</a> ) and it&#8217;s legit (even though when I first seen the Firefox browser asking me to give permissions I was a bit concerned initially as I never seen that happen before and looked into it a bit before allowing it). but in general I figure keep ones extensions minimal as it limits the attack surface as those people who randomly install a bunch of extensions are just asking for trouble. a person should not need more than a few or so tops. but I consider a ad-blocking extension to be a must (I suggest uBlock Origin by Raymond Hill) and I figure while it increases the attack surface, it&#8217;s probably worth the minimal risk because of the junk it blocks online and can be further tweaked to block even more junk but the risk of breaking sites increases. but it&#8217;s default state is pretty good with no configuration needed.</p>
<p>I figure for the more paranoid types one can run their browser in a sandbox and use a separate browser (in a sandbox) with NO extensions installed for doing more sensitive things online like banking. I am just assuming ones computer is virus/junk free to begin with.</p>
<p>hell, if a person has more than one computer, especially if one has a bit of a older computer I suggest one installs Linux to it as that should give a person a secure internet machine for doing more sensitive things online and I would assume the average person is a lot less likely to get a virus or other random junk on their computer using that than Windows. so by default, it&#8217;s sort of protects the user against themselves to some degree vs Windows.</p>

				<p>I have always been SUPER afraid of any chrome extensions. </p>
<p>Something many dont think about, is that there is no way to DISABLE auto extension updates. </p>
<p>So just because xyz extension is safe/trusted today, does not mean that TOMOROW the author of the extension wont add some malicious code to xyz extension that quickly gets pushed to ALL users of xyz extension. (or a malicious actor who gains access to the author&#8217;s pc or their google dev account).</p>
<p>I have asked google about this directly several months ago, and they said there is no way to disable auto extension updates (for &#8220;security&#8221; purposes), but that google does have a &#8220;extensive&#8221; auditing process for all code that appears on its chrome extensions store.  I dont trust that!<br />
tks brian!</p>
