                <h1 class="c-page-title__title">
<span>FiveHands Ransomware</span>
</h1>
                                                          <div class="c-page-title__fields">  




<div  class="c-field c-field--name-field-last-updated c-field--type-datetime c-field--label-above">
  <div  class="c-field__label">Last Revised</div><div class="c-field__content"><time datetime="2021-05-06T12:00:00Z">May 06, 2021</time></div></div>

  




<div  class="c-field c-field--name-field-alert-code c-field--type-string c-field--label-above">
  <div  class="c-field__label">Alert Code</div><div class="c-field__content">AR21-126A</div></div>

</div>
                        
        
      </div>

<div  class="l-page-section l-page-section--rich-text">
      <div class="l-constrain">
  
  
  <div class="l-page-section__content">
          <h4><strong>Initial Access</strong></h4>
<p>The initial access vector was a zero-day vulnerability in a virtual private network (VPN) product (<em>Exploit Public-Facing Application</em> [<a href="https://attack.mitre.org/versions/v9/techniques/T1190/">T1190</a>]).</p>
<h4><strong>Publicly Available Tool: SoftPerfect Network Scanner</strong></h4>
<p>The cyber actor used <em>SoftPerfect Network Scanner for Discovery</em> [<a href="https://attack.mitre.org/versions/v9/tactics/TA0007/">TA0007</a>] of hostnames and network services (<em>Network Service Scanning</em> [<a href="https://attack.mitre.org/versions/v9/techniques/T1046/">T1046</a>]).</p>
<p>Details on the SoftPerfect Network Scanner artifacts are below.</p>
<h4><strong><em>netscan.exe</em></strong></h4>
<p>The <code>netscan.exe</code> artifact is a stand-alone version of the SoftPerfect Network Scanner, version 7.2.9 for 64-bit operating systems. The SoftPerfect website states that the "SoftPerfect Network Scanner can ping computers, scan ports, discover shared folders, and retrieve practically any information about network devices, via Windows Management Instrumentation (WMI), Simple Network Management Protocol (SNMP), Hypertext Transfer Protocol (HTTP), Secure Shell (SSH), and PowerShell. It also scans for remote services, registry, files and performance counters; offers flexible filtering and display options and exports NetScan results to a variety of formats from XML to JSON."</p>
<p>The utility can also be used with Nmap for vulnerability scanning. The utility will generate a report of its findings called <code>netscan.xml</code>.</p>
<h4><strong><em>netscan.xml</em></strong></h4>
<p>The <code>netscan.xml </code>artifact is an Extensible Markup Language (XML) document reporting scanning results for the SoftPerfect Network Scanner program. The XML document indicates that a random scan was conducted to identify hostnames on a network and to search for:</p>
<ul>
<li>web servers,&nbsp;</li>
<li>file servers,&nbsp;</li>
<li>database servers, and</li>
<li>any open Remote Desktop Protocol (RDP) ports for several subnets of unrouteable Internet Protocol (IP) addresses.</li>
</ul>
<h4><strong><em>netscan.lic</em></strong></h4>
<p>A license is required to unlock all of the features of the SoftPerfect Network Scanner. The <code>netscan.lic</code> artifact is the Network Scanner license that was included with this submission. The license name is <code>DeltaFoX</code>.</p>
<h4><strong>FiveHands Ransomware</strong></h4>
<p>The malicious cyber actor used <a href="https://attack.mitre.org/versions/v9/software/S0029/">PsExec</a> to execute <code>ServeManager.exe</code>, which CISA refers to as FiveHands ransomware (<em>Execution</em> [<a href="https://attack.mitre.org/versions/v9/tactics/TA0002/">TA0002</a>], <em>System Services: Service Execution</em> [<a href="https://attack.mitre.org/versions/v9/techniques/T1569/002/">T1569.002</a>], <em>Impact </em>[<a href="https://attack.mitre.org/versions/v9/tactics/TA0040/">TA0040</a>]). FiveHands is a novel ransomware variant that uses a public key encryption scheme called <code>NTRUEncrypt</code>. <strong>Note: </strong>the NTRUEncrypt public key cryptosystem encryption algorithm (NTRU), is a lattice-based alternative to Rivest-Shamir-Adleman, known as RSA, and Elliptic-curve cryptography, or ECC, and is based on the shortest vector problem in a lattice.</p>
<p>To prevent data recovery, FiveHands uses WMI to first enumerate then delete Volume Shadow copies (<em>Inhibit System Recovery</em> [<a href="https://attack.mitre.org/versions/v9/techniques/T1490/">T1490</a>]; <em>Windows Management Instrumentation</em> [<a href="https://attack.mitre.org/versions/v9/techniques/T1047/">T1047</a>]). The malware also encrypts files in the recovery folder (<em>Data Encrypted for Impact</em> [<a href="https://attack.mitre.org/versions/v9/techniques/T1486/">T1486</a>]). After the files are encrypted, the program will write a ransom note to each folder and directory on the system.</p>
<p>Details on the ransomware artifacts are below.</p>
<h4><strong><em>PsExec.exe</em></strong></h4>
<p>The <code>PsExec.exe</code> artifact is the legitimate remote administration program. This tool is part of Microsoft's Sysinternals tool suite. This utility was used to execute the program <code>ServeManager.exe</code> with the following arguments:</p>
<p><code>psexec.exe -d @comps.txt -s -relatime -c ServeManager.exe -key</code>&nbsp;</p>
<p>The arguments are defined as follows:</p>
<p>&nbsp; &nbsp; &nbsp; &nbsp; &nbsp; <code>-d</code> --&gt; Run psexec.exe without any prompts.</p>
<p>&nbsp; &nbsp; &nbsp; &nbsp; &nbsp; <code>@</code> --&gt; Remotely access this list of hostnames/IP addresses.</p>
<p>&nbsp; &nbsp; &nbsp; &nbsp; &nbsp; <code>-s</code> --&gt; Run the program with system level privileges.</p>
<p>&nbsp; &nbsp; &nbsp; &nbsp; &nbsp; <code>-relatime</code> --&gt; This is a typo. This should be -realtime, or run this process before&nbsp;any other process.</p>
<p>&nbsp; &nbsp; &nbsp; &nbsp; &nbsp; <code>-c</code> --&gt; Copy the program to the remote system before executing.</p>
<h4><strong>ServeManager.exe</strong></h4>
<p>The <code>ServeManager.exe</code> artifact is a 32-bit executable file that is executed using the Microsoft Sysinternals remote administration tool, <code>PsExec.exe</code>. When the program is executed it will attempt to load into memory a large embedded module that is decoded with a supplied key. The module is decoded in memory and checked to verify that it has a portable executable (PE) header. If the header is verified, the payload is executed.</p>
<p>The payload is a 32-bit executable file that is used to encrypt files on the victim’s system to extort a ransom. When the ransomware is executed, it will enumerate files and folders on the system and encrypt files with the extensions, <code>.txt</code>, <code>.chm</code>, <code>.dat</code>, <code>.ocx</code>, <code>.js</code>, <code>.tlb</code>, <code>.vbs</code>, <code>.sys</code>, <code>.lnk</code>, <code>.xml</code>, <code>.jpg</code>, <code>.log</code>, <code>.zip</code>, <code>.htm</code>, <code>.ini</code>, <code>.gif</code>, <code>.html</code>, <code>.css</code>, and others (<em>File and Directory Discovery</em> [<a href="https://attack.mitre.org/versions/v9/techniques/T1083/">T1083</a>]). Key system files are not encrypted.</p>
<p>To thwart the recovery of the data, the ransomware uses Windows Management Instrumentation (WMI) to enumerate Volume Shadow copies using the command <code>select * from Win32_ShadowCopy</code> and then deletes copies by ID (<code>Win32_ShadowCopy.ID</code>). The malware will also encrypt files in the recovery folder at <code>C:\Recovery</code>. After the files are encrypted the program will write a ransom note to each folder and directory on the system called <code>read_me_unlock.txt</code>.</p>
<p>Figure 1 displays the ransom note (redacted for privacy).</p>

<h4><br><br>
<strong>Remote Access Trojan: SombRAT&nbsp;</strong></h4>
<p>The threat actors used batch and text files to execute and invoke PowerShell scripts that decoded a SombRAT loader and enabled PowerShell to bypass the organization’s anti-malware program (<em>Command and Scripting Interpreter: Windows Command Shell</em> [<a href="https://attack.mitre.org/versions/v9/techniques/T1059/003/">T1059.003</a>], <em>Command and Scripting Interpreter: PowerShell</em> [<a href="https://attack.mitre.org/versions/v9/techniques/T1059/001/">T1059.001</a>], <em>Defense Evasion</em> [<a href="https://attack.mitre.org/versions/v9/tactics/TA0005/">TA0005</a>]). SombRAT is a custom remote access Trojan (RAT) used to download and execute malicious payloads.[<a href="https://blogs.blackberry.com/en/2020/11/the-costaricto-campaign-cyber-espionage-outsourced">1</a>]&nbsp;</p>
<p>The SombRAT loader recovered in this incident was a 64-bit variant that allowed the malicious actor to remotely download and load executable dynamic-link libraries (DLL) plugins on the affected system (<em>Ingress Tool Transfer</em> [<a href="https://attack.mitre.org/versions/v9/techniques/T1105/">T1105</a>]). The loader used hardcoded public RSA keys for command and control (C2) sessions (<em>Command and Control</em> [<a href="https://attack.mitre.org/versions/v9/tactics/TA0011/">TA0011</a>]). The C2 communications were encrypted using Advanced Encryption Standard (AES), resulting in a Secure Sockets Layer tunnel with the threat actors (<em>Encrypted Channel: Asymmetric Cryptography</em> [<a href="https://attack.mitre.org/versions/v9/techniques/T1573/002/">T1573.002</a>]).</p>
<p>Details on the SombRAT artifacts are below.</p>
<h4><strong><em>WwanSvc.bat</em></strong></h4>
<p>The <code>WwanSvc.bat</code> artifact is a batch file. When executed, it will invoke PowerShell, which decodes and executes a base64-encoded PowerShell script called <code>WwanSvc.txt</code> in the path <code>C:\ProgramData\Microsoft\WwanSvc\</code> (<em>Deobfuscate/Decode Files or Information</em> [<a href="https://attack.mitre.org/versions/v9/techniques/T1140/">T1140</a>], <em>Obfuscated Files or Information</em> [<a href="https://attack.mitre.org/versions/v9/techniques/T1027/">T1027</a>]).</p>
<h4><strong><em>WwanSvc.txt</em></strong></h4>
<p>The <code>WwanSvc.txt</code> artifact is a base64-encoded PowerShell script that is decoded and executed by <code>WwanSvc.bat</code>. The script allows PowerShell to run without system restrictions while bypassing the Microsoft anti-malware program. Next, the script decodes the file <code>WwanSvc.c</code> using a bitwise Exclusive OR (XOR) with a 256-byte key that is found in <code>WwanSvc.a</code>. Both <code>WwanSvc.a</code> and <code>WwanSvc.c</code> are located in <code>C:\ProgramData\Microsoft\</code>. The newly decoded script is then executed using the <code>InvokeExpression</code> command.</p>
<h4><strong><em>WwanSvc.a</em></strong></h4>
<p>The <code>WwanSvc.a</code> artifact contains a 256-byte key that is used by the base64-encoded script in <code>WwanSvc.txt</code> to decode a new PowerShell script in <code>WwanSvc.c</code>. The key is also used to decode the reflectively loaded payload in <code>WwanSvc.b</code>.</p>
<h4><strong><em>WwanSvc.c</em></strong></h4>
<p>The <code>WwanSvc.c</code> artifact is an XOR-encoded PowerSploit reflective loader program.[<a href="https://attack.mitre.org/software/S0194/">2</a>] The program is decoded using the 256-byte key found in <code>WwanSvc.a</code>. The script will decode the content of <code>WwanSvc.b</code> and then check to confirm that it has a valid PE header. The script will also check the system environment for a 64-bit architecture (<em>System Information Discovery</em> [<a href="https://attack.mitre.org/versions/v9/techniques/T1082/">T1082</a>]). The executable is not written to disk but loaded directly into memory.</p>
<h4><strong><em>WwanSvc.b</em></strong></h4>
<p>The <code>WwanSvc.b</code> artifact, when decoded, is a 64-bit variant of the SombRAT loader. The primary purpose of the loader is to allow a remote operator to securely download and load executable plugins on a target system. Given this plugin structure, the author can easily mold the RAT to provide additional functionalities and capabilities. The application contains the following two hardcoded public RSA keys, which it will utilize to secure its C2 sessions with the remote operator. Static analysis indicates that the C2 communications will also be encrypted using AES resulting in a secure Secure Sockets Layer (SSL) tunnel with the remote operator.</p>
<p>The configuration file <code>59fb3174bb34e803</code>, located in <code>C:\ProgramData</code>, contains the data the malware requires at runtime, including the operator-controlled remote C2 address. The malware decrypts this configuration file with the hardcoded AES key <code>ujnchdyfngtreaycnbjgi837157fncae</code>. See figure 2.<br><br>
&nbsp;</p>
<p><img class="align-center"></p>
<p><em>Figure 2: Hardcoded AES key</em></p>
<p>&nbsp;</p>
<p>The malware contains numerous encoded strings, including the AES key used to decrypt the malware configuration file. The malware decrypts these strings by first XORing them with the first byte. The malware then decrypts the rest of the string by XORing it with the single byte XOR key <code>0xDE</code>.&nbsp;</p>
<p>This string can be decrypted by XORing the entire string with the value <code>0x78</code> and then XORing the result with <code>0xDE</code>.</p>
<p>The RAT provides most of its C2 capabilities to the remote operator by allowing the remote operator to securely transfer executable DLL plugins to the target system—via a protected SSL session—and load these plugins at will via the embedded plugin framework. The native malware itself does not provide much actual functionality to the operator without the code provided by the plugins. Some of the native functionality that the malware provides without the use of a plugin includes collecting system data—such as computer name, username, current process, operating system (OS) version, local system time, and the current process that the malware is masquerading as (<em>System Owner/User Discovery</em> [<a href="https://attack.mitre.org/versions/v9/techniques/T1033">T1033</a>], <em>Process Discovery</em> [<a href="https://attack.mitre.org/versions/v9/techniques/T1057">T1057</a>], <em>System Time Discovery</em> [<a href="https://attack.mitre.org/versions/v9/techniques/T1124">T1124</a>], <em>Masquerading</em> [<a href="https://attack.mitre.org/versions/v9/techniques/T1036">T1036</a>]). The program also contains native C2 capabilities allowing it to communicate with the remote operator using an embedded SOCKS proxy or via domain name system (DNS) tunneling (<em>Proxy</em> [<a href="https://attack.mitre.org/versions/v9/techniques/T1090">T1090</a>]).</p>
<p>The malware does contain hardcoded commands that it uses to evaluate against operator-provided data. These commands are encoded within the binary, and they are not encoded before being compared against operator-provided data—indicating the malware expects the remote operator to encode the commands before passing them to the RAT.&nbsp;</p>
<p>The malware contains an encoded note, presumably designed for malware analysts who analyze the code. See figure 3.</p>
<p>&nbsp;</p>
<p><img class="align-center"></p>
<p>&nbsp;</p>
<p><em>Figure 3: Encoded&nbsp;note</em></p>
<p><br><br>
<strong><em>59fb3174bb34e803</em></strong></p>
<p>The <code>59fb3174bb34e803</code> artifact is an encrypted configuration file that is read by the WwanSvc program. The configuration file contains the hardcoded domain, <code>feticost[.]com</code>. The program attempts DNS queries for this domain prepending a third level domain that consists of seven to nine random hexadecimal characters (e.g., <code>bb95058f1[.]feticost.com</code>).</p>
<p>The file <code>feticost[.]com</code> resolved to the IP address <code>51.89.50[.]152</code> at the time of analysis.</p>
<h4><strong>Publicly Available Tool: RouterScan.exe</strong></h4>
<p>The <code>RouterScan.exe</code> artifact is Router Scan v2.60 by Stas'M. This utility is used to identify network routers and proxy servers on a network (<em>Discovery</em> [<a href="https://attack.mitre.org/versions/v9/tactics/TA0007/">TA0007</a>]). The latest release of this program (v2.60) contains a list of common admin names and passwords that can be used for a dictionary attack to gain access to a network router (<em>Credential Access</em> [<a href="https://attack.mitre.org/versions/v9/tactics/TA0006/">TA0006</a>], <em>Brute Force: Password Guessing</em> [<a href="https://attack.mitre.org/versions/v9/techniques/T1110/001/">T1110.001</a>]). The program also contains code to identify common vulnerabilities and leverage exploits against many popular routers (<em>Active Scanning: Vulnerability Scanning</em> [<a href="https://attack.mitre.org/versions/v9/techniques/T1595/002/">T1595.002</a>]). The program can be customized to scan any subnet and any particular port, or protocol (<em>Network Service Scanning</em> [<a href="https://attack.mitre.org/versions/v9/techniques/T1046/">T1046</a>]). The latest version also contains software to scan for wireless network access points (<em>System Network Connections Discovery</em> [<a href="https://attack.mitre.org/versions/v9/techniques/T1049/">T1049</a>]).&nbsp;</p>
<p>To execute this program, two libraries are required: <code>librouter.dll</code> and <code>libeay32.dll</code>. Upon execution, the program will generate several telemetry files that are dropped in the current directory. These files are named <code>RouterScan.log</code>, <code>Config.ini</code>, <code>filter.txt</code>, <code>exclusions.txt</code>, <code>ports.txt</code>, and <code>ranges.txt</code>.</p>
<h4><strong>Open-Source Tool: grabff.exe</strong></h4>
<p>The <code>grabff.exe</code> artifact is a 32-bit .NET executable called grabff and is used for <em>Credential Access</em> [<a href="https://attack.mitre.org/versions/v9/tactics/TA0006/">TA0006</a>]. The program uses a command line interface to extract Firefox stored passwords and authentication information from the user’s profile located at <code>C:\Users\&lt;user&gt;\AppData\Roaming\Mozilla\Firefox\Profiles</code> (<em>Command and Scripting Interpreter: Windows Command Shell</em> [<a href="https://attack.mitre.org/versions/v9/techniques/T1059/003/">T1059.003</a>], Credentials from Password Stores: Credentials from Web Browsers [<a href="https://attack.mitre.org/versions/v9/techniques/T1555/003/">T1555.003</a>]). The program will extract the password databases found in <code>key3.db</code>, <code>key4.db</code>, and <code>logins.json</code> as well as the SQLite-based certificate database, <code>cert9.db</code>. The data can be copied to any designated directory.</p>
<h4><strong>Open-Source Tool: rclone.exe</strong></h4>
<p>The <code>rclone.exe</code> artifact is an open-source cloud content management program called Rclone. The program uses a command line interface to manage files in cloud storage. The program is capable of uploading and downloading files, verifying file integrity, and providing file encryption. The program can use any of the following protocols: SSH File Transfer Protocol (SFTP), Web Distributed Authoring and Versioning (WebDAV), File Transfer Protocol (FTP), and Digital Living Network Alliance (DLNA).</p>
<h4><strong>s3browser-9-5-3.exe</strong></h4>
<p>The <code>s3browser-9-5-3.exe</code> artifact is the free version of the S3 Browser program used to upload and download data from a cloud account. The program can fully configure a cloud account, modify HTTP headers and object tags, enable multiple simultaneous uploads and downloads, and provide server-side encryption (<em>Create Account: Cloud Account</em> [<a href="https://attack.mitre.org/versions/v9/techniques/T1136/003/">T1136.003</a>]). By default, the installed components of the program are stored in the path <code>C:\Program Files\S3 Browser</code>. Activity logs are created on a daily basis and are stored in the path <code>C:\Users\&lt;user&gt;\AppData\Roaming\S3Browser\logs</code> in the format <code>s3browser-win32-YYYY-MM-DD-log.txt</code>.</p>
<div>
<h3>Summary</h3>
</div>
<p><em>This Analysis Report&nbsp;uses the MITRE Adversarial Tactics, Techniques, and Common Knowledge (ATT&amp;CK®) framework, Version 9. See the <a href="https://attack.mitre.org/versions/v7/matrices/enterprise/">ATT&amp;CK for Enterprise</a> framework for all referenced threat actor tactics and techniques.</em></p>
<p>The Cybersecurity and Infrastructure Security Agency (CISA) is aware of a recent successful cyberattack against an organization using a new ransomware variant, which CISA refers to as FiveHands. Threat actors used publicly available penetration testing and exploitation tools, FiveHands ransomware, and SombRAT remote access trojan (RAT), to steal information, obfuscate files, and demand a ransom from the victim organization. Additionally, the threat actors used publicly available tools for network discovery and credential access.</p>
<p>This report provides the tactics, techniques, and procedures the threat actors used in this attack as well as indicators of compromise (IOCs). It also includes CISA’s recommended mitigations to protect networks from ransomware attacks and to detect—and respond to—these attacks.</p>
<p>Refer to Malware Analysis Report <a href="https://us-cert.cisa.gov/ncas/analysis-reports/ar21-126b">AR21-126B</a> for full technical details and associated IOCs.</p>
<p>For a PDF copy of this report, <a href="/sites/default/files/publications/AR21-126A_FiveHands_Ransomware.pdf">click here</a>.</p>
<p><strong>Note:</strong> the analysis of FiveHands ransomware is ongoing; CISA will update this report as new information becomes available.</p>
<div>
<h3>Solution</h3>
</div>
<p>Should your organization be a victim of ransomware, CISA strongly recommends responding by using the Ransomware Response Checklist located in the <a href="/resources-tools/resources/stopransomware-guide" title="Ransomware Guide">Joint Ransomware Guide</a>, co-authored by CISA and the Multi-State Information Sharing and Analysis Center. The guide contains steps for detection and analysis as well as containment and eradication.</p>
<p>CISA recommends organizations implement the following practices to strengthen the security posture of their systems.&nbsp;</p>
<ul>
<li>Maintain up-to-date antivirus signatures and engines.</li>
<li>Keep operating system patches up to date.</li>
<li>Disable file and printer sharing services. If these services are required, use strong passwords or Active Directory authentication.</li>
<li>Restrict users' ability (permissions) to install and run unwanted software applications. Do not add users to the local administrators group unless required.</li>
<li>Implement multi-factor authentication (MFA), particularly on all VPN connections, external-facing services, and privileged accounts. Where MFA is not implemented, enforce a strong password policy and implement regular password changes.</li>
<li>Decommission unused VPN servers, which may act as a point of entry for attackers.&nbsp;</li>
<li>Monitor network traffic for unexpected and unapproved protocols, especially outbound to the internet (e.g., SSH, SMB, RDP).</li>
<li>Exercise caution when opening email attachments even if the attachment is expected and the sender appears to be known.</li>
<li>Enable a personal firewall on agency workstations, configured to deny unsolicited connection requests.</li>
<li>Disable unnecessary services on agency workstations and servers.</li>
<li>Scan for—and remove—suspicious e-mail attachments; ensure the scanned attachment is its "true file type" (i.e., the extension matches the file header).</li>
<li>Monitor users' web browsing habits; restrict access to sites with unfavorable content.</li>
<li>Exercise caution when using removable media (e.g., USB thumb drives, external drives, CDs).</li>
<li>Scan all software downloaded from the internet prior to executing.</li>
<li>Maintain situational awareness of the latest threats and implement appropriate access control lists (ACLs).</li>
</ul>
