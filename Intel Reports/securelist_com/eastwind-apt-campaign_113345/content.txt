												<div class="c-wysiwyg">
													<p>In late July 2024, we detected a series of ongoing targeted cyberattacks on dozens of computers at Russian government organizations and IT companies. The threat actors infected devices using phishing emails with malicious shortcut attachments. These shortcuts were used to deliver malware that received commands via the Dropbox cloud service. Attackers used this malware to download additional payloads onto infected computers, in particular tools used by the APT31 group and an updated CloudSorcerer backdoor. We dubbed this campaign EastWind.</p>
<p>Below are the most interesting facts about the implants used in this campaign:</p>
<ul>
<li>The malware downloaded by the attackers from Dropbox has been used by APT31 since at least 2021. We named it GrewApacha.</li>
<li>The attackers updated the The CloudSorcerer backdoor (<a href="https://securelist.com/cloudsorcerer-new-apt-cloud-actor/113056/" target="_blank" rel="noopener">described</a> by us in early July 2024) ) after we published our blogpost. It currently uses LiveJournal (a social network popular in Russia) and Quora profiles as initial C2 servers.</li>
<li>The attacks additionally deploy a previously unknown implant with a classic backdoor functionality, which we dubbed PlugY. It is loaded via the CloudSorcerer backdoor, and its command set is quite extensive. It supports three different protocols for communicating with C2, and what&#8217;s more, its code resembles that of the DRBControl backdoor (aka Clambling), which <a href="https://documents.trendmicro.com/assets/white_papers/wp-uncovering-DRBcontrol.pdf" target="_blank" rel="noopener">several</a> <a href="https://cdn.prod.website-files.com/660e348ec87579d74705013a/66562c2c7fbdb8cb33a5c396_APT27_turns_to_ransomware.pdf" target="_blank" rel="noopener">companies</a> attribute to the APT27 group.</li>
</ul>
<h2 id="technical-information">Technical information</h2>
<p>As mentioned above, the attackers used spear phishing to gain an initial foothold into the organizations. They sent malicious emails with attached RAR archives to target organizational email addresses. These archives had the following names:</p>
<ul>
<li><em>инициативная группа из Черниговского района Приморского края.rar </em>(translates as <em>advocacy group from Chernigov district of Primorsky Krai.rar</em>)</li>
<li><em>вх.rar</em></li>
</ul>
<p>They contained the following files:</p>
<ul>
<li><em>.con</em> folder, which contained:
<ul>
<li><em>1.docx</em>, a legitimate decoy document</li>
<li><em>desktop.exe</em>, a legitimate file</li>
<li><em>VERSION.dll</em>, a malicious file</li>
</ul>
</li>
<li>A malicious shortcut with a name similar to that of the archive.</li>
</ul>

Once connected to the Dropbox cloud, the backdoor reads commands to be executed from the file <em>&lt;computer name&gt;/a.psd</em> contained in the storage. <h3 id="grewapacha-a-rat-used-by-apt31-since-2021">GrewApacha: a RAT used by APT31 since 2021</h3>

<ul>
<li><em>msedgeupdate.exe</em>, a legitimate executable file signed by Microsoft</li>
<li><em>msedgeupdate.dll</em>, a malicious library</li>
<li><em>wd</em>, a file with an encrypted payload</li>
</ul>

<p>While this set of three files resembles the &#8220;sideloading triad&#8221; that is typical of attacks involving <a href="https://securelist.com/plugx-malware-a-good-hacker-is-an-apologetic-hacker/74150/" target="_blank" rel="noopener">PlugX</a>analysis of these files revealed that the malware inside them is a RAT of the APT31 group, already described in <a href="https://www.ptsecurity.com/ww-en/analytics/pt-esc-threat-intelligence/apt31-new-attacks/?sphrase_id=322222" target="_blank" rel="noopener">2021</a> and <a href="https://habr.com/ru/companies/solarsecurity/articles/723526/" target="_blank" rel="noopener">2023</a>. <p>The behavior of the loader (<em>msedgeupdate.dll</em>) hasn&#8217;t changed since the 2023 post was published. As before, it decrypts the payload stored on the drive using the XOR key 13 18 4F 29 0F, and loads it into the <em>dllhost.exe</em> process.</p>
<p>While the GrewApacha loader has not changed since last year, there have been minor differences introduced to the RAT itself. Specifically, the new version now uses two C2 servers instead of one. Through network communications, the cybercriminals first retrieve a webpage with a profile bio on GitHub. This profile contains a string encoded with the Base64 algorithm:</p>
<div id="attachment_113356" style="width: 392px" class="wp-caption aligncenter"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/25120641/EastWind_05.png" class="magnificImage"><img loading="lazy" decoding="async" aria-describedby="caption-attachment-113356" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2024/09/25120641/EastWind_05.png" alt="" width="382" height="570" class="size-full wp-image-113356" /></a><p id="caption-attachment-113356" class="wp-caption-text">Profile of a user created by the attackers on GitHub</p></div>

<p>This library was programmed to read the previously mentioned <em>.ini file</em>, which contains:</p>
<ul>
<li>The ciphertext of a four-byte number generated and encrypted by the <em>GetKey.exe</em> utility</li>

<p>Accordingly, the library proceeded to decrypt the four-byte number using the CryptUnprotectData function, use it to decrypt the <em>.ini</em> file, and then load the decrypted file into the memory of the current process.</p>
<p>Analysis of the decrypted <em>.ini</em> files revealed them to be updated versions of the CloudSorcerer backdoor. After we publicly described this backdoor in early July 2024, the attackers modified it: the new version of CloudSorcerer uses profile pages on the Russian-language social network LiveJournal and the Q&#038;A site Quora as the initial C2 servers:</p>

<p>As with past versions of CloudSorcerer, the profile bios contain an encrypted authentication token for interaction with the cloud service.</p>
<h3 id="plugy-an-implant-that-overlaps-with-apt27-tools">PlugY: an implant that overlaps with APT27 tools</h3>
<p>Having analyzed the behavior of the newly found CloudSorcerer samples, we found that the attackers used it to download a previously unknown implant. This implant connects to the C2 server by one of three methods:</p>
<ul>
<li>TCP protocol</li>
<li>UDP protocol</li>
<li>Named pipes</li>
</ul>
<p>The set of commands this implant can handle is quite extensive, and implemented commands range from manipulating files and executing shell commands to logging keystrokes and monitoring the screen or the clipboard.</p>
<p>Analysis of the implant is still ongoing, but we can conclude with a high degree of confidence that the code of the DRBControl (aka Clambling) backdoor was used to develop it. This backdoor was described in 2020 by <a href="https://documents.trendmicro.com/assets/white_papers/wp-uncovering-DRBcontrol.pdf">Trend Micro</a> and <a href="https://www.talent-jump.com/article/2020/02/17/CLAMBLING-A-New-Backdoor-Base-On-Dropbox-en/" target="_blank" rel="noopener">Talent-Jump Technologies</a>. Later, Security Joes and Profero linked it to the <a href="https://cdn.prod.website-files.com/660e348ec87579d74705013a/66562c2c7fbdb8cb33a5c396_APT27_turns_to_ransomware.pdf" target="_blank" rel="noopener">APT27</a> group. The backdoor also has similarities to PlugX.</p>
<p>Our comparison of samples of the PlugY implant (MD5 example: <a href="https://opentip.kaspersky.com/faf1f7a32e3f7b08017a9150dccf511d/?icid=gl_OpenTIP_acq_ona_smm__onl_b2b_securelist_lnk_sm-team_______d4d6e81351dbd72c?utm_source=SL&amp;utm_medium=SL&amp;utm_campaign=SL">faf1f7a32e3f7b08017a9150dccf511d</a>) and the DRBControl backdoor (MD5: <a href="https://opentip.kaspersky.com/67cfecf2d777f3a3ff1a09752f06a7f5/?icid=gl_OpenTIP_acq_ona_smm__onl_b2b_securelist_lnk_sm-team_______4a292b7ac6b56950?utm_source=SL&amp;utm_medium=SL&amp;utm_campaign=SL" target="_blank" rel="noopener">67cfecf2d777f3a3ff1a09752f06a7f5</a>) revealed that these two samples have the exact same architecture. Additionally, many commands in them are implemented almost identically, as evidenced by the screenshots below:<br />

<p>Thus, the code previously observed in attacks by APT27 was likely used in developing the implant.</p>
<p>While analyzing the PlugY implant we also noticed that it uses a unique malicious library to communicate with the C2 server via UDP. We found the very same library in the DRBControl backdoor, as well as several samples of the PlugX backdoor, which is popular among Chinese-speaking groups. Apart from DRBControl and PlugX, this library has not been detected in any other malware.</p>

<h2 id="tips-for-attack-detection">Tips for attack detection</h2>
<p>The implants identified during the attack significantly differ from each other. As such, it&#8217;s necessary to use a separate set of IoCs for each malware used in any compromise.</p>
<p>The backdoor that uses Dropbox and is delivered via email can be found by looking for relatively large DLL files (> 5 MB) located in the directory <em>C:\Users\Public</em>. Regular access to the Dropbox cloud in network traffic can serve as an additional indicator of this backdoor&#8217;s operation.</p>
<p>The GrewApacha Trojan can be detected by searching for an unsigned file named <em>msedgeupdate.dll</em> in the file system. This file also reaches several megabytes in size.<br />
The PlugY implant that is delivered using the CloudSorcerer backdoor launches a process named msiexec.exe for each user signed to the OS, and also creates named pipes with the name template <em>\.\PIPE\Y<number></em>. The presence of these two indicators in the system is strong evidence of an infection.</p>
<h2 id="conclusion">Conclusion</h2>
<p>In attacks on government organizations, threat actors often use toolkits that implement a wide variety of techniques and tactics. In developing these tools, they go to the greatest lengths possible to hide malicious activity in network traffic. For instance, the attackers behind the EastWind campaign, for instance, used popular network services (GitHub, Dropbox, Quora, LiveJournal and Yandex.Disk) as C2 servers. </p>
<p>Notably, the EastWind campaign bore traces of malware from two different Chinese-speaking groups: APT27 and APT31. This clearly shows that APT groups very often team up, actively sharing knowledge and tools. To successfully counter such collaborations, we closely monitor the techniques and tactics of APT groups operating around the world. </p>
