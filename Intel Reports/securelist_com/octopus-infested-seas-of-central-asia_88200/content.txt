																							<h2>Russian-language actor exploits hype over Telegram ban in Central Asia</h2>											
											<div class="js-reading-content">
												<div class="c-wysiwyg">
													<p>For the last two years we have been monitoring a Russian-language cyberespionage actor that focuses on Central Asian users and diplomatic entities. We named the actor DustSquad and have provided private intelligence reports to our customers on four of their campaigns involving custom Android and Windows malware. In this blogpost we cover a malicious program for Windows called Octopus that mostly targets diplomatic entities.</p>
<p>The name was originally coined by ESET in 2017 after the 0ct0pus3.php script used by the actor on their old C2 servers. We also started monitoring the malware and, using Kaspersky Attribution Engine based on similarity algorithms, discovered that Octopus is related to DustSquad, something we reported in April 2018. In our telemetry we tracked this campaign back to 2014 in the former Soviet republics of Central Asia (still mostly Russian-speaking), plus Afghanistan.</p>
<p>In the case of Octopus, DustSquad used Delphi as their programming language of choice, which is unusual for such an actor. Among others exceptions are the Russian-language Zebrocy (Sofacy&#8217;s Delphi malware), the Hindi-language DroppingElephant and the Turkish-language StrongPity. Although we detected Octopus victims that were also infected with Zebrocy/Sofacy, we didn&#8217;t find any strong similarities and we don&#8217;t consider the two actors to be related.</p>
<h2 id="what-happened">What happened?</h2>
<p>In April 2018 we discovered a new Octopus sample pretending to be Telegram Messenger with a Russian interface. We couldn´t find any legitimate software that this malware appears to be impersonating; in fact, we don´t believe it exists. The Trojan uses third-party Delphi libraries like The Indy Project for JSON-based C2 communications and TurboPower Abbrevia (sourceforge.net/projects/tpabbrevia) for compression. Malware persistence is basic and achieved via the system registry. The server side uses commercial hosting in different countries with .php scripts deployed. Kaspersky Lab products detect the Octopus Trojan as Trojan.Win32.Octopus.gen. For more information, please contact: <a target="_blank" href="mailto:intelreports@kaspersky.com" rel="noopener">intelreports@kaspersky.com</a>.</p>
<h2 id="technical-details">Technical details</h2>
<p>The attackers used the potential Telegram ban in Kazakhstan to push its dropper as an alternative communication software for the political opposition.</p>
<div style="width: 1010px" class="wp-caption aligncenter"><a target="_blank" href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2018/10/12104635/181012-octopus-1.png" rel="noopener" class="magnificImage"><img decoding="async" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2018/10/12104635/181012-octopus-1.png" style="max-height:60vh" alt="" class="aligncenter size-full wp-image-81520" /></a><p class="wp-caption-text"><em>&#8216;Telegram messenger&#8217; establishes network module persistence in the simplest way and starts the module</em></p></div>
<p>We can&#8217;t confirm how this malware is being distributed, although it clearly uses some form of social engineering. This actor previously used spear phishing to spread malware.</p>

<p>All network modules consist of hardcoded IP addresses belonging to commercial web-hosting services based in different countries. The operators simply deploy their first-stage .php script in them, which will check the connection and get the actual C2 server domain name using an HTTP GET request.</p>
<div style="width: 1010px" class="wp-caption aligncenter"><a target="_blank" href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2018/10/12104631/181012-octopus-5.png" rel="noopener" class="magnificImage"><img decoding="async" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2018/10/12104631/181012-octopus-5.png" style="max-height:60vh" alt="" class="aligncenter size-full wp-image-81520" /></a><p class="wp-caption-text"><em>After the initial connection check, the malware receives a JSON with the actual C2 domain name</em></p></div>
<div style="width: 1010px" class="wp-caption aligncenter"><a target="_blank" href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2018/10/12104631/181012-octopus-6.png" rel="noopener" class="magnificImage"><img decoding="async" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2018/10/12104631/181012-octopus-6.png" style="max-height:60vh" alt="" class="aligncenter size-full wp-image-81520" /></a><p class="wp-caption-text"><em>Then the network module checks against the hardcoded victim&#8217;s id</em></p></div>
<p>The network module checks against a 32-digit hardcoded victim id and sends the gathered data to the C2 using a HTTP POST request. In terms of programming, this id is strange, because the malware simultaneously &#8216;fingerprints&#8217; its victim with an MD5 hash of its system data.</p>
<div style="width: 1010px" class="wp-caption aligncenter"><a target="_blank" href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2018/10/12104631/181012-octopus-7.png" rel="noopener" class="magnificImage"><img decoding="async" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2018/10/12104631/181012-octopus-7.png" style="max-height:60vh" alt="" class="aligncenter size-full wp-image-81520" /></a><p class="wp-caption-text"><em>JSON-based gathered data sent in a HTTP POST base64-encoded request</em></p></div>
<p>All communication with the C2s is based on JSON-formatted data and the HTTP protocol. For that, the developers used The Indy Project (indyproject.org) publicly available library as well as the third-party TurboPower Abbrevia (sourceforge.net/projects/tpabbrevia) for compression.</p>
<p>After all the initial HTTP GET requests, the malware starts to gather JSON-formatted system data. For all the fixed drives in the system, the network module stores the disk name and size, as well as computer and user name, Windows directory, host IP, etc. One interesting field is &#8220;vr&#8221;:&#8221;2.0&#8243; which appears to be the malware version encoded in the communication protocol.</p>
<p>The &#8216;id&#8217; field is the victim&#8217;s fingerprint for which the malware actively uses the Windows Management Instrumentation mechanism. The Trojan runs WMIC.exe with the following arguments:</p>
<table>
<tbody>
<tr></tr>
<tr>
<td>C:\WINDOWS\system32\wbem\WMIC.exe computersystem get Name /format:list</td>
</tr>
<tr>
<td>C:\WINDOWS\system32\wbem\WMIC.exe os get installdate /format:list</td>
</tr>
<tr>
<td>C:\WINDOWS\system32\wbem\WMIC.exe path CIM_LogicalDiskBasedOnPartition get Antecedent,Dependent</td>
</tr>
</tbody>
</table>
<p>Then the module concatenates the gathered ids and computes an MD5 hash, which will be the victim&#8217;s final id.  The &#8220;act&#8221; field numbers the communication stage (0 for initial fingerprinting). After this, the HTTP POST control server returns a JSON {&#8220;rt&#8221;:&#8221;30&#8243;} and the client continues with the next &#8220;act&#8221; in the HTTP POST:</p>
<p><a target="_blank" href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2018/10/12104631/181012-octopus-8.png" rel="noopener" class="magnificImage"><img decoding="async" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2018/10/12104631/181012-octopus-8.png" style="max-height:60vh" alt="" class="aligncenter size-full wp-image-81520" /></a></p>
<p>At this point the C2 sends a JSON with commands to execute, including uploading/downloading files, taking a screenshot and finding *.rar archives on the host.</p>
<h3><strong>Other software</strong></h3>
<p>Besides the Trojan itself, the Octopus developers used the password dumping utility <a target="_blank" href="http://foofus.net/goons/fizzgig/fgdump/" rel="noopener">fgdump</a>.</p>

<h2 id="conclusions">Conclusions</h2>
<p>Political entities in Central Asia have been targeted throughout 2018 by different actors, including IndigoZebra, Sofacy (with Zebrocy malware) and most recently by DustSquad (with Octopus malware). Interestingly, we observed some victims who are &#8216;threat magnets&#8217; targeted by all of them. From our experience we can say that the interest shown by threat actors in this region is now high, and the traditional &#8216;players&#8217; have been joined by relative newcomers like DustSquad that have sprung up locally.</p>
