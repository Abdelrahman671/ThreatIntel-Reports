lg:col-span-3"><div class="prose lg:prose-lg prose-invert w-full article-content"><div><h2 class="font-bold text-2xl md:text-4xl relative"><span id="preamble" class="absolute -top-32"></span>Preamble</h2>
<p>Elastic Security Labs has identified an intrusion set incorporating several malicious modules and leveraging vulnerable drivers to disable known security solutions (EDRs) for crypto mining. Additionally, the team discovered capabilities to establish persistence, install a previously undocumented backdoor, and execute a crypto-miner. We refer to this intrusion set as REF4578 and the primary payload as GHOSTENGINE (tangental research by the team at Antiy has named parts of this intrusion set <a href="https://www.antiy.com/response/HideShoveling.html">HIDDENSHOVEL</a>).</p>
<h2 class="font-bold text-2xl md:text-4xl relative"><span id="key-takeaways" class="absolute -top-32"></span>Key takeaways</h2>
<ul>
<li>Malware authors incorporated many contingency and duplication mechanisms</li>
<li>GHOSTENGINE leverages vulnerable drivers to terminate and delete known EDR agents that would likely interfere with the deployed and well-known coin miner</li>
<li>This campaign involved an uncommon amount of complexity to ensure both the installation and persistence of the XMRIG miner</li>
</ul>
<h2 class="font-bold text-2xl md:text-4xl relative"><span id="code-analysis" class="absolute -top-32"></span>Code analysis</h2>
<p></p>
<p>On May 6, 2024, at 14:08:33 UTC,  the execution of a PE file named <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">Tiworker.exe</code> (masquerading as the legitimate Windows <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">TiWorker.exe</code> file) signified the beginning of the REF4578 intrusion. The following alerts were captured in telemetry, indicating a known vulnerable driver was deployed.</p>
<p></p>
<p>Upon execution, this file downloads and executes a PowerShell script that orchestrates the entire execution flow of the intrusion. Analysis revealed that this binary executes a hardcoded PowerShell command line to retrieve an obfuscated script, <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">get.png,</code> which is used to download further tools, modules, and configurations from the attacker C2– as depicted in the screenshot below.</p>
<p></p>
<h3 class="font-bold leading-tight text-xl md:text-3xl relative"><span id="ghostengine" class="absolute -top-32"></span>GHOSTENGINE</h3>
<p>GHOSTENGINE is responsible for retrieving and executing modules on the machine. It primarily uses HTTP to download files from a configured domain, with a backup IP in case domains are unavailable. Additionally, it employs FTP as a secondary protocol with embedded credentials. The following is a summary of the execution flow:</p>
<p></p>

<h3 class="font-bold leading-tight text-xl md:text-3xl relative"><span id="ghostengine-modules" class="absolute -top-32"></span>GHOSTENGINE modules</h3>
<p>GHOSTENGINE deploys several modules that can tamper with security tools, create a backdoor, and check for software updates.</p>
<h4 class="font-bold leading-tight text-lg md:text-2xl relative"><span id="edr-agent-controller-and-miner-module-smartsscreenexe" class="absolute -top-32"></span>EDR agent controller and miner module: smartsscreen.exe</h4>
<p>This module primarily terminates any active EDR agent processes before downloading and installing a crypto-miner.</p>
<p></p>
<p>The malware scans and compares all the running processes with a hardcoded list of known EDR agents. If there are any matches, it first terminates the security agent by leveraging the Avast Anti-Rootkit Driver file <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">aswArPots.sys</code> with the IOCTL <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">0x7299C004</code> to terminate the process by PID.</p>
<p><code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">smartscreen.exe</code> is then used to delete the security agent binary with another vulnerable driver, <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">iobitunlockers.sys</code> from IObit, with the IOCTL <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">0x222124</code>.</p>
<p><code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">smartscreen.exe</code> then downloads the XMRig client mining program (<code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">WinRing0x64.png</code>) from the C2 server as <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">taskhostw.png</code>. Finally, it executes XMRig, its drivers, and the configuration file <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">config.json</code>, starting the mining process.</p>
<p></p>
<h4 class="font-bold leading-tight text-lg md:text-2xl relative"><span id="updatepersistence-module-ocidll" class="absolute -top-32"></span>Update/Persistence module: oci.dll</h4>
<p>The PowerShell script creates a service DLL (<code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">oci.dll</code>), a phantom DLL loaded by <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">msdtc</code>. The DLL&#x27;s architecture varies depending on the machine; it can be 32-bit or 64-bit. Its primary function is to create system persistence and download any updates from the C2 servers by downloading the <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">get.png</code> script from the C2 and executing it.</p>
<p></p>
<p>Every time the <code>msdtc<strong> </strong></code>service starts, it will load <code>oci.dll</code> to spawn the PowerShell one-liner that executes <code>get.png</code> :</p>
<p></p>
<h4 class="font-bold leading-tight text-lg md:text-2xl relative"><span id="edr-agent-termination-module-killpng" class="absolute -top-32"></span>EDR agent termination module: <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">kill.png</code></h4>
<p><code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">kill.png</code> is a PowerShell script that injects shellcode into the current process, decrypting and loading a PE file into memory.</p>
<p></p>
<p>This module is written in C++, and the authors have integrated redundancy into its operation. This redundancy is evident in the replication of the technique used in <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">smartsscreen.exe</code> to terminate and delete EDR agent binaries; it continuously scans for any new processes.</p>
<p></p>
<h4 class="font-bold leading-tight text-lg md:text-2xl relative"><span id="powershell-backdoor-module-backuppng" class="absolute -top-32"></span>Powershell backdoor module: <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">backup.png</code></h4>
<p>The PowerShell script functions like a backdoor, enabling remote command execution on the system. It continually sends a Base64-encoded JSON object containing a unique ID, derived from the current time and the computer name while awaiting base64-encoded commands. The results of those commands are then sent back.</p>
<p></p>
<p>In this example <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">eyJpZCI6IjE3MTU2ODYyNDA3MjYyNiIsImhvc3QiOiJhbmFseXNpcyJ9</code> is the Base64-encoded JSON object:</p>
<p></p>
<pre><code>$ echo &quot;eyJpZCI6IjE3MTU2ODYyNDA3MjYyNiIsImhvc3QiOiJhbmFseXNpcyJ9&quot; | base64 -D
{&quot;id&quot;:&quot;171568624072626&quot;,&quot;host&quot;:&quot;analysis&quot;}</code></pre>

<h2 class="font-bold text-2xl md:text-4xl relative"><span id="malware-and-mitre-attck" class="absolute -top-32"></span>Malware and MITRE ATT&amp;CK</h2>
<p>Elastic uses the <a href="https://attack.mitre.org/">MITRE ATT&amp;CK</a> framework to document common tactics, techniques, and procedures that threats use against enterprise networks.</p>
<h3 class="font-bold leading-tight text-xl md:text-3xl relative"><span id="tactics" class="absolute -top-32"></span>Tactics</h3>

<h2 class="font-bold text-2xl md:text-4xl relative"><span id="mitigating-ghostengine" class="absolute -top-32"></span>Mitigating GHOSTENGINE</h2>
<h3 class="font-bold leading-tight text-xl md:text-3xl relative"><span id="detection" class="absolute -top-32"></span>Detection</h3>
<p>The first objective of the GHOSTENGINE malware is to incapacitate endpoint security solutions and disable specific Windows event logs, such as Security and System logs,  which record process creation and service registration. Therefore, it is crucial to prioritize the detection and prevention of these initial actions:</p>
<ul>
<li>Suspicious PowerShell execution</li>
<li>Execution from unusual directories</li>
<li>Elevating privileges to system integrity</li>
<li>Deploying vulnerable drivers and establishing associated kernel mode services.</li>
</ul>
<p>Once the vulnerable drivers are loaded, detection opportunities decrease significantly, and organizations must find compromised endpoints that stop transmitting logs to their SIEM.</p>
<p>Network traffic may generate and be identifiable if DNS record lookups point to <a href="https://miningpoolstats.stream/monero">known mining pool</a> domains over well-known ports such as HTTP (<code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">80</code>) and HTTPS  (<code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">443</code>). Stratum is also another popular network protocol for miners, by default, over port <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">4444</code>.</p>
