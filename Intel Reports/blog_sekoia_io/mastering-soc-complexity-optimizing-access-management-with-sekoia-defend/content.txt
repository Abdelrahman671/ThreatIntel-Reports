<p></p>



<p>In today’s evolving cybersecurity landscape, organizations face the challenge of monitoring and responding to threats efficiently through their Security Operations Centers (SOC). Depending on an organization&#8217;s size and security priorities, different SOC models are adopted, each with its own approach to security management.</p>



<span id="more-10388"></span>



<p>Some companies opt for a <strong>fully in-house SOC</strong>, with internal teams managing all aspects of security operations. Large corporations, in particular, benefit from this model, given their resources to maintain a dedicated security staff. On the other hand, <strong>hybrid SOCs</strong> combine internal expertise with external Managed Security Service Providers (MSSPs), allowing businesses to maintain control over some aspects of security while outsourcing monitoring and threat detection. Smaller organizations often rely on <strong>outsourced SOCs</strong>, entirely managed by MSSPs, which offer scalability and expert monitoring without the need for a full in-house team.</p>



<p>Each SOC model offers unique advantages:</p>



<ul class="wp-block-list">
<li><strong>In-house SOCs</strong> provide complete control over security operations.</li>



<li><strong>Hybrid SOCs</strong> offer a balance of internal oversight and outsourced expertise.</li>



<li><strong>Outsourced SOCs</strong> bring cost-effective security solutions with scalability.</li>
</ul>



<p>However, each SOC model presents challenges, particularly when it comes to managing access to sensitive security data across multiple teams.</p>



<h2 class="wp-block-heading" id="h-managing-access-across-teams-the-challenge-in-soc-models">Managing access across teams: The challenge in SOC models</h2>



<p>In hybrid and outsourced SOC models, managing access for different stakeholders—including internal security teams, MSSP personnel, and other IT departments—can be complex. Even different teams than security ones may need access to specific data, such as network logs for infrastructure team, without being exposed to broader security intelligence. This situation calls for <strong>fine-grained control</strong> over permissions, ensuring teams only access the data necessary for their role, while maintaining strict data segregation to prevent unauthorized exposure.</p>



<p>Effective access management becomes increasingly complex as organizations rely on both internal and external teams, requiring <strong>robust security tools</strong> to manage permissions and safeguard sensitive information.</p>



<h2 class="wp-block-heading" id="h-how-sekoia-defend-streamlines-soc-access-management">How Sekoia Defend streamlines SOC access management</h2>



<p>Sekoia Defend addresses these access challenges with two standout features: <strong>multi-tenant support</strong> and <strong>data segregation by Intakes (data sources)</strong>. These features offer flexibility and control, allowing organizations to segment security environments and manage data access with precision.</p>



<h3 class="wp-block-heading" id="h-multi-tenant-architecture-enhancing-large-operations">Multi-Tenant architecture: Enhancing large operations</h3>



<p>For MSSPs managing multiple clients or large companies managing multiple subsidiaries, <strong>multi-tenant architecture</strong> is crucial. Sekoia Defend allows to operate a <strong>parent tenant</strong> (called “<strong>workspace</strong>”) to oversee all clients, while each client is provided with an isolated <strong>child tenant</strong> (called ****“<strong>community</strong>” ****in the product). This ensures complete data segregation between clients while enabling centralized management from the parent tenant. MSSPs or large companies can customize security configurations for each client, maintaining full control without cross-contaminating environments. This structure simplifies complex environments, enabling flexible management of security operations across multiple structures.</p>



<h3 class="wp-block-heading" id="h-entities-amp-intakes-the-foundation-for-segmentation">Entities &amp; Intakes: The foundation for segmentation</h3>



<p>In Sekoia Defend, an <strong>entity</strong> represents a distinct organizational unit within a community. Each entity serves as a functional container for grouping related resources and data, which simplifies both data management and access control. This structure allows organizations to manage multiple departments, teams, or subsidiaries within a single platform. <strong>Intakes</strong> in Sekoia Defend act as the primary gateway for data ingestion. Each intake is a specialized entry point that processes various types of data, such as logs and alerts, from different sources. This system ensures data is efficiently parsed and normalized for accurate analysis. Each Intake needs to be configured within an entity in the platform, allowing accurate segregation of data through Sekoia Defend’s RBAC model.</p>



<h3 class="wp-block-heading" id="h-role-based-access-control-rbac-ensuring-secure-permissions">Role-Based Access Control (RBAC): Ensuring secure permissions</h3>



<p>Sekoia Defend’s <strong>RBAC</strong> is an essential security model designed to restrict system access to authorized users based on their specific roles within a community. Each role corresponds with a set of permissions that dictate what actions a user can perform and which part of the product they can access. This method <strong>enhances overall security</strong> by ensuring that users have only the permissions necessary to perform their job functions, thereby reducing the risk of data breaches or unauthorized actions. The platform offers three <strong>built-in</strong> roles: <strong>Admin, Analyst, and Guest</strong>. Additionally, in Sekoia Defend’s <em><strong>Prime</strong></em> plan, administrators have the flexibility to create custom and/or restricted roles that are tailored to the specific needs of their organization.</p>



<h3 class="wp-block-heading" id="h-restricted-roles-to-segregate-access-to-data-by-intakes">Restricted roles to segregate access to data by Intakes</h3>



<p>Thanks to this RBAC model, Sekoia Defend allows <strong>data segregation through Intakes</strong>. Access to these Intakes can be controlled by intake restricted roles, enabling organizations to compartmentalize data streams so that only users with the appropriate permissions can access specific subsets of data in a read-only mode, accessing to this specific events &amp; alerts only. This integrated approach between custom roles and Intakes adds an additional layer of security by ensuring that data segregation aligns with organizational requirements. By combining RBAC with data segregation through Intakes, Sekoia Defend provides a <strong>robust framework for managing permissions</strong>, aligning closely with both security best practices and compliance requirements.</p>



<p>This architecture of entities, intakes and RBAC model forms the backbone of Sekoia Defend&#8217;s flexible and secure data handling system, enabling organizations to efficiently manage and analyze their security data while maintaining strict control over data access and processing.</p>

<p></p>



<h3 class="wp-block-heading" id="h-use-case-1-mssp-managing-multiple-clients">Use Case 1: MSSP managing multiple clients</h3>



<p>As the head of a Security Operations Center (SOC) within an MSSP, you are tasked with protecting the IT infrastructures of multiple clients, each with unique requirements. Sekoia Defend’s multi-tenant feature allows you to configure a <strong>parent tenant</strong> (Workspace) to centralize your operations while creating <strong>child tenants</strong> (communities) for each client.</p>



<p>With this architecture, you can:</p>



<ul class="wp-block-list">
<li>Completely separate the environments of each client, ensuring their logs, incidents, and data remain isolated.</li>



<li>Administer each child tenant independently while maintaining global control over all environments from the parent tenant.</li>



<li>Tailor detection rules and playbooks to the specific needs of each client without risking cross-contamination of security configurations.</li>
</ul>

<h3 class="wp-block-heading" id="h-use-case-2-large-companies-with-subsidiaries">Use Case 2: Large companies with subsidiaries</h3>

As the Chief Information Security Officer (CISO), you need to ensure that each subsidiary’s security is managed appropriately while maintaining centralized oversight from headquarters. <ul class="wp-block-list">
<li>Grant local teams in each subsidiary control over their security environment only while maintaining centralized visibility and governance from headquarters.</li>



<li>Customize security strategies based on the regional or operational specifics of each subsidiary while ensuring global consistency in incident management.</li>
</ul>



<p>This approach simplifies incident response management and aligns security practices with the specific needs of each entity.</p>

<h2 class="wp-block-heading" id="h-data-segregation-by-intakes-fine-tuned-data-access-control">Data segregation by Intakes: Fine-tuned data access control</h2>



<h3 class="wp-block-heading" id="h-use-case-3-managing-access-for-it-and-security-teams">Use Case 3: Managing access for IT and security teams</h3>



<p>Not all IT teams need access to the same data. For example, a <strong>network infrastructure team</strong> may only require access to network logs, whereas a <strong>security team</strong> needs broader access across all logs to monitor for threats. Sekoia Defend’s <strong>data segregation by Intakes</strong> ensures:</p>



<ul class="wp-block-list">
<li>Specific teams can only access data relevant to their responsibilities.</li>



<li>Security teams can monitor the entire environment without compromising data segregation.</li>



<li>Each team operates within its scope, reducing unauthorized access risks.</li>
</ul>



<p>This feature enables organizations to efficiently manage responsibilities and ensure data confidentiality.</p>

<h2 class="wp-block-heading" id="h-maximizing-efficiency-by-combining-multi-tenant-and-data-segregation">Maximizing Efficiency by Combining Multi-Tenant and Data Segregation</h2>



<h3 class="wp-block-heading" id="h-use-case-4-managing-global-security-in-international-groups">Use Case 4: Managing global security in international groups</h3>



<p>By combining the multi-tenant and data segregation features, you can optimize security management in complex environments. Take the example of an international group with multiple subsidiaries, each having specialized IT teams (network, security, systems administration).</p>



<p>With this combination:</p>



<ul class="wp-block-list">
<li><strong>The parent tenant</strong> (headquarters) oversees global security with comprehensive visibility across all subsidiaries.</li>



<li><strong>Each child tenant</strong> (subsidiary) has its own local IT teams, with restricted access to specific data based on their responsibilities. For example, the network infrastructure team at a subsidiary can access only network logs, while the security team has access to all logs.</li>
</ul>



<p>This hybrid approach not only segments environments between different entities but also provides fine-grained access control within each subsidiary. Teams can work autonomously within their respective areas, while benefiting from the central protection and supervision of the parent tenant.</p>

<ul class="wp-block-list">
<li><strong>Granular data access control</strong> to protect sensitive information.</li>



<li><strong>Centralized management</strong> for better oversight of distributed teams or clients.</li>



<li><strong>Operational flexibility</strong> to customize security workflows to specific needs.</li>
</ul>



<p>By leveraging these features, organizations can optimize their SOC operations, enhance data security, and streamline access management, ensuring a robust and adaptable cybersecurity infrastructure.</p>
