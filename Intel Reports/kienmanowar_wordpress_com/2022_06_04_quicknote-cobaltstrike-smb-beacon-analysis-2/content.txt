<h4 class="wp-block-heading">4. Analyze Lotes.dll</h4>



<p>Load the Dll file into IDA for analysis, the code in the <strong><code>ReflectiveLoader </code></strong>function is similar to the code <a rel="noreferrer noopener" href="https://github.com/stephenfewer/grinder/blob/master/node/source/logger/ReflectiveLoader.c" target="_blank">here</a>, but it has been modified a bit related to processing import table . It first reads the <strong><code>NumberOfSymbols </code></strong>value from the <strong><code>File Header</code></strong> and stores it in a variable. This variable will be used as the<strong><code> xor_key</code></strong>. Then, when processing the import table , it uses the obtained <strong><code>xor_key</code></strong> value to decode the names of the dlls, as well as the names of the API functions that the malicious code will use:</p>


<div class="wp-block-image">
<figure class="aligncenter size-full"><a href="https://kienmanowar.wordpress.com/wp-content/uploads/2022/06/img_10.png"><img data-attachment-id="6294" data-permalink="https://kienmanowar.wordpress.com/2022/06/04/quicknote-cobaltstrike-smb-beacon-analysis/img_10/" data-orig-file="https://kienmanowar.wordpress.com/wp-content/uploads/2022/06/img_10.png" data-orig-size="1150,842" data-comments-opened="1" data-image-meta="{&quot;aperture&quot;:&quot;0&quot;,&quot;credit&quot;:&quot;&quot;,&quot;camera&quot;:&quot;&quot;,&quot;caption&quot;:&quot;&quot;,&quot;created_timestamp&quot;:&quot;0&quot;,&quot;copyright&quot;:&quot;&quot;,&quot;focal_length&quot;:&quot;0&quot;,&quot;iso&quot;:&quot;0&quot;,&quot;shutter_speed&quot;:&quot;0&quot;,&quot;title&quot;:&quot;&quot;,&quot;orientation&quot;:&quot;0&quot;}" data-image-title="img_10" data-image-description="" data-image-caption="" data-medium-file="https://kienmanowar.wordpress.com/wp-content/uploads/2022/06/img_10.png?w=300" data-large-file="https://kienmanowar.wordpress.com/wp-content/uploads/2022/06/img_10.png?w=1150" loading="lazy" width="1150" height="842" src="https://kienmanowar.wordpress.com/wp-content/uploads/2022/06/img_10.png" alt="" class="wp-image-6294" srcset="https://kienmanowar.wordpress.com/wp-content/uploads/2022/06/img_10.png 1150w, https://kienmanowar.wordpress.com/wp-content/uploads/2022/06/img_10.png?w=128&amp;h=94 128w, https://kienmanowar.wordpress.com/wp-content/uploads/2022/06/img_10.png?w=300&amp;h=220 300w, https://kienmanowar.wordpress.com/wp-content/uploads/2022/06/img_10.png?w=768&amp;h=562 768w, https://kienmanowar.wordpress.com/wp-content/uploads/2022/06/img_10.png?w=1024&amp;h=750 1024w" sizes="(max-width: 1150px) 100vw, 1150px" /></a></figure></div>

<p>After completing the Loader process, it will call the entry point of the Dll file to execute:</p>


<div class="wp-block-image">

<p class="has-text-align-justify">The code at <strong><code>DllEntryPoint </code></strong>will call <strong><code>DllMain</code></strong>, and then calls the function <strong><code>f_decrypt_and_parse_beacon_config</code></strong>. The reason I know this is a CobaltStrike Beacon is because the <strong><code>f_decrypt_and_parse_beacon_config </code></strong>function will perform decode the config with a hard-coded value of <strong><code>0x2e </code></strong>(as xor_key). The value <strong><code>0x2e </code></strong>is used in Beacon version 4.</p>

The result shows that this is an SMB Beacon:</p>
