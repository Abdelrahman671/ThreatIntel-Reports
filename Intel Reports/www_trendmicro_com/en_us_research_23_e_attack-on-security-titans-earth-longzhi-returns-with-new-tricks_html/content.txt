	<h1 class="article-details__title">Attack on Security Titans: Earth Longzhi Returns With New Tricks</h1>
	<p class="article-details__description">After months of dormancy, Earth Longzhi, a subgroup of advanced persistent threat (APT) group APT41, has reemerged using new techniques in its infection routine. This blog entry forewarns readers of Earth Longzhi’s resilience as a noteworthy threat. </p>
	<p class="article-details__author-by">By: Ted Lee, Hara Hiroaki
		
			<time class="article-details__date">May 02, 2023</time>
		
		
		<span>Read time:&nbsp;</span><span class="eta"></span> (<span class="words"></span> words)

            <div class="richText">
	
    


	
		<div>
			<p>We discovered a new campaign by Earth Longzhi (a <a href="/en_us/research/22/k/hack-the-real-box-apt41-new-subgroup-earth-longzhi.html" target="_blank">subgroup of APT41</a>) that targets organizations based in Taiwan, Thailand, the Philippines, and Fiji. This recent campaign, which follows months of dormancy, abuses a Windows Defender executable to perform DLL sideloading while also exploiting a vulnerable driver, <i>zamguard64.sys</i>, to disable security products installed on the hosts via a bring-your-own-vulnerable-driver (BYOVD)<i> </i>attack. We also found that Earth Longzhi uses a new way to disable security products, a technique we’ve dubbed “stack rumbling” via Image File Execution Options (IFEO), which is a new denial-of-service (DoS) technique.</p>
<p>In addition, we’ve noticed that this campaign installs drivers as kernel-level services by using Microsoft <a href="https://learn.microsoft.com/en-us/windows/win32/rpc/rpc-start-page" target="_blank">Remote Procedure Call</a> (RPC) instead of using general Windows application programming interfaces (APIs). This is a stealthy way to evade typical API monitoring. We also found some interesting samples in our investigation that contained information not only on Earth Longzhi’s potential targets, but also techniques for possible use in future campaigns. This blog entry seeks to forewarn readers that Earth Longzhi remains active and continues to improve its tactics, techniques, and procedures (TTPs).</p>
<h2><span class="body-subhead-title">Attack vectors</span></h2>
<p>Earth Longzhi’s new campaign samples showed a tendency to exploit public-facing applications, Internet Information Services (IIS) servers, and Microsoft Exchange servers to install <a href="https://github.com/rebeyond/Behinder">Behinde</a><a href="http://r/">r</a>, a well-known web shell, rather than send pieces of document-based malware through email. As seen in this campaign, Behinder proved to be a powerful web shell variant that can support multiple backdoor functions, including file operation, remote command execution (RCE), interactive shell, and Socks5 proxy.</p>
<p> Malicious actors use this web shell to discover intranet information and deploy other pieces of malware and hacking tools on a compromised machine.</p>

		</div>
	

</div>
            <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/23/e/attack-on-security-titans-earth-longzhi-returns-with-new-tricks/earthlongzhi_01.jpg" alt="Figure 1. Infection routine used by Earth Longzhi"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 1. Infection routine used by Earth Longzhi</figcaption>
			
		</div>
	</figure>

</div>
            <div>




    
    
    <div class="richText">
	
    


	
		<div>
			<h2><span class="body-subhead-title">New tricks for DLL sideloading</span></h2>
<p>In the group’s new campaign, the malware was launched through legitimate Windows Defender binaries, <i>MpDlpCmd.exe</i> and <i>MpCmdRun.exe</i>, instead of using document-based samples. The malware was disguised as a legitimate DLL, <i>MpClient.dll</i> and was loaded by Microsoft Defender’s binaries. Our investigation showed two different types of malware that were launched through this technique: One is a new variant of <a href="https://www.trendmicro.com/vinfo/tmr/?/us/threat-encyclopedia/malware/Trojan.Win64.CROXLOADER.ZTKC/" target="_blank">Croxloader</a>, and the other is a tool that can disable security products, which we dubbed “SPHijacker.”</p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/23/e/attack-on-security-titans-earth-longzhi-returns-with-new-tricks/earthlongzhi_02.jpg" alt="Figure 2. Legitimate files used for DLL sideloading"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 2. Legitimate files used for DLL sideloading</figcaption>
			
		</div>
	</figure>

</div>


    
    
    <div class="richText">
	
    


	
		<div>
			<h3><span class="body-subhead-title">New Croxloader variant</span></h3>
<p>Earth Longzhi’s new campaign launched Windows Defender binaries as a system service. The new Croxloader variant, disguised as <i>MpClient.dll,</i> was subsequently loaded. Once launched, Croxloader reads the payload named <i>MpClient.bin</i> and decrypts its content. The new variant is almost identical to the older ones, except that it uses a different decryption algorithm. The algorithm used in the original variant is <i>(SUB 0xA) XOR 0xCC,</i> while the algorithm for the new variant is <i>(ADD 0x70) XOR 0xDD. </i>The final payload is identified as a Cobalt Strike beacon, which we detected as Backdoor.Win64.COBEACON.ZYKB.</p>

		</div>
	

</div>


    
    
    <div class="image">

    <div class="richText">
	
    


	
		<div>
			<h3><span class="body-subhead-title">SPHijacker</span></h3>
<p>SPHijacker, a new tool designed to disable security products, adopts two approaches to achieve this purpose. One approach terminates the security product process by using a vulnerable driver, <i>zamguard64.sys,</i> published by Zemana (vulnerability designated as <a href="https://nvd.nist.gov/vuln/detail/CVE-2018-5713" target="_blank">CVE-2018-5713</a>). Meanwhile, another approach disables process launching by using a new technique that we named stack rumbling, which we will discuss in detail in succeeding paragraphs. Notably, this is the first time we’ve seen such a technique being used in the wild.  </p>
<h4><span class="body-subhead-title">Technical analysis</span></h4>
<p>Based on our analysis, the <i>mmmm.sys</i> file (originally named <i>Zamguard64.sys)</i> is decrypted and dropped, after which it is registered as a service. It then creates and starts the service through RPC as opposed to calling general Windows APIs to set up the service, as shown in Figure 6. We reckon that such a technique enables malicious actors to evade API call monitoring.</p>

		</div>
	

</div>


    
    
    <div class="image">
	
    


	<figure class="image-figure">
		
			<img src="/content/dam/trendmicro/global/en/research/23/e/attack-on-security-titans-earth-longzhi-returns-with-new-tricks/earthlongzhi_06.png" alt="Figure 6. Code showing service started via RPC"/>
		
		<div class="caption-image-container ">
			<figcaption>Figure 6. Code showing service started via RPC</figcaption>
			
		</div>
	</figure>

</div>

<p>Once the process termination is completed, SPHijacker disables process execution by forcefully causing the targeted applications to crash upon launching, a technique we referred to earlier as stack rumbling. This technique is a type of DoS attack that abuses undocumented <i>MinimumStackCommitInBytes</i> values in the IFEO registry key via the following steps:</p>
<ol>
<li>Modifying the registry <i>HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Image File Execution Options\{target process name}.</i></li>
<li>Creating a new value, <i>MinimumStackCommitInBytes,</i> with <i>0x88888888</i> as its data. Any value deemed large enough is acceptable.</li>
<li>Waiting for the next process launch to take place. It’s important to note that this depends on whether the targeted process is antivirus-related. There is usually a need to wait for the operating system to reboot.</li>
<li>Once the targeted process is launched, it will soon crash due to stack overflow.</li>
</ol>

<p>Another notable component that we found in our threat hunting is <i>dwm.exe</i>, a new tool used for privilege escalation. It is implemented based on an open-source proof of concept on <a href="https://github.com/zcgonvh/TaskSchedulerMisc/blob/master/schuac.cs">GitHub</a>. First, <i>dwm.exe</i> replaces the image path name and the command-line information with <i>C:\Windows\explorer.exe</i> for defense evasion. Then, the Component Object Model (COM) object, <i>IElevatedFactoryServer,</i> is used to bypass the Windows User Account Control (UAC) mechanism and register the given payload as a scheduled task with the highest privilege. This approach enables the specified binary to be launched with system privileges. This is the first time that we’ve seen Earth Longzhi actors use this relatively new technique in its operations. </p>

			<h2><span class="body-subhead-title">Conclusion</span></h2>
<p>In the fourth quarter of 2022, we discovered a new <a href="/en_us/research/22/k/hack-the-real-box-apt41-new-subgroup-earth-longzhi.html">subgroup of APT41</a> that we tracked as Earth Longzhi. In the process, we revealed two different campaigns that took place from 2020 to 2022. This follow-up article to our previous <a href="/en_us/research/22/k/hack-the-real-box-apt41-new-subgroup-earth-longzhi.html" target="_blank">report</a> aims to flag readers that Earth Longzhi remains in circulation and is expected to improve its TTPs. Here, we revealed that the campaign deployed a fake <i>mpclient.dll, </i>launched through signed Windows Defender binaries, to decrease its risk of exposure. To evade and disable security products, Earth Longzhi adopted the following approaches:</p>
<ol>
<li>It used Microsoft Windows RPC to create a system service instead of standard Windows APIs.</li>
<li>It terminated running security products via a vulnerable driver, <i>zamguard64.sys,</i> which is essentially a BYOVD attack.</li>
<li>It modified IFEO registries to restrict the execution of security products.</li>
</ol>
<p>We also shared some interesting threat-hunting findings. Although the samples that we’ve collected resemble testing files, they can still be useful because they contain information on Earth Longzhi’s potential targets and new techniques that it might employ in the future. From the embedded documents that we’ve collected, we can infer that Vietnam and Indonesia are the countries that they will likely aim at next. Notably, the group’s possible abuse of Task Scheduler to escalate privileges for persistence is a new technique that it might use in future campaigns.</p>
<p>Another noteworthy insight is that the threat actors showed an inclination for using open-source projects to implement their own tools. There is evidence to suggest that the group spruces up its toolset during periods of inactivity. With this knowledge in mind, organizations should stay vigilant against the continuous development of new stealthy schemes by cybercriminals.</p>
