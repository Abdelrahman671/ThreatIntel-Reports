Infections</a></li></ul></div></div><h1 class="entry-title">Windigo Linux Analysis &#8211; Ebury and Cdorked</h1><div class="post-header"><div class="post-header-container"><div class="author-avatar"> <a href="https://blog.sucuri.net/author/dcid" rel="author"> <img alt='' src='https://secure.gravatar.com/avatar/df3ec5506ba59d2ed3b951b7057e97d0?s=60&#038;d=mm&#038;r=g' srcset='https://secure.gravatar.com/avatar/df3ec5506ba59d2ed3b951b7057e97d0?s=120&#038;d=mm&#038;r=g 2x' class='avatar avatar-60 photo' height='60' width='60' decoding='async'/> </a></div><div class="author-details"><div class="author-wrap"><div class="author"> <span class="fn"> <a href="https://blog.sucuri.net/author/dcid" rel="author"> Daniel Cid </a> </span></div></div><ul class="post-meta"><li>March 18, 2014</li></ul></div></div></div></header><div class="entry-container"><div class="entry-content-wrap"><div class="entry-content"><p>Our friends over at ESET released a very detailed document about the Windigo Operation. The Windigo Operation has been responsible for the compromise of thousands of Linux servers over the years. When you hear terms like <strong>Ebury, CDorked, Calfbot</strong> and others, they are all related to each other.</p><p>Over the last few years, our team has been handling and fixing compromised servers and we can attest to how complex the clean up for this infection can be. We’ve seen that the servers we’ve fixed have been misused for distribution of malware, SPAM and, in some cases, to steal credit cards on compromised web servers used for e-commerce.</p><h5>Windigo Timeline</h5><p>The timeline released by ESET matches what we have been seeing and it goes back to 2011 when Linux/eBury was first seen. It goes through many evolutions, including our joint analysis of <a href="https://blog.sucuri.net/2013/04/apache-binary-backdoors-on-cpanel-based-servers.html">CDORKED</a> on 2013 and the <a href="https://blog.sucuri.net/2013/01/server-side-iframe-injections-via-apache-modules-and-sshd-backdoor.html">SSH backdoors</a>:</p><p><a href="https://blog.sucuri.net/wp-content/uploads/2014/03/Screen-Shot-2014-03-18-at-6.07.01-PM.jpg" rel="lightbox"><img fetchpriority="high" decoding="async" src="https://blog.sucuri.net/wp-content/uploads/2014/03/Screen-Shot-2014-03-18-at-6.07.01-PM-600x274.png" alt="Windigo Timeline" width="600" height="274" class="aligncenter size-medium wp-image-9040" /></a></p><h5>Windigo Indicators of Compromise (IOC)</h5><p>If you run a Linux server and you are worried it might be infected, they provide a few techniques (indicators of compromise) to check if the server is hacked.</p><ul><li><b>For Linux/Ebury.</b> Run the <strong>ssh -g</strong> command. If it returns an error about missing argument, you know you are compromised.</li><li><b>For Linux/CDorked.</b> Run <strong>curl</strong> to <strong>favicon.iso</strong> and see if you get redirected to Google.com. If you do, you know you are compromised.</li></ul><p>These apply to the latest versions of the malware. Old versions have different indicators and we explain them on our previous blog posts. Note that with the release of this document, the malware authors will likely change operations and the behavior of the code.
