<a href="https://blogger.googleusercontent.com/img/b/R29vZ2xl/AVvXsEi16ORoh0JlLuEE54g3XdABewivfk2ARW-ZGV4fFTTYET_57gO3LYRJJAcb5zlX1LN1B3KX_oCMwTytlworkapNKspRUVHRCnxGMy47tgtsu7BRWWLQwLaH-gteXGSlJdD8RRuZhkbg6tAj2vHkwNbxg-jICgXadNlYRjtiZFrVvagxensmtEmuMPuF4ole/s867/lemonduckk.png" style="margin-left: 1em; margin-right: 1em; text-align: center;"><img border="0" data-original-height="867" data-original-width="800" src="https://blogger.googleusercontent.com/img/b/R29vZ2xl/AVvXsEi16ORoh0JlLuEE54g3XdABewivfk2ARW-ZGV4fFTTYET_57gO3LYRJJAcb5zlX1LN1B3KX_oCMwTytlworkapNKspRUVHRCnxGMy47tgtsu7BRWWLQwLaH-gteXGSlJdD8RRuZhkbg6tAj2vHkwNbxg-jICgXadNlYRjtiZFrVvagxensmtEmuMPuF4ole/s16000/lemonduckk.png" /></a><br /><p style="text-align: justify;">This post was authored by Aufa and NetbyteSEC Interns (Irham, Idham, Adnin, Nabiha, Haiqal, Amirul)</p><p style="text-align: justify;">This blog post is intended to give an overall picture of LemonDuck malware exploited SMB by leveraging&nbsp;EternalBlue vulnerability (CVE-2017-0144) for cryptocurrency mining, based on observations from our observatory farm. EternalBlue is a significant vulnerability in Microsoft's Server Message Block (SMB) protocol that has been exploited by numerous malware variants. One such strain, LemonDuck, is a well-known cryptomining malware that takes advantage of weaknesses in SMB services to breach networks, disable security measures, and utilize system resources for cryptomining purposes. It spreads through phishing emails, exploiting SMB vulnerabilities, and using brute-force password attacks. Additionally, LemonDuck employs PowerShell to avoid detection, deploy various malicious payloads, and target systems for cryptojacking.</p><p style="text-align: justify;">This blog post might be useful for security professionals with insights into current malware trends and attack techniques observed in the wild. By the end of this blog post, readers will gain a comprehensive understanding of the tactics and techniques used by this LemonDuck malware operator that compromised users via the SMB domain admin. Furthermore, security analyst can assess the provided Indicator of Compromise (IOCs) associated with LemonDuck malware variant to assess whether their environment has been compromised or not.</p><h2 style="text-align: justify;"><span data-darkreader-inline-color="" style="--darkreader-inline-color: #4ef6f6; color: #01ffff;">Case Summary</span></h2><p style="text-align: justify;"><span style="text-align: left;">The attacker initiated a brute force attack in the vulnerable SMB service leveraging Eternal Blue (</span>CVE-2017-0144) vulnerability<span style="text-align: left;">&nbsp;</span><span style="text-align: left;">which was running on our observatory farm</span><span style="text-align: left;">, successfully compromising the user "Administrator". Upon gaining initial access, the attacker creates a hidden administrative share</span><span style="font-family: &quot;Times New Roman&quot;, serif; font-size: 12pt; text-align: left;">&nbsp;for the C: drive&nbsp;</span><span style="text-align: left;">enabling&nbsp;</span><span style="text-align: left;">them to remotely access the entire drive without the victim's knowledge. The attacker created a&nbsp;</span><span style="text-align: left;">batch file named</span><span style="text-align: left;">&nbsp;</span><i style="text-align: left;">p.bat,&nbsp;</i><span style="text-align: left;">designed to execute a series of malicious actions aimed at gaining persistent control over a system.&nbsp;</span><span style="text-align: left;">It first creates and&nbsp;</span><span style="text-align: left;">copies malicious file <i>msInstall.exe</i>, then rename it into <i>FdQn.exe,&nbsp;</i></span><span style="text-align: left;">opens specific firewall ports and sets up port forwarding to redirect traffic to a remote address (1.1.1.1) on port 53.&nbsp;</span><span style="text-align: left;">The script checks for PowerShell and, if found, runs a base64-encoded PowerShell script to download and execute script from remote URL and schedule the execution of <i>FdQn.exe&nbsp;</i></span><span style="text-align: left;">regularly. If PowerShell isn't available, it creates scheduled tasks to run malicious payloads from a remote URL using <i>mshta</i>,&nbsp;</span><span style="text-align: left;">and executes the malware <i>installed.exe</i>.&nbsp;</span><span style="text-align: left;">It also includes anti-detection mechanisms by monitoring the number of open command prompts and forcing a system reboot if more than 10 are detected, likely to disrupt analysis or debugging efforts.&nbsp;</span></p><p style="text-align: justify;"><span style="text-align: left;">Finally, it cleans up by deleting the batch file itself after execution.&nbsp;</span><span style="text-align: left;">Additionally,&nbsp;</span><span style="text-align: left;">the attacker also create another malicious executable that disguised a legitimate system service named <i>svchost.exe.</i></span><span style="text-align: left;">&nbsp;This malware was able to&nbsp;</span><span style="text-align: left;">disable Windows Defender&#8217;s real-time protection and create exclusions&nbsp;</span><span style="text-align: left;">for the entire C: drive and the PowerShell process to avoid detection. Similarly with the previous, the script manipulates the firewall and sets up port forwarding to redirect traffic to a remote address (1.1.1.1) on port 53, commonly used for DNS.&nbsp;</span><span style="text-align: left;">It copies and renames malicious executables (<i>svchost.exe, dig.exe</i>) into (<i>HbxbVCnn.exe, SgcwqM.exe</i>)</span><span style="text-align: left;">&nbsp;</span><span style="text-align: left;">and sets up scheduled tasks to run these files at regular intervals, ensuring persistence. The script also interacts with other scheduled tasks, including deleting and recreating them for continued execution of the malware. This malware attempts to bypass security measures, gain network control, and maintain persistence on the system while executing hidden tasks.&nbsp;</span><span style="text-align: left;">By the end of their operation, the attacker delete the administrative share in order to hide their tracks or eliminate the administrative shares where they can maintain control of the compromised system exclusively for themselves.</span></p><p style="text-align: justify;"></p><div class="separator" style="clear: both; text-align: center;"><a href="https://blogger.googleusercontent.com/img/b/R29vZ2xl/AVvXsEjFoEsH2Q-85sPBt8Wrb5nhxPrzTPpzaI1eAJRqy_JxkK1vqEaRz4wfs_N-EvGbirv2zLUsjdyOY-OU7oaNXsSCiv5vx2zg9P9zY_hNPASlRn8Mzg69aNwrJ-CqgAd9-TZHEUS3Iwt2ZrEkQ4WlJBTt63yVVh33vpzgB1WsWc_eh49BDYMUFTvHfm5JJnNH/s1501/SMB%20exploitation.drawio%20(1).png" style="margin-left: 1em; margin-right: 1em;"><img border="0" data-original-height="941" data-original-width="1501" src="https://blogger.googleusercontent.com/img/b/R29vZ2xl/AVvXsEjFoEsH2Q-85sPBt8Wrb5nhxPrzTPpzaI1eAJRqy_JxkK1vqEaRz4wfs_N-EvGbirv2zLUsjdyOY-OU7oaNXsSCiv5vx2zg9P9zY_hNPASlRn8Mzg69aNwrJ-CqgAd9-TZHEUS3Iwt2ZrEkQ4WlJBTt63yVVh33vpzgB1WsWc_eh49BDYMUFTvHfm5JJnNH/s16000/SMB%20exploitation.drawio%20(1).png" /></a><span style="text-align: left;"><i><span style="color: #01ffff;">Figure 1: Graph flow of the attack</span></i></span></div><p></p><p></p><h2 style="text-align: justify;"><span style="color: #01ffff;"><br /></span></h2><h2 style="text-align: justify;"><span style="color: #01ffff;">Technical Analysis</span></h2><div><h2 style="text-align: justify;"><span style="color: #01ffff;">Brute-force SMB for Initial Access</span></h2></div><p style="text-align: left;">In the initial access phase, the attacker with IP address 211.22.131.99 initiated brute-forcing activities targeting the SMB machine and successfully logged in as a local user named Administrator.
