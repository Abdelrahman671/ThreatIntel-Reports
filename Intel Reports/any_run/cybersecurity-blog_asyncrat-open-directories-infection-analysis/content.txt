            <div class="custom-breadcrumbs custom-breadcrumbs_main"><a class="custom-breadcrumbs__item" href="/cybersecurity-blog/">Home</a><a class="custom-breadcrumbs__item" href="/cybersecurity-blog/category/malware-analysis/">Malware Analysis</a><div class="custom-breadcrumbs__item" >AsyncRAT&#8217;s Infection Tactics <br>via Open Directories: Technical AnalysisÂ </div></div>
      
<p><em>Editorâs note: The current article is authored by RacWatchin8872, who is a threat intelligence analyst. You can find him on </em><a href="https://x.com/RacWatchin8872" target="_blank" rel="noreferrer noopener"><em>X</em></a><em>.</em>Â </p>



<p>This article covers two distinct methods used to infect systems with AsyncRAT via open directories. These techniques show how attackers are constantly adapting, finding new ways to use publicly accessible files to broaden AsyncRAT&#8217;s impact and reach.&nbsp;</p>



<h2 class="wp-block-heading">Overview&nbsp;</h2>



<p><a href="https://any.run/malware-trends/asyncrat" target="_blank" rel="noreferrer noopener">AsyncRAT</a> is a type of <a href="https://any.run/malware-trends/rat" target="_blank" rel="noreferrer noopener">Remote Access Trojan</a> (RAT) malware designed to stealthily infiltrate systems and give attackers remote control over infected devices. It is commonly used for spying, data theft, and manipulation of compromised systems.&nbsp;&nbsp;</p>



<p>Recently, two open directories surfaced, each employing unique methods to distribute and infect victims with AsyncRAT. These techniques highlight the persistent threat posed by this malware and its diverse infection strategies.&nbsp;</p>



<h2 class="wp-block-heading">First Technique&nbsp;</h2>



<h3 class="wp-block-heading">Open Directory&nbsp;</h3>



<p>While investigating malicious open directories exposed to the internet, I discovered one with an unusual structure.&nbsp;&nbsp;</p>



<p>The directory contained the following files:&nbsp;</p>



<ul class="wp-block-list">
<li>A text file with an extensive string that turned out to be a <a href="https://any.run/cybersecurity-blog/macros-in-malware/" target="_blank" rel="noreferrer noopener">VBS script</a>&nbsp;</li>
</ul>



<ul class="wp-block-list">
<li>A JPG file that was actually a disguised ZIP archive&nbsp;</li>
</ul>


<div class="wp-block-image">
<figure class="aligncenter size-full"><img fetchpriority="high" decoding="async" width="1010" height="811" src="/cybersecurity-blog/wp-content/uploads/2024/11/image1.png" alt="" class="wp-image-9697" srcset="/cybersecurity-blog/wp-content/uploads/2024/11/image1.png 1010w, /cybersecurity-blog/wp-content/uploads/2024/11/image1-300x241.png 300w, /cybersecurity-blog/wp-content/uploads/2024/11/image1-768x617.png 768w, /cybersecurity-blog/wp-content/uploads/2024/11/image1-370x297.png 370w, /cybersecurity-blog/wp-content/uploads/2024/11/image1-270x217.png 270w, /cybersecurity-blog/wp-content/uploads/2024/11/image1-740x594.png 740w" sizes="(max-width: 1010px) 100vw, 1010px" /><figcaption class="wp-element-caption"><em>Figure <em>1</em>: Open directory structure&nbsp;</em></figcaption></figure></div>


<h3 class="wp-block-heading">Analysis of the Txt file&nbsp;</h3>



<p>The text fileâs extensive string conceals an obfuscated VBS script. It uses random variables to store parts of the text that will be used to download the JPG file.</p>


<div class="wp-block-image">
<figure class="aligncenter size-large"><img decoding="async" width="1024" height="417" src="/cybersecurity-blog/wp-content/uploads/2024/11/image2-2-1024x417.png" alt="" class="wp-image-9698" srcset="/cybersecurity-blog/wp-content/uploads/2024/11/image2-2-1024x417.png 1024w, /cybersecurity-blog/wp-content/uploads/2024/11/image2-2-300x122.png 300w, /cybersecurity-blog/wp-content/uploads/2024/11/image2-2-768x313.png 768w, /cybersecurity-blog/wp-content/uploads/2024/11/image2-2-370x151.png 370w, /cybersecurity-blog/wp-content/uploads/2024/11/image2-2-270x110.png 270w, /cybersecurity-blog/wp-content/uploads/2024/11/image2-2-740x301.png 740w, /cybersecurity-blog/wp-content/uploads/2024/11/image2-2.png 1377w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption class="wp-element-caption"><em>Figure <em>2</em>: Obfuscated VBS code&nbsp;</em></figcaption></figure></div>


<p>To make it easier to read we just need to make a few changes:&nbsp;</p>



<ol class="wp-block-list" start="1">
<li>Replace the variables with the actual text</li>
</ol>



<ol class="wp-block-list" start="2">
<li>Use intuitive names for variables that are used to write or download files</li>
</ol>


<div class="wp-block-image">
<figure class="aligncenter size-large"><img decoding="async" width="1024" height="498" src="/cybersecurity-blog/wp-content/uploads/2024/11/image3-3-1024x498.png" alt="" class="wp-image-9699" srcset="/cybersecurity-blog/wp-content/uploads/2024/11/image3-3-1024x498.png 1024w, /cybersecurity-blog/wp-content/uploads/2024/11/image3-3-300x146.png 300w, /cybersecurity-blog/wp-content/uploads/2024/11/image3-3-768x374.png 768w, /cybersecurity-blog/wp-content/uploads/2024/11/image3-3-370x180.png 370w, /cybersecurity-blog/wp-content/uploads/2024/11/image3-3-270x131.png 270w, /cybersecurity-blog/wp-content/uploads/2024/11/image3-3-740x360.png 740w, /cybersecurity-blog/wp-content/uploads/2024/11/image3-3.png 1377w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption class="wp-element-caption"><em>Figure <em>3</em>: Deobfuscated VBS code&nbsp;</em></figcaption></figure></div>


<p>Now we see that the VBS script creates an XML file <strong>OMjRRRRRRRRRRRRRRRRRRRRvbK.xml </strong>located at <strong>C:\Users\Public</strong>. The content of the XML file contains a PowerShell script that downloads the disguised JPG file, saves it, and extracts it to the same directory.&nbsp;</p>



<p>Once extracted, the process continues by executing another script, <strong>TesKKKeLAvaYdAfbBS.vbs. </strong>Then, it cleans up by deleting both the XML and ZIP files.&nbsp;</p>



<h3 class="wp-block-heading">Analysis of the VBS file&nbsp;</h3>



<p>The VBS script is also obfuscated and uses the same technique as the other text file. By examining the file, we can understand a few parts of its execution:</p>


<div class="wp-block-image">

<h3 class="wp-block-heading">Analysis of the Powershell file&nbsp;</h3>



<p>The PowerShell file performs 2 main functions:&nbsp;</p>



<ol class="wp-block-list" start="1">
<li><strong>File creation and content writing</strong>: Creates three files essential to the infection process</li>
</ol>



<ol class="wp-block-list" start="2">
<li><strong>Scheduled task setup</strong>: Schedules a task to ensure repeated execution, thereby maintaining the AsyncRAT infection</li>
</ol>



<h3 class="wp-block-heading">File Creation&nbsp;</h3>



<p>The Powershell creates 3 files.</p>



<p><strong>First file</strong></p>



<p>This obfuscated file stores and executes the values of EXE and DLL files related to AsyncRAT directly in memory.&nbsp;</p>
