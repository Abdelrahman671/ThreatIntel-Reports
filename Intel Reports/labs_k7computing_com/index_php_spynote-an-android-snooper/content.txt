<p>
<p></p>



<p>Threat actors are constantly using new tricks and tactics to target users across the globe. This blog is about SpyNote,&nbsp; an Android RAT targeting Indian Defense personnel. The initial attack vector information was found on the <a href="https://www.newindianexpress.com/cities/delhi/2022/jul/17/security-at-risk-whatsapp-malware-targeting-army-personnel-detected-2477529.html">newindianexpress</a> website.</p>



<p>Let’s now get into the details of how this SpyNote works.</p>



<p>This RAT is propagated via WhatsApp&nbsp; with the name &#8220;<strong>CSO_SO on Deputation DRDO. apk</strong>&#8220;.&nbsp;&nbsp;</p>



<p>Once the user falls prey to this RAT and&nbsp; installs this malicious<strong> &#8220;CSO_SO on Deputation DRDO. apk&#8221;,</strong> this app pretends to be the genuine Adobe reader icon in the device app drawer as shown in Figure 1.</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" width="512" height="1024" src="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure1-1-512x1024.png" alt="" class="wp-image-24527" srcset="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure1-1-512x1024.png 512w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure1-1-150x300.png 150w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure1-1.png 720w" sizes="(max-width: 512px) 100vw, 512px" /><figcaption>Figure 1: Fake Adobe Reader icon of&nbsp; the malware</figcaption></figure></div>



<p></p>



<p>Upon launching , this application opens a Google Drive URL that is hardcoded in the app&#8217;s “strings.xml” file&nbsp; and displays the images as shown in Figure 2. Google Drive URL hardcoded in the app&#8217;s “strings.xml” file as shown in Figure 3.</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" width="576" height="1024" src="https://labs.k7computing.com/wp-content/uploads/2022/08/unnamed-576x1024.png" alt="" class="wp-image-24554" srcset="https://labs.k7computing.com/wp-content/uploads/2022/08/unnamed-576x1024.png 576w, https://labs.k7computing.com/wp-content/uploads/2022/08/unnamed-169x300.png 169w, https://labs.k7computing.com/wp-content/uploads/2022/08/unnamed-768x1365.png 768w, https://labs.k7computing.com/wp-content/uploads/2022/08/unnamed-864x1536.png 864w, https://labs.k7computing.com/wp-content/uploads/2022/08/unnamed.png 1080w" sizes="(max-width: 576px) 100vw, 576px" /><figcaption> Figure 2: Images from Google Drive </figcaption></figure></div>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="847" height="272" src="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure3-1.png" alt="" class="wp-image-24529" srcset="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure3-1.png 847w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure3-1-300x96.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure3-1-768x247.png 768w" sizes="(max-width: 847px) 100vw, 847px" /><figcaption>Figure 3: Hardcoded Google Drive URL string</figcaption></figure></div>



<p></p>



<h2><strong>Technical Analysis</strong></h2>



<p>Figure 4 shows that this malware refers to services in the AndroidManifest.xml file but not defined in the classes.dex in the APK’s root folder. This indicates that the services’ classes&nbsp; or another dex containing the classes would be loaded in memory at run-time using any one of the dynamic loading techniques.</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" width="1024" height="399" src="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure4-1-1024x399.png" alt="" class="wp-image-24530" srcset="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure4-1-1024x399.png 1024w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure4-1-300x117.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure4-1-768x299.png 768w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure4-1.png 1308w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption>Figure 4: Undefined Class Names in AndroidManifest.xml</figcaption></figure></div>



<p></p>



<p>SpyNote sample which we analyzed employs the technique of using the “base application context” to the class “com.android.protector.ProtectApplication” as shown in Figure 5.&nbsp;</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" width="1024" height="131" src="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure5-1-1024x131.png" alt="" class="wp-image-24531" srcset="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure5-1-1024x131.png 1024w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure5-1-300x38.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure5-1-768x98.png 768w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure5-1.png 1218w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption>Figure 5: AndroidManifest.XML showing base context to “com.android.protector.ProtectApplication”</figcaption></figure></div>



<p></p>



<p>Hence, when the application’s launcher activity is triggered, “attachbasecontext” function from the class “com.android.protector.ProtectApplication” is executed and the other classes.dex (carried within the APK) are loaded and functions in those classes.dex files are&nbsp; invoked using reflection and MultiDex support as shown in Figure 6 and 7.&nbsp;</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="1007" height="310" src="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure6-1.png" alt="" class="wp-image-24532" srcset="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure6-1.png 1007w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure6-1-300x92.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure6-1-768x236.png 768w" sizes="(max-width: 1007px) 100vw, 1007px" /><figcaption>Figure 6: “attachbasecontext” using reflection to load the secondary dex files in the APK</figcaption></figure></div>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="920" height="451" src="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure7-1.png" alt="" class="wp-image-24533" srcset="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure7-1.png 920w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure7-1-300x147.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure7-1-768x376.png 768w" sizes="(max-width: 920px) 100vw, 920px" /><figcaption>Figure 7: classLoader API loading secondary dex files using MultiDex support</figcaption></figure></div>



<p></p>



<p>Looking at the logcat at runtime,&nbsp; with MultiDex support, secondary dex files are loaded as&nbsp; base.apk.classes1.zip and converted as executable ‘base.apk.classes1.odex’ as shown in Figure 8.</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" width="1024" height="83" src="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure8-1-1024x83.png" alt="" class="wp-image-24536" srcset="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure8-1-1024x83.png 1024w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure8-1-300x24.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure8-1-768x62.png 768w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure8-1.png 1074w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption>Figure 8: The logcat image showing the base.apk.classes1.odex file at runtime</figcaption></figure></div>



<p></p>



<h2><strong>Analyzing the Payload</strong></h2>



<p>The payload file base.apk.classes1.zip as shown in Figure 9 has the references to services’ classes declared in the AndroidManifest.xml.&nbsp;</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="506" height="397" src="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure9-3.png" alt="" class="wp-image-24537" srcset="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure9-3.png 506w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure9-3-300x235.png 300w" sizes="(max-width: 506px) 100vw, 506px" /><figcaption>Figure 9: Defined Class Name from AndoridManifest.xml</figcaption></figure></div>



<p></p>



<p></p>



<p>This malware collects location information like altitude, latitude, longitude, precision and even the speed at which the device is moving as shown in Figure 10.</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" width="1024" height="307" src="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure10-1-1024x307.png" alt="" class="wp-image-24538" srcset="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure10-1-1024x307.png 1024w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure10-1-300x90.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure10-1-768x230.png 768w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure10-1.png 1071w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption>Figure 10: Collects the device location information</figcaption></figure></div>



<p></p>



<p>SpyNote then proceeds to combine all the aforementioned data and compresses (using <strong>gZIPOutputStream</strong> API) them before forwarding it to the C2 server as shown in Figure 11.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="587" height="424" src="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure11-1.png" alt="" class="wp-image-24539" srcset="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure11-1.png 587w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure11-1-300x217.png 300w" sizes="(max-width: 587px) 100vw, 587px" /><figcaption>Figure 11: DATA compression using gZIPOutputStream</figcaption></figure></div>



<p></p>



<h2><strong>C2 Communication</strong></h2>



<p>This RAT contacts the C2 server at IP 213.136.80[.]208, which is hardcoded in the “strings.xml” file (refer Figure 3). Figure 12 shows the connection established with the C2.&nbsp;</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="971" height="334" src="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure12-1.png" alt="" class="wp-image-24540" srcset="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure12-1.png 971w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure12-1-300x103.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure12-1-768x264.png 768w" sizes="(max-width: 971px) 100vw, 971px" /><figcaption>Figure 12: TCP connection with the C2 server</figcaption></figure></div>



<p></p>



<p>After the connection is established, the malware sends the gzip compressed data to the C2 as evident from the network packet’s header in Figure 13.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="455" height="161" src="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure13-1.png" alt="" class="wp-image-24541" srcset="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure13-1.png 455w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure13-1-300x106.png 300w" sizes="(max-width: 455px) 100vw, 455px" /><figcaption>Figure 13: gzip data sent by the device after establishing the connection with the C2 Server</figcaption></figure></div>



<p></p>



<p>The decompressed content of the data is shown below in Figure 14.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="764" height="318" src="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure14.png" alt="" class="wp-image-24542" srcset="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure14.png 764w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure14-300x125.png 300w" sizes="(max-width: 764px) 100vw, 764px" /><figcaption>Figure 14: Decompressed gzip data showing IP address</figcaption></figure></div>



<h2><strong>Decode packets from the C2</strong></h2>



<p>The C2 responds by sending a series of compressed data,&nbsp; which when decompressed, is revealed to be system commands and the related APK payload as shown in Figure 15. In our case, the APK was extracted using Cyberchef.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="1012" height="544" src="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure15.png" alt="" class="wp-image-24544" srcset="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure15.png 1012w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure15-300x161.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure15-768x413.png 768w" sizes="(max-width: 1012px) 100vw, 1012px" /><figcaption>Figure 15: Getting commands and APK file from C&amp;C server</figcaption></figure></div>



<p></p>



<p>We analyzed the C&amp;C command ‘info’ and the associated APK. This command collects the clipboard data and verifies the victims’ device for the presence of a hardcoded list of mobile security products, may be with the aim of disabling them or forwarding the info to the C2.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="867" height="366" src="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure16-1.png" alt="" class="wp-image-24545" srcset="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure16-1.png 867w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure16-1-300x127.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure16-1-768x324.png 768w" sizes="(max-width: 867px) 100vw, 867px" /><figcaption>Figure 16: Collects the clipboard information</figcaption></figure></div>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="528" height="467" src="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure17.png" alt="" class="wp-image-24546" srcset="https://labs.k7computing.com/wp-content/uploads/2022/08/Figure17.png 528w, https://labs.k7computing.com/wp-content/uploads/2022/08/Figure17-300x265.png 300w" sizes="(max-width: 528px) 100vw, 528px" /><figcaption>Figure 17: Checks for the presence of security related products</figcaption></figure></div>
