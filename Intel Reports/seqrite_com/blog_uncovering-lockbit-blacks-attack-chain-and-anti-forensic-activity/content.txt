			<span class="rt-reading-time" style="display: block;"><span class="rt-label">Estimated reading time: </span><span class="rt-time">6</span><span class="rt-label"> minutes</span></span><p>Since the infamous Conti ransomware group disbanded due to source code leaks during the Russia-Ukraine war, the LockBit group has claimed dominance. The group has adopted new extortion techniques and added a first-of-its-kind bug-bounty program, along with many features, to advance their new leak site.Â Upon investigation and analysis, we have determined that the new LockBit 3.0 variant has a high infection vector and attack chain exhibiting substantial anti-forensic activity.</p>
<h2>Attack Overview</h2>
<p>LockBit&#8217;s new Black variant showed anti-forensic activities which cleared event logs, killed multiple tasks, and deleted services simultaneously. It obtains initial access to the victim&#8217;s network via SMB brute forcing from various IPs.</p>
<p><img class=" wp-image-5549 aligncenter" src="https://www.seqrite.com/blog/wp-content/uploads/2023/01/lockbit3.drawio-1-300x84.png" alt="" width="414" height="116" srcset="https://www.seqrite.com/blog/wp-content/uploads/2023/01/lockbit3.drawio-1-300x84.png 300w, https://www.seqrite.com/blog/wp-content/uploads/2023/01/lockbit3.drawio-1.png 754w" sizes="(max-width: 414px) 100vw, 414px" /></p>
<p style="text-align: center;"><em><strong>Fig. 1 â Attack Chain</strong></em></p>
<p>The sys-internal tool PSEXEC is used to execute malicious BAT files on a single system which were later cleaned off. These files indicate activity related to modifying RDP &amp; authentication settings while disabling antivirus at the same time:</p>
<ul>
<li>C:\Windows\system32\cmd.exe /c &#8220;&#8221;openrdp.bat&#8221; &#8220;</li>
<li>C:\Windows\system32\cmd.exe /c &#8220;&#8221;mimon.bat&#8221; &#8220;</li>
<li>C:\Windows\system32\cmd.exe /c &#8220;&#8221;auth.bat&#8221; &#8220;</li>
<li>C:\Windows\system32\cmd.exe /c &#8220;&#8221;turnoff.bat&#8221; &#8220;</li>
</ul>
<p>PSEXEC is also used to spread laterally across the victim&#8217;s network to execute the ransomware payload. The encryption is done using a multi-threaded approach where only shared drives got encrypted. The executed payload must have a valid key passed along with the command-line option &#8216;-pass.&#8217; The encrypted files are appended with the <em>.zbzdbs59d</em> extension, which suggests that the builder generates each payload with a random static string.</p>
<h2>Payload Analysis</h2>
<p>The ransomware payload is dropped inside the <em>Windows</em> directory, where every variant requires a unique key to be passed as an argument. This feature was previously known to be used by other ransomware groups like <em>BlackCat</em> and <em>Egregor</em>. Even if the name of the payload is changed from &#8216;Lock.exe&#8217; to anything else or put in any other directory, it does not run. The pass key used in this case is <strong>60c14e91dc3375e4523be5067ed3b111</strong>.</p>
<p>Let us look at a few stages of the payload below:</p>
<h3>Decrypting Sections</h3>
<p><img class=" wp-image-5547 aligncenter" src="https://www.seqrite.com/blog/wp-content/uploads/2023/01/decrypt_func_2-300x220.png" alt="" width="633" height="464" srcset="https://www.seqrite.com/blog/wp-content/uploads/2023/01/decrypt_func_2-300x220.png 300w, https://www.seqrite.com/blog/wp-content/uploads/2023/01/decrypt_func_2-230x168.png 230w, https://www.seqrite.com/blog/wp-content/uploads/2023/01/decrypt_func_2-768x564.png 768w, https://www.seqrite.com/blog/wp-content/uploads/2023/01/decrypt_func_2-150x110.png 150w, https://www.seqrite.com/blog/wp-content/uploads/2023/01/decrypt_func_2.png 809w" sizes="(max-width: 633px) 100vw, 633px" /></p>
<p style="text-align: center;"><strong><em>Fig. 2 â Pseudo code for decrypting PE Sections</em></strong></p>
<p>The key passed in the argument is taken from the command line and verified. If it passes verification, this key is further processed to obtain a 1-byte key to decrypt specific sections obtained by traversing the PEB structure. The three sections decrypted in memory are â TEXT, DATA, and PDATA.</p>
<h3>Resolving Obfuscated APIs</h3>
<p>Being packed and having only a few imports, Win32 APIs are resolved by decrypting the obfuscated string with XOR using the key <strong>0x3A013FD5</strong>, which is again unique to each payload.</p>
<p><img class=" wp-image-5546 aligncenter" src="https://www.seqrite.com/blog/wp-content/uploads/2023/01/win32api-300x41.png" alt="" width="366" height="50" srcset="https://www.seqrite.com/blog/wp-content/uploads/2023/01/win32api-300x41.png 300w, https://www.seqrite.com/blog/wp-content/uploads/2023/01/win32api.png 413w" sizes="(max-width: 366px) 100vw, 366px" /></p>
<p style="text-align: center;"><strong><em>Fig. 3 â Resolving APIs</em></strong></p>
<h3>Privilege Escalation</h3>
<p>When Admin privileges are not present during execution, it uses <strong>CMSTPLUA COM</strong> to bypass the UAC prompt, a legitimate Windows Connection Manager Service. This elevates the rights from the user to the administrator level with another instance of the ransomware payload, terminating the current process.</p>
<p><img class=" wp-image-5545 aligncenter" src="https://www.seqrite.com/blog/wp-content/uploads/2023/01/cmlua_1-300x92.png" alt="" width="694" height="213" srcset="https://www.seqrite.com/blog/wp-content/uploads/2023/01/cmlua_1-300x92.png 300w, https://www.seqrite.com/blog/wp-content/uploads/2023/01/cmlua_1-768x236.png 768w, https://www.seqrite.com/blog/wp-content/uploads/2023/01/cmlua_1-1024x315.png 1024w, https://www.seqrite.com/blog/wp-content/uploads/2023/01/cmlua_1.png 1030w" sizes="(max-width: 694px) 100vw, 694px" /></p>
<p style="text-align: center;"><strong><em>Fig. 4 â UAC Bypass using CMSTPLUA</em></strong></p>
<h3>Anti-Debugging Technique</h3>
<p>Threads used for file encryption are hidden from the debugger by calling <strong>NtSetInformationThread</strong> Win32 API via <strong>ThreadInformationClass</strong> with an undocumented value <strong>0x11</strong> that denotes <strong>ThreadHideFromDebugger</strong>. This hinders dynamic analysis by not allowing debug information from the current ransomware&#8217;s thread to reach the attached debugger.</p>
<p><img class=" wp-image-5544 aligncenter" src="https://www.seqrite.com/blog/wp-content/uploads/2023/01/ntsetinfothread-300x149.png" alt="" width="470" height="234" srcset="https://www.seqrite.com/blog/wp-content/uploads/2023/01/ntsetinfothread-300x149.png 300w, https://www.seqrite.com/blog/wp-content/uploads/2023/01/ntsetinfothread.png 547w" sizes="(max-width: 470px) 100vw, 470px" /></p>
<p style="text-align: center;"><strong><em>Fig. 5 â Anti-Debugging technique to hide threads</em></strong></p>
<h3>Anti-Forensic Activity</h3>
<p>As part of wiping out its traces, lots of anti-forensic activity is observed where Windows Event Logs are disabled by setting multiple registry subkeys to value 0.</p>
<ul>
<li><strong>HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WINEVT\Channels\*</strong></li>
</ul>
<p>Specifically, Windows Defender is disabled for evasion. An exhaustive list of <a href="https://sebsauvage.net/paste/?b0443b90f3fc5ffc#UomoaFa+vkJS9tcLpJwcH1X38YjfbBZDDf3BBk1abPE=" data-wpel-link="external" target="_blank" rel="nofollow">Events Cleared</a>.</p>
<h4>Service Deletion and Process Termination</h4>
<p>Process terminated included <em>SecurityHealthSystray.exe</em> and the mutex created during execution was <strong>13fd9a89b0eede26272934728b390e06</strong>. Services were enumerated using a pre-defined list and deleted or killed if found on the machine:</p>
<ol>
<li>Sense</li>
<li>Sophos</li>
<li>Sppsvc</li>
<li>Vmicvss</li>
<li>Vmvss</li>
<li>Vss</li>
<li>Veeam</li>
<li>Wdnissvc</li>
<li>Wscsvc</li>

<h3>Ransom Note</h3>
<p>Before encryption, the ransom note is created in every directory except the <em>Program Files</em> and the <em>Windows</em> directory, which arenât encrypted. We can see that they have moved the naming convention of ransom notes from â<em>Restore-My-Files.txt</em>â to a static string format â<em>zbzdbs59d.README.txt</em>â.</p>
<p><img class=" wp-image-5543 aligncenter" src="https://www.seqrite.com/blog/wp-content/uploads/2023/01/ransom_note_1-300x147.png" alt="" width="737" height="361" srcset="https://www.seqrite.com/blog/wp-content/uploads/2023/01/ransom_note_1-300x147.png 300w, https://www.seqrite.com/blog/wp-content/uploads/2023/01/ransom_note_1-768x376.png 768w, https://www.seqrite.com/blog/wp-content/uploads/2023/01/ransom_note_1-1024x501.png 1024w" sizes="(max-width: 737px) 100vw, 737px" /></p>
<p style="text-align: center;"><strong><em>Fig. 6 â Ransom Note</em></strong></p>
<p>The ransom note contains instructions to install the TOR browser, links for a chat, and the personal ID unique to the victim to communicate with the attackers. It also includes the threat message to leak the stolen data if the ransom amount is not paid and ends with the warnings as usual. Multiple TOR mirrors for their leak site can be seen in the ransom note, which is used to reduce redundancy.</p>
<h3>File Encryption</h3>
<p>Before starting file encryption, a registry key for <em>DefaultIcon</em> is created to associate an icon to all the encrypted files. Along with this ICO file (zbzdbs59d.ico), a BMP file is also dropped in the <strong>C:\ProgramData</strong> directory. Files are encrypted by creating multiple threads where each filename is replaced with a random string generated and appending the extension to them. With full encryption completed under 2 minutes it still has the fastest encryption process since LockBit 2.0.</p>
<p><img class=" wp-image-5542 aligncenter" src="https://www.seqrite.com/blog/wp-content/uploads/2023/01/encr-297x300.png" alt="" width="430" height="434" srcset="https://www.seqrite.com/blog/wp-content/uploads/2023/01/encr-297x300.png 297w, https://www.seqrite.com/blog/wp-content/uploads/2023/01/encr-150x150.png 150w, https://www.seqrite.com/blog/wp-content/uploads/2023/01/encr-55x55.png 55w, https://www.seqrite.com/blog/wp-content/uploads/2023/01/encr-45x45.png 45w, https://www.seqrite.com/blog/wp-content/uploads/2023/01/encr.png 498w" sizes="(max-width: 430px) 100vw, 430px" /></p>

<h2>Conclusion</h2>
<p>Unprotected systems in the network were brute-forced to run the PSEXEC tool for lateral movement across the systems. This was done to execute LockBitâs latest Black ransomware variant. With LockBit 3.0 introducing its bug bounty program and adopting new extortion tactics, it is mandatory to takeÂ precautions like downloading applications only from trusted sources, using antivirus for enhanced protection, and avoiding clicking on any links received through email or social media platforms. As threat actors create their own variants from the leaked LockBit Blackâs builder, <a href="https://blogs.quickheal.com/proactive-measures-to-safeguard-against-the-ransomware-menace/" data-wpel-link="internal" target="_self" rel="follow">proactive measures</a> must be taken to stay protected.</p>
