<h2>Introduction</h2>
<p>In this post, we cover the CinaRAT loader's evasive tactics, techniques, and procedures (TTPs), as identified and prevented by Morphisec’s zero-trust endpoint security solution powered by Moving Target Defense technology.&nbsp;</p>
<p><!--more--></p>
<p>We review different versions of multi-staged loaders that attempt to inject and execute <strong>CinaRAT</strong> within a victim’s host memory. <em>CinaRAT</em> code is available on GitHub for download; generally it's just a rebranded QuasarRAT.&nbsp;</p>
<p>We focus on the evasive components that allowed attackers to sustain zero detection for such a long period of time (VirusTotal).</p>
<h2>First stage - ISO/VBS script</h2>
<p>Our investigation begins with a Visual Basic script identified in a customer environment. We observed its delivery through an ISO archive file.</p>
<p>The script implements reflective loading, persistence, and evasion functionality.&nbsp;</p>
<p>The first step is a reflective loading of a remote .NET DLL executable, usually from a GitHub user account represented by an image download.</p>

<h2>Second stage: .NET loader</h2>
<p>As described in a previous stage, the Visual Basic script delivers a second .NET file.</p>
<p>The purpose of this .NET file is to decode the final stage .NET <strong>CinaRAT</strong> payload and inject it into a legitimate Windows process using process hollowing. This is done with several evasion techniques in order to avoid detection and AI.&nbsp;</p>
<p>The RAT payload resides in a base64 encoded string that, during the execution, is decoded and XORed with string as the key. Once the .NET DLL decodes the RAT, it hollows legitimate images in memory and injects it.</p>
