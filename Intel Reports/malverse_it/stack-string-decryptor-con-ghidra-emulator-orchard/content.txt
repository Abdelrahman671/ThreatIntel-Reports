<h2 class="wp-block-heading">Introduzione</h2>



<p>Ciao a tutti! Oggi vedremo come realizzare uno script sfruttando le API di <strong>Ghidra </strong>per decifrare le stringhe di <strong>Orchard</strong>. In particolare analizzeremo il sample V3 (MD5: <strong>cb442cbff066dfef2e3ff0c56610148f</strong>) sviluppato in C++. Questo malware sfrutta inoltre un&#8217;interessante tecnica <strong>DGA </strong>il cui seed non Ã¨ deterministico e dipende dal balance del genesis block; l&#8217;analisi tecnica di questo aspetto si puÃ² trovare sui blog <a rel="noreferrer noopener" href="https://bin.re/blog/a-dga-seeded-by-the-bitcoin-genesis-block/" data-type="URL" data-id="https://bin.re/blog/a-dga-seeded-by-the-bitcoin-genesis-block/" target="_blank">bin.re</a> e <a rel="noreferrer noopener" href="https://blog.netlab.360.com/a-new-botnet-orchard-generates-dga-domains-with-bitcoin-transaction-information/" target="_blank">360 Netlab</a>.</p>



<p>Orchard per la decryption delle stringhe memorizza nello stack l&#8217;<strong>offset del carattere all&#8217;interno dell&#8217;alfabeto</strong>; vedremo quindi come Ã¨ possibile realizzare un semplice script che sfrutta l&#8217;<strong>EmulatorHelper </strong>di Ghidra per la decifratura.</p>



<p>Iniziamo!</p>



<h2 class="wp-block-heading">Analisi</h2>



<p>Il malware inserisce nello stack gli offset dell&#8217;alfabeto e viene costruito ottenendo degli int da variabili globali e valori immediati; successivamente vengono chiamate due funzioni, la prima che si occupa di creare un oggetto e la seconda di fare la decryption della stringa. </p>



<p>La tecnica utilizzata dal malware Ã¨ molto semplice, tuttavia Ã¨ possibile complicare maggiormente l&#8217;analisi facendo sÃ¬ che i valori nello stack dipendando dal risultato di funzioni, come ad esempio spiegato <a rel="noreferrer noopener" href="https://rioasmara.com/2020/10/20/evade-strings-detection-with-stack-based/" data-type="URL" data-id="https://rioasmara.com/2020/10/20/evade-strings-detection-with-stack-based/" target="_blank">qui</a>; in questo caso non potremmo escludere dall&#8217;esecuzione tutte le chiamate a funzione come fatto successivamente per Orchard, in quanto non ci permetterebbe di ottenere il valore corretto della stringa.</p>

<p>Per l&#8217;emulation successiva ci interessa come vengono utilizzati i registri e in particolare per la prima funzione:</p>



<ul class="wp-block-list"><li><strong>EDX</strong>: puntatore al primo offset all&#8217;interno dello stack.</li><li><strong>EAX</strong>: contiene il valore che sottratto all&#8217;indirizzo del primo offset e diviso per 4 permette di ottenere la lunghezza della stringa.</li><li><strong>ECX</strong>: puntatore this.</li></ul>



<p>Ad esempio in questo caso EDX punta a <strong>0x19F8E4 </strong>e quindi 33,39,33,34,25,2D,24,32,29,36,25 che equivale alla stringa <strong>SYSTEMDRIVE</strong>. In EAX invece abbiamo 0x19F910, quindi (0x19F910 &#8211; 0x19F8E4)/ 4 = <strong>11 </strong>che Ã¨ il numero di offset presenti.</p>

<h2 class="wp-block-heading">Emulation con Ghidra</h2>



<p>Per effettuare Emulation ci sono diverse soluzioni come Unicorn, Flare-Emu, Qiling e Dumpulator. In questo caso utilizzeremo le API di Ghidra attraveso la classe <a href="https://ghidra.re/ghidra_docs/api/ghidra/app/emulator/EmulatorHelper.html" data-type="URL" data-id="https://ghidra.re/ghidra_docs/api/ghidra/app/emulator/EmulatorHelper.html">EmulatorHelper</a> . Per approfondire l&#8217;argomento: <a rel="noreferrer noopener" href="https://www.youtube.com/watch?v=xz5qHmc41LI" target="_blank">QUI</a>, <a rel="noreferrer noopener" href="https://saml98.github.io/jekyll/update/2022/05/23/ghidra-stack-strings-emulation.html" target="_blank">QUI</a> e <a rel="noreferrer noopener" href="https://maxkersten.nl/binary-analysis-course/analysis-scripts/ghidra-script-to-handle-stack-strings/" target="_blank">QUI</a>.</p>



<p>In particolare, i passaggi effettuati per l&#8217;emulazione sono:</p>



<ul class="wp-block-list"><li><strong>Ottenere tutte le chiamate alla funzione di Decryption</strong><ul><li>Ottenibile facilmente con le API Ghidra getReferencesTo e getFromAddress</li></ul></li><li><strong>Determinare quale istruzioni devono essere emulate</strong><ul><li>Funzione getStartAndEndAddress</li><li>Una stessa funzione puÃ² contenere piÃ¹ chiamate alla funzione di Decryption, quindi l&#8217;inizio dell&#8217;emulation Ã¨ il prologo della funzione chiamante oppure l&#8217;indirizzo dell&#8217;istruzione successiva a una precedente chiamata di Decryption</li></ul></li><li><strong>Emulare solamente</strong> <strong>le istruzioni che ci interessano</strong><ul><li>Escludere chiamate che possono modificare il flusso (call, jmp, ecc)</li><li>L&#8217;API Ghidra getFlowType permette di ottenere il <a rel="noreferrer noopener" href="https://ghidra.re/ghidra_docs/api/ghidra/program/model/symbol/FlowType.html" target="_blank">FlowType</a> dell&#8217;istruzione</li></ul></li><li><strong>Leggere l&#8217;output dopo l&#8217;emulation</strong><ul><li>Indirizzo contenuto in EDX permette di ottenere il primo valore dell&#8217;offset nello stack</li><li>(Valore contenuto in EAX &#8211; valore contento in EDX) / 4 = size delle stringa</li><li>Le API Ghidra readRegister e readMemory permettono di leggere i valori memorizzati nei registri e nel range di memoria specificato</li></ul></li></ul>



<p>Il codice per effettuare l&#8217;emulation:</p>



<pre class="wp-block-code"><code>from ghidra.app.emulator import EmulatorHelper
from ghidra.program.model.symbol import SymbolUtilities
     
def decrypt(data):
    alphabet = "!\"#$%&amp;'()*+,-./0123456789:;&lt;=&gt;?@ABCDEFGHIJKLMNOPQRSTUVWXYZ&#91;\]^_`abcdefghijklmnopqrstuvwxyzw"
  
    decrypted = ""
      
    for b in data:
       if(b != 0):  
            decrypted += alphabet&#91;b-1]
    
    return decrypted


class Emulator(object):

    def __init__(self):
        self.emulator = EmulatorHelper(currentProgram)

    def run(self):

        references = getReferencesTo(toAddr("DecryptStackStrings"))
        
        decryptOk = 0
        numCall = len(references)
         
        # Ottengo l'indirizzo delle chiamate alla funzione di Decryption
        for ref in references:
        
            callAddr = ref.getFromAddress()
 
            # Ottengo l'indirizzo di partenza e finale per l'emulation
            start, end = self.getStartAndEndAddress(callAddr)
        
            print("--------------------------------------------------------------")
            print("Start: " + start.toString() )
            print("End: " + end.toString() )
            print("--------------------------------------------------------------")
            
            # Imposto i registri e avvio l'emulation con gli indirizzi precedentemente trovati
            self.setupRegister(start)
            self.runEmulation(start, end)

            # Ottengo i valori dei registri EBP, EDX, EAX per calcolare la size e l'indirizzo di partenza degli offset
            valueEBP = self.emulator.readRegister("EBP")
            valueEDX = self.emulator.readRegister("EDX")
            valueEAX = self.emulator.readRegister("EAX")

            # Calcolo la dimensione della stringa
            size = valueEAX - valueEDX

            addr = toAddr(self.emulator.readRegister("EDX"))
            code = bytes(self.emulator.readMemory(addr, size))

            decryptedString = decrypt(code)
             
            if(len(decryptedString) != 0):
                #hexdump.hexdump(code)
                #print("{%s} Decrypted: %s " % (callAddr, decryptedString) )
                
                print("{%s} - %s " % (callAddr, decryptedString) )
                
                decryptOk += 1
                
                # Imposto il commento su Ghidra
                codeUnit = currentProgram.getListing().getCodeUnitAt(callAddr)
                codeUnit.setComment(codeUnit.PLATE_COMMENT, decryptedString)


            
        print("Decryption done:" + str(decryptOk))
        print("Call number: " + str(numCall))
        
        self.emulator.dispose()

     
    def setEIPNextInstruction(self):
    
        nextInstr = getInstructionAt(self.emulator.getExecutionAddress()).getNext()
        self.emulator.writeRegister(self.emulator.getPCRegister(), nextInstr.getAddress().getOffset())
        return nextInstr
              
      
    # Funzione che permette di ottenere l'indirizzo di partenza e termine dell'emulation      
    def getStartAndEndAddress(self, fromAddr):

        # Ottengo il prologo della funzione attuale
        functionAddr = getFunctionContaining(fromAddr).getEntryPoint()

        while True:
        
            instr = getInstructionBefore(fromAddr)
            addr = instr.getAddress()

            mnemonic = instr.getMnemonicString().lower()
    
            if(len(instr.getOpObjects(0)) &gt; 0):   
                op1 = instr.getOpObjects(0)&#91;0]
                op1_str = op1.toString().lower()
            
                op2 = instr.getOpObjects(0)&#91;1]
                op2_str = op2.toString().lower()
     
            # Chiamata funzione CreateObject, indirizzo dove l'emulation deve terminare
            if mnemonic == "call" and op1_str == "00703390":
                end = addr
                
            
            # Se siamo arrivati al prologo oppure a un'altra chiamata di decryption, quello Ã¨ l'indirizzo di partenza dell'emulation
            if functionAddr == addr or (mnemonic == "call" and op1_str == "0070af90"):
                return addr, end
         
            fromAddr = instr.getAddress()

            

    def setupRegister(self, start):
    
        emulator = self.emulator

        # Ottengo la memoria per lo stack e inizializzo ESP e EBP
        stack_address = (start.getAddressSpace().getMaxAddress().getOffset() &gt;&gt; 1) - 0x7fff
        
        emulator.writeRegister("ESP", stack_address)
        emulator.writeRegister("EBP", stack_address)
        emulator.writeRegister(emulator.getPCRegister(), start.getOffset())


    def runEmulation(self, start, end):

        emulationDone = False
        
        while not monitor.isCancelled():
            
            executionAddr = self.emulator.getExecutionAddress()   
            currentInstruction = getInstructionAt(executionAddr)


            # Eseguo solamente istruzioni di tipo FALL_THROUGH escludendo quelle che possono modificare il flusso (call, jmp, ja, ecc)
            op = str(currentInstruction).lower()  
            flowType = currentInstruction.getFlowType().toString()
            prefixes = &#91;"lock"]
            
            while(flowType != "FALL_THROUGH" or op in prefixes):
            
                newInstruction = self.setEIPNextInstruction()   
                flowType = newInstruction.getFlowType().toString()
                op = newInstruction.toString()
                
            if self.emulator.step(monitor) == False:
                raise Exception("Emulation Error: '{}'".format(self.emulator.getLastError()))


            if (executionAddr == end):
                emulationDone = True
                break   
                
                
        if not emulationDone:
            raise Exception("&#91;EMULATION] Error Emulation! ")
    
        print("&#91;EMULATION] Done!")


if __name__=="__main__":

    print(" &#91;EMULATION] Starting.. ")

    Emulator().run()</code></pre>
