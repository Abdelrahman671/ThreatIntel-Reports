<b>/index.php?pop=7</b>.</p><p>While the<b> /index.php </b>file is not infected per se, it <i>is</i> the main PrestaShop file responsible for loading the rest of the CMS scripts on the website — including <b>controllers/front/IndexController.php</b> and <b>classes/tools.php </b>which happen to contain the server-side part of the malware.</p><p>Let’s take a look at how the attacker processes If <b>pop=7, </b>the malware activates the function that processes the stolen data by calling the <b>Tools::redirectErrorPage()</b> that is defined by the attacker in <b>classes/tools.php</b>.</p><p><img loading="lazy" decoding="async" class="aligncenter wp-image-29224 size-full" src="https://blog.sucuri.net/wp-content/uploads/2022/07/function-processes-stolen-data.png" alt="Stolen data processed by calling the Tools::redirectErrorPage(" width="1600" height="867" srcset="https://blog.sucuri.net/wp-content/uploads/2022/07/function-processes-stolen-data.png 1600w, https://blog.sucuri.net/wp-content/uploads/2022/07/function-processes-stolen-data-600x325.png 600w, https://blog.sucuri.net/wp-content/uploads/2022/07/function-processes-stolen-data-650x352.png 650w, https://blog.sucuri.net/wp-content/uploads/2022/07/function-processes-stolen-data-768x416.png 768w, https://blog.sucuri.net/wp-content/uploads/2022/07/function-processes-stolen-data-1536x832.png 1536w, https://blog.sucuri.net/wp-content/uploads/2022/07/function-processes-stolen-data-390x211.png 390w, https://blog.sucuri.net/wp-content/uploads/2022/07/function-processes-stolen-data-820x444.png 820w, https://blog.sucuri.net/wp-content/uploads/2022/07/function-processes-stolen-data-1180x639.png 1180w, https://blog.sucuri.net/wp-content/uploads/2022/07/function-processes-stolen-data-560x303.png 560w, https://blog.sucuri.net/wp-content/uploads/2022/07/function-processes-stolen-data-80x43.png 80w" sizes="auto, (max-width: 1600px) 100vw, 1600px" /></p><p>The <b>redirectErrorPage() </b>function first encrypts the data using <b>openssl_public_encrypt </b>with the attacker’s public key. It then initiates a <b>curl </b>request to exfiltrate the harvested credit card information to <b>hxxps://fastfixtuning[.]nl/cache/cache.php</b> via <b>POST</b> request.</p><p>To make detection and troubleshooting more challenging for the website owner, the malware sets the following cookie and displays the warning “Selected payment method is currently unavailable, please try again.”  as soon as the payment details have been stolen:</p><pre>Presta_Shop=433e5179e5aa1923.1653055359.1.1653055379.1653055359</pre><p>Once set, users with this cookie will no longer see the fake payment form during the checkout process.</p><h2>Website Backdoor in config/alias.php</h2><p>Attackers regularly plant <a href="https://blog.sucuri.net/2022/05/examining-emerging-backdoors.html">backdoors</a> in compromised environments to maintain access long after the initial infection has occurred. Backdoors can come in a variety of flavors — and may use HTTP requests to upload files and web shells or remotely execute code.</p><p>During our analysis, we also found the following line of code in the website’s <b>config/alias.php</b> file.</p><p><img loading="lazy" decoding="async" class="aligncenter wp-image-29223 size-full" src="https://blog.sucuri.net/wp-content/uploads/2022/07/website-backdoor.png" alt="Backdoor in in the website’s config/alias.php " width="841" height="40" srcset="https://blog.sucuri.net/wp-content/uploads/2022/07/website-backdoor.png 841w, https://blog.sucuri.net/wp-content/uploads/2022/07/website-backdoor-600x29.png 600w, https://blog.sucuri.net/wp-content/uploads/2022/07/website-backdoor-650x31.png 650w, https://blog.sucuri.net/wp-content/uploads/2022/07/website-backdoor-768x37.png 768w, https://blog.sucuri.net/wp-content/uploads/2022/07/website-backdoor-390x19.png 390w, https://blog.sucuri.net/wp-content/uploads/2022/07/website-backdoor-820x39.png 820w, https://blog.sucuri.net/wp-content/uploads/2022/07/website-backdoor-560x27.png 560w, https://blog.sucuri.net/wp-content/uploads/2022/07/website-backdoor-80x4.png 80w" sizes="auto, (max-width: 841px) 100vw, 841px" /></p><p>While small and easily confused for normal code, this snippet allows the attacker to upload a webshell or execute malicious code in a <strong>POST</strong> request onto the compromised environment.</p><p>It’s worth noting that while the <b>eval </b>function itself isn’t malicious, searching for <b>eval( </b>can in fact help you identify many backdoors on your website; attackers regularly use this method to inject malicious content or run malicious code on the server. But since backdoors come in so many shapes and sizes, it’s not likely to find everything.</p><h2>Attack &amp; Exfiltration Sequence</h2><p>Our analysis revealed five distinct parts to this skimming attack.</p><ol><li><b> Initial injection into the compromised environment.<br /> </b>The attacker injects malicious obfuscated <b>dfasdf3124sfcad2.js </b>script into the <b>payment.tpl</b> file<b>.</b></li><li><b> JavaScript creates and overlays form.<br /> </b>The contents of <b>dfasdf3124sfcad2.js </b>create a fake form which is overlaid on top of the existing checkout cart.</li><li><b> Second JavaScript skims data.<br /> </b>A skimmer from <b>onepagecheckoutps.js</b> monitors changes in the payment form and sends the payment details to the server-side part of the malware.</li><li><b> Injected code checks for parameter and activates function.<br /> </b>A single line of malicious code injected into <b>IndexController.php </b>file checks and intercepts POST requests containing payment details and activates the function <b>redirectErrorPage()</b>.</li><li><b> Function exfiltrates data to a third party server</b>.<br /> The <b>redirectErrorPage() </b>function from <b>tools.php</b> encrypts harvested data then initiates a <b>curl </b>request to send stolen credit card details to the hacker controlled <b>hxxps://fastfixtuning[.]nl/cache/cache.php</b> via <b>POST</b> request.</li></ol><h2>Conclusion &amp; Mitigation Steps</h2><p>It’s not typical for the same skimmer to be found on thousands of websites. Credit card skimming attacks are often highly targeted and customized campaigns — and we regularly find malware hand-crafted for just one (or a small handful) of sites.</p><p>Attackers clearly went to great lengths to craft this swiper specifically for this particular PrestaShop installation. The customized credit card form and handler function to process stolen data along with obfuscation techniques and naming conventions to evade detection showcase the steps that attackers will take to steal sensitive information from e-commerce websites.</p><p>Fortunately, there are a number of initiatives you can take to protect your website, checkout process, and customers from targeted skimmer attacks like these.</p><ul><li><b>Always keep your website software updated with the latest patches. </b><br /> It cannot be stressed enough how important patching your software is. Regardless of what CMS you use, you should always keep all of your website software up-to-date with the latest patches and security updates — including modules, plugins, themes, and core CMS.</li><li><b>Use strong passwords. </b><br /> Always use <a href="https://blog.sucuri.net/2024/01/how-to-make-strong-password.html" target="_blank" rel="noopener">unique, complex passwords</a> for your CMS, database, FTP/SSH, hosting and cPanel accounts to protect against brute force and dictionary attacks.</li><li><b>Follow the principle of least privilege.</b><br /> Ensure that all user accounts are only assigned to their needed roles by following the<a href="https://blog.sucuri.net/2017/04/the-principle-of-least-privilege.html"> principle of least privilege</a>.</li><li><b>Check your file integrity regularly. </b><br /> Monitoring for changes on your website is an important component of identifying indicators of compromise. There are a number of tools you can use to alert you of any unexpected changes to your website’s files or environment.</li><li><b>Use a web application firewall. </b><br /> Leverage a firewall to filter traffic between your server and visitors and virtually patch known vulnerabilities in the event that you forget to patch an update.</li></ul><p>As always, if you think your website has been hacked or may contain a credit card skimmer and you need a hand cleaning it up, <a href="https://sucuri.net/website-hack-protection/">we’re here to help</a>.</p></div><div class="post-author"><div class="author-wrap"><div class="author"><div class="author-avatar"> <a href="https://blog.sucuri.net/author/matt-morrow" rel="author"> <img alt='' src='https://secure.gravatar.com/avatar/890e38d7466587bdcb96bd18543e2b88?s=120&#038;d=mm&#038;r=g' srcset='https://secure.gravatar.com/avatar/890e38d7466587bdcb96bd18543e2b88?s=240&#038;d=mm&#038;r=g 2x' class='avatar avatar-120 photo' height='120' width='120' loading='lazy' decoding='async'/> </a></div><div class="author-description"><h5 class="title-author"> <span class="fn"> <a href="https://blog.sucuri.net/author/matt-morrow" rel="author"> Matt Morrow </a> </span></h5><p class="note">Matt is a Cyber Security analyst who joined Sucuri in 2018.
