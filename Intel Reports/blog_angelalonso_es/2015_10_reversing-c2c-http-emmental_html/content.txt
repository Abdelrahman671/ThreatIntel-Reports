<h3 class='post-title entry-title' itemprop='name'>
Reversing the C2C HTTP Emmental communication
</h3>
<div class='post-header'>
<div class='post-header-line-1'></div>
</div>
<div class='post-body entry-content' id='post-body-1645279914442541240' itemprop='description articleBody'>
<span style="font-family: Arial, Helvetica, sans-serif;">In last&nbsp;<a href="http://blog.angelalonso.es/2015/10/decrypting-emmental-blowfish-and-base64.html">post</a>&nbsp;l explained how it was possible to decrypt the&nbsp;initial&nbsp;C&amp;C communication&nbsp;from the data dumped from memory, with the support of a python script. In this post, I am going to follow the same approach, but using the information from the captured network traffic.</span><br />
<span style="font-family: Arial, Helvetica, sans-serif;">For that I will capture with Wireshark all the communication with the C&amp;C while the malware is running. Then I can export all the 'objects' in the HTTP connection, which means the content of the HTTP request and response.</span><br />
<span style="font-family: Arial, Helvetica, sans-serif;"><br />
</span><br />
<div class="separator" style="clear: both; text-align: center;">
<a href="https://blogger.googleusercontent.com/img/b/R29vZ2xl/AVvXsEh_VauoCReKHahCF4p53XrvHMmd1CUQHi3j9Cv4sxkvirmY8MRP3MCtM7rf_LrWhHKWz_QncveO6AZTkrarb8YqyqrS9qih2OBNasPJxd5-eU98LNIPTZosfTe5CzcQ_Am9JsA13QNzm9lR/s1600/Untitled.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><span style="font-family: Arial, Helvetica, sans-serif;"><img border="0" height="640" src="https://blogger.googleusercontent.com/img/b/R29vZ2xl/AVvXsEh_VauoCReKHahCF4p53XrvHMmd1CUQHi3j9Cv4sxkvirmY8MRP3MCtM7rf_LrWhHKWz_QncveO6AZTkrarb8YqyqrS9qih2OBNasPJxd5-eU98LNIPTZosfTe5CzcQ_Am9JsA13QNzm9lR/s640/Untitled.png" width="496" /></span></a></div>
<div class="separator" style="clear: both; text-align: center;">
<span style="font-family: Arial, Helvetica, sans-serif;"><br /></span></div>
<div class="separator" style="clear: both; text-align: left;">
<span style="font-family: Arial, Helvetica, sans-serif;">Now, I have e in a folder all the files with the objects from the HTTP request:</span></div>
<div class="separator" style="clear: both; text-align: left;">
<span style="font-family: Arial, Helvetica, sans-serif;"><br /></span></div>

<span style="font-family: Courier New, Courier, monospace;">$ <b>more main.php</b></span></div>
<div class="separator" style="clear: both;">
<span style="font-family: Courier New, Courier, monospace;">i=McsZtRV7Bv7ZjMSzwk5aIyZEiijP8F38NJcxd5VNElaIVxctxxX9UWCGbUaOIYRxhMxTtA8nBYmT%0A%2FkgJOPilsUZZyvc2swCziOJC5ae17wUorRhyx48b3kIReFjFdcomTsuyE8PNXnthpE3oWi%2F%2BV%2Btg%0AT%2Bcp2CCNstrLbeKReraPgFcgZKAlluZKoVG6SxwgKzzt0MxQFlobMu21L%2BmlA2DJ2pj8buhKtEOs%0AGAH9H6kSzdxaKks6lIynIzHLsLfyIKpHqWeTAynnJdBzcKsCIFeNvWzDdrD7Q3By3JgNN6RrgE2l%0A1z5FuY6CihbBi4hUWN36%2F2Gvtg%2BOpSz6********************zk4US3U2GPhp4C3L5%2FcA4a0nroBhndxpaDIjN4ftS0%0ACxg2100kZN9YsaLUXHvJ6Yg9jY7TJzsd8PhlNq%2FWyl1pP5YoOejbO1cFA7jj0RG%2F9Tp60zj1HpDy%0AIFOb9z5FNRI3JjGh3P4V%2Bg0iVWee2FM5Jk4k2ABcGEK5mS8MJnax3iiLEitzTnYb2zgO2V9uO9Bu%0AMq%2BswCEDk2yLmZJJ1ptAUIwj0pXqchauQ1cbikRrMOD5xTprWuQ62iS6kVtvQosA%2BfnxyresiWbs%0Am4B7s3xb%2BSV7ORouF9tQMk68thyu0NXfnSNJxuXm00FXsXMoFt%2Biw4IbtLLyhPjL3QfpMJqX4X0g%0AMmWMMQ81GIbF5Wv%2FR416%2BRCo%2BXBWDabDnwMWDY0M%2FD3G5DcXa5mVRtDJ8CDQol05s0BH9RsBNQsB%0AMFEev%2BHVGfKFCntlAy2If8lNL5k0Rx39g5QKfGIKEOxbxUfsIe0MG1TEs5ABUZBR3g3Ipz0AY9Ax%0ATekUoKpSdg4l7W7v0DFPoHmKD6WqhpPF%2B%2BFPo2CvkhWRKn0IzWcUbyGK%2Fg6ngmIv2%2F8w8nwSt3w0%0AVEZYdQns5zBFPGo6%2B9pZyceFt77QcBcrTGBheKMCsDlGfua3pGKBcxitkvWWdNmcKEP11TjLkEpb%0A4B83AHatDzOwTmh7iYz07GOU7CQXxI93q65Dc1crEEpH7rycKhc7KUZvkWYcmn8dayifCXDpEr%2Bl%0Akonl7bWWLqGDvFJ617dZ1pe9IGUCLCKmV5aunHGC8Zk9iVwELhfkMEv0a9SoxsnNnUIzSGV5hZZu%0A&amp;s=&amp;</span></div>
<span style="font-family: Arial, Helvetica, sans-serif;"><br />
<br />
As the HTTP request is URL encoded, I need first to decode it, so I will adapt the python script created in this <a href="http://blog.angelalonso.es/2015/10/decrypting-emmental-blowfish-and-base64.html">post</a> to do it automatically. This is the script:</span><br />
<br />
<br />
<span style="font-family: Courier New, Courier, monospace;">#!/usr/bin/python</span><br />
<span style="font-family: Courier New, Courier, monospace;"><br />
from Crypto.Cipher import Blowfish</span><br />
<span style="font-family: Courier New, Courier, monospace;">from Crypto import Random</span><br />
<span style="font-family: Courier New, Courier, monospace;">from struct import pack</span><br />
<span style="font-family: Courier New, Courier, monospace;">from binascii import hexlify, unhexlify</span><br />
<span style="font-family: Courier New, Courier, monospace;">import sys</span><br />
<span style="font-family: Courier New, Courier, monospace;">import urllib</span><br />
<span style="font-family: Courier New, Courier, monospace;"><br />
file1 = &nbsp;sys.argv[1]</span><br />
<span style="font-family: Courier New, Courier, monospace;">file_out = sys.argv[2]</span><br />
<span style="font-family: Courier New, Courier, monospace;"><br />
blfs_key = &nbsp;open('/path/to/the/blfs.key','r')</span><br />
<span style="font-family: Courier New, Courier, monospace;"><br />
url_encode = open(file1,'r')</span><br />
<span style="font-family: Courier New, Courier, monospace;">url_encode_2 = url_encode.read()</span><br />
<span style="font-family: Courier New, Courier, monospace;"><br />
url_decode = urllib.unquote(url_encode_2).decode('utf8')</span><br />
<span style="font-family: Courier New, Courier, monospace;"><br />
file_ciphertext_base64 = url_decode</span><br />
<span style="font-family: Courier New, Courier, monospace;">file_blfs_key = blfs_key.read()</span><br />
<span style="font-family: Courier New, Courier, monospace;">ciphertext_raw = file_ciphertext_base64.decode("base64")</span><br />
<span style="font-family: Courier New, Courier, monospace;"><br />
IV = "12345678"</span><br />
<span style="font-family: Courier New, Courier, monospace;">_KEY = file_blfs_key</span><br />
<span style="font-family: Courier New, Courier, monospace;">ciphertext = ciphertext_raw</span><br />
<span style="font-family: Courier New, Courier, monospace;">KEY = hexlify(_KEY)[:50]</span><br />
<span style="font-family: Courier New, Courier, monospace;">cipher = Blowfish.new(KEY, Blowfish.MODE_CBC, IV)</span><br />
<span style="font-family: Courier New, Courier, monospace;">message = cipher.decrypt(ciphertext)</span><br />
<span style="font-family: Courier New, Courier, monospace;">config_plain = open(file_out,'w')</span><br />
<span style="font-family: Courier New, Courier, monospace;">config_plain.write(message)</span><br />
<span style="font-family: Arial, Helvetica, sans-serif;"><br /></span>
<span style="font-family: Arial, Helvetica, sans-serif;"><br />
With this script it is easy to run a shell command with a loop 'for' to decrypt all the files in the directory. Bare in mind than the HTTP response are not URL&nbsp;encoded, so I will not need to perform that step on some of the files.&nbsp;</span><br />
<span style="font-family: Arial, Helvetica, sans-serif;"><br />
Now I should have&nbsp;decrypted&nbsp;all the information from each object. Looking at the first two HTTP POST requests I see this is the case, but for the third one, this is not the case and the data is still encrypted. What's going on here?</span><br />
<span style="font-family: Arial, Helvetica, sans-serif;"><br />
</span><br />
<div class="separator" style="clear: both; text-align: center;">
<a href="https://blogger.googleusercontent.com/img/b/R29vZ2xl/AVvXsEjuZ6QCWQghM7xKJO-I9NNg1K3jSE0GU_4YVTrifzAA6BmFlXuE9RgYniCasGiXMWjbk0N1si9zI4RS-5MNS7k2rZ8crfDgEDFyWR-dsfGWU-wzgbj1cBtHmg6e16_0ZjJkRgfD7nFd_C0V/s1600/Screen+Shot+2015-10-28+at+09.25.42.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><span style="font-family: Arial, Helvetica, sans-serif;"><img border="0" height="352" src="https://blogger.googleusercontent.com/img/b/R29vZ2xl/AVvXsEjuZ6QCWQghM7xKJO-I9NNg1K3jSE0GU_4YVTrifzAA6BmFlXuE9RgYniCasGiXMWjbk0N1si9zI4RS-5MNS7k2rZ8crfDgEDFyWR-dsfGWU-wzgbj1cBtHmg6e16_0ZjJkRgfD7nFd_C0V/s640/Screen+Shot+2015-10-28+at+09.25.42.png" width="640" /></span></a></div>
<div class="separator" style="clear: both; text-align: center;">
<span style="font-family: Arial, Helvetica, sans-serif;"><br /></span></div>
<span style="font-family: Arial, Helvetica, sans-serif;">I am going to take a look to the HTTP response from the server, what information is being sent?</span><br />
<span style="font-family: Arial, Helvetica, sans-serif;"><br />
</span><br />
<div class="separator" style="clear: both; text-align: center;">

<div class="separator" style="clear: both; text-align: center;">
<span style="font-family: Arial, Helvetica, sans-serif;"><br /></span></div>
<span style="font-family: Arial, Helvetica, sans-serif;"><br />
&nbsp;<b>A Public Key!! </b>really interesting stuff...</span><br />
<span style="font-family: Arial, Helvetica, sans-serif;"><br />
Actually, if I look further in the second HTTP request from the screenshot above I can see the following:</span><br />
<br />
<br />
<span style="font-family: Courier New, Courier, monospace;">$ more "main(3).php"</span><br />
<span style="font-family: Courier New, Courier, monospace;">a:4....</span><br />
<span style="font-family: Courier New, Courier, monospace;">.....</span><br />
<span style="font-family: Courier New, Courier, monospace;">cjogVGhlIEFuZHJvaWQgUHJvamVjdCB8IGphdmEudmVyc2lvbjogMA==</span><br />

<span style="font-family: Courier New, Courier, monospace;">";s:3:"cmd";s:7:"<span style="background-color: yellow;">get_key</span>";s:3:"rid";s:2:"25";s:4:"data";s:0:"";}</span><br />
<span style="font-family: Arial, Helvetica, sans-serif;"><br />

<span style="font-family: Arial, Helvetica, sans-serif;"><br />

<ol>

<li><span style="font-family: Arial, Helvetica, sans-serif;">The C&amp;C server can decrypt with it's private key</span></li>
