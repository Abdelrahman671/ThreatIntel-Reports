    <div class="node-full__body blog-content__body">  <h3>Key findings&nbsp;</h3>

<ul>
	<li>Proofpoint observed advanced persistent threat (APT) TA397 targeting a Turkish defense sector organization with a lure about public infrastructure projects in Madagascar.&nbsp;&nbsp;</li>
	<li>The attack chain used alternate data streams in a RAR archive to deliver a shortcut (LNK) file that created a scheduled task on the target machine to pull down further payloads.&nbsp;&nbsp;</li>
	<li>TA397 was observed manually delivering WmRAT and MiyaRAT malware families in the final stages of this attack chain. Both malware families are designed to enable intelligence gathering and exfiltration.&nbsp;&nbsp;</li>
	<li>Proofpoint assesses TA397 campaigns are almost certainly intelligence collection efforts in support of a South Asian government’s interests.&nbsp;</li>
</ul>

<h3>Overview&nbsp;</h3>

<p>On November 18, 2024, TA397 (also known by third-party researchers as <a href="https://attack.mitre.org/groups/G1002/" rel="noreferrer noopener" target="_blank">Bitter</a>) targeted a defense sector organization in Turkey with a spearphishing lure. The email included a compressed archive (RAR) file attachment containing a decoy PDF (~tmp.pdf) file detailing a World Bank public initiative in Madagascar for infrastructure development, a shortcut (LNK) file masquerading as a PDF (PUBLIC INVESTMENTS PROJECTS 2025.pdf.lnk), and an Alternate Data Stream (ADS) file that contained PowerShell code.&nbsp;&nbsp;</p>

<p>The lure contained the subject “PUBLIC INVESTMENTS PROJECTS 2025 _ MADAGASCAR” which closely matched the LNK file name masquerading as a PDF within the RAR archive: “PUBLIC INVESTMENTS PROJECTS 2025.pdf.lnk”. This subject line theme is very common for TA397, as the majority of the organizations they target are either in the public sector or receive public investments and is indicative of the targeted nature of their campaigns.&nbsp;&nbsp;</p>

<p>The usage of RAR archives is a staple tactic of TA397 payload delivery. Throughout the first half of 2024, Proofpoint has observed TA397 utilizing Microsoft Compiled Help Files (CHM) files within RAR archives as a means of creating scheduled tasks on target machines.&nbsp;&nbsp;</p>

<p>This blog post details TA397’s usage of NTFS alternate data streams (ADS) in combination with PDF and LNK files to gain persistence, which facilitates the deployment of further malware. This research also looks at the continued usage of wmRAT by TA397, the recently discovered MiyaRAT - a contemporary addition to the threat actor’s arsenal – and the associated infrastructure of TA397.&nbsp;&nbsp;</p>

<h3 aria-level="1" paraeid="{8704eae8-3111-4738-b1ea-5b84a8c6f028}{210}" paraid="1672810117" role="heading">Infection chain&nbsp;</h3>

<p paraeid="{8704eae8-3111-4738-b1ea-5b84a8c6f028}{221}" paraid="1263734997">The spearphishing email originated from a compromised email account belonging to a government organization and contained a RAR archive with a variety of artifacts inside. Alongside the LNK file, was a “~tmp.pdf” file and two NTFS alternate data streams (ADS), one titled “Participation” and the other a “Zone.Identifier”.&nbsp;&nbsp;</p>

By default, Windows hides the real extension of a file. However, if the RAR is opened in 7-Zip, the user can view and extract the NTFS ADS streams on Windows systems (NTFS file formatted system):&nbsp;</p>

<p paraeid="{9cf0fba9-951d-4159-88a5-fe0c3eb77c10}{224}" paraid="778570999">ADS streams are a feature of the NTFS file system in Windows that allows users to attach data streams to a file. There are certain archive formats and software that allow ADS streams to be included into the archive container along with the file. The archive format used in this attack chain is RAR <a href="https://www.rarlab.com/technote.htm" rel="noreferrer noopener" target="_blank">v5</a> which allows the storage of NTFS ADS streams.&nbsp;&nbsp;</p>

<p paraeid="{9d759cb1-9f44-4b08-9f23-1cf9515e435d}{24}" paraid="160926731">The Zone.Identifier stream is an ADS introduced in older Windows versions as a <a href="https://www.digital-detective.net/forensic-analysis-of-zone-identifier-stream/" rel="noreferrer noopener" target="_blank">security feature</a>. It stores information about the origin of a file, such as the URL Security Zone (e.g., Zone 3 for the internet) to determine if the file is trustworthy. Files downloaded via browsers automatically receive this stream. Additionally, when files are extracted from a downloaded archive using Windows Explorer, the extracted files inherit a Zone.Identifier stream with a ReferrerUrl pointing to the original archive. A Zone.Identifier ADS is very common and is not required for the success of this attack chain, nevertheless it can provide useful information for forensic analysis.&nbsp;&nbsp;</p>

<p paraeid="{9d759cb1-9f44-4b08-9f23-1cf9515e435d}{81}" paraid="1115205773">The Zone.Identifier ADS contains information about the origin of the “~tmp.pdf” file, and can be seen below:&nbsp;</p>

Below is Proofpoint’s detailed analysis of WmRAT and MiyaRAT.&nbsp;&nbsp;</p>

<h3>WmRAT<b>&nbsp;</b></h3>

<p>WmRAT is a remote access trojan (RAT) written in C++ that uses sockets for communications and has standard RAT functionality. The RAT can gather basic host information, upload or download files, take screenshots, get geolocation data of the target machine, enumerate directories and files, and run arbitrary commands via cmd or PowerShell. The malware also generates a number of junk threads, potentially to mislead researchers or responders investigating the samples.&nbsp;</p>

<p>The malware starts by copying timezone information from calling <a href="https://learn.microsoft.com/en-us/windows/win32/api/timezoneapi/nf-timezoneapi-getdynamictimezoneinformation">GetDynamicTimeZoneInformation.</a> Rather than any unique or interesting sleep techniques, this malware uses the classic method of directly calling Sleep. This is done throughout the malware at various stages as well as having a dedicated function serving the purpose of just initiating a long sleep.&nbsp;&nbsp;</p>

<p>The sample then creates a thread which gathers some basic host information:&nbsp;</p>

<ul>
	<li paraeid="{9d759cb1-9f44-4b08-9f23-1cf9515e435d}{214}" paraid="93645046">Username&nbsp;</li>
	<li paraeid="{d8b04385-68b5-4dd3-8fac-6a29e8613aef}{54}" paraid="243779820">Hostname&nbsp;</li>
	<li paraeid="{d8b04385-68b5-4dd3-8fac-6a29e8613aef}{60}" paraid="1025489906">The list of logical drives for the host&nbsp;</li>
</ul>

<p paraeid="{d8b04385-68b5-4dd3-8fac-6a29e8613aef}{66}" paraid="1771576051">While this information is gathered, nothing is done with it and this process is repeated 1,000 times. Most likely this is just to fill up behavior logs or create copious amounts of noise within the environment.&nbsp;&nbsp;&nbsp;</p>

<p paraeid="{d8b04385-68b5-4dd3-8fac-6a29e8613aef}{84}" paraid="1229008460"><img alt="Figure 8" data-entity-type="file" data-entity-uuid="b70f3b6d-a303-4593-9bb9-6e7b3d8f83e5" height="520" src="/sites/default/files/inline-images/Screenshot%202024-12-11%20at%209.43.37%E2%80%AFAM.png" width="535" loading="lazy"></p>

<p paraeid="{d8b04385-68b5-4dd3-8fac-6a29e8613aef}{89}" paraid="1972801968"><em>Malware gathering basic disk information.&nbsp;&nbsp;</em></p>

<p paraeid="{d8b04385-68b5-4dd3-8fac-6a29e8613aef}{99}" paraid="1369523855">Soon after creating this thread, another thread is created that executes the same exact function gathering the same information.&nbsp;&nbsp;</p>

<p paraeid="{d8b04385-68b5-4dd3-8fac-6a29e8613aef}{111}" paraid="1503100340">With the threads created, the malware begins attempts to communicate with the C2. In the samples observed by Proofpoint at this stage a socket has not been initialized yet, so these specific communications fail. Although later in the execution the socket is initialized properly, so it will indeed be able to communicate with the C2 after that point.&nbsp; The malware decrypts the C2 server address academymusica[.]com by taking each character from an encrypted blob and subtracting 0x25 from it.&nbsp;&nbsp;</p>

<p paraeid="{d8b04385-68b5-4dd3-8fac-6a29e8613aef}{173}" paraid="148109276"><img alt="Figure 9" data-entity-type="file" data-entity-uuid="bd72883d-b301-42c6-8ebe-8b081bfdd7db" height="284" src="/sites/default/files/inline-images/Screenshot%202024-12-11%20at%209.43.47%E2%80%AFAM.png" width="547" loading="lazy"></p>

<p paraeid="{d8b04385-68b5-4dd3-8fac-6a29e8613aef}{178}" paraid="2092285202"><em>Decrpytion of server address academymusica[.]com.</em></p>

<p paraeid="{d8b04385-68b5-4dd3-8fac-6a29e8613aef}{184}" paraid="2116239262">This subtraction cipher is how all string decryption is done for the malware. Each character has a set value added or subtracted from it, resulting in the expected output. Generally when malware sends command identifiers or data identifiers to the C2, its done in plaintext or is some numeric value. But in the case of this malware, it sends seemingly random values as identifiers. Like the “*****” below.&nbsp;&nbsp;</p>

<p paraeid="{d8b04385-68b5-4dd3-8fac-6a29e8613aef}{222}" paraid="695600310"><img alt="Figure 10" data-entity-type="file" data-entity-uuid="563b3862-2e3c-47a9-ae86-13abf79c2877" height="153" src="/sites/default/files/inline-images/Screenshot%202024-12-11%20at%209.43.54%E2%80%AFAM.png" width="569" loading="lazy"></p>

<p paraeid="{d8b04385-68b5-4dd3-8fac-6a29e8613aef}{227}" paraid="1718550105"><em>Cleartext string of asterisks.&nbsp;&nbsp;</em></p>

<p paraeid="{d8b04385-68b5-4dd3-8fac-6a29e8613aef}{237}" paraid="1323981375">Assuming all the junk threads have started, the malware does a connectivity check by making a request to microsoft[.]com. After this, the socket is finally initialized to connect to the C2 decrypted earlier in the program. This is set to communicate with a hardcoded port, which in this sample is set to 47408.&nbsp;</p>

<p paraeid="{af0926b9-2800-478f-9842-2fed4d5bd481}{8}" paraid="346001608"><img alt="Figure 11" data-entity-type="file" data-entity-uuid="db353572-5c60-4b74-afe7-28f4b8436adb" height="634" src="/sites/default/files/inline-images/Screenshot%202024-12-11%20at%209.55.12%E2%80%AFAM_0.png" width="559" loading="lazy"></p>

<h3 aria-level="1" paraeid="{af0926b9-2800-478f-9842-2fed4d5bd481}{230}" paraid="1782957632" role="heading">MiyaRAT&nbsp;</h3>

<p paraeid="{af0926b9-2800-478f-9842-2fed4d5bd481}{236}" paraid="1923412177">MiyaRAT is also written in C++ and contains similar functionality to WmRAT. The malware starts by decrypting its hardcoded C2 server. This domain is decoded by taking the encoded value and subtracting the matching character in the string “doobiedoodooziezzz” from it.&nbsp;&nbsp;</p>

<p paraeid="{68160603-f327-4b48-87f1-cc7a2bd79b66}{5}" paraid="199956299"><img alt="Figure 13" data-entity-type="file" data-entity-uuid="b962f9cb-4b3b-4a21-9260-5805b882d990" height="417" src="/sites/default/files/inline-images/Screenshot%202024-12-11%20at%2010.00.59%E2%80%AFAM.png" width="594" loading="lazy"></p>

<p paraeid="{68160603-f327-4b48-87f1-cc7a2bd79b66}{10}" paraid="122086497"><em>Decoding C2 server samsnewlooker[.]com.&nbsp;&nbsp;</em></p>

<p paraeid="{68160603-f327-4b48-87f1-cc7a2bd79b66}{50}" paraid="347520383">Doing this results in the C2 domain samsnewlooker[.]com. Along with this, the sample has as hardcoded port value set as a string which determines which port the implant connects to. Strangely the malware then runs its own implementation of Mersenne twister, which is a common random number generator implementation, which is used to generate values to sleep for a designated period of time.&nbsp;&nbsp;</p>

<p paraeid="{68160603-f327-4b48-87f1-cc7a2bd79b66}{54}" paraid="1582148538">With the C2 decoded, a global socket is then created with the hardcoded port of 56189.&nbsp;&nbsp;</p>

<p paraeid="{68160603-f327-4b48-87f1-cc7a2bd79b66}{66}" paraid="1189849088"><img alt="Figure 14" data-entity-type="file" data-entity-uuid="0404b765-8efc-4a22-8960-c1ef5db3801b" height="232" src="/sites/default/files/inline-images/Screenshot%202024-12-11%20at%2010.01.05%E2%80%AFAM.png" width="748" loading="lazy"></p>

<p paraeid="{68160603-f327-4b48-87f1-cc7a2bd79b66}{71}" paraid="361208235"><em>Socket creation with hardcoded port 56189.</em>&nbsp;&nbsp;</p>

<p paraeid="{68160603-f327-4b48-87f1-cc7a2bd79b66}{77}" paraid="1387350156">After the socket is initialized, MiyaRAT gathers basic system information which is sent in the first communication to the C2.&nbsp;&nbsp;</p>

<p paraeid="{68160603-f327-4b48-87f1-cc7a2bd79b66}{85}" paraid="1388534620"><img alt="Figure 15" data-entity-type="file" data-entity-uuid="6c343e7d-f972-45f8-b9ee-9b91b35441b9" height="204" src="/sites/default/files/inline-images/Screenshot%202024-12-11%20at%2010.01.11%E2%80%AFAM.png" width="545" loading="lazy"></p>

<p paraeid="{68160603-f327-4b48-87f1-cc7a2bd79b66}{90}" paraid="2020814898"><em>Basic information gathered to send to the threat actor.&nbsp;&nbsp;</em></p>

<p paraeid="{68160603-f327-4b48-87f1-cc7a2bd79b66}{96}" paraid="1481211942">One of the final fields sent within the initial communication is the version of the malware, in this case being 3. &nbsp;</p>

<p paraeid="{68160603-f327-4b48-87f1-cc7a2bd79b66}{96}" paraid="1481211942"><img alt="Figure 16" data-entity-type="file" data-entity-uuid="60017751-657a-49f6-836b-d289b929a7c4" height="161" src="/sites/default/files/inline-images/Screenshot%202024-12-11%20at%2010.01.17%E2%80%AFAM.png" width="531" loading="lazy"></p>

<p paraeid="{68160603-f327-4b48-87f1-cc7a2bd79b66}{96}" paraid="1481211942"><em>Malware version information.&nbsp;&nbsp;</em></p>

<p paraeid="{68160603-f327-4b48-87f1-cc7a2bd79b66}{109}" paraid="1215103989">This data is then encrypted by XORing each byte of the data with 0x43. An example of the entire set of data sent to the C2 can be seen below.&nbsp;&nbsp;</p>

<p paraeid="{68160603-f327-4b48-87f1-cc7a2bd79b66}{123}" paraid="901909303">```&nbsp;</p>

<p paraeid="{68160603-f327-4b48-87f1-cc7a2bd79b66}{129}" paraid="322304383">C:\--3  [106.07/238.47] GB FREE&nbsp;<br>
|&lt;username&gt;|&lt;hostname&gt;|C:\Windows\cnstaller\MScCA69.tmp|C:\Users\&lt;user&gt;|10.0 125 19044|3.0|&nbsp;</p>

<p paraeid="{68160603-f327-4b48-87f1-cc7a2bd79b66}{145}" paraid="1154864671">```&nbsp;</p>

<p paraeid="{68160603-f327-4b48-87f1-cc7a2bd79b66}{149}" paraid="1609876398">The C2 can respond commands that MiyaRAT supports:&nbsp;</p>

<ul>
	<li paraeid="{68160603-f327-4b48-87f1-cc7a2bd79b66}{165}" paraid="726717374">GDIR – get directory tree&nbsp;</li>
	<li paraeid="{68160603-f327-4b48-87f1-cc7a2bd79b66}{173}" paraid="573786145">DELz – remove directory/file&nbsp;</li>
	<li paraeid="{68160603-f327-4b48-87f1-cc7a2bd79b66}{189}" paraid="144889870">GFS – enumerate all files from a specific directory&nbsp;</li>
	<li paraeid="{68160603-f327-4b48-87f1-cc7a2bd79b66}{195}" paraid="1661203888">SH1start_cmd – reverse shell using CMD&nbsp;</li>
	<li paraeid="{68160603-f327-4b48-87f1-cc7a2bd79b66}{209}" paraid="981716211">SH1start_ps – reverse shell using PowerShell&nbsp;</li>
	<li paraeid="{68160603-f327-4b48-87f1-cc7a2bd79b66}{221}" paraid="2129176868">SH1 – interact with reverse shell&nbsp;</li>
	<li paraeid="{68160603-f327-4b48-87f1-cc7a2bd79b66}{233}" paraid="1549828464">SFS – connect to new socket to upload and download files via UPL/DWNL&nbsp;</li>
	<li paraeid="{68160603-f327-4b48-87f1-cc7a2bd79b66}{249}" paraid="1098735996">GSS – take screenshot and exfil&nbsp;&nbsp;</li>
	<li paraeid="{68160603-f327-4b48-87f1-cc7a2bd79b66}{255}" paraid="202543328">SH1exit_client – close infection&nbsp;</li>
	<li paraeid="{97e8a8c6-2de2-438b-8c93-20d46a52d039}{12}" paraid="453659506">SH2 - interact with reverse shell&nbsp;</li>
</ul>

<h3 aria-level="1" paraeid="{97e8a8c6-2de2-438b-8c93-20d46a52d039}{28}" paraid="1840265059" role="heading">Network analysis&nbsp;</h3>

<p paraeid="{97e8a8c6-2de2-438b-8c93-20d46a52d039}{34}" paraid="191820171">TA397’s infrastructure usage throughout this campaign is divided between the implant domains and the staging domains. The jacknwoods[.]com domain acted as their staging domain to distribute WmRAT and MiyaRAT, and academymusica[.]com and samsnewlooker[.]com as C2 domains for each implant.&nbsp;&nbsp;</p>

<p paraeid="{97e8a8c6-2de2-438b-8c93-20d46a52d039}{60}" paraid="452896199">The staging domain jacknwoods[.]com resolved to 185.244.151[.]84, registered with a Let’s Encrypt certificate with GoDaddy as a provider. Proofpoint has observed this combination of domain registration patterns in several previous TA397 campaigns when deploying staging infrastructure. The IP is multi-tenanted, and not controlled by TA397.&nbsp;&nbsp;</p>

<p paraeid="{97e8a8c6-2de2-438b-8c93-20d46a52d039}{94}" paraid="1030574260">The WmRAT C2 academymusica[.]com resolved to 38.180.142[.]228 at the time of analysis, while theMiyaRAT C2&nbsp; samsnewlooker[.]comresolved to 96.9.215[.]155. Proofpoint assesses it is likely these two IPs are attacker-owned infrastructure.&nbsp;&nbsp;&nbsp;</p>

<h3 aria-level="1" paraeid="{97e8a8c6-2de2-438b-8c93-20d46a52d039}{150}" paraid="573955414" role="heading">Attribution&nbsp;</h3>

<p paraeid="{97e8a8c6-2de2-438b-8c93-20d46a52d039}{156}" paraid="509584938">Whilst this blog has revealed an interesting evolution of TTPs in TA397’s capability set; this activity still demonstrates signature markers of behavior Proofpoint have observed from TA397 historically:&nbsp;</p>

<ul>
	<li paraeid="{97e8a8c6-2de2-438b-8c93-20d46a52d039}{196}" paraid="238380151">The usage of RAR archives to deliver a payload that creates a scheduled task on target machines, with a command line structure that seldom changes.&nbsp;&nbsp;</li>
	<li paraeid="{97e8a8c6-2de2-438b-8c93-20d46a52d039}{208}" paraid="322026829">The infrastructure and hosting providers in use bears similarities to historical TA397 infrastructure.&nbsp;&nbsp;</li>
	<li paraeid="{97e8a8c6-2de2-438b-8c93-20d46a52d039}{222}" paraid="368464062">Targeting organizations in the defense sector in the EMEA and APAC regions.&nbsp;</li>
	<li paraeid="{97e8a8c6-2de2-438b-8c93-20d46a52d039}{236}" paraid="1062889067">Across observed campaigns, activity consistently falls within the working hours of UTC+5:30. Proofpoint has observed the manual deployment of TA397 malware multiple times across different campaigns that align with the working hours of this timezone.&nbsp;&nbsp;</li>
	<li paraeid="{d27fd394-0d4e-4876-90a7-7aea2fd4e009}{7}" paraid="1260969722">Both RATs analyzed in this blog have been historically attributed to TA397 (<a href="https://www.secrss.com/articles/57351" rel="noreferrer noopener" target="_blank">wmRAT</a> and <a href="https://www.secrss.com/articles/71129" rel="noreferrer noopener" target="_blank">MiyaRAT</a> respectively), and Proofpoint concurs with this assessment.&nbsp;&nbsp;</li>
</ul>

<p paraeid="{cc9e2c0d-e254-40f2-9a29-fafd26871e62}{20}" paraid="590392697">Proofpoint assesses WmRAT and MiyaRAT are two distinct malware families in active, operational use by TA397. We believe it is a realistic possibility that both families were written by the same developer(s). It is also clear that MiyaRAT is the newer of the two tools in their arsenal. Proofpoint assesses MiyaRAT may be reserved for targets TA397 deems high value due to the observed sporadic deployment of the malware in only a certain number of campaigns.&nbsp;&nbsp;</p>

<p paraeid="{d27fd394-0d4e-4876-90a7-7aea2fd4e009}{89}" paraid="493079033">Based on Proofpoint's analysis of the characteristics of the observed campaign, previously observed lure content, historical targeting, time-based analysis of TA397 campaigns, and a review of industry reporting on activity associated to Bitter, Proofpoint researchers assess these campaigns are almost certainly intelligence collection efforts in support of a South Asian government’s interests. &nbsp;</p>

<h3 aria-level="1" paraeid="{d27fd394-0d4e-4876-90a7-7aea2fd4e009}{167}" paraid="359525239" role="heading">Why it matters&nbsp;</h3>

<p paraeid="{d27fd394-0d4e-4876-90a7-7aea2fd4e009}{173}" paraid="1238005490">TA397 is a prominent South-Asian nexus, espionage-focused APT that frequently and consistently targets government, energy, telecommunications, defense and engineering organizations throughout the EMEA and APAC regions. They persistently utilize scheduled tasks to communicate with their staging domains to deploy malicious backdoors into target organizations, for the purpose of gaining access to privileged information and intellectual property. &nbsp;<br>
&nbsp;<br>
This blog provides defenders with the knowledge and tools necessary to identify and defend against intrusions from TA397.&nbsp;</p>
