Websites with Parrot TDS have malicious scripts injected into existing JavaScript code hosted on the server. This TDS is easily identifiable by keywords found in the injected JavaScript that we will explore to show the evolution of this threat.\r\n\r\nThis injected script consists of two components: an initial landing script that profiles the victim, and a payload script that can direct the victim\u2019s browser to a malicious location or piece of content. To help the reader better understand Parrot TDS, this article provides in-depth analysis of the landing scripts and payload scripts we have collected from this campaign.\r\n\r\nPalo Alto Networks customers are better protected from the threats discussed in this article through our Next-Generation Firewall with Cloud-Delivered Security Services, including Advanced WildFire, DNS Security, Advanced Threat Prevention and Advanced URL Filtering. If you think you might have been compromised or have an urgent matter, contact the Unit 42 Incident Response team.\r\n\r\n\r\n\r\nRelated Unit 42 Topics\r\nWeb Threats\r\n\r\n\r\n\r\nBackground\r\nIn early September 2023, we investigated a notification concerning a compromised website based in Brazil. Our investigation revealed that this website served pages with injected JavaScript identified as Parrot TDS. Further research uncovered many variations of Parrot TDS script from various servers worldwide.\r\n\r\nOur research reveals several versions of injected JavaScript associated with this campaign. Before reviewing all variations of this script, we should better understand the basic nature of Parrot TDS.\r\nParrot TDS Overview\r\nWhile campaigns involving malicious or injected JavaScript code are fairly common, Parrot TDS is notable due to its wide scope and ability to threaten millions of potential victims. This TDS is easily identifiable by keywords from the injected JavaScript, such as:\r\n\r\n \tNdsj\r\n \tNdsw\r\n \tNdsx\r\n\r\nThe threat operators have consistently used these keywords for Parrot TDS. The presence of these keywords makes it easier for researchers to group samples from this campaign together, making it one of the most investigated campaigns in recent years.\r\n\r\nAlthough its origin remains unclear and public reports indicate Parrot TDS started in 2021, our data indicates it first appeared as early as 2019, with full samples available by August of that year. This relatively high-profile campaign would in that case have been active for more than four years.\r\n\r\nOur investigation into Parrot TDS has revealed different versions of injected JavaScript that illustrate its evolution. Throughout its evolution, the chain of events used by Parrot TDS has remained consistent.\r\nChain of Events for Parrot TDS Payload Distribution\r\nAlthough we have observed different versions of Parrot TDS, the attack chain follows the same basic pattern as shown below in Figure 1.\r\n\r\n[caption id=\"attachment_132076\" align=\"aligncenter\" width=\"600\"] Figure 1. Chain of events for payload distribution through Parrot TDS.[\/caption]\r\n\r\nIn most cases, a web server compromised by Parrot TDS injects a landing JavaScript code snippet into existing JavaScript files. This code usually contains keywords such as ndsj or ndsw.\r\n\r\nWe call this the Parrot TDS \u201clanding script\u201d as shown above in Steps 3 and 4 from Figure 1. The landing script conducts environment checks as a way to avoid detection.\r\n\r\nIf conditions set by the landing script are successfully met, the victim\u2019s web browser queries a payload server. This payload server then returns a JavaScript payload containing keywords such as ndsx.\r\n\r\nWe call this second script the \u201cpayload script\u201d as shown in Steps 5 and 6 from Figure 1 above. The Parrot TDS payload script can direct the victim\u2019s browser to a malicious webpage or other potentially harmful content.\r\n\r\nUltimately, the two components we have identified from Parrot TDS traffic are:\r\n\r\n \tLanding scripts (usually containing keywords ndsj or ndsw)\r\n \tPayload scripts (containing keywords such as ndsx)\r\n\r\nTo better understand these two components, we must first examine the landing script.\r\nParrot TDS Landing Script\r\nWe analyzed more than 10,000 Parrot TDS landing scripts from internal and external data sources. The range of this dataset is from August 2019 through October 2023.\r\n\r\nThese samples reveal four versions of Parrot TDS landing script that represent approximately 95.8% of the collected data as indicated in Figure 2 below. The remaining 4.2% could be the future of this campaign, since the characteristics of these samples do not match the four versions of landing script we have identified so far.\r\n\r\n[caption id=\"attachment_132112\" align=\"aligncenter\" width=\"600\"] Figure 2. Pie chart showing Parrot TDS landing script distribution.[\/caption]\r\n\r\nThe four versions of Parrot TDS landing script from 95.8% of our samples use either the keyword ndsw or ndsj, while the other 4.2% use the keyword ndsj. Scripts with the keyword ndsj use more obfuscation techniques such as Canvas, decodeURI or WebAssembly.\r\n\r\nMost Parrot TDS landing scripts from earlier in the campaign were injected as a single line of code, often appended at the end of JavaScript files served from the compromised website. We identify this as Version 1 (V1), and Figure 3 shows one such example. Note: We will acronymize each version as V paired with its sequential version number for the remainder of the article.\r\n\r\n[caption id=\"attachment_132080\" align=\"aligncenter\" width=\"900\"] Figure 3. Example of landing script V1. Source of sample: VirusTotal.[\/caption]\r\n\r\nThe example in Figure 3 indicates a single line of injected code, with the JavaScript normalized and beautified above it.\r\n\r\nThe different landing script versions have no significant differences in the core function of the injected script. Later versions include more obfuscation. The V1 sample from Figure 3 above shows the core function quite clearly. The sixth line of beautified JavaScript code shows an XMLHTTPRequest that interacts with the payload server and executes the response as a payload.\r\n\r\nThe major function and workflow of the landing script of V2 are almost the same as those of V1. The only difference is that V2 appends a token every time it interacts with the payload server. This token contains two random strings as noted below in Figure 4, and the token is usually 21-22 bytes long.\r\n\r\n[caption id=\"attachment_132082\" align=\"aligncenter\" width=\"900\"] Figure 4. Example of Parrot TDS landing script V2. Source of sample: VirusTotal.[\/caption]\r\n\r\nCompared to V1 and V2, the landing script for V3 looks very different. V3 includes a new function that primarily serves as storage for strings, noted as \u201cserving strings\u201d in Figure 5 below.\r\n\r\n[caption id=\"attachment_132084\" align=\"aligncenter\" width=\"900\"] Figure 5. Example of Parrot TDS landing script V3, part 1 of 3. Source of sample: VirusTotal.[\/caption]\r\n\r\nParrot TDS landing script V3 hosts a long array of strings. Each string in the array could be a word or part of a word used by other functions to dynamically construct a keyword or string at runtime.\r\n\r\nAlso shown in Figure 5, another function modifies the string array from the previously-noted function. This makes static deobfuscation for analysis more difficult. Other than that, the core function of V3 is not much different from previous versions.\r\n\r\nThe remaining portions of our V3 landing script example are shown below in Figures 6 and 7.\r\n\r\n[caption id=\"attachment_132086\" align=\"aligncenter\" width=\"900\"] Figure 6. Example of Parrot TDS landing script V3, part 2 of 3.[\/caption]\r\n\r\n[caption id=\"attachment_132088\" align=\"aligncenter\" width=\"900\"] Figure 7. Example of Parrot TDS landing script V3, part 3 of 3.[\/caption]\r\n\r\nCompared to V3, V4 landing scripts contain additional obfuscation and use somewhat different array indexes. V4 also implements additional changes affecting how its JavaScript handles numbers and strings. Despite these changes, V4 has the same overall functionality as V3 landing scripts.\r\n\r\nFigures 8 through 11 below show an example of a V4 landing script.\r\n\r\n[caption id=\"attachment_132090\" align=\"aligncenter\" width=\"900\"] Figure 8. Example of Parrot TDS landing script V4, part 1 of 4. Source of sample: VirusTotal.[\/caption]\r\n\r\n[caption id=\"attachment_132092\" align=\"aligncenter\" width=\"900\"] Figure 9. Example of Parrot TDS landing script V4, part 2 of 4.[\/caption]\r\n\r\n[caption id=\"attachment_132094\" align=\"aligncenter\" width=\"900\"] Figure 10. Example of Parrot TDS landing script V4, part 3 of 4.[\/caption]\r\n\r\n[caption id=\"attachment_132096\" align=\"aligncenter\" width=\"900\"] Figure 11. Example of Parrot TDS landing script V4, part 4 of 4.[\/caption]\r\n\r\nParrot TDS landing script samples using an ndsj keyword are much rarer than ndsw in our collected data. We treat the majority of ndsj landing script samples as minor versions among V3 and V4.\r\n\r\nIn reviewing our collected landing script samples, we found other versions that do not fully fit V1 through V4 or the ndsj landing scripts. These samples include:\r\n\r\n \tA special version that loads its payload with a Canvas object\r\n \tAdvanced versions that involve more obfuscation and WebAssembly code such as decodeURIComponent and String.fromCharCode\r\n \tSamples that also contain injected JavaScript code from different campaigns (if a web server remains vulnerable for an extended period of time, its JavaScript files could be injected with many different snippets of malicious code)\r\n \tSeveral minor versions that apply interchangeable obfuscation, such as using a number value or string value, or using [] or a period to access the property of an object \u2013 the numeric or string values can also be represented as decimal or hexadecimal numbers\r\n\r\nWhile earlier samples of the injected landing script consist of a single line of JavaScript code, we observed an increasing number of Parrot TDS samples with multiple lines of injected JavaScript code since August 2022. This is likely an evasion technique, since a single long line of malicious code is easier to spot in a script file than multiple lines of shorter malicious code.\r\n\r\nParrot TDS landing scripts profile the victim\u2019s web browser, and if all conditions are successfully met, they direct the victim\u2019s browser to retrieve a payload script.\r\nParrot TDS Payload Script\r\nParrot TDS payload scripts use an ndsx keyword, making them relatively easy to identify.\r\n\r\nCompared to the landing scripts, we found fewer unique samples of Parrot TDS payload scripts. We have classified these into nine versions, compared to the four major versions of Parrot TDS landing scripts.\r\n\r\nThese payload scripts are mostly malicious, but V1 only sets a cookie value for the victim and is basically benign. The other eight major versions of the Parrot TDS payload script are malicious.\r\n\r\nV2 is the most common payload script, representing more than 70% of our sample set. Figure 12 shows a column chart revealing the Parrot TDS payload script distribution.\r\n\r\n[caption id=\"attachment_132098\" align=\"aligncenter\" width=\"900\"] Figure 12. Column chart showing Parrot TDS payload script distribution.[\/caption]\r\n\r\nV1 is the simplest version of the Parrot TDS payload script, and it merely sets a cookie that expires after one year as shown below in Figure 13. This payload script is effectively benign.\r\n\r\nA Parrot TDS landing script will only query the payload server if the victim\u2019s browser has no cookie set by a previous payload script. This V1 payload script basically removes the current browser from any follow-up actions for one year.\r\n\r\n[caption id=\"attachment_132100\" align=\"aligncenter\" width=\"900\"] Figure 13. Example of Parrot TDS payload script V1. Source of sample: VirusTotal.[\/caption]\r\n\r\nThe V2 payload script is straightforward. Without any obfuscation, it creates a new script tag to load JavaScript from a malicious URL as shown in Figure 14.\r\n\r\nThis payload script is the most common version we see for Parrot TDS. Around 70% of our collected payload samples are V2.\r\n\r\n[caption id=\"attachment_132102\" align=\"aligncenter\" width=\"633\"] Figure 14. Example of Parrot TDS payload script V2. Source of sample: VirusTotal.[\/caption]\r\n\r\nParrot TDS payload script V3 contains obfuscation and only targets victims running Microsoft Windows. Figure 15 shows an example of a V3 payload script.\r\n\r\nIn the bottom third of the script, ls represents a decode function that decodes several strings in the script. Our investigation revealed that V3 payload scripts will check for the following conditions:\r\n\r\n \tA referrer\r\n \tAcceptable URL format\r\n \tA platform identifier of \u201cwindows\u201d\r\n \tThat Parrot TDS had not previously set a cookie\r\n\r\nAfter passing all checks, the V3 payload script functions the same as V2, loading an additional script from a malicious URL.\r\n\r\n[caption id=\"attachment_132104\" align=\"alignnone\" width=\"900\"] Figure 15. Example of Parrot TDS payload script V2. Source of sample: VirusTotal.[\/caption]\r\n\r\nV4 and V5 payload scripts are similar. V4 is effectively a V1 payload script plus additional code as shown in Figure 16.\r\n\r\nV5 is effectively a V2 payload script plus additional code (see Figure 17). In both cases, the additional code appears before the original V1 or V2 functions.\r\n\r\n[caption id=\"attachment_132106\" align=\"aligncenter\" width=\"900\"] Figure 16. Example of Parrot TDS payload script V4. Source of sample: VirusTotal.[\/caption]\r\n\r\n[caption id=\"attachment_132108\" align=\"aligncenter\" width=\"900\"] Figure 17. Example of Parrot TDS payload script V5. Source of sample: VirusTotal.[\/caption]\r\n\r\nWith V4 and V5, Parrot TDS payload scripts involve more obfuscation, which is similar to the obfuscation seen in V3 landing scripts. The core function of this extra payload script code is to hook all clickable links in the landing page. Whenever a visitor to the webpage clicks a link, the script will create a new image object and load from a specific URL.\r\n\r\nV6 through V9 of the payload script include more obfuscation. These are very rare in our dataset.\r\nTargets of Parrot TDS\r\nParrot TDS is part of an ongoing campaign targeting victims across the globe. We see landing script or payload script samples daily from a variety of websites compromised through this campaign. While our study began with a tip about a compromised Brazilian website, the variety of compromised websites we found serving Parrot TDS indicates victims are not limited to a single industry, nationality or geographic area.\r\n\r\nThe attackers likely use automatic tools to exploit known vulnerabilities. The majority of the compromised servers use WordPress, Joomla or other content management systems (CMS) to host a website. Even websites without CMS could be compromised through this campaign, since server-side vulnerabilities are not limited to CMS.\r\nConclusion\r\nParrot TDS is a notable part of our threat landscape. This campaign has lasted more than four years, and it keeps evolving with new techniques and obfuscations. Most websites compromised through this campaign use some sort of CMS like WordPress or Joomla.\r\n\r\nWebsite administrators can detect if Parrot TDS has compromised their sites by searching files hosted on the associated web server. For example, they can search server content for the keywords associated with Parrot TDS, like ndsj, ndsw and ndsx. <p><span style="font-weight: 400;">A traffic direction system (TDS) nicknamed Parrot TDS has been publicly reported as active since October 2021. Websites with Parrot TDS have malicious scripts injected into existing JavaScript code hosted on the server. This TDS is easily identifiable by keywords found in the injected JavaScript that we will explore to show the evolution of this threat.</span></p>
<p>This injected script consists of two components: an initial landing script that profiles the victim, and a payload script that can direct the victim’s browser to a malicious location or piece of content. To help the reader better understand Parrot TDS, this article provides in-depth analysis of the landing scripts and payload scripts we have collected from this campaign.</p>

<h2>Background</h2>
<p>In early September 2023, we investigated a notification concerning a compromised website based in Brazil. Our investigation revealed that this website served pages with injected JavaScript identified as <a href="https://decoded.avast.io/janrubin/parrot-tds-takes-over-web-servers-and-threatens-millions/" target="_blank" rel="noopener">Parrot TDS</a>. Further research uncovered many variations of Parrot TDS script from various servers worldwide.</p>
<p>Our research reveals several versions of injected JavaScript associated with this campaign. Before reviewing all variations of this script, we should better understand the basic nature of Parrot TDS.</p>
<h2><a id="post-132073-_wven14kmgum2"></a>Parrot TDS Overview</h2>
<p>While campaigns involving malicious or injected JavaScript code are fairly common, Parrot TDS is notable due to its <a href="https://thehackernews.com/2022/06/researchers-uncover-malware-controlling.html" target="_blank" rel="noopener">wide scope</a> and <a href="https://cymulate.com/threats/parrot-tds-takes-over-web-servers-and-threatens-millions/" target="_blank" rel="noopener">ability to threaten millions of potential victims</a>. This TDS is easily identifiable by keywords from the injected JavaScript, such as:</p>
<ul>
<li><span style="font-family: 'courier new', courier, monospace;">Ndsj</span></li>
<li><span style="font-family: 'courier new', courier, monospace;">Ndsw</span></li>
<li><span style="font-family: 'courier new', courier, monospace;">Ndsx</span></li>
</ul>
<p>The threat operators have consistently used these keywords for Parrot TDS. The presence of these keywords makes it easier for researchers to group samples from this campaign together, making it one of the most investigated campaigns in recent years.</p>
<p>Although its origin remains unclear and public reports indicate Parrot TDS started in 2021, our data indicates it first appeared as early as 2019, with full samples available by August of that year. This relatively high-profile campaign would in that case have been active for more than four years.</p>
<p>Our investigation into Parrot TDS has revealed different versions of injected JavaScript that illustrate its evolution. Throughout its evolution, the chain of events used by Parrot TDS has remained consistent.</p>
<h3><a id="post-132073-_t15qbgjw2n2o"></a><strong>Chain of Events for Parrot TDS Payload Distribution</strong></h3>
<p>Although we have observed different versions of Parrot TDS, the attack chain follows the same basic pattern as shown below in Figure 1.</p>
<figure id="attachment_132076" aria-describedby="caption-attachment-132076" style="width: 600px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-132076" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2024/01/word-image-132073-1.jpeg" alt="Image 1 is an attack chain diagram of Parrot TDS. Step 1: Attacker compromises a legitimate server and sets it up with Parrot TDS. Icon of attacker with arrow at server. Compromised web server song Parrot TDS. Step 2: Victim browses to compromised site. Icon of unsuspecting victim at computer. Arrow to server. Step 3: Web server returns malicious landing script. Step 4: Landing script causes victim’s web browser to retrieve payload script from payload server. Arrow to payload server. Arrow to victim. Step 5: Payload server returns payload script requested by landing script. Step 6: Payload script directs victim’s web browser to malicious location or content. " width="600" height="535" /><figcaption id="caption-attachment-132076" class="wp-caption-text">Figure 1. Chain of events for payload distribution through Parrot TDS.</figcaption></figure>
<p>In most cases, a web server compromised by Parrot TDS injects a landing JavaScript code snippet into existing JavaScript files. This code usually contains keywords such as <span style="font-family: 'courier new', courier, monospace;">ndsj</span> or <span style="font-family: 'courier new', courier, monospace;">ndsw</span>.</p>
<p>We call this the Parrot TDS “landing script” as shown above in Steps 3 and 4 from Figure 1. The landing script conducts environment checks as a way to avoid detection.</p>
<p>If conditions set by the landing script are successfully met, the victim’s web browser queries a payload server. This payload server then returns a JavaScript payload containing keywords such as <span style="font-family: 'courier new', courier, monospace;">ndsx</span>.</p>
<p>We call this second script the “payload script” as shown in Steps 5 and 6 from Figure 1 above. The Parrot TDS payload script can direct the victim’s browser to a malicious webpage or other potentially harmful content.</p>
<p>Ultimately, the two components we have identified from Parrot TDS traffic are:</p>
<ul>
<li>Landing scripts (usually containing keywords <span style="font-family: 'courier new', courier, monospace;">ndsj</span> or <span style="font-family: 'courier new', courier, monospace;">ndsw</span>)</li>
<li>Payload scripts (containing keywords such as <span style="font-family: 'courier new', courier, monospace;">ndsx</span>)</li>
</ul>
<p>To better understand these two components, we must first examine the landing script.</p>
<h3><a id="post-132073-_3rnjygcfjr4m"></a><strong>Parrot TDS Landing Script</strong></h3>
<p>We analyzed more than 10,000 Parrot TDS landing scripts from internal and external data sources. The range of this dataset is from August 2019 through October 2023.</p>
<p>These samples reveal four versions of Parrot TDS landing script that represent approximately 95.8% of the collected data as indicated in Figure 2 below. The remaining 4.2% could be the future of this campaign, since the characteristics of these samples do not match the four versions of landing script we have identified so far.</p>
<figure id="attachment_132112" aria-describedby="caption-attachment-132112" style="width: 600px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-132112" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2024/01/Parrot-TDS-F2.png" alt="Image 2 is a pie chart of the Parrot TDS landing script distribution. Clockwise from left to right: Other is 4.2%. V1 is 0.5% is 2.7%. V3 is 18.2%. V4 is 74.4%." width="600" height="387" /><figcaption id="caption-attachment-132112" class="wp-caption-text">Figure 2. Pie chart showing Parrot TDS landing script distribution.</figcaption></figure>
<p>The four versions of Parrot TDS landing script from 95.8% of our samples use either the keyword <span style="font-family: 'courier new', courier, monospace;">ndsw</span> or <span style="font-family: 'courier new', courier, monospace;">ndsj</span>, while the other 4.2% use the keyword <span style="font-family: 'courier new', courier, monospace;">ndsj</span>. Scripts with the keyword <span style="font-family: 'courier new', courier, monospace;">ndsj</span> use more obfuscation techniques such as <a href="https://developer.mozilla.org/en-US/docs/Web/API/Canvas_API/Tutorial" target="_blank" rel="noopener">Canvas</a>, <a href="https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/decodeURI" target="_blank" rel="noopener">decodeURI</a> or <a href="https://developer.mozilla.org/en-US/docs/WebAssembly/Using_the_JavaScript_API" target="_blank" rel="noopener">WebAssembly</a>.</p>
<p>Most Parrot TDS landing scripts from earlier in the campaign were injected as a single line of code, often appended at the end of JavaScript files served from the compromised website. We identify this as Version 1 (V1), and Figure 3 shows one such example. Note: We will acronymize each version as V paired with its sequential version number for the remainder of the article.</p>
<figure id="attachment_132080" aria-describedby="caption-attachment-132080" style="width: 900px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-132080" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2024/01/word-image-132073-3.png" alt="Image 3 is an example of a landing script from version one. Two screenshots are stacked on top of each other. Some of the information is redacted. Highlighted in the top screenshot are the conditions, which are 1: Referrer exists. 2: the referrer doesn't contain a host name. 3: No cookie. An arrow points to the line that executes if the response contains ndsx." width="900" height="1085" /><figcaption id="caption-attachment-132080" class="wp-caption-text">Figure 3. Example of landing script V1. Source of sample: <a href="https://www.virustotal.com/gui/file/705bda35a217b92f67ef6b190b2a701fda99fb862b9fd7af7856b48d5be75d26" target="_blank" rel="noopener">VirusTotal</a>.</figcaption></figure>
<p>The example in Figure 3 indicates a single line of injected code, with the JavaScript normalized and beautified above it.</p>
<p>The different landing script versions have no significant differences in the core function of the injected script. Later versions include more obfuscation. The V1 sample from Figure 3 above shows the core function quite clearly. The sixth line of beautified JavaScript code shows an <span style="font-family: 'courier new', courier, monospace;">XMLHTTPRequest</span> that interacts with the payload server and executes the response as a payload.</p>
<p>The major function and workflow of the landing script of V2 are almost the same as those of V1. The only difference is that V2 appends a token every time it interacts with the payload server. This token contains two random strings as noted below in Figure 4, and the token is usually 21-22 bytes long.</p>
<figure id="attachment_132082" aria-describedby="caption-attachment-132082" style="width: 900px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-132082" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2024/01/word-image-132073-4.jpeg" alt="Image 4 is an example of version two of a Parrot TDS landing script. Conditions: 1: Referrer exists. 2: the referrer doesn't contain a host name. 3: No cookie. Some of the information is redacted." width="900" height="841" /><figcaption id="caption-attachment-132082" class="wp-caption-text">Figure 4. Example of Parrot TDS landing script V2. Source of sample: <a href="https://www.virustotal.com/gui/file/a49489d3138bf382a4dc683d227d340adfb26a46d894f3c9492c0532923d20c2" target="_blank" rel="noopener">VirusTotal</a>.</figcaption></figure>
<p>Compared to V1 and V2, the landing script for V3 looks very different. V3 includes a new function that primarily serves as storage for strings, noted as “serving strings” in Figure 5 below.</p>
<figure id="attachment_132084" aria-describedby="caption-attachment-132084" style="width: 900px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-132084" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2024/01/word-image-132073-5.jpeg" alt="Image 5 is a screenshot of version three of an example Parrot TDS landing script. Some of the information is redacted. Arrows indicate, from top to bottom, the serving strings and the change order of the strings." width="900" height="845" /><figcaption id="caption-attachment-132084" class="wp-caption-text">Figure 5. Example of Parrot TDS landing script V3, part 1 of 3. Source of sample: <a href="https://www.virustotal.com/gui/file/553cef0f3b7cf70e1786c4e3d10ae990edb9020038f1b7ffe27a4d236b631fbc" target="_blank" rel="noopener">VirusTotal</a>.</figcaption></figure>
<p>Parrot TDS landing script V3 hosts a long array of strings. Each string in the array could be a word or part of a word used by other functions to dynamically construct a keyword or string at runtime.</p>
<p>Also shown in Figure 5, another function modifies the string array from the previously-noted function. This makes static deobfuscation for analysis more difficult. Other than that, the core function of V3 is not much different from previous versions.</p>
<p>The remaining portions of our V3 landing script example are shown below in Figures 6 and 7.</p>
<figure id="attachment_132086" aria-describedby="caption-attachment-132086" style="width: 900px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-132086" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2024/01/word-image-132073-6.jpeg" alt="Image 6 is a screenshot of version three of an example Parrot TDS landing script. <figure id="attachment_132088" aria-describedby="caption-attachment-132088" style="width: 900px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-132088" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2024/01/word-image-132073-7.jpeg" alt="Image 7 is a screenshot of version three of an example Parrot TDS landing script. An arrow points to the line that executes if the response contains ndsx. A second arrow points to indexOf. " width="900" height="226" /><figcaption id="caption-attachment-132088" class="wp-caption-text">Figure 7. Example of Parrot TDS landing script V3, part 3 of 3.</figcaption></figure>
<p>Compared to V3, V4 landing scripts contain additional obfuscation and use somewhat different array indexes. V4 also implements additional changes affecting how its JavaScript handles numbers and strings. Despite these changes, V4 has the same overall functionality as V3 landing scripts.</p>
<p>Figures 8 through 11 below show an example of a V4 landing script.</p>
<figure id="attachment_132090" aria-describedby="caption-attachment-132090" style="width: 900px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-132090" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2024/01/word-image-132073-8.png" alt="Image 8 is a screenshot of version 4 of the example Parrot TDS landing script. <p>Parrot TDS landing script samples using an <span style="font-family: 'courier new', courier, monospace;">ndsj</span> keyword are much rarer than <span style="font-family: 'courier new', courier, monospace;">ndsw</span> in our collected data. We treat the majority of <span style="font-family: 'courier new', courier, monospace;">ndsj</span> landing script samples as minor versions among V3 and V4.</p>
<p>In reviewing our collected landing script samples, we found other versions that do not fully fit V1 through V4 or the <span style="font-family: 'courier new', courier, monospace;">ndsj</span> landing scripts. These samples include:</p>
<ul>
<li>A special version that loads its payload with a Canvas object</li>
<li>Advanced versions that involve more obfuscation and WebAssembly code such as <span style="font-family: 'courier new', courier, monospace;">decodeURIComponent</span> and <span style="font-family: 'courier new', courier, monospace;">String.fromCharCode</span></li>
<li>Samples that also contain injected JavaScript code from different campaigns (if a web server remains vulnerable for an extended period of time, its JavaScript files could be injected with many different snippets of malicious code)</li>
<li>Several minor versions that apply interchangeable obfuscation, such as using a number value or string value, or using <span style="font-family: 'courier new', courier, monospace;">[]</span> or a period to access the property of an object – the numeric or string values can also be represented as decimal or hexadecimal numbers</li>
</ul>
<p>While earlier samples of the injected landing script consist of a single line of JavaScript code, we observed an increasing number of Parrot TDS samples with multiple lines of injected JavaScript code since August 2022. This is likely an evasion technique, since a single long line of malicious code is easier to spot in a script file than multiple lines of shorter malicious code.</p>
<p>Parrot TDS landing scripts profile the victim’s web browser, and if all conditions are successfully met, they direct the victim’s browser to retrieve a payload script.</p>
<h3><a id="post-132073-_upb70tuvosiu"></a><strong>Parrot TDS Payload Script</strong></h3>
<p>Parrot TDS payload scripts use an <span style="font-family: 'courier new', courier, monospace;">ndsx</span> keyword, making them relatively easy to identify.</p>
<p>Compared to the landing scripts, we found fewer unique samples of Parrot TDS payload scripts. We have classified these into nine versions, compared to the four major versions of Parrot TDS landing scripts.</p>
<p>These payload scripts are mostly malicious, but V1 only sets a cookie value for the victim and is basically benign. The other eight major versions of the Parrot TDS payload script are malicious.</p>
<p>V2 is the most common payload script, representing more than 70% of our sample set. Figure 12 shows a column chart revealing the Parrot TDS payload script distribution.</p>

<p>With V4 and V5, Parrot TDS payload scripts involve more obfuscation, which is similar to the obfuscation seen in V3 landing scripts. Whenever a visitor to the webpage clicks a link, the script will create a new image object and load from a specific URL.</p>
<p>V6 through V9 of the payload script include more obfuscation. These are very rare in our dataset.</p>
<h3><a id="post-132073-_ta33v22khb50"></a><strong>Targets of Parrot TDS</strong></h3>
<p>Parrot TDS is part of an ongoing campaign targeting victims across the globe. We see landing script or payload script samples daily from a variety of websites compromised through this campaign. While our study began with a tip about a compromised Brazilian website, the variety of compromised websites we found serving Parrot TDS indicates victims are not limited to a single industry, nationality or geographic area.</p>
<p>The attackers likely use automatic tools to exploit known vulnerabilities. The majority of the compromised servers use WordPress, Joomla or other content management systems (CMS) to host a website. Even websites without CMS could be compromised through this campaign, since server-side vulnerabilities are not limited to CMS.</p>
<h2><a id="post-132073-_2an8ryq91inv"></a>Conclusion</h2>
<p>Parrot TDS is a notable part of our threat landscape. This campaign has lasted more than four years, and it keeps evolving with new techniques and obfuscations. Most websites compromised through this campaign use some sort of CMS like WordPress or Joomla.</p>
<p>Website administrators can detect if Parrot TDS has compromised their sites by searching files hosted on the associated web server. For example, they can search server content for the keywords associated with Parrot TDS, like <span style="font-family: 'courier new', courier, monospace;">ndsj</span>, <span style="font-family: 'courier new', courier, monospace;">ndsw</span> and <span style="font-family: 'courier new', courier, monospace;">ndsx</span>. Administrators can also conduct an audit to discover any extra <span style="font-family: 'courier new', courier, monospace;">.php</span> files on a web server.</p>

		if(error_type == 1) {

	function isEmail(email) {

	function maskEmailAddress (emailAddress) {
		function mask(str) {
			var strLen = str.length;
			if (strLen > 4) {
				return str.substr(0, 1) + str.substr(1, strLen - 1).replace(/\w/g, '*') + str.substr(-1,1);

			return str.replace(/\w/g, '*');
		}

			return mask(p1) + '@' + mask(p2) + p3;
		});

		return emailAddress;
	}
