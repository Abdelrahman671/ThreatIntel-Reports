                            <div class="tech-info-content">
                                <h3 id="threatHead" class="threatSubContentHead" style="">Threat behavior</h3> <div class="threat"><div xmlns="http://www.w3.org/1999/xhtml">
<h5 xmlns="http://www.w3.org/1999/xhtml">Installation</h5>
<p xmlns="http://www.w3.org/1999/xhtml">Threats in this family can be installed on your PC by&#160;<a href="http://www.microsoft.com/security/portal/mmpc/shared/glossary.aspx#exploit" target="_blank">exploit</a> kits such as <a href="http://www.microsoft.com/security/portal/threat/encyclopedia/Entry.aspx?Name=JS%2fNeclu" target="_blank">JS/Neclu</a>, spam email attachments, or infected removable drives. They can also be&#160;downloaded by other malware such as <a href="http://www.microsoft.com/security/portal/threat/encyclopedia/Entry.aspx?Name=Win32%2fGamarue" target="_blank">Win32/Gamarue</a> and <a href="http://www.microsoft.com/security/portal/threat/encyclopedia/Entry.aspx?Name=Win32%2fDorkbot" target="_blank">Win32/Dorkbot</a>.</p>
<p xmlns="http://www.w3.org/1999/xhtml">This malware&#160;can create a file on your PC using the name of any of the files it finds in&#160;<em><span class="notranslate"><a title="Default location for the variable folder &#34;%SystemRoot%&#34;" href="https://www.microsoft.com/en-us/wdsi/help/folder-variables#systemroot" target="_blank">%SystemRoot%</a></span></em>&#160;and sets the attributes to read-only and hidden. For example,&#160;we have seen it use the following files names:</p>
<ul xmlns="http://www.w3.org/1999/xhtml">
<li><em>explorer.exe</em></li>
<li><em>bfsvc.exe</em></li>
<li><em>fveupdate.exe</em></li>
<li><em>helppane.exe</em></li>
<li><em>hh.exe</em></li>
<li><em>isuninst.exe</em></li>
<li><em>notepad.exe</em></li>
<li><em>regedit.exe</em></li>
<li><em>slrundll.exe</em></li>
<li><em>splwow64.exe</em></li>
<li><em>svchost.exe</em></li>
<li><em>taskman.exe</em></li>
<li><em>twunk_16.exe</em></li>
<li><em>twunk_32.exe</em></li>
<li><em>winhelp.exe</em></li>
<li><em>winhlp32.exe</em></li>
<li><em>write.exe</em></li>
</ul>
<p xmlns="http://www.w3.org/1999/xhtml">It then creates this file in the following location:</p>
<ul xmlns="http://www.w3.org/1999/xhtml">
<li><em><span class="notranslate"><a title="Default location for the variable folder &#34;%APPDATA%&#34;" href="https://www.microsoft.com/en-us/wdsi/help/folder-variables#appdata" target="_blank">%APPDATA%</a></span>\&#60;PC name&#62;\&#60;file name&#62;, </em>for example <em><span class="notranslate"><a title="Default location for the variable folder &#34;%APPDATA%&#34;" href="https://www.microsoft.com/en-us/wdsi/help/folder-variables#appdata" target="_blank">%APPDATA%</a></span></em><em>\mymachine\explorer.exe</em></li>
</ul>
<p xmlns="http://www.w3.org/1999/xhtml">Some variants use a random folder name instead of <em>&#60;PC name&#62;. </em>Some variants can also create a shortcut link to <em><span class="notranslate"><a title="Default location for the variable folder &#34;&#60;startup folder&#62;&#34;" href="https://www.microsoft.com/en-us/wdsi/help/folder-variables#startupfolder" target="_blank">&#60;startup folder&#62;</a></span></em> that points to its dropped copy.</p>
<p xmlns="http://www.w3.org/1999/xhtml">The malware creates the following registry entries so that it runs each time you start your <span class="notranslate">PC</span>:</p>
<p xmlns="http://www.w3.org/1999/xhtml" style="padding-left: 30px;">In subkey:&#160;<em>HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Run</em> <br />Sets value: "<em><span class="notranslate"><a title="Default location for the variable folder &#34;%APPDATA%&#34;" href="https://www.microsoft.com/en-us/wdsi/help/folder-variables#appdata" target="_blank">%APPDATA%</a></span>\&#60;PC name&#62;\&#60;file name&#62;</em>", for example "<em><span class="notranslate"><a title="Default location for the variable folder &#34;%APPDATA%&#34;" href="https://www.microsoft.com/en-us/wdsi/help/folder-variables#appdata" target="_blank">%APPDATA%</a></span></em><em>\mymachine\explorer.exe</em>" <br />With data: <em>"&#60;file name&#62;",</em> for example "explorer.exe"<br />&#160; <br />In subkey:&#160;<em>HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run</em> <br />Sets value: "<em><span class="notranslate"><a title="Default location for the variable folder &#34;%APPDATA%&#34;" href="https://www.microsoft.com/en-us/wdsi/help/folder-variables#appdata" target="_blank">%APPDATA%</a></span>\&#60;PC name&#62;\&#60;file name&#62;</em>", for example "<em><span class="notranslate"><a title="Default location for the variable folder &#34;%APPDATA%&#34;" href="https://www.microsoft.com/en-us/wdsi/help/folder-variables#appdata" target="_blank">%APPDATA%</a></span></em><em>\mymachine\explorer.exe</em>" <br />With data: <em>"&#60;file name&#62;",</em> for example "explorer.exe"</p>
<h5 xmlns="http://www.w3.org/1999/xhtml">Spreads through...</h5>
<p xmlns="http://www.w3.org/1999/xhtml">It can create the following copies&#160;on removable drives, such as&#160;<span class="notranslate">USB</span> flash drives:</p>
<ul xmlns="http://www.w3.org/1999/xhtml">
<li><em>&#60;drive&#62;:\WinSystemKB001.exe</em></li>
</ul>
<p xmlns="http://www.w3.org/1999/xhtml">It also creates an <em><span class="notranslate">autorun.inf</span></em> file in the root folder of the removable&#160;drive. The file has instructions to launch the malware automatically when the removable drive is connected to a <span class="notranslate">PC</span> with the <span class="notranslate">Autorun</span> feature turned on.</p>
<p xmlns="http://www.w3.org/1999/xhtml">This is a&#160;common way for malware to spread. However, <em><span class="notranslate">autorun.inf</span></em> files on their own are not necessarily a sign of infection; they are also used by legitimate programs.</p>
<h5 xmlns="http://www.w3.org/1999/xhtml">Payload</h5>
<p xmlns="http://www.w3.org/1999/xhtml"><strong>Steals your sensitive information</strong></p>
<p xmlns="http://www.w3.org/1999/xhtml">This threat can collect the&#160;following information from your PC:</p>
<ul xmlns="http://www.w3.org/1999/xhtml">
<li>PC name</li>
<li>User name</li>
<li>Operating system version</li>
<li>Product ID</li>
<li>Installed antivirus products</li>
<li>Local IP address</li>
</ul>
<p xmlns="http://www.w3.org/1999/xhtml">It also checks to see if you PC is running:</p>
<ul xmlns="http://www.w3.org/1999/xhtml">
<li>64-bit <span class="notranslate">Windows</span></li>
<li>with administrator privilege</li>
</ul>
<p xmlns="http://www.w3.org/1999/xhtml">The malware can also search running processes for credit card data. It skips the following <span class="notranslate">Windows</span> processes:</p>
<ul xmlns="http://www.w3.org/1999/xhtml">
<li>csrss.exe</li>
<li>devenv.exe</li>
<li>lsass.exe</li>
<li>smss.exe</li>
<li>spoolsv.exe</li>
<li>winlogon.exe</li>
</ul>
<p xmlns="http://www.w3.org/1999/xhtml"><strong>Contacts a remote host</strong></p>
<p xmlns="http://www.w3.org/1999/xhtml">The malware connects to a remote command and control (C&#38;C) server&#160;with a 'ping' message. If the server is available, it will reply a &#8216;pong&#8217; message. The malware then sends information about your PC to, including:</p>
<ul xmlns="http://www.w3.org/1999/xhtml">
<li>The current logged on user name</li>
<li>PC name</li>
<li>Windows operating system version</li>
<li>Whether an antivirus is found or not</li>
<li>Whether or not IP is using Network Address Translation (NAT)</li>
<li>Bot version</li>
<li>Windows serial&#160;number</li>
</ul>
<p xmlns="http://www.w3.org/1999/xhtml">Below is an example of the HTTP request:</p>
<ul xmlns="http://www.w3.org/1999/xhtml">
<li><em>GET /fav.php?getcmd=1&#38;uid=&#60;user name&#62;&#38;cn=&#60;PC name&#62;:&#60;user name&#62;&#38;os=&#60;Windows operating system version&#62;&#38;av=&#60;Not+installed&#62;&#38;nat=yes&#38;version=2.0&#38;serial=&#60;Windows serial&#62;</em></li>
</ul>
<p xmlns="http://www.w3.org/1999/xhtml">The remote server sends encoded data, usually in base64 format, that contains commands for the malware. This can include instructions to:</p>
<ul xmlns="http://www.w3.org/1999/xhtml">
<li>Download and run files</li>
<li>Record which keys you press</li>
<li>Participate in <a href="https://www.microsoft.com/security/portal/mmpc/shared/glossary.aspx#dos" target="_blank">DoS</a> attacks&#160;</li>
<li>Update itself</li>
<li>Delete files and registry entries</li>
<li>Find files on your PC</li>
<li>Modify the system Hosts file</li>
<li>Visit a URL using a hidden desktop</li>
<li>Set the interval for retrieving commands from C&#38;C</li>
</ul>
<p xmlns="http://www.w3.org/1999/xhtml">We have seen the malware connect to following servers:</p>
<ul xmlns="http://www.w3.org/1999/xhtml">
<li><em><em>count&#60;removed&#62;.com</em></em></li>
<li><em><em>count&#60;removed&#62;.net</em></em></li>
<li><em><em>hotlog2&#60;removed&#62;.net</em></em></li>
<li><em><em>redtd&#60;removed&#62;.com</em></em></li>
<li><em><em>traficins&#60;removed&#62;.net</em></em></li>
</ul>

<p xmlns="http://www.w3.org/1999/xhtml"><strong>Checks for virtual environments</strong></p>
<p xmlns="http://www.w3.org/1999/xhtml">This malware checks if it's running in virtualized, sandbox environment by looking for processes and analysis tools such as: &#160;</p>

<li>Sandbox</li>

<p xmlns="http://www.w3.org/1999/xhtml">If any of the above checks is true the malware will stop running and exit.</p>
