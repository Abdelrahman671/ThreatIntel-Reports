			<h1 class="entry-title">WebLogic RCE Leads to XMRig</h1>
		<div class="entry-meta">
			<span class="posted-on"><a href="https://thedfirreport.com/2021/06/03/weblogic-rce-leads-to-xmrig/" rel="bookmark"><time class="entry-date published" datetime="2021-06-03T00:57:26+00:00">June 3, 2021</time></a></span>		</div><!-- .entry-meta -->

	<div class="entry-content">
		
<h3>Intro</h3>
<p>This report will review an intrusion where, the threat actor took advantage of a WebLogic remote code execution vulnerability (CVE-2020–14882) to gain initial access to the system before installing a coin miner (XMRig).</p>



<h3>Summary </h3>



<p>In the attack, the threat actor took around 2 minutes from initial access, to running a persistent coin miner. The speed and number of times the exploit was ran against the target make it likely that this was an automated attack, rather than human operated intrusion.</p>
<p>The threat actor exploited CVE-2020-14882 by making a request to the WebLogic server, which allowed the attacker to execute code. The payload then ran a PowerShell command to download a PowerShell script from a remote server. This script downloaded a loader and executed it, then it enabled persistence mechanisms. Next, the loader downloaded XMRig, and started the mining process.</p>
<p>While this attack took place around a month ago the payloads used in the intrusion continue to be hosted as of this publication, indicating that this threat is still active and finding vulnerable WebLogic hosts to continue exploiting.</p>
<p>&nbsp;</p>
<p>Analysis and reporting completed by <a href="https://twitter.com/tas_kmanager" target="_blank" rel="noopener">@tas_kmanager</a> and <a href="https://twitter.com/TheDFIRReport" target="_blank" rel="noopener">@TheDFIRReport</a></p>
<p>Reviewed by <a href="https://twitter.com/iiamaleks" target="_blank" rel="noopener">@iiamaleks</a> and <a href="https://twitter.com/samaritan_o" target="_blank" rel="noopener">@samaritan_o</a></p>
<p>&nbsp;</p>







<h3>MITRE ATT&amp;CK</h3>



<h3>Initial Access </h3>



<p>Before jumping into the details of the attack, it is pertinent to understand CVE-2020-14882. This vulnerability was discovered back in October 2020, which effects Oracle’s WebLogic product. [<a href="https://www.oracle.com/security-alerts/cpuoct2020traditional.html" target="_blank" rel="noopener">1</a>] We previously <a href="https://thedfirreport.com/2020/11/12/cryptominers-exploiting-weblogic-rce-cve-2020-14882/" target="_blank" rel="noopener">reported</a> on an intrusion that also used this exploit and ended in a coin miner. This vulnerability is easy to exploit, and leads to remote code execution (RCE) without authentication.  </p>



<p>Just by invoking URL path <em>/console/images/%252E%252E%252Fconsole.portal</em> and a little bit of tweaking of the packet sent to the server, the threat actor will be able to run their code on the server.</p>
<p>In this case, we can clearly see from network logs that the attacker was probing the portal page and followed that by sending the crafted URL request.</p>



<figure class="wp-block-image size-large"><img fetchpriority="high" decoding="async" class="alignnone size-full wp-image-3156" src="https://thedfirreport.com/wp-content/uploads/2021/06/2-WireSharkTImeline.png" alt="" width="3171" height="987" srcset="https://thedfirreport.com/wp-content/uploads/2021/06/2-WireSharkTImeline.png 3171w, https://thedfirreport.com/wp-content/uploads/2021/06/2-WireSharkTImeline-300x93.png 300w, https://thedfirreport.com/wp-content/uploads/2021/06/2-WireSharkTImeline-1024x319.png 1024w, https://thedfirreport.com/wp-content/uploads/2021/06/2-WireSharkTImeline-768x239.png 768w, https://thedfirreport.com/wp-content/uploads/2021/06/2-WireSharkTImeline-1536x478.png 1536w, https://thedfirreport.com/wp-content/uploads/2021/06/2-WireSharkTImeline-2048x637.png 2048w" sizes="(max-width: 3171px) 100vw, 3171px" /></figure>





<p>The detailed payload in the header can be observed below:</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2021/05/3580-11.png"><img decoding="async" class="alignnone size-full wp-image-3137" src="https://thedfirreport.com/wp-content/uploads/2021/05/3580-11.png" alt="" width="1332" height="241" srcset="https://thedfirreport.com/wp-content/uploads/2021/05/3580-11.png 1332w, https://thedfirreport.com/wp-content/uploads/2021/05/3580-11-300x54.png 300w, https://thedfirreport.com/wp-content/uploads/2021/05/3580-11-1024x185.png 1024w, https://thedfirreport.com/wp-content/uploads/2021/05/3580-11-768x139.png 768w" sizes="(max-width: 1332px) 100vw, 1332px" /></a></p>





<p>Interestingly, we can see the threat actor chose the <em>FileSystemXmlApplicationContext </em>class to carry out this attack, executing a file named <em>poc.xml</em>. This technique is listed in PoC developed by Jang. [<a href="https://github.com/jas502n/CVE-2020-14882#filesystemxmlapplicationcontext" target="_blank" rel="noopener">2</a>] </p>



<p>Below we can see the payload inside of the xml file hosted in the attacker server. The payload will execute the PowerShell command on the vulnerable system.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2021/05/3580-10.png"><img decoding="async" class="alignnone size-full wp-image-3136" src="https://thedfirreport.com/wp-content/uploads/2021/05/3580-10.png" alt="" width="1054" height="225" srcset="https://thedfirreport.com/wp-content/uploads/2021/05/3580-10.png 1054w, https://thedfirreport.com/wp-content/uploads/2021/05/3580-10-300x64.png 300w, https://thedfirreport.com/wp-content/uploads/2021/05/3580-10-1024x219.png 1024w, https://thedfirreport.com/wp-content/uploads/2021/05/3580-10-768x164.png 768w" sizes="(max-width: 1054px) 100vw, 1054px" /></a></p>





<h3>Execution </h3>



<p>The threat actor leveraged the WebLogic vulnerability to spawn a command shell from the server running in the Java process, which then in turn, was used to run PowerShell and collect the final payload that the threat actor wished to run on the system.</p>
<p><img loading="lazy" decoding="async" class="alignnone size-full wp-image-3161" src="https://thedfirreport.com/wp-content/uploads/2021/06/WebLogic-RCE-Leads-to-XMRig.png" alt="" width="1377" height="721" srcset="https://thedfirreport.com/wp-content/uploads/2021/06/WebLogic-RCE-Leads-to-XMRig.png 1377w, https://thedfirreport.com/wp-content/uploads/2021/06/WebLogic-RCE-Leads-to-XMRig-300x157.png 300w, https://thedfirreport.com/wp-content/uploads/2021/06/WebLogic-RCE-Leads-to-XMRig-1024x536.png 1024w, https://thedfirreport.com/wp-content/uploads/2021/06/WebLogic-RCE-Leads-to-XMRig-768x402.png 768w" sizes="auto, (max-width: 1377px) 100vw, 1377px" /></p>



<p>Next, let us investigate the content of the PowerShell script (ldr.ps1) </p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2021/05/3580-2.png"><img loading="lazy" decoding="async" class="alignnone wp-image-3104 size-full" src="https://thedfirreport.com/wp-content/uploads/2021/05/3580-2.png" alt="" width="645" height="63" srcset="https://thedfirreport.com/wp-content/uploads/2021/05/3580-2.png 645w, https://thedfirreport.com/wp-content/uploads/2021/05/3580-2-300x29.png 300w" sizes="auto, (max-width: 645px) 100vw, 645px" /></a></p>





<p>On line one, we can see var $cc (can be pronounced as C2?) that indicates the server where it will get all the other components such as the payload <em>sys.exe.</em> The next line sets up random characters to be appended to the payload executable.</p>



<p>This binary (renamed to <em>sysvr013.exe</em>) was then executed by the same script.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2021/05/3580-3.png"><img loading="lazy" decoding="async" class="alignnone size-full wp-image-3105" src="https://thedfirreport.com/wp-content/uploads/2021/05/3580-3.png" alt="" width="544" height="71" srcset="https://thedfirreport.com/wp-content/uploads/2021/05/3580-3.png 544w, https://thedfirreport.com/wp-content/uploads/2021/05/3580-3-300x39.png 300w" sizes="auto, (max-width: 544px) 100vw, 544px" /></a></p>



<h3>Persistence </h3>



<p>There are several persistence mechanisms observed. The first one utilizes schtasks to create a task called “<em>BrowserUpdate</em>”. </p>
<pre>schtasks /create /F /sc minute /mo 1 /tn "BrowserUpdate" /tr "$dst"</pre>





<p>It also created a registry run key to execute the miner binary on reboot.</p>





<p>We also observed the Loader binary adding npf.sys (WinPcap binary) as a scheduled task.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2021/05/3580-4-1.png"><img loading="lazy" decoding="async" class="alignnone size-full wp-image-3121" src="https://thedfirreport.com/wp-content/uploads/2021/05/3580-4-1.png" alt="" width="1326" height="144" srcset="https://thedfirreport.com/wp-content/uploads/2021/05/3580-4-1.png 1326w, https://thedfirreport.com/wp-content/uploads/2021/05/3580-4-1-300x33.png 300w, https://thedfirreport.com/wp-content/uploads/2021/05/3580-4-1-1024x111.png 1024w, https://thedfirreport.com/wp-content/uploads/2021/05/3580-4-1-768x83.png 768w" sizes="auto, (max-width: 1326px) 100vw, 1326px" /></a></p>





<h3>Privilege Escalation </h3>



<p>No privilege escalation was observed, but note that the Java.exe process was running as a <strong>high</strong> integrity process from the start.  </p>





<h3>Defense Evasion </h3>



<p>From the PowerShell script above (ldr.ps1), we can see that name is being randomized to avoid detection by blue team. We can also see that the attacker disabled firewall rules to make sure connection to the mining pool was not blocked.<strong> </strong> </p>





<pre>netsh advfirewall set allprofiles state off</pre>
<p>As noted above, the attacker created a task called “<em>BrowserUpdate</em>” to masquerade the scheduled task as a web browser update task. </p>





<h3>Discovery </h3>



<p>We can see several local discovery commands executed from the PowerShell script.  They are looking for existing mining process on the machine, such as <em>kthreaddi</em>, <em>sysrv</em>, etc. They also utilize netstat to find if any process is using mining related ports, such as 3333, 4444, etc.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2021/05/3580-5.png"><img loading="lazy" decoding="async" class="alignnone size-full wp-image-3124" src="https://thedfirreport.com/wp-content/uploads/2021/05/3580-5.png" alt="" width="1007" height="182" srcset="https://thedfirreport.com/wp-content/uploads/2021/05/3580-5.png 1007w, https://thedfirreport.com/wp-content/uploads/2021/05/3580-5-300x54.png 300w, https://thedfirreport.com/wp-content/uploads/2021/05/3580-5-768x139.png 768w, https://thedfirreport.com/wp-content/uploads/2021/05/3580-5-1000x182.png 1000w" sizes="auto, (max-width: 1007px) 100vw, 1007px" /></a></p>
<p>This is likely partially to kill other rival miners but also to prevent duplicate miners on the same host as the exploit was executed against the target repeatedly in quick succession, at least 12 times.</p>
<p>

</p>





<p><a href="https://thedfirreport.com/wp-content/uploads/2021/05/3580-7.png"><img loading="lazy" decoding="async" class="alignnone size-full wp-image-3126" src="https://thedfirreport.com/wp-content/uploads/2021/05/3580-7.png" alt="" width="1140" height="486" srcset="https://thedfirreport.com/wp-content/uploads/2021/05/3580-7.png 1140w, https://thedfirreport.com/wp-content/uploads/2021/05/3580-7-300x128.png 300w, https://thedfirreport.com/wp-content/uploads/2021/05/3580-7-1024x437.png 1024w, https://thedfirreport.com/wp-content/uploads/2021/05/3580-7-768x327.png 768w" sizes="auto, (max-width: 1140px) 100vw, 1140px" /></a></p>











<h3>Impact </h3>



<p>We saw that the additional <em>sys.exe</em> binary and <em>ldr.ps1</em> PowerShell script was brought in to support the Mining operation from IP address <em>194[.]145[.]227[.]21</em> </p>
<p>

</p>
<p>We also observed connection from [kthreaddi].exe talking to crypto mining pools</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2021/05/3580-8.png"><img loading="lazy" decoding="async" class="alignnone size-full wp-image-3130" src="https://thedfirreport.com/wp-content/uploads/2021/05/3580-8.png" alt="" width="755" height="334" srcset="https://thedfirreport.com/wp-content/uploads/2021/05/3580-8.png 755w, https://thedfirreport.com/wp-content/uploads/2021/05/3580-8-300x133.png 300w" sizes="auto, (max-width: 755px) 100vw, 755px" /></a></p>
<p>Eventually, the crypto mining operation will cause performance issue with the WebLogic server.</p>
