<p>On May 25, 2021, Volexity identified a phishing campaign targeting multiple organizations based in the United States and Europe. The following industries have been observed being targeted thus far:</p>
<ul>
<li>NGOs</li>
<li>Research Institutions</li>
<li>Government Agencies</li>
<li>International Agencies</li>
</ul>
<p>The campaign’s phishing e-mails purported to originate from the USAID government agency and contained a malicious link that resulted in an ISO file being delivered. This file contained a malicious LNK file, a malicious DLL file, and a legitimate lure referencing foreign threats to the 2020 US Federal Elections.</p>
<p>This blog post provides details on the observed activity and outlines possible justification that this campaign could be related to APT29.</p>
<h2>Phishing Email Campaign</h2>
<p>The original e-mails looked like the following:</p>
<p style="text-align: center;"><img decoding="async" class="aligncenter wp-image-2126 size-large" src="https://www.volexity.com/wp-content/uploads/2021/05/phish_email-1024x817.png" alt="" width="1024" height="817" srcset="https://www.volexity.com/wp-content/uploads/2021/05/phish_email-1024x817.png 1024w, https://www.volexity.com/wp-content/uploads/2021/05/phish_email-300x239.png 300w, https://www.volexity.com/wp-content/uploads/2021/05/phish_email-768x613.png 768w, https://www.volexity.com/wp-content/uploads/2021/05/phish_email.png 1187w" sizes="(max-width: 1024px) 100vw, 1024px" /></p>
<p style="text-align: center;"><em>Figure 1. Phishing e-mails sent to numerous organizations</em></p>
<p>Volexity also observed a smaller campaign from the same sender with largely the same content several hours earlier, but with the subject line "<strong>USAID Special Alert!</strong>". Most of the hyperlinks in the e-mail are of the following format.</p>
<blockquote><p>https://r20.rs6[.]net/tn.jsp?f=001R6x5duwxLa513iT3wolVtyZj3Ojypr9nwPwZKB3X68SGRFzUVNUR4MdENUXj_c4poo1hx_rFF79P1NsazE-FONIrA9G0ypkCwKTRfL95fp3xUyuceYYrPAtcDp20R1wmw-XZ197ks1FH22V3BIcZYlAfIHdUZQ3M&amp;c=[Random Data]&amp;ch=[Random Data]&amp;__=[Victim Email]</p></blockquote>
<p>While each link was consistent in a given e-mail, parts of the e-mail varied.</p>
<p>The “rs6.net” domain is used by Constant Contact e-mail marketing software to track click-throughs on links, meaning the attacker should be able to map the success of their campaign. On the newsletter page of USAID's website, their sign-up link is through Constant Contact. As a result of using this software, the attacker is able to generate a more convincing spear-phish e-mail. Additionally, the e-mail appears to originate from USAID; however, news-related e-mails from this organization are traditionally sent via the <a href="/cdn-cgi/l/email-protection" class="__cf_email__" data-cfemail="2c5c5e495f5f6c595f4d4548024b435a">[email&#160;protected]</a> e-mail address.</p>
<p>When a recipient clicks the Constant Contact URL referenced above, their network request will be redirected to the following URL:</p>
<blockquote><p>https://usaid.theyardservice[.]com/d/[Victim Email]</p></blockquote>
<p>This network request will initiate a download for the malicious ISO file “ICA-declass.iso” to the victim's system. The same file is delivered to every user irrespective of their e-mail or referring URL.</p>

<p>The PDF file appears to have been pulled directly from the <a href="https://www.dni.gov/index.php/newsroom/reports-publications/reports-publications-2021/item/2192-intelligence-community-assessment-on-foreign-threats-to-the-2020-u-s-federal-elections">dni.gov website</a> and acts as a decoy; its contents are shown in Figure 2.</p>
<p><img decoding="async" class="alignnone size-full wp-image-2122 aligncenter" src="https://www.volexity.com/wp-content/uploads/2021/05/image002.png" alt="" width="809" height="1051" srcset="https://www.volexity.com/wp-content/uploads/2021/05/image002.png 809w, https://www.volexity.com/wp-content/uploads/2021/05/image002-231x300.png 231w, https://www.volexity.com/wp-content/uploads/2021/05/image002-788x1024.png 788w, https://www.volexity.com/wp-content/uploads/2021/05/image002-768x998.png 768w" sizes="(max-width: 809px) 100vw, 809px" /></p>

<p>If a user opened the embedded LNK file, it would run the Document.dll file and use its exported function “Open”.</p>
<p><img loading="lazy" decoding="async" class="alignnone size-full wp-image-2123 aligncenter" src="https://www.volexity.com/wp-content/uploads/2021/05/image003.png" alt="" width="938" height="360" srcset="https://www.volexity.com/wp-content/uploads/2021/05/image003.png 938w, https://www.volexity.com/wp-content/uploads/2021/05/image003-300x115.png 300w, https://www.volexity.com/wp-content/uploads/2021/05/image003-768x295.png 768w" sizes="auto, (max-width: 938px) 100vw, 938px" /></p>

<p>While the PDB string contains the Korean word for “develop,” Volexity does not believe Korean-speaking threat actors or developers are responsible for this malware family. Volexity instead believes this to be a false flag. Additionally, the compile timestamp dating to the year 2019 is likely to have been falsified.</p>
<p>The DLL is equipped with a number of anti-sandbox and anti-vm checks based on the presence of registry keys commonly found in Virtual Machine environments, as shown in Figure 4:</p>
<p><img loading="lazy" decoding="async" class="alignnone size-full wp-image-2124 aligncenter" src="https://www.volexity.com/wp-content/uploads/2021/05/image004.png" alt="" width="648" height="468" srcset="https://www.volexity.com/wp-content/uploads/2021/05/image004.png 648w, https://www.volexity.com/wp-content/uploads/2021/05/image004-300x217.png 300w" sizes="auto, (max-width: 648px) 100vw, 648px" /></p>
<p style="text-align: center;"><em>Figure 4. Decompiled virtual machine checks found within the malicious DLL</em></p>

<p>The exported function "WaitPrompt" of this DLL launches the malicious behavior. The malware begins by attempting to open a mutex with the name "UlswcXJJWhtHIHrVqWJJ". If this mutex already exists on the system the malware will exit, otherwise it will proceed. The malware then queries the file attributes of the file "C:\dell.sdr" and appends data from these file attributes to the above mutex name. This combined string is then hashed with MD5 and used to generate a Triple DES decryption key.</p>
<p style="text-align: center;"><img loading="lazy" decoding="async" class="size-full wp-image-2150 aligncenter" src="https://www.volexity.com/wp-content/uploads/2021/05/figure5.png" alt="" width="1114" height="398" srcset="https://www.volexity.com/wp-content/uploads/2021/05/figure5.png 1114w, https://www.volexity.com/wp-content/uploads/2021/05/figure5-300x107.png 300w, https://www.volexity.com/wp-content/uploads/2021/05/figure5-1024x366.png 1024w, https://www.volexity.com/wp-content/uploads/2021/05/figure5-768x274.png 768w" sizes="auto, (max-width: 1114px) 100vw, 1114px" /></p>
<p style="text-align: center;">Figure 5. Encryption routine leveraged by the malware</p>
<p>The sample then uploads a timestamp to <a href="https://firebase.google.com/" target="_blank" rel="noopener">Firebase</a> and downloads a blob from Firebase storage. This data is base64 decoded and decrypted using the generated key. Then, the data is executed in a separate thread, and an HTTP DELETE request is sent to the Firebase storage address used to download the payload.</p>
<p style="text-align: center;"><img loading="lazy" decoding="async" class="size-full wp-image-2151 aligncenter" src="https://www.volexity.com/wp-content/uploads/2021/05/figure6.png" alt="" width="1382" height="306" srcset="https://www.volexity.com/wp-content/uploads/2021/05/figure6.png 1382w, https://www.volexity.com/wp-content/uploads/2021/05/figure6-300x66.png 300w, https://www.volexity.com/wp-content/uploads/2021/05/figure6-1024x227.png 1024w, https://www.volexity.com/wp-content/uploads/2021/05/figure6-768x170.png 768w" sizes="auto, (max-width: 1382px) 100vw, 1382px" /><br />

<p>Volexity was able to capture an encrypted payload from the Firebase URL and are currently in the process of analyzing it.</p>
<h2>Attribution</h2>
<p>While Volexity cannot say with certainty who is behind these attacks, it does believe it has the earmarks of a known threat actor it has dealt with on several previous occasions. However, a number of attack attributes are consistent with previous tactics used by APT29:</p>
<ul>
<li>The use of an <a href="https://www.fireeye.com/blog/threat-research/2018/11/not-so-cozy-an-uncomfortable-examination-of-a-suspected-apt29-phishing-campaign.html">archive file format containing an LNK</a> to deliver the initial payload (2018)</li>
<li>The use of a <a href="https://www.virustotal.com/gui/file/2d2fa32f928f8abf31b9e79153422d65fe72cd5ad0d1f815a9d2ffa42fc8d224/relations">US election-themed lure document</a> sent from a spoofed US government source address (2016)</li>
<li>The use of <a href="https://www.fireeye.com/blog/threat-research/2018/11/not-so-cozy-an-uncomfortable-examination-of-a-suspected-apt29-phishing-campaign.html">CobaltStrike with a custom malleable profile</a> as an initial payload (2018)</li>
<li>The relatively widespread nature of the campaign, with many targets receiving the same spear phishing content at the same time</li>
</ul>
<p>Notably, in the 2018 case, FireEye highlighted the same MAC address had been used to create the LNKs observed in 2016 and 2018, while the newest LNK detailed in this blog post has had this metadata scrubbed. This is perhaps an indication that the attacker is learning from public reports on their work.</p>
<div>From an infrastructure point of view, the domains used bear some similarity to the Dark Halo campaign reported by Volexity. In the case of Dark Halo, domains were bought at auction or through marketplace transactions which meant they appeared to be created long ago in WHOIS records. This is the case again with the domains used for command and control in these attacks. Following the Volexity publication, it has been alleged that the Dark Halo campaign was also the work of APT29; however, Volexity has not reached that conclusion at this time.</div>
<p>Volexity cannot be completely certain this new activity is the work of APT29, but it is believed with moderate confidence that it is.</p>
<h2>Conclusion</h2>
<p>Volexity believes the APT29 threat actor is likely responsible for a phishing campaign against numerous organizations within the United States and Europe. It is currently unclear how many organizations have been targeted, but several of Volexity's customers—as well as a number of organizations submitting to VirusTotal—have been attacked.</p>
<p>After a relatively long hiatus with no publicly detailed spear phishing activity, APT29 appears to have returned with only slight changes to its historical TTPs. In this instance, the attacker purports to be from USAID, enticing victims into clicking an embedded file to download and execute a malicious ISO file. In doing so, the CobaltStrike Beacon implant is executed, providing remote access to the attackers.</p>
<p>At the time of writing, all files involved have relatively low static detection rates on VirusTotal. This suggests the attacker is likely having some success in breaching targets.</p>
<p>Organizations are encouraged to perform the following actions to protect against this threat:</p>
<ul>
<li>Block the following network indicators identified as part of this phishing campaign:
<ul>
<li>theyardservice[.]com</li>
<li>worldhomeoutlet[.]com</li>
<li>83.171.237.173</li>
<li>192.99.221.77</li>
<li>refreshauthtoken-default-rtdb[.]firebaseio.com</li>
</ul>
</li>
