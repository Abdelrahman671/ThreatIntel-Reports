<p>SafeBreach Labs researchers are constantly monitoring the hacker underground, sourcing intelligence feeds, and conducting original research to uncover new threats and ensure our Hacker&rsquo;s Playbook provides the most comprehensive collection of attacks. As part of this ongoing effort, we recently discovered a new targeted attack we believe is compelling for four main reasons:&nbsp;</p>



<ol class="wp-block-list">
<li>It appears to target Farsi-speaking code developers by using a Microsoft Word document that includes a Microsoft Dynamic Data Exchange (DDE) exploit.</li>



<li>It leverages a previously undiscovered remote access trojan (RAT)&mdash;dubbed CodeRAT by SafeBreach Labs researchers&mdash;that supports ~50 commands.</li>



<li>We were able to identify the developer of CodeRAT who, after being confronted by us, decided to publish the source code of CodeRAT in his public GitHub account.</li>



<li>CodeRAT is using a unique exfiltration and command and control mechanism. Instead of using a dedicated C2 server, CodeRAT is using a public anonymous file upload API.&nbsp;&nbsp;&nbsp;</li>
</ol>



<p>In this research report, we will provide a high-level overview of CodeRAT, including when it first appeared, what it does, the type of communications it uses, and who might be behind it. We&rsquo;ll also provide a deep-dive into the technical details behind the RAT, including its operational modes and available commands. Finally, we&rsquo;ll provide insight into our conversation with the developer of CodeRAT and details about how SafeBreach is sharing this information with the security community.&nbsp;</p>



<h1 class="wp-block-heading"><strong>CodeRAT Overview&nbsp;</strong></h1>



<p>For initial access, the threat actor uses a Microsoft Word document that includes a DDE exploit, a <a href="https://www.sentinelone.com/blog/malware-embedded-microsoft-office-documents-dde-exploit-macroless/" target="_blank" rel="noreferrer noopener">well-known technique</a> used by threat actors to deliver malicious code within a macro in the document. The document used in this attack contains information regarding hardware design languages &#8203;&#8203;like Verilog and very high-speed integrated circuit hardware description language (VHDL).</p>


<div class="wp-block-image is-style-default">
<figure class="aligncenter size-full"><img fetchpriority="high" decoding="async" width="752" height="426" src="https://www.safebreach.com/wp-content/uploads/2022/08/Figure-1.png" alt="" class="wp-image-104170" srcset="https://www.safebreach.com/wp-content/uploads/2022/08/Figure-1.png 752w, https://www.safebreach.com/wp-content/uploads/2022/08/Figure-1-300x170.png 300w" sizes="(max-width: 752px) 100vw, 752px"></figure></div>


<p class="has-text-align-center"><em>Figure 1: Sample of content in Word document used in attack&nbsp;</em></p>



<p>The file, named <em>432gsbse5, </em>was first uploaded to the <a href="https://raw.githubusercontent.com/alberfrancis/camo/main/432gsbse5" target="_blank" rel="noreferrer noopener">alberfrancis GitHub repository</a> on April 22, 2022&mdash;the exploit downloads and executes CodeRAT from this repository. The file was updated on July 10, 2022, and subsequently deleted and uploaded again 15 times by the threat actor.</p>


<div class="wp-block-image is-style-default">
<figure class="aligncenter size-full"><img decoding="async" width="936" height="382" src="https://www.safebreach.com/wp-content/uploads/2022/08/Figure-2.png" alt="" class="wp-image-104184" srcset="https://www.safebreach.com/wp-content/uploads/2022/08/Figure-2.png 936w, https://www.safebreach.com/wp-content/uploads/2022/08/Figure-2-300x122.png 300w, https://www.safebreach.com/wp-content/uploads/2022/08/Figure-2-768x313.png 768w" sizes="(max-width: 936px) 100vw, 936px"></figure></div>


<p class="has-text-align-center"><em>Figure 2: The albertfrancis GitHub repository, including two versions of the RAT and two libraries</em></p>



<p>This exploits document was first submitted to VirusTotal from Iran on July 5, 2022.&nbsp;</p>


<div class="wp-block-image is-style-default">
<figure class="aligncenter size-full"><img decoding="async" width="548" height="298" src="https://www.safebreach.com/wp-content/uploads/2022/08/Figure-3.png" alt="" class="wp-image-104185" srcset="https://www.safebreach.com/wp-content/uploads/2022/08/Figure-3.png 548w, https://www.safebreach.com/wp-content/uploads/2022/08/Figure-3-300x163.png 300w" sizes="(max-width: 548px) 100vw, 548px"></figure></div>


<p class="has-text-align-center"><em>Figure 3: VirusTotal submission</em></p>



<p>Once executed, the main goal of CodeRAT is to monitor the victim&rsquo;s activity on social networks and on local machines. The monitoring capabilities include almost 50 commands and allow the attacker to monitor webmail, Microsoft Office documents, databases, social networks, games, <a href="https://www.techtarget.com/searchsoftwarequality/definition/integrated-development-environment#:~:text=An%20IDE%20typically%20contains%20a,is%20executable%20for%20a%20computer." target="_blank" rel="noreferrer noopener">integrated development environments</a> (IDEs) for Windows and Android, and pornographic sites. Moreover, CodeRAT monitors a large number of browser window titles, two of which are unique to Iranian victims: a popular Iranian e-commerce site and a web messenger in Farsi.</p>



<p>This type of monitoring&mdash;specifically of pornographic sites, use of anonymous browsing tools, and social network activities&mdash;leads us to believe CodeRAT is an intelligence tool used by a threat actor tied to a government. It is commonly seen in attacks operated by the Islamic regime of Iran to monitor illegal/immoral activities of their citizens.&nbsp;</p>



<p>The communication methods of CodeRAT are versatile and quite unique. CodeRAT supports communication over Telegram groups using the bot API or through USB flash drive. It can also act in silent mode, which includes no report back. CodeRAT uses an anonymous, public uploading site, rather than a dedicated C2 server, and uses anti-detection techniques to limit its usage to 30 days. In addition, it will use the HTTP Debugger website as a proxy to communicate with its C2 Telegram group.&nbsp;</p>



<h1 class="wp-block-heading"><strong>CodeRAT Detailed Analysis</strong></h1>



<p><strong>Operation Modes</strong></p>



<p>CodeRAT has five modes of operation derived from a command line argument:</p>



<ol class="wp-block-list">
<li><strong>&ldquo;father&rdquo;</strong> &ndash; Get a process ID (PID) from a second command line argument, then kill it and start it with the &ldquo;continue&rdquo; command line argument.&nbsp;</li>



<li><strong>&ldquo;Continue&rdquo;</strong> &ndash; Get a PID from a second command line argument, then kill it and delete its <em>.exe</em>, <em>.pdb</em>, and <em>.exe.config</em> files.</li>



<li><strong>&ldquo;Word&rdquo;</strong> &ndash; Check if the last modified date of the RAT binary is below 30 days.</li>
</ol>


<div class="wp-block-image is-style-default">
<figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="758" height="122" src="https://www.safebreach.com/wp-content/uploads/2022/08/Figure-4-1.png" alt="" class="wp-image-104188" srcset="https://www.safebreach.com/wp-content/uploads/2022/08/Figure-4-1.png 758w, https://www.safebreach.com/wp-content/uploads/2022/08/Figure-4-1-300x48.png 300w" sizes="auto, (max-width: 758px) 100vw, 758px"></figure></div>


<p class="has-text-align-center"><em>Figure 4: Compile time anti-detection technique</em></p>



<p>It will copy itself to <em>%appdata%\desktopmgr.exe</em>. If it fails to copy, it will copy itself to <em>myPictures\deskmgr.exe</em>. If the copy works, it will execute it with the &ldquo;Wordbetraied&rdquo; argument (below) and its own working directory path as a second argument.</p>



<ol class="wp-block-list" start="4">
<li><strong>&ldquo;Wordbetraied&rdquo;</strong> &ndash; Download <em>Aspose.Words.dll</em> from the same GitHub repository and check if a file <em>pass.exe</em> exists in the directory received in the second command line argument.
<ol class="wp-block-list">
<li>If the <em>pass.exe</em> file does not exist, it will try to delete files received in second and third arguments.&nbsp;</li>



<li>If the <em>pass.exe</em> file exists, it will search all the <em>.docx</em> files in his current directory. For each <em>.docx</em> file, it will rename it to <em>working.docx</em> and will use the LoadOptions class exported by the <em>Aspose.Words.dll</em> to load the document into a <a href="https://reference.aspose.com/words/net/aspose.words/document" target="_blank" rel="noreferrer noopener">Document</a> object and set a password from the <em>pass.txt</em> file on the document. Next, it will search the winword process by enumerating all processes and searching for a process with a Window title that contains the name of the <em>.docx</em> file. It will delete the <em>.docx</em> file and save the object with the password to a new file. Then it will terminate the winword process and start the new Word with the password file.</li>
</ol>
</li>



<li>If none of the four command arguments was used, it will execute a file with the same name but that ends with <em>.exe.bak</em>.<br><br>CodeRAT generates a unique ID for each victim with this formula:<br><em>from cpu_id(13) + cpu_id(1,4) + the hard drive volume serial number + cpu_id(4)&nbsp;</em></li>
</ol>



<p><strong>Commands</strong></p>



<p>Commands can be received in three methods:</p>



<ol class="wp-block-list">
<li><strong>Local file</strong> &ndash; CodeRAT will check if the file <em>command.txt</em> exists under <em>myPictures</em> folder. The content should end with &ldquo;EOF&rdquo;.<br><br>If it exists, it will read the last command before &ldquo;EOF&rdquo;. If it&rsquo;s equal to &ldquo;silence&rdquo;, it won&rsquo;t report back; if it&rsquo;s not silent, it will use the usbFlash to report.<br><br>Supported USB commands are: flashextentioncopy, flashcopyfilelist, flashcopyfolderlist (see details in the next section).</li>



<li><strong>Manual UI</strong> &ndash; CodeRAT will get the command from the main UI window (see details in the next section) and use usbFlash to report back by copying the exfiltrated data to the USB. The USB drive letter will be received from a combobox in the UI. There are two buttons: one will hide the UI and one doesn&rsquo;t hide it.&nbsp;</li>



<li><strong>Telegram bot API</strong> &ndash; CodeRAT will use getUpdates Telegram bot API to get messages/commands and for exfiltration. An interesting feature is that it uses a proxy instead of directly querying the Telegram bot API. The proxy used is: <a href="http://www.httpdebugger.com" target="_blank" rel="noreferrer noopener">www.httpdebugger.com</a></li>
</ol>

<p>CodeRAT parses the HTML response of the HTTP Debugger proxy and extracts the original response of the Telegram bot API:</p>



<p><em>&lt;h3 class=&rsquo;ls1 t400 h2long&rsquo;&gt;Response Content&lt;/h3&gt;&lt;pre class=&rsquo;brush: html; toolbar: false; wrap-lines: true;&rsquo;&gt;{&ldquo;ok&rdquo;:true,&rdquo;result&rdquo;:[]}&lt;/pre&gt;</em><br><br>Each message will be between brackets and contain at least one &ldquo;-&rdquo;, which is the separator between messages.</p>



<p>If the message is the MD5 of * or includes the unique ID of the attackers machine, it will upload files, screen captures, and thumbnail images using the public anonymous file upload API: <a href="https://api.anonfile.com/upload" target="_blank" rel="noreferrer noopener">https://api.anonfile.com/upload</a></p>

<p><strong>CodeRAT &ldquo;boss&rdquo; Mode</strong></p>



<p>CodeRAT will check for &ldquo;boss&rdquo; mode every two seconds.</p>


<div class="wp-block-image is-style-default">
<figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="572" height="430" src="https://www.safebreach.com/wp-content/uploads/2022/08/Figure-10.png" alt="" class="wp-image-104194" srcset="https://www.safebreach.com/wp-content/uploads/2022/08/Figure-10.png 572w, https://www.safebreach.com/wp-content/uploads/2022/08/Figure-10-300x226.png 300w" sizes="auto, (max-width: 572px) 100vw, 572px"></figure></div>


<p class="has-text-align-center"><em>Figure 10: BossWatch calls CheckBoss function</em></p>

<h1 class="wp-block-heading"><strong>Conclusion&nbsp;</strong></h1>



<p>SafeBreach is passionate about improving security on a global level and, as an organization, we are committed to openly sharing our research with the broader security community. By sharing information specifically about our discovery of CodeRAT, our goal is to raise awareness about this new, unrecognized type of malware that leverages a relatively new technique of using an anonymous uploads site as a C2 server. Finally, we hope organizations and individuals can use the indicators of compromise (IOCs) and YARA rules provided in Appendix A to better detect and protect themselves against this threat.&nbsp;</p>
