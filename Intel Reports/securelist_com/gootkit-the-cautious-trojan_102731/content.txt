												<div class="c-wysiwyg">
													<p>Gootkit is complex multi-stage banking malware that was discovered for the first time by Doctor Web in 2014. Initially it was distributed via spam and exploits kits such as Spelevo and RIG. In conjunction with spam campaigns, the adversaries later switched to compromised websites where the visitors are tricked into downloading the malware.</p>
<p>Gootkit is capable of stealing data from the browser, performing man-in-the-browser attacks, keylogging, taking screenshots and lots of other malicious actions. Its loader performs various virtual machine and sandbox checks and uses sophisticated persistence algorithms. In 2019, Gootkit stopped operating after it experienced a <a href="https://www.zdnet.com/article/gootkit-malware-crew-left-their-database-exposed-online-without-a-password/" target="_blank" rel="noopener">data leak</a>, but has been <a href="https://www.bleepingcomputer.com/news/security/gootkit-malware-returns-to-life-alongside-revil-ransomware/" target="_blank" rel="noopener">active again</a> since November 2020.</p>
<p>Gootkit&#8217;s victims are mainly located in EU countries such as Germany and Italy. In this article we analyze a recent sample of Gootkit.</p>
<h2 id="technical-details">Technical Details</h2>
<p>Gootkit consists of a (down)loader component written in C++ and the main body written in JS and interpreted by Node.js. The main body is a modular framework, containing registration, spyware, VMX detection and other modules.</p>
<h3 id="loader">Loader</h3>
<p>The sample (MD5 <a href="https://opentip.kaspersky.com/97713132e4ea03422d3915bab1c42074/?utm_source=SL&amp;utm_medium=SL&amp;utm_campaign=SL" target="_blank" rel="noopener">97713132e4ea03422d3915bab1c42074</a>) is packed by a custom-made multi-stage packer which decrypts the final payload step by step. The last stage is a shellcode that decrypts the original loader executable and maps it into memory. After mapping, the original entry point is called. Hence, we can easily unpack the original executable and analyze it. We detect the Gootkit loader with the verdicts listed in the table below.</p>

Matching of the process name is done by calculating the CRC32 value of the process name. For a list of supported browsers, see the table below.</p>

<p>        The injected code is called from the main body web injection and traffic sniffing routines to perform a <a href="https://encyclopedia.kaspersky.com/glossary/man-in-the-browser-attack/?utm_source=securelist&amp;utm_medium=blog&amp;utm_campaign=termin-explanation" target="_blank" rel="noopener">man-in-the-browser attack</a>. To do so, the code patches standard browser functions responsible for certificate validation to allow self-signed certificates. As a result, attackers are able to inject custom JS code and modify or redirect traffic.</li>
<li><strong>Persistence_service</strong><br />
        If a sample is running under LOCAL_SYSTEM account, the Gootkit persistence mechanism abuses the pending GPO Windows feature. When a user modifies Pending GPO registry values, he/she has to specify the following parameters:</p>
<ul>
<li>count – count of pending GPOs;</li>
<li>path1, path2, … – path to the special .inf file that contains instructions on how to load GPO;</li>
<li>Section1, Section2, … – name of the section from the INF file.</li>
</ul>
<p>        So Gootkit creates an .inf file in the same directory as the sample and writes the following values to the Software\Microsoft\IEAK\GroupPolicy\PendingGPOs registry key:</p>
<ul>
<li>count – 0x1</li>
<li>path1 – .inf file location</li>
<li>Section1 – DefaultInstall</li>
</ul>

<p style="text-align: center"><strong><em>INF file content</em></strong></p>
<p>        Now explorer.exe will load the Group Policy Objects (GPO) whenever it is loaded. Gootkit creates a pending GPO for the Internet Explorer Administration Kit (IEAK), which points directly at the INF file. When explorer.exe is loaded at runtime, it will execute the [DefaultInstall] inside the created file, which will run the Gootkit executable.</p>
<p>        If the sample is running under another account, it creates a service with a random name chosen from %SystemRoot%, copies itself into the %SystemRoot% folder with the chosen name and deletes itself from the disk.</li>
<li><strong>Stop_switch</strong><br />
        The thread looks for a file named uqjckeguhl.tmp in the \AppData\Local\Temp and \Local Settings\Temp folders. When the file is found, the malware will stop.</li>
</ul>
<h4 id="main-body-download">Main body download</h4>
<p>Before downloading the main body from the C&amp;C, the loader tries to find registry keys with the following format: <em>HKCU\Software\AppDataLow\&lt;pr_string&gt;_&lt;i&gt;</em>, where i is a number starting from 0 and pr_string is a pseudo-random string generated when the bot starts. Generation is based on the victim&#8217;s PC parameters, so the same value is generated for the same PC each time.</p>
<p>Each key contains a maximum chunk of 512,000 bytes (500KB) of encrypted data. If the aforementioned keys were found, their contents will be saved in a newly allocated buffer (used for decryption and decompression). The buffer is then decrypted using the same function used for decrypting the configuration, after which the buffer is decompressed.</p>
<p>After the unpacking routine, the loader will download the main body from the C&amp;C, calculate its CRC32 and compare it with the registry payload CRC (if one exists). If the CRCs are different, the loader will execute the newer version downloaded from the C&amp;C. The C&amp;C server will not send the DLL module without the appropriate UserAgent header that is hardcoded into the sample. The current hardcoded value is: Mozilla/5.0 (Windows NT 6.1; Win64; x64; rv:25.0) Gecko/21006101 Firefox/25.0.</p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/03133138/Gootkit_banking_malware_05.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-102740" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/03133138/Gootkit_banking_malware_05.png" alt="" width="353" height="375" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/03133138/Gootkit_banking_malware_05.png 353w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/03133138/Gootkit_banking_malware_05-282x300.png 282w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/03133138/Gootkit_banking_malware_05-165x175.png 165w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/03133138/Gootkit_banking_malware_05-264x280.png 264w" sizes="(max-width: 353px) 100vw, 353px" /></a></p>

<h3 id="main-body">Main body</h3>
<p>The main body (MD5 <a href="https://opentip.kaspersky.com/20279d99ee402186d1e3a16d6ab9398a/?utm_source=SL&amp;utm_medium=SL&amp;utm_campaign=SL" target="_blank" rel="noopener">20279d99ee402186d1e3a16d6ab9398a</a>, verdict HEUR:Trojan.Win32.Generic) is a Node.js interpreter with bundled encrypted JS files. On startup, the main body decrypts the JavaScript files using an RC4-like algorithm with hardcoded keystream.</p>
<p>Information about the embedded modules is stored in an array of special file structures that have the following format: BYTE* name_pointer, BYTE* encrypted_data, DWORD data_size, DWORD encr_flag. These structures are used within the decryption routine that reads <strong>data_size</strong> bytes starting from <strong>encrypted_data</strong>. This routine decrypts <strong>encrypted_data</strong> if <strong>encr_flag</strong> is set and writes the result into a file with name *<strong>name_pointer</strong>. The decryption routine iterates over all entries in the file information array. Then the decryption execution is transferred to the Node.js interpreter.</p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/03134746/Gootkit_banking_malware_06.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-102741" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/03134746/Gootkit_banking_malware_06.png" alt="" width="624" height="227" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/03134746/Gootkit_banking_malware_06.png 624w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/03134746/Gootkit_banking_malware_06-300x109.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/03134746/Gootkit_banking_malware_06-481x175.png 481w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/03134746/Gootkit_banking_malware_06-370x135.png 370w" sizes="(max-width: 624px) 100vw, 624px" /></a></p>

<p>There are six types of internal event handlers and corresponding packet formats.</p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/03134817/Gootkit_banking_malware_07.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-102742" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/03134817/Gootkit_banking_malware_07.png" alt="" width="592" height="290" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/03134817/Gootkit_banking_malware_07.png 592w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/03134817/Gootkit_banking_malware_07-300x147.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/03134817/Gootkit_banking_malware_07-357x175.png 357w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/03134817/Gootkit_banking_malware_07-370x181.png 370w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/03134817/Gootkit_banking_malware_07-572x280.png 572w" sizes="(max-width: 592px) 100vw, 592px" /></a></p>
<p style="text-align: center"><strong><em>Event handlers</em></strong></p>
<p>The general packet structure is as follows:</p>
<ul>
<li>Length + 8 (4 bytes)</li>
<li>Packet magic (0xEDB88320 XOR length+8)</li>
<li>Packet data (different for each package type, serialized using protobuf)</li>
<li>Packet magic</li>
</ul>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/03134837/Gootkit_banking_malware_08.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-102743" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/03134837/Gootkit_banking_malware_08.png" alt="" width="576" height="326" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/03134837/Gootkit_banking_malware_08.png 576w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/03134837/Gootkit_banking_malware_08-300x170.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/03134837/Gootkit_banking_malware_08-309x175.png 309w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/03134837/Gootkit_banking_malware_08-370x209.png 370w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/06/03134837/Gootkit_banking_malware_08-495x280.png 495w" sizes="(max-width: 576px) 100vw, 576px" /></a></p>
<p style="text-align: center"><strong><em>Packet generation routine</em></strong></p>
<p>Kaspersky products detect this family as Trojan-Downloader.Win32.Injecter, HEUR:Trojan.Win32.Generic, Trojan-Downloader.Win32.Gootkit, Trojan-Banker.Win32.Gootkit. All the details, IoCs, MITRE ATT&amp;CK Framework data, Yara rules and hashes related to this threat are available to the users of our <a href="https://www.kaspersky.com/enterprise-security/threat-intelligence" target="_blank" rel="noopener">Financial Threat Intelligence services</a>.
