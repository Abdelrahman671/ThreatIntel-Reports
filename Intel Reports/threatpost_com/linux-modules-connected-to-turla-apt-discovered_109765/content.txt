																														<div class="c-article__intro" name="overview">
											<p>Researchers at Kaspersky Lab have found two Linux modules connected to the Turla APT campaigns. </p>
										</div>
																				<div class="c-article__content js-reading-content">
											<p>The <a href="https://threatpost.com/epic-operation-kicks-off-multistage-turla-apt-campaign/107612">Turla APT campaigns</a> have a broader reach than initially anticipated after the recent discovery of two modules built to infect servers running Linux. Until now, every Turla sample in captivity was designed for either 32- or 64-bit Windows systems, but researchers at Kaspersky Lab have discovered otherwise.</p>
<p>&#8220;The attack tool takes us further into the set alongside the Snake rootkit and components first associated with this actor a couple years ago,&#8221; <a href="https://securelist.com/blog/research/67962/the-penquin-turla-2/">wrote</a> Kurt Baumgartner and Costin Raiu, researchers with Kasperky&#8217;s Global Research and Analysis Team. &#8220;We suspect that this component was running for years at a victim site, but do not have concrete data to support that statement just yet.&#8221;</p>
<p>Like its Windows brethren, this version of Turla is a backdoor used to open communication to a command and control server—Kaspersky said it has sink-holed one such domain, which is based on UDP packets, used by one of the Linux modules—for file exfiltration, remote management and remote code execution.</p>
<p>Turla has been used in espionage campaigns against municipal governments, embassies, militaries and other industrial targets, primarily in the Middle East and Europe. In August, another component to these stealthy attacks called Epic Turla was disclosed; Epic is a multistage attack in which victims are compromised via spearphishing emails and other social engineering scams, or watering hole attacks.</p>
<div class="pullquote">The Epic Turla campaigns combined commodity exploits with zero-day attacks against Windows XP and Windows Server 2003 machines, as well as an Adobe Reader zero day.</div>
<p>The <a href="https://securelist.com/analysis/publications/65545/the-epic-turla-operation/">Epic Turla campaigns</a> combined commodity exploits with zero-day attacks against Windows XP and Windows Server 2003 machines, as well as an Adobe Reader zero day have been used to elevate an attacker&#8217;s privileges on the underlying system.</p>
<p>More than 100 websites were reported to be infected in Epic Turla attacks, including the website for City Hall in Pinor, Spain, an entrepreneurial site in Romania and the Palestinian Authority Ministry of Foreign Affairs.</p>
<p>All of the sites were built using the TYPO3 content management system, indicating the attackers had access to a vulnerability on that platform.</p>
<p>Once compromised, the websites then loaded remote JavaScript that performs a number of tasks, including dropping exploits for flaws in Internet Explorer 6-8, recent Java or Flash bugs, or a phony Microsoft Security Essentials application signed with a legitimate certificate from Sysprint AG.</p>
<p>Kaspersky&#8217;s Raiu and Baumgartner said most of the code in the Linux version of Turla comes from public sources. The backdoor, for example, is based on <a href="http://www.phenoelit.org/stuff/cd00r.c">cd00r</a>, Baumgartner and Raiu wrote. It includes an ELF executable that is statically linked against the GNU C library, an older version of OpenSSL and libpcap, the tcpdump network capture library.</p>
<p>The use of the cd00r backdoor enables the attack to go undetected, researchers said, because it does not require elevated privileges while running remote commands.</p>
<p>&#8220;It can&#8217;t be discovered via netstat, a commonly used administrative tool. It uses techniques that don&#8217;t require root access, which allows it to be more freely run on more victim hosts,&#8221; the researchers wrote. &#8220;Even if a regular user with limited privileges launches it, it can continue to intercept incoming packets and run incoming commands on the system.&#8221;</p>
<p>Turla was uncovered early this year and researchers also found a connection to the <a href="https://threatpost.com/agent-btz-malware-may-have-served-as-starting-point-for-red-october-turla/104735">Agent.btz worm</a> which infected U.S. military networks and led to a government mandate banning the use of USB drives. While <a href="https://securelist.com/blog/virus-watch/58551/agent-btz-a-source-of-inspiration/">Agent.btz </a>and Turla share characteristics, no one has linked the authors. Turla uses the same XOR key and log file names as Agent.btz, for example. Kaspersky&#8217;s Baumgartner and Raiu said that Linux variants were known to exist, but this is the first sample caught in the wild.</p>
<p>&#8220;Some of the malicious code appears to be inactive, perhaps leftovers from older versions of the implant,&#8221; the said. &#8220;Perhaps the most interesting part here is the unusual command and control mechanism based on TCP/UDP packets, as well as the C&amp;C hostname which fits previously known Turla activity.&#8221;</p>
