                    <div class="entry-content entry-single clearfix">
                        
<p>We’ve been closely monitoring an information stealer called ViperSoftX. ViperSoftX was first <a href="https://twitter.com/c3rb3ru5d3d53c/status/1227071037633945600" target="_blank" rel="noreferrer noopener">reported</a> on Twitter in 2020, and by <a href="https://www.fortinet.com/blog/threat-research/vipersoftx-new-javascript-threat" target="_blank" rel="noreferrer noopener">Fortinet</a> in the same year. Some aspects of ViperSoftX were also described previously by <a href="https://www.th3protocol.com/2022/ViperSoftX-Stealer" target="_blank" rel="noreferrer noopener">Colin Cowie</a>. However, it has undergone very intensive development since then, intensifying throughout 2022. The malware authors’ constant game of hide-and-seek in which they continually improve their strategies and techniques to avoid detections shows no signs of stopping. We, therefore, decided to put the pieces together to provide a comprehensive analysis.</p>



<p>This multi-stage stealer exhibits interesting hiding capabilities, concealed as small PowerShell scripts on a single line in the middle of otherwise innocent-looking large log files, among others. ViperSoftX focuses on stealing cryptocurrencies, clipboard swapping, fingerprinting the infected machine, as well as downloading and executing arbitrary additional payloads, or executing commands.</p>



<p>One of the payloads ViperSoftX distributes is a specific information stealer in the form of a browser extension for Chromium-based browsers. Due to its standalone capabilities and uniqueness, we decided to give it its own name, VenomSoftX. The malicious extension provides full access to every page the victim visits, carries out man-in-the-browser attacks to perform cryptocurrency addresses swapping by tampering with API requests’ data on popular cryptocurrency exchanges, steals credentials and clipboard content, tampers with crypto addresses on visited websites, reports events using MQTT to the C&amp;C server, and more.</p>



<p>ViperSoftX is mostly spread via cracked software such as <code>Adobe Illustrator</code>, <code>Corel Video Studio</code>, <code>Microsoft Office</code>, and more, commonly distributed over torrents.</p>



<h1 class="wp-block-heading">Campaign overview</h1>



<p>Since the beginning of 2022, we have protected more than 93,000 of our users. As the malware is mostly spread via torrents and software-sharing sites, ViperSoftX activity is distributed all over the world. The most impacted countries are India (7,000+), USA (6,000+), and Italy (5,000+).</p>

<h2 class="wp-block-heading">From cracked software to fake logs</h2>



<p>In the beginning, ViperSoftX is served to victims when they download what they believe to be cracked software. It is commonly named <code>Activator.exe</code> or <code>Patch.exe</code>. Upon execution, however, the victim is infected with ViperSoftX.</p>



<h3 class="wp-block-heading">Activator.exe extraction</h3>



<p><code>Activator.exe</code> is in fact a loader that decrypts data from itself using AES in CBC mode.</p>



<p>The decryption algorithm performs a checksum as follows:</p>



<ol class="wp-block-list">
<li>Read 4 bytes at offset <code>0x24</code> from the end of the file which gives an offset</li>



<li>Hash the offset value using SHA256</li>



<li>Read the rest of the bytes (from <code>-0x20</code>) and compare it to the hash</li>
</ol>



<p>If the checksum holds, the offset points to the location where the data is stored at <code>offset+0x24</code> from the end of the binary. Since the data is stored from the end of the binary, offset is also the size of the data blob. This blob can be decrypted straight away using a hardcoded key as well as IV inside the binary:</p>



<figure class="wp-block-table noheader"><table><tbody><tr><td>Key</td><td><code>71C54C3BCFFCE591A70C0B5BA6448327BC975D89F3021053125F1CB9A7C0AF72</code></td></tr><tr><td>IV</td><td><code>C0BA0B56EAC742AFD4CB680EE0EB4FB0</code></td></tr></tbody></table></figure>



<p>The decrypted data blob is a serialized protocol buffer structure. The structure template contains two protobuf messages as shown below:</p>


<div class="wp-block-image">
<figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="850" height="336" src="https://decoded.avast.io/wp-content/uploads/sites/2/2022/11/image-2.png" alt="" class="wp-image-6814" srcset="https://decoded.avast.io/wp-content/uploads/sites/2/2022/11/image-2.png 850w, https://decoded.avast.io/wp-content/uploads/sites/2/2022/11/image-2-300x119.png 300w, https://decoded.avast.io/wp-content/uploads/sites/2/2022/11/image-2-768x304.png 768w" sizes="(max-width: 850px) 100vw, 850px" /></figure></div>


<p>We can use this template to deserialize the structure, revealing five different files:</p>



<ul class="wp-block-list">
<li>A log file with a hidden additional payload resulting in the <code>ViperSoftX PowerShell</code> (see next subsections)</li>



<li><code>XML file</code> for the task scheduler</li>



<li><code>SyncAppvPublishingServer.vbs</code> (clean) that is used to create a scheduled task for persistence</li>



<li>Application <code>binary</code> (usually clean) that is supposed to be cracked</li>



<li><code>Manifest</code> file</li>
</ul>



<p>Last but not least, you can find the extraction Python script in our <a href="https://github.com/avast/ioc/blob/master/ViperSoftX/extras/extraction_script/extract_files.py" target="_blank" rel="noreferrer noopener">GitHub repository</a>.</p>



<h3 class="wp-block-heading">Analyzing the files</h3>



<p>The most interesting file is the aforementioned “log file” which is usually more than <code>5 MB</code> in size and contains a single malicious line of code (usually from <code>17,302</code> lines in total, but this might vary in different versions).</p>



<p>This log file is usually named and stored as:<br><code>C:\Windows\Logs\system-logs.txt</code><br>although we saw variants dropping the same “log” files disguising as a “driver” or a “text” file, e.g.<br><code>C:\Windows\System32\Drivers\p4kizn\e12de1ae-2139-45f6-b883-3c58243f23d6.sys</code><br><code>C:\Windows\2ZQ2UoL\5A7C4B54-4404-4424-83DA-CC696BED43D3.txt</code><br>where the subfolder names and the GUIDs are randomized.</p>



<p>An example of such a malicious line can be found below. As we will see later, this line actually contains a script that is decoded and executed.</p>

<h2 class="wp-block-heading">ViperSoftX Information Stealer</h2>



<p>When the payload is dropped, an obfuscated <code>ViperSoftX PowerShell script</code> is presented to us. We have seen multiple variants of ViperSoftX, suggesting that the malware is under active development. In the text below, we will cover the stealer’s features as a whole.</p>



<h3 class="wp-block-heading">Stealing Capabilities</h3>



<p>First, let’s have a look at what ViperSoftX is actually capable of stealing. ViperSoftX performs fingerprinting of the infected machine, focusing on various types of information, including:</p>



<ul class="wp-block-list">
<li>Computer name</li>



<li>Username</li>



<li>OS information and its architecture</li>



<li>Installed antivirus or other security software and whether the solution is active</li>
</ul>



<p>The malware focuses on stealing cryptocurrencies. To do so, it searches the typical locations for web browser extensions and locally stored wallets. The full list of locations can be found in the <a href="#appendix-locations">Appendix</a>. More generic information, such as OS, architecture, and username, is obtained using <code>WMI</code> and system variables.</p>



<p>ViperSoftX searches for cryptocurrencies stored locally on the infected device in cryptocurrency software and browser extensions, and monitors the clipboard for cryptocurrency wallet addresses to perform clipboard swapping.</p>



<p>The gathered data, as well as the fingerprint, is then concatenated together into a single string, encoded by base64, and sent to the hardcoded C&amp;C server in the <code>User-Agent</code> HTTP header. Note that C&amp;C servers vary across versions:<br><code>http://api.private-chatting[.]com/connect</code></p>



<p>Each time the victim copies anything to their clipboard, ViperSoftX scans the content using predefined regular expressions in the background. If the expression matches one of the configured wallet addresses belonging to the specific cryptocurrency, the malware replaces the content with the attacker&#8217;s address and sends a notification to the C&amp;C server in the <code>X-Notify</code> HTTP header in a form of three values:<br><code>Cryptocurrency type - victim’s address - attacker’s address</code></p>



<p>The cryptocurrency type reflects what type of cryptocurrency was matched and can be one of the following: <code>BTC</code>, <code>BCH</code>, <code>BNB</code>, <code>ETH</code>, <code>XMR</code>, <code>XRP</code>, <code>DOGE</code>, or <code>DASH</code>.</p>



<p>The malware also checks title texts of opened windows (<code>MainWindowTitle</code> property) and if it spots an application focused on cryptocurrencies or finance, it logs its presence into:<br><code>%SystemDrive%\Users\Public\log.dat</code><br>The full list of searched keywords can be found in the <a href="#appendix-keywords">Appendix</a>.</p>



<p>On top of the information-stealing core, ViperSoftX provides RAT functionalities as well, like executing arbitrary commands on the command line, downloading an additional arbitrary payload provided by the C&amp;C server, and executing it, as well as removing itself completely from the system. The RAT functionality can also be used, for example, to steal cryptocurrencies from their locations, which it previously identifies and sends to the C&amp;C server.</p>



<h3 class="wp-block-heading">Host Header Spoofing</h3>



<p>Aside from trying to steal cryptocurrencies, the malware spoofs host headers to obfuscate its communication with the C&amp;C servers.</p>



<p>The spoofed host header consists of five to 10 lowercase alpha letters, but the true destination is in the hardcoded <code>$meta_host</code> variable. This way, the real C&amp;C server address is obfuscated by a random-looking domain that doesn’t exist.</p>


<div class="wp-block-image">
<figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="850" height="168" src="https://decoded.avast.io/wp-content/uploads/sites/2/2022/11/image-6.png" alt="" class="wp-image-6819" srcset="https://decoded.avast.io/wp-content/uploads/sites/2/2022/11/image-6.png 850w, https://decoded.avast.io/wp-content/uploads/sites/2/2022/11/image-6-300x59.png 300w, https://decoded.avast.io/wp-content/uploads/sites/2/2022/11/image-6-768x152.png 768w" sizes="(max-width: 850px) 100vw, 850px" /></figure></div>


<h2 class="wp-block-heading">VenomSoftX Browser Extensions</h2>



<p>Newer versions of ViperSoftX information stealer are capable of loading a custom malicious browser extension to <code>Chromium-based browsers</code> installed on infected computers. The extension is provided by the C&amp;C server. The extension is basically another standalone information stealer, we are calling VenomSoftX, but is installed by ViperSoftX, as described below. The extension disguises itself as various popular browser extensions to avoid user detection.&nbsp;</p>



<p>The main goal VenomSoftX is also to steal cryptocurrencies from the unsuspecting victim. The difference is, VenomSoftX mainly does this by hooking API requests on a few very popular crypto exchanges victims visits/have an account with. When a certain API is called, for example, to send money, VenomSoftX tampers with the request before it is sent to redirect the money to the attacker instead. Although similar in principle to what information stealers like ViperSoftX do and rather a common clipboard swapping, this technique is performed at a lower level, meaning the victim has little to no chance of noticing the money is being transferred elsewhere.</p>



<h3 class="wp-block-heading">Installing the extension</h3>



<p>ViperSoftX&#8217;s approach is simple. The malware downloads a <code>VenomSoftX PowerShell installer</code> from the C&amp;C server e.g. by base64-decoding a hardcoded request metadata directly from the PowerShell script, following a request to:<br><code>http://apps-analyser[.]com/api/v1/&lt;HASH&gt;</code><br>depending on the malware version. This can hold different payloads, but we will focus on the VenomSoftX browser extension.</p>



<p>After the installer script is downloaded from the C&amp;C server and the <code>VenomSoftX browser extension</code> is extracted, the installer searches several locations for .lnk files and if such a link file belongs to <code>Chrome</code>, <code>Brave</code>, <code>Opera</code>, or <code>Edge</code>, it modifies the link file with a parameter <code>--load-extension=&lt;path_to_the_malicious_extension&gt;</code>. This way, when the user starts their favorite browser, they actually load the malicious extension with it.</p>



<p>These locations are checked for link files leading to browsers:</p>



<ul class="wp-block-list">
<li><code>%USERPROFILE%\Desktop</code></li>



<li><code>%USERPROFILE%\OneDrive\Desktop</code></li>



<li><code>%PUBLIC%\Desktop</code></li>



<li><code>%ALLUSERSPROFILE%\Microsoft\Windows\Start Menu\Programs</code></li>



<li><code>%APPDATA%\Microsoft\Windows\Start Menu\Programs</code></li>



<li><code>%APPDATA%\Microsoft\Internet Explorer\Quick Launch\User Pinned\TaskBar</code></li>
</ul>



<p>The extension ID is randomly generated, provided random lowercase characters to represent the extension folder and randomly generated version number of the extension:</p>



<p><code>$(Get-Random -Minimum 1 -Maximum 10).$(Get-Random -Minimum 1 -Maximum 10).$(Get-Random -Minimum 1 -Maximum 10)._0</code></p>



<p>We observed further versions of ViperSoftX containing a full update mechanism. These versions were able to walk through the modified <code>.lnk</code> files, parse the <code>manifest.json</code> file of the malicious extension, and when an older version or an old write timestamp of the extension was detected on the infected system than what was advertised by the C&amp;C server, the malware requested an update and replaced the extension files to the newest version by a dedicated command (provided by the C&amp;C server).</p>



<p>The extension tries to disguise itself as well known and common browser extensions such as <code>Google Sheets</code>. In reality, the VenomSoftX is yet another information stealer deployed onto the unsuspecting victim with full access permissions to every website the user visits from the infected browser.</p>

<h3 class="wp-block-heading">Diving into javascript shenanigans</h3>



<p>The extension contains several files, as shown in the table below. Each file has a different purpose:</p>



<figure class="wp-block-table"><table><tbody><tr><td><strong>File name</strong></td><td><strong>Purpose</strong></td></tr><tr><td>128.png</td><td>Google Sheets icon to disguise the extension</td></tr><tr><td>content.bootstrap.js</td><td>Orchestrates the malicious activity, sends results using MQTT</td></tr><tr><td>manifest.json</td><td>The extension’s manifest file</td></tr><tr><td>rules.json</td><td>URL filter rules for Kucoin</td></tr><tr><td>webpack_block.js</td><td>Request tampering and credential theft on Blockchain.com</td></tr><tr><td>webpack_bnb.js</td><td>Request tampering and cryptocurrency theft on Binance</td></tr><tr><td>webpack_cb.js</td><td>Request tampering and cryptocurrency theft on Coinbase</td></tr><tr><td>webpack_common.js</td><td>Contains an address book for pattern matching, clipboard monitoring and stealing</td></tr><tr><td>webpack_content.js</td><td>When no exchange is detected &#8211; attacker’s addresses are replaced on the visited websites with the ones that victim entered earlier</td></tr><tr><td>webpack_gt.js</td><td>Request tampering and cryptocurrency theft on Gate.io</td></tr><tr><td>webpack_kuc.js</td><td>Request tampering and cryptocurrency theft on Kucoin</td></tr></tbody></table></figure>



<p>The malware focuses on five big cryptocurrency exchanges, reflected by abbreviations in the modules names.</p>



<p>In the paragraphs below, we’ll focus on what each of the modules do, in detail.</p>



<h4 class="wp-block-heading">content.bootstrap.js</h4>



<p>This module is the starting point of VenomSoftX and it is loaded with every site visit. It orchestrates what modules to load and it is also responsible for sending stolen data to the C&amp;C server.</p>



<p>The scripts are loaded depending on the visited domain. The bootstrap checks what site is being loaded and if it is one of the following: <code>Blockchain.com</code>, <code>Binance</code>, <code>CoinBase</code>, <code>Gate.io</code>, or <code>Kucoin</code>, the module loads an appropriate “<code>webpack</code>”. If the user is on any other site, <code>webpack_content.js</code> is loaded. The module <code>webpack_common.js</code> is loaded by default regardless of which site the victim visits.</p>


<div class="wp-block-image">
<figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="850" height="624" src="https://decoded.avast.io/wp-content/uploads/sites/2/2022/11/image-8.png" alt="" class="wp-image-6821" srcset="https://decoded.avast.io/wp-content/uploads/sites/2/2022/11/image-8.png 850w, https://decoded.avast.io/wp-content/uploads/sites/2/2022/11/image-8-300x220.png 300w, https://decoded.avast.io/wp-content/uploads/sites/2/2022/11/image-8-768x564.png 768w" sizes="(max-width: 850px) 100vw, 850px" /><figcaption class="wp-element-caption"><em>Determination process which modules to load (deobfuscated)</em></figcaption></figure></div>


<p>All the modules serve their own purpose. However, two of the modules, <code>webpack_common.js</code> and <code>webpack_block.js</code>, are capable of sending data back to the collector server using a <code>Paho MQTT client</code> present in the <code>content.bootstrap.js</code>. The MQTT client has an event listener set to a hardcoded value <code>b8b0becb-080a-46af-9688-e3671fcc4166</code> that indicates data should be sent to an MQTT broker, harvesting the data:<br><code>broker.emqx[.]io</code><br>Note that the collector address can vary in different versions.</p>



<p>The sent data are structured as follows:<br><code>MSG: time: &lt;utc_datetime&gt;<br>ip: &lt;client_ip&gt;<br>data: &lt;data&gt;</code></p>



<p>The <code>time</code> and <code>ip</code> fields are obtained using a public service (<code>https://worldtimeapi.org/api/ip</code>).</p>



<p>The <code>data</code> field is either clipboard content with a cryptowallet and other metadata, or stolen credentials for <code>blockchain.com</code>. See further sections below for details.</p>



<h4 class="wp-block-heading">webpack_common.js</h4>



<p>The “<code>common</code>” module is loaded to every website, regardless of whether it is a cryptocurrency exchange or something else. It is used to define an address book with regular expression patterns, which is used for crypto address matching on several occasions in other modules.</p>



<p>The structure of the address book is in the form of a dictionary, where the key is the regular expression and the value is yet another dictionary containing three values: <code>coin</code>, <code>address</code>, and <code>network</code>. An example of such an address book can be seen in the below snippet. For the full address book, see <a href="#appendix-addressbook">Appendix</a>.</p>


<div class="wp-block-image">
<figure class="aligncenter size-full"><img loading="lazy" decoding="async" width="827" height="887" src="https://decoded.avast.io/wp-content/uploads/sites/2/2022/11/image-9.png" alt="" class="wp-image-6822" srcset="https://decoded.avast.io/wp-content/uploads/sites/2/2022/11/image-9.png 827w, https://decoded.avast.io/wp-content/uploads/sites/2/2022/11/image-9-280x300.png 280w, https://decoded.avast.io/wp-content/uploads/sites/2/2022/11/image-9-768x824.png 768w" sizes="(max-width: 827px) 100vw, 827px" /><figcaption class="wp-element-caption"><em>A snippet of a possible address book (incomplete, deobfuscated)</em></figcaption></figure></div>


<p>Furthermore, each time the user pastes anything into any website (except the malicious address from the address book), this module checks whether the clipboard content matches any of the regular expressions from the address book and if they do, it sends the following data to the collector server encoded using base64. The data for the MQTT message has a following construct:</p>



<p><code>Action<br><br>Site: &lt;URL&gt;<br>Browser: &lt;USER-AGENT&gt;<br>Clipboard: &lt;CLIPBOARD-WALLET-ADDRESS&gt;<br>Time: &lt;TIMESTAMP&gt;</code></p>



<h4 class="wp-block-heading">webpack_content.js</h4>



<p>This module monitors two input elements for content the user fills into websites and it is loaded when the victim is outside of any of the mentioned exchange sites:</p>



<ul class="wp-block-list">
<li><code>HTMLInputElement</code></li>



<li><code>HTMLTextAreaElement</code></li>
</ul>



<p>This is done by implementing hooks in getters of these elements to try to find a compatible crypto address for the user’s input and if found, the malware creates new <code>localStorage</code> entry with a combination of visited site and the compatible address:<br><code>website_attackerAddress: userAddress</code><br>The compatible address is found when the provided address matches any of the regular expressions from the <a href="#appendix-addressbook">address book</a> described in the previous section.</p>



<p>After that, a custom <code>MutationObserver</code> watches for dynamic changes in the site (e.g. loading the page, displaying a message sent earlier in the user’s messenger client, etc.) and if such a change occurs, the malware replaces all mentions of the malicious address (if found) with the user’s address using the <code>localStorage</code>. This effectively hides all traces of the malicious address in the website&#8217;s body.</p>



<p>Note that since the information is stored directly in the persistent <code>localStorage</code>, the functionality survives a browser restart, PC restart, or re-visiting the page anytime in the future. The victim has to clear the user data in their browser or uninstall the malware extension altogether to get rid of the malicious behavior.</p>



<p>For the sake of demonstrating this behavior, we decided to create a <code>demo PoC</code> of a simple page that illustrates the whole process of content tampering while the malware is active:</p>



<ol class="wp-block-list">
<li>First two buttons, <code>Attacker address</code> and <code>User address</code>, fill in attacker address or user address to the “<code>Dynamic content to be pasted</code>” line on the webpage</li>



<li>“<code>Get value</code>” button triggers the getter but the hook is inactive since the provided address doesn’t match cryptowallet address pattern (it is too short)</li>



<li>The user fills in a <code>compatible address</code></li>



<li>“<code>Get value</code>” button creates a <code>localStorage</code> entry with the entered address</li>



<li>The attacker’s address is always replaced with the one from <code>localStorage</code></li>
</ol>

<h3 class="wp-block-heading">Draining the victim&#8217;s accounts</h3>



<p>As we already mentioned, VenomSoftX focuses on five different crypto exchanges/websites, namely on <code>Blockchain.com</code>, <code>Binance</code>, <code>Coinbase</code>, <code>Gate.io</code>, and <code>Kucoin</code>.</p>



<p>In these modules, the malware tries to tamper with API requests the sites use for several actions like money withdrawal or sending security codes. This is done by creating hooks on the API calls (or rather functions responsible for sending those requests), parsing their structure, and replacing the response’s body with the desired attacker’s contents &#8211; commonly meaning that the recipient&#8217;s address is replaced by the attacker’s one as well as the amount is set to the value of an available account balance if known. The request is then processed by the malware without the user noticing anything, completely draining the victim’s portfolio as a result.</p>



<p>Note that the user has little to no chance to notice this. In comparison to a rather common clipboard swapping, for example, this “swapping” is performed on a lower level. Since the transactions on blockchains/ledgers are inherently irreversible, when the user checks the transaction history of payments afterward, it is already too late.</p>



<p>The complete list of hooked API functions can be found in the <a href="#appendix-hooked-api-functions">Appendix</a>. The only file that differs from the rest is <code>webpack_block.js</code>. This module focuses on <code>www.blockchain.com</code> and it tries to hook <code>https://blockchain.info/wallet</code>. It also modifies the getter of the password field to steal entered passwords. Once the request to the API endpoint is sent, the wallet address is extracted from the request, bundled with the password, and sent to the collector as a base64-encoded JSON via MQTT.</p>



<p>Since the rest of the hooks are the same on the higher level if we ignore API differences, we will use the Binance module as an illustration example.</p>



<p>The Binance module recognizes six different API calls invoking malicious interactions. When the user logs in to the site, it is expected that at some point the API function<br><code>https://www.binance.com/bapi/asset/v3/private/asset-service/asset/get-user-asset</code><br>will be called. VenomSoftX then parses and saves all assets available on the victim’s account. When the user tries to manipulate their savings, e.g. withdrawing their money, the malware intercepts the request<br><code>https://www.binance.com/bapi/capital/v3/private/capital/withdraw/apply</code><br>and tampers with the request body, modifying the address to redirect the money to the attacker’s address if a compatible attacker’s address was found. The amount of the request is also set to the maximum amount available obtained by the previous step. After the tampering, the request is passed further like nothing happened, effectively draining the victim’s wallet.</p>



<h1 class="wp-block-heading">Conclusion</h1>



<p>In this blog post, we took a closer look at ViperSoftX, a long-standing information stealer, and its malicious browser extension payload VenomSoftX. We described both information stealers’ infection chains, and how the original payload hides and decrypts on the infected system.</p>



<p>We described what both ViperSoftX and VenomSoftX steal and how the browser extension leverages its full access to every page the victim visits and carries out man-in-the-browser attacks by silently tampering with the API requests popular cryptocurrency exchanges use, resulting in draining the victim’s accounts.</p>
