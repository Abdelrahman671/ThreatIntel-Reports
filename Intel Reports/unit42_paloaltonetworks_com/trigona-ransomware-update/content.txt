Changing the default language of NetScan to Russian is an option that can be configured upon initial installation.\r\n\r\n[caption id=\"attachment_127268\" align=\"aligncenter\" width=\"300\"] Figure 6. NetScan output that operator(s) left on disk containing Cyrillic characters.[\/caption]\r\n\r\nAfter conducting reconnaissance, Trigona operators used Splashtop \u2013 a remote access and management (RMM) tool \u2013 to transfer the following malware into the target\u2019s environment.\r\n\r\nThreat actors often abuse, take advantage of or subvert legitimate products for malicious purposes. This does not necessarily imply a flaw or malicious quality to the legitimate product being abused.\r\nStart.bat\r\nStart.bat is a batch script that performs the following activities:\r\n\r\n \tIt creates a new folder at C:\\temp\r\n \tIt copies other malicious batch and EXE files from a compromised internal Server Message Block (SMB) server to the newly created temp folder\r\n \tIt executes Turnoff.bat\r\n\r\nTurnoff.bat\r\nTurnoff.bat is a cleanup script used to remove evidence of the attack on a system. It does so by performing the following activities:\r\n\r\n \tClearing the Recycle Bin of any mounted drive\r\n \tAttempting to use sc stop and taskkill to stop over 100 services related to various areas ranging from remote desktop tools to Windows Defender\r\n \tAttempting to stop services related to VMware, Hyper-V and SQL\r\n \tEnding several running tasks related to the stopped services mentioned above\r\n \tClear Windows Event Logs (using wevutil cl)\r\n \tDeleting Volume Shadow Copies\r\n \tDisconnecting all network drives\r\n\r\nUnit 42 researchers have observed that cleanup scripts from other threat actors are usually smaller and more specific to the tools used by that actor. The scattershot variety of services and tasks that turnoff.bat stops could suggest that the tool is attempting to ensure that a wider variety of systems are encrypted.\r\nNewuser.bat\r\nNewuser.bat is a batch script that creates a new user with the name fredla and the password Qw123456. It then adds the fredla user to the local groups Administrator and Remote Desktop Users. Threat actors sometimes create privileged user accounts to keep access to target systems without having to install persistent remote access tools on the system.\r\nDC2.exe\r\nDC2.exe contains a password protected version of Mimikatz, which is a tool used for extracting sensitive information such as passwords and authentication credentials from a Windows operating system.\r\n\r\nThis version of Mimikatz has been compressed using UPX. While UPX is often legitimately used to reduce file size, we have observed threat actors utilizing UPX and other packing programs to evade static detection of the underlying payload.\r\n\r\nThe tool is also password protected, which adds an extra layer of complexity when ascertaining the program\u2019s functionality.\r\n\r\nWhen the executable is run, the threat actor is prompted for a password to continue. The MD5 hash of the password is then calculated, and if it is equal to 4dbf44c6b1be736ee92ef90090452fc2, the program will continue running.\r\n\r\nThe password required to achieve the MD5 hash is boris.\r\n\r\nAmong its many legitimate uses, Unit 42 researchers have most often observed Mimikatz being leveraged maliciously by threat actors in the following ways:\r\n\r\n \tCredential Loading\r\n\r\n \tMimikatz loads credentials from various sources such as Windows memory, Local Security Authority Subsystem Service (LSASS) process and the Windows registry.\r\n\r\n\r\n \tCredential Dumping\r\n\r\n \tThe tool then extracts and dumps the credentials, including usernames and passwords, hashes, and Kerberos tickets to the screen or to a file.\r\n\r\n\r\n \tCredential Manipulation\r\n\r\n \tMimikatz allows the user to manipulate the dumped credentials, such as changing passwords, creating new user accounts and adding users to groups.\r\n\r\n\r\n \tCredential Injection\r\n\r\n \tThe tool can also inject the manipulated credentials into other processes, allowing the user to impersonate another user and gain access to restricted resources.\r\n\r\n\r\n\r\nDC4.exe\r\nDC4.exe is a small, UPX-packed password protected binary that generates and executes an embedded batch file. Like DC2.exe, the password to allow the binary to run is boris.\r\n\r\nUpon execution, the batch file makes the following changes to the system:\r\n\r\n \tDisables the User Account Control (UAC) and sets cmd.exe as a debugger for HelpPane.exe, utilman.exe, Magnify.exe and sethc.exe. This is a common method of creating a \u201cSticky Keys backdoor\u201d that allows for the creation of a command prompt with NT AUTHORITY\\SYSTEM privileges.\r\n \tOpens specific ports on the firewall to allow remote desktop connections using the netsh command.\r\n \tModifies the Windows registry to allow remote desktop connections.\r\n \tCreates a new user account with the username sys and password Mm1518061+-, and adds this user to the Administrator\u00a0and Remote Desktop Users groups.\r\n\r\nDC6.exe\r\nDC6.exe is an installer for the publicly available tool Advanced Port Scanner, wrapped up in an Inno Setup installer package. Inno Setup is a free software installer for Windows programs. Advanced Port Scanner is a tool that is commonly abused by threat actors for network scanning and mapping, for lateral movement and discovery purposes.\r\n\r\nWrapping Advanced Port Scanner in Inno Setup adds an additional layer of obfuscation to the code, and it is likely to evade static signature detection, forcing dynamic analysis to determine functionality rather than relying on traditional static code signatures.\r\nTTPs\r\n\r\n\r\n\r\nTactic \/ Technique\r\nNotes\r\n\r\n\r\nTA0002 Execution\r\n\r\n\r\nT1072. Software Deployment Tools\r\nTrigona operators use Splashtop to move laterally and transfer malware between compromised hosts in the victim\u2019s environment.\r\n\r\n\r\nTA0003 Persistence\r\n\r\n\r\nT1546.008. Accessibility Features\r\nDC4.exe creates a batch script that, when executed, creates a \u201cSticky Keys backdoor\u201d that allows for creation of a command prompt with NT AUTHORITY\\SYSTEM privileges.\r\n\r\n\r\nT1136. Create Account\r\nNewuser.bat creates a new user with the username fredla and password Qw123456.\r\n\r\n\r\nT1098. Account Manipulation\r\nTrigona operators compromise administrator accounts and use them to conduct malicious activities, such as executing NetScan.\r\n\r\n\r\nTA0005 Defense Evasion\r\n\r\n\r\nT1027. Obfuscated Files or Information\r\nTrigona operators use UPX to pack DC2.exe and DC4.exe to avoid static signature detection. For DC6.exe, Trigona hid the installer for Advanced Port Scanner within Inno Setup installer to evade static signature detection.\r\n\r\n\r\nT1112. Modify Registry\r\nDC4.exe creates a batch script that, when executed, modifies the Windows Registry to allow remote desktop connections.\r\n\r\n\r\nT1562.004. Disable or Modify System Firewall\r\nTrigona operators open up an Remote Desktop Protocol (RDP) port in the firewall with DC4.exe.\r\n\r\n\r\nT1070.001. Mim32.exe is associated with Mimikatz while zam.exe and zam.bat are associated with NetScan.\r\n\r\n\r\nT1036.004. Network Service Discovery\r\nTrigona operators use NetScan to enumerate hosts within victims\u2019 domains that might be vulnerable to remote software exploitation.\r\n\r\n\r\nT1069. Permission Groups Discovery\r\nTrigona operators use NetScan to enumerate the security-enabled local group membership of the Administrators group.\r\n\r\n\r\nT1021.001. <p>Trigona ransomware is a relatively new strain that security researchers first discovered in late October 2022. By analyzing Trigona ransomware binaries and ransom notes obtained from VirusTotal, as well as information from Unit 42 incident response, we determined that Trigona was very active during December 2022, with at least 15 potential victims being compromised. Affected organizations are in the manufacturing, finance, construction, agriculture, marketing and high technology industries.</p>
<p>Unit 42 researchers identified two new Trigona ransom notes in January 2023 and two in February 2023. Trigona’s ransom notes are unique; rather than the usual text file, they are instead presented in an HTML Application with embedded JavaScript containing unique computer IDs (CID) and victim IDs (VID).</p>
<p>Palo Alto Networks helps detect and prevent Trigona ransomware with the following products and services: <a href="https://docs-cortex.paloaltonetworks.com/p/XDR" target="_blank" rel="noopener">Cortex XDR</a>, <a href="https://docs.paloaltonetworks.com/prisma/prisma-cloud" target="_blank" rel="noopener">Prisma Cloud</a> and <a href="https://docs.paloaltonetworks.com/ngfw" target="_blank" rel="noopener">Next-Generation Firewalls</a> (including cloud-delivered security subscriptions such as <a href="https://docs.paloaltonetworks.com/wildfire" target="_blank" rel="noopener">WildFire</a>) and through <a href="https://start.paloaltonetworks.com/contact-unit42.html" target="_blank" rel="noopener">incident response</a>.</p>

<h2><a id="post-127253-_ny2qloj26cvd"></a>Trigona Overview</h2>
<p>The first mention of Trigona, also the name of a family of stingless bees, comes from <a href="https://twitter.com/malwrhunterteam/status/1587581807595249666" target="_blank" rel="noopener">a tweet by security researchers</a> in late October 2022. Malware samples were passed to BleepingComputer, which in turn published <a href="https://www.bleepingcomputer.com/news/security/trigona-ransomware-spotted-in-increasing-attacks-worldwide/" target="_blank" rel="noopener">a blog post on the ransomware</a> on Nov. 29, 2022. Unit 42 consultants also have seen Trigona firsthand in the course of incident response.</p>
<p>Unit 42 researchers have observed Trigona’s threat operator engaging in behavior such as obtaining initial access to a target’s environment, conducting reconnaissance, transferring malware via remote monitoring and management (RMM) software, creating new user accounts and deploying ransomware.</p>
<h2><a id="post-127253-_3b42rk4nwqtv"></a>Ransomware Analysis</h2>
<h3><a id="post-127253-_n19f2o1xfs39"></a>Ransomware Binary</h3>
<p>Unit 42 obtained and analyzed a sample of the Trigona ransomware binary, named <span style="font-family: 'courier new', courier, monospace;">svhost.exe</span>. Upon execution, the ransomware binary uses <span style="font-family: 'courier new', courier, monospace;">TDCP_rijndael</span> (a Delphi AES library) to encrypt files. The ransomware then appends the <span style="font-family: 'courier new', courier, monospace;">._locked</span> file extension, modifies registry keys to maintain persistence, and drops ransom notes.</p>
<p>The ransomware binary supports the following command line arguments:</p>

<p>The ransomware establishes persistence through the creation of two keys in <span style="font-family: 'courier new', courier, monospace;">CurrentVersion\Run</span>. Keys found in <span style="font-family: 'courier new', courier, monospace;">CurrentVersion\Run</span> contain references to programs that will execute when a user logs in.</p>
<p>One key executes the ransomware binary whenever the user logs in, ensuring that the encryption process would resume upon reboot. The other key ensures that the ransom note is opened every time the user logs in.</p>
<h3><a id="post-127253-_d3myajqe88xh"></a>Ransom Note</h3>
<p>Trigona’s ransom note is dropped to the system with the name <span style="font-family: 'courier new', courier, monospace;">how_to_decrypt.hta</span>. The HTML code in this file contains embedded JavaScript functionality, which displays ransom note details as shown below in Figure 1.</p>
<figure id="attachment_127258" aria-describedby="caption-attachment-127258" style="width: 600px" class="wp-caption aligncenter"><img  class="wp-image-127258 lozad"  data-src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/03/word-image-127253-1.png" alt="Image 1 is a screenshot of a sample Trigona ransom note that tells a business its network is encrypted, the three steps of instructions for data recovery, and tips to make the price cheaper. There is also a “Need help?” link." width="600" height="531" /><figcaption id="caption-attachment-127258" class="wp-caption-text">Figure 1. Sample Trigona ransom note.</figcaption></figure>
<p>Unit 42 researchers observed that the JavaScript within the ransom note contains the following information:</p>
<ul>
<li>A uniquely generated CID and VID</li>
<li>A link to the negotiation Tor portal</li>
<li>An email address to contact.</li>
</ul>
<p>The contact email shown below in Figure 2 is <span style="font-family: 'courier new', courier, monospace;">phandaledr@onionmail[.]org</span>. We have also seen <span style="font-family: 'courier new', courier, monospace;">farusbig@tutanota[.]com</span> used as the contact email in other Trigona ransom notes.</p>
<figure id="attachment_127260" aria-describedby="caption-attachment-127260" style="width: 716px" class="wp-caption aligncenter"><img  class="wp-image-127260 lozad"  data-src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/03/word-image-127253-2.png" alt="Image 2 is a screenshot of JavaScript code showing the unique computer and victim IDs, which here have been redacted. <h3><a id="post-127253-_68menvpdiyi1"></a>Victimology</h3>
<p>By looking at the victim ID in the embedded JavaScript in the Trigona ransom notes, we were able to identify at least 15 potential victims that were compromised in December 2022. We also identified two new Trigona ransom notes in January 2023 and two in February 2023.</p>
<p>Trigona ransomware has been linked to compromises impacting multiple organizations worldwide, in sectors including manufacturing, finance, construction, agriculture, marketing and high technology. The companies impacted were in the United States, Italy, France, Germany, Australia and New Zealand.</p>
<h3><a id="post-127253-_3oad56y6f3i7"></a>Leak Site Analysis</h3>
<p>When Trigona was first observed, there was no evidence of this group using a leak site for double extortion. Their ransom note pointed the victims to their negotiation portal instead. During the investigation of this ransomware family, we observed that <a href="https://twitter.com/paul_eubanks/status/1628497550679351303?cxt=HHwWjoCxnZ35ypktAAAA" target="_blank" rel="noopener">a researcher identified a leak site attributed to Trigona</a> hosted on the IP address <span style="font-family: 'courier new', courier, monospace;">45.227.253[.]99</span>.</p>
<p>Unit 42 researchers pivoted on the SSH key for <span style="font-family: 'courier new', courier, monospace;">45.227.253[.]99</span> and identified three other IP addresses related to Trigona’s infrastructure:</p>
<ul>
<li><span style="font-family: 'courier new', courier, monospace;">45.227.253[.]106</span></li>
<li><span style="font-family: 'courier new', courier, monospace;">45.227.253[.]98</span></li>
<li><span style="font-family: 'courier new', courier, monospace;">45.227.253[.]107</span></li>
</ul>
<p>Each IP shares the same SSH key of <span style="font-family: 'courier new', courier, monospace;">ecdsa-sha2-nistp256 AAAAE2VjZHNhLXNoYTItbmlzdHAyNTYAAAAIbmlzdHAyNTYAAABBBMjqeyIfJyuimtE414TBCxN+lHleN5/P3CNiD4uln5xyHjyw4muLePQj2y3yOJ7GLlTvjrheWlrot3REko99eKQ=</span>.</p>
<p>IPs <span style="font-family: 'courier new', courier, monospace;">45.227.253[.]99</span> and <span style="font-family: 'courier new', courier, monospace;">45.227.253[.]106</span> hosted web servers on port 8000, while <span style="font-family: 'courier new', courier, monospace;">45.227.253[.]98</span> and <span style="font-family: 'courier new', courier, monospace;">45.227.253[.]107</span> hosted no web services.</p>
<p>We identified that <span style="font-family: 'courier new', courier, monospace;">45.227.253[.]99</span> hosted a web server between Dec. 6, 2022, and Jan. 27, 2023. On Feb. 13, 2023, <span style="font-family: 'courier new', courier, monospace;">45.227.253[.]106</span> started hosting a web server with the HTML title <span style="font-family: 'courier new', courier, monospace;">Trigona Leaks</span> that was active until March 3, 2023.</p>
<p>As shown in Figure 3, each post contained the following information:</p>
<ul>
<li>A description of the company</li>
<li>The victim’s ZoomInfo page</li>
<li>A description of the stolen data</li>
<li>Links to screenshots of example files</li>
<li>A countdown timer</li>
<li>A button to bid for the data.</li>
</ul>
<p>The “@ Place a bid” button contained a <span style="font-family: 'courier new', courier, monospace;">mailto</span> link to <span style="font-family: 'courier new', courier, monospace;">auction@mailthink[.]net</span>. Mailthink is a service that allows users to create temporary, disposable email addresses.</p>
<figure id="attachment_127262" aria-describedby="caption-attachment-127262" style="width: 900px" class="wp-caption aligncenter"><img  class="wp-image-127262 lozad"  data-src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/03/word-image-127253-3.png" alt="Image 3 is a screenshot of the Trigona leak site. It details current leaks, views, if the leak is active, and a counter showing how much time is left. Details including screenshots are available, as well as the ransom amounts. There is a green button to place a bid. " width="900" height="825" /><figcaption id="caption-attachment-127262" class="wp-caption-text">Figure 3. Trigona leak site.</figcaption></figure>
<p>While the leak site was active, there were four victims:</p>
<ul>
<li>Victim 1 has a near-duplicate post on the <a href="https://unit42.paloaltonetworks.com/blackcat-ransomware/" target="_blank" rel="noopener">BlackCat (ALPHV)</a> leak site and a countdown timer of over 300 days. <a href="https://areteir.com/report/observations-on-new-trigona-ransomware/" target="_blank" rel="noopener">Security researchers at Arete Incident Response recently observed</a> Trigona leveraging BlackCat’s reputation and data leak site to pressure and extort victims. It’s <strong>unclear</strong> whether Victim 1 was impacted by Trigona.</li>
<li>Victim 2 has a duplicate post on the BlackCat (ALPHV) leak site and a countdown timer of over 300 days. It’s <strong>unclear</strong> whether Victim 2 was impacted by Trigona.</li>
<li>Victim 3 has an associated ransom note on VirusTotal and a countdown timer of just over 30 days. Unit 42 assesses with high confidence that Victim 3 <strong>was impacted</strong> by Trigona.</li>
<li>Victim 4 is not mentioned on any other ransomware gang’s leak site and has a countdown timer of over 300 days. Unit 42 did not identify any associated ransom notes and it’s <strong>unclear</strong> whether Victim 4 was impacted by Trigona.</li>
</ul>
<p>The countdown timers of over 300 days for Victims 1, 2 and 4 were well beyond the usual timeframe that we have observed in incident response cases where attackers demand payment, which is between two and four weeks.</p>
<p>Given the following features, the Unit 42 team believes with moderate confidence that the surface web leak page was a development environment to test out features before a possible move to the dark web:</p>
<ul>
<li>Several posts appear to be duplicates from the BlackCat leak site (as shown in Figure 4)</li>
<li>Several of the countdown timers are considerably longer</li>
<li>The leak site is no longer available on the surface web</li>
</ul>
<figure id="attachment_127264" aria-describedby="caption-attachment-127264" style="width: 900px" class="wp-caption aligncenter"><img  class="wp-image-127264 lozad"  data-src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/03/word-image-127253-4.png" alt="Image 4 is a screenshot comparing Trigona’s leak site (left) to BlackCat (ALPHV) on the right. Highlighted in red in both screenshots is the duplicate text that Trigona copied from the BlackCat site. " width="900" height="684" /><figcaption id="caption-attachment-127264" class="wp-caption-text">Figure 4. Comparison between Trigona leak site (left) and BlackCat (ALPHV) leak site (right).</figcaption></figure>
<h2><a id="post-127253-_ka9kzoe4ky9r"></a>Similarities to CryLock Ransomware</h2>
<p>Trigona operators share overlap in tactics, techniques and procedures (TTPs) with CryLock ransomware operators, suggesting that ransomware threat actors that once deployed CryLock ransomware might have moved on to deploying Trigona ransomware. The email associated with Trigona ransom notes analyzed by Unit 42 (<span style="font-family: 'courier new', courier, monospace;">phandaledr@onionmail[.]org</span>) was mentioned in an online forum discussing CryLock ransomware, as shown below in Figure 5.</p>
<figure id="attachment_127266" aria-describedby="caption-attachment-127266" style="width: 900px" class="wp-caption aligncenter"><img  class="wp-image-127266 lozad"  data-src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/03/word-image-127253-5.jpeg" alt="Image 5 is a screenshot of the Russian antimalware forum SafeZone where someone has posted asking for help with Crylock. Highlighted in red is an email address. " width="900" height="483" /><figcaption id="caption-attachment-127266" class="wp-caption-text">Figure 5. A user on SafeZone, a Russian anti-malware forum, seeking help for Crylock ransomware.</figcaption></figure>
<p>Both ransomware families also drop ransom notes in HTML Application format, named <span style="font-family: 'courier new', courier, monospace;">how_to_decrypt.hta</span>. There are also similarities in the ransom message, including:</p>
<ul>
<li>Their claim that all “documents, databases, backups, and other critical” files and data were encrypted</li>
<li>AES as their choice of cryptographic algorithm</li>
<li>Their statement that “the price depends on how soon you will contact us”</li>
</ul>
<h2><a id="post-127253-_pugaeqkgy4cz"></a>Tools and Techniques</h2>
<p>Unit 42 has seen evidence of malicious activity associated with Trigona originating from a compromised Windows 2003 server, followed by the threat operators executing NetScan for internal reconnaissance.</p>
<h3><a id="post-127253-_u0di1l4110g1"></a>NetScan</h3>
<p>Unit 42 analysts recovered the NetScan output and noticed that it contained Cyrillic characters, as shown below in Figure 6. Changing the default language of NetScan to Russian is an option that can be configured upon initial installation.</p>
<figure id="attachment_127268" aria-describedby="caption-attachment-127268" style="width: 300px" class="wp-caption aligncenter"><img  class="wp-image-127268 lozad"  data-src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/03/word-image-127253-6.png" alt="Image 6 is a screenshot of many lines of code showing the NetScan output containing Cyrillic characters." width="300" height="259" /><figcaption id="caption-attachment-127268" class="wp-caption-text">Figure 6. NetScan output that operator(s) left on disk containing Cyrillic characters.</figcaption></figure>
<p>After conducting reconnaissance, Trigona operators used Splashtop – a remote access and management (RMM) tool – to transfer the following malware into the target’s environment.</p>
<p>Threat actors often abuse, take advantage of or subvert legitimate products for malicious purposes. This does not necessarily imply a flaw or malicious quality to the legitimate product being abused.</p>
<h3><a id="post-127253-_n0oc3sikogwo"></a>Start.bat</h3>
<p><span style="font-family: 'courier new', courier, monospace;">Start.bat</span> is a batch script that performs the following activities:</p>
<ul>
<li>It creates a new folder at <span style="font-family: 'courier new', courier, monospace;">C:\temp</span></li>
<li>It copies other malicious batch and EXE files from a compromised internal Server Message Block (SMB) server to the newly created temp folder</li>
<li>It executes <span style="font-family: 'courier new', courier, monospace;">Turnoff.bat</span></li>
</ul>
<h3><a id="post-127253-_bpmj0vc7ts45"></a>Turnoff.bat</h3>
<p><span style="font-family: 'courier new', courier, monospace;">Turnoff.bat</span> is a cleanup script used to remove evidence of the attack on a system. It does so by performing the following activities:</p>
<ul>
<li>Clearing the Recycle Bin of any mounted drive</li>
<li>Attempting to use <span style="font-family: 'courier new', courier, monospace;">sc stop</span> and <span style="font-family: 'courier new', courier, monospace;">taskkill</span> to stop over 100 services related to various areas ranging from remote desktop tools to Windows Defender</li>
<li>Attempting to stop services related to VMware, Hyper-V and SQL</li>
<li>Ending several running tasks related to the stopped services mentioned above</li>
<li>Clear Windows Event Logs (using <span style="font-family: 'courier new', courier, monospace;">wevutil cl</span>)</li>
<li>Deleting Volume Shadow Copies</li>
<li>Disconnecting all network drives</li>
</ul>
<p>Unit 42 researchers have observed that cleanup scripts from other threat actors are usually smaller and more specific to the tools used by that actor. The scattershot variety of services and tasks that <span style="font-family: 'courier new', courier, monospace;">turnoff.bat</span> stops could suggest that the tool is attempting to ensure that a wider variety of systems are encrypted.</p>
<h3><a id="post-127253-_ih0kdfunr4js"></a>Newuser.bat</h3>
<p><span style="font-family: 'courier new', courier, monospace;">Newuser.bat</span> is a batch script that creates a new user with the name <span style="font-family: 'courier new', courier, monospace;">fredla</span> and the password <span style="font-family: 'courier new', courier, monospace;">Qw123456</span>. It then adds the <span style="font-family: 'courier new', courier, monospace;">fredla</span> user to the local groups <span style="font-family: 'courier new', courier, monospace;">Administrator</span> and <span style="font-family: 'courier new', courier, monospace;">Remote Desktop Users</span>. Threat actors sometimes create privileged user accounts to keep access to target systems without having to install persistent remote access tools on the system.</p>
<h3><a id="post-127253-_qj63gkoqod9k"></a>DC2.exe</h3>
<p><span style="font-family: 'courier new', courier, monospace;">DC2.exe</span> contains a password protected version of Mimikatz, which is a tool used for extracting sensitive information such as passwords and authentication credentials from a Windows operating system.</p>
<p>This version of Mimikatz has been compressed using UPX. While UPX is often legitimately used to reduce file size, we have observed threat actors utilizing UPX and other packing programs to evade static detection of the underlying payload.</p>
<p>The tool is also password protected, which adds an extra layer of complexity when ascertaining the program’s functionality.</p>
<p>When the executable is run, the threat actor is prompted for a password to continue. The MD5 hash of the password is then calculated, and if it is equal to <span style="font-family: 'courier new', courier, monospace;">4dbf44c6b1be736ee92ef90090452fc2</span>, the program will continue running.</p>
<p>The password required to achieve the MD5 hash is <span style="font-family: 'courier new', courier, monospace;">boris</span>.</p>
<p>Among its many legitimate uses, Unit 42 researchers have most often observed Mimikatz being leveraged maliciously by threat actors in the following ways:</p>
<ul>
<li>Credential Loading
<ul>
<li>Mimikatz loads credentials from various sources such as Windows memory, Local Security Authority Subsystem Service (LSASS) process and the Windows registry.</li>
</ul>
</li>
<li>Credential Dumping
<ul>
<li>The tool then extracts and dumps the credentials, including usernames and passwords, hashes, and Kerberos tickets to the screen or to a file.</li>
</ul>
</li>
<li>Credential Manipulation
<ul>
<li>Mimikatz allows the user to manipulate the dumped credentials, such as changing passwords, creating new user accounts and adding users to groups.</li>
</ul>
</li>
<li>Credential Injection
<ul>
<li>The tool can also inject the manipulated credentials into other processes, allowing the user to impersonate another user and gain access to restricted resources.</li>
</ul>
</li>
</ul>
<h3><a id="post-127253-_4aingk6ibb8l"></a>DC4.exe</h3>
<p><span style="font-family: 'courier new', courier, monospace;">DC4.exe</span> is a small, UPX-packed password protected binary that generates and executes an embedded batch file. Like <span style="font-family: 'courier new', courier, monospace;">DC2.exe</span>, the password to allow the binary to run is <span style="font-family: 'courier new', courier, monospace;">boris</span>.</p>
<p>Upon execution, the batch file makes the following changes to the system:</p>
<ol>
<li>Disables the User Account Control (UAC) and sets <span style="font-family: 'courier new', courier, monospace;">cmd.exe</span> as a debugger for <span style="font-family: 'courier new', courier, monospace;">HelpPane.exe</span>, <span style="font-family: 'courier new', courier, monospace;">utilman.exe</span>, <span style="font-family: 'courier new', courier, monospace;">Magnify.exe</span> and <span style="font-family: 'courier new', courier, monospace;">sethc.exe</span>. This is a common method of creating a “Sticky Keys backdoor” that allows for the creation of a command prompt with <span style="font-family: 'courier new', courier, monospace;">NT AUTHORITY\SYSTEM</span> privileges.</li>
<li>Opens specific ports on the firewall to allow remote desktop connections using the <span style="font-family: 'courier new', courier, monospace;">netsh</span> command.</li>
<li>Modifies the Windows registry to allow remote desktop connections.</li>
<li>Creates a new user account with the username <span style="font-family: 'courier new', courier, monospace;">sys</span> and password <span style="font-family: 'courier new', courier, monospace;">Mm1518061+-</span>, and adds this user to the <span style="font-family: 'courier new', courier, monospace;">Administrator</span> and <span style="font-family: 'courier new', courier, monospace;">Remote Desktop Users</span> groups.</li>
</ol>
<h3><a id="post-127253-_u6rhm48xp9cp"></a>DC6.exe</h3>
<p><span style="font-family: 'courier new', courier, monospace;">DC6.exe</span> is an installer for the publicly available tool Advanced Port Scanner, wrapped up in an Inno Setup installer package. Inno Setup is a free software installer for Windows programs. Advanced Port Scanner is a tool that is commonly abused by threat actors for network scanning and mapping, for lateral movement and discovery purposes.</p>
<p>Wrapping Advanced Port Scanner in Inno Setup adds an additional layer of obfuscation to the code, and it is likely to evade static signature detection, forcing dynamic analysis to determine functionality rather than relying on traditional static code signatures.</p>

<h2><a id="post-127253-_sv1mtpyc8nnf"></a>Conclusion</h2>
<p>Trigona is a newer strain of ransomware that, to date, has had minimal coverage by security news articles. This lack of security community awareness allows Trigona to discreetly attack victims while other higher-profile ransomware operations dominate the news headlines. We hope that shining a light on Trigona and its uncommon technique of using password-protected executables to obfuscate malware helps defenders better protect their organizations against this threat.</p>
<p>Due to the stream of victims identified by the Unit 42 team and Trigona’s currently developing leak site, the operator and/or affiliates behind the ransomware likely will continue (and possibly even ramp up) its malicious activity.</p>

<li>Prisma Cloud will detect any instance of this malware being executed through properly configured Defender agents using Wildfire.
