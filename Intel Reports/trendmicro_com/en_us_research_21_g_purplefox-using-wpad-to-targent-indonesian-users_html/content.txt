	<h1 class="article-details__title">PurpleFox Using WPAD to Target Indonesian Users</h1>
	<p class="article-details__description">The PurpleFox Exploit Kit is now being distributed via WPAD attacks targeting Indonesian users.</p>
	<p class="article-details__author-by">By: William Gamazo Sanchez
		
			<time class="article-details__date">July 01, 2021</time>
		
		
		<span>Read time:&nbsp;</span><span class="eta"></span> (<span class="words"></span> words)
	</p>

            <div class="richText">
	
    


	
		<div>
			<p>In September 2020, we published a blog describing how the <a href="/en_us/research/20/i/purple-fox-ek-relies-on-cloudflare-for-stability.html">PurpleFox Exploit Kit used Cloudflare services</a> to maintain an infrastructure resilient to blocking and detection attempts. Since then, PurpleFox has been maintaining this strategy while at same time improving its attack chain by incorporating the latest <a href="/en_us/research/19/i/purple-fox-fileless-malware-with-rookit-component-delivered-by-rig-exploit-kit-now-abuses-powershell.html">public vulnerabilities</a> into its arsenal.</p>
<p>Recently, we found that PurpleFox added a very old tactic to increase its delivering performance. This time PurpleFox EK is <a href="https://www.trendmicro.com/en_us/research/16/h/badwpad-doubtful-legacy-wpad-protocol.html">making use of WPAD domains</a> to infect users. While a WPAD abuse attack is a technique that has been around for approximately 14 years, it still works. <a href="https://www.wpadblock.com/" target="_blank">Initiatives to prevent this attack</a> help, but they are not sufficient.</p>
<p>Our systems started detecting victims accessing the “wpad.id” domain, which makes use of the Indonesian top level domain (*.id). We did not find any other country top level domain affected. Using this technique, a zero-click attack can be implemented, as the WPAD URL is accessed whenever the system starts, without any user input.</p>
<p><b>PurpleFox WPAD landing page</b></p>
<p>To abuse WPAD, the PurpleFox authors registered the domain “wpad.id” with Cloudflare. They then load the URL for WPAD services, which is located at <i>http://wpad[.]id/wpad[.]dat</i>. At the time of analysis, this would return a standalone JavaScript version of the <a href="https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2019-1367" target="_blank">CVE-2019-1367</a> with custom shellcode to follow the attack chain setup for the WPAD attack. Figure 1 shows the WPAD resolution and malicious sample delivery.</p>

		</div>
	

</div>

    <div class="richText">
	
    


	
		<div>
			<p>The custom shellcode downloads the next stage from the URL <i>http://9kf[.]me/in[.]php?id=1</i>. The domain “9kf.me” was no longer accessible by the time we analyzed the samples, but we were be able to find two more active domains, “2kf.me” and “6kf.me,” that contained the same payload.</p>
<p>Following the request logic, we retrieved the full chain used in this deployment. The PurpleFox chain is designed with multiple complicated stages abusing PowerShell and MSI files as previously described by Trend Micro and <a href="https://labs.sentinelone.com/purple-fox-ek-new-cves-steganography-and-virtualization-added-to-attack-flow/" target="_blank">other researchers</a>. This post will not go into the details; we will limit ourselves to showing how the two domains are chained to deliver the full attack chain.</p>

		</div>
	

</div>

		<div>
			<p>The domain resolution and access to the attack chain artifacts are all being proxied through Cloudflare servers, as shown in the Figure 4.</p>

    <div class="richText">
	
    


	
		<div class="responsive-table-wrap">
			<p>Analysis of the full chain revealed that the following CVEs were being exploited: <a href="https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2020-1054" target="_blank">CVE-2020-1054</a>, <a href="https://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2018-8120" target="_blank">CVE-2018-8120</a>, as well as an <a href="https://github.com/euphrat1ca/ms15-051" target="_blank">exploit for MS15-051</a>. The binary exploiting the MS15-051 leak the symbols path <i>C:\Users\K8team\Desktop\ms15-051\ms15-051\ms15-051\Win32\ms15-05, </i>suggesting that PurpleFox is reusing tools from <a href="https://github.com/k8gege/K8tools" target="_blank">K8team</a>, which is responsible for maintaining public repositories of CVE exploits POCs and hack tools.</p>
<p><b>Defending against PurpleFox</b></p>
<p>The PurpleFox Exploit Kit continues to be very active and appear to be looking for new infection tactics. Our feedback shows that this specific attempt is not only affecting Indonesian victims, as users in other countries who are using the Indonesian TLD are being affected as well. At same time, PurpleFox is trying to reach servers where the user interaction is minimal but are potentially affected by the WPAD technique, such as unattended machines.</p>
<p>Continuous vigilance against threat groups is an important aspect of keeping up with — if not staying one step ahead of — threats. To protect systems from this type of threat, users can use multilayered security solutions like <a href="/en_us/business/products/user-protection.html">Trend Micro Protection Suites</a> that help detect and block attacks. <a href="/en_us/business/products/detection-response.html">Trend Micro Vision One™️</a> also provides visibility, correlated detection, and behavior monitoring across multiple layers, such as emails, endpoints, servers, and cloud workloads. This ensures that no significant incidents go unnoticed and allows faster response to threats before they can do any real damage to the system. </p>
