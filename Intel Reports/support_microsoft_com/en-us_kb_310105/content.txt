	<header class="ocpArticleTitleSection" aria-labelledby="page-header" role="banner">
		<h1 id="page-header" class="">How to use the SysKey utility to secure the Windows Security Accounts Manager database</h1>
	</header>
	
		<div class="ocpArticleContent">
			<div class="ucsPromotionContainer"></div>
		</div>
	<div id="ucsRailInlineContainer"></div>
	

	<article class="ocpArticleContent"><section class="ocpIntroduction">
        <p></p>
        <p>For a Microsoft Windows NT version of this article, see <a href="http://support.microsoft.com/en-us/help/143475" class="ocpExternalLink" target="_blank" data-bi-type="anchor">143475</a>.</p>
      </section><section aria-labelledby="ID0EDF" class="ocpSection">
        <h2 id="ID0EDF" class=""> Summary</h2>
        
          <p>The Microsoft Windows 2000, Microsoft Windows XP, and Microsoft Windows 2003 Security Accounts Management Database (SAM) stores hashed copies of user passwords. This database is encrypted with a locally stored system key. To keep the SAM database secure, Windows requires that the password hashes are encrypted. Windows prevents the use of stored, unencrypted password hashes. <br aria-hidden="true"><br aria-hidden="true">You can use the SysKey utility to additionally secure the SAM database by moving the SAM database encryption key off the Windows-based computer. The SysKey utility can also be used to configure a start-up password that must be entered to decrypt the system key so that Windows can access the SAM database. This article describes how to use the SysKey utility to secure the Windows SAM database.</p>
        
      </section><section aria-labelledby="ID0EFD" class="ocpSection">
        <h2 id="ID0EFD" class=""> More Information</h2>
        
        
          <section aria-labelledby="ID0EDBBD" class="ocpSection">
            <h3 id="ID0EDBBD" class="">Configure Windows System Key Protection</h3>
            
              <p>To Configure Windows System Key Protection, follow these steps:</p>
              <ol type="1" itemscope="" itemtype="http://schema.org/ItemList">
      <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                  <p>At a command prompt, type syskey, and then press ENTER.</p>
                <meta itemprop="position" content="1">
      </li>
      <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                  <p>In the <b class="ocpLegacyBold">Securing the Windows Account Database</b> dialog box, note that the <b class="ocpLegacyBold">Encryption Enabled</b> option is selected and is the only option available. When this option is selected, Windows will always encrypt the SAM database.</p>
                <meta itemprop="position" content="2">
      </li>
      <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                  <p>Click <b class="ocpLegacyBold">Update</b>.</p>
                <meta itemprop="position" content="3">
      </li>
      <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                  <p>Click <b class="ocpLegacyBold">Password Startup</b> if you want to require a password to start Windows. Use a complex password that contains a combination of upper case and lower case letters, numbers, and symbols. The startup password must be at least 12 characters long and can be up to 128 characters long.<br aria-hidden="true"><br aria-hidden="true">Note If you must remotely restart a computer that requires a password (if you use the <b class="ocpLegacyBold">Password Startup</b> option), a person must be at the local console during the restart. Use this option only if a trusted security administrator will be available to type the Startup password.</p>
                <meta itemprop="position" content="4">
      </li>
      <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                  <p>Click <b class="ocpLegacyBold">System Generated Password</b> if you do not want to require a startup password.<br aria-hidden="true"><br aria-hidden="true">Select either of the following options:</p>
                  <ul>
          <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                      <p>Click <b class="ocpLegacyBold">Store Startup Key on Floppy Disk</b> to store the system startup password on a floppy disk. This requires  that someone insert the floppy disk to start the operating system.</p>
                    <meta itemprop="position" content="1">
          </li>
          <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                      <p>Click <b class="ocpLegacyBold">Store Startup Key Locally</b> to store the encryption key on the hard disk of the local computer. This is the default option.</p>
                    <meta itemprop="position" content="2">
          </li>
        </ul>
                  <p>Click <b class="ocpLegacyBold">OK</b> two times to complete the procedure.<br aria-hidden="true"><br aria-hidden="true">Remove the SAM encryption key from the local hard disk by using the <b class="ocpLegacyBold">Store Startup Key on Floppy Disk</b> option for optimum security. This provides the highest level of protection for the SAM database. <br aria-hidden="true"><br aria-hidden="true">Always create a back-up floppy disk if you use the <b class="ocpLegacyBold">Store Startup Key on Floppy Disk</b> option. You can restart the system remotely if someone is available to insert the floppy disk into the computer when it restarts.</p>
                <meta itemprop="position" content="5">
      </li>
    </ol>
              <p>Note The Microsoft Windows NT 4.0 SAM database was not encrypted by default. You can encrypt the Windows NT 4.0 SAM database by using the SysKey utility.</p>
