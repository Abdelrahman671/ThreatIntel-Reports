<script type="application/ld+json">{"@context":"https:\/\/schema.org","@type":"BlogPosting","headline":"Threat Actors Rapidly Adopt Web3 IPFS Technology","name":"Threat Actors Rapidly Adopt Web3 IPFS Technology","description":"Web3 technologies are seeing widespread adoption \u2014 including by TAs. We discuss Web3 technology InterPlanetary File System (IPFS), and malicious use of it.","url":"https:\/\/unit42.paloaltonetworks.com\/ipfs-used-maliciously\/","mainEntityOfPage":"https:\/\/unit42.paloaltonetworks.com\/ipfs-used-maliciously\/","datePublished":"April 19, 2023","articleBody":"Executive Summary\r\nDuring 2022, analysts from Unit 42 observed the rampant adoption of the InterPlanetary File System (aka IPFS) being used as a vehicle for malicious intent. IPFS is a Web3 technology that decentralizes and distributes the storage of files and other data into a peer-to-peer network.\r\n\r\nLike any technology, IPFS can be abused by malicious threat actors. However, because the hosted content on IPFS is decentralized and distributed, there are challenges in locating and removing malicious content from the ecosystem, making it akin to bullet-proof hosting.\r\n\r\nFrom the last quarter of 2021 through the end of the last quarter of 2022, Palo Alto Networks detected an 893% increase in IPFS-related traffic. Our calculations also show that VirusTotal reported over 27,000% increase during the same period.\r\n\r\nThis increase in IPFS-related traffic is rife with notable increases in malicious activity, as can be expected. Analysts from Unit 42 observed numerous threat campaigns in 2022 that covered the gamut of phishing, credential theft, malicious payload distribution and general adoption by threat actors.\r\n\r\nSome of the key takeaways from our analysis include the following:\r\n\r\n \tThere was a significant jump in IPFS-related traffic at the beginning of 2022. Palo Alto Networks detected a 178% increase in IPFS-related traffic from the last quarter of 2021 to the first quarter of 2022, while VirusTotal reported more than 6,500% increase during that same reporting period.\r\n \tWe have identified threat actors discussing the adoption of the technology on forums in the dark web as well as threat actors selling services hosted on IPFS.\r\n \tWe have seen several types of cyberthreats using IPFS, including phishing, credential theft, command and control (C2) communications, and malicious payload distribution.\r\n\r\nPalo Alto Networks customers receive protections from the malware families discussed as well as their malicious components in the following ways:\r\n\r\n \tCortex XDR\r\n \tThe Next-Generation Firewall with Cloud-Delivered Security Services including WildFire and Advanced Threat Prevention.\r\n \tAdvanced URL Filtering and DNS Security can analyze and block malicious IPFS domains, including malware and phishing hosting URLs, and C2 domains.\r\n\r\n\r\n\r\n\r\nRelated Unit 42 Topics\r\nPhishing\r\n\r\n\r\n\r\nWhat Are Web3 and IPFS?\r\nBefore we delve into our observations around threat actor behavior associated with IPFS, it is important to have a general understanding of Web3 and also IPFS. IPFS is one of the technologies that supports Web3 infrastructures.\r\n\r\nWeb3 \u2013 or the third iteration of the web \u2013 is a new version of the internet that prioritizes decentralization using blockchain technology and tokens. With Web3, users can safeguard their data against censorship and manipulation without the need for a central authority.\r\n\r\nThis decentralization allows individuals to have ownership and control over their own content, which can be posted without fear of governments or tech companies removing it. However, cybercriminals can also exploit these same benefits to further their malicious activities.\r\n\r\nIPFS is a distributed file sharing system that was released in 2015. It is open source and uses a peer-to-peer (P2P) hypermedia protocol, making the internet faster, safer and more open.\r\n\r\nUnlike the traditional web, IPFS is content-oriented and searches for content identifiers in the form of hashes through a decentralized network, rather than specific locations. Content on IPFS can be accessed by establishing your own node on the IPFS network or using IPFS gateways, which are third-party web-based interfaces between the web and the IPFS network. IPFS is available through any IPFS gateway, and is not gateway-specific. These gateways allow users to view and pull content through HTTP requests, but they cannot alter or add to the content.\r\n\r\nMore information about Web3 and IPFS can be found at the following sites:\r\n\r\n \tWeb3\r\n \tIPFS\r\n\r\nIncrease in Overall Traffic\r\nDespite its relative obscurity, we observed a notable increase in IPFS traffic across Palo Alto Networks starting the first quarter of 2022, as seen in Figure 1. In the first quarter of 2022, our products detected a 178% increase in IPFS traffic as compared to what we logged at the end of the last quarter of 2021.\r\n\r\nThis traffic continued to increase following this initial observation:\r\n\r\n \tAn increase of 85% in the second quarter\r\n \tA 62% increase in the third quarter\r\n \tA 19% increase in the last quarter of 2022\r\n\r\nThis equated to an overall increase of 893%.\r\n\r\n[caption id=\"attachment_127747\" align=\"aligncenter\" width=\"900\"] Figure 1. Palo Alto Networks IPFS traffic.[\/caption]\r\n\r\nWe had also identified that IPFS-related traffic saw a similar increase on VirusTotal for the first quarter of 2022, with an increase of 6,503% when compared to the last quarter of 2021.\r\n\r\nSimilar to what was seen across Palo Alto Networks, VirusTotal continued to see the following increases quarter over quarter (as shown in Figure 2):\r\n\r\n \tA 68% increase in the second quarter\r\n \tA 69% increase in the third quarter\r\n \tA 48% increase in the last quarter of 2022\r\n\r\n[caption id=\"attachment_127749\" align=\"aligncenter\" width=\"900\"] Figure 2. VirusTotal IPFS submissions.[\/caption]\r\n\r\nAs shown in the observed traffic for IPFS represented in Figures 1 and 2, this increase can be directly attributed to the adoption of IPFS as a technology. Unfortunately, with the adoption of any new technology, there are always people who aim to use it with malicious intent. The notable increase in IPFS traffic we observed in Palo Alto Networks and VirusTotal submissions also includes substantial increases in malicious activity using IPFS.\r\n\r\nAnalysts from Unit 42 have observed adoption of IPFS for the following types of malicious activities over the past year:\r\n\r\n \tAdoption by threat actors\r\n \tUtilization for phishing and account credential theft\r\n \tUtilization by several malware intrusion sets for malicious payload distribution\r\n \tC2 communication\r\n\r\nObservations of Threat Actors Adopting IPFS\r\nUnit 42 researchers have observed threat actors discussing their use of IPFS, or their customers\u2019 adoption of this technology, particularly as it pertains to phishing and similar types of cybercrime. Threat actors often advertise their scam services, citing a variety of selling points including that sites were \u201cnot suspended online for a long time\u201d and that their \u201clink has no downtime.\u201d This is to say that IPFS has provided them with protection and longevity to their campaigns due to the nature of IPFS\u2019 distributed file system. These discussions are shown in Figures 3, 4, 5 and 6.\r\n\r\n[caption id=\"attachment_127751\" align=\"aligncenter\" width=\"900\"] Figure 3. Threat actor selling scam services using customer IPFS links.[\/caption]\r\n\r\n[caption id=\"attachment_127753\" align=\"aligncenter\" width=\"900\"] Figure 4. Threat actor selling IPFS phishing page.[\/caption]\r\n\r\n[caption id=\"attachment_127755\" align=\"aligncenter\" width=\"900\"] Figure 5. Threat actor selling IPFS phishing page.[\/caption]\r\n\r\n[caption id=\"attachment_127757\" align=\"aligncenter\" width=\"900\"] Figure 6. Threat actor selling IPFS phishing page.[\/caption]\r\n\r\nThreat actors are using public IPFS gateways as a means to deliver their malicious content. Without these internet accessible gateways, threat actors would not be able to reach their targets successfully at the scale necessary when using the IPFS network as part of their threat campaigns. This trend is seen in the use of internet accessible IPFS links found in numerous phishing and cybercrime campaigns where initial attack vectors are often email lures.\r\n\r\nIn the following sections, we will detail some of the threat campaigns we have seen during our analysis of malicious use of IPFS technology.\r\nPhishing\r\nFigure 7 shows that IPFS saw exponential growth in phishing-related network traffic, particularly in the last quarter of the year. Unlike traditional phishing pages hosted on the web, a hosting provider or moderating organization cannot easily remove IPFS phishing content.\r\n\r\nOnce published to the IPFS network, anyone can obtain and republish content on their own node. Phishing content can be hosted on multiple nodes and requests would have to be made to each host for content removal. Should any one host not agree to the removal, the content would be virtually impossible to remove.\r\n\r\nPhishing campaigns typically have shorter times to live (TTL) than other types of cybercrime due to content removal or suspension by site owners, hosting providers or moderators. The structure of IPFS allows criminals to prolong their campaign by making it more resilient to takedowns.\r\n\r\n[caption id=\"attachment_127759\" align=\"aligncenter\" width=\"900\"] Figure 7. Palo Alto Networks IPFS phishing URLs.[\/caption]\r\n\r\nIPFS phishing campaigns are similar to those of traditional phishing, where attackers mimic legitimate services and software (such as DHL, DocuSign and Adobe) to increase the potential of landing in someone\u2019s inbox. The ability to block these lures depends on what email security the receiving organization has in place. While some organizations set very strict rules within their secure email gateway and other security products, others do not out of fear that legitimate emails will be affected.\r\n\r\nNote that the names and logos shown below are the work of a threat actor attempting to impersonate a legitimate organization and do not represent an actual affiliation with that organization. The threat actor\u2019s impersonation does not imply a vulnerability in the legitimate organization\u2019s products or services.\r\n\r\nIn the example below, an email lure mimicking the DHL brand contains an attachment. Within that attachment, there is an IPFS link to the actual phishing page.\r\n\r\n[caption id=\"attachment_127761\" align=\"aligncenter\" width=\"900\"] Figure 8. DHL-themed email lure with attachment submitted to VirusTotal.[\/caption]\r\n\r\nOnce the user clicks the attachment shown in Figure 8, a fake invoice mimicking Adobe PDF branding is previewed, as shown in Figure 9. Instead of opening a PDF, the \u201cOpen\u201d button is actually an IPFS link that redirects the user to the actual phishing page. This page can be accessed via the internet through an IPFS gateway (also highlighted in Figure 9).\r\n\r\n[caption id=\"attachment_127763\" align=\"aligncenter\" width=\"900\"] Figure 9. Attachment with link from the DHL lure.[\/caption]\r\n\r\nThe IPFS URL takes the user to an Adobe phishing page (shown in Figure 10) via the ipfs[.]io gateway, which then attempts to steal the user\u2019s login credentials.\r\n\r\n[caption id=\"attachment_127765\" align=\"aligncenter\" width=\"900\"] Figure 10. IPFS phishing link found in the attachment from the DHL-themed lure.[\/caption]\r\n\r\nAs in other Web3 technologies, typical data exfiltration methods are not possible on the IPFS network. There is no way for the threat actor to receive the data the victim inputs into the form meant to steal their credentials.\r\n\r\nStandard web forms use HTML frontends to display content for presentation and backend form processors to collect, process and send the data to a web server. IPFS does not contain the same or similar technology to handle these dynamic functions.\r\n\r\nPeople using IPFS are simply pulling or retrieving a read-only copy of the data, not interacting with it. Phishing pages hosted behind IPFS gateways rely on a number of other techniques instead.\r\n\r\nFor example, attackers can use embedded script code in webpages that collect account credentials. They can also use headless forms, which are static user forms that can be filled out and collected. The form fields are mapped to a JSON model to facilitate API-driven theft by sending to a backend system via an API. The information is gathered via HTTP POST requests, which are sent to the attacker, where it can be used for additional nefarious purposes.\r\n\r\nLet\u2019s examine the use of both of these credential theft techniques.\r\nUnescaped Script in Nillis.html\r\nFigure 10 shows an example of a phish mimicking Microsoft that is hosted as an HTML page. The IPFS URL to contact this page is\r\n\r\nhxxps[:\/\/]bafybeicw4jjag57bk3czji7wjznkkpbocg27qk3fjvqh5krbrfiqbksr2a[.]ipfs[.]w3s[.]link\/Nills[.]html.\r\n\r\n[caption id=\"attachment_127767\" align=\"aligncenter\" width=\"900\"] Figure 11. Phishing page for Nills.html.[\/caption]\r\n\r\nTo understand how the credentials will be exfiltrated, let\u2019s review the source content of the webpage.\r\n\r\nFigure 12 shows a function named WriteHTMLtoJS. The purpose of this function is to write HTML to JavaScript (JS), and to also unescape the contents. The unescape JS function is responsible for decoding hexadecimal sequences with their actual ASCII character values. This representation can be seen on lines 3 and 4 of Figure 12.\r\n\r\n[caption id=\"attachment_127769\" align=\"aligncenter\" width=\"900\"] Figure 12. Source view of Nills.html webpage.[\/caption]\r\n\r\nDecoding and analyzing the unescaped content (shown in Figure 13) reveals a pair of script tags and an observed URL, which is app[.]headlessforms[.]cloud. The phishing page appears to contact this URL.\r\n\r\nA review of the headless forms indicates this phishing page is using a method for managing user forms where data can be captured on a third-party backend without having to design and\/or develop a frontend web application.\r\n\r\n[caption id=\"attachment_127771\" align=\"aligncenter\" width=\"900\"] Figure 13. Decoded view of Nills.html containing the location of credential exfiltration.[\/caption]\r\n\r\nAfter a victim enters an account username and password credential, it will be sent via an HTTP POST request (shown in Figure 14). The string of characters at the end of the URI (GjCP9S9nke) is a unique identifying token associated with the actor on the headless forms platform.\r\n\r\n[caption id=\"attachment_127773\" align=\"aligncenter\" width=\"900\"] Figure 14. HTTP POST and captured credentials.[\/caption]\r\n\r\nLet\u2019s move on to examining another phishing campaign using another collection technique.\r\nHTTP POST in new.html\r\nAnother phishing page (shown in Figure 15) that we discovered mimicking the Microsoft brand, which was also hosted on IPFS, is called new.html. The associated IPFS phishing URL is:\r\n\r\nhxxp[:\/\/]bafybeifm5vcoj35hhuxf7ha3gg6asrrlrwu3bvcysgmrvygnm3qjmugwxq[.]ipfs[.]w3s[.]link\/new[.]html?email=\r\n\r\n&nbsp;\r\n\r\n[caption id=\"attachment_127775\" align=\"aligncenter\" width=\"900\"] Figure 15. Phishing page for new.html.[\/caption]\r\n\r\nReview of the webpage source indicates a similar JS function to the one referenced previously, where it unescapes the contents, decoding them to the ASCII values. The unescape function can be seen in Figure 16.\r\n\r\n[caption id=\"attachment_127777\" align=\"aligncenter\" width=\"900\"] Figure 16. Source view of new.html.[\/caption]\r\n\r\nDecoding the content reveals a snippet of interest that is located near the bottom of a sizable amount of code, as shown in Figure 17.\r\n\r\n[caption id=\"attachment_127779\" align=\"aligncenter\" width=\"763\"] Figure 17. new.html URL where data is POSTed.[\/caption]\r\n\r\nThe snippet in Figure 17 shows the external URL (fairpartner[.]ru) registered to a click function event for the submit button. This URL will be contacted with the data provided by an HTTP POST request, as shown in Figure 18.\r\n\r\n[caption id=\"attachment_127781\" align=\"aligncenter\" width=\"900\"] Figure 18. DNS request for fairpartner.ru.[\/caption]\r\n\r\nAt the time this research was compiled, we were unable to capture the credential theft as the domain was no longer reachable. This highlights the resiliency of using a third-party service such as headless forms versus the traditional hosting used by this campaign for capturing credentials.\r\n\r\nIPFS is not only used by threat actors for phishing. It is also used for malware intrusion sets and ransomware.\r\n\r\nRansomware-as-a-service (RaaS) operators such as RansomEXX are known to use the IPFS network to release the stolen data of their victims. Malware families such as Smoke Loader, XLoader, XMRig and OriginLogger often use IPFS links for malicious payload delivery. IPStorm and Dark Utilities use the IPFS network for infrastructure and staging.\r\n\r\nLet\u2019s review how each of these groups uses IPFS.\r\nMalware and Red Team Tools\r\nThreat actors use IPFS gateways in a couple different ways. These can be categorized as either delivery methods or for use as infrastructure for hosting or staging payloads, or for acting as a decentralized C2 channel.\r\n\r\nThe following malware families have been using IPFS throughout 2022. The malware family Dark Utilities has been using IPFS gateways for staging malicious payloads. IPStorm uses IPFS gateways as a C2 channel for P2P communications.\r\n\r\nAttackers have also used IPFS gateways to deliver various red team tools and malware such as the following:\r\n\r\n \tOriginLogger\r\n \tXLoader\r\n \tXMRig\r\n \tMetasploit\r\n\r\nWe\u2019ll describe how these threats operate at a high level, including how IPFS was used to facilitate malicious operations.\r\nOriginLogger\r\nThe OriginLogger malware family has been around since 2019. It is the continuation of the Agent Tesla remote access Trojan. It is written in .NET and functions as a highly evasive information stealer. This threat typically targets keystrokes and clipboard data, which is communicated back to an actor-controlled server through a C2 channel.\r\n\r\nUnit 42 researchers discovered an email lure masquerading as an overdue invoice, with an XLL attachment (as shown in Figure 19). Upon opening the XLL file, an HTTP GET request is sent to the IPFS URL:\r\n\r\nhxxps[:\/\/]ipfs[.]io\/ipfs\/QmXtVwamvHvXZzuEZcMn2xDsPRKN8uS17YCUzTiGx1rYnv?filename=file-05-2022.exe\r\n\r\nThis URL is used for downloading the OriginLogger payload via IPFS gateway.\r\n\r\n[caption id=\"attachment_127783\" align=\"aligncenter\" width=\"796\"] Figure 19. Attachment - 30-MAY-22.xll.[\/caption]\r\n\r\nFigure 20 shows the second delivery example of OriginLogger. It is important to note that there is no single solution for removing malicious content from the IPFS networks. Different approaches might be more or less effective depending on the specific circumstances and the participation of the owners of the decentralized networks in which the content is ultimately hosted.\r\n\r\nThe significant increase in IPFS-related traffic we observed during 2022 at Palo Alto Networks and VirusTotal data highlights the growing popularity of this technology among threat actors. The threat campaigns observed by analysts from Unit 42 demonstrate the versatility of IPFS for carrying out a variety of malicious activities, including phishing, credential theft, C2 communications and payload distribution.\r\n\r\nThreat actors\u2019 adoption of IPFS, as well as the sale of services hosted on IPFS, underscores the need for continued vigilance and proactive measures to detect and mitigate threats on this platform. <p>During 2022, analysts from Unit 42 observed the rampant adoption of the InterPlanetary File System (aka IPFS) being used as a vehicle for malicious intent. IPFS is a Web3 technology that decentralizes and distributes the storage of files and other data into a peer-to-peer network.</p>
<p>Like any technology, IPFS can be abused by malicious threat actors. However, because the hosted content on IPFS is decentralized and distributed, there are challenges in locating and removing malicious content from the ecosystem, making it akin to bullet-proof hosting.</p>
<p>From the <strong>last quarter of 2021 through the end of the last quarter of 2022</strong>, Palo Alto Networks detected an 893% increase in IPFS-related traffic. Our calculations also show that VirusTotal reported over 27,000% increase during the same period.</p>
<p>This increase in IPFS-related traffic is rife with notable increases in malicious activity, as can be expected. Analysts from Unit 42 observed numerous threat campaigns in 2022 that covered the gamut of phishing, credential theft, malicious payload distribution and general adoption by threat actors.</p>
<p>Some of the key takeaways from our analysis include the following:</p>
<ul>
<li>There was a significant jump in IPFS-related traffic at the beginning of 2022. Palo Alto Networks detected a <strong>178% increase</strong> in IPFS-related traffic from the <strong>last quarter of 2021 to the first quarter of 2022</strong>, while <strong>VirusTotal reported more than 6,500% increase</strong> during that same reporting period.</li>
<li>We have identified threat actors discussing the adoption of the technology on forums in the dark web as well as threat actors selling services hosted on IPFS.</li>
<li>We have seen several types of cyberthreats using IPFS, including phishing, credential theft, command and control (C2) communications, and malicious payload distribution.</li>
</ul>

<h2><a id="post-127743-_wven14kmgum2"></a>What Are Web3 and IPFS?</h2>
<p>Before we delve into our observations around threat actor behavior associated with IPFS, it is important to have a general understanding of Web3 and also IPFS. IPFS is one of the technologies that supports Web3 infrastructures.</p>
<p>Web3 – or the third iteration of the web – is a new version of the internet that prioritizes decentralization using blockchain technology and tokens. With Web3, users can safeguard their data against censorship and manipulation without the need for a central authority.</p>
<p>This decentralization allows individuals to have ownership and control over their own content, which can be posted without fear of governments or tech companies removing it. However, cybercriminals can also exploit these same benefits to further their malicious activities.</p>
<p>IPFS is a distributed file sharing system that was released in 2015. It is open source and uses a peer-to-peer (P2P) hypermedia protocol, making the internet faster, safer and more open.</p>
<p>Unlike the traditional web, IPFS is content-oriented and searches for content identifiers in the form of hashes through a decentralized network, rather than specific locations. Content on IPFS can be accessed by establishing your own node on the IPFS network or using IPFS gateways, which are third-party web-based interfaces between the web and the IPFS network. IPFS is available through any IPFS gateway, and is not gateway-specific. These gateways allow users to view and pull content through HTTP requests, but they cannot alter or add to the content.</p>
<p>More information about Web3 and IPFS can be found at the following sites:</p>
<ul>
<li><a href="https://en.wikipedia.org/wiki/Web3">Web3</a></li>
<li><a href="https://github.com/ipfs/ipfs">IPFS</a></li>
</ul>

<p>Threat actors are using public IPFS gateways as a means to deliver their malicious content. Without these internet accessible gateways, threat actors would not be able to reach their targets successfully at the scale necessary when using the IPFS network as part of their threat campaigns. This trend is seen in the use of internet accessible IPFS links found in numerous phishing and cybercrime campaigns where initial attack vectors are often email lures.</p>

<h2><a id="post-127743-_5rgxzypfamvk"></a>Phishing</h2>
<p>Figure 7 shows that IPFS saw exponential growth in phishing-related network traffic, particularly in the last quarter of the year. Unlike traditional phishing pages hosted on the web, a hosting provider or moderating organization cannot easily remove IPFS phishing content.</p>
<p>Once published to the IPFS network, anyone can obtain and republish content on their own node. Phishing content can be hosted on multiple nodes and requests would have to be made to each host for content removal. Should any one host not agree to the removal, the content would be virtually impossible to remove.</p>
<p>Phishing campaigns typically have shorter times to live (TTL) than other types of cybercrime due to content removal or suspension by site owners, hosting providers or moderators. The structure of IPFS allows criminals to prolong their campaign by making it more resilient to takedowns.</p>
<figure id="attachment_127759" aria-describedby="caption-attachment-127759" style="width: 900px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-127759" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/04/chart-2.png" alt="Image 7 is a graph of the increase of Palo Alto Networks IPFS phishing URLs change over time. There is an enormous increase, especially from quarter three to quarter four of 2022." width="900" height="568" /><figcaption id="caption-attachment-127759" class="wp-caption-text">Figure 7. Palo Alto Networks IPFS phishing URLs.</figcaption></figure>
<p>IPFS phishing campaigns are similar to those of traditional phishing, where attackers mimic legitimate services and software (such as DHL, DocuSign and Adobe) to increase the potential of landing in someone’s inbox. The ability to block these lures depends on what email security the receiving organization has in place. While some organizations set very strict rules within their secure email gateway and other security products, others do not out of fear that legitimate emails will be affected.</p>
<p>Note that the names and logos shown below are the work of a threat actor attempting to impersonate a legitimate organization and do not represent an actual affiliation with that organization. The threat actor’s impersonation does not imply a vulnerability in the legitimate organization’s products or services.</p>
<p>In the example below, an email lure mimicking the DHL brand contains an attachment. Within that attachment, there is an IPFS link to the actual phishing page.</p>

<figure id="attachment_127765" aria-describedby="caption-attachment-127765" style="width: 900px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-127765" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/04/word-image-127743-10.png" alt="Image 10 is a screenshot of an IPFS phishing link. <p>As in other Web3 technologies, typical data exfiltration methods are not possible on the IPFS network. There is no way for the threat actor to receive the data the victim inputs into the form meant to steal their credentials.</p>
<p>Standard web forms use HTML frontends to display content for presentation and backend form processors to collect, process and send the data to a web server. IPFS does not contain the same or similar technology to handle these dynamic functions.</p>
<p>People using IPFS are simply pulling or retrieving a read-only copy of the data, not interacting with it. Phishing pages hosted behind IPFS gateways rely on a number of other techniques instead.</p>
<p>For example, attackers can use embedded script code in webpages that collect account credentials. They can also use headless forms, which are static user forms that can be filled out and collected. The form fields are mapped to a JSON model to facilitate API-driven theft by sending to a backend system via an API. The information is gathered via HTTP POST requests, which are sent to the attacker, where it can be used for additional nefarious purposes.</p>
<p>Let’s examine the use of both of these credential theft techniques.</p>

<p>IPFS is not only used by threat actors for phishing. It is also used for malware intrusion sets and ransomware.</p>
<p>Ransomware-as-a-service (RaaS) operators such as RansomEXX are known to use the IPFS network to release the stolen data of their victims. Malware families such as Smoke Loader, XLoader, XMRig and OriginLogger often use IPFS links for malicious payload delivery. IPStorm and Dark Utilities use the IPFS network for infrastructure and staging.</p>
<p>Let’s review how each of these groups uses IPFS.</p>
<h2><a id="post-127743-_r9zdk4klab1v"></a>Malware and Red Team Tools</h2>
<p>Threat actors use IPFS gateways in a couple different ways. These can be categorized as either delivery methods or for use as infrastructure for hosting or staging payloads, or for acting as a decentralized C2 channel.</p>
<p>The following malware families have been using IPFS throughout 2022. The malware family Dark Utilities has been using IPFS gateways for staging malicious payloads. IPStorm uses IPFS gateways as a C2 channel for P2P communications.</p>
<p>Attackers have also used IPFS gateways to deliver various red team tools and malware such as the following:</p>
<ul>
<li>OriginLogger</li>
<li>XLoader</li>
<li>XMRig</li>
<li>Metasploit</li>
</ul>
<p>We’ll describe how these threats operate at a high level, including how IPFS was used to facilitate malicious operations.</p>
<h3><a id="post-127743-_nzzf4jjrtwrv"></a>OriginLogger</h3>
<p>The <a href="https://unit42.paloaltonetworks.com/atoms/agent-tesla/">OriginLogger</a> malware family has been around since 2019. It is the continuation of the Agent Tesla remote access Trojan. It is written in .NET and functions as a highly evasive information stealer. This threat typically targets keystrokes and clipboard data, which is communicated back to an actor-controlled server through a C2 channel.</p>
<p>Unit 42 researchers discovered an email lure masquerading as an overdue invoice, with an XLL attachment (as shown in Figure 19). Upon opening the XLL file, an HTTP GET request is sent to the IPFS URL:</p>
<p><span style="font-family: 'courier new', courier, monospace;">hxxps[://]ipfs[.]io/ipfs/QmXtVwamvHvXZzuEZcMn2xDsPRKN8uS17YCUzTiGx1rYnv?filename=file-05-2022.exe</span></p>
<p>This URL is used for downloading the OriginLogger payload via IPFS gateway.</p>

<p><span style="font-family: 'courier new', courier, monospace;">hxxps[://]ipfs[.]io/ipfs/QmXtVwamvHvXZzuEZcMn2xDsPRKN8uS17YCUzTiGx1rYnv?filename=file-05-2022.exe</span></p>

<h3><a id="post-127743-_ghfjxc7bt5j"></a>XLoader</h3>
<p>The <a href="https://blogs.blackberry.com/en/2021/09/threat-thursday-xloader-infostealer" target="_blank" rel="noopener">XLoader</a> malware family has been around since 2020, as a rebrand of FormBook. XLoader has been advertised as a malware-as-a-service (MaaS) offering that can run on Windows as well as macOS. XLoader functions as a stealer that targets browser and login credentials, and it’s also capable of keylogging and capturing screenshots.</p>
<p>Unit 42 researchers observed the following in the wild (ITW) URLs related to the hosting of XLoader payloads:</p>
<p><span style="font-family: 'courier new', courier, monospace;">hxxp[://]bafybeiafb63z73aoz3d4jdpve2rhlwo6ujlzjyri26z63flqnara2dqwoa[.]ipfs[.]dweb[.]link/order.exe</span></p>

<h3><a id="post-127743-_kzucpjrdjj13"></a>XMRig</h3>
<p><a href="https://www.welivesecurity.com/2017/09/28/monero-money-mining-malware/" target="_blank" rel="noopener">XMRig</a> is a cryptomining malware family that has been around since 2017. XMRig is an open-source utility that various threat groups have adapted to be delivered either in malspam or in campaigns that exploit vulnerable public facing applications, leading to XMRig being deployed.</p>

<h3><a id="post-127743-_grfslpp4tt85"></a>Metasploit</h3>
<p>The <a href="https://malpedia.caad.fkie.fraunhofer.de/details/win.meterpreter" target="_blank" rel="noopener">Metasploit Framework</a> is a penetration testing toolkit that has been around since the early 2000s. Metasploit contains exploits, modules, payloads and auxiliary capabilities. The primary use case for this toolkit is to generate payloads and achieve code execution to set up a communication channel with a victim machine. This allows someone using the toolkit to gain access to and control of an environment or user.</p>
<p>In late March 2023, Unit 42 researchers observed an ITW IPFS domain:</p>
<p><span style="font-family: 'courier new', courier, monospace;">hxxps[://]bafybeihtuhj7ezvjbtig75qpv43t7eh6dmcnarlm454fx5yjb4uzqbidpy[.]ipfs[.]infura-ipfs[.]io</span></p>

<h3><a id="post-127743-_8c5rrs1toy5v"></a>IPStorm</h3>
<p>The IPStorm malware family has been around since 2019, as reported by researchers at <a href="https://www.anomali.com/blog/the-interplanetary-storm-new-malware-in-wild-using-interplanetary-file-systems-ipfs-p2p-network" target="_blank" rel="noopener">Anomali</a>. <a href="https://www.bitdefender.com/files/News/CaseStudies/study/376/Bitdefender-Whitepaper-IPStorm.pdf" target="_blank" rel="noopener">IPStorm</a> is a notable botnet as it uses IPFS as a C2 channel.</p>
<p>IPStorm uses the open-source library <a href="https://docs.libp2p.io/concepts/introduction/overview/" target="_blank" rel="noopener">libp2p</a> for its networking stack and P2P communication over the IPFS network. The executable is written in Golang and contains multiple notable package names that can be used to identify the malware family. This threat uses the folder name <span style="font-family: 'courier new', courier, monospace;">/storm</span> before the module name, as shown in Figure 25.</p>

<h3><a id="post-127743-_iddhtlit0rfn"></a>Dark Utilities</h3>
<p>The <a href="https://blog.talosintelligence.com/dark-utilities/" target="_blank" rel="noopener">Dark Utilities</a> malware family was initially identified in 2022 by Cisco Talos. Dark Utilities is a C2-as-a-service product offering that uses IPFS as its delivery channel of choice.</p>
<p>This threat can be compiled for Windows, macOS, Linux or various embedded systems. The product offers cryptomining and DDoS capabilities, as well as typical remote access functionality.</p>

<h2><a id="post-127743-_psff5d5qkobe"></a>Conclusion</h2>
<p>Threat actors increasing their adoption of IPFS is a growing concern because, as a decentralized and distributed storage technology, IPFS poses unique challenges in locating and removing malicious content from the ecosystem. It is important to note that there is no single solution for removing malicious content from the IPFS networks. Different approaches might be more or less effective depending on the specific circumstances and the participation of the owners of the decentralized networks in which the content is ultimately hosted.</p>
<p>The significant increase in IPFS-related traffic we observed during 2022 at Palo Alto Networks and VirusTotal data highlights the growing popularity of this technology among threat actors. The threat campaigns observed by analysts from Unit 42 demonstrate the versatility of IPFS for carrying out a variety of malicious activities, including phishing, credential theft, C2 communications and payload distribution.</p>
<p>Threat actors’ adoption of IPFS, as well as the sale of services hosted on IPFS, underscores the need for continued vigilance and proactive measures to detect and mitigate threats on this platform. It is imperative that the cybersecurity community remains vigilant and takes proactive steps to stay ahead of evolving threats on IPFS and other emerging technologies.</p>

(function($) {
	// Migrated from the unit42-v5 + Modifications
	var subscribeSuccess = false;
	var email = document.getElementById('newsletter-email');
	var subscription_form = document.getElementById('unit42footerSubscription_form');
	var subscription_form_button = document.getElementById('unit42footerSubscription_form_button');
	window.captchaComplete = function() {
		subscribeSuccess = true;
		if ($(mail).val() != '' && isEmail($(mail).val())) {
			$(subscription_form_button).removeClass('is-disabled');
		}
		setTimeout(function() {
			$(email).focus();
			$('.g-recaptcha iframe').attr('tabindex', '-1');
		}, 100)
	}
	window.captchaExpires = function() {
		subscribeSuccess = false;
		$(subscription_form_button).addClass('is-disabled', true);
	} 
	$(subscription_form).submit(function(e) {
		e.preventDefault();
		e.stopImmediatePropagation();
		updateEmailMask();
		var success = true;
		var form = $(this);
		var mail = form.find('input[name="Email"]');
		
		if (mail.val() === '') {
			mail.addClass('has-error');
			showError(1);
			success = false;	
		}  else if (!isEmail(mail.val())){
			showError(2);
			success = false;
		}
		else {
			mail.removeClass('has-error');
			$('.error-mail').addClass('d-none');
		}

		if (!subscribeSuccess) {
			$('.error-recaptcha').removeClass('d-none');
		} else {
			$('.error-recaptcha').addClass('d-none');
		}

		if (success && subscribeSuccess) {
			$.ajax({
				type: 'POST',
				url: form.attr('action'),
				data: form.serialize(),
				beforeSend: function() {
                 form.find('button').addClass('is-loading');
    			},
				success: function(msg) {
					form.find('.form-success-message').html('<p class="success-message">You have been successfully subscribed</p>');
					form.find('button').removeClass('is-loading');
					$(email).val('');
					clearError();
				},
				error: function(jqXHR, textStatus, errorThrown) {
					$(subscription_form_button).addClass('is-disabled', true);
					form.find('button').removeClass('is-loading');
					
				}
			});
		}
		return false;
	});
	function showError(error_type){
		if(error_type == 1) {
			$('.error-mail').text("Please enter the email address.").addClass('error-show');
			$(subscription_form_button).addClass('is-disabled');
		} else if(error_type == 2){
			$('.error-mail').text("Please provide a valid e-mail address.").addClass('error-show');
			$(subscription_form_button).addClass('is-disabled');
		}	
		$(subscription_form_button).removeClass('is-loading');
	}
	function clearError(){
		$('.error-mail').text("").removeClass('error-show');;
		$(subscription_form_button).removeClass('is-loading');
		$(subscription_form_button).removeClass('is-disabled');
	}
	
	$(email).on('input', function (event) { 
		var email = $(this).val();
		if (isEmail(email) ) {			
			clearError();
		} else if(email == ""){
			clearError();
		} else{
			showError(2);			  
		}
    });

	function isEmail(email) {
		var re = /^(([^<>()\[\]\\.,;:\s@"]+(\.[^<>()\[\]\\.,;:\s@"]+)*)|(".+"))@((\[[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\])|(([a-zA-Z\-0-9]+\.)+[a-zA-Z]{2,}))$/;
		return re.test(String(email).toLowerCase());
	}
	
	var captcha_loaded = false;
         if(!captcha_loaded){
            // recaptcha on foucs call
            $(document).on('change paste keyup', '#newsletter-email', function () {
               if($('.g-recaptcha').hasClass('d-none')){
                $('.g-recaptcha').removeClass('d-none');
               }
               if(!captcha_loaded ){
                  captcha_loaded  = true;
                  // trigger loading api.js (recaptcha.js) script
                  var head = document.getElementsByTagName('head')[0];
                  var script = document.createElement('script');
                  script.type = 'text/javascript';
                  script.src = 'https://www.google.com/recaptcha/api.js?hl=en_US';
                  head.appendChild(script);
               }
            });
        }
	
	function updateEmailMask() {
		var email = $("#unit42footerSubscription_form input[name='Email']").val();
		if (email && email.trim() != '') {
			var maskedEmail = maskEmailAddress(email);
			$("#unit42footerSubscription_form input[name='emailFormMask']").val(maskedEmail);
		}
	}

	function maskEmailAddress (emailAddress) {
		function mask(str) {
			var strLen = str.length;
			if (strLen > 4) {
				return str.substr(0, 1) + str.substr(1, strLen - 1).replace(/\w/g, '*') + str.substr(-1,1);
			}
			return str.replace(/\w/g, '*');
		}
		return emailAddress.replace(/([\w.]+)@([\w.]+)(\.[\w.]+)/g, function (m, p1, p2, p3) {
			return mask(p1) + '@' + mask(p2) + p3;
		});

		return emailAddress;
	}

}(jQuery));
