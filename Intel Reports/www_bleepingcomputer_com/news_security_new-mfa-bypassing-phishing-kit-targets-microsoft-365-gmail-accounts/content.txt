<article><div class="article_section">
<h1>New MFA-bypassing phishing kit targets Microsoft 365, Gmail accounts</h1>
<div class="cz-news-story-title-section">
<div class="cz-news-title-left-area">
By <h6><a rel="author" href="https://www.bleepingcomputer.com/author/bill-toulas/" class="author"><span itemprop="author" itemscope itemtype="https://schema.org/Person"><span itemprop="name">Bill Toulas</span></span></a></h6>
</div>
<meta itemprop="articleSection" content="Security"><div class="cz-news-title-right-area">
<ul><li class="cz-news-date">March 25, 2024</li>
<li class="cz-news-time">12:56 PM</li>
<li class="cz-news-comment"><a href="#comments">3</a></li>
</ul></div>
</div>
<div class="articleBody">
<p style="text-align:center"><img alt="New MFA-bypassing phishing kit targets Microsoft 365, Gmail accounts" height="900" src="https://www.bleepstatic.com/content/hl-images/2024/03/25/phishing.jpg" width="1600"></p>
<p>Cybercriminals have been increasingly using a new phishing-as-a-service (PhaaS) platform named 'Tycoon 2FA' to target&nbsp;Microsoft 365 and Gmail accounts and bypass two-factor authentication (2FA) protection.</p>
<p>Tycoon 2FA was discovered by <a href="https://blog.sekoia.io/tycoon-2fa-an-in-depth-analysis-of-the-latest-version-of-the-aitm-phishing-kit/" target="_blank" rel="nofollow noopener">Sekoia analysts</a> in October 2023 during routine threat hunting, but it has been active since at least August 2023, when the Saad Tycoon group&nbsp;offered it through private Telegram channels.</p>
<p>The PhaaS kit shares similarities with other adversary-in-the-middle (AitM) platforms, such as Dadsec OTT, suggesting possible code reuse or a collaboration between developers.</p>
<p>In 2024, Tycoon 2FA released a new version that is stealthier,&nbsp;indicating a continuous effort to improve the kit. Currently, the service leverages 1,100 domains and has been observed in&nbsp;thousands of phishing attacks.</p>
<div style="text-align:center">
<figure class="image" style="display:inline-block"><img alt="Tycoon 2FA website" height="427" src="https://www.bleepstatic.com/images/news/u/1220909/2024/Phishing/tycoon-site.png" width="658"><figcaption><strong>Tycoon 2FA website</strong> <em>(Sekoia)</em></figcaption></figure></div>
<h2>Tycoon 2FA attacks</h2>
<p>Tycoon 2FA attacks involve a multi-step process where the threat actor steals session cookies by using a reverse proxy server hosting the phishing web page, which intercepts the victim's input and relays them to the legitimate service.</p>
<p>"Once the user completes the MFA challenge, and the authentication is successful, the server in the middle captures session cookies," Skoia explains. This way, the attacker can replay a user's session and bypass multi-factor authentication (MFA) mechanisms.</p>
<p>Sekoia's report describes the attacks in seven distinct stages as described below:</p>
<ul><li><strong>Stage 0</strong> &ndash; Attackers distribute malicious links via emails with embedded URLs or QR codes, tricking victims into accessing phishing pages.</li>
<li><strong>Stage 1</strong> &ndash; A security challenge (Cloudflare Turnstile) filters out bots, allowing only human interactions to proceed to the deceptive phishing site.</li>
<li><strong>Stage 2</strong> &ndash; Background scripts extract the victim's email from the URL to customize the phishing attack.</li>
<li><strong>Stage 3</strong> &ndash; Users are quietly redirected to another part of the phishing site, moving them closer to the fake login page.</li>
<li><strong>Stage 4</strong> &ndash; This stage presents a fake Microsoft login page to steal credentials, using WebSockets for data exfiltration.</li>
<li><strong>Stage 5</strong> &ndash; The kit mimics a 2FA challenge, intercepting the 2FA token or response to bypass security measures.</li>
<li><strong>Stage 6</strong> &ndash; Finally, victims are directed to a legitimate-looking page, obscuring the phishing attack's success.</li>
</ul><p>An overview of the attack is described with the diagram below, which includes all the steps of the process.</p>
<div style="text-align:center">
<figure class="image" style="display:inline-block"><img alt="Tycoon 2FA attack overview" height="600" src="https://www.bleepstatic.com/images/news/u/1220909/2024/Phishing/diagram.png" width="1191"><figcaption><strong>Tycoon 2FA attack overview</strong> <em>(Sekoia)</em></figcaption></figure></div>
<h2>Evolution and scale</h2>
<p>Sekoia reports that the latest version of the Tycoon 2FA phishing kit, released this year, has introduced significant modifications that improve the phishing and evasion capabilities.</p>
<p>Key changes include updates to the JavaScript and HTML code, alterations in the order of resource retrieval, and more extensive filtering to block traffic from bots and analytical tools.</p>
<p>For example, the kit now delays loading malicious resources until after the Cloudflare Turnstile challenge is resolved, using pseudorandom names for URLs to obscure its activities.</p>
<p>Also, Tor network traffic or IP addresses linked to data centers are now better identified, while traffic is rejected based on specific user-agent strings.</p>
<p>Regarding the scale of operations, Sekoia reports that it's substantial, as there's evidence of a broad user base of cybercriminals currently utilizing Tycoon 2FA for phishing operations.</p>
<p>The Bitcoin wallet linked to the operators has recorded over 1,800 transactions since October 2019, with a notable increment starting August 2023, when the kit was launched.</p>
<p>Over 530 transactions were over $120, which is the entry price for a 10-day phishing link. By mid-March 2024, the threat actors' wallet had received a total of $394,015 worth of cryptocurrency.</p>
<p>Tycoon 2FA is just a recent addition to a PhaaS space that already offers cybercriminals plenty of options. Other notable platforms that can bypass 2FA protections include <a href="https://www.bleepingcomputer.com/news/security/labhost-cybercrime-service-lets-anyone-phish-canadian-bank-users/" target="_blank">LabHost</a>, <a href="https://www.bleepingcomputer.com/news/security/new-greatness-service-simplifies-microsoft-365-phishing-attacks/" target="_blank">Greatness</a>, and <a href="https://www.bleepingcomputer.com/news/security/robin-banks-phishing-service-returns-to-steal-banking-accounts/" target="_blank">Robin Banks</a>.</p>
<p>For a <a href="http://github.com/SEKOIA-IO/Community/blob/main/IOCs/tycoon2fa/tycoon2fa_iocs_20240325.csv" target="_blank" rel="nofollow noopener">list of the indicators of compromise</a> (IoCs) linked to the Tycoon 2FA operation, Sekoia makes available a repository with over 50 entries.</p>
<hr><p><em>Update 3/27</em> - A Google spokesperson has sent BleepingComputer the following comment regarding the Tycoon 2FA threat:</p>
<blockquote>
<p>These are well-known types of phishing attacks, and we have numerous protections to combat such attacks, including passkeys, which substantially reduce the impact of phishing and other social engineering attacks.</p>
<p>Google research has shown that security keys provide a stronger protection against automated bots, bulk phishing attacks, and targeted attacks than SMS, app-based one-time passwords, and other forms of traditional two-factor authentication (2FA).</p>
</blockquote>

function cz_strip_tags(input, allowed) {
  allowed = (((allowed || '') + '')
    .toLowerCase()
    .match(/<[a-z][a-z0-9]*>/g) || [])
    .join(''); // making sure the allowed arg is a string containing only tags in lowercase (<a><b><c>)
  var tags = /<\/?([a-z][a-z0-9]*)\b[^>]*>/gi,
    commentsAndPhpTags = /<!--[\s\S]*?-->|<\?(?:php)?[\s\S]*?\?>/gi;
  return input.replace(commentsAndPhpTags, '')
    .replace(tags, function($0, $1) {
      return allowed.indexOf('<' + $1.toLowerCase() + '>') > -1 ? $0 : '';
    });
}
function cz_br2nl(str) {
	var regex = /<br\s*[\/]?>/gi;
    //var pure_str = str.replace(regex,"\n");
    var pure_str = str.replace(regex,"");
    return cz_strip_tags(pure_str,'');
}
