													<h2 id="introduction">Introduction</h2>
<p>Stealing cryptocurrencies is nothing new. For example, the Mt. Gox exchange was robbed of many bitcoins <a href="https://www.wired.com/2014/02/bitcoins-mt-gox-implodes-2/" target="_blank" rel="noopener">back in the beginning of 2010s</a>. Attackers such as those behind the Coinvault ransomware were <a href="https://securelist.com/coinvault-are-we-reaching-the-end-of-the-nightmare/72187/" target="_blank" rel="noopener">after your Bitcoin </a><a href="https://securelist.com/coinvault-are-we-reaching-the-end-of-the-nightmare/72187/">wallets</a>, too. Since then, stealing cryptocurrencies has continued to occupy cybercriminals.</p>
<p>One of the latest additions to this phenomenon is the multi-stage DoubleFinger loader delivering a cryptocurrency stealer. DoubleFinger is deployed on the target machine, when the victim opens a malicious PIF attachment in an email message, ultimately executing the first of DoubleFinger&#8217;s loader stages.</p>
<h2 id="doublefinger-stage-1">DoubleFinger stage 1</h2>
<p>The first stage is a modified &#8220;espexe.exe&#8221; (MS Windows Economical Service Provider Application) binary, where the DialogFunc is patched so that a malicious shellcode is executed. After resolving API functions by hash, which were added to DialogFunc, the shellcode downloads a PNG image from Imgur.com. Next, the shellcode searches for the magic bytes (0xea79a5c6) in the downloaded image, locating the encrypted payload within the image.</p>

<p style="text-align: center"><em><strong>Real DialogFunc function (left) and patched function with shellcode (right)</strong></em></p>
<p>The encrypted payload consists of:</p>
<ol>
<li>A PNG with the fourth-stage payload;</li>
<li>An encrypted data blob;</li>
<li>A legitimate java.exe binary, used for DLL sideloading;</li>
<li>The DoubleFinger stage 2 loader.</li>
</ol>
<h2 id="doublefinger-stage-2">DoubleFinger stage 2</h2>
<p>The second-stage shellcode is loaded by executing the legitimate Java binary located in the same directory as the stage 2 loader shellcode (the file is named msvcr100.dll). Just as the first stage, this file is a legitimate patched binary, having similar structure and functionality as the first stage.</p>
<p>To no one&#8217;s surprise, the shellcode loads, decrypts and executes the third stage shellcode.</p>
<h2 id="doublefinger-stage-3">DoubleFinger stage 3</h2>
<p>The third-stage shellcode differs greatly from the first and second stages. For example, it uses low-level Windows API calls, and ntdll.dll is loaded and mapped in the process memory to bypass hooks set by security solutions.</p>
<p>Next step is to decrypt and execute the fourth-stage payload, located in the aforementioned PNG file. Unlike the downloaded PNG file, which does not display a valid image, this PNG file does. The steganography method used is, however, rather simple, as the data is retrieved from specific offsets.</p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/06/09152105/Crimeware_report_DoubleFinger_03.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-109988" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/06/09152105/Crimeware_report_DoubleFinger_03.png" alt="The aa.png file with embedded Stage 4" width="703" height="698" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/06/09152105/Crimeware_report_DoubleFinger_03.png 703w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/06/09152105/Crimeware_report_DoubleFinger_03-300x298.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/06/09152105/Crimeware_report_DoubleFinger_03-150x150.png 150w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/06/09152105/Crimeware_report_DoubleFinger_03-176x175.png 176w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/06/09152105/Crimeware_report_DoubleFinger_03-370x367.png 370w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/06/09152105/Crimeware_report_DoubleFinger_03-282x280.png 282w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/06/09152105/Crimeware_report_DoubleFinger_03-50x50.png 50w" sizes="(max-width: 703px) 100vw, 703px" /></a></p>
<p style="text-align: center"><em><strong>The aa.png file with embedded Stage 4</strong></em></p>
<h2 id="doublefinger-stage-4">DoubleFinger stage 4</h2>
<p>The stage 4 shellcode is rather simple. It locates the fifth stage within itself and then uses the <a href="https://encyclopedia.kaspersky.com/glossary/process-doppelganging/?utm_source=securelist&amp;utm_medium=blog&amp;utm_campaign=termin-explanation" target="_blank" rel="noopener">Process Doppelgänging</a> technique to execute it.</p>
<h2 id="doublefinger-stage-5">DoubleFinger stage 5</h2>
<p>The fifth stage creates a scheduled task that executes the GreetingGhoul stealer every day at a specific time. It then downloads another PNG file (which is actually the encrypted GreetingGhoul binary prepended with a valid PNG header), decrypts it and then executes it.</p>
<h2 id="greetingghoul-remcos">GreetingGhoul &amp; Remcos</h2>
<p>GreetingGhoul is a stealer designed to steal cryptocurrency-related credentials. It essentially consists of two major components that work together:</p>
<ol>
<li>A component that uses MS WebView2 to create overlays on cryptocurrency wallet interfaces;</li>
<li>A component that detects cryptocurrency wallet apps and steals sensitive information (e.g. recovery phrases).</li>
</ol>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/06/12095612/Crimeware_report_DoubleFinger_04-1.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-109989" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/06/12095612/Crimeware_report_DoubleFinger_04-1.png" alt="Examples of fake windows" width="372" height="265" /></a></p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/06/12095605/Crimeware_report_DoubleFinger_05-1.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-109990" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2023/06/12095605/Crimeware_report_DoubleFinger_05-1.png" alt="Examples of fake windows" width="376" height="266" /></a></p>
<p style="text-align: center"><em><strong>Examples of fake windows</strong></em></p>
<p>With hardware wallets, a user should never fill their recovery seed on the computer. A hardware wallets vendor will never ask for that.</p>
<p>Next to GreetingGhoul we also found several DoubleFinger samples that downloaded the Remcos RAT. Remcos is a well-known commercial RAT often used by cybercriminals. We&#8217;ve seen it being utilized in targeted attacks against businesses and organizations.</p>
<h2 id="victims-attribution">Victims &amp; Attribution</h2>
<p>We found several pieces of Russian text in the malware. The first part of the C2 URL is &#8220;Privetsvoyu&#8221; which is a misspelled transliteration of the Russian word for &#8220;Greetings.&#8221; Secondly, we found the string &#8220;salamvsembratyamyazadehayustutlokeretodlyagadovveubilinashusferu.&#8221; Despite the weird transliteration, it roughly translates to: &#8220;Greetings to all brothers, I&#8217;m suffocating here, locker is for bastards, you&#8217;ve messed up our area of interest.&#8221;</p>
<p>Looking at the victims, we see them in Europe, the USA and Latin America. This is in accordance with the old adage that cybercriminals from CIS countries don&#8217;t attack Russian citizens. Although the pieces of Russian text and the victimology are not enough to conclude that the ones behind this campaign are indeed from the post-Soviet space.</p>
<h2 id="conclusion">Conclusion</h2>
<p>Our analysis of the DoubleFinger loader and GreetingGhoul malware reveals a high level of sophistication and skill in crimeware development, akin to advanced persistent threats (APTs). The multi-staged, shellcode-style loader with steganographic capabilities, the use of Windows COM interfaces for stealthy execution, and the implementation of Process Doppelgänging for injection into remote processes all point to well-crafted and complex crimeware. The use of Microsoft WebView2 runtime to create counterfeit interfaces of cryptocurrency wallets further underscores the advanced techniques employed by the malware.</p>
