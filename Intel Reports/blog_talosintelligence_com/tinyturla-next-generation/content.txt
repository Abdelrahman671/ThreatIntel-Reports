    <h1 class="text-center">TinyTurla Next Generation - Turla APT spies on Polish NGOs</h1>

    <div class="text-center m-1">
        <div class="post-author">
            <span>By </span>
                <a href="https://blog.talosintelligence.com/author/asheer-malhotra/">Asheer Malhotra</a>,

    <section class="post-content-wrapper mt-5">
        <div class="post-content">
            <ul><li>Cisco Talos has identified a new backdoor authored and operated by the Turla APT group, a Russian cyber espionage threat group. This new backdoor we’re calling “TinyTurla-NG” (TTNG) is similar to Turla’s previously disclosed implant, <a href="https://blog.talosintelligence.com/tinyturla/"><u>TinyTurla</u></a>, in coding style and functionality implementation.</li><li>Talos assesses with high confidence that TinyTurla-NG, just like TinyTurla, is a small “last chance” backdoor that is left behind to be used when all other unauthorized access/backdoor mechanisms have failed or been detected on the infected systems.</li><li>TinyTurla-NG was seen as early as December 2023 targeting a Polish non-governmental organization (NGO) working on improving Polish democracy and supporting Ukraine during the Russian invasion.&nbsp;</li><li>We’ve also discovered previously unknown PowerShell scripts we’re calling “TurlaPower-NG '' that are meant to act as file exfiltrators. TinyTurla-NG deployed these scripts to exfiltrate key material used to secure the password databases of popular password management software, indicating a concerted effort for Turla to steal login credentials.</li></ul><p>Talos, in cooperation with <a href="https://cert.ngo/"><u>CERT.NGO</u></a>, investigated another compromise by the Turla threat actor, with a new backdoor quite similar to <a href="https://blog.talosintelligence.com/tinyturla/"><u>TinyTurla</u></a>, that we are calling TinyTurla-NG (TTNG). Our findings indicate that Polish non-governmental organizations (NGOs) are actively being targeted, with at least one of them supporting Ukraine. While NGOs aren’t directly involved in conflicts they frequently participate in providing aid to entities suffering through the conflicts. Aggressor parties may deem it strategically beneficial to monitor such NGOs to keep track of ongoing and potentially new aid packages for their victims.</p><p>Turla has been widely known to target entities across the world using a huge set of offensive tools in geographies including the U.S., European Union, Ukraine and Asia. They’ve previously used malware families such as <a href="https://cert.gov.ua/article/5213167"><u>CAPIBAR and KAZUAR</u></a> to target Ukrainian defense forces. After <a href="https://www.welivesecurity.com/2020/12/02/turla-crutch-keeping-back-door-open/"><u>Crutch</u></a> and <a href="https://blog.talosintelligence.com/tinyturla/"><u>TinyTurla</u></a>, Turla has now expanded its arsenal to include the TinyTurla-NG and TurlaPower-NG malware families, while also widening its net of targets to NGOs. This activity signals the adversary’s intention to expand both their suite of malware as well as a set of targets to support Russia’s strategic and political goals.</p><p>Talos identified the existence of three different TinyTurla-NG samples, but only obtained access to two of them. This campaign’s earliest compromise date was Dec. 18, 2023, and was still active as recently as Jan. 27, 2024. However, we assess that the campaign may have started as early as November 2023 based on malware compilation dates.&nbsp;</p><p>In this campaign, Turla uses compromised WordPress-based websites as command and control endpoints (C2) for the TTNG backdoor. The operators used different websites running vulnerable WordPress versions (versions including 4.4.20, 5.0.21, 5.1.18 and 5.7.2), which allowed the upload of PHP files containing the C2 code consisting of names such as: rss-old[.]php, rss[.]old[.]php or block[.]old[.]php</p><h1 id="tinyturla-ng-uses-powershell-and-a-command-line-to-run-arbitrary-commands">TinyTurla-NG uses PowerShell and a command line to run arbitrary commands</h1><p>During the campaign’s three-month run, different C2 servers were also used to host PowerShell scripts and arbitrary commands that could then be executed on the victim machine.</p><p>Like TinyTurla, the malware is a service DLL, which is started via svchost.exe. The malware code itself is different and new. Different malware features are distributed via different threads. The malware is using Windows events for synchronization. In the DLL’s ServiceMain function, the first main malware thread is started.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://lh7-us.googleusercontent.com/EJ_Be_vqrWWqVIjn5l4m0UbQsd1Ltc9HpD6KZQPFqH4WYnGK4POcgtoCS42591IcCCK0O1QkevED0qyYuAdX9VVVOrOVju3hufmtwa8Dw-hbdRMYsrRz0CMfVG_CyWzB5LFIAH-NNaEBnRils_snOGk" class="kg-image" alt="" loading="lazy" width="1245" height="98"><figcaption><span style="white-space: pre-wrap;">TinyTurla-NG DLL starting the main infection thread.</span></figcaption></figure><p>The InitCfgSetupCreateEvent function initializes the config variables and the event which is used for synchronization later on.&nbsp;</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://lh7-us.googleusercontent.com/UJgF146MvnlBJaL6n2Nvz4aMq8KHshF_7rr0jxF8rW_U1Gk3cDVqQTT5GLJYtwpF1LKoMA5mYmyU3EKTK4aR5Zr6uBv1y4oN4hLqY9oGLRpRJ1jo3NxxuTJ9-NV6RjWLff3EqAGizYkQh0tvuKmFHd8" class="kg-image" alt="" loading="lazy" width="944" height="369"><figcaption><span style="white-space: pre-wrap;">De-facto main function of the DLL calling code to initiate threads.</span></figcaption></figure><p>This thread then starts two more threads via the CheckOSVersion_StartWorkerThreads function.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://lh7-us.googleusercontent.com/Atydn1ORKCE1QIeJROBp54Fv5eVYTJOk1l-3C26gF-15guWS7RrqqWSRf2Y6Qs01BWCkx_PkAOYhNfTLMF2U2tenZTGAUNVzIg7sVU_1mgxT74lZvXwrsZMb5zSKXB3KtCJvQWALUGR6Yt-0wzOEIf0" class="kg-image" alt="" loading="lazy" width="1067" height="925"><figcaption><span style="white-space: pre-wrap;">CheckOSVersion_Start_WorkerThreads function.</span></figcaption></figure><p>After checking the PowerShell and Windows versions, the first thread starts to beacon to the C2 by sending a campaign identifier (“id”) and the message “Client Ready” to register the successful infection with the C2. This is done in the C2_client_ready function in the screenshot below.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://lh7-us.googleusercontent.com/J47spUToI7jZtO6c5aZXKnOvpdCAMJlBEmBoXmldI6gSj9mOb51x0rdyROY1GsbbpKzASNyRTnIKFz5j4i5veSfED8SIAFJ5AhfbOUAjo42Mt_bnr6B1uYfF2w7llEiedY-lX2OwK-i104JCep8zk7E" class="kg-image" alt="" loading="lazy" width="904" height="535"><figcaption><span style="white-space: pre-wrap;">Thread No. 1: C2 beaconing thread.</span></figcaption></figure><p>If the registration is successful, the TTNG backdoor will ask the C2 for a task to execute (gettask_loop function). The second thread, which was started by the CheckOSVersion_Start_WorkerThreads function, is responsible for executing the task command sent from the C2. It waits until the TTNG backdoor has received the response from the C2. The synchronization between the two threads is performed via the Windows event mentioned earlier. The first thread triggers the event (in the thread1_function) once it has successfully received the task from the C2.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://lh7-us.googleusercontent.com/Z-11UPqrlMIKGIHRXI-lf4t9cYKsfJ24xSEE5wL9oc393-36xnhkWGCXS-CL_icw3aTGvVfVVTSSt2R7oZNi6faCUBjHr3PCWsgHdgxls4AJ83OAWLdidb4YzGdYMI1utr90EhAXgsW1dd3LZTNGzgU" class="kg-image" alt="" loading="lazy" width="852" height="120"><figcaption><span style="white-space: pre-wrap;">Thread No. 1 signals Thread No. 2 to handle the task/command received from the C2.</span></figcaption></figure><p>The tasks can be executed either using a PowerShell or command (cmd.exe) shell. The decision is made based on the PowerShell version running on the victim machine.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://lh7-us.googleusercontent.com/TVQrsMLE9h34mcsRt0Rb6ZknDt9fpRR8dMtxzW-J_S3gWfuOiXxnPJLLh_Rw0c7-HNxiYAA2ONuBPKiGYf7QTuLcyfEHXLoFzM9OfrYRqKkVj7e25hen27Ho3sAYZV17Byyni9kxFkAQ2N_ayqEUouM" class="kg-image" alt="" loading="lazy" width="1056" height="568"><figcaption><span style="white-space: pre-wrap;">Thread No. 2: Windows command execution function.</span></figcaption></figure><p>When executing commands via cmd.exe or PowerShell.exe, TinyTurla-NG will create pipes to input and read the output of the commands. While executing commands via cmd.exe, the backdoor first executes the command <code>chcp 437 &gt; NUL</code>execute to set the active console page to 437, i.e., the U.S., and then execute the commands issued by the C2.&nbsp;</p><p>However, while executing commands via PowerShell.exe, TinyTurla-NG will additionally execute the following PowerShell cmdlet to prevent the recording of command history:</p><p><code>Set-PSReadLineOption -HistorySaveStyle SaveNothing</code></p><p>In addition to executing the content of the task received from the C2 directly e.g., <code>C:\windows\system32\malware.exe</code>, the backdoor will accept the following command codes from the C2. These command codes can be meant for administering the implant or for file management:</p><ul><li><strong>“timeout”</strong>: Change the number of minutes the backdoor sleeps between asking the C2 for new tasks. The new timeout is one minute multiplied by the timeout parameter sent by the C2. For example, if the C2 sends the task “timeout 10”, then the backdoor will now sleep for 10 minutes. If it is given a third parameter, the fail counter is changed, too.</li></ul><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://lh7-us.googleusercontent.com/1frUezT1Ig8gceqF-B3JDOAuvnO_QlO7XPKOMtCv_TwDLMeTRtt9vutt7pWkkLJddHPLGvLiiYjJMFx4_vrL61GD5os2Lc_7ZNOnrbXRbSNlAe5LUm9w3xODYthE72yzG-SfNgoVRvWaeB0RIZNvUIE" class="kg-image" alt="" loading="lazy" width="855" height="53"><figcaption><span style="white-space: pre-wrap;">TTNG setting a timeout value for C2 communication.</span></figcaption></figure><ul><li><strong>“changeshell”</strong>: This command will instruct the backdoor to switch the current shell being used to execute commands, i.e., from cmd.exe to PowerShell.exe, or vice versa.</li><li><strong>“changepoint”</strong>: This command code is used by the C2 to retrieve the result of command(s) executed on the infected endpoint. The endpoint will also return logging messages to the C2 server it has collected for administrative commands executed since "changepoint" was last issued such as:</li></ul>

<figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://lh7-us.googleusercontent.com/noWKpCBCOBHY0YHFgaPfFSS7EKaWdRQtuh77akFj_DcjbjQ39U_wQvLpRvdcRC0IZ7itRPvOT2OTXMAAzceJqRCgrTSVBTDwkITz9W55qu1cHYGMBLiJPe4rwV674pMnyTEBSmGfEnC6DXFAg_Comuw" class="kg-image" alt="" loading="lazy" width="1600" height="651"><figcaption><span style="white-space: pre-wrap;">BAT file contents template.</span></figcaption></figure><p>The BAT file is created from the template where the first two “%s” are replaced with the DLL name and the last one with the name of the BAT file itself to delete both artifacts from the disk.</p><h1 id="turlapower-ng-and-its-exfiltration-capabilities">TurlaPower-NG and its exfiltration capabilities</h1><p>Talos also discovered malicious PowerShell scripts we’re calling “TurlaPower-NG”, written to infected endpoints via the TTNG backdoor. The scripts consist of the C2 URL and target file paths. For each file path specified, the script will recursively enumerate files and add them to an archive on disk. TurlaPower-NG takes specific care to exclude files with the “.mp4” extension from being added to the archive. The attackers had a specific interest in key material used to secure the password databases and popular password management software, adding related files to the archive:</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://lh7-us.googleusercontent.com/FYoRK-2fgjcMlMHf27wfdb3-Qf55Q5AyIbVuKnlRfHKJMT65vYAzYYv3cfAViLOTKPi8RRSl2MngRS4ussQuSPbD_K0E4PWvmiHrc1L1eNalYyHG3JP6vyQP84nV92HSTw1SNeZ2H_e61a8qn4-zCtw" class="kg-image" alt="" loading="lazy" width="1458" height="1600"><figcaption><span style="white-space: pre-wrap;">TurlaPower-NG’s file archiving function.</span></figcaption></figure><p>The archive is a “.zip” extension whose name is generated on the fly by generating a new GUID which is used as the archive name. The archive file is then exfiltrated to the C2 using HTTP/S POST requests along with a log of the activity performed being sent to the C2 as well. The log consists of:</p><ul><li>Name of the archive file (or part) POSTed to the C2.</li><li>Number of files in the archive along with the archive size.</li></ul><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://lh7-us.googleusercontent.com/zPdLw8VT3FZV8jQ4td_N3rdiimbpv6TBZdKrqULFYzvrZb91ymtEE8BVTOXk0u04Im1B2qROciTG8OxJrGtjycGnzZGJvpD91SvBFCYBps8SQix72_gMVrpW7RRVcn-izhNOaE1WxSwfHULXdnU_bag" class="kg-image" alt="" loading="lazy" width="1600" height="613"><figcaption><span style="white-space: pre-wrap;">TurlaPower-NG’s archive filename generation and log generation for C2.</span></figcaption></figure><h1 id="c2-setup-and-operations">C2 setup and operations</h1><p>All of the C2 servers discovered so far consist of legitimate, vulnerable WordPress-based websites compromised by Turla to set up their C2 servers. Once compromised the operators set up scripts, logging and data directories to operate their C2 servers.</p><h2 id="directory-and-file-structure">Directory and file structure</h2><p>The C2’s directories and files setup consists of three key components:</p><ul><li><strong>C2 scripts</strong>: Turla set up PHP scripts ending with extensions — “.old.php” — in certain directories of the compromised websites. The URLs for these PHP-based C2s were then coded into the TTNG backdoors consisting of two C2 URLs per sample.</li><li><strong>Logging</strong>: In addition to the C2 PHP scripts, the adversary also set up the logging of infections to keep track of infected systems and commands being issued to them. The logging mechanism of the C2 generates three log files on the C2 server:<ul><li><strong>_log[.]txt</strong>: A log of all infected endpoints beaconing into the C2.</li><li><strong>result[.]txt</strong>: A log of all messages received from the TTNG backdoor.</li><li><strong>tasks[.]txt</strong>: A log of all commands issued to the infected hosts.</li></ul></li><li><strong>Data directories</strong>: TTNG and TurlaPower-NG both support the exfiltration of files to the C2 server. The C2 server stores stolen data in directories separate from the logging directories.</li></ul><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://lh7-us.googleusercontent.com/S6sN64okaRX1gOEcetykzMyemJ50jL6Lezjxc0TTkAgg32rw74BPJ2QJwenTIX6V8BMl4yTQMufI8WJKQ9lR_hN47PE2lTFy6ih19WO0EEojMXzBmP0tBs8aMwiprqz_wwb9km7uIllZ7nwwFz7lGf8" class="kg-image" alt="" loading="lazy" width="927" height="357"><figcaption><span style="white-space: pre-wrap;">Sample directory listing of the logs of the C2 server.</span></figcaption></figure><h2 id="c2-communication-process">C2 communication process</h2><p>The TinyTurla-NG backdoor uses a specific Identifier, “id” value in its HTTP form data whenever it communicates with the C2 server. This ID value is an eight-character phrase hardcoded into the backdoor.&nbsp;</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://lh7-us.googleusercontent.com/qDXXMORRSjuOHqML2Lf9xQiVQy7VNmsFBTdTgLJEmCvFAABGNmKmHwbCV6-3wlU1hfdnyTxy-UV9udddLckB7S_sx1KbJrCqeIBfuzwY2TJZBSXprAno1WLGCXjtpIjDghXMzbct7eRWIKkghDcgnkw" class="kg-image" alt="" loading="lazy" width="1319" height="710"><figcaption><span style="white-space: pre-wrap;">Network capture displaying the Identifier value and “Client Ready” message.</span></figcaption></figure><p>This same identifier value is then used to create directories for log files on the C2 server indicating that the C2 server maintains different log files for different identifiers.</p><p>After registering the victim on the C2 server, the backdoor sends out a <code>gettask</code> request, similar to the one below. The C2 can answer this with special commands or just the file that is supposed to be executed on the infected machine.&nbsp;</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://lh7-us.googleusercontent.com/dU8y2bTot2OL42uzlutLtVw2slR65Id8jHry86NT4Ww9Jn9gkuuO5cXWe42MN-VSkQysrZKK8pW1stVdkhGPnro1l3DAOjG7VKzBjvofc4PnSar0ACe6dR8mfmtRgdyFq-Av0Qc5QNQQFp8c_jr4au4" class="kg-image" alt="" loading="lazy" width="1143" height="732"><figcaption><span style="white-space: pre-wrap;">TTNG’s C2 communication to fetch tasks to perform on the infected endpoint.</span></figcaption></figure><p>Depending on the PowerShell version running on the victim machine, the C2 task commands are piped into a PowerShell or cmd[.]exe shell.&nbsp;</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://lh7-us.googleusercontent.com/XQsBlQrVqyG_Eflc9h4VPgqUPs31xKbJSxvK0GXrU2ZpsoJanFY-iEl39ImqBL7HChIowSsk0XCaJfhGJpXJKIfYwHAnEftPbBxj8AGARBC0FG4JXvhU6k3cizVS9ME9aqFCsBONdLUSO0mfrf3G4Ik" class="kg-image" alt="" loading="lazy" width="1122" height="565"><figcaption><span style="white-space: pre-wrap;">TinyTurla-NG’s shell selection between PowerShell or cmd[.]exe.</span></figcaption></figure><h1 id="coverage">Coverage</h1><p>Ways our customers can detect and block this threat are listed below.</p><figure class="kg-card kg-image-card"><img src="https://lh7-us.googleusercontent.com/fP2OMCl7Rz7iARuuo3kPaost_QHBeoam5z2Iebs3ZvlFqpe97Dv04S24x7c95k3d0nR4kpBndpypaNt8ifVjYNTIXCP5RtwurUMgOCZXq06Lf0zB1MJSWfkgsjtM5-h1-9IVG0RZ9FGYEm1VvpUpOwg" class="kg-image" alt="" loading="lazy" width="1600" height="323"></figure><p><a href="https://www.cisco.com/c/en/us/products/security/amp-for-endpoints/index.html?ref=cisco-talos-blog"><u>Cisco Secure Endpoint</u></a> (formerly AMP for Endpoints) is ideally suited to prevent the execution of the malware detailed in this post.
