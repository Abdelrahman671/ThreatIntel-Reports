							<div class="content "><h1 id="dir">Dir</h1><div class="display-flex justify-content-space-between align-items-center flex-wrap-wrap page-metadata-container">
										<div class="margin-right-xxs">
											<ul class="metadata page-metadata" data-bi-name="page info" lang="en-us" dir="ltr"><li>Article</li><li>
														<time class="is-invisible" data-article-date aria-label="Article review date" datetime="2016-08-31T00:00:00.000Z" data-article-date-source="ms.date">08/31/2016</time>
													</li></ul>
										</div></div><nav id="center-doc-outline" class="doc-outline is-hidden-desktop display-none-print margin-bottom-sm" data-bi-name="intopic toc" aria-label="In this article">
											<h2 id="ms--in-this-article" class="title is-6 margin-block-xs">In this article</h2>
										</nav><!-- <content> --><p>&nbsp;</p>
<p>Applies To: Windows Vista, Windows Server 2008, Windows Server 2012, Windows 8</p>
<p>Displays a list of a directory's files and subdirectories. If used without parameters, <strong>dir</strong> displays the disk's volume label and serial number, followed by a list of directories and files on the disk (including their names and the date and time each was last modified). For files, <strong>dir</strong> displays the name extension and the size in bytes. <strong>Dir</strong> also displays the total number of files and directories listed, their cumulative size, and the free space (in bytes) remaining on the disk.</p>
<p>For examples of how to use this command, see <a href="">Examples</a>.</p>
<h2 id="syntax">Syntax</h2>
<pre><code>dir [&lt;Drive&gt;:][&lt;Path&gt;][&lt;FileName&gt;] [...] [/p] [/q] [/w] [/d] [/a[[:]&lt;Attributes&gt;]][/o[[:]&lt;SortOrder&gt;]] [/t[[:]&lt;TimeField&gt;]] [/s] [/b] [/l] [/n] [/x] [/c] [/4]
</code></pre>
<h2 id="parameters">Parameters</h2>
<table>
<colgroup>
<col>
<col>
</colgroup>
<thead>
<tr class="header">
<th><p>Parameter</p></th>
<th><p>Description</p></th>
</tr>
</thead>
<tbody>
<tr class="odd">
<td><p>[&lt;Drive&gt;:][&lt;Path&gt;]</p></td>
<td><p>Specifies the drive and directory for which you want to see a listing.</p></td>
</tr>
<tr class="even">
<td><p>[&lt;FileName&gt;]</p></td>
<td><p>Specifies a particular file or group of files for which you want to see a listing.</p></td>
</tr>
<tr class="odd">
<td><p>/p</p></td>
<td><p>Displays one screen of the listing at a time. To see the next screen, press any key on the keyboard.</p></td>
</tr>
<tr class="even">
<td><p>/q</p></td>
<td><p>Displays file ownership information.</p></td>
</tr>
<tr class="odd">
<td><p>/w</p></td>
<td><p>Displays the listing in wide format, with as many as five file names or directory names on each line.</p></td>
</tr>
<tr class="even">
<td><p>/d</p></td>
<td><p>Displays the listing in the same format as <strong>/w</strong>, but the files are sorted by column.</p></td>
</tr>
<tr class="odd">
<td><p>/a[[:]&lt;Attributes&gt;]</p></td>
<td><p>Displays only the names of those directories and files with the attributes that you specify. If you omit <strong>/a</strong>, <strong>dir</strong> displays the names of all files except hidden and system files. If you use <strong>/a</strong> without specifying <em>Attributes</em>, <strong>dir</strong> displays the names of all files, including hidden and system files.</p>
<p>The following list describes each of the values that you can use for <em>Attributes</em>. Using a colon (:) is optional. Use any combination of these values, and do not separate the values with spaces.</p>
<p>&nbsp;&nbsp;&nbsp;<strong>d</strong> Directories</p>
<p>&nbsp;&nbsp;&nbsp;<strong>h</strong> Hidden files</p>
<p>&nbsp;&nbsp;&nbsp;<strong>s</strong> System files</p>
<p>&nbsp;&nbsp;&nbsp;<strong>l</strong> Reparse points</p>
<p>&nbsp;&nbsp;&nbsp;<strong>r</strong> Read-only files</p>
<p>&nbsp;&nbsp;&nbsp;<strong>a</strong> Files ready for archiving</p>
<p>&nbsp;&nbsp;&nbsp;<strong>i</strong> Not content indexed files</p>
<p>&nbsp;&nbsp;&nbsp;<strong>-</strong> Prefix meaning "not"</p></td>
</tr>
<tr class="even">
<td><p>/o[[:]&lt;SortOrder&gt;]</p></td>
<td><p>Sorts the output according to <em>SortOrder</em>, which can be any combination of the following values:</p>
<p>&nbsp;&nbsp;&nbsp;<strong>n</strong> By name (alphabetical)</p>
<p>&nbsp;&nbsp;&nbsp;<strong>e</strong> By extension (alphabetical)</p>
<p>&nbsp;&nbsp;&nbsp;<strong>g</strong> Group directories first</p>
<p>&nbsp;&nbsp;&nbsp;<strong>s</strong> By size (smallest first)</p>
<p>&nbsp;&nbsp;&nbsp;<strong>d</strong> By date/time (oldest first)</p>
<p>&nbsp;&nbsp;&nbsp;<strong>-</strong> Prefix to reverse order</p>
<div class="alert">
<table>
<colgroup>
<col>
</colgroup>
<thead>
<tr class="header">
<th><img src="images/dn486833.s-e6f6a65cf14f462597b64ac058dbe1d0-system-media-system-caps-note(ws.11).jpeg" data-linktype="relative-path">Note</th>
</tr>
</thead>
<tbody>
<tr class="odd">
<td><p>Using a colon is optional. Multiple values are processed in the order in which you list them. Do not separate multiple values with spaces.</p></td>
</tr>
</tbody>
</table>
</div>
<p>If <em>SortOrder</em> is not specified, <strong>dir /o</strong> lists the directories in alphabetic order, followed by the files, which are also sorted in alphabetic order.</p></td>
</tr>
<tr class="odd">
<td><p>/t[[:]&lt;TimeField&gt;]</p></td>
<td><p>Specifies which time field to display or use for sorting. The following list describes each of the values you can use for <em>TimeField</em>:</p>
<p><strong>c</strong> Creation</p>
<p><strong>a</strong> Last access</p>
<p><strong>w</strong> Last written</p></td>
</tr>
<tr class="even">
<td><p>/s</p></td>
<td><p>Lists every occurrence of the specified file name within the specified directory and all subdirectories.</p></td>
</tr>
<tr class="odd">
<td><p>/b</p></td>
<td><p>Displays a bare list of directories and files, with no additional information. <strong>/b</strong> overrides <strong>/w</strong>.</p></td>
</tr>
<tr class="even">
<td><p>/l</p></td>
<td><p>Displays unsorted directory names and file names in lowercase.</p></td>
</tr>
<tr class="odd">
<td><p>/n</p></td>
<td><p>Displays a long list format with file names on the far right of the screen.</p></td>
</tr>
<tr class="even">
<td><p>/x</p></td>
<td><p>Displays the short names generated for non-8dot3 file names. The display is the same as the display for <strong>/n</strong>, but the short name is inserted before the long name.</p></td>
</tr>

<h2 id="remarks">Remarks</h2>
<ul>
<li><p>To use multiple <em>FileName</em> parameters, separate each file name with a space, comma, or semicolon.</p>
</li>
<li><p>You can use wildcard characters (<strong>*</strong> or**?**), to represent one or more characters of a file name and to display a subset of files or subdirectories.</p>
<p><strong>Asterisk (*):</strong> Use the asterisk as a substitute for any string of characters, for example:</p>
<ul>
<li><p><strong>dir *.txt</strong> lists all files in the current directory with extensions that begin with .txt, such as .txt, .txt1, .txt_old.</p>
</li>
<li><p><strong>dir read*.txt</strong> lists all files in the current directory that begin with "read" and with extensions that begin with .txt, such as .txt, .txt1, or .txt_old.</p>
</li>
<li><p><strong>dir read*.*</strong> lists all files in the current directory that begin with "read" with any extension.</p>
</li>
</ul>
<p>The asterisk wildcard always uses short file name mapping, so you might get unexpected results. For example, the following directory contains two files (t.txt2 and t97.txt):</p>
<pre><code>C:\test&gt;dir /x
Volume in drive C has no label.
Volume Serial Number is B86A-EF32

Directory of C:\test

11/30/2004  01:40 PM &lt;DIR&gt;  .
11/30/2004  01:40 PM &lt;DIR&gt; ..
11/30/2004  11:05 AM 0 T97B4~1.TXT t.txt2
11/30/2004  01:16 PM 0 t97.txt
</code></pre>
<p>You might expect that typing <strong>dir t97*</strong> would return the file t97.txt. However, typing <strong>dir t97*</strong> returns both files, because the asterisk wildcard matches the file t.txt2 to t97.txt by using its short name map T97B4~1.TXT. Similarly, typing <strong>del t97*</strong> would delete both files.</p>
<p><strong>Question mark (?):</strong> Use the question mark as a substitute for a single character in a name. For example, typing <strong>dir read???.txt</strong> lists any files in the current directory with the .txt extension that begin with "read" and are followed by up to three characters. This includes Read.txt, Read1.txt, Read12.txt, Read123.txt, and Readme1.txt, but not Readme12.txt.</p>
</li>
<li><p>Specifying file display attributes</p>
<p>If you use <strong>/a</strong> with more than one value in <em>Attributes</em>, <strong>dir</strong> displays the names of only those files with all the specified attributes. For example, if you use <strong>/a</strong> with <strong>r</strong> and <strong>-h</strong> as attributes (by using either <strong>/a:r-h</strong> or <strong>/ar-h</strong>), <strong>dir</strong> will only display the names of the read-only files that are not hidden.</p>
</li>
<li><p>Specifying file name sorting</p>
<p>If you specify more than one <em>SortOrder</em> value, <strong>dir</strong> sorts the file names by the first criterion, then by the second criterion, and so on. For example, if you use <strong>/o</strong> with the <strong>e</strong> and <strong>-s</strong> values for <em>SortOrder</em> (by using either <strong>/o:e-s</strong> or <strong>/oe-s</strong>), <strong>dir</strong> sorts the names of directories and files by extension, with the largest first, and then displays the final result. The alphabetic sorting by extension causes file names with no extensions to appear first, then directory names, and then file names with extensions.</p>
</li>
<li><p>Using redirection symbols and pipes</p>
<p>When you use the redirection symbol (<strong>&gt;</strong>) to send <strong>dir</strong> output to a file or a pipe (<strong>|</strong>) to send <strong>dir</strong> output to another command, use <strong>/a:-d</strong> and <strong>/b</strong> to list the file names only. You can use <em>FileName</em> with <strong>/b</strong> and <strong>/s</strong> to specify that <strong>dir</strong> is to search the current directory and its subdirectories for all file names that match <em>FileName</em>. <strong>Dir</strong> lists only the drive letter, directory name, file name, and file name extension (one path per line), for each file name it finds. Before you use a pipe to send <strong>dir</strong> output to another command, you should set the TEMP environment variable in your Autoexec.nt file.</p>
</li>
<li><p>The <strong>dir</strong> command, with different parameters, is available from the Recovery Console.</p>
</li>
</ul>
<h2 id="examples">Examples</h2>
<p>To display all directories one after the other, in alphabetical order, in wide format, and pausing after each screen, make sure that the root directory is the current directory, and then type:</p>
<pre><code>dir /s/w/o/p
</code></pre>
<p><strong>Dir</strong> lists the root directory, the subdirectories, and the files in the root directory, including extensions. Then, <strong>dir</strong> lists the subdirectory names and file names in each subdirectory in the tree.</p>
<p>To alter the preceding example so that <strong>dir</strong> displays the file names and extensions, but omits the directory names, type:</p>
<pre><code>dir /s/w/o/p/a:-d
</code></pre>
<p>To print a directory listing, type:</p>
<pre><code>dir &gt; prn
</code></pre>
<p>When you specify <strong>prn</strong>, the directory list is sent to the printer that is attached to the LPT1 port. If your printer is attached to a different port, you must replace <strong>prn</strong> with the name of the correct port.</p>
<p>You can also redirect output of the <strong>dir</strong> command to a file by replacing <strong>prn</strong> with a file name. You can also type a path. For example, to direct <strong>dir</strong> output to the file dir.doc in the Records directory, type:</p>
<pre><code>dir &gt; \records\dir.doc
</code></pre>
<p>If dir.doc does not exist, <strong>dir</strong> creates it, unless the Records directory does not exist. In that case, the following message appears:</p>
<p>File creation error</p>
<p>To display a list of all the file names with the .txt extension in all directories on drive C, type:</p>
<pre><code>dir c:\*.txt /w/o/s/p
</code></pre>
<p><strong>Dir</strong> displays, in wide format, an alphabetized list of the matching file names in each directory, and it pauses each time the screen fills until you press any key to continue.</p>
