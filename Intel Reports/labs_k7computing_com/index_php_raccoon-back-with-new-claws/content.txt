<p>
<p>Raccoon infostealer was first released in April 2019, the initial Version1(V1) was distributed in telegram groups and other forums as Malware-as-a-service (MaaS). The detailed blog on V1 can be seen <a href="https://labs.k7computing.com/index.php/raccoon-the-info-stealer/">here</a>. Now the stealer has been updated with new features, and comes packed with Commercial packers. It has a stealthy way of gaining information from the system using Windows API’s. This blog discusses in depth on the Version2(V2) of Racoon Stealer and its method to obtain the information. </p>



<p>The Stealer is usually downloaded when a user tries to download cracked software, thus the malware is added with around 400MB of junk in the overlay along with an invalid digital signature from AVG.</p>



<div style="height:15px" aria-hidden="true" class="wp-block-spacer"></div>



<h4 class="has-text-align-left"><strong>Analysis</strong></h4>



<p>The sample is around <strong>417MB </strong>disguises itself as <strong>Windows File System Proxy, </strong>has an invalid digital Signature and comes packed with VMProtect. The analysis is based on the unpacked binary.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="474" height="272" src="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-1.png" alt="" class="wp-image-24338" srcset="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-1.png 474w, https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-1-300x172.png 300w" sizes="(max-width: 474px) 100vw, 474px" /><figcaption>Figure&nbsp; 1: Sample Version Information</figcaption></figure></div>



<h4>Dynamic API Resolving</h4>



<p>The malware begins with resolving the required API’s dynamically through LoadLibrary and GetProcAddress.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="544" height="606" src="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-2.png" alt="" class="wp-image-24337" srcset="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-2.png 544w, https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-2-269x300.png 269w" sizes="(max-width: 544px) 100vw, 544px" /><figcaption>Figure 2: Dynamic API resolving procedure</figcaption></figure></div>



<p>It uses LoadLibrary to get the handles of kernel32.dll, shell32.dll, user32.dll, advapi32.dll, wininet.dll, ole32.dll, crypt32.dll and pass on the returned handle as an argument to LoadLibrary to the get the address of the required WinAPI and stores them at a memory offset.&nbsp;</p>



<div style="height:15px" aria-hidden="true" class="wp-block-spacer"></div>



<h4>String Decryption</h4>



<p>The sample uses the RC4 algorithm for decrypting the base64 strings stored in binary. At first the string is base64 decoded using CryptStringToBinary API passing the dwFlags argument as <strong>CRYPT_STRING_BASE64(0x1).</strong></p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="367" height="458" src="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-3.png" alt="" class="wp-image-24336" srcset="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-3.png 367w, https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-3-240x300.png 240w" sizes="(max-width: 367px) 100vw, 367px" /><figcaption>Figure 3: Base64 decode using CryptStringToBinaryA</figcaption></figure></div>



<p>The decoded base64 string is saved in a variable and it is passed as an argument to the function which RC4 decrypts the string using the hardcoded symmetric key “<strong>edinayarossiya</strong>”(“United Russia” &#8211;&nbsp; a political party in Russia)</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" width="1024" height="415" src="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-4-1024x415.png" alt="" class="wp-image-24335" srcset="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-4-1024x415.png 1024w, https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-4-300x122.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-4-768x311.png 768w, https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-4.png 1091w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption>Figure 4: String decryption procedure</figcaption></figure></div>



<p>Complete list of strings decrypted is listed in Appendix A.</p>



<div style="height:15px" aria-hidden="true" class="wp-block-spacer"></div>



<h4>Retrieve C2 URL</h4>



<p>The binary uses the same string decryption method discussed above to retrieve the C2 URL. For the decryption of the C2 it uses a different hardcoded RC4 symmetric key “<strong>b616297870490e1028b141f53eb3afe8</strong>” which is later used as config ID when initial information is sent.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="939" height="512" src="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-5.png" alt="" class="wp-image-24334" srcset="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-5.png 939w, https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-5-300x164.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-5-768x419.png 768w" sizes="(max-width: 939px) 100vw, 939px" /><figcaption>Figure 5: Decryption of Command and control server</figcaption></figure></div>



<h4>Checks system locale</h4>



<p>The malware then proceeds to check the locale of the system using&nbsp;</p>



<p>GetUserDefaultLocaleName API, and checks the returned string with a dword from virtual address 0x40E000.&nbsp; In this variant, this locale check does not affect the behaviour of the malware. Usually threat actors opt for an option for excluding victims from certain geolocale. Seems like the threat actors here have that option but are not using it.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="747" height="493" src="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-6.png" alt="" class="wp-image-24333" srcset="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-6.png 747w, https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-6-300x198.png 300w" sizes="(max-width: 747px) 100vw, 747px" /><figcaption>Figure 6: Get Locale of Execution system</figcaption></figure></div>



<p></p>



<h4>Checks mutex</h4>



<p>The malware checks for a mutex with name “<strong>8724643052</strong>”, if not, then creates one. If the mutex exists then it kills itself to stop itself from running multiple times.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="746" height="435" src="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-7.png" alt="" class="wp-image-24332" srcset="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-7.png 746w, https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-7-300x175.png 300w" sizes="(max-width: 746px) 100vw, 746px" /><figcaption>Figure 7: Malware checks if Mutex Exists</figcaption></figure></div>



<p></p>



<h4>Checks for system privilege</h4>



<p>The malware retrieves the Current Process access token and compares it to the SID of<strong> NTAuthority\System(“S-1-5-18”)</strong>. If it matches it executes the function to enumerate the active process list.&nbsp;</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="830" height="481" src="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-8.png" alt="" class="wp-image-24331" srcset="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-8.png 830w, https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-8-300x174.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-8-768x445.png 768w" sizes="(max-width: 830px) 100vw, 830px" /><figcaption>Figure 8: Check System Privilege</figcaption></figure></div>



<p></p>



<p>Similar to locale check, there is no change in behaviour&nbsp;</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="431" height="258" src="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-9.png" alt="" class="wp-image-24330" srcset="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-9.png 431w, https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-9-300x180.png 300w" sizes="(max-width: 431px) 100vw, 431px" /><figcaption>Figure 9: Enumerate process list if it has System privilege</figcaption></figure></div>



<p></p>



<h4>Gather Initial Information</h4>



<p>The malware initially collects machine GUID, username and sends it to C2 and awaits response from C2 for further information gathering.</p>



<p>Machine GUID is obtained from the registry key “HKEY_LOCAL_MACHINE\\SOFTWARE\\Microsoft\\Cryptography” under “MachineGUID”</p>



<p>The malware sends the initial information to C2 in the following syntax</p>



<p><strong>machineId=&lt;machineGUID&gt;|&lt;username&gt;&amp;configid=&lt;RC4_key used to decrypt C2&gt;</strong></p>



<div style="height:15px" aria-hidden="true" class="wp-block-spacer"></div>



<h4>Sends initial collected data</h4>



<p>After converting the collected initial data into Unicode string. It sends a POST request to the decrypted C2 using an unusual User-Agent String “<strong>record</strong>”. The data is sent in form data format.</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" width="1024" height="640" src="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-10-1024x640.png" alt="" class="wp-image-24329" srcset="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-10-1024x640.png 1024w, https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-10-300x188.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-10-768x480.png 768w, https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-10.png 1241w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption>Figure 10: Procedure to send request to C2 and wait for response</figcaption></figure></div>



<p></p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="768" height="168" src="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-11.png" alt="" class="wp-image-24328" srcset="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-11.png 768w, https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-11-300x66.png 300w" sizes="(max-width: 768px) 100vw, 768px" /><figcaption>Figure 11: Request sent to C2</figcaption></figure></div>



<p></p>



<p>After making the request the connection handle is kept open until it&nbsp; receives a data response. It waits for the POST response until the size of response is greater than 64 bytes.&nbsp;</p>



<div style="height:15px" aria-hidden="true" class="wp-block-spacer"></div>



<h4>Process C2 Response</h4>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="672" height="323" src="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-12.png" alt="" class="wp-image-24327" srcset="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-12.png 672w, https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-12-300x144.png 300w" sizes="(max-width: 672px) 100vw, 672px" /><figcaption>Figure 12: C2 response</figcaption></figure></div>



<p></p>



<p>The C2 response contains the urls of the dlls which are needed to collect detailed information</p>



<p>A GET request is made to download all the Dll and it is saved in the APPDATA_LOCAL folder.The path to APPDATA_LOCAL is retrieved using the API SHGetFolderPath with CSIDL passed as “<strong>CSIDL_LOCAL_APPDATA</strong>”<strong>(0x1c).</strong>.If the response doesn’t have the String “<strong>Token</strong>” in it the malware kills itself.</p>



<div style="height:15px" aria-hidden="true" class="wp-block-spacer"></div>



<h4>Collect detailed information</h4>



<p>After downloading the required dlls, it changes the current working directory and adds the path to the APPDATA_LOCAL directory to “PATH” Environment Variable using SetEnvironmentVariableW.</p>



<div style="height:15px" aria-hidden="true" class="wp-block-spacer"></div>



<h4><strong>System Info.txt</strong></h4>



<p>The malware first collects the system information and sends it as a POST request to the C2. Let us see what and how the system information is collected using WinAPI.</p>



<ul><li><strong>Locale</strong> : The malware collects the current locale using the API <strong>GetLocaleInfoW</strong></li></ul>



<ul><li><strong>TimeZone</strong> : Timezone is retrieved using API GetTimeZoneInformation</li></ul>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="877" height="444" src="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-13.png" alt="" class="wp-image-24326" srcset="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-13.png 877w, https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-13-300x152.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-13-768x389.png 768w" sizes="(max-width: 877px) 100vw, 877px" /><figcaption>Figure 13: collection of Locale and TimeZone using API</figcaption></figure></div>



<p></p>



<ul><li><strong>Product Name</strong> : Windows version is retrieved by querying the registry key <strong>“HKEY_LOCAL_MACHINE\software\microsoft\windows nt\currentversion\” </strong>and data <strong>“ProductName”</strong></li></ul>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="958" height="523" src="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-14.png" alt="" class="wp-image-24325" srcset="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-14.png 958w, https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-14-300x164.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-14-768x419.png 768w" sizes="(max-width: 958px) 100vw, 958px" /><figcaption>Figure 14: retrieve productname from registry</figcaption></figure></div>



<p></p>



<ul><li><strong>Architecture</strong> : The malware checks if SYSWOW64 directory exists on the system, if it is unavailable it considers the architecture as 32bit, else architecture is 64bit.</li></ul>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="535" height="569" src="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-15.png" alt="" class="wp-image-24324" srcset="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-15.png 535w, https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-15-282x300.png 282w" sizes="(max-width: 535px) 100vw, 535px" /><figcaption>Figure 15: Find system architecture</figcaption></figure></div>



<p></p>



<ul><li><strong>Processor&nbsp;</strong>: The processor information is obtained with the usage of ASM instruction “CPUID”(CPU Identification).</li></ul>



<ul><li><strong>RAM</strong> : The Exact amount of physical storage is retrieved using the API GlobalMemoryStatusEx,which returns the “LPMEMORYSTATUSEX” structure. From the returned structure the malware takes the field “ullTotalPhys” and right shift by 20 bits to convert it into MB.</li></ul>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="768" height="600" src="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-16.png" alt="" class="wp-image-24323" srcset="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-16.png 768w, https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-16-300x234.png 300w" sizes="(max-width: 768px) 100vw, 768px" /><figcaption>Figure 16: Get RAM information</figcaption></figure></div>



<p></p>



<ul><li><strong>Display height and width</strong> : Display height and width is obtained using the API “GetSystemMetrics” by passing the argument<strong> 0x0(SM_CXSCREEN)</strong> to retrieve width and <strong>0x1(SM_CYSCREEN)</strong> to get height.</li></ul>



<ul><li><strong>Display Devices</strong> : The display enumerated and saved using the API “<strong>EnumDisplayDevicesW</strong>”</li></ul>



<p>Screen dimension and display devices could be checked at the server if the malware is executed in a VM or sandbox.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="959" height="643" src="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-17.png" alt="" class="wp-image-24322" srcset="https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-17.png 959w, https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-17-300x201.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/07/Figure-17-768x515.png 768w" sizes="(max-width: 959px) 100vw, 959px" /><figcaption>Figure 17: Get Display information</figcaption></figure></div>



<p></p>



<ul><li><strong>List of Installed Products</strong> : The complete list of products which are installed are obtained by looping through all the subkeys under “HKLM\Software\Microsoft\Windows\CurrentVersion\Uninstall”</li></ul>



<p>All the collected information about the System is sent immediately to the C2 without saving it to a file.</p>



<div style="height:15px" aria-hidden="true" class="wp-block-spacer"></div>



<h4>Cookies.txt</h4>



<p>After collecting all the information related to system, it proceeds to collect browser saved passwords, credit card details and cookies using the following dll</p>



<ol><li>Sqlite3.dll &#8211; to collect login id and passwords from chrome(ium) based browsers</li><li>mozglue.dll/nss3.dll &#8211; to collects login id and passwords from firefox</li></ol>



<p>The following queries are used to query the required information.</p>



<ul><li>SELECT origin_url, username_value, password_value FROM logins</li><li>SELECT host_key, path, is_secure , expires_utc, name, encrypted_value FROM cookies</li><li>SELECT name, value FROM autofill</li><li>SELECT host, path, isSecure, expiry, name, value FROM moz_cookies</li><li>SELECT fieldname, value FROM moz_formhistory</li><li>SELECT name_on_card, card_number_encrypted, expiration_month, expiration_year FROM credit_cards</li></ul>



<p>The Stealer even has the capability to collect the crypto wallets if found on the system and sends all the collected information to C2 immediately.</p>



<div style="height:15px" aria-hidden="true" class="wp-block-spacer"></div>



<h4>Captures screenshot</h4>



<p>A series of Windows API is used to capture the screenshot of the infected machine, and is sent to C2. The flow is similar to the example code given by microsoft <a href="https://docs.microsoft.com/en-us/windows/win32/gdi/capturing-an-image">here</a>.&nbsp;</p>



<div style="height:15px" aria-hidden="true" class="wp-block-spacer"></div>
