	<article class="ocpArticleContent"><section class="ocpIntroduction">
        <p id="bmbacktotop">Files from the Internet and from other potentially unsafe locations can contain viruses, worms, or other kinds of malware that can harm your computer. To help protect your computer, files from these potentially unsafe locations are opened as read only or in Protected View. By using Protected View, you can read a file, see its contents and enable editing while reducing the risks.</p>
        <div class="ocpAlert">
    <p class="ocpAlertSection"><b class="ocpNote">Note: </b>If your machine has <a href="/en-us/office/application-guard-for-office-9e0fb9c2-ffad-43bf-8ba3-78f785fdba46" class="ocpArticleLink" data-bi-type="anchor">Application Guard for Microsoft 365</a> enabled, documents that previously opened in Protected View will now open in Application Guard for Microsoft 365.</p>
  </div>

        <h2 id="ID0EDX" class="">Why is my file opening in Protected View?</h2>
        
          <p>Protected View is a read-only mode where most editing functions are disabled. There are several reasons why a file opens in Protected View:</p>
          <ul>
    <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>
                <b class="ocpRunInHead">The file was opened from an Internet location</b>   - When you see the message in Protected View that says <b class="ocpLegacyBold"><i class="ocpItalic">"Be careful - files from the Internet can contain viruses. Unless you need to edit, it's safer to stay in Protected View."</i></b>, the file is being opened from the Internet. Files from the Internet can have viruses and other harmful content embedded in them. We recommend you only edit the document if you trust its contents.</p>
              <p>
                 <picture><img class="fluentImage" src="https://support.content.office.net/en-us/media/55aad12e-2bef-4bb7-93a7-22e5cce88202.png" loading="lazy" alt="Protected View for document from internet"></picture>
              </p>
            <meta itemprop="position" content="1">
    </li>
    <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>
                <b class="ocpRunInHead">The file was received as an Outlook attachment and your computer policy has defined the sender as unsafe</b>   - When you see the message in Protected View that says <i class="ocpItalic"><b class="ocpLegacyBold">"Be careful - email attachments can contain viruses. Unless you need to edit, it's safer to stay in Protected View."</b></i>, the file was received from a potentially unsafe sender. We recommend you only edit the document if you trust its contents.</p>
              <p>
                 <picture><img class="fluentImage" src="https://support.content.office.net/en-us/media/5d3f1ec6-5542-43d5-b3b6-b3c4df31ed61.png" loading="lazy" alt="Protected View for untrusted email attachments"></picture>
              </p>
            <meta itemprop="position" content="2">
    </li>
    <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>
                <b class="ocpRunInHead">The file was opened from an unsafe location</b>   - When you see the message in Protected View that says <i class="ocpItalic"><b class="ocpLegacyBold">"This file was opened from a potentially unsafe location. Click for more details."</b></i>, the file was opened from a folder that is unsafe. An example of an unsafe location is your Temporary Internet Files folder. We recommend you only edit the document if you trust its contents.</p>
              <p>
                 <picture><img class="fluentImage" src="https://support.content.office.net/en-us/media/aa842f91-8731-4f94-9ecf-03ea76d971a5.jpg" loading="lazy" alt="Protected View from unsafe location"></picture>
              </p>
            <meta itemprop="position" content="3">
    </li>
    <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>
                <b class="ocpRunInHead">The file is blocked by File Block</b>   - The following images are examples. <a href="/en-us/office/what-is-file-block-10d0e0ab-fecf-4605-befd-1e6563e7686d" class="ocpArticleLink" data-bi-area="bm1" data-bi-type="anchor">Learn more about File Block</a></p>
              <p>
                 <picture><img class="fluentImage" src="https://support.content.office.net/en-us/media/3d816214-8036-417d-bdd7-17d7bac0d75a.png" loading="lazy" alt="Protected View for files blocked by File Block and when editing is not allowed"></picture>
              </p>
              <p>Editing isn’t allowed.</p>
              <p>
                 <picture><img class="fluentImage" src="https://support.content.office.net/en-us/media/a88ef5b7-8b7b-44fa-b851-bf12cda59814.png" loading="lazy" alt="Protected View for documents blocked by File Block and editing is allowed"></picture>
              </p>
              <p>Editing is allowed, but not recommended unless you completely trust its contents.</p>
            <meta itemprop="position" content="4">
    </li>
    <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>
                <b class="ocpRunInHead">File validation failure</b>   - When you see a message in Protected View that says "<i class="ocpItalic"><b class="ocpLegacyBold">Microsoft 365 has detected a problem with this file. Editing it may harm your computer. Click for more details."</b></i>, the file didn’t pass file validation. File validation scans file for security problems that can result from changes in the file structure.</p>
              <p>
                 <picture><img class="fluentImage" src="https://support.content.office.net/en-us/media/3e94e9e7-a555-40d0-8ba1-1e43013d5c2b.png" loading="lazy" alt="Protected View for documents failing Office File Validation"></picture>
              </p>
            <meta itemprop="position" content="5">
    </li>
    <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>
                <b class="ocpRunInHead">The file was opened in Protected View by using the <b class="ocpUI">Open in Protected View</b> option</b>   - When you see the message in Protected View that says <i class="ocpItalic"><b class="ocpLegacyBold">"This file was opened in Protected View. Click for more details.</b></i><b class="ocpLegacyBold">"</b>, you chose to open the file in Protected View. This can be done by using the <b class="ocpUI">Open in Protected View</b> option:</p>
              <p>1. Click <b class="ocpUI">File</b> &gt; <b class="ocpUI">Open</b>.</p>
              <p>2. On the <b class="ocpUI">Open</b> dialog box, click the arrow next to the <b class="ocpUI">Open</b> button.</p>
              <p>3. From the list, click <b class="ocpUI">Open in Protected View</b>.</p>
              <p>
                 <picture><img class="fluentImage" src="https://support.content.office.net/en-us/media/d834a3ee-72ea-4921-b8d3-72c639c014a5.png" loading="lazy" alt="Protected View for documents that are forced to open in Protected View by the user"></picture>
              </p>
            <meta itemprop="position" content="6">
    </li>
    <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>
                <b class="ocpLegacyBold">The file was opened from someone else's OneDrive storage﻿</b>- When you see the message in Protected View that says <b class="ocpLegacyBold"><i class="ocpItalic">"Be careful - This file is from someone else's OneDrive. Unless you trust this person and want to continue collaborating with them, it is safer to stay in Protected View."</i></b>, you opened a document from a OneDrive folder other than your own, for example, when someone has shared a file in OneDrive with you. Such files may be untrusted and could beused to cause harm to your computer. We recommend you trust documents only if you trust the person to whom this OneDrive location belongs.</p>
              <div class="ocpAlert">
        <p class="ocpAlertSection"><b class="ocpNote">Notes: </b></p>
        <ul class="ocpAlertSectionListContainer">
          <li>
            <p>This functionality is currently only available in Microsoft 365 clients.</p>
          </li>
          <li>
            <p>Once you click "Trust Documents From This Person" all subsequent documents shared from this OneDrive location will no longer open in Protected View.</p>
          </li>
        </ul>
      </div>
              <p>
                 <picture><img class="fluentImage" src="https://support.content.office.net/en-us/media/c4decafd-74ad-4140-8c47-e1e0d43403e4.png" loading="lazy" alt="Protected View for documents opened from someone else's OneDrive storage"></picture>
              </p>
            <meta itemprop="position" content="7">
    </li>
  </ul>
          <div class="ocpAlert">
    <p class="ocpAlertSection"><b class="ocpImportant">Important: </b>Administrators can customize the list of potentially unsafe locations to include additional folders they also consider unsafe.</p>
  </div>
        
      </section><section aria-labelledby="ID0EFV" class="ocpSection" id="bm2" data-bi-id="bm2">
        <h2 id="ID0EFV" class="">How do I exit Protected View so that I can edit, save, or print?</h2>
        
          <p>If you must read the file, and don't have to edit it, you can remain in Protected View. If you know the file is from a trustworthy source, and you want to edit, save, or print the file, you can exit Protected View. After you leave Protected View, you've effectively remove read only, and the file becomes a <a href="/en-us/office/trusted-documents-cf872bd8-47ec-4c02-baa5-1fdba1a11b53" class="ocpArticleLink" data-bi-area="bm2" data-bi-type="anchor">trusted document</a>.</p>
        
        
          <section aria-labelledby="ID0EDDBV" class="ocpSection">
            <h3 id="ID0EDDBV" class="">Exit Protected View and edit when the yellow Message Bar appears</h3>
            
              <ul>
      <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                  <p>On the <b class="ocpUI">Message Bar</b>, click <b class="ocpUI">Enable Editing</b>.</p>
                <meta itemprop="position" content="1">
      </li>
    </ul>
            
          </section>
          <section aria-labelledby="ID0EDBBV" class="ocpSection">
            <h3 id="ID0EDBBV" class="">Exit Protected View and edit when the red Message Bar appears</h3>
            
              <ol type="1" itemscope="" itemtype="http://schema.org/ItemList">
      <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                  <p>Click <b class="ocpUI">File</b> &gt; <b class="ocpUI">Edit Anyway</b>.</p>
                <meta itemprop="position" content="1">
      </li>
    </ol>
              <div class="ocpAlert">
      <p class="ocpAlertSection"><b class="ocpCaution">Caution: </b>We recommend you only do this if the file's source and content are trusted by you.</p>
    </div>
            
          </section>
        
      </section><section aria-labelledby="ID0EDT" class="ocpSection" id="bm3" data-bi-id="bm3">
        <h2 id="ID0EDT" class="">Why can't I exit Protected View?</h2>
        
          <p>If you can’t exit Protected View, it's possible that your systems administrator has rules established that prevent leaving Protected View. Speak to your administrator to determine whether such rules have been made.</p>
        
      </section><section aria-labelledby="ID0EFR" class="ocpSection" id="bm4" data-bi-id="bm4">
        <h2 id="ID0EFR" class="">A problem was detected with my file</h2>
        
          <p>
            Microsoft 365 found a problem with your file and it might be a security risk. Opening the file in Protected View helps protect your computer and we recommend that you edit the file only if you trust the person who sent it to you, and if the file doesn’t look suspicious.</p>
        
        
          <section aria-labelledby="ID0EDFBR" class="ocpSection">
            <h3 id="ID0EDFBR" class="">Why do I see this message?</h3>
            
              <p>This message can appear for a malicious file, which was created by a hacker to infect your computer with a virus or steal important information. This message means that editing the file could be dangerous. Sometimes the message appears for files that are damaged, for example:</p>
              <ul>
      <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                  <p>The disk where the file is stored could be worn out or broken.</p>
                <meta itemprop="position" content="1">
      </li>
      <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                  <p>The file was created or edited with a program that has a problem.</p>
                <meta itemprop="position" content="2">
      </li>
      <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                  <p>An unexpected error occurred while copying the file to your computer, which can be caused by a problem with your Internet connection.</p>
                <meta itemprop="position" content="3">
      </li>
      <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                  <p>There could be a problem with how Microsoft 365 looks for problems in files. We work to make it better, but it’s not perfect.</p>
                <meta itemprop="position" content="4">
      </li>
    </ul>
            
          </section>
          <section aria-labelledby="ID0EDDBR" class="ocpSection">
            <h3 id="ID0EDDBR" class="">Can I edit the file?</h3>
            
              <p>If the file is from someone you know and trust, you can choose to edit it. But we recommend that you avoid editing a file that seems suspicious. For example:</p>
              <ul>
      <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                  <p>The file came from someone you don’t know or trust.</p>
                <meta itemprop="position" content="1">
      </li>
      <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                  <p>You weren’t expecting to receive the file or it doesn't seem like the kind of file that person would send you.</p>
                <meta itemprop="position" content="2">
      </li>
      <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                  <p>The content of the file seems unusual, for example, it appears to be a bill for something you never bought, or it’s causing your computer to display errors.</p>
                <meta itemprop="position" content="3">
      </li>
    </ul>
              <p>If the file seems suspicious, close the file and delete it from your computer. We recommend you don’t edit it. To ease suspicion, you can call or email the person who sent you the file to confirm.</p>
              <p>The following image is an example of the <b class="ocpUI">Edit Anyway</b> button in the Microsoft 365 Backstage view.</p>
              <p>
                 <picture><img class="fluentImage" src="https://support.content.office.net/en-us/media/d6c7365f-4dbf-4a53-a1ce-b3578340d538.png" loading="lazy" alt="Protected View Office File Validation Failure Backstage"></picture>
              </p>
            
          </section>

                    Excel 97-2003 files (.xls, .xla, .xlt, .xlm, .xlb, .xlt)</p>

        <h2 id="ID0EDP" class="">I want to change my Protected View settings</h2>
        
          <p>We advise speaking with your administrator before you make changes to your Protected View settings.</p>
          <ol type="1" itemscope="" itemtype="http://schema.org/ItemList">
    <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">

            <meta itemprop="position" content="2">

             <picture><img class="fluentImage" src="https://support.content.office.net/en-us/media/c7681ab5-fa2e-4944-97d2-88affe6776aa.png" loading="lazy" alt="Trust Center Protected View settings"></picture>
          </p>
        
      </section><section aria-labelledby="ID0EDN" class="ocpSection" id="bm6" data-bi-id="bm6">
        <h2 id="ID0EDN" class="">Protected View Trust Center settings explained</h2>
        
          <p></p>
          <ul>
    <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>
                <b class="ocpRunInHead">Enable Protected View for files originating from the Internet</b>    The Internet is considered an unsafe location because of its many opportunities for malicious intent.</p>
            <meta itemprop="position" content="1">
    </li>
    <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>
                <b class="ocpRunInHead">Enable Protected View for files that are located in potentially unsafe locations</b>    This refers to folders on your computer or network that are considered unsafe, such as the Temporary Internet folder or other folders assigned by your administrator.</p>
            <meta itemprop="position" content="2">
    </li>
    <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>
                <b class="ocpRunInHead">Enable Protected View for Outlook attachments</b>    Attachments in emails can come from unreliable or unknown sources.</p>
            <meta itemprop="position" content="3">
    </li>
    <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>
                <b class="ocpLegacyBold">Always open untrusted Text-Based files</b>
                <b class="ocpUI">(.csv, .dif and .sylk) </b>
                <b class="ocpLegacyBold">in protected view  </b> If this Excel-specific setting is enabled, text-based files opened from an untrusted location are always opened in Protected View. If you disable or don’t configure this setting, text-based files opened from an untrusted location are not opened in Protected View.  </p>
              <p>This setting can also be configured by an administrator as a policy via Group Policy or the <a href="https://docs.microsoft.com/en-us/DeployOffice/overview-office-cloud-policy-service" target="_blank" class="ocpExternalLink" data-bi-area="bm6" data-bi-type="anchor">Microsoft 365 cloud policy service</a>. </p>
              <div class="ocpAlert">
        <p class="ocpAlertSection"><b class="ocpNote">Note: </b>This policy setting only applies to Microsoft 365 products.</p>
      </div>
            <meta itemprop="position" content="4">
    </li>
    <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>
                <b class="ocpLegacyBold">Always open untrusted Database files (.dbf) in protected view  </b>If this Excel-specific setting is enabled, database files opened from an untrusted location are always opened in Protected View. If you disable or don’t configure this setting, database files opened from an untrusted location are not opened in Protected View.  </p>
              <p>This setting can also be configured by an administrator as a policy via Group Policy or the <a href="https://docs.microsoft.com/en-us/DeployOffice/overview-office-cloud-policy-service" target="_blank" class="ocpExternalLink" data-bi-area="bm6" data-bi-type="anchor">Microsoft 365 cloud policy service</a>. </p>
              <div class="ocpAlert">
        <p class="ocpAlertSection"><b class="ocpNote">Note: </b>This policy setting only applies to Microsoft 365 products.</p>
      </div>
              <p></p>
            <meta itemprop="position" content="5">
    </li>
  </ul>
        
      </section><section aria-labelledby="ID0EDL" class="ocpSection" id="bmrevoke" data-bi-id="bmrevoke">
        <h2 id="ID0EDL" class="">I want to revoke trust from a document/documents that I've previously trusted to not open in Protected View</h2>
        
          <p>If you previously trusted a document or documents to open outside of Protected View by either (1) clicking "Enable Editing" or "Trust Documents From This Person" in the message bar or (2) clicking on "Edit Anyway" when the file fails validation, please refer to guidance under <a href="/en-us/office/trusted-documents-cf872bd8-47ec-4c02-baa5-1fdba1a11b53" class="ocpArticleLink" data-bi-area="bmRevoke" data-bi-type="anchor">Trusted documents</a> for removing this trust decision and making such documents re-open in Protected View.</p>
        
      </section><section aria-labelledby="ID0EDJ" class="ocpSection" id="bm7" data-bi-id="bm7">
        <h2 id="ID0EDJ" class="">What happens to add-ins in Protected View?</h2>
        
          <p>Add-ins may run when a file opens in Protected View, but may not function as expected. If your add-ins aren’t running correctly, contact the add-in's author. An updated version, which is compatible with Protected View, may be needed.</p>
        
      </section><section aria-labelledby="ID0EDH" class="ocpSection" id="bm8" data-bi-id="bm8">
        <h2 id="ID0EDH" class="">What happens to cloud fonts in Protected View?</h2>
        
          <p>It's possible the person who sent you the document used a cloud font, which is a font that doesn't ship with Windows or Microsoft 365 but which must be downloaded from the Internet the first time it's used. If so, and it's a font you don't already have installed, that font won't download while you're in Protected View. Word will try to substitute another font that will hopefully look okay. If you're confident that the document is safe, and want to see it the way the author intended, you'll need to enable editing so that Word can download and install the correct font.</p>
          <div class="ocpAlert">
    <p class="ocpAlertSection"><b class="ocpNote">Note: </b>If Word is unable to find any compatible fonts to substitute you might see black boxes where the text should be. Enabling editing so the correct font can download should fix the problem.</p>
  </div>
        
      </section><section aria-labelledby="ID0EDF" class="ocpSection" id="bkmk_screenreader" data-bi-id="bkmk_screenreader">
        <h2 id="ID0EDF" class="">How do I use Protected View with a screen reader?</h2>
        
          <p>When you are in protected view, editing is locked, so you can't cursor around the document as expected. However, if you need to navigate through a document in Protected View with a screen reader, you can press F7 to turn on caret browsing. This action should allow you to navigate through the text without being in edit mode.</p>
