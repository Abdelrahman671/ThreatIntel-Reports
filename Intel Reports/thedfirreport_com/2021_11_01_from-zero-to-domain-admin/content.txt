			<h1 class="entry-title">From Zero to Domain Admin</h1>
		<div class="entry-meta">
			<span class="posted-on"><a href="https://thedfirreport.com/2021/11/01/from-zero-to-domain-admin/" rel="bookmark"><time class="entry-date published" datetime="2021-11-01T00:57:41+00:00">November 1, 2021</time></a></span>		</div><!-- .entry-meta -->

<h3 id="intro">Intro</h3>
<p>This report will go through an intrusion from July that began with an email, which included a link to Google&#8217;s Feed Proxy service that was used to download a malicious Word document. Upon the user enabling macros, a Hancitor dll was executed, which called the usual suspect, Cobalt Strike.</p>
<p>Various different enumeration and lateral movement tactics were observed on the network, along with the exploitation of Zerologon to elevate to domain administrator and gain full control over the domain. The threat actor was able to go from zero access to domain admin, in just under one hour.</p>
<h3 id="case-summary">Case Summary</h3>
<p>Like with many infections today, the threat actors gained initial access on a system through a malicious document email campaign, which made use of the Hancitor downloader. The document, upon opening and enabling of macros, would write and then execute a dll file from the users appdata folder.</p>
<p>The Hancitor dll downloaded and executed multiple payloads including a Cobalt Strike stager and Ficker Stealer. The threat actors then began port scanning for SMB and a few backup systems such as Synology, Veeam and Backup Exec.</p>
<p>After that, a battery of Windows utilities were run to check the windows domain trusts, domain administrators, domain controllers, and test connectivity. They then checked access to remote systems by connecting to the C$ share.</p>
<p>The threat actors proceeded to move laterally to multiple other servers on the network by making use of existing local administrative rights of a compromised user. Cobalt Strike beacons were deployed to each server to facilitate remote access. Furthermore, the threat actors dropped an obfuscated PowerShell script on one of the machines to further their access. The PowerShell script loaded the malicious code into memory and started beaconing out to the remote command and control server.</p>
<p>Next, the threat actors used a custom implementation of the Zerologon (CVE-2020-1472) exploit (zero.exe) against one of the domain controllers. The domain controllers were vulnerable, and as a result, the operators managed to dump the domain administrator&#8217;s NTLM hash. The threat actors then pivoted to the two domain controllers and deployed Cobalt Strike beacons.</p>
<p>The threat actors continued pivoting to key systems including additional domain controllers, backup servers, and file shares, using Cobalt Strike. Once on these systems, additional scanning occurred using a binary called check.exe that ran ICMP sweeps across the network.</p>
<p>Within two hours of the initial malicious document execution, the threat actors had a foothold on all key systems in the environment. Similar to a <a href="https://thedfirreport.com/2021/06/28/hancitor-continues-to-push-cobalt-strike/" target="_blank" rel="nofollow noopener">previous case</a>, the threat actors were evicted before completing their mission and as a result their final actions could not be observed.</p>
<h3 id="services">Services</h3>
<p>We offer multiple services including a <a href="https://thedfirreport.com/services/" target="_blank" rel="nofollow noopener">Threat Feed service</a> which tracks Command and Control frameworks such as Cobalt Strike, Metasploit, Empire, PoshC2, BazarLoader, etc. More information on this service and others can be found <a href="https://thedfirreport.com/services/" target="_blank" rel="nofollow noopener">here</a>.</p>
<p>The Cobalt Strike servers in this case were added to the Threat Feed on 5/16 and 7/15 .</p>
<p>We also have artifacts and IOCs available from this case such as pcaps, memory captures, files, event logs including Sysmon, Kape packages, and more, under our <a href="https://www.patreon.com/thedfirreport" target="_blank" rel="nofollow noopener">Security Researcher and Organization</a> services.</p>

<h3 id="execution">Execution</h3>
<p>Three files were downloaded by Hancitor from 4a5ikol[.]ru (8.211.241.0) including two Cobalt Strike stagers and Ficker Stealer.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/10/02_2-in-mem-download.png" alt="enter image description here" /></p>
<p>Hancitor then launched multiple instances of svchost.exe and process injected them with Cobalt Strike.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/10/03-hanictor-injects-CS.png" alt="enter image description here" /></p>
<p>The following diagram shows the initial execution process from the WINWORD.exe to the Cobalt Strike Beacons that were injected into memory by Hancitor.</p>
<p><img decoding="async" class="alignnone size-full wp-image-4414" src="https://thedfirreport.com/wp-content/uploads/2021/11/Hancitor-Process-Tree.png" alt="" width="1276" height="690" srcset="https://thedfirreport.com/wp-content/uploads/2021/11/Hancitor-Process-Tree.png 1276w, https://thedfirreport.com/wp-content/uploads/2021/11/Hancitor-Process-Tree-300x162.png 300w, https://thedfirreport.com/wp-content/uploads/2021/11/Hancitor-Process-Tree-1024x554.png 1024w, https://thedfirreport.com/wp-content/uploads/2021/11/Hancitor-Process-Tree-768x415.png 768w" sizes="(max-width: 1276px) 100vw, 1276px" /></p>
<p>Lastly, a Cobalt Strike command and control server was pinged before they copied the Cobalt Strike DLL and batch file, which were used to facilitate lateral movement.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/10/5-CS-ping-and-run.png" alt="enter image description here" /></p>

<h3 id="privilege-escalation">Privilege Escalation</h3>
<p>The threat actor made use of a custom developed implementation of Zerologon (CVE-2020-1472) executed from a file named “zero.exe”.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/10/00-Zero_exe.png" alt="enter image description here" /></p>
<pre><code><span class="hljs-tag">zero</span><span class="hljs-class">.exe</span> 10<span class="hljs-class">.10</span><span class="hljs-class">.10</span><span class="hljs-class">.10</span> <span class="hljs-tag">DomainControllerHostName</span> <span class="hljs-tag">domain</span><span class="hljs-class">.name</span> <span class="hljs-tag">administrator</span> <span class="hljs-tag">-c</span> "<span class="hljs-tag">powershell</span><span class="hljs-class">.exe</span>" 
</code></pre>
<p>Once “zero.exe” is run it will provide the threat actor with the NTLM hash of the specified username, a Domain Administrator account in this case.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/10/01-zero-exe-running-example.png" alt="enter image description here" /></p>
<p>On the Domain Controller a service (Event ID 7045) will be created that will run the Reset-ComputerMachinePassword PowerShell Cmdlet.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/10/02-Zerologon-Reset-Machine-Password-Service.png" alt="enter image description here" /></p>
<p>The service will then be executed and the machine account password will be reset.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/10/03-zerologon-reset-machine-password-4688.png" alt="enter image description here" /></p>
<p>Zerologon will create an Event ID 4624 for the domain controller computer account attempting to authenticate. The main red flag is the source network address IP differing from the IP of the domain controller, which in this case is set to the beachhead workstation on which zero.exe was executed.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/10/04-4024-DC1-account-from-BH-IP-source.png" alt="enter image description here" /></p>
<p>Lastly, Event ID 4648 will be logged on the beachhead machine indicating the zero.exe process was used to connect to a domain controller.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/10/05-4648-on-BH.png" alt="enter image description here" /></p>
<p>A blog post by Blackberry can be referenced to learn more about this custom developed Zerologon file used: <a href="https://blogs.blackberry.com/en/2021/03/zerologon-to-ransomware" target="_blank" rel="nofollow noopener">https://blogs.blackberry.com/en/2021/03/zerologon-to-ransomware</a>.</p>
<p>For more information on detecting Zerologon check out Kroll&#8217;s <a href="https://www.kroll.com/en/insights/publications/cyber/cve-2020-1472-zerologon-exploit-detection-cheat-sheet" target="_blank" rel="noopener">Zerologon Exploit Detect Cheat Sheet</a>.</p>
<h3 id="defense-evasion">Defense Evasion</h3>
<p>Upon Hancitor launching on the system, it process injected into multiple instances of svchost.exe and rundll32.exe. Memory segments can be seen allocated with Execute, Read, and Write permissions, indicating that executable code is stored.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/10/1-CS-injected-into-memory-on-rundll32.png" alt="enter image description here" /></p>
<p>Anomalous parent and child process relationships can be seen on the system that Hancitor was executed on, including rundll32.exe spawning svchost.exe and svchost.exe spawning cmd.exe.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/10/1.2-odd-parent-child-process.png" alt="enter image description here" /></p>
<p>Moreover, the Cobalt Strike DLL stager was executed with a specific command line parameter which is used as a sandbox evasion feature. In this case it is the number 11985756.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/10/02-cor-batch-script.png" alt="enter image description here" /></p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/10/2-CS-sandbox-avoid.png" alt="enter image description here" /></p>
<p>Lastly, a PowerShell loader named agent1.ps1 used heavy obfuscation to conceal the execution flow and hide the final shellcode. After many iterations, the script would deobfuscate and run-in memory. The shellcode is responsible for loading a PE file into memory and calling out to 64.235.39[.]32 for further instructions.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/10/3-Obfuscated-PowerShell.png" alt="enter image description here" /></p>
<h3 id="credential-access">Credential Access</h3>
<p>The only credential access observed was through Zerologon, which was used to retrieve the domain administrator&#8217;s NTLM hash.</p>
<h3 id="discovery">Discovery</h3>
<p>Discovery started with a port scan initiated by the Hancitor dll.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/10/5295-discovery1.png" alt="enter image description here" /></p>
<p>After SMB was scanned we saw scans of 5000/tcp, 9392/tcp, 6106/tcp. The threat actors were scanning for backup products such as Synology, Backup Exec and Veeam.</p>
<p><img decoding="async" class="alignnone size-full wp-image-4397" src="https://thedfirreport.com/wp-content/uploads/2021/10/1-3.png" alt="" width="1422" height="531" srcset="https://thedfirreport.com/wp-content/uploads/2021/10/1-3.png 1422w, https://thedfirreport.com/wp-content/uploads/2021/10/1-3-300x112.png 300w, https://thedfirreport.com/wp-content/uploads/2021/10/1-3-1024x382.png 1024w, https://thedfirreport.com/wp-content/uploads/2021/10/1-3-768x287.png 768w" sizes="(max-width: 1422px) 100vw, 1422px" /></p>
<p>This was followed by a battery of discovery command using the built in Microsoft utilities to discover domain controllers, administrators, connectivity checks and other items.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/10/01-Inital-Enum.png" alt="enter image description here" /></p>
<pre><code>C:\Windows\system32\cmd.exe /C net time

<p>Notice above, the threat actors pinged 190.114.254[.]116 which is one of the Cobalt Strike servers they later used.</p>
<p>The threat actors enumerated local administrative access on remote systems by checking access to the C$ share for hosts discovered after the port scan.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/10/5295-discovery2.png" alt="enter image description here" /></p>
<p>We observed a PowerShell script named comp2.ps1 that was executed on every Domain Controller in the environment. This script used the Active Directory RSAT module to get a list of computers and place them in a file named ‘comps.txt.’</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/10/02_1-Generate-comps-txt-for-check-program.png" alt="enter image description here" /></p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/10/02_2-comps-txt-powershell-script.png" alt="enter image description here" /></p>
<p>A program named check.exe was observed using the comps.txt text file. This program will take a list of IP addresses and hostnames from comps.txt and check if they are online using ICMP. The online hosts will then be directed to the check.txt text file.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/10/02_3-check_exe-run.png" alt="enter image description here" /></p>
<p>The check.exe file contains three parameters that can be used one at a time:</p>

<h3 id="lateral-movement">Lateral Movement</h3>
<p>The threat actors pivoted towards multiple hosts on the domain from the beachhead. The main actions involved copying a Cobalt Strike DLL beacon and a batch script to run the DLL (cor.dll, cor.bat, GAS.dll, GAS.bat). Operators executed the batch script through a remotely created service on the target system.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/10/01-Lateral-Moveemnt-to-DC2-using-cor_bat.png" alt="enter image description here" /></p>
<p>The following shows one of the batch scripts used to run a Cobalt Strike payload.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/10/02-cor-batch-script-1.png" alt="enter image description here" /></p>
<p>An obfuscated PowerShell script named ‘agent1.ps1’ was dropped on a machine through a Cobalt Strike Beacon. The PowerShell script had instructions to deobfuscate shellcode and run it in memory as a thread in the same PowerShell process.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/10/03-snippet-of-agent-ps1-code.png" alt="enter image description here" /></p>
<p>The shellcode itself also has a PE file embedded inside of itself. Once the shellcode is running this PE file will be loaded into memory and executed. You can see this from the memory dump MZ header denoting the PE binary loaded into the PowerShell process.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/10/04-PE-in-memory-of-powershell.png" alt="enter image description here" /></p>
<p>The PE file is of a small size and has the capability to beacon out at regular intervals to a command-and-control server on 64.235.39[.]32 to retrieve instructions.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/10/05_curl_http.png" alt="enter image description here" /></p>
<p>The Visual C# Command Line Compiler was observed being invoked by the PowerShell script where the shellcode was executed. This is most likely instructions that the previously discussed PE file retrieved from the remote command and control server.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/10/06-agent-powershell-script-run.png" alt="enter image description here" /></p>
<h3 id="command-and-control">Command and Control</h3>
<p>Hancitor contacted its servers over HTTP and advertised details about the compromised machine, user, and domain while also retrieving instructions from the command and control server (1). From another dedicated location, 4a5ikol[.]ru, two Cobalt Strike beacons and Ficker Stealer malware were downloaded through HTTP (2).</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/10/1.1-Hancitor-inital-traffic.png" alt="enter image description here" /></p>
<p>A successful connection from Ficker Stealer was not observed. A domain was queried; however, the response returned an error.</p>
<p><img decoding="async" title="enter image title here" src="https://thedfirreport.com/wp-content/uploads/2021/10/1.2-Ficker-Stealer-dns-error.png" alt="enter image description here" /></p>
<p>Cobalt Strike was also observed to be making use of HTTP.</p>
<p><img loading="lazy" decoding="async" class="alignnone size-full wp-image-4428" src="https://thedfirreport.com/wp-content/uploads/2021/11/1-1.png" alt="" width="958" height="280" srcset="https://thedfirreport.com/wp-content/uploads/2021/11/1-1.png 958w, https://thedfirreport.com/wp-content/uploads/2021/11/1-1-300x88.png 300w, https://thedfirreport.com/wp-content/uploads/2021/11/1-1-768x224.png 768w" sizes="auto, (max-width: 958px) 100vw, 958px" /></p>
<p>Lastly, the shellcode executed by the agent1.ps1 PowerShell loader, was observed loading a PE file into memory that would beacon out at consistent intervals to 64.235.39[.]32. Further encrypted network activity was also observed to this IP address. Unfortunately, the tool sending these connections could not be definitively determined.</p>
<p>The user agent for this was curl/7.55.1</p>
