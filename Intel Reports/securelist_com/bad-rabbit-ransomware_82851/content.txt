<h3 id="what-happened">What happened?</h3>
<p>On October 24th we observed notifications of mass attacks with ransomware called Bad Rabbit. It has been targeting organizations and consumers, mostly in Russia but there have also been reports of victims in Ukraine. Here&#8217;s what a ransom message looks like for the unlucky victims:</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07170246/bad_rabbit_ransomware_01.png" class="magnificImage"><img decoding="async" class="size-full wp-image-82843 aligncenter" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07170246/bad_rabbit_ransomware_01.png" alt="" width="644" height="392" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07170246/bad_rabbit_ransomware_01.png 644w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07170246/bad_rabbit_ransomware_01-300x183.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07170246/bad_rabbit_ransomware_01-330x200.png 330w" sizes="(max-width: 644px) 100vw, 644px" /></a></p>
<h3 id="what-is-bad-rabbit">What is Bad Rabbit?</h3>
<p>Bad Rabbit is a previously unknown ransomware family.</p>
<h3 id="how-is-bad-rabbit-distributed">How is Bad Rabbit distributed?</h3>
<p>The ransomware <a href="https://encyclopedia.kaspersky.com/glossary/dropper/" rel="noopener noreferrer" target="_blank">dropper</a> was distributed with the help of <a href="https://encyclopedia.kaspersky.com/glossary/drive-by-attack/?utm_source=securelist&#038;utm_medium=blog&#038;utm_campaign=termin-explanation" rel="noopener noreferrer" target="_blank">drive-by attacks</a>. While the target is visiting a legitimate website, a malware dropper is being downloaded from the threat actor&#8217;s infrastructure. No exploits were used, so the victim would have to manually execute the malware dropper, which pretends to be an Adobe Flash installer. However, our analysis confirmed that Bad Rabbit uses the EternalRomance exploit as an infection vector to spread within corporate networks. The same exploit was used in the ExPetr.</p>
<p>We&#8217;ve detected a number of compromised websites, all of which were news or media websites.</p>
<h3 id="whom-does-it-target">Whom does it target?</h3>
<p>Most of the targets are located in Russia. Similar but fewer attacks have also been seen in other countries &#8211; Ukraine, Turkey and Germany. Overall, there are almost 200 targets, according to the KSN statistics.</p>
<h3 id="since-when-does-kaspersky-lab-detect-the-threat">Since when does Kaspersky Lab detect the threat?</h3>
<p>We have been proactively detecting the original vector attack since it began on the morning of October 24. The attack lasted until midday, although ongoing attacks were detected at 19.55 Moscow time. The server from which the Bad rabbit dropper was distributed went down in the evening (Moscow time).</p>
<h3 id="how-is-it-different-to-expetr-or-it-is-the-same-malware">How is it different to ExPetr? Or it is the same malware?</h3>
<p>Our observations suggest that this been a targeted attack against corporate networks, using methods similar to those used during <a href="https://securelist.com/schroedingers-petya/78870/" rel="noopener noreferrer" target="_blank">the ExPetr attack</a>. What&#8217;s more, the code analysis showed a notable similarity between the code of ExPetr and Bad Rabbit binaries.</p>
<h3 id="technical-details">Technical details</h3>
<p>According to our telemetry, the ransomware is spread via a drive-by attack.</p>
<p>The ransomware dropper is distributed from hxxp://1dnscontrol[.]com/flash_install.php</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07170242/bad_rabbit_ransomware_02.png" class="magnificImage"><img loading="lazy" decoding="async" class="size-full wp-image-82844 aligncenter" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07170242/bad_rabbit_ransomware_02.png" alt="" width="663" height="516" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07170242/bad_rabbit_ransomware_02.png 663w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07170242/bad_rabbit_ransomware_02-300x233.png 300w" sizes="(max-width: 663px) 100vw, 663px" /></a></p>

<h3 id="encryption-scheme">Encryption scheme</h3>
<p>As we mentioned, the Bad Rabbit ransomware encrypts a victim&#8217;s files and disk. Files are encrypted with the following algorithms:</p>
<ol>
<li>AES-128-CBC</li>
<li>RSA-2048</li>
</ol>
<p>It is a default encryption scheme for ransomware. </p>
<p>An interesting fact is that the ransomware enumerates all running processes and compares the hashed name of each process with embedded hash values. It is important to mention that the hashing algorithm is similar to the ExPetr one.</p>

<p>The partitions on the victim&#8217;s disks are encrypted with the help of the DiskCryptor driver dcrypt.sys (which is installed into C:Windowscscc.dat). The ransomware sends the necessary IOCTL codes to this driver. Some functions are taken as is from the sources of DiskCryptor (drv_ioctl.c), others seem to be implemented by the malware developers.</p>
<p>The disk partitions on the infected machine are encrypted by the DiskCryptor driver using the AES cipher in XTS mode. The password is generated by dispci.exe using the WinAPI function CryptGenRandom and has a length of 32 symbols.</p>
<h3 id="decryption-opportunity-assessment">Decryption opportunity assessment</h3>
<p>Unlike ExPetr, the evidence suggests that Bad Rabbit is not intended as a wiper. Previously, <a href="https://securelist.com/expetrpetyanotpetya-is-a-wiper-not-ransomware/78902/">in our article</a> we wrote that the threat actors behind ExPetr were technically unable to decrypt MFT that was encrypted with the GoldenEye component. In the case of Bad Rabbit, however, the malware algorithm suggests that the threat actors have the technical means to decrypt the password necessary for disk decryption.</p>
<p>The data shown on the screen of an infected machine as &#8220;personal installation key#1&#8221; is an encrypted by RSA-2048 and base64-encoded binary structure that contains the following information gathered from the infected system:</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07165808/171025-badrabbit-upd-2-2.png" class="magnificImage"><img loading="lazy" decoding="async" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07165808/171025-badrabbit-upd-2-2.png" alt="" width="720" height="177" class="aligncenter size-full wp-image-82946" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07165808/171025-badrabbit-upd-2-2.png 720w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07165808/171025-badrabbit-upd-2-2-300x74.png 300w" sizes="(max-width: 720px) 100vw, 720px" /></a></p>
<p>The threat actors can use their own private RSA key to decrypt this structure. After decryption they can send this information to the victim.</p>
<p><strong>Please note that, despite what it says in other vendors&#8217; reports, the value of the id field which is passed to dispci.exe is just a 32-bit number used to distinguish different infected machines, and not the AES key which is used for disk encryption.</strong></p>
<p>As part of our analysis, we extracted the password generated by the malware during a debugging session and attempted to enter this password when the system was locked after reboot. The password indeed worked and the boot-up process continued.</p>
<p><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07165805/171025-badrabbit-upd-2-3.png" class="magnificImage"><img loading="lazy" decoding="async" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07165805/171025-badrabbit-upd-2-3.png" alt="" width="633" height="399" class="aligncenter size-full wp-image-82948" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07165805/171025-badrabbit-upd-2-3.png 633w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07165805/171025-badrabbit-upd-2-3-300x189.png 300w" sizes="(max-width: 633px) 100vw, 633px" /></a></p>
<p>Unfortunately, we have to conclude that at this point there&#8217;s no way to decrypt disk and victim files without the threat actor&#8217;s RSA-2048 private key. The symmetric encryption keys are securely generated on the ransomware side which makes attempts to guess the keys unfeasible in practice.</p>
<p>However, we found a flaw in the code of dispci.exe: the malware doesn&#8217;t wipe the generated password from the memory, which means that there is a slim chance to extract it before the dispci.exe process terminates. In the picture below, note that while the variable dc_pass (which will be passed to the driver) is securely erased after use, that&#8217;s not the case for the variable rand_str which holds the original copy of the password.</p>
<div id="attachment_82891" style="width: 631px" class="wp-caption aligncenter"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07170057/171025-badrabbit-upd-13.png" class="magnificImage"><img loading="lazy" decoding="async" aria-describedby="caption-attachment-82891" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07170057/171025-badrabbit-upd-13.png" alt="" width="621" height="677" class="size-full wp-image-82891" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07170057/171025-badrabbit-upd-13.png 621w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07170057/171025-badrabbit-upd-13-275x300.png 275w" sizes="(max-width: 621px) 100vw, 621px" /></a><p id="caption-attachment-82891" class="wp-caption-text">Pseudocode of the procedure that generates the password and encrypts the disk partitions</p></div>
<h3 id="file-encryption">File encryption</h3>
<p>As we wrote before, the trojan uses a common file encryption scheme. It generates a random 32-bytes-length string and uses it in the key derivation algorithm. Unfortunately, the trojan uses the CryptGenRandom function when generating this string.</p>
<div id="attachment_82949" style="width: 552px" class="wp-caption aligncenter"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07165801/171025-badrabbit-upd-2-4.png" class="magnificImage"><img loading="lazy" decoding="async" aria-describedby="caption-attachment-82949" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07165801/171025-badrabbit-upd-2-4.png" alt="" width="542" height="425" class="size-full wp-image-82949" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07165801/171025-badrabbit-upd-2-4.png 542w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07165801/171025-badrabbit-upd-2-4-300x235.png 300w" sizes="(max-width: 542px) 100vw, 542px" /></a><p id="caption-attachment-82949" class="wp-caption-text">Key derivation algorithm</p></div>
<p>The encrypted password, along with information about the infected system is written into Readme file as &#8220;personal installation key#2&#8221;.</p>
<div id="attachment_82950" style="width: 821px" class="wp-caption aligncenter"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07165756/171025-badrabbit-upd-2-5.png" class="magnificImage"><img loading="lazy" decoding="async" aria-describedby="caption-attachment-82950" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07165756/171025-badrabbit-upd-2-5.png" alt="" width="811" height="766" class="size-full wp-image-82950" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07165756/171025-badrabbit-upd-2-5.png 811w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07165756/171025-badrabbit-upd-2-5-300x283.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07165756/171025-badrabbit-upd-2-5-768x725.png 768w" sizes="(max-width: 811px) 100vw, 811px" /></a><p id="caption-attachment-82950" class="wp-caption-text">Ransom note creation routine</p></div>
<p>An interesting fact is that the trojan cannot encrypt files which have a Read-only attribute.</p>
<h3 id="file-recovery-possibility">File recovery possibility</h3>
<p>We have discovered that Bad Rabbit does not delete shadow copies after encrypting the victim&#8217;s files. It means that if the shadow copies had been enabled prior to infection and if the full disk encryption did not occur for some reason, then the victim can restore the original versions of the encrypted files by the means of the standard Windows mechanism or 3rd-party utilities.</p>
<div id="attachment_82950" style="width: 930px" class="wp-caption aligncenter"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07165751/bad_rabbit_ransomware_09.png" class="magnificImage"><img loading="lazy" decoding="async" aria-describedby="caption-attachment-82950" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07165751/bad_rabbit_ransomware_09.png" alt="" width="920" height="584" class="aligncenter size-full wp-image-82960" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07165751/bad_rabbit_ransomware_09.png 920w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07165751/bad_rabbit_ransomware_09-300x190.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2017/10/07165751/bad_rabbit_ransomware_09-768x488.png 768w" sizes="(max-width: 920px) 100vw, 920px" /></a><p id="caption-attachment-82950" class="wp-caption-text"><br />
Shadow copies remain unharmed by Bad Rabbit</p></div>
<h3 id="recommendations">Recommendations</h3>
<p>Kaspersky Lab corporate customers are also advised to:</p>
<ul>
<li>make sure that all protection mechanisms are activated as recommended; and that KSN and System Watcher components (which are enabled by default) are not disabled.</li>
<li>update the antivirus databases immediately.</li>
</ul>
<p>The abovementioned measures should be sufficient. However, as additional precautions we advise the following:</p>
<ul>
<li>restricting execution of files with the paths <strong>c:windowsinfpub.dat</strong> and <strong>C:Windowscscc.dat</strong> in Kaspersky Endpoint Security.</li>
<li>configuring and enabling Default Deny mode in the Application Startup Control component of Kaspersky Endpoint Security to ensure and enforce proactive defense against this and other attacks.</li>
</ul>
<p>Kaspersky Lab products detect this threat with the following verdicts:</p>
<ul>
<li>Trojan-Ransom.Win32.Gen.ftl</li>
<li>Trojan-Ransom.Win32.BadRabbit</li>
<li>DangerousObject.Multi.Generic</li>
<li>PDM:Trojan.Win32.Generic</li>
<li>Intrusion.Win.CVE-2017-0147.sa.leak</li>
</ul>
<p><strong>IOCs:<br />
</strong>http://1dnscontrol[.]com/<br />
fbbdc39af1139aebba4da004475e8839 &#8211; install_flash_player.exe<br />
1d724f95c61f1055f0d02c2154bbccd3 &#8211; C:Windowsinfpub.dat<br />
b14d8faf7f0cbcfad051cefe5f39645f &#8211; C:Windowsdispci.exe</p>
