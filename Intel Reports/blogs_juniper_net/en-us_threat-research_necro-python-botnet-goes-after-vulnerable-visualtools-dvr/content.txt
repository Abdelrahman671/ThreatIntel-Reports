			<div class="dfd-blog-title">Necro Python Botnet Goes After Vulnerable VisualTools DVR</div>
							<div class="entry-meta meta-bottom">
	<span class="entry-date">October 11, 2021</span>	<div class="meta-author">
	<span class="before-author">by</span>

	<span class="byline author vcard">
	<a href="https://blogs.juniper.net/author/paulkim" title="Posts by Paul Kimayong" class="author url fn" rel="author">Paul Kimayong</a>	</span>

	</div>

	<div class="entry-content">

		<p><img loading="lazy" class="alignnone size-full wp-image-25873" src="https://blogs.juniper.net/wp-content/uploads/2021/10/Necro-Python-Botnet.gif" alt="" width="1024" height="571" /></p>
<p>In the last week of September 2021, Juniper Threat Labs detected a new activity from Necro Python (a.k.a N3Cr0m0rPh , Freakout, Python.IRCBot) that is actively exploiting some services, including a new exploit added to its arsenal. This new exploit targets Visual Tools DVR VX16 4.2.28.0 from visual-tools.com (no CVE number is assigned to this vulnerability). Successful exploitation will download the bot into the system and install a Monero miner.</p>
<p>Necro was first discovered in <a href="https://research.checkpoint.com/2021/freakout-leveraging-newest-vulnerabilities-for-creating-a-botnet/">January</a>. The threat actor made a move in <a href="https://blog.netlab.360.com/necro-upgrades-again-using-tor-dynamic-domain-dga-and-aiming-at-both-windows-linux/">March</a> and in <a href="https://blog.talosintelligence.com/2021/06/necro-python-bot-adds-new-tricks.html">May, </a>adding new exploits to its arsenal.</p>
<p>Necro bot is an interesting python bot that has many functions which include the following:</p>
<ul>
<li>Network Sniffer</li>
<li>Spreading by exploits</li>
<li>Spreading by brute-force</li>
<li>Using Domain Generation Algorithm</li>
<li>Installing a Windows rootkit</li>
<li>Receiving and executing bot commands</li>
<li>Participating in DDoS attacks</li>
<li>Infecting HTML, JS, PHP files</li>
<li>Installing Monero Miner</li>
</ul>
<p>The script can run in both Windows and Linux environments. The script has its own polymorphic engine to morph itself every execution which can bypass signature-based defenses. This works by reading every string in its code and encrypting it using a hardcoded key.</p>
<figure id="attachment_25872" class="thumbnail wp-caption alignnone" style="width: 725px"><img loading="lazy" class="wp-image-25872 " src="https://blogs.juniper.net/wp-content/uploads/2021/10/Morph-1024x462.png" alt="" width="725" height="327" srcset="https://blogs.juniper.net/wp-content/uploads/2021/10/Morph-1024x462.png 1024w, https://blogs.juniper.net/wp-content/uploads/2021/10/Morph-300x135.png 300w, https://blogs.juniper.net/wp-content/uploads/2021/10/Morph-768x346.png 768w, https://blogs.juniper.net/wp-content/uploads/2021/10/Morph.png 1178w" sizes="(max-width: 725px) 100vw, 725px" /><figcaption class="caption wp-caption-text">Necro Python’s polymorphism, before and after</figcaption></figure>
<h3></h3>
<h4>Domain Generation Algorithm</h4>
<p>Necro uses DGA for both its CnC and download server. It selects from a list of dynamic DNS services as its domain, e.g., <strong>ddns.net</strong> and prefixes that with 10-19 random characters. E.g., ‘<strong>3ood3dfcqchro.ddns.net’</strong></p>
<p>The domains are pseudo-randomly generated using a hardcoded seed, <strong>0xFAFFDED00001,</strong> and a counter is added until <strong>0xFD</strong> (253 in decimal) before the counter is reset to 0. The seed controls the domain to be generated.  In effect, it can generate up to 253 unique domains.</p>
<p>This seed is different from the previous campaigns. For instance, the sample used in the March attack used a different seed, <strong>0x7774DEAD.</strong></p>
<p>From this list of generated domains, it connects to them one by one to see which one is online. During our analysis, the following DGA domain was active:</p>
<ul>
<li>gtmpbeaxruxy[.]myftp.org</li>
</ul>

<p>We have noted a few changes on this bot from the previous version. First, it removed the SMB scanner which was observed in the May 2021 attack. Second, it changed the url that it injects to script files on the compromised system. Previously, it used a hardcoded url, ‘<strong>ublock-referer[.]dev/campaign.js</strong>’ and injects this on the scripts and now it uses the DGA for its url, i.e., <strong>‘DGA_DOMAIN/campaign.js’</strong>. As noted in the previous reports, this bot will find HTML, PHP, JS and HTM files in the system and will inject a javascript code in every file. This is an attempt for that attacker to not only compromise the server but also clients connecting to it. Using a DGA domain to host the javascript makes it more resilient against defenses.</p>
