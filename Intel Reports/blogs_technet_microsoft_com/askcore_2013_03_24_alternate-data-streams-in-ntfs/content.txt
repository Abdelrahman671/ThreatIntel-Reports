										</nav><!-- <content> --><p>This blog has been a long time coming. There is a bit of confusion about the subject of <em><strong>alternate data streams</strong></em> (ADS) and no small amount of suspicion. So I want to take a few minutes to set the record straight on ADS.</p>
<p>A couple years ago I wrote a blog on NTFS attributes.</p>
<p><a href="../askcore/ntfs-file-attributes" data-linktype="relative-path">https://blogs.technet.com/b/askcore/archive/2010/08/25/ntfs-file-attributes.aspx</a></p>
<p>You might want to review that blog before continuing. Iâll waitâ¦.</p>
<p>Welcome back.</p>
<p>One of the common questions I get is, âRobert. What is an alternate data stream?â</p>
<p>My reply is always the same, âIt is a data stream that is alternateâ.</p>
<p>I donât mean to be smart aleck about itâ¦but thatâs what it is. We know from my older blog that a file is divided up into âattributesâ and one of these attributes is $DATA or simply called the data attribute. It is the part of the file we put data into. So if I have a text file that says, âThis is my textâ, then if I look at the data attribute, it will contain a stream of data that reads, âThis is my textâ. However, this is the normal data stream, sometimes called the primary data stream, but more accurately it is called the <em><strong>unnamed data stream</strong></em>. Why? Because it is a data stream that has no name. In the jolly land of programming it is referred to as $DATA:ââ</p>
<p><a href="https://msdntnarchive.z22.web.core.windows.net/media/TNBlogsFS/prod.evol.blogs.technet.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/59/75/metablogapi/3482.image_106A3119.png" data-linktype="external"><img src="https://msdntnarchive.z22.web.core.windows.net/media/TNBlogsFS/prod.evol.blogs.technet.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/59/75/metablogapi/4578.image_thumb_6FE2F166.png" alt="image" title="image" data-linktype="external"></a></p>
<p>The name of the stream will appear between the quotes. Since this is an unnamed data stream, there isnât anything there.</p>
<p>Now that we know what the unnamed data stream looks like, we can start thinking in terms of alternates. Knowing that the place we normally store data is the unnamed data stream, if a stream has a name, it is alternate. So if I had a file with an ADS named SecondStream, its full name would be, $DATA:âSecondStreamâ</p>
<p><a href="https://msdntnarchive.z22.web.core.windows.net/media/TNBlogsFS/prod.evol.blogs.technet.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/59/75/metablogapi/7536.image_4F5BB1B4.png" data-linktype="external"><img src="https://msdntnarchive.z22.web.core.windows.net/media/TNBlogsFS/prod.evol.blogs.technet.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/59/75/metablogapi/6560.image_thumb_00E71F4A.png" alt="image" title="image" data-linktype="external"></a></p>
<p>This is all good and fine, but unlike the unnamed data stream, we canât see the ADS. Or can we? The answer is, yes we can. But you have to use a method different than just opening the file in NotePad.</p>
<p>There are a number of tools out there that will allow you to view and manipulate ADS. One that Microsoft has provided for years is called STREAMS.EXE.</p>
<p><a href="https://technet.microsoft.com/en-us/sysinternals/bb897440.aspx" data-linktype="external">https://technet.microsoft.com/en-us/sysinternals/bb897440.aspx</a></p>
<p>STREAMS.EXE will display any ADS the file has.</p>
<p><a href="https://msdntnarchive.z22.web.core.windows.net/media/TNBlogsFS/prod.evol.blogs.technet.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/59/75/metablogapi/4527.clip_image005_6E322592.png" data-linktype="external"><img src="https://msdntnarchive.z22.web.core.windows.net/media/TNBlogsFS/prod.evol.blogs.technet.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/59/75/metablogapi/3365.clip_image005_thumb_58D4702A.png" alt="clip_image005" title="clip_image005" data-linktype="external"></a></p>
<p>The formatting is a little different.</p>
<p><a href="https://msdntnarchive.z22.web.core.windows.net/media/TNBlogsFS/prod.evol.blogs.technet.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/59/75/metablogapi/1738.image_2D23A62E.png" data-linktype="external"><img src="https://msdntnarchive.z22.web.core.windows.net/media/TNBlogsFS/prod.evol.blogs.technet.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/59/75/metablogapi/1643.image_thumb_5EAF13C3.png" alt="image" title="image" data-linktype="external"></a></p>
<p>STREAMS.EXE is fine, and Iâve used it for years, but with the release of Win8/Server 2012, Iâve discovered a new way of dealing with ADSâ¦.Windows PowerShell. Using the cmdlet, Get-Item, I can get more information than I did with STREAM.EXE.</p>
<p><a href="https://msdntnarchive.z22.web.core.windows.net/media/TNBlogsFS/prod.evol.blogs.technet.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/59/75/metablogapi/4863.clip_image008_1E0CC754.png" data-linktype="external"><img src="https://msdntnarchive.z22.web.core.windows.net/media/TNBlogsFS/prod.evol.blogs.technet.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/59/75/metablogapi/6567.clip_image008_thumb_4B8DE717.png" alt="clip_image008" title="clip_image008" data-linktype="external"></a></p>
<p>The output shows not only the name of the ADS and its size, but also the unnamed data stream and its size is also listed (shown as :$DATA). And now that I know the name of the ADS, I can use the Get-Content cmdlet to query its contents.</p>
<p><a href="https://msdntnarchive.z22.web.core.windows.net/media/TNBlogsFS/prod.evol.blogs.technet.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/59/75/metablogapi/5383.clip_image009_0AEB9AA8.png" data-linktype="external"><img src="https://msdntnarchive.z22.web.core.windows.net/media/TNBlogsFS/prod.evol.blogs.technet.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/59/75/metablogapi/6318.clip_image009_thumb_432A11C0.png" alt="clip_image009" title="clip_image009" data-linktype="external"></a></p>
<p>STREAM.EXE canât display whatâs actually in an ADS. Hereâs another trick that STREAM.EXE canât doâ¦.create data streams. Using Set-Content, Iâll create a second ADS in the same file and add a line of text.</p>
<p><a href="https://msdntnarchive.z22.web.core.windows.net/media/TNBlogsFS/prod.evol.blogs.technet.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/59/75/metablogapi/1538.clip_image010_0287C551.png" data-linktype="external"><img src="https://msdntnarchive.z22.web.core.windows.net/media/TNBlogsFS/prod.evol.blogs.technet.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/59/75/metablogapi/2234.clip_image010_thumb_3008E514.png" alt="clip_image010" title="clip_image010" data-linktype="external"></a></p>
<p>And again, we can query for the streams using Get-Item.</p>
<p><a href="https://msdntnarchive.z22.web.core.windows.net/media/TNBlogsFS/prod.evol.blogs.technet.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/59/75/metablogapi/1050.clip_image011_417945EC.png" data-linktype="external"><img src="https://msdntnarchive.z22.web.core.windows.net/media/TNBlogsFS/prod.evol.blogs.technet.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/59/75/metablogapi/2654.clip_image011_thumb_410D12F7.png" alt="clip_image011" title="clip_image011" data-linktype="external"></a></p>
<p>And finally, we can remove an ADS using the Remove-Item cmdlet.</p>
<p><a href="https://msdntnarchive.z22.web.core.windows.net/media/TNBlogsFS/prod.evol.blogs.technet.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/59/75/metablogapi/4336.clip_image012_527D73CF.png" data-linktype="external"><img src="https://msdntnarchive.z22.web.core.windows.net/media/TNBlogsFS/prod.evol.blogs.technet.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/59/75/metablogapi/6708.clip_image012_thumb_2DEBE64B.png" alt="clip_image012" title="clip_image012" data-linktype="external"></a></p>
<p>Now we know what ADS is, how to query for ADS, how to create ADS, and how to delete ADS. So what is the big deal?</p>
<p>The big deal is that since ADS isnât easily visible, it has become a cute way to hide data. Unfortunately it has also been used in the past to hide malicious code. This is how ADS got a bad name. In fact, a number of people that approach me about ADS already know that they have files with alternate data streams and they think they are infected with viruses.</p>
<p>Calm down. The mere presence of an ADS doesnât mean that there is a problem. In fact, Microsoft uses ADS for a number of functions. I can almost guarantee that if you are reading this, you probably have some ADS on your computer. Letâs take a look at a couple examples.</p>
<p><strong>Internet Explorer:</strong> Ever download an executable file from the Internet and then get warned about it when you ran it? How does that work?</p>
<p>When the file is downloaded, IE slaps an ADS on it. The stream will store a tag that tells Windows what zone the file was downloaded from.</p>
<p><a href="https://msdntnarchive.z22.web.core.windows.net/media/TNBlogsFS/prod.evol.blogs.technet.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/59/75/metablogapi/6811.clip_image013_3F5C4723.png" data-linktype="external"><img src="https://msdntnarchive.z22.web.core.windows.net/media/TNBlogsFS/prod.evol.blogs.technet.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/59/75/metablogapi/4111.clip_image013_thumb_3E83E139.png" alt="clip_image013" title="clip_image013" data-linktype="external"></a></p>
<p><em>Look Familiar?</em></p>
<p>So using what Iâve learned so far, I can look at one of the files Iâve downloaded from the internet and see if there is an ADS on it.</p>
<p><a href="https://msdntnarchive.z22.web.core.windows.net/media/TNBlogsFS/prod.evol.blogs.technet.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/59/75/metablogapi/7331.clip_image014_7DE194C9.png" data-linktype="external"><img src="https://msdntnarchive.z22.web.core.windows.net/media/TNBlogsFS/prod.evol.blogs.technet.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/59/75/metablogapi/4532.clip_image014_thumb_7D7561D4.png" alt="clip_image014" title="clip_image014" data-linktype="external"></a></p>
<p>Yes, it is called âZone.Identifierâ. And then we can query the contents of the âZone.Indentifierâ ADS.</p>
<p><a href="https://msdntnarchive.z22.web.core.windows.net/media/TNBlogsFS/prod.evol.blogs.technet.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/59/75/metablogapi/4150.clip_image016_51C497D8.png" data-linktype="external"><img src="https://msdntnarchive.z22.web.core.windows.net/media/TNBlogsFS/prod.evol.blogs.technet.com/CommunityServer.Blogs.Components.WeblogFiles/00/00/00/59/75/metablogapi/6523.clip_image016_thumb_4A39286B.png" alt="clip_image016" title="clip_image016" data-linktype="external"></a></p>
<p>Now we know that the file was downloaded from zone 3. Using the zone chart we can see it came from the Internet zone.</p>
<blockquote>
<p>Value Setting<br>
------------------------------<br>
0 My Computer<br>
1 Local Intranet Zone<br>
2 Trusted sites Zone<br>
3 Internet Zone<br>
4 Restricted Sites Zone</p>
</blockquote>
<p>Notice that my test download file is in a test directory. This means I moved the file here from my download directory. This is the cool thing about ADS, since it is part of the file, it moves with the file. Even if I copied it, the ADS would be on the new copy as well.</p>
<p>Other Internet browsers use ADS in a similar fashion.</p>
<p><strong>File Classification Infrastructure</strong>: FCI is very dependent on ADS. The way that the classification works is that it puts tags on your files that allows you to keep track of what the file was classified as, no matter what happens with the file. It could be edited, copied, moved to another server, and its classification tags remain intact.</p>
<p><strong>Others</strong>: Office files and Outlook Express file use ADS. And it isnât limited to Microsoft programs. Numerous programs utilize the ADS functionality.</p>
<p>The point is that if you discover ADS on your system, it isnât necessarily a bad thing. And just blindly stripping these data streams out of files can actually do a great deal of harm.</p>
<p>And now that you have some tools to use for querying alternate data streams, they wonât be so scary.</p>
