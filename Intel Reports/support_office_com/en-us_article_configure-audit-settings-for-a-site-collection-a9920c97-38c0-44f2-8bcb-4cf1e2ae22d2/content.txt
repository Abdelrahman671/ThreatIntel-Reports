	<article class="ocpArticleContent"><section class="ocpIntroduction">
        <p>The audit feature for Microsoft SharePoint and SharePoint Server lets you track user activity on content types like lists and libraries within your site collection. Knowing which users have accessed specific content at any given time is critical for many business requirements, such as regulatory compliance and records management.</p>
        <p></p>
        <div class="ocpAlert">
    <p class="ocpAlertSection"><b class="ocpNote">Notes: </b></p>
    <ul class="ocpAlertSectionListContainer">
      <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>Only site-collection owners can run audit log reports.</p>
            <meta itemprop="position" content="1">
      </li>
      <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>If the Audit log Reports link is not available under Site Settings, make sure you have the Site Collection Feature “Reporting” activated.</p>
            <meta itemprop="position" content="2">
      </li>
      <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>Pages (such as .aspx) aren't considered documents so they can't be audited in SharePoint Servers 2019 or 2016.</p>
            <meta itemprop="position" content="3">
      </li>
      <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>SharePoint SharePoint audit log reports function differently than classic audit log reports.</p>
            <meta itemprop="position" content="4">
      </li>
      <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
              <p>SharePoint administrators can directly download data from the <a href="https://docs.microsoft.com/en-us/office/office-365-management-api/office-365-management-activity-api-reference" target="_blank" class="ocpExternalLink" data-bi-type="anchor">Microsoft 365 Management Activity API.</a></p>
            <meta itemprop="position" content="5">
      </li>
    </ul>

                  <section aria-labelledby="ID0EFBBBDBBH" class="ocpSection">
                    <h3 id="ID0EFBBBDBBH" class="">Changes to SharePoint audit settings</h3>
                    
                      <p>
                        Microsoft 365 Unified Audit Logging now supports SharePoint and OneDrive. Unified auditing provides access to event logs (like view, create, edit, upload, download, and delete) and sharing actions like invitation and access requests, and synchronization activity. For more information about what Unified Audit Logging in Microsoft 365 provides for SharePoint, see <a href="https://docs.microsoft.com/en-us/office365/securitycompliance/search-the-audit-log-in-security-and-compliance#audited-activities" target="_blank" class="ocpExternalLink" data-bi-type="anchor">Audited activities</a>.</p>
                    
                    
                      <section aria-labelledby="ID0EDFBBBBDBBH" class="ocpSection">
                        <h4 id="ID0EDFBBBBDBBH" class="">Summary of Audit Log Report changes:</h4>
                        
                          <ul>
              <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                              <p>Trimming audit log reports is no longer a feature but you can still access files that have been trimmed in the past in the document library that you have set for this purpose. </p>
                            <meta itemprop="position" content="1">
              </li>
              <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                              <p>Audit Log Reports will no longer allow for specific events to be edited. Audit Log Reports will still be available but will be limited to <a href="https://docs.microsoft.com/en-us/microsoft-365/compliance/search-the-audit-log-in-security-and-compliance" target="_blank" class="ocpExternalLink" data-bi-type="anchor">specific types of events</a>.</p>
                            <meta itemprop="position" content="2">
              </li>
              <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                              <p>For Customize Reports, the location option has been hidden because reports scoped at the sub-site level are no longer supported. </p>
                            <meta itemprop="position" content="3">
              </li>
            </ul>
                        
                      </section>

                          <ol type="1" itemscope="" itemtype="http://schema.org/ItemList">
              <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                              <p>Select <b class="ocpLegacyBold">Settings</b> &gt; <b class="ocpLegacyBold">Site settings</b>. If you don't see <b class="ocpUI">Site settings</b>, select <b class="ocpUI">Site information</b> and then select <b class="ocpUI">View all site settings</b>. Some pages might require you to select <b class="ocpUI">Site contents</b>, and then <b class="ocpUI">Site settings</b>.</p>
                            <meta itemprop="position" content="1">
              </li>
              <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                              <p>If you are not at the root of your site collection, under <b class="ocpLegacyBold">Site Collection Administration</b>, select <b class="ocpLegacyBold">Go to top level site settings</b>.</p>
                              <div class="ocpAlert">
                  <p class="ocpAlertSection"><b class="ocpNote">Notes: </b></p>
                  <ul class="ocpAlertSectionListContainer">
                    <li>
                      <p>The Site Collection Administration section won't be available if you don't have the necessary permissions. You need to be a member of the default Site Collections Administrators group or a farm administrator.</p>
                    </li>
                    <li>
                      <p>If the Audit log Reports link is not available under Site Settings, make sure you have the Site Collection Feature “Reporting” activated.</p>
                    </li>
                  </ul>
                </div>
                            <meta itemprop="position" content="2">
              </li>
              <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                              <p>On the Site Settings page, under <b class="ocpLegacyBold">Site Collection Administration</b>, select <b class="ocpLegacyBold">Audit log reports</b>.</p>

You can also specify the maximum number of days that items will be retained. By default all items are removed at the end of the month.</p>

          <p class="ocpAlertSection"><b class="ocpNote">Note: </b>When multiple users are co-editing a document, auditing events from multiple authors or editors can be difficult to interpret. If this is a concern, consider limiting editing permission to a minimum number of users.</p>

                      <ol type="1" itemscope="" itemtype="http://schema.org/ItemList">
                <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                          <p>Select <b class="ocpLegacyBold">Settings</b> &gt; <b class="ocpLegacyBold">Site settings</b>. If you don't see <b class="ocpUI">Site settings</b>, select <b class="ocpUI">Site information</b> and then select <b class="ocpUI">View all site settings</b>. Some pages might require you to select <b class="ocpUI">Site contents</b>, and then <b class="ocpUI">Site settings</b>.</p>
                        <meta itemprop="position" content="1">
                </li>
                <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                          <p>If you are not at the root of your site collection, under <b class="ocpLegacyBold">Site Collection Administration</b>, select <b class="ocpLegacyBold">Go to top level site settings</b>.</p>
                          <div class="ocpAlert">
                    <p class="ocpAlertSection"><b class="ocpNote">Note: </b>The Site Collection Administration section won't be available if you don't have the necessary permissions. You need to be a member of the default Site Collections Administrators group or a farm administrator.</p>
                  </div>
                        <meta itemprop="position" content="2">
                </li>
                <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                          <p>On the Site Settings page, under <b class="ocpLegacyBold">Site Collection Administration</b>, select <b class="ocpLegacyBold">Site collection audit settings</b>.</p>
                          <picture><img class="fluentImage" src="https://support.content.office.net/en-us/media/ae4c9638-089b-4674-b6a6-afc328802085.png" loading="lazy" alt="Site collection audit settings selected in Site Settings dialog."></picture>
                        <meta itemprop="position" content="3">
                </li>
                <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                          <p>On the Configure Audit Settings page, in the <b class="ocpLegacyBold">Documents and Items</b> and <b class="ocpLegacyBold">List, Libraries, and Site sections</b>, select the events you want to audit, and then select <b class="ocpLegacyBold">OK</b>.</p>
                        <meta itemprop="position" content="4">
                </li>
              </ol>
                      <p>Which events you audit depends on your auditing needs. For example, regulatory compliance usually has specific requirements that will dictate which events you need to audit. We recommend that you only audit the events required to meet your needs. Additional, unnecessary auditing can affect the performance and other aspects of your site collection(s).</p>
                      <div class="ocpAlert">
                <p class="ocpAlertSection"><b class="ocpNote">Note: </b>For SharePoint Server 2013, we recommend that you only select the <b class="ocpLegacyBold">Opening </b>or<b class="ocpLegacyBold"> downloading documents, viewing items in lists, or viewing item properties</b> event when absolutely necessary. This option is likely to generate a large number of events that will potentially degrade the performance and other aspects of your site collection(s).</p>
              </div>
                    </div>
          </div>
                  </div>
                  <div class="ocpSection" id="__bkmkshowsup" data-bi-id="__bkmkshowsup">
                    <h3 class="ocpExpandoHead"><button ms.interactiontype="11" aria-expanded="false" class="expandoButtonLink" type="button" id="ID0EFBBBBBH-button" aria-controls="ID0EFBBBBBH-panel"><div class="ocpExpandoHeadChevronContainer" aria-hidden="true"><i class="ms-Icon--expandoCollapsed ocpExpander" role="none"></i></div>
              <div class="ocpExpandoHeadTitleContainer" role="none">What shows up in the audit log report ?</div></button></h3>
                    <div class="ocpebw" id="ID0EFBBBBBH-panel" aria-labelledby="ID0EFBBBBBH-button" role="region">
            <div class="ocpExpandoBody">
                      <p>Audited events are captured in audit reports that are based on Microsoft Excel 2013 and are available from the Auditing Reports page. You can also create a custom report that includes a number of these events over a specified date range, within a specific area of the site collection, or filtered to an individual user. You cannot modify events once they are logged, but site collection administrators can delete items from the audit log and configure automatic trimming of the audit log data.</p>
                      <p>The audit log captures the following information for the events that are selected to be audited:</p>
                      <ul>
                <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                          <p>Site from which an event originated</p>
                        <meta itemprop="position" content="1">
                </li>
                <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                          <p>Item ID, type, name, and location</p>
                        <meta itemprop="position" content="2">
                </li>
                <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                          <p>User ID associated with the event</p>
                        <meta itemprop="position" content="3">
                </li>
                <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                          <p>Event type, date, time, and source</p>
                        <meta itemprop="position" content="4">
                </li>
                <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                          <p>Action taken on the item</p>
                        <meta itemprop="position" content="5">
                </li>
              </ul>
                      <p>Following is an example of the data in a Deletion audit log report. With this report, you can determine who deleted and restored data across the site collection. You can use the features of Excel to filter, sort, and analyze the data.</p>
                      <p>
                         <picture><img class="fluentImage" src="https://support.content.office.net/en-us/media/d5c99bd0-cb5c-469d-b04e-207ca4044e8b.jpg" loading="lazy" alt="Delete Audit Report Report Data"></picture>
                      </p>
                    </div>
          </div>
                  </div>
                  <div class="ocpSection" id="__bkmktrimaudit" data-bi-id="__bkmktrimaudit">
                    <h3 class="ocpExpandoHead"><button ms.interactiontype="11" aria-expanded="false" class="expandoButtonLink" type="button" id="ID0EDBBBBBH-button" aria-controls="ID0EDBBBBBH-panel"><div class="ocpExpandoHeadChevronContainer" aria-hidden="true"><i class="ms-Icon--expandoCollapsed ocpExpander" role="none"></i></div>
              <div class="ocpExpandoHeadTitleContainer" role="none">Trimming the audit log reports</div></button></h3>
                    <div class="ocpebw" id="ID0EDBBBBBH-panel" aria-labelledby="ID0EDBBBBBH-button" role="region">
            <div class="ocpExpandoBody">
                      <p>When you select an event to audit for a site collection, the report will audit every item in the site collection each time the event occurs. Auditing can potentially generate a large number of audit events, creating a large audit log that could fill the hard drive, impacting performance and other aspects of a site collection.</p>
                      <div class="ocpAlert">
                <p class="ocpAlertSection"><b class="ocpImportant">Important: </b>To prevent the audit log from filling the hard drive and potentially degrading the performance of the site collection, we recommended that you enable audit log trimming for site collections with extensive auditing.</p>
              </div>
                      <p>To manage the size of the audit log report you can configure it to automatically trim, and optionally archive, the current audit log data in a document library before the data is trimmed. The schedule for audit log trimming is configured by your server administrator in Central Administration. The default is the end of the month.</p>
                    </div>
          </div>
                  </div>
                  <div class="ocpSection" id="__bkmkviewaudit" data-bi-id="__bkmkviewaudit">
                    <h3 class="ocpExpandoHead"><button ms.interactiontype="11" aria-expanded="false" class="expandoButtonLink" type="button" id="ID0EBBBBBBH-button" aria-controls="ID0EBBBBBBH-panel"><div class="ocpExpandoHeadChevronContainer" aria-hidden="true"><i class="ms-Icon--expandoCollapsed ocpExpander" role="none"></i></div>
              <div class="ocpExpandoHeadTitleContainer" role="none">View audit log reports</div></button></h3>
                    <div class="ocpebw" id="ID0EBBBBBBH-panel" aria-labelledby="ID0EBBBBBBH-button" role="region">
            <div class="ocpExpandoBody">
                      <p>To view an audit log report:</p>
                      <ol type="1" itemscope="" itemtype="http://schema.org/ItemList">
                <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                          <p>Click <b class="ocpLegacyBold">Settings</b> <picture><img class="fluentImage" src="https://support.content.office.net/en-us/media/4b83e9cb-c7e4-46c8-b3d1-cfee017123ae.png" loading="lazy" alt="Settings: update your profile, install software and connect it to the cloud"></picture>, and then click <b class="ocpLegacyBold">Site settings</b>. If you don't see <b class="ocpUI">Site settings</b>, click <b class="ocpUI">Site information</b> and then click <b class="ocpUI">View all site settings</b>. Some pages might require you to select <b class="ocpUI">Site contents</b>, and then <b class="ocpUI">Site settings</b>.</p>
                          <div class="ocpAlert">
                    <p class="ocpAlertSection"><b class="ocpNote">Note: </b>The Site Collection Administration section won't be available if you don't have the necessary permissions. You need to be a member of the default Site Collections Administrators group or a farm administrator.</p>
                  </div>
                        <meta itemprop="position" content="1">
                </li>
                <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                          <p>Click <b class="ocpLegacyBold">Audit log reports</b> in the <b class="ocpLegacyBold">Site Collection Administration</b> section.</p>
                        <meta itemprop="position" content="2">
                </li>
                <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                          <p>Select the report that you want on the <b class="ocpLegacyBold">View Auditing Reports page</b>.</p>
                        <meta itemprop="position" content="3">
                </li>
                <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                          <p>Type a URL or <b class="ocpLegacyBold">Browse</b> to the library where you want to save the report, and then select <b class="ocpLegacyBold">OK</b>.</p>
                        <meta itemprop="position" content="4">
                </li>
                <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                          <p>On the Operation Completed Successfully page, select <b class="ocpLegacyBold">click here to view this report</b>. If you get an error, it may be because audit logs weren't enabled or there was no data to show.</p>
                          <div class="ocpAlert">
                    <p class="ocpAlertSection"><b class="ocpNote">Notes: </b></p>
                    <ul class="ocpAlertSectionListContainer">
                      <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                                <p>At a minimum, Excel version 2013 must be installed to view audit log reports by clicking <b class="ocpLegacyBold">Click here to view this report</b>.</p>
                              <meta itemprop="position" content="1">
                      </li>
                      <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                                <p>Alternatively, if opening documents in the browser is enabled for the library, go to the library where you saved the audit log report, point to the audit log report, select the down arrow, and then select <b class="ocpLegacyBold">View in Browser</b>.</p>
                              <meta itemprop="position" content="2">
                      </li>
                    </ul>
                  </div>
                        <meta itemprop="position" content="5">
                </li>
              </ol>
                      <p>You can use standard Excel features to narrow the reports to the information you want. Some ways in which you can analyze and view the log data include:</p>
                      <ul>
                <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                          <p>Filtering the audit log report for a specific site.</p>
                        <meta itemprop="position" content="1">
                </li>
                <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                          <p>Filtering the audit log report for a particular date range.</p>
                        <meta itemprop="position" content="2">
                </li>
                <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                          <p>Sorting the audit log report.</p>
                        <meta itemprop="position" content="3">
                </li>
                <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                          <p>Determining who has updated content.</p>
                        <meta itemprop="position" content="4">
                </li>
                <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                          <p>Determining which content has been deleted but not restored.</p>
                        <meta itemprop="position" content="5">
                </li>
                <li itemprop="itemListElement" itemscope="" itemtype="http://schema.org/ListItem">
                          <p>Viewing the changes to permissions on an item.</p>
                        <meta itemprop="position" content="6">
                </li>
              </ul>
                      <p></p>
                    </div>
