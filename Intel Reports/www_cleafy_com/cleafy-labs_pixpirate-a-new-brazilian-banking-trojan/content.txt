Cyber attacks continued to surge in the latter half of 2021 and 2022, not only in quantity but also in impact. Financial groups are expanding their reach, targeting organizations worldwide mainly through <strong>ransomware</strong> but also increasing the range of their activities.</p><p>â<br/>That said, on top of this evolution, one of the most crucial elements which have been disrupting the current state-of-art of anti-fraud departments is Instant Payments. <strong>Instant Payments</strong> areelectronic money transfers that make transferred funds available in real-time from one account to another, bringing speed to money transfers and increasing the underlined risk of unhandled frauds and, consequently, monetary losses. In recent years, the adoption of Instant Payments has been rapidly growing in Europe, America, and, more recently, also in Brazil, with the introduction of Pix, an instant payment platform created and managed by the monetary authority of Brazil, the Central Bank of Brazil (BCB), which enables the quick execution of payments and transfers and now counting over <a href="https://www.spglobal.com/marketintelligence/en/news-insights/latest-news-headlines/pix-breaks-ground-in-brazil-shakes-up-payments-market-68147884">100 million registered accounts</a>.</p><p>â<br/>One such threat recently discovered in the wild is a <strong>brand-new mobile malware</strong> targeting LATAM countries, specifically Brazil. The primary goal of this malware is to steal sensitive information and perform fraud on users that regularly use Pix platform. This report will provide a detailed analysis of this malware in the following chapters.</p><p>[1]<a href="https://www.interpol.int/en/News-and-Events/News/2022/INTERPOL-Working-Group-highlights-cyber-threats-across-the-Americas"> https://www.interpol.int/en/News-and-Events/News/2022/INTERPOL-Working-Group-highlights-cyber-threats-across-the-Americas Â </a></p></div></section><section id="3" class="bigmarginbottom"><h3 class="titlesectionclab">Technical Analysis: Overview</h3><div class="blog w-richtext"><p>PixPirate hides its malicious purposes with familiar names and icons, posing as a legitimate application to the victims. At the end of 2022, we intercepted the following decoys, which appear to be pretty consolidated by TAs for delivering their malicious samples:</p><figure style="max-width:1900pxpx" class="w-richtext-align-fullwidth w-richtext-figure-type-image"><div><img src="https://cdn.prod.website-files.com/60201cc2b6249b0358f70f8a/63dcdf79dce28a1f64e74391_1.png" loading="lazy" alt="Figure 1 - Main names/icons used by PixPirate"/></div><figcaption>Figure 1 - Main names/icons used by PixPirate</figcaption></figure><p>PixPirate is usually delivered using a dropper application, used to download (or in some cases just to unpack) and install the banking trojan. During its installation, PixPirate immediately tries to enable Accessibility Services that keep being requested persistently with fake pop-ups until the victim accepts. </p><figure style="max-width:1876pxpx" class="w-richtext-align-fullwidth w-richtext-figure-type-image"><div><img src="https://cdn.prod.website-files.com/60201cc2b6249b0358f70f8a/63dcdfa4ba28139f695fd300_2.png" loading="lazy" alt="Figure 2 - Permissions used by the dropper application"/></div><figcaption>Figure 2 - Permissions used by the dropper application</figcaption></figure><figure style="max-width:1574pxpx" class="w-richtext-align-fullwidth w-richtext-figure-type-image"><div><img src="https://cdn.prod.website-files.com/60201cc2b6249b0358f70f8a/63dcdfcc800642aaa1026744_3.png" loading="lazy" alt="Figure 3 - Installation phases of PixPirate "/></div><figcaption>Figure 3 - Installation phases of PixPirate</figcaption></figure><p>Abusing the Accessibility Services is a standard routine for banking trojans since they provide features to interact with other apps. After the victim gives the permissions, PixPirate will enable all its malicious functionalities.</p><p>After inspecting PixPirate code, we identified a few references related to a framework called <strong>Auto.js</strong>[<a href="https://pro.autojs.org/">2</a>]. This is an open-source tool for automating tasks on Android devices using JavaScript. It allows developers to write scripts that can interact with the device&#x27;s UI and perform actions like finding and interacting with UI elements, entering text, scrolling through lists, simulating touch events, etc. Auto.js also provides a built-in JavaScript interpreter, which allows scripts to run on the device itself without the need for an external runtime. The following list is an example of the built-in functions available on Auto.js:</p><ul role="list"><li><strong>sleep()</strong>: pause the execution</li><li><strong>toast():</strong> show a toast notification</li><li><strong>swipe():</strong> execute a swipe gesture</li><li><strong>click():</strong> clicks on the screen</li><li><strong>device.width:</strong> get the device screen width</li><li><strong>device. height:</strong> get the device screen height</li></ul><figure style="max-width:1406pxpx" class="w-richtext-align-fullwidth w-richtext-figure-type-image"><div><img src="https://cdn.prod.website-files.com/60201cc2b6249b0358f70f8a/63dce0d6be397f74892a7533_4.png" loading="lazy" alt="Figure 4 - Auto.js home page "/></div><figcaption>Figure 4 - Auto.js home page</figcaption></figure><p>Since Auto.js represents a new framework for mobile banking trojan, we wanted to understand the reason behind this choice. By inspecting the framework capabilities, it was possible to identify some features that could speed up the development phase:</p><ul role="list"><li><strong>Non-Android coders</strong> can write automation scripts using JavaScript code and create standalone applications.</li><li><strong>Web communication management</strong> within the application.</li><li>Built-in<strong> mechanism to encrypt/obfuscate the code</strong>.</li></ul><p>TAs were able to adapt this legitimate framework and building-up the entire malicious routines in JavaScript, executed on top of the Auto.js stack. For slowing down analysts, TAs adopt a heavy layer of code obfuscation, including multiple techniques such as string array encoding, control flow flattering, etc.</p><p>The following features have been observed:</p><ul role="list"><li>Preventing uninstall</li><li>Disabling Google Play Protect </li><li>Intercepting SMS messages</li><li>Intercepting banking credentials</li><li>Monitoring victimâs financial activities</li><li>Malvertising via push notifications</li><li>Perform ATS attacks via PIX payments</li></ul><figure style="max-width:1872pxpx" class="w-richtext-align-fullwidth w-richtext-figure-type-image"><div><img src="https://cdn.prod.website-files.com/60201cc2b6249b0358f70f8a/63dce16d31782fcc56c2687a_5.png" loading="lazy" alt="Figure 5 - Overview of the malicious JavaScript routines"/></div><figcaption>Figure 5 - Overview of the malicious JavaScript routines</figcaption></figure><p>The following chapters explore and discuss all PixPirateâs main features.</p><p>[2] <a href="https://pro.autojs.org/">https://pro.autojs.org/</a></p></div></section><section id="4" class="bigmarginbottom"><h3 class="titlesectionclab">Password Stealer</h3><div class="blog w-richtext"><p>One of the JavaScript modules of PixPirate is used to steal the banking password with the help of the well-known accessibility services of Android. Inside this module, a specific function was created for each targeted bank, since every banking application has a different layout.</p><p>In fact, through the Accessibility Services, PixPirate can recognize the different UI elements of the bankâs activity [3] and the password element displayed on the screen. If it detects some changes in the password input text, it grabs the password of the user (if it hasn&#x27;t already been stolen previously).</p><figure style="max-width:1862pxpx" class="w-richtext-align-fullwidth w-richtext-figure-type-image"><div><img src="https://cdn.prod.website-files.com/60201cc2b6249b0358f70f8a/63dce1d2a821e7857667461b_6.png" loading="lazy" alt="Figure 6 - Example of a portion of code used to steal the password of the targeted bank "/></div><figcaption>Figure 6 - Example of a portion of code used to steal the password of the targeted bank</figcaption></figure><p>[3] An Activity in Android represents a single screen with a user interface and is used to display and interact with content.</p></div></section><section id="5" class="bigmarginbottom"><h3 class="titlesectionclab">ATS landed on Pix System</h3><div class="blog w-richtext"><p>The Banco Central do Brasil (BCB) created Pix, an instant payment method that enables users like people, companies, and governmental entities, to send or receive payment transfers in a few seconds at any time, including non-business days. Furthermore, Pix transactions can be performed between any payment institutions or financial institutions that comply with this ecosystem.</p><p>In recent years, different Android banking trojans have been created to perform fraudulent transactions on this ecosystem, like PixStealer or the most recent BrasDex.</p><p>PixPirateâs TAs created a specific javascript file for each targeted bank to manage the different phases of the fraudulent transaction, which can be summarized with the following steps:</p><ul role="list"><li><strong>The identification of the UI elements of the targeted app</strong>, like buttons, texts, or inputs, to establish which activity is displayed on the victim device.</li></ul><figure style="max-width:1878pxpx" class="w-richtext-align-fullwidth w-richtext-figure-type-image"><div><img src="https://cdn.prod.website-files.com/60201cc2b6249b0358f70f8a/63dce2b70cf4564682b024ff_7.png" loading="lazy" alt="Figure 7 - An example of code used to identify the elements of the targeted bank"/></div><figcaption>Figure 7 - An example of code used to identify the elements of the targeted bank</figcaption></figure><ul role="list"><li>The <strong>balance discovery</strong> of the account, used to set up the amount of money to steal (balance available * 0.95) or discard accounts considered invalid. Followed by the single steps to carry out the transaction.</li></ul><figure style="max-width:1890pxpx" class="w-richtext-align-fullwidth w-richtext-figure-type-image"><div><img src="https://cdn.prod.website-files.com/60201cc2b6249b0358f70f8a/63dce307be397f398f2c8070_8.png" loading="lazy" alt="Figure 8 - A portion of code used to identify the balance of the targeted bank "/></div><figcaption>Figure 8 - A portion of code used to identify the balance of the targeted bank</figcaption></figure></div></section><section id="6" class="bigmarginbottom"><h3 class="titlesectionclab">SMS Hijack</h3><div class="blog w-richtext"><p>PixPirate also contains a script designed to<strong> delete SMS messages</strong> with specific text content. To perform this action, the malware can observe and detect when the default SMS app is in the foreground and perform actions such as long clicking, clicking the delete button, and confirming the deletion. To hide these activities from the user, PixPirate can display a loading window and mutes the device during the deletion of the messages.</p><p>This kind of feature is also present in other Android banking trojans and has the dual function of hiding suspicious actions carried out by the malware and removing specific details from analysts carrying out IR operations.</p><figure style="max-width:1864pxpx" class="w-richtext-align-fullwidth w-richtext-figure-type-image"><div><img src="https://cdn.prod.website-files.com/60201cc2b6249b0358f70f8a/63dce347bef5986a956b6ea5_9.png" loading="lazy" alt="Figure 9 - A portion of code used in the sms hijack method"/></div><figcaption>Figure 9 - A portion of code used in the sms hijack method</figcaption></figure></div></section><section id="7" class="bigmarginbottom"><h3 class="titlesectionclab">Code Protection</h3><div class="blog w-richtext"><p>Among the main countermeasures adopted by PixPirate to slow down the analysis are code obfuscation and encryption, other than classic functionalities Â that try to avoid application removal at runtime. Â Speaking about obfuscation, it has been implemented wisely, making the code quite challenging to be analyzed directly. In fact, before proceeding with the analysis, it was required to remove garbage functions and rename variables other than proceeding with multiple steps of deobfuscation. At the end of this process, it was possible to have a clearer understanding of the code.</p><figure style="max-width:1864pxpx" class="w-richtext-align-fullwidth w-richtext-figure-type-image"><div><img src="https://cdn.prod.website-files.com/60201cc2b6249b0358f70f8a/63dce38a2d5b3f66c816c6f6_10.png" loading="lazy" alt="Figure 10 - Comparison of the same function obfuscated (left) and partially deobfuscated (right)"/></div><figcaption>Figure 10 - Comparison of the same function obfuscated (left) and partially deobfuscated (right)</figcaption></figure><p>Moreover, TAs have adopted an <strong>encryption</strong> routine provided by <strong>Auto.js</strong> using the Rhino engine and the Common Encryption method. Through this method, most of the strings within the code have been encrypted through a xor operation. Additionally, PixPirate has been observed to target the instant payment platform Pix, adopted by multiple Brazilian banks. </p><p>Although PixPirate seems to be still in the early stages of development because of the IOCs observed (e.g. logs sent to C2, comments in the code and more variants with very few differences), it&#x27;s not possible to exclude that in the next future, there will be even more threats that are going to follow the PixPirate example, targeting other LATAM countries or even moving their eyes towards different regions.</p><h4>Appendix 1: IOCs</h4><div class="w-embed"><style type="text/css">
