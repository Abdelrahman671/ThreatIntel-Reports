              <span id="hs_cos_wrapper_post_body" class="hs_cos_wrapper hs_cos_wrapper_meta_field hs_cos_wrapper_type_rich_text" style="" data-hs-cos-general-type="meta_field" data-hs-cos-type="rich_text"><p><em>Research by: Shilpesh Trivedi and Tejaswini Sandapolla</em></p>
<p>&nbsp;</p>
<p>The Uptycs <a href="/products/use-cases/threat-hunting" rel="noopener">Threat Research</a> Team has discovered a new infostealer. Spread by multiple bundlers and new on cybercrime forums, HookSpoofer has keylogging and clipper abilities. (A bundler combines two or more files in a single package.) It sends its stolen data to a Telegram bot.<br><br>Uptycs threat research has determined that while it includes enhanced features, HookSpoofer is coded in C# and is inspired by StormKitty, an open source program found on GitHub.</p>
<!--more-->
<p>&nbsp;</p>
<h2 style="font-size: 24px;">Infection Vector</h2>
<p>HookSpoofer is spread by bundlers (MD5: de90466d983da595e863339c34ee4b6b) named “Spotify proxyless checker 2022.rar.” It contains 17 files, with the main executable (the HookSpoofer stealer) being circulated as Spotify checker.exe.</p>
<p>&nbsp;</p>
<h2>HookSpoofer Infection Flow</h2>
<p>&nbsp;</p>
<p><img src="https://www.uptycs.com/hs-fs/hubfs/Figure1%20(1).png?width=3307&amp;height=1873&amp;name=Figure1%20(1).png" width="3307" height="1873" loading="lazy" alt="Figure1 (1)" style="height: auto; max-width: 100%; width: 3307px; margin-left: auto; margin-right: auto; display: block;" srcset="https://www.uptycs.com/hs-fs/hubfs/Figure1%20(1).png?width=1654&amp;height=937&amp;name=Figure1%20(1).png 1654w, https://www.uptycs.com/hs-fs/hubfs/Figure1%20(1).png?width=3307&amp;height=1873&amp;name=Figure1%20(1).png 3307w, https://www.uptycs.com/hs-fs/hubfs/Figure1%20(1).png?width=4961&amp;height=2810&amp;name=Figure1%20(1).png 4961w, https://www.uptycs.com/hs-fs/hubfs/Figure1%20(1).png?width=6614&amp;height=3746&amp;name=Figure1%20(1).png 6614w, https://www.uptycs.com/hs-fs/hubfs/Figure1%20(1).png?width=8268&amp;height=4683&amp;name=Figure1%20(1).png 8268w, https://www.uptycs.com/hs-fs/hubfs/Figure1%20(1).png?width=9921&amp;height=5619&amp;name=Figure1%20(1).png 9921w" sizes="(max-width: 3307px) 100vw, 3307px"></p>
<p style="text-align: center;"><span style="font-size: 14px; color: #ffffff;"><em>Figure 1 - Hookspoofer infection flow</em></span></p>
<p>&nbsp;</p>
<h2>Technical Analysis</h2>
<p>The analyzed program has an md5 hash: 7FCE055A581C0B116A9679291BF89B7D. It’s .NET compiled and packed by Confuser, a .NET obfuscator.</p>
<p>&nbsp;</p>
<p><img src="https://www.uptycs.com/hs-fs/hubfs/Figure2%20(1).png?width=1258&amp;height=498&amp;name=Figure2%20(1).png" width="1258" height="498" loading="lazy" alt="Figure2 (1)" style="height: auto; max-width: 100%; width: 1258px; margin-left: auto; margin-right: auto; display: block;" srcset="https://www.uptycs.com/hs-fs/hubfs/Figure2%20(1).png?width=629&amp;height=249&amp;name=Figure2%20(1).png 629w, https://www.uptycs.com/hs-fs/hubfs/Figure2%20(1).png?width=1258&amp;height=498&amp;name=Figure2%20(1).png 1258w, https://www.uptycs.com/hs-fs/hubfs/Figure2%20(1).png?width=1887&amp;height=747&amp;name=Figure2%20(1).png 1887w, https://www.uptycs.com/hs-fs/hubfs/Figure2%20(1).png?width=2516&amp;height=996&amp;name=Figure2%20(1).png 2516w, https://www.uptycs.com/hs-fs/hubfs/Figure2%20(1).png?width=3145&amp;height=1245&amp;name=Figure2%20(1).png 3145w, https://www.uptycs.com/hs-fs/hubfs/Figure2%20(1).png?width=3774&amp;height=1494&amp;name=Figure2%20(1).png 3774w" sizes="(max-width: 1258px) 100vw, 1258px"></p>
<p style="text-align: center; font-size: 14px;"><em>Figure 2 -&nbsp; Array initialization</em></p>
<p>&nbsp;</p>
<p>Looking at the file contents, a large chunk of data is stored in the variable array that is later decrypted and stored in array2.</p>
<p>&nbsp;</p>
<p><img src="https://www.uptycs.com/hs-fs/hubfs/Figure3%20(1).png?width=1015&amp;height=822&amp;name=Figure3%20(1).png" width="1015" height="822" loading="lazy" alt="Figure3 (1)" style="height: auto; max-width: 100%; width: 1015px; margin-left: auto; margin-right: auto; display: block;" srcset="https://www.uptycs.com/hs-fs/hubfs/Figure3%20(1).png?width=508&amp;height=411&amp;name=Figure3%20(1).png 508w, https://www.uptycs.com/hs-fs/hubfs/Figure3%20(1).png?width=1015&amp;height=822&amp;name=Figure3%20(1).png 1015w, https://www.uptycs.com/hs-fs/hubfs/Figure3%20(1).png?width=1523&amp;height=1233&amp;name=Figure3%20(1).png 1523w, https://www.uptycs.com/hs-fs/hubfs/Figure3%20(1).png?width=2030&amp;height=1644&amp;name=Figure3%20(1).png 2030w, https://www.uptycs.com/hs-fs/hubfs/Figure3%20(1).png?width=2538&amp;height=2055&amp;name=Figure3%20(1).png 2538w, https://www.uptycs.com/hs-fs/hubfs/Figure3%20(1).png?width=3045&amp;height=2466&amp;name=Figure3%20(1).png 3045w" sizes="(max-width: 1015px) 100vw, 1015px"></p>
<p style="text-align: center; font-size: 14px;"><em>Figure 3 - Decryption loop</em></p>
<p>&nbsp;</p>
<p>The decrypt function then decrypts the long array2 object (fig. 3). The result is a GCHandle object; its content creates a “koi” module, after which one of its methods is invoked.</p>
<p>&nbsp;</p>
<p><img src="https://www.uptycs.com/hs-fs/hubfs/Figure4%20(1).png?width=1506&amp;height=475&amp;name=Figure4%20(1).png" width="1506" height="475" loading="lazy" alt="Figure4 (1)" style="height: auto; max-width: 100%; width: 1506px; margin-left: auto; margin-right: auto; display: block;" srcset="https://www.uptycs.com/hs-fs/hubfs/Figure4%20(1).png?width=753&amp;height=238&amp;name=Figure4%20(1).png 753w, https://www.uptycs.com/hs-fs/hubfs/Figure4%20(1).png?width=1506&amp;height=475&amp;name=Figure4%20(1).png 1506w, https://www.uptycs.com/hs-fs/hubfs/Figure4%20(1).png?width=2259&amp;height=713&amp;name=Figure4%20(1).png 2259w, https://www.uptycs.com/hs-fs/hubfs/Figure4%20(1).png?width=3012&amp;height=950&amp;name=Figure4%20(1).png 3012w, https://www.uptycs.com/hs-fs/hubfs/Figure4%20(1).png?width=3765&amp;height=1188&amp;name=Figure4%20(1).png 3765w, https://www.uptycs.com/hs-fs/hubfs/Figure4%20(1).png?width=4518&amp;height=1425&amp;name=Figure4%20(1).png 4518w" sizes="(max-width: 1506px) 100vw, 1506px"></p>
<p style="text-align: center; font-size: 14px;"><em>Figure 4 - Code for Memory Loading of decrypted .Net Module</em></p>
<p>&nbsp;</p>
<p>The figure 4 decryption logic is also related to Confuser. After decryption another .NET-compiled PE file is packed by Confuserex v1.0.0. Figure 4 shows how the “koi” module is loaded internally to the main stub.exe file inside memory.<br><br>After the module is loaded into the memory, there is a call to invoke a fake error message, (fig. 5) stored as a string in the “koi” module. This is done to divert the victim as the “koi” module performs its activity in the background.</p>
<p>&nbsp;</p>
<p><img src="https://www.uptycs.com/hs-fs/hubfs/Figure5%20(1).png?width=512&amp;height=220&amp;name=Figure5%20(1).png" width="512" height="220" loading="lazy" alt="Figure5 (1)" style="height: auto; max-width: 100%; width: 512px; margin-left: auto; margin-right: auto; display: block;" srcset="https://www.uptycs.com/hs-fs/hubfs/Figure5%20(1).png?width=256&amp;height=110&amp;name=Figure5%20(1).png 256w, https://www.uptycs.com/hs-fs/hubfs/Figure5%20(1).png?width=512&amp;height=220&amp;name=Figure5%20(1).png 512w, https://www.uptycs.com/hs-fs/hubfs/Figure5%20(1).png?width=768&amp;height=330&amp;name=Figure5%20(1).png 768w, https://www.uptycs.com/hs-fs/hubfs/Figure5%20(1).png?width=1024&amp;height=440&amp;name=Figure5%20(1).png 1024w, https://www.uptycs.com/hs-fs/hubfs/Figure5%20(1).png?width=1280&amp;height=550&amp;name=Figure5%20(1).png 1280w, https://www.uptycs.com/hs-fs/hubfs/Figure5%20(1).png?width=1536&amp;height=660&amp;name=Figure5%20(1).png 1536w" sizes="(max-width: 512px) 100vw, 512px"></p>
<p style="text-align: center; font-size: 14px;"><em>Figure 5 - Fake error message</em></p>
<p>&nbsp;</p>
<p>Figure 6 shows how the error message is invoked:</p>
<p>&nbsp;</p>
<p><img src="https://www.uptycs.com/hs-fs/hubfs/Figure6%20(1).png?width=1905&amp;height=558&amp;name=Figure6%20(1).png" width="1905" height="558" loading="lazy" alt="Figure6 (1)" style="height: auto; max-width: 100%; width: 1905px; margin-left: auto; margin-right: auto; display: block;" srcset="https://www.uptycs.com/hs-fs/hubfs/Figure6%20(1).png?width=953&amp;height=279&amp;name=Figure6%20(1).png 953w, https://www.uptycs.com/hs-fs/hubfs/Figure6%20(1).png?width=1905&amp;height=558&amp;name=Figure6%20(1).png 1905w, https://www.uptycs.com/hs-fs/hubfs/Figure6%20(1).png?width=2858&amp;height=837&amp;name=Figure6%20(1).png 2858w, https://www.uptycs.com/hs-fs/hubfs/Figure6%20(1).png?width=3810&amp;height=1116&amp;name=Figure6%20(1).png 3810w, https://www.uptycs.com/hs-fs/hubfs/Figure6%20(1).png?width=4763&amp;height=1395&amp;name=Figure6%20(1).png 4763w, https://www.uptycs.com/hs-fs/hubfs/Figure6%20(1).png?width=5715&amp;height=1674&amp;name=Figure6%20(1).png 5715w" sizes="(max-width: 1905px) 100vw, 1905px"></p>
<p style="text-align: center; font-size: 14px;"><em>Figure 6 - Code showing the error message&nbsp;</em></p>
<p>&nbsp;</p>
<p>Here’s an in-depth look into the “koi” module (md5: BD4345C3A7CC6F6E261986E1F5F1E8BC). As it’s packed by Confuserex, to ease our analysis we’ve removed the obfuscation to obtain an unpacked file (md5: 474E0CD6BC1F0FB71BBFFA1AE7DD8E66).<br><br>After the error message appears, the code creates a mutex, then checks if the file is in the Startup subfolder (it sets file attributes to Hidden if not there). The malware then checks for a hardcoded, obfuscated Telegram API and Telegram ID (as follows). If they’re not found it deletes itself, after which dependent DLLs (DotNetZip.dll and AnonFileApi.dll) are downloaded from the following locations:</p>
<p><br><span style="font-size: 16px; color: #ffffff;"><span style="font-family: 'Courier New', Courier, monospace;"><em>hxxps://raw.githubusercontent.com/LimerBoy/StormKitty/master/StormKitty/stub/packages/DotNetZip.1.13.8/lib/net40/DotNetZip.dl</em></span></span><span style="font-size: 16px; color: #ffffff;"><em><em style="font-family: 'Courier New', Courier, monospace; letter-spacing: 0.05em; background-color: transparent; display: inline !important;">hxxps://raw.githubusercontent.com/LimerBoy/StormKitty/master/StormKitty/stub/packages/AnonFileApi.1.14.6/lib/net40/AnonFileApi.dll</em><br></em><span style="font-family: 'Courier New', Courier, monospace;"><em> respectively.</em></span></span><img src="https://www.uptycs.com/hs-fs/hubfs/Figure7%20(1).png?width=1121&amp;height=746&amp;name=Figure7%20(1).png" width="1121" height="746" loading="lazy" alt="Figure7 (1)" style="height: auto; max-width: 100%; width: 1121px; margin-left: auto; margin-right: auto; display: block;" srcset="https://www.uptycs.com/hs-fs/hubfs/Figure7%20(1).png?width=561&amp;height=373&amp;name=Figure7%20(1).png 561w, https://www.uptycs.com/hs-fs/hubfs/Figure7%20(1).png?width=1121&amp;height=746&amp;name=Figure7%20(1).png 1121w, https://www.uptycs.com/hs-fs/hubfs/Figure7%20(1).png?width=1682&amp;height=1119&amp;name=Figure7%20(1).png 1682w, https://www.uptycs.com/hs-fs/hubfs/Figure7%20(1).png?width=2242&amp;height=1492&amp;name=Figure7%20(1).png 2242w, https://www.uptycs.com/hs-fs/hubfs/Figure7%20(1).png?width=2803&amp;height=1865&amp;name=Figure7%20(1).png 2803w, https://www.uptycs.com/hs-fs/hubfs/Figure7%20(1).png?width=3363&amp;height=2238&amp;name=Figure7%20(1).png 3363w" sizes="(max-width: 1121px) 100vw, 1121px"></p>
<p style="text-align: center; font-size: 14px;"><em>Figure 7 - Self deletion code</em></p>

<h3 style="font-size: 20px;"><strong>StormKitty&nbsp;</strong><strong>Similarities</strong></h3>
<p style="font-size: 16px;">Uptycs has discovered that the “HookSpoofer” code is inspired from open source StormKitty with a few enhanced features (e.g., error code popup, memory loading). Misuse of open source stealers has become common, with StormKitty code being observed earlier in Typhon, WorldWind stealer, and Prynt Stealer malware. <h2>Conclusion</h2>
<p style="font-size: 16px;">Stealer bundlers can spread through various channels, including email attachments, fake software downloads, and social engineering techniques. To defend against <a href="/products/use-cases/threat-hunting" rel="noopener">malware attacks</a> like Hookspoofer, Uptycs recommends:</p>
<ul style="font-size: 18px;">
<li style="color: #ffffff;" aria-level="1"><span style="color: #000000;">Users should keep their operating system and security software up to date and avoid downloading files or clicking links from unknown sources.</span></li>
<li style="color: #ffffff;" aria-level="1"><span style="color: #000000;">Businesses should ensure that all staff receive cybersecurity training, and implement measures such as two-factor authentication and data encryption to protect against data theft.</span></li>
</ul>
