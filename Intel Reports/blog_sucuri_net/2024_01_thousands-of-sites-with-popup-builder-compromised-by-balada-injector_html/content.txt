We described this behavior in detail in the article about the <a href="https://blog.sucuri.net/2023/10/balada-injector-targets-unpatched-tagdiv-plugin-newspaper-theme-wordpress-admins.html">attack exploiting the tagDiv Composer vulnerability</a> a few months ago.</p><p>This time, the injected script checks for keywords present in admin related cookies “<b>wp-settings</b>” and “<b>loggen_in</b>” (probably they meant “logged_in”). The file is uploaded to a temporary directory and then deleted immediately after execution.</p><p>In our experience, the attackers use this backdoor to execute the following code:</p><p><img loading="lazy" decoding="async" class="wp-image-32875 size-large aligncenter" src="https://blog.sucuri.net/wp-content/uploads/2024/01/code-executed-from-backdoor-650x50.png" alt="Attackers execute this code from the backdoor" width="650" height="50" srcset="https://blog.sucuri.net/wp-content/uploads/2024/01/code-executed-from-backdoor-650x50.png 650w, https://blog.sucuri.net/wp-content/uploads/2024/01/code-executed-from-backdoor-600x46.png 600w, https://blog.sucuri.net/wp-content/uploads/2024/01/code-executed-from-backdoor-768x59.png 768w, https://blog.sucuri.net/wp-content/uploads/2024/01/code-executed-from-backdoor-390x30.png 390w, https://blog.sucuri.net/wp-content/uploads/2024/01/code-executed-from-backdoor-820x63.png 820w, https://blog.sucuri.net/wp-content/uploads/2024/01/code-executed-from-backdoor-1180x91.png 1180w, https://blog.sucuri.net/wp-content/uploads/2024/01/code-executed-from-backdoor-560x43.png 560w, https://blog.sucuri.net/wp-content/uploads/2024/01/code-executed-from-backdoor.png 1324w" sizes="(max-width: 650px) 100vw, 650px" /></p><p>It’s another backdoor layer that fetches malicious code from <b>http://get.specialcraftbox[.]com/loc/gr.txt</b>, saves it to the <b>sasas</b> file in the system temporary directory, executes it and then deletes it from disk.</p><h3>Secondary wp-blog-header.php injection</h3><p>The <b>sasas</b> file is responsible for the secondary infection. It checks up to 3 levels above the current directory, looking for the root directory of the current site and any other sites that may share the same server account. Then, in the detected site root directories, it modifies the <b>wp-blog-header.php</b> file to inject the same Balada JavaScript malware as was originally injected via the Popup Builder vulnerability.</p><figure id="attachment_32876" aria-describedby="caption-attachment-32876" style="width: 650px" class="wp-caption aligncenter"><img loading="lazy" decoding="async" class="wp-image-32876 size-large" src="https://blog.sucuri.net/wp-content/uploads/2024/01/balada-injection-wp-blog-header-php-650x650.png" alt="Balada injection in wp-blog-header.php" width="650" height="650" srcset="https://blog.sucuri.net/wp-content/uploads/2024/01/balada-injection-wp-blog-header-php-650x650.png 650w, https://blog.sucuri.net/wp-content/uploads/2024/01/balada-injection-wp-blog-header-php-600x600.png 600w, https://blog.sucuri.net/wp-content/uploads/2024/01/balada-injection-wp-blog-header-php-150x150.png 150w, https://blog.sucuri.net/wp-content/uploads/2024/01/balada-injection-wp-blog-header-php-768x767.png 768w, https://blog.sucuri.net/wp-content/uploads/2024/01/balada-injection-wp-blog-header-php-80x80.png 80w, https://blog.sucuri.net/wp-content/uploads/2024/01/balada-injection-wp-blog-header-php-390x389.png 390w, https://blog.sucuri.net/wp-content/uploads/2024/01/balada-injection-wp-blog-header-php-820x819.png 820w, https://blog.sucuri.net/wp-content/uploads/2024/01/balada-injection-wp-blog-header-php-560x559.png 560w, https://blog.sucuri.net/wp-content/uploads/2024/01/balada-injection-wp-blog-header-php.png 1162w" sizes="(max-width: 650px) 100vw, 650px" /><figcaption id="caption-attachment-32876" class="wp-caption-text">Balada injection in wp-blog-header.php</figcaption></figure><p>Instead of using the Popup Builder event handlers, this variation adds the <b>hook_csssss</b> (sometimes <b>hook_csss</b>) function to the WordPress <b>wp_head</b> action hook.</p><pre class="code_syntax" style="color: #d1d1d1; background: #000000;"><span class="line_wrapper">add_action<span style="color: #d2cd86;">(</span><span style="color: #02d045;">'</span><span style="color: #00c4c4;">wp_head</span><span style="color: #02d045;">'</span><span style="color: #d2cd86;">,</span> <span style="color: #02d045;">'</span><span style="color: #00c4c4;">hook_csssss</span><span style="color: #02d045;">'</span><span style="color: #d2cd86;">)</span><span style="color: #b060b0;">;</span></span></pre><p>The <b>hook_csssss</b> function injects the script whose only difference from the one injected into <b>wp_postmeta</b> is it doesn’t depend on any Popup Builder events and works on any page load.</p><h3>Balada Domains</h3><p>Balada operators registered <b>specialcraftbox[.]com</b> and <b>greenfastline[.]com</b> on December 13, 2023 specifically for the Popup Builder campaign and put them behind the CloudFlare firewall to mask their real IP addresses.</p><p>The attack uses multiple subdomains of these domains for different stages of the attack.</p><p>For example:</p><ul><li aria-level="1">service.specialcraftbox[.]com</li><li aria-level="1">soft.specialcraftbox[.]com</li><li aria-level="1">get.specialcraftbox[.]com</li><li aria-level="1">rest.greenfastline[.]com</li><li aria-level="1">fine.greenfastline[.]com</li></ul><p>On January 6, 2024, CloudFlare appeared to have blocked the specialcraftbox[.]com domain so it was moved to the <b>80.66.79[.]248</b> server in Moldova. On January 9, 2024 greenfastline[.]com also had to move to <b>80.66.79[.]248.</b></p><p>As <a href="https://infosec.exchange/@rmceoin/111665788958104483">Randy McEoin mentioned on Mastodon</a>, the redirects to push notification scam involve the following domains:</p><ul><li><b>solohostering[.]com</b> (registered on October 2, 2023)</li><li><b>blueperfectballon[.]com</b></li><li><b>redperfectballon[.]com</b></li><li><b>sunshineblackcolor[.]com</b></li><li><b>sunshinewhitecolor[.]com</b></li></ul><p>These domains are all currently behind the CloudFlare firewall. The last four domains were registered on November 22, 2023.</p><h3>Balada redirect URL patterns</h3><p>It’s quite common for Balada redirects to have a specific redirect URL pattern.</p><p><img loading="lazy" decoding="async" class="size-full wp-image-32877 aligncenter" src="https://blog.sucuri.net/wp-content/uploads/2024/01/balada-redirect-url-patterns.png" alt="Balada Injector redirect URL patterns" width="1500" height="200" srcset="https://blog.sucuri.net/wp-content/uploads/2024/01/balada-redirect-url-patterns.png 1500w, https://blog.sucuri.net/wp-content/uploads/2024/01/balada-redirect-url-patterns-600x80.png 600w, https://blog.sucuri.net/wp-content/uploads/2024/01/balada-redirect-url-patterns-650x87.png 650w, https://blog.sucuri.net/wp-content/uploads/2024/01/balada-redirect-url-patterns-768x102.png 768w, https://blog.sucuri.net/wp-content/uploads/2024/01/balada-redirect-url-patterns-390x52.png 390w, https://blog.sucuri.net/wp-content/uploads/2024/01/balada-redirect-url-patterns-820x109.png 820w, https://blog.sucuri.net/wp-content/uploads/2024/01/balada-redirect-url-patterns-1180x157.png 1180w, https://blog.sucuri.net/wp-content/uploads/2024/01/balada-redirect-url-patterns-560x75.png 560w" sizes="(max-width: 1500px) 100vw, 1500px" /></p><p>You can notice that:</p><ul><li aria-level="1">The redirect chain ends with the <b>0.</b> subdomain of the current malicious domain</li><li aria-level="1">The URL usually contains parameters <b>sub1</b> and or <b>sub2</b> that specify the name of a subcampaign.</li><li aria-level="1">The domain name usually involves combinations of three English words. mitigating threats</h2><p>This current wave of attacks from Balada Injector is a prime example of how dangerous exploitation of known vulnerabilities can be. The stored XSS vulnerability in the Popup Builder plugin was quickly resolved, but not before older vulnerable versions were compromised, leading to over 6,200 infected websites.</p><p>In just the first week of January of 2024, 2+ weeks later after the patched version had been released, our SiteCheck scanner detected this malware on over 3,000 web sites. Many users have updated the Popup Builder plugin since then, but by the time they had updated they already had been infected. It doesn’t take long for Balada operators to identify vulnerable sites and infect thousands of them in a matter of hours.</p><p>This massive infection highlights the importance of promptly updating your WordPress plugins and themes, along with any other website software in your environment. Those who are unable to apply automatic updates or promptly patch to the latest version should consider using a <a href="https://blog.sucuri.net/2022/09/a-guide-to-virtual-patching-for-website-vulnerabilities.html">web application firewall to virtually patch any known vulnerabilities</a>.</p><p>Here are some additional steps to protect your website against malware infections like Balada Injector:</p><ol><li style="list-style-type: none;"><ol><li aria-level="1"><b>Monitor for signs of malicious WordPress admins and plugins:</b> Infections like Balada Injector often create malicious admin users and install fake plugins. Make sure to check the <b>wp_users</b> table and wp-content/plugins/ folder directly since the malware is known to hide its presence in the WordPress dashboard</li><li aria-level="1"><b>Protect WordPress admins:</b> Use <a href="https://blog.sucuri.net/2022/08/how-to-create-secure-passwords-for-your-website.html">unique passwords for all accounts</a>, employ 2FA, and practice the <a href="https://blog.sucuri.net/2024/01/what-is-the-principle-of-least-privilege.html">principle of least privilege</a>. Reset your admin credentials as soon as you’ve become aware of compromise.</li><li aria-level="1"><b>Prevent cross-site contamination: </b>Regularly inspect, clean, and protect all websites under the same server account. Isolate important websites using separate server accounts to curb malware propagation from neighboring websites.</li><li aria-level="1"><b>Promptly remove website backdoors: </b>Thoroughly <a href="https://sucuri.net/website-malware-removal/">clean up malware infections</a> from affected JavaScript and PHP files. Use a <a href="https://blog.sucuri.net/2021/05/server-side-scans-and-file-integrity-monitoring.html" target="_blank" rel="noopener">file integrity control system</a> and leverage professional malware cleanup services.<b> </b></li></ol></li></ol><p>If you&#8217;re a website admin, be sure to check out our <a href="https://sucuri.net/guides/website-security/">website security guide</a>.
