<h2 class="wp-block-heading">Introduction</h2>



<p>After we discovered in <a href="https://blog.fox-it.com/2022/03/03/sharkbot-a-new-generation-android-banking-trojan-being-distributed-on-google-play-store/">February 2022 the SharkBotDropper in Google Play</a> posing as a fake Android antivirus and cleaner, now we have detected a new version of this dropper active in the Google Play and dropping a new version of Sharkbot.<br>This new dropper doesn&#8217;t rely Accessibility permissions to automatically perform the installation of the dropper Sharkbot malware. Instead, this new version ask the victim to install the malware as a fake update for the antivirus to stay protected against threats.<br>We have found two SharkbotDopper apps active in Google Play Store, with 10K and 50K installs each of them.</p>

<p>The Google Play droppers are downloading the full featured Sharkbot V2, discovered some time ago by <a href="https://twitter.com/ThreatFabric/status/1524767906780831750">ThreatFabric</a>. On the 16th of August 2022, Fox-IT&#8217;s Threat Intelligence team observed new command-and-control servers (C2s), that were providing a list of targets including banks outside of United Kingdom and Italy. The new targeted countries in those C2s were: Spain, Australia, Poland, Germany, United States of America and Austria.</p>



<p>On the 22nd of August 2022, Fox-IT&#8217;s Threat Intelligence team found a new Sharkbot sample with version <strong>2.25</strong>; communicating with command-and-control servers mentioned previously. This Sharkbot version introduced a new feature to steal session cookies from the victims that logs into their bank account.</p>



<h2 class="wp-block-heading">The new SharkbotDropper in Google Play</h2>



<p>In the previous versions of SharkbotDropper, the dropper was abusing accessibility permissions in order to install automatically the dropper malware. To do this, the dropper made a request to its command-and-control server, which provided an URL to download the full featured Sharkbot malware and a list of steps to automatically install the malware, as we can see in the following image.</p>

<p>Abusing the accessibility permissions, the dropper was able to automatically click all the buttons shown in the UI to install Sharkbot. But this not the case in this new version of the dropper for Sharkbot. The dropper instead will make a request to the C2 server to directly receive the APK file of Sharkbot. It won&#8217;t receive a download link alongside the steps to install the malware using the &#8216;Automatic Transfer Systems&#8217; (ATS) features, which it normally did.</p>

<p>In order to complete the installation on the infected device, the dropper will ask the user to install this APK as an update for the fake antivirus. Which results in the malware starting an Android Intent to install the fake update.</p>

<p>This way, the new version of the Sharkbot dropper is now installing the payload in a non automatic way, which makes it more difficult to get installed &#8211; since it depends on the user interaction to be installed -, but it is now more difficult to detect before being published in Google Play Store, since it doesn&#8217;t need the accessibility permissions which are always suspicious.<br>Besides this, the dropper has also removed the &#8216;Direct Reply&#8217; feature, used to automatically reply to the received notifications on the infected device. This is another feature which needs suspicious permissions, and which once removed makes it more difficult to detect.</p>

<p>If we look carefully at the code used to check the installed apps against the targeted apps, we can realize that it first makes another check in the first lines:</p>



<pre class="wp-block-code"><code>String lowerCase = ((TelephonyManager) App.f7282a.getSystemService("phone")).getSimCountryIso().toLowerCase();
    if (!lowerCase.isEmpty() &amp;&amp; this.f.getString(0).contains(lowerCase)) </code></pre>



<p>Besides having at least one of the targeted apps installed in the device, the SharkbotDropper is checking if the SIM provider&#8217;s country code is one of the ones included in the configuration &#8211; in this campaign it must be <strong>GB</strong> or <strong>IT</strong>. If it matches and the device has installed any of the targeted apps, then the dropper can request the full malware download from the C2 server. This way, it is much more difficult to check if the app is dropping something malicious. But this is not the only way to make sure only targeted users are infected, the app published in Google Play is only available to install in United Kingdom and Italy.</p>

<h2 class="wp-block-heading">Sharkbot 2.25-2.26: New features to steal cookies</h2>

<ul class="wp-block-list">
<li>Injections (overlay attacks): this feature allows Sharkbot to steal credentials by showing a fake website (phishing) inside a WebView. It is shown as soon as the malware detects one of the banking application has been opened.</li>

<p>Those features were present in Sharkbot 1, but also in Sharkbot 2, which didn&#8217;t change too much related to the implemented features to steal information. <p>We have not observed any big changes until version 2.25, in which the developers of Sharkbot have introduced a new and interesting feature: <strong>Cookie Stealing</strong> or <strong>Cookie logger</strong>. This new feature allows Sharkbot to receive an URL and an User-Agent value &#8211; using a new command &#8216;logsCookie&#8217; -, these will be used to open a WebView loading this URL &#8211; using the received User-Agent as header &#8211; as we can see in the following images of the code.</p>

<h2 class="wp-block-heading">New campaigns in new countries</h2>



<p>During our research, we observed that the newer C2 servers are providing new targeted applications in Sharkbot&#8217;s configuration. The list of targeted countries has grown including <strong>Spain</strong>, <strong>Australia</strong>, <strong>Poland</strong>, <strong>Germany</strong>, <strong>United States of America</strong> and <strong>Austria</strong>. But the interesting thing is the new targeted applications are not targeted using the typical webinjections, instead, they are targeted using the <strong>keylogging</strong> &#8211; grabber &#8211; features. This way, the malware is stealing information from the text showed inside the official app. As we can see in the following image, the focus seems to be getting the account balance and, in some cases, the password, by reading the content of specific TextFields.</p>



<figure class="wp-block-image size-full"><a href="https://i0.wp.com/blog.fox-it.com/wp-content/uploads/2022/08/grabber_config_start.png?ssl=1"><img data-recalc-dims="1" loading="lazy" decoding="async" width="495" height="334" data-attachment-id="3705" data-permalink="https://blog.fox-it.com/2022/09/02/sharkbot-is-back-in-google-play/grabber_config_start/" data-orig-file="https://i0.wp.com/blog.fox-it.com/wp-content/uploads/2022/08/grabber_config_start.png?fit=495%2C334&amp;ssl=1" data-orig-size="495,334" data-comments-opened="0" data-image-meta="{&quot;aperture&quot;:&quot;0&quot;,&quot;credit&quot;:&quot;&quot;,&quot;camera&quot;:&quot;&quot;,&quot;caption&quot;:&quot;&quot;,&quot;created_timestamp&quot;:&quot;0&quot;,&quot;copyright&quot;:&quot;&quot;,&quot;focal_length&quot;:&quot;0&quot;,&quot;iso&quot;:&quot;0&quot;,&quot;shutter_speed&quot;:&quot;0&quot;,&quot;title&quot;:&quot;&quot;,&quot;orientation&quot;:&quot;0&quot;}" data-image-title="grabber_config_start" data-image-description="" data-image-caption="" data-medium-file="https://i0.wp.com/blog.fox-it.com/wp-content/uploads/2022/08/grabber_config_start.png?fit=300%2C202&amp;ssl=1" data-large-file="https://i0.wp.com/blog.fox-it.com/wp-content/uploads/2022/08/grabber_config_start.png?fit=495%2C334&amp;ssl=1" src="https://i0.wp.com/blog.fox-it.com/wp-content/uploads/2022/08/grabber_config_start.png?resize=495%2C334&#038;ssl=1" alt="" class="wp-image-3705" srcset="https://i0.wp.com/blog.fox-it.com/wp-content/uploads/2022/08/grabber_config_start.png?w=495&amp;ssl=1 495w, https://i0.wp.com/blog.fox-it.com/wp-content/uploads/2022/08/grabber_config_start.png?resize=300%2C202&amp;ssl=1 300w" sizes="auto, (max-width: 495px) 100vw, 495px" /></a></figure>

<p>Also, for some of the targeted applications, the malware is providing within the configuration a list of ATS configurations used to avoid the log in based on fingerprint, which should allow to show the usual username and password form. This allows the malware to steal the credentials using the previously mentioned &#8216;keylogging&#8217; features, since log in via fingerprint should ask for credentials.</p>

<h2 class="wp-block-heading">Conclusion</h2>



<p>Since we published our first blog post about Sharkbot in March 2022, in which we detected the SharkbotDropper campaigns within Google Play Store, the developers have been working hard to improve their malware and the dropper. In May, <a href="https://twitter.com/ThreatFabric/status/1524767906780831750">ThreatFabric found a new version of Sharkbot</a>, the version 2.0 of Sharkbot that was a refactor of the source code, included some changes in the communication protocol and in the DGA.</p>



<p>Until now, Sharkbot&#8217;s developers seem to have been focusing on the dropper in order to keep using Google Play Store to distribute their malware in the latest campaigns. These latest campaigns still use fake antivirus and Android cleaners to install the dropper from the Google Play.</p>



<p>With all these the changes and new features, we are expecting to see more campaigns, targeted applications, targeted countries and changes in Sharkbot this year.</p>
