											<div class="text-block bullets-listing">
											<p>By Securonix Threat Research: D. Iuzvyk, T. Peck, O. Kolesnikov</p>
<h2>tldr:</h2>
<p>The Securonix Threat Research team has uncovered an elaborate multi-stage attack campaign likely associated with the North Korean <a href="https://www.cisa.gov/news-events/cybersecurity-advisories/aa20-301a">Kimsuky </a>group.</p>
<p><a href="https://www.securonix.com/wp-content/uploads/2024/03/image16.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="aligncenter wp-image-24826 size-full" src="https://www.securonix.com/wp-content/uploads/2024/03/image16.png" alt="" width="954" height="718" /></a></p>
<p>The Securonix Threat Research (STR) team has been monitoring a new campaign tracked as DEEP#GOSU likely associated with the Kimsuky group, which features some new code/stagers as well as some recycled code and TTPs that were <a href="https://www.genians.co.kr/hubfs/blogfile/20231030_threat_inteligence_report_Kimsuky.pdf?hsLang=ko">reported in the past</a>. While the targeting of South Korean victims by the Kimsuky group <a href="https://thehackernews.com/2024/02/kimsukys-new-golang-stealer-troll-and.html">happened before</a>, from the tradecraft observed it’s apparent that the group has shifted to using a new script-based attack chain that leverages multiple PowerShell and VBScript stagers to quietly infect systems. The later-stage scripts allow the attackers to monitor clipboard, keystroke, and other session activity.</p>
<p>The threat actors also employed a remote access trojan (RAT) software to allow for full control over the infected hosts, while the background scripts continued to provide persistence and monitoring capabilities.</p>
<p>All of the C2 communication is handled through legitimate services such as Dropbox or Google Docs allowing the malware to blend undetected into regular network traffic. Since these payloads were pulled from remote sources like Dropbox, it allowed the malware maintainers to dynamically update its functionalities or deploy additional modules without direct interaction with the system .</p>
<p>The malware used in the DEEP#GOSU campaign likely enters the system through typical means where the user downloads a malicious email attachment containing a zip file with a single disguised file using the extension: pdf.lnk, (IMG_20240214_0001.pdf.lnk) in this case.</p>
<h3>Stage 1: Initial execution: LNK files [T1204.002]</h3>
<p>The use of shortcut files, or .lnk files by threat actors is nothing new. However, in the case of DEEP#GOSU, the methodology behind the code execution is quite different from what we have typically seen in the past.</p>
<p>First, as seen in the figure below, the length of the command is quite impressive and it’s clear that the executed PowerShell is designed to perform several complex functions. Additionally, standing at about 2.2MB, it’s clear that there is more to this shortcut file than what meets the eye.</p>
<p><a href="https://www.securonix.com/wp-content/uploads/2024/03/image9.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="aligncenter wp-image-24798 size-large" src="https://www.securonix.com/wp-content/uploads/2024/03/image9-1024x602.png" alt="" width="640" height="376" /></a></p>
<p>Figure 1: IMG_20240214_0001.pdf.lnk – command line execution</p>
<p>The embedded PowerShell script contained within the shortcut file is designed to take byte data from itself, which extracts embedded files, AESDecrypt and executes further malicious code downloaded from the internet (/step2/ps.bin) and clean up traces of its execution. The use of encryption and cloud services for payload retrieval indicates some level of sophistication intended to evade detection and analysis. This type of infrastructure typically takes much more time to set up and maintain versus simply hosting files on rented servers.</p>
<p>Let’s first analyze the reason that the shortcut file is over 2MB in size. Upon close analysis, the shortcut file appears to have an entire embedded PDF concatenated to it after tens of thousands of “A” characters. Those characters may be an attempt to pad the size of the file to evade AV detections.</p>
<p>The figure below demonstrates how this looks when using a hexadecimal editor to view the file’s raw data. On the left, we can see the end of the shortcut code which calls cmd.exe (to eventually call powershell.exe) and the start of the sequence of “A” characters. Over on the left the A’s terminate and the start of a PDF header appears!</p>

<p>So, the shortcut file has a concatenated PDF file attached to it. The PowerShell code contains a clever function that performs a few tasks. The PowerShell code below is taken from the code from within the shortcut file (figure 1) and then cleaned up a bit so it’s easier to read:</p>
<p><a href="https://www.securonix.com/wp-content/uploads/2024/03/image8.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="aligncenter wp-image-24794 size-full" src="https://www.securonix.com/wp-content/uploads/2024/03/image8.png" alt="" width="2048" height="1204" /></a></p>

<ul>
<li>This portion of the script extracts the PDF portion of the .lnk file&#8217;s content based on specific byte positions which exists between byte values 2105824 and 2282653 (<span style="color: #980000;">$len1</span> to <span style="color: #980000;">$len2</span>).</li>

<li>This extracted content is then eventually saved to a new variable $path, and then executed using the PowerShell Start-Process commandlet.</li>

<p>The PDF lure document is in Korean and appears to be an announcement regarding the son of Korean Airlines CEO Choi Hyun (the late Choi Yul) and states that the son has passed away due to a car accident. The rest contains details and dates of the funeral hall.</p>

<p>In addition to extracting and executing the PDF document, the shortcut file also executes the malware’s next stage payload from a Dropbox URL (<span style="color: #980000;">hxxps://content.dropboxapi[.]com/2/files/download/step2/ps.bin</span>). Despite its name, the ps.bin file is actually another PowerShell script which we’ll dive into later.</p>
<p>Since Dropbox requires authentication, all of the required parameters are embedded into the shortcut’s original PowerShell script (figure 1). <p>To sum up, the PowerShell script contained with the shortcut file is designed to silently find and execute the specifically crafted malicious .lnk file (itself), extract and execute the embedded PDF lure document, authenticate, decrypt and execute further malicious code downloaded from Dropbox, and then clean up traces of its execution. The use of encryption and cloud services for payload retrieval indicates a level of sophistication intended to evade detection and analysis.</p>
<h3>Stage 2: Invoked code from Dropbox [T1102]</h3>
<p>At this stage, the initial shortcut file has downloaded and invoked a remote payload from Dropbox called ps3.bin. The PowerShell code contained within the .bin file defines a function (<span style="color: #980000;">Load</span>) that performs several operations which includes downloading, decompressing, and dynamically loading and executing .NET assembly code from a different Dropbox URL.</p>
<ul>
<li>Define a decompression helper function (<span style="color: #980000;">GzExtract</span>):
<ul>
<li>This inner function takes a byte array as input in the form of GZIP compressed data.</li>
<li>Decompresses this data and return the resulting byte array</li>
</ul>
</li>
<li>Dynamically loading .NET assemblies:
<ul>
<li>The script dynamically loads assemblies related to <span style="color: #980000;">System.Drawing</span>, <span style="color: #980000;">System.Windows.Forms</span>, and <span style="color: #980000;">PresentationCore</span></li>
<li>This enables the script to use advanced graphical UI capabilities which have been used in the past for features such as screenshots or screen recording by <a href="https://www.group-ib.com/blog/dark-pink-apt/">Dark Pink malware</a> among others.</li>
</ul>
</li>
<li>Authenticating with Dropbox and downloading next-stage remote payload:
<ul>
<li>Similar to the shortcut file&#8217;s PowerShell script, it authenticates with Dropbox once again using a refresh token, client ID, and client secret to obtain an access token.</li>
<li>A file named r_enc.bin is downloaded from Dropbox (stage 3).</li>
<li>After downloading the file, it attempts to decompress the payload using the GzExtract function defined earlier.</li>
<li>The script implies this payload is a .NET assembly in binary form, though compressed to evade detection.</li>
</ul>
</li>
<li>Dynamically loading and executing the .NET assembly:
<ul>
<li>It loads the decompressed .NET assembly into memory without writing it to disk which can help cut down AV detections.</li>
<li>It iterates through types and methods within the loaded assembly to find and invoke a specific method (<span style="color: #980000;">makeProbe1</span>). The invocation is commented out, but it suggests that the method would execute with a hardcoded parameter, which is partially shown and then truncated.</li>
<li>This dynamic loading and execution allow the malware to perform virtually any action the .NET framework supports, based on the code within the downloaded assembly.</li>
</ul>
</li>
</ul>
<p><a href="https://www.securonix.com/wp-content/uploads/2024/03/image3.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="aligncenter wp-image-24774 size-large" src="https://www.securonix.com/wp-content/uploads/2024/03/image3-1024x602.png" alt="" width="640" height="376" /></a></p>
<p>Figure 6: Example of PowerShell ps.bin</p>
<p>In addition to the above the script also invokes a method on an object instance using reflection in PowerShell, with a parameter that appears to be a Base64-encoded string. The string can be seen in the figure below.</p>
<p><a href="https://www.securonix.com/wp-content/uploads/2024/03/image5.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="aligncenter wp-image-24782 size-large" src="https://www.securonix.com/wp-content/uploads/2024/03/image5-1024x602.png" alt="" width="640" height="376" /></a></p>
<p>Figure 7: ps.bin PowerShell invokes next stage payloads</p>
<p>The $method variable is set up and holds a reference to a “<span style="color: #980000;">MethodInfo</span>” object, which represents a specific method of a class. The “<span style="color: #980000;">$instance</span>” variable contains the instance of the class which in turn contains the method you want to invoke. The string is encoded in Base64 and then passed as an argument to the method.</p>
<p>Since at this point the code is doing two pretty interesting things simultaneously, let’s follow the loading and execution of “r_enc.bin” from Dropbox further down (Stage 4) which is loaded from the following Dropbox URL:</p>
<p><span style="color: #980000;">hxxps://content.dropboxapi[.]com/2/files/download/step2/r_enc.bin</span></p>
<p>We’ll continue with the invocation of the new Base64 encoded method (Stage 4) further down.</p>
<h3>Stage 3: TutClient [C# RAT] (r_enc.bin)</h3>
<p>When analyzing the PowerShell script in Stage 2 we determined that the script once again reached back out to Dropbox and downloaded a compressed Base64 string. The file itself is indeed a binary file which can be easily confirmed using a tool such as CyberChef. If we place the large Base64 string (r_enc.bin) inside the input field, select “From Base64” and “Gunzip”, we see the MZ header and other common strings for Windows executables inside the output.</p>
<p><a href="https://www.securonix.com/wp-content/uploads/2024/03/image22.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="aligncenter wp-image-24850 size-large" src="https://www.securonix.com/wp-content/uploads/2024/03/image22-1024x602.png" alt="" width="640" height="376" /></a></p>
<p>Figure 8: Decoding r_enc.bin in CyberChef</p>
<p>The decompressed binary file ends up being an open source RAT (remote access trojan), known as TruRat, TutRat or C# R.A.T. which generates a commonly named client called TutClient.exe.</p>
<p>As the name suggests, the RAT is coded in C# and is open source. Since the source of the application can be found online, we won’t go too deep into the binary code analysis portion as it’s <a href="https://github.com/AdvancedHacker101/C-Sharp-R.A.T-Client/blob/master/TutClient/Program.cs">available online</a>, but rather discuss its capabilities.</p>
<p><a href="https://www.securonix.com/wp-content/uploads/2024/03/image19.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="aligncenter wp-image-24838 size-large" src="https://www.securonix.com/wp-content/uploads/2024/03/image19-1024x602.png" alt="" width="640" height="376" /></a></p>
<p>Figure 9: C# RAT executable client overview</p>
<p>Currently this particular RAT software is quite old and likely to be picked up by most antivirus vendors. However, given the unique method in which this binary is loaded and executed directly into memory (stage2), it’s likely to skirt some detections.</p>
<p>Execution of the payload in memory, also known as &#8220;fileless&#8221; execution, is a technique used by attackers to evade detection by traditional file-based antivirus solutions. Since the payload does not touch the disk, it leaves fewer traces, making it harder for security tools to detect and mitigate the threat.</p>
<p>According to the C# Rat’s GitHub page, the malware supports a wide range of features including:</p>
<ul>
<li>Keylogger</li>
<li>Remote desktop</li>
<li>Mic and cam spy</li>
<li>Remote Cmd prompt</li>
<li>Process and file manager</li>
<li>Fun menu (hiding desktop icons, clock, taskbar, showing messagebox, triggering Windows sound effects)</li>
<li>DDoS with target validation</li>
<li>Password manager (supporting: Internet Explorer, Google Chrome, Firefox)</li>
</ul>
<p>Interestingly enough, this is not the first time that we’ve seen this RAT used against Korean targets. A year ago the <a href="https://asec.ahnlab.com/wp-content/uploads/2023/04/ATIP_2023_Feb_Threat-Trend-Report-on-Kimsuky-Group.pdf">Kimsuky group was </a><a href="https://asec.ahnlab.com/wp-content/uploads/2023/04/ATIP_2023_Feb_Threat-Trend-Report-on-Kimsuky-Group.pdf">identified</a> delivering TutRAT and xRAT payloads through other methods.</p>
<h3>Stage 4: VBScript execution (invoked code from stage 2) [T1059.005]</h3>
<p>Circling back to Stage 2, if you recall, we observed a large Base64 encoded string getting invoked. After decoding the string we reveal a VBScript code segment which once again is designed to connect back to Dropbox by interacting with specific web APIs.</p>
<p><a href="https://www.securonix.com/wp-content/uploads/2024/03/image15.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="aligncenter wp-image-24822 size-large" src="https://www.securonix.com/wp-content/uploads/2024/03/image15-1024x602.png" alt="" width="640" height="376" /></a></p>
<p>Figure 10: Stage 4 VBScript execution – download info_sc.txt from Dropbox (from stage 2)</p>
<p>The next stage is downloaded from Dropbox in the same manner we observed during the last several stages. Using a unique client ID, refresh token and secret, the file “info_sc.txt” is downloaded from the URL:</p>
<p><span style="color: #980000;">hxxps://content.dropboxapi[.]com/2/files/download/step2/info_sc.txt</span></p>
<p>Once the file is downloaded, it is written to a VB Stream object then switches the stream&#8217;s type to text and reads it as a UTF-8 encoded string. This is a method to convert binary data (the downloaded file content) into a readable string.</p>
<p>The crucial part of this script is the “<span style="color: #980000;">Execute</span>” statement, which executes the string read from the stream as VBScript code. This means the downloaded content is not just data but executable code, which makes the purpose for Stage 4 run arbitrary VBScript code fetched from Dropbox.</p>
<p><a href="https://www.securonix.com/wp-content/uploads/2024/03/image6.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="aligncenter wp-image-24786 size-large" src="https://www.securonix.com/wp-content/uploads/2024/03/image6-1024x602.png" alt="" width="640" height="376" /></a></p>
<p>Figure 11: Stage 4 VBScript execution execute downloaded code</p>
<p>With the code downloaded from Dropbox, parsed and then converted, it’s placed inside “<span style="color: #980000;">convertedString</span>” and then executed.</p>
<p>Lastly, the script dynamically writes a PowerShell file on the disk and then executes it (<a href="#_bf1x1ed2naqv">Stage 7</a>). This file was written to:</p>
<p><span style="color: #980000;">c:\users\[redacted]\appdata\roaming\microsoft\windows\w568232.ps1</span></p>
<p>Originally the script dropped the file named <span style="color: #980000;">w568232.ps12x</span> , however it was immediately renamed to <span style="color: #980000;">w568232.ps1</span> using the following command:</p>
<p><span style="color: #980000;">cmd /c rename c:\users\[redacted]\appdata\roaming\microsoft\windows\w568232.ps12x w568232.ps1</span></p>
<h3>Stage 5: VBScript execution (info_sc.txt) [T1059.005]</h3>
<p>If you thought at this point we were done with Dropbox stages, you might be right, depending on the OS version the victim system is running. But for now, a closer look at this script reveals several indications of more traditional malware such as persistence indicators and WMI (Windows Management Instrumentation) activity.</p>
<p>The script is quite complex, though it did not feature any form of obfuscation which needed to be decoded. Let’s go over some of the more interesting routines and functions to better understand its capabilities.</p>
<h4>WMI Execution [T1047]</h4>
<p>At the beginning of the script there is a WMProc Subroutine which uses WMI to execute commands on the system. It takes a single parameter <span style="color: #980000;">p_cmd</span> which specifies the executable or script that is launched by the WMI service.</p>
<p>Additionally, there is a commented out line with instructions to download, save and execute a remote .hwp document file. Kimsuky <a href="https://asec.ahnlab.com/en/54736/">has been known to use disguised hwp files in the past</a>, so this could be an artifact of an older attack chain. The commented out line references a remote server at regard.co[.]kr, however we did not observe any network communication to that domain throughout the course of the DEEP#GOSU campaign.</p>
<p><a href="https://www.securonix.com/wp-content/uploads/2024/03/image14.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="aligncenter wp-image-24818 size-large" src="https://www.securonix.com/wp-content/uploads/2024/03/image14-1024x602.png" alt="" width="640" height="376" /></a></p>
<p>Figure 12: Stage 5 VBScript execution – WMProc and TF functions</p>
<h4>Scheduled tasks [T1053]</h4>
<p>The <span style="color: #980000;">TF</span> function works with the <span style="color: #980000;">Reg</span> and <span style="color: #980000;">Reg1</span> subroutines which are used to schedule tasks on the system. Additionally, the <span style="color: #980000;">TF</span> function formats a timestamp for scheduling, and the Reg subroutine actually schedules a new task. This task is configured to execute a script or command at a later time, ensuring that the malware maintains persistence on the system.</p>
<p><a href="https://www.securonix.com/wp-content/uploads/2024/03/image18.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="aligncenter wp-image-24834 size-large" src="https://www.securonix.com/wp-content/uploads/2024/03/image18-1024x602.png" alt="" width="640" height="376" /></a></p>
<p>Figure 13: Stage 5 VBScript execution – Reg and Reg1 functions</p>
<h4>Remote payload download</h4>
<p>At this point the script checks the version of the operating system and branches its behavior accordingly. For OS versions prior to Windows 10, it uses Internet Explorer functionality to download and execute a script fetched from a remote server at</p>
<p><span style="color: #980000;">hxxp://gbionet[.]com/inc/basl/up1/list.php?query=6</span></p>
<p>After contacting the URL above, the script captures the “innerText” of the page&#8217;s body, which is the text content of the response from the server, excluding any HTML tags.</p>
<p>For systems running Windows 10 or later, it uses a PowerShell script which is saved into a single VBScript variable to download and execute a payload from Dropbox using similar methods we witnessed prior.</p>
<p><a href="https://www.securonix.com/wp-content/uploads/2024/03/image13.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="aligncenter wp-image-24814 size-large" src="https://www.securonix.com/wp-content/uploads/2024/03/image13-1024x602.png" alt="" width="640" height="376" /></a></p>
<p>Figure 14: Stage 5 VBScript execution – Next stage download</p>
<p>The inclusion of Google Docs URLs in the PowerShell script encapsulated within the <span style="color: #980000;">psTxt</span> variable is a method used to dynamically retrieve configuration data for the Dropbox connection. This could be useful for when payloads, or Dropbox account data needs to be changed, without having to change the script itself.</p>
<p>As we witnessed previously, the PowerShell script uses a hard-coded password (pa55w0rd), and then executes the decrypted content. This also helps reduce the malware’s detection footprint. Using these types of services to fetch configuration data or payloads can blend in with legitimate network traffic, reducing the likelihood of network-based detection.</p>
<p><a href="https://www.securonix.com/wp-content/uploads/2024/03/image7.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="aligncenter wp-image-24790 size-large" src="https://www.securonix.com/wp-content/uploads/2024/03/image7-1024x602.png" alt="" width="640" height="376" /></a></p>
<p>Figure 15: Stage 5 VBScript/PowerShell execution – invoke next stage</p>
<p>The decrypted content uses a predefined password and AES decryption. Since the downloaded content is encrypted another layer of protection against detection is added.</p>
<p>Interestingly enough, the <span style="color: #980000;">$uh</span> variable is not defined anywhere in the script. This is used by the Invoke-Command alias (icm) to execute a PowerShell scriptblock. This could be a mistake by the malware authors, or used in context with other more broad malware operations where it could be used with portions of code not included in the samples identified by the team.</p>
<p>Lastly, the decrypted content is then executed directly in memory using a PowerShell invoke-expression, which leads us into Stage 6!</p>
<h3>Stage 6: PowerShell execution &#8211; system enumeration [T1082]</h3>

<h3>Stage 7: stealth and persistence in PowerShell [T1041]</h3>
<p>If you recall, this script is created and saved to the disk from Stage 5 (<span style="color: #980000;">\appdata\roaming\microsoft\windows\w568232.ps1</span>). The purpose of this script appears to be designed to serve as a tool for periodic communication with a command and control (C2) server via Dropbox. Its main purposes include encrypting and exfiltrating or downloading data.</p>
<p>Most of the script once again contains PowerShell code for handling Dropbox connections and AES encryption/decryptors however there are a few interesting functions worth mentioning.</p>
<p><a href="https://www.securonix.com/wp-content/uploads/2024/03/image20.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="aligncenter wp-image-24842 size-large" src="https://www.securonix.com/wp-content/uploads/2024/03/image20-1024x602.png" alt="" width="640" height="376" /></a></p>
<p>Figure 18: stage 7 various functions inside w568232.ps1</p>
<p>To ensure persistent, stealthy operation, it contains unique functions for both mutex-based singleton execution (<span style="color: #980000;">$bMute</span>) and variable intervals for network connectivity (<span style="color: #980000;">GetTimeInterval</span>). The time is set to a random interval between 10000 seconds (2.78 hours). Essentially, the script acts as a versatile backdoor that allows attackers to continuously monitor and control their infected systems.</p>
<h3>Stage 8: Keylogging [T1056.001]</h3>
<p>The purpose of this (and final) script is to act as a keylogging and clipboard monitoring component to monitor and log user activity on the compromised system.</p>
<p>It achieves this by first obtaining access to Windows native APIs using .NET assemblies, and then using the Add-Type PowerShell module to call the Core class within the session. The script uses some targeted variable substitution obfuscation throughout the defined strings.</p>
<p><a href="https://www.securonix.com/wp-content/uploads/2024/03/image4.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="aligncenter wp-image-24778 size-large" src="https://www.securonix.com/wp-content/uploads/2024/03/image4-1024x602.png" alt="" width="640" height="376" /></a></p>
<p>Figure 19: stage 8 obfuscated .NET assemblies</p>
<p>The script uses functions such as <span style="color: #980000;">GetAsyncKeyState</span> to monitor the state of individual keys on the keyboard, capturing key presses and releases.</p>
<p><a href="https://www.securonix.com/wp-content/uploads/2024/03/image12.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="aligncenter wp-image-24810 size-large" src="https://www.securonix.com/wp-content/uploads/2024/03/image12-1024x602.png" alt="" width="640" height="376" /></a></p>
<p>Figure 20: stage 8 PowerShell keylogging functions</p>
<p>The PowerShell script includes functionality to monitor and log changes in the clipboard content. It does this by using the <span style="color: #980000;">GetClipboardSequenceNumber</span> function to retrieve the current clipboard sequence number, which changes anytime the content of the clipboard changes.</p>
<p>It then compares the current clipboard sequence number in<span style="color: #980000;"> $curClip</span> with the previously stored sequence number in <span style="color: #980000;">$oldClip</span>. If they differ, it indicates the clipboard content has changed. If the format is verified as “text” it then uses <span style="color: #980000;">[Windows.Clipboard]::GetText()</span> to retrieve the new clipboard text.</p>
<p>Lastly, it appends the content into the <span style="color: #980000;">$Path</span> (Version.xml) variable using <span style="color: #980000;">[System.IO.File]::AppendAllText</span>.</p>
<p><a href="https://www.securonix.com/wp-content/uploads/2024/03/image1.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="aligncenter wp-image-24766 size-large" src="https://www.securonix.com/wp-content/uploads/2024/03/image1-1024x602.png" alt="" width="640" height="376" /></a></p>
<p>Figure 21: stage 8 PowerShell clipboard monitoring</p>
<h4>Additional functionality:</h4>
<ul>
<li><strong>Window monitoring:</strong> It uses both <span style="color: #980000;">GetForegroundWindow</span> and <span style="color: #980000;">GetWindowText</span> to track the active window and its title, enabling the script to log which application the user is interacting with alongside the captured keystrokes or clipboard.</li>
<li><strong>System tick count:</strong> <span style="color: #980000;">GetTickCount</span> is also used to manage the timing of log entries (clipboard, keystrokes, etc), ensuring that entries are spaced out and potentially reducing the volume of logged data to focus on periods of activity.</li>
<li><strong>Encoding and file writing:</strong> All of the captured data is saved into the variable path <span style="color: #980000;">$Path</span> (&#8220;<span style="color: #980000;">$env:appdata\Microsoft\Windows\Themes\version.xml</span>&#8220;), using UTF-8 encoding (created and exfiltrated in stage 7.)</li>
</ul>
<h2>Wrapping up…</h2>
<p>The malware payloads used in the DEEP#GOSU represent a sophisticated, multi-stage threat designed to operate stealthily on Windows systems especially from a network-monitoring standpoint. It relied on both PowerShell and VBScript for its execution which interestingly enough used very minimal obfuscation. Each stage was encrypted using AES and a common password and IV which should minimize network, or flat file scanning detections.</p>
<p>Its capabilities included keylogging, clipboard monitoring, dynamic payload execution, and data exfiltration, and persistence using both RAT software for full remote access, scheduled tasks as well as self-executing PowerShell scripts using jobs.</p>
<h2>Securonix recommendations</h2>
<p>Since many malware infections begin outside the organization, exercise caution especially around unsolicited emails, especially when the email is unexpected or employs a sense of urgency. When it comes to prevention and detection, the Securonix Threat Research Team recommends:</p>
<ul>
<li>Avoid downloading files or attachments from external sources, especially if the source was unsolicited.</li>
<li>Monitor common malware staging directories, especially script-related activity in world-writable directories. In the case of this campaign the threat actors staged in subdirectories in %APPDATA%</li>
<li>Since all of the network communication in the DEEP#GOSU campaign is encrypted and employs legitimate services such as Dropbox or Google Docs, we strongly recommend deploying robust endpoint logging capabilities. This includes leveraging additional process-level logging such as <a href="https://www.securonix.com/blog/improving-blue-team-threat-detection-with-enhanced-siem-telemetry/">Sysmon and PowerShell logging</a> for additional log detection coverage.</li>
<li>Securonix customers can scan endpoints using the Securonix hunting queries below.</li>
</ul>
