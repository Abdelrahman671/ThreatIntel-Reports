This malware has a lot of capabilities and runs its payload mostly in memory which makes its presence stealthy in infected machines. The fileless infection of the malware is mainly using PowerShell modules. Phishing emails with a malicious document, SMB Remote Code Execution Vulnerability (CVE-2017-0144), and brute-force attacks were used to conduct internal network spreading while a malicious document was used to infect external victims. They also leverage some open source tools like XMRig, PingCastle, PowerSploit to achieve their goals.</p><a name='more'></a>
<p></p><h2 style="text-align: justify;">Initial access</h2><div>The infection of this crypto miner begin on the victim in many ways.</div><div><ul style="text-align: left;"><li>Phishing email with Malicious document as an attachment</li><li>SMB exploit</li><li>RDP brute-force</li><li>USB infection</li><li>SSH brute-force</li><li>Pass the hash</li><li>MS-SQL brute-force</li><li>Redis remote command</li><li>Yarn remote command</li></ul></div><p style="text-align: justify;">Our Splunk detection team first detect a lot of suspicious communication were made to a domain name <i>t[.]bb3u9[.]com</i> as shown in below:</p>
