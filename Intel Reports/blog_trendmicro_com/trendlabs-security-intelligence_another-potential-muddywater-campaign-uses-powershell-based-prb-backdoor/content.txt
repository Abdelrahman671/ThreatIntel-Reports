	<h1 class="article-details__title">Potential MuddyWater Campaign uses PRB-Backdoor</h1>
	<p class="article-details__description">We found a new sample that may be related to the MuddyWater campaign. The sample does not directly download the Visual Basic Script and PowerShell component files, and instead encode all the scripts on the document itself.</p>
	<p class="article-details__author-by">By: Michael Villanueva, Martin Co

		<span>Read time:&nbsp;</span><span class="eta"></span> (<span class="words"></span> words)

            <div class="richText">
	
    


	
		<div>
			<p>The <a href="https://www.reuters.com/article/us-saudi-cyber/saudi-agency-says-country-targeted-in-cyber-spying-campaign-idUSKBN1DK27M" target="_blank">MuddyWater campaign</a> was first sighted in 2017 when it targeted the Saudi government using an attack involving PowerShell scripts deployed via Microsoft Office Word macro. In March 2018, we provided a detailed analysis of <a href="/en_us/research/18/c/campaign-possibly-connected-muddywater-surfaces-middle-east-central-asia.html" target="_self">another campaign</a> that bore the hallmarks of MuddyWater.<br />
</p>
<p>In May 2018, we found a new sample (Detected as W2KM_DLOADR.UHAOEEN) that may be related to this campaign. Like the previous campaigns, these samples again involve a Microsoft Word document embedded with a malicious macro that is capable of executing PowerShell (PS) scripts leading to a backdoor payload. One notable difference in the analyzed samples is that they do not directly download the Visual Basic Script(VBS) and PowerShell component files, and instead encode all the scripts on the document itself. The scripts will then be decoded and dropped to execute the payload without needing to download the component files.</p>
<p>As mentioned earlier, our analysis of the sample revealed characteristics that likely connect it to the MuddyWater campaign, in particular:</p>
<ul>
<li><span class="rte-red-bullet">The delivery method, which involves the use of a malicious document with an embedded macro as a lure for potential victims</span></li>
<li><span class="rte-red-bullet">The obfuscation method for the macro scripts, which will result in an intended backdoor payload. This method is commonly used in samples that were used in the MuddyWater campaign</span></li>
</ul>
<p> </p>
<p><b style=""><span class="body-subhead-title">Infection chain</span></b><br />
</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/06/MuddyWater-New.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/06/MuddyWater-New.png"/></a></p>
<p style="text-align: center;">Figure 1. Comparison of the infection chains used in the previous and current campaigns</p>
<p><b><span class="body-subhead-title">Technical details</span></b></p>
<p>The sample we analyzed was a Word document used as a lure for unsuspecting victims. However, unlike the samples from the previous campaigns, the lure document deals with a different subject matter. Instead of using government or telecommunications-related documents, the new lure document presents itself as a reward or promotion, which could indicate that the targets are no longer limited to specific industries or organizations.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/06/MuddyWater-New2.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/06/MuddyWater-New2.png"/></a></p>
<p style="text-align: center;">Figure 2. Sample lure document used in the new campaign</p>
<p>The document is designed to trick users into enabling the macro to view its full content. However, the macro's true purpose is to allow it to execute malicious routines without the user’s knowledge.</p>
<p>Once the macro is enabled, it will use the <b>Document_Open()</b> event to automatically execute the malicious routine if either a new document using the same template is opened or when the template itself is opened as a <i>document0</i>.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/06/MuddyWater-New3.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/06/MuddyWater-New3.png" alt=" Figure 3. Executing the malicious routine via Document_Open()"/></a></p>
<p style="text-align: center;">Figure 3. Executing the malicious routine via Document_Open()</p>
<p>The malicious macro's code snippet uses three main functions, specifically:</p>
<ul>
<li><span class="rte-red-bullet">The function contained in the RED box is the Document_Open() event, where all the sub-functions will be executed/called.</span></li>
<li><span class="rte-red-bullet">The code inside the GREEN box manipulates the images shown in the document's body.</span></li>
<li><span class="rte-red-bullet">The code inside the BLUE box constructs the main Powershell commands and scripts. These will be executed to perform the main routine.</span></li>
</ul>
<p><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/06/MuddyWater-New4.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/06/MuddyWater-New4.png"/></a><br />
</p>
<p style="text-align: center;">Figure 4. A snippet of the malicious macro’s code, marked with colored boxes to show the different functions</p>
<p><b><span class="body-subhead-title">Decoding and deobfuscation</span></b></p>
<p>Analysis of the code revealed a PowerShell script capable of decoding the contents of the malicious document, which results in the execution of yet another encoded PowerShell script.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/06/MuddyWater-New5.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/06/MuddyWater-New5.png" alt=" Figure 5. The Powershell script contained in the sample's code"/></a></p>
<p style="text-align: center;">Figure 5. The Powershell script contained in the sample's code</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/06/MuddyWater-New6.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/06/MuddyWater-New6.png" alt=" Figure 6. The second encoded PowerShell script, which is executed after the first script is decoded"/></a></p>
<p style="text-align: center;">Figure 6. The second encoded PowerShell script, which is executed after the first script is decoded</p>
<p>This will then result in more readable PowerShell scripts capable of dropping various components in the %Application Data%\Microsoft\CLR\* directory. The main PowerShell file <i>invoker.ps1</i> uses these components to run the final payload, PRB-Backdoor, previously analyzed by <a href="https://sec0wn.blogspot.com/2018/05/prb-backdoor-fully-loaded-powershell.html" target="_blank">other security researchers in May 2018</a>.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/06/MuddyWater-New7.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/06/MuddyWater-New7.png"/></a></p>
<p style="text-align: center;">Figure 7: The components dropped in the %Application Data%\Microsoft\CLR\* directory</p>
<p>PRB-Backdoor is a backdoor that takes its name from the function used in the final PowerShell script payload, as seen in the figure below.</p>

<p><b><span class="body-subhead-title">Countermeasures and Trend Micro Solutions</span></b></p>
<p>Given the use of lure documents designed with social engineering in mind, it is likely that the attackers use phishing or spam to target users who are unaware of these documents' malicious nature. Awareness can effectively mitigate or stop these kinds of attacks from being successful. The first step is to be able to <a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/news/cybercrime-and-digital-threats/best-practices-identifying-and-mitigating-phishing-attacks" target="_blank">identify phishing attacks</a> and <a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/news/cybercrime-and-digital-threats/infosec-guide-email-threats" target="_blank">distinguish legitimate emails</a> from malicious ones. Telltale signs of social engineering include “too-good-to-be-true” offers and messages that lack context. In general, users should always practice caution when it comes to email. This includes avoiding clicking on links or downloading any documents unless certain that these are legitimate.</p>
