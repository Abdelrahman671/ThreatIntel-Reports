<p><strong>Netwalker DLL</strong></p>



<p>The first thing the DLL does is resolve all of its needed functions.</p>



<div class="wp-block-image"><figure class="aligncenter"><img src="https://lh5.googleusercontent.com/sxoocHkTfAqvinatGAJsX8iLeXRinKejGL2rY-zNXVBXRRU-vv5g75e_eeVQnMIfRHXhWklE8yOW2eOOApdEeOQ5SyZkBCNoLZiCc4dURcNgDqZMiIn8s-Fw8dhQ0f6y6hWV1cYR" alt="" /></figure></div>



<p>Dynamic function resolution such as this normally involves some form of string hashing, in this case it is CRC32.</p>



<p class="has-text-align-center"><img src="https://lh5.googleusercontent.com/eocjQT9UPgvkmRBlt5U1mDBSujaTQ57QWTTGR87Ae5xIfO4DPihAG-XayoPWoIUPrQAx6oZSS5WwtGtxgOVO2_ysQncIQeMRhPmBkWYS8E6lRep1nzqTQxOmPTHFyS-Gu8keaUis" width="346" height="127"></p>



<p>In the picture above you can see the hash value for FindResourceA being passed to a function that will resolve it and then store it in an array.</p>



<figure class="wp-block-table"><table><tbody><tr><td>&gt;&gt;&gt; hex(zlib.crc32(&#8216;FindResourceA&#8217;) % (1&lt;&lt;32))&#8217;0x3e006b7a&#8217;</td></tr></tbody></table></figure>



<p>After resolving there is a function responsible for simply getting the beginning address of the previous table. Shortly after resolving the needed functions the DLL gets its resource section.</p>



<p class="has-text-align-center"><img src="https://lh5.googleusercontent.com/kGRGiL2E-nsG2aSVredZqQuvwFvfJiRdRsXEsGeN5Eh9OSsBSapEFQo6T5xW1QFGubNIU2I6AkoG5lgO2tSufjzohQEP3tcOgQ1oAZ_8t9Ucl9G5J47cGlFjGUjh7_b7_KG1Iy3S" width="396" height="135"></p>



<p>A little ways down this data is passed off to another function which has a noticeable loop inside of it that is RC4 KSA[2]. If you are wanting to be able to identify some of these common routines I frequently recommend that people compile some files themselves and then take a look at them in a debugger or a disassembler. Routines I would recommend becoming familiar with will be included at the end of this blog.</p>



<p class="has-text-align-center"><img src="https://lh6.googleusercontent.com/UjBpLtQALjp7EPF1uLKsnis12LJUZ2Uiibz0M1Z-zavjHSYZFFCxVo8uT727mXAGjhd1MQ9SDMfxDDG21V4sxbo1HXGxuS7fCLH24xhaUUsmkNkCvhiS1Lr9tpo95z77SPmXY-5P" width="260" height="238"></p>



<p>So since the the resource section appears to be RC4 decrypted, let’s take a look at the data.</p>



<figure class="wp-block-table"><table><tbody><tr><td>&gt;&gt;&gt; r = get_rsrc(pe)<br>&gt;&gt;&gt; data = r[0][1]<br>&gt;&gt;&gt; data[:100]&#8221;\x07\x00\x00\x00cZu-H!&lt;\x9b\xc5E\xder\x88&amp;D\xf0\xd7{@\x00\xa9m\xa1\x1e\x8b\xac\xf7Z\xb4|\xc9&amp;[\x03\x1cY\xde\xf5\x8dA\x97C:8y\xb5&amp;\x07\xb8q\xed\xe1^\xfd&lt;\x98\x92\x95\x1a\xa6\xf7\xea&nbsp; \x07\xea\x18v\xe1\x0f\xa1\xdcmR\x8c\x85&#8217;I\t\xc9\x12\x8e\xe5\x9d+\xc2\x7f\x00G\x01-Jp\xd48 &#8220;</td></tr></tbody></table></figure>



<p>The first 4 bytes appear to be a DWORD value stored as little endian, since I didn’t see a reference to any hardcoded data being used as a RC4 key from the DLL and a small dword value is the first thing in this resource section I’m going to assume it is some sort of a length value. Perhaps a key length?</p>



<figure class="wp-block-table"><table><tbody><tr><td>&gt;&gt;&gt; import struct<br>&gt;&gt;&gt; from Crypto.Cipher import ARC4<br>&gt;&gt;&gt; import struct<br>&gt;&gt;&gt; struct.unpack_from(&#8216;&lt;I&#8217;, data)(7,)<br>&gt;&gt;&gt; key = data[4:4+7]<br>&gt;&gt;&gt; rc4 = ARC4.new(key)<br>&gt;&gt;&gt; t = rc4.decrypt(data[4+7:])<br>&gt;&gt;&gt; t[:100]'{&#8220;mpk&#8221;:&#8221;+1KtL9ibbeqaChhoz4iEHeTtRtw8pNA5yC034\\/3klSA=&#8221;,&#8221;mode&#8221;:0,&#8221;spsz&#8221;:15360,&#8221;thr&#8221;:1500,&#8221;namesz&#8221;:8,&#8221;&#8216;</td></tr></tbody></table></figure>



<p>It’s always nice when your hypothesis turns out to be correct on the first try, definitely isn’t the norm though.</p>



<p>Now that we have decrypted the config we need to find more samples and see if we can decode more configs, I use VirusTotal sometimes and I understand it’s not something everyone has access to so I will upload all the samples I end up going through onto MalwareBazaar.&nbsp;</p>



<p>For pivoting in order to find more samples I will normally turn to YARA and use a service that allows for scanning malware repositories, most of these services are either paid or private so you will normally obtain access to them as you progress into the world of malware research. VirusTotal has a convenient way to quickly search for hex strings using content searching so I pick a string that looks like it will remain static in this malware stub or template but will not give me a bunch of false positives. Sometimes this is harder than not and in this case the string I went with was actually my 3rd attempt.</p>



<p class="has-text-align-center"><img src="https://lh4.googleusercontent.com/LYeOBgKvf3muHNIPI8-x4mqwgXfuMFGA2JGu0RVW46I5Pslf66yOnzMzQPxcGjmHLmvl8doUPV1hlhUCHiUD66jj8UMJLGfhKwFZ5DiadhcEih6KT8zLm1I40kDRQyq_a5od_ufR" width="282" height="575"></p>



<p>Here we have two constant strings associated with SALSA20 or CHACHA20 encryption and following it is a dword value associated with hashing, kind of odd to see them back to back that way in the binary.</p>



<p>We can convert this to a hexlified string for doing VT content searching:</p>



<figure class="wp-block-table"><table><tbody><tr><td>content:&#8221;{657870616e642033322d62797465206b657870616e642031362d62797465206b982f8a42}&#8221;</td></tr></tbody></table></figure>



<p>Searching in VirusTotal shows this to be a pretty good string for finding Netwalker related samples! Surprisingly enough it turns up lots of samples as well.</p>



<p>So now we need a decoder, or a way to automate decoding the config data from the files.</p>



<p>Let&#8217;s build out a decoder, we need to:</p>



<ul class="wp-block-list"><li>Read a file in</li><li>Fix up MZ stomping</li><li>Retrieve the resources</li><li>Find the key length</li><li>RC4 decode</li><li>Print config</li></ul>



<p>Let&#8217;s setup for the first part for retrieving the resources:</p>



<p class="has-text-align-center"><em>The full script can be found as part of the <a rel="noreferrer noopener" href="http://courses.zero2auto.com" target="_blank"><strong>Zero2Automated Advanced Malware Analysis Course</strong></a></em></p>
