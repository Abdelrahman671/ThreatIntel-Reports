											<div class="text-block bullets-listing">
											<p><strong>Securonix Threat Research Security Advisory</strong></p>
<p><strong>By Securonix Threat Research: D.Iuzvyk, T. Peck, O.Kolesnikov</strong></p>
<p>The Securonix Threat Research (STR) team has identified the use of a stealthy backdoor payload likely targeting Pakistani victims via unsolicited messages.</p>
<p><img loading="lazy" decoding="async" class="alignnone size-full wp-image-29286 lazyload" data-src="https://www.securonix.com/wp-content/uploads/2024/06/phantom_advisory_1C.jpg" alt="" width="1344" height="760" src="data:image/gif;base64,R0lGODlhAQABAAAAACH5BAEKAAEALAAAAAABAAEAAAICTAEAOw==" style="--smush-placeholder-width: 1344px; --smush-placeholder-aspect-ratio: 1344/760;" /><br />
In an attack campaign tracked by the Securonix Threat Research team as PHANTOM#SPIKE, threat actors are making use of military-related phishing documents to lure their victims into executing a simple RAT binary payload. While there are many methods used today to deploy malware, the threat actors made use of ZIP files with a password-protected payload archive contained within.</p>
<p>Based on gathered telemetry, the attack appears to be primarily targeting victims associated with Pakistan. Some of the attack payloads identified appear to be involving the United States and other Western countries. While the origin of the attack was never verified, the attackers are likely politically motivated.</p>
<p>The document uses a military-themed meeting title to entice potential targets related to military or defense, increasing the likelihood of executing the file due to its seemingly important or even confidential content.</p>
<p>The attack chain makes use of the seemingly innocuous Compiled HTML Help (CHM) files to deploy and execute next-stage payloads. The use of these types of files highlights a troubling trend where attackers exploit trusted file formats to bypass user and technological defenses.</p>
<h2>Lure analysis &amp; initial access</h2>
<p>The initial lure leverages an archive file titled “Minutes of meeting Intl Military Technical Forum Army 2024.zip” containing a single directory with the same name. Inside that directory, there are two files, another archive file “Minutes of meeting Intl Military Technical Forum Army 2024.rar” and “Minutes of meeting Intl Military Technical Forum Army 2024 Password.txt.”</p>
<p>Once the password is supplied to the RAR file, the contents are displayed to the user, which contains two files, “Minutes of meeting Intl Military Technical Forum Army 2024.pdf.chm” and “RuntimeIndexer.exe”. The EXE file has hidden attributes applied in hopes that the file will not be visible to the user, only the CHM file. The overall directory structure can be seen in the figure below.</p>
<div>
<dl id="attachment_29275">
<dt><img loading="lazy" decoding="async" src="https://www.securonix.com/wp-content/uploads/2024/06/phantom_advisory_2.png" alt="" width="1536" height="903" /></dt>
</dl>
</div>
<h6>Figure 1: Phishing lure directory structure.</h6>
<p>The usage of CHM files in phishing emails is nothing new. Last year, the research team also observed CHM files used in phishing emails against Ukraine during the <a href="https://www.securonix.com/blog/threat-labs-security-advisory-new-starkvortex-attack-campaign-threat-actors-use-drone-manual-lures-to-deliver-merlinagent-payloads/">STARK#VORTEX campaign</a>.</p>
<p>CHM, or Microsoft compiled HTML help files, are a compressed format developed by Microsoft which contains various binary and HTML files typically used for software documentation. Threat actors can abuse this format as it allows for embedded script execution inside its HTML pages, which easily allows for stealthy code execution once the document is opened, or a specific action is taken.</p>
<h2>CHM file analysis</h2>
<p><span style="font-size: 16px;">The purpose of the CHM file is to provide the victim with a convincing phishing lure document while also silently executing code in the background. The lure hopes to pique the victim’s curiosity by supposedly containing documents containing minutes regarding an International Military-Technical Forum in Moscow this past March. The lure consisted of four related images, a sample of which can be seen below.</span></p>
<div>
<dl id="attachment_29274">
<dt><img loading="lazy" decoding="async" src="https://www.securonix.com/wp-content/uploads/2024/06/phantom_advisory_3.png" alt="" width="1536" height="903" /></dt>
</dl>
<h6>Figure 2: CHM file contents</h6>
</div>
<p>These four images within the CHM file can be seen highlighted in the figure below. When the file is opened, The “Document.html” file is displayed, along with the embedded images. Additionally, a small snippet of JavaScript is also executed which is triggered on a user “click” action. This then triggers a set of objects and parameters that executes the binary file “RuntimeIndexer.exe” which was included in the same directory, but hidden from the user.</p>
<div>
<dl id="attachment_29273">
<dt><img loading="lazy" decoding="async" src="https://www.securonix.com/wp-content/uploads/2024/06/phantom_advisory_4.png" alt="" width="1536" height="903" /></dt>
</dl>
</div>
<h6>Figure 3: Main Document.html file contents</h6>
<h2>Breakdown of the HTML Content</h2>
<p>The HTML begins with a standard structure including a &lt;title&gt;, &lt;head&gt;, and &lt;body&gt; tags. The &lt;title&gt; tag indicates the supposed context or theme of the document (&#8220;Minutes of meeting Intl Military Technical Forum Army 2024&#8221;), which again is used to lure the victim into trusting the file&#8217;s contents.</p>
<p>Next, the &lt;body&gt; of the document contains a series of four images which are intended to provide a legitimate look to the document to legitimize the user’s expectations.</p>
<h2>Malicious Code Analysis</h2>
<p>OBJECT Tag with Class ID: The use of an &lt;OBJECT&gt; tag with a specific classid is critical. This classid corresponds to a COM object that can be used to perform actions not typical of your average CHM file. In this case, it&#8217;s configured to create a shortcut referencing a single command action.</p>
<p>PARAM Tags: Inside the &lt;OBJECT&gt; tag, there are &lt;PARAM&gt; tags specifying the properties for the shortcut. First, the command is set to &#8220;ShortCut&#8221;, indicating the creation of a shortcut.</p>
<p>Item1 includes the command to execute, referencing &#8220;RuntimeIndexer,&#8221;. As the locally contained executable file is named “RuntimeIndexer.exe”, this will execute the specified file regardless of whether or not an extension is provided.</p>
<p>SCRIPT Tag and Execution: The &lt;SCRIPT&gt; tag calls the Click() method on the shortcut object created by the &lt;OBJECT&gt; tag. This script triggers the execution of the action defined in the &lt;OBJECT&gt; tag, executing &#8220;RuntimeIndexer.exe&#8221;.</p>
<p>In summary, as soon as the user clicks anywhere on the document, the bundled executable runs silently on the system.</p>
<h2>Binary file analysis</h2>
<p>The binary payload “RuntimeIndexer.exe” is overall quite tiny at only 21KB and was written in CSharp. Fortunately, the executable is a .NET payload so we should be able to extract its source code and analyze it further.</p>
<p>The file is digitally signed however the signature is invalid. While the executable name is RuntimeIndexer.exe, the original filename metadata indicates that the file was renamed from “SearchApp.exe”. Perhaps in some stages, it is designed to masquerade as the legitimate SearchApp process which is a legitimate Windows process which handles functions related to the Windows search bar.</p>
<dl id="attachment_29272">
<dt><img loading="lazy" decoding="async" src="https://www.securonix.com/wp-content/uploads/2024/06/phantom_advisory_5.png" alt="" width="1536" height="903" /></dt>
<dt></dt>
<dt>
<h6>Figure 4: RuntimeIndexer.exe binary file overview</h6>
</dt>

<dd></dd>

<h2>Binary source code analysis</h2>
<p>Delving into the executable&#8217;s source code, we concluded that the malware is designed to function as a backdoor, which connects back to a command and control (C2) server, and allows remote execution of commands on the infected system. Here&#8217;s a breakdown of the code&#8217;s key components and functionalities:</p>
<h2>Network communication and data transmission:</h2>
<p>The malware first attempts to establish a TCP connection to the C2 server at the IP address 162.252.172[.]67 over port 443, which is commonly used for HTTPS. Threat actors typically use common ports in an effort to blend into standard web traffic. Additionally, these ports are typically allowed out by corporate firewalls.</p>
<p>It uses the <a href="https://learn.microsoft.com/en-us/dotnet/api/system.net.security.sslstream?view=net-8.0">SslStream</a> class for encrypted communication over this connection. Despite using SSL, the ValidateServerCertificate method always returns true, indicating it does not verify the C2 server&#8217;s SSL certificate.</p>
<p>Once a successful connection is established, the malware transmits the hostname and username of the infected machine to the C2 server, providing identifying information about the victim. If an error occurs (like loss of connection), the malware prints an error message and tries to reconnect every 5 seconds. This persistent connection attempt makes it more resilient and harder to disable without removing the malware.</p>
<p><img loading="lazy" decoding="async" class="size-full wp-image-29271" src="https://www.securonix.com/wp-content/uploads/2024/06/phantom_advisory_6.png" alt="" width="1536" height="903" /></p>
<h6>Figure 5: RuntimeIndexer.exe &#8211; secure connection details</h6>
<p>At this point, a connection is established and the attacker at the other end of the C2 server is able to perform next-stage operations.</p>
<h2>Command execution:</h2>
<p>The ReceiveAndExecuteCommandsAsync method listens for commands from the C2 server, reading data into a buffer and converting it into a string command.</p>
<p>Any commands received from the C2 server are executed locally on the infected machine using the ExecuteCommandAsync method. This method executes commands via cmd.exe, captures the output, and sends it back to the C2 server. This functionality gives the attacker the ability to remotely execute arbitrary commands.</p>
<p>&nbsp;</p>
<p><img loading="lazy" decoding="async" class="wp-image-29270 size-full" src="https://www.securonix.com/wp-content/uploads/2024/06/phantom_advisory_7.png" alt="" width="1536" height="903" /></p>
<h6>Figure 6: RuntimeIndexer.exe &#8211; command execution details</h6>
<h2>Asynchronous and hidden execution:</h2>
<p>The malware operates asynchronously, making it efficient and responsive. It uses <a href="https://learn.microsoft.com/en-us/dotnet/api/system.threading.tasks.task.run?view=net-8.0">Task.Run</a> to handle each command as it is executed by the attacker which allows each to run without interfering with other commands or operations. Additionally, each spawned process is set to have no window and not use “shell execute”, minimizing its visibility on the infected machine.</p>
<p>To summarize, this backdoor essentially functions as a command line-based remote access trojan (RAT) that provides the attacker with persistent, covert, and secure access to the infected system. The ability to execute commands remotely and relay the results back to the C2 server allows the attacker to control the infected system, steal sensitive information or execute additional malware payloads.</p>
<h2>Post exploitation</h2>
<p>Once the backdoor was established, we observed the attackers perform some general enumeration commands to get a feel for the environment. This included the usage of systeminfo, tasklist and pulling the public IP using ip-api[.]com:</p>
<p>cmd.exe /c systeminfo</p>
<p>cmd.exe /c tasklist</p>
<p>cmd.exe /c dir &#8220;%userprofile%\Downloads\&#8221;</p>
<p>cmd.exe /c curl ip-api[.]com</p>
<p>Next, the binary payload backdoor was copied into the system&#8217;s PUBLIC documents directory. Permissions were also adjusted using attrib.exe to set the file as hidden.</p>
<p>cmd.exe /c copy RuntimeIndexer.exe C:\Users\Public\Documents\</p>
<p>cmd.exe /c attrib +s +h RuntimeIndexer.exe</p>
<p>At this stage we observed the attacker establish persistence manually through Windows scheduled tasks by issuing the following command:</p>
<p>cmd.exe /c \schtasks /CREATE /SC ONEVENT /DELAY 0004:00 /EC Microsoft-Windows-NetworkProfile/Operational /MO &#8220;*[System[Provider[@Name=&#8217;Microsoft-Windows-NetworkProfile&#8217;]]]&#8221; /TN RuntimeTaskMachinesCore /tr &#8220;C:\Users\Public\Documents\RuntimeIndexer.exe&#8221; /F</p>
<h2>Wrapping up…</h2>
<p>What makes this particular attack stand out from many others that we’ve taken a look at over the years, is the use of relatively simple payloads and modularity. The attack does not make use of large or complex sequences of stages. Contrary to other more complex attack chains such as <a href="https://www.securonix.com/blog/detecting-steepmaverick-new-covert-attack-campaign-targeting-military-contractors/">STEEP#MAVERICK</a> or <a href="https://www.securonix.com/blog/threat-labs-security-advisory-new-starkvortex-attack-campaign-threat-actors-use-drone-manual-lures-to-deliver-merlinagent-payloads/">STARK#VORTEX</a>, the benefit of this (from the attacker’s perspective) can be a significant reduction in attack surface as very few artifacts are present on the system which could get picked up by AV or EDR.</p>
<h2>Securonix recommendations</h2>
<p>The PHANTOM#SPIKE campaign, like so many others, begins with a message containing attractive lures to deliver the malware. It’s critical to always be mindful and extra vigilant regarding common tactics threat actors use when delivering phishing emails, or unsolicited messages.</p>
<ul>
<li aria-level="1">Avoid downloading files or attachments from external sources, from email, social media or externally connected messaging applications, especially if the source was unsolicited. ZIP files are an extremely common malware delivery method.</li>
<li aria-level="1">Double-check file extensions before executing them to make sure they are what you expect. These can be enabled in Windows folder options, or simply right-click on the file and select Properties.</li>
<li aria-level="1">Monitor common malware staging directories, especially script-related activity in world-writable directories. In the case of this campaign the threat actors staged in subdirectories in C:\ProgramData as well as the user’s %APPDATA% directory.</li>
<li aria-level="1">Through various phases of the PHANTOM#SPIKE campaign, the threat actors leveraged encrypted channels over port 443 in an effort to blend in with typical SSL/TLS traffic. Because of this, we strongly recommend deploying robust endpoint logging capabilities. This includes leveraging additional process-level logging such as <a href="https://www.securonix.com/blog/improving-blue-team-threat-detection-with-enhanced-siem-telemetry/">Sysmon and PowerShell logging</a> for additional log detection coverage.</li>
<li aria-level="1">Securonix customers can scan endpoints using the Securonix hunting queries below.</li>
</ul>
