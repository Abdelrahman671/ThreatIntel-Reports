<p>During our day to day hunting to protect our customers, we came across two Command and Control servers (C2s) of the North Korea-nexus intrusion set <strong>Reaper (aka APT37)</strong> with open directories, allowing us to observe <strong>hosted implants</strong> as well as <strong>victim’s exfiltrated data</strong>.</p>



<p>Reaper is active since at least 2012, primarily conducting cyberespionage <strong>campaigns against NGOs</strong> and civil society (dissidents, journalists, DPRK defectors). Reaper’s assessed missions are <strong>surveillance and counter intelligence in support of DPRK’s strategic interests</strong>, notably the <strong>Ministry of State Security</strong> (MSS) aka. Bowibu. Reaper used infection vectors in past campaigns include watering holes exploiting 0 day vulnerabilities, and phishing emails with malicious attachment.</p>



<p>SEKOIA.IO analysts’ investigation led to the uncovering of several <strong>phishing webpages</strong>, a CHM infection vector, new <strong>PowerShell implants</strong> and <strong>Chinotto malware modules</strong>, a Reaper signature malware <a href="https://securelist.com/scarcruft-surveilling-north-korean-defectors-and-human-rights-activists/105074/" target="_blank" rel="noreferrer noopener">documented by Kaspersky</a> in November 2021. We assess the recently observed activity almost certainly pertains to the surveillance of North Korea defectors, and associate it to Reaper with high confidence.</p>



<span id="more-4379"></span>



<h2 class="wp-block-heading" id="h-credential-harvesting-via-phishing-webpages">Credential harvesting via phishing webpages</h2>



<p>Our first finding was phishing webpages targeting multiple email and cloud services such as <strong>Naver</strong>, <strong>iCloud</strong>, <strong>Kakao</strong>, <strong>Mail.ru</strong> and <strong>163.com</strong>. In some cases, we were able to retrieve the phishing web pages and analyse their source code.</p>

<p>SEKOIA.IO analysts identified two types of phishing operated by Reaper. The first one, used against 163.com users, is quite standard: the collected credentials are sent to a PHP script and the victim is redirected to the real service via Javascript, following a fake error message. Interestingly, the variables used to send the stolen credentials (<span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">atotsuke</span>, <span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">akaunto</span>, <span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">pasuwado</span>) are in Japanese. SEKOIA.IO analysts assess it is a possible attempt to run a <strong>false flag operation</strong>.</p>



<p>The second one, used against iCloud, Naver and Kakao, is more complex as it can rely on four different technologies (HTTP, websockets, and real time messaging public services Ably and Pubnub) to bypass 2 factors authentication (2FA) mechanism. SEKOIA.IO analysts assess Reaper possibly interface <strong><a href="https://ably.com/" target="_blank" rel="noreferrer noopener">Ably</a></strong> and <strong><a href="https://www.pubnub.com/" target="_blank" rel="noreferrer noopener">Pubnum</a></strong> services with browser automation libraries (such as <strong><a href="https://pptr.dev/" target="_blank" rel="noreferrer noopener">Puppeteer</a></strong>) on the server side. In this case, the browser automation script would be used to check whether the data provided is valid and steal the authentication tokens / cookies, as indicated below.</p>

<p>As the API keys used by Reaper are directly available inside the client-side JavaScript code, <strong>anyone can subscribe to the communication channels and look at Reaper operations</strong> which represents an opportunity for cybersecurity researchers to gain insight into Reaper&#8217;s victimology. </p>



<p>It is worth to note that <strong>these phishing webpages were not resolved by any domain name</strong>. Therefore, we still don&#8217;t know how they were accessed by the victims. It is possible that they were embedded inside <strong>iframes</strong> by using vulnerabilities affecting some services, inserted in attacker&#8217;s controlled websites or in HTML files sent by email. </p>



<h2 class="wp-block-heading" id="h-infection-vector-and-powershell-backdoors">Infection vector and Powershell backdoors</h2>



<p>The infection vectors retrieved during our investigation came from a Github repository online since 2021 and used as a staging infrastructure by Reaper. <h3 class="wp-block-heading" id="h-cku-command">CKU command</h3>



<p>The CKU command is a combination of three functionalities:</p>



<ul class="wp-block-list">
<li>screen capture</li>



<li>keylogging</li>



<li>data collection</li>



<li>etc.</li>
</ul>



<h4 class="wp-block-heading" id="h-screen-capture">Screen capture</h4>



<p>Screenshots are made every 5 seconds. The name of the screenshot contains the date in the following format: <span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">YYYY-MM-DD HH:MM:SS.jpg</span> (example: <span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">2023-01-12 13:23:45.jpg</span>)</p>



<h4 class="wp-block-heading" id="h-keylogging">Keylogging</h4>



<p>The keylogging functionality uses the <span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">GetAsyncKeyState</span> API. Each keystroke is logged with the name of the current foreground windows in a file named <span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">C:\Users\Public\Key.ini</span></p>



<h4 class="wp-block-heading" id="h-data-collection">Data collection</h4>



<p>As described by Kaspersky, to select data to extract, Chinotto needs a filter containing:</p>



<ul class="wp-block-list">
<li>a path</li>



<li>at least one regex describing the files to be extracted.</li>
</ul>



<p>The malware will then gather all files matching the regex from the directory path and its subdirectories.</p>



<p>The CKU command:</p>



<ul class="wp-block-list">
<li>collects <span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">.ini</span> files from <span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">C:\Users\Public</span> (including subdirectories). This includes the <span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">Key.ini</span> file. The corresponding filter is <span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">C:\Users\Public&gt;ini|</span></li>



<li>checks if removable devices are attached to the computer. If so, all files are dumped. The filter is <span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">:\&gt;<em>.</em>|</span></li>



<li>checks if the directory <span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">C:\ProgramData\Phone</span> exists. If so, all files are also dumped. The filter is <span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">C:\ProgramData\Phone&gt;<em>.</em>|</span>.</li>
</ul>



<h3 class="wp-block-heading" id="h-data-command">DATA command</h3>



<p>We only identified a few “DATA” samples, and found these variants are of lesser complexity than the CKU variant. The data collection uses the same principle as for the CKU command, with different filters. The variants we found don’t have the same behaviour, for instance here are two filters found in different data variants:</p>



<ul class="wp-block-list">

<h3 class="wp-block-heading" id="h-data-extraction">Data extraction</h3>



<p>Data extraction (for CKU and DATA commands) consists in creating a ZIP archive of files with the following name format <span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">e_[ckupd]-[a-zA-Z0-9]{8}.zip</span>. <ul class="wp-block-list">
<li><span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">d</span>: documents collected by the “data” DLL</li>

<li><span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">u</span>: the archive contains files from removable disks</li>

<h2 class="wp-block-heading" id="h-additional-findings-found-on-reaper-s-c2">Additional findings found on Reaper&#8217;s C2</h2>



<h3 class="wp-block-heading" id="h-ablygo-backdoor">AblyGo Backdoor</h3>



<p>AblyGo is a simple backdoor written in Go also found on Reaper’s C2. It uses the Ably framework to receive commands. The sample SEKOIA.IO analysts observed subscribes to a channel by providing:</p>



<ul class="wp-block-list">
<li>an application ID</li>



<li>an authentication key</li>



<li>a channel name.</li>
</ul>



<p>Similar to the 2FA bypass, we try to subscribe to the channel. But in this case an error is returned: <span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">Application [REDACTED] disabled</span>.</p>



<h3 class="wp-block-heading" id="h-loaders">Loaders</h3>



<p>We also found several loaders on the C2. These loaders are a modified version of the <span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">mfc42u.dll</span> DLL. The purpose of these DLLs is to load a DLL named <span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">evc.dll</span>. The path of this loaded DLL is hardcoded. Several paths were found:</p>



<ul class="wp-block-list">
<li><span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">c:\users\public\data\evc.dll</span></li>



<li><span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">c:\users\public\librairies\evc.dll</span></li>



<li><span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">c:\users\public\vnc\evc.dll</span></li>



<li><span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">c:\programdata\evc.dll</span></li>
</ul>



<p>Unfortunately, we were unable to get a sample of <span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">evc.dll</span> during this investigation.</p>



<p>We also retrieved a customised VNC instance. At the time of writing, SEKOIA.IO analysts were not able to assess whether or how these additional resources were used by Reaper (APT37) in this campaign.</p>



<h3 class="wp-block-heading">ExtremeVNC</h3>



<p>The final sample we found is named ExtremeVNC (from its PDB path). As its name suggests, this has VNC capability. It communicates with the C2 via HTTP and exchanges json data every second. The protocol’s commands are:</p>



<ul class="wp-block-list">
<li><span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">BROWSER_REQ</span>: executes a command with cmd.exe /c start &lt;command&gt; &#8211;no-sandbox &#8211;allow-no-sandbox-job &#8211;disable-3d-apis &#8211;disable-gpu &#8211;disable-d3d11</li>



<li><span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">SC_REQ</span>: take screenshot</li>



<li><span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">CLIP_REQ</span>: set clipboard data</li>



<li><span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">EVENT_REQ</span>: simulate mouse and keyboard</li>



<li><span style="background-color:#FAFAFA;border-radius:3px;padding3px;border:1px solid #CCC;padding-right:5px;padding-left:5px;font-family:monospace">CLOSE_REQ</span>: stop ExtremeVNC execution</li>
</ul>



<p>After each command, a response is sent to the C2.</p>



<h2 class="wp-block-heading" id="h-conclusion">Conclusion</h2>



<p>SEKOIA.IO analysts assess that recent observed activity was almost certainly part of a cyberespionage campaign by Reaper, highly likely targeting North Korean defectors in South Korea.</p>



<p>Our analysis provides insight into Reaper’s use of Chinotto, a malware solely associated to this intrusion set, as well as its continuous development efforts, as indicated by the new variants documented. SEKOIA.IO analysts assess Reaper will continue leveraging Chinotto in cyberespionage campaigns in the short to near term.</p>



<p>SEKOIA.IO will continue to track and report on this intrusion set’s activities, notably via our <a href="https://app.sekoia.io/intelligence/" target="_blank" rel="noreferrer noopener">Intelligence Center</a>.</p>
