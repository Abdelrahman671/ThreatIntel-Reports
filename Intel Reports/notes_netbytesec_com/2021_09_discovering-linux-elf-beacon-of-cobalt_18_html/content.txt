<a name='more'></a><h2 style="text-align: left;">Introduction</h2><p style="text-align: justify;">Cobalt Strike is a tool used for red teaming and penetration testing to demonstrate the cyber attack. Cobalt Strike is a commercial, full-featured, remote access tool that bills itself as "adversary simulation software designed to execute targeted attacks and emulate the post-exploitation actions of advanced threat actors". Cobalt Strike&#8217;s interactive post-exploit capabilities cover the full range of ATT&amp;CK tactics, all executed within a single, integrated system.<br /></p><p style="text-align: justify;">Nowadays, real attackers and cyber threat actors have been used this tool a lot in their operations to conduct a cyber-attack against their target. Today, we see another evolution of Cobalt Strike where the threat actor has developed a Linux version of a payload of Cobalt Strike where it gets 0 detection in the VirusTotal and remains stealthy in our client premises for more than 3 months.</p><h2 style="text-align: justify;">Graph flow</h2><div style="text-align: justify;"><table align="center" cellpadding="0" cellspacing="0" class="tr-caption-container" style="margin-left: auto; margin-right: auto;"><tbody><tr><td style="text-align: center;"><a href="https://blogger.googleusercontent.com/img/b/R29vZ2xl/AVvXsEh1QQGrWkb5P8MJ1eYZZjLL5_wbb66ks1GF15UTa7HHnNARcNFH_k2hMUlTMR4IF76HKJpevyS2ORRXBenvtDajVMyS4MS2jZbel-vcDqS4uC9x8GbdrMTs361Y9Y2Iy-ZdJ2Z50EOWsDJc/s467/image3.jpeg" style="margin-left: auto; margin-right: auto;"><img border="0" data-original-height="111" data-original-width="467" src="https://blogger.googleusercontent.com/img/b/R29vZ2xl/AVvXsEh1QQGrWkb5P8MJ1eYZZjLL5_wbb66ks1GF15UTa7HHnNARcNFH_k2hMUlTMR4IF76HKJpevyS2ORRXBenvtDajVMyS4MS2jZbel-vcDqS4uC9x8GbdrMTs361Y9Y2Iy-ZdJ2Z50EOWsDJc/s16000/image3.jpeg" /></a></td></tr><tr><td class="tr-caption" style="text-align: center;">Figure 1: Graph flow of the malware</td></tr></tbody></table></div><div style="text-align: justify;"></div><div style="text-align: justify;">&nbsp;</div><div style="text-align: justify;">Based on figure 1, the malware has a simple function routine. The core function of this malware is to make the beacon connection to the attacker Cobalt Strike server via DNS using the Cobalt Strike Malleable C2 config embedded in the malware. The most important and interesting case is case 2 where all the beacon connections are created and make to communicate with the C2 server.</div><div><br /></div><table align="center" cellpadding="0" cellspacing="0" class="tr-caption-container" style="margin-left: auto; margin-right: auto;"><tbody><tr><td style="text-align: center;"><a href="https://blogger.googleusercontent.com/img/b/R29vZ2xl/AVvXsEgHM5npPRGNsJd7pkc3KWLlL13pQ7uDAxZlwfxOFIkbsD7oCx5Me5fHaI8N0hE2riQOhcySXE5Yt9oflIRG6vbDGwhyphenhyphenjrEi9oSGWOsxXrIW3f8muYmo_DHAiVsqi-uF0OB4XualAjQCfNE0/s1463/Snipaste_2021-09-17_10-46-56.png" style="margin-left: auto; margin-right: auto;"><img border="0" data-original-height="650" data-original-width="1463" src="https://blogger.googleusercontent.com/img/b/R29vZ2xl/AVvXsEgHM5npPRGNsJd7pkc3KWLlL13pQ7uDAxZlwfxOFIkbsD7oCx5Me5fHaI8N0hE2riQOhcySXE5Yt9oflIRG6vbDGwhyphenhyphenjrEi9oSGWOsxXrIW3f8muYmo_DHAiVsqi-uF0OB4XualAjQCfNE0/s16000/Snipaste_2021-09-17_10-46-56.png" /></a></td></tr><tr><td class="tr-caption" style="text-align: center;">Figure 17: Part of case 2 code<br /></td></tr></tbody></table><br /></div></div><div style="text-align: justify;">In the <i>sub_40D440</i> subroutine function, the Netbytesec team

IP address to the request header &#8220;<i>Host: </i>&#8221;. As seen in the figure below, IP address <i>160.202.163.100</i> is matching with the PCAP communication. This IP address is matching with the previous OSINT research result on both domains. <br /></div><p></p><table align="center" cellpadding="0" cellspacing="0" class="tr-caption-container" style="margin-left: auto; The IP was also flagged as malicious which has communicated with a malicious file recently.</div><div style="text-align: justify;">&nbsp;</div><div style="text-align: justify;"><table align="center" cellpadding="0" cellspacing="0" class="tr-caption-container" style="margin-left: auto; margin-right: auto;"><tbody><tr><td style="text-align: center;"><a href="https://blogger.googleusercontent.com/img/b/R29vZ2xl/AVvXsEhcEWAO2mmC3EIlRjCfyWuTRgzv4K9soE5dyhrvKJhag0josc9tKvXZU7wAbfNtHIcLDaiK4ua16AZBrIZGGcs_H9yArLJ9wcbgOflwZb9YBFfnnkngNyewngGKRYaGHhFGcWDjNHGgTK_V/s717/image27.png" style="margin-left: auto; margin-right: auto;"><img border="0" data-original-height="591" data-original-width="717" src="https://blogger.googleusercontent.com/img/b/R29vZ2xl/AVvXsEhcEWAO2mmC3EIlRjCfyWuTRgzv4K9soE5dyhrvKJhag0josc9tKvXZU7wAbfNtHIcLDaiK4ua16AZBrIZGGcs_H9yArLJ9wcbgOflwZb9YBFfnnkngNyewngGKRYaGHhFGcWDjNHGgTK_V/s16000/image27.png" /></a></td></tr><tr><td class="tr-caption" style="text-align: center;">Figure 20: Virustotal result of the malicious IP</td></tr></tbody></table><br /></div><div style="text-align: justify;">Finally, the connection is officially create using the function 
<i>BIO_s_connect()</i>. This is a wrapper around the platform's TCP/IP socket 
connection routines. The connection is created and set up using the 
strings and data like URL and cookies append along with the GET request 
headers to complete the connection to the attacker infrastructure.</div><div style="text-align: justify;">&nbsp;</div><table align="center" cellpadding="0" cellspacing="0" class="tr-caption-container" style="margin-left: auto; margin-right: auto;"><tbody><tr><td style="text-align: center;"><a href="https://blogger.googleusercontent.com/img/b/R29vZ2xl/AVvXsEihoXvjx3Msv1LM3Qj4C9xaiWufR1xWWlGiBJlff-UHxEKl0ZLZIIrzE9oetzJuWE9O3ngvxDS-UQ2D9qFkHTX3mg8RUKbmVZPc4DyDGMnlBipwRpi2qYeAkoWI7KSwDAEniyRwIcV6WrQU/s1023/bio_connect.png" style="margin-left: auto; margin-right: auto;"><tbody><tr><td style="text-align: center;"><a href="https://blogger.googleusercontent.com/img/b/R29vZ2xl/AVvXsEhNFWZzYBfxtggVL7WvZ5TPlmww8hSzFMbq0EX5atkGHJX-UGV-lkhHaqFEM7AUPY_0wSIEM5YfoM5bBlU9d_vSFX0JLD45CWtHb9JQKl_dy2V1OIEelylzp83f8tAHN-Flz38PRw7kgkIk/s1920/index.png" style="margin-left: auto; For example:</p><p></p><table align="center" cellpadding="0" cellspacing="0" class="tr-caption-container" style="margin-left: auto; margin-right: auto;"><tbody><tr><td style="text-align: center;"><a href="https://blogger.googleusercontent.com/img/b/R29vZ2xl/AVvXsEiaIUUmQZ5MSVQ2xFEyVx948_MCTJfgAQq0IYUUq-Hu4Yv-G4_eBOQZ4NI_t0taE5dEpwNE1JiIwChSREr6bS1SIyALRukr5I-E7mhjPSF5qO9nIWprFti0dqQncNV_5YX1Ib2pM-G_E74v/s1920/index.png" style="margin-left: auto; margin-right: auto;"><img border="0" data-original-height="796" data-original-width="1920" src="https://blogger.googleusercontent.com/img/b/R29vZ2xl/AVvXsEiaIUUmQZ5MSVQ2xFEyVx948_MCTJfgAQq0IYUUq-Hu4Yv-G4_eBOQZ4NI_t0taE5dEpwNE1JiIwChSREr6bS1SIyALRukr5I-E7mhjPSF5qO9nIWprFti0dqQncNV_5YX1Ib2pM-G_E74v/s16000/index.png" /></a></td></tr><tr><td class="tr-caption" style="text-align: center;">Figure 26: Pcap capture of the TXT query and response<br /></td></tr></tbody></table><p></p><p>As
 explained from the CS blog, when Cobalt Strike&#8217;s DNS server gets the 
request, it checks if any tasks are available for that Beacon instance. 
If no tasks are available, it returns a response that tells Beacon to go
 to sleep. If a task is available, the Cobalt Strike DNS server returns 
an IP address. The compromised system then connects to that IP address 
and makes an HTTP GET request to download its tasks. The tasks are 
encrypted. That's why we see there is HTTP GET request construction in 
the code. This is the hybrid communication model. The idea here is that 
DNS requests on a regular interval are less likely to get noticed than, 
say, HTTP requests on a regular interval.</p></div><div style="text-align: justify;">To track the compromise events, the NetByteSec Splunk analyst discovers that a few servers of our client's
 have made the DNS beaconing to the Cobalt Strike since April 2021.</div><div style="text-align: justify;">&nbsp;<table align="center" cellpadding="0" cellspacing="0" class="tr-caption-container" style="margin-left: auto; margin-right: auto;"><tbody><tr><td style="text-align: center;"><a href="https://blogger.googleusercontent.com/img/b/R29vZ2xl/AVvXsEiiXTs1bCurqqlMZI77JePDHg2XlFidyc43E1t3cVsSOwFW5pPP3yUMWZ2DixoRis4judZ90p5MmVmJVAwv1vSHo3H2TIeR68VTePHB1YMtynZ35r24J-xbRRdJXUsMi34V9UXpY4gV3qmq/s3375/1.png" style="margin-left: auto; margin-right: auto;"><img border="0" data-original-height="932" data-original-width="3375" src="https://blogger.googleusercontent.com/img/b/R29vZ2xl/AVvXsEiiXTs1bCurqqlMZI77JePDHg2XlFidyc43E1t3cVsSOwFW5pPP3yUMWZ2DixoRis4judZ90p5MmVmJVAwv1vSHo3H2TIeR68VTePHB1YMtynZ35r24J-xbRRdJXUsMi34V9UXpY4gV3qmq/s16000/1.png" /></a></td></tr><tr><td class="tr-caption" style="text-align: center;"><i>Figure 27: Splunk detection on the IP</i></td></tr></tbody></table><br /></div><div style="text-align: justify;"><h2>Conclusion</h2><div>The attacker has dropped their malicious 
software in the servers and run the malware. The malware has the ability
 to run in the background and create a DNS beacon connection to the 
Cobalt Strike C2 server hosted on IP 160.202.163.100. Before the malware
 is set up and creates the connection, the malware will decrypt a lot of
 strings and data include Cobalt Strike config, and then parse and 
append it to the function that will make the connection function. Most 
of the findings of OSINT research of the found IP address and Domain 
tell that these IOCs are positive belong to an attacker that is using 
the Cobalt Strike tool to conduct the attack. The Netbytesec team 
believes that this cyber attack was conducted by an actor who were able to tweak and port the Cobalt Strike payload to Linux's
 based version and remain stealthy after compromised our client.<br /></div><div>&nbsp;</div><div><div><h2>Indicator of Compromises&nbsp;</h2><div><b>IP address</b></div><div><ul><li>160.202.163.100 <br /></li></ul></div><div><b>Domains</b></div><div><ul><li>microsoftkernel.com</li><li>microsofthk.com <br /></li></ul></div><div><b>Subdomains</b><br /><ul><li>update.microsoftkernel.com</li><li>update.microsofthk.com </li></ul></div><div><b>Hash </b><br /></div></div><div><ul><li>3db3e55b16a7b1b1afb970d5e77c5d98</li><li>c5718ec198d13ef5e3f81cecd0811c98<br /><br /></li></ul></div></div></div>
