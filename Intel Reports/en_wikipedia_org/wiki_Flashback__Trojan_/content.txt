					<div id="contentSub"><div id="mw-content-subtitle"></div></div>
					
					
					<div id="mw-content-text" class="mw-body-content"><div class="mw-content-ltr mw-parser-output" lang="en" dir="ltr"><div class="shortdescription nomobile noexcerpt noprint searchaux" style="display:none">Malware targeting Mac OS X systems</div>
<p><b>OSX.FlashBack</b>,<sup id="cite_ref-1" class="reference"><a href="#cite_note-1"><span class="cite-bracket">&#91;</span>1<span class="cite-bracket">&#93;</span></a></sup> also known as the <b>Flashback</b> Trojan, <b>Fakeflash</b>, or <b>Trojan BackDoor.Flashback</b>, is a <a href="/wiki/Trojan_horse_(computing)" title="Trojan horse (computing)">Trojan horse</a> affecting personal computer systems running Mac <a href="/wiki/MacOS" title="MacOS">OS X</a>.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2"><span class="cite-bracket">&#91;</span>2<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-3" class="reference"><a href="#cite_note-3"><span class="cite-bracket">&#91;</span>3<span class="cite-bracket">&#93;</span></a></sup> The first variant of Flashback was discovered by antivirus company <a href="/wiki/Intego" title="Intego">Intego</a> in September 2011.<sup id="cite_ref-intego_4-0" class="reference"><a href="#cite_note-intego-4"><span class="cite-bracket">&#91;</span>4<span class="cite-bracket">&#93;</span></a></sup>
</p>
<meta property="mw:PageProp/toc" />
<div class="mw-heading mw-heading2"><h2 id="Infection">Infection</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Flashback_(Trojan)&amp;action=edit&amp;section=1" title="Edit section: Infection"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>According to the Russian antivirus company <a href="/wiki/Dr._Web" class="mw-redirect" title="Dr. Web">Dr. Web</a>, a modified version of the "BackDoor.Flashback.39" variant of the Flashback Trojan had infected over 600,000 Mac computers, forming a <a href="/wiki/Botnet" title="Botnet">botnet</a> that included 274 bots located in <a href="/wiki/Cupertino,_California" title="Cupertino, California">Cupertino, California</a>.<sup id="cite_ref-arstechnica_5-0" class="reference"><a href="#cite_note-arstechnica-5"><span class="cite-bracket">&#91;</span>5<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-drweb_6-0" class="reference"><a href="#cite_note-drweb-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup> The findings were confirmed one day later by another computer security firm, <a href="/wiki/Kaspersky_Lab" title="Kaspersky Lab">Kaspersky Lab</a>.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7"><span class="cite-bracket">&#91;</span>7<span class="cite-bracket">&#93;</span></a></sup> This variant of the <a href="/wiki/Malware" title="Malware">malware</a> was first detected in April 2012<sup id="cite_ref-bbc_infected_8-0" class="reference"><a href="#cite_note-bbc_infected-8"><span class="cite-bracket">&#91;</span>8<span class="cite-bracket">&#93;</span></a></sup> by Finland-based computer security firm <a href="/wiki/F-Secure" title="F-Secure">F-Secure</a>.<sup id="cite_ref-9" class="reference"><a href="#cite_note-9"><span class="cite-bracket">&#91;</span>9<span class="cite-bracket">&#93;</span></a></sup><sup id="cite_ref-10" class="reference"><a href="#cite_note-10"><span class="cite-bracket">&#91;</span>10<span class="cite-bracket">&#93;</span></a></sup> Dr. Web estimated that in early April 2012, 56.6% of infected computers were located within the <a href="/wiki/United_States" title="United States">United States</a>, 19.8% in <a href="/wiki/Canada" title="Canada">Canada</a>, 12.8% in the <a href="/wiki/United_Kingdom" title="United Kingdom">United Kingdom</a> and 6.1% in <a href="/wiki/Australia" title="Australia">Australia</a>.<sup id="cite_ref-drweb_6-1" class="reference"><a href="#cite_note-drweb-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading2"><h2 id="Details">Details</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Flashback_(Trojan)&amp;action=edit&amp;section=2" title="Edit section: Details"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p>The original variant used a fake installer of <a href="/wiki/Adobe_Flash_Player" title="Adobe Flash Player">Adobe Flash Player</a> to install the malware, hence the name "Flashback".<sup id="cite_ref-intego_4-1" class="reference"><a href="#cite_note-intego-4"><span class="cite-bracket">&#91;</span>4<span class="cite-bracket">&#93;</span></a></sup>
</p><p>A later variant targeted a <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> vulnerability on Mac OS X. The system was infected after the user was redirected to a compromised bogus site, where <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a> code caused an <a href="/wiki/Applet" title="Applet">applet</a> containing an exploit to load. An executable file was saved on the local machine, which was used to download and run malicious code from a remote location. The malware also switched between various servers for optimized load balancing. Each bot was given a unique ID that was sent to the control server.<sup id="cite_ref-drweb_6-2" class="reference"><a href="#cite_note-drweb-6"><span class="cite-bracket">&#91;</span>6<span class="cite-bracket">&#93;</span></a></sup> The trojan, however, would only infect the user visiting the infected web page, meaning other users on the computer were not infected unless their user accounts had been infected separately.<sup id="cite_ref-11" class="reference"><a href="#cite_note-11"><span class="cite-bracket">&#91;</span>11<span class="cite-bracket">&#93;</span></a></sup>
</p>
<div class="mw-heading mw-heading2"><h2 id="Resolution">Resolution</h2><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Flashback_(Trojan)&amp;action=edit&amp;section=3" title="Edit section: Resolution"><span>edit</span></a><span class="mw-editsection-bracket">]</span></span></div>
<p><a href="/wiki/Oracle_Corporation" title="Oracle Corporation">Oracle</a>, the company that develops Java, fixed the vulnerability exploited to install Flashback on February 14, 2012.<sup id="cite_ref-bbc_infected_8-1" class="reference"><a href="#cite_note-bbc_infected-8"><span class="cite-bracket">&#91;</span>8<span class="cite-bracket">&#93;</span></a></sup> However, at the time of Flashback's release, Apple maintained the Mac OS X version of Java and did not release an update containing the fix until April 3, 2012,<sup id="cite_ref-Apple_About_Flashback_malware_12-0" class="reference"><a href="#cite_note-Apple_About_Flashback_malware-12"><span class="cite-bracket">&#91;</span>12<span class="cite-bracket">&#93;</span></a></sup> after the flaw had already been exploited to install Flashback on 600,000 Macs.<sup id="cite_ref-flashbackcheck_13-0" class="reference"><a href="#cite_note-flashbackcheck-13"><span class="cite-bracket">&#91;</span>13<span class="cite-bracket">&#93;</span></a></sup> On April 12, 2015, the company issued a further update to remove the most common Flashback variants.<sup id="cite_ref-Apple_Java_2012-003_14-0" class="reference"><a href="#cite_note-Apple_Java_2012-003-14"><span class="cite-bracket">&#91;</span>14<span class="cite-bracket">&#93;</span></a></sup> The updated Java release was only made available for <a href="/wiki/Mac_OS_X_Lion" class="mw-redirect" title="Mac OS X Lion">Mac OS X Lion</a> and <a href="/wiki/Mac_OS_X_Snow_Leopard" title="Mac OS X Snow Leopard">Mac OS X Snow Leopard</a>; the removal utility was released for Intel versions of <a href="/wiki/Mac_OS_X_Leopard" title="Mac OS X Leopard">Mac OS X Leopard</a> in addition to the two newer operating systems.  Users of older operating systems were advised to disable Java.<sup id="cite_ref-Apple_About_Flashback_malware_12-1" class="reference"><a href="#cite_note-Apple_About_Flashback_malware-12"><span class="cite-bracket">&#91;</span>12<span class="cite-bracket">&#93;</span></a></sup> There are also some third party programs to detect and remove the Flashback trojan.<sup id="cite_ref-flashbackcheck_13-1" class="reference"><a href="#cite_note-flashbackcheck-13"><span class="cite-bracket">&#91;</span>13<span class="cite-bracket">&#93;</span></a></sup> Apple worked on a new process that would eventually lead to a release of a Java Runtime Environment (JRE) for Mac OS X at the same time it would be available for Windows, Linux, and Solaris users.<sup id="cite_ref-Mac_Security:_A_Myth?_15-0" class="reference"><a href="#cite_note-Mac_Security:_A_Myth?-15"><span class="cite-bracket">&#91;</span>15<span class="cite-bracket">&#93;</span></a></sup>  As of January 9, 2014, about 22,000 Macs were still infected with the Flashback trojan.<sup id="cite_ref-It&#39;s_alive!_Once-prolific_Flashback_trojan_still_infecting_22,000_Macs_16-0" class="reference"><a href="#cite_note-It&#39;s_alive!_Once-prolific_Flashback_trojan_still_infecting_22,000_Macs-16"><span class="cite-bracket">&#91;</span>16<span class="cite-bracket">&#93;</span></a></sup>
</p>
