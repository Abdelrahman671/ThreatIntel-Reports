<h3>Reports validate the importance of API threats</h3>
<h4>Akamai</h4>
<p><a href="/resources/state-of-the-internet/lurking-in-the-shadows" target="_self" title="Akamai's 2024 State of the Internet report on API threats">Akamai's 2024 State of the Internet report on API threats</a> not only brings attention to the various attacks directed at APIs, including traditional web attacks, but also highlights the proactive measures being taken to address these risks.</p>
<p>This report underscores the risks that API misuse poses in common problem areas such as posture and runtime challenges. A total of 29% of web attacks targeted APIs over the 12 months from January through December 2023, indicating that APIs are a focus area for cybercriminals. The report further mentions that business logic abuse is a critical concern, as it is difficult to detect abnormal API activity without having first established a baseline for API behavior.</p>
<h4>Gartner</h4>
<p>In May 2024, <a href="https://www.gartner.com/en/documents/5471595" target="_blank" title="Gartner" rel="nofollow noreferrer">Gartner</a> published a Market Guide for API Protection, which states that the average API breach results in at least 10 times more leaked data than the average security breach. However, high-profile API incidents are more common than ever.</p>
<h4>OWASP</h4>
<p><a href="https://owasp.org/www-project-top-ten/" target="_blank" title="The Open Web Application Security Project (OWASP)" rel="nofollow noreferrer">The Open Web Application Security Project (OWASP),</a> a nonprofit organization known for its list of top 10 security risks, released an updated version of API-specific risks, the OWASP API Security Top 10 in 2023, which not only recognizes the unique threats posed by APIs but also validates the importance of this issue in the industry.</p>
<h3>Common posture and runtime problems</h3>
<p>Akamai monitors and evaluates 20% to 30% of global internet traffic, which involves processing more than a trillion content requests daily from users in more than 200 countries and regions. Akamai collaborates with businesses worldwide to gather comprehensive data on API use and to conduct sophisticated behavioral analysis to detect security weaknesses and signs of API misuse. Based on Akamai's observations of API activity, two main issues typically arise: posture and runtime problems.</p>
<p>API implementation flaws can lead to posture problems, while runtime problems are urgent threats or behaviors, often of a critical nature, that demand immediate attention.</p>
<p>According to <a href="/resources/state-of-the-internet/lurking-in-the-shadows" target="_self" title="Akamai SOTI report">Akamai SOTI report</a>, the most common posture issues include:</p>
<ul>
<li>Shadow endpoints (i.e., unattended or outdated endpoints)</li>
<li>Unauthenticated resource access</li>
<li>Sensitive data in a URL</li>
<li>A permissive cross-origin resource sharing (CORS) policy</li>
<li>Excessive client errors</li>
</ul>
<p>Common runtime problems include:</p>
<ul>
<li>Unauthenticated resource access attempts</li>
<li>Abnormal JSON property</li>
<li>Path parameter fuzzing attempts</li>
<li>Impossible time travel</li>

    <p><a href="/glossary/what-is-api-security" target="_self" title="API security">API security</a> is not just a variation of “traditional” application security; it requires a distinct approach. Application security is concerned with protecting the entire application, while API security is specifically focused on safeguarding the APIs that enable data exchange in modern applications.</p>
<p>The most significant difference between an API and an application is the user base. APIs are built for software applications, while software applications are built for human interaction. This underscores the importance of having different security controls for each. <a href="https://thehackernews.com/2023/02/application-security-vs-api-security.html#:~:text=With%20that%20in%20mind%2C%20in,the%20need%20for%20developers%20and" target="_blank" title="Gartner" rel="nofollow noreferrer">Gartner</a> highlights this by stating that “traditional network and web protection tools are insufficient to defend against the diverse security threats that APIs face, including many of those outlined in the OWASP API Security Top 10.”</p>

    <p>Having a clear view of your environment and understanding normal activities and data distribution across APIs is the first crucial step in safeguarding against data breaches caused by API vulnerabilities. This involves securing all APIs with appropriate security controls, implementing automated measures to respond to and prevent attacks, and notifying the security operations team.</p>

DevOps security practices are designed to address security issues before they escalate, thereby reducing the time and resources needed for mitigation.</p>

<p>Implementing robust security practices is not just about protecting sensitive data; it's also about preventing potential damage to reputation. By prioritizing security, organizations demonstrate commitment to customers and the situation's urgency, helping them to retain customer loyalty and avoid reputational damage.</p>

    <p>API development is changing significantly due to the emergence of low-code and no-code platforms like APIDNA, Mendix, and OutSystems. These innovative solutions enable developers to build and integrate APIs with minimal traditional coding, thereby creating new opportunities for rapid development. With APIDNA, for example, developers can simply drag-and-drop prebuilt API components to create their own APIs.</p>
<h3>The API-first approach</h3>
<p>The API landscape is expanding, with the number of developers using APIs increasing by 61% in recent years. <a href="https://www.postman.com/state-of-api/" title="The 2023 State of the API Report" target="_blank" rel="nofollow noreferrer">The 2023 State of the API Report</a> by Postman indicates a growing trend of non-developers, including chief technology officers, managers, and directors, entering the API industry. This trend underscores the strong interest of leaders in adopting an API-first approach.</p>
<p>In the next 12 months, organizations will either increase their investment of time and resources into APIs or maintain the current level. Importantly, 53% of CEOs anticipate increased API investments in the coming year, providing a reassuring outlook for the industry's future.</p>
<p>The <a href="https://www.postman.com/state-of-api/apis-and-generative-ai/#apis-and-generative-ai" title="Postman report" target="_blank" rel="nofollow noreferrer">Postman report</a> found that only 51% of respondents in the government and defense sectors are using artificial intelligence (AI) tools, such as machine learning algorithms and natural language processing systems, making them the least likely industries to do so. A few respondents mentioned concerns regarding security risks and corporate restrictions on sharing data with third-party AI tools.</p>
<h3>Rapid evolution of APIs requires robust security</h3>
<p>The rapid evolution and expansion of APIs have created a new and urgent challenge: securing this prime opportunity for exploitation. APIs will continue to play a central role in companies' digital transformation. According to <a title="Akamai's 2024 State of the Internet report" href="/resources/state-of-the-internet/lurking-in-the-shadows" target="_self">Akamai's 2024 State of the Internet report</a>, 44.2% of web attacks that impact organizations in the commerce industry targeted APIs, followed by organizations in the business services industry at 31.8%.</p>
<p>This emphasis on <a href="/resources/infographic/commerce-execs-weigh-in-on-the-state-of-api-security" target="_self" title="commerce">commerce</a> is due to various factors, such as the complex nature of the industry’s ecosystem, heavy reliance on APIs, and large amounts of sensitive customer data. The escalating number of API attacks underscore the pressing need for robust API security in all companies.</p>

Implementing robust security in all stages of API development empowers businesses to navigate the evolving digital landscape with confidence and success.</p>
