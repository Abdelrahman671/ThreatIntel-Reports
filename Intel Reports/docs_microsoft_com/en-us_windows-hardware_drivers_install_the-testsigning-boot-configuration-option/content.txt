											<h2 id="ms--in-this-article" class="title is-6 margin-block-xs">In this article</h2>
										</nav><!-- <content> --><p>By default, Windows does not load test-signed kernel-mode drivers. To change this behavior and enable test-signed drivers to load, use the boot configuration data editor, BCDEdit.exe, to enable or disable TESTSIGNING, a boot configuration option. You must have Administrator rights to enable this option.</p>
<div class="NOTE">
<p>Note</p>
<p>Starting in Windows Vista, the kernel-mode code signing policy requires that all kernel-mode code have a digital signature to load on 64-bit versions of Windows. However, in most cases, an unsigned driver can be installed and loaded on 32-bit versions of Windows. For more information, see <a href="kernel-mode-code-signing-policy--windows-vista-and-later-" data-linktype="relative-path">Driver Signing Policy</a>.</p>
</div>
<h2 id="administrator-rights-required">Administrator rights required</h2>
<p>To use BCDEdit, you must be a member of the Administrators group on the system and run the command from an elevated command prompt. To open an elevated Command Prompt window, type <strong>cmd</strong> into the search box in the Windows taskbar, select and hold (or right-click) <strong>Command Prompt</strong> in the search results, and then select <strong>Run as administrator</strong>.</p>
<div class="WARNING">
<p>Warning</p>
<p>Administrative rights are required to use BCDEdit to modify boot configuration data. Changing some boot entry options by using <strong>BCDEdit /set</strong> could render your computer inoperable. As an alternative, use System Configuration utility (MSConfig.exe) to change boot settings.</p>
</div>
<h2 id="enable-or-disable-use-of-test-signed-code">Enable or disable use of test-signed code</h2>
<p>Run BCDEdit command lines to enable or disable the loading of test-signed code. For a change to take effect, whether enabling or disabling the option, you must restart the computer after changing the configuration.</p>
<p>To enable test-signed code, use the following BCDEdit command line:</p>
<pre><code class="lang-cmd">:: If this command results in "The value is protected by Secure Boot policy and cannot be modified or deleted"
:: Then reboot the PC, go into BIOS settings, and disable Secure Boot. BitLocker may also affect your ability to modify this setting.
Bcdedit.exe -set TESTSIGNING ON
</code></pre>
<div class="NOTE">
<p>Note</p>
<p>Starting in Windows 10, version 1507, if you have Memory Integrity / HVCI (Hypervisor Code Integrity) enabled, you must test-sign the binary using any self-created test cert. An unsigned binary is not supported.</p>
</div>
<p>To disable use of test-signed code, use the following BCDEdit command line:</p>
<pre><code class="lang-cmd">Bcdedit.exe -set TESTSIGNING OFF
</code></pre>
<p>The following figure shows the result of using the BCDEdit command line to enable test-signing.</p>
<p><img src="images/driver-signing-enable-vista-test-signing.png" alt="Screen shot of the results of using testsigning, a boot configuration option." data-linktype="relative-path"></p>
<h2 id="behavior-of-windows-when-loading-test-signed-code-is-enabled">Behavior of Windows when loading test-signed code is enabled</h2>
<p>When loading test-signed code is enabled, Windows does the following:</p>
<ul>
<li><p>Displays a watermark with the text "Test Mode" in the lower right corner of the desktop to remind users that the system has test-signing enabled.</p>
</li>
<li><p>The operating system loader and the kernel load drivers that are signed by any certificate. The certificate validation is not required to chain up to a trusted root certification authority. However, each driver image file must have a digital signature.</p>
</li>
</ul>
