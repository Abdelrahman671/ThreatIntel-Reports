<span id="hs_cos_wrapper_post_body" class="hs_cos_wrapper hs_cos_wrapper_meta_field hs_cos_wrapper_type_rich_text" style="" data-hs-cos-general-type="meta_field" data-hs-cos-type="rich_text"><p>&nbsp;Hworm/njRAT is a Remote Access Tool (RAT) that first appeared in 2013 in targeted attacks against the international energy industry, primarily&nbsp;in the Middle East. It was soon commoditized and is now part of a constantly evolving family of RATs that <a href="https://unit42.paloaltonetworks.com/unit42-houdinis-magic-reappearance/" rel="noopener" target="_blank">pop-up in various new formats</a>. Today we see this attack employed on a regular basis as part of widespread spam phishing campaigns - if successful, <em>Hworm</em> gives the attacker complete control of the victim’s system. Morphisec Labs recently observed a new version with a minor modification to its obfuscation technique.</p>
<!--more-->
<p><img src="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/blogpost-header-hworm.png?width=718&amp;height=338&amp;name=blogpost-header-hworm.png" width="718" height="338" loading="lazy" alt="blogpost-header-hworm" style="height: auto; max-width: 100%; width: 718px;" srcset="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/blogpost-header-hworm.png?width=359&amp;height=169&amp;name=blogpost-header-hworm.png 359w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/blogpost-header-hworm.png?width=718&amp;height=338&amp;name=blogpost-header-hworm.png 718w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/blogpost-header-hworm.png?width=1077&amp;height=507&amp;name=blogpost-header-hworm.png 1077w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/blogpost-header-hworm.png?width=1436&amp;height=676&amp;name=blogpost-header-hworm.png 1436w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/blogpost-header-hworm.png?width=1795&amp;height=845&amp;name=blogpost-header-hworm.png 1795w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/blogpost-header-hworm.png?width=2154&amp;height=1014&amp;name=blogpost-header-hworm.png 2154w" sizes="(max-width: 718px) 100vw, 718px"></p>
<h2>Technical Description:</h2>
<p>The attack uses the kind of fileless VBScript injector, leveraging DynamicWrapperX, that has been seen used in the wild by RATs such as <strong>HWorm</strong>, DarkComet, KilerRAT and others. We observed a new obfuscation level, as the distribution of this RAT is still changing and running. We will describe the injector stage and how it used to load Hworm/Houdini RAT.</p>
<h3>Stage 1</h3>
<p>The payload is a VBS file, which, in some cases, comes obfuscated or encoded with couple of layers.</p>
<p><img src="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-1-900px.png?width=900&amp;name=hworm-1-900px.png" alt="hworm-1-900px" style="width: 900px; margin: 0px;" width="900" srcset="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-1-900px.png?width=450&amp;name=hworm-1-900px.png 450w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-1-900px.png?width=900&amp;name=hworm-1-900px.png 900w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-1-900px.png?width=1350&amp;name=hworm-1-900px.png 1350w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-1-900px.png?width=1800&amp;name=hworm-1-900px.png 1800w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-1-900px.png?width=2250&amp;name=hworm-1-900px.png 2250w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-1-900px.png?width=2700&amp;name=hworm-1-900px.png 2700w" sizes="(max-width: 900px) 100vw, 900px"></p>
<h6 style="text-align: center;"><strong><em>Figure 1: Obfuscated VBScript</em></strong></h6>
<p>The next stage VBS file contains 3 chunks of base64 streams:</p>
<p><strong><em>DCOM_DATA</em></strong>:&nbsp; Holds a PE file, which is <a href="http://www.script-coding.com/dynwrapx_eng.html" rel="noopener nofollow" target="_blank">DynamicWrapperX</a>. It allows to call functions exported by DLL libraries, in particular Windows API functions, from JScript and VBScript.</p>
<p><strong><em>LOADER_DATA</em></strong>: Holds RunPE shellcode.</p>
<p><strong><em>FILE_DATA</em></strong>: Holds the shellcode that is injected to the host process. This will be discussed later.</p>
<p>As the script executes, it drops a copy of itself into %appdata%\Microsoft and gains persistence by editing the registry key:&nbsp; ‘<em>HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run</em>’.</p>
<ul>
<li>The script checks whether the current environment is 64bit or not. If it is, it will execute the script with a 32-bit version of wscript.exe (from SysWOW64).</li>
</ul>
<p><img src="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-2-1000px.png?width=1000&amp;name=hworm-2-1000px.png" alt="hworm-2-1000px" style="width: 1000px; margin-top: 0px; margin-bottom: 0px;" width="1000" srcset="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-2-1000px.png?width=500&amp;name=hworm-2-1000px.png 500w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-2-1000px.png?width=1000&amp;name=hworm-2-1000px.png 1000w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-2-1000px.png?width=1500&amp;name=hworm-2-1000px.png 1500w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-2-1000px.png?width=2000&amp;name=hworm-2-1000px.png 2000w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-2-1000px.png?width=2500&amp;name=hworm-2-1000px.png 2500w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-2-1000px.png?width=3000&amp;name=hworm-2-1000px.png 3000w" sizes="(max-width: 1000px) 100vw, 1000px"></p>
<h6 style="text-align: center;"><em><strong>Figure 2 : Execute with 32-bit version of wsscript.exe</strong></em></h6>
<ul>
<li>It determines the path for the host process that <strong><em>FILE_DATA </em></strong>will be injected into. There are two options – ‘wscript.exe’ or ‘msbuild.exe’. In our samples, the flag that decided which path to use was hardcoded (set to True), thus, always chose msbuild.exe.</li>
</ul>
<h6 style="text-align: center;"><em><strong><img src="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-3.png?width=906&amp;name=hworm-3.png" alt="hworm-3" style="width: 906px; margin-top: 0px; margin-bottom: 0px;" width="906" srcset="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-3.png?width=453&amp;name=hworm-3.png 453w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-3.png?width=906&amp;name=hworm-3.png 906w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-3.png?width=1359&amp;name=hworm-3.png 1359w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-3.png?width=1812&amp;name=hworm-3.png 1812w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-3.png?width=2265&amp;name=hworm-3.png 2265w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-3.png?width=2718&amp;name=hworm-3.png 2718w" sizes="(max-width: 906px) 100vw, 906px"></strong></em><em><strong>Figure 3 choose host process</strong></em></h6>
<ul>
<li><strong><em>DCOM_DATA</em></strong> is decoded and dropped to %temp% directory under the name “HOUDINI.BIN” and registered with regsvr32.exe. It creates an object instance named “DynamicWrapperX” and registers two DLL functions: “CallWindowProcW” from “User32.dll” and “VirtualAlloc” from “Kernel32.dll”. It uses VirtualAlloc to allocate memory for the RunPE shellcode and <strong><em>FILE_DATA </em></strong>shellcode, then, invokes it using CallWindowProcW.</li>
</ul>
<p><img src="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-4.png?width=742&amp;name=hworm-4.png" alt="hworm-4" style="width: 742px; margin-top: 0px; margin-bottom: 0px;" width="742" srcset="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-4.png?width=371&amp;name=hworm-4.png 371w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-4.png?width=742&amp;name=hworm-4.png 742w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-4.png?width=1113&amp;name=hworm-4.png 1113w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-4.png?width=1484&amp;name=hworm-4.png 1484w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-4.png?width=1855&amp;name=hworm-4.png 1855w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-4.png?width=2226&amp;name=hworm-4.png 2226w" sizes="(max-width: 742px) 100vw, 742px"></p>
<h6 style="text-align: center;"><em><strong>Figure 4 invoke injection procedure</strong></em></h6>
<h3>Stage 2</h3>
<p>The second stage is basically <strong><em>FILE_DATA </em></strong>which is injected to ‘msbuild.exe’ using<strong><em> LOADER_DATA </em></strong>(RunPE). <strong><em>FILE_DATA </em></strong>is base64 encoded – trying to decode and look at it does not yield information, as there is another layer of encoding.</p>
<p><img src="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-5.png?width=519&amp;name=hworm-5.png" alt="hworm-5" style="width: 519px; margin: 0px auto; display: block;" width="519" srcset="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-5.png?width=260&amp;name=hworm-5.png 260w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-5.png?width=519&amp;name=hworm-5.png 519w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-5.png?width=779&amp;name=hworm-5.png 779w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-5.png?width=1038&amp;name=hworm-5.png 1038w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-5.png?width=1298&amp;name=hworm-5.png 1298w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-5.png?width=1557&amp;name=hworm-5.png 1557w" sizes="(max-width: 519px) 100vw, 519px"></p>
<h6 style="text-align: center;"><em><strong>Figure 5 FILE_DATA base64 decoded</strong></em></h6>
<p><strong><em>LOADER_DATA </em></strong>(RunPE shellcode) is responsible for the second decoding routine.</p>
<p><img src="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-6.png?width=512&amp;name=hworm-6.png" alt="hworm-6" style="width: 512px; margin: 0px auto; display: block;" width="512" srcset="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-6.png?width=256&amp;name=hworm-6.png 256w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-6.png?width=512&amp;name=hworm-6.png 512w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-6.png?width=768&amp;name=hworm-6.png 768w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-6.png?width=1024&amp;name=hworm-6.png 1024w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-6.png?width=1280&amp;name=hworm-6.png 1280w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-6.png?width=1536&amp;name=hworm-6.png 1536w" sizes="(max-width: 512px) 100vw, 512px"></p>
<h6 style="text-align: center;"><em><strong>Figure 6 After LOADER_DATA decoding</strong></em></h6>
<p>Eventually, we see&nbsp;<strong><em>FILE_DATA</em></strong> is a portable executable, written in Dot Net. Looking at the decompiled source code we can see Hworm (njRAT) configuration.</p>
<p><img src="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-7.png?width=559&amp;name=hworm-7.png" alt="hworm-7" style="width: 559px; margin: 0px auto; display: block;" width="559" srcset="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-7.png?width=280&amp;name=hworm-7.png 280w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-7.png?width=559&amp;name=hworm-7.png 559w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-7.png?width=839&amp;name=hworm-7.png 839w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-7.png?width=1118&amp;name=hworm-7.png 1118w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-7.png?width=1398&amp;name=hworm-7.png 1398w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/hworm-7.png?width=1677&amp;name=hworm-7.png 1677w" sizes="(max-width: 559px) 100vw, 559px"></p>
<h6 style="text-align: center;"><em><strong>Figure 7</strong></em></h6>
<ul>
<li>“svchost.exe” - Trojan exe.</li>
<li>“AppData” - Installation path.</li>
<li>“183d24d29354086f9c19c24368929a8c” - Mutex name.</li>

</ul>
<h2>Conclusion</h2>
<p>Morphisec protects against Hworm and similar attacks. By applying Moving target defense technology, we deterministically prevent this attack without relation to signatures / patterns or obfuscation techniques.&nbsp;</p>
