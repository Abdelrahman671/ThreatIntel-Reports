										</nav><!-- <content> --><h2 id="summary">Summary</h2>
<p>This article describes how to disable Microsoft Visual Basic for Applications during the deployment of the versions of Microsoft Office that are listed in the "Applies To" section.</p>
<h2 id="more-information">More Information</h2>
<p>Visual Basic for Applications, although not a security risk in itself, can be used by other users to compromise security. However, you can install Office without Visual Basic for Applications support.</p>
<p>The following methods can be used to remove Visual Basic for Applications support in Office.</p>
<p>Note The Custom Maintenance Wizard and the Custom Installation Wizard can be used only with the Enterprise edition of Microsoft Office.</p>
<h3 id="method-1-custom-installation-wizard">Method 1: Custom Installation Wizard</h3>
<div class="NOTE">
<p>Note</p>
<p>This method can be used only with the Enterprise edition of Microsoft Office.</p>
</div>
<p>The Custom Installation Wizard is included with the Microsoft Office Resource Kit (ORK). The wizard can be used to create a custom TRANSFORM file that is used with Setup.exe to customize the installation of Office features during Setup. To create a TRANSFORM file that removes Visual Basic for Applications support during installation, follow these steps.</p>
<div class="NOTE">
<p>Note</p>
<p>To create the TRANSFORM file , you must install the ORK. You can install the ORK from the ORK folder on the CD for an Enterprise edition of Office.</p>
</div>
<ol>
<li><p>Start the Custom Installation Wizard, and then click <strong>Next</strong>.</p>
</li>
<li><p>In the <strong>Name and path of MSI file to open</strong> box, type the full path and file name of your .msi file, and then click Next two times.</p>
<p><strong>Note</strong> An .msi file is included with the Office installation CD. For example, type D:\PROPLUS.MSI.</p>
</li>
<li><p>In the <strong>Name and path of MST file</strong> box, type the path and file name with which you want to save your custom Setup file, and then click Next three times.</p>
</li>
<li><p>In the Set Feature Installation States page of the Custom Installation Wizard, in the <strong>For each of the following Microsoft Office features, click to select the default installation state</strong> list, expand Office Shared Features.</p>
</li>
<li><p>Click <strong>Visual Basic for Applications</strong>, and then click Not Available, Hidden, Locked.</p>
<p>You receive the following message:</p>
<p><strong>You have chosen not to install Visual Basic for Applications. Microsoft Access requires this component and will not be installed if you continue. Additional features in Microsoft Office, including some wizards and templates, will not work properly.</strong></p>
<p><strong>Do you want to install Microsoft Office without Visual Basic for Applications?</strong></p>
</li>
<li><p>Click <strong>Yes</strong>, and then click <strong>Finish</strong>. After the transform file is created, click <strong>Exit</strong> to exit the Custom Installation Wizard.</p>
</li>
</ol>
<p>For more information about how to cutomize Office installations and about how to use the Custom Installation Wizard, go to the following Microsoft websites:</p>
<p>Office 2016</p>
<p><a href="https://www.microsoft.com/download/details.aspx?id=49030" data-linktype="external">(Preview) Office 2016 Administrative Template files (ADMX/ADML) and Office Customization Tool</a></p>

<p>Office 2010</p>
<p><a href="https://technet.microsoft.com/library/cc179132%28v=office.14%29.aspx" data-linktype="external">Customize Office 2010</a></p>
<p>The 2007 Office system</p>

<h3 id="method-2-custom-maintenance-wizard">Method 2: Custom Maintenance Wizard</h3>
<div class="NOTE">
<p>Note</p>
<p>This method can be used only with the Enterprise edition of Microsoft Office.</p>
</div>
<p>The Custom Maintenance Wizard is included with the Microsoft Office Resource Kit (ORK). The wizard is used to create a Custom Maintenance Wizard customization file (CMW file) that can be used to update an existing Office installation. To create a CMW file to remove Visual Basic for Applications support, follow these steps.</p>
<div class="NOTE">
<p>Note</p>
<p>To create the CMW file , you must install the ORK. You can install the ORK from the ORK folder on the CD for an Enterprise edition of Office.</p>
</div>
<ol>
<li><p>Start the Custom Maintenance Wizard, and then click Next.</p>
</li>
<li><p>In the <strong>Name and path of MSI file to open</strong> box, type the full path and file name of your .msi file, and then click Next two times.</p>
<p><strong>Note</strong> An .msi file is included with the Office installation CD. For example, for Microsoft Office XP Professional Plus, you would type     D:\PROPLUS.MSI.</p>
</li>
<li><p>Click Next two times, and then on the Set Feature Installation States page, in the <strong>For each of the following Microsoft Office features, click to select the desired installation state</strong> list, expand Office Shared Features.</p>
</li>
<li><p>Click Visual Basic for Applications, and then click Not Available, Hidden, Locked.</p>
<p>You receive the following message:</p>
<p><strong>You have chosen not to install Visual Basic for Applications. Microsoft Access requires this component and will not be installed if you continue. Additional features in Microsoft Office, including some wizards and templates, will not work properly.</strong></p>
<p><strong>Do you want to install Microsoft Office without Visual Basic for Applications?</strong></p>
</li>
<li><p>Click Yes, and then click Finish. After the Custom Maintenance Wizard configuration file (CMW file) is created, click Exit to exit the Custom Maintenance Wizard.</p>
</li>
</ol>
<h3 id="method-3-system-policy">Method 3: System Policy</h3>
<p>System Policy Editor is included with the Microsoft Office Resource Kit (ORK). System Policy Editor can be used to turn off Visual Basic for Applications support for Office programs.</p>
<p>Turning on the <strong>Disable VBA for Office applications</strong> policy sets the VBAOFF DWORD value to 1 in the following registry subkey:</p>
<p>Office 2016</p>
<p><strong>HKEY_LOCAL_MACHINE\Software\Policies\Microsoft\Office\16.0\Common</strong></p>
<p>Office 2013</p>
<p><strong>HKEY_LOCAL_MACHINE\Software\Policies\Microsoft\Office\15.0\Common</strong></p>
<p>Office 2010</p>
<p><strong>HKEY_LOCAL_MACHINE\Software\Policies\Microsoft\Office\14.0\Common</strong></p>
<p>The 2007 Office system</p>
<p><strong>HKEY_LOCAL_MACHINE\Software\Policies\Microsoft\Office\12.0\Common</strong></p>
<p>Office 2003</p>
<p><strong>HKEY_LOCAL_MACHINE\Software\Policies\Microsoft\Office\11.0\Common</strong></p>
<p>Office XP</p>
<p><strong>HKEY_LOCAL_MACHINE\Software\Policies\Microsoft\Office\10.0\Common</strong></p>
<p>This registry setting prevents Microsoft Excel, Microsoft FrontPage, Microsoft Outlook, Microsoft PowerPoint, Microsoft Publisher, and Microsoft Word from using Visual Basic for Applications.</p>
<h3 id="method-4-stand-alone-office-installation">Method 4: Stand-alone Office installation</h3>
<p>During or after an Office installation, you can specify the features that you want installed. To do this, follow these steps:</p>
<ol>
<li>In Office Setup, on the <strong>Choose installation options for all Office applications and tools</strong> page, in the <strong>Features to install</strong> list, expand Office Shared Features.</li>
<li>Click <strong>Visual Basic for Applications</strong>, and then click Not Available.</li>
<li>Continue the Office installation or update.</li>
</ol>
<p>Microsoft Access requires Visual Basic for Applications. To enable the installation of Access and disable Visual Basic for Applications in all other Office programs, install a stand-alone version of Microsoft Access, and then install Office as a custom installation that has Visual Basic for Applications disabled.</p>
<h3 id="method-5-manually-add-the-vbaoff-registry-subkey">Method 5: Manually add the VBAOff registry subkey</h3>
<p>The VBAOff registry subkey can be added manually to disable Visual Basic for Applications functionality for all users on a computer after Office is installed. To add the VBAOff registry key, follow these steps:</p>
<ol>
<li><p>Exit your version of all Office programs.</p>
</li>
<li><p>Click <strong>Start</strong>, click <strong>Run</strong>, type regedit, and then click <strong>OK</strong>.</p>
</li>
<li><p>Locate and then click to select one of the following registry keys, depending on the version of the product that you are using:</p>
<ul>
<li><p>For Office 2016, locate and then click to select <strong>HKEY_LOCAL_MACHINE\Software\Microsoft\Office\16.0\Common</strong>.</p>
</li>
<li><p>For Office 2013, locate and then click to select <strong>HKEY_LOCAL_MACHINE\Software\Microsoft\Office\15.0\Common</strong>.</p>
</li>
<li><p>For Office 2010, locate and then click to select <strong>HKEY_LOCAL_MACHINE\Software\Microsoft\Office\14.0\Common</strong>.</p>
</li>
<li><p>For the 2007 Office system, locate and then click to select <strong>HKEY_LOCAL_MACHINE\Software\Microsoft\Office\12.0\Common</strong>.</p>
</li>
<li><p>For Office 2003, locate and then click to select <strong>HKEY_LOCAL_MACHINE\Software\Microsoft\Office\11.0\Common</strong>.</p>
</li>
<li><p>For Office XP, locate and then click to select <strong>HKEY_LOCAL_MACHINE\Software\Microsoft\Office\10.0\Common</strong>.</p>
</li>
</ul>
</li>
<li><p>After you select the key that is specified in step 3, point to <strong>New</strong> on the <strong>Edit</strong> menu, and then click <strong>DWORD Value</strong>.</p>
</li>
<li><p>Type VBAOff, and then press ENTER.</p>
</li>
<li><p>Right-click <strong>VBAOff</strong>, and then click <strong>Modify</strong>.</p>
</li>
<li><p>In the <strong>Value data</strong> box, type 1, and then click <strong>OK</strong>.</p>
</li>
<li><p>On the <strong>File</strong> menu, click <strong>Exit</strong> to quit Registry Editor.</p>
</li>
</ol>
<p><strong>Note</strong> You can also disable Visual Basic for Applications functionality for the current user only. You can do this by selecting the <strong>HKEY_CURRENT_USER</strong> path for the appropriate product that is shown in step 3.</p>
<p>For more information, view the article in the Microsoft Knowledge Base:</p>
<p><a href="https://support.microsoft.com/help/287567" data-linktype="external">287567</a> Considerations for disabling VBA in Office XP</p>
