	<h1 class="article-details__title">AutoIt-Compiled Worm Sends Fileless BLADABINDI/njRAT</h1>
	<p class="article-details__description">BLADABINDI’s customizability and seeming availability in the underground make it a prevalent threat. Last week, we came across a worm (Worm.Win32.BLADABINDI.AA) that propagates through removable drives and installs a fileless version of BLADABINDI.
</p>
	<p class="article-details__author-by">By: Carl Maverick Pascual, Michael Jhon Ofiaza, Patrick Angelo Roderno
		
			<time class="article-details__date">November 27, 2018</time>

	</p>

		<div>
			<p><span class="rte-red-bullet"></span>BLADABINDI, also known as <a href="/en_us/research/15/a/new-rats-emerge-from-leaked-njw0rm-source-code.html" style="background-color: rgb(255,255,255);">njRAT/Njw0rm</a><span style="">, is a remote access tool (RAT) with a myriad of backdoor capabilities — from keylogging to carrying out </span><a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/definition/distributed-denial-of-service" style="background-color: rgb(255,255,255);" target="_self">distributed denial of service</a><span style=""> (DDoS) — and has been rehashed and </span><a href="https://www.zdnet.com/article/njrat-secures-top-spot-as-most-active-malware-on-networks-in-2017/" style="background-color: rgb(255,255,255);" target="_blank">reused</a><span style=""> in various </span><a href="/en_us/research/17/j/from-cybercrime-to-cyberpropaganda.html" style="background-color: rgb(255,255,255);">cyberespionage</a><span style=""> </span><a href="/en_us/research/17/l/cyberespionage-campaign-sphinx-goes-mobile-anubisspy.html" style="background-color: rgb(255,255,255);">campaigns</a><span style=""> since it first emerged. Indeed, BLADABINDI’s </span><a href="https://www.darkreading.com/attacks-breaches/botnets-serving-up-more-multipurpose-malware-/d/d-id/1332709" style="background-color: rgb(255,255,255);" target="_blank">customizability</a><span style=""> and seeming </span><a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/news/cybercrime-and-digital-threats/the-middle-eastern-and-north-african-underground-where-culture-and-cybercrime-meet" style="background-color: rgb(255,255,255);">availability</a><span style=""> in the underground make it a </span><a href="https://www.cbronline.com/news/kaspersky-botnet-activity" style="background-color: rgb(255,255,255);" target="_blank">prevalent</a><span style=""> threat. Case in point: Last week, we came across a worm (detected by Trend Micro as Worm.Win32.BLADABINDI.AA) that propagates through removable drives and installs a fileless version of the BLADABINDI backdoor.</span></p>
<p>While it is still unknown how the malicious file actually arrives in the infected system, its propagation routine suggests that it enters systems through removable drives. Apart from being a flexible and easy-to-use scripting language, BLADABINDI’s use of <a href="/en_us/research/13/e/autoit-used-to-spread-malware-and-toolsets.html" style="background-color: rgb(255,255,255);">AutoIt</a><span style=""> is notable. It uses </span><a href="/en_us/research/17/f/information-stealer-found-hitting-israeli-hospitals.html" style="background-color: rgb(255,255,255);">AutoIt</a><span style=""> (the </span><i style="">FileInstall</i><span style=""> command) to compile the payload and the main script into a single executable, which can make the payload — the backdoor — difficult to detect.</span><br />
</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/11/bladabindi-njrat-1.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/11/bladabindi-njrat-1.png"/></a></p>
<p style="text-align: center;">Figure 1: Screenshot showing a common indicator of a compiled AutoIt script (highlighted)</p>
<p><b><span class="body-subhead-title">Technical analysis</span></b></p>
<p>We used an AutoIt script decompiler to break down the executable’s AutoIt script and found that the script’s main function first deletes any file named <i>Tr.exe</i> from the system’s %TEMP% directory so it can install its own version of <i>Tr.exe</i> on it. The dropped file is executed after terminating any process with the same name. It will also drop a copy of itself in the same directory. For persistence, it adds a shortcut for the file at the <i>%STARTUP%</i> directory.</p>
<p>For propagation, it installs a hidden copy of itself on any removable drive found on the infected system. It will also drop a shortcut file (.LNK) and move all original files of the removable drive from its root to a created folder named <i>sss</i>.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/11/bladabindi-njrat-2.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/11/bladabindi-njrat-2.png"/></a></p>
<p style="text-align: center;">Figure 2: Code snapshot showing the decompiled script</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/11/bladabindi-njrat-3.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/11/bladabindi-njrat-3.png"/></a></p>
<p style="text-align: center;">Figure 3: Code snapshot showing how the AutoIt’s FileInstall command is used to bundle an AutoIt script with any file then load the file during the script’s execution</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/11/bladabindi-njrat-4-1.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/11/bladabindi-njrat-4-1.png"/></a></p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/11/bladabindi-njrat-4-2.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/11/bladabindi-njrat-4-2.png"/></a></p>
<p style="text-align: center;">Figure 4: Code snapshots showing how the shortcut is added (top) and how it propagates through removable drives (bottom)</p>
<p>The dropped <i>Tr.exe</i> is actually another AutoIt-compiled executable script (Trojan.Win32.BLADABINDI.AA). Decompiling it reveals that it contains a base-64 encoded executable, which it will write in a registry value named <i>Valuex</i> in the registry HKEY_CURRENT_USER\Software.</p>
<p>It will also create another value for persistence. It will use an auto-run registry (HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run) named <i>AdobeMX</i> that will execute PowerShell to load the encoded executable via reflective loading (loading an executable from memory rather than from the system’s disks).</p>
<p>Since the executable is loaded directly from the registry to the memory of PowerShell, we were able to dump the specific address where the malicious executable is located. And we found out that it is .NET-compiled, which uses a commercial code protector software for obfuscation.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/11/bladabindi-njrat-5.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/11/bladabindi-njrat-5.png"/></a></p>
<p style="text-align: center;">Figure 5: Screenshots showing PowerShell loading the encoded executable</p>
<p><b><span class="body-subhead-title">BLADABINDI/njRAT payload</span></b></p>
<p>The variant of the BLADABINDI backdoor uses <i>water-boom[.]duckdns[.]org</i> as its command-and-control (C&amp;C) server, on port 1177. As with other and <a href="https://www.fortinet.com/blog/threat-research/bladabindi-remains-a-constant-threat-by-using-dynamic-dns-services.html" target="_blank">previous</a> iterations of BLADABINDI, this fileless version’s C&amp;C-related URL uses dynamic domain name system (DNS). This could potentially allow the attackers to hide the server’s actual IP address or change/update it as necessary.</p>
<p>All files downloaded from C&amp;C server are stored in the %TEMP% folder as <i>Trojan.exe</i>. It uses the string 5cd8f17f4086744065eb0992a09e05a2 as its mutex as well as its registry hive in the affected machine. It uses the value <i>tcpClient_0</i> as its HTTP server, where it will receive all stolen information from the infected machine. However, since the value was set to null, all stolen information will be sent to the same C&amp;C server<i>.</i></p>
<p>When the backdoor runs, it creates a firewall policy that adds PowerShell’s process to the list of allowed programs in the system. BLADABINDI’s backdoor capabilities are shown in Figure 7, which includes keylogging, retrieving and executing files, and stealing credentials from web browsers.</p>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/11/bladabindi-njrat-6-1.png"/></p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/11/bladabindi-njrat-6-2.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/11/bladabindi-njrat-6-2.png"/></a></p>
<p style="text-align: center;">Figure 6: Code snapshots showing the configurations of the BLADABINDI variant (top) and how it creates a firewall policy to add PowerShell to the list of programs allowed to run (bottom)</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/11/bladabindi-njrat-7.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2018/11/bladabindi-njrat-7.png"/></a>Figure 7: The backdoor capabilities of the BLADABINDI variant</p>
<p><b><span class="body-subhead-title">Best practices and Trend Micro solutions</span></b></p>
<p>The worm’s payload, propagation, and <a href="/en_us/research/17/h/usb-malware-implicated-fileless-attacks.html">technique</a> of <a href="https://www.trendmicro.com/vinfo/tmr/?/us/security/news/security-technology/security-101-the-rise-of-fileless-threats-that-abuse-powershell">filelessly</a> delivering the backdoor in the affected system make it a significant threat. Users and especially businesses that still use removable media in the workplace should practice security hygiene. Restrict and secure the use of removable media or USB functionality, or tools like PowerShell (particularly on systems with sensitive data), and proactively monitor the gateway, endpoints, networks, and servers for anomalous behaviors and indicators such as C&amp;C communication and information theft.</p>
<p>Users and businesses can also use Trend Micro endpoint solutions such as <a href="/en_us/forHome/products.html">Trend Micro™ Security</a>, <a href="/en_us/business/products/user-protection/sps.html">Smart Protection Suites</a>, and <a href="/en_us/small-business.html">Worry-Free Business Security</a> all include behavior monitoring to detect fileless malware attacks. This helps organizations look out for malicious behavior that can block the malware before the behavior is executed or performed. OfficeScan can also include a <a href="https://success.trendmicro.com/solution/1056026" target="_blank">device control</a> feature that can prevent USB and optical drives from being accessed, preventing an attack similar to the one discussed in this post. Trend Micro™ OfficeScan™ with XGen™ endpoint security infuses high-fidelity machine learning with other detection technologies and global threat intelligence for comprehensive protection against advanced malware.</p>
