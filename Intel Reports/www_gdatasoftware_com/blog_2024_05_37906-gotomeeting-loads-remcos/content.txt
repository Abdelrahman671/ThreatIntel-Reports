id="blogpage-content-container"> <div class="row"> <div class="column fce small-12"> <p class="intro-text">Legitimate applications can unwittingly become conduits for malware execution. This is also the case for recent malware loaders which abuse GoTo Meeting, an online meeting software, to deploy Remcos RAT. Their lures include porn downloads, software setup files as well as tax forms with file names in Russian and English language.</p> </div> </div> <div class="row hide-for-large"> <div class="column small-12"> <div class="pagenews-progress"> <div class="reading-time"> <span class="eta"></span> <span class="word-count"></span> </div> <div data-sticky-container id="mobile-progress-container" class="hide-on-large"> <progress value="0" class="sticky" data-sticky data-margin-top="0" data-sticky-on="small" data-margin-bottom="0" data-top-anchor="mobile-progress-container" data-btm-anchor="site-footer:top"></progress> </div> </div> <nav class="mobile-menu"> <ul class="pagenews-menu"> <li> <a href="#c232793" data-scroll-link> <span>LNK Execution Chain</span> </a> </li> <li> <a href="#c232797" data-scroll-link> <span>Rust Loader and Shellcode</span> </a> </li> <li> <a href="#c232800" data-scroll-link> <span>JS Infection Chain</span> </a> </li> <li> <a href="#c232802" data-scroll-link> <span>Indicators of Compromise</span> </a> </li> <li> <a href="#pagenews-related-articles"> <span>Related articles</span> </a> </li> </ul> </nav> </div> </div> <div class="row"> <div data-magellan-target="c232793" class="small-12 column"><div class="row"> <div id="c232793" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><header class=" "><h2 class=" "> LNK Execution Chain </h2></header><p>The following image demonstrates the LNK execution chain, taking myrecentfiles23.zip<sup>[1]</sup> as an example. The start of the chain is myrecentfiles.lnk<sup>[6]</sup>. The other files are hidden from the user. The windows shortcut myrecentfiles.lnk has a PDF icon.&nbsp;</p></div><div class="ce-gallery row small-up-1 medium-up-2 large-up-2"><div class="column column-block"><figure class="image"><a href="/fileadmin/web/general/images/blog/2024/04/GDATA_GotoMeeting_Remcos_execution_chain.png" class="fancybox"><img class="image-embed-item thumbnail" src="/fileadmin/web/general/images/blog/2024/04/GDATA_GotoMeeting_Remcos_execution_chain.png" width="615" height="360" loading="lazy" alt="" /></a><figcaption class="image-caption"> Execution chain of Remcos RAT abusing a legitimate GoTo Meeting executable </figcaption></figure></div><div class="column column-block"><figure class="image"><a href="/fileadmin/web/general/images/blog/2024/04/GDATA_GotoMeeting_myrecentfileszip_folder_contents.png" class="fancybox"><img class="image-embed-item thumbnail" src="/fileadmin/_processed_/6/a/GDATA_GotoMeeting_myrecentfileszip_folder_contents_e516b2a2d4.png" width="711" height="189" loading="lazy" alt="" /></a><figcaption class="image-caption"> Contents of myrecentfiles23.zip. The shortcut myrecentfiles.lnk is the only non-hidden item and has a PDF icon. </figcaption></figure></div></div></div></div> <div id="c232795" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><p>When myrecentfiles.lnk<sup>[6]</sup> is double-clicked, it will open a harmless PDF file named MLD.pdf<sup>[5]</sup> and execute winsys.odt<sup>[3]</sup> at the same time. The winsys.odt is a validly signed GoTo Meeting PE32 file and not malicious.</p></div><div class="ce-gallery row small-up-1 medium-up-2 large-up-2"><div class="column column-block"><figure class="image"><a href="/fileadmin/web/general/images/blog/2024/04/GDATA_GotoMeeting_winsys_exe_properties.png" class="fancybox"><img class="image-embed-item " src="/fileadmin/web/general/images/blog/2024/04/GDATA_GotoMeeting_winsys_exe_properties.png" width="532" height="342" loading="lazy" alt="" /></a><figcaption class="image-caption"> After appending an &quot;.exe&quot; extension, we can see that winsys.odt is a validly signed executable. </figcaption></figure></div><div class="column column-block"><figure class="image"><a href="/fileadmin/web/general/images/blog/2024/04/GDATA_GotoMeeting_winsys_odt_after_renaming.png" class="fancybox"><img class="image-embed-item " src="/fileadmin/_processed_/8/c/GDATA_GotoMeeting_winsys_odt_after_renaming_4751177224.png" width="715" height="177" loading="lazy" alt="" /></a><figcaption class="image-caption"> The file winsys.odt shows a GoTo meeting icon after changing the extension to &quot;.exe&quot; </figcaption></figure></div></div></div></div> <div id="c232796" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><p>The MLD.pdf<sup>[5] </sup>is there to lower suspicion that the system has been infected because the user will expect a PDF when they click on a PDF icon. It is questionable how well that works with an outdated tax form from 2022, though.&nbsp;</p></div><div class="ce-gallery row small-up-1 medium-up-1 large-up-1"><div class="column column-block"><figure class="image"><a href="/fileadmin/web/general/images/blog/2024/04/GDATA_GotoMeeting_mld_pdf.png" class="fancybox"><img class="image-embed-item thumbnail" src="/fileadmin/web/general/images/blog/2024/04/GDATA_GotoMeeting_mld_pdf.png" width="1102" height="598" loading="lazy" alt="" /></a><figcaption class="image-caption"> MLD.pdf is a decoy document with U.S. 2022 tax form </figcaption></figure></div></div></div></div> <div id="c232815" class="fce column large-12 small-12 frame-layout-0 " ><p>The GoTo Meeting executable loads a DLL file from the same folder which is named g2m.dll<sup>[4]</sup>. Under normal circumstances GoTo Meeting would load a clean g2m.dll file, but this infection chain uses DLL sideloading to execute the malware DLL instead.</p></div> </div></div> <div data-magellan-target="c232797" class="small-12 column"><div class="row"> <div id="c232797" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><header class=" "><h2 class=" "> Rust Loader and Shellcode </h2></header><p class="Paragraph SCXW3569969 BCX8">The malware DLL is written in Rust which is discernible by the statically linked Rust libraries. It exports 20 functions for GoTo Meeting but these all lead to the very same address with an empty function. Because the functions are not implemented, the GoTo Meeting executable cannot work, but the DLLMain of g2m.dll<sup>[4] </sup>will still run and allow the malware loader to execute.</p><p class="Paragraph SCXW3569969 BCX8">The Rust code reads the data.bin<sup>[2]</sup> file, allocates RWX memory for it and creates a new thread to execute it (see right image below).</p></div><div class="ce-gallery row small-up-1 medium-up-2 large-up-2"><div class="column column-block"><figure class="image"><a href="/fileadmin/web/general/images/blog/2024/04/GDATA_GotoMeeting_exports_gotomeeting.png" class="fancybox"><img class="image-embed-item " src="/fileadmin/_processed_/5/1/GDATA_GotoMeeting_exports_gotomeeting_711869213e.png" width="715" height="379" loading="lazy" alt="" /></a><figcaption class="image-caption"> The export table entries point to the same address of an empty function (click to enlarge) </figcaption></figure></div><div class="column column-block"><figure class="image"><a href="/fileadmin/_processed_/f/9/GDATA_GotoMeeting_databin_loading_0f6ed4bfa5.png" class="fancybox"><img class="image-embed-item " src="/fileadmin/_processed_/f/9/GDATA_GotoMeeting_databin_loading_43475a287e.png" width="715" height="461" loading="lazy" alt="" /></a><figcaption class="image-caption"> Function that loads data.bin shellcode (click to enlarge) </figcaption></figure></div></div></div></div> <div id="c232799" class="fce column large-12 small-12 frame-layout-0 " ><p>The file data.bin<sup>[2]</sup> does not only consist of shellcode but also contains the encrypted data for the payload. The shellcode takes care of decrypting and executing the payload.</p><p><a href="https://github.com/mandiant/speakeasy">Speakeasy </a>unpacks the payload with the following command line.</p><p>&nbsp;</p><pre><code class="language-bash">speakeasy.exe -r -a x86 -t data.bin -q 3000 -d dump.zip -o report.txt </code></pre><p>&nbsp;</p><p>The resulting payload is a Remcos RAT executable.&nbsp;</p></div> </div></div> <div data-magellan-target="c232800" class="small-12 column"><div class="row"> <div id="c232800" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><header class=" "><h2 class=" "> JS Infection Chain </h2></header><p class="Paragraph SCXW249514006 BCX8">After analysing the LNK execution chain, I started looking for more cases like these via related files on VirusTotal. I discovered initial infectors in various languages and for several target groups just by looking for archives that contain either the same g2m.dll<sup>[4] </sup>or the same decoy PDF<sup>[5]</sup> by hash.&nbsp;</p><p class="Paragraph SCXW249514006 BCX8">Among others I found a fake LeonardoAI2 Setup<sup>[14]</sup>, a fake OnlyFans Livestreams setup<sup>[13]</sup>, fake tax documents<sup>[16]</sup>, fake tax organizers<sup>[17]</sup> and a Russian ZIP file named Заявка_на_Геоприборы.rar.zip<sup>[15]</sup> which translates to “application for geodevices”.&nbsp;</p><p class="Paragraph SCXW249514006 BCX8">Some feature a slightly different execution chain than the previous one, e.g., the JScript file Teen Girl Leak Porn 10.js<sup>[7]</sup> whose name indicates that it targets porn consumers. The image below visualizes the infection chain for the this JScript file<sup>[7] </sup>and serves as an example for a different mode of execution.</p></div><div class="ce-gallery row small-up-1 medium-up-1 large-up-1"><div class="column column-block"><figure class="image"><img class="image-embed-item " src="/fileadmin/web/general/images/blog/2024/04/GDATA_GotoMeeting_Remcos_JS_infection_chain.png" width="614" height="425" loading="lazy" alt="" /><figcaption class="image-caption"> Infection chain that starts with a JScript file </figcaption></figure></div></div></div></div> <div id="c232801" class="fce column large-12 small-12 frame-layout-0 " ><div class="ce-textpic ce-center ce-below"><div class="ce-bodytext"><p class="Paragraph SCXW40333501 BCX8">The JScript file starts the chain. It downloads a PowerShell script<sup>[9] </sup>from rentry.co<sup>[8]</sup>, which is a markdown paste service. The PowerShell script downloads a file2.zip<sup>[19]</sup> and is responsible for creating persistence via a RunBatchFile.lnk<sup>[20]</sup> in the startup folder. This windows shortcut executes run.bat<sup>[11] </sup>from the archive which in turn will start the same execution chain as before.&nbsp;</p></div><div class="ce-gallery row small-up-1 medium-up-1 large-up-1"><div class="column column-block"><figure class="image"><a href="/fileadmin/_processed_/f/d/GDATA_GotoMeeting_powershell_script_88eadcd9a9.png" class="fancybox"><img class="image-embed-item " src="/fileadmin/_processed_/f/d/GDATA_GotoMeeting_powershell_script_8c74b1375d.png" width="1440" height="1020" loading="lazy" alt="" /></a><figcaption class="image-caption"> PowerShell script raw.ps1 </figcaption></figure></div></div></div></div> <div id="c232807" class="fce column large-12 small-12 frame-layout-0 " ><p>Utility.exe<sup>[3]</sup> is the very same GotoMeeting executable as winsys.odt. MLD.pdf<sup>[5]</sup>.&nbsp;&nbsp;</p></div> </div></div> <div data-magellan-target="c232802" class="small-12 column"><div class="row"> <div id="c232802" class="fce column large-12 small-12
