                            <div class="post-view__body fr-view ck-content " itemprop="articleBody">
                                <p>In recent weeks, we have observed a spike in infections involving the STOP ransomware variant. STOP is also known as DJVU by other vendors in the industry. In this article, we’ve looked at the latest version circulating in the wild. We will look at some of the main characteristics of this malware variant, along with detections that can be used to prevent infection and IOCs that we were able to extract during analysis.</p><p>The STOP ransomware has been around for some time, dating back to 2019. The latest version has been found to be distributed broadly in the past few weeks. Like the ones in the past, this variant is a portable executable that uses a public key to encrypt data on the victim’s machine and drops a ransom note in folder directories as it goes through the entire file system encrypting files using the Salsa20 encryption algorithm. The threat actors behind STOP have gone for a flat rate of USD $980 to provide the decryption keys to victims and have also offered a ‘discounted’ rate of USD $490 if the victims contact them within 72 hours of the attack occurring. This tactic is consistent with what has been observed in the past for this ransomware group.</p><p>Based on the tactics and techniques used by the malware, the threat actors behind the variant are likely from the Russian region as the malware avoids encryption explicitly on systems geo-located in or near Russia.</p><p style="margin-left: 40px"><img src="https://passle-net.s3.amazonaws.com/Passle/602651b953548812c0fa5fe2/MediaLibrary/Images/618a83ce17af570b2c25d016/2021-12-22-01-08-02-477-61c27a7249b2040fb8a38598.png" width="624" height="155" class="fr-fic fr-dii"></p><p align="center"><em>Figure 1: Quick Snapshot of STOP Ransomware</em></p><h2>Mitigation</h2><p>This section provides information that can be used to prevent infection by the STOP ransomware. We have included detections, IOC list, and YARA Rules that can be used to defend against this threat.</p><h3>YARA Rule</h3><p>This YARA Rule can be used to detect STOP Ransomware. Download the entire ruleset <a href="https://github.com/vithakur/Yara-rules/blob/main/ransomware/win_ransomware_stop.yar">here</a>.</p><figure class="image ck-widget image-style-block-align-center" contenteditable="false"><img src="https://passle-net.s3.amazonaws.com/Passle/602651b953548812c0fa5fe2/MediaLibrary/Images/618a83ce17af570b2c25d016/2021-12-22-01-08-50-497-61c27aa25354891344aab352.png" style="width: 728px" class="fr-fic fr-dib"></figure><p style="margin-left: 200px"><em>Figure 2: YARA Ruleset for STOP Ransomware</em></p><h3>Detections</h3><p>The following figure has the information that can be used to create detections for this malware. Download the entire list <a href="https://github.com/vithakur/detections/blob/main/STOP-ransomware-djvu/detections-memory">here</a>.</p><p>The following strings are from the unpacked malware, and these can be found in memory during and after the malware has been fully executed. This information can be used to create detections for EDR tools that can access and read memory and take actions based on detection rules applied.</p><figure class="image ck-widget image-style-block-align-center" contenteditable="false"><a href="https://github.com/vithakur/detections/blob/main/ransomware_stop.md"><img src="https://passle-net.s3.amazonaws.com/Passle/602651b953548812c0fa5fe2/MediaLibrary/Images/618a83ce17af570b2c25d016/2021-12-21-11-28-11-530-61c1ba4b5354890b6cc10be3.png" style="width: 600px" class="fr-fic fr-dib"></a></figure><p align="center"><em>Figure 3: Detections</em></p><h3>IOC List</h3><p>Download the entire list <a href="https://github.com/vithakur/detections/blob/main/STOP-ransomware-djvu/IOC-list">here</a>.</p><figure class="image ck-widget image-style-block-align-center" contenteditable="false"><img src="https://passle-net.s3.amazonaws.com/Passle/602651b953548812c0fa5fe2/MediaLibrary/Images/618a83ce17af570b2c25d016/2021-12-22-01-09-53-657-61c27ae18cb5d30ab8c38e02.png" style="width: 600px" class="fr-fic fr-dib"></figure><p align="center"><em>Figure 4: IOC list</em></p><h2>Execution</h2><p>Once the STOP ransomware executes, it attempts to make a few network connections over the Internet for various purposes, such as; geo-checking, key retrieval, and further infection by downloading different malware. First, let’s look at the start of the execution of this malware.</p><p style="margin-left: 40px"><img src="https://passle-net.s3.amazonaws.com/Passle/602651b953548812c0fa5fe2/MediaLibrary/Images/618a83ce17af570b2c25d016/2021-12-21-11-34-51-095-61c1bbdb8cb5d30dd8e2fae3.png" class="fr-fic fr-dii" style="width: 600px"></p><p align="center"><em>Figure 5: &nbsp;Malware Entry-point</em></p><p>Upon execution, the malware copies itself to the ‘C:\Users\[username]\AppData\Local\[GUID]’ directory on disk and tries to execute with escalated privileges, as shown in the figures below.</p><p style="margin-left: 40px"><img src="https://passle-net.s3.amazonaws.com/Passle/602651b953548812c0fa5fe2/MediaLibrary/Images/618a83ce17af570b2c25d016/2021-12-21-11-35-24-431-61c1bbfc49b2040bb0b1a05f.png" width="606" height="102" class="fr-fic fr-dii"></p><p align="center"><em>Figure 6: Malware copies itself to a different location</em></p><p style="margin-left: 40px"><img src="https://passle-net.s3.amazonaws.com/Passle/602651b953548812c0fa5fe2/MediaLibrary/Images/618a83ce17af570b2c25d016/2021-12-21-11-35-59-944-61c1bc1f8cb5d30dd8e2fbec.png" width="624" height="36" class="fr-fic fr-dii"></p><p align="center"><em>Figure 7: Spawning new process with elevated privileges</em></p><p>The malware then attempts to connect over the Internet to “<a href="https://api.2ip.ua/geo.json">https://api.2ip.ua/geo.json</a>” to verify the victim’s geolocation. This link leads to a Russian site (screenshot below) that provides geolocation services based on public Internet IP addresses which the malware uses to ascertain the location of its victims. The malware has a hard-coded country codes list that is checked before it continues executing on the victim’s system and will avoid encrypting victims within these countries.</p><p style="margin-left: 40px"><img src="https://passle-net.s3.amazonaws.com/Passle/602651b953548812c0fa5fe2/MediaLibrary/Images/618a83ce17af570b2c25d016/2021-12-21-11-38-28-406-61c1bcb45354890f603e0810.png" width="624" height="252" class="fr-fic fr-dii"></p><p align="center"><em>Figure 8: Geo-location service used by the malware</em></p><p>The site also offers an API-based service that the malware uses to determine the geolocation of the victim machines.</p><p style="margin-left: 40px"><img src="https://passle-net.s3.amazonaws.com/Passle/602651b953548812c0fa5fe2/MediaLibrary/Images/618a83ce17af570b2c25d016/2021-12-22-01-14-13-486-61c27be55354891344aab5a5.png" width="624" height="284" class="fr-fic fr-dii"></p><p align="center"><em>Figure 9: The specific API-based service the malware uses</em></p><p>The country code list can be seen in the figure below, showing the codes in memory during execution.</p><p style="margin-left: 100px"><img src="https://passle-net.s3.amazonaws.com/Passle/602651b953548812c0fa5fe2/MediaLibrary/Images/618a83ce17af570b2c25d016/2021-12-21-11-39-14-656-61c1bce25354890f603e0841.png" width="526" height="140" class="fr-fic fr-dii"></p><p align="center"><em>Figure 10: Country codes of locations this malware avoids</em></p><p>Next, the malware tries to connect to a command and control URI to get the public key for encryption. As we can see in the figure below, it sends a request to this URI with a PID created for the victim.</p><figure class="image ck-widget image-style-block-align-center" contenteditable="false"><img src="https://passle-net.s3.amazonaws.com/Passle/602651b953548812c0fa5fe2/MediaLibrary/Images/618a83ce17af570b2c25d016/2021-12-22-01-17-50-220-61c27cbe49b2040fb8a38a22.png" style="width: 600px" class="fr-fic fr-dib"></figure><p align="center"><em>Figure 11: URI loaded into the Stack for processing</em></p><p align="center"><img src="https://passle-net.s3.amazonaws.com/Passle/602651b953548812c0fa5fe2/MediaLibrary/Images/618a83ce17af570b2c25d016/2021-12-22-01-19-01-966-61c27d055354891344aab84b.png" width="624" height="103" class="fr-fic fr-dii"></p><p align="center"><em>Figure 12: Connection to the C2 for public key</em></p><p>Once the request is successful, the malware uses the public key with the ID to encrypt the victim's data.</p><p style="margin-left: 40px"><img src="https://passle-net.s3.amazonaws.com/Passle/602651b953548812c0fa5fe2/MediaLibrary/Images/618a83ce17af570b2c25d016/2021-12-21-11-40-14-891-61c1bd1e49b2040bb0b1a2eb.png" width="624" height="155" class="fr-fic fr-dii"></p><p align="center"><em>Figure 13: Public Key for encryption served by the C2</em></p><p>The malware uses a standard encryption sequence, calling in the functions required to encrypt data from start to finish. The complete sequence can be seen in the figure below, in the order of called functions.</p><p style="margin-left: 40px"><img src="https://passle-net.s3.amazonaws.com/Passle/602651b953548812c0fa5fe2/MediaLibrary/Images/618a83ce17af570b2c25d016/2021-12-21-12-09-33-779-61c1c3fd49b2040b0449675b.png" width="624" height="173" class="fr-fic fr-dii"></p><p align="center"><em>Figure 14: Encryption Sequence of function calls</em></p><h2>CSP – Cryptography Service Provider</h2><p>The malware queries the Registry on the victim machine to set the CSP and CSP type. Note that type shown in the figure below is 'Type 001' which is the 'RSA Full' provider.</p><p style="margin-left: 40px"><img src="https://passle-net.s3.amazonaws.com/Passle/602651b953548812c0fa5fe2/MediaLibrary/Images/618a83ce17af570b2c25d016/2021-12-22-01-21-52-503-61c27db05354891344aab8f5.png" width="624" height="37" class="fr-fic fr-dii"></p><p align="center"><em>Figure 15: Malware query to Registry for the Type of CSP</em></p><p>The malware uses the Registry to set the provider type and subsequently the actual provider, which in this case happens to be RSA Full.</p><h3>RegOpenKey</h3><p style="margin-left: 40px"><img src="https://passle-net.s3.amazonaws.com/Passle/602651b953548812c0fa5fe2/MediaLibrary/Images/618a83ce17af570b2c25d016/2021-12-21-12-10-48-417-61c1c44849b2040b044967dc.png" width="624" height="189" class="fr-fic fr-dii"></p><p align="center"><em>Figure 16: Registry functions used to determine the CSP</em></p><h3>RegOpenKeyExA</h3><p>Next, the malware queries the Registry to determine the actual CSP as can be seen in the figure below. </p><p align="center"><img src="https://passle-net.s3.amazonaws.com/Passle/602651b953548812c0fa5fe2/MediaLibrary/Images/618a83ce17af570b2c25d016/2021-12-21-12-11-48-722-61c1c48449b2040b04496894.png" width="624" height="15" class="fr-fic fr-dii"></p><p align="center"><em>Figure 17: The absolute Registry path passing through the Registers</em></p><p style="margin-left: 60px"><img src="https://passle-net.s3.amazonaws.com/Passle/602651b953548812c0fa5fe2/MediaLibrary/Images/618a83ce17af570b2c25d016/2021-12-21-12-12-08-970-61c1c4988cb5d30d9ccada38.png" width="553" height="83" class="fr-fic fr-dii"></p><p align="center"><em>Figure 18: The CSP highlighted in the Registry</em></p><p style="margin-left: 60px"><img src="https://passle-net.s3.amazonaws.com/Passle/602651b953548812c0fa5fe2/MediaLibrary/Images/618a83ce17af570b2c25d016/2021-12-21-12-12-26-841-61c1c4aa8cb5d30d9ccada5d.png" width="536" height="105" class="fr-fic fr-dii"></p><p align="center"><em>Figure 19: DLL image path to be called for the CSP</em></p><p>The malware uses the public key obtained from the command and control server to start the process of encryption on the victim’s system.</p><p style="margin-left: 100px"><img src="https://passle-net.s3.amazonaws.com/Passle/602651b953548812c0fa5fe2/MediaLibrary/Images/618a83ce17af570b2c25d016/2021-12-21-12-12-52-477-61c1c4c48cb5d30d9ccada9a.png" width="533" height="247" class="fr-fic fr-dii"></p><p align="center"><em>Figure 20: Second function to be called in the Encryption Sequence</em></p><figure class="image ck-widget image-style-block-align-center" style="width: 600px" class="fr-fic fr-dib"></figure><p align="center"><em>Figure 21: Public key loaded</em></p><p>Once the entire encryption sequence is completed for a directory, the final step is to write a ransom note to the directory with instructions on how to pay the ransom.</p><p style="margin-left: 100px"><img src="https://passle-net.s3.amazonaws.com/Passle/602651b953548812c0fa5fe2/MediaLibrary/Images/618a83ce17af570b2c25d016/2021-12-21-12-13-50-294-61c1c4fe8cb5d30d9ccadb59.png" width="512" height="159" class="fr-fic fr-dii"></p><p align="center"><em>Figure 22: Ransom note 'write' initiated</em></p><p>The figure below shows the ransom note as strings being passed onto the Stack before it is written to the disk.</p><figure class="image ck-widget image-style-block-align-center" contenteditable="false"><em><img src="https://passle-net.s3.amazonaws.com/Passle/602651b953548812c0fa5fe2/MediaLibrary/Images/618a83ce17af570b2c25d016/2021-12-21-12-14-12-056-61c1c5148cb5d30d9ccadb83.png" style="width: 600px" class="fr-fic fr-dib"></em></figure><p align="center"><em>Figure 23: Ransom note loaded into the Stack</em></p><p>Finally, the ransom note is written as a ‘txt’ file to the disk. This process is repeated for all directories in which the malware encrypts data. The figure below shows the newly created ransom note “_readme.txt”.</p><figure class="image ck-widget image-style-block-align-center" contenteditable="false"><img src="https://passle-net.s3.amazonaws.com/Passle/602651b953548812c0fa5fe2/MediaLibrary/Images/618a83ce17af570b2c25d016/2021-12-21-12-14-54-663-61c1c53e8cb5d30d9ccadc08.png" style="width: 300px" class="fr-fic fr-dib"></figure><p style="margin-left: 160px"><em>Figure 24: Ransom note file written to the current directory</em></p><p>The ransom note has the instructions on how the victims can pay to get the decryption key and provides a unique ID that the victim needs to use to get the decryption key for their machine. There is also a link to a demo video showing how the decryption tool works. The note also provides a couple of email addresses for the victims to contact the ransomware group if needed.</p><p style="margin-left: 40px"><img src="https://passle-net.s3.amazonaws.com/Passle/602651b953548812c0fa5fe2/MediaLibrary/Images/618a83ce17af570b2c25d016/2021-12-21-12-15-23-864-61c1c55b5354890f603e193a.png" width="624" height="376" class="fr-fic fr-dii"></p><p align="center"><em>Figure 25: Ransom note with instructions on next steps</em></p><p>This version of the STOP ransomware variant encrypts the file and replaces the file-extensions to “.shgv”, as seen in the figure below.</p><p style="margin-left: 260px"><img src="https://passle-net.s3.amazonaws.com/Passle/602651b953548812c0fa5fe2/MediaLibrary/Images/618a83ce17af570b2c25d016/2021-12-21-12-15-33-696-61c1c56549b2040b04496b09.png" width="211" height="215" class="fr-fic fr-dii"></p><p style="margin-left: 240px"><em>Figure 26: Files successfully encrypted</em></p><h2>Downloader Module</h2><figure class="image ck-widget image-style-block-align-center" contenteditable="false">Aside from performing common ransomware activities, this malware also tries to download and execute other malware:<img src="https://passle-net.s3.amazonaws.com/Passle/602651b953548812c0fa5fe2/MediaLibrary/Images/618a83ce17af570b2c25d016/2021-12-21-12-16-11-325-61c1c58b5354890f603e197a.png" style="width: 600px" class="fr-fic fr-dib"></figure><p align="center"><em>Figure 27: Downloaded malware - Vidar Stealer</em></p><p>This downloaded PE is a variant of the <a href="https://malware.news/t/deep-analysis-of-vidar-stealer/49591">Vidar malware family</a>.</p><p>Vidar Stealer is malware designed to steal information, mainly distributed as spam mail or cracked versions of commercial software and keygen programs. When installed, data such as infected device information, account, and history recorded in the browser is collected and sent to a command and control server.</p><p>The group behind the development or distribution (or both) of STOP ransomware may be working with the group responsible for developing the Vidar malware.</p><h2>Conclusion</h2><p>STOP ransomware has been around for quite some time now. Early occurrences of infections by this ransomware can be traced back to 2019.</p><p>Compared to some other ransomware families, the execution standard is low and it’s clear that this ransomware model is affiliation-leaning (working with other malware groups). We were able to link this malware to a different malware, the Vidar Stealer, which has been the case for quite some time.</p><p>The encryption is straightforward, with the threat actors not bothering to create their encryption algorithm or deploying any additional modules other than a downloader for a separate malware. The malware uses the Salsa20 algorithm for encryption. It is capable of both online and offline encryption.</p><p>This ransomware avoids infecting victims in and near Russia.</p><p>The ransomware seems to be targeted towards individuals or small businesses at best, as the asking price for the decryption key is not that high. They even offer an ‘early bird’ discount to top it all off.</p><figure class="image ck-widget image-style-block-align-center" contenteditable="false"><img src="https://passle-net.s3.amazonaws.com/Passle/602651b953548812c0fa5fe2/MediaLibrary/Images/6094020e53548503949f9ec4/2021-12-22-14-06-13-885-61c330d549b2040da0ce7458.png" style="width: 691px" class="fr-fic fr-dib"></figure><p><a href="https://medium.com/s2wblog/deep-analysis-of-vidar-stealer-ebfc3b557aed">Deep Analysis of Vidar Stealer</a> - Sojun Ryu<br><a href="https://github.com/vithakur/Yara-rules/blob/main/ransomware/win_ransomware_stop.yar" rel="noopener noreferrer" target="_blank">YAYA ruleset for STOP Ransomware</a> - Vishal Thakur<br><a href="https://github.com/vithakur/detections/blob/main/STOP-ransomware-djvu/detections-memory" rel="noopener noreferrer" target="_blank">Detections list for STOP Ransomware</a> - Vishal Thakur<br><a href="https://github.com/vithakur/detections/blob/main/STOP-ransomware-djvu/IOC-list" rel="noopener noreferrer" target="_blank">IOC list of STOP Ransomware</a> - Vishal Thakur</p><p><em>© Copyright 2021.
