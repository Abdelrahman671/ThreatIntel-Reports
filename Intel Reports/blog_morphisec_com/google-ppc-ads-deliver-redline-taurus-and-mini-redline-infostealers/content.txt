<p>In the past month, Morphisec has investigated the origin of several increasingly prevalent infostealers. These include Redline, Taurus, Tesla, and Amadey.</p>
<p>As part of our research, we identified pay-per-click (PPC) ads in Google’s search results that lead&nbsp; to downloads of malicious <strong>AnyDesk</strong>, <strong>Dropbox </strong>and <strong>Telegram </strong>packages wrapped as ISO images.</p>
<p><!--more--></p>
<p>The PPC ads targeted specific IP ranges in the US and probably some other countries. Non-targeted IPs are redirected to legitimate pages that download the correct applications.</p>
<p>The advertisements being on the first page indicates the need for constant vigilance on all levels. Adversaries will clearly take all opportunities possible to target their chosen victims, even going so far as to use legitimate services such as Google Adwords.&nbsp;</p>
<p>In this threat post we will go through three attack chains that lead to Redline, Taurus and a new mini-Redline infostealer compromise. We will focus on two adversaries because of similarities in patterns, certificates, and C2s. The first adversary leverages Redline and the second uses Taurus and mini-Redline.</p>
<p>We will cover Amadey in a separate blog post.</p>
<h2>Technical Introduction</h2>
<p>All of these attack chains start with one of a dozen paid Google ads that lead to a website with an ISO image download. The ISO image size is larger than 100MB, which allows the image to evade some scanning solutions that are optimized on throughput and size.</p>
<p>Mounting the ISO image leads to executables that are usually, but not always, digitally signed and legitimately verified.</p>
<p>Adversary One delivers the <span style="font-weight: bold;">Redline infostealer</span>.</p>
<ul>
<li aria-level="1">.Net executables are obfuscated with known obfuscators such as DeepSea, which leads to a custom obfuscated .Net DLL loader that eventually leads to a custom obfuscated Redline stealer .Net executable.</li>
</ul>
<p>Adversary Two delivers Taurus and a <strong>mini-Redline</strong> <span style="font-weight: bold;">infostealer</span>.</p>
<ul>
<li aria-level="1"><strong>Taurus AutoIt</strong> - 7fx executables that recreate and execute a legitimate AutoIt compiler with a malicious AutoIt script and a malicious encrypted Taurus executable that will be hollowed into the AutoIt process.&nbsp;</li>
<li aria-level="1"><strong>Mini-Redline</strong> - A minimized .Net version of the Redline stealer with some common functionality for stealing data from browsers. It features&nbsp; different configuration and communication patterns wrapped in four layers of obfuscation.</li>
</ul>

<h2>Taurus Infostealer</h2>
<p>The <span style="font-weight: bold;">Taurus infostealer</span> is delivered in a similar way and appears as the third paid ad in a search for the popular applications mentioned in the introduction.&nbsp;</p>
<p>This time the website is signed with a legitimate Cloudflare certificate. Like the Sectigo certificate used with Redline, the Taurus certificate is not older than two weeks</p>
<p>In the Taurus case, we did not see any redirects to additional websites. As can be seen in the image below, the download results from a submitted form that is handled by “get.php” and in turn delivers the ISO image directly from the website.</p>
<p><img src="https://lh4.googleusercontent.com/y4wTqB_BgEiTd0OWa2jm4adzARINxGbTmJKTetOpbBAikCzURNaTk2xwEeMlGBX18656tZY1O4v2LlATACjvBOWhKuGUWaEyoY2eUgtNCaIKeTT27oQbNr9IhZk7NOI-irh0QZLw" width="624" height="277" loading="lazy" style="margin-left: auto; margin-right: auto; display: block;" alt="Taurus infostealer"></p>
<p>If the target is not within the range of interesting IP addresses, users will see a normal redirect to the legitimate application website like in the Redline infostealer.</p>
<p><img src="https://lh6.googleusercontent.com/y2mgxp4wLsGgC_o9kOs908UXcwPIa9HECRTyR7ne4ehMsxgEOoh_QXOZ0uNl4p4pYQPweTZih1VG7R506x50BKszImreiLo7YAf07hFX2yWwNwFwRCdwP-b-b6DcP8S-S344kczd" width="624" height="224" loading="lazy" style="margin-left: auto; margin-right: auto; display: block;" alt="infostealer redirects"></p>
<p>The downloaded ISO image consists of a 7z SFX executable.&nbsp;</p>
<p><img src="https://lh4.googleusercontent.com/ozI5qMmPBFx_UvY_DSUwMHfATARG9TX96QgGz8U2K-A8Ux3QGGtLO5rKs6CoMy_T8g2AbI8ABvABFJjkOF_bT4HzyFMvj__K2OWguDkLQUGIJ_RqM2UHahvgUyyOzXYB08ZDEFS6" width="467" height="412" loading="lazy" style="margin-left: auto; margin-right: auto; display: block;" alt="7z SFX executable"></p>
<p>The executable includes either 4 “flv” or 4 “bmp” files in the examples we cover below. Sfx is configured to start the execution from the first batch file (masquerading as either flv, bmp or any other unique extension). The batch script is then redirected as input into cmd.exe.</p>
<p><img src="https://lh6.googleusercontent.com/_NepyR8cn27D8MNfUfFST8ynTW7L9AeUcqLYB_RXLoC0raWWvjNoC5XqWyi74zJ7y2fUJyIFoAbGur7JgPuJfva0R63A3qAYlY8_DEVL04CPJhgtYsAvVwkJQqM-prus2XtfW04z" width="624" height="275" loading="lazy" style="margin-left: auto; margin-right: auto; display: block;" alt="batch script is then redirected as input into cmd.exe"></p>
<p><img src="https://lh5.googleusercontent.com/av3ZdRiGCJ_Wjk2N7KgSDtPvPGAZ4GpDzLjUMPl7D9zLoihYYLkRuM5MGiz-RHZ-HMey_LdUOK01GfrzMBHkqxobNXUOQ3ENmYhk3C35hsZMpkMmicUCUkfn9ZLcOqlD3M0acL_E" width="624" height="388" loading="lazy" style="margin-left: auto; margin-right: auto; display: block;" alt="batch script"></p>
<p><img src="https://lh5.googleusercontent.com/nQIK3bZ7avZ0nue42rFVIi5VNkS7-UUCIjHOTfujxl0ibJiVeprcyBKFFObAHcpY17Rhn30SuDAvPDjTWbT8wQI5TRBMIMZ9qzz79XWHXKOEAfOavQ7kqTAYMAb3VX8Rzs-fe-jd" width="624" height="227" loading="lazy" style="margin-left: auto; margin-right: auto; display: block;" alt="re-creation of the legitimate AutoIt compiler"></p>
<p>This batch script is well documented. It is responsible for the re-creation of the legitimate AutoIt compiler (Ali.exe.com or the Dio.exe.com in the examples above) and the execution of the malicious AutoIt script (Pramide.flv or the Debbano.bmp). Through the re-created compiler, it will fail to execute upon detection of a known sandbox provider. A VirusTotal search for additional 7z SFX archives with a similar evasion will lead to more than 400 different files uploaded in the past month.</p>
<p><img src="https://lh5.googleusercontent.com/10f-4ShC_p0Hj8XNzXJIP9UoQCXYiVmslr3FJvfzHwNbxjMEGnJ5n4CkB5IRnq4VF8mtMrFayfdyPhxli5E2RxiC85tW0Lr6a9oRMeZRuHSkNAxZ7EmNSPwaiebg61uni2fE0_mj" width="624" height="404" loading="lazy" style="margin-left: auto; margin-right: auto; display: block;" alt="files upload"></p>
<p>The AutoIt script supports both 32 and 64 bit processes (slightly deobfuscated).</p>
<p><img src="https://lh3.googleusercontent.com/tN4wrJmog_-iL1-8nmcpTR5ySCLQwnkQO-ma-oeTDfcvk3JMavjo5cO8lk1IlTqchSngccCotv41W3aLDzsruwxrrggwaEUyGrb3FYA4OnIHNEvbMaXXn1u7NSDb9kHjdqmvH1NK" width="624" height="352" loading="lazy" style="margin-left: auto; margin-right: auto; display: block;" alt="AutoIt script"></p>
<p>It also implements persistence through a URL link directly in the startup folder. The link executes Javascript from a hidden folder under roaming (use attrib -H to unhide).&nbsp;</p>
<p><img src="https://lh3.googleusercontent.com/XNirKnE9vrlxNR1ju_orlKnOjFToT3uLUuJOadnqRUrVb5aR-EEpNh_2G1MkL00oKRdX2Lb2CqETyxnhmPwkPJHdlHYcSTghdvGtafAZKC-RHjigDMvnKDCyO9zMtMpKs-J31KXD" width="624" height="536" loading="lazy" style="margin-left: auto; margin-right: auto; display: block;" alt="Javascript from a hidden folder"></p>
<p>As in the previous batch file execution, the Javascript file executes the AutoIt compiler with the copied Taurus AutoIt script.</p>
<p><img src="https://lh6.googleusercontent.com/IT7cAeJibH-Fx5QIr6JAN7tEFSyA38yybmR06C2K4672XiriaFwj5_XXwZOK4E6DlIU4kTz8CUWK0tThi7HngoKLI1siokeSdxdKaPNye6mh6XQIowTlbq3xjqT_6W8yzKORSUN9" width="624" height="303" loading="lazy" style="margin-left: auto; margin-right: auto; display: block;" alt="Taurus AutoIt script"></p>
<h2>Mini-Redline Infostealer</h2>

<h2>Conclusion:</h2>
<p>Adversaries will use any method possible to gather targets, even paying Google top dollar for their paid search results to surface a malicious website as a top search result. This inventiveness on the part of threat actors means that organizations need to be constantly vigilant in all aspects of their operations. There’s no telling when an adversary will set up a website with a signed, legitimate certificate designed to mislead website visitors.&nbsp;</p>
