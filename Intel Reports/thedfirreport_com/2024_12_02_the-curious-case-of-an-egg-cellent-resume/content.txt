<h2 id="key" class="wp-block-heading"><a href="#key">Key Takeaways</a></h2>



<ul class="wp-block-list">
<li>Initial access was via a resume lure as part of a TA4557/FIN6 campaign.</li>



<li>The threat actor abused LOLbins like ie4uinit.exe and msxsl.exe to run the more_eggs malware.</li>



<li>Cobalt Strike and python-based C2 Pyramid were employed by the threat actor for post-exploitation activity.</li>



<li>The threat actor abused CVE-2023-27532 to exploit a Veeam server and facilitate lateral movement and privilege escalation activities.</li>



<li>The threat actor installed Cloudflared to assist in tunneling RDP traffic.</li>



<li>This case was first published as a <a href="https://thedfirreport.com/services/threat-intelligence/#threat-brief" target="_blank" rel="noreferrer noopener">Private Threat Brief</a> for customers in April of 2024.</li>



<li>Eight new rules were created from this report and added to our <a href="https://thedfirreport.com/services/detection-rules/" target="_blank" rel="noreferrer noopener">Private Detection Ruleset</a>.</li>
</ul>

<h2 id="case-summary" class="wp-block-heading"><a href="#case-summary">Case Summary</a></h2>
<!-- /wp:heading -->

<!-- wp:paragraph -->
<p>In March 2024, an investigation took place after malicious activity was detected. Upon analysis, it was identified that a threat actor was able to infect and pivot from a user endpoint to two servers in the environment.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2024/12/27899_001.png"><img fetchpriority="high" decoding="async" class="aligncenter size-full wp-image-40701" src="https://thedfirreport.com/wp-content/uploads/2024/12/27899_001.png" alt="" width="1927" height="850" srcset="https://thedfirreport.com/wp-content/uploads/2024/12/27899_001.png 1927w, https://thedfirreport.com/wp-content/uploads/2024/12/27899_001-300x132.png 300w, https://thedfirreport.com/wp-content/uploads/2024/12/27899_001-1024x452.png 1024w, https://thedfirreport.com/wp-content/uploads/2024/12/27899_001-768x339.png 768w, https://thedfirreport.com/wp-content/uploads/2024/12/27899_001-1536x678.png 1536w" sizes="(max-width: 1927px) 100vw, 1927px" /></a></p>
<!-- /wp:paragraph -->

<!-- wp:paragraph --><!-- /wp:paragraph -->

<!-- wp:paragraph --><!-- /wp:paragraph -->

<!-- wp:paragraph --><!-- /wp:paragraph -->

<!-- wp:image -->
<figure class="wp-block-image"></figure>
<!-- /wp:image -->

<!-- wp:paragraph -->
<p>The threat actor was able to gain access by submitting a job application that pointed to a resume lure. This initial access campaign was observed by Proofpoint who attribute it to the group they track as TA4557. This group has historically overlapped with <a href="https://attack.mitre.org/groups/G0037/" target="_blank" rel="noreferrer noopener">FIN6</a> activity, and has tooling overlaps with <a href="https://attack.mitre.org/groups/G0080/" target="_blank" rel="noreferrer noopener">Cobalt Group</a> and <a href="https://attack.mitre.org/groups/G0120/" target="_blank" rel="noreferrer noopener">Evilnum</a>.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>After being directed to an online resume site from the job posting notice, the victim downloaded the fake resume zip and executed the malicious .lnk file within the zip. This started an execution flow with the threat actor using the ie4uinit.exe Microsoft executable to side-load a malicious .inf file. After that, the process dropped a malicious DLL which was executed using WMI. This then created a scheduled task, followed by another WMI process to load malicious JScript using the msxsl.exe Microsoft binary. This was the final more_eggs payload that established beacon activity to the command and control server.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>Some initial discovery commands were then run using Microsoft binaries like nltest, net, and whoami. Activity mostly ceased until approximately one and a half days later when Cobalt Strike was dropped on the beachhead. First we observed the threat actor create shadow copies using vssadmin, which we assess was likely for trying to access credentials. This was followed up with some initial discovery using Microsoft utilities and the creation of a new user on the system.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>The threat actor then used <a href="https://github.com/djhohnstein/SharpShares" target="_blank" rel="noreferrer noopener">SharpShares</a> and <a href="https://github.com/GhostPack/Seatbelt" target="_blank" rel="noreferrer noopener">Seatbelt</a> to further enumerate the host and the environment. The threat actor then attempted to deploy <a href="https://github.com/naksyn/Pyramid" target="_blank" rel="noreferrer noopener">Pyramid</a> on the beachhead, and while it was executed after much trouble, we observed little action from it or communication to its command and control server. After Pyramid, the threat actor began looking for lateral movement options. They decided to target a backup server running Veeam software. They were able to exploit the vulnerability, CVE-2023-27532, on the server and used the access to create a new local administrator account.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>Using the new account on the backup server they used RDP to connect from the beachhead. During this and later RDP pivoting activity, the threat actor leaked several host names that tie to other intrusions that ended with a Fog ransomware deployment, as reported by <a href="https://arcticwolf.com/resources/blog/lost-in-the-fog-a-new-ransomware-threat/" target="_blank" rel="noreferrer noopener">Arctic Wolf</a>. On the backup server the threat actor deployed their Cobalt Strike payload and continued discovery activity with more SharpShares and Seatbelt activity and then AdFind. At this time, on the backup server, LSASS memory was accessed for credentials.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>Following this, the threat actor targeted a second server and this time used a remote service to replicate the creation of a new local administrator account on this second server. This service was created using a domain administrator account indicating the previous LSASS access was successful. The threat actor then checked various privileged users in the environment before locating a disabled domain administrator account that they re-enabled.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>On the backup server, the threat actor then used the browser to access the file sharing site temp.sh to download a zip file containing a <a href="https://github.com/cloudflare/cloudflared" target="_blank" rel="noreferrer noopener">Cloudflared</a> installer. The MSI installer in the zip was then run and Cloudflared was installed as a service on the server. They then connected to the second server using RDP and repeated the install process there. On this server, they then created another new user and added them to the local administrator group.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>The threat actor then started a new RDP session with the new user and then dropped and executed <a href="https://www.softperfect.com/products/networkscanner/" target="_blank" rel="noreferrer noopener">SoftPerfect Network Scanner</a>. After the scan, the threat actor opened a few files from a remote file share and then activity ceased for the day.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>The following day the threat actor returned, pinging a host on a remote network. After this, they then removed the more_eggs files and persistence task on the beachhead and beaconing to more_eggs command and control ceased. The Cobalt Strike and Cloudflared tunnels remained active but no further activity was observed before the threat actor was evicted.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>Further open-source investigation into the fake resume campaign identified numerous other lure sides with the same templates and images following the same &lt;name&gt;.com format.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2024/12/27899_002.png"><img decoding="async" class="aligncenter size-full wp-image-40702" src="https://thedfirreport.com/wp-content/uploads/2024/12/27899_002.png" alt="" width="1386" height="765" srcset="https://thedfirreport.com/wp-content/uploads/2024/12/27899_002.png 1386w, https://thedfirreport.com/wp-content/uploads/2024/12/27899_002-300x166.png 300w, https://thedfirreport.com/wp-content/uploads/2024/12/27899_002-1024x565.png 1024w, https://thedfirreport.com/wp-content/uploads/2024/12/27899_002-768x424.png 768w" sizes="(max-width: 1386px) 100vw, 1386px" /></a></p>

<h2 id="initial-access" class="wp-block-heading"><a href="#initial-access">Initial Access</a></h2>
<!-- /wp:heading -->

<!-- wp:paragraph -->
<p>Proofpoint has been tracking TA4557 since 2018 as a skilled, financially motivated threat actor known to distribute the exclusive more_eggs backdoor, which can profile the endpoint and send additional payloads. TA4557 notably differs from other priority threat actors due to the unique tool and malware usage, campaign targeting, job candidate-themed lures, sophisticated evasive measures, distinct attack chains, and notable infrastructure patterns.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2024/12/27899_003.png"><img decoding="async" class="aligncenter size-full wp-image-40703" src="https://thedfirreport.com/wp-content/uploads/2024/12/27899_003.png" alt="" width="590" height="720" srcset="https://thedfirreport.com/wp-content/uploads/2024/12/27899_003.png 590w, https://thedfirreport.com/wp-content/uploads/2024/12/27899_003-246x300.png 246w" sizes="(max-width: 590px) 100vw, 590px" /></a></p>

<p>Throughout 2024, Proofpoint has observed TA4557 use multiple delivery approaches when targeting employees involved in the hiring process, including:</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<ul>
<li>Sending emails directly to employees containing instructions guiding the recipient to navigate to a resume-themed website that ultimately leads to malware delivery. The actor used various methods to prevent security tools from recognizing the domain in the email body, seemingly to avoid automated analysis. Examples include leaving a space or underscore before the TLD in the email body.</li>
<li>Sending benign emails directly to employees, waiting for a response, then replying with an email containing instructions guiding the recipient to navigate to a resume-themed website that ultimately leads to malware delivery masquerading as a fake candidate and applying to legitimate job postings on various employment websites. The actor typically uploaded a resume to the job application containing instructions that guided users to a fake candidate web page.</li>
<li>In March 2024, Proofpoint observed the use of the third approach in an email from a job board notifying a user that a new candidate (TA4557) had applied to the user’s open job posting. Our intrusion files and lure site matched this campaign.</li>
</ul>

<h2 id="execution" class="wp-block-heading"><a href="#execution">Execution</a></h2>
<!-- /wp:heading -->

<!-- wp:paragraph --><!-- /wp:paragraph -->

<!-- wp:paragraph -->

<p><a href="https://thedfirreport.com/wp-content/uploads/2024/12/27899_010.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-40710" src="https://thedfirreport.com/wp-content/uploads/2024/12/27899_010.png" alt="" width="277" height="246" /></a></p>

<p><strong>Abusing ie4uinit.exe (LOLBin)</strong></p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->
<p>The flow uses a documented execution hijack of IE4uinit.exe (<a href="https://lolbas-project.github.io/lolbas/Binaries/Ie4uinit/" target="_blank" rel="noreferrer noopener">https://lolbas-project.github.io/lolbas/Binaries/Ie4uinit/</a>). By supplying a &#8220;side-loaded&#8221; .inf file to IE4uinit.exe, it can be used to load and execute COM scriptlets (SCT) from remote servers. In this case it was observed to reach out to the URL:</p>
<!-- /wp:paragraph -->

<!-- wp:preformatted -->

</pre>

<p>The threat actor later used the Cobalt Strike beacon to execute numerous other discovery commands:</p>
<!-- /wp:paragraph -->

<!-- wp:preformatted -->
<pre class="wp-block-preformatted">%WINDIR%\system32\cmd.exe /C query session 
%WINDIR%\system32\cmd.exe /C nslookup 
%WINDIR%\system32\cmd.exe /C ping -n 1 &lt;DOMAIN&gt;

%WINDIR%\system32\cmd.exe /C net user REDACTED /dom

<p>The victim organization was recording certain key ETW providers which allows us to capture the threat actor loading certain modules. The following was recorded, which highlighted the threat actor using the known enumeration tool <a href="https://github.com/GhostPack/Seatbelt" target="_blank" rel="noreferrer noopener">Seatbelt</a> and <a href="https://github.com/djhohnstein/SharpShares" target="_blank" rel="noreferrer noopener">SharpShares</a>.</p>
<!-- /wp:paragraph -->

<!-- wp:preformatted -->
<pre class="wp-block-preformatted">Microsoft-Windows-DotNETRuntimeRundown {A669021C-C450-4609-A035-5AF59AF4DF18}

Seatbelt
"EventID":153
"CommandLine":"C:\\Windows\\system32\\rundll32.exe"
"ModuleILPath":"Seatbelt"
"ManagedPdbBuildPath":"Z:\\Agressor\\github.com-GhostPack\\Seatbelt-master\\Seatbelt\\obj\\Debug\\Seatbelt.pdb"

SharpShares
"EventID":153
"CommandLine":"C:\\Windows\\system32\\rundll32.exe"
"ModuleILPath":"SharpShares"
"ManagedPdbBuildPath":"C:\\Users\\mmoser\\source\\repos\\SharpShares\\SharpShares\\obj\\Release\\SharpShares.pdb"
</pre>
<!-- /wp:preformatted -->

<!-- wp:paragraph -->
<p>The execution of these tools generated the files seatinfo.txt and share.txt, which the threat actor viewed on host.</p>
<!-- /wp:paragraph -->

<!-- wp:preformatted -->
<pre class="wp-block-preformatted">type C:\programdata\shares.txt (A list of file shares in the network)
type c:\programdata\seatinfo.txt (Output of Seatbelt)
</pre>
<!-- /wp:preformatted -->

<!-- wp:paragraph -->
<p>Invoking SharpShares or SeatBelt (both .NET/managed applications) within an unmanaged code-based process (e.g., RunDLL32) results in the creation of a Common Language Runtime (CLR) usage log file as a side effect.</p>
<!-- /wp:paragraph -->

<!-- wp:preformatted -->
<pre class="wp-block-preformatted">C:\Users\&lt;REDACTED&gt;\AppData\Local\Microsoft\CLR_v4.0\UsageLogs\rundll32.exe.log
</pre>
<!-- /wp:preformatted -->

<!-- wp:paragraph -->
<p>The presence of this file for a process such as RunDLL32.exe indicates that a .Net compiled payload has been executed. In this case we can correlate this activity to the execute-assembly event.</p>
<!-- /wp:paragraph -->

<!-- wp:paragraph -->

<!-- /wp:paragraph -->

<pre class="wp-block-preformatted">adfind.exe -f "(objectcategory=person)"
adfind.exe -f "objectcategory=computer"
adfind.exe -subnets -f (objectCategory=subnet) 
</pre>

<!-- wp:preformatted -->
<pre class="wp-block-preformatted">C:\ProgramData\scaner\scaner\netscan.exe
</pre>
