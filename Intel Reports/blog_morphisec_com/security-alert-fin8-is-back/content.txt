<p>During the period of March to May 2019, Morphisec Labs observed a new, highly sophisticated variant of the ShellTea / PunchBuggy backdoor malware that attempted to infiltrate a number of machines within the network of a customer in the hotel-entertainment industry. It is believed that the malware was deployed as a result of several phishing attempts.</p>
<!--more-->
<p>The last documented version of <span style="font-weight: bold;">ShellTea</span> was in 2017, <a href="https://www.root9b.com/sites/default/files/whitepapers/PoS%20Malware%20ShellTea%20PoSlurp.pdf" rel=" noopener">in a <span style="font-weight: normal;">POS malware attack</span></a><span style="font-weight: normal;">.</span> Given the nature of the industry targeted in the attack uncovered by Morphisec, we assume that this was also an attempted POS attack. As the attack was prevented by the Morphisec solution, the POS malware could not be downloaded to the machines.&nbsp;</p>
<p>This is the first cyberattack observed during 2019 that can be attributed to <em>FIN8</em> with high probability, although there are a few indicators that overlap with known FIN7 attacks (URLs and infrastructure).</p>
<p>In this report, we investigate this latest variant of ShellTea, together with the artifacts it downloaded after the Morphisec Labs team detonated a sample in a safe environment.</p>
<h2>FIN8 - Technical Details</h2>
<p>We begin by examining the different stages of the fileless dropper in detail.</p>
<h2>Fileless execution</h2>
<p>Following successful infiltration, the malware persists through registry: HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run</p>
<p>&nbsp;</p>
<p><img id="__hsnew" src="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic1.png?width=1593&amp;name=shelltea-pic1.png" alt="Fileless execution" style="width: 1593px;" width="1593" srcset="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic1.png?width=797&amp;name=shelltea-pic1.png 797w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic1.png?width=1593&amp;name=shelltea-pic1.png 1593w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic1.png?width=2390&amp;name=shelltea-pic1.png 2390w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic1.png?width=3186&amp;name=shelltea-pic1.png 3186w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic1.png?width=3983&amp;name=shelltea-pic1.png 3983w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic1.png?width=4779&amp;name=shelltea-pic1.png 4779w" sizes="(max-width: 1593px) 100vw, 1593px"></p>
<p>The command line execution leads to PowerShell code executed from a different registry value: <em>HKEY_CURRENT_USER\\Software\\ [random name]</em>. The attacker abuses the PowerShell wildcard mechanism with the assumption that there are no additional keys in registry under HKCU the match the <em>S???ware</em> string beside <em>Software</em>. This may minimize the effectiveness of certain detection or intro inspection tools when looking for the next stage execution.</p>
<p><img id="__hsnew" src="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic2-900px.png?width=900&amp;name=shelltea-pic2-900px.png" alt="HKEY CURRENT USER WINDOWS REGISTRY EDITOR" style="width: 900px;" width="900" srcset="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic2-900px.png?width=450&amp;name=shelltea-pic2-900px.png 450w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic2-900px.png?width=900&amp;name=shelltea-pic2-900px.png 900w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic2-900px.png?width=1350&amp;name=shelltea-pic2-900px.png 1350w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic2-900px.png?width=1800&amp;name=shelltea-pic2-900px.png 1800w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic2-900px.png?width=2250&amp;name=shelltea-pic2-900px.png 2250w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic2-900px.png?width=2700&amp;name=shelltea-pic2-900px.png 2700w" sizes="(max-width: 900px) 100vw, 900px"></p>
<p>The code from the [<em>random name</em>] key executes an additional PowerShell command that decodes base64 assembly and invokes it within the memory while passing it as parameters an additional [<em>random name</em>] registry value – the <em>ShellTea</em> shellcode:</p>
<p><em>"Xshuzugewogazi"="$null=[System.Reflection.Assembly]::Load([System.Convert]::FromBase64String("TVq….</em>”<em>));[LExR.J9fL]::YHvszr((Get-ItemProperty -Path HKCU:Software\\Fpkakesude -Name Glyavonubafi).Glyavonubafi, 0);"</em></p>
<h3>Executing .NET assembly for shellcode execution</h3>
<p>The base64 encoded assembly that we saw in the previous stage is a .NET stager that self-injects a shellcode (the parameter from the previous stage) by creating a new thread with the shellcode entry. The thread also needs a parameter (the same shellcode) for proper execution.</p>
<p><img id="__hsnew" src="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic3.png?width=1381&amp;name=shelltea-pic3.png" alt="base64 encoded" style="width: 1381px;" width="1381" srcset="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic3.png?width=691&amp;name=shelltea-pic3.png 691w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic3.png?width=1381&amp;name=shelltea-pic3.png 1381w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic3.png?width=2072&amp;name=shelltea-pic3.png 2072w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic3.png?width=2762&amp;name=shelltea-pic3.png 2762w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic3.png?width=3453&amp;name=shelltea-pic3.png 3453w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic3.png?width=4143&amp;name=shelltea-pic3.png 4143w" sizes="(max-width: 1381px) 100vw, 1381px"></p>
<h2>Shellcode</h2>
<h3>Custom function resolution</h3>
<p>To operate and evade standard analysis tools, most of the functions are hashed. The hashing algorithm has a high degree of similarity to the previous ShellTea version, with a slight modification of the seeds and constants. In this version, the attacker also utilizes functions from ole32 for stream processing.</p>
<p><img id="__hsnew" src="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic4.png?width=934&amp;name=shelltea-pic4.png" alt="Custom function resolution" style="width: 934px;" width="934" srcset="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic4.png?width=467&amp;name=shelltea-pic4.png 467w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic4.png?width=934&amp;name=shelltea-pic4.png 934w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic4.png?width=1401&amp;name=shelltea-pic4.png 1401w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic4.png?width=1868&amp;name=shelltea-pic4.png 1868w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic4.png?width=2335&amp;name=shelltea-pic4.png 2335w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic4.png?width=2802&amp;name=shelltea-pic4.png 2802w" sizes="(max-width: 934px) 100vw, 934px"></p>
<h3>Inject into Explorer</h3>
<p>As in the previous version, ShellTea continues the operation after persisting through the explorer.exe process. While it includes multiple ways to find Explorer, the preferred method is to get the process id from the current desktop window.</p>
<p><img id="__hsnew" src="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic5.png?width=686&amp;name=shelltea-pic5.png" alt="Inject into Explorer" width="686" srcset="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic5.png?width=343&amp;name=shelltea-pic5.png 343w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic5.png?width=686&amp;name=shelltea-pic5.png 686w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic5.png?width=1029&amp;name=shelltea-pic5.png 1029w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic5.png?width=1372&amp;name=shelltea-pic5.png 1372w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic5.png?width=1715&amp;name=shelltea-pic5.png 1715w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic5.png?width=2058&amp;name=shelltea-pic5.png 2058w" sizes="(max-width: 686px) 100vw, 686px"></p>
<p>After finding the process id, the shellcode uses standard functions to allocate and write memory within Explorer and then uses low-level API <em>RtlCreateUserThread </em>for thread injection.</p>
<p><img id="__hsnew" src="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic6.png?width=973&amp;name=shelltea-pic6.png" alt="shelltea-pic6" style="width: 973px;" width="973" srcset="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic6.png?width=487&amp;name=shelltea-pic6.png 487w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic6.png?width=973&amp;name=shelltea-pic6.png 973w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic6.png?width=1460&amp;name=shelltea-pic6.png 1460w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic6.png?width=1946&amp;name=shelltea-pic6.png 1946w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic6.png?width=2433&amp;name=shelltea-pic6.png 2433w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic6.png?width=2919&amp;name=shelltea-pic6.png 2919w" sizes="(max-width: 973px) 100vw, 973px"></p>
<h3>Virtual Environment and Sandbox Bypass</h3>
<p>ShellTea utilizes a number of techniques to identify if it is running within a virtual environment or is being monitored.</p>
<h4>Virtual environment by firmware</h4>
<p>ShellTea extracts the firmware string information using <em>NtQuerySystenInformation</em> with the <em>SystemFirmwareTableInformation</em> flag. Then it searches for a set of known strings as described in this <a href="https://blog.checkpoint.com/wp-content/uploads/2016/10/DefeatingSandBoxEvasion-VB2016_CheckPoint.pdf">article</a> by Checkpoint (additional strings have been added in this case, e.g. Visual studio).</p>
<h4><img id="__hsnew" src="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic7.png?width=763&amp;name=shelltea-pic7.png" alt="Shelltea Virtual Environment and Sandbox Bypass" style="width: 763px;" width="763" srcset="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic7.png?width=382&amp;name=shelltea-pic7.png 382w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic7.png?width=763&amp;name=shelltea-pic7.png 763w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic7.png?width=1145&amp;name=shelltea-pic7.png 1145w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic7.png?width=1526&amp;name=shelltea-pic7.png 1526w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic7.png?width=1908&amp;name=shelltea-pic7.png 1908w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic7.png?width=2289&amp;name=shelltea-pic7.png 2289w" sizes="(max-width: 763px) 100vw, 763px"></h4>
<h4>Looking for monitoring processes</h4>
<p>As part of the anti-debugging or anti-monitoring techniques, ShellTea iterates over all the running processes, applies CRC32 on each process name (after converting the string to capital letters), and then compares the value against a predefined set of CRCs. Note that a bug in the previous version has been fixed and more CRCs were added.</p>
<p><img id="__hsnew" src="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic8.png?width=764&amp;name=shelltea-pic8.png" alt="Shelltea " style="width: 764px;" width="764" srcset="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic8.png?width=382&amp;name=shelltea-pic8.png 382w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic8.png?width=764&amp;name=shelltea-pic8.png 764w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic8.png?width=1146&amp;name=shelltea-pic8.png 1146w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic8.png?width=1528&amp;name=shelltea-pic8.png 1528w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic8.png?width=1910&amp;name=shelltea-pic8.png 1910w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic8.png?width=2292&amp;name=shelltea-pic8.png 2292w" sizes="(max-width: 764px) 100vw, 764px"></p>
<p>We wrote a python script which is based on a set of known processes and identified the list of the processes that are being searched for. These are:</p>
<p>WINDBG.EXE, WIRESHARK.EXE, PROCEXP.EXE, PROCMON.EXE, TCPVIEW.EXE, OLLYDBG.EXE, IDAG.EXE, IDAG64.EXE, DUMPCAP.EXE, FILEMON.EXE, IDAQ64.EXE, IDAQ.EXE, IMMUNITYDEBUGGER.EXE, PETOOLS.EXE, REGMON.EXE, SYSER.EXE, TCPDUMP.EXE, WINDUMP.EXE, APIMONITOR.EXE, APISPY32.EXE, IRIS.EXE, NETSNIFFER.EXE, WINAPIOVERRIDE32.EXE, WINSPY.EXE</p>
<h4>Validate Hard Disk Volume</h4>
<p>The hard disk volume name is hashed with SHA1 and compared to a preconfigured SHA1.</p>

<h3>FIN8 Persistency</h3>
<p>Upon successful bypassing of sandboxes, the shellcode executes a persistency module. If the attack is yet to be persistent (validates beforehand), it decrypts the PowerShell base64 command, then decrypts the CMD command for persistency and writes those into the registry as described in the first step. Note that every string is decrypted with different XOR parameter which may fail some of the automatic analyzers.</p>
<p><img id="__hsnew" src="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic10.png?width=1168&amp;name=shelltea-pic10.png" alt="FIN8 Persistency" style="width: 1168px;" width="1168" srcset="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic10.png?width=584&amp;name=shelltea-pic10.png 584w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic10.png?width=1168&amp;name=shelltea-pic10.png 1168w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic10.png?width=1752&amp;name=shelltea-pic10.png 1752w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic10.png?width=2336&amp;name=shelltea-pic10.png 2336w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic10.png?width=2920&amp;name=shelltea-pic10.png 2920w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic10.png?width=3504&amp;name=shelltea-pic10.png 3504w" sizes="(max-width: 1168px) 100vw, 1168px"></p>
<h2>C2 protocol</h2>
<h3>Commands</h3>
<p>The ShellTea backdoor communicates on top of HTTPS and supports a number of commands based on what is returned from the C2 server.</p>
<ul>
<li>It may write data/shellcode it received from the C2 into the registry</li>
<li>It may reflectively load the delivered executable into the process (and then execute it)</li>
<li>It may create a file and execute it as a process, then mark it as deleted (after restart)</li>
<li>It may execute the shellcode as is by creating additional thread</li>
<li>It may execute any PowerShell command using downloaded native Empire ReflectivePicker (will be described later).</li>
</ul>
<p><img id="__hsnew" src="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic11.png?width=917&amp;name=shelltea-pic11.png" alt="PowerShell command using downloaded native Empire ReflectivePicker" style="width: 917px;" width="917" srcset="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic11.png?width=459&amp;name=shelltea-pic11.png 459w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic11.png?width=917&amp;name=shelltea-pic11.png 917w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic11.png?width=1376&amp;name=shelltea-pic11.png 1376w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic11.png?width=1834&amp;name=shelltea-pic11.png 1834w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic11.png?width=2293&amp;name=shelltea-pic11.png 2293w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic11.png?width=2751&amp;name=shelltea-pic11.png 2751w" sizes="(max-width: 917px) 100vw, 917px"></p>
<h3>Communication</h3>
<p>ShellTea is proxy aware malware – if direct communication fails it will try to execute the proxy aware API. Most of the API are standard and are mapped from wininet. Following a decryption of the embedded domains we get the following list:</p>
<p>telemerty-cdn-cloud[.]host, reservecdn[.]pro, wsuswin10[.]us, telemetry[.]host</p>
<p>At the time of the investigation, telemerty-cdn-cloud[.]host was used and mapped to 104.193.252[.]162.</p>
<p>Note that cdn substrings are also used in recent Fin7 attacks and it is a convenient method to bypass some of the network filters and DLP solutions.</p>
<p>ShellTea also uses ole32 Stream object functions (e.g. CMemStm::Write) to manipulate the downloaded memory stream (downloaded by InternetReadFile directly into the stream).</p>
<p><img id="__hsnew" src="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic12.png?width=961&amp;name=shelltea-pic12.png" alt="InternetReadFile stream" style="width: 961px;" width="961" srcset="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic12.png?width=481&amp;name=shelltea-pic12.png 481w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic12.png?width=961&amp;name=shelltea-pic12.png 961w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic12.png?width=1442&amp;name=shelltea-pic12.png 1442w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic12.png?width=1922&amp;name=shelltea-pic12.png 1922w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic12.png?width=2403&amp;name=shelltea-pic12.png 2403w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic12.png?width=2883&amp;name=shelltea-pic12.png 2883w" sizes="(max-width: 961px) 100vw, 961px"></p>
<p><span style="background-color: transparent;">The C2 identifies the post request using the additional optional data that is sent immediately after the headers request using </span><em style="background-color: transparent;">HttpSendRequestA</em><span style="background-color: transparent;">.</span></p>
<p>For example, in the case of ReflectivePicker download, the optional data will consist of embedded cookie and byte ‘b’ as a command.</p>
<p><img id="__hsnew" src="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic13.png?width=684&amp;name=shelltea-pic13.png" alt="ReflectivePicker download" style="width: 684px;" width="684" srcset="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic13.png?width=342&amp;name=shelltea-pic13.png 342w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic13.png?width=684&amp;name=shelltea-pic13.png 684w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic13.png?width=1026&amp;name=shelltea-pic13.png 1026w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic13.png?width=1368&amp;name=shelltea-pic13.png 1368w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic13.png?width=1710&amp;name=shelltea-pic13.png 1710w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic13.png?width=2052&amp;name=shelltea-pic13.png 2052w" sizes="(max-width: 684px) 100vw, 684px"></p>
<p>In case a buffer needs to be sent back (in case of recon data collected on the endpoint), a magic header cookie is attached to the data and sent as is (encrypted of course) through the optional buffer.</p>
<p><img id="__hsnew" src="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic14.png?width=798&amp;name=shelltea-pic14.png" alt="recon data collected on the endpoint" style="width: 798px;" width="798" srcset="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic14.png?width=399&amp;name=shelltea-pic14.png 399w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic14.png?width=798&amp;name=shelltea-pic14.png 798w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic14.png?width=1197&amp;name=shelltea-pic14.png 1197w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic14.png?width=1596&amp;name=shelltea-pic14.png 1596w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic14.png?width=1995&amp;name=shelltea-pic14.png 1995w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic14.png?width=2394&amp;name=shelltea-pic14.png 2394w" sizes="(max-width: 798px) 100vw, 798px"></p>
<h2>Recon stage</h2>
<p>One of the first artifacts the shellcode downloads is a PowerShell code and a .NET native ReflectivePicker. Because PowerShell was executed outside PowerShell (from within the Explorer process) it will bypass many of the blacklisting defenses.</p>
<h3>PowerShell download</h3>
<p>The PowerShell script collects all possible information on the user and the network, including snapshots, computer and user names, emails from registry, tasks in task scheduler, system information, AVs registered in the system, privileges, domain and workgroup information.</p>
<p><img id="__hsnew" src="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic15.png?width=1449&amp;name=shelltea-pic15.png" alt="powershell download" style="width: 1449px;" width="1449" srcset="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic15.png?width=725&amp;name=shelltea-pic15.png 725w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic15.png?width=1449&amp;name=shelltea-pic15.png 1449w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic15.png?width=2174&amp;name=shelltea-pic15.png 2174w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic15.png?width=2898&amp;name=shelltea-pic15.png 2898w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic15.png?width=3623&amp;name=shelltea-pic15.png 3623w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic15.png?width=4347&amp;name=shelltea-pic15.png 4347w" sizes="(max-width: 1449px) 100vw, 1449px"></p>
<p>The results are Gzipped and saved under random file in the temp folder. Following successful collection of information, the data is send back to the C2 and the file is deleted.</p>
<h3>Empire ReflectivePicker</h3>
<p>As mentioned previously, the PowerShell is executed using reflectively loaded ReflectivePicker from the Empire project – it loads CLR by using CorBindToRuntime that is loaded dynamically within the shellcode.</p>
<p><img id="__hsnew" src="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic16.png?width=728&amp;name=shelltea-pic16.png" alt="Empire ReflectivePicker" style="width: 728px;" width="728" srcset="https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic16.png?width=364&amp;name=shelltea-pic16.png 364w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic16.png?width=728&amp;name=shelltea-pic16.png 728w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic16.png?width=1092&amp;name=shelltea-pic16.png 1092w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic16.png?width=1456&amp;name=shelltea-pic16.png 1456w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic16.png?width=1820&amp;name=shelltea-pic16.png 1820w, https://blog.morphisec.com/hs-fs/hubfs/Blog_images/1-Q2-2019/shelltea-pic16.png?width=2184&amp;name=shelltea-pic16.png 2184w" sizes="(max-width: 728px) 100vw, 728px"></p>
<h2>Conclusion</h2>
<p>The hospitality industry, and particularly their POS networks, continues to be one of the industries most targeted by cybercrime groups. In addition to this attack by <span style="font-weight: bold;">FIN8</span>, we’ve seen multiple attacks by <a href="//blog.morphisec.com/new-global-attack-on-point-of-sale-systems">FIN6</a>, <a href="//blog.morphisec.com/fin7-not-finished-morphisec-spots-new-campaign">FIN7</a>, and others.</p>
<p>Many POS networks are running on the POS version of Windows 7, making them more susceptible to vulnerabilities. What’s more, attackers know that many POS systems run with only rudimentary security as traditional antivirus is too heavy and requires constant updating that can interfere with system availability.</p>
<p>As we see here, attack syndicates are constantly innovating and learn from their mistakes – the numerous improvements and bug fixes from the previous version of ShellTea are evident. The techniques implemented can easily evade standard POS defenses.</p>
<p>Morphisec immediately prevented this attack from ever getting to the point where it could access POS endpoints. It is lightweight, with no need for updates, and does not need to be online to provide full protection. Moreover, it serves as a compensating control for Windows 7 systems, providing a virtual patch that protects vulnerabilities.</p>
