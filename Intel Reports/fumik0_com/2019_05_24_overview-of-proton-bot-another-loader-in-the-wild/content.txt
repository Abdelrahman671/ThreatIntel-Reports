							<div itemprop="articleBody">
								<p>Loaders nowadays are part of the malware landscape and it is common to see on sandbox logs results with &#8220;loader&#8221; tagged on. Specialized loader malware like Smoke or Hancitor/Chanitor are facing more and more with new alternatives like Godzilla loader, stealers, miners and plenty other kinds of malware with this developed feature as an option. This is easily catchable and already explained in earlier articles that I have made.</p>
<p>Since a few months, another dedicated loader malware appears from multiple sources with the name of &#8220;Proton Bot&#8221; and on my side, first results were coming from a v0.30 version. For this article, the overview will focus on the latest one, the v1.</p>
<p>Sold 50$ (with C&amp;C panel) and developed in C++, its cheaper than Smoke (usually seen with an average of 200$/300$) and could explain that some actors/customers are making some changes and trying new products to see if it&#8217;s worth to continue with it. The developer behind (glad0ff), is not as his first malware, he is also behind <a href="https://tracker.fumik0.com/malware/Acrux">Acrux</a> &amp; Decrux.</p>
<p><em>[Disclamer: This article is not a deep in-depth analysis]</em></p>
<h2>Analyzed sample</h2>
<ul>
<li><a href="https://tracker.fumik0.com/sample/5cdd8a707a324f0bb7ae3451">1AF50F81E46C8E8D49C44CB2765DD71A</a> [Packed]</li>
<li><a href="https://www.virustotal.com/#/file/43a9d0ebdd41f21f8252ddb4576c7577478a14083aadeb48bff7331e49ec1580/detection">4C422E9D3331BD3F1BB785A1A4035BBD</a> [Unpacked]</li>
</ul>
<p>Something that I am finally glad by reversing this malware is that I&#8217;m not in pain for unpacking a VM protected sample. By far this is the &#8220;only one&#8221; that I&#8217;ve analyzed from this developer this is not using Themida, VMprotect or Enigma Protector.</p>
<p>So seeing finally a clean PE is some kind of heaven.</p>
<h2>Behavior</h2>

<h2>A unique way to perform data interaction</h2>
<p>This loader has an odd and unorthodox way to manipulate the data access and storage by using the <a href="https://www.win7dll.info/ktmw32_dll.html">Windows KTM library</a>. This is way more different than most of the malware that is usually using easier ways for performing tasks like creating a folder or a file by the help of the <a href="https://docs.microsoft.com/en-us/windows/desktop/api/fileapi/index">FileAPI</a> module.</p>
<p>The idea here, it is permitting a way to perform actions on data with the guarantee that there is not even a single error during the operation. For this level of reliability and integrity, the Kernel Transaction Manager (KTM) comes into play with the help of the <a href="https://technet.microsoft.com/fr-fr/office/bb968806(v=vs.80)">Transaction NTFS</a> (TxF).</p>
<p>For those who aren&#8217;t familiar with this, there is an example here :</p>
<p><img data-recalc-dims="1" loading="lazy" decoding="async" data-attachment-id="773" data-permalink="https://fumik0.com/2019/05/24/overview-of-proton-bot-another-loader-in-the-wild/transaction/" data-orig-file="https://i0.wp.com/fumik0.com/wp-content/uploads/2019/05/Transaction.png?fit=898%2C689&amp;ssl=1" data-orig-size="898,689" data-comments-opened="1" data-image-meta="{&quot;aperture&quot;:&quot;0&quot;,&quot;credit&quot;:&quot;&quot;,&quot;camera&quot;:&quot;&quot;,&quot;caption&quot;:&quot;&quot;,&quot;created_timestamp&quot;:&quot;0&quot;,&quot;copyright&quot;:&quot;&quot;,&quot;focal_length&quot;:&quot;0&quot;,&quot;iso&quot;:&quot;0&quot;,&quot;shutter_speed&quot;:&quot;0&quot;,&quot;title&quot;:&quot;&quot;,&quot;orientation&quot;:&quot;0&quot;}" data-image-title="Transaction" data-image-description="" data-image-caption="" data-medium-file="https://i0.wp.com/fumik0.com/wp-content/uploads/2019/05/Transaction.png?fit=300%2C230&amp;ssl=1" data-large-file="https://i0.wp.com/fumik0.com/wp-content/uploads/2019/05/Transaction.png?fit=898%2C689&amp;ssl=1" class="alignnone size-full wp-image-773" src="https://i0.wp.com/fumik0.com/wp-content/uploads/2019/05/Transaction.png?resize=898%2C689&#038;ssl=1" alt="Transaction" width="898" height="689" srcset="https://i0.wp.com/fumik0.com/wp-content/uploads/2019/05/Transaction.png?w=898&amp;ssl=1 898w, https://i0.wp.com/fumik0.com/wp-content/uploads/2019/05/Transaction.png?resize=300%2C230&amp;ssl=1 300w, https://i0.wp.com/fumik0.com/wp-content/uploads/2019/05/Transaction.png?resize=768%2C589&amp;ssl=1 768w" sizes="auto, (max-width: 898px) 100vw, 898px" /></p>
<ol>
<li><a href="https://docs.microsoft.com/en-us/windows/desktop/api/ktmw32/nf-ktmw32-createtransaction">CreateTransaction</a> is called for starting the transaction process</li>
<li>The requested task is now called</li>
<li>If everything is good, the Transaction is finalized with a commit (<a href="https://docs.microsoft.com/en-us/windows/desktop/api/ktmw32/nf-ktmw32-committransaction">CommitTransaction</a>) and confirming the operation is a success</li>
<li>If a single thing failed (even 1 among 10000 tasks), the transaction is rolled back with <a href="https://docs.microsoft.com/en-us/windows/desktop/api/ktmw32/nf-ktmw32-rollbacktransaction">RollbackTransaction</a></li>
</ol>
<p>In the end, this is the task list used by ProtonBot are:</p>
<ul>
<li><a href="https://docs.microsoft.com/en-us/windows/desktop/api/winbase/nf-winbase-deletefiletransacteda">DeleteFileTransactedA</a></li>
<li><a href="https://docs.microsoft.com/en-us/windows/desktop/api/winbase/nf-winbase-copyfiletransacteda">CopyFileTransactedA</a></li>
<li><a href="https://docs.microsoft.com/en-us/windows/desktop/api/winbase/nf-winbase-setfileattributestransacteda">SetFileAttributesTransactedA</a></li>
<li><a href="https://docs.microsoft.com/en-us/windows/desktop/api/winbase/nf-winbase-createdirectorytransacteda">CreateDirectoryTransactedA</a></li>
</ul>
<p>This different way to interact with the Operating System is a nice way to escape some API monitoring or avoiding triggers from sandboxes &amp; specialized software. It&#8217;s a matter time now to hotfix and adjusts this behavior for having better results.</p>
<p><em>The API used has been also used for another technique with analysis of the banking malware <a href="https://blog.malwarebytes.com/threat-analysis/2018/08/process-doppelganging-meets-process-hollowing_osiris/">Osiris</a> by <a href="https://twitter.com/hasherezade">@hasherezade</a></em></p>

<h2>Other features</h2>
<h3>Clipper</h3>
<p>Clipper fundamentals are always the same and at that point now, I&#8217;m mostly interested in how the developer decided to organize this task. On this case, this is simplest but enough to performs accurately some stuff.</p>
<p>The first main thing to report about it, it that the wallets and respective regular expressions for detecting them are not hardcoded into the source code and needs to perform an HTTP request only once on the C&amp;C for setting-up this :</p>
<pre>/page.php?id=<strong>%GUID%</strong>&amp;clip=get</pre>
<p>The response is a consolidated list of a homemade structure that contains the configuration decided by the attacker. The format is represented like this:</p>
<pre>[
  id,             <span style="color: #008000;"><em><strong># ID on C&amp;C</strong></em></span>
  name,           <em><strong><span style="color: #008000;"># ID Name (i.e: Bitcoin)</span></strong></em>
  regex,          <em><strong><span style="color: #008000;"># Regular Expression for catching the Wallet</span></strong></em>
  attackerWallet  <span style="color: #008000;"><em><strong># Switching victim wallet with this one</strong></em></span>
]</pre>
<p>At first, I thought, there is a request to the C&amp;C when the clipper triggered a matched regular expression, but it&#8217;s not the case here.</p>
<p>On this case, the attacker has decided to target some wallets:</p>
<ul>
<li>Bitcoin</li>
<li>Dash</li>
<li>Litecoin</li>
<li>Zcash</li>
<li>Ethereum</li>
<li>DogeCoin</li>
</ul>
<p><em>if you want an in-depth analysis of a clipper task, I recommend you to check my other articles that mentioned in details this (<a href="https://fumik0.com/2019/05/03/lets-nuke-megumin-trojan/">Megumin</a> &amp; <a href="https://fumik0.com/2019/03/25/lets-play-with-qulab-an-exotic-malware-developed-in-autoit/">Qulab</a>).</em></p>

<h2><strong>Conclusion</strong></h2>
<p>Young malware means fresh content and with time and luck, could impact the malware landscape. This loader is cheap and will probably draw attention to some customers (or even already the case), to have less cost to maximize profits during attacks. ProtonBot is not a sophisticated malware but it&#8217;s doing its job with extra modules for probably being more attractive. Let&#8217;s see with the time how this one will evolve, but by seeing some kind of odd cases with plenty of different malware pushed by this one, that could be a scenario among others that we could see in the future.</p>
