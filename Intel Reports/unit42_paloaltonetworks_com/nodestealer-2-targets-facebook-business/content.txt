We will provide recommendations for how Facebook business account owners can protect their accounts.\r\n\r\nWhile this specific campaign is no longer active, we have indications that the threat actors behind it may continue to use and evolve NodeStealer or use similar techniques to continue targeting Facebook business accounts. It is also possible that there may be ongoing effects for previously compromised organizations.\r\n\r\nPalo Alto Networks customers also receive protections against NodeStealer in the following ways:\r\n\r\n \tOrganizations can engage the Unit 42 Incident Response team for specific assistance with this threat and others.\r\n \tCortex XDR and XSIAM agents help protect against the threats discussed in this article, providing a multilayer defense that includes behavioral threat protection and exploit protection.\r\n \tThe Advanced WildFire cloud-delivered malware analysis service accurately identifies known samples related to these threats as malicious.\r\n \tAdvanced URL Filtering and DNS Security identify URLs and domains associated with this campaign as malicious.\r\n \tNext-Generation Firewall with Advanced Threat Prevention security subscriptions can help block samples.\r\n\r\n\r\n\r\n\r\nRelated Unit 42 Topics\r\nInfostealer, Phishing\r\n\r\n\r\n\r\nPhishing Campaign\r\nFrom the telemetry available to us, the main infection vector for the infostealer was a phishing campaign. The phishing campaign took place around December of 2022 and was used for delivering two variants of the stealer, which we will refer to as Variant #1 and Variant #2. The differences between them will be described in the next sections of this article.\r\n\r\nThe main theme of the campaign was advertising materials for businesses. The threat actor used multiple Facebook pages and users to post information luring victims to download a link from known cloud file storage providers. After clicking on it, a .zip file was downloaded to the machine, containing the malicious infostealer executable.\r\n\r\n[caption id=\"attachment_129365\" align=\"aligncenter\" width=\"900\"] Figure 1. Facebook phishing post luring victims to download the infected .zip file.[\/caption]\r\nVariant #1 Analysis\r\nThe first variant of the infostealer in the campaign was internally named word.exe. It was compiled with Nuitka, and the threat actor used a unique product name for the files: Peguis.\r\n\r\n[caption id=\"attachment_129367\" align=\"aligncenter\" width=\"305\"] Figure 2. Metadata for word.exe.[\/caption]\r\n\r\nVariant #1\u2019s process tree is quite \u201cnoisy,\u201d meaning it creates multiple processes and performs many actions that are considered as indications of abnormal activity, and not very clandestine, including pop-up windows presented to the user.\r\nMain Features\r\nAs mentioned earlier, NodeStealer targets Facebook business accounts. Variant #1 has some additional features that enable it to do much more than that. Here are the main features of Variant #1:\r\n\r\n \tStealing Facebook business account information\r\n \tDownloading additional malware\r\n \tDisabling Windows Defender via GUI (graphical user interface)\r\n \tMetaMask (cryptocurrency wallet) theft\r\n\r\nStealing Facebook Business Account Information\r\nThe first thing the malware does when executing is check if there is a Facebook business account logged in to the default browser on the infected machine. It does that by connecting to https:\/\/business.facebook.com\/ads\/ad_limits\/ and checking the header.\r\n\r\n[caption id=\"attachment_129369\" align=\"aligncenter\" width=\"900\"] Figure 3. Stealing information using Facebook\u2019s Graph API.[\/caption]\r\n\r\nIf there is indeed a Facebook business account logged in, the malware connects to the Graph API \u2013 graph.facebook.com \u2013 with the user ID and the access token stolen from the header.\r\n\r\nAccording to Meta, \u201cThe Graph API is the primary way to get data into and out of the Facebook platform. It's an HTTP-based API that apps can use to programmatically query data, post new stories, manage ads, upload photos, and perform a wide variety of other tasks.\u201d\r\n\r\nNodeStealer uses the Graph API to steal information about the target, including: followers count, user verification status, account credit balance, if the account is prepaid, and ads information.\r\n\r\nThe malware also gets the content of a Facebook JavaScript module AdsLWIDescribeCustomersContainer by sending a request to https:\/\/www.facebook.com\/ajax\/bootloader-endpoint\/?modules=AdsLWIDescribeCustomersContainer.react.\r\n\r\nThis JavaScript module is a part of Facebook's advertising platform and is used for describing and managing custom audiences in Facebook Ads. Custom audiences allow advertisers to target specific groups of people based on their demographics, interests, behaviors or other criteria. The malware steals this information and sends it to its command and control server (C2).\r\n\r\nIn addition to stealing information about the Facebook business account, the malware also aims to steal those accounts credentials. In order to do so, it checks for Facebook users and passwords within the cookies and local databases of the following browsers: Chrome, Edge, C\u1ed1c C\u1ed1c, Brave and Firefox.\r\n\r\n[caption id=\"attachment_129371\" align=\"aligncenter\" width=\"900\"] Figure 4. Stealing passwords from browsers\u2019 databases.[\/caption]\r\n\r\n[caption id=\"attachment_129373\" align=\"aligncenter\" width=\"900\"] Figure 5. Alerts for the execution of NodeStealer, as shown in Cortex XDR.[\/caption]\r\n\r\nThe malware then exfiltrates the output files through Telegram and deletes the files to remove its tracks:\r\n\r\n[caption id=\"attachment_129375\" align=\"aligncenter\" width=\"684\"] Figure 6. Exfiltration through Telegram.[\/caption]\r\n\r\n[caption id=\"attachment_129377\" align=\"aligncenter\" width=\"474\"] Figure 7. Tracks removal by NodeStealer.[\/caption]\r\nDownloading Additional Malware\r\nVariant #1 is configured to download two .zip files from the following URLs:\r\n\r\n \thxxps:\/\/tinyurl[.]com\/batkyc, which redirects to hxxp:\/\/adgowin66[.]site\/ratkyc\/4\/bat.zip\r\n \thxxps:\/\/tinyurl[.]com\/ratkyc2, which redirects to hxxp:\/\/adgowin66[.]site\/ratkyc\/4\/ratkyc.zip\r\n\r\nBat.zip contains the ToggleDefender batch script that disables Windows Defender, and Ratkyc.zip contains three pieces of malware:\r\n\r\n \tBitRAT named COM Surrogate.exe\r\n \tA hidden virtual network computing (hVNC) RAT named Antimalware Service Executable.exe\r\n \tXWorm named Host Process for Windows Tasks.exe\r\n\r\nIn order to download the .zip files, the malware implements the FodHelper UAC bypass. Using this method, the attackers attempt to bypass User Account Control (UAC) and execute the PowerShell scripts used to download the above-mentioned zip files.\r\n\r\n[caption id=\"attachment_129379\" align=\"aligncenter\" width=\"900\"] Figure 8. This is a very noisy approach, since the end user would be able to see the Windows Defender GUI pop up on the machine and the malware acting to disable it.\r\n\r\nThe commands used to open the GUI and disable Windows Defender are shown in Figure 10 below.\r\n\r\n[caption id=\"attachment_129385\" align=\"aligncenter\" width=\"774\"] Figure 10. Commands used to disable Windows Defender.[\/caption]\r\nMetaMask Theft\r\nThe malware also tries to maximize financial gain by stealing MetaMask credentials from Chrome, C\u1ed1c C\u1ed1c and Brave browsers.\r\n\r\nMetaMask is an extension for accessing Ethereum Wallets through the browser. Stealing credentials for this application allows the attackers to steal cryptocurrency from the user\u2019s wallets.\r\n\r\nJust as it did in stealing Facebook cookies and credentials, the malware extracts the local databases used to store browsers\u2019 information. It searches within them for the extension nkbihfbeogaeaoehlefnkodbefgpgknn, which is the extension of MetaMask when installed directly from the extension store.\r\n\r\nThen, the malware copies the data into a file and exfiltrates it using Telegram, in the same fashion it did with the Facebook credentials.\r\n\r\n[caption id=\"attachment_129387\" align=\"aligncenter\" width=\"833\"] Figure 11. Stealing MetaMask credentials from a Brave browser.[\/caption]\r\nVariant #2 Analysis\r\nThe second variant of the infostealer in the campaign was internally named MicrosofOffice.exe and was compiled with Nuitka, same as the first variant. Unlike the first variant, it does not generate a lot of activity visible to the unsuspecting user. For this variant, the threat actor used the product name \u201cMicrosoft Coporation\u201d (originally misspelled by the malware authors).\r\n\r\n[caption id=\"attachment_129389\" align=\"aligncenter\" width=\"357\"] Figure 12. Metadata of Variant #2 masquerading as MicrosofOffice.exe.[\/caption]\r\nMain Features\r\nLike the first variant, Variant #2 targets Facebook business account information and MetaMask wallets, but it goes beyond by:\r\n\r\n \tAttempting to take over the Facebook account\r\n \tImplementing anti-analysis features\r\n \tStealing emails\r\n\r\nTaking Over the Facebook Account\r\nVariant #2 attempts to purchase an online email service provided by a legitimate Vietnamese website (hotmailbox[.]me). It attempts to do so using an embedded API key that holds a credit balance for that specific service: https:\/\/api.hotmailbox[.]me\/mail\/buy?apikey=&lt;redacted&gt;&amp;mailcode=HOTMAIL&amp;quantity=1.\r\n\r\n[caption id=\"attachment_129391\" align=\"aligncenter\" width=\"900\"] Figure 13. Purchasing mailbox service from hotmailbox[.]me.[\/caption]\r\n\r\n[caption id=\"attachment_129393\" align=\"aligncenter\" width=\"900\"] Figure 14. Credit balance for the API key used by the malware.[\/caption]\r\n\r\nIf the purchase attempt is unsuccessful, the malware tries to purchase a mailbox service from another Vietnamese website (dongvanfb[.]net), again, using an API key that holds a dedicated credit balance \u2014 https:\/\/api.dongvanfb[.]net\/user\/buy?apikey=&lt;redacted&gt;&amp;account_type=1&amp;quality=1.\r\n\r\n[caption id=\"attachment_129395\" align=\"aligncenter\" width=\"900\"] Figure 15. Purchasing mailbox service from dongvanfb[.]net.[\/caption]If the purchase attempt succeeds, the malware saves the email and password for the new mailbox, which will be used in the next phase of the campaign.\r\n\r\nNext, the malware modifies the account email address for the Facebook business account of the victim, using a technique that doesn\u2019t require verifying the password using the following URL: https:\/\/www.facebook[.]com\/add_contactpoint\/dialog\/submit\/.\r\n\r\nIf needed, the malware sends a request to get the Facebook authentication code via email by sending a request to: https:\/\/getcode.hotmailbox[.]me.\r\n\r\n[caption id=\"attachment_129397\" align=\"aligncenter\" width=\"900\"] Figure 16. Code for requesting the Facebook authentication code from hotmailbox[.]me.[\/caption]The malware then checks the updated email to see if the modification was successful:\r\n\r\n[caption id=\"attachment_129399\" align=\"aligncenter\" width=\"900\"] Figure 17. Checking the updated email for the Facebook account.[\/caption]\r\n\r\nIf successful, the attackers have now taken over the Facebook account by replacing the legitimate user\u2019s email address with a mailbox under their control.\r\nReading Emails\r\nIn addition, the malware has a function that parses emails, so it can read the victim\u2019s emails. It is possible that the threat actor added this functionality to potentially interfere with any Facebook alerts notifying the victim of the configuration changes, though we did not directly observe activity of this kind.\r\n\r\n[caption id=\"attachment_129401\" align=\"aligncenter\" width=\"752\"] Figure 18. Function that is responsible for reading emails.[\/caption]\r\nAnti Analysis and Anti VM\r\nIn several samples of Variant #2 that were analyzed, the threat actor added a simple function to check for the presence of several malware analysis tools and virtual machine processes. Anti-VM and anti-analysis function.[\/caption]\r\nDifferences Between the NodeStealer Variants\r\nAs mentioned above, there are similarities between the two variants of NodeStealer analyzed in this article, but there are many differences as well. Comparison of NodeStealer and the two variants.\r\n\r\nVietnamese Threat Actor\r\nInterestingly, both Ducktail and NodeStealer were previously suspected by Meta to originate from threat actors based in Vietnam.\r\n\r\nThe suspected connection between the NodeStealer malware and a Vietnamese threat actor can be explained in different ways.\r\n\r\nThe first finding that may indicate this connection is that in the Python script of both variants analyzed in this blog, we came across many strings in Vietnamese. For example, see Figures 20 and 21.\r\n\r\n[caption id=\"attachment_129459\" align=\"aligncenter\" width=\"600\"] Figure 20. Translation of the string \u201cTongChiTieu\u201d found in NodeStealer.[\/caption]\r\n\r\n[caption id=\"attachment_129461\" align=\"aligncenter\" width=\"600\"] Figure 21. Wikipedia description for C\u1ed1c C\u1ed1c software.[\/caption]\r\n\r\nThe third indication of a suspected Vietnamese connection to NodeStealer was found in Variant #2. This variant, as described earlier in the article, attempts to purchase an online mailbox service from two different Vietnamese websites: Hotmailbox[.]me and Dongvanfb[.]net.\r\nConclusion\r\nIn this article, we uncovered a campaign of the NodeStealer malware that targets Facebook business accounts. As part of the campaign, two variants of NodeStealer were discovered, Variant #1 and Variant #2. Analyzing the two variants revealed some interesting behavior of the malware that includes doing much more than its original intentions, all likely to increase the potential profit for the threat actor.\r\n\r\nThe threat actor, who is suspected to be of Vietnamese origin, provided the new variants with cryptocurrency stealing capabilities, downloader capabilities and the ability to fully take over Facebook business accounts. The potential damage for both individuals and organizations can be reflected not only in financial loss, but also in reputation damage for a target.\r\n\r\nWe encourage all organizations to review their protection policies and use the indicators of compromise (IoCs) provided in this report in order to address this threat. Facebook business account owners are encouraged to use strong passwords and enable multifactor authentication. Take the time to provide education for your organization on phishing tactics, especially modern, targeted approaches that play off current events, business needs and other appealing topics.\r\nProtections and Mitigations\r\nSmartScore, a unique ML-driven scoring engine that translates security investigation methods and their associated data into a hybrid scoring system, scored an incident involving NodeStealer an 86 out of 100, as shown in Figure 23. <p>Unit 42 researchers have recently discovered a previously unreported phishing campaign that distributed an infostealer equipped to fully take over Facebook business accounts. Facebook business accounts were targeted with a phishing lure offering tools such as spreadsheet templates for business. This is part of a growing trend of threat actors targeting Facebook business accounts – for advertising fraud and other purposes –  which emerged around July 2022 with the discovery of the <a href="https://www.withsecure.com/en/expertise/research-and-innovation/research/ducktail-an-infostealer-malware" target="_blank" rel="noopener">Ducktail</a> infostealer.</p>
<p>About eight months later, in March 2023, <a href="https://labs.guard.io/fakegpt-new-variant-of-fake-chatgpt-chrome-extension-stealing-facebook-ad-accounts-with-4c9996a8f282" target="_blank" rel="noopener">FakeGPT</a>, a new variant of a fake ChatGPT Chrome extension that steals Facebook Ad accounts, was reported. Unit 42 also reported on <a href="https://unit42.paloaltonetworks.com/chatgpt-scam-attacks-increasing/" target="_blank" rel="noopener">ChatGPT-themed scam attacks</a> in April 2023. In May 2023, a report from Meta of new information-stealing malware named <a href="https://engineering.fb.com/2023/05/03/security/malware-nodestealer-ducktail/" target="_blank" rel="noopener">NodeStealer</a> surfaced, which described malware that was <a href="https://www.virustotal.com/gui/file/fbb887dbcbac146adde68ae15bf0fadab144f2ed60e2db62a6acf5f13e31b80e/detection" target="_blank" rel="noopener">compiled in July 2022</a> and malicious activity involving NodeStealer that was identified in January 2023. NodeStealer allowed threat actors to steal browser cookies to hijack accounts on the platform, specifically aiming toward business accounts.</p>
<p>While investigating the growing trend, we came across a campaign that started around December 2022, and has not been previously reported.</p>
<p>The infostealer distributed in the campaign shares multiple similarities with the NodeStealer variant compiled in July 2022 that Meta analyzed, which was written in JavaScript. However, the new campaign involved two variants written in Python, improved with additional features to benefit the threat actors. The threat actor equipped these variants with cryptocurrency stealing capabilities, downloader capabilities and the ability to fully take over Facebook business accounts.</p>
<p>NodeStealer poses great risk for both individuals and organizations. Besides the direct impact on Facebook business accounts, which is mainly financial, the malware also steals credentials from browsers, which can be used for further attacks.</p>
<p>In this article, we will shed some light on the unreported phishing campaign targeting Facebook business accounts and will provide a deep dive analysis of the malware. In addition, we will show the execution of the malware through the lens of Cortex XDR (set to detect-only mode). We will provide recommendations for how Facebook business account owners can protect their accounts.</p>
<p>While this specific campaign is no longer active, we have indications that the threat actors behind it may continue to use and evolve NodeStealer or use similar techniques to continue targeting Facebook business accounts. It is also possible that there may be ongoing effects for previously compromised organizations.</p>

<h2><a id="post-129362-_bvpzvte0l38"></a><strong>Phishing Campaign</strong></h2>
<p>From the telemetry available to us, the main infection vector for the infostealer was a phishing campaign. The phishing campaign took place around December of 2022 and was used for delivering two variants of the stealer, which we will refer to as Variant #1 and Variant #2. The differences between them will be described in the next sections of this article.</p>
<p>The main theme of the campaign was advertising materials for businesses. The threat actor used multiple Facebook pages and users to post information luring victims to download a link from known cloud file storage providers. After clicking on it, a <span style="font-family: 'courier new', courier, monospace;">.zip</span> file was downloaded to the machine, containing the malicious infostealer executable.</p>

<h2><a id="post-129362-_imglkjgcpaxt"></a><strong>Variant #1 Analysis</strong></h2>
<p>The first variant of the infostealer in the campaign was internally named <span style="font-family: 'courier new', courier, monospace;">word.exe</span>. It was compiled with <a href="https://github.com/Nuitka/Nuitka" target="_blank" rel="noopener">Nuitka</a>, and the threat actor used a unique product name for the files: Peguis.</p>
<figure id="attachment_129367" aria-describedby="caption-attachment-129367" style="width: 305px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-129367" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/07/word-image-129362-2.png" alt="Image 2 is a screenshot is a screenshot of the metadata for word.exe. <p>Variant #1’s process tree is quite “noisy,” meaning it creates multiple processes and performs many actions that are considered as indications of abnormal activity, and not very clandestine, including pop-up windows presented to the user.</p>
<h3><a id="post-129362-_x556i87m8f36"></a>Main Features</h3>
<p>As mentioned earlier, NodeStealer targets Facebook business accounts. Variant #1 has some additional features that enable it to do much more than that. Here are the main features of Variant #1:</p>
<ul>
<li>Stealing Facebook business account information</li>
<li>Downloading additional malware</li>
<li>Disabling Windows Defender via GUI (graphical user interface)</li>
<li>MetaMask (cryptocurrency wallet) theft</li>
</ul>
<h4><a id="post-129362-_1v8riw2bvhwb"></a>Stealing Facebook Business Account Information</h4>
<p>The first thing the malware does when executing is check if there is a Facebook business account logged in to the default browser on the infected machine. It does that by connecting to <span style="font-family: 'courier new', courier, monospace;">https://business.facebook.com/ads/ad_limits/</span> and checking the header.</p>
<figure id="attachment_129369" aria-describedby="caption-attachment-129369" style="width: 900px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-129369" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/07/word-image-129362-3.png" alt="Image 3 screenshot of many lines of code of Facebook's Graph API. Three areas are highlighted in red where information is being stolen." width="900" height="374" /><figcaption id="caption-attachment-129369" class="wp-caption-text">Figure 3. Stealing information using Facebook’s Graph API.</figcaption></figure>
<p>If there is indeed a Facebook business account logged in, the malware connects to the Graph API – <span style="font-family: 'courier new', courier, monospace;">graph.facebook.com</span> – with the user ID and the access token stolen from the header.</p>
<p><a href="https://developers.facebook.com/docs/graph-api/" target="_blank" rel="noopener">According to Meta</a>, “The Graph API is the primary way to get data into and out of the Facebook platform. It's an HTTP-based API that apps can use to programmatically query data, post new stories, manage ads, upload photos, and perform a wide variety of other tasks.”</p>
<p>NodeStealer uses the Graph API to steal information about the target, including: followers count, user verification status, account credit balance, if the account is prepaid, and ads information.</p>
<p>The malware also gets the content of a Facebook JavaScript module <span style="font-family: 'courier new', courier, monospace;">AdsLWIDescribeCustomersContainer</span> by sending a request to <span style="font-family: 'courier new', courier, monospace;">https://www.facebook.com/ajax/bootloader-endpoint/?modules=AdsLWIDescribeCustomersContainer.react</span>.</p>
<p>This JavaScript module is a part of Facebook's advertising platform and is used for describing and managing custom audiences in Facebook Ads. Custom audiences allow advertisers to target specific groups of people based on their demographics, interests, behaviors or other criteria. The malware steals this information and sends it to its command and control server (C2).</p>
<p>In addition to stealing information about the Facebook business account, the malware also aims to steal those accounts credentials. In order to do so, it checks for Facebook users and passwords within the cookies and local databases of the following browsers: Chrome, Edge, Cốc Cốc, Brave and Firefox.</p>

<h4><a id="post-129362-_tjfharfzhopj"></a>Downloading Additional Malware</h4>
<p>Variant #1 is configured to download two <span style="font-family: 'courier new', courier, monospace;">.zip</span> files from the following URLs:</p>
<ul>
<li><span style="font-family: 'courier new', courier, monospace;">hxxps://tinyurl[.]com/batkyc</span>, which redirects to <span style="font-family: 'courier new', courier, monospace;">hxxp://adgowin66[.]site/ratkyc/4/bat.zip</span></li>
<li><span style="font-family: 'courier new', courier, monospace;">hxxps://tinyurl[.]com/ratkyc2</span>, which redirects to <span style="font-family: 'courier new', courier, monospace;">hxxp://adgowin66[.]site/ratkyc/4/ratkyc.zip</span></li>
</ul>
<p><span style="font-family: 'courier new', courier, monospace;">Bat.zip</span> contains the <a href="https://github.com/AveYo/LeanAndMean" target="_blank" rel="noopener">ToggleDefender</a> batch script that disables Windows Defender, and <span style="font-family: 'courier new', courier, monospace;">Ratkyc.zip</span> contains three pieces of malware:</p>
<ul>
<li><a href="https://malpedia.caad.fkie.fraunhofer.de/details/win.bit_rat" target="_blank" rel="noopener">BitRAT</a> named <span style="font-family: 'courier new', courier, monospace;">COM Surrogate.exe</span></li>
<li>A hidden virtual network computing (hVNC) RAT named <span style="font-family: 'courier new', courier, monospace;">Antimalware Service Executable.exe</span></li>
<li><a href="https://malpedia.caad.fkie.fraunhofer.de/details/win.xworm" target="_blank" rel="noopener">XWorm</a> named <span style="font-family: 'courier new', courier, monospace;">Host Process for Windows Tasks.exe</span></li>
</ul>
<p>In order to download the <span style="font-family: 'courier new', courier, monospace;">.zip</span> files, the malware implements the <a href="https://winscripting.blog/2017/05/12/first-entry-welcome-and-uac-bypass/" target="_blank" rel="noopener">FodHelper UAC bypass</a>. Using this method, the attackers attempt to bypass User Account Control (UAC) and execute the PowerShell scripts used to download the above-mentioned zip files.</p>

<h4><a id="post-129362-_1kmu84noag0z"></a>MetaMask Theft</h4>
<p>The malware also tries to maximize financial gain by stealing <a href="https://metamask.io/" target="_blank" rel="noopener">MetaMask</a> credentials from Chrome, Cốc Cốc and Brave browsers.</p>
<p>MetaMask is an extension for accessing Ethereum Wallets through the browser. Stealing credentials for this application allows the attackers to steal cryptocurrency from the user’s wallets.</p>
<p>Just as it did in stealing Facebook cookies and credentials, the malware extracts the local databases used to store browsers’ information. It searches within them for the extension <span style="font-family: 'courier new', courier, monospace;">nkbihfbeogaeaoehlefnkodbefgpgknn</span>, which is the extension of MetaMask when installed directly from the extension store.</p>
<p>Then, the malware copies the data into a file and exfiltrates it using Telegram, in the same fashion it did with the Facebook credentials.</p>

<h2><a id="post-129362-_ctjelejo6iag"></a><strong>Variant #2 Analysis</strong></h2>
<p>The second variant of the infostealer in the campaign was internally named <span style="font-family: 'courier new', courier, monospace;">MicrosofOffice.exe</span> and was compiled with <a href="https://github.com/Nuitka/Nuitka" target="_blank" rel="noopener">Nuitka</a>, same as the first variant. Unlike the first variant, it does not generate a lot of activity visible to the unsuspecting user. For this variant, the threat actor used the product name “Microsoft Coporation” (originally misspelled by the malware authors).</p>
<figure id="attachment_129389" aria-describedby="caption-attachment-129389" style="width: 357px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-129389" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/07/word-image-129362-13.png" alt="Image 12 is a screenshot of the metadata of variant 2 of the NodeStealer malware. <h3><a id="post-129362-_ykr1ev3sxf70"></a>Main Features</h3>
<p>Like the first variant, Variant #2 targets Facebook business account information and MetaMask wallets, but it goes beyond by:</p>
<ul>
<li>Attempting to take over the Facebook account</li>
<li>Implementing anti-analysis features</li>
<li>Stealing emails</li>
</ul>
<h4><a id="post-129362-_w09mtcl0qo6i"></a>Taking Over the Facebook Account</h4>
<p>Variant #2 attempts to purchase an online email service provided by a legitimate Vietnamese website (<span style="font-family: 'courier new', courier, monospace;">hotmailbox[.]me</span>). It attempts to do so using an embedded API key that holds a credit balance for that specific service: <span style="font-family: 'courier new', courier, monospace;">https://api.hotmailbox[.]me/mail/buy?apikey=&lt;redacted&gt;&amp;mailcode=HOTMAIL&amp;quantity=1.</span></p>
<figure id="attachment_129391" aria-describedby="caption-attachment-129391" style="width: 900px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-129391" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/07/word-image-129362-14.png" alt="Image 13 is a screenshot of many lines of code. Here is where the NodeStealer variant is attempting to purchase a mailbox service." width="900" height="318" /><figcaption id="caption-attachment-129391" class="wp-caption-text">Figure 13. Purchasing mailbox service from <span style="font-family: 'courier new', courier, monospace;">hotmailbox[.]me.</span></figcaption></figure>
<figure id="attachment_129393" aria-describedby="caption-attachment-129393" style="width: 900px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-129393" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/07/word-image-129362-15.png" alt="Image 14 is a screenshot of a browser address where some of the information has been redacted and is blurred. <p>If the purchase attempt is unsuccessful, the malware tries to purchase a mailbox service from another Vietnamese website (<span style="font-family: 'courier new', courier, monospace;">dongvanfb[.]net</span>), again, using an API key that holds a dedicated credit balance — <span style="font-family: 'courier new', courier, monospace;">https://api.dongvanfb[.]net/user/buy?apikey=&lt;redacted&gt;&amp;account_type=1&amp;quality=1</span>.</p>

<p>Next, the malware modifies the account email address for the Facebook business account of the victim, using a <a href="https://en.veryfb.com/d/45-modifying-the-account-email-address-without-verifying-the-password" target="_blank" rel="noopener">technique</a> that doesn’t require verifying the password using the following URL: <span style="font-family: 'courier new', courier, monospace;">https://www.facebook[.]com/add_contactpoint/dialog/submit/</span>.</p>
<p>If needed, the malware sends a request to get the Facebook authentication code via email by sending a request to: <span style="font-family: 'courier new', courier, monospace;">https://getcode.hotmailbox[.]me</span>.</p>

<h4><a id="post-129362-_dcps7che58j"></a>Reading Emails</h4>

It is possible that the threat actor added this functionality to potentially interfere with any Facebook alerts notifying the victim of the configuration changes, though we did not directly observe activity of this kind.</p>

<h2><a id="post-129362-_old5wbt0byny"></a><strong>Conclusion</strong></h2>
<p>In this article, we uncovered a campaign of the NodeStealer malware that targets Facebook business accounts. As part of the campaign, two variants of NodeStealer were discovered, Variant #1 and Variant #2. Analyzing the two variants revealed some interesting behavior of the malware that includes doing much more than its original intentions, all likely to increase the potential profit for the threat actor.</p>
<p>The threat actor, who is suspected to be of Vietnamese origin, provided the new variants with cryptocurrency stealing capabilities, downloader capabilities and the ability to fully take over Facebook business accounts. The potential damage for both individuals and organizations can be reflected not only in financial loss, but also in reputation damage for a target.</p>
<p>We encourage all organizations to review their protection policies and use the indicators of compromise (IoCs) provided in this report in order to address this threat. Facebook business account owners are encouraged to use strong passwords and enable multifactor authentication. Take the time to provide education for your organization on phishing tactics, especially modern, targeted approaches that play off current events, business needs and other appealing topics.</p>

		}

		$('.error-mail').text("").removeClass('error-show');;

	function isEmail(email) {
		var re = /^(([^<>()\[\]\\.,;:\s@"]+(\.[^<>()\[\]\\.,;:\s@"]+)*)|(".+"))@((\[[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\])|(([a-zA-Z\-0-9]+\.)+[a-zA-Z]{2,}))$/;

	function maskEmailAddress (emailAddress) {
		function mask(str) {
			var strLen = str.length;
			if (strLen > 4) {
				return str.substr(0, 1) + str.substr(1, strLen - 1).replace(/\w/g, '*') + str.substr(-1,1);
			}
			return str.replace(/\w/g, '*');
		}
		return emailAddress.replace(/([\w.]+)@([\w.]+)(\.[\w.]+)/g, function (m, p1, p2, p3) {
			return mask(p1) + '@' + mask(p2) + p3;
		});

		return emailAddress;
	}
