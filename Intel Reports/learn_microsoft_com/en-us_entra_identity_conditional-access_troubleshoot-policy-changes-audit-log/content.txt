											<h2 id="ms--in-this-article" class="title is-6 margin-block-xs">In this article</h2>
										</nav><!-- <content> --><p>The Microsoft Entra audit log is a valuable source of information when troubleshooting why and how Conditional Access policy changes happened in your environment.</p>
<p>Audit log data is only kept for 30 days by default, which might not be long enough for every organization. Organizations can store data for longer periods by changing diagnostic settings in Microsoft Entra ID to:</p>
<ul>
<li>Send data to a Log Analytics workspace</li>
<li>Archive data to a storage account</li>
<li>Stream data to Event Hubs</li>
<li>Send data to a partner solution</li>
</ul>
<p>Find these options under <strong>Identity</strong> &gt; <strong>Monitoring &amp; health</strong> &gt; <strong>Diagnostic settings</strong> &gt; <strong>Edit setting</strong>. If you don't have a diagnostic setting, follow the instructions in the article <a href="/en-us/azure/azure-monitor/essentials/diagnostic-settings" data-linktype="absolute-path">Create diagnostic settings to send platform logs and metrics to different destinations</a> to create one.</p>
<h2 id="use-the-audit-log">Use the audit log</h2>
<ol>
<li><p>Sign in to the <a href="https://entra.microsoft.com" data-linktype="external">Microsoft Entra admin center</a> as at least a <a href="../role-based-access-control/permissions-reference#reports-reader" data-linktype="relative-path">Reports Reader</a>.</p>
</li>
<li><p>Browse to <strong>Identity</strong> &gt; <strong>Monitoring &amp; health</strong> &gt; <strong>Audit logs</strong>.</p>
</li>
<li><p>Select the <strong>Date</strong> range you want to query.</p>
</li>
<li><p>From the <strong>Service</strong> filter, select <strong>Conditional Access</strong> and select the <strong>Apply</strong> button.</p>
<p>The audit logs display all activities, by default. Open the <strong>Activity</strong> filter to narrow down the activities. For a full list of the audit log activities for Conditional Access, see the <a href="../monitoring-health/reference-audit-activities#conditional-access" data-linktype="relative-path">Audit log activities</a>.</p>
</li>
<li><p>To view the details, select a row. The <strong>Modified Properties</strong> tab lists the modified JSON values for the selected audit activity.</p>
</li>
</ol>

</span>

<h2 id="use-log-analytics">Use Log Analytics</h2>
<p>Log Analytics allows organizations to query data using built in queries or custom created Kusto queries, for more information, see <a href="/en-us/azure/azure-monitor/logs/get-started-queries" data-linktype="absolute-path">Get started with log queries in Azure Monitor</a>.</p>
<p><span class="mx-imgBorder">
<a href="media/troubleshoot-policy-changes-audit-log/log-analytics-new-old-value.png#lightbox" data-linktype="relative-path">
<img src="media/troubleshoot-policy-changes-audit-log/log-analytics-new-old-value.png" alt="Log Analytics query for updates to Conditional Access policies showing new and old value location" data-linktype="relative-path">
</a>
</span>
</p>
<p>Once enabled find access to Log Analytics in the <strong>Identity</strong> &gt; <strong>Monitoring &amp; health</strong> &gt; <strong>Log Analytics</strong>. The table of most interest to Conditional Access administrators is <strong>AuditLogs</strong>.</p>
<pre><code class="lang-kusto">AuditLogs 
| where OperationName == "Update Conditional Access policy"
</code></pre>
<p>Changes can be found under <strong>TargetResources</strong> &gt; <strong>modifiedProperties</strong>.</p>
