This plugin includes an interesting sample of malicious code which goes to great lengths to conceal itself from the website owner.</p><p>In this post, we&#8217;ll review how the malware worked as well as how ecommerce website owners can protect themselves from such attacks.</p><h2>Malware injection</h2><p>Let&#8217;s start with how the malware is first introduced into the environment. The malware was lodged into the following <strong>mu-plugins/wp_services.php</strong> file in the WordPress website:</p><pre class="code_syntax" style="color: #d1d1d1; background: #000000;"><span class="line_wrapper">./wp-content/<strong><span style="color: #008080;">mu-plugins</span></strong>/<strong><span style="color: #008000;">wp_services_.php</span></strong></span></pre><p>As with many other malicious or fake WordPress plugins it contains some deceptive information at the top of the file to give it a veneer of legitimacy. In this case, comments claim the code to be &#8220;<strong>WordPress Cache Addons</strong>&#8220;:</p><p><a href="https://blog.sucuri.net/wp-content/uploads/2023/12/wordpress_cache_addons.png" target="_blank" rel="noopener"><img decoding="async" class="wp-image-32817 size-large aligncenter" src="https://blog.sucuri.net/wp-content/uploads/2023/12/wordpress_cache_addons-483x650.png" alt="Fake WordPress Cache Addons plugin" width="483" height="650" srcset="https://blog.sucuri.net/wp-content/uploads/2023/12/wordpress_cache_addons-483x650.png 483w, https://blog.sucuri.net/wp-content/uploads/2023/12/wordpress_cache_addons-446x600.png 446w, https://blog.sucuri.net/wp-content/uploads/2023/12/wordpress_cache_addons-390x525.png 390w, https://blog.sucuri.net/wp-content/uploads/2023/12/wordpress_cache_addons-560x754.png 560w, https://blog.sucuri.net/wp-content/uploads/2023/12/wordpress_cache_addons.png 680w" sizes="(max-width: 483px) 100vw, 483px" /></a></p><p>Initially this gets installed as a regular plugin from the wp-admin panel, likely the result of a compromised admin user (demonstrating the importance of <a href="https://blog.sucuri.net/2023/07/how-to-harden-wordpress-a-basic-overview.html" target="_blank" rel="noopener">securing your WordPress dashboard</a>). It then replicates itself to the <a href="https://developer.wordpress.org/advanced-administration/plugins/mu-plugins/" target="_blank" rel="noopener">mu-plugins directory</a>, or &#8220;must use&#8221; plugins). These don&#8217;t require activation from any user and are automatically executed by WordPress itself.</p><p>This year alone we have cleaned over 87,000 malicious files from this directory. Plugin files in this directory will also become active on all websites if they are part of a <a href="https://wordpress.org/documentation/article/create-a-network/">multisite</a> installation.</p><p>This is often where hosting providers will place plugins that are required to use in their specific environment. A list of installed mu-plugins is normally accessible from the admin panel by selecting the following:</p><p><img loading="lazy" decoding="async" class="alignnone wp-image-32819 size-large" srcset="https://blog.sucuri.net/wp-content/uploads/2023/12/add-new-plugin-650x131.png 650w, https://blog.sucuri.net/wp-content/uploads/2023/12/add-new-plugin-600x121.png 600w, https://blog.sucuri.net/wp-content/uploads/2023/12/add-new-plugin-768x155.png 768w, https://blog.sucuri.net/wp-content/uploads/2023/12/add-new-plugin-390x79.png 390w, https://blog.sucuri.net/wp-content/uploads/2023/12/add-new-plugin.png 849w" sizes="(max-width: 650px) 100vw, 650px" /></p><p>However, a portion of the injection also conceals this from view in order restrictions</h2><p>Speaking of which, there is an interesting addition to this malware which restricts the usage of file manager plugins:</p><p><a href="https://blog.sucuri.net/wp-content/uploads/2023/12/file_manager_restrictions.png" target="_blank" rel="noopener"><img loading="lazy" decoding="async" class="wp-image-32821 size-full aligncenter" sizes="(max-width: 632px) 100vw, 632px" /></a></p><p>Since the only way to remove any of the mu-plugins is by manually removing the file the malware goes out of its way to prevent this. If the website administrator has access only to the wp-admin panel (or if they are unfamiliar with FTP clients) then the only way you can remove one of these is through something like the &#8220;<a href="https://wordpress.org/plugins/file-manager-advanced/">Advanced File Manager</a>&#8220;, a popular plugin with 100,000+ active installations.</p><p>The malware accomplishes this by unregistering callback functions for hooks that plugins like this normally use. Some of the more generic hooks that are disabled here here such as &#8220;<i>admin_menu</i>&#8221; and &#8220;<i>admin_enqueue_scripts</i>&#8221; might also cause some collateral damage and break other plugin functionality but I don&#8217;t think that the attackers are too worried about that.</p><h2>Administrator creation and concealment</h2><p>Another addition to this infection is something that WordPress malware commonly does: creates and conceals an administrator user account. This provides the attackers sustained access to the environment (assuming that <a href="https://docs.sucuri.net/website-firewall/whitelist-and-blacklist/protected-page/">access restrictions</a> have not been put in place by the site owner) and hides it from the legitimate website admin.</p><p><img loading="lazy" decoding="async" class="wp-image-32822 size-full aligncenter" src="https://blog.sucuri.net/wp-content/uploads/2023/12/malicious_admin_user.png" alt="Malicious admin user and credentials" width="632" height="212" srcset="https://blog.sucuri.net/wp-content/uploads/2023/12/malicious_admin_user.png 632w, https://blog.sucuri.net/wp-content/uploads/2023/12/malicious_admin_user-600x201.png 600w, https://blog.sucuri.net/wp-content/uploads/2023/12/malicious_admin_user-390x131.png 390w, https://blog.sucuri.net/wp-content/uploads/2023/12/malicious_admin_user-560x188.png 560w" sizes="(max-width: 632px) 100vw, 632px" /></p><p>In addition to hiding the user from view in the admin panel so as to not arouse suspicion it also reduces the total admin count in the panel. WordPress offers a variety of different <a href="https://wordpress.org/documentation/article/roles-and-capabilities/">user roles</a> such as &#8220;<strong>Contributor</strong><i>&#8220;</i> and &#8220;<strong>Editor</strong><i>&#8220;</i> so as to better manage access control and security.
