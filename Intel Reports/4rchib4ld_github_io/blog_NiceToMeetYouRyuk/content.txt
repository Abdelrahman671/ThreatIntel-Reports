        <p>This work is a comprehensive analysis of the ransomware Ryuk. It will be break down into three parts :</p>
<ul>
  <li>What are the Ryuk operator interested in ?</li>
  <li>What tactics and tools do they typically use ?</li>
  <li>How can defenders detect those tools or tactics ?</li>
</ul>

<h2 id="from-netflix-to-your-enterprise">From Netflix to your enterprise</h2>

<p>Ryuk is a Ransomware, <a href="https://www.securitymagazine.com/articles/93769-ryuk-ransomware-responsible-for-one-third-of-all-ransomware-attacks-in-2020">responsible for a third of all ransomware attacks in 2020</a>. Ryuk is specifically used to target enterprise environments, and specially in the recent times Big Game Hunting – going after bigger, more secure targets in tailored operations and potentially extract larger ransoms –.
Like almost all ransomwares, Ryuk goal is to generate as much money as possible, in the shortest amount of time. They often succeed according to the reports, because the average ransom seems to be near $1Millions, and the highest was $34Millions in 2018.</p>

<p>For this, Ryuk will targets huge corporations and specially Windows assets (which represents the most part of their infrastructure). Clients and servers will (sadly) go the same way, disregarding of their version.</p>

<p>At the time of writing, there is no way to decrypt Ryuk Encryption. That’s because it’s the AES/RSA combo and restoring data without the keys is impossible (unless there is a failure in the implementation).</p>

<p>Ryuk is under constant development, new modules and functionalities are added over time.</p>

<h2 id="how-does-it-operates-">How does it operates ?</h2>

<p>Ryuk is most of the time delivered by TrickBot and more recently Bazar. (**), so with a phishing mail <a href="https://attack.mitre.org/techniques/T1566">(T1566)</a>.</p>

<p class="notice--info">This two malwares are <a href="https://www.zdnet.com/article/new-bazar-backdoor-linked-to-trickbot-banking-trojan-campaigns/">attributed to the same group</a></p>

<p>However, it was also seen to exploit accessible Remote Desktop Services on targets <a href="https://attack.mitre.org/techniques/T1021/001/">(T1021.001)</a>.</p>

<p>Ryuk can also exploit vulnerabilities like <a href="https://threatpost.com/ryuk-ransomware-gang-zerologon-lightning-attack/160286/">ZeroLogon</a> or <a href="https://www.computerweekly.com/news/252489779/Ryuk-attack-downs-private-health-provider-in-major-incident">EternalBlue</a> to propagate itself into the networks <a href="https://attack.mitre.org/techniques/T1210/">(T1210)</a>.</p>

<p>There is actually some works done before Ryuk is run on the victim’s infrastructure.
There is an Active Directory reconnaissance <a href="https://attack.mitre.org/techniques/T1018/">(T1018)</a> done using tools like :</p>
<ul>
  <li>Mimikatz</li>
  <li>PowerShell PowerSploit</li>
  <li>AdFind</li>
  <li>Bloodhound</li>
  <li>PsExec</li>
</ul>

<p class="notice--info">Investigating the Active Directory <a href="https://www.bleepingcomputer.com/news/security/trickbot-now-steals-windows-active-directory-credentials/">is also a behavior seen for TrickBot</a>.</p>

<p>The goal here for Ryuk is to have enough knowledge and privileges to impact as much hosts and shares as possible, as well as estimating the victims infrastructure : A big Active Directory most of the time involves a big entreprise.</p>

<p>There is also <a href="https://www.advanced-intel.com/post/anatomy-of-attack-inside-bazarbackdoor-to-ryuk-ransomware-one-group-via-cobalt-strike">instances where Cobalt Strike was used</a>, as it <a href="https://www.bleepingcomputer.com/news/security/bazarbackdoor-trickbot-gang-s-new-stealthy-network-hacking-malware/">sometimes comes with Bazar</a>.</p>

<p>Once the Active Directory has delivered all of its secrets, Ryuk is executed.</p>

<p class="notice--info"><strong>About the delivery :</strong> Even when Emotet is involved, it actually delivers Trickbot which itself delivers Ryuk. This reinforce the accountancy between this two and the attribution to the same group : Wizard Spider</p>
<h3 id="malware-analysis">Malware Analysis</h3>

<p>I was able to find a Ryuk Sample (5e1e2920736e1c00104e24ee). I will describe points that seems interesting to have in mind when dealing with Ryuk, as well as how they can be detected/mitigated.</p>

<p class="notice--info"><strong>About Hermes :</strong> Ryuk is most likely built on the Hermes ransomware, which had <a href="https://research.checkpoint.com/2018/ryuk-ransomware-targeted-campaign-break/">it source code shared on various underground forums</a>. That’s why you’ll see strings that relates to Hermes.</p>

<h4 id="dropping-itself">Dropping itself</h4>

<p>Upon launching, it copies itself in the current directory with a random 7 characters length and then launch the new file with the parameters “8 Lan” (Wake On Lan)</p>

<p><img src="/assets/images/Pasted%20image%2020210221085933.png" alt="" /></p>

<h4 id="anti-everything">Anti everything</h4>

<p>It then creates a thread that loop thought all running processes and services and kill those who might disturb it : like excel, backup services, virtual machine and such. The full list is in the Appendix. The goal is to make sure not file are opened in memory <a href="https://attack.mitre.org/techniques/T1489/">(T1489)</a>, as well as performing AntiVM action <a href="https://attack.mitre.org/techniques/T1562/001">(T1562.001)</a>.</p>

<ul>
  <li><code class="language-plaintext highlighter-rouge">net stop</code> is used for services :</li>
</ul>

<p><img src="/assets/images/Pasted%20image%2020210226113746.png" alt="" /></p>

<ul>
  <li><code class="language-plaintext highlighter-rouge">taskkill</code> is used for processes :</li>
</ul>

<p><img src="/assets/images/Pasted%20image%2020210226113834.png" alt="" /></p>

<p class="notice--primary"><strong>Detection opportunity</strong> : It’s highly suspicious that a process kills a high number of processes and/or services. Maybe some legit apps do it, but this behavior can be a sign of a malware trying to setup its playground.</p>

<p>It will also performs anti forensics/recovery by executing various commands (<a href="https://attack.mitre.org/techniques/T1059/003">T1059.003</a> and <a href="https://attack.mitre.org/techniques/T1490">T1490</a>) :</p>

<p><img src="/assets/images/Pasted%20image%2020210218091816.png" alt="" /></p>

<p class="notice--info">Note the typo for the first command, there is a missing ‘e’ on ‘delete’</p>

<p class="notice--primary"><strong>Detection opportunity</strong> : This kind of commands are HIGHLY suspicious, and there should be a rule on the SIEM or EDR that monitor their usages.
Also, having a lot of this commands executed in the same interval can be determined in a rule</p>

<h4 id="privilege-escalation">Privilege Escalation</h4>

<p>Ryuk will check for two privileges :</p>
<ul>
  <li><code class="language-plaintext highlighter-rouge">SEBackupPrivilege</code> : In order to access to any file and bypass ACL</li>
  <li><code class="language-plaintext highlighter-rouge">SEDebugPrivilege</code> : In order to execute the process injection</li>
</ul>

<p>In order to do so, Ryuk will open it current thread and retrieve the Thread Token.</p>

<p><img src="/assets/images/Pasted%20image%2020210226090247.png" alt="" /></p>

<p>It will then enable the wanted privilege with the function <code class="language-plaintext highlighter-rouge">AdjustTokenPrivileges</code> (<a href="https://attack.mitre.org/techniques/T1134">T1134</a>)</p>

<p><img src="/assets/images/Pasted%20image%2020210226090526.png" alt="" /></p>

<p>The privileges will be used on the next steps</p>

<h4 id="process-injection">Process Injection</h4>

<p>Ryuk injects itself into multiples processes, in order to be very fast.</p>

<p><img src="/assets/images/Pasted%20image%2020210222084700.png" alt="" /></p>

<p>It enumerates running process (<a href="https://attack.mitre.org/techniques/T1057">T1057</a>)checks if the process is one of <code class="language-plaintext highlighter-rouge">csrss.exe</code>, <code class="language-plaintext highlighter-rouge">explorer.exe</code>, <code class="language-plaintext highlighter-rouge">lsaas.exe</code> or is run under NT Admin and then injects into it, using <code class="language-plaintext highlighter-rouge">WriteProcessMemory</code> and <code class="language-plaintext highlighter-rouge">CreateRemoteThread</code> (<a href="https://attack.mitre.org/techniques/T1055">T1055</a>).</p>

<p class="notice--primary"><strong>Detection opportunity</strong> : It’s pretty uncommon for a process to inject itself into another, or even writing to its memory. This kind of behavior can be detected by an EDR, or by Windows Sysmon Event ID 8, for example.</p>

<h4 id="encryption">Encryption</h4>

<p>The encryption (<a href="https://attack.mitre.org/techniques/T1486">T1486</a>) is done with the <code class="language-plaintext highlighter-rouge">CryptEncrypt</code> Windows API with AES_256.</p>

<p><img src="/assets/images/Pasted%20image%2020210222082250.png" alt="" /></p>

<p>All encrypted files have the string “HERMES” at their end. It is used as a marker to know whether the file has already been encrypted. The file itself is encrypted, so there is no deletion involves and so recovery is unlikely. A file named <code class="language-plaintext highlighter-rouge">RyukReadMe.html</code> is dropped on every encrypted directory.</p>

<p><img src="/assets/images/Pasted%20image%2020210218092446.png" alt="" /></p>

<p>There is a bunch of directory that are whitelisted by the malware :</p>

<p><img src="/assets/images/Pasted%20image%2020210221114747.png" alt="" /></p>

<p>As well as :</p>
<ul>
  <li>Chrome</li>
  <li>Mozilla</li>
  <li>Recycle.bin</li>
  <li>Windows</li>
  <li>Microsoft</li>
  <li>AhnLab</li>
</ul>

<p>All encrypted files are appended the <code class="language-plaintext highlighter-rouge">.RYK</code> file extension.</p>

<p>Any mounted drives (<a href="https://attack.mitre.org/techniques/T1547/001">T1547.001</a>)will suffer the same fate, first by having rights :</p>

<p><img src="/assets/images/Pasted%20image%2020210223203130.png" alt="" /></p>

<p>And then the encryption :</p>

<p><img src="/assets/images/Pasted%20image%2020210223203344.png" alt="" /></p>

<p>Ryuk will also scan the ARP table in order to find any files or folders in remote hosts. (<a href="https://attack.mitre.org/techniques/T1016">T1016</a>)</p>

<p><img src="/assets/images/Pasted%20image%2020210223202849.png" alt="" /></p>

<h4 id="persistence">Persistence</h4>

<p>Create a registry key named ‘svchos’ on the famous <code class="language-plaintext highlighter-rouge">HKEY\_CURRENT\_USER\\SOFTWARE\\Microsoft\\Windows\\CurrentVersion\\Run</code> registry (<a href="https://attack.mitre.org/techniques/T1547/001">T1547.001</a>)with the full path of the executable using <code class="language-plaintext highlighter-rouge">cmd.exe</code> (<a href="https://attack.mitre.org/techniques/T1059/003">T1059.003</a>)</p>

<p><img src="/assets/images/Pasted%20image%2020210221114253.png" alt="" /></p>

<p class="notice--primary"><strong>Detection opportunity</strong> : This registry is a well known location for persistence. A rule can be easily setup to monitor any suspicious addition.</p>

<h2 id="whats-next-">What’s next ?</h2>

<p>Wizard Spider seems to have built an ecosystem of malware, and still continues to develop it, with the addition to Conti, a new ransomware. Some reports seems to <a href="https://www.bleepingcomputer.com/news/security/ryuk-successor-conti-ransomware-releases-data-leak-site/">indicate that Conti is a new version of Ryuk</a>. Maybe that’s a question I will tackle during another analysis.
For now, Ryuk is still going, and should be kept in mind.</p>
