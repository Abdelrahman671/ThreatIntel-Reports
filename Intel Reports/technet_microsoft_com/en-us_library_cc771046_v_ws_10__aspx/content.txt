							<div class="content "><h1 id="netsh-commands-for-windows-firewall">Netsh Commands for Windows Firewall</h1><div class="display-flex justify-content-space-between align-items-center flex-wrap-wrap page-metadata-container">
										<div class="margin-right-xxs">
											<ul class="metadata page-metadata" data-bi-name="page info" lang="en-us" dir="ltr"><li>Article</li><li>
														<time class="is-invisible" data-article-date aria-label="Article review date" datetime="2012-07-02T00:00:00.000Z" data-article-date-source="ms.date">07/02/2012</time>
													</li></ul>

											<h2 id="ms--in-this-article" class="title is-6 margin-block-xs">In this article</h2>
										</nav><!-- <content> --><p>Applies To: Windows 7, Windows Server 2008, Windows Server 2008 R2, Windows Vista</p>
<p>The Netsh commands for Windows Firewall provide a command-line alternative to the capabilities of the Windows Firewall Control Panel utility. By using the Netsh firewall commands, you can configure and view Windows Firewall exceptions and configuration settings.</p>
<div class="IMPORTANT">
<p>Important</p>
<p>The <strong>firewall</strong> context of the netsh command-line tool is provided only for backwards-compatibility with earlier versions of Windows. The <strong>firewall</strong> context works on computers that are running WindowsÂ® 7, Windows ServerÂ® 2008 R2, Windows VistaÂ®, and Windows ServerÂ® 2008, but it does not allow you to manage or interact with any of the firewall features that are new to those newer versions of Windows. This context does not allow you to work remotely on a computer to directly configure its firewall.
<br>We recommend that you instead use the <strong>advfirewall</strong> context unless you are using this tool in a mixed environment and must maintain backwards-compatibility with earlier versions of Windows. To use the new firewall features that are included with Windows Vista and later versions of Windows, you must use the <strong>advfirewall</strong> context instead. For more information, see <a href="cc771920(v=ws.10)" data-linktype="relative-path">Netsh Commands for Windows Firewall with Advanced Security</a>.
<br>We recommend that you do not use this context on a computer that is running Windows Vista or a later version of Windows, because by using it you can create and modify firewall rules only for the domain and private profiles. Earlier versions of Windows only supported a domain and standard profile. On Windows Vista and later versions of Windows, standard maps to the private profile and domain continues to map to the domain profile. Rules for the public profile can only be manipulated when the computer is actually attached to a public network and the command is run against the "current" profile.
<br>Starting with Windows 7 and Windows Server 2008 R2, if you run any command in the firewall context, the command still works, but is accompanied by the message:
<br>IMPORTANT: ânetsh firewallâ is deprecated; use ânetsh advfirewall firewallâ instead. For more information on using ânetsh advfirewall firewallâ commands instead of ânetsh firewallâ, see <a href="https://go.microsoft.com/fwlink/?linkid=121488" data-linktype="external">KB article 947709</a> at <a href="https://go.microsoft.com/fwlink/?linkid=121488" data-linktype="external">https://go.microsoft.com/fwlink/?linkid=121488</a>.
<br></p>
</div>
<div class="IMPORTANT">
<p>Important</p>
<p>To use the netsh firewall commands remotely on another computer by using the netsh âr parameter, the Remote Registry service must be running on the remote computer. If it is not, then Windows displays a âNetwork Path Not Foundâ error message.
<br></p>
</div>
<p>You can run these commands from within the netsh tool at the <strong>netsh firewall&gt;</strong> prompt.</p>
<p>For these commands to work at a standard Windows command prompt, you must preface each command with <strong>netsh firewall</strong>, followed by the specific command and parameters as they appear in the syntax below.</p>
<div class="NOTE">
<p>Note</p>
<p>If User Account Control is enabled on your computer and you want to run any netsh firewall command that changes the firewall configuration, you must run the command from a command prompt that was started with the <strong>Run as administrator</strong> option. If you try to change the firewall state without having administrator permissions available to the command-line tool, it fails with the message "The requested operation requires elevation."
<br></p>
</div>
<p>For more information about <strong>netsh</strong>, see <a href="cc732279(v=ws.10)" data-linktype="relative-path">Netsh Overview</a> and <a href="cc754580(v=ws.10)" data-linktype="relative-path">Enter a Netsh Context</a>.</p>

<ul>
<li><p>add allowedprogram</p>
</li>
<li><p>set allowedprogram</p>

</li>

</li>
<li><p>set notifications</p>
</li>

<div class="NOTE">
<p>Note</p>
<p>In earlier versions of Windows, many of these command accepted a parameter called <strong>interface</strong>. This parameter is not supported in the firewall context in Windows Vista or later versions of Windows.
<br></p>
</div>
<h2 id="add-allowedprogram">add allowedprogram</h2>
<p>Adds a program-based exception to the firewall.</p>
<h2 id="syntax">Syntax</h2>
<p><strong>add allowedprogram</strong> [ <strong>program =</strong> ] <em>PathAndFileName</em> [ <strong>name =</strong> ] <em>ProgramName</em> [ [ <strong>mode =</strong> ] { <strong>enable</strong> | <strong>disable</strong> } ] [ [ <strong>scope =</strong> ] { <strong>all</strong> | <strong>subnet</strong> | <strong>custom</strong> } ] [ [ <strong>addresses =</strong> ] { <em>IPAddress</em> | <em>IPRange</em> | <em>Subnet</em> | <strong>localsubnet</strong> }[,â¦] ] [ [ <strong>profile =</strong> ] { <strong>current</strong> | <strong>domain</strong> | <strong>standard</strong> | <strong>all</strong> } ]</p>
<h2 id="parameters">Parameters</h2>
<ul>
<li><strong>[ <strong>program =</strong> ] <em>PathAndFileName</em></strong><br>

<p><strong>add portopening</strong> [ <strong>protocol =</strong> ] { <strong>tcp</strong> | <strong>udp</strong> | <strong>all</strong> } [ <strong>port =</strong> ] <em>Integer</em> [ <strong>name =</strong> ] <em>ExceptionName</em> [ [ <strong>mode =</strong> ] { <strong>enable</strong> | <strong>disable</strong> } ] [ [ <strong>scope =</strong> ] <strong>all</strong> | <strong>subnet</strong> | <strong>custom</strong> } ] [ [ <strong>addresses</strong> = ] addresses ] [ [ <strong>profile =</strong> ] <strong>current</strong> | <strong>domain</strong> | <strong>standard</strong> | <strong>all</strong> } ]</p>
<h2 id="parameters-8">Parameters</h2>
<ul>
<li><strong>[ <strong>protocol =</strong> ] { <strong>tcp</strong> | <strong>udp</strong> | <strong>all</strong> }</strong><br>
<strong>Required</strong>. Specifies whether the port number refers to TCP, UDP, or both.</li>
</ul>
<!-- end list -->
<ul>
<li><strong>[ <strong>port =</strong> ] <em>Integer</em></strong><br>
<strong>Required</strong>. Specifies the port number to be excepted. Must be an integer value from 1 to 65535. Only a single value can be specified and port ranges are not supported.</li>
</ul>
<!-- end list -->
<ul>
<li><strong>[ <strong>name =</strong> ] <em>ExceptionName</em></strong><br>
<strong>Required</strong>. Specifies the name of the exception. This value is displayed in the Firewall control panel exception list.</li>
</ul>
<!-- end list -->
<ul>
<li><strong>[ [ <strong>mode =</strong> ] { <strong>enable</strong> | <strong>disable</strong> } ]</strong><br>
Specifies whether this exception is currently applied and active on the local computer.</li>
</ul>
<!-- end list -->
<ul>
<li><strong>[ <strong>scope =</strong> ] { <strong>all</strong> | <strong>subnet</strong> | <strong>custom</strong> }</strong><br>
Specifies the scope of the allowed network traffic from remote computers. <strong>all</strong> indicates that traffic is allowed from any computer, including those on the Internet. <strong>subnet</strong> indicates that traffic is allowed from computers on the local computer's subnet only. <strong>custom</strong> indicates that traffic is allowed from only those computers whose IP address matches the <strong>addresses</strong> parameter. The default value is <strong>all</strong>.</li>
</ul>
<!-- end list -->
<ul>
<li><p><strong>[ <strong>addresses =</strong> ] { <em>IPAddress</em> | <em>IPRange</em> | <em>Subnet</em> | <strong>localsubnet</strong> }[,â¦]</strong><br>
Specifies a custom list of addresses for the <strong>scope=custom</strong> parameter. Each entry can be:</p>
<ul>
<li><p>An IPv4 or IPv6 address. For example, <strong>192.168.0.15</strong>.</p>
</li>
<li><p>An IPv4 or IPv6 range with start and end addresses separated by a '-'. For example, <strong>192.168.0.1-192.168.0.50</strong>.</p>
</li>
<li><p>A subnet indicated by the subnet address and subnet mask separated by a '/'. For example, <strong>192.168.0.0/255.255.255.0</strong>.</p>
</li>
<li><p>A subnet indicated by the subnet address and a subnet prefix separated by a '/'. For example, <strong>10.1.0.0/16</strong>.</p>
</li>
<li><p>The keyword <strong>localsubnet</strong>, which includes all addresses that are on the local computer's current subnet.</p>
</li>
</ul>
<p>Multiple entry types can be combined on a command line by separating them with commas: <strong>172.16.0.0/16, 10.0.0.0/255.0.0.0, 21AB:0000:0000:CD30::/60, localsubnet</strong></p>
</li>
</ul>
<!-- end list -->
<ul>
<li><p><strong>[ <strong>profile =</strong> ] { <strong>current</strong> | <strong>domain</strong> | <strong>standard</strong> | <strong>all</strong> }</strong><br>
Specifies the firewall profile to which the command applies. The profile is determined by the detected network location types accessible through the computer's network adapters.</p>
<ul>
<li><strong>current</strong> specifies that the command applies to the profile that is currently active on the computer.</li>
</ul>
</li>
</ul>
<div class="NOTE">
<p>Note</p>
<p>On Windows 7 and Windows Server 2008 R2, this option applies to all profiles that are currently active on the computer.
<br></p>
</div>
<pre><code>  - **domain** specifies that the command applies only to the domain profile.  
      
  - **standard** specifies that the command applies only to the private profile.  
      
  - **all** specifies that the command applies to all profiles except the private profile.  
      

The default value is **current**.
</code></pre>
<h2 id="remarks-8">Remarks</h2>
<ul>
<li><p>You must specify <strong>scope=custom</strong> to specify <strong>addresses</strong>. If <strong>scope=custom</strong> is used, then <strong>addresses</strong> cannot be blank.</p>
</li>
<li><p>To specify the profile associated with the public network location type, you must specify <strong>profile=current</strong> when the computer is attached to a public network.</p>
</li>
<li><p>The <strong>addresses</strong> parameter cannot contain an unspecified IPv6 address, a loopback address, or a multicast address.</p>
</li>
</ul>
<h2 id="examples-8">Examples</h2>
<p>Each example must be entered as a single command line. The examples may be displayed on multiple lines below for space reasons.</p>
<p><strong>add portopening tcp 80 MyWebPort</strong></p>
<p><strong>add portopening udp 500 "IKE Exception" enable all</strong></p>
<p><strong>add portopening all 53 DNS enable custom 157.60.0.1,172.16.0.0/16,10.0.0.0/255.0.0.0,localsubnet</strong></p>
