          <h1>New Android NoviSpy spyware linked to Qualcomm zero-day bugs</h1>
          <div class="cz-news-story-title-section">
            <div class="cz-news-title-left-area">
                            By <h6><a rel='author' href='https://www.bleepingcomputer.com/author/bill-toulas/' class='author'><span itemprop="author" itemscope itemtype="https://schema.org/Person"><span itemprop="name">Bill Toulas</span></span></a></h6>
            </div>
	    <meta itemprop="articleSection" content="Security"><meta itemprop="articleSection" content="Mobile">
            <div class="cz-news-title-right-area">
              <ul>
                <li class="cz-news-date">December 16, 2024</li>
                <li class="cz-news-time">10:06 AM</li>
                
                <li class="cz-news-comment"><a href="#comments">1</a></li>
              </ul>
            </div>
          </div>
	  <div class="articleBody">
	   
<p style="text-align:center"><img alt="Qualcomm" height="900" src="https://www.bleepstatic.com/content/hl-images/2024/10/07/Qualcomm.jpg" width="1600"></p>

<p>The Serbian government exploited Qualcomm zero-days to unlock and infect&nbsp;Android devices with a new spyware named 'NoviSpy,' used to spy on activists, journalists, and protestors.</p>

<p>One of the Qualcomm flaws linked to the attacks is CVE-2024-43047, which was marked as an actively exploited zero-day vulnerability <a href="https://www.bleepingcomputer.com/news/security/qualcomm-patches-high-severity-zero-day-exploited-in-attacks/" target="_blank" rel="nofollow noopener">by Google Project Zero</a> in October 2024 and <a href="https://www.bleepingcomputer.com/news/security/google-fixes-two-android-zero-days-used-in-targeted-attacks/" target="_blank" rel="nofollow noopener">received a fix</a> on Android in November.</p>

<p>The spyware, which appears to have been deployed by Serbian authorities, based on its communications, was discovered by Amnesty International's Security Lab on a journalist's phone after police returned it.</p>

<p>"In February 2024, Slavi&scaron;a Milanov, an independent journalist from Dimitrovgrad in Serbia who covers local interest news stories, was brought into a police station after a seemingly routine traffic stop," reads a <a href="http://securitylab.amnesty.org/latest/2024/12/a-digital-prison-surveillance-and-the-suppression-of-civil-society-in-serbia/" target="_blank" rel="nofollow noopener">report by Amnesty International</a>.</p>

<p>"After Slavi&scaron;a was released, he noticed that his phone, which he had left at the police station reception at the request of the officers, was acting strangely &ndash; the data and wi-fi settings were turned off. Aware that this can be a sign of hacking, and mindful of the surveillance threats facing journalists in Serbia, Slavi&scaron;a contacted Amnesty International's Security Lab to request an analysis of his phone."</p>

<p>Subsequently, the researchers provided Google's Threat Analysis Group (TAG) with exploit artifacts, leading to uncovering the flaws in Qualcomm's DSP (Digital Signal Processor) driver ('adsprpc'), which is used for offloading multimedia processing to the DSP core.</p>

<p>While Google is unsure about which vulnerabilities are leveraged by NoviSpy, the evidence suggests that the spyware employs an exploit chain to bypass Android security mechanisms and install itself persistently at the kernel level.</p>

<h2>NoviSpy deployed in Serbia</h2>

<p>Amnesty International&nbsp;reports that NoviSpy was deployed by the Serbian Security Information Agency (BIA) and the Serbian police after a phone was unlocked using the Cellebrite unlocking tools during physical custody of the devices.</p>

<p>According to forensic evidence on tampered devices, the researchers believe that Cellebrite exploited Qualcomm zero-days to unlock Android phones.</p>

<p>"While conducting research for this report, the Security Lab also uncovered forensic evidence leading to the identification of a zero-day Android privilege escalation vulnerability used to escalate privileges on the device<br>
an activist from Serbia," reads Amnesty International's report.</p>

<p>"The vulnerability, identified in collaboration with security researchers at Androidmaker Google, affected numerous Android devices using popular Qualcomm chipsets impacting millions of Android devices worldwide."</p>

<p>The spyware communicated with servers on IP ranges tied directly to BIA, while configuration data in the samples identified a specific person linked to the country's prior spyware procurement programs.</p>

<p>The targets include journalists, human rights activists, and government dissidents. Specific examples mentioned in the Amnesty report include journalist Slavi&scaron;a Milanov, a member of the Krokodil NGO, and three activists.</p>

<p>However, Amnesty says that technical evidence suggests NoviSpy was installed on dozens, if not hundreds, of Android devices in Serbia over the last few years.</p>

<p>Regarding the initial compromise, Amnesty International says the recovered artifacts point to a zero-click attack leveraging Android calling features such as Voice-over-Wifi or Voice-over-LTE (VoLTE) functionality.</p>

<p>These were&nbsp;active on the examined compromised devices, used as part of the&nbsp;Rich Communication Suite (RCS) calling.</p>

<p>Amnesty International suspects some activists may have been targeted using a zero-click Android vulnerability that could be exploited by receiving phone calls from invalid phone numbers of many digits, as shown&nbsp;below.</p>

<div style="text-align:center">
<figure class="image" style="display:inline-block"><img alt="Suspicious calls believed to have triggered a zero-click exploit" height="427" src="https://www.bleepstatic.com/images/news/u/1220909/2024/Linux/05/long-numbers.png" width="282"><figcaption><strong>Suspicious calls that could have triggered a zero-click exploit</strong><br><em>Source: Amnesty International</em></figcaption></figure></div>

<h2>Google finds Qualcomm flaws</h2>

<p>Google's TAG received kernel panic logs generated by exploits captured by Amnesty International and worked backwards to <a href="https://googleprojectzero.blogspot.com/2024/12/qualcomm-dsp-driver-unexpectedly-excavating-exploit.html" target="_blank" rel="nofollow noopener">identify six vulnerabilities</a> in Qualcomm's adsprpc driver, used in millions of Android devices.</p>

<p>The six flaws are summarized as follows:</p>

<ol><li><strong>CVE-2024-38402</strong>: A reference counting issue in the driver can lead to use-after-free (UAF) exploitation and arbitrary code execution in the kernel space.</li>
	<li><strong>CVE-2024-21455</strong>: A flawed 'is_compat' flag handling allows user-controlled pointers to be treated as kernel pointers, creating arbitrary read/write primitives and leading to privilege escalation.</li>
	<li><strong>CVE-2024-33060</strong>: A race condition in 'fastrpc_mmap_create' exposes the driver to UAF vulnerabilities, especially when handling global memory maps, leading to kernel memory corruption.</li>
	<li><strong>CVE-2024-49848</strong>: A logic error in handling persistent mappings causes a UAF scenario when references to mappings are improperly released, providing a persistence mechanism.</li>
	<li><strong>CVE-2024-43047</strong>: Overlapping memory mappings in 'fastrpc_mmap' can lead to corrupted object references, potentially leading to memory corruption.</li>
	<li>No CVE: Improper validation in fastrpc_mmap_find leaks kernel address space information, allowing to bypass kernel address space layout randomization (KASLR).</li>
</ol><p>Google researchers confirmed the exploitation of CVE-2024-43047 and hypothesize that the rest were exploited in a complex attack chain.</p>

<p>At the time of writing, Qualcomm has not released a patch for CVE-2024-49848, despite Google having reported the issue to them 145 days back.</p>

<p>Google also noted that Qualcomm delayed patching CVE-2024-49848 and CVE-2024-21455 over the industry-standard period of 90 days.</p>

<p>BleepingComputer contacted Qualcomm to ask about the status of those the six flaws, and a spokesperson has provided the below statement:</p>

<p>"Developing technologies that endeavor to support robust security and privacy is a priority for Qualcomm Technologies,"&nbsp;Qualcomm told BleepingComputer.</p>

<p>"We commend the researchers from Google Project Zero and Amnesty International Security Lab for using coordinated disclosure practices. Regarding their FastRPC driver research, fixes have been made available to our customers as of September 2024. We encourage end users to apply security updates as they become available from device makers."</p>

<p>Regarding&nbsp;CVE-2024-49848, Qualcomm told BleepingComputer that a fix has been developed and is going through its disclosure process, with the related&nbsp;security bulletin&nbsp;coming in January 2025.</p>

<p>Regarding the vulnerability that lacks a CVE identifier, Qualcomm says the issue was packaged along with the CVE-2024-33060 fix&nbsp;in September 2024, and hence has been fixed.</p>

<p><em>Update 12/16/24</em>: Added new information from Qualcomm about upcoming fixes.</p>

function cz_strip_tags(input, allowed) {
  allowed = (((allowed || '') + '')
    .toLowerCase()
    .match(/<[a-z][a-z0-9]*>/g) || [])
    .join(''); // making sure the allowed arg is a string containing only tags in lowercase (<a><b><c>)
  var tags = /<\/?([a-z][a-z0-9]*)\b[^>]*>/gi,
    commentsAndPhpTags = /<!--[\s\S]*?-->|<\?(?:php)?[\s\S]*?\?>/gi;
  return input.replace(commentsAndPhpTags, '')
    .replace(tags, function($0, $1) {
      return allowed.indexOf('<' + $1.toLowerCase() + '>') > -1 ? $0 : '';
    });
}
function cz_br2nl(str) {
	var regex = /<br\s*[\/]?>/gi;
    //var pure_str = str.replace(regex,"\n");
    var pure_str = str.replace(regex,"");
    return cz_strip_tags(pure_str,'');
}
