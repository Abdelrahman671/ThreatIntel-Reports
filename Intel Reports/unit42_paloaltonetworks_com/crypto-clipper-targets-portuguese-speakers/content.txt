To do this, the campaign uses a type of malware known as a cryptocurrency clipper, which monitors the victim\u2019s clipboard for signs that a cryptocurrency wallet address is being copied.\r\n\r\nThe malware, which we call CryptoClippy, seeks to replace the user\u2019s actual wallet address with the threat actor\u2019s, causing the user to inadvertently send cryptocurrency to the threat actor. Unit 42 Managed Threat Hunting found victims across manufacturing, IT services, and real estate industries, though they likely impacted the personal wallet addresses of someone using their work machine.\r\n\r\nTo deliver the malware to users\u2019 computers, threat actors in this campaign used both Google Ads and traffic distribution systems (TDS) to redirect victims to malicious domains that are impersonating the legitimate WhatsApp Web application. They use this to ensure victims are real users, and also that they\u2019re Portuguese speakers. For users who are sent to malicious domains, the threat attempts to trick them into downloading malicious files, including either .zip or .exe files, that lead to the final payload.\r\n\r\nPalo Alto Networks customers receive protections against this campaign through Cortex XDR. The Advanced URL Filtering and DNS Security cloud-delivered security services for the Next-Generation Firewall identify domains associated with the CryptoClippy campaign as malicious.\r\n\r\n\r\n\r\nRelated Unit 42 Topics\r\nCryptocurrency, Bitcoin\r\n\r\n\r\n\r\nWhat Are Cryptocurrency Clippers?\r\nCryptocurrency malware aims to redirect cryptocurrency funds away from legitimate users\u2019 wallets and into wallets that are controlled by the threat actor. When a computer is infected with a cryptocurrency clipper, the malware continually examines the victim\u2019s clipboard to see if they have copied a cryptocurrency wallet address. The idea behind this is that if a person copies a wallet address to the clipboard, it indicates they might be in the process of transferring cryptocurrency from one wallet to another.\r\n\r\nThe clipper malware uses regular expressions (regexes) to identify what type of cryptocurrency the address pertains to. It then replaces the clipboard entry with a visually similar but adversary-controlled wallet address for the appropriate cryptocurrency. Later, when the victim pastes the address from the clipboard to conduct a transaction, they actually are sending cryptocurrency directly to the threat actor.\r\n\r\nSince wallet addresses typically are very long \u2013 sometimes exceeding 40 alphanumeric characters \u2013 many people will not notice this change in address. This contributes to the effectiveness of clipper malware.\r\nCryptoClippy Overview\r\nUnit 42 Managed Threat Hunting found victims across manufacturing, IT services, and real estate industries. This threat does not target specific industries, and the devices impacted were hit opportunistically.\r\n\r\nA CryptoClippy infection begins with SEO poisoning, so that when a person searches for \u201cWhatsApp Web,\u201d the result leads them to a threat actor-controlled domain. Once there, victims are prompted to download a .zip file that contains a .lnk file, which is composed of malicious scripts. These scripts set off a chain of events that installs the clipper malware. This threat then monitors the victim\u2019s clipboard activity for bitcoin transactions, such that their valid crypto wallet address will be replaced with one controlled by the threat actor.\r\n\r\nThe variant of CryptoClippy we investigated has a variety of additional capabilities to help the threat actor accomplish their crypto stealing activities. This includes being able to establish a Remote Desktop Protocol (RDP) backdoor through execution of an RC4-encrypted PowerShell script.\r\n\r\nThis script contains elements of Windows Management Instrumentation (WMI), terminal service registry manipulation, icacls, net commands and log clearing. These allow the attackers to maintain access outside the in-memory payload.\r\n\r\nIn addition, the variant also has functionality related to targeting Ethereum and bitcoin cryptocurrency wallets. This is not surprising, given the growing popularity of digital currencies in Latin America.\r\n\r\nAt the time of writing this post, the actor controlled wallets show recent activity. The bitcoin address has shown evidence of receiving 0.039954 BTC, which roughly equals $982.83. This bitcoin (BTC) balance came from four different BTC transactions. The Ethereum (ETH) address shows evidence of having received funds as well, with 0.110915556631181819 ETH (roughly equal to $186.32) being sent from three different ETH addresses.\r\n\r\nThe threat actors in this campaign employed a multi-stage approach to try to bypass signature- and heuristic-based security engines. This approach included obfuscation techniques to evade detection, including using obfuscated PowerShell scripts and encoded payloads.\r\n\r\nThe current low detection rate of this malware highlights this approach's effectiveness, as only a handful of vendors appear to be detecting this malware in VirusTotal. The vendors that do detect this have assigned generic names to this malware.\r\nInfection Chain for CryptoClippy\r\nThe infection flow for CryptoClippy starts with delivering an initial .zip file that contains a .lnk file that is composed of an obfuscated PowerShell command line script. Once the .lnk file is double-clicked by the victim, the PowerShell script will be executed, which will download the second stage and several obfuscated\/encrypted payloads (as discussed later in Stage 1 - Loader).\r\n\r\nWhen the second stage PowerShell script is executed, it deobfuscates\/decrypts the CryptoClippy loader and executes it. The loader will then inject its stealer component into svchost.exe.\r\n\r\nCryptoClippy will set up user-mode event based hooks and a callback function into clipboard APIs to replace the victim\u2019s Ethereum\/bitcoin crypto wallet with the actor\u2019s crypto wallet, when it is copied to the clipboard. It also contains functionality to communicate with C2 servers.\r\n\r\nFigure 1 shows the attack diagram for CryptoClippy.\r\n\r\n[caption id=\"attachment_127637\" align=\"alignnone\" width=\"1310\"] Figure 1. CryptoClippy attack diagram.[\/caption]\r\nDelivery via Malvertising\r\nWe\u2019ve observed this malware campaign leveraging Google Ads and TDS. First the threat actor used Google Search ads that appeared in search results when users search for \u201cWhatsApp Web.\u201d This allowed them to trick victims into opening and downloading their malicious compressed .zip file. After being alerted to the issue in early 2022, Google implemented additional protections and the company says its systems have improved at detecting and preventing renewed attack attempts.\r\n\r\nIn addition, threat actors used TDS to filter victims and bots on the malicious landing page. The TDS filters exclude bots and internet crawlers by using various criteria to determine if a client device is a real user and also a Portuguese speaker.\r\n\r\nFirst, the TDS checks if the connecting device is coming from a virtual private network (VPN) IP address. Using a VPN can make it difficult for the attacker to determine the location and characteristics of the victim's device, and can also make it more difficult to deliver malicious content.\r\n\r\nNext, the TDS makes user agent checks. The user agent is a string of text that a web browser sends to a website to identify itself and its capabilities. By checking the user agent of the client device, the TDS determines if the preferred browser language is Portuguese by checking the Accept-Language header. The TDS can also check for other information such as device type, operating system, browser version and geolocation.\r\n\r\nFinally, the TDS follows the Hypertext Transfer Protocol (HTTP) GET header criteria. The HTTP GET header is a request message sent by a web browser to a server to retrieve a resource. The header can contain various information, such as the requested URL, the user agent, the Accept-Language header and the referer header. Checking the criteria in the HTTP GET header is another way the TDS can determine if the preferred language for the browser is Portuguese\r\n\r\nIf the TDS determines that the connection does not meet the above criteria, the victim is redirected to a different landing page (shown in Figure 2) and prompted to click on \"Continuar para o WhatsApp Web\" (\"Continue to WhatsApp Web\" in Portuguese). This will redirect to the legitimate WhatsApp Web domain without any further malicious activity, effectively avoiding detection by the victim and security software.\r\n\r\n[caption id=\"attachment_127556\" align=\"aligncenter\" width=\"900\"] Figure 2. The content of the malicious domain.[\/caption]\r\n\r\nHowever, if the TDS determines that the connection does meet the above criteria, the victim is redirected to the malicious landing page where they are prompted to download the malicious .zip file (shown in Figures 3 and 4).\r\n\r\n[caption id=\"attachment_127558\" align=\"aligncenter\" width=\"900\"] Figure 3. The victim\u2019s browsing history.[\/caption]\r\n\r\nThis site disguises itself as the official WhatsApp Web site, offering WhatsApp Desktop. Downloading this file from mydigitalrevival[.]com resulted in the initial infection \u2014 WhatsApp.zip (shown in Figure 4).\r\n\r\n[caption id=\"attachment_127560\" align=\"aligncenter\" width=\"900\"] Figure 4. The mydigitalrevival[.]com site.[\/caption]Other domains we discovered belonging to this campaign are preflightdesign[.]com and pickconferences[.]com, which had the same content in Portuguese.\r\n\r\nWhen a victim successfully loaded the malicious webpage and downloaded the offered .zip file, it contained a .lnk file, as shown in Figure 5.\r\n\r\n[caption id=\"attachment_127562\" align=\"aligncenter\" width=\"900\"] Figure 5. The initial ZIP file contains the LNK file inside.[\/caption]\r\n\r\nFor more technical details on the .lnk file, please see Infection via LNK File.\r\n\r\nFollowing the execution of the .lnk file, Unit 42 researchers observed the following files downloaded to C:\\Users\\&lt;USERNAME&gt;\\AppData\\Roaming\\Ricoly:\r\n\r\n \tRicoly.bat\r\n \tRicoly.ps1\r\n \tTwo obfuscated encrypted files (ps, sc)\r\n \tOne obfuscated encrypted auxiliary config file (pf)\r\n\r\nThe first PowerShell script loader Ricoly.ps1 is launched and executed by the Ricoly.bat batch file. The Ricoly.ps1 script's purpose is to decrypt the second-stage obfuscated\/encrypted script ps. More in-depth information about these loader files can be found in Stage 1 - Loader and Stage 2 - Loader.\r\n\r\nThe functionality of the second stage PowerShell script ps is to act as a reflective PE loader. The ps file will target an encrypted EXE file named sc that is written to the file system. The sc file is reflectively loaded and eventually functions as the main loader for CryptoClippy.\r\nMain CryptoClippy Executable\r\nAfter the execution of the .lnk file and the two subsequent PowerShell scripts, the CryptoClippy operation consists of an executable loader and the main CryptoClippy executable. The loader for CryptoClippy is a 64-bit EXE file, where the main stealer EXE file is embedded inside of the .data section. The loader uses syscalls and appears to overlap with SysWhispers2 implementation for execution.\r\n\r\nSysWhispers2 is a stealthier implementation for code execution, which is used to bypass inspection of user-mode APIs. This bypass is possible because user-mode APIs are typically used by applications, including introspection by AV\/EDR products, through user-mode hooks.\r\n\r\nThe main executable for CryptoClippy is written in C and it is statically compiled with the Transport Layer Security (TLS) library Mbed-TLS. The main feature of the executable includes its own algorithm for name generation (i.e., the file path, mutex and event object). For more in-depth information on this algorithm, see Folder Name and Mutex String Generator.\r\n\r\nThe main executable also makes extensive use of a local auxiliary file called pf, which contains an encrypted certificate. CryptoClippy uses its own algorithm to deobfuscate the character lookup table from the auxiliary file to build the plaintext certificate from pointers to values in the lookup table.\r\n\r\nThis threat also uses two different RC4 keys for various string decryption functions. The string decryption routine will use a structure built in memory for the index location into the structure, string offset and string size. These strings are related to object names, domains, crypto wallets and persistence scripts.\r\n\r\n \tRC4 Key: 1b43233d5a054808061c190336320e46\r\n \tRC4 Key: 4646070B47445451604F291809444703\r\n\r\nCrypto Stealing Operation\r\nThe primary objective of CryptoClippy is to identify crypto wallet strings being copied into the clipboard. CryptoClippy sets up user-mode event hooks and uses a callback function to execute replacing Ethereum or bitcoin based wallets copied into the Windows clipboard with hardcoded wallet values owned by the threat actor. For more technical details, please see User-Mode Event Hook Setup.\r\n\r\nFigure 6 illustrates the window capturing the clipboard input. Based on the execution flow of the loader, this particular application would normally not be visible.\r\n\r\n[caption id=\"attachment_127564\" align=\"aligncenter\" width=\"986\"] Figure 6. Capturing the clipboard input.[\/caption]\r\n\r\nThe decompiled view of CryptoClippy\u2019s clipboard function (shown in Figure 7) reveals that the wallet addresses used by the threat actor will be RC4 decrypted to be used later on, in any replacement of crypto wallets that take place.\r\n\r\nCryptoClippy will perform a string check against the data being pushed into the clipboard. It will check for common leading characters associated with Ethereum or bitcoin related wallets. The string copied into the clipboard will be looped through to determine the character count. On line 27 of Figure 7, 25 is subtracted from the total character count and compared to see if it is less than or equal to 9.\r\n\r\n[caption id=\"attachment_127566\" align=\"aligncenter\" width=\"874\"] Figure 7. This section will roughly follow the order of activities as described in the infection diagram in Figure 1.\r\nInfection via LNK File\r\nThe .lnk file found within the .zip file that is initially downloaded by victims contains a truncated command, shown in Figure A1. The LNK file with the target field containing the command to run.[\/caption]\r\n\r\nThe .lnk file utilizes a few different character padding methods for obfuscation, which include the following character sets:\r\n\r\n \t^\r\n \t!!\r\n \t:~\r\n\r\nFigure A2 shows how this padding is used in the .lnk file.\r\n\r\n[caption id=\"attachment_127574\" align=\"aligncenter\" width=\"719\"] Figure A2. These files are usually contained in a .zip archive. However, during pivoting in this campaign, we observed another variation to the infection flow where the initial payload is an .exe file (reported on VirusTotal) that is linked to the previously mentioned domain mydigitalrevival[.]com.\r\n\r\nWhen we examined the .exe file that impersonates WhatsApp Web, we found that once it's executed, it will contact the previously mentioned tunneldrive[.]com domain. It will then drop and execute a .bat file to delete itself.\r\n\r\nThis .bat file (shown in Figure A4) uses an obfuscation that gradually builds up its payload similar to the one detailed in this blog, but with more obfuscation. In addition, this obfuscation type appears to be adopted from Daniel Bohannon\u2019s Invoke-Obfuscation project, which is a technique widely used by Brazilian threat actors.\r\n\r\n[caption id=\"attachment_127578\" align=\"aligncenter\" width=\"900\"] Figure A4. BAT file dropped as first stage.[\/caption]\r\nStage 1 - Loader\r\nThe first PowerShell script loader Ricoly.ps1 is launched and executed by the Ricoly.bat batch file. This keys the payload and adds an additional layer of anti-analysis, as shown in Figure A5.\r\n\r\n[caption id=\"attachment_127580\" align=\"aligncenter\" width=\"900\"] Figure A5. This EXE file will then be the target and focus of the second stage ps script.\r\n\r\nThe functionality of the second stage PowerShell script ps is to act as a reflective PE loader. It does so using .NET methods and D\/invoke to work with .NET delegates to assist in evasion and dynamic resolving.\r\n\r\nThe ps script will determine if the operating system is 32-bit or 64-bit. It will also determine the API functionality needed from kernel32.dll to act as a reflective loader. It does this by using GetDelegateForFunctionPointer for resolving the needed APIs such as VirtualAlloc and CreateThread.\r\n\r\nThe ps script will then XOR decrypt the payload and invoke the EXE file sc (as shown in Figure A7).\r\n\r\n[caption id=\"attachment_127584\" align=\"aligncenter\" width=\"900\"] Figure A7. This is achieved by using the APIs GetWindowsDirectoryA, GetVolumeInformationA and SHGetFolderPathA.\r\n\r\nThe stealer contains a function that will uniquely create alphanumeric strings. This string generator is used in several locations within the main program. It is first used to create a unique folder name under the AppData path. To do this, a constant value is used as a parameter to the string generator (shown in Figure A8).\r\n\r\nIn this instance, the constant is 0x79FE6D, which will be used within the string generator function and mapped to the format string %08x-%08x.\r\n\r\n[caption id=\"attachment_127586\" align=\"aligncenter\" width=\"900\"] Figure A8. The first operation will be an XOR performed by the first character of the string against the value FFFFFFFF.\r\n\r\nThe resulting operation will then be shifted right by 1 and XORed by the constant 0x82F63B78. The resulting XOR operation will then follow the same set of operations a total of eight times per each character in the 16 character example string 079fe6d-de786dd1.\r\n\r\nThe completion of the algorithm returns the folder name to be added to the AppData directory (shown in Figure A10). The folder name in this instance was 55abf82d and the full path was C:\\Users\\xxx\\AppData\\Roaming\\55abf82d. <p>Unit 42 recently discovered a malware campaign targeting Portuguese speakers, which aims to redirect cryptocurrency away from legitimate users’ wallets and into wallets controlled by threat actors instead. To do this, the campaign uses a type of malware known as a cryptocurrency clipper, which monitors the victim’s clipboard for signs that a cryptocurrency wallet address is being copied.</p>
<p>The malware, which we call CryptoClippy, seeks to replace the user’s actual wallet address with the threat actor’s, causing the user to inadvertently send cryptocurrency to the threat actor. Unit 42 Managed Threat Hunting found victims across manufacturing, IT services, and real estate industries, though they likely impacted the personal wallet addresses of someone using their work machine.</p>
<p>To deliver the malware to users’ computers, threat actors in this campaign used both Google Ads and traffic distribution systems (TDS) to redirect victims to malicious domains that are impersonating the legitimate WhatsApp Web application. They use this to ensure victims are real users, and also that they’re Portuguese speakers. For users who are sent to malicious domains, the threat attempts to trick them into downloading malicious files, including either <span style="font-family: 'courier new', courier, monospace;">.zip</span> or <span style="font-family: 'courier new', courier, monospace;">.exe</span> files, that lead to the final payload.</p>
<p>Palo Alto Networks customers receive protections against this campaign through <a href="https://docs-cortex.paloaltonetworks.com/p/XDR" target="_blank" rel="noopener">Cortex XDR</a>. The <a href="https://docs.paloaltonetworks.com/advanced-url-filtering/administration" target="_blank" rel="noopener">Advanced URL Filtering </a>and <a href="https://docs.paloaltonetworks.com/dns-security" target="_blank" rel="noopener">DNS Security</a> cloud-delivered security services for the Next-Generation Firewall identify domains associated with the CryptoClippy campaign as malicious.</p>

<h2><a id="post-127548-_5fz8g9aiqqtl"></a>What Are Cryptocurrency Clippers?</h2>
<p>Cryptocurrency malware aims to redirect cryptocurrency funds away from legitimate users’ wallets and into wallets that are controlled by the threat actor. When a computer is infected with a cryptocurrency clipper, the malware continually examines the victim’s clipboard to see if they have copied a cryptocurrency wallet address. The idea behind this is that if a person copies a wallet address to the clipboard, it indicates they might be in the process of transferring cryptocurrency from one wallet to another.</p>
<p>The clipper malware uses regular expressions (regexes) to identify what type of cryptocurrency the address pertains to. It then replaces the clipboard entry with a visually similar but adversary-controlled wallet address for the appropriate cryptocurrency. Later, when the victim pastes the address from the clipboard to conduct a transaction, they actually are sending cryptocurrency directly to the threat actor.</p>
<p>Since wallet addresses typically are very long – sometimes exceeding 40 alphanumeric characters – many people will not notice this change in address. This contributes to the effectiveness of clipper malware.</p>
<h2><a id="post-127548-_6w2fque06ys"></a>CryptoClippy Overview</h2>
<p>Unit 42 Managed Threat Hunting found victims across manufacturing, IT services, and real estate industries. This threat does not target specific industries, and the devices impacted were hit opportunistically.</p>
<p>A CryptoClippy infection begins with SEO poisoning, so that when a person searches for “WhatsApp Web,” the result leads them to a threat actor-controlled domain. Once there, victims are prompted to download a <span style="font-family: 'courier new', courier, monospace;">.zip</span> file that contains a <span style="font-family: 'courier new', courier, monospace;">.lnk</span> file, which is composed of malicious scripts. These scripts set off a chain of events that installs the clipper malware. This threat then monitors the victim’s clipboard activity for bitcoin transactions, such that their valid crypto wallet address will be replaced with one controlled by the threat actor.</p>
<p>The variant of CryptoClippy we investigated has a variety of additional capabilities to help the threat actor accomplish their crypto stealing activities. This includes being able to establish a Remote Desktop Protocol (RDP) backdoor through execution of an RC4-encrypted PowerShell script.</p>
<p>This script contains elements of Windows Management Instrumentation (WMI), terminal service registry manipulation, <a href="https://learn.microsoft.com/en-us/windows-server/administration/windows-commands/icacls" target="_blank" rel="noopener">icacls</a>, net commands and log clearing. These allow the attackers to maintain access outside the in-memory payload.</p>
<p>In addition, the variant also has functionality related to targeting Ethereum and bitcoin cryptocurrency wallets. This is not surprising, given the growing popularity of digital currencies in Latin America.</p>
<p>At the time of writing this post, the actor controlled wallets show recent activity. The bitcoin address has shown evidence of receiving 0.039954 BTC, which roughly equals $982.83. This bitcoin (BTC) balance came from four different BTC transactions. The Ethereum (ETH) address shows evidence of having received funds as well, with 0.110915556631181819 ETH (roughly equal to $186.32) being sent from three different ETH addresses.</p>
<p>The threat actors in this campaign employed a multi-stage approach to try to bypass signature- and heuristic-based security engines. This approach included obfuscation techniques to evade detection, including using obfuscated PowerShell scripts and encoded payloads.</p>
<p>The current low detection rate of this malware highlights this approach's effectiveness, as only a handful of vendors appear to be detecting this malware in VirusTotal. The vendors that do detect this have assigned generic names to this malware.</p>
<h2><a id="post-127548-_o5h614e21dya"></a><strong>Infection Chain for CryptoClippy</strong></h2>
<p>The infection flow for CryptoClippy starts with delivering an initial <span style="font-family: 'courier new', courier, monospace;">.zip</span> file that contains a <span style="font-family: 'courier new', courier, monospace;">.lnk</span> file that is composed of an obfuscated PowerShell command line script. Once the <span style="font-family: 'courier new', courier, monospace;">.lnk</span> file is double-clicked by the victim, the PowerShell script will be executed, which will download the second stage and several obfuscated/encrypted payloads (as discussed later in <a href="#post-127548-_dmbu726584d4">Stage 1 - Loader</a>).</p>
<p>When the second stage PowerShell script is executed, it deobfuscates/decrypts the CryptoClippy loader and executes it. The loader will then inject its stealer component into <span style="font-family: 'courier new', courier, monospace;">svchost.exe</span>.</p>
<p>CryptoClippy will set up user-mode <a href="https://learn.microsoft.com/en-us/windows/win32/api/winuser/nf-winuser-setwineventhook">event based hooks</a> and a callback function into clipboard APIs to replace the victim’s Ethereum/bitcoin crypto wallet with the actor’s crypto wallet, when it is copied to the clipboard. It also contains functionality to communicate with C2 servers.</p>

<h2><a id="post-127548-_pbx71hiwsgwk"></a>Delivery via Malvertising</h2>
<p>We’ve observed this malware campaign leveraging Google Ads and TDS. First the threat actor used Google Search ads that appeared in search results when users search for “WhatsApp Web.” This allowed them to trick victims into opening and downloading their malicious compressed <span style="font-family: 'courier new', courier, monospace;">.zip</span> file. After being alerted to the issue in early 2022, Google implemented additional protections and the company says its systems have improved at detecting and preventing renewed attack attempts.</p>
<p>In addition, threat actors used TDS to filter victims and bots on the malicious landing page. The TDS filters exclude bots and internet crawlers by using various criteria to determine if a client device is a real user and also a Portuguese speaker.</p>
<p>First, the TDS checks if the connecting device is coming from a virtual private network (VPN) IP address. Using a VPN can make it difficult for the attacker to determine the location and characteristics of the victim's device, and can also make it more difficult to deliver malicious content.</p>
<p>Next, the TDS makes user agent checks. The user agent is a string of text that a web browser sends to a website to identify itself and its capabilities. By checking the user agent of the client device, the TDS determines if the preferred browser language is Portuguese by checking the Accept-Language header. The TDS can also check for other information such as device type, operating system, browser version and geolocation.</p>
<p>Finally, the TDS follows the Hypertext Transfer Protocol (HTTP) GET header criteria. The HTTP GET header is a request message sent by a web browser to a server to retrieve a resource. The header can contain various information, such as the requested URL, the user agent, the Accept-Language header and the referer header. Checking the criteria in the HTTP GET header is another way the TDS can determine if the preferred language for the browser is Portuguese</p>
<p>If the TDS determines that the connection <em>does not</em> meet the above criteria, the victim is redirected to a different landing page (shown in Figure 2) and prompted to click on "Continuar para o WhatsApp Web" ("Continue to WhatsApp Web" in Portuguese). This will redirect to the legitimate WhatsApp Web domain without any further malicious activity, effectively avoiding detection by the victim and security software.</p>
<figure id="attachment_127556" aria-describedby="caption-attachment-127556" style="width: 900px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-127556" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/04/word-image-127548-2.png" alt="Image 2 is a screenshot of the content of the malicious domain, showing a couple paragraphs in Portuguese and a button with the text “Continuar para o WhatsApp Web&quot; (&quot;Continue to WhatsApp Web&quot; in Portuguese)" width="900" height="675" /><figcaption id="caption-attachment-127556" class="wp-caption-text">Figure 2. The content of the malicious domain.</figcaption></figure>
<p>However, if the TDS determines that the connection <em>does </em>meet the above criteria, the victim is redirected to the malicious landing page where they are prompted to download the malicious <span style="font-family: 'courier new', courier, monospace;">.zip</span> file (shown in Figures 3 and 4).</p>
<figure id="attachment_127558" aria-describedby="caption-attachment-127558" style="width: 900px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-127558" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/04/word-image-127548-3.png" alt="Image 3 is a screenshot of victim browsing history, showing their searches for WhatsApp web " width="900" height="56" /><figcaption id="caption-attachment-127558" class="wp-caption-text">Figure 3. The victim’s browsing history.</figcaption></figure>
<p>This site disguises itself as the official WhatsApp Web site, offering WhatsApp Desktop. Downloading this file from <span style="font-family: 'courier new', courier, monospace;">mydigitalrevival[.]com</span> resulted in the initial infection — <span style="font-family: 'courier new', courier, monospace;">WhatsApp.zip</span> (shown in Figure 4).</p>
<p><figure id="attachment_127560" aria-describedby="caption-attachment-127560" style="width: 900px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-127560" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/04/word-image-127548-4.png" alt="Image 4 is a screenshot of the mydigitialrevival[.]com website. <p>When a victim successfully loaded the malicious webpage and downloaded the offered <span style="font-family: 'courier new', courier, monospace;">.zip</span> file, it contained a <span style="font-family: 'courier new', courier, monospace;">.lnk</span> file, as shown in Figure 5.</p>
<figure id="attachment_127562" aria-describedby="caption-attachment-127562" style="width: 900px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-127562" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/04/word-image-127548-5.png" alt="Image 5 is a screenshot of the contents of the .zip file containing the .lnk file. <p>For more technical details on the <span style="font-family: 'courier new', courier, monospace;">.lnk</span> file, please see <a href="#post-127548-_eel2zvjg0ygs">Infection via LNK File</a>.</p>
<p>Following the execution of the <span style="font-family: 'courier new', courier, monospace;">.lnk</span> file, Unit 42 researchers observed the following files downloaded to <span style="font-family: 'courier new', courier, monospace;">C:\Users\&lt;USERNAME&gt;\AppData\Roaming\Ricoly</span>:</p>
<ul>
<li><span style="font-family: 'courier new', courier, monospace;">Ricoly.bat</span></li>
<li><span style="font-family: 'courier new', courier, monospace;">Ricoly.ps1</span></li>
<li>Two obfuscated encrypted files (<span style="font-family: 'courier new', courier, monospace;">ps</span>, <span style="font-family: 'courier new', courier, monospace;">sc</span>)</li>
<li>One obfuscated encrypted auxiliary config file (<span style="font-family: 'courier new', courier, monospace;">pf</span>)</li>
</ul>
<p>The first PowerShell script loader <span style="font-family: 'courier new', courier, monospace;">Ricoly.ps1</span> is launched and executed by the <span style="font-family: 'courier new', courier, monospace;">Ricoly.bat</span> batch file. The <span style="font-family: 'courier new', courier, monospace;">Ricoly.ps1</span> script's purpose is to decrypt the second-stage obfuscated/encrypted script <span style="font-family: 'courier new', courier, monospace;">ps</span>. More in-depth information about these loader files can be found in <a href="#post-127548-_dmbu726584d4">Stage 1 - Loader</a> and <a href="#post-127548-_3bcvjj59dymm">Stage 2 - Loader</a>.</p>
<p>The functionality of the second stage PowerShell script ps is to act as a reflective PE loader. The ps file will target an encrypted EXE file named <span style="font-family: 'courier new', courier, monospace;">sc</span> that is written to the file system. The <span style="font-family: 'courier new', courier, monospace;">sc</span> file is reflectively loaded and eventually functions as the main loader for CryptoClippy.</p>
<h2><a id="post-127548-_6qmmuwuwnxtm"></a>Main CryptoClippy Executable</h2>
<p>After the execution of the <span style="font-family: 'courier new', courier, monospace;">.lnk</span> file and the two subsequent PowerShell scripts, the CryptoClippy operation consists of an executable loader and the main CryptoClippy executable. The loader for CryptoClippy is a 64-bit EXE file, where the main stealer EXE file is embedded inside of the <span style="font-family: 'courier new', courier, monospace;">.data</span> section. The loader uses <span style="font-family: 'courier new', courier, monospace;">syscalls</span> and appears to overlap with <a href="https://github.com/jthuraisamy/SysWhispers2" target="_blank" rel="noopener">SysWhispers2</a> implementation for execution.</p>
<p>SysWhispers2 is a stealthier implementation for code execution, which is used to bypass inspection of user-mode APIs. This bypass is possible because user-mode APIs are typically used by applications, including introspection by AV/EDR products, through user-mode hooks.</p>
<p>The main executable for CryptoClippy is written in C and it is statically compiled with the Transport Layer Security (TLS) library <a href="https://github.com/Mbed-TLS/mbedtls" target="_blank" rel="noopener">Mbed-TLS</a>. The main feature of the executable includes its own algorithm for name generation (i.e., the file path, mutex and event object). For more in-depth information on this algorithm, see <a href="#post-127548-_bry43oioklcd" target="_blank" rel="noopener">Folder Name and Mutex String Generator</a>.</p>
<p>The main executable also makes extensive use of a local auxiliary file called <span style="font-family: 'courier new', courier, monospace;">pf</span>, which contains an encrypted certificate. CryptoClippy uses its own algorithm to deobfuscate the character lookup table from the auxiliary file to build the plaintext certificate from pointers to values in the lookup table.</p>
<p>This threat also uses two different RC4 keys for various string decryption functions. The string decryption routine will use a structure built in memory for the index location into the structure, string offset and string size. These strings are related to object names, domains, crypto wallets and persistence scripts.</p>
<ul>
<li>RC4 Key: <span style="font-family: 'courier new', courier, monospace;">1b43233d5a054808061c190336320e46</span></li>
<li>RC4 Key: <span style="font-family: 'courier new', courier, monospace;">4646070B47445451604F291809444703</span></li>
</ul>
<h2><a id="post-127548-_xwpf6tcpkl6"></a>Crypto Stealing Operation</h2>
<p>The primary objective of CryptoClippy is to identify crypto wallet strings being copied into the clipboard. CryptoClippy sets up user-mode event hooks and uses a callback function to execute replacing Ethereum or bitcoin based wallets copied into the Windows clipboard with hardcoded wallet values owned by the threat actor. For more technical details, please see <a href="#post-127548-_8d9pcjfia23t" target="_blank" rel="noopener">User-Mode Event Hook Setup</a>.</p>
<p>Figure 6 illustrates the window capturing the clipboard input. Based on the execution flow of the loader, this particular application would normally not be visible.</p>
<figure id="attachment_127564" aria-describedby="caption-attachment-127564" style="width: 986px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-127564" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/04/word-image-127548-6.png" alt="Image 6 is a screenshot of the clipboard input. It shows many lines of code." width="986" height="559" /><figcaption id="caption-attachment-127564" class="wp-caption-text">Figure 6. Capturing the clipboard input.</figcaption></figure>
<p>The decompiled view of CryptoClippy’s clipboard function (shown in Figure 7) reveals that the wallet addresses used by the threat actor will be RC4 decrypted to be used later on, in any replacement of crypto wallets that take place.</p>
<p>CryptoClippy will perform a string check against the data being pushed into the clipboard. It will check for common leading characters associated with Ethereum or bitcoin related wallets. The string copied into the clipboard will be looped through to determine the character count. On line 27 of Figure 7, 25 is subtracted from the total character count and compared to see if it is less than or equal to 9.</p>
<figure id="attachment_127566" aria-describedby="caption-attachment-127566" style="width: 874px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-127566" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/04/word-image-127548-7.png" alt="Image 7 is a screenshot of the decompiled view of CryptoClippy’s clipboard function. Otherwise the code path will be skipped, and the string in the clipboard will be checked to see if it starts with either <span style="font-family: 'courier new', courier, monospace;">bc1</span> (related to bitcoin) or <span style="font-family: 'courier new', courier, monospace;">0x</span> (related to Ethereum).</p>
<p>The debugging view shown in Figure 8 shows the API <span style="font-family: 'courier new', courier, monospace;">GetClipboardData</span> and the return value of the API, which contains the example Ethereum wallet copied.</p>
<figure id="attachment_127568" aria-describedby="caption-attachment-127568" style="width: 900px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-127568" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/04/word-image-127548-8.png" alt="Image 8 is a screenshot of an Ethereum wallet. <p>Once the clipboard data is obtained, the string checks noted in the decompiled view above will be performed, looking for specific crypto wallet-like characters used. In Figure 9, an Ethereum wallet was provided as input and was subsequently checked for the starting character value of <span style="font-family: 'courier new', courier, monospace;">0x</span>.</p>

<p>To maintain access to the victim computer, CryptoClippy obtains persistence by adding <span style="font-family: 'courier new', courier, monospace;">Ricoly.lnk</span> to the Startup folder (<span style="font-family: 'courier new', courier, monospace;">\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup\Ricoly.lnk</span>). The <span style="font-family: 'courier new', courier, monospace;">Ricoly.lnk</span> would be launched as the user logs in, which would execute the <span style="font-family: 'courier new', courier, monospace;">Ricoly.bat</span> script, which would then execute <span style="font-family: 'courier new', courier, monospace;">Ricoly.ps1</span>.</p>
<p>Additional efforts to maintain access are contained in an RC4-encrypted PowerShell script called <span style="font-family: 'courier new', courier, monospace;">Tozzia.ps1</span>. The CryptoClippy payload injected into <span style="font-family: 'courier new', courier, monospace;">svchost.exe</span> extracts the <span style="font-family: 'courier new', courier, monospace;">Tozzia.bat</span> and <span style="font-family: 'courier new', courier, monospace;">Tozzia.ps1</span> files.</p>
<p>Along with two other scripts, the <span style="font-family: 'courier new', courier, monospace;">Tozzia.ps1</span> script contains various capabilities related to registry modifications of terminal services, local account creation, net commands and event log clearing. These three scripts are covered in more depth in <a href="#post-127548-_hkr3os9mll5b">Backdoor</a>.</p>
<h2><a id="post-127548-_2an8ryq91inv"></a>Conclusion</h2>
<p>The overall proliferation of cryptocurrency-focused malware has been gaining momentum the last few years. The threat actor in this particular instance has implemented the following activities in their campaigns:</p>
<ul>
<li>Incorporating malvertising</li>
<li>Using TDS</li>
<li>Limiting the scope of targeting</li>
<li>Keying the payloads to the environment</li>
<li>Employing more complex and stealthier techniques to hide the overall functionality of the loader and clipper</li>
<li>Creating custom algorithms for data manipulation</li>
</ul>
<p>These practices highlight an interesting shift, especially in cryptocurrency-targeting malware. We often see two particular methods for the distribution of such threats. They may be pushed using a botnet like with <a href="https://blog.cyble.com/2022/11/02/new-laplas-clipper-distributed-by-smokeloader/" target="_blank" rel="noopener">Laplas Clipper and Smokeloader</a>, or there may be an underlying malware-as-a-service (MaaS) element such as with <a href="https://blogs.blackberry.com/en/2022/06/threat-spotlight-eternity-project-maas-goes-on-and-on" target="_blank" rel="noopener">Eternity Clipper</a>.</p>

<h3><a id="post-127548-_eel2zvjg0ygs"></a>Infection via LNK File</h3>
<p>The <span style="font-family: 'courier new', courier, monospace;">.lnk</span> file found within the <span style="font-family: 'courier new', courier, monospace;">.zip</span> file that is initially downloaded by victims contains a truncated command, shown in Figure A1. Double-clicking the file will execute an obfuscated command located in the target field of the shortcut responsible for checking into the actor-controlled domain, shown in Figure A3.</p>
<figure id="attachment_127572" aria-describedby="caption-attachment-127572" style="width: 403px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-127572" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/04/word-image-127548-10.png" alt="Image 10 is a screenshot of the WhatsApp-RKQT .lnk file Properties pane. The Target field contains the command to run. " width="403" height="544" /><figcaption id="caption-attachment-127572" class="wp-caption-text">Figure A1. The LNK file with the target field containing the command to run.</figcaption></figure>
<p>The <span style="font-family: 'courier new', courier, monospace;">.lnk</span> file utilizes a few different character padding methods for obfuscation, which include the following character sets:</p>
<ul>
<li><span style="font-family: 'courier new', courier, monospace;">^</span></li>

<p>The LNK deobfuscates to the following PowerShell command, which will upload the string <span style="font-family: 'courier new', courier, monospace;">uiPX</span> to the attacker-controlled domain <span style="font-family: 'courier new', courier, monospace;">tunneldrive[.]com</span> via HTTP protocol (shown in Figure A3).</p>

<p>It should be noted that in cases observed by Unit 42 researchers, the infection chain traces to files with the extension <span style="font-family: 'courier new', courier, monospace;">.lnk</span>. These files are usually contained in a <span style="font-family: 'courier new', courier, monospace;">.zip</span> archive. However, during pivoting in this campaign, we observed another variation to the infection flow where the initial payload is an <span style="font-family: 'courier new', courier, monospace;">.exe</span> file <a href="https://www.virustotal.com/gui/domain/mydigitalrevival.com/relations" target="_blank" rel="noopener">(reported on VirusTotal)</a> that is linked to the previously mentioned domain <span style="font-family: 'courier new', courier, monospace;">mydigitalrevival[.]com</span>.</p>
<p>When we examined the <span style="font-family: 'courier new', courier, monospace;">.exe</span> file that impersonates WhatsApp Web, we found that once it's executed, it will contact the previously mentioned <span style="font-family: 'courier new', courier, monospace;">tunneldrive[.]com</span> domain. It will then drop and execute a <span style="font-family: 'courier new', courier, monospace;">.bat</span> file to delete itself.</p>
<p>This <span style="font-family: 'courier new', courier, monospace;">.bat</span> file (shown in Figure A4) uses an obfuscation that gradually builds up its payload <a href="https://www.linkedin.com/pulse/malware-dropper-gabriel-sulbaran/" target="_blank" rel="noopener">similar to the one detailed in this blog,</a> but with more obfuscation. In addition, this obfuscation type appears to be adopted from Daniel Bohannon’s <a href="https://github.com/danielbohannon/Invoke-Obfuscation/search?q=environment&amp;unscoped_q=environment" target="_blank" rel="noopener">Invoke-Obfuscation</a> project, which is a technique widely used by Brazilian threat actors.</p>

<h3><a id="post-127548-_dmbu726584d4"></a>Stage 1 - Loader</h3>
<p>The first PowerShell script loader <span style="font-family: 'courier new', courier, monospace;">Ricoly.ps1</span> is launched and executed by the <span style="font-family: 'courier new', courier, monospace;">Ricoly.bat</span> batch file. The <span style="font-family: 'courier new', courier, monospace;">Ricoly.ps1</span> script's purpose is to decrypt a dropped second-stage obfuscated/encrypted script <span style="font-family: 'courier new', courier, monospace;">ps</span> also residing in <span style="font-family: 'courier new', courier, monospace;">C:\Users\&lt;USERNAME&gt;\AppData\Roaming\Ricoly</span>.</p>
<p>The <span style="font-family: 'courier new', courier, monospace;">Ricoly.ps1</span> script utilizes an XOR key that is mostly hardcoded (statically set), while one portion of the XOR key is dynamically derived from the processor ID of the computer. This keys the payload and adds an additional layer of anti-analysis, as shown in Figure A5.</p>
<figure id="attachment_127580" aria-describedby="caption-attachment-127580" style="width: 900px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-127580" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/04/word-image-127548-14.png" alt="Image 14 is a screenshot of many lines of code. It is the PowerShell script that responsible for the XOR decrypting. Highlighted is the XOR key in red. Highlighted in purple is the invoking .ps file. " width="900" height="278" /><figcaption id="caption-attachment-127580" class="wp-caption-text">Figure A5. PowerShell script responsible for XOR decrypting the next script to be executed.</figcaption></figure>
<h3><a id="post-127548-_3bcvjj59dymm"></a>Stage 2 - Loader</h3>
<p>We used the following CyberChef XOR recipe (also shown in Figure A6) to decrypt the second stage payload:</p>
<ul>
<li>XOR KEY: <span style="font-family: 'courier new', courier, monospace;">b8 55 2e 7c 48 f6 44 ea c1 7c 9e 09 eb b0 f5 f1 23 f4 22 42 02 15 a1 cb ed 69 50 c1 1b 46 bf eb fb ff 00 08 06 ec bc c5 55 c1 de 96 14 3b bb a8 85 59 13 70 0b 28</span></li>
</ul>
<figure id="attachment_127582" aria-describedby="caption-attachment-127582" style="width: 900px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-127582" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/04/word-image-127548-15.png" alt="Image 15 is a screenshot of many lines of code showing the decoded second stage PowerShell .ps file after running the recipe in CyberChef. " width="900" height="581" /><figcaption id="caption-attachment-127582" class="wp-caption-text">Figure A6. CyberChef recipe to decode second stage PowerShell script file <span style="font-family: 'courier new', courier, monospace;">ps</span>.</figcaption></figure>
<p>An encrypted EXE file named <span style="font-family: 'courier new', courier, monospace;">sc</span> is written to the file system when it’s downloaded during the first stage mentioned above. This EXE file will then be the target and focus of the second stage <span style="font-family: 'courier new', courier, monospace;">ps</span> script.</p>
<p>The functionality of the second stage PowerShell script <span style="font-family: 'courier new', courier, monospace;">ps</span> is to act as a reflective PE loader. It does so using <span style="font-family: 'courier new', courier, monospace;">.NET</span> methods and <span style="font-family: 'courier new', courier, monospace;">D/invoke</span> to work with <span style="font-family: 'courier new', courier, monospace;">.NET</span> delegates to assist in evasion and dynamic resolving.</p>
<p>The <span style="font-family: 'courier new', courier, monospace;">ps</span> script will determine if the operating system is 32-bit or 64-bit. It will also determine the API functionality needed from <span style="font-family: 'courier new', courier, monospace;">kernel32.dll</span> to act as a reflective loader. It does this by using <span style="font-family: 'courier new', courier, monospace;">GetDelegateForFunctionPointer</span> for resolving the needed APIs such as <span style="font-family: 'courier new', courier, monospace;">VirtualAlloc</span> and <span style="font-family: 'courier new', courier, monospace;">CreateThread</span>.</p>
<p>The <span style="font-family: 'courier new', courier, monospace;">ps</span> script will then XOR decrypt the payload and invoke the EXE file <span style="font-family: 'courier new', courier, monospace;">sc</span> (as shown in Figure A7).</p>
<figure id="attachment_127584" aria-describedby="caption-attachment-127584" style="width: 900px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-127584" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/04/word-image-127548-16.png" alt="Image 16 is a screenshot of many lines of code. The red highlight code shows the APIs, and the code highlighted in purple is the XOR key. " width="900" height="494" /><figcaption id="caption-attachment-127584" class="wp-caption-text">Figure A7. Second stage PowerShell script using <span style="font-family: 'courier new', courier, monospace;">D/invoke</span> methods and XOR operation.</figcaption></figure>
<p>The EXE file <span style="font-family: 'courier new', courier, monospace;">sc</span>, reflectively loaded into memory, acts as the main loader using <span style="font-family: 'courier new', courier, monospace;">syscalls</span> to inject code into another newly created process. The newly created <span style="font-family: 'courier new', courier, monospace;">svchost.exe</span> process contains CryptoClippy.</p>
<h3><a id="post-127548-_bry43oioklcd"></a>Folder Name and Mutex String Generator</h3>
<p>The first few functions of the stealer will first map out the file system to create a folder to be used. This is achieved by using the APIs <span style="font-family: 'courier new', courier, monospace;">GetWindowsDirectoryA</span>, <span style="font-family: 'courier new', courier, monospace;">GetVolumeInformationA</span> and <span style="font-family: 'courier new', courier, monospace;">SHGetFolderPathA</span>.</p>
<p>The stealer contains a function that will uniquely create alphanumeric strings. This string generator is used in several locations within the main program. It is first used to create a unique folder name under the <span style="font-family: 'courier new', courier, monospace;">AppData</span> path. To do this, a constant value is used as a parameter to the string generator (shown in Figure A8).</p>
<p>In this instance, the constant is <span style="font-family: 'courier new', courier, monospace;">0x79FE6D</span>, which will be used within the string generator function and mapped to the format string <span style="font-family: 'courier new', courier, monospace;">%08x-%08x</span>.</p>
<figure id="attachment_127586" aria-describedby="caption-attachment-127586" style="width: 900px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-127586" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/04/word-image-127548-17.png" alt="Image 17 is a screenshot of two panes. On the left highlighted in red is the constant used for the folder. On the right highlighted in purple are the 11 references to the generator function. " width="900" height="286" /><figcaption id="caption-attachment-127586" class="wp-caption-text">Figure A8. Constant value referenced and xrefs of function alphanumeric generator.</figcaption></figure>
<p>The string-generating function will use the constant value provided to the function and the volume serial number queried from the victim’s system. It will create the following string as an example: <span style="font-family: 'courier new', courier, monospace;">079fe6d-de786dd1</span> (shown in Figure A9).</p>
<figure id="attachment_127588" aria-describedby="caption-attachment-127588" style="width: 900px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-127588" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/04/word-image-127548-18.png" alt="Image 18 is a screenshot of many lines of code. Highlighted in red is the constant value. Highlighted in yellow is the volume serial. " width="900" height="469" /><figcaption id="caption-attachment-127588" class="wp-caption-text">Figure A9. Constant value and volume serial number concatenated into format string.</figcaption></figure>
<p>The algorithm will shuffle the string <span style="font-family: 'courier new', courier, monospace;">079fe6d-de786dd1</span> by splitting the string by each character. The first operation will be an XOR performed by the first character of the string against the value <span style="font-family: 'courier new', courier, monospace;">FFFFFFFF</span>.</p>
<p>The resulting operation will then be shifted right by 1 and XORed by the constant <span style="font-family: 'courier new', courier, monospace;">0x82F63B78</span>. The resulting XOR operation will then follow the same set of operations a total of eight times per each character in the 16 character example string <span style="font-family: 'courier new', courier, monospace;">079fe6d-de786dd1</span>.</p>
<p>The completion of the algorithm returns the folder name to be added to the <span style="font-family: 'courier new', courier, monospace;">AppData</span> directory (shown in Figure A10). The folder name in this instance was <span style="font-family: 'courier new', courier, monospace;">55abf82d</span> and the full path was<span style="font-family: 'courier new', courier, monospace;"> C:\Users\xxx\AppData\Roaming\55abf82d</span>. This will vary based on the constant and volume serial number provided as input to create unique alphanumeric strings.</p>
<figure id="attachment_127590" aria-describedby="caption-attachment-127590" style="width: 900px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-127590" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/04/word-image-127548-19.png" alt="Image 19 is a screenshot of many lines of code in two panes. In the left pane highlighted in red is the algorithm. Highlighted in purple on the right is the result, which is 94ee4d61. " width="900" height="662" /><figcaption id="caption-attachment-127590" class="wp-caption-text">Figure A10. Result of the algorithm used, which is the folder name to be created.</figcaption></figure>

<figure id="attachment_127592" aria-describedby="caption-attachment-127592" style="width: 855px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-127592" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/04/word-image-127548-20.png" alt="Image 20 is a screenshot of the constant value used for mutex creation with the volume serial number. <h3><a id="post-127548-_8d9pcjfia23t"></a>User-Mode Event Hook Setup</h3>
<p>In order for the stealer to facilitate clipping out crypto wallets copied to the clipboard and replacing them with the hardcoded wallets included in the binary, it will start by calling the API <span style="font-family: 'courier new', courier, monospace;">SetWinEventHook</span> to set up event hooks for calling the function responsible for interacting with clipboard data when the <a href="https://learn.microsoft.com/en-us/windows/win32/winauto/event-constants" target="_blank" rel="noopener">specific events</a> are triggered.</p>
<ul>
<li><span style="font-family: 'courier new', courier, monospace;">EVENT_OBJECT_FOCUS</span></li>
<li><span style="font-family: 'courier new', courier, monospace;">EVENT_OBJECT_VALUECHANGE</span></li>
<li><span style="font-family: 'courier new', courier, monospace;">EVENT_SYSTEM_FOREGROUND</span></li>
</ul>
<p>The event hooks setup will have a process scope of receiving events from all processes as well as all existing threads, and then skip hooking the owning process responsible for creating the hooks. Following the setup of event hooks is the creation of a Windows event object (shown in Figure A12). It does this by calling the Windows API <span style="font-family: 'courier new', courier, monospace;">CreateEventA</span>, which is used for synchronization, and it will signal to a thread when a particular event has occurred and completed.</p>
<figure id="attachment_127594" aria-describedby="caption-attachment-127594" style="width: 900px" class="wp-caption aligncenter"><img decoding="async" class="wp-image-127594" src="https://unit42.paloaltonetworks.com/wp-content/uploads/2023/04/word-image-127548-21.png" alt="Image 21 is a screenshot of the decompiled Windows event hooks and registering class. " width="900" height="1295" /><figcaption id="caption-attachment-127594" class="wp-caption-text">Figure A12. Decompiled view of windows event hooks and registering class.</figcaption></figure>
<p>Following the Windows object event creation, the stealer will use the API <span style="font-family: 'courier new', courier, monospace;">RegisterClassExA</span>. This API is provided a pointer to the <span style="font-family: 'courier new', courier, monospace;">WNDCLASSEX</span> structure containing <span style="font-family: 'courier new', courier, monospace;">wcbClass</span> and various structure fields.</p>
<p>After this, <span style="font-family: 'courier new', courier, monospace;">CreateWindowExA</span> is called, which will create a window with the structure pointed to by the previous API. The important field in this structure for our analysis is the field <span style="font-family: 'courier new', courier, monospace;"><a href="https://learn.microsoft.com/en-us/windows/win32/api/winuser/nc-winuser-wndproc" target="_blank" rel="noopener">lpfnWndProc</a>,</span> which points to the malware author’s clipboard function.</p>
<h3><a id="post-127548-_hkr3os9mll5b"></a>Backdoor</h3>

The batch file <span style="font-family: 'courier new', courier, monospace;">Tozzia.bat</span> is executed, which in turn will execute <span style="font-family: 'courier new', courier, monospace;">Tozzia.ps1</span> (shown in Figure A13).</p>

(function($) {
	// Migrated from the unit42-v5 + Modifications
	var subscribeSuccess = false;
	var email = document.getElementById('newsletter-email');
	var subscription_form = document.getElementById('unit42footerSubscription_form');
	var subscription_form_button = document.getElementById('unit42footerSubscription_form_button');
	window.captchaComplete = function() {
		subscribeSuccess = true;
		if ($(mail).val() != '' && isEmail($(mail).val())) {
			$(subscription_form_button).removeClass('is-disabled');
		}
		setTimeout(function() {
			$(email).focus();
			$('.g-recaptcha iframe').attr('tabindex', '-1');
		}, 100)
	}
	window.captchaExpires = function() {
		subscribeSuccess = false;
		$(subscription_form_button).addClass('is-disabled', true);
	} 
	$(subscription_form).submit(function(e) {
		e.preventDefault();
		e.stopImmediatePropagation();
		updateEmailMask();
		var success = true;
		var form = $(this);
		var mail = form.find('input[name="Email"]');
		
		if (mail.val() === '') {
			mail.addClass('has-error');
			showError(1);
			success = false;	
		}  else if (!isEmail(mail.val())){
			showError(2);
			success = false;
		}
		else {
			mail.removeClass('has-error');
			$('.error-mail').addClass('d-none');
		}

		if (!subscribeSuccess) {
			$('.error-recaptcha').removeClass('d-none');
		} else {
			$('.error-recaptcha').addClass('d-none');
		}

		if (success && subscribeSuccess) {
			$.ajax({
				type: 'POST',
				url: form.attr('action'),
				data: form.serialize(),
				beforeSend: function() {
                 form.find('button').addClass('is-loading');
    			},
				success: function(msg) {
					form.find('.form-success-message').html('<p class="success-message">You have been successfully subscribed</p>');
					form.find('button').removeClass('is-loading');
					$(email).val('');
					clearError();
				},
				error: function(jqXHR, textStatus, errorThrown) {
					$(subscription_form_button).addClass('is-disabled', true);
					form.find('button').removeClass('is-loading');
					
				}
			});
		}
		return false;
	});
	function showError(error_type){
		if(error_type == 1) {
			$('.error-mail').text("Please enter the email address.").addClass('error-show');
			$(subscription_form_button).addClass('is-disabled');
		} else if(error_type == 2){
			$('.error-mail').text("Please provide a valid e-mail address.").addClass('error-show');
			$(subscription_form_button).addClass('is-disabled');
		}	
		$(subscription_form_button).removeClass('is-loading');
	}
	function clearError(){
		$('.error-mail').text("").removeClass('error-show');;
		$(subscription_form_button).removeClass('is-loading');
		$(subscription_form_button).removeClass('is-disabled');
	}
	
	$(email).on('input', function (event) { 
		var email = $(this).val();
		if (isEmail(email) ) {			
			clearError();
		} else if(email == ""){
			clearError();
		} else{
			showError(2);			  
		}
    });

	function isEmail(email) {
		var re = /^(([^<>()\[\]\\.,;:\s@"]+(\.[^<>()\[\]\\.,;:\s@"]+)*)|(".+"))@((\[[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\])|(([a-zA-Z\-0-9]+\.)+[a-zA-Z]{2,}))$/;
		return re.test(String(email).toLowerCase());
	}
	
	var captcha_loaded = false;
         if(!captcha_loaded){
            // recaptcha on foucs call
            $(document).on('change paste keyup', '#newsletter-email', function () {
               if($('.g-recaptcha').hasClass('d-none')){
                $('.g-recaptcha').removeClass('d-none');

                  // trigger loading api.js (recaptcha.js) script

                  var script = document.createElement('script');
                  script.type = 'text/javascript';
                  script.src = 'https://www.google.com/recaptcha/api.js?hl=en_US';
                  head.appendChild(script);
               }
            });
        }
	
	function updateEmailMask() {
		var email = $("#unit42footerSubscription_form input[name='Email']").val();
		if (email && email.trim() != '') {
			var maskedEmail = maskEmailAddress(email);
			$("#unit42footerSubscription_form input[name='emailFormMask']").val(maskedEmail);
		}
	}

	function maskEmailAddress (emailAddress) {
		function mask(str) {
			var strLen = str.length;
			if (strLen > 4) {
				return str.substr(0, 1) + str.substr(1, strLen - 1).replace(/\w/g, '*') + str.substr(-1,1);
			}
			return str.replace(/\w/g, '*');
		}
		return emailAddress.replace(/([\w.]+)@([\w.]+)(\.[\w.]+)/g, function (m, p1, p2, p3) {
			return mask(p1) + '@' + mask(p2) + p3;
		});

		return emailAddress;
	}
