											<h2 id="ms--in-this-article" class="title is-6 margin-block-xs">In this article</h2>
										</nav><!-- <content> --><blockquote>
<p>Applies to: Windows Server 2022, Windows Server 2019, Windows Server 2016, Windows Server 2012 R2, Windows Server 2012</p>
</blockquote>
<p>Expands one or more compressed files. You can also use this command to retrieve compressed files from distribution disks.</p>
<p>The <strong>expand</strong> command can also run from the Windows Recovery Console, using different parameters. For more information, see <a href="/en-us/windows-hardware/manufacture/desktop/windows-recovery-environment--windows-re--technical-reference" data-linktype="absolute-path">Windows Recovery Environment (WinRE)</a>.</p>
<h2 id="syntax">Syntax</h2>
<pre><code>expand [-r] &lt;source&gt; &lt;destination&gt;
expand -r &lt;source&gt; [&lt;destination&gt;]
expand -i &lt;source&gt; [&lt;destination&gt;]
expand -d &lt;source&gt;.cab [-f:&lt;files&gt;]
expand &lt;source&gt;.cab -f:&lt;files&gt; &lt;destination&gt;
</code></pre>
<h3 id="parameters">Parameters</h3>
<table>
<thead>
<tr>
<th>Parameter</th>
<th>Description</th>
</tr>
</thead>
<tbody>
<tr>
<td>-r</td>
<td>Renames expanded files.</td>
</tr>
<tr>
<td>source</td>
<td>Specifies the files to expand. <em>Source</em> can consist of a drive letter and colon, a directory name, a file name, or a combination of these. You can use wildcards (<strong>*</strong> or <strong>?</strong>).</td>
</tr>
<tr>
<td>destination</td>
<td>Specifies where files are to be expanded.<p>If <em>source</em> consists of multiple files and you don't specify <strong>-r</strong>, the <em>destination</em> must be a directory that already exists. <em>Destination</em> can consist of a drive letter and colon, a directory name, a file name, or a combination of these. Destination <code>file \| path</code> specification.</p></td>
</tr>
