      <h2 class="cmp-title__text">Executive summary</h2>
    
</div>

    


</div>
<div class="text text__pt-none aem-GridColumn aem-GridColumn--default--12">
<div data-cmp-is="text" class="cmp-text" data-cmp-data-layer="{&#34;text-42e5051f3e&#34;:{&#34;@type&#34;:&#34;akamai/components/text&#34;,&#34;repo:modifyDate&#34;:&#34;2023-08-08T19:40:33Z&#34;,&#34;xdm:text&#34;:&#34;&lt;ul>\r\n&lt;li>&lt;p>Akamai researchers have discovered an ongoing server-side template injection (CVE-2022-24086) campaign that is exploiting digital commerce websites. This campaign targets Magento 2 shops, and we have dubbed it Xurum in reference to the domain name of the attacker’s command and control (C2) server.&amp;nbsp;&lt;/p>\r\n&lt;/li>\r\n&lt;li>&lt;p>We have observed activity in this campaign since at least January 2023. The attacker seems to be interested in payment stats from the orders in the victim&#39;s Magento store placed in the past 10 days.&amp;nbsp;&lt;/p>\r\n&lt;/li>\r\n&lt;li>&lt;p>The attacker registers a new Magento component and masks it as “GoogleShoppingAds.”&lt;/p>\r\n&lt;/li>\r\n&lt;li>&lt;p>The attacker uses an advanced web shell named “wso-ng” that is activated only when the attacker sends the cookie “magemojo000” to the backdoor “GoogleShoppingAds” component.&amp;nbsp;&lt;/p>\r\n&lt;/li>\r\n&lt;li>&lt;p>The web shell login page masquerades as an error page containing a hidden login form that attempts to glean victim credentials.&lt;/p>\r\n&lt;/li>\r\n&lt;li>&lt;p>The attacker creates a backdoor admin user in Magento, named “mageplaza” or “mageworx,” as another deception trick as those are the names of the popular Magento extensions stores.&lt;/p>\r\n&lt;/li>\r\n&lt;li>&lt;p>The attacker uses the older Dirty COW exploit (CVE-2016-5195) to attempt privilege escalation within Linux.&amp;nbsp;&lt;/p>\r\n&lt;/li>\r\n&lt;li>&lt;p>Evidence indicates Russian origins for this threat.&amp;nbsp;&lt;/p>\r\n&lt;/li>\r\n&lt;li>&lt;p>Some of the websites involved in this campaign were observed to be infected with simple JavaScript-based skimmers with no attempt to obfuscate or hide its existence.&lt;/p>\r\n&lt;/li>\r\n&lt;/ul>\r\n&#34;}}">
    <ul>
<li><p>Akamai researchers have discovered an ongoing server-side template injection (CVE-2022-24086) campaign that is exploiting digital commerce websites. This campaign targets Magento 2 shops, and we have dubbed it Xurum in reference to the domain name of the attacker’s command and control (C2) server. </p>
</li>
<li><p>We have observed activity in this campaign since at least January 2023. The attacker seems to be interested in payment stats from the orders in the victim's Magento store placed in the past 10 days. </p>
</li>
<li><p>The attacker registers a new Magento component and masks it as “GoogleShoppingAds.”</p>
</li>
<li><p>The attacker uses an advanced web shell named “wso-ng” that is activated only when the attacker sends the cookie “magemojo000” to the backdoor “GoogleShoppingAds” component. </p>
</li>
<li><p>The web shell login page masquerades as an error page containing a hidden login form that attempts to glean victim credentials.</p>
</li>
<li><p>The attacker creates a backdoor admin user in Magento, named “mageplaza” or “mageworx,” as another deception trick as those are the names of the popular Magento extensions stores.</p>
</li>
<li><p>The attacker uses the older Dirty COW exploit (CVE-2016-5195) to attempt privilege escalation within Linux. </p>
</li>
<li><p>Evidence indicates Russian origins for this threat. </p>
</li>
<li><p>Some of the websites involved in this campaign were observed to be infected with simple JavaScript-based skimmers with no attempt to obfuscate or hide its existence.</p>
</li>
</ul>

<div data-cmp-is="text" class="cmp-text" data-cmp-data-layer="{&#34;text-635abe8069&#34;:{&#34;@type&#34;:&#34;akamai/components/text&#34;,&#34;repo:modifyDate&#34;:&#34;2024-07-16T12:20:21Z&#34;,&#34;xdm:text&#34;:&#34;&lt;p>&lt;a href=\&#34;/content/akamai/en/lp/soti/2024/web-scraping-report-2024.html\&#34; target=\&#34;_self\&#34;>Digital commerce is a widely targeted industry&lt;/a> for cybercriminals in general, but Magento’s widespread popularity has unfortunately made it an especially appealing (and lucrative) target.     <p><a href="/lp/soti/web-scraping-report-2024" target="_self">Digital commerce is a widely targeted industry</a> for cybercriminals in general, but Magento’s widespread popularity has unfortunately made it an especially appealing (and lucrative) target. The most notable specific attack is <a href="/glossary/what-is-magecart" target="_self">Magecart</a>,  in which the attackers aim to deploy JavaScript-based skimmers to illicitly acquire sensitive user information. These <a href="/blog/security-research/new-magecart-hides-behind-legit-domains" target="_self">Magecart threat actors</a> take advantage of well-known web vulnerabilities to achieve this acquisition.</p>
<p><b>There have been at least seven threat groups that have targeted Magento shops since 2015</b>, which speaks to the prominence of the platform and the success the threat actors have achieved through this exploit.</p>
<p>In early 2022, the CVE-2022-24086 vulnerability came to light, enabling attackers to exploit the Magento template engine and execute arbitrary PHP code on susceptible targets. The exploit operates through multiple steps, with common attack vectors <b>involving the abuse of either the check-out process or the wishlist functionality</b>. Since its disclosure, this vulnerability has emerged as a primary entry point for numerous <a href="/blog/security/magecart-skimmers-are-alive-and-well-constant-vigilance-is-required" target="_self">Magecart actors</a> who are targeting vulnerable Magento 2 shops.</p>
<p>Over the past few months, Akamai has been closely monitoring a focused campaign that specifically targets a relatively small number of Magento deployments. We dubbed the campaign Xurum to reference the domain name of the C2 server utilized by the attackers.</p>

    <p>The malicious PHP script that is being downloaded from the xurum server and executed on the victim machine has several infection stages.</p>

</li>

    <p>A web shell is a malicious script or piece of code that attackers upload and execute on a web server to gain unauthorized access and control over the server and its underlying files and data in a persistent manner.</p>
<p>In this operation, the attackers are leveraging the very advanced web shell named wso-ng, which as mentioned above, was created by a security researcher several years ago. As stated by the author, wso-ng is a new generation of the older and famous WSO (which stands for Web Shell by Orb). </p>

    <p>Our investigation into this campaign indicates that it dates back to at least the end of January 2023. The attackers have shown a meticulous approach, targeting specific Magento 2 instances rather than indiscriminately spraying their exploits across the internet. They demonstrate a high level of expertise in Magento and invest considerable time in understanding its internals, setting up attack infrastructure, and testing their exploits on real targets.</p>

<div data-cmp-is="text" class="cmp-text" data-cmp-data-layer="{&#34;text-bf2b045964&#34;:{&#34;@type&#34;:&#34;akamai/components/text&#34;,&#34;repo:modifyDate&#34;:&#34;2023-10-03T14:07:01Z&#34;,&#34;xdm:text&#34;:&#34;&lt;p>To prevent initial server access, security practitioners are advised to stay updated with the latest patches and complement them with the implementation of a WAF, such as &lt;a href=\&#34;/content/akamai/en/products/app-and-api-protector.html\&#34; target=\&#34;_self\&#34;>Akamai App &amp;amp; API Protector&lt;/a>.&lt;/p>\r\n&lt;p>Since the primary objective of these &lt;a href=\&#34;/content/akamai/en/blog/security/magecart-attack-disguised-as-google-tag-manager.html\&#34; target=\&#34;_self\&#34;>Magecart threat actors&lt;/a> is to infect Magento pages with web skimmers and steal customers&#39; credit card information, additional dedicated security solutions are strongly recommended.     <p>To prevent initial server access, security practitioners are advised to stay updated with the latest patches and complement them with the implementation of a WAF, such as <a href="/products/app-and-api-protector" target="_self">Akamai App &amp; API Protector</a>.</p>
<p>Since the primary objective of these <a href="/blog/security/magecart-attack-disguised-as-google-tag-manager" target="_self">Magecart threat actors</a> is to infect Magento pages with web skimmers and steal customers' credit card information, additional dedicated security solutions are strongly recommended. These solutions should provide visibility into browser script behaviors and offer defense against client-side attacks.</p>
<p>An effective approach involves deploying security measures closer to where the actual attacks on clients occur. This includes identifying attempted reads from sensitive input fields and data exfiltration. We suggest promptly collecting these events to facilitate rapid and efficient mitigation with the help of security products such as <a href="/products/client-side-protection-compliance" target="_self">Client-Side Protection &amp; Compliance</a>.</p>
