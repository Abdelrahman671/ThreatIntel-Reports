    <h1 class="text-center">Picking Apart Remcos Botnet-In-A-Box</h1>

    <div class="text-center m-1">
        <div class="post-author">
            <span>By </span>
                <a href="https://blog.talosintelligence.com/author/edmund-brumaghin/">Edmund Brumaghin</a>
        </div>

        <br/>
        <time class="post-datetime" datetime="August 22, 2018 12:00">
            Wednesday, August 22, 2018 12:00
        </time>

        <div class="m-3">
                            <a href="/category/rat/" class="category primary-category">
                                RAT
                            </a>
                                </div>
    </div>

    <section class="post-content-wrapper mt-5">
        <div class="post-content">
            <p><i>This blog post was authored by <a href="https://blog.talosintelligence.com/author/edmund-brumaghin/">Edmund Brumaghin</a> and <a href="http://blogs.cisco.com/author/holgerunterbrink">Holger Unterbrink</a> with contributions from <a href="https://twitter.com/E191145">Eric Kuhla</a> and <a href="https://www.linkedin.com/in/lilia-elena-gonz%25C3%25A1lez-80207050/">Lilia Gonzalez Medina</a>.</i></p><h2 id="overview">Overview </h2><p>Cisco Talos has recently observed multiple campaigns using the Remcos remote access tool (RAT) that is offered for sale by a company called <a href="https://breaking-security.net/shop/remcos/">Breaking Security</a>. While the company says it will only sell the software for legitimate uses as described in comments in response to the article <a href="https://krabsonsecurity.com/2018/03/02/analysing-remcos-rats-executable/">here</a> and will revoke the licenses for users not following their EULA, the sale of the RAT gives attackers everything they need to establish and run a potentially illegal botnet.</p><p>Remcos' prices per license range from €58 to €389. Breaking Security also offers customers the ability to pay for the RAT using a variety of digital currencies. This RAT can be used to fully control and monitor any Windows operating system, from Windows XP and all versions thereafter, including server editions.</p><p>In addition to Remcos, Breaking Security is also offering <a href="https://breaking-security.net/octopus/">Octopus Protector</a>, a cryptor designed to allow malicious software to bypass detection by anti-malware products by encrypting the software on the disk. A YouTube <a href="https://www.youtube.com/watch?v=BVxQxSfNJXQ">video</a> available on the Breaking Security channel demonstrates the tool's ability to facilitate the bypass of several antivirus protections. Additional products offered by this company include a <a href="https://breaking-security.net/keylogger/">keylogger</a>, which can be used to record and send the keystrokes made on an infected system, a <a href="https://breaking-security.net/poseidon/">mass mailer</a> that can be used to send large volumes of spam emails, and a <a href="https://breaking-security.net/dns-service/">DynDNS service</a> that can be leveraged for post-compromise command and control (C2) communications. These tools, when combined with Remcos provide all the tools and infrastructure needed to build and maintain a botnet.</p><p>Within Cisco's Advanced Malware Protection (AMP) telemetry, we have observed several instances of attempts to install this RAT on various endpoints. As described below, we have also seen multiple malware campaigns distributing Remcos, with many of these campaigns using different methods to avoid detection. To help people who became victims of a harmful use of Remcos, Talos is providing a <a href="https://github.com/Cisco-Talos/remcos-decoder/blob/master/remcos_decryptor.py">decoder</a> script that can extract the C2 server addresses and other information from the Remcos binary. Please see the Technical Details section below for more information.</p><h2 id="technical-details">Technical Details</h2><h3 id="remcos-distribution-in-the-wild">Remcos distribution in the wild </h3><p>Talos has observed several malware campaigns attempting to spread Remcos to various victims. Since Remcos is advertised and sold on numerous hacking-related forums, we believe it is likely that multiple unrelated actors are leveraging this malware in their attacks using a variety of different methods to infect systems. Earlier this year, RiskIQ published a <a href="https://www.riskiq.com/blog/labs/spear-phishing-turkish-defense-contractors/">report</a> regarding an attacker who was reportedly targeting defense contractors in Turkey. Since then, this threat actor has continued to operate and has been observed targeting specific types of organizations. Talos has confirmed that in addition to defense contractors, this attacker has also targeted other organizations such as:</p><ul><li>International news agencies;</li><li>Diesel equipment manufacturers and service providers operating within the maritime and energy sector; and</li><li>HVAC service providers operating within the energy sector. In all of the observed campaigns, the attack begins with specially crafted spear phishing emails written in Turkish. The emails appear as if they were sent from a Turkish government agency and purport to be related to tax reporting for the victim's organization. Below is an example of one of these email messages:</li></ul><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-QCTFsWe9vf8/W31-ZnxCI9I/AAAAAAAAAtA/T-nKOVUJluMfkP4_y3AQNZNDN7_8hivKgCLcBGAs/s640/image18.jpg" class="kg-image" alt loading="lazy"></figure><p>The attacker put effort into making the emails look as if they were official communications from Gelir İdaresi Başkanlığı (GIB), the Turkish Revenue Administration, which operates under the Ministry of Finance and is responsible for handling taxation functions in Turkey. The attacker even went as far to include official GIB graphics and the text at the bottom which translates to:</p><p><i>"Thank you for your participation in the e-mail notification system of [the] Department of Revenue Administration's e-mail service. This message has been sent to you by GIB Mail Notification System. Please do not reply to this message."</i></p><p>As is common with many spear phishing campaigns, malicious Microsoft Office documents are attached to the emails. While the majority of these documents have been Excel spreadsheets, we have also observed the same attacker leveraging Word documents. In many cases, the contents of the document have been intentionally blurred as way to entice victims to enable macros and view the content. Below is an example of a Word document associated with one of these campaigns that have been made to look as if it is a tax bill:</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-ZqdD1T7H5yc/W31-fTLdCrI/AAAAAAAAAtE/kE6axzHR-FQQ1RRl8e4o8FU31jLCiuT4gCLcBGAs/s640/image1.jpg" class="kg-image" alt loading="lazy"></figure><p>Many of the Excel spreadsheets we analyzed were mostly blank, and only included the following image and warning prompting the victim to enable macros in Turkish:</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-hKk1Tah0ht8/W31-j3KuBeI/AAAAAAAAAtI/4FRRK4gQ7ugrCldGK0NQMrNgfKl1dNFKACLcBGAs/s640/image15.jpg" class="kg-image" alt loading="lazy"></figure><p>We have also observed campaigns that appear to be targeting English-speaking victims. Below is an example of one of the malicious attachments that were made to appear as if it was an invoice on letterhead associated with Iberia, which is the flagship airline in Spain.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-BVwEZVhdlIo/W31-pFX0D8I/AAAAAAAAAtM/bTLp66s1jNoOMJrnLLxlQERl6oIzH_EzwCLcBGAs/s640/image6.jpg" class="kg-image" alt loading="lazy"></figure><p>In addition to the Iberia-themed malicious documents, we uncovered multiple malicious documents that were created to appear as if they were invoices associated with AMC Aviation, a Polish charter airline. Talos has observed the following same itinerary decoy image used across both Excel and Word documents:</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-6lSg8rY3HD0/W31-vplI_SI/AAAAAAAAAtU/1jCA1BSH-2UB1YFyQXT-FL56kzLvgxWogCLcBGAs/s640/image29.jpg" class="kg-image" alt loading="lazy"></figure><p>As described in the RiskIQ report, the macros in these files contain a small executable that is embedded into the document in the form of a series of arrays. When executed, the macros reconstruct the executable, save it to a specific location on the system and execute it. The file location specified changes across malicious documents, but includes directories commonly used by malware authors such as %APPDATA% and %TEMP%. The executable filename also changes across documents.</p><p>The extracted executable is simple and functions as the downloader for the Remcos malware. It is a very basic program and is used to retrieve Remcos from an attacker-controlled server and execute it, thus infecting the system. An example of this is below:</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-xC_sywUjdnw/W31-8x4BejI/AAAAAAAAAtc/Z_llySKDLBo9nv0I-c2BKQMBjA4pLQITgCLcBGAs/s640/image21.jpg" class="kg-image" alt loading="lazy"></figure><p>Remcos is a robust RAT that can be used to monitor keystrokes, take remote screen captures, manage files, execute commands on infected systems and more. In several cases, the distribution servers associated with these campaigns have been observed hosting several other malicious binaries in addition to Remcos.</p><h3 id="who-is-behind-remcos">Who is behind Remcos? </h3><p>As previously mentioned, a company called Breaking Security has been offering Remcos and other questionable software for purchase on their website. There are no details about the company or the people behind it listed on its website. The website does, however, list a value-added tax (VAT) number (DE308884780) which shows the company is registered in Germany. Interestingly, you can look up the name and address of companies in almost any European Union (EU) country except Germany on this <a href="http://ec.europa.eu/taxation_customs/vies/vatRequest.html">website</a>. Germany does not share this information due to privacy concerns. Because Breaking Security was registered in Germany, we were unable to identify the name and address of the individual behind this company. Nevertheless, we were able to identify several artifacts that give us an idea as to who might be behind the company.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-U3GJM_YdpaY/W31_B6ueaWI/AAAAAAAAAtg/nrc3VW5fzZU4H5Fi4kZc1qjZJ5iW6nGtQCLcBGAs/s640/image4.png" class="kg-image" alt loading="lazy"></figure><p><b>Comparison of Public and Private VAT Entries</b> <br>The Breaking Security domain is hosted behind Cloudflare currently, and Whois privacy protects the registrant information. Quite a bit of effort has been put into attempting to mask who is behind this company and the associated software. During our analysis, we were able to uncover several clues about the individual that we believe is behind this organization, either due to mistakes or very well organized false evidence on the internet.</p><p>The first thing we identified was the following email address and domain present in the Viotto Keylogger screenshot below:</p><p><i>logs@viotto[.]it</i><br><i>viotto-security[.]net</i></p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-8DEYA3BVGcU/W31_GQGsZhI/AAAAAAAAAtk/xKItbSrSrlM-2OojPf30yIURUhS03HTdwCLcBGAs/s640/image5.png" class="kg-image" alt loading="lazy"></figure><p>While the viotto-security[.]net domain server and registrant information is protected similar to what was seen with the breaking-security[.]net domain, the domain viotto[.]it listed in the "Sender's e-mail" text field is not. The Whois information associated with this domain can be seen in the screenshot below:</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-WSxHab-B8No/W31_MwYsXCI/AAAAAAAAAts/rJwjHroIUAMKeDt7uxN7BR4IeoT9S9ZogCLcBGAs/s640/image22.png" class="kg-image" alt loading="lazy"></figure><p>Normally Talos would obfuscate this data however since it is public in so many places we have elected not to. We also identified additional email, Jabber, and XMPP addresses that appear to be used by the author of Remcos by leveraging the data we collected from the website, as well as other sources:</p><p><i>viotto@null[.]pm</i><br><i>viotto24@hotmail[.]it</i><br><i>viotto@xmpp[.]ru</i></p><p>In multiple cases, the domains investigated were leveraging the Cloudflare service. This often obscures the address of servers hosting domains, as the DNS configuration typically points the name resolution to Cloudflare IPs rather than the IP of the web servers themselves. One common mistake is that while the domain itself may be protected by Cloudflare, in many cases, a subdomain exists that does not point to Cloudflare servers, allowing the server IP address to be unmasked.</p><p>This was the case with the breaking-security[.]net domain. While Cloudflare shields the domain, their mail subdomains are not protected. The A record that was configured for the mail subdomains is as follows:</p><p><i>mail[.]breaking-security[.]net. A 146.66.84[.]79</i><br><i>webmail[.]breaking-security[.]net A 146.66.84[.]79</i></p><p>The IP address 146.66.84[.]79 is hosted at <a href="https://www.siteground.com/">SiteGround Amsterdam</a>. After various testing, we are confident that this is also the IP address where the main breaking-security[.]net website is hosted.</p><p>One of the other domains we identified as being associated with Remcos was viotto-security[.]net. This domain is currently configured to redirect traffic to the main breaking-security[.]net domain. However, this was not always the case. Searching for pages associated with this domain in the Wayback Machine, a website that allows users to view past versions of a web page, yields the following result in the form of a personal biography. There are multiple clear overlaps between the interests of this individual and the developer of the various tools the company sells:</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-zvQ5oWl45zw/W31_SwbPO9I/AAAAAAAAAt0/HVoloultvdUONPT1_s0WtnEiVj35NMbXwCLcBGAs/s640/image28.png" class="kg-image" alt loading="lazy"></figure><p>We also identified several instances where Viotto was advertising, selling and supporting Remcos on various hacking forums, including HackForums since at least 2016, which makes their intentions questionable. Below is an example of one of these threads.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-Iz8QaJfQM90/W31_emNk2_I/AAAAAAAAAuA/_DAcYPdw3kY_-7TUjNbKKYhOUNoZjcGrwCLcBGAs/s640/image12.png" class="kg-image" alt loading="lazy"></figure><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-XcLw-6ngKzw/W31_mINtjZI/AAAAAAAAAuE/3FLd1deOlqUg9WQ-8wd2548X-Bvia_VvgCLcBGAs/s640/image19.png" class="kg-image" alt loading="lazy"></figure><p>While the company states that they revoke user licenses if they were to use Remcos for illegal activity, as illustrated by the thread below the purported official reseller of Remcos doesn't seem to mind another user informing it that they are using the software to control 200 bots.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-bkIv_Suq3YI/W31_rgP5X5I/AAAAAAAAAuM/mCeao0GBsf8eUPw-Zzax8dyG6Gz3V1aGQCLcBGAs/s640/image24.png" class="kg-image" alt loading="lazy"></figure><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-v9-m5rirosw/W31_wuRMQhI/AAAAAAAAAuQ/PhJx-hMzDu0zxKPwneZUnAFFCt_YVy7_gCLcBGAs/s640/image23.png" class="kg-image" alt loading="lazy"></figure><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-Wjqs36an2Zo/W31_09vcW4I/AAAAAAAAAuU/b0vPfUbiPXg7QJAnrZXWbkiKe7V3064CACLcBGAs/s640/image9.png" class="kg-image" alt loading="lazy"></figure><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-Ujmo-NMyRk8/W31_508PicI/AAAAAAAAAuc/0ExrMmCj6eoUpz0o44Dng47vlZMEYFzGwCLcBGAs/s640/image26.png" class="kg-image" alt loading="lazy"></figure><p>Viotto also appears to be active on other hacking forums, including OpenSC, where he is a moderator. Below is a thread where this user is advertising Remcos and Octopus Protector.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-Pcmr75vwO24/W31_-Thm8OI/AAAAAAAAAuk/qPpnAjVZSLkURkYARUiROuxR8HFVTPQXwCLcBGAs/s640/image13.png" class="kg-image" alt loading="lazy"></figure><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-cwPAPESEca4/W32ACd2KtvI/AAAAAAAAAuo/SXY-zVhRkskkq-0Vk9HaZ03lIuxzvGVTgCLcBGAs/s640/image16.png" class="kg-image" alt loading="lazy"></figure><h3 id="remcos-technical-details-">Remcos Technical Details: </h3><p>As described in other <a href="https://krabsonsecurity.com/2018/03/02/analysing-remcos-rats-executable/">blog posts</a>, Remcos appears to be developed in C++.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-pD844WQrsoQ/W32AH_N9I8I/AAAAAAAAAuw/o_ZP_CsJYoQ34kXjXD2i6tKjETYABP5nACLcBGAs/s640/image25.png" class="kg-image" alt loading="lazy"></figure><p>As the release notes show, it is actively maintained. The authors release new versions on almost a monthly basis:</p><p>v2.0.5 – July 14, 2018<br>v2.0.4 – April 6, 2018<br>v2.0.3 – March 29, 2018<br>v2.0.1 – Feb. 10, 2018<br>v2.0.0 – Feb. 2, 2018<br>v1.9.9 – Dec. 17, 2017</p><p>Remcos has the functionalities that are typical of a RAT. It is capable of hiding in the system and using malware techniques that make it difficult for the typical user to detect the existence of Remcos.</p><p>Several routines are looking like they were just copied and (best case) slightly modified from publicly available sources. A good example is the anti-analysis section:</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-obalSlTUuAU/W32ANWFXSBI/AAAAAAAAAu0/Mg9ki-UjZw8xHv9DidAzhHBxnKgJa4jkQCLcBGAs/s640/image27.png" class="kg-image" alt loading="lazy"></figure><p>It is checking for an outdated artifact, the 'SbieDll.dll'. In our opinion, there are not many analysts using Sandboxie these days anymore. A closer look at the other functions is also showing a high code similarity to publicly available projects. Below you can see the Remcos VMware detection code:</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-cMC58D-NALo/W32ASOxr7DI/AAAAAAAAAvA/VrQwmCFILTMu7i2itr0jgJRPldTRGGK2QCLcBGAs/s640/image30.png" class="kg-image" alt loading="lazy"></figure><p>The following is a code sample from <a href="https://www.aldeid.com/wiki/VMXh-Magic-Value">aldeid.com</a>:</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-8dXqLUMbMJw/W32AXEMj53I/AAAAAAAAAvE/_AguGq-47uoE_9-FFY-KT42kh42n6IsgwCLcBGAs/s640/image7.png" class="kg-image" alt loading="lazy"></figure><p>The blog referenced above has already described several functions of Remcos features in detail. We would like to focus on Remcos' cryptographic implementation. It uses RC4 pretty much everywhere when there is a need to decode or encode any data. Examples are registry entries, C2 server network communication or file paths shown below:</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-JDR30NvxUQQ/W32Ae6jaC-I/AAAAAAAAAvQ/d8NC5OPFM4wk8O6xzXkW0EAyKdPeFPKEgCLcBGAs/s640/image2.png" class="kg-image" alt loading="lazy"></figure><p>The exepath registry data is base64-encoded, RC4-encrypted data. Decoded, it is the path of the executable:<br> <i>C:\TEMP\1cc8f8b1487893b2b0ff118faa2333e1826ae1495b626e206ef108460d4f0fe7.exe</i></p><p>The RC4 implementation is the standard RC4 implementation that can be found in many code examples on the internet. They are first building the Key Scheduling Algorithms (KSA) S_array at 00402F01.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-cgkFW-Yi4GU/W32AjHPH__I/AAAAAAAAAvU/roG_54jGBrEy0CfZwR1orxc46xtUefcXwCLcBGAs/s400/image31.png" class="kg-image" alt loading="lazy"></figure><p>This can be converted into the typical RC4 pseudo code:</p><pre><code>for i from 0 to 255
        S[i] := i
endfor
j := 0
for i from 0 to 255
        j := (j + S[i] + key[i mod keylength]) mod 256
        swap values of S[i] and S[j]
endfor
</code></pre><p>Which is followed by the RC4 Pseudo-random generation algorithm (PRGA) at 00402F5B.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-TDb5TglxOME/W32Aor3eMSI/AAAAAAAAAvc/cYMHmTx4X4M6UjP42JFiO-tqj-bVozR-wCLcBGAs/s400/image20.png" class="kg-image" alt loading="lazy"></figure><p>Which looks in pseudo code like this:</p><pre><code>i := 0
j := 0
while GeneratingOutput:
        i := (i + 1) mod 256
        j := (j + S[i]) mod 256
        swap values of S[i] and S[j]
        K := S[(S[i] + S[j]) mod 256]
        output K
endwhile</code></pre><p><br>As the screenshots above illustrate, Remcos is using RC4 to encrypt and decrypt its data, and it is using the PE resource section to store the initial encryption key in the 'SETTINGS' resource. This key can have a variable length — we have seen short keys from 40 bytes to keys with more than 250 bytes.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-5MfRGa0ongw/W32AuuSMjuI/AAAAAAAAAvk/oYaxeLDRntgkm9oBnOorMj9eocglP1_UwCLcBGAs/s640/image32.png" class="kg-image" alt loading="lazy"></figure><p>They are storing the data in the following format:</p><p>[Length of key]<br>[Encryption Key]<br>[Encrypted configuration data]</p><p>This encrypted configuration data section contains the command and control servers, RAT commands to execute and other data. Decoded, it looks like this:</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-APGGH-i_aig/W32Az2b0aBI/AAAAAAAAAvs/n2GSzTXXKo0IRH9sOR7IaveFXJfmcD5zACLcBGAs/s640/image8.png" class="kg-image" alt loading="lazy"></figure><p>The decoded data contains the C2 server, e.g. ejiroprecious[.]ddns[.]net, and the corresponding port number, followed by a password. This password is used to generate a separate S_array for the RC4 encrypted C2 communication. The picture shows the relevant part of the RC4 Key Scheduling Algorithms (KSA) from above.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-E1ZYVjfvk4Y/W32A4l2OYqI/AAAAAAAAAvw/77MeaChBULQALAk-ODHyVzeYY9fbR99PwCLcBGAs/s640/image17.png" class="kg-image" alt loading="lazy"></figure><p>Even if a stronger password is used than in the example above, using such a weak encryption algorithm means that everyone who gets his or her hands on the binary file can extract the password and decrypt the C2 traffic or inject their own commands into the C2 channel to control the RAT. The good news is that companies who became a victim of Remcos have a good chance to analyse the threat if they have stored the network traffic and the Remcos binary file.</p><p>To make the life of forensic investigators easier, we are providing a small <a href="https://github.com/Cisco-Talos/remcos-decoder/blob/master/remcos_decryptor.py">decoder Python script</a> that can decode the config data from the resource section:</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-wdX7bNsh_AY/W32A_kfYKfI/AAAAAAAAAv8/fF5mKMklbSwJwNo0kApTfDR5AapMReDKgCLcBGAs/s640/image14.png" class="kg-image" alt loading="lazy"></figure><p>As mentioned above, Remcos is using the same encryption routine for all kinds of other functions, too. For this reason, the decoder program also offers an option to hand over encrypted bytes manually. This can be used to decode, for example, the exepath registry key.</p><p>We have used this tool to extract all the IOCs below. It is tested with the latest 2.0.4 and 2.0.5 versions of Remcos, but likely also works with other versions.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-3ASJftgNMjY/W32BE0CeW3I/AAAAAAAAAwA/VnGuhtNPNs8iP9TbgX-SlD_SKC6kBZQkACLcBGAs/s640/image10.png" class="kg-image" alt loading="lazy"></figure><p>The user can also copy bytes from a network sniffer to a binary file, and hand it over to decrypt the bytes from the C2 communication to see which commands the C2 server has sent to the victim. Keep in mind to use the extracted password, e.g. "pass."</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-NQqKcySIX9I/W32BKLFQqjI/AAAAAAAAAwI/iOuB867_WvE2jWVbf7auK1h8VPWPIzE_QCLcBGAs/s640/image3.png" class="kg-image" alt loading="lazy"></figure><h2 id="conclusion">Conclusion </h2><p>While the organization that sells Remcos claims that the application is only for legal use, our research indicates it is still being used extensively by malicious attackers, as well. In some cases, attackers are strategically targeting victims to attempt to gain access to organizations that operate as part of the supply chain for various critical infrastructure sectors. Organizations should ensure that they are implementing security controls to combat Remcos, as well as other threats that are being used in the wild. Remcos is a robust tool that is being actively developed to include new functionality increasing what the attackers can gain access to. To combat this, organizations should continue to be aware of this threat, as well as others like this that may be circulated on the internet.</p><h2 id="coverage">Coverage </h2><p>Additional ways our
