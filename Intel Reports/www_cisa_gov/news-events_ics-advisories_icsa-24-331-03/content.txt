                <h1 class="c-page-title__title">
<span>Schneider Electric EcoStruxure Control Expert, EcoStruxure Process Expert, and Modicon M340, M580 and M580 Safety PLCs</span>
</h1>
                                                          <div class="c-page-title__fields">  




<div  class="c-field c-field--name-field-release-date c-field--type-datetime c-field--label-above">
  <div  class="c-field__label">Release Date</div><div class="c-field__content"><time datetime="2024-11-26T12:00:00Z">November 26, 2024</time></div></div>

  




<div  class="c-field c-field--name-field-alert-code c-field--type-string c-field--label-above">
  <div  class="c-field__label">Alert Code</div><div class="c-field__content">ICSA-24-331-03</div></div>

</div>
                        
                  <div class="c-page-title__topic">
            <div class="c-topic__label"> Related topics: </div>
            <div class="c-top__topics">
                                                                              <a href="/topics/industrial-control-systems/industrial-control-system-vulnerabilities">Industrial Control System Vulnerabilities</a>,                                                                                 <a href="/topics/industrial-control-systems">Industrial Control Systems</a>                                          </div>
          </div>
        
      </div>
          </div>
    <div class="c-page-title__decoration"></div>
  </div>
</div>
    </div>
    <div class="l-full__main">
                      

<div  class="l-page-section l-page-section--rich-text">
      <div class="l-constrain">
  
  
  <div class="l-page-section__content">
          <p><a href="https://github.com/cisagov/CSAF" target="_blank"><strong>View CSAF</strong></a></p>
<h2 id="1-executive-summary">1. EXECUTIVE SUMMARY</h2>
<ul>
<li><strong>CVSS v3 8.1</strong></li>
<li><strong>ATTENTION</strong>: Exploitable remotely</li>
<li><strong>Vendor</strong>: Schneider Electric</li>
<li><strong>Equipment</strong>: EcoStruxure Control Expert, EcoStruxure Process Expert and Modicon M340, M580 and M580 Safety PLCs</li>
<li><strong>Vulnerabilities</strong>: Improper Enforcement of Message Integrity During Transmission in a Communication Channel, Use of Hard-coded Credentials, Insufficiently Protected Credentials</li>
</ul>
<h2 id="2-risk-evaluation">2. RISK EVALUATION</h2>
<p>Successful exploitation of these vulnerabilities could allow a denial of service, a loss of confidentiality, and threaten the integrity of controllers.</p>
<h2 id="3-technical-details">3. TECHNICAL DETAILS</h2>
<h3 id="31-affected-products">3.1 AFFECTED PRODUCTS</h3>
<p>The following Schneider Electric products are affected:</p>
<ul>
<li>Modicon M340 CPU (part numbers BMXP34*): Versions prior to sv3.60 (CVE-2023-6408)</li>
<li>Modicon M580 CPU (part numbers BMEP* and BMEH* excluding M580 CPU Safety): Versions prior to SV4.20 (CVE-2023-6408)</li>
<li>Modicon M580 CPU Safety: Versions prior to SV4.21 (CVE-2023-6408)</li>
<li>EcoStruxure Control Expert: Versions prior to v16.0</li>
<li>EcoStruxure Process Expert: Versions prior to v2023</li>
<li>Modicon MC80 (part numbers BMKC80): All versions (CVE-2023-6408)</li>
<li>Modicon Momentum Unity M1E Processor (171CBU*): All versions (CVE-2023-6408)</li>
</ul>
<h3 id="32-vulnerability-overview">3.2 Vulnerability Overview</h3>
<h4 id="321-improper-enforcement-of-message-integrity-during-transmission-in-a-communication-channel-cwe-924"><strong>3.2.1 </strong><a href="https://cwe.mitre.org/data/definitions/924.html" target="_blank"><strong>IMPROPER ENFORCEMENT OF MESSAGE INTEGRITY DURING TRANSMISSION IN A COMMUNICATION CHANNEL CWE-924</strong></a></h4>
<p>An improper enforcement of message integrity during transmission in a communication channel vulnerability exists that could cause a denial of service, a loss of confidentiality, and threaten the integrity of controllers through a man-in-the-middle attack.</p>
<p><a href="https://www.cve.org/CVERecord?id=CVE-2023-6408" target="_blank">CVE-2023-6408</a> has been assigned to this vulnerability. A CVSS v3 base score of 8.1 has been calculated; the CVSS vector string is (<a href="https://www.first.org/cvss/calculator/3.1#CVSS:3.1/AV:N/AC:H/PR:N/UI:N/S:U/C:H/I:H/A:H" target="_blank">CVSS:3.1/AV:N/AC:H/PR:N/UI:N/S:U/C:H/I:H/A:H</a>).</p>
<h4 id="322-use-of-hard-coded-credentials-cwe-798"><strong>3.2.2 </strong><a href="https://cwe.mitre.org/data/definitions/798.html" target="_blank"><strong>USE OF HARD-CODED CREDENTIALS CWE-798</strong></a></h4>
<p>A use of hard-coded credentials vulnerability exists that could cause unauthorized access to a project file protected with application password when opening the file with EcoStruxure Control Expert.</p>
<p><a href="https://www.cve.org/CVERecord?id=CVE-2023-6409" target="_blank">CVE-2023-6409</a> has been assigned to this vulnerability. A CVSS v3 base score of 7.7 has been calculated; the CVSS vector string is (<a href="https://www.first.org/cvss/calculator/3.1#CVSS:3.1/AV:L/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:N" target="_blank">CVSS:3.1/AV:L/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:N</a>).</p>
<h4 id="323-insufficiently-protected-credentials-cwe-522"><strong>3.2.3 </strong><a href="https://cwe.mitre.org/data/definitions/522.html" target="_blank"><strong>INSUFFICIENTLY PROTECTED CREDENTIALS CWE-522</strong></a></h4>
<p>An insufficiently protected credentials vulnerability exists that could cause unauthorized access to the project file in EcoStruxure Control Expert when a local user tampers with the memory of the engineering workstation.</p>
<p><a href="https://www.cve.org/CVERecord?id=CVE-2023-27975" target="_blank">CVE-2023-27975</a> has been assigned to this vulnerability. A CVSS v3 base score of 7.1 has been calculated; the CVSS vector string is (<a href="https://www.first.org/cvss/calculator/3.1#CVSS:3.1/AV:L/AC:L/PR:L/UI:N/S:U/C:H/I:H/A:N" target="_blank">CVSS:3.1/AV:L/AC:L/PR:L/UI:N/S:U/C:H/I:H/A:N</a>).</p>
<h3 id="33-background">3.3 BACKGROUND</h3>
<ul>
<li><strong>CRITICAL INFRASTRUCTURE SECTORS:</strong> Critical Manufacturing</li>
<li><strong>COUNTRIES/AREAS DEPLOYED:</strong> Worldwide</li>
<li><strong>COMPANY HEADQUARTERS LOCATION:</strong> France</li>
</ul>
<h3 id="34-researcher">3.4 RESEARCHER</h3>
<p>Gao Jian, Jianshuang Ding, and Kaikai Yang reported these vulnerabilities to Schneider Electric.</p>
<h2 id="4-mitigations">4. MITIGATIONS</h2>
<p>Schneider Electric has identified the following remediations and mitigations users can apply to reduce risk:</p>
<p>Modicon M340 CPU (part numbers BMXP34*):</p>
<ul>
<li><a href="https://www.se.com/ww/en/product-range/1468-modicon-m340" target="_blank">Firmware Version SV3.60</a> includes a fix for this vulnerability and is available for download.</li>
<li>Set up an application password in the project properties.</li>
<li>Set up network segmentation and implement a firewall to block all unauthorized access to Port 502/TCP.</li>
<li>Configure the access control list following the recommendations of the user manuals: <a href="https://www.se.com/ww/en/download/document/31007131K01000/" target="_blank">"Modicon M340 for Ethernet Communications Modules and Processors User Manual"</a> in chapter "Messaging Configuration Parameters":</li>
<li>Set up a secure communication according to the following guideline <a href="https://www.se.com/ww/en/download/document/EIO0000001999/" target="_blank">"Modicon Controllers Platform Cyber Security Reference Manual,"</a> in chapter "Set up secured communications":</li>
<li>Consider use of external firewall devices such as EAGLE40-07 from Belden to establish VPN connections for M340 &amp; M580 architectures. For more details refer to the chapter <a href="https://www.se.com/ww/en/download/document/EIO0000001999/" target="_blank">"How to protect M580 and M340 architectures with EAGLE40 using VPN"</a></li>
<li>Ensure the M340 CPU is running with the memory protection activated by configuring the input bit to a physical input, for more details refer to the following guideline <a href="https://www.se.com/ww/en/download/document/EIO0000001999/" target="_blank">"Modicon Controllers Platform Cyber Security Reference Manual"</a>, <a href="https://www.se.com/ww/en/download/document/31007131K01000/" target="_blank">"CPU Memory Protection section"</a>.</li>
</ul>
<p>Modicon M580 CPU (part numbers BMEP* and BMEH* excluding M580 CPU Safety):</p>
<ul>
<li><a href="https://www.se.com/ww/en/product-range/62098-%20modicon-m580-epac/%20-%20software-and-firmware" target="_blank">Firmware Versions SV4.20</a> includes a fix for this vulnerability and is available for download.</li>
<li>Set up an application password in the project properties</li>
<li>Setup network segmentation and implement a firewall to block all unauthorized access to Port 502/TCP.</li>
<li>Configure the access control list following the recommendations of the user manuals: <a href="https://www.se.com/ww/en/product-range/62098-%20modicon-m580-epac/%20-%20software-and-firmware" target="_blank">"Modicon M580, Hardware, Reference Manual"</a>.</li>
<li>Set up a secure communication according to the following guideline <a href="https://www.se.com/ww/en/download/document/EIO0000001999/" target="_blank">"Modicon Controllers Platform Cyber Security Reference Manual,"</a> in chapter "Set up secured communications":</li>
<li>Use a BMENOC module and follow the instructions to configure IPSEC feature as described in the guideline <a href="https://www.se.com/ww/en/download/document/HRB62665/" target="_blank">"Modicon M580 - BMENOC03.1 Ethernet Communications Schneider Electric Security Notification Module, Installation and Configuration Guide"</a> in the chapter "Configuring IPSEC communications":</li>
<li>Use a BMENUA0100 module and follow the instructions to configure IPSEC feature as described in the chapter <a href="https://www.se.com/ww/en/download/document/PHA83350" target="_blank">"Configuring the BMENUA0100 Cybersecurity Settings"</a>.</li>
<li>Consider use of external firewall devices such as EAGLE40-07 from Belden to establish VPN connections for M340 &amp; M580 architectures. For more details refer to the chapter <a href="https://www.se.com/ww/en/download/document/EIO0000001999/" target="_blank">"How to protect M580 and M340 architectures with EAGLE40 using VPN"</a>.</li>
<li>Ensure the M580 CPU is running with the memory protection activated by configuring the input bit to a physical input, for more details refer to the following guideline <a href="https://www.schneiderelectric.com/en/download/document/EIO0000001999/" target="_blank">"Modicon Controllers Platform Cyber Security Reference Manual", "CPU Memory Protection section"</a>.</li>
<li>The CPU memory protection cannot be configured with M580 Hot Standby CPUs. In such cases, use IPsec <a href="https://www.se.com/ww/en/download/document/EIO0000001578/" target="_blank">encrypted communication</a> .</li>
</ul>
<p>Modicon M580 CPU Safety (part numbers BMEP58<em>S and BMEH58</em>S):</p>
<ul>
<li><a href="https://www.se.com/ww/en/product-range/62098-modicon-m580-pac-controller/#software-andfirmware" target="_blank">Firmware SV4.21</a> includes a fix for CVE-2023-6408 and is available for download. Important: users needs to use version of <a href="https://www.se.com/ww/en/product-range/548-ecostruxure-control-expert-unity-pro/#software-andfirmware" target="_blank">EcoStruxure Control Expert v16.0 HF001 minimum</a> to connect with the latest version of M580 CPU Safety.</li>
<li>If users choose not to apply the remediation, they are encouraged to immediately apply the following mitigations to reduce the risk of exploit:</li>
<li>Set up an application password in the project properties.</li>
<li>Setup network segmentation and implement a firewall to block all unauthorized access to Port 502/TCP.</li>
<li>Configure the Access Control List following the recommendations of <a href="https://www.se.com/ww/en/download/document/EIO0000001578/" target="_blank">"Modicon M580, Hardware, Reference Manual"</a></li>
<li>Set up a secure communication according to the following guideline <a href="https://www.se.com/ww/en/download/document/EIO0000001999/" target="_blank">"Modicon Controllers Platform Cyber Security Reference Manual,"</a> in chapter "Set up secured communications".</li>
<li>Use a BMENOC module and follow the instructions to configure IPSEC feature as described in the guideline "Modicon M580 - BMENOC03.1 Ethernet Communications Schneider Electric Security Notification Module, Installation and Configuration Guide" in the chapter "Configuring IPSEC communications": <a href="https://www.se.com/ww/en/download/document/HRB62665/" target="_blank">https://www.se.com/ww/en/download/document/HRB62665/</a></li>
<li>Use a BMENUA0100 module and follow the instructions to configure IPSEC feature as described in the chapter <a href="https://www.se.com/ww/en/download/document/PHA83350" target="_blank">"Configuring the BMENUA0100 Cybersecurity Settings"</a></li>
<li>Consider use of external firewall devices such as EAGLE40-07 from Belden to establish VPN connections for M340 &amp; M580 architectures. For more details refer to the chapter <a href="https://www.se.com/ww/en/download/document/EIO0000001999/" target="_blank">"How to protect M580 and M340 architectures with EAGLE40 using VPN"</a></li>
<li>Ensure the M580 CPU is running with the memory protection activated by configuring the input bit to a physical input, for more details refer to the following guideline <a href="https://www.schneiderelectric.com/en/download/document/EIO0000001999/" target="_blank">"Modicon Controllers Platform Cyber Security Reference Manual", "CPU Memory Protection section"</a></li>
<li>NOTE: The CPU memory protection cannot be configured with M580 Hot Standby CPUs. In such cases, use IPsec encrypted communication.</li>
<li>To further reduce the attack surface on Modicon M580 CPU Safety: Ensure the CPU is running in Safety mode and maintenance input is configured to maintain this Safety mode during operation – refer to the document <a href="https://www.se.com/ww/en/download/document/EIO0000001578/" target="_blank">Modicon M580 - Safety System Planning Guide</a> - in the chapter "Operating Mode Transitions".</li>
<li>Schneider Electric is establishing a remediation plan for all future versions of EcoStruxure Process Expert that will include a fix for CVE-2023-6409 and CVE-2023-27975. They will update SEVD-2024-317-04 when the remediation is available. Until then, users should immediately apply the above mitigations to reduce the risk of exploit.</li>
</ul>
<p>Modicon MC80 (part numbers BMKC80):</p>
<ul>
<li>Set up an application password in the project properties.</li>
<li>Set up network segmentation and implement a firewall to block all unauthorized access to Port 502/TCP.</li>
<li>Configure the access control list following the recommendations of <a href="https://usdhs-my.sharepoint.com/personal/grayson_gaylor_associates_cisa_dhs_gov1/_layouts/15/:https://www.se.com/ww/en/download/document/EIO0000002071/Setup" target="_blank">"Modicon MC80 Programmable Logic Controller (PLC) manual"</a> in the chapter "Access Control List (ACL)" a secure communication according to <a href="https://www.se.com/ww/en/download/document/EIO0000001999/" target="_blank">"Modicon Controller Systems Cybersecurity, User Guide"</a> in chapter "Set Up Encrypted Communication".</li>
<li>(CVE-2023-6408) Schneider Electric Modicon Momentum Unity M1E Processor (171CBU*) All versions: Setup an application password in the project properties
<ul>
<li>Setup network segmentation and implement a firewall to block all unauthorized access to Port 502/TCP</li>
<li>Setup a secure communication according to the following guideline "Modicon Controller Systems Cybersecurity, User Guide" in chapter "Set Up <a href="https://www.se.com/ww/en/download/document/EIO0000001999/" target="_blank">Encrypted Communication":</a></li>
</ul>
</li>
</ul>
<p>EcoStruxure Control Expert:</p>
<ul>
<li><a href="https://www.se.com/in/en/product-range/548-ecostruxure-control-expert-unity-pro/" target="_blank">Version 16.0</a> includes a fix for these vulnerabilities and is available for download. Reboot the computer after installation is completed.</li>
<li>Enable encryption on application project and store application files in secure location with restricted access only for legitimate users.</li>
<li>Schneider Electric recommends using McAfee Application and Change Control software for application control. Refer to the <a href="https://www.se.com/ww/en/download/document/EIO0000004778/" target="_blank">Cybersecurity Application Note</a>.</li>
<li>Follow workstation, network and site-hardening guidelines in the <a href="https://www.se.com/ww/en/download/document/EIO0000004778/" target="_blank">Recommended Cybersecurity Best Practices</a>.</li>
</ul>
<p>EcoStruxure Process Expert:</p>
<ul>
<li><a href="https://www.se.com/ww/en/product-range/548-ecostruxure-control-expert-unity-pro/" target="_blank">Version 15.3 HF008</a> includes the fix for these vulnerabilities and is available for download.</li>
<li>EcoStruxure Process Expert manages application files within its database in secure way. Do not export &amp; store them outside the application.</li>
<li>Schneider Electric recommends using McAfee Application and Change Control software for application control. Refer to the <a href="https://www.se.com/ww/en/download/document/EIO0000004778/" target="_blank">Cybersecurity Application Note</a>.</li>
<li>Follow workstation, network and site-hardening guidelines in the <a href="https://www.se.com/ww/en/download/document/EIO0000004778/" target="_blank">Recommended Cybersecurity Best Practices</a>.</li>
</ul>
<p>For more information refer to the <a href="https://www.se.com/us/en/download/document/7EN52-0390/" target="_blank">Schneider Electric Recommended Cybersecurity Best Practices</a> and the associated Schneider Electric Security Notification SEVD-2024-044-01 in <a href="https://download.schneider-electric.com/files?p_Doc_Ref=SEVD-2024-044-01&amp;p_enDocType=Security+and+Safety+Notice&amp;p_File_Name=SEVD-2024-044-01.pdf" target="_blank">PDF</a> and <a href="https://download.schneider-electric.com/files?p_Doc_Ref=SEVD-2024-044-01&amp;p_enDocType=Security+and+Safety+Notice&amp;p_File_Name=SEVD-2024-044-01.json" target="_blank">CSAF.</a></p>
<p>CISA recommends users take defensive measures to minimize the risk of exploitation of these vulnerabilities. CISA reminds organizations to perform proper impact analysis and risk assessment prior to deploying defensive measures.</p>
<p>CISA also provides a section for <a href="https://www.cisa.gov/resources-tools/resources/ics-recommended-practices" target="_blank">control systems security recommended practices</a> on the ICS webpage on <a href="https://www.cisa.gov/topics/industrial-control-systems" target="_blank">cisa.gov</a>. Several CISA products detailing cyber defense best practices are available for reading and download, including <a href="https://us-cert.cisa.gov/sites/default/files/recommended_practices/NCCIC_ICS-CERT_Defense_in_Depth_2016_S508C.pdf" target="_blank">Improving Industrial Control Systems Cybersecurity with Defense-in-Depth Strategies</a>.</p>
<p>CISA encourages organizations to implement recommended cybersecurity strategies for <a href="https://www.cisa.gov/sites/default/files/publications/Cybersecurity_Best_Practices_for_Industrial_Control_Systems.pdf" target="_blank">proactive defense of ICS assets</a>.</p>
<p>Additional mitigation guidance and recommended practices are publicly available on the ICS webpage at <a href="https://www.cisa.gov/topics/industrial-control-systems" target="_blank">cisa.gov</a> in the technical information paper, <a href="https://www.cisa.gov/uscert/ics/tips/ICS-TIP-12-146-01B" target="_blank">ICS-TIP-12-146-01B--Targeted Cyber Intrusion Detection and Mitigation Strategies</a>.</p>
<p>Organizations observing suspected malicious activity should follow established internal procedures and report findings to CISA for tracking and correlation against other incidents.</p>
<p>No known public exploitation specifically targeting these vulnerabilities has been reported to CISA at this time. These vulnerabilities have a high attack complexity.</p>
