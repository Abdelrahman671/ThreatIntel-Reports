<h3>Summary</h3><br />
<p>The REvil ransomware-as-a-service (RaaS) operation continues to impact businesses worldwide. The threat actors responsible for developing and maintaining the malware have released an updated ransomware, namely version 2.2. In this short blog post, we will cover the significant changes from the previous version, which we covered in detail in an earlier blog post (see: https://blog.intel471.com/2020/03/31/revil-ransomware-as-a-service-an-analysis-of-a-ransomware-affiliate-operation/).</p><br />
<h3>Persistence mechanism</h3><br />
<p>REvil ransomware persists on a machine if the <strong>arn</strong> configuration field is set to <strong>true</strong>. It writes its path to the registry key <strong>SOFTWARE\Microsoft\Windows\CurrentVersion\Run</strong>. An example of the value name of the registry key entry is <strong>mjOObKp0yy</strong>.</p><figure><img src="https://d39ec1uo9ktrut.cloudfront.net/revil.png" alt="" /></figure><p>In version 2.1, first collected by our systems March 15, 2020, this persistence mechanism was removed. It seems this little experiment didn’t go as planned, because the new version 2.2 brings the same persistence mechanism back!</p><br />
<h3>Restart Manager to terminate processes</h3><br />
<p>One of the more interesting new features of REvil version 2.2 is the use of the Windows Restart Manager to terminate processes and services that can lock files targeted for encryption. If a process has an open file handle for a specific file, then writes to that file by another process (in this case, a ransomware) it will be prevented by the Windows operating system (OS). To circumvent this, the REvil developers have implemented a technique using the Windows Restart Manager also used by other ransomware such as SamSam and LockerGoga (see: <a href="https://www.crowdstrike.com/blog/an-in-depth-analysis-of-samsam-ransomware-and-boss-spider/">https://www.crowdstrike.com/blog/an-in-depth-analysis-of-samsam-ransomware-and-boss-spider/</a>).</p><br />
<p>REvil ransomware opens files for encryption with no sharing (dwShareMode equals 0). As a result, the Restart Manager is invoked whenever a sharing violation occurs when opening an already opened file.</p><figure><img src="https://d39ec1uo9ktrut.cloudfront.net/pasted-image-0-6.png" alt="" /></figure><p>The function prototype for <strong>rvl_restart_manager</strong> is:</p><br />
<pre>VOID rvl_restart_manager(LPCWSTR Filename, BOOL DoEndSession)</pre><br />
<p>The following explains how REvil employs this technique:</p><br />
<ul><li>Call <strong>OpenSCManagerW</strong> to open the “ServicesActive” database.</li><li>Start a new Restart Manager session by calling <strong>RmStartSession</strong> and save the returned handle in a global variable for future calls.</li><li>Invoke <strong>RmRegisterResources</strong> with the target file name to register it to the Restart Manager session.</li><li>Retrieve the list of all applications currently using the file by calling <strong>RmGetList</strong>. This application programming interface (API) returns an array of <strong>RM_PROCESS_INFO</strong> structures.</li><li>If a normal process is using the file, it is terminated by a call to <strong>TerminateProcess</strong>.</li><li>If a service is encountered, <strong>ControlService</strong> is invoked with the <strong>SERVICE_CONTROL_STOP</strong> control code to stop the service followed by a call to <strong>DeleteService</strong>.</li><li>If a critical process is encountered, its critical status is removed by calling <strong>ZwSetInformationProcess</strong> with the information class <strong>ProcessBreakOnTermination</strong> before terminating it. This may lead to undefined behavior on the victim system.</li></ul><h4>New ‘-silent’ flag</h4><br />
<p>A new command-line option -silent was added that skips termination of blacklisted processes, services and shadow copy deletion. However, this flag does not impact the new Restart Manager functionality.</p><figure><img src="https://d39ec1uo9ktrut.cloudfront.net/pasted-image-0-7.png" alt="" /></figure><h3>Indicators of compromise</h3><figure><img src="https://d39ec1uo9ktrut.cloudfront.net/Screen-Shot-2020-09-03-at-11.48.30-am.png" alt="" /></figure></div>
