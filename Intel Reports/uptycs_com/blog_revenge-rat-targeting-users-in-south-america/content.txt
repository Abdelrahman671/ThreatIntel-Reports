        <div class="content-wrapper blog--post-content-section">
          <article class="blog-post">
            
            <div class="blog-post__tags">
              Tags
              
              <a class="blog-post__tag-link" href="https://www.uptycs.com/blog/threat-research-report-team/tag/threats" rel="tag">Threats</a>
              
            </div>
             
            <div class="blog-post__body">
              <span id="hs_cos_wrapper_post_body" class="hs_cos_wrapper hs_cos_wrapper_meta_field hs_cos_wrapper_type_rich_text" style="" data-hs-cos-general-type="meta_field" data-hs-cos-type="rich_text"><p>The Uptycs <a href="/products/use-cases/threat-hunting" rel="noopener" target="_blank">threat research</a> team recently came across multiple document samples that download Revenge RAT. The campaign currently seems to be active in Brazil. All of the malware samples we received have the same properties. One of the samples we received has the name “Rooming List Reservas para 3 Familias.docx” (SHA-256: 91611ac2268d9bf7b7cb2e71976c630f6b4bfdbb68774420bf01fd1493ed28c7). The document has only a few detections in VirusTotal.</p>
<!--more-->
<p style="text-align: center;"><img src="https://www.uptycs.com/hs-fs/hubfs/figure1.png?width=624&amp;name=figure1.png" alt="VirusTotal detections for the document. (Image via VirusTotal.)" width="624" style="width: 624px;" srcset="https://www.uptycs.com/hs-fs/hubfs/figure1.png?width=312&amp;name=figure1.png 312w, https://www.uptycs.com/hs-fs/hubfs/figure1.png?width=624&amp;name=figure1.png 624w, https://www.uptycs.com/hs-fs/hubfs/figure1.png?width=936&amp;name=figure1.png 936w, https://www.uptycs.com/hs-fs/hubfs/figure1.png?width=1248&amp;name=figure1.png 1248w, https://www.uptycs.com/hs-fs/hubfs/figure1.png?width=1560&amp;name=figure1.png 1560w, https://www.uptycs.com/hs-fs/hubfs/figure1.png?width=1872&amp;name=figure1.png 1872w" sizes="(max-width: 624px) 100vw, 624px"></p>
<p style="text-align: center;"><em>Figure 1: VirusTotal detections for the document. (Image via VirusTotal.)</em></p>
<p>&nbsp;</p>
<p>Upon opening the document, a series of events happen that lead to the download of Revenge RAT malware hosted on a Brazilian website (hxxp://azulviagens[.]online). Azul Viagens is a legitimate hotel chain in Brazil and the official website of the hotel can be found <a href="https://www.azulviagens.com.br/" rel="noopener" target="_blank">here</a>.</p>
<p>&nbsp;</p>
<p>Attackers registered the fake domain name and used a room reservation document file to infect the end user. The attack is multi-stage with the components used in the attack spread across multiple files on the attacker’s server. The WHOIS records for hxxp://azulviagens[.]online seems to have been registered on December 10, 2020 with the email ID <a href="mailto:mmpereiramm30@gmail.com.">mmpereiramm30@gmail.com</a>.</p>
<p>&nbsp;</p>
<h2>The Attack Flow</h2>
<p>The components of the attack span multiple stages. Figure 2 (below) shows the steps involved in the attack.</p>
<p><img src="https://www.uptycs.com/hs-fs/hubfs/figure2-crop.png?width=800&amp;name=figure2-crop.png" alt="The Revenge RAT attack flow." width="800" style="width: 800px; margin-left: auto; margin-right: auto; display: block;" srcset="https://www.uptycs.com/hs-fs/hubfs/figure2-crop.png?width=400&amp;name=figure2-crop.png 400w, https://www.uptycs.com/hs-fs/hubfs/figure2-crop.png?width=800&amp;name=figure2-crop.png 800w, https://www.uptycs.com/hs-fs/hubfs/figure2-crop.png?width=1200&amp;name=figure2-crop.png 1200w, https://www.uptycs.com/hs-fs/hubfs/figure2-crop.png?width=1600&amp;name=figure2-crop.png 1600w, https://www.uptycs.com/hs-fs/hubfs/figure2-crop.png?width=2000&amp;name=figure2-crop.png 2000w, https://www.uptycs.com/hs-fs/hubfs/figure2-crop.png?width=2400&amp;name=figure2-crop.png 2400w" sizes="(max-width: 800px) 100vw, 800px"></p>
<p style="text-align: center;"><em>Figure 2: The attack flow.</em></p>
<p>&nbsp;</p>
<ul>
<li><strong>Step 1</strong>: The DOCX file (“Rooming List Reservas para 3 Familias.docx”) used in the attack vector downloads the “1.docx” (template) from the CnC server</li>
<li><strong>Steps 2 and 3</strong>: The embedded “Microsoft_Excel_Macro-Enabled_Worsksheet1.xlsm” file in "1.docx" (template) downloads the PowerShell code “A.txt” from the CnC server and executes it in memory.</li>
<li><strong>Step 4</strong>: The PowerShell code in “A.txt” downloads “index.mp3” from the CnC server and saves it as “index.vbs.”</li>
<li><strong>Step 5</strong>: Upon execution, “index.vbs” creates “opera.vbs,” which contains code to execute “opera.ps1” created in the next step.</li>
<li><strong>Step 6</strong>: “index.vbs” downloads “1.txt” and saves it as “opera.ps1,” which has obfuscated Revenge RAT in it.</li>
<li><strong>Step 7</strong>: “opera.vbs” executes “opera.ps1.”</li>
</ul>
<p>A detailed analysis of files used during various stages of the attack is provided below.</p>
<p>&nbsp;</p>
<h2>The Initial Document</h2>
<p>The initial document, “Rooming List Reservas para 3 Familias.docx,” used as the attack vector is a DOCX file. The document uses a technique known as Dynamic Office Template Injection to bypass security products. This allows the attacker to store the malicious file on a remote server. This technique can evade anti-malware solutions that rely on static detection.</p>
<p>&nbsp;</p>
<p>The document has the structure shown in Figure 3 (below). The structure contains a file named “footer.xml.rels.” The “target” fields in the file point to the templates hosted on the CnC server. There are several URLs in the “target” fields that point to files “1.docx” all the way to “9.docx” hosted on the CnC server. Each of the files has the same content (the same SHA-256: 338b2d8d76f4028bfbd177127371b2509971606553d606c534316dc40cfa8fb9).</p>
<p>&nbsp;</p>
<p style="text-align: center;"><a href="https://f.hubspotusercontent00.net/hubfs/2617658/figure3-crop.png" rel="nofollow noopener" target="_blank"><img src="https://www.uptycs.com/hs-fs/hubfs/figure3-crop.png?width=800&amp;name=figure3-crop.png" alt=" Structure of the DOCX and footer.xml.res pointing to the malicious template. " width="800" style="width: 800px; margin-left: auto; margin-right: auto; display: block;" srcset="https://www.uptycs.com/hs-fs/hubfs/figure3-crop.png?width=400&amp;name=figure3-crop.png 400w, https://www.uptycs.com/hs-fs/hubfs/figure3-crop.png?width=800&amp;name=figure3-crop.png 800w, https://www.uptycs.com/hs-fs/hubfs/figure3-crop.png?width=1200&amp;name=figure3-crop.png 1200w, https://www.uptycs.com/hs-fs/hubfs/figure3-crop.png?width=1600&amp;name=figure3-crop.png 1600w, https://www.uptycs.com/hs-fs/hubfs/figure3-crop.png?width=2000&amp;name=figure3-crop.png 2000w, https://www.uptycs.com/hs-fs/hubfs/figure3-crop.png?width=2400&amp;name=figure3-crop.png 2400w" sizes="(max-width: 800px) 100vw, 800px"></a></p>
<p style="text-align: center;"><em>Figure 3: Structure of the DOCX and footer.xml.res pointing to the malicious template. (<a href="https://f.hubspotusercontent00.net/hubfs/2617658/figure3-crop.png" rel="nofollow noopener" target="_blank">Click to see larger version</a>.)</em></p>
<p>&nbsp;</p>
<p>When the victim opens the document, one of the templates is downloaded and executed.</p>
<p>&nbsp;</p>
<h2>The Template File</h2>
<p>The template file("1.docx" ... "9x.docx") follows the structure shown in Figure 4 (below). The settings.xml in the structure have the “target” fields that point to XLSM files, which are present in the “embeddings'' directory in the structure of the DOCX file.</p>
<p>&nbsp;</p>
<p>The XLSM files “Microsoft_Excel_Macro-Enabled_Worksheet.xlsm” to “Microsoft_Excel_Macro-Enabled_Worksheet9.xlsm” have the same contents (same SHA-256: 32f1a502126b1932e1def04b98d8be235c8d25ef7268f8cb35d460cd073a88b2). When the template file ("1.docx" ... "9x.docx") is executed by Microsoft Word, it executes one of the XLSM files (“Microsoft_Excel_Macro-Enabled_Worksheet.xlsm” to “Microsoft_Excel_Macro-Enabled_Worksheet9.xlsm”).</p>
