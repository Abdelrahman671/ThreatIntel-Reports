			<h1 class="entry-title">Dridex &#8211; From Word to Domain Dominance</h1>
		<div class="entry-meta">
			<span class="posted-on"><a href="https://thedfirreport.com/2020/08/03/dridex-from-word-to-domain-dominance/" rel="bookmark"><time class="entry-date published" datetime="2020-08-03T10:19:10+00:00">August 3, 2020</time></a></span>		</div><!-- .entry-meta -->

		
	</header><!-- .entry-header -->

	<div class="entry-content">
		
<figure>Are you familiar with the <a href="https://detect-respond.blogspot.com/2013/03/the-pyramid-of-pain.html" target="_blank" rel="noopener noreferrer">pyramid of pain</a>? The idea is that to make life harder for attackers we need to push up the pyramid in our defenses. If they have a playbook, then they will keep using it until we break it.</figure>
<p>&nbsp;</p>
<p><img fetchpriority="high" decoding="async" class="alignnone size-full wp-image-616" src="https://thedfirreport.com/wp-content/uploads/2020/07/Pyramid-of-Pain-v2.png" alt="" width="720" height="405" srcset="https://thedfirreport.com/wp-content/uploads/2020/07/Pyramid-of-Pain-v2.png 720w, https://thedfirreport.com/wp-content/uploads/2020/07/Pyramid-of-Pain-v2-300x169.png 300w" sizes="(max-width: 720px) 100vw, 720px" /></p>
<p>We see a good example of this in our Dridex sample. We executed a malicious Word doc in our honeypot which later elevated to multiple Empire shells across the domain as well as additional Dridex installations. The threat actors used well known tools, moved like they were running a playbook, and used an Empire C2 server known to the community for 8 months.</p>
<p><span style="color: #000000;">So, what is Dridex?</span></p>
<blockquote>
<p><span style="color: #000000;">Dridex also known as Bugat and Cridex is a form of malware that specializes in stealing bank credentials via a system that utilizes macros from Microsoft Word. </span><a style="font-size: 16px;" href="https://en.wikipedia.org/wiki/Dridex" target="_blank" rel="noopener noreferrer">https://en.wikipedia.org/wiki/Dridex</a></p>
</blockquote>
<p>As we&#8217;ve seen over the last few years, Dridex is more than just a banking Trojan. </p>
<blockquote>
<p><span style="color: #000000;">Cybersecurity industry reporting attributes Dridex, BitPaymer, and Locky campaigns, as well as other massive malware spam (malspam) campaigns to actors known alternately as Evil Corp or TA505. Actors distributing Dridex likely employ ransomware with similar configurations. Code for BitPaymer, also known as Friedex, includes numerous similarities to Dridex, despite its function as ransomware rather than data extraction. The two malwares use the same mechanics for several functions, and the authors compiled the codes at nearly the same time. The ransomware distributed through these malwares has targeted U.S. financial institutions and resulted in data and financial loss. </span><a style="font-size: 16px;" href="https://us-cert.cisa.gov/ncas/alerts/aa19-339a" target="_blank" rel="noopener noreferrer">https://us-cert.cisa.gov/ncas/alerts/aa19-339a</a></p>
</blockquote>
<p>More info on Dridex:</p>
<p><a href="https://www.alertlogic.com/resources/threat-reports/dridex-malware-has-evolved-to-locky-ransomware/" target="_blank" rel="noopener noreferrer">Dridex &amp; Locky</a></p>
<p><a href="https://lifars.com/2019/11/from-dridex-to-bitpaymer-ransomware-to-doppelpaymerthe-evolution/" target="_blank" rel="noopener noreferrer">Dridex &amp; BitPaymer</a></p>
<p><a href="https://www.zdnet.com/article/us-charges-two-members-of-the-dridex-malware-gang/" target="_blank" rel="noopener noreferrer">Dridex &amp; EvilCorp</a></p>
<p><a href="https://malpedia.caad.fkie.fraunhofer.de/details/win.dridex" target="_blank" rel="noopener noreferrer">Additional Dridex write-ups on malpedia</a></p>
<h3>Report Structure:</h3>
<p>We&#8217;ll be trying out the <a href="https://www.csacademy.nl/images/scripties/2018/Paul-Pols---The-Unified-Kill-Chain.pdf">Unified Kill Chain</a> as our flow today.</p>
<p><img decoding="async" class="alignnone size-full wp-image-645" src="https://thedfirreport.com/wp-content/uploads/2020/07/unified-killchain.png" alt="" width="818" height="300" srcset="https://thedfirreport.com/wp-content/uploads/2020/07/unified-killchain.png 818w, https://thedfirreport.com/wp-content/uploads/2020/07/unified-killchain-300x110.png 300w, https://thedfirreport.com/wp-content/uploads/2020/07/unified-killchain-768x282.png 768w" sizes="(max-width: 818px) 100vw, 818px" /></p>
<h2>Initial Foothold:</h2>
<h3>Delivery:</h3>
<p>Like many threat activities today, this started with delivery of a malicious Word document. See:</p>
<p><a href="https://app.any.run/tasks/ced14735-0ed0-49c4-a4d7-f7fe169a81ee/" target="_blank" rel="noopener noreferrer">July2020_2485413825.doc</a> e3589aa5d687e58ee97bda2c501bcba9d5e942fe929644602dd1645b3c7f0e94 </p>
<p>Using <a href="https://github.com/decalage2/oletools/wiki/olevba" target="_blank" rel="noopener noreferrer">olevba</a> to look at the file, we get some clues on the file&#8217;s linage and probable execution activity.</p>

<p>After pulling down PsExec, the threat actor ran the following command.</p>
<pre class="c-mrkdwn__pre" data-stringify-type="pre">pse.exe -accepteula @a.txt -c -f -d -s ufo.exe &gt; log.txt</pre>
<p>Here&#8217;s what that command does:</p>
<p>-accepteula: Suppresses the display of the license dialog.</p>
<p>@file: Execute the command on each of the computers listed in the file.</p>
<p>-c: Copy the specified executable to the remote system for execution.</p>
<p>-f: Copy the specified program even if the file already exists on the remote system.</p>
<p>-d: Don&#8217;t wait for process to terminate (non-interactive).</p>
<p>-s: Run the remote process as the System account.</p>
<p>As you can see above, ufo.exe is executed as System across multiple hosts, which is all logged to log.txt. The below snippet is from log.txt, showing that the file was copied/run.</p>
<pre>Starting PSEXESVC service on hostname...<br />Connecting with PsExec service on hostname...<br />Copying ufo.exe to hostname...<br />Starting ufo.exe on hostname...</pre>
<p>Unfamiliar with how PsExec works? Here&#8217;s a good summation by <a href="https://guyrleech.wordpress.com/tag/psexesvc/" target="_blank" rel="noopener noreferrer">Guy Leech</a></p>
<blockquote>
<p>So when psexec is used to run something on a remote system, it works by creating a new service executable called psexesvc.exe which is embedded within the original psexec.exe file. This is copied to the Windows folder on the remote machine via the admin$ default share (hence why you need to be an admin to get psexec to work remotely). It then creates the PSEXESVC service with this, now local, executable, starts it and then runs the specified command.</p>
</blockquote>
<p>Here we can see psexesvc.exe being copied to the machine over SMB. You can rename but you can&#8217;t hide&#8230;</p>
