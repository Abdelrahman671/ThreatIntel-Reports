<p><em>Original KB number:</em> &nbsp; 556003</p>
<p>This article was written by <a href="https://mvp.microsoft.com/en-US/PublicProfile/33635?fullName=Nirmal%20K%20Sharma%20%28Ratawa%29" data-linktype="external">Nirmal Sharma</a>, Microsoft MVP.</p>
<h2 id="summary">Summary</h2>
<p><em>The following knowledgebase will explain the uses of Net commands in Windows Operating Systems.</em></p>
<h2 id="more-information">More information</h2>
<p>Net Commands On Windows Operating Systems</p>
<p>The following Net Commands can be used to perform operations on Groups, users, account policies, shares, and so on.</p>
<h3 id="net">NET</h3>
<p>ACCOUNTS<br>
COMPUTER<br>
CONFIG<br>
CONTINUE<br>
FILE<br>
GROUP<br>
HELP<br>
HELPMSG<br>
LOCALGROUP<br>
NAME<br>
PAUSE<br>
PRINT<br>
SEND<br>
SESSION<br>
SHARE<br>
START<br>
STATISTICS<br>
STOP<br>
TIME<br>
USE<br>
USER<br>
VIEW</p>
<p>The "Net Accounts" command is used to set the policy settings on local computer, such as Account policies and password policies. This command can't be used on domain controller. This command is only used on local computer.</p>
<p>When you type Net Accounts, you will see the default settings of the Account Lockout policy and Password Policy in local computer show as:</p>
<pre><code class="lang-output">Force user logoff how long after time expires?: Never  
Minimum password age (days): 1  
Maximum password age (days): 90  
Minimum password length: 8  
Length of password history maintained: 5  
Lockout threshold: 4  
Lockout duration (minutes): 4  
Lockout observation window (minutes): 4  
Computer role: WORKSTATION
</code></pre>
<p>The above settings displayed as per the role of the computer. If computer is joined to a domain, the domain settings will take effect and only the settings coming from domain will be displayed. The rest settings will be the local settings if it's not coming from the Domain GPO.</p>
<p>You can change the following use the following options in Net Accounts option:</p>
<pre><code class="lang-console">NET ACCOUNTS  
[/FORCELOGOFF:{minutes | NO}]  
[/MINPWLEN:length]  
[/MAXPWAGE:{days | UNLIMITED}]  
[/MINPWAGE:days]  
[/UNIQUEPW:number] [/DOMAIN]
</code></pre>
<p>Two conditions are required in order for options used with NET ACCOUNTS to take effect:</p>
<p><code>/FORCELOGOFF:{minutes | NO}</code> Sets the number of minutes a user has before being forced to log off when the account expires or valid logon hours expire. NO, the default, prevents forced logoff.</p>
<p><code>/MINPWLEN:length</code> Sets the minimum number of characters for a password. The range is 0-14 characters; the default is six characters.</p>
<p><code>/MAXPWAGE:{days | UNLIMITED}</code> Sets the maximum number of days that a
password is valid. No limit is specified by using UNLIMITED. /MAXPWAGE can't be less than /MINPWAGE. The range is 1-999; the default is 90 days.</p>
<p><code>/MINPWAGE:days</code> Sets the minimum number of days that must pass before a user can change a password. A value of zero sets no minimum time. The range is 0-999; the default is zero days. /MINPWAGE can't be more than /MAXPWAGE.\</p>
<p><code>/UNIQUEPW:number</code> Requires that a user's passwords be unique through the specified number of password changes. The maximum value is 24.</p>
<p><code>/DOMAIN</code> Performs the operation on a domain controller of the current domain. Otherwise, the operation is performed on the local computer.</p>
