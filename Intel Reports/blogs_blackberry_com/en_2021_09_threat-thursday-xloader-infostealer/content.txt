    <div class="text">    
    
    <h3><b>Background</b></h3>
<p>xLoader is an information-stealing malware targeting both macOS® and Windows®. Previously sold on underground forums under the name <a title="FormBook" href="https://www.blackberry.com/us/en/solutions/endpoint-security/ransomware-protection/formbook">FormBook</a>, xLoader surfaced in early 2020 shortly after FormBook was shut down by its author.</p>
<p>xLoader is sold under a Malware-as-a-Service (MaaS) agreement. Subscribers have access to the administrative panel and executable builds for both Windows and macOS. The authors of xLoader retain full control of command-and-control (C2) infrastructure and malware builds for each target environment.</p>
<h3><b>Operating System</b></h3>

    <div class="text">    
    
    <h3><b>Technical Analysis</b></h3>
<p>For this report, we analyzed a Windows build of xLoader.</p>
<p><b>Sample hash: </b>9f7b903ab126b2a3a0ca3c5977bbf84111f52a6e3a6e43aa127763e1a46b8f2d</p>
<p>The file is a 32-bit Windows executable, which shows a compile timestamp in 2001. Given what we know of xLoader’s history, this is clearly a false date and an indication of this threat’s evasive behavior.</p>
<p>Basic static analysis reveals more abnormal features:</p>
<p style="margin-left: 40.0px;">1.&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Zero imported libraries and functions<br>
2.&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; No data directory entries<br>
3.&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; A single executable “.text” section<br>
4.&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; A very small number of legible strings</p>
<p>This sample is a very feature-limited build. Perhaps the author adopted a “less is more” mindset, believing that a reduced analysis “surface” would help to avoid suspicion. When placed under the analysis microscope, this decision was largely counter-productive as the malware’s simplicity only serves to highlight its unusual features.</p>
<p>Disassembly shows the code is heavily obfuscated:</p>

    
    
</div>


    
    
    <div class="image">
  <div data-cmp-is="image" data-cmp-src="/en/2021/09/threat-thursday-xloader-infostealer/_jcr_content/main/par/sectionblog/par/image_1778095965.coreimg{.width}.png/1677529423690/ttxloader-fig01.png" data-asset="/content/dam/blogs-blackberry-com/images/blogs/2021/09/ttxloader-fig01.png" data-asset-id="a95d0ce3-b186-4508-984a-c85287f821dd" class="cmp-image" itemscope itemtype="http://schema.org/ImageObject">

    <div class="text">    
    
    <p><i>Figure 1: Obfuscated code flow within the xLoader binary</i></p>
<p>Turning to dynamic analysis, when xLoader is launched, it will first manually load ntdll.dll from disk. This approach of manual mapping is an evasive technique that aims to bypass hooks put in place by (poorly implemented) endpoint protection and automated malware analysis environments.</p>
<p>With access to ntdll, xLoader performs a battery of anti-analysis checks. This includes looking for the following conditions:</p>
<p style="margin-left: 40.0px;">1.&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; The presence of a user-mode debugger<br>
2.&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; The presence of a kernel-mode debugger<br>
3.&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Execution timing checks (rdtsc)<br>
4.&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; The presence of blacklisted process names<br>
5.&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; The presence of blacklisted modules<br>
6.&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; The presence of blacklisted usernames<br>
7.&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; The Thread Environment Block (TEB) Wow32Reserved field points to a 64-bit module</p>
<p>As this last point is a bit more complicated, let’s investigate it a little further.</p>
<p>In this instance, xLoader is a 32-bit process. Under normal operation, Wow32Reserved in the TEB resolves to a function located in wow64cpu.dll. xLoader will validate the PE header of whatever Wow32Reserved points to, and confirm that the expected offset for PE “Magic Number” is 0x20B, indicating a 64-bit (PE32+) module.</p>
<p>Any naïve attempts to intercept API calls made by xLoader, by overwriting Wow32Reserved, would likely be met with having it point to a user allocated block of memory containing shellcode. This would lack the proper PE header and expected “Magic Number” value.</p>
<p>The success or failure of each of the seven anti-analysis checks listed above is recorded in a 16-byte field. Once all checks are completed, the hash value of this field is used to decrypt API strings. Rudimentary attempts to bypass the checks result in incorrect decryption and deliberate self-destruction by way of an unhandled exception.</p>

    
    
</div>

    <p><i>Figure 6: Self-deletion using cmd.exe</i></p>
<h3><b>Persistence<br>
</b></h3>
<p>Persistence is achieved using the Windows registry, by creating an entry in the HKey_Current_User hive: HKCU\Software\Microsoft\Windows\CurrentVersion\Run\&lt;RANDOM&gt; REG_SZ</p>
<p>This entry points to a copy of itself dropped in a randomly-named subfolder of C:\Program Files (x86):</p>

<h3><b>Command &amp; Control:</b><br>
</h3>
<p>Handling of C2 requests remains the responsibility of the first-stage code running inside explorer.exe. The initial check-in with the channel comprises the following basic host information:</p>
<p style="margin-left: 40.0px;">1.&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Handshake bytes (“XLNG”)<br>
2.&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Hash of the user’s Windows Security Identifier (SID) (“28BED20C”)<br>
3.&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Version of xLoader (“2.3”)<br>
4.&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Operating system version and “bitness” (x64)<br>
5.&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; Username, Base64 encoded</p>

<p>This host “fingerprint” is encrypted and Base64 encoded prior to transmission.</p>
<p>Repeated sandbox analysis shows xLoader randomly selecting 16 C2 domains from an embedded pool of 43, with only one being the genuine C2 host. Other domains may be included, to make C2 traffic less obvious, or harder to identify against a backdrop of benign HTTP requests. Each domain in the set is contacted an equal amount in a round-robin fashion. Check-in attempts are made at five-second intervals and continue indefinitely.</p>
<p>The campaign ID for the analyzed build is “ipa8,” which is included in the GET request:</p>

    <p><i>Figure 9: GET request to xLoader C2 with encoded host identification</i></p>
<p>When check-in succeeds, xLoader can be instructed to:</p>
<ul>
<li>Download and launch additional payloads</li>
<li>Upload screenshots</li>
<li>Intercept and steal user credentials or cookies</li>
<li>Commence keystroke logging</li>
<li>Terminate and uninstall</li>
<li>Steal clipboard contents</li>
</ul>
<h3><b>One Tool to Bind Them</b><br>
</h3>
<p>Subscription to xLoader includes access to a free Java-based tool called xBinder, that combines Windows and macOS executables into a single, portable .jar file. Given Java’s prevalence, this is an attractive feature. It makes crafting email attachments for social engineering or watering hole downloads possible avenues for attack.</p>
<p>Within the .jar file, the macOS and Windows executables are bundled and encrypted using AES with a static key. When launched, a runtime probe identifies the host operating system. It then drops and executes the corresponding payload.</p>
<p>A version of xBinder can be found in VirusTotal, indicating a possible leak or accidental upload. A handful of analyzed xBinder-generated .jar files contained benign macOS command line tools, such as “/bin/ls,” together with malicious Windows executables from these different families:</p>
<ul>
<li>Azorult (info stealer)</li>
<li>Snake (info stealer)</li>
<li>Neshta (file infector)</li>
<li>Guloader (downloader)</li>
</ul>
<p>This mismatched bundling of benign files together with malicious code from disparate malware families suggests unsanctioned use of xBinder by groups or individuals, rather than paying subscribers.</p>

<h3><b>Conclusion</b></h3>
<p>xLoader promotes itself among the rank-and-file of underground MaaS offerings as cheap, highly functional and resilient. With its evasive and mature binaries for both macOS and Windows, together with the provision of a free binding tool that caters to more indiscriminate targeting, xLoader’s offering has appeal for both fledgling cyber criminals and larger, more organized groups.</p>
