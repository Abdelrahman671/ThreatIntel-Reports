srcset="https://blog.comodo.com/wp-content/uploads/versions-of-psixbot-455x397.jpg 455w, https://blog.comodo.com/wp-content/uploads/versions-of-psixbot-310x270.jpg 310w" sizes="100vw" /> <span class="rt-reading-time" style="display: block;"><span class="rt-label">Reading Time: </span> <span class="rt-time">4</span> <span class="rt-label rt-postfix">minutes</span></span><h2><strong><u>Introduction of PSIXBOT:</u></strong></h2><p>PsiXBot is data-stealing trojan capable of harvesting confidential data and passwords from a victim’s computer. It can steal cookies, extract logins/passwords from applications like Firefox and Microsoft Outlook, record the victim’s keystrokes, allow criminals to remotely view/interact with the victim’s desktop, and can even add the victim’s computer to a botnet. It is most often spread via infected email attachments, via online adverts which contain the bot, and via other social engineering methods.</p><p>The original PsixBot malware surfaced in November 2017 but underwent significant development before arriving in beta format in 2019.  It has since been developed further and currently stands at version 1.1.0.4 in February 2020:</p><p><img decoding="async" loading="lazy" class="aligncenter size-medium wp-image-24462" src="https://blog.comodo.com/wp-content/uploads/psixbot-malware-300x102.png" alt="" width="300" height="102" style="margin: 20px 0;" srcset="https://blog.comodo.com/wp-content/uploads/psixbot-malware-300x102.png 300w, https://blog.comodo.com/wp-content/uploads/psixbot-malware.png 411w" sizes="(max-width: 300px) 100vw, 300px" /></p><p>PsixBot was generated in .NET framework. This blog takes you through the various iterations of PsixBot to illustrate how online criminals constantly update their <a href="https://freethreatanalysis.comodo.com/?track=16053&amp;af=16053" target="_blank" rel="noopener" title="Free Threat Analysis Scorecard">malware</a> to improve its performance and features.</p><h2><strong><u>Behaviour of PsixBot</u></strong></h2><p>PsixBot changes the system certificate settings, which gives it virtually unlimited user access rights on the host machine:</p><p><strong>Keys added:</strong></p><p style="word-wrap: anywhere;">KEY_LOCAL_MACHINE\SOFTWARE\Microsoft\SystemCertificates\TrustedPeople\Certificates\636D2838EB7A7F3A8E6B6F7CD035375E7E704248</p><p><strong>Values added:</strong></p><p style="word-wrap: anywhere;">HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\SystemCertificates\TrustedPeople\Certificates\636D2838EB7A7F3A8E6B6F7CD035375E7E704248\Blob: 02 00 00 ……..</p><p><strong>Files added:</strong></p><p style="word-wrap: anywhere;">C:\Documents and Settings\Administrator\Application Data\</p><p style="word-wrap: anywhere;">Microsoft\SystemCertificates\My\Certificates\636D2838EB7A7F3A8E6B6F7CD035375E7E704248</p><h3><strong><u>Beta 1.0.0</u></strong></h3><p>The first version of PsixBot covered in this blog is Beta 1.0.0 with the core class 11. Each class has its individual task. The following basic classes are used in all versions of PsixBot:</p><ul><li><strong>Servertalk</strong> &#8211; used to initialize the global variable, create the connection with the mothership server, and send results back and forth.</li><li><strong>RunInMemory </strong>&#8211; used to actually execute the file.</li><li><strong>SysInfo </strong>&#8211; used to obtain information about the user’s system, including antivirus name, CPU, Windows version, user type and user permissions.</li><li><strong>CatchEndSession</strong> &#8211; used to create hidden autoruns.</li><li><strong>DeleteAttrib </strong>&#8211; used to kill the system’s <a href="https://www.comodo.com/home/internet-security/antivirus.php" target="_blank" rel="noopener">antivirus software</a>, Windows Explorer, and any system error alerts.</li><li><strong>IsAdmin</strong> &#8211; used to assume membership of the admin group.</li><li><strong>IsVm</strong> &#8211; detects the presence of any virtual machines.</li><li><strong>ResolveBit </strong>&#8211; used to resolve DNS requests from the user.</li><li><strong>RC4</strong> &#8211; the algorithm used to encrypt and decrypt data.</li><li><strong>Install</strong> &#8211; installs the bot file and sets up the file’s security and update modules.</li></ul><p><img decoding="async" loading="lazy" class="size-full wp-image-24471 aligncenter" src="https://blog.comodo.com/wp-content/uploads/Beta-1.0.0.png" alt="" width="197" height="209" style="margin: 20px 0;"/></p><h3><strong><u>Version 1.0.2</u></strong></h3><p>Beta 1.0.2 retained the basic class functionality of the first version, but renamed some of the classes as follows:</p><ul><li><strong>ServerTalk &#8211; </strong>renamed as <strong>CpWorker</strong></li><li><strong>RunInMemory &#8211; </strong>renamed as <strong>MemoryModulesWorker</strong></li><li><strong>SysInfo &#8211; </strong>renamed as<strong> SysHelper</strong></li></ul><p>&#8230; and added the following class:</p><ul><li><strong>DNSWorker</strong> – used to get the host entry and ping the host to check whether or not it is up.</li></ul><p><img decoding="async" loading="lazy" class="size-medium wp-image-24492 aligncenter" src="https://blog.comodo.com/wp-content/uploads/Version-1.0.2-300x70.png" alt="" width="300" height="70" style="margin: 20px 0;" srcset="https://blog.comodo.com/wp-content/uploads/Version-1.0.2-300x70.png 300w, https://blog.comodo.com/wp-content/uploads/Version-1.0.2.png 464w" sizes="(max-width: 300px) 100vw, 300px" /></p><h3><strong><u>Version 1.1</u></strong></h3><p>Version 1.1 again retained the same class structure as its predecessor but added the following task to the features list:</p><ul><li><strong>Forfg &#8211;
