							<div class="content "><h1 id="net-use">Net use</h1><div class="display-flex justify-content-space-between align-items-center flex-wrap-wrap page-metadata-container">
										<div class="margin-right-xxs">
											<ul class="metadata page-metadata" data-bi-name="page info" lang="en-us" dir="ltr"><li>Article</li><li>
														<time class="is-invisible" data-article-date aria-label="Article review date" datetime="2009-09-11T00:00:00.000Z" data-article-date-source="ms.date">09/11/2009</time>
													</li></ul>
										</div></div><nav id="center-doc-outline" class="doc-outline is-hidden-desktop display-none-print margin-bottom-sm" data-bi-name="intopic toc" aria-label="In this article">
											<h2 id="ms--in-this-article" class="title is-6 margin-block-xs">In this article</h2>
										</nav><!-- <content> --><p>Connects a computer to or disconnects a computer from a shared resource, or displays information about computer connections. The command also controls persistent net connections. Used without parameters, <strong>net use</strong> retrieves a list of network connections.</p>
<h3 id="syntax">Syntax</h3>
<p><strong>net use</strong>&nbsp;[{<em>DeviceName</em> | <strong>*</strong>}] [<strong>\\<em><strong>ComputerName</strong></em>\<strong><em>ShareName</em>[</strong>\<strong><em>volume</em>]] [{<em>Password</em> | <strong>*</strong>}]] [</strong>/user:</strong>[<em>DomainName</em>**\<strong>]<em>UserName</em>] [</strong>/user:<strong>[<em>DottedDomainName</em></strong>\<strong>]<em>UserName</em>] [</strong>/user:** [<em>UserName</em>**@<strong><em>DottedDomainName</em>] [</strong>/savecred**] [<strong>/smartcard</strong>] [{<strong>/delete</strong> | <strong>/persistent:</strong>{<strong>yes</strong> | <strong>no</strong>}}]</p>
<p><strong>net use</strong> [<em>DeviceName</em>&nbsp;[<strong>/home</strong>[{<em>Password</em> | <strong>*</strong>}] [<strong>/delete:</strong>{<strong>yes</strong> | <strong>no</strong>}]]</p>
<p><strong>net use</strong>&nbsp;[<strong>/persistent:</strong>{<strong>yes</strong> | <strong>no</strong>}]</p>
<h3 id="parameters">Parameters</h3>
<p><em><strong>DeviceName</strong></em> &nbsp; <strong>:</strong> Assigns a name to connect to the resource or specifies the device to be disconnected. There are two kinds of device names: disk drives (that is, D: through Z:) and printers (that is, LPT1: through LPT3:). Type an asterisk (*) instead of a specific device name to assign the next available device name.</p>
<p><strong>\\</strong> <em><strong>ComputerName</strong></em> <strong>\</strong> <em><strong>ShareName</strong></em> &nbsp; <strong>:</strong> Specifies the name of the server and the shared resource. If <em>ComputerName</em> contains spaces, use quotation marks around the entire computer name from the double backslash (\\) to the end of the computer name (for example, <strong>"\\<em><strong>Computer&nbsp;Name</strong></em>\<em><strong>Share&nbsp;Name</strong></em>"</strong>). The computer name can be from 1 to 15 characters long.</p>
<p><strong>\</strong> <em><strong>volume</strong></em> &nbsp; <strong>:</strong> Specifies a NetWare volume on the server. You must have Client Service for NetWare installed and running to connect to NetWare servers.</p>
<p><em><strong>Password</strong></em> &nbsp; <strong>:</strong> Specifies the password needed to access the shared resource. Type an asterisk (*) to produce a prompt for the password. The password is not displayed when you type it at the password prompt.</p>
<p><strong>/user</strong> &nbsp; <strong>:</strong> Specifies a different user name with which the connection is made.</p>
<p><em><strong>DomainName</strong></em> &nbsp; <strong>:</strong> Specifies another domain. If you omit <em>DomainName</em>, <strong>net use</strong> uses the current logged on domain.</p>
<p><em><strong>UserName</strong></em> &nbsp; <strong>:</strong> Specifies the user name with which to log on.</p>
<p><em><strong>DottedDomainName</strong></em> &nbsp; <strong>:</strong> Specifies the fully-qualified domain name for the domain where the user account exists.</p>
<p><strong>/savecred</strong> &nbsp; <strong>:</strong> Stores the provided credentials for reuse.</p>
<p><strong>/smartcard</strong> &nbsp; <strong>:</strong> Specifies the network connection is to use the credentials on a smart card. If multiple smart cards are available, you are asked to specify the credential.</p>
<p><strong>/delete</strong> &nbsp; <strong>:</strong> Cancels the specified network connection. If you specify the connection with an asterisk (*), all network connections are canceled.</p>
<p><strong>/persistent:</strong> <strong>{</strong> <strong>yes</strong> <strong>|</strong> <strong>no</strong> <strong>}</strong> <strong>:</strong> Controls the use of persistent network connections. The default is the setting used last. Deviceless connections are not persistent. <strong>Yes</strong> saves all connections as they are made, and restores them at next logon. <strong>No</strong> does not save the connection being made or subsequent connections. Existing connections are restored at the next logon. Use <strong>/delete</strong> to remove persistent connections.</p>
<p><strong>/home</strong> &nbsp; <strong>:</strong> Connects a user to the home directory.</p>
<p><strong>net help</strong> &nbsp; <em><strong>command</strong></em> &nbsp; <strong>:</strong> Displays help for the specified <strong>net</strong> command.</p>
<h3 id="remarks">Remarks</h3>
<ul>
<li><p>Connecting and disconnecting from a network resource</p>
<p>Use <strong>net use</strong> to connect to and disconnect from a network resource, and to view your current connections to network resources. You cannot disconnect from a shared directory if you use it as your current drive or an active process is using it.</p>
</li>
<li><p>Viewing connection information</p>
<p>To view information about a connection, you can do either of the following:</p>
<ul>
<li><p>Type <strong>net use</strong>&nbsp;<em>DeviceName</em> to get information about a specific connection.</p>
</li>
<li><p>Type <strong>net use</strong> to get a list of all the computer's connections.</p>
</li>
</ul>
</li>
<li><p>Using deviceless connections</p>
<p>Deviceless connections are not persistent.</p>
</li>
<li><p>Connecting to NetWare servers</p>
<p>After you install and run Client Service for NetWare, you can connect to a NetWare server on a Novell network. Use the same syntax that you use to connect to a Windows Networking server, except you must include the volume you to which you want to connect.</p>
</li>
<li><p>Using quotation marks</p>
<p>If the <em>ServerName</em> that you supply contains spaces, use quotation marks around the text (that is, <strong>"<em><strong>Server&nbsp;Name</strong></em>"</strong>). If you omit quotation marks, an error message appears.</p>
</li>
</ul>
<h3 id="examples">Examples</h3>
<p>To assign the disk-drive device name E: to the Letters shared directory on the \\Financial server, type:</p>
<p><strong>net use e: \\financial\letters</strong>&nbsp;</p>
<p>To assign (map) the disk-drive device name M: to the directory Mike within the Letters volume on the \\Financial NetWare server, type:</p>
<p><strong>net use m: \\financial\letters\mike</strong>&nbsp;</p>
<p>To connect the user identifier Dan as if the connection were made from the Accounts domain, type:</p>
<p><strong>net use d:\\server\share /user:Accounts\Dan</strong>&nbsp;</p>
<p>To disconnect from the \\Financial\Public directory, type:</p>
<p><strong>net use f: \\financial\public /delete</strong>&nbsp;</p>
<p>To connect to the resource memos shared on the \\Financial 2 server, type:</p>
<p><strong>net use k: "\\financial 2" \memos</strong>&nbsp;</p>
<p>To restore the current connections at each logon, regardless of future changes, type:</p>
<p><strong>net use /persistent:yes</strong>&nbsp;</p>
