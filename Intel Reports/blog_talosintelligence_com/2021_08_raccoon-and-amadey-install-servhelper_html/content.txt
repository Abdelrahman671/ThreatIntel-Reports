    <h1 class="text-center">Signed MSI files, Raccoon and Amadey are used for installing ServHelper RAT</h1>

    <div class="text-center m-1">

    <section class="post-content-wrapper mt-5">
        <div class="post-content">
            <p><i>By <a href="https://twitter.com/vanjasvajcer">Vanja Svajcer</a>.</i><br></p><h3 id="news-summary">News summary</h3><ul><li>Group TA505 has been active for at least seven years, making wide-ranging connections with other threat actors involved in ransomware, stealing credit card numbers and exfiltrating data. One of the common tools in TA505's arsenal is ServHelper. In mid-June, Cisco Talos detected an increase in ServHelper's activity. We investigated the activity and discovered a set of intertwined malware families and TTPs.</li><li>We found that ServHelper is being installed onto the targeted systems using several different mechanisms, ranging from fake installers for popular software to using other malware families such as Raccoon and Amadey as the installation proxies.</li><li>This threat demonstrates several techniques of the <a href="https://attack.mitre.org/">MITRE ATT&amp;CK framework</a>, most notably Scripting - <a href="https://attack.mitre.org/techniques/T1064/">T1064</a>, PowerShell - <a href="https://attack.mitre.org/techniques/T1059/001/">T1059.001</a>, Process Injection - <a href="https://attack.mitre.org/techniques/T1055/">T1055</a>, Non-Standard Port - <a href="https://attack.mitre.org/techniques/T1571/">T1571</a>, Remote Access Software - <a href="https://attack.mitre.org/techniques/T1219/">T1219</a>, Input Capture - <a href="https://attack.mitre.org/techniques/T1056/">T1056</a>, Obfuscated Files or Information - <a href="https://attack.mitre.org/techniques/T1027/">T1027</a>, Ingress Tool Transfer - <a href="https://attack.mitre.org/techniques/T1105/">T1105</a>, and Registry Run Keys/Startup Folder - <a href="https://attack.mitre.org/techniques/T1547/">T1547.001</a>.</li></ul><h4 id="what-s-new-although-servhelper-has-existed-since-at-least-early-2019-we-detected-the-use-of-other-malware-families-to-install-it-the-installation-comes-as-a-golang-dropper-net-dropper-or-powershell-script-its-activity-is-generally-linked-to-group-ta505-but-we-cannot-be-certain-that-they-are-the-exclusive-users-of-this-rat-">What's new? Although ServHelper has existed since at least early 2019, we detected the use of other malware families to install it. The installation comes as a GoLang dropper, .NET dropper or PowerShell script. Its activity is generally linked to Group TA505, but we cannot be certain that they are the exclusive users of this RAT.</h4><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-MtaKW5nbMqA/YQLHIVoA0TI/AAAAAAAAAdY/Za6bCJWras095XwyUOVgcMOoQqSxRwhuwCPcBGAYYCw/s0/blog-onepager-button.jpg" class="kg-image" alt loading="lazy"></figure><p><br> ServHelper will also sometimes install a module that includes either Monero or Ethereum cryptocurrency mining tools.</p><h4 id="how-did-it-work-one-path-for-infection-starts-with-the-compromise-of-a-legitimate-site-that-hosts-cryptographically-signed-msi-installers-these-install-popular-software-such-as-discord-however-they-also-launch-a-variant-of-the-raccoon-stealer-which-downloads-and-installs-a-servhelper-rat-if-instructed-by-the-command-and-control-c2-server-">How did it work? One path for infection starts with the compromise of a legitimate site that hosts cryptographically signed MSI installers. These install popular software such as Discord. However, they also launch a variant of the Raccoon stealer, which downloads and installs a ServHelper RAT if instructed by the command and control (C2) server.</h4><p>Attackers also deploy the ServHelper RAT with a variant of the Amadey malware which gets a full command line from the server to install an initial PowerShell downloader component for ServHelper.</p><p>ServHelper includes the functionality to remotely control the infected system, log keystrokes, exfiltrate users' confidential data, launch RDP sessions, install cryptomining software and install the NetSupport remote access tool.</p><h4 id="so-what-although-many-threat-actors-such-as-ta505-or-its-associated-groups-to-which-we-attribute-these-campaigns-with-moderate-confidence-have-been-affected-by-the-arrests-of-several-clop-members-in-ukraine-they-continued-to-operate-using-a-different-set-of-tools-these-attacks-are-geared-toward-taking-control-over-the-infected-systems-and-stealing-confidential-data-which-the-group-will-likely-leverage-for-financial-gain-later-on-">So what? Although many threat actors, such as TA505 or its associated groups — to which we attribute these campaigns with <a href="https://blog.talosintelligence.com/on-conveying-doubt/">moderate confidence</a> — have been affected by <a href="https://duo.com/decipher/ukrainian-police-arrest-suspected-cl0p-ransomware-operators">the arrests of several CLOP</a> members in Ukraine, they continued to operate using a different set of tools. These attacks are geared toward taking control over the infected systems and stealing confidential data which the group will likely leverage for financial gain later on.</h4><p>Users need to make sure they install software only from trusted sources. Even if installers are signed with a valid certificate, that does not mean that the functionality is legitimate.</p><h3 id="technical-details">Technical details</h3><h4 id="introduction-this-investigation-started-as-a-single-and-simple-call-to-an-ip-address-hosting-a-powershell-script-found-in-cisco-secure-product-telemetry-initially-while-looking-at-the-code-we-thought-it-is-just-one-of-many-cryptomining-malware-campaigns-">Introduction This investigation started as a single and simple call to an IP address hosting a PowerShell script found in Cisco Secure product telemetry. Initially, while looking at the code, we thought it is just one of <a href="https://blog.talosintelligence.com/cryptomining-analysis/">many cryptomining malware campaigns</a>.</h4><p>However, we realized soon that the main payload is a variant of a ServHelper backdoor. The usage of the tool is generally linked with the activity of the threat group <a href="https://malpedia.caad.fkie.fraunhofer.de/actor/ta505">TA505</a>. TA505 is a crimeware-focused group active since at least 2014. They are mostly known for their usage of the Dridex and CLOP ransomware families in their campaigns. There are some theories that CLOP ransomware gang members, <a href="https://krebsonsecurity.com/2021/06/ukrainian-police-nab-six-tied-to-clop-ransomware/">arrested in Ukraine in June 2021</a>, originated from the group TA505.</p><p>What started as an investigation into an Ethereum cryptominer, turned out to be a never-ending whirlwind of different malware families. Apart from the usual suspects, often attributed to TA505, such as the Raccoon stealer and Amadey stealer/loader, we have encountered a couple of newer techniques from this group. TA505 now uses an MSI installer signed with a valid certificate and a GoLang go-clr-based dropper that can load a .NET assembly from memory.</p><h4 id="initial-discovery-while-looking-at-the-daily-report-of-suspicious-command-lines-we-encountered-a-call-to-a-powershell-script-hosted-at-the-ip-address-94-158-245-88-">Initial discovery While looking at the daily report of suspicious command lines, we encountered a call to a PowerShell script hosted at the IP address 94.158.245.88.</h4><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-p8IF55y0pWw/YRI_3_s3fCI/AAAAAAAAAwA/7iHQGEtiuW4woguVFYc3LF-MjTfUhNT5ACLcBGAsYHQ/s16000/image7.png" class="kg-image" alt loading="lazy"></figure><p><i>Initial command in product telemetry.</i></p><p><br> The downloaded script was a simple PowerShell script to download additional scripts and launch a VBScript file, "start.vbs." We first assumed this was another cryptocurrency miner, but we decided to analyze the scripts because, as a cryptocurrency miner, it was not immediately familiar.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-pa5NJnrMqkw/YRJAmaTK6nI/AAAAAAAAAwI/d6fewQyodoYPclEO7T35LD6Sh1o1zTFDACLcBGAsYHQ/s16000/image6.png" class="kg-image" alt loading="lazy"></figure><p><i>Downloaded PowerShell script, "bi.ps1."</i> <br> Start.vbs is a simple driver that seems to check for the size of the System Management BIOS memory to avoid executing within virtual machines, and then launches the script "ready.ps1."<br></p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-b0PXt_l1o9w/YRJBEeUHW_I/AAAAAAAAAwQ/KDy8J2ZKYakLqtNHjbQVhML9m0qsnQZBQCLcBGAsYHQ/s16000/image18.png" class="kg-image" alt loading="lazy"></figure><p><i>Start.vbs first checks for the amount of SMBIOS memory before it launches the initial loader.</i> <br> Before executing the next stage, ready.ps1 decodes a base64-encoded string to reveal a tool for disabling the Microsoft AMSI interface, which is often used by anti-malware software for scanning the content of obfuscated PowerShell, VBA, VBS and scripts written in other Microsoft scripting languages. Once AMSI is disabled, ready.ps1 launches the next stage, which is the main PowerShell loader, resolve-domain.ps1.<br></p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-hi62MknzPYQ/YRJCuHla4wI/AAAAAAAAAwY/1SoN1WsvZmUYnXL3JTnoyuizW_pycPYoQCLcBGAsYHQ/s16000/image16.png" class="kg-image" alt loading="lazy"></figure><p><i>AMSI bypass in ready.ps1.</i> <br> The AMSI bypass first loads the amsi.dll library and then gets the address of the function AmsiScanBuffer. The first six bytes of the function are patched with 2 x86 instructions "MOV EAX,80070057;RET" to return the value 0x80070057 (INVALID ARG) to the caller so that it seems as if the scan has failed. <br></p><h5 id="main-powershell-loader-the-loader-starts-with-a-function-for-decrypting-the-content-of-a-variable-containing-the-real-powershell-code-of-the-loader-the-function-derives-a-tripledes-password-from-the-parameters-submitted-to-the-function-call-">Main PowerShell loader The loader starts with a function for decrypting the content of a variable containing the real PowerShell code of the loader. The function derives a TripleDES password from the parameters submitted to the function call.</h5><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-FW9VQgJdTvA/YRJDC-oOAiI/AAAAAAAAAwg/mazMi7qPQfQnYTd3bJ8ptl3e1KxnCC0CACLcBGAsYHQ/s16000/image15.png" class="kg-image" alt loading="lazy"></figure><p><i><i>TripleDES is used to decrypt the actual PowerShell loader.</i></i></p><p>When decrypted, the main loader contains several base64-encoded buffers and a logic to drop and load files decoded from those buffers. Before it's dropped to disc, the buffers need to be decrypted using a simple byte XOR scheme with a fixed seven-byte XOR key.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-l3sjtrW5aww/YRJD7Fp8dDI/AAAAAAAAAwo/04biTdhmOo8EKTfnlwuclekF1BPgPW8xACLcBGAsYHQ/s16000/image2.png" class="kg-image" alt loading="lazy"></figure><p><i>The main loader needs to deobfuscate the file content after Base64 decoding.</i> <br> The loader will, depending on the characteristics of the system, attempt to drop the payloads into the following locations:</p><ul><li>C:\Windows\branding\mediasvc.png</li><li>C:\Windows\branding\mediasrv.png</li><li>C:\Windows\branding\wupsvc.jpg</li><li>C:\Windows\system32\rdpclip.exe</li><li>C:\Windows\system32\rfxvmt.dll</li></ul><p>These locations are a telltale sign of ServHelper infection, a RAT that's been active since at least early 2019.</p><p>While reading the deobfuscated PowerShell script we noticed the download and execution of another URL: hxxp://45[.]61[.]136[.]223/get/m5.php. The call to this URL will be executed only if the amount of memory in the graphics card is larger than 4MB, which may be a way to evade some virtual machines or just a check for a machine with sufficient capacity to download the module, which is used for cryptocurrency mining.</p><h5 id="cryptomining-module-m5-php-the-cryptomining-module-hosted-on-hxxp-45-61-136-223-get-m5-php-is-what-prompted-us-to-initially-assume-that-the-final-payload-of-the-campaign-is-just-a-cryptominer-and-not-confidential-information-stealing-and-remote-control-of-the-infected-systems-which-may-be-attributed-based-on-the-known-ttps-to-group-ta505-">Cryptomining module m5.php The cryptomining module hosted on hxxp://45[.]61[.]136[.]223/get/m5.php is what prompted us to initially assume that the final payload of the campaign is just a cryptominer and not confidential information stealing and remote control of the infected systems which may be attributed, based on the known TTPs, to Group TA505.</h5><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-f1hMOlZCqig/YRJEydSnibI/AAAAAAAAAww/uOKD3QBbNEo3Vw3Zh4h7YCR8oNR5SJH7ACLcBGAsYHQ/s16000/image3.png" class="kg-image" alt loading="lazy"></figure><p><br></p><p><i>m5.php will only be downloaded if the size of the video controller memory is larger than 4MB.</i> <br> The m5 is lightly obfuscated, first by adding a decimal 92 to each of the characters and then by base64 encoding it. The script creates another PowerShell script in the C:\Windows\system32\update-request.ps1. The update-request module will be launched through a UAC bypass attempt which abuses a scheduled task \Microsoft\Windows\DiskCleanup\SilentCleanup by modifying the registry value HKCU:\Environment\windir to contain the call to the modified slmgr.vbs script containing the call to launch update-request.ps1.<br></p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-4b-HeQTs9tY/YRJFCbtW4VI/AAAAAAAAAw4/SglgLZ57k4Y1gKmQ5dcNUuVcIoPZP3CYwCLcBGAsYHQ/s16000/image4.png" class="kg-image" alt loading="lazy"></figure><p><i><i>UAC bypass to launch update-request.ps1.</i></i></p><p>Update-request.ps1 is obfuscated in a similarly simple manner to m5.php content. The script contains another layer of obfuscation with TripleDES similar to the one seen in resolve-domain.ps1. An interesting fact is that the decrypting function has a name decra, which is the same name as an export of a ServHelper service loader, which will be described a bit later.</p><p>Once decoded, update-request contains instructions to download and run cryptomining software. In some previous instances, ServHelper installations were linked to <a href="https://www.gdatasoftware.com/blog/2020/07/36122-hidden-miners">Monero mining</a> but in this campaign, a choice is made between mining Monero with XMRig and Ethereum. Ethermine and its configuration file are downloaded if Ethereum is the chosen miner.</p><p>The Ethminer payload is saved into the drive as C:\windows\system32\mui_pack_es.json as a base64-encoded file, which is also encrypted using XOR with the byte-based key "Asfianweiw". The Monero mining XMRig payload may be downloaded into C:\windows\system32\mui_pack.json and decrypted before loading into memory using the same method. This tactic is likely designed to avoid the detection of PE files by anti-malware software.</p><p>The mining payloads will only exist in the memory as PE modules and it will be loaded by injecting them reflectively into the legitimate process c:\windows\system32\msiexec.exe. The method to hollow msiexec seems to be a PowerShell port of the <a href="https://github.com/d35ha/RunPE/blob/master/RunPE.cpp">C++ RunPE process hollowing</a> project. It loads an embedded base64-encoded custom .NET assembly which is simply used as a trampoline to avoid having to import all the required functions from the kernel32.dll module.</p><p>Looking at the Ethereum payouts for the address 0x12420E4083F1E37b91AFA0E054682d049F9505C6 at the blockchain explorer on ethermine.org, we can see that the pool of miners with this address earned just over 17 Ethereums over the period of fewer than four months, which at the time of writing this post is worth slightly more than $33,000 USD. <br></p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-bepoXkuCDFk/YRJFhUZY4WI/AAAAAAAAAxA/kioQbwXGpHIaQUs2I0Gl439GqA60Zj0WACLcBGAsYHQ/s16000/image9.png" class="kg-image" alt loading="lazy"></figure><p><i>Ethermine payouts to the address associated with the TA505 ServHelper campaigns.</i> <br></p><h4 id="servhelper-installation-in-this-campaign-the-powershell-loader-resolve-domain-ps1-is-responsible-for-dropping-and-installing-the-servhelper-payload-in-this-case-the-main-loader-is-downloaded-from-a-website-but-in-some-other-cases-the-main-powershell-loader-is-dropped-by-an-executable-dropper-">ServHelper installation  In this campaign, the PowerShell loader resolve-domain.ps1 is responsible for dropping and installing the ServHelper payload. In this case, the main loader is downloaded from a website, but in some other cases, the main PowerShell loader is dropped by an executable dropper.</h4><p>The PowerShell loader uses the same SilentCleanup UAC bypass seen in the mining module to execute the registry. The file system can change without the UAC prompt.</p><p>The loading of ServHelper consists of two main DLL modules combined with helper modules to allow communication over RDP. The PowerShell module changes the registry keys related to the Windows service Terminal Server. The Terminal Server service is changed so that it listens on a non-default TCP port 7201 instead of the default port 3389. A new Remote Desktop service Termservice is created and its ServiceDLL value is set to point into the ServHelper DLL loader C:\Windows\branding\mediasrv.png.</p><p>The loader DLL mediasrv.png is packed with UPX and employs a simple XOR decryption of strings that are dynamically constructed in memory. The loader is a service DLL with the ServiceMain function containing the ability to load the main ServHelper DLL module from C:\Windows\branding\mediasvc.png.</p><p>The mediasrv.png contains the usual exports for a svchost-based service DLL, such as ServiceMain, SvcHostPushServiceGlobals and an additional export decra, which seems to be used for decryption. SvcHostPushServiceGlobals seems to be used to load the standard Remote Desktop DLL termserv.dll and loads an RDPWrapper configuration from the dropped file C:\Windows\branding\wupsvc.jpg. Some <a href="https://www.binarydefense.com/an-updated-servhelper-tunnel-variant/">earlier analyses</a> indicate that the first stage of the ServHelper DLL loader is in fact a modified variant of RDPWrapper which allows for remote concurrent RDP sessions on the host. This would also explain the loading of the RDPWrap configuration file. <br></p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-3zJwptorhNQ/YRJFytpmtuI/AAAAAAAAAxI/V7A-0NTPaHc6WjUvDrslAvCN4M3ud72dwCLcBGAsYHQ/s16000/image11.png" class="kg-image" alt loading="lazy"></figure><p><i>Modified RDPWrap DLL first decrypts the name and then loads the main ServHelper module.</i> <br> The main ServHelper RAT module is written in Delphi using the compilation to native code. While searching for other related samples, we discovered another dropper written in GoLang.</p><h4 id="golang-go-clr-dropper-the-dropper-uses-the-go-clr-loader-which-allows-a-golang-executable-to-load-a-net-assembly-from-memory-or-from-the-disk-the-dropper-extracts-a-base64-encoded-buffer-from-its-own-rdate-section-decompresses-the-buffer-using-gzip-deflate-and-loads-the-net-servhelper-dropper-into-memory-to-execute-it-the-base64-buffer-is-corrupt-to-avoid-automatic-extraction-by-anti-malware-software-it-automatically-replaces-itself-before-it-is-base64-decoded-and-decompressed-">GoLang go-clr dropper The dropper uses the <a href="https://github.com/ropnop/go-clr">go-clr</a> loader, which allows a GoLang executable to load a <a href="https://blog.ropnop.com/hosting-clr-in-golang/">.NET assembly from memory</a> or from the disk. The dropper extracts a base64-encoded buffer from its own rdate section, decompresses the buffer using gzip deflate, and loads the .NET ServHelper dropper into memory to execute it. The base64 buffer is corrupt to avoid automatic extraction by anti-malware software. It automatically replaces itself <a href="https://medium.com/walmartglobaltech/ta505-adds-golang-crypter-for-delivering-miners-and-servhelper-af70b26a6e56">before it is base64 decoded</a> and decompressed.<br></h4><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-REDOSlzc-h4/YRJGWxbpoyI/AAAAAAAAAxY/o5CVJLcoNEEnWMzkKVrJVi9wxNL6ZAQdACLcBGAsYHQ/s16000/image22.png" class="kg-image" alt loading="lazy"></figure><p><i>Gzip buffer is corrected before it is base64 decoded into a binary buffer containing a .NET assembly.</i> <br>The loaded assembly is a usual dropper for ServHelper. It extracts the PowerShell loader components for the ServHelper into the temporary files folder and runs ready.ps1, which launches the infection as described above.</p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-3ILuezi3ynA/YRJGoCYyoEI/AAAAAAAAAxg/-EK1jRNhKl0iKJ92NrHIEpMCVjy1uEutQCLcBGAsYHQ/s16000/image14.png" class="kg-image" alt loading="lazy"></figure><p><br></p><p><i>.NET ServHelper dropper drops and executes the PowerShell loader components.</i> <br> As with every investigation, we tried to find similar files and their locations on the internet and found that GoLang droppers are downloaded and installed as a consequence of running a group of MSI installer packages signed with a valid certificate and not detected by any antimalware software. The installers purport to be MSI files that install various applications, including a video editor, a wiki program and Discord. <br></p><h4 id="msi-installer-downloaders-the-certificate-belongs-to-a-russian-company-sgp-geologiya-ooo-the-first-sample-has-been-submitted-to-virustotal-on-may-2-and-again-on-july-15-">MSI installer downloaders The certificate belongs to a Russian company SGP-GEOLOGIYA, OOO. A logical assumption is that the filename.exe is a downloader of the ServHelper dropper.<br></p><h4 id="raccoon-and-amadey-stealers-entering-the-servhelper-play-the-sample-filename-exe-sha256-7516b2271e4a887156d52f661cdfc561fded62338a72b56f50bf188c2f5f222a-downloaded-by-the-msi-installer-is-a-variant-of-the-raccoon-information-stealing-malware-raccoon-is-a-stealer-that-attempts-to-find-and-exfiltrate-information-that-can-be-used-by-the-attacker-to-gain-financial-benefits-such-as-user-credentials-cookies-cryptocurrency-wallet-details-and-credit-card-numbers-">Raccoon and Amadey stealers entering the ServHelper play The sample filename.exe, (SHA256 7516b2271e4a887156d52f661cdfc561fded62338a72b56f50bf188c2f5f222a), downloaded by the MSI installer is a variant of the <a href="https://malpedia.caad.fkie.fraunhofer.de/details/win.raccoon">Raccoon information stealing</a> malware. Raccoon is a stealer that attempts to find and exfiltrate information that can be used by the attacker to gain financial benefits, such as user credentials, cookies, cryptocurrency wallet details and credit card numbers.</h4><p>The sample we analysed, referenced by the MSI installer, communicates with a C2 server hosted on 34[.]76[.]8[.]115.</p><p>Raccoon uses HTTP for communication with the C2 server. The data submitted and received is encrypted with RC4 using an encryption key hardcoded within the body of the stealer. When run for the first time, the Raccoon executable sends a POST request to the C2 server. The request contains the basic information about the bot, such as a unique bot identifier, username, configuration identifier and the format of data expected and the C2 server responds with the configuration in a format specified by the bot, usually JSON. <br></p><figure class="kg-card kg-image-card"><img src="https://blog.talosintelligence.com/content/images/-vejG3Qvjc6M/YRJIMjrI_PI/AAAAAAAAAyA/-E02A-msaZgVSzpvKgl5CcMptUKOQnY4wCLcBGAsYHQ/s16000/image12.png" class="kg-image" alt loading="lazy"></figure><p><i>The RC4 passphrase and all other Raccoon strings are encrypted with a simple XOR scheme.</i> <br> The analysed sample RC4 passphrase was "$Z2s`ten\@bE9vzR".
