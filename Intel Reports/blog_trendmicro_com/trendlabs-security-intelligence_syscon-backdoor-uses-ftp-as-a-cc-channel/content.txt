		<div>
			<p>Bots can use various methods to establish a line of communication between themselves and their command-and-control (C&amp;C) server. Usually, these are done via HTTP or other TCP/IP connections. However, we recently encountered a botnet that uses a more unusual method: an FTP server that, in effect, acts as a C&amp;C server.</p>
<p>Using an FTP server has some advantages. It is less common, and this fact may allow it to slip unnoticed by administrators and researchers. However, this also leaves the C&amp;C traffic open for monitoring by others, including security researchers. In addition, thanks to a coding mistake by the attackers, this particular backdoor does not always run the right commands.</p>
<p><b><span class="body-subhead-title">Infection Chain</span></b>The infection chain starts with a malicious document with macros. The examples of delivery documents are shown below. The documents suggest that targeted individuals may be connected to the <a href="http://www.ifrc.org/what-we-do/health/cbhfa/about-cbhfa/" target="_blank">Red Cross</a> and the <a href="http://www.who.int/pmnch/about/continuum_of_care/en/" target="_blank">World Health Organization</a>. All the documents mention North Korea as well. We detect these delivery documents as W2KM_SYSCON.A.</p>
<p style="text-align: center;"><a href="http://blog.trendmicro.com/trendlabs-security-intelligence/files/2017/10/syscon1.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2017/10/syscon1tb.png"/></a><a href="http://blog.trendmicro.com/trendlabs-security-intelligence/files/2017/10/syscon2.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2017/10/syscon2tb.png"/></a><a href="http://blog.trendmicro.com/trendlabs-security-intelligence/files/2017/10/syscon3.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2017/10/syscon3tb.png"/></a></p>
<p style="text-align: center;"><i>Figures 1-3. Delivery documents with macros (Click to enlarge)</i></p>
<p>Each document contains two long strings, with Base64 encoding using a custom alphabet. This same technique was used to deliver the <a href="https://www.theregister.co.uk/2012/12/12/russian_cyberespionage_attack/" target="_blank">Sanny malware family</a> in late 2012.</p>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2017/10/syscon4.png"/></p>
<p style="text-align: center;"><i>Figure 4. Base64 decoded function with custom alphabet highlighted</i></p>
<p>Its similarities with the earlier Sanny attack are interesting. Both attacks used relatively unusual techniques for their C&amp;C server, their structure is similar, and the encoding key is identical. Documents somehow tied to North Korea were also used. We cannot eliminate the possibility that both Sanny and this new malware family were the work of the same threat actor.</p>
<p>Decoding each Base64 string results in a <a href="https://msdn.microsoft.com/en-us/library/windows/desktop/aa367841(v=vs.85).aspx" target="_blank">cabinet</a> file. One string contains a 32-bit version of the malware; the other contains a 64-bit version. The appropriate version (based on OS version) is extracted using the <i>expand</i> command into the <i>%Temp%</i> folder, and <i>uacme.exe</i> (one of the files in the cabinet file) is executed. (We detect the malicious files in these cabinet files under the following detection names: BAT_SYSCON.A, <a href="https://www.trendmicro.com/vinfo/tmr/?/us/threat-encyclopedia/malware/BKDR_SYSCON.A">BKDR_SYSCON.A</a>, and TROJ_SYSCON.A.)</p>
<p style="text-align: center;"><a href="http://blog.trendmicro.com/trendlabs-security-intelligence/files/2017/10/syscon5.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2017/10/syscon5tb.png"/></a></p>
<p style="text-align: center;"><i>Figure 5. Command to extract cabinet archive and execute its contents (Click to enlarge)</i></p>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2017/10/syscon6.png"/></p>
<p style="text-align: center;"><i>Figure 6. Listing of all files in cabinet archive</i></p>
<p><b><span class="body-subhead-title">UAC bypass and installation</span></b><i style="">Uacme.exe</i>, as the name suggests, determines the operating system version. Based on that information, it either directly executes <i style="">install.bat</i> (for older Windows versions) or injects <i style="">dummy.dll </i>into the <i style="">taskhost(ex)</i> process, which attempts to execute <i style="">install.bat</i> without a <a href="https://msdn.microsoft.com/en-us/library/cc505883.aspx" target="_blank">UAC prompt</a> appearing.</p>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2017/10/syscon7.png"/></p>
<p style="text-align: center;"><i>Figure 7. uacme.exe and UAC bypass</i></p>
<p><i>Install.bat</i> copies two files: <i>ipnet.dll</i> (the main file) and <i>ipnet.ini</i> (configuration file) into <i>%Windows%\System32</i>, configures new malicious <i>COMSysApp</i> service using the <a href="https://technet.microsoft.com/en-us/library/bb490995.aspx" target="_blank"><i>sc</i></a> command line utility, adds the service parameters into the registry, starts the malicious service, and deletes all previously created files in the <i>%Temp%</i> directory. This does two things: it sets up the backdoor’s autostart routine, and deletes some traces of its previous activity, making detection more difficult.</p>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2017/10/syscon8.png"/></p>
<p style="text-align: center;"><i>Figure 8. Service installation</i></p>
<p>Important parameters for the service configuration are “<i>type=own”</i>, which according to MSDN documentation means that it has its own <i>svchost.exe</i> process in which the <i>ipnet.dll</i> runs. The parameter “<i>start = auto</i>” starts the service every time the computer is restarted.</p>
<p><i>Ipnet.ini</i> is a text file encoded in the same way as the two cabinet files.</p>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2017/10/syscon9.png"/></p>
<p style="text-align: center;"><i>Figure 9. Configuration file</i></p>
<p>Decoding the configuration reveals a URL for the <a href="https://byet.host/" target="_blank">byethost</a> free FTP service provider, as well as a set of login credentials.</p>
<p><b><span class="body-subhead-title">The backdoor</span></b>The malware first gets the computer name, which it uses as the affected machine’s identifier. It then logs into the FTP server using the credentials in the configuration file, enters the <i>/htdocs/</i> directory, and monitors existing <i>.txt</i> file names. If the file name contains “To EVERYONE”, it means that the file should be processed by everyone. If it contains “to computer_name”, then the file should be processed (and later deleted) only by the victim computer with the matching computer_name.</p>
<p>After the backdoor processes the command, it lists all the currently running processes by calling “<i>cmd /c tasklist &gt;%ws</i>”, which is then packed, encoded and uploaded to FTP under following name pattern “<i>From %s (%02d-%02d %02d-%02d-%02d).txt</i>”, i.e. computer name followed by date and time of the task execution.</p>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2017/10/syscon10.png"/></p>
<p style="text-align: center;"><i>Figure 10. Construction of name of the uploaded file</i></p>
<p>This shows that the communication between the victim’s computer and the bot master is done via uploaded files. However, the files are generally zipped and encoded with the same custom Base64 encoding used earlier.</p>
<p>Compression of the files is done using the <a href="https://msdn.microsoft.com/en-us/library/windows/desktop/bb776890(v=vs.85).aspx" target="_blank">Shell Automation Service</a>, which is part of Windows itself. No external library is needed. The malware first creates a 22-byte long empty <i>.zip</i> file by executing the instructions below. The content of the newly created <i>.zip</i> file in hex is “<i>504b0506000000000000000000000000000000000000</i>”, which is basically an empty <i>.zip</i> archive.</p>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2017/10/syscon11.png"/></p>
<p style="text-align: center;"><i>Figure 11. Instructions creating empty .zip file</i></p>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2017/10/syscon12.png"/></p>
<p style="text-align: center;"><i>Figure 12. Empty <i>.zip</i> file opened in Explorer</i></p>
<p>The malware then uses the <i>Folder.CopyHere</i> method to copy files into the empty <i>.zip</i> archive, so these files are compressed by the operating system. According to MSDN documentation, these are the flags used while copying:</p>
<ul>
<li><span class="rte-red-bullet">0x1000 = Only operate in the local directory. Do not operate recursively into subdirectories.</span></li>
<li><span class="rte-red-bullet">0x0400 = Do not display a user interface if an error occurs.</span></li>
<li><span class="rte-red-bullet">0x0010 = Respond with “Yes to All” for any dialog box that is displayed.</span></li>
<li><span class="rte-red-bullet">0x0004 = Do not display a progress dialog box.</span></li>
</ul>
<p> </p>
<p style="text-align: center;"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2017/10/syscon13.png"/></p>
<p style="text-align: center;"><i>Figure 13. Flags used by Folder.CopyHere  method</i></p>
<p>These flags ensure that no dialog box appears and no errors are shown to the victim. All these operations remain hidden in the background. After compression is finished, the compressed files are encoded with Base64 and uploaded to the FTP server. For downloading files from FTP server, the reverse operations would be performed by the threat actor.</p>

<p><b><span class="bs-modal"><span class="body-subhead-title">Mistakes in malware code</span></span></b>The command processing loop contains what appears to be a typo or mistake. The malware treats the command-and-control commands as strings in <a href="https://msdn.microsoft.com/en-us/library/z207t55f.aspx" target="_blank">wide character</a> format. The second <a href="https://msdn.microsoft.com/en-us/library/windows/desktop/ms647550(v=vs.85).aspx" target="_blank"><i>wsprintfA</i></a> function in the snippet below has the first format parameter <i>“%s</i>”, which should actually be <i>“%ws”.</i> Because of this, only the first character of <i>lpExistingFileName</i> is propagated into <i>&amp;CommandLine</i>, preventing the process from executing because of the incorrect file name.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2017/10/syscon6.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2017/10/syscon18tb.png"/></a></p>
<p style="text-align: center;"><i>Figure 15. Code snippet with a typo (Click to enlarge)</i></p>
<p><b><span class="body-subhead-title">Conclusion</span></b>It is interesting to see something atypical, like C&amp;C communication via FTP. While the malware authors probably used this method in an attempt to avoid security solutions inspection and/or blocking, they may not have realized this would make it very easy to monitor their actions and victims’ data.</p>
<p>IT administrators should be aware that connections to external FTP servers can signify not just data extraction, but C&amp;C activity as well. Either way, if this kind of network activity is not necessary for business functions, blocking it should be considered.</p>
