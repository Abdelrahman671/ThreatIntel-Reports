			<h1 class="entry-title">Bazar, No Ryuk?</h1>
		<div class="entry-meta">
			<span class="posted-on"><a href="https://thedfirreport.com/2021/01/31/bazar-no-ryuk/" rel="bookmark"><time class="entry-date published" datetime="2021-01-31T23:58:23+00:00">January 31, 2021</time></a></span>		</div><!-- .entry-meta -->

		
	</header><!-- .entry-header -->

	<div class="entry-content">
		<h3>Intro</h3>
<p>In the fall of 2020, Bazar came to prominence when several campaigns delivered Ryuk ransomware. While Bazar appeared to drop-off in December, new campaigns have sprung up recently, using similar TTP&#8217;s.</p>
<p>In this case, we will describe how the threat actor went from a DocuSign themed, malicious document, to domain wide compromise, using Bazar aka KEGTAP and Cobalt Strike.</p>
<h3>Case Summary</h3>
<p>This investigation began as many do, with a malicious document delivered via email. The email and accompanying Excel file purported to be a DocuSign request, which entices the user to enable macros. This lead to Bazar being dropped on the system, which created a run key for persistence.</p>
<p>On the first day, after the initial activity, nothing else was seen. On the second day, we observed DNS requests to .bazar domain names (the hallmark of the Bazar malware family). The malware also executed some basic nltest domain discovery, and a short ping to a Cobalt Strike server, but no additional activity was observed.</p>
<p>On the third day, more communication was observed between the Bazar and Cobalt Strike infrastructure, but again, no downloads or follow-on activity was observed.</p>
<p>On the fourth day, Bazar pulled down a Cobalt Strike Beacon in the form of a DLL, which was executed via rundll32 and injected into various system processes. One of those processes injected into, was dllhost, which then ran various PowerSploit commands for discovery activity and dumped credentials from lsass. Shortly thereafter, the threat actors began moving laterally using multiple techniques, such as:</p>
<p style="padding-left: 40px;">Pass the Hash</p>
<p style="padding-left: 40px;">SMB executable transfer and exec</p>
<p style="padding-left: 40px;">RDP</p>
<p style="padding-left: 40px;">Remote service execution</p>
<p>The threat actors then continued pivoting and collecting more information about the environment. About an hour after beginning their lateral movement, they had compromised a domain controller. On that domain controller, they executed AdFind, and then dropped a custom PowerShell script named Get-DataInfo.ps1. This script looks for all active machines and queries installed software, i.e., backup software, security software, etc. We first saw this script about a year ago when threat actors deployed Ryuk ransomware across a domain. Other <a href="https://blog.talosintelligence.com/2020/04/IR-quarterly-threat-report-spring-2020.html" target="_blank" rel="noopener">public data</a> has also linked this TTP to Ryuk threat actors.</p>
<p>However, in this case, about 15 minutes after running the script, the threat actor dropped their access and left the environment. We do not know what caused them to leave, but we have some ideas. Based on the TTP&#8217;s of this intrusion, we assess, with medium to high confidence, that Ryuk would have been the likely ransomware deployed. Total time in the environment was around 4 days.</p>

If you&#8217;re interested in pricing or interested in a trial please use <a href="https://thedfirreport.com/contact/" target="_blank" rel="noopener">Contact Us</a> to get in touch.</em></p>
<h3>Timeline</h3>
<p><a href="https://thedfirreport.com/wp-content/uploads/2021/01/Bazar-No-Ryuk-5.png"><img fetchpriority="high" decoding="async" class="alignnone size-full wp-image-2258" src="https://thedfirreport.com/wp-content/uploads/2021/01/Bazar-No-Ryuk-5.png" alt="" width="800" height="2135" srcset="https://thedfirreport.com/wp-content/uploads/2021/01/Bazar-No-Ryuk-5.png 800w, https://thedfirreport.com/wp-content/uploads/2021/01/Bazar-No-Ryuk-5-112x300.png 112w, https://thedfirreport.com/wp-content/uploads/2021/01/Bazar-No-Ryuk-5-384x1024.png 384w, https://thedfirreport.com/wp-content/uploads/2021/01/Bazar-No-Ryuk-5-768x2050.png 768w, https://thedfirreport.com/wp-content/uploads/2021/01/Bazar-No-Ryuk-5-576x1536.png 576w, https://thedfirreport.com/wp-content/uploads/2021/01/Bazar-No-Ryuk-5-767x2048.png 767w" sizes="(max-width: 800px) 100vw, 800px" /></a></p>
<h3>MITRE ATT&amp;CK</h3>
<h3><a>Initial Access</a></h3>
<p>Initial access to the environment was via a malicious email that entices a user to download an Excel document with macros using a DocuSign social engineering theme.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2021/01/bazar-1013-xlsm.png"><img decoding="async" class="alignnone size-full wp-image-2083" src="https://thedfirreport.com/wp-content/uploads/2021/01/bazar-1013-xlsm.png" alt="" width="1015" height="705" srcset="https://thedfirreport.com/wp-content/uploads/2021/01/bazar-1013-xlsm.png 1015w, https://thedfirreport.com/wp-content/uploads/2021/01/bazar-1013-xlsm-300x208.png 300w, https://thedfirreport.com/wp-content/uploads/2021/01/bazar-1013-xlsm-768x533.png 768w" sizes="(max-width: 1015px) 100vw, 1015px" /></a></p>
<h3><a>Execution</a></h3>
<p>The Excel document required the user to enable content to execute. The embedded macro in the file was using an Excel 4.0 macro, which at time of execution had a detection rate of 1/63 in <a href="https://www.virustotal.com/gui/file/571c32689719ba00f0d60918ae70a8edc185435ce3201413c75da1dbd269f88c/detection">Virustotal</a>.</p>
