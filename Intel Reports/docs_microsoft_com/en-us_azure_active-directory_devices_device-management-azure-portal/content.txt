											<h2 id="ms--in-this-article" class="title is-6 margin-block-xs">In this article</h2>
										</nav><!-- <content> --><p>Microsoft Entra ID provides a central place to manage device identities and monitor related event information.</p>
<p><a href="media/manage-device-identities/devices-azure-portal.png#lightbox" data-linktype="relative-path"><img src="media/manage-device-identities/devices-azure-portal.png" alt="Screenshot that shows the devices overview." data-linktype="relative-path"></a></p>
<p>You can access the devices overview by completing these steps:</p>
<ol>
<li>Sign in to the <a href="https://entra.microsoft.com" data-linktype="external">Microsoft Entra admin center</a> as a user with at least <a href="../../fundamentals/users-default-permissions" data-linktype="relative-path">default user permissions</a>.</li>
<li>Go to <strong>Identity</strong> &gt; <strong>Devices</strong> &gt; <strong>Overview</strong>.</li>
</ol>
<p>In the devices overview, you can view the number of total devices, stale devices, noncompliant devices, and unmanaged devices. It provides links to Intune, Conditional Access, BitLocker keys, and basic monitoring. Other features like Conditional Access and Microsoft Intune require additional role assignments</p>
<p>Device counts on the overview page don't update in real time. Changes should be reflected every few hours.</p>
<p>From there, you can go to <strong>All devices</strong> to:</p>
<ul>
<li>Identify devices, including:
<ul>
<li>Devices joined or registered in Microsoft Entra ID.</li>
<li>Devices deployed via <a href="/en-us/autopilot/windows-autopilot" data-linktype="absolute-path">Windows Autopilot</a>.</li>
<li>Printers that use <a href="/en-us/universal-print/fundamentals/universal-print-getting-started" data-linktype="absolute-path">Universal Print</a>.</li>
</ul>
</li>
<li>Complete device identity management tasks like enable, disable, delete, and manage.
<ul>
<li>The management options for <a href="/en-us/universal-print/fundamentals/" data-linktype="absolute-path">Printers</a> and <a href="/en-us/autopilot/windows-autopilot" data-linktype="absolute-path">Windows Autopilot</a> are limited in Microsoft Entra ID. These devices must be managed from their respective admin interfaces.</li>
</ul>
</li>
<li>Configure your device identity settings.</li>
<li>Enable or disable enterprise state roaming.</li>
<li>Review device-related audit logs.</li>
<li>Download devices.</li>
</ul>
<p><a href="media/manage-device-identities/all-devices-azure-portal.png#lightbox" data-linktype="relative-path"><img src="media/manage-device-identities/all-devices-azure-portal.png" alt="Screenshot that shows the All devices view." data-linktype="relative-path"></a></p>
<div class="TIP">
<p>Tip</p>
<ul>
<li><p>Microsoft Entra hybrid joined Windows 10 or newer devices don't have an owner unless the primary user is set in Microsoft Intune. If you're looking for a device by owner and don't find it, search by the device ID.</p>
</li>
<li><p>If you see a device that's <strong>Microsoft Entra hybrid joined</strong> with a state of <strong>Pending</strong> in the <strong>Registered</strong> column, the device has been synchronized from Microsoft Entra Connect and is waiting to complete registration from the client. See <a href="hybrid-join-plan" data-linktype="relative-path">How to plan your Microsoft Entra hybrid join implementation</a>. For more information, see <a href="faq" data-linktype="relative-path">Device management frequently asked questions</a>.</p>
</li>
<li><p>For some iOS devices, device names that contain apostrophes can use different characters that look like apostrophes. So searching for such devices is a little tricky. If don't see correct search results, be sure the search string contains the matching apostrophe character.</p>
</li>
</ul>
</div>
<h2 id="manage-an-intune-device">Manage an Intune device</h2>
<p>If you have rights to manage devices in Intune, you can manage devices for which mobile device management is listed as <strong>Microsoft Intune</strong>. If the device isn't enrolled with Microsoft Intune, the <strong>Manage</strong> option isn't available.</p>
<h2 id="enable-or-disable-a-microsoft-entra-device">Enable or disable a Microsoft Entra device</h2>
<p>There are two ways to enable or disable devices:</p>
<ul>
<li>The toolbar on the <strong>All devices</strong> page, after you select one or more devices.</li>
<li>The toolbar, after you drill down for a specific device.</li>
</ul>
<div class="IMPORTANT">
<p>Important</p>
<ul>
<li>You must be a Intune Administrator or Cloud Device Administrator to enable or disable a device.</li>
<li>Disabling a device prevents it from authenticating via Microsoft Entra ID. This prevents it from accessing your Microsoft Entra resources that are protected by device-based Conditional Access and from using Windows Hello for Business credentials.</li>
<li>Disabling a device revokes the Primary Refresh Token (PRT) and any refresh tokens on the device.</li>
<li>Printers can't be enabled or disabled in Microsoft Entra ID.</li>
</ul>
</div>
<h2 id="delete-a-microsoft-entra-device">Delete a Microsoft Entra device</h2>
<p>There are two ways to delete a device:</p>
<ul>
<li>The toolbar on the <strong>All devices</strong> page, after you select one or more devices.</li>
<li>The toolbar, after you drill down for a specific device.</li>
</ul>
<div class="IMPORTANT">
<p>Important</p>
<ul>
<li>You must be a Cloud Device Administrator, Intune Administrator or Windows 365 Administrator to delete a device.</li>
<li>Printers can't be deleted before they are deleted from Universal Print.</li>
<li>Windows Autopilot devices can't be deleted before they are deleted from Intune.</li>
<li>Deleting a device:
<ul>
<li>Prevents it from accessing your Microsoft Entra resources.</li>
<li>Removes all details attached to the device. For example, BitLocker keys for Windows devices.</li>
<li>Is a nonrecoverable activity. We don't recommend it unless it's required.</li>
</ul>
</li>
</ul>
</div>
<p>If a device is managed in another management authority, like Microsoft Intune, be sure it's wiped or retired before you delete it. See <a href="manage-stale-devices" data-linktype="relative-path">How to manage stale devices</a> before you delete a device.</p>
<h2 id="view-or-copy-a-device-id">View or copy a device ID</h2>
<p>You can use a device ID to verify the device ID details on the device or to troubleshoot via PowerShell. To access the copy option, select the device.</p>
<p><img src="media/manage-device-identities/device-details.png" alt="Screenshot that shows a device ID and the copy button." data-linktype="relative-path"></p>
<h2 id="view-or-copy-bitlocker-keys">View or copy BitLocker keys</h2>
<p>You can view and copy BitLocker keys to allow users to recover encrypted drives. These keys are available only for Windows devices that are encrypted and store their keys in Microsoft Entra ID. You can find these keys when you view a device's details by selecting <strong>Show Recovery Key</strong>. Selecting <strong>Show Recovery Key</strong> generates an audit log entry, which you can find in the <code>KeyManagement</code> category.</p>
<p><img src="media/manage-device-identities/show-bitlocker-key.png" alt="Screenshot that shows how to view BitLocker keys." data-linktype="relative-path"></p>
<p>To view or copy BitLocker keys, you need to be the owner of the device or have one of these roles:</p>
<ul>
<li><a href="../role-based-access-control/permissions-reference#cloud-device-administrator" data-linktype="relative-path">Cloud Device Administrator</a></li>
<li><a href="../role-based-access-control/permissions-reference#helpdesk-administrator" data-linktype="relative-path">Helpdesk Administrator</a></li>
<li><a href="../role-based-access-control/permissions-reference#intune-administrator" data-linktype="relative-path">Intune Administrator</a></li>
<li><a href="../role-based-access-control/permissions-reference#security-administrator" data-linktype="relative-path">Security Administrator</a></li>
<li><a href="../role-based-access-control/permissions-reference#security-reader" data-linktype="relative-path">Security Reader</a></li>
</ul>
<div class="NOTE">
<p>Note</p>
<p>When devices that utilize <a href="/en-us/mem/autopilot/windows-autopilot" data-linktype="absolute-path">Windows Autopilot</a> are reused, <strong>and there is a new device owner</strong>, that new device owner must contact an administrator to acquire the BitLocker recovery key for that device. Custom role or administrative unit scoped administrators will lose access to BitLocker recovery keys for those devices that have undergone device ownership changes. These scoped administrators will need to contact a non-scoped administrator for the recovery keys. For more information, see the article <a href="/en-us/mem/intune/remote-actions/find-primary-user#change-a-devices-primary-user" data-linktype="absolute-path">Find the primary user of an Intune device</a>.</p>
</div>
<h2 id="view-and-filter-your-devices">View and filter your devices</h2>
<p>You can filter the device list by these attributes:</p>
<ul>
<li>Enabled state</li>
<li>Compliant state</li>
<li>Join type (Microsoft Entra joined, Microsoft Entra hybrid joined, Microsoft Entra registered)</li>
<li>Activity timestamp</li>
<li>OS type and OS version
<ul>
<li>Windows is displayed for Windows 11 and Windows 10 devices (with KB5006738).</li>
<li>Windows Server is displayed for <a href="/en-us/mem/intune/protect/mde-security-integration#supported-platforms" data-linktype="absolute-path">supported versions managed with Microsoft Defender for Endpoint</a>.</li>
</ul>
</li>
<li>Device type (printer, secure VM, shared device, registered device)</li>
<li>MDM</li>
<li>Autopilot</li>
<li>Extension attributes</li>
<li>Administrative unit</li>
<li>Owner</li>
</ul>
<h2 id="download-devices">Download devices</h2>
<p>Cloud Device Administrators and Intune Administrators can use the <strong>Download devices</strong> option to export a CSV file that lists devices. You can apply filters to determine which devices to list. If you don't apply any filters, all devices are listed. An export task might run for as long as an hour, depending on your selections. If the export task exceeds 1 hour, it fails, and no file is output.</p>
<p>The exported list includes these device identity attributes:</p>
<p><code>displayName,accountEnabled,operatingSystem,operatingSystemVersion,joinType (trustType),registeredOwners,userNames,mdmDisplayName,isCompliant,registrationTime,approximateLastSignInDateTime,deviceId,isManaged,objectId,profileType,systemLabels,model</code></p>
<p>The following filters can be applied for the export task:</p>
<ul>
<li>Enabled state</li>
<li>Compliant state</li>
<li>Join type</li>
<li>Activity timestamp</li>
<li>OS type</li>
<li>Device type</li>
</ul>
<h2 id="configure-device-settings">Configure device settings</h2>
<p>If you want to manage device identities by using the Microsoft Entra admin center, the devices need to be either <a href="overview" data-linktype="relative-path">registered or joined</a> to Microsoft Entra ID. As an administrator, you can control the process of registering and joining devices by configuring the following device settings.</p>
<p>You must be assigned one of the following roles to read or modify device settings:</p>
<ul>
<li><a href="../role-based-access-control/permissions-reference#cloud-device-administrator" data-linktype="relative-path">Cloud Device Administrator</a> (read and modify)</li>
<li><a href="../role-based-access-control/permissions-reference#intune-administrator" data-linktype="relative-path">Intune Administrator</a> (read only)</li>
<li><a href="../role-based-access-control/permissions-reference#windows-365-administrator" data-linktype="relative-path">Windows 365 Administrator</a> (read only)</li>
</ul>
<p><img src="media/manage-device-identities/device-settings-azure-portal.png" alt="Screenshot that shows device settings related to Microsoft Entra ID." data-linktype="relative-path"></p>
<ul>
<li><p><strong>Users may join devices to Microsoft Entra ID</strong>: This setting enables you to select the users who can register their devices as Microsoft Entra joined devices. The default is <strong>All</strong>.</p>
<div class="NOTE">
<p>Note</p>
<p>The <strong>Users may join devices to Microsoft Entra ID</strong> setting is applicable only to Microsoft Entra join on Windows 10 or newer. This setting doesn't apply to Microsoft Entra hybrid joined devices, <a href="howto-vm-sign-in-azure-ad-windows#enable-azure-ad-login-for-a-windows-vm-in-azure" data-linktype="relative-path">Microsoft Entra joined VMs in Azure</a>, or Microsoft Entra joined devices that use <a href="/en-us/autopilot/self-deploying" data-linktype="absolute-path">Windows Autopilot self-deployment mode</a> because these methods work in a userless context.</p>
</div>
</li>
<li><p><strong>Users may register their devices with Microsoft Entra ID</strong>: You need to configure this setting to allow users to register Windows 10 or newer personal, iOS, Android, and macOS devices with Microsoft Entra ID. If you select <strong>None</strong>, devices aren't allowed to register with Microsoft Entra ID. Enrollment with Microsoft Intune or mobile device management for Microsoft 365 requires registration. If you've configured either of these services, <strong>ALL</strong> is selected, and <strong>NONE</strong> is unavailable.</p>
</li>
<li><p><strong>Require multifactor authentication to register or join devices with Microsoft Entra ID</strong>:</p>
<ul>
<li>We recommend organizations use the <a href="../conditional-access/concept-conditional-access-cloud-apps#user-actions" data-linktype="relative-path">Register or join devices user action</a> in Conditional Access to enforce multifactor authentication. You must configure this toggle to <strong>No</strong> if you use a <a href="../conditional-access/policy-all-users-device-registration" data-linktype="relative-path">Conditional Access policy to require multifactor authentication</a>.</li>
<li>This setting allows you to specify whether users are required to provide another authentication factor to join or register their devices to Microsoft Entra ID. The default is <strong>No</strong>. We recommend that you require multifactor authentication when a device is registered or joined. Before you enable multifactor authentication for this service, you must ensure that multifactor authentication is configured for users that register their devices. For more information on Microsoft Entra multifactor authentication services, see <a href="../authentication/concept-mfa-howitworks" data-linktype="relative-path">getting started with Microsoft Entra multifactor authentication</a>. This setting might not work with third-party identity providers.</li>
</ul>
<div class="NOTE">
<p>Note</p>
<p>The <strong>Require multifactor authentication to register or join devices with Microsoft Entra ID</strong> setting applies to devices that are either Microsoft Entra joined (with some exceptions) or Microsoft Entra registered. This setting doesn't apply to Microsoft Entra hybrid joined devices, <a href="howto-vm-sign-in-azure-ad-windows#enable-azure-ad-login-for-a-windows-vm-in-azure" data-linktype="relative-path">Microsoft Entra joined VMs in Azure</a>, or Microsoft Entra joined devices that use <a href="/en-us/autopilot/self-deploying" data-linktype="absolute-path">Windows Autopilot self-deployment mode</a>.</p>
</div>
</li>
<li><p><strong>Maximum number of devices</strong>: This setting enables you to select the maximum number of Microsoft Entra joined or Microsoft Entra registered devices that a user can have in Microsoft Entra ID. If users reach this limit, they can't add more devices until one or more of the existing devices are removed. The default value is <strong>50</strong>. You can increase the value up to 100. If you enter a value above 100, Microsoft Entra ID sets it to 100. You can also use <strong>Unlimited</strong> to enforce no limit other than existing quota limits.</p>
<div class="NOTE">
<p>Note</p>
<p>The <strong>Maximum number of devices</strong> setting applies to devices that are either Microsoft Entra joined or Microsoft Entra registered. This setting doesn't apply to Microsoft Entra hybrid joined devices.</p>
</div>
</li>
<li><p><strong>Manage Additional local administrators on Microsoft Entra joined devices</strong>: This setting allows you to select the users who are granted local administrator rights on a device. These users are added to the Device Administrators role in Microsoft Entra ID.</p>
</li>
<li><p><strong>Enable Microsoft Entra Local Administrator Password Solution (LAPS) (preview)</strong>: LAPS is the management of local account passwords on Windows devices. LAPS provides a solution to securely manage and retrieve the built-in local admin password. With cloud version of LAPS, customers can enable storing and rotation of local admin passwords for both Microsoft Entra ID and Microsoft Entra hybrid join devices. To learn how to manage LAPS in Microsoft Entra ID, see <a href="howto-manage-local-admin-passwords" data-linktype="relative-path">the overview article</a>.</p>
</li>
<li><p><strong>Restrict non-admin users from recovering the BitLocker key(s) for their owned devices</strong>: Admins can block self-service BitLocker key access to the registered owner of the device. Default users without the BitLocker read permission are unable to view or copy their BitLocker key(s) for their owned devices. You must be at least a <a href="../role-based-access-control/permissions-reference#privileged-role-administrator" data-linktype="relative-path">Privileged Role Administrator</a> to update this setting.</p>
</li>
<li><p><strong>Enterprise State Roaming</strong>: For information about this setting, see <a href="enterprise-state-roaming-enable" data-linktype="relative-path">the overview article</a>.</p>
</li>
</ul>
<h2 id="audit-logs">Audit logs</h2>
<p>Device activities are visible in the activity logs. These logs include activities triggered by the device registration service and by users:</p>
<ul>
<li>Device creation and adding owners/users on the device</li>
<li>Changes to device settings</li>
<li>Device operations like deleting or updating a device</li>
<li>Bulk operations like downloading all devices</li>
</ul>
<div class="NOTE">
<p>Note</p>
<p>When performing bulk operations, such as import or create, you may encounter a problem if the bulk operation does not complete within the hour. To work around this issue, we recommend splitting the number of records processed per batch. For example, before starting an export you could limit the result set by filtering on a group type or user name to reduce the size of the results. By refining your filters, essentially you are limiting the data returned by the bulk operation. For more information, see <a href="../../fundamentals/bulk-operations-service-limitations" data-linktype="relative-path">Bulk operations service limitations</a>.</p>
</div>
<p>The entry point to the auditing data is <strong>Audit logs</strong> in the <strong>Activity</strong> section of the <strong>Devices</strong> page.</p>
<p>The audit log has a default list view that shows:</p>
<ul>
<li>The date and time of the occurrence.</li>
<li>The targets.</li>
<li>The initiator/actor of an activity.</li>
<li>The activity.</li>
</ul>
<p><img src="media/manage-device-identities/63.png" alt="Screenshot that shows a table in the Activity section of the Devices page. The table shows the date, target, actor, and activity for four audit logs." data-linktype="relative-path">
</p>
<p>You can customize the list view by selecting <strong>Columns</strong> in the toolbar:</p>
<p><img src="media/manage-device-identities/64.png" alt="Screenshot that shows the toolbar of the Devices page." data-linktype="relative-path">
</p>
<p>To reduce the reported data to a level that works for you, you can filter it by using these fields:</p>
<ul>
<li><strong>Category</strong></li>
<li><strong>Activity Resource Type</strong></li>
<li><strong>Activity</strong></li>
<li><strong>Date Range</strong></li>
<li><strong>Target</strong></li>
<li><strong>Initiated By (Actor)</strong></li>
</ul>
<p>You can also search for specific entries.</p>
<p><img src="media/manage-device-identities/65.png" alt="Screenshot that shows audit data filtering controls." data-linktype="relative-path">
</p>
