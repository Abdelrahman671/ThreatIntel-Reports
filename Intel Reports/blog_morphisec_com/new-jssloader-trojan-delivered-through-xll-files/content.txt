<p>Morphisec Labs has observed a new wave of <a href="https://blog.morphisec.com/the-evolution-of-the-fin7-jssloader"><span>JSSLoader</span></a> infections this year. We’ve tracked JSSLoader activity since December 2020 and published a thorough report on the Russian criminal hacking group FIN7’s JSSLoader: <a href="https://www.morphisec.com/hubfs/eBooks_and_Whitepapers/FIN7%20JSSLOADER%20FINAL%20WEB.pdf"><span>The Evolution of the FIN7 JSSLoader</span></a>. JSSLoader is a small, very capable .NET remote access trojan (RAT). Its capabilities include data exfiltration, persistence, auto-updating, additional payload delivery, and more.&nbsp;</p>
<p>Attackers are now using .XLL files to deliver a new, obfuscated version of JSSLoader. We explain how this new malware variant utilizes the Excel add-ins feature to load the malware and inspect the changes inside.</p>
<p><!--more--></p>
<h2>Infection Chain</h2>
<p style="text-align: center; font-size: 14px;"><img src="https://blog.morphisec.com/hs-fs/hubfs/From%20.xll%20file%20to%20JSSLoader.png?width=712&amp;name=From%20.xll%20file%20to%20JSSLoader.png" alt="From .xll file to JSSLoader" width="712" loading="lazy" style="width: 712px;" srcset="https://blog.morphisec.com/hs-fs/hubfs/From%20.xll%20file%20to%20JSSLoader.png?width=356&amp;name=From%20.xll%20file%20to%20JSSLoader.png 356w, https://blog.morphisec.com/hs-fs/hubfs/From%20.xll%20file%20to%20JSSLoader.png?width=712&amp;name=From%20.xll%20file%20to%20JSSLoader.png 712w, https://blog.morphisec.com/hs-fs/hubfs/From%20.xll%20file%20to%20JSSLoader.png?width=1068&amp;name=From%20.xll%20file%20to%20JSSLoader.png 1068w, https://blog.morphisec.com/hs-fs/hubfs/From%20.xll%20file%20to%20JSSLoader.png?width=1424&amp;name=From%20.xll%20file%20to%20JSSLoader.png 1424w, https://blog.morphisec.com/hs-fs/hubfs/From%20.xll%20file%20to%20JSSLoader.png?width=1780&amp;name=From%20.xll%20file%20to%20JSSLoader.png 1780w, https://blog.morphisec.com/hs-fs/hubfs/From%20.xll%20file%20to%20JSSLoader.png?width=2136&amp;name=From%20.xll%20file%20to%20JSSLoader.png 2136w" sizes="(max-width: 712px) 100vw, 712px">Figure 1: From .xll file to JSSLoader</p>
<p>This infection chain is similar to other XLL infections. The victim receives a malicious attachment, either an XLM or XLL file, inside an email. Once the attachment is downloaded and executed, Excel loads and executes the malicious code inside the .xll file, which then downloads the payload from a remote server. The payload is a new, similar variant of JSSLoader.</p>
<h2 style="font-size: 28px;">XLL Excel Add-in</h2>
<p>The first stage of the malware responsible for downloading JSSLoader into an infected machine uses an Excel add-in file, denoted by an XLL file extension. Because the file isn’t signed, a popup displays for the user before executing:</p>
<p style="text-align: center;"><img src="https://blog.morphisec.com/hs-fs/hubfs/Microsoft%20security%20popup%20.png?width=412&amp;name=Microsoft%20security%20popup%20.png" alt="Microsoft security popup " width="412" loading="lazy" style="width: 412px; margin-left: auto; margin-right: auto; display: block;" srcset="https://blog.morphisec.com/hs-fs/hubfs/Microsoft%20security%20popup%20.png?width=206&amp;name=Microsoft%20security%20popup%20.png 206w, https://blog.morphisec.com/hs-fs/hubfs/Microsoft%20security%20popup%20.png?width=412&amp;name=Microsoft%20security%20popup%20.png 412w, https://blog.morphisec.com/hs-fs/hubfs/Microsoft%20security%20popup%20.png?width=618&amp;name=Microsoft%20security%20popup%20.png 618w, https://blog.morphisec.com/hs-fs/hubfs/Microsoft%20security%20popup%20.png?width=824&amp;name=Microsoft%20security%20popup%20.png 824w, https://blog.morphisec.com/hs-fs/hubfs/Microsoft%20security%20popup%20.png?width=1030&amp;name=Microsoft%20security%20popup%20.png 1030w, https://blog.morphisec.com/hs-fs/hubfs/Microsoft%20security%20popup%20.png?width=1236&amp;name=Microsoft%20security%20popup%20.png 1236w" sizes="(max-width: 412px) 100vw, 412px"><span style="font-size: 14px;">Figure 2: Microsoft security popup</span></p>
<p>Each XLL file must implement and export the xlAutoOpen function. This function is called by Excel whenever an XLL is activated. In our case, the malicious activity is located at the end of xlAutoOpen:</p>
<p><img src="https://blog.morphisec.com/hs-fs/hubfs/Malicious%20code%20inside%20xlAutoOpen%20.png?width=241&amp;name=Malicious%20code%20inside%20xlAutoOpen%20.png" alt="Malicious code inside xlAutoOpen " width="241" loading="lazy" style="width: 241px; margin-left: auto; margin-right: auto; display: block;" srcset="https://blog.morphisec.com/hs-fs/hubfs/Malicious%20code%20inside%20xlAutoOpen%20.png?width=121&amp;name=Malicious%20code%20inside%20xlAutoOpen%20.png 121w, https://blog.morphisec.com/hs-fs/hubfs/Malicious%20code%20inside%20xlAutoOpen%20.png?width=241&amp;name=Malicious%20code%20inside%20xlAutoOpen%20.png 241w, https://blog.morphisec.com/hs-fs/hubfs/Malicious%20code%20inside%20xlAutoOpen%20.png?width=362&amp;name=Malicious%20code%20inside%20xlAutoOpen%20.png 362w, https://blog.morphisec.com/hs-fs/hubfs/Malicious%20code%20inside%20xlAutoOpen%20.png?width=482&amp;name=Malicious%20code%20inside%20xlAutoOpen%20.png 482w, https://blog.morphisec.com/hs-fs/hubfs/Malicious%20code%20inside%20xlAutoOpen%20.png?width=603&amp;name=Malicious%20code%20inside%20xlAutoOpen%20.png 603w, https://blog.morphisec.com/hs-fs/hubfs/Malicious%20code%20inside%20xlAutoOpen%20.png?width=723&amp;name=Malicious%20code%20inside%20xlAutoOpen%20.png 723w" sizes="(max-width: 241px) 100vw, 241px"></p>
<p style="text-align: center; font-size: 14px;">Figure 3: Malicious code inside xlAutoOpen</p>
<p>Before exiting from the function, the malware loads itself, the .XLL file, into memory (not relevant to the attack) and calls the mw_download_and_execute function.</p>
<p>This function is responsible for downloading the payload from a remote server. An attacker uses a different User-Agent between samples to help avoid network signature-based security solutions.</p>
<p style="text-align: center;"><img src="https://blog.morphisec.com/hs-fs/hubfs/User%20Agent%20changes%20between%20samples%20%20.png?width=731&amp;name=User%20Agent%20changes%20between%20samples%20%20.png" alt="User Agent changes between samples  " width="731" loading="lazy" style="width: 731px; margin-left: auto; margin-right: auto; display: block;" srcset="https://blog.morphisec.com/hs-fs/hubfs/User%20Agent%20changes%20between%20samples%20%20.png?width=366&amp;name=User%20Agent%20changes%20between%20samples%20%20.png 366w, https://blog.morphisec.com/hs-fs/hubfs/User%20Agent%20changes%20between%20samples%20%20.png?width=731&amp;name=User%20Agent%20changes%20between%20samples%20%20.png 731w, https://blog.morphisec.com/hs-fs/hubfs/User%20Agent%20changes%20between%20samples%20%20.png?width=1097&amp;name=User%20Agent%20changes%20between%20samples%20%20.png 1097w, https://blog.morphisec.com/hs-fs/hubfs/User%20Agent%20changes%20between%20samples%20%20.png?width=1462&amp;name=User%20Agent%20changes%20between%20samples%20%20.png 1462w, https://blog.morphisec.com/hs-fs/hubfs/User%20Agent%20changes%20between%20samples%20%20.png?width=1828&amp;name=User%20Agent%20changes%20between%20samples%20%20.png 1828w, https://blog.morphisec.com/hs-fs/hubfs/User%20Agent%20changes%20between%20samples%20%20.png?width=2193&amp;name=User%20Agent%20changes%20between%20samples%20%20.png 2193w" sizes="(max-width: 731px) 100vw, 731px"><span style="font-size: 14px;">Figure 4: User Agent changes between samples</span></p>
<p style="text-align: left; font-size: 18px;">Once downloaded, the XLL file creates a temp file with a DNA prefix using a GetTempFileNameW API call and executes it as a new process.</p>
<p style="text-align: left; font-size: 18px;"><img src="https://blog.morphisec.com/hs-fs/hubfs/Temporary%20file%20creation.png?width=751&amp;name=Temporary%20file%20creation.png" alt="Temporary file creation" width="751" loading="lazy" style="width: 751px;" srcset="https://blog.morphisec.com/hs-fs/hubfs/Temporary%20file%20creation.png?width=376&amp;name=Temporary%20file%20creation.png 376w, https://blog.morphisec.com/hs-fs/hubfs/Temporary%20file%20creation.png?width=751&amp;name=Temporary%20file%20creation.png 751w, https://blog.morphisec.com/hs-fs/hubfs/Temporary%20file%20creation.png?width=1127&amp;name=Temporary%20file%20creation.png 1127w, https://blog.morphisec.com/hs-fs/hubfs/Temporary%20file%20creation.png?width=1502&amp;name=Temporary%20file%20creation.png 1502w, https://blog.morphisec.com/hs-fs/hubfs/Temporary%20file%20creation.png?width=1878&amp;name=Temporary%20file%20creation.png 1878w, https://blog.morphisec.com/hs-fs/hubfs/Temporary%20file%20creation.png?width=2253&amp;name=Temporary%20file%20creation.png 2253w" sizes="(max-width: 751px) 100vw, 751px"></p>

<h2 style="font-size: 28px;">New Obfuscation Layer</h2>
<p style="text-align: left;">Look carefully at the dropped sample and compare it with a JSSLoader sample. They share the exact same execution flow. So, what's different? This variant introduces a new layer of string obfuscation, renaming all functions and variables names.</p>
<p style="text-align: center;"><img src="https://blog.morphisec.com/hs-fs/hubfs/Comparison%20of%20Samples.png?width=1109&amp;name=Comparison%20of%20Samples.png" alt="Comparison of Samples" width="1109" loading="lazy" style="width: 1109px;" srcset="https://blog.morphisec.com/hs-fs/hubfs/Comparison%20of%20Samples.png?width=555&amp;name=Comparison%20of%20Samples.png 555w, https://blog.morphisec.com/hs-fs/hubfs/Comparison%20of%20Samples.png?width=1109&amp;name=Comparison%20of%20Samples.png 1109w, https://blog.morphisec.com/hs-fs/hubfs/Comparison%20of%20Samples.png?width=1664&amp;name=Comparison%20of%20Samples.png 1664w, https://blog.morphisec.com/hs-fs/hubfs/Comparison%20of%20Samples.png?width=2218&amp;name=Comparison%20of%20Samples.png 2218w, https://blog.morphisec.com/hs-fs/hubfs/Comparison%20of%20Samples.png?width=2773&amp;name=Comparison%20of%20Samples.png 2773w, https://blog.morphisec.com/hs-fs/hubfs/Comparison%20of%20Samples.png?width=3327&amp;name=Comparison%20of%20Samples.png 3327w" sizes="(max-width: 1109px) 100vw, 1109px"> <span style="font-size: 14px;">Figure 6: Comparison of Samples</span></p>

<h2 style="font-size: 28px;">This New Malware Variant Evades Traditional Security</h2>
<p>Morphisec Labs will continue to monitor the evolution of JSSLoader and its delivery methods. Although it didn't present new capabilities, this new JSSLoader variant is a worry. Especially for organizations relying on their next-generation antivirus (NGAV) or endpoint detection and response (EDR) to stop it. Most NGAV and EDR solutions won’t detect day zero .XLL files hiding a JSSLoader. It can take days or weeks before signatures are deployed, all while attackers have free reign inside your network.</p>
<p>However, Morphisec’s Moving Target Defense (MTD) technology instantly stops these and other unknown and zero-day attacks. It uses system polymorphism to unpredictably hide application targets, operating system targets, and other critical asset targets from adversaries. This leads to a dramatically reduced attack surface.</p>
<p><span style="font-size: 18px;">Gartner analysts have called Moving Target Defense a “game changer.” MTD can uniquely detect and stop ransomware, zero-day, and other advanced attacks that bypass NGAV, EDR, and other defenses.
