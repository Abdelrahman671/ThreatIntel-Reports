<p style="font-size:12px;text-align:left">In August 2018, Group-IB <a href="https://www.group-ib.com/resources/threat-research/silence.html">published </a>research (available in translated form <a href="https://xz.aliyun.com/t/2722">here</a>) regarding a financially-motivated group referred to by the community as Silence. Included in this report is the mention of a proxy tool that the group uses to route traffic to and from devices on an infected network that are normally isolated from the Internet. <br><br>Although the tool is simple (and in development), it has not yet been well-documented in the public space. This may partly be because the tool is relatively rare: Group-IB describes Silence as a small group performing a limited set of activities. For researchers to obtain a copy, the Silence proxy would have to be deployed post-compromise, identified during incident response, and uploaded online. Given the rarity, some notes on the .NET version of this tool are below as a reference to future analysts.</p>



<p><u>Technical Details</u> </p>



<span id="more-103"></span>



<p style="font-size:12px">Files examined:<br>50565c4b80f41d2e7eb989cd24082aab (New)<br>8191dae4bdeda349bda38fd5791cb66f (Old)</p>



<p style="font-size:12px;text-align:left">The Silence proxy is written in .NET and known versions are packed with SmartAssembly. This can be unpacked automatically using a tool such as <a href="https://github.com/0xd4d/de4dot">De4Dot </a>to facilitate static analysis; however, this can lead to issues during debugging (with a tool such as <a href="https://github.com/0xd4d/dnSpy">DnSpy</a>) that prevent the malware from properly executing. In the two known samples, the malware&#8217;s source code is readable even without this step.<br><br>The Silence proxy performs four basic tasks:<br><br>1) The malware reads and parses its configuration<br>2) The malware enters a switch/case statement based on a configuration value<br>3) The malware opens a connection to a specified C2, the type of which depends on task 2<br>4) (Optional) The malware can perform status logging (to a local location that varies by sample)</p>



<div class="wp-block-image"><figure class="aligncenter"><img fetchpriority="high" decoding="async" width="1024" height="337" src="https://norfolkinfosec.com/wp-content/uploads/2019/02/silence_config_net-1024x337.png" alt="" class="wp-image-104" srcset="https://norfolkinfosec.com/wp-content/uploads/2019/02/silence_config_net-1024x337.png 1024w, https://norfolkinfosec.com/wp-content/uploads/2019/02/silence_config_net-300x99.png 300w, https://norfolkinfosec.com/wp-content/uploads/2019/02/silence_config_net-768x253.png 768w, https://norfolkinfosec.com/wp-content/uploads/2019/02/silence_config_net-210x69.png 210w, https://norfolkinfosec.com/wp-content/uploads/2019/02/silence_config_net-1536x506.png 1536w, https://norfolkinfosec.com/wp-content/uploads/2019/02/silence_config_net.png 1725w" sizes="(max-width: 980px) 100vw, 980px" /><figcaption>The Silence proxy configuration (right click, open in new tab)</figcaption></figure></div>



<p style="font-size:12px;text-align:left">In the sample analyzed, the configuration specifies the following:<br><br>BackConnectServerIP &#8211; C2 server IP<br>BackConnectServerPort &#8211; C2 server port<br>ConfiguredAs &#8211; Type of connection (used in the Case/Switch statement)<br>DomainName &#8211; Used for NtlmAuth case<br>PortToListen &#8211; Listening port<br>ProxyIP &#8211; Endpoint IP<br>ProxyPort &#8211; Endpoint Port<br>UserName &#8211; Used for authentication cases<br>UserPassword &#8211; Used for authentication cases<br><br>From here, the tool passes the ConfiguredAs value into a Case/Switch statement that determines what type of connection to open. This routine is not fully implemented, and thus serves as an excellent example for malware that is &#8220;under development.&#8221; The first image below shows a portion of the Case/Switch statement with several cases that have not yet been populated with code. The second image below is from a &#8220;newer&#8221; sample. While there is still an empty case, the ProxyBackConnector case has been filled in.</p>



<figure class="wp-block-image"><img decoding="async" width="836" height="574" src="https://norfolkinfosec.com/wp-content/uploads/2019/02/silence_case.png" alt="" class="wp-image-105" srcset="https://norfolkinfosec.com/wp-content/uploads/2019/02/silence_case.png 836w, https://norfolkinfosec.com/wp-content/uploads/2019/02/silence_case-300x206.png 300w, https://norfolkinfosec.com/wp-content/uploads/2019/02/silence_case-768x527.png 768w, https://norfolkinfosec.com/wp-content/uploads/2019/02/silence_case-210x144.png 210w" sizes="(max-width: 836px) 100vw, 836px" /><figcaption>Older sample with ProxyBackConnector not populated</figcaption></figure>



<figure class="wp-block-image"><img decoding="async" width="838" height="503" src="https://norfolkinfosec.com/wp-content/uploads/2019/02/silence_case_2.png" alt="" class="wp-image-106" srcset="https://norfolkinfosec.com/wp-content/uploads/2019/02/silence_case_2.png 838w, https://norfolkinfosec.com/wp-content/uploads/2019/02/silence_case_2-300x180.png 300w, https://norfolkinfosec.com/wp-content/uploads/2019/02/silence_case_2-768x461.png 768w, https://norfolkinfosec.com/wp-content/uploads/2019/02/silence_case_2-210x126.png 210w" sizes="(max-width: 838px) 100vw, 838px" /><figcaption>&#8220;Newer&#8221; sample with ProxyBackConnector implemented</figcaption></figure>



<p style="font-size:12px;text-align:left">In total, the tool supports (or likely will support) the following cases, which represent the functionality of the malware:<br><br>SocksServer &#8211; Act as a listener for network traffic<br>DirectBackConnector &#8211; Open a connection to a specified IP and accept the response<br>ProxyBackConnector &#8211; Open a connection to a specified IP and route the response to another device<br>ProxyBackConnectorWithAuth- Not implemented, likely intended as proxy + regular (non-domain) credentials<br>ProxyBackConnectorWithNtlmAuth &#8211; Proxy with an implementation to pass domain credentials<br><br>As mentioned at the start of the post, this is not a complex tool. Despite this, its appearance on the network should be cause for concern, as it is indicative of an adversary that is attempting to route traffic to and from a specific isolated device.</p>
