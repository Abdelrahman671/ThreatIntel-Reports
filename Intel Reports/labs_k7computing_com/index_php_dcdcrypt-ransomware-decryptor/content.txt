<p>
<p></p>



<p>We at K7 Labs came across DcDcrypt ransomware sample, that had infected a user machine and encrypted all user files. Usually the chances of getting back the encrypted files in such a scenario is 0, given the level of sophistication involved in the encryption. Also paying up the ransomware does not guarantee the files be decrypted as “promised” in the ransomware note. Usually the ransom notes and the encrypted are only left behind in a victimized system, the ransomware sample usually gets self-deleted. This case wasn’t so, the ransomware binary was available and naturally we had a closer look at it. Turns out we were able to decrypt the user files after having looked at the malware. The malware and the decryptor would be henceforth discussed.</p>



<p><strong>Analysing Ransomware</strong></p>



<p>This is a basic ransomware written in C#. It encrypts the user files and writes a ransom note in every directory. It does not delete backups, does not create persistence,does not even self-delete(like we mentioned earlier).</p>



<p>Upon execution, this ransomware first checks for a file named “<strong>ID.cl”</strong>. If the file is not present in the same directory, it creates the file and writes a randomly generated ID (used as Victim ID in this case) into it and also stores the ID for further use in a variable named <strong>userID</strong>. If the <strong>ID.cl</strong> file is already present in the same directory it will read and store the content (ID) into the same variable <strong>userID </strong>as shown in Figure 1<strong>.</strong></p>



<p></p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="759" height="345" src="https://labs.k7computing.com/wp-content/uploads/2022/09/Figure1.png" alt="" class="wp-image-24657" srcset="https://labs.k7computing.com/wp-content/uploads/2022/09/Figure1.png 759w, https://labs.k7computing.com/wp-content/uploads/2022/09/Figure1-300x136.png 300w" sizes="(max-width: 759px) 100vw, 759px" /><figcaption>Figure 1: Creating a userID for the victim</figcaption></figure></div>



<p></p>



<p>This ransomware then asks the user to write yes and press enter to continue the encryption through the command line. Once obliged before the start of the encryption the ransomware first uses the <strong>userID </strong>and a hardcoded <strong>salt </strong>to create a password. It uses the method <strong>GetHashCode </strong>of <strong>passwordHasher </strong>class which is then stored in a variable named <strong>password </strong>as shown in Figure 2.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="870" height="215" src="https://labs.k7computing.com/wp-content/uploads/2022/09/Figure2.png" alt="" class="wp-image-24659" srcset="https://labs.k7computing.com/wp-content/uploads/2022/09/Figure2.png 870w, https://labs.k7computing.com/wp-content/uploads/2022/09/Figure2-300x74.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/09/Figure2-768x190.png 768w" sizes="(max-width: 870px) 100vw, 870px" /><figcaption>Figure 2: Generating a password</figcaption></figure></div>



<p></p>



<p><strong>GetHashCode </strong>method just adds the userID and salt and sends it to another method called <strong>Hasher </strong>which will create a Sha512 hash of the added userID and salt. Then the sha512 hash is converted into base64 as shown in Figure 3. The resultant value is stored in the variable named <strong>password </strong>of the class <strong>CoreEncrypter</strong>.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="853" height="390" src="https://labs.k7computing.com/wp-content/uploads/2022/09/figure3-1.png" alt="" class="wp-image-24660" srcset="https://labs.k7computing.com/wp-content/uploads/2022/09/figure3-1.png 853w, https://labs.k7computing.com/wp-content/uploads/2022/09/figure3-1-300x137.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/09/figure3-1-768x351.png 768w" sizes="(max-width: 853px) 100vw, 853px" /><figcaption>Figure 3: GetHashCode and Hasher method</figcaption></figure></div>



<p></p>



<p>After generating the password, the ransomware starts encrypting the contents of the&nbsp; current directory as shown in Figure 4. <strong>Enc</strong> method takes the current directory path as an argument and starts encrypting the files within, traversing all the folders inside the current directory. This ransomware does not encrypt anything other than the contents of the current directory it has been run from.</p>



<p></p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="854" height="136" src="https://labs.k7computing.com/wp-content/uploads/2022/09/figure4.png" alt="" class="wp-image-24663" srcset="https://labs.k7computing.com/wp-content/uploads/2022/09/figure4.png 854w, https://labs.k7computing.com/wp-content/uploads/2022/09/figure4-300x48.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/09/figure4-768x122.png 768w" sizes="(max-width: 854px) 100vw, 854px" /><figcaption>Figure 4: <strong>Enc </strong>method being called with the current directory path as argument</figcaption></figure></div>



<p>The <strong>Enc</strong> method contains two “for” loops, Figure 4: <strong>Enc </strong>method being called with the current directory path as argumentone to traverse the sub-directories within the current directory using recursive call to <strong>Enc </strong>method and the other one to encrypt the files of the current directory. Before encrypting the file, it first checks if the file name contains any of the following strings:&nbsp;</p>



<ul><li>‘<strong>.[Enc]</strong>’ which is present in the extension of the encrypted file.</li><li>&nbsp;‘<strong>.hta’</strong> which is the extension of ransom note,’<strong>ID.cl</strong>’ is the userID file that ransomware creates in the beginning.</li><li>’<strong>desktop.ini</strong>’ is a configuration file.&nbsp;</li><li>‘<strong>Encrypter.exe</strong>’ is the name of ransomware.&nbsp;</li></ul>



<p>If any one of these strings is present in the filename then the ransomware won&#8217;t encrypt that file. Check Figure 5 for details.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="868" height="476" src="https://labs.k7computing.com/wp-content/uploads/2022/09/Figure5.jpg" alt="" class="wp-image-24664" srcset="https://labs.k7computing.com/wp-content/uploads/2022/09/Figure5.jpg 868w, https://labs.k7computing.com/wp-content/uploads/2022/09/Figure5-300x165.jpg 300w, https://labs.k7computing.com/wp-content/uploads/2022/09/Figure5-768x421.jpg 768w" sizes="(max-width: 868px) 100vw, 868px" /><figcaption>Figure 5: Implementation of <strong>Enc </strong>method</figcaption></figure></div>



<p></p>



<p><strong>EncryptFile </strong>contains the encryption routine which is being called in the For loop 1 with the filename as an argument as shown in Figure 5. Let&#8217;s analyse this method so that we can figure out how the encryption is done.</p>



<p>As we can see in Figure 6 <strong>EncryptFile </strong>method uses <strong>Rfc2898DeriveBytes </strong>to generate bytes using a password (generated previously using userID and salt) and a salt which can later be used to derive the key and IV(Initialization Vector) for the encryption algorithm. This ransomware uses <strong>Rijndael </strong>as encryption algorithm which is the predecessor of AES algorithm, at default block size (128 bytes) and in cbc mode it works like AES itself.</p>



<p>Microsoft declared this algorithm as <a href="https://learn.microsoft.com/en-us/dotnet/api/system.security.cryptography.rijndaelmanaged?view=net-6.0">obsolete </a>and suggests using AES instead.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="848" height="544" src="https://labs.k7computing.com/wp-content/uploads/2022/09/figure6.png" alt="" class="wp-image-24665" srcset="https://labs.k7computing.com/wp-content/uploads/2022/09/figure6.png 848w, https://labs.k7computing.com/wp-content/uploads/2022/09/figure6-300x192.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/09/figure6-768x493.png 768w" sizes="(max-width: 848px) 100vw, 848px" /><figcaption>Figure 6: Encryption routine and ransom note</figcaption></figure></div>



<p>&nbsp;</p>



<p>The password that is generated using userID and hardcoded salt remains the same for the same userID (check Figure 2 for reference)&nbsp; and since the password is the same, the key and IV generated using <strong>Rfc2898DeriveBytes </strong>also remains the same for the same userID. <strong>Rijndael </strong>is a symmetric algorithm so the Key-IV pair used for encryption can be used for decryption also.</p>



<p>After initialising all the variables related to encryption, it writes a ransom note in the current directory. Then the ransomware checks for the file size. If it is less than 1000000 bytes, it will create a new file with the same name + encryption extension and then encrypt the original file and put the encrypted content in new file and deletes the original file but if it is larger than the mentioned size it will just XOR the 1st byte of the file with 255 and renames the file with original name + encryption extension, where the encryption extension is<strong> .[Enc][dc.dcrypt@cyberfear.com And dc.dcrypt@mailfence.com (send both) ATTACK ID = %userID%&nbsp; Telegram ID = @decryptionsupport1]</strong>.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="797" height="496" src="https://labs.k7computing.com/wp-content/uploads/2022/09/figure7.png" alt="" class="wp-image-24666" srcset="https://labs.k7computing.com/wp-content/uploads/2022/09/figure7.png 797w, https://labs.k7computing.com/wp-content/uploads/2022/09/figure7-300x187.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/09/figure7-768x478.png 768w" sizes="(max-width: 797px) 100vw, 797px" /><figcaption>Figure 7: Checking file size encrypting file</figcaption></figure></div>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="523" height="255" src="https://labs.k7computing.com/wp-content/uploads/2022/09/figure8.jpg" alt="" class="wp-image-24669" srcset="https://labs.k7computing.com/wp-content/uploads/2022/09/figure8.jpg 523w, https://labs.k7computing.com/wp-content/uploads/2022/09/figure8-300x146.jpg 300w" sizes="(max-width: 523px) 100vw, 523px" /><figcaption>Figure 8: XORing 1st byte of large files</figcaption></figure></div>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="838" height="984" src="https://labs.k7computing.com/wp-content/uploads/2022/09/figure9.png" alt="" class="wp-image-24670" srcset="https://labs.k7computing.com/wp-content/uploads/2022/09/figure9.png 838w, https://labs.k7computing.com/wp-content/uploads/2022/09/figure9-255x300.png 255w, https://labs.k7computing.com/wp-content/uploads/2022/09/figure9-768x902.png 768w" sizes="(max-width: 838px) 100vw, 838px" /><figcaption>Figure 9: Ransomware execution</figcaption></figure></div>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="640" height="493" src="https://labs.k7computing.com/wp-content/uploads/2022/09/figure10.png" alt="" class="wp-image-24671" srcset="https://labs.k7computing.com/wp-content/uploads/2022/09/figure10.png 640w, https://labs.k7computing.com/wp-content/uploads/2022/09/figure10-300x231.png 300w" sizes="(max-width: 640px) 100vw, 640px" /><figcaption>Figure 10: Ransom Note</figcaption></figure></div>



<p></p>



<p><strong>Decryptor Internals</strong></p>



<p>As we already know the ransomware generates a password using userID that it created randomly and a hardcoded salt, since every infected user have a unique userID and we know it will be stored in a file called <strong>ID.cl</strong> so in our decryptor, instead of creating the userID we would instead read ID.cl file to get the userID and use the same method for generating the password (GetHashCode in Figure 3) as ransomware did to generate the password.&nbsp;</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="812" height="299" src="https://labs.k7computing.com/wp-content/uploads/2022/09/figure11.png" alt="" class="wp-image-24672" srcset="https://labs.k7computing.com/wp-content/uploads/2022/09/figure11.png 812w, https://labs.k7computing.com/wp-content/uploads/2022/09/figure11-300x110.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/09/figure11-768x283.png 768w" sizes="(max-width: 812px) 100vw, 812px" /><figcaption>Figure 11: Generating password with userID</figcaption></figure></div>



<p></p>



<p>After that this tool would scan all the drives for any encrypted files, all the encrypted files have .[Enc] in their extension hence&nbsp; identifying the encrypted file wouldn&#8217;t be too hard.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="733" height="264" src="https://labs.k7computing.com/wp-content/uploads/2022/09/figure12.png" alt="" class="wp-image-24673" srcset="https://labs.k7computing.com/wp-content/uploads/2022/09/figure12.png 733w, https://labs.k7computing.com/wp-content/uploads/2022/09/figure12-300x108.png 300w" sizes="(max-width: 733px) 100vw, 733px" /><figcaption>Figure 12: Scanning all drives</figcaption></figure></div>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="538" height="209" src="https://labs.k7computing.com/wp-content/uploads/2022/09/figure13.png" alt="" class="wp-image-24674" srcset="https://labs.k7computing.com/wp-content/uploads/2022/09/figure13.png 538w, https://labs.k7computing.com/wp-content/uploads/2022/09/figure13-300x117.png 300w" sizes="(max-width: 538px) 100vw, 538px" /><figcaption>Figure 13: Checking file name for <strong>.[Enc]</strong> and sending it for decryption</figcaption></figure></div>



<p></p>



<p>Since the ransomware used built-in classes and methods provided by .Net for encrypting the files (RijndaelManaged) we can use the same for decrypting the files. We will generate the Key and IV same way ransomware generated with the password using <strong>Rfc2898DeriveBytes</strong>.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="806" height="476" src="https://labs.k7computing.com/wp-content/uploads/2022/09/figure14.png" alt="" class="wp-image-24675" srcset="https://labs.k7computing.com/wp-content/uploads/2022/09/figure14.png 806w, https://labs.k7computing.com/wp-content/uploads/2022/09/figure14-300x177.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/09/figure14-768x454.png 768w" sizes="(max-width: 806px) 100vw, 806px" /><figcaption>Figure 14: Initialising Key and IV for decryption</figcaption></figure></div>



<p></p>



<p>In order to decrypt the file size is checked if less than 1000000 bytes then the tool would proceed to decrypt it. Ransomware uses <strong>rijndaelManaged.CreateEncryptor </strong>to create encryption stream in the same manner we can use <strong>rijndaelManaged.CreateDecryptor </strong>to create the<strong> </strong>decryption stream it would use the Key-IV generated before for decryption.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="972" height="442" src="https://labs.k7computing.com/wp-content/uploads/2022/09/figure15.png" alt="" class="wp-image-24676" srcset="https://labs.k7computing.com/wp-content/uploads/2022/09/figure15.png 972w, https://labs.k7computing.com/wp-content/uploads/2022/09/figure15-300x136.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/09/figure15-768x349.png 768w" sizes="(max-width: 972px) 100vw, 972px" /><figcaption>Figure 15: Decrypting file</figcaption></figure></div>



<p></p>



<p>After decryption, we will write the decrypted bytes into a new file and keep the encrypted file as .[backup] in case the file is not decrypted correctly. For files above 1000000 bytes we will XOR the 1st byte with 255 again to get the original byte and rename the file as the original name (without encryption extension).</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><img loading="lazy" width="542" height="233" src="https://labs.k7computing.com/wp-content/uploads/2022/09/figure16.png" alt="" class="wp-image-24677" srcset="https://labs.k7computing.com/wp-content/uploads/2022/09/figure16.png 542w, https://labs.k7computing.com/wp-content/uploads/2022/09/figure16-300x129.png 300w" sizes="(max-width: 542px) 100vw, 542px" /><figcaption>Figure 16: XORing 1st byte with 255 for larger files</figcaption></figure></div>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" width="1024" height="636" src="https://labs.k7computing.com/wp-content/uploads/2022/09/figure17-1024x636.png" alt="" class="wp-image-24678" srcset="https://labs.k7computing.com/wp-content/uploads/2022/09/figure17-1024x636.png 1024w, https://labs.k7computing.com/wp-content/uploads/2022/09/figure17-300x186.png 300w, https://labs.k7computing.com/wp-content/uploads/2022/09/figure17-768x477.png 768w, https://labs.k7computing.com/wp-content/uploads/2022/09/figure17.png 1274w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption>Figure 17: Decryption Tool</figcaption></figure></div>
