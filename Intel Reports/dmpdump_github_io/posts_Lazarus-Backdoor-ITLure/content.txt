data-toc-skip>Lazarus Backdoor with IT Lure</h1><div class="post-meta text-muted"> <span> Posted <time data-ts="1739059200" data-df="ll" data-bs-toggle="tooltip" data-bs-placement="bottom" > Feb 9, 2025 </time> </span> <span> Updated <time data-ts="1739420627" data-df="ll" data-bs-toggle="tooltip" data-bs-placement="bottom" > Feb 12, 2025 </time> </span><div class="d-flex justify-content-between"> <span> By <em> <a href="https://github.com/dmpdump">dmpdump</a> </em> </span><div> <span class="readtime" data-bs-toggle="tooltip" data-bs-placement="bottom" title="794 words" > <em>4 min</em> read</span></div></div></div></header><div class="content"><p>On January 27, 2025, <a href="https://x.com/smica83/status/1883855708963442892">@smica83</a> shared a sample on X indicating that it looked like Lazarus malware. I reviewed the sample and concluded that, indeed, it is a North Korean backdoor, likely the latest version of a backdoor publicly tracked as PEBBLEDASH.</p><p><a href="/assets/images/dprk_itsector/smica83.png" class="popup img-link shimmer"><img src="/assets/images/dprk_itsector/smica83.png" alt="sshot" loading="lazy"></a></p><p>The file shared by @smica83 is a portable executable named <code class="language-plaintext highlighter-rouge">iconcache.tmp.pif</code>, with SHA2:<code class="language-plaintext highlighter-rouge">d0a41dfe8f5b5c8ba6a5d0bdc3754543210ec2d36290564d9a774e9d22e3ad97 </code>. Reviewing connected samples in VirusTotal, I could see 2 droppers leading to this executable:</p><p><a href="/assets/images/dprk_itsector/pivot.png" class="popup img-link shimmer"><img src="/assets/images/dprk_itsector/pivot.png" alt="sshot" loading="lazy"></a></p><p>Taking <code class="language-plaintext highlighter-rouge">6744ca5d49833c9b90aee0f3be39d28dec94579b028b05c647354ec5e1ab53e1</code> as a sample dropper, we can see it is a 64-bit portable executable with a PDF icon as a lure. The dropper is obfuscated and it drops and opens a decoy PDF named <code class="language-plaintext highlighter-rouge">2025년 01월 오라클 정기점검(서명완).pdf</code>, which translates to <code class="language-plaintext highlighter-rouge">Oracle Scheduled Maintenance in January 2025 (Seo Myeong-wan)</code>. It also drops the executable shared by @smica83, <code class="language-plaintext highlighter-rouge">iconcache.tmp.pif</code>, both under <code class="language-plaintext highlighter-rouge">C:\ProgramData</code>.</p><p><a href="/assets/images/dprk_itsector/drop1.png" class="popup img-link shimmer"><img src="/assets/images/dprk_itsector/drop1.png" alt="sshot" loading="lazy"></a></p><p>The PDF seems to be a monthly inspection report from South Korean IT comany DBWorks, potentially for their client Unison Co Ltd., a South Korean wind turbine manufacturer. The report is dated 2025-01-23, which coincides with the VirusTotal upload date of <code class="language-plaintext highlighter-rouge">iconcache.tmp.pif</code> and its dropper. It is unclear which of the two companies were targeted, but Unison was likely the target.</p><p><a href="/assets/images/dprk_itsector/pdf.png" class="popup img-link shimmer"><img src="/assets/images/dprk_itsector/pdf.png" alt="sshot" loading="lazy"></a></p><p>Upon execution, the dropper runs <code class="language-plaintext highlighter-rouge">iconcache.tmp.pif</code>, setting its persistence via a registry run key:</p><p><a href="/assets/images/dprk_itsector/drop2.png" class="popup img-link shimmer"><img src="/assets/images/dprk_itsector/drop2.png" alt="sshot" loading="lazy"></a></p><h1 id="backdoor">Backdoor</h1><p><code class="language-plaintext highlighter-rouge">iconcache.tmp.pif</code> is a backdoor that implements an argument check to determine its execution path:</p><p><a href="/assets/images/dprk_itsector/initial_decision.png" class="popup img-link shimmer"><img src="/assets/images/dprk_itsector/initial_decision.png" alt="sshot" loading="lazy"></a></p><p>If the backdoor is executed with the <code class="language-plaintext highlighter-rouge">--start</code> argument, it skips the execution of its main logic and establishes persistence for itself via the registry run key, as observed in the initial execution. If the backdoor is executed without any arguments (as is the case during persistence), the main backdoor logic is executed.</p><p>Across the backdoor, we can see the resolution of APIs via the computation of FNV1-a hashes using embedded strings, as can be seen during the creation of the malware persistence:</p><p><a href="/assets/images/dprk_itsector/fnv1a-res.png" class="popup img-link shimmer"><img src="/assets/images/dprk_itsector/fnv1a-res.png" alt="sshot" loading="lazy"></a></p><p><a href="/assets/images/dprk_itsector/persist_debug.png" class="popup img-link shimmer"><img src="/assets/images/dprk_itsector/persist_debug.png" alt="sshot" loading="lazy"></a></p><p><a href="/assets/images/dprk_itsector/persist.png" class="popup img-link shimmer"><img src="/assets/images/dprk_itsector/persist.png" alt="sshot" loading="lazy"></a></p><p>When the main backdoor logic is executed, it sets up a connection to the C2 via the initialization of Winsock structures using <code class="language-plaintext highlighter-rouge">WSAStartup</code>.</p><p><a href="/assets/images/dprk_itsector/initializewinsock.png" class="popup img-link shimmer"><img src="/assets/images/dprk_itsector/initializewinsock.png" alt="sshot" loading="lazy"></a></p><p>It then proceeds to decrypt the C2 domain: <code class="language-plaintext highlighter-rouge">http://www.addfriend[.]kr/board/userfiles/temp/index.html</code></p><p><a href="/assets/images/dprk_itsector/decryptc2.png" class="popup img-link shimmer"><img src="/assets/images/dprk_itsector/decryptc2.png" alt="sshot" loading="lazy"></a></p><p>The backdoor uses the following User Agent string, obfuscated via stack strings: <code class="language-plaintext highlighter-rouge">Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/104.0.5112.102 Safari/537.36</code></p><p><a href="/assets/images/dprk_itsector/uastring.png" class="popup img-link shimmer"><img src="/assets/images/dprk_itsector/uastring.png" alt="sshot" loading="lazy"></a></p><p>Connections to the command and control use the following values: <code class="language-plaintext highlighter-rouge">sep</code>, a hard-coded string, <code class="language-plaintext highlighter-rouge">uid</code>, based on the volume drive information, and <code class="language-plaintext highlighter-rouge">sid</code>, random.</p><p><a href="/assets/images/dprk_itsector/postreq.png" class="popup img-link shimmer"><img src="/assets/images/dprk_itsector/postreq.png" alt="sshot" loading="lazy"></a></p><p>After decrypting the C2 URL and connecting to the C2, the backdoor parses an HTML response before entering a loop to accept remote commands.</p><p><a href="/assets/images/dprk_itsector/mainloop1.png" class="popup img-link shimmer"><img src="/assets/images/dprk_itsector/mainloop1.png" alt="sshot" loading="lazy"></a></p><p>The backdoor implements 14 commands, some of which were not very clear to me. It makes extensive use of files with the “PMS” prefix written to AppData\Local\Temp. These files are used by the backdoor to save the output of commands executed remotely, to save screenshots, and for the registration of DLLs.</p><div class="table-wrapper"><table><thead><tr><th>Command<th>Action<tbody><tr><td>3<td>Changes/sets the current directory<tr><td>4<td>Creates a process<tr><td>5<td>Creates a process impersonating a token<tr><td>6<td>Writes a file with the “PMS” prefix in AppData\Local\Temp, registers it with regsvr32.exe /s<tr><td>7<td>Loads a DLL<tr><td>8<td>Reads a file<tr><td>9<td>Writes a file<tr><td>10<td>Gets System time and updates a global variable, possibly introducing a delay<tr><td>11<td>Gets System time and updates a global variable, possibly introducing a delay<tr><td>12<td>Deletes persistence in the registry and runs a self-deletion batch file<tr><td>13<td>Gets the hostname, computer name, and adapters information<tr><td>14<td>Executes shell commands with cmd.exe, writing the output to a file with the “PMS” prefix<tr><td>15<td>Makes new request to the C2, unclear what the purpose is<tr><td>16<td>Takes a screenshot and saves it to a file with the “PMS” prefix in AppData\Local\Temp</table></div><p>The batch file dropped by command 12 is named <code class="language-plaintext highlighter-rouge">ico.bat</code> and it is dropped to <code class="language-plaintext highlighter-rouge">AppData\Local\Temp</code>. The response data seems to be AES encrypted with key <code class="language-plaintext highlighter-rouge">aqjNWSmPkmpYnZJT</code> and then base64-encoded.</p><p><a href="/assets/images/dprk_itsector/encryptencode.png" class="popup img-link shimmer"><img src="/assets/images/dprk_itsector/encryptencode.png" alt="sshot" loading="lazy"></a></p><p>We can test this by reversing the <code class="language-plaintext highlighter-rouge">data=</code> value when a screenshot is exfiltrated:</p><p><a href="/assets/images/dprk_itsector/cyberchef.png" class="popup img-link shimmer"><img src="/assets/images/dprk_itsector/cyberchef.png" alt="sshot" loading="lazy"></a></p><h1 id="previous-reports">Previous reports</h1><p>There are multiple reports documenting the use of PEBBLEDASH in the past, including great work from <a href="https://asec.ahnlab.com/en/30022/">Ahnlab</a>. One of the most detailed reports I could find was from <a href="https://ti.qianxin.com/blog/articles/Kimsuky-Weapon-Update:-Analysis-of-Attack-Activity-Targeting-Korean-Region/">Qianxing Threat Intelligence</a>. The backdoor described in their report is very similar to the one analyzed here, although it uses different string encryption and there are differences in the commands.
