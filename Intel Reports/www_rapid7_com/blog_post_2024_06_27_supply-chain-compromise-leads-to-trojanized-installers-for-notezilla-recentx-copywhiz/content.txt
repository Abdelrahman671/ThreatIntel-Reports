                            <p>The following Rapid7 analysts contributed to this research: Leo Gutierrez, Tyler McGraw, Sarah Lee, and Thomas Elkins.</p><!--kg-card-begin: markdown--><h2 id="executivesummary">Executive Summary</h2>
<p>On Tuesday, June 18th, 2024, Rapid7 initiated an investigation into suspicious activity in a customer environment. Our investigation identified that the suspicious behavior was emanating from the installation of Notezilla, a program that allows for the creation of sticky notes on a Windows desktop. Installers for Notezilla, along with tools called RecentX and Copywhiz, are distributed by the India-based company Conceptworld at the official domain <code>conceptworld[.]com</code>. After analyzing the installation packages for all three programs, Rapid7 discovered that the installers had been trojanized to execute information-stealing malware that has the capability to download and execute additional payloads.</p>
<h2 id="disclosure">Disclosure</h2>
<p>On Monday, June 24th, 2024, Rapid7 contacted Conceptworld to disclose the backdoored installers being hosted on <code>conceptworld[.]com</code> in accordance with <a href="https://www.rapid7.com/security/disclosure/">Rapid7’s vulnerability disclosure policy</a>. Within 12 hours, Conceptworld confirmed and remediated the issue by removing the malicious installers from <code>conceptworld[.]com</code> and replacing them with legitimate, signed copies. Rapid7 is grateful to Conceptworld for their prompt action on this issue.</p>
<h2 id="overview">Overview</h2>
<p>Conceptworld is an India-based company offering three different software products: Notezilla, which allows users to create sticky notes on a Windows desktop; RecentX, which stores recently used files/applications/clipboard data; and Copywhiz, which improves file copying and backup operations. A free trial download is available on the official <code>conceptworld[.]com</code> site for each software package.</p>
<p>The installation packages being served by <code>conceptworld[.]com</code> at the time of investigation, however, executed malware alongside the legitimate installer, were not signed, and did not match the file size stated on the download page. The differences in the file sizes are due to the malware and its dependencies, which increases the size of the compromised installation packages.</p>

<!--kg-card-end: markdown--><!--kg-card-begin: markdown--><p>The malware Rapid7 observed contains the functionality to steal browser credentials and crypto currency wallet information, log clipboard contents and keystrokes, and download and execute additional payloads. After infecting a system, the malware persists via a scheduled task that executes the primary payload every three hours.</p>
<p>Based on file submissions to VirusTotal, the malicious copies of the installers have existed since early June of 2024. The malware payloads delivered by the trojanized installers, however, seem to belong to a nameless malware family that has been in distribution since at least January of 2024. Rapid7 internally refers to this malware family as <strong>dllFake</strong> because of the naming scheme used for several of the malware payloads.</p>

<!--kg-card-end: markdown--><!--kg-card-begin: markdown--><h2 id="technicalanalysis">Technical analysis</h2>
<p>To take a deeper look at the malware payloads, we will analyze the malicious installer that was served for Notezilla.</p>
<h3 id="initialaccess">Initial Access</h3>
<p>Rapid7 determined that trojanized installers for the 32-bit and 64-bit versions of Notezilla, Copywhiz, and RecentX were, at the time of investigation, being served from the official website <code>conceptworld[.]com</code>. Any users searching for this software via a popular search engine at the time were most likely to find the official domain as the first result, which would then have directed them to download the malware.</p>
<h3 id="execution">Execution</h3>
<p>The installer served by <code>conceptworld[.]com</code> for Notezilla at the time of investigation was <code>NotezillaSetup.exe</code>, which, based on static analysis, is packed using software called <code>Smart Install Maker(5.04)</code>.</p>
<!--kg-card-end: markdown--><figure class="kg-card kg-image-card"><img src="https://blog.rapid7.com/content/images/2024/06/figure1.png" class="kg-image"></figure><!--kg-card-begin: markdown--><p><em>Figure 1. Software Properties of <code>NotezillaSetup.exe</code>.</em></p>
<!--kg-card-end: markdown--><!--kg-card-begin: markdown--><p>Using the <code>sim_unpacker</code> plugin for the tool <a href="https://github.com/Bioruebe/UniExtract2">UniExtract2</a>, we were able to unpack and acquire most of the contents of the installation package, such as the embedded files and configuration information. The configuration file contains references to the legitimate software installer for Notezilla, which is dropped into <code>%TEMP%</code> during execution, and multiple files that are dropped into the installation directory (i.e., staging folder) <code>%LOCALAPPDATA%\Microsoft\WindowsApps\</code> during execution.</p>

<!--kg-card-end: markdown--><!--kg-card-begin: markdown--><p>Once executed, <code>NotezillaSetup.exe</code> will then execute the file <code>dllCrt32.exe</code> from the staging directory <code>%LOCALAPPDATA%\Microsoft\WindowsApps\</code> via a WINAPI call to <code>ShellExecuteA</code> with the verb <code>open</code>. A second call is then made to <code>ShellExecuteA</code> to execute the file <code>NotezillaSetup.exe</code>, a copy of the legitimate installer, from <code>%TEMP%</code>. As a result, the only thing seen by the end user after initial execution is the installation window pop-up for the legitimate installer, prompting the user to proceed with the installation process for Notezilla.</p>
<!--kg-card-end: markdown--><figure class="kg-card kg-image-card"><img src="https://blog.rapid7.com/content/images/2024/06/figure4.png" class="kg-image"></figure><!--kg-card-begin: markdown--><p><em>Figure 4. Typical Process Tree for Initial Execution of the Trojanized Installer.</em></p>
<!--kg-card-end: markdown--><figure class="kg-card kg-image-card"><img src="https://blog.rapid7.com/content/images/2024/06/figure5.png" class="kg-image"></figure><!--kg-card-begin: markdown--><p><em>Figure 5. The User’s View after the Infection has Already Begun in the Background.</em></p>
<!--kg-card-end: markdown--><!--kg-card-begin: markdown--><p>The file <code>dllCrt32.exe</code> is a relatively small (~10KB) program that only serves as a wrapper to call <code>CreateProcessA</code> to execute the file <code>dllCrt.bat</code>.</p>
<!--kg-card-end: markdown--><figure class="kg-card kg-image-card"><img src="https://blog.rapid7.com/content/images/2024/06/figure6.png" class="kg-image"></figure><!--kg-card-begin: markdown--><p><em>Figure 6. The Contents of <code>dllCrt.bat</code>.</em></p>
<!--kg-card-end: markdown--><!--kg-card-begin: markdown--><p>The batch file <code>dllCrt.bat</code> will then create a hidden scheduled task named <code>Check dllHourly32</code> using <code>schtasks.exe</code> and an XML file that was previously dropped into the staging directory at <code>%LOCALAPPDATA%\Microsoft\WindowsApps\dllCrt.xml</code>. The scheduled task <code>Check dllHourly32</code> will then execute the file <code>%LOCALAPPDATA%\Microsoft\WindowsApps\dllBus32.exe</code> every three hours after being initially created, which means that the primary malware payload will not be executed until at least three hours after the user originally executed the trojanized installer.</p>
<!--kg-card-end: markdown--><figure class="kg-card kg-image-card"><img src="https://blog.rapid7.com/content/images/2024/06/figure7.png" class="kg-image"></figure><!--kg-card-begin: markdown--><p><em>Figure 7. Command Line Assembly within <code>dllBus32.exe</code>.</em></p>
<!--kg-card-end: markdown--><!--kg-card-begin: markdown--><p>When <code>dllBus32.exe</code> is executed, it also serves as a small wrapper for calling <code>CreateProcessA</code>, though it initially retrieves several important command line parameters. First, a call to the CRT library function <code>sprintf</code> concatenates a hard-coded IPv4 address. Then, a second call to <code>sprintf</code> concatenates the assembled IPv4 address with several other arguments to be passed to the batch file <code>dllBus.bat</code>. Finally, <code>CreateProcessA</code> is called with the fully assembled command line.</p>
<!--kg-card-end: markdown--><figure class="kg-card kg-image-card"><img src="https://blog.rapid7.com/content/images/2024/06/figure8.png" class="kg-image"></figure><!--kg-card-begin: markdown--><p><em>Figure 8. The Initial Lines of <code>dllBus.bat</code>.</em></p>
<!--kg-card-end: markdown--><!--kg-card-begin: markdown--><p>The command line arguments passed to <code>dllBus.bat</code> via <code>dllBus32.exe</code> contain an IPv4 address, an SFTP port, a password for ZIP archive payloads, two sets of SFTP credentials, and the staging directory where the majority of the malware’s files are located.</p>

<!--kg-card-end: markdown--><!--kg-card-begin: markdown--><p>The batch file <code>dllBus.bat</code> contains functionality to facilitate the theft of information from Google Chrome, Mozilla Firefox, and multiple cryptocurrency wallets. The copy of <code>curl.exe</code> dropped by the installer is also used to connect to a list of command-and-control (C2) addresses hosting SFTP servers. The curl commands are used to download an updated list of C2 addresses, stored as plaintext within the file <code>dll_srv.txt</code>, and to download and execute additional payloads saved within encrypted ZIP archives named <code>Updt.zip</code>, <code>Apps.zip</code>, and <code>BB.zip</code>. The batch script will also attempt to compress all files on the infected system that have specific file extensions and exist in directories that are not on a hardcoded blacklist (for exfiltration). All stolen data is ultimately compressed using <code>7z.exe</code> and uploaded directly to the selected C2 SFTP server using curl.</p>

<h2 id="mitigationguidance">Mitigation Guidance</h2>
<p>Rapid7 recommends verifying the file integrity of freely available software. Check that the file hash and properties of the downloaded file(s) match those provided by the official distributor and/or that they contain a valid and relevant signature. The malicious installers observed in this case are unsigned and have a file size that is inconsistent with copies of the legitimate installer, even as noted on the official download page.</p>
<p>If an installer for Notezilla, RecentX, or Copywhiz has been executed on a system within the last month, Rapid7 recommends checking for signs of compromise due to the malicious installers detailed in this blog. The primary indicators of infection include the hidden scheduled task <code>Check dllHourly32</code> and a persistent running instance of the Windows Command Prompt, <code>cmd.exe</code>, which makes outbound network connections via <code>curl.exe</code>.</p>
<p>If evidence of compromise is found, Rapid7 recommends re-imaging affected systems to a known good baseline to eradicate any changes made by the malware.</p>
