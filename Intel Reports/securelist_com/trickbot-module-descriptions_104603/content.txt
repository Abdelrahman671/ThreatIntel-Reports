												<div class="c-wysiwyg">
													<p>Trickbot (aka TrickLoader or Trickster), is a successor of the <a href="https://threats.kaspersky.com/en/threat/Trojan-Banker.Win32.Dyre/" target="_blank" rel="noopener">Dyre</a> banking Trojan that was active from 2014 to 2016 and performed <a href="https://encyclopedia.kaspersky.com/glossary/man-in-the-browser-attack/?utm_source=securelist&amp;utm_medium=blog&amp;utm_campaign=termin-explanation" target="_blank" rel="noopener">man-in-the-browser</a> attacks in order to steal banking credentials. Trickbot was first discovered in October 2016. Just like Dyre, its main functionality was initially the theft of online banking data. However, over time, its tactics and goals have changed. Currently Trickbot is focused on penetration and distribution over the local network, providing other malware (such as <a href="https://securelist.com/story-of-the-year-2019-cities-under-ransomware-siege/95456/" target="_blank" rel="noopener">Ryuk ransomware</a>) with access to the infected system, though that&#8217;s not the only functionality it supports.</p>
<p>Over the years, Trickbot has acquired dozens of auxiliary modules that steal credentials and sensitive information, spread it over the local network using stolen credentials and vulnerabilities, provide remote access, proxy network traffic, perform brute-force attacks and download other malware. In this document, we decided to provide a brief description of the Trickbot modules. Despite the fact the dates indicated in the PE headers of some modules are quite old, these modules are still available for download and can be used by threat actors. Such information should simplify analysis of any activity related to Trickbot.</p>
<h2 id="technical-details">Technical details</h2>
<h3 id="how-to-obtain-trickbot-modules-for-analysis">How to obtain Trickbot modules for analysis</h3>
<p>Modules can be downloaded from one of Trickbot&#8217;s C2s using simple GET requests like https://&lt;CC_IP&gt;:&lt;CC_PORT&gt;/&lt;gtag&gt;/&lt;bot_ID&gt;/5/&lt;module_name&gt;/. Keep in mind that module names are case sensitive, and although we describe 32-bit modules in this article in most cases 64-bit versions can be downloaded by replacing &#8217;32&#8217; with &#8217;64&#8217; in the module name. In most cases valid values of &lt;gtag&gt; and &lt;bot_ID&gt; are not needed for successful download. Here is an example of the URL to download the pwgrab64 module:</p>
<p style="text-align: center">https[:]//87.97.178[.]92:447/asdasdasd/asdasdasd_asdasdasd.asdasdasd/5/pwgrab64/</p>
<p>Downloaded modules are encrypted, and can be decrypted with the Python script below.</p>

<p><strong><a name="adll32"></a>adll32</strong></p>
<p>The new Trickbot module ADLL dumps Active Directory database files (ntds.dit and ntds.jfm) and registry hives using the ntdsutil and reg tools. For example, it utilizes the &#8220;Install from Media (IFM)&#8221; ntdsutil command to dump the Active Directory database and various registry hives to the %Temp% folder. These files are then compressed and sent back to the attackers.</p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150508/Trickbot_report_02.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-104610" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150508/Trickbot_report_02.png" alt="Part of the adll32 function of collecting information using the ntdsutil tool" width="475" height="123" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150508/Trickbot_report_02.png 475w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150508/Trickbot_report_02-300x78.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150508/Trickbot_report_02-370x96.png 370w" sizes="(max-width: 475px) 100vw, 475px" /></a></p>
<p style="text-align: center"><strong><em>Part of the adll32 function of collecting information using the ntdsutil tool</em></strong></p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150519/Trickbot_report_03.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-104611" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150519/Trickbot_report_03.png" alt="Part of the adll32 function collecting information using the reg tool" width="429" height="76" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150519/Trickbot_report_03.png 429w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150519/Trickbot_report_03-300x53.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150519/Trickbot_report_03-370x66.png 370w" sizes="(max-width: 429px) 100vw, 429px" /></a></p>
<p style="text-align: center"><strong><em>Part of the adll32 function collecting information using the reg tool</em></strong></p>
<p>The module uses the ntdsutil and reg tools with the following command line arguments:</p>
<ul>
<li>ntdsutil &#8220;ac in ntds&#8221; &#8220;ifm&#8221; &#8220;cr fu %TEMP%\&lt;random&gt;0.dat&#8221; q q</li>
<li>reg save HKLM\SAM %TEMP%\&lt;random&gt;1.dat /y</li>
<li>reg save HKLM\SECURITY %TEMP%\&lt;random&gt;2.dat /y</li>
<li>reg save HKLM\SYSTEM %TEMP%\&lt;random&gt;3.dat /y</li>
</ul>
<p>These commands will dump the Active Directory database as well as the SAM, SECURITY, and SYSTEM hives. Threat actors can decrypt these files and dump the usernames, password hashes, computer names, groups, and other data. This data can then be used for spreading further across the network.</p>
<p><strong><a name="aexecDll32"></a>aexecDll32</strong></p>
<p>This module is a simple downloader. It downloads a payload (e.g., another Trickbot module or third-party malware) by hardcoded URL and executes it.</p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150539/Trickbot_report_04.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-104613" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150539/Trickbot_report_04.png" alt="Part of the aexecDll32 download routine" width="753" height="304" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150539/Trickbot_report_04.png 753w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150539/Trickbot_report_04-300x121.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150539/Trickbot_report_04-433x175.png 433w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150539/Trickbot_report_04-370x149.png 370w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150539/Trickbot_report_04-694x280.png 694w" sizes="(max-width: 753px) 100vw, 753px" /></a></p>

<p><strong><a name="anubisDll32"></a>anubisDll32</strong></p>
<p>This is a man-in-the-browser module. It contains a full implementation of the <a href="https://securelist.com/malicious-spam-campaigns-delivering-banking-trojans/102917/" target="_blank" rel="noopener">IcedID</a> main module. It can intercept web traffic on the victim machine. This module also contains embedded binary Anubis VNC (also known as HDESK Bot), which is a custom RAT based on the VNC (Virtual Network Computing) protocol that&#8217;s used by IcedID for remote control.</p>
<p><strong><a name="bcClientDll32"></a>bcClientDll32</strong></p>
<p>This is a reverse proxy module with custom implementation of SOCKS5 protocol. It is used to bypass traffic through the compromised host.</p>
<p><strong><a name="bvncDll32"></a>bvncDll32</strong></p>
<p>This module is an implementation of Hidden VNC, which is a custom remote administration tool based on the VNC (Virtual Network Computing) protocol. Hidden means that this modification of the VNC creates an additional desktop hidden from the user, allowing attackers to work with the compromised system in the same way as via an RDP connection without attracting the user&#8217;s attention. <p><strong><a name="tabDll32"></a>tabDll32</strong></p>
<p>This module propagates Trickbot via the EternalRomance exploit. It enables WDigest Authentication by modifying the UseLogonCredential value in the HKLM\SYSTEM\CurrentControlSet\Control\SecurityProviders\WDigest registry key. This modification is forced to save credentials in lsass.exe memory (Local Security Authority Subsystem Service). The tabDll32 module then injects the embedded module screenLocker in explorer.exe and locks the workstation with the lock screen, forcing the user to login again. It waits for next user login and scrapes the credentials from LSASS memory utilizing Mimikatz functionality. Stolen credentials are sent back to the C2. After that tabDll32 downloads the payload from hardcoded URLs – usually the Trickbot loader (downloader) – starts up to 256 threads and uses the EternalRomance exploit to rapidly spread the downloaded payload over the network. Another embedded module, ssExecutor_x86, is used to set up persistence for the downloaded payload on exploited systems. This module also contains the main code of the shareDll32 module, and uses it to spread over the network.</p>
<p>The module ssExecutor_x86 copies the payload into the C:\WINDOWS\SYSTEM32\ and C:\WINDOWS\SYSTEM32\TASKS folders, and executes the payload. It enumerates user profiles in registry, copies the payload to C:\Users\&lt;User Profile&gt;\AppData\Roaming\ and establishes persistence by creating shortcuts in the profile&#8217;s Startup folder and creating a &#8216;Run&#8217; key in the profile&#8217;s registry.</p>
<p>The tabDll32 module also exists in the form of a standalone executable with the alias &#8216;spreader&#8217;.</p>
<p><strong><a name="systeminfo32"></a>systeminfo32</strong></p>
<p>This module gathers basic system information. It uses WQL to get information about the OS name, architecture, version, CPU and RAM information. It also collects user account names on the local computer and gathers information about installed software and services by enumerating HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall&#8217; and &#8216;HKLM\SYSTEM\CurrentControlSet\Services registry keys.</p>
<p><strong><a name="vpnDll32"></a>vpnDll32</strong></p>
<p>This module uses an RAS (Remote Access Service) API to establish a VPN (Virtual Private Network) connection. The destination VPN server, and credentials for connection are provided by the C2&#8217;s command and configuration files.</p>

<p style="text-align: center"><strong><em>Setting the RAS (Remote Access Service) entry.</em></strong></p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150925/Trickbot_report_14.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-104626" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150925/Trickbot_report_14.png" alt="The vpnDll32 module establishes a VPN connection" width="384" height="171" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150925/Trickbot_report_14.png 384w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150925/Trickbot_report_14-300x134.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150925/Trickbot_report_14-370x165.png 370w" sizes="(max-width: 384px) 100vw, 384px" /></a>&lt;</p>
<p style="text-align: center"><strong><em>The vpnDll32 module establishes a VPN connection</em></strong></p>
<p><strong><a name="webiDll32"></a>webiDll32</strong></p>
<p>This module is used to intercept activity related to banking websites in browsers. It uses web injects to steal financial information. In addition to typical static and dynamic injections, this modification also supports web injects in the Zeus format, and can modify pages on the client side.</p>
<p style="text-align: center"><a href="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150939/Trickbot_report_15.png" class="magnificImage"><img loading="lazy" decoding="async" class="aligncenter size-full wp-image-104627" src="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150939/Trickbot_report_15.png" alt="Trickbot web inject configuration file in Zeus format" width="891" height="400" srcset="https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150939/Trickbot_report_15.png 891w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150939/Trickbot_report_15-300x135.png 300w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150939/Trickbot_report_15-768x345.png 768w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150939/Trickbot_report_15-390x175.png 390w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150939/Trickbot_report_15-370x166.png 370w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150939/Trickbot_report_15-624x280.png 624w, https://media.kasperskycontenthub.com/wp-content/uploads/sites/43/2021/10/18150939/Trickbot_report_15-800x359.png 800w" sizes="(max-width: 891px) 100vw, 891px" /></a></p>
<p style="text-align: center"><strong><em>Trickbot web inject configuration file in Zeus format</em></strong></p>
<p><strong><a name="wormDll32"></a>wormDll32</strong></p>
<p>This module propagates Trickbot with the EternalBlue exploit. It enumerates computers using Network Management API and Active Directory Service Interfaces. It uses the EternalBlue exploit to inject shellcode in LSASS process memory. The injected shellcode downloads Trickbot from a hardcoded URL and executes it.</p>

<h2 id="geography-of-trickbot-attacks">Geography of Trickbot attacks</h2>
<div class="js-infogram-embed" data-id="_/5IYcPFh7x7fZV3zOkMMH" data-type="interactive" data-title="01 EN Trickbot map" style="min-height:;"></div>

<p>We analyzed Trickbot detections that occurred between January 2021 and early October 2021. Most of the affected users were located in the USA (13.21%), Australia (10.25%) and China (9.77%), followed by Mexico (6.61%) and France (6.30%).</p>
