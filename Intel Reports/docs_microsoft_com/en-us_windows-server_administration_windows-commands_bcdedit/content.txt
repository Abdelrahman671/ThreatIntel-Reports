											<h2 id="ms--in-this-article" class="title is-6 margin-block-xs">In this article</h2>
										</nav><!-- <content> --><p>Boot Configuration Data (BCD) files provide a store that is used to describe boot applications and boot application settings. The objects and elements in the store effectively replace Boot.ini.</p>
<p>BCDEdit is a command-line tool for managing BCD stores. It can be used for a variety of purposes, including creating new stores, modifying existing stores, adding boot menu parameters, and so on. BCDEdit serves essentially the same purpose as Bootcfg.exe on earlier versions of Windows, but with two major improvements:</p>
<ul>
<li><p>Exposes a wider range of boot parameters than Bootcfg.exe.</p>
</li>
<li><p>Has improved scripting support.</p>
</li>
</ul>
<div class="NOTE">
<p>Note</p>
<p>Administrative privileges are required to use BCDEdit to modify BCD.</p>
</div>
<p>BCDEdit is the primary tool for editing the boot configuration of Windows Vista and later versions of Windows. It is included with the Windows Vista distribution in the %WINDIR%\System32 folder.</p>
<p>BCDEdit is limited to the standard data types and is designed primarily to perform single common changes to BCD. For more complex operations or nonstandard data types, consider using the BCD Windows Management Instrumentation (WMI) application programming interface (API) to create more powerful and flexible custom tools.</p>
<h2 id="syntax">Syntax</h2>
<pre><code>bcdedit /command [&lt;argument1&gt;] [&lt;argument2&gt;] ...
</code></pre>
<h3 id="parameters">Parameters</h3>
<h3 id="general-bcdedit-command-line-options">General BCDEdit Command-Line Options</h3>
<table>
<thead>
<tr>
<th>Option</th>
<th>Description</th>
</tr>
</thead>
<tbody>
<tr>
<td>/?</td>
<td>Displays a list of BCDEdit commands. Running this command without an argument displays a summary of the available commands. To display detailed help for a particular command, run <strong>bcdedit /?</strong> <code>&lt;command&gt;</code>, where <code>&lt;command&gt;</code> is the name of the command you are searching for more information about. For example, <strong>bcdedit /? createstore</strong> displays detailed help for the Createstore command.</td>
</tr>
</tbody>
</table>
<h4 id="parameters-that-operate-on-a-store">Parameters that Operate on a Store</h4>
<table>
<thead>
<tr>
<th>Option</th>
<th>Description</th>
</tr>
</thead>
<tbody>
<tr>
<td>/createstore</td>
<td>Creates a new empty boot configuration data store. The created store is not a system store.</td>
</tr>
<tr>
<td>/export</td>
<td>Exports the contents of the system store into a file. This file can be used later to restore the state of the system store. This command is valid only for the system store.</td>
</tr>
<tr>
<td>/import</td>
<td>Restores the state of the system store by using a backup data file previously generated by using the <strong>/export</strong> option. This command deletes any existing entries in the system store before the import takes place. This command is valid only for the system store.</td>
</tr>
<tr>
<td>/store</td>
<td>This option can be used with most BCDedit commands to specify the store to be used. If this option is not specified, then BCDEdit operates on the system store. Running the <strong>bcdedit /store</strong> command by itself is equivalent to running the <strong>bcdedit /enum active</strong> command.</td>
</tr>

If a well-known identifier is specified, then the <strong>/application</strong>, <strong>/inherit</strong>, and <strong>/device</strong> parameters cannot be specified.
