		<div>
			<p>We recently caught a malvertising attack distributing the <a href="https://www.trendmicro.com/vinfo/us/threat-encyclopedia/malware/trojan.win32.glupteba.ta" target="_blank">malware Glupteba</a>. This is an older malware that was previously connected to a campaign named <a href="https://www.welivesecurity.com/2014/03/18/operation-windigo-the-vivisection-of-a-large-linux-server-side-credential-stealing-malware-campaign/" target="_blank">Operation Windigo</a> and distributed through exploit kits to Windows users. In 2018, a security company <a href="https://www.welivesecurity.com/2018/03/22/glupteba-no-longer-windigo/" target="_blank">reported</a> that the Glupteba botnet may have been independent from Operation Windigo and had moved to a <a href="https://blog.trendmicro.com/trendlabs-security-intelligence/pop-up-ads-and-over-a-hundred-sites-are-helping-distribute-botnets-cryptocurrency-miners-and-ransomware/" target="_blank">pay-per-install</a> adware service to distribute it in the wild. The activities of the actors behind Glupteba have been varied: they were suspected of providing proxy services in the underground, and were <a href="https://s.tencent.com/research/report/482.html" target="_blank">identified</a> as using the EternalBlue exploit to move into local networks and run Monero (XMR) cryptocurrency miners.</p>
<p>After looking into the recent variant of the Glupteba dropper delivered from the malvertising attack, we found that the dropper downloaded two undocumented components aside from the Glupteba malware:</p>
<ul>
<li><span class="rte-red-bullet">A browser stealer that can steal sensitive data, for example, browsing history, website cookies, and account names and passwords from browsers and send the information to a remote server.</span></li>
<li><span class="rte-red-bullet">A router exploiter that attacks MikroTik routers in local network with the <a href="https://blog.mikrotik.com/security/winbox-vulnerability.html" target="_blank">CVE-2018-14847</a> vulnerability. It will schedule a task on the router for command and control (C&amp;C) and upload the stolen administrator credentials to a remote server. A compromised router will be configured as a SOCKS proxy to relay malicious traffic, matching the original purpose of the Glupteba botnet on Windows.</span></li>
</ul>
<p>In addition, an interesting feature we found inside the Glupteba dropper can retrieve the latest C&amp;C domain from Bitcoin transactions. We explain this feature further in the next sections. It seems the operators are still improving their malware and may be trying to extend their proxy network to internet of things (IoT) devices.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/Glupteba-01.jpg"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/Glupteba-01.jpg" alt="Figure. 1"/></a></p>
<p style="text-align: center;"><i>Figure 1. Glupteba campaign attack flow</i></p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/Fig-2.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/Fig-2.png" alt="Figure. 2"/></a></p>
<p style="text-align: center;"><i>Figure 2. Pop-up malvertising on file-sharing websites downloaded Glupteba dropper</i></p>
<p><b><span class="body-subhead-title">Analysis of the Glupteba dropper</span></b></p>
<p>The downloaded dropper binary is packed with a custom packer, written in Go programming language, and compiled to executable. The dropper first initializes ‘config information’ by acquiring current application information, operating information, hardware information, as well as some information hardcoded in binary. It creates a registry key <i>HKEY_USERS\&lt;sid&gt;\Software\Microsoft\TestApp</i> to store all the acquired information. The result of running the config initialization function is shown in the figure below.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/Fig-3.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/Fig-3.png" alt="Figure. 3"/></a></p>
<p style="text-align: center;"><i>Figure 3. Registries created by the Glupteba dropper</i></p>
<p>Then, the function <i>sendParentProcesses</i> acquires machine_guid from the registry, as well as distributor id and campaign id from the file name, product identification (PID), and names of parent processes. It then embeds this information in a POST request, encrypts it with an AES cipher, and uploads to the C&amp;C server: <i>hxxps://&lt;server&gt;/api/parent-processes</i>.</p>
<p>After that, the dropper checks if process is elevated and running as a SYSTEM user. If process is not elevated, it tries to exploit the <a href="https://pentestlab.blog/2017/06/07/uac-bypass-fodhelper/" target="_blank">fodhelper method</a> to get it elevated. If it is elevated but not running as a SYSTEM user, it uses the “Run as Trusted Installer” method, likely inspired by <a href="https://github.com/nfedera/run-as-trustedinstaller/blob/master/run-as-trustedinstaller/main.cpp" target="_blank">this code</a>, which uses a stolen winlogon process token to run process as SYSTEM.</p>
<p>The main dropper binary has embedded a few rootkit drivers used for hiding files and processes (WinMon32.sys, WinMon64.sys, WinMonFs32.sys, WinMonFs64.sys, WinMonprocessmonitor32.sys, WinMonProcessMonitor64.sys, WinmonSystemMonitor-10-64.sys, WinmonSystemMonitor-7-10-32.sys, WinmonSystemMonitor-7-64.sys) and a few other tools taken from GitHub used to help installing the necessary drivers (<a href="https://github.com/hfiref0x/DSEFix" target="_blank">dsefix.exe</a> and <a href="https://github.com/hfiref0x/UPGDSED" target="_blank">patch.exe</a>.) Function <i>executeTask</i> processes these main commands:</p>

<p style="text-align: left;">Function <i>mainInstall</i> checks for installed antivirus (AV) programs, adds firewall rules, and adds defender exclusions.</p>
<p style="text-align: left;">Function <i>mainPoll</i> regularly polls the C&amp;C server for new commands. It sends a POST request to hxxps://&lt;server&gt;/api/poll. POST parameters look like the following (before encryption):</p>
<blockquote>challenge=e94e354daf5f48ca&amp;cloudnet_file=1&amp;cloudnet_process=1&amp;lcommand=0&amp;mrt=1&amp;pgdse=0&amp;sb=1&amp;sc=0&amp;uuid=&amp;version=145&amp;wup_process=1&amp;wupv=0.</blockquote>

<p><b><span class="body-subhead-title">Notable C&amp;C update capability</span></b></p>
<p>The backdoor mostly has standard capabilities, but one interesting feature stands out: This malware can update its C&amp;C server address through the blockchain via the function <i>discoverDomain</i>.</p>
<p>The <i>discoverDomain</i> function can be run either by sending a backdoor command, or automatically by the dropper. <i>DiscoverDomain</i> first enumerates Electrum Bitcoin wallet servers using a publicly available list, then tries to query the blockchain script hash history of the script with a hardcoded hash.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/1-body.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/1-body.png" alt="Fig 1"/></a></p>
<p>This command then reveals all the related transactions.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/2-body.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/2-body.png" alt="Fig 2"/></a></p>
<p>Then each transaction is parsed, searching for the <a href="https://en.bitcoin.it/wiki/OP_RETURN" target="_blank">OP_RETURN</a> instruction.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/3-body.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/3-body.png" alt="Fig 3"/></a></p>
<p>The pieces of data followed by OP_RETURN instruction are then used as parameters for AES decryption routine — the first 12 bytes are used as the AES GCM tag, and the following 32 bytes are the encrypted data. The 32-byte long AES key is hardcoded in binary file.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/4-body.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/4-body.png" alt="Fig 4"/></a></p>
<p>Therefore, 0f8f7cd39e1a5231b49f986b877befce0c2f558f0c1a9844833ac702cb3eba6e gets decoded to venoxcontrol[.]com, which is the current C&amp;C server at the time of writing this publication.</p>
<p>This technique makes it more convenient for the threat actor to replace C&amp;C servers. If they lose control of a C&amp;C server for any reason, they simply need to add a new Bitcoin script and the infected machines obtain a new C&amp;C server by decrypting the script data and reconnecting.</p>
<p><b><span class="body-subhead-title">Browser stealer component</span></b></p>
<p>One observed component from the recent Glupteba variant is called “updateprofile”, which is a browser profile, cookies, and password extractor. The Chrome, Opera, and Yandex browsers are targeted — cookies, history, and other profile files are zipped and uploaded to the information collection server to path <i>/api/log</i>. Similar to the main dropper, this component is also written in Go, compiled to be executable, and packed with a UPX packer.</p>
<p>Another version of the browser stealer is called “vc.exe”. Its goal is to extract browser passwords and cookies and post the extracted data to the information collection server to path <i>/bots/post-en-data?uuid=</i>.</p>
<p><b><span class="body-subhead-title">Router exploiter component</span></b></p>
<p>Another component we found downloaded by the Glupteba dropper is a router exploiter, which is also developed with Go language. It looks into the default gateway of the victim’s network. The list of default IP gateways is obtained by calling WMI command “SELECT DefaultIPGateway FROM Win32_NetworkAdapterConfiguration WHERE IPEnabled = true”.</p>
<p>In addition to these addresses, the following three default addresses are added: 192.168.88.11, 192.168.0.1, 192.168.1.1.</p>
<p>Once the component successfully connects to the device listening on port 8291, it then attempts to exploit the device with the <a href="https://nvd.nist.gov/vuln/detail/CVE-2018-14847" target="_blank">CVE-2018-14847</a> vulnerability, which affects the RouterOS system used on MikroTik routers. The exploit code was likely inspired by <a href="https://www.exploit-db.com/exploits/45170" target="_blank">this code on exploit-db</a>. It allows the attackers to grab the administrator’s credentials from unpatched routers. The grabbed account names and passwords are stored in a JSON object, encrypted, and POSTed to /api/router path of the C&amp;C server.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/5-body.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/5-body.png" alt="Fig 5"/></a></p>
<p>Once credentials are successfully obtained, a task is added to the scheduler of the router. There are three methods implemented to add a scheduler task: using WinBox protocol, using SSH, or using API.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/Fig-4.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/Fig-4.png" alt="Figure. 4"/></a></p>
<p style="text-align: center;"><i>Figure 4. Example of scheduled task added by Glupteba campaign to compromised MikroTik routers</i></p>
<p>The router exploiter component scheduled a task named “U6” on compromised routers for command and control. The task will regularly check a C&amp;C URL every 10 minutes and execute the content downloaded from it. The C&amp;C URL is appended with a unique UUID, which is the same as the bot ID of Glupteba on the victim’s machine. The URL usually returns an HTTP 404 error; but when the bot master sends a command, it returns an <a href="https://wiki.mikrotik.com/wiki/Manual:Configuration_Management" target="_blank">RSC</a> file, which is the format of the RouterOS configuration.</p>
<p>The file is the command from the bot master to control the router. In the attack we analyzed, we saw the C&amp;C server send multiple RSC files step by step to configure the compromised router to be a SOCKS proxy. Here are the steps:</p>
<ul>
<li><span class="rte-red-bullet">The first configuration changed the period of “U6” task schedule from 10 minutes to every 15 seconds.</span></li>
</ul>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/6-body.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/6-body.png" alt="Fig 6"/></a></p>
<ul>
<li><span class="rte-red-bullet">The second configuration disabled services, including winbox, telnet, api, and api-ssl. This is most likely to prevent the router from being compromised by other attackers through the same vulnerability. Then it opened the SSH and SOCKS services, which are listening on randomly assigned ports, and created a firewall rule to accept external connection to the SOCKS port.</span></li>
</ul>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/7-body.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/7-body.png" alt="Fig 7"/></a></p>
<ul>
<li><span class="rte-red-bullet">The third configuration removed the existing SOCKS access list on the compromised router.</span></li>
</ul>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/8-body.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/8-body.png" alt="Fig 8"/></a></p>
<ul>
<li><span class="rte-red-bullet">The fourth configuration added a new SOCKS access list to limit the service so that it only accepts connections from specified IP ranges. These ranges are probably where the attackers’ servers are.</span></li>
</ul>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/9-body.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/9-body.png" alt="Fig 9"/></a></p>
<p><b><span class="body-subhead-title">Relayed traffic on compromised routers</span></b></p>
<p>After the above-mentioned setups, the compromised router became a SOCKS proxy for the attackers to relay traffic. We monitored a compromised router to see what kind of traffic is transferred. The first remote connection routed through the SOCKS proxy is from a server, which likely belongs to the attackers. This server queries “http://ip-api[.]com/json”, which returns the IP address of the current SOCKS proxy server. This query is sent repeatedly, probably to monitor the SOCKS proxy service.</p>
<p>After the first check on the router status, we started seeing different servers with two types of traffic connected to the proxy. The first one is spam traffic. We saw a remote server establish <a href="https://en.wikipedia.org/wiki/Simple_Mail_Transfer_Protocol" target="_blank">SMTP</a> connections to different mail servers through the SOCKS proxy of compromised routers. If a mail server accepted the connection, that remote server started to send spam mail. The spam mail delivered seems to be related to the notorious “<a href="https://krebsonsecurity.com/2011/01/battling-the-zombie-web-site-armies/" target="_blank">Canadian Pharmacy</a>” spam.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/Fig-5.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/Fig-5.png" alt="Figure. 5"/></a></p>
<p style="text-align: center;"><i>Figure 5. Example of spam traffic sent through a compromised router</i></p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/Fig-6.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/Fig-6.png" alt="Figure. 6"/></a></p>
<p style="text-align: center;"><i>Figure 6. “Canada Pharmacy” website redirected from spam mail</i></p>
<p>Besides the spam traffic, we saw other traffic from a set of remote servers that were repeatedly connecting to Instagram. However, the traffic sent through was protected by HTTPS encryption. We can’t decrypt it and don’t know what exactly these connections are for. One theory is that it is the password-reuse attack hitting Instagram. It was previously <a href="https://www.welivesecurity.com/2018/03/22/glupteba-no-longer-windigo/" target="_blank">reported</a> to be one type of malicious traffic proxied through the Glupteba botnet.</p>
<p style="text-align: center;"><a href="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/Fig-7.png"><img src="/content/dam/trendmicro/global/en/migrated/security-intelligence-migration-spreadsheet/trendlabs-security-intelligence/2019/09/Fig-7.png" alt="Figure. 7"/></a></p>
<p style="text-align: center;"><i>Figure 7. Example of Instagram connection (HTTPS-encrypted) relayed by a compromised router</i></p>
<p>As mentioned, Glupteba still seems to be evolving and adding capabilities. New techniques, such as updating C&amp;C servers through data obtained from Bitcoin transactions, show that the malicious actors behind this malware are adopting little-used techniques to try and keep their malware active. Since it is already proven to be an information stealer and a proxy for malicious spam, users and enterprises should be wary of this threat.</p>
<p><b><span class="body-subhead-title">Security recommendations</span></b></p>
<p>Malvertising is a widespread threat that can affect users and businesses alike. A multilayered approach to security is important — from the gateway, endpoints, networks, and servers. Trend Micro solutions powered by XGen™ security, such as <a href="/en_us/forHome/products.html" target="_blank">Trend Micro™ Security</a> and <a href="/en_us/business/products/network.html" target="_blank">Trend Micro Network Defense</a>, can detect related malicious files and URLs and protect users’ systems. <a href="/en_us/business/products/user-protection/sps.html" target="_blank">Trend Micro Smart Protection Suites</a> and <a href="/en_us/small-business.html" target="_blank">Trend Micro Worry-Free™ Business Security</a>, which have <a href="https://success.trendmicro.com/solution/1122593-configuring-behavior-monitoring-settings-in-apex-one" target="_blank">behavior monitoring capabilities</a>, can additionally protect from these types of threats by detecting malicious files, as well as blocking all related malicious URLs.</p>
<p>Security should be top of mind when setting up routers — most devices across homes and offices are connected to these devices and can be affected if a router is compromised. Although manufacturers <a href="https://blog.trendmicro.com/trendlabs-security-intelligence/internet-things-ecosystem-broken-fix/" target="_blank">play</a> important roles in securing routers and other devices, users and businesses can adopt <a href="https://www.trendmicro.com/vinfo/us/security/news/internet-of-things/protect-home-network-securing-router" target="_blank">good security practices</a> to defend against threats. Also, deploying tools that provide <a href="/en_us/forHome/products/homenetworksecurity.html" target="_blank">additional security</a> to home networks and devices connected to them further strengthens defenses.</p>
<p>For a full list of the Indicators of Compromise for this malware, please see this <a href="https://documents.trendmicro.com/images/TEx/pdf/Appendix%20Glupteba%20Campaign%20Hits%20Network%20Routers%20and%20Updates%20C&amp;C%20Servers%20with%20Data%20from%20Bitcoin%20Transactions.pdf">document</a>.</p>
