<p>Often, malware uses platforms like — Telegram and Discord for data exfiltration. Due to its simplicity and the lack of need for building a server architecture, this exfiltration method has gained significant popularity. However, this very simplicity is also its weakness.&nbsp;</p>

<p>Now, let&#8217;s use the attacker&#8217;s chat_id and bot token. The chat_id can refer to either a group chat or direct messages.&nbsp;First, we check if the bot has a webhook:&nbsp;</p>



<p>https://api.telegram.org/bot&lt;token&gt;/getWebhookInfo</p>



<p>The presence of a webhook means a high chance of early detection of abuse.</p>


<div class="wp-block-image">

<p>If a webhook is present, we save its data and delete it using /deleteWebhook.&nbsp;</p>



<p>NOTE! The webhook may have a secret token which could reveal the substitution.&nbsp;</p>

<h2 class="wp-block-heading">How to Copy the Entire Chat&nbsp;</h2>



<p>If you want to copy a chat entirely, you need to understand how message_id works.&nbsp;</p>



<p>This id is actually the index of the message.&nbsp;</p>



<p>For private chats and each group (group/supergroup), the indices run in parallel.&nbsp;Message_id for private chats is shared across all chats with users.&nbsp;With each message received from an individual user or sent to an individual user, the message_id increments by one.&nbsp;</p>



<p>Thus, the first message in a chat with one user might have a message_id of 4096, even though in the context of the chat it should have a message_id of 1.&nbsp;In groups, however, message_id works as expected, starting from 1.&nbsp;</p>

<li>If it is a group (group/supergroup), there shouldn&#8217;t be any significant issues&nbsp;</li>

</ul>



<p>There is no simple way to retrieve all messages from a chat; some messages may not be accessible to the bot, but it will definitely have access to the messages it has sent.</p>



<p>In the malware request, we can see the message_id of the message from the malware, allowing us to estimate the number of messages.&nbsp;</p>



<p>Next, we iterate through all messages from 1 to the required number.&nbsp;Telegram allows for a stable rate of 20 requests per minute with short bursts.&nbsp;</p>



<p>To copy multiple messages at once, you can use /forwardMessages, which allows copying up to 100 messages in a single request.&nbsp;Thus, in one minute, you can stably copy 2000 messages or more if you utilize bursts.&nbsp;</p>



<p>Using a Python script, we can copy the entire chat</p>



<p>We recommend saving the server responses, as they contain additional data useful for research: the date of the original message, its ID, and the ID of the original chat.&nbsp;</p>

<h2 class="wp-block-heading">Parsing Discord&nbsp;</h2>



<p>Replicating the same method with Discord is challenging due to the use of webhooks.&nbsp;</p>



<p>A Discord webhook allows sending messages to a chat for which it was designated. Retrieving a message without knowing the message_id is difficult because Discord uses a <a href="https://en.wikipedia.org/wiki/Snowflake_ID" target="_blank" rel="noreferrer noopener">snowflake ID</a>, which includes the timestamp of the message and service information for identification.&nbsp;</p>



<p>The only known message IDs for you will be those you managed to intercept.&nbsp;</p>



<p>Among the methods that can be executed directly in the browser, there are only two:&nbsp;</p>



<ol class="wp-block-list" start="1">
<li>Retrieving webhook data:&nbsp;</li>
</ol>



<p>https://discord.com/api/webhooks/&lt;webhook_id&gt;/&lt;webhook_token&gt;</p>



<ol class="wp-block-list" start="2">
<li>Retrieving a message:&nbsp;</li>
</ol>



<p>https://discord.com/api/webhooks/&lt;webhook_id&gt;/&lt;webhook_token&gt;/messages/&lt;message_id&gt;</p>



<p>For example, consider the <a href="https://app.any.run/tasks/189ce54d-7b1a-4d6f-a3ab-c6ea88d1aa5b/?utm_source=anyrunblog&amp;utm_medium=article&amp;utm_campaign=intercepting_malware_data&amp;utm_term=250924&amp;utm_content=linktoservice" target="_blank" rel="noreferrer noopener">following sandbox session</a>.&nbsp;</p>



<p>We once again run it with the MITM Proxy enabled.</p>
