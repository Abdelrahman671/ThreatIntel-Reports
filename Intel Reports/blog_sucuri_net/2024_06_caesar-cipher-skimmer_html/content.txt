We recently wrote about this plugin being <a href="https://blog.sucuri.net/2024/04/javascript-malware-switches-to-server-side-redirects-dns-txt-records-tds.html">misused</a> by attackers to insert server-side redirects within website code.</p><p>On Magento websites they appear to be using their old favourite <b>core_config_data</b> database table. Credit card skimming JavaScript is found there a lot because that is the database table that custom code inserted into the Magento admin panel is stored.</p><p>As for OpenCart, we&#8217;ve not yet had a client come through with this particular infection using OpenCart, so we&#8217;re not entirely sure where in the back end it is getting lodged, but we&#8217;ll update this post if we&#8217;re able to determine that.</p><h2 id="mitigation">Mitigation steps</h2><p>If you are a website owner <span class="BxUVEf ILfuVd" lang="en"><span class="hgKElc">—</span></span> particularly if you manage an ecommerce website <span class="BxUVEf ILfuVd" lang="en"><span class="hgKElc">—</span></span> let this analysis serve as a reminder to always keep website security as a priority. You don&#8217;t want to be receiving a call from Visa or Mastercard having identified your website as a common point of purchase for stolen cards.</p><p>Let&#8217;s take a look at some steps you can take to protect your ecommerce site from credit card skimmers:</p><ul><li aria-level="1"><b>Keep your site up to date</b>. The most common method of infection is via outdated software: attackers are always checking for vulnerabilities in outdated plugins and themes and leverage automation to quickly exploit vulnerable environments. You can prevent this by simply <a href="https://blog.sucuri.net/2022/09/a-guide-to-virtual-patching-for-website-vulnerabilities.html" target="_blank" rel="noopener">patching your site</a> and ensuring software is using the latest security updates. Or, use a <a href="https://sucuri.net/website-firewall/" target="_blank" rel="noopener">WAF</a> to virtually patch against known vulnerabilities.</li><li aria-level="1"><b>Review admin accounts and keep passwords updated.</b> Attackers commonly gain access via accounts with weak passwords. Once they have gained access using an admin account, they can make unlimited changes to the site content. Attackers will often add additional admin accounts that they can use later. It is important to routinely review the admin accounts that are configured to confirm they are all valid, and to change passwords on a regular basis. Ensure all <a href="https://blog.sucuri.net/2024/01/how-to-make-strong-password.html" target="_blank" rel="noopener">passwords are strong</a> and unique from other credentials.</li><li aria-level="1"><b>Leverage file integrity and website monitoring.</b> <a href="https://blog.sucuri.net/2021/05/server-side-scans-and-file-integrity-monitoring.html">File integrity monitoring</a> checks and compares files against a known baseline to detect any suspicious or unexpected changes. This essentially functions as an early detection system. If a hacker gains access to your website and modifies files, or if malware alters your site’s code, it will flag these changes, allowing for rapid response and minimizing potential damage.</li><li aria-level="1"><b>Protect your site with a web application firewall. </b>A properly configured <a href="https://sucuri.net/website-firewall/" target="_blank" rel="noopener">website firewall</a> can help to block malicious traffic by preventing hack attempts from reaching the hosting server.</li></ul><p>If you believe that your site is infected with skimmers or other malware, reach out or <a href="https://sucuri.net/live-chat/">start a chat</a>!
