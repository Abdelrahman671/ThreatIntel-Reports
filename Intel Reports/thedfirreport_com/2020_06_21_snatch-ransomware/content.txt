			<h1 class="entry-title">Snatch Ransomware</h1>
		<div class="entry-meta">
			<span class="posted-on"><a href="https://thedfirreport.com/2020/06/21/snatch-ransomware/" rel="bookmark"><time class="entry-date published" datetime="2020-06-21T22:58:39+00:00">June 21, 2020</time></a></span>		</div><!-- .entry-meta -->

		
	</header><!-- .entry-header -->

	<div class="entry-content">
		
<p>Another RDP brute force ransomware strikes again, this time, Snatch Team! Snatch Team was able to go from brute forcing a Domain Administrator (DA) account via RDP, to running a Meterpreter reverse shell and a RDP proxy via Tor on a Domain Controller (DC), to encrypting all Domain joined systems in under 5 hours.</p>
<p><img fetchpriority="high" decoding="async" class="alignnone size-large wp-image-532" src="https://thedfirreport.com/wp-content/uploads/2020/06/Snach-Ransomware-Event-UTC-1024x564.png" alt="" width="1024" height="564" srcset="https://thedfirreport.com/wp-content/uploads/2020/06/Snach-Ransomware-Event-UTC-1024x564.png 1024w, https://thedfirreport.com/wp-content/uploads/2020/06/Snach-Ransomware-Event-UTC-300x165.png 300w, https://thedfirreport.com/wp-content/uploads/2020/06/Snach-Ransomware-Event-UTC-768x423.png 768w, https://thedfirreport.com/wp-content/uploads/2020/06/Snach-Ransomware-Event-UTC.png 1178w" sizes="(max-width: 1024px) 100vw, 1024px" /></p>



<p>Snatch is a widely known variant due to it causing systems to reboot into safe mode before encrypting the system. <a href="https://news.sophos.com/en-us/2019/12/09/snatch-ransomware-reboots-pcs-into-safe-mode-to-bypass-protection/" target="_blank" rel="noreferrer noopener">SophosLabs</a> has an excellent write up on Snatch which was very similar to what we witnessed.</p>



<h2 class="has-text-align-left wp-block-heading">Initial Access:</h2>



<p>Snatch Team logged into a DA account from 193.70.12.240 around 0515 UTC. Initially with that access they performed a simple arp -a.</p>
<p>At 0753 UTC the threat actors made the next move running ipconfig and quser. Just minutes later they began lateral movement initiating an RDP session with a DC.</p>
<h2>Lateral Movement and Persistence:</h2>
<p>Once on the DC the threat actor moved quickly deploying a tool set in C:\Windows. This tool set included 2 executable that masqueraded as Windows Management Instrumentation files. One was executed with the following command parameters.</p>
<p><img decoding="async" class="alignnone wp-image-518 size-full" src="https://thedfirreport.com/wp-content/uploads/2020/06/snatch1.png" alt="" width="863" height="217" srcset="https://thedfirreport.com/wp-content/uploads/2020/06/snatch1.png 863w, https://thedfirreport.com/wp-content/uploads/2020/06/snatch1-300x75.png 300w, https://thedfirreport.com/wp-content/uploads/2020/06/snatch1-768x193.png 768w" sizes="(max-width: 863px) 100vw, 863px" /></p>
<p>The .dat file turned out to be a configuration file with the executable being TOR creating an RDP tunnel. (Wouldn&#8217;t this be really really slow?)</p>
<p><img decoding="async" class="alignnone wp-image-519 size-full" src="https://thedfirreport.com/wp-content/uploads/2020/06/snatch2.png" alt="" width="1469" height="240" srcset="https://thedfirreport.com/wp-content/uploads/2020/06/snatch2.png 1469w, https://thedfirreport.com/wp-content/uploads/2020/06/snatch2-300x49.png 300w, https://thedfirreport.com/wp-content/uploads/2020/06/snatch2-1024x167.png 1024w, https://thedfirreport.com/wp-content/uploads/2020/06/snatch2-768x125.png 768w" sizes="(max-width: 1469px) 100vw, 1469px" /></p>
<p>The other executable file in the wmis folder was a Go executable of unknown providence potentially related to utorrent capability?</p>





<p>The next thing they did was create a reverse shell using what we think is Meterpreter. C2 initiated over HTTPS/443 to 91.229.77.161 via cplXen.exe</p>
<p>The presence of logs indicating the use of <a href="https://blog.cobaltstrike.com/2014/04/02/what-happens-when-i-type-getsystem/" target="_blank" rel="noopener noreferrer">named pipe services</a> also increases the likelihood of Meterpreter or possibly Cobaltstrike. We didn&#8217;t see any ET Pro signatures fire for this activity but we also didn&#8217;t have SSL inspection on at the time.</p>



<figure class="wp-block-image"><img decoding="async" src="https://thedfirreport.com/wp-content/uploads/2020/06/image-17.png" alt="This image has an empty alt attribute; its file name is image-17.png" /></figure>



<p>A separate executable was then dropped for stealthy persistence of cplXen.exe. X3.exe is a loader that uses the 3 DLLs (which are ini files) below to run cplXen.</p>
<p><img loading="lazy" decoding="async" class="alignnone wp-image-522 size-full" src="https://thedfirreport.com/wp-content/uploads/2020/06/snatch4.png" alt="" width="1546" height="369" srcset="https://thedfirreport.com/wp-content/uploads/2020/06/snatch4.png 1546w, https://thedfirreport.com/wp-content/uploads/2020/06/snatch4-300x72.png 300w, https://thedfirreport.com/wp-content/uploads/2020/06/snatch4-1024x244.png 1024w, https://thedfirreport.com/wp-content/uploads/2020/06/snatch4-768x183.png 768w, https://thedfirreport.com/wp-content/uploads/2020/06/snatch4-1536x367.png 1536w" sizes="auto, (max-width: 1546px) 100vw, 1546px" /></p>
<pre>jd4ob7162ns.dll: C:\windows\system32\cplXen.exe /F<br />fw0a53482aa.dll: 443<br />kb05987631s.dll: 91.229.77.161</pre>
<p>Two Scheduled Tasks were created to launch the loader, which in turn persists the loading of cplXen.exe.</p>
