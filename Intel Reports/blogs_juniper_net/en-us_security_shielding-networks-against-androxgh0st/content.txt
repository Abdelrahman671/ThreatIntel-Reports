<p><span data-contrast="none">AndroxGh0st is a Python-based malware designed to target Laravel applications. It works by scanning and taking out important information from .env files, revealing login details linked to AWS and Twilio. Classified as an SMTP cracker, it exploits SMTP using various strategies such as credential exploitation, web shell deployment and vulnerability scanning. <p><span data-contrast="none">Laravel is an open-source PHP web application development framework known for its MVC architecture. It simplifies development with built-in features for database interaction, authentication, routing, sessions and caching. Laravel is widely used for designing web applications such as e-commerce platforms, social networking platforms, APIs (Application Programming Interface)</span><span data-contrast="none">,</span> <span data-contrast="none">and Content Management Systems (CMS).</span><span data-ccp-props="{&quot;201341983&quot;:0,&quot;335551550&quot;:6,&quot;335551620&quot;:6,&quot;335559685&quot;:-20,&quot;335559737&quot;:-20,&quot;335559739&quot;:160,&quot;335559740&quot;:259}"> </span></p>

<h2><b><span data-contrast="none"><br />

<p><span data-contrast="auto">CVE-2018-15133, recognized as the &#8220;Laravel App Key Deserialization RCE,&#8221; unveils a critical vulnerability affecting Laravel applications within the version spectrum of 5.5.40 to 5.6.29. This exploit allows malevolent actors to execute unauthorized code on vulnerable systems, posing a substantial risk of complete compromise.</span><span data-ccp-props="{&quot;201341983&quot;:0,&quot;335551550&quot;:6,&quot;335551620&quot;:6,&quot;335559737&quot;:-20,&quot;335559739&quot;:0,&quot;335559740&quot;:276}"> </span></p>
<p><b><span data-contrast="auto">Exploiting the Deserialization Weakness:</span></b><span data-contrast="auto"> The vulnerability resides in the Illuminate\Encryption\Encrypter class, responsible for encrypting and decrypting data in Laravel. By leveraging the unserialize function for handling serialized data, a process fraught with risks if not meticulously validated, attackers find a gateway to unauthorized exe</span><span data-contrast="none">cution.</span><span data-ccp-props="{&quot;201341983&quot;:0,&quot;335551550&quot;:6,&quot;335551620&quot;:6,&quot;335557856&quot;:16777215,&quot;335559685&quot;:-20,&quot;335559737&quot;:-20,&quot;335559739&quot;:0,&quot;335559740&quot;:259}"> </span></p>
<p><b><span data-contrast="auto">Crafting a Malicious Payload:</span></b><span data-contrast="auto"> Perpetrators can construct a malevolent payload featuring a specially tailored XSRF-TOKEN value. The success of this exploit hinges on the lack of proper validation, enabling the crafted payload to undergo deserialization by the Encrypter class.</span><span data-ccp-props="{&quot;201341983&quot;:0,&quot;335551550&quot;:6,&quot;335551620&quot;:6,&quot;335557856&quot;:16777215,&quot;335559685&quot;:-20,&quot;335559737&quot;:-20,&quot;335559739&quot;:0,&quot;335559740&quot;:259}"> </span></p>
<p><b><span data-contrast="auto">Executing the Exploit:</span></b><span data-contrast="auto"> The deserialization process, triggered by the manipulated payload, leads to the execution of arbitrary code embedded within the token. Notably, successful exploitation of this vulnerability requires the attacker to possess the API Key.</span><span data-ccp-props="{&quot;201341983&quot;:0,&quot;335551550&quot;:6,&quot;335551620&quot;:6,&quot;335557856&quot;:16777215,&quot;335559685&quot;:-20,&quot;335559737&quot;:-20,&quot;335559739&quot;:0,&quot;335559740&quot;:259}"> </span></p>

<h2><b></b><strong>CVE-2021-41773 (Apache HTTP Server RCE):  </strong></h2>
<p><span data-contrast="auto">A </span><span data-contrast="auto">flaw </span><span data-contrast="auto">in Apache httpd introduces a vulnerability facilitating directory traversal. This flaw appears from the normalization process of the URI path in HTTP requests. The ap_normalize_path() function, responsible for resolving &#8220;../&#8221; sequences by traversing to the parent directory, fails to interpret the sequence when the second dot character (&#8216;.&#8217;) is URL-encoded (e.g., &#8220;. %2e/&#8221;). Consequently, the &#8220;../&#8221; sequence persists in the normalized path.</span><span data-ccp-props="{&quot;201341983&quot;:0,&quot;335551550&quot;:6,&quot;335551620&quot;:6,&quot;335557856&quot;:16777215,&quot;335559739&quot;:0,&quot;335559740&quot;:259}"> </span></p>
<p><span data-contrast="auto">Upon combining this path with the server root path using the </span><i><span data-contrast="auto">apr_filepath_merge()</span></i><span data-contrast="auto"> function, the resulting path stored in the filename field of the </span><i><span data-contrast="auto">request_rec</span></i><span data-contrast="auto"> structure may traverse beyond the server root path. If the server&#8217;s root directory is accessible, and the </span><i><span data-contrast="auto">mod_cgi</span></i><span data-contrast="auto"> module is enabled, a remote attacker could invoke an arbitrary executable on the server, leading to arbitrary code execution.</span><span data-ccp-props="{&quot;201341983&quot;:0,&quot;335551550&quot;:6,&quot;335551620&quot;:6,&quot;335557856&quot;:16777215,&quot;335559739&quot;:0,&quot;335559740&quot;:259}"> </span></p>
<p><span data-contrast="auto">To exploit this vulnerability, a remote attacker could dispatch a request with a specially crafted URI to the target service. Successful exploitation could unveil the contents of arbitrary files on the server. If the </span><i><span data-contrast="auto">mod_cgi</span></i><span data-contrast="auto"> module is enabled, the attacker gains the ability to execute arbitrary code on the target server.</span><span data-ccp-props="{&quot;201341983&quot;:0,&quot;335551550&quot;:6,&quot;335551620&quot;:6,&quot;335557856&quot;:16777215,&quot;335559739&quot;:0,&quot;335559740&quot;:259}"> </span></p>
