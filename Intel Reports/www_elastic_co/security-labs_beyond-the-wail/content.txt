lg:col-span-3"><div class="prose lg:prose-lg prose-invert w-full article-content"><div><h2 class="font-bold text-2xl md:text-4xl relative"><span id="preamble" class="absolute -top-32"></span>Preamble</h2>
<p>In August 2024, a novel macOS malware named &quot;BANSHEE Stealer&quot; emerged, catching the attention of the cybersecurity community. Reportedly developed by Russian threat actors, BANSHEE Stealer was introduced on an underground forum and is designed to function across both macOS x86_64 and ARM64 architectures.</p>
<p>This malware presents a severe risk to macOS users, targeting vital system information, browser data, and cryptocurrency wallets.</p>
<p>With a steep monthly subscription price of $3,000, BANSHEE Stealer stands out in the market, particularly compared to known stealers like AgentTesla.</p>
<p>As macOS increasingly becomes a prime target for cybercriminals, BANSHEE Stealer underscores the rising observance of macOS-specific malware. This analysis explores the technical details of BANSHEE Stealer, aiming to help the community understand its impact and stay informed about emerging threats.</p>
<p></p>
<h2 class="font-bold text-2xl md:text-4xl relative"><span id="key-takeaways" class="absolute -top-32"></span>Key takeaways</h2>
<ul>
<li>BANSHEE Stealer highlights the growing number of macOS malware samples as the OS becomes a more attractive target for cyber threats.</li>
<li>BANSHEE Stealer&#x27;s $3,000 monthly price is notably high compared to Windows-based stealers.</li>
<li>BANSHEE Stealer targets a wide range of browsers, cryptocurrency wallets, and around 100 browser extensions, making it a highly versatile and dangerous threat.</li>
</ul>
<h2 class="font-bold text-2xl md:text-4xl relative"><span id="malware-analysis" class="absolute -top-32"></span>Malware Analysis</h2>
<p>The malware we analyzed in this research contained all the C++ symbols, which is interesting as we can guess the project&#x27;s code structure by knowing these source code file names, as seen in the picture below. Looking into the C++-generated global variable initialization functions, we can find values set automatically/manually by the user during the build process, like the remote IP, encryption key, build ID, etc.</p>

<h3 class="font-bold leading-tight text-xl md:text-3xl relative"><span id="debugger-vm-detection-and-language-checks" class="absolute -top-32"></span>Debugger, VM Detection, and Language Checks</h3>
<p></p>
<p>BANSHEE Stealer uses basic techniques to evade detection. It detects debugging by utilizing the <a href="https://developer.apple.com/library/archive/documentation/System/Conceptual/ManPages_iPhoneOS/man3/sysctl.3.html">sysctl</a> API.</p>
<p></p>
<p>For virtualization detection, it runs the command <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">system_profiler SPHardwareDataType | grep &#x27;Model Identifier&#x27;</code> to determine whether the string <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">Virtual</code> appears in the hardware model identifier, which suggests a virtual machine. These methods are relatively simple and can be easily circumvented by advanced sandboxes and malware analysts.</p>
<p></p>
<p>Additionally, It parses the user-preferred canonicalized language returned from the <a href="https://developer.apple.com/documentation/corefoundation/1542887-cflocalecopypreferredlanguages">CFLocaleCopyPreferredLanguages </a> API and looks for the string <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">ru</code>. This tactic helps the malware avoid infecting systems where Russian is the primary language.</p>
<h3 class="font-bold leading-tight text-xl md:text-3xl relative"><span id="system-information-collection" class="absolute -top-32"></span>System information collection</h3>
<h4 class="font-bold leading-tight text-lg md:text-2xl relative"><span id="user-password" class="absolute -top-32"></span>User password</h4>
<p>The malware creates an <a href="https://ss64.com/mac/osascript.html">Osascript</a> password prompt with a dialog saying that to launch the application, you need to update the system settings. Please enter your password.</p>
<p>When the user enters the password, it will be validated using the <a href="https://ss64.com/mac/dscl.html">dscl</a> command by running <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">dscl Local/Default -authonly &lt;username&gt; &lt;password&gt;</code></p>
<p>If valid, the password will be written to the following file <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">/Users/&lt;username&gt;/password-entered</code>.</p>
<p></p>
<p>These credentials can be leveraged to decrypt the keychain data stored on the system, granting access to all saved passwords.</p>
<h4 class="font-bold leading-tight text-lg md:text-2xl relative"><span id="file-software-and-hardware-information-collection" class="absolute -top-32"></span>File, software, and hardware information collection</h4>
<p>The function <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">System::collectSystemInfo</code> collects system information and serializes it in a JSON object. It executes the command <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">system_profiler SPSoftware DataType SPHardwareDataType</code>, which provides details about the system’s software and hardware. It gets the machine&#x27;s public IP by requesting it from <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">freeipapi.com</code> through the built-in macOS <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">cURL</code> command.</p>
<p>The JSON file will be saved under <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">&lt;temporary_path&gt;/system_info.json</code></p>
<p>BANSHEE stealer executes AppleScripts; interestingly, it writes the AppleScripts to the same file <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">/tmp/tempAppleScript</code>.</p>
<p>The first script to be executed first mutes the system sound with <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">osascript -e &#x27;set volume with output muted&#x27;</code> command. It then collects various files from the system, which are listed below:</p>
<ul>
<li>Safari cookies</li>
<li>Notes database</li>
<li>Files with the following extensions <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">.txt</code>, <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">.docx</code>, <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">.rtf</code>, <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">.doc</code>, <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">.wallet</code>, <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">.keys</code>, or <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">.key</code> from the Desktop and Documents folders.</li>
</ul>
<h4 class="font-bold leading-tight text-lg md:text-2xl relative"><span id="dump-keychain-passwords" class="absolute -top-32"></span>Dump keychain passwords</h4>
<p>It copies the keychain of the system <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">/Library/Keychains/login.keychain-db</code> to <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">&lt;temporary_path&gt;/Passwords</code></p>
<h3 class="font-bold leading-tight text-xl md:text-3xl relative"><span id="browser-collection" class="absolute -top-32"></span>Browser collection</h3>
<p>BANSHEE collects data from 9 different browsers currently, including browser history, cookies, logins, etc:</p>
<ul>
<li>Chrome</li>
<li>Firefox</li>
<li>Brave</li>
<li>Edge</li>
<li>Vivaldi</li>
<li>Yandex</li>
<li>Opera</li>
<li>OperaGX</li>
</ul>
<p>Regarding Safari, only the cookies are collected by the AppleScript script for the current version.</p>
<p></p>
<p>Additionally, data from approximately 100 browser plugins are collected from the machine. A list of these extension IDs is provided at the end of the blog post.</p>
<p>The collected files are saved under <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">&lt;temporary_path&gt;/Browsers</code>.</p>
<h3 class="font-bold leading-tight text-xl md:text-3xl relative"><span id="wallet-collection" class="absolute -top-32"></span>Wallet collection</h3>
<ul>
<li>Exodus</li>
<li>Electrum</li>
<li>Coinomi</li>
<li>Guarda</li>
<li>Wasabi Wallet</li>
<li>Atomic</li>
<li>Ledger</li>
</ul>
<p>The collected wallets are stored under <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">&lt;temporary_path&gt;/Wallets</code>.</p>
<h3 class="font-bold leading-tight text-xl md:text-3xl relative"><span id="exfiltration" class="absolute -top-32"></span>Exfiltration</h3>
<p>After the malware finishes collecting data, it first ZIP compresses the temporary folder using the <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">ditto</code> command. The zip file is then XOR encrypted and base64 encoded and sent through a post request to the URL: <code class="px-1.5 py-1 rounded not-prose bg-[var(--tw-prose-invert-pre-bg)] whitespace-break-spaces text-[85%] text-emerald-600">http://45.142.122[.]92/send/</code> with the built-in cURL command.</p>

<h2 class="font-bold text-2xl md:text-4xl relative"><span id="conclusion" class="absolute -top-32"></span>Conclusion</h2>
<p>BANSHEE Stealer is macOS-based malware that can collect extensive data from the system, browsers, cryptocurrency wallets, and numerous browser extensions. Despite its potentially dangerous capabilities, the malware&#x27;s lack of sophisticated obfuscation and the presence of debug information make it easier for analysts to dissect and understand. While BANSHEE Stealer is not overly complex in its design, its focus on macOS systems and the breadth of data it collects make it a significant threat that demands attention from the cybersecurity community.</p>
