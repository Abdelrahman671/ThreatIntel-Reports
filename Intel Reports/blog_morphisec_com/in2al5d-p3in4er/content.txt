<h2 aria-level="2"><strong><span data-contrast="auto">Technical Analysis</span></strong><span data-ccp-props="{&quot;134245418&quot;:true,&quot;134245529&quot;:true,&quot;201341983&quot;:0,&quot;335559738&quot;:40,&quot;335559739&quot;:0,&quot;335559740&quot;:259}">&nbsp;</span></h2>
<p><span data-contrast="auto">This loader uses a surprisingly simple yet highly effective evasion technique. It leverages the usage of CreateDXGIFactory function of dxgi.dll library to query the vendor ID of the graphics card installed on a system. The ID is then compared against a whitelist of vendor IDs; specifically, the 0x10de (NVIDIA), 0x1002 (AMD), and 0x8086 (Intel) graphics cards. If the vendor ID doesn’t match the whitelisted values, the loader makes it appear as a benign application by terminating itself.</span><span data-ccp-props="{&quot;201341983&quot;:0,&quot;335559739&quot;:160,&quot;335559740&quot;:259}">&nbsp;</span></p>
<p style="text-align: center;"><span><span role="presentation"><img unselectable="on" src="https://blog.morphisec.com/hubfs/undefined-Apr-18-2023-12-31-35-8313-AM.png" loading="lazy" alt="The anti-VM function checks the graphics vendor ID&nbsp;"></span><br></span><em>The anti-VM function checks the graphics vendor ID&nbsp;</em></p>
<p><span data-contrast="auto">After checking the vendor IDs, the loader decrypts the final payload in separate chunks and injects it into `sihost.exe` using a process hollowing technique.</span><span data-ccp-props="{&quot;201341983&quot;:0,&quot;335559739&quot;:160,&quot;335559740&quot;:259}">&nbsp;</span></p>
<p style="text-align: center;"><span><span role="presentation"><img unselectable="on" src="https://blog.morphisec.com/hubfs/undefined-Apr-18-2023-12-32-29-0930-AM.png" loading="lazy" alt="Payload decryption routine&nbsp;"></span><span data-contrast="auto" xml:lang="EN-US" lang="EN-US"></span>&nbsp;<br></span><em>Payload decryption routine&nbsp;</em></p>
<p><span data-contrast="auto">Some samples of the loader do not use the process hollowing technique. Instead, they allocate memory to write the decrypted payload into the allocated memory, and then make a call to the payload entry point (EP).</span><span data-ccp-props="{&quot;201341983&quot;:0,&quot;335559739&quot;:160,&quot;335559740&quot;:259}">&nbsp;</span></p>
<p style="text-align: center;"><span><span role="presentation"><img unselectable="on" src="https://blog.morphisec.com/hubfs/undefined-Apr-18-2023-12-33-06-4246-AM.png" loading="lazy" alt="Payload injection&nbsp;"></span><span data-contrast="auto" xml:lang="EN-US" lang="EN-US"></span> </span><em>Payload injection&nbsp;</em></p>
<p><span data-contrast="auto">During the injection process, all loader samples resolve the necessary Win APIs dynamically and decrypt these names using a XOR key: "</span><strong><span data-contrast="auto">in2al5d p3in4er</span></strong><span data-contrast="auto">" (invalid printer), as illustrated in the figure below.</span><span data-ccp-props="{&quot;201341983&quot;:0,&quot;335559739&quot;:160,&quot;335559740&quot;:259}">&nbsp;</span></p>
<p style="text-align: center;"><span><span role="presentation"><img unselectable="on" src="https://blog.morphisec.com/hubfs/undefined-Apr-18-2023-12-34-05-5981-AM.png" loading="lazy" alt="String decryption routine using the &quot;invalid printer&quot; key&nbsp;"></span><span data-contrast="auto" xml:lang="EN-US" lang="EN-US"></span>&nbsp;<br></span><em>String decryption routine using the "invalid printer" key&nbsp;</em></p>
<p><span data-contrast="auto">Another element helping the low detection rate on VirusTotal is the threat actor using </span><a href="https://www.embarcadero.com/products/rad-studio" rel="nofollow noopener" target="_blank"><span data-contrast="none">Embarcadero RAD Studio</span></a><span data-contrast="auto"> to generate executables. Embarcadero RAD Studio is an integrated development environment (IDE) for building software applications that runs on various platforms and operating systems.&nbsp;</span><span data-ccp-props="{&quot;201341983&quot;:0,&quot;335559739&quot;:160,&quot;335559740&quot;:259}"> </span></p>
<p><span data-ccp-props="{&quot;201341983&quot;:0,&quot;335559739&quot;:160,&quot;335559740&quot;:259}">Examining samples from our investigation, we found the threat actor is compiling executables using several options from the Embarcadero IDE. Those with the lowest detection rate on VirusTotal are compiled using “BCC64.exe”, a new Clang based C++ compiler from Embarcadero. This compiler is based on a different code base than a “Standard Library” (Dinkumware) or “Runtime Library” (compiler-rt) and generates optimized code which changes the entry point and execution flow. This breaks security vendors’ indicators, such as signatures composed from “malicious/suspicious code block.”&nbsp;</span></p>

<h2 aria-level="2"><strong><span data-contrast="auto">Defending Against in2al5d p3in4er</span></strong><span data-ccp-props="{&quot;134245418&quot;:true,&quot;134245529&quot;:true,&quot;201341983&quot;:0,&quot;335559738&quot;:40,&quot;335559739&quot;:0,&quot;335559740&quot;:259}">&nbsp;</span></h2>
<p><span data-contrast="auto">Malware loaders like in2al5d p3in4er are essentially basic delivery. <p><span data-contrast="auto">The attackers behind in2al5d p3in4er are combining it with widely accessible social engineering tools for a high impact campaign that takes over popular YouTube accounts and directs viewers to convincing looking fake websites. <p><span data-contrast="auto">One of the first steps to avoid compromise by in2al5d p3in4er is training organization employees how to detect social engineering campaigns: ensuring URLs are legitimate, and not downloading cracked versions of software.&nbsp;</span><span data-ccp-props="{&quot;201341983&quot;:0,&quot;335559739&quot;:160,&quot;335559740&quot;:259}"></span></p>
