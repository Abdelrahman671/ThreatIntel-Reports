							SystemBC Being Used by Various Attackers						</h1>

					<div class="entry-content clearfix">
						
						<p>SystemBC is a proxy malware that has been used by various attackers for the last few years. While it is recently distributed through SmokeLoader or Emotet, this malware has steadily been used in various ransomware attacks in the past. When an attacker attempts to access a certain address with malicious intent, the system can be used as a passage if the infected system utilizes SystemBC, which acts as a Proxy Bot. Because it can also act as a downloader to install additional malware externally, attackers can also use it to install additional payloads.</p>
<h3>Previous Distribution Cases</h3>
<p>SystemBC&#8217;s distribution using RIG exploit kit and Fallout exploit kit was first discovered in 2019. <a href="https://www.proofpoint.com/us/threat-insight/post/systembc-christmas-july-socks5-malware-and-exploit-kits">[1]</a> The initial version found in 2019 focused mainly on Socks5 Proxy features and had a small size. According to ProofPoint which first discovered SystemBC, the developer of the malware had a history of selling it under the name &#8220;socks5 backconnect system.&#8221;</p>
<p>SystemBC discovered in 2020 was used with Ryuk or Egregor in ransomware attacks. It was also the malware used by the DarkSide ransomware group, which used it to attack Colonial Pipeline, a U.S. pipeline company. <a href="https://news.sophos.com/en-us/2021/05/11/a-defenders-view-inside-a-darkside-ransomware-attack/">[2]</a> Unlike ransomware distributed through exploit kits, web browsers, or spam emails, attackers using this type of malware install ransomware after dominating the company environment system, then demand money. In other words, they dominate the internal network using tools such as Cobalt Strike after the initial infiltration and infect various systems within a company by installing ransomware.</p>
<p>The role of SystemBC in such an attack is not known in detail. Yet as it can act as a proxy and install additional payloads after downloading them, it might download and execute malicious payloads or be installed in internal networks to perform the role of a proxy. In fact, according to a report made by F-Secure <a href="https://labs.f-secure.com/blog/prelude-to-ransomware-systembc/">[3]</a> that found an attack using SystemBC, the malware was used for downloading and running PsExec and scripts for lateral movement attacks.</p>
<h3>
<p>Recent Distribution Cases</h3>
<p>In March 2022, it was found that SystemBC was being installed as an additional payload by Emotet. Emotet is a banking malware that installs additional modules or malware strains to steal credentials from the infected system. Normally, the attackers install Cobalt Strike through Emotet to dominate the infected system, but recently, SystemBC is also being distributed.</p>
<blockquote class="twitter-tweet" data-width="550" data-dnt="true">
<p lang="en" dir="ltr"><a href="https://twitter.com/hashtag/Emotet?src=hash&amp;ref_src=twsrc%5Etfw">#Emotet</a> E5 Update &#8211; Within the last several hours, we have seen some bots on the Epoch 5 botnet begin to drop SystemBC now as a module and execute it. This is the first drop beyond Cobalt Strike that we have seen since Emotet returned. This is a significant change 1/x</p>
<p>&mdash; Cryptolaemus (@Cryptolaemus1) <a href="https://twitter.com/Cryptolaemus1/status/1502069552246575105?ref_src=twsrc%5Etfw">March 10, 2022</a></p></blockquote>
<p><script async src="https://platform.twitter.com/widgets.js" charset="utf-8"></script></p>
<p>According to AhnLab&#8217;s ASD infrastructure, most of the recent cases involving SystemBC have the malware installed by SmokeLoader. SmokeLoader operates by being injected into explorer.exe (Windows Explorer that is currently being run) and can install additional modules or malware. The figure below shows the log of the injected Explorer process installing SystemBC.</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2022/03/image-88.png?resize=724%2C226&amp;ssl=1" alt=""></figure>
<p>SmokeLoader is recently installed through Muldrop, an NSIS dropper malware distributed through malicious websites disguised as cracks and serial download pages of commercial software. Besides Muldrop, CryptBot and PseudoManuscrypt are also distributed in such a method.</p>
<ul>
<li><a href="https://asec.ahnlab.com/en/26576/"><strong>[ASEC Blog] Changed Form of CryptBot Infostealer Disguised as Software Crack Download</strong></a></li>
<li><a href="https://asec.ahnlab.com/en/31683/"><strong>[ASEC Blog] PseudoManuscrypt Being Distributed in the Same Method as Cryptbot</strong></a></li>
</ul>
<h3>
<p>Analysis of SystemBC</h3>
<p>SystemBC has a number of variants. The exact order is not confirmed, but the variants are categorized based on their additional features. Unlike Type 1 which is an early version and can only update itself, Type 2 can run scripts such as Batch, VBS, and PowerShell after downloading them. It can also download malware in DLL and Shellcode forms to execute them in the memory. In addition, the malware can communicate with the C&amp;C server through the Tor network. <a href="https://news.sophos.com/en-us/2020/12/16/systembc/">[4]</a> Type 3, the second variant, lacks certain features including being able to use the Tor network and execute DLL and Shellcode after downloading them.</p>
<p>This post will discuss the analysis of SystemBC type that can currently communicate with the C&amp;C server. To be more precise, it is an analysis of Type 2, which has most of the features of Type 1 and Type 3. The malware was found to be installed through RedLine, packed with the packer that was used for the type distributed through SmokeLoader. SystemBC known to be installed through Emotet is Type 3.</p>
<p>Initial Routine</p>
<p>When SystemBC is initially run, it first checks if the argument is &#8220;start&#8221;. It will not have an argument when it is executed for the first time. In this case, it checks the windows of the currently running processes. If there is a process with &#8220;Microsoft&#8221; as the window name and &#8220;win32app&#8221; as the class name, it will send the message &#8220;WM_COPYDATA&#8221; and goes dormant for a certain amount of time. Afterward, it deletes the file for the process.</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2022/03/image-90.png?resize=411%2C305&amp;ssl=1" alt=""></figure>
<p>SystemBC first registered a window class and created a window. The name of the window and class is &#8220;Microsoft&#8221; and &#8220;win32app&#8221; respectively. As shown in the figure below, the following windows and classes can be seen when SystemBC is executed.</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2022/03/image-91.png?resize=672%2C82&amp;ssl=1" alt=""></figure>
<p>The message handling function registered at this moment deletes and terminates a process registered as &#8220;certain random string&#8221; when it receives the message &#8220;WM_COPYDATA&#8221;. In summary, SystemBC checks for the SystemBC process that has been running when it is executed for the first time. If there is one, it sends a message to terminate the old SystemBC. The previous SystemBC that received the message deletes the task it is registered to and terminates itself, and SystemBC that was executed later deletes the binary of the previous one.</p>
<p>It then scans the process named &#8220;a2guard.exe&#8221; which is assumed to be a product of Emisoft. If the process is running, it terminates itself and will no longer perform malicious behaviors. Lastly, it copies the binary of the currently running SystemBC as a random name in %ALLUSERSPROFILE% (in the random folder of the ProgramData path) and registers it as a task named &#8220;certain random string&#8221; again. The process uses COM objects, TaskScheduler class, and methods of the Task class.</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2022/04/image-11.png" alt=""></figure>
<p>The task starts 2 minutes after the current time and is run every 2 minutes. The target that is executed is SystemBC, and designates &#8220;start&#8221; as an argument. SystemBC can download payloads in exe form from the C&amp;C server and run them. If the downloaded executable is SystemBC with the latest version, the process then becomes a binary update for SystemBC.</p>
<p>C&amp;C Communications</p>
<p>SystemBC executed with the &#8220;start&#8221; argument attempts to communicate with the C&amp;C server. It has the URL of the C&amp;C server in the data section in XOR-encrypted form. The malware decrypts the C&amp;C server address and port number before communicating with the C&amp;C server. If it cannot access the first URL, it will attempt to communicate with the second one. Since the current analysis target does not have its settings data encrypted, one can check it in its plain form. If the &#8220;xordata&#8221; string exists below the settings data, the XOR encoding will not be processed. The 0x32 byte-sized data that has the string is the value for the RC4 key. If a normal RC4 key value exists, the XOR encoding will be processed.</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2022/03/image-93.png?resize=591%2C197&amp;ssl=1" alt=""></figure>
<pre><code class="language-plaintext">- C&amp;C Server URL 1 : 31.44.185[.]6:4001
- C&amp;C Server URL 2 : 31.44.185[.]11:4001</code></pre>

<p>If the downloaded payload is DLL, it assigns memory and loads it to run as a new thread. If the &#8220;#&#8221; string is behind the URL sent from the C&amp;C server, it calls the export function from the downloaded DLL. For Shellcode, the malware also runs it as a new thread going through the decoding routine. As a result, DLL and Shellcode are not created as files but run in the memory of SystemBC.</p>
<p>TOR Communications</p>
<p>Because the current analysis target does not have a Tor URL, the team will discuss a previous case where Tor network communication was possible. The malware in this case has the C&amp;C server URLs encoded as shown below. If it cannot access both servers, it uses Tor to access another server.</p>
<pre><code class="language-plaintext">- C&amp;C Server URL 1: admex175x[.]xyz:4044
- C&amp;C Server URL 2: servx278x[.]xyz:4044</code></pre>
<p>To do so, it accesses the following URLs to obtain a public IP address. The address is then encoded with the data that will be sent to the C&amp;C server and sent.</p>
<pre><code class="language-plaintext">https://api.ipify.org/
https://ip4.seeip.org/</code></pre>
<p>SystemBC is known to utilize the mini-tor<a href="https://github.com/wbenny/mini-tor">[5]</a> library to use the Tor network.<a href="https://news.sophos.com/en-us/2020/12/16/systembc/">[6]</a> It first goes through the reset process to access Tor. By randomly selecting one of the IP addresses of the hard-coded Authoritative Directory Server, it gets the Consensus data for the Tor network. Then it will start Tor communications based on the settings data it received.</p>

The part that comes after the &#8220;TOR:&#8221; string is the Tor C&amp;C URL that is decrypted for the first time. The actual URL will be revealed through the additional decryption process.</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2022/03/image-101.png?resize=516%2C150&amp;ssl=1" alt=""></figure>

<p>After normally accessing the Tor network, the malware will send the information of the infected system including the public IP address that was mentioned earlier. This method is identical to other methods of using Raw TCP socket communications, except that it sends data by using the Tor network. So the malware will send the data encrypted with RC4 algorithm and receive C&amp;C commands encrypted with the same key as in previous cases. The case is also the same for the HTTP communications used for downloading additional payloads.</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2022/03/image-103.png?resize=517%2C149&amp;ssl=1" alt=""></figure>

<p>SOCKS5 PROXY</p>
<p>Besides downloader, the main features of SystemBC include being able to operate as Proxy Bot. The figure below shows the commands related to proxies that were mentioned above. Each line creates a socket for the proxy and processes certain proxy packets.</p>
<figure class="image"><img decoding="async" src="https://image.ahnlab.com/atip/content/atcp/2022/03/image-104.png?resize=543%2C451&amp;ssl=1" alt=""></figure>
<p>If the attacker wants to use an infected system as Proxy Bot (using SystemBC of the infected system when accessing a certain address), a command to create proxies will be sent first. SystemBC creates a socket depending on the type when it receives a command to create proxies. The created socket will be managed by index.</p>
<p>After the socket is created, the malware will create a new thread and connect to the address it received. The reason the attacker initially named the malware BackConnect is because SystemBC first connects to the attacker&#8217;s server instead of the attacker manually accessing SystemBC to attempt Socks5 proxy connection. Since SystemBC cannot be accessed externally if it is installed in the system of a private IP band, malware strains with the Proxy feature mainly use the Reverse Proxy method.</p>
<p>Should the attacker send requests to a certain address later, they will send the created proxy socket with the assigned index. SystemBC will then send the data it received to the address. The data received will be sent to the C&amp;C server through SystemBC. SystemBC thus acts as Proxy Bot, allowing the attacker to hide the IP when performing attacks. If the malware operates in the system that can access internal networks, the networks can be accessed by the external attacker through SystemBC.</p>

<h3>
<p>Conclusion</h3>
<p>Ever since SystemBC was distributed through exploit kits in the past, the malware has been installed through other malware strains from malicious websites disguised as download pages for cracks and serials of commercial software until recently. While it was used for attacks targeting normal users, it was also employed by attackers in multiple ransomware attacks targeting companies to achieve their goals.</p>
<p>After it is installed, SystemBC stays in the infected system to download additional payloads. Moreover, it can also act as Proxy Bot, meaning that the system can become a passageway for other attackers. Users should apply the latest patch for OS and programs such as Internet browsers, and update V3 to the latest version to prevent malware infection in advance.</p>
<p>AhnLab&#8217;s anti-malware software, V3, detects and blocks the malware above using the aliases below.</p>
