											<h2 id="ms--in-this-article" class="title is-6 margin-block-xs">In this article</h2>
										</nav><!-- <content> --><blockquote>
<p>Applies to: Windows Server 2022, Windows Server 2019, Windows Server 2016, Windows Server 2012 R2, Windows Server 2012</p>
</blockquote>
<p>Transfers files to and from a computer running a File Transfer Protocol (ftp) server service. This command can be used interactively or in batch mode by processing ASCII text files.</p>
<h2 id="syntax">Syntax</h2>
<pre><code>ftp [-v] [-d] [-i] [-n] [-g] [-s:&lt;filename&gt;] [-a] [-A] [-x:&lt;sendbuffer&gt;] [-r:&lt;recvbuffer&gt;] [-b:&lt;asyncbuffers&gt;][-w:&lt;windowssize&gt;][&lt;host&gt;] [-?]
</code></pre>
<h3 id="parameters">Parameters</h3>
<table>
<thead>
<tr>
<th>Parameter</th>
<th>Description</th>
</tr>
</thead>
<tbody>
<tr>
<td>-v</td>
<td>Suppresses display of remote server responses.</td>
</tr>
<tr>
<td>-d</td>
<td>Enables debugging, displaying all commands passed between the FTP client and FTP server.</td>
</tr>
<tr>
<td>-i</td>
<td>Disables interactive prompting during multiple file transfers.</td>
</tr>
<tr>
<td>-n</td>
<td>Suppresses auto-login upon initial connection.</td>
</tr>
<tr>
<td>-g</td>
<td>Disables file name globbing.  <strong>Glob</strong> permits the use of the asterisk (*) and question mark (?) as wildcard characters in local file and path names.</td>
</tr>
<tr>
<td>-s:<code>&lt;filename&gt;</code></td>
<td>Specifies a text file that contains <strong>ftp</strong> commands. These commands run automatically after <strong>ftp</strong> starts. This parameter allows no spaces. Use this parameter instead of redirection (<code>&lt;</code>). <strong>Note:</strong> In Windows 8 and  Windows Server 2012  or later operating systems, the text file must be written in UTF-8.</td>
</tr>
<tr>
<td>-a</td>
<td>Specifies that any local interface can be used when binding the ftp data connection.</td>
</tr>
<tr>
<td>-A</td>
<td>Logs onto the ftp server as anonymous.</td>
</tr>
<tr>
<td>-x:<code>&lt;sendbuffer&gt; </code></td>
<td>Overrides the default SO_SNDBUF size of 8192.</td>
</tr>
<tr>
<td>-r:<code>&lt;recvbuffer&gt;</code></td>
<td>Overrides the default SO_RCVBUF size of 8192.</td>
</tr>
<tr>
<td>-b:<code>&lt;asyncbuffers&gt;</code></td>
<td>Overrides the default async buffer count of 3.</td>
</tr>
<tr>
<td>-w:<code>&lt;windowssize&gt;</code></td>
<td>Specifies the size of the transfer buffer. The default window size is 4096 bytes.</td>
</tr>
<tr>
<td><code>&lt;host&gt;</code></td>
<td>Specifies the computer name, IP address, or IPv6 address of the ftp server to which to connect. The host name or address, if specified, must be the last parameter on the line.</td>
</tr>
<tr>
<td>-?</td>
<td>Displays help at the command prompt.</td>
</tr>
</tbody>
</table>
<h4 id="remarks">Remarks</h4>
<ul>
<li><p>The <strong>ftp</strong> command-line parameters are case-sensitive.</p>
</li>
<li><p>This command is available only if the <strong>Internet Protocol (TCP/IP)</strong> protocol is installed as a component in the properties of a network adapter in Network Connections.</p>
</li>
<li><p>The <strong>ftp</strong> command can be used interactively. After it is started, <strong>ftp</strong> creates a sub-environment in which you can use <strong>ftp</strong> commands. You can return to the command prompt by typing the <strong>quit</strong> command. When the <strong>ftp</strong> sub-environment is running, it is indicated by the <code>ftp &gt;</code> command prompt. For more information, see the <strong>ftp</strong> commands.</p>
</li>
<li><p>The <strong>ftp</strong> command supports the use of IPv6 when the IPv6 protocol is installed.</p>
</li>
</ul>
<h3 id="examples">Examples</h3>
<p>To log on to the ftp server named <code>ftp.example.microsoft.com</code>, type:</p>
<pre><code>ftp ftp.example.microsoft.com
</code></pre>
<p>To log on to the ftp server named <code>ftp.example.microsoft.com</code> and run the <strong>ftp</strong> commands contained in a file named <em>resync.txt</em>, type:</p>
<pre><code>ftp -s:resync.txt ftp.example.microsoft.com
</code></pre>
